# impact — 改前轻量预览

## 触发

```
/woworldidea-design impact <设计意图描述或概念名+改动>
```

两种输入形式：
- 精确概念: `impact Vitals 新增hydration字段` → 直接跑
- 设计意图: `impact 建筑和市民的生成顺序互换` → 先解析再跑

## 执行流程

### 对于精确概念名+改动描述

用户指名了概念和改动内容 → 跳过阶段0，直接进入 S1。

### 对于设计意图描述

用户描述了一个设计决策，未指名具体概念 → 先跑阶段0：

#### 阶段0a: 术语消歧 + 关键词扫描

1. 从用户描述中提取关键词（如"建筑/市民/生成/顺序"）
2. 在模块接头总览全量出口条目中搜索匹配关键词
3. 术语消歧——"建筑"可能指 Building 模块或泛指建筑物，"市民"可能指 NPC 或特定概念
4. 去噪——关键词匹配可能命中无关概念（"生成"命中世界生成/NPC生成/经济生成...）
5. 产出粗候选概念清单

#### 阶段0b: 深度阅读

对粗候选进行采样阅读模块文档，判断是否真的与用户意图相关。

为每个候选标注：相关度（高/中/低）+ 理由。

#### 阶段0c: 用户确认

展示候选清单：
```
推测以下概念可能受"建筑和市民生成顺序互换"影响：
  ☑ WorldGen.生成管线排序 — 高 — 世界生成阶段顺序
  ☑ Building.建筑放置 — 高 — 建筑生成时机
  ☑ Life.NPC初始化 — 高 — NPC生成依赖世界状态
  ☑ NPC.初始住房分配 — 中 — 市民生成时建筑是否存在
  ☑ History.早期文明事件序列 — 低 — 历史模拟中的时序
勾选确认受影响的，取消不相关的。
```

#### 阶段0d: 完整性自审

skill 自问："还有什么可能被影响但不在清单上？"

搜索可能遗漏的消费方，补充到清单。

### S1: 查消费方

对确认清单中每个概念 → 读接头总览出口条目 → 统计消费方数量。

### S2: 粗分 Hard/Soft/Ref

基于接头总览"变更影响链"中的信息，粗分级别。

### S3: 输出轻量报告

```
━━━━━━━━━━━━━━━━━━━━━━
影响预览: <用户描述>
━━━━━━━━━━━━━━━━━━━━━━

受影响概念: 5个
直接消费方: ~18个模块引用
  Hard (类型级): ~8处
  Soft (设计级): ~15处
  Ref (仅wikilink): ~6处

⚠️ 可能涉及的概念间交互:
  • Building.建筑放置 ↔ Life.NPC初始化 (时序依赖)

━━━━━━━━━━━━━━━━━━━━━━
💡 这是轻量预览（数量级估计）。
   详细冲突分析请在改完设计后跑 /woworldidea-design sync。
━━━━━━━━━━━━━━━━━━━━━━
```

## 重要限制

- impact **不跑子代理**，不读消费方文档做深度比对
- 数量级来自接头总览条目统计，可能因腐化而偏差
- 所有 Hard/Soft 分级是粗估，实际级别以 sync 结果为准
- 如果你的设计改动涉及 4+ 个概念，建议不跑 impact 直接改 → 改完跑 sync
