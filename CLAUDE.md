## 项目概述

**WoWorld (Wonder World)** — 与其说是一个游戏，不如说是在以游戏的方式创建另一个"人世"。程序化生成的体素开放世界RPG，独立开发中。核心卖点：100,000个力求"完整"的NPC——有记忆（上限2000条）、预测、情感、思想、人际关系、欲望、习惯、偏好。

**核心设计哲学**：WoWorld 是一个**故事生成器**，不是一个故事容器。传统RPG（如D&D系列）用开发者手搓的剧情驱动游戏，故事写到哪里游戏寿命就终止在哪里。WoWorld 不预设任何剧情——开发者只定义世界的底层规则（人格参数、行动规则、情绪规律、文化传播率），具体的故事由NPC之间的互动以概率和统计的方式自然涌现。每个存档都是一个完全不同的世界，不同世界之间的文明形态、文化格局、历史轨迹可能天差地别。游戏没有"通关"的概念——只要世界还在运转，故事就永不停歇。

**"人世"的关键维度**：
- **完整的人**：NPC 有记忆、预测、情感、思想、欲望、习惯——与"人"打交道是核心，与"物"打交道是手段
- **历史深度**：玩家进入时世界已运转千百万年，书本和遗迹中保留着先民活过的证据。也可以从零开始建立文明
- **冒险与生活平等**：组建冒险小队去无穷远方探索文明与奇迹，或做个铁匠、商人、政客度过一生——每条路都是"主线"
- **全球多元文化**：西方剑与魔法为基底，世界各地的文化元素（东方、中东、非洲、北欧、美洲、南亚……）自由融入。各文明可自行演化出独一无二的文化特色

**尚无游戏代码**，本仓库纯粹是设计文档。项目当前处于理念设计阶段，无固定的阶段化实现计划。

> ⚠️ **重要说明**：WoWorld 目前仅处于理念设计阶段，具体代码实现层面的技术选型、架构细节、数据结构等均未最终确定。设计文档中提及的任何代码方案、伪代码、数据结构示例等，仅用于辅助阐明设计理念，在实际开发中随时可能大幅重构甚至推翻重写。不应将文档中的实现描述视为已确定的工程规范。

## 技术栈（已规划，尚未实现）

> 📘 **权威技术方案见 `参考文档/015-正式技术栈方案-20260610/`**。以下为概要。

- **引擎**: Godot 4.6 LTS
- **模拟语言**: Rust (stable 1.80+) — 通过 GDExtension 与 Godot 集成
- **体素**: 自建 Transvoxel（Rust 侧）→ Godot ArrayMesh 渲染。分层密度场（基础地形+奇幻+文化+NPC修改+玩家修改+天气），8级LOD覆盖0-6.4km+
- **画面**: 3D低多边形 + 像素纹理
- **NPC AI**: GOAP规划（安全网，占9%决策）+ 概率行为树（日常，占90%）+ 可选LLM增强（1%）。SoA数据布局 + rayon并行
- **动画**: UAF启发的模块化GPU动画——Rust CPU批量骨骼矩阵 → Godot GPU skinning shader。无AnimationTree节点
- **世界生成**: 多层噪声 → 生物群系 → 聚落 → 建筑(WFC) → 道路 → NPC初始化 → 历史模拟。种子确定性+增量存档
- **数据库**: LMDB — 双层记忆架构（事件事实库 + 主观记忆库）
- **时间/天气**: Rust侧TimeManager + WeatherManager；Godot Sky shader天体渲染；季节+天气影响NPC行为权重
- **Mod**: TOML数据驱动 + Rhai脚本引擎（远期）；参数调节API（非行为编写）；5级Mod支持
- **架构**: Rust模拟核心（NPC/世界生成/时间天气/Mod）→ GDExtension → Godot客户端（渲染/UI/音频/输入/玩家物理）
- **美术**: AI生成(Stable Diffusion) + 手动调整(Blender)
- **平台**: Windows / Linux / macOS

## 文档元数据约定（Frontmatter）

