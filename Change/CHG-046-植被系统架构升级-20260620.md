# CHG-046 植被系统架构升级

> **日期**: 2026-06-20
> **类型**: 架构重构 + 模块接口新增
> **影响范围**: 生命模块 · 世界生成 · 建筑模块 · 模块接头总览 · CLAUDE.md · CLAUDE-INTERFACES.md
> **关联**: CHG-045 (世界生成 v2.0) · CHG-043 (建筑模块) · CHG-013 (基座契约)

---

## 变更原因

当前植被设计散落在两处（`生命/009-植物.md` 1850行 + `世界生成/010-生命世界初始化.md` P3.5），存在四个结构性问题：

1. **双重所有权模糊**：植物物种归Life，植物群落归World Gen，但L6.5密度场在P2生成而物种放置却在P3.5——中间隔着文化(P2.5)和资源(P3)
2. **木材跨5模块无正式合同**：Life(ProductCategory::Wood) → Items(WoodMat 0x24) → World Gen(MaterialAvailability) → NPC物理原子(MaterialProperties) → Building(MaterialClass::Wood)——链路长但无单一权威契约
3. **植被无独立模块接头**：其他22个模块都有标准的模块接头条目，植被作为Life子项缺少可被其他模块正式消费的接口
4. **P3.5打包了四类不相关的生命**：植物、动物、怪物、亡灵——它们的依赖链完全不同

---

## 变更内容

### 1. 模块拓扑：Life保持Owner + 植被提取为独立子crate

- **不做**：拆出独立的顶层"植被模块"目录
- **设计文档层**：
  - `生命/009-植物.md` 保留——个体植物生物学（8维生成、PlantSpecies、产物）
  - `生命/010-植被群落与覆盖.md` ★新建——群落生态学（VegetationProvider trait、木材合同、演替动态）
- **Rust crate 层**（未来实现时）：
  - `woworld_vegetation` (lib crate) — VegetationProvider trait实现
  - `woworld_core::vegetation` 模块 — trait定义 + TimberAvailability + GroundCoverMap等共享类型

### 2. 管线重构：P2.25植被覆盖

```
旧: P2 → P2.5 → P3 → P3.5(植物+动物+怪物+亡灵)
新: P2 → P2.25(植被覆盖) → P2.5 → P3 → P3.5(动物+怪物+亡灵)
```

- `世界生成/012-植被覆盖生成.md` ★新建——P2.25管线阶段完整规格
- `世界生成/001-世界生成总流程.md` 修改——插入P2.25，15阶段管线
- `世界生成/010-生命世界初始化.md` 修改——植物迁移至P2.25
- P3木材zone从VegetationCover推导（非独立噪声）
- P3.5动物初级生产力从VegetationCoverMap读取

### 3. VegetationProvider trait（woworld_core · Pattern D）

```rust
pub trait VegetationProvider: Send + Sync {
    fn query_community(&self, pos: Vec2, radius: f32) -> PlantCommunitySnapshot;
    fn canopy_closure(&self, pos: Vec2) -> f32;
    fn timber_availability(&self, pos: Vec2) -> TimberAvailability;
    fn ground_cover(&self, pos: Vec2) -> GroundCoverMap;
    fn query_harvestable(&self, pos: Vec2, radius: f32) -> Vec<HarvestableInfo>;
    fn fuel_load(&self, pos: Vec2) -> f32;
    fn root_interference(&self, pos: Vec3) -> f32;
}
```

消费方：建筑/NPC/天气/音频/战斗/经济/历史/生命(动物) ——共8个模块

### 4. 木材形式化合同（WoodMaterialContract）

- `TimberAvailability` 和 `TimberQuality` 在 `woworld_core` 中单一定义
- Building模块不import任何Life/植物类型
- `TimberQuality = f(wood_type, max_trunk_diameter, spirit_capacity)` 从形态参数推导，非手工标注
- 新增魔法木材种 → 只需在woworld_vegetation添加映射规则 → Building零改动

### 5. 涌现深度保障

- **香农熵加权优势种筛选**：丰富度 = f(H, soil_fertility)，非硬编码"3个树种"
- **Voronoi林窗检测**：Voronoi cell面积异常大 = 林窗，零硬编码
- **并查集斑块连通性**：森林斑块从连通分量涌现，非预定义ForestRegion
- **演替纯函数**：`succession_phase = f(years_since_disturbance, soil, climate)`，不存枚举标签
- **木材品质推导**：从形态参数涌现，非手工标注timber_quality字段

## 模块接口变更

### 新增接口
| 接口 | 定义位置 | 实现位置 | 消费方 |
|------|---------|---------|--------|
| `VegetationProvider` trait | woworld_core | woworld_vegetation | 建筑/NPC/天气/音频/战斗/经济/历史 |
| `TimberAvailability` | woworld_core | — | 建筑/经济 |
| `TimberQuality` | woworld_core | — | 建筑/经济 |
| `GroundCoverMap` | woworld_core | — | 音频/NPC |

### 接口消费变更
| 消费模块 | 旧方式 | 新方式 |
|----------|--------|--------|
| 建筑 | MaterialAvailability（World Gen独立噪声生成） | VegetationProvider::timber_availability() |
| NPC | 直接查PlantCommunitySnapshot（依赖Life类型） | VegetationProvider::query_harvestable() |
| 天气 | 直接读L6.5密度场（耦合World Gen数据结构） | VegetationProvider::canopy_closure() |
