# DEVLOG — 2026-07-06 (Sprint 032 系列: VoxelChunk LOD 0 bug 修复攻势)

## 摘要

Sprint 032 经过 **7 轮迭代**（032 → 032-B → … → 032-G），修复了 VoxelChunk LOD 0 的三个根因问题。过程中三次错误诊断（精度损失、水下地形、CaveDensity），最终通过系统性数据日志锁定并修复。72 测试全绿。

## 背景

从 Sprint 031 交接（handoff-024）起步。VoxelChunk LOD 0 替代 clipmap 后，玩家脚下区域渲染异常。交接文档描述为"彩色碎片，chunk 内部有 1m 水平间隙"。已有 5 次修复尝试（法线反转、Material 延迟、y-origin 统一、顶点坐标系修正、cull_disabled 临时方案），均未根治。

## 迭代记录

### Sprint 032-A（❌ 失败 — 前提假设错误）

**假设**：CaveDensity（3D Worley 洞穴）在 VoxelChunk 密度栈中产生地表洞穴 → 碎片化。

**修复**：
- `submit_voxel_job` 改为构造仅含 TerrainBaseDensity 的 surface-only 密度栈（排除 CaveDensity）
- Per-chunk 垂直范围替代统一 wy（避免 vy 膨胀）
- 三角形绕序反转 + cull_back
- CLAUDE.md 更新（测试分布、新源文件）

**结果**：问题未修复。用户报告"正方形区域透明 + 绿色小盒子 2-3 层"。

**教训**：CaveDensity 不是根因。在修复前应先验证前提假设（地面是否真的在海拔 >0）。

### Sprint 032-B（❌ 失败 — 再次错误诊断）

**新假设**：`.max(0.0)` 将 wy clamp 到 ≥0，水下地形不渲染 → 透明区域。

**修复**：
- init_voxel_grid：删除 Pass 1 统一扫描 → per-chunk wy，去掉 `.max(0.0)`
- drift loop：去掉 `.max(0.0)`
- 删除死字段 `voxel_wy`/`voxel_vy`
- Cargo.toml：`opt-level=0`（debug mode）

**结果**：用户报告"问题变化了——不是整个区域透明，而是一块一块的三角形变透明，绿盒子消失，光照偏暗"。用户明确指出**问题在陆地上，非水下**。

**教训**：`.max(0.0)` 不是根因。症状变化说明 per-chunk wy 有影响——绿盒子消失是进步，三角形缺口是新现象。需用数据诊断替代猜测。

### Sprint 032-C（🔬 诊断 — 排除面剔除）

**测试**：将 cull_back 改回 cull_disabled。

**结果**：三角形缺口未消失。**排除面剔除假说**。

### Sprint 032-D（🔬 诊断 — 数据日志）

**添加**：upload_voxel_mesh 中 NaN/Inf 检测 + 顶点计数 + 样本法线。

**结果**：
- NaN_v=0, Inf_v=0, NaN_n=0 → **mesh 数据完全干净**
- OOB_idx=0 → **索引全部有效，无越界**
- 法线 Y=0.85~0.98 → **法线指向正确**
- Vertex 1200-1800/chunk, Triangle 2300-3500/chunk → 数量合理

**关键结论**：问题不在 transvoxel 提取、不在 mesh 上传、不在面剔除、不在法线计算。根因在更上层。

### Sprint 032-E（📋 计划）

系统性列出 4 个待验证假说（索引越界、法线偏差、ArrayMesh 格式、clipmap 重叠 z-fighting）。添加 OOB 检测和法线采样诊断。

### Sprint 032-F（✅ 部分修复 — 统一 wy 回退）

**假设**：per-chunk wy 导致相邻 chunk 垂直错位 → 三角形缺口。

**修复**：
- 恢复 init_voxel_grid 25-chunk 统一 wy 扫描（无 `.max(0.0)`）
- 恢复 drift 使用 `voxel_wy`/`voxel_vy`
- 恢复死字段

**结果**：**三角形缺口消失！** 但两个新问题：
1. LOD 0 正方形区域颜色与其他 LOD 明显不同
2. 移动到陡坡区域时部分 VoxelChunk 变透明

**结论**：per-chunk wy 确实导致了边界不连续。统一 wy 是正确的方向，但需要修复剩余两个问题。

### Sprint 032-G（✅ 最终修复）

**根因 1 — 颜色差异**：VoxelChunk 用 `material_from_height()`（纯高度分类），clipmap 用 biome 分类器（温度×降水→群系→材质）。两个材质路径不同。

**修复**：`TerrainBaseDensity` 新增 `biome: Option<BiomeClassifier>`。`material_at()` 调用 `classifier.classify()` → biome surface_material，与 clipmap 一致。`submit_voxel_job` 传入 biome_classifier。

**根因 2 — 陡坡透明**：drift 时复用 init 时的 `voxel_wy`/`voxel_vy`，不会根据玩家新位置的地形海拔重算。移动到不同海拔区后垂直范围不够。

**修复**：drift recenter 时重算统一 wy（扫描新 25 chunk 的 y_min/y_max → 更新 `voxel_wy`/`voxel_vy`）。

**结果**：颜色一致、陡坡不透明。**Sprint 032 完成。**

**最终保留的修改**：

- ✅ 三角形绕序反转（MC CCW → Godot CW）——为 cull_back 做准备
- ✅ 统一 wy + drift 重算
- ✅ biome 材质统一
- ✅ `cull_disabled`（当前 shader 状态——绕序反转后部分三角形面朝向不一致，需后续修复）

**当前 shader 为 cull_disabled**：Paul Bourke MC 表的已知不一致性导致反转后仍有部分三角形朝内。后续需要逐 case 修复绕序，才能恢复 cull_back。

## 关键决策

- **统一 wy > per-chunk wy**：对于 16m 间距的 chunk 网格，统一垂直原点避免了边界错位。per-chunk 方案仅在大海拔跨度时有利（避免 vy 膨胀），但引入了更严重的边界裂缝。
- **数据驱动诊断 > 假设驱动**：Sprint 032-D 的数据日志用 4 行输出排除了 4 个假说。在复杂 mesh 管线中，花 10 分钟加诊断比花 2 小时追错误假设更高效。
- **TerrainBaseDensity 持有 BiomeClassifier**：材质查询走与 clipmap 相同的路径——biome 分类器优先，高度回退。`material_at()` 现在的行为与 `sample_vertex()` 一致。
