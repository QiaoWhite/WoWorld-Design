## 一、变更动机

`想法/世界生成/` 中的 22 份文档包含大量高质量设计（9 阶段管线、四级聚落体系、WFC 建筑布局、政治实体、矿田集中分布、河流分级、过渡带等），但它们建立在以下已过时的技术基础上：

- **体素引擎**：Zylann Voxel Tools 插件 + Marching Cubes → v3.0 要求自建 Transvoxel（Rust）
- **世界尺度**：16km×16km → v3.0 要求 25 万 km²（≈英国）+ 海陆 7:3
- **语言**：全部 GDScript 伪代码 → v3.0 要求 Rust 模拟核心
- **LOD 体系**：3 级简单 LOD → v3.0 要求 8 级 Clipmap LOD
- **Chunk**：均匀 32³ → v3.0 要求垂直稀疏 Chunk
- **海洋**：设计中不存在 → v3.0 70% 海洋 + Gerstner 波
- **玩家中心密度控制**：围绕玩家调整景观密度 → 25 万 km² 下不可行

本次重构目标：**吸收原方案中优秀的设计逻辑，以 v3.0 技术栈方案为权威依据，产出正式开发规格**。

---

## 二、核心决策清单

### 决策 1：体素引擎——自建 Transvoxel（Rust）✓

**废弃**：Zylann Voxel Tools 插件（GDScript API）、Marching Cubes、SVO 存储。
**采用**：Rust 自建 Transvoxel，Godot ArrayMesh 输出，垂直稀疏 Chunk，Clipmap LOD 8 级，3km 处切换 Signed Heightfield + 法线烘焙。

**理由**：Zylann 插件设计上限 16km²，无法支撑 25 万 km² + 1500m 高程范围 + 10km 远景。自建 Transvoxel 与 Rust 世界生成管线同侧，消除 GDScript 桥接开销，无架构天花板。

### 决策 2：世界尺度——25 万 km² + 海陆 7:3（可调）✓

**废弃**：16km×16km 最大、4km×4km 初始、纯陆地假设。
**采用**：25 万 km² 初步（≈英国，约 500×500km），海陆比 7:3 为默认基准，允许种子参数自定义调节。

**理由**：v3.0 设计哲学要求"无限探索"。所有涉及数量的参数（聚落数量、矿田数量、道路长度、NPC 密度）按新尺度重算，保留原方案中的相对比例逻辑（如聚落等级比例 60/25/10/5）。

### 决策 3：生成管线——插入 P0 全球大陆架构 + P1 海洋系统 ✓

**废弃**："围绕玩家加载区块进行跨区块自然景观分布规划"（P3 玩家中心逻辑）。
**采用**：在现有 9 阶段管线头部插入两个新阶段：

```
P0：全球大陆架构
  全球噪声(频率 0.00005-0.0002) → 大陆板块形状 → 海陆掩码(7:3)
  → 造山带/裂谷/热点岛链 → 大陆骨架

P1：海洋系统
  洋流模型 → 海洋深度分级(浅海/深海/海沟)
  → 海岸线分类(构造/沉积/珊瑚礁/红树林/峡湾)
  → 岛屿链/群岛 → 可航行海域标记

P2-P9：原 P1-P9 重构
  原 P3 改为全局均匀生成 + Clipmap LOD 分层加载
```

**理由**：70% 海洋是"内容"而非"空地"——航海、岛屿发现、海上贸易、跨洋文明碰撞都依赖海洋系统在整个管线中优先于陆地生成。

### 决策 4：自然景观——保留+扩展+幻想地形细分 ✓

**保留**：全部地貌类型定义（9 大系统）、河流 5 级 Strahler 分级、过渡带逻辑、矿田分布逻辑。
**调整**：海拔数值体系对齐 v3.0（~1500m 总范围），重新定义山体分类阈值。
**扩展**：
- 海岸系统大幅扩充（构造/沉积/珊瑚礁/红树林/峡湾海岸详细分类）
- 新增海洋内部景观（浅海/深海/海沟/珊瑚海/海藻森林/冰海/沉没遗迹）
- 新增纬度-气候带映射
- 现实地形系统更细致的分级
- 幻想地形按特性深入分类（浮空型/能量浸染型/巨型生物遗迹型/元素异常型/维度重叠型——而非简单列表）

### 决策 5：聚落体系——保留核心逻辑 + 海洋/游牧扩展 ✓

**保留**：四级聚落链（村庄→小镇→城市→城堡）、选址评分函数、聚落群域→群落→政治实体三级组织、7 权力来源+9 统治结构+6 命名类别。

**扩展**：
- **港口城市**：半圆形沿海布局、码头区/仓库区/海关/灯塔/造船厂——作为主要聚落类型之一
- **游牧聚落**：不适用固定四级链，有移动/季节迁徙/氏族组织逻辑
- **海上航线网络**：洋流+季风+港口间距驱动——替代陆地 MST 的海上等价算法
- **海防体系**：防波堤/海墙/炮台/封锁链——对应不同聚落等级
- **岛屿文化隔离**：未发现大陆标记为 `unknown`，发现后触发文化碰撞事件
- **承载容量模型**（新增）：聚落人口上限 = min(粮食承载, 水源承载, 建材承载, 防御承载)，从资源禀赋自动推导上限
- **经济关系推导**（新增）：从聚落间资源禀赋差异自动推导经济关系（互补→贸易，重叠→竞争，双向依赖不论等级）
  - 城堡依赖村庄粮食（等级高的依赖低的）是正常的——依赖方向由禀赋决定，不由等级决定

### 决策 6：建筑与房间——WFC 保留并扩展到城市级 ✓

**保留**：WFC 作为核心布局算法、三级矛盾处理（回溯策略）、模板建筑路径、质量衰减模型。

**关键澄清**：模板 ≠ 预制 3D 模型 ≠ 跳过 WFC。模板 = **预定义的 WFC 约束集**（瓦片集 + 邻接规则 + 全局约束）。模板路径和个性化路径都跑 WFC，区别仅在于约束集的来源（查表 vs 动态生成）。

**扩展**：
- WFC 从单纯"建筑内房间布局"扩展到两层：城市级（功能区→建筑类型填充）+ 建筑级（房间类型→房间布局）
- 模板路径的约束集可由动态生成器产出后缓存，减少手写成本
- 初始手写 3-5 套核心模板覆盖验证，逐步过渡到生成器

### 决策 7：体素设计决策——原则保留，技术重写，资源扩展 ✓

**保留**：体素 vs 网格根本划分原则（体素=大地素材，网格=地形之上一切）、体素材质类型体系、矿田三级储量、三种采矿方式（露天/地下/水域）、建筑材质与文化关联。

**重写**：体素网格规格（对齐 v3.0 Transvoxel）、LOD 体系（8 级 Clipmap）、Chunk 规格（垂直稀疏）、数据格式、SDF 编辑接口（Rust trait）、确定性保证机制（Rust seed→RNG）。

**扩展**：
- **完整可开采资源体系**：矿物（金属/非金属/宝石）、岩料（花岗岩/大理石/石灰岩/沙岩/玄武岩/板岩）、木材（硬木/软木/竹材/特殊木材）、土沙（粘土/沙/腐殖土/泥炭）、液体（淡水/盐卤/魔法泉）、有机（染料植物/药草/香料/纤维植物）、魔法资源（魔力水晶/元素精华/龙遗物）
- `ResourceProvider` trait 预留未来新资源类型扩展
- 新增海洋底质类型（沙底/泥底/岩底/珊瑚底）

### 决策 8：NPC 认知模型——移出世界生成 ✓

`05-NPC认知模型.md` 的内容（本体论分类、智能↔野蛮光谱、7 因子心理加工）属于 NPC 系统。**从世界生成文档中移出**，其设计精华融入 NPC 活人感模块未来版本。世界生成文档仅作引用。

### 决策 9：文档结构——9 篇 + README ✓

在 `开发阶段/世界生成/` 下建立：

```
开发阶段/世界生成/
├── README.md                    # 模块索引 + 快速参考
├── 001-世界生成总流程.md         # 重构后 11 阶段生成管线
├── 002-自然景观.md               # 地形+生物群系+河流+海洋景观+幻想地形+气候带+生物栖息地
├── 003-资源与矿产.md             # 完整可开采资源体系 + 矿田生成 + ResourceProvider trait
├── 004-聚落生成.md               # 四级聚落+港口城市+游牧+承载容量+群域/群落/政治实体+经济推导
├── 005-建筑与房间生成.md         # WFC 两层 + 模板约束集 + 文化参数→建筑风格
├── 006-交通网络.md               # 陆地 MST + 海上航线 + 洋流/季风
├── 007-体素设计决策.md           # 自建 Transvoxel + 体素/网格边界 + 材质 + 采矿 + SDF

```

### 决策 10：生物生成——栖息地接口预留 ✓

世界生成只负责产出**栖息地标记**（群系/初级生产力/水体/特殊标记），不决定具体生物 spawn。预留 `CreatureSpawnTable`、`MonsterSpawnCondition`、`MigrationCorridor` 接口供未来生物系统独立读取。

### 决策 11：天气/天象——边界划分 ✓

世界生成只负责**静态气候基底**（温度带/湿度带/纬度带/洋流——P2 产出）。天气/天象/体积云的运行时行为留给独立的天气系统文档。体素临时层（积雪/积水）在 `007-体素设计决策` 中记录接口。
