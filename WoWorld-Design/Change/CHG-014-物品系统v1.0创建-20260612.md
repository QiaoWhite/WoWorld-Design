# CHG-014 — 物品系统 v1.0 创建

## 概述

创建 WoWorld **物品系统**（Item System）——四大地基模块（物品/技能/天气/对话）之首。物品系统定义了游戏中一切可持有、可交换、可装备、可消耗的事物的 ID 命名空间、属性规范、装配规则和跨模块接口。

### 设计决策（8 项用户裁决）

1. **ItemId 命名空间**：两层——ItemDefId(u64,全局恒定) + ItemEntId(u64,存档内唯一)。8+56 bit 编码。物品装配框架支持刚性/铰链/链/柔性四种连接
2. **品质与稀有度**：分离。Quality(4档)=工艺精度，Rarity(5档)=自然稀缺度
3. **装备槽位**：BodyPlan 自动派生——不预定义物种类型。双套 Outfit——NPC 自主决定穿战甲还是常服（不是开发者 hardcode）
4. **附魔**：两条路径——卡槽附魔(日常消费) + 直接附魔(永久，历史锚点)。可共存
5. **库存**：槽位为主(30基础+容器)，重量为安全网。五层仓储体系驱动经济行为
6. **堆叠**：全部物品可堆叠，stack_size 控制上限
7. **耐久**：可修理但每次降最大耐久 3-10%
8. **货币**：铜/银/金 100:1 进制，独立于库存（CharacterWallet）

## 新增跨模块接口契约

物品系统作为 Owner：

| 概念 | 权威 Owner（新增） | 消费方 | 关键约定 |
|------|-----------------|--------|---------|
| ItemDefId | **物品系统 002** | 全部模块 | u64, 8+56 bit, 0x00-0x7F 核心/0x80-0xFF Mod |
| ItemEntId | **物品系统 001** | NPC/History/Combat | u64, 存档内唯一, 永不回收 |
| ItemProperties | **物品系统 003** | 全部模块 | 核心属性+品质+稀有度+审美——各模块在此之上叠加 |
| Quality | **物品系统 003** | Magic/Combat/Life | 4档——合并 Magic 的品质体系为全物品通用 |
| Rarity | **物品系统 003** | World Gen/Life | 5档——源于 World Gen 的地质分布 |
| EquipmentSlots | **物品系统 004** | NPC | BodyPlan 自动派生——不是预定义物种表格 |
| OutfitMode | **物品系统 004** | NPC | 切换由 NPC 决定——物品系统只提供数据操作 |
| AestheticProps | **物品系统 003** | NPC | 客观属性——主观判断在 NPC 侧(AestheticTaste) |
| CraftingRecipe | **物品系统 006** | NPC/Magic | TOML 数据驱动——不是代码 |
| CharacterWallet | **物品系统 005** | NPC/Player | 独立于库存——货币不占槽位 |
