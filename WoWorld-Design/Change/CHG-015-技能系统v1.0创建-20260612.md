# CHG-015 -- 技能系统 v1.0 创建

## 概述

创建 WoWorld **技能系统**（Skill System）--四大地基模块之二。技能系统定义了游戏中一切"熟练度"的计量、累积、提升和传授机制。它是纯基础设施--只定义"熟练度怎么计量、怎么提升、怎么传授"，不定义"剑术80级能放什么招式"（那是 Combat 的职责）。

**核心原则**：技能系统管辖 5 大类（Combat/Magic/Artisan/Academic/Survival）。社交能力和经济能力不在管辖范围。

### 关键设计决策

| 决策 | 结论 |
|------|------|
| **管辖范围** | 5大类：Combat（战斗）/ Magic（魔法）/ Artisan（工艺）/ Academic（学术）/ Survival（生存）。社交/经济不在管辖内 |
| **升级模型** | 指数减速累积XP（base=100, k=0.04）--初期快（0到30约2周游戏时间），后期极慢（90到100约半年）。用进不退--技能只升不降 |
| **硬上限** | level = 100（绝对人体极限）。99.9% NPC一辈子到不了任何技能的90+ |
| **天赋三层叠加** | 1) MentalAccess trait（从NPC活人感的MentalAttributes派生，range 0.7-1.0）2) 天生经验倍率（每NPC每技能独立roll，0.7-1.3，出生时锁定）3) 交叉训练（同组技能被动获得微量XP，上限=min(40, source_level x 0.5)） |
| **种族天赋不硬编码** | 种族对技能的影响通过心智属性分布间接体现--世界生成规则决定 |
| **玩家** | 完全相同的SkillEntry和add_xp机制。MentalAccess + PhysicalAccess返回全1.0--不受虚拟属性削弱 |
| **交叉训练** | 非递归。天花板=min(40, source_level x 0.5)。默认比率：同子组(0.12-0.20) / 同大类(0.10) / 元素亲和(0.15) / 跨类逻辑(0.05) / 元素排斥(0.04) |
| **教学** | 统一在技能系统。四种路径：自学(1.0x) / 师承(1.2-3.0x，上限=导师95%) / 学院(1.3x) / 秘传（调用Magic模块add_xp_direct） |
| **教学风险** | TeachingRisk trait--默认空实现。Combat可选（对战轻伤），Magic可选（契约条款）。大多数技能教学无特殊风险 |
| **技能定义** | TOML数据驱动--不做枚举。增删技能不需要修改代码 |
| **前置条件** | 不做--世界规则自然产生依赖（没人教+没熔炉=学不了锻造）。不硬编码 |
| **基础XP值** | 不由技能系统定义--各消费模块各自决定（Combat: 命中5XP/击杀15XP，Magic: 施法5-20XP，Artisan: 配方数据定义） |

## 新增跨模块接口契约

技能系统作为 Owner：

| 概念 | 权威 Owner | 消费方 | 关键约定 |
|------|-----------|--------|---------|
| SkillId (u64) | **技能系统** 001/002 | 全部模块 | u64全局恒定(8+8+16+32bit, 5分类22子组)。NPC/Combat/Magic/物品系统的SkillId引用全部以技能系统为权威 |
| SkillEntry | **技能系统** 001 | 全部模块 | xp/level/innate_aptitude(0.7-1.3天生一次roll)/total_xp_earned/times_used。稀疏存储，不存在=untrained |
| 累积XP公式 | **技能系统** 001 | Combat/Magic/NPC | total_xp(L)=100 x (e^(0.04L)-1)/0.04。[TUNING]可调。用进不退--无衰减 |
| 天赋模型 | **技能系统** 001/002 | NPC | 三层叠加：MentalAccess trait(0.7-1.0) / 天生倍率(0.7-1.3每NPC每技能独立) / 交叉训练(同组0.04-0.20, 天花板min(40, 0.5L), 非递归) |
| 技能管辖范围 | **技能系统** 001 | NPC/物品/Combat/Magic | 5大类(Combat/Magic/Artisan/Academic/Survival)——社交/经济不在管辖。玩家MentalAccess+PhysicalAccess均返回1.0。★ v1.1升级：新增FineArts第6大类(4子组:Visual/Musical/Literary/Performing) |
| 教学系统 | **技能系统** 003 | NPC/Magic | 四种路径统一在技能系统：自学(1.0x) / 师承(1.2-3.0x) / 学院(1.3x) / 秘传(调用Magic的add_xp_direct)。TeachingRisk trait空默认 |
| 交叉训练 | **技能系统** 002 | 全部模块 | 仅DirectAction/Teaching/DirectInfusion触发——CrossTraining不递归。天花板min(40, source_level x 0.5)。元素组边界由Magic 002权威定义 |
| SkillCategory | **技能系统** 002 | NPC | ★ v1.0:5类枚举 → v1.1:6类枚举（新增FineArts，从Artisan分离）。NPC文档以技能系统为权威 |

## 对现有模块的影响

| 受影响模块 | 影响类型 | 说明 |
|-----------|---------|------|
| **NPC 活人感** | MentalAccess trait实现 | NPC模块定义MentalAccess trait--从MentalAttributes派生，技能系统消费。SkillCategory从7类缩减为5类 |
| **战斗系统** | 消费SkillId | skill_proficiency (=level/100)用于parry/dodge/counter公式。基础XP值定义（命中/击杀/格挡/弹反） |
| **魔法系统** | 消费SkillId + 秘传路径 | 秘传路径调用add_xp_direct。法术本能化阈值（times_used>=500）。元素组边界权威 |
| **物品系统** | 消费SkillId | SkillRequirement（配方所需技能门槛+quality_influence）。配方基础XP值 |
| **世界生成** | 消费generate_starting_skills | NPC出生/移民时的初始技能生成 |
| **历史系统** | 消费skills_history | 技能里程碑事件（Lv50/70/90/100） |
| **生命系统** | PhysicalAccess trait实现 | 身体属性到体能技能关联 |
