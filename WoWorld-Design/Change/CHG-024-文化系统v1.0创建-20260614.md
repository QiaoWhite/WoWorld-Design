## 背景

文化是 WoWorld 中**引用最多的缺失模块**——53 个文件、327+ 次引用，跨越 14 个现有模块。`[[../Happy Game/想法/WoW World/文化设计|想法/文化设计.md]]` 中有较完整的概念草稿（10 个霍夫斯泰德式参数、四层结构、文化传播模型），`[[../Happy Game/开发阶段/魔法/06-人文层/018-魔法与文化|魔法/018-魔法与文化.md]]` 有文化对魔法偏好的轻量处理。`[[../参考文档/021-WoWorld设计文档补全规划-20260613/001-设计文档补全总体规划|021 规划]]` §13.2 制定了文化模块的初步规划（5-6 篇、~2,000-2,500 行）。

经与用户 @QiaoWhite 进行深入设计访谈（/grill-me），对所有关键架构决策达成共识后，文化系统正式创建。

## 关键设计决策

以下决策在设计与用户讨论过程中达成共识：

| # | 决策 | 理由 |
|---|------|------|
| 1 | 三层参数架构（核心→第一级派生→第二级派生） | 10 个核心参数是不可分原子；派生参数确保一致性和涌现性 |
| 2 | `competition_orientation` 替换 `masculinity` | WoWorld 的多智能种族有多样化的性别动态——去除性别标签混淆 |
| 3 | CultureId = 扁平 u32 + 独立谱系图 | 4 字节优于 8 字节——CultureId 被大量存储；谱系查询极低频 |
| 4 | 障碍 Voronoi 离散区域 + 渐变边界 | 离散 CultureId 提供身份锚点；边界带保证空间连续性 |
| 5 | P2.5 管线插入点 | P2 提供全部自然上下文；P3+ 全部受益于真实文化数据 |
| 6 | TechnologyProfile 8 独立领域 | 取消单线时代标签——矮人冶金 90 但航海 15 是合法涌现 |
| 7 | 信仰 = 独立模块 | 文化只提供 religiosity 单参数；信仰类型/教义/仪式属于信仰系统 |
| 8 | CommunicationNorms 所有权转移至文化系统 | norms 是文化参数——定义权归文化，使用权归语言表达 |
| 9 | Survival 技能不受技术天花板限制 | 技术不影响追踪/采集/攀爬——孤立文化可以有 Lv95 猎人 |
| 10 | CultureQuery（pub）/ CultureMut（pub(crate)）分离 | 消费模块只能只读查询——文化修改仅通过世界生成/历史引擎 |
| 11 | SettlementTechQuery trait 由技能系统定义、文化系统实现 | 技能系统不知道 TechnologyProfile 存在——仅依赖自身的抽象 trait |
| 12 | CultureName 双层体系（Endonym + Exonym） | 音系规则生成自名 + 5 种来源加权随机他称——沉浸感与涌现性 |

## 性能影响

- **模块增量**：~3,200 行设计规格，7 个 Markdown 文件
- **运行时性能**（理论——基于现有技术栈方案各模块预算的推断）：
  - Rust CPU：文化模块本身 < 0.05ms/帧（四叉树查询 + SoA 缓存，远低于 16.7ms/帧 60fps 预算）
  - 内存：100 文化 ≈ 38KB 存储 + 20KB 缓存 + 25KB 四叉树/谱系 ≈ <100KB（极低）
  - VRAM：无直接影响（文化数据通过 GDExtension 推送至 Godot——仅为 NPC/UI 提供字符串和浮点值）

## 理念走通验证场景

在群聊中已经过概念验证的 3 个场景和额外识别的多个场景：

- [x] 场景 A：两个文化 NPC 相遇 → norms 差异（一方直视=尊重，另一方低头=尊重）→ 初始关系微减 → 玩家充当"文化翻译"
- [x] 场景 B：openness 高的城市 → 新技术/时尚先在此出现 → 沿贸易路线扩散 → conservatism 高的区域最后接受或拒绝
- [x] 场景 C：玩家穿北方文化服饰进南方城市 → NPC 反应（好奇/轻视/敌视——取决于两文化关系 + southern norms × 玩家服饰文化标签）
- [x] 场景 D：玩家在 `metallurgy=75, navigation=10` 的文化城买钢剑 → 附近港口只有小渔船 → 无法渡海
- [x] 场景 E：两个文化被山脉隔离 500 年 → 参数分化 > 0.15 → P9 历史模拟中自动创建子文化
- [x] 场景 F：边界带聚落存续 300+ 年 → 混合参数与双亲都不同 → 新克里奥尔文化结晶 → 历史事件记录

## 向后兼容

文化系统是**新增**模块——不删除或修改任何已有模块的核心逻辑。`CommunicationNorms` 的所有权转移是"定义权"的转移——语言表达模块中的 norms 消费代码不需要改变（仅需从 `CultureQuery` 而非本地定义获取 norms 值）。

已有模块中所有"文化"的幽灵引用现在有了确定的类型名称和获取路径，但**在幽灵类型替换编辑完成前**，幽灵类型继续作为"待设计"标记存在——不破坏现有文档的可读性。
