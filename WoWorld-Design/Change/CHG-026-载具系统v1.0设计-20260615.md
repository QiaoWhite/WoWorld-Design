## 〇、背景

载具系统是 WoWorld 中尚未正式设计的模块。现有文档中散落着对载具的引用和假设——有些互相矛盾（天气系统说"帆船航速=f(风速)"，技术栈方案说"魔力驱动，不考虑风力"），有些是未定义的幽灵类型（NPC 文档中的 CrewRank/CrewDuty）。通过 grill-me 深度访谈，全面设计了载具系统并修复了所有相关矛盾。

---

## 一、决策汇总

### 1.1 核心哲学（6 条原则）

1. **载具 = 无意志的移动平台**——马不是载具（有独立 AI），船/火车/马车是
2. **载具是社会的延伸**——载具上的 NPC 继续过日子
3. **只创造规则，不创造故事**——航线/运费/船员全由 NPC 心智涌现
4. **日常全自动，灾害半自动**——对标战斗系统的 L1-L3 模型
5. **只定义物理事实**——不碰价格/裁决/建造执行
6. **记忆优先，契书可选**——所有权在记忆中成立，契书是辅助证据

### 1.2 关键架构决策

| 决策 | 内容 |
|------|------|
| **载具身份** | VehicleId（世界实体）+ 可选 ItemEntId 契书（产权锚）。载具永不进背包 |
| **载具分类** | 从参数涌现——域 × 规模 × 动力 × 路径约束 × 能力层。非枚举 |
| **动力类型** | 5 种：MusclePowered / AnimalTowed / WindPowered / MagicEngine / Hybrid |
| **日常航行** | 不受自然力影响——NPC 想走什么路线就走什么路线。灾害级天气施加减速+损伤 |
| **魔力引擎** | 走魔法系统+物品系统——大型魔法物品（直接附魔）+ 符文槽（卡槽附魔）|
| **操控模型** | L1-L3 半自动。NPC 和玩家实现同一 VehicleController trait。仅小载具+玩家可手动 |
| **损伤模型** | 三通道连续值（hull/propulsion/steering）——对标战斗伤势 |
| **所有权** | 5 条获取路径（口头/契书/夺取/建造/继承）。契书可伪造/篡改/检测。裁决走权力系统 Adjudicate |
| **货运** | 移动容器——cargo_hold: ContainerId，走物品系统标准 Container 接口 |
| **铁路** | 不由世界生成——P9 历史模拟中极低概率涌现（[TUNING] ~3%/世纪） |
| **载具定义** | 三层涌现：原型 TOML(~20-30) × 文化参数 × 种子 → VehicleDef(~200-500) |

### 未改动的模块（经确认无需改动）

- **经济系统**：运营成本由 NPC 心智自行计算——走已有市场/钱包/Contract
- **文化系统**：CulturalShipbuildingTradition 从 CultureCoreParams 派生——不修改文化系统
- **物品系统**：契书复用 Financial 类别；Container 接口无需扩展
- **魔法系统**：ManaConsumer trait 由载具系统定义——魔法系统无需改动
- **技能系统**：5 个载具相关技能归入已有分类——不新增 SkillCategory
- **信仰系统**：仅作为命名素材提供者——无接口改动
- **历史系统**：消费 VehicleQuery——无接口改动
- **权力系统**：Adjudicate 原子处理所有权争议——无接口改动

## 三、关键接口契约

### VehicleQuery trait（pub——所有模块消费）

```
基础: vehicle_def() / vehicle_state() / position() / damage_state() / motion_vector()
战斗: collision_bodies() / cover_surfaces()
空间: vehicles_in_radius() / vehicles_at_port()
所有权: vehicles_owned_by() / is_crew_on_duty() / ownership_record()
运输: available_transport()
经济: physical_profile() / cargo_space()
```

### VehicleMut trait（pub(crate)——仅授权模块）

```
update_navigation() / apply_damage() / perform_emergency_patch() / port_repair()
assign_crew() / load_cargo() / unload_cargo()
register_vehicle() / deregister_vehicle()
set_status() / set_owner() / set_name()
```
