# CHG-028: 进阶需求系统 v1.0 创建

## 变更摘要

在基本需求系统（7维，CHG-027）的基础上，创建 NPC 进阶需求系统 v1.0。新增 2 个心理需求维度（尊重/认可 + 胜任挫折），形式化 IntrinsicGoal → action_weight 公式，新增 2 个跨层桥接机制（生存抑制 + 挫折回归），将 NPC 需求框架从 7 维扩展至 9 维。

## 设计过程

通过 8 篇参考讨论草稿（[[../参考文档/025-NPC进阶需求系统设计探讨-20260615/|025-进阶需求系统设计探讨]]）深入讨论了理论框架（马斯洛/ERG/SDT/赫兹伯格）、维度选择（从 6+ 候选维度中精选 2 个）、详细公式设计、跨模块接口审计、性能预算分析。

## 新增接口契约

遵循 CLAUDE.md 跨模块接口契约模式：

| 概念 | 权威 Owner | 消费方 | 关键约定 |
|------|-----------|--------|---------|
| `esteem_deficit` | **NPC.NpcData** | NPC 决策器、情绪引擎 | 心理状态——不入 Life.Vitals。+0.01/游戏日。社交钦佩事件恢复。**零新跨模块推送** |
| `competence_frustration` | **NPC.NpcData** | NPC 决策器、情绪引擎、SelfNarrative | Allostatic 模型——设定点 = aspiration skill gap。每游戏日更新。无 SkillMastery aspiration → 恒为 0 |
| `honor_weight_for_domain()` | **文化系统**（CultureQueryExt 新增方法） | NPC 模块（esteem 计算） | ★ **唯一新跨模块方法**。`domain_code: u8` 参数（非类型依赖）。从已有 CultureCoreParams 派生。零新存储 |
| `NeedSensitivity::esteem` | **NPC 模块** | NPC 决策器 | `0.2 + E×0.5 + (1-A)×0.3`。终身不变 |
| `NeedSensitivity::competence` | **NPC 模块** | NPC 决策器 | `0.2 + C×0.5 + N×0.3`。终身不变 |
| `survival_suppression()` | **NPC 模块** | 仅概率引擎 | sigmoid `1/(1+e^(-10(x-0.7)))`。**不影响 GOAP** |
| `frustration_regression()` | **NPC 模块** | NeedSensitivity 临时调制 | avg_frustration > 0.6 触发。**不持久化** |
| `intrinsic_motivation_weight()` | **NPC 模块** | 仅概率引擎 | `∏(1 + relevance×commitment×0.5)`, clamp [1,3] |
| `NeedTag::Esteem` / `NeedTag::Competence` | **NPC 模块** | ActionType 定义 | 新增 2 个需求标签 |

## 依赖方向

```
进阶需求系统（NPC 模块 — 权威 Owner）
├──→ CultureQueryExt::honor_weight_for_domain()  ← 唯一新方法
├──→ PowerTopologyQuery::perceived_legitimacy()   (已存在)
├──→ npc.skills, npc.relationships, npc.memories (NPC 内部——零新依赖)
└──→ SelfNarrative / IntrinsicGoal                (NPC 内部)

其他模块 → 本模块：零新依赖 ✅
```

## GOAP 边界确认

**不修改** `should_activate()`。心理需求不进 GOAP 安全网——不被认可不会死。

## 性能影响

- 内存: +20 bytes/NPC (+1% NpcData)
- 热路径: +66 ns/决策 (+1.3% 决策器预算)
- VRAM: 0
- 新 trait 对象: 0
- 评估: ✅ 无风险

## 向后兼容

- 所有已有公式保持有效
- `need_action_match` 签名新增 2 个参数——调用方需更新
- `NeedSensitivity` 新增 2 字段——已有构造代码需补充
- `honor_weight()` 不受影响——`honor_weight_for_domain()` 是独立新方法
