# CHG-029: 审美与艺术系统 v1.0 创建

## 变更概述

创建了 WoWorld 的**审美与艺术系统**——一个无状态引擎，定义了所有模块共享的审美"通用语"。这是模块体系中填补"艺术/娱乐系统尚未设计"（参考文档 025/006）空白的首次正式规格。

### 2. 核心架构

- **三层管道**：`AestheticSignal`（6维客观信号）→ `AestheticTaste`（主观滤镜）→ `AestheticJudgment`（4维输出）
- **4 个事件原子**：React（内省·不自主动作）· Articulate（人际·外化）· Adopt（品味传播）· Embellish（审美产出）
- **12 个 HasAestheticSignal 实现者**：覆盖物品/建筑/生物/NPC/载具/景观/表演/技艺过程/战斗/魔法/仪式
- **零状态引擎**：审美模块不存储任何持久数据——对标 `WeatherQuery::sample()` 模式
- **零反向依赖**：审美模块不 import 任何业务模块

### 4. 关键设计决策

1. **6 信号维而非 8**：somatic_impact 和 symbolic_depth 是涌现产物，非原子
2. **judge() 不做生存压制**：压制在注意力层（NPC 感知系统）和效应层（React/Articulate 内部）
3. **FineArts 独立于 Artisan**：工艺≠艺术——交叉训练基础不同（材料 vs 感知），技能进步机制不同（重复 vs 灵感）
4. **确定性噪声**：jitter seed = hash 三元组，±2%，存档可复现
5. **不新增需求维度**：审美不是 deficit-driven——已确认 025/001 的"审美需求被拒绝"结论正确
6. **情欲审美不归入审美模块**：情欲 = 审美×吸引力×libido×社会——四系统交叉涌现
7. **所有复杂现象（艺术市场/时尚/流派/共识）从原子涌现**：零硬编码"画家职业""艺术市场""时尚系统"

### 5. 性能

- 热路径 <0.002ms/帧（1万NPC规模）
- 存储增量 32B/NPC
- VRAM 零增量
- judge() 纯函数，完全可并行化
