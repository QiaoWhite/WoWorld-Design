## 一、概述

创建感官与知觉系统模块正式开发规格。12 文件（11 规格 + 1 README），覆盖视觉、嗅觉、注意与显著性、知觉噪声与偏差四大子域。

**核心定位**：感官系统是 WoWorld NPC 感知世界的**统一入口**——介于世界物理真值与 NPC 心智之间。不做决策、不存记忆、不辨识实体，只做信息压缩与变形。

**一句话**：将世界真值转化为每个 NPC 独一无二的知觉流（`PerceptBatch`），供给日常决策、战斗情报、审美判断和大日志。

## 三、核心设计决策

| # | 决策 | 选择 | 理由 |
|---|------|------|------|
| 1 | 模块定位 | **无状态纯函数管线**——`perceive(params) → PerceptBatch` | 不拥有世界状态。所有状态通过参数传入 |
| 2 | 辨识边界 | **感官不辨识**——只提取物理特征 | "这是龙"或"他受伤了"是信念推导层的事 |
| 3 | 跨模态整合 | **感官不整合跨模态** | 听到脚步声+看到影子→是不是同一个人→信念推导层 |
| 4 | 事件边界 | **感官不定义事件边界** | 记忆系统自己决定什么时候编码事件 |
| 5 | 统一产出 | **同一份 `PerceptBatch` 供给所有消费者** | 日常决策、战斗情报、审美判断、大日志——不区分战斗/日常 |
| 6 | 视觉通道 | **感官 crate 从零实现 `VisionQuery`** | 此前为缺失通道。背景/信号二分类→自适应射线→分级细节→运动检测双路径 |
| 7 | 嗅觉通道 | **感官 crate 从零实现 `ScentQuery`** | 懒采样模型——查询时计算浓度，不模拟扩散。对标 WeatherQuery::sample() |
| 8 | 感知退化 | **距离的连续函数**——LOD 层级只是调度分桶 | 层级数量可随意增减，不改变感知算法 |
| 9 | 噪声确定性 | **PerceptualSeed 三部分 hash** | world_seed × npc_permutation × time_quantum × modality → Pcg64。同存档同时刻同知觉 |
| 10 | 听觉/天气/内感受 | **编排消费已有 trait**——不重新实现 | AudioQuery / WeatherQuery / Physiology::from_vitals() / perceive_sky() 等已由其他模块实现 |
| 11 | trait 所有权 | **woworld_core 定义所有 trait，各 crate 实现，NPC crate 做组合** | 对标 CHG-013 基座契约的 Pattern D（trait inversion） |

## 四、核心数据结构

### PerceptBatch（统一知觉产出）

```
PerceptBatch {
    entities: Vec<PerceptEntry>,       // 可见/可闻实体
    visual_scene: VisualScene,          // 场景光照/天气/能见度
    auditory_scene: AuditoryScene,      // 环境噪声/混响/风
    olfactory_landscape: OlfactoryLandscape, // 当前气味景观
    body_state: BodyPerception,              // 饥饿/疲劳/冷热/疼痛
    environment: EnvironmentPerception,      // 温度/降水/时间/天气
}
```

### PerceptualCache

- 有界（64 实体），LRU 淘汰，不持久化
- 存储于 `SensoryState.perceptual_cache`
- 空缓存 → 新奇驱动探索行为涌现

### VisionParams

- 对标 HearingModel 模式——从 race + perception + age + personality 派生
- 由 Life 模块初始化，存储于 NPC 的 `SensoryState`
- 包含：视觉范围、暗适应参数、运动检测灵敏度、颜色感知

### DarkAdaptation

- 指数松弛模型
- 亮→暗：慢（~30 分钟），暗→亮：快（~30 秒）
- 不持久化——每次决策周期从当前光照条件重新计算

### SensoryState（NPC 组合结构）

NPC crate 不做任何字段设计——只做组合：

```rust
SensoryState {
    hearing: HearingModel,          // 音频系统定义
    vision: VisionParams,           // 感官系统定义
    perceptual_cache: PerceptualCache,  // 感官系统定义
    dark_adaptation: DarkAdaptation,    // 感官系统定义
    voice: VoiceProfile,            // 音频系统定义
    action_log: ActionRingBuffer,   // 音频系统定义
    silence: Silence,               // 音频系统定义
    current_speech: CurrentSpeech,  // 语言表达定义
}
```

## 五、跨模块契约

详见 [[../CLAUDE-INTERFACES|CLAUDE-INTERFACES.md CHG-031 契约表]]。摘要如下：

| 概念 | 消费方 | 关键约定 |
|------|--------|---------|
| `PerceptualCache` | NPC 决策引擎、记忆系统 | 64 实体上限，LRU 淘汰，不持久化 |
| `VisionParams` | VisionQuery 消费 | 从 race + perception + age + personality 派生。对标 HearingModel 模式 |
| `DarkAdaptation` | VisionQuery 消费 | 指数松弛。亮→暗慢，暗→亮快。不持久化 |
| `VisionQuery` trait | NPC crate | 纯函数——不依赖 NpcData。背景/信号二分类→自适应射线→分级细节→运动检测双路径 |
| `ScentQuery` trait | NPC crate | 懒采样模型——查询时计算浓度。气味源由各模块管理，SpatialIndex 存储+过期 |
| `SpatialQuery` trait | 感官/战斗/寻路/音频 | 纯几何+索引——不含业务语义。woworld_core 定义，世界空间管理 crate 实现 |
| `SpatialEntity` | 全部模块 | id+位置+速度+粗分类+隐蔽度+发光量+透明度。不含业务语义 |
| `SpatialEvent` | 感官（L1+L2 消费） | 16m Chunk ring buffer（64），自动过期 max(intensity×10s, 5s) |
| `PerceptBatch` | NPC 决策引擎/战斗情报/审美系统/大日志 | 统一知觉产出——不区分战斗/日常。同一份数据供给所有消费者 |
| `PerceptualModifiers` | VisionQuery/AudioQuery | 状态效果→感知调制。NPC crate 做桥接——零循环依赖 |
| `SensoryState` | 感官系统/音频系统 | NPC crate 只做组合——各子模块拥有字段定义权 |
| GOAP 事实知识源 | GOAP/行为树 | FactSet 每决策周期临时构建——三源：感官缓存+长期记忆+文化基线。GOAP 永不直接查询世界真值 |
| 大日志感官输入 | 大日志（历史 005 消费） | 玩家角色跑轻量感官管线——不做注意筛选/噪声注入/内感受 |
| 战斗情报层级 | 战斗 AI | 消费 PerceptBatch → 派生 CombatantAssessment。战斗不重复感知 |
| 感知退化 | NPC 调度 | 距离的连续函数——LOD 层级只是调度分桶 |
| 感官噪声确定性 | 全部感官模态 | PerceptualSeed（world_seed × npc_permutation × time_quantum × modality）→ Pcg64 |
| `AestheticFrameworks` | 审美系统 judge() | 四独立过程：获取/深化/衰减/激活。NPC 存储，审美系统定义 |
| `CulturalKnowledgeBase` trait | prepare_facts() | 与文化参数查询分离——不同目的，不同 trait |

## 七、设计原则

1. **无状态** — 感官系统是纯函数管线。不拥有世界状态
2. **不辨识** — 感官只提取物理特征，不做语义判断
3. **不整合跨模态** — 视觉和听觉保持独立输出
4. **不定义事件边界** — 记忆系统自己决定何时编码事件
5. **统一产出** — 同一份 PerceptBatch 供给所有消费者
6. **感知退化是连续函数** — 层级只是调度分桶，不改变算法
7. **噪声确定性** — 同一存档同一时刻知觉完全相同
8. **trait 定义在 woworld_core** — Pattern D trait inversion，各 crate 实现，NPC crate 做组合
