## 一、变更概述

植被系统架构升级——将植被从生命层（P3.5）提升至自然基底层（P2.25），与地形/水文同级。核心洞察：植被是"画布"而非"画中之物"——它和山、河一样，是动物/NPC/建筑在其上操作的自然基底。

核心变更六项：

1. **VMC双层空间架构**——75K个1km²植被宏观Chunk（Vegetation Macro-Chunk）存储群落类型/覆盖度/演替阶段；运行时32m×32m Tile Chunk确定性展开，T0实例零存储
2. **群落层从个体层分离**——009-植物定义"原子"（单株橡树的生物学属性），010-植被群落与覆盖定义"统计力学"（千万棵树如何组成一片森林）
3. **VegetationProvider trait（Pattern D trait inversion）**——定义在woworld_core，由woworld_vegetation实现。7方法：query_community / canopy_closure / timber_availability / ground_cover / query_harvestable / fuel_load / root_interference
4. **木材形式化合同（WoodMaterialContract）**——TimberAvailability / TimberQuality在woworld_core单一定义，5模块隐式链路（建筑/经济/物品/NPC物理原子/世界生成）收敛为VegetationProvider单一trait枢纽
5. **P3.5植物迁移至P2.25**——世界生成管线从14阶段扩展至15阶段（插入P2.25），植物从生命世界初始化迁至自然基底生成
6. **群落涌现算法**——香农熵加权优势种筛选（丰富度非硬编码整数）· Voronoi林窗检测 · 并查集斑块连通性 · 演替纯函数推导（f(years_since_disturbance)，零状态标签）

## 四、核心设计决策

1. **植被是自然基底，非生命层**——植被覆盖在地形生成后立即确定（P2.25，位于P2和P2.5之间），与地形/水文同级。个体植物在细节层展开（P2.25 Poisson盘近场预展开 + 运行时TC确定性展开）。P3.5生命世界初始化不再生成植物——改为消费VegetationCover。

2. **VMC双层空间架构**——75K个1km²植被宏观Chunk（VMC）存储群落类型/覆盖度/演替阶段/木材品质，模板约100B/VMC，全量~7.5MB。运行时32m×32m Tile Chunk（TC）从父VMC+种子确定性展开：Poisson盘个体 + 垂直分层 + L6.5密度场（1m²分辨率）+ 地面覆盖。T0实例零存储——PlantInstanceId从hash(seed, "plant_instance", cx, cy, instance_index)推导。

3. **VegetationProvider trait（Pattern D trait inversion）**——定义在woworld_core（与TerrainQuery/SpatialQuery/VisibilityQuery/EntityIndex四trait同级），由woworld_vegetation crate实现。消费方依赖woworld_core trait，不依赖Life crate植物类型。7方法覆盖所有跨模块植被查询。

4. **木材形式化合同（WoodMaterialContract）**——TimberAvailability / TimberQuality在woworld_core单一定义，是"木材可获得性"的唯一事实来源。5模块隐式链路（Life::ProductCategory::Wood → Items::WoodMat → World Gen::MaterialAvailability → NPC物理原子::MaterialProperties → Building::MaterialClass::Wood）收敛为VegetationProvider::timber_availability()单一trait枢纽。Building crate的Cargo.toml不含woworld_life依赖——Building完全不import植物类型。

5. **TimberQuality从形态参数涌现**——不在PlantSpecies中硬编码timber_quality字段。derive_timber_quality()纯函数：f(wood_type, max_trunk_diameter, growth_rate, spirit_capacity)。新增"金属木"植物→只需woworld_vegetation内添加映射规则→Building/Items/Economy零改动。

6. **香农熵加权优势种筛选**——群落优势种不硬编码"这个群系有3个树种"。丰富度=香农熵×土壤肥力的连续函数：高熵+高肥力→5个优势种共存（热带雨林）；低熵+低肥力→1-2个优势种（泰加林）。过渡带→斑块镶嵌涌现。

7. **演替纯函数推导**——不在PlantCommunitySnapshot中存储succession_stage枚举标签。演替阶段=f(years_since_disturbance, soil_fertility, climate)，sigmoid过渡（非硬阈值）。同一片森林50年后重访→自动从YoungForest过渡到MatureForest。NPC砍伐→T3写入disturbance→years_since_disturbance重置→该Chunk回到Pioneer阶段。

8. **森林斑块从连通性涌现**——不做预定义ForestRegion硬边界多边形。并查集UnionFind在冠层密度图上4邻域连通→连通分量=森林斑块。不同连通分量被道路/河流/草原自然隔开。

9. **Voronoi林窗检测**——每棵树=Voronoi cell中心。正常密林Voronoi面积≈物种典型间距²。林窗=Voronoi面积远大于典型间距（此处应有树但缺失）。密丛=Voronoi面积远小于典型间距（竞争过密，自疏将发生）。大树周围Voronoi面积大→自动识别"这棵是林中之王"。

10. **种子确定性全链路**——每个VMC的RNG从hash(world_seed, "vegetation", vmc_x, vmc_y)派生。每个TC从父VMC种子+TC坐标展开。PlantInstanceId从hash(world_seed, "plant_instance", chunk_x, chunk_y, instance_index)推导。同一种子→同一群落→同一Poisson盘→同一个体→T1/T3 Delta正确匹配。

### 6.2 P2.25 生成算法（5步）

| 步骤 | 内容 | 粒度 |
|------|------|------|
| Step 1 | 优势种筛选：PlantSpeciesRegistry::query() → 香农熵加权 → 1-5个优势种 | VMC (1km²) |
| Step 2 | 多层噪声群落密度：第1层大尺度密度(~50m)→密林/空地；第2层演替阶段(~100m)→成熟/再生/林窗；第3层物种混合(~30m)→优势种空间分布 | VMC |
| Step 3 | 垂直分层光竞争：冠层→林下→灌木→地被 依序放置，每层光照=f(上层覆盖度) | VMC |
| Step 4 | Poisson盘个体放置（仅出生点近场~10km半径·~200 VMC·~195K TC） | TC (32m²) |
| Step 5 | 推导输出：冠层郁闭度→TimberAvailability·地表覆盖→GroundCoverMap·垂直结构→L6.5密度场·物种比例→香农熵 | VMC |

## 七、关键隔离与设计原则

### 7.1 trait间接消费三角

```
woworld_core
  ├── VegetationProvider trait（定义）
  ├── TimberAvailability / TimberQuality / GroundCoverMap（共享类型）
  │
  ├─▲ 实现
  │     woworld_vegetation crate
  │       从PlantSpecies推导木材品质
  │       从群落模板推导冠层密度
  │       读取Life crate::PlantSpeciesRegistry（只读）
  │
  └─▼ 消费
       Building crate
         BuildContext.materials: HashMap<MaterialClass, Availability>
         Cargo.toml: 不含woworld_life依赖
         仅知道MaterialClass::Wood（woworld_core定义）
```

### 7.2 六项原则

| 原则 | 内容 |
|------|------|
| **植被是自然基底，不是生命层** | P2.25位于P2和P2.5之间——和地形一样是画布 |
| **trait隔离** | 消费方依赖woworld_core trait，不依赖Life crate植物类型。Building完全不知道PlantSpecies的存在 |
| **木材品质涌现** | TimberQuality = f(wood_type, max_trunk_diameter, spirit_capacity)。新增金属木植物→只需woworld_vegetation内添加映射规则 |
| **群落从数学涌现** | 香农熵、Voronoi、并查集——从连续参数场自然推导，零硬编码分类 |
| **演替是纯函数** | 不存succession_stage标签。同一片林随years_since_disturbance流逝自动演进 |
| **种子确定性** | 每Chunk RNG从hash(seed, "vegetation", cx, cy)派生。T0实例零存储 |

## 九、自检清单

- [x] VegetationProvider trait 定义在woworld_core（不依赖Life crate）
- [x] Building crate 的 Cargo.toml 不含woworld_life依赖
- [x] TimberQuality 从PlantSpecies形态参数推导，不硬编码
- [x] 演替阶段=f(years_since_disturbance)纯函数，不存枚举标签
- [x] 香农熵加权优势种——丰富度非硬编码整数
- [x] Voronoi林窗检测——零预定义区域
- [x] 并查集斑块连通性——森林从连通性涌现
- [x] Poisson盘仅出生点近场预展开（~200 VMC），远距离延迟至运行时
- [x] PlantInstanceId 从 hash(seed, "plant_instance", cx, cy, idx) 推导——T0实例零存储
- [x] P2.25 不消费 P2.25 之后的任何阶段数据（仅消费P0-P2）
- [x] 木材从 VegetationCover 推导，P3 不重复生成木材zone
- [x] GroundCoverMap 覆盖所有陆地Chunk（无空值）
- [x] 魔法植物仅在 magic_concentration > 0.4 时出现
- [x] 世界生成耗时增量在预算内（P2.25初始前台~16s + 后台~20s = 全量~36s，16线程rayon）
- [x] 运行时查询在帧预算内（~0.1ms/帧）
