# CHG-052 — 玩家游玩内容全貌设计

> **创建日期**: 2026-06-20
> **状态**: 设计变更追踪
> **范围**: 36 问玩家旅程全貌设计 + 法律涌现分析

---

## 〇、动机

WoWorld 当前 24 个模块覆盖了物理世界、NPC 心智、社会系统、交互系统——但"玩家实际玩什么"从未被系统性地从玩家体验角度串联。本次变更不创建"玩家旅程系统"独立模块，而是：

1. 验证已有 24 个模块的交汇点已覆盖全部玩家游玩场景
2. 创建少量补充文档填补真实缺口
3. 识别出需要扩展的现有模块接口
4. 将遗漏项归档供用户后续解决

## 三、核心设计决策汇总

### 3.1 架构决策

| 决策 | 结论 |
|------|------|
| 是否需要"玩家旅程系统"模块 | **否**。全部从已有24个模块交汇涌现 |
| 是否需要独立法律模块 | **否**。法律从 PrescribeRule+Sanction+Adjudicate+NPC决策+信息传播 涌现 (17原子已全覆盖) |
| 是否需要"队伍/伙伴系统" | **否**。伙伴关系 = 权力系统17原子 + NPC自有决策 |
| 是否需要"家系统" | **否**。家 = 建筑+物品五层容器+权力所有权+文化待客+NPC邻里 |
| 是否需要独立战争系统 | **否**。宣战(权力GOAP)+动员(Compel)+后勤(经济)+交锋(战斗)+伤亡(生命)。唯一新内容=方阵统计优化层 |
| 是否需要"海洋系统" | **否**。游泳(SWIM原子)+造船(配方/Blueprint)+风暴(天气×波)+潜水(憋气×压力)+岛屿(世界生成)。全是已有系统的延伸 |
| 是否需要"农业系统" | **否**。农业 = TILL/PLANT/HARVEST复合原子 + 植物生长公式 + 动物繁衍 |
| 是否需要独立制造系统 | **否**。制造 = 配方表驱动 + 物理原子序列 + 玩家决策点穿插。生活制造走配方表(开发负担低) |
| 是否需要"欺诈/计谋系统" | **否**。欺诈 = NPC自主决策。识破 = MentalModel+非语言信号+多来源印证 |
| 是否需要"名声系统" | **否**。名声 = 分散式 NPC记忆 × 信息传播 × 共识。不聚合为"声望分数" |
| 魔法是否需要独立结算通道 | **否**。魔法 = 复合原子层(CAST/CHANNEL等)消费物理原子层(IGNITE/WET/CONDUCT)。火球烧木门和火把烧木门同一行代码 |

### 3.2 数据流决策

| 决策 | 结论 |
|------|------|
| 大日志归属 | 角色知识库——共享给两个可控角色使用。不衰减、无容量上限(区别于NPC记忆) |
| 名声存储 | 分散式——不做聚合分数。存在每个NPC的EventMemory + trust数据中 |
| 玩家角色创建 | 完全自由填写——从零到龙傲天。数值范围=各数据结构的字段类型约束。无"角色创建专属限制层" |
| 人格修改 | 和NPC一样——终身极少变动。仅极高冲击力创伤(>0.9)或长期环境冲突(>100日)时微调(Δt≤0.05) |
| 死亡继承 | 血亲(一代以内)→大日志任何人→全新角色。大日志可选继承/封印/清除 |
| 双角色关系 | 和NPC间关系同机制涌现——不被系统特殊对待 |

### 3.3 玩家体验决策

| 决策 | 结论 |
|------|------|
| 任务系统 | 不存在。WoWorld没有任务 |
| 驱动力 | 世界本身→地形文化(探索动力)+活人感(社交动力)+怪物(战斗动力) |
| 引导 | 小精灵可选辅助(可关闭)。除此之外无引导层 |
| 旅行 | 不跳过时间。走在路上就是走在路上。遭遇=世界本来就有的动态实体 |
| 战斗 | 半自动(龙之信条/Kenshi式)——角色自己打,玩家插手关键决策 |
| 制造 | 玩家是策略者,角色是执行者。决策点穿插在AI自动序列中。无进度条 |
| 信息发现 | 感官系统+对话/传闻+大日志=全覆盖。无额外层 |
| 探索回报 | 生成器统计尾部的稀有配置+历史残留+信息跨空间搬运+纯美学瞬间 |
| 时间跳跃 | 仅当双角色都睡/不可活动时可选。尽可能避免跳过/模糊模拟 |
