## 变更动机

### 发现的问题

1. **P8 存在 7 个一致性 bug**：
   - Bug 1：婚姻配对使用 `social_standing ±1 tier`，但 `social_standing` 在 Phase C 才填充 → 循环依赖
   - Bug 2：`blood_distance ≥ 2` 约束在 FamilyTree 几乎为空时不可计算
   - Bug 3：Step 3（生成子女）可能将活人 NPC 错误地匹配给多对不同的夫妻
   - Bug 4：反推祖先时用 `mortality_rate[parent_age_at_child_birth]` 查死亡率——应查父母当前年龄的死亡率
   - Bug 5：递归停止条件 `父母年龄 < settlement_founding_year` 类型不匹配——"年龄"是岁数，"年份"是 i32
   - Bug 6：跨聚落婚姻导致人口迁移，但聚落人口计数未更新
   - Bug 7：已故祖先没有 `death_age`——Gompertz 预采样缺失

2. **P8 引用不存在的字段**：`culture.fertility_rate`、`culture.marriage_age_avg` 在 `CultureCoreParams` 和 `FertilityNorms` 中均不存在。`settlement_founding_year` 在 `SettlementData` 中不存在。

3. **P9a 性能严重不足**：天真年步长算法 `O(年×聚落²×技艺)` ≈ 2000×1000²×20 = 400 亿次迭代，预估 1-3s 严重偏低估。

4. **婚姻体系不完整**：P8 仅处理异性一夫一妻。缺少同性伴侣、多配偶制、非婚生子女、跨信仰婚姻约束。

## 变更内容

### 一、文化系统 004 — 新增类型和方法

**FertilityNorms 新增派生方法**（不新增 struct 字段）：
- `annual_fertility_rate(&self) → f32` — 从 `ideal_family_size` 反推年生育率
- `marriage_age_typical(core: &CultureCoreParams) → (u8, u8)` — 从 `CoreParams` 派生初婚年龄

**新增 RelationshipNorms struct**（第一级派生参数，扁平字段，不依赖 NPC 模块类型）：
- 数量结构：`monogamous_default`、`polygyny_acceptance`、`polyandry_acceptance`、`polyamory_acceptance`、`group_marriage_acceptance`、`max_spouses`
- 形成方式：`arranged_marriage_tendency`、`contractual_marriage_acceptance`
- 性别与取向：`homosexuality_acceptance`、`same_sex_marriage_legal`
- 生物与社会：`conjugal_birth_norm`
- 离婚：`divorce_acceptance`
- 居住模式：`residence_pattern: ResidencePattern`（Patrilocal/Matrilocal/Neolocal/Ambilocal）
- 完整派生公式（从 10 个 CoreParams 推导）

### 二、世界生成 P4 — SettlementData 新增字段

- `founding_year: i32` — 聚落建立年份。从文化起源年份 + 障碍距离延迟 + 等级延迟推导。消费方：P8 Phase B Pass 1（祖先回溯截止）· P9a（知识传播起始）

### 三、世界生成 P8 — Phase B 五 Pass 重构（v2.0 → v2.1）

**Phase A 新增**：
- `rough_standing: u8`（1-4 层，从文化分布 + 聚落等级采样）
- `gender_attraction: BTreeMap<BiologicalSex, f32>`（取向，从 `RelationshipNorms` 分布采样）

**Phase B 五个 Pass**（替代原"婚姻→子女→祖先"顺序）：
1. **Pass 1 — 祖先创建**：向上创建父母实体，聚类合并同胞父母（动态半径：小聚落 3 年/大聚落 1 年）。递归 ≤ 3 代，截止于 `founding_year`。验证 `parent.birth_year < child.birth_year - 14`。
2. **Pass 2 — 配偶池合并**：小聚落（<200 人）按距离概率合并邻近聚落候选池。
3. **Pass 3 — 社会关系初始化**：贪心 O(N) 统计分配。消费 `RelationshipNorms` + `ReligiousReproductionNorms`。规则：生物学父母确定性结婚（`conjugal_birth_norm ≥ 0.8`）· 异性婚配（阶层 ±1·血亲距离 ≥ 2·跨信仰检查）· 同性伴侣（仅取向兼容者，MarriedTo vs PartneredTo 取决于 `same_sex_marriage_legal`）· 多配偶制· 配偶池枯竭处理。
4. **Pass 4 — 已故子女生成**：消费 Pass 1 父母对，泊松生成已故同胞。子女出生年份在双方存活且年龄 ≥ 14 期间。
5. **Pass 5 — DAG 验证**：无环检查· MarriedTo/PartneredTo 对称·父母年龄约束。

**Phase D 新增**：婚姻质量审计——Phase C 精确 `social_standing` 对比 Phase A `rough_standing`，标记跨阶层婚姻（`MarriageFlag`）。

**新增消费接口**：`FertilityNorms`、`RelationshipNorms`（文化系统 004）、`ReligiousReproductionNorms`（信仰系统 002）

**新增产出**：`Vec<MarriageFlag>` — 跨阶层婚姻标记，供传说/八卦系统消费

**消除的 bug 对应表**：

| Bug | 修复方式 |
|-----|---------|
| 1 (social_standing 循环) | Phase B 使用 `rough_standing`（Phase A 产出，不依赖 Phase C）。Phase D 审计微调 |
| 2 (blood_distance) | Pass 1 先行建立父子关系，Pass 3 使用 `blood_distance ≥ 2` 已可用 |
| 3 (子女错误认领) | 活人 NPC 的父母在 Pass 1 创建（不被 Pass 4 覆盖）。Pass 4 仅生成已故子女 |
| 4 (死亡率年龄) | 查父母当前年龄 = parent_age_at_child_birth + child_current_age |
| 5 (递归停止) | `parent.birth_year < settlement.founding_year`（配合 P4 新增字段） |
| 6 (人口泄漏) | Pass 3 跨聚落婚姻按 `ResidencePattern` 迁移 → 更新聚落人口。P13 按当前居住地验证 |
| 7 (祖先无年龄) | Gompertz 预采样 `death_age`，Pass 1 创建实体时赋值 |

### 四、世界生成 P9a — 事件驱动优化

从年步长 O(年×聚落²×技艺) 改为优先队列事件驱动 O(E×logE)，E ≈ 300-500 传播事件。

新算法：
1. 三类发现（Inevitable/Conditional/Breakthrough）直接采样发现年份并入队
2. 优先队列驱动——每个到达事件触发传播调度
3. 传播延迟 = distance / propagation_speed，仅当 `propagation_year ≤ 0` 时入队
4. 产出完全相同的 `SettlementTechState`（含精确 `tech_arrivals` 年份和来源）

性能：~1-3s → ~0.1-0.3s

### 五、世界生成 P13 — 新增校验与死因装饰

**新增校验**：
- §六之二：FamilyTree 结构完整性（DAG 无环· 婚姻对称· 父母年龄 ≥ child.birth_year - 14）

**新增终组步骤**：
- §7.0：`decorate_death_causes()` — 遍历 EventLog 大规模死亡事件，覆盖受影响死者的 `death_cause`（Natural → Battlefield/Plague/Starvation）

**人口验证修正**：
- 按当前居住地（`npc.current_settlement()`）非出生地（`npc.birth_settlement`）验证——跨聚落婚姻迁移导致两者不同

### 六、性能预算更新

| 阶段 | 原预算 | 新预算 |
|------|--------|--------|
| P8 Phase B | ~200ms（四阶段） | ~180ms（五 Pass + 贪心 O(N)） |
| P9a 知识传播 | ~1-3s（年步长轮询） | ~0.1-0.3s（事件驱动） |
| **P8+P9a 合计** | **~1.7-4.5s** | **~0.8-1.0s** |
| **总生成时间** | **5-13s（典型 ~8s）** | **3-10s（典型 ~5-6s）** |

## 已知限制

以下问题在文档中明确声明，不在此次 CHG 中修复：
- 已故子女的后代不在 FamilyTree 中（Pass 4 生成的是终端节点）
- 祖先链深度 ≤ 3 代（~75 年）。超过 3 代的祖先不追溯
- 祖先技能水平使用 year-0 天花板（非其出生年代的实际技术水平）
- 独身要求（祭司等）在生成时不处理——交予运行时。Phase C 后才知道独身职业
- 收养/继亲关系不在 FamilyTree 中表达
- Phase C 的 `family_inheritance_bonus` 对已故祖先使用懒推导 fallback（非预存职业数据）
