## 一、背景与动机

### 1.1 `/grill-me` 深度访谈

用户对 NPC 心智的六个原始问题——想象、幻觉、群体心理、记忆篡改、记忆压缩、知识抽象——触发了 12 轮深度讨论。讨论中暴露出 10 个设计缺陷：5 个错误、5 个遗漏。

### 1.2 核心空白（10 个缺陷中 6 个关键空白见下表；其余 4 个为架构细节——见自审 §三）

| 空白 | 严重度 | 影响 |
|------|--------|------|
| `PatternExpression` 被 MentalModel 引用但从未定义 | 关键 | 整个认知系统的数据结构地基缺失 |
| `EmotionalCharge` 被 PerceptEntry 引用但从未定义 | 关键 | 情绪传染、思维触发、信念评估的输入类型为空 |
| `formalize_innovation()` 11/17 域返回 None | 高 | 知识领域（史学、数学、哲学）的创新管线断裂 |
| 感官→认知的桥梁层未指定 | 高 | 跨模态绑定、视线感知、情绪读取无明确位置 |
| 没有修辞/比喻的语言机制 | 中 | NPC 能做类比思维但不能表达为比喻 |
| 没有数学技能支持数学职业 | 中 | Mathematician/TaxCollector 职业无技能支撑 |

## 二、统一设计原则

### 回顾性原则

**NPC 心智是回顾性的记忆加工引擎，不是前瞻性的世界模拟器。**

```
正确：已编码经验 ──[数学变换]──→ 认知产物
错误：当前状态 ──[世界模拟]──→ 假想场景 ──→ 认知产物
```

适用范围：
- **想象** = 已存储模式的再组合（Creative Leap），不是空白生成
- **预测** = 将已有 MentalModel 应用于候选路径（模式匹配），不是世界仿真
- **反事实后悔** = 已有模式的预测结果与实际结果的比较，不是模拟替代时间线
- **深思熟虑** = GOAP 效用评估 + MentalModel 预测的加权比较，不是 Monte Carlo 搜索

例外：GOAP 的 A* 搜索做的是**物理可行性预演**（AgentSnapshot 参数 + 材料属性），不是认知模拟。两者在不同层。

### 设计检验

任何新 NPC 心智功能需回答：
1. 数学输入全部来自已有数据吗？→ 通过。
2. 需要模拟尚未发生的事件吗？→ 不通过。不属于 NPC 心智。

## 三、核心架构裁决

### 3.1 PatternExpression 数学地基

**裁决**：在 woworld_core 中定义 PatternExpression 的数据结构、LSH 结构指纹、DomainSignature 领域签名。

```
PatternExpression {
    steps: ArrayVec<PatternStep, 8>,    // ≤8 因果步骤，栈分配
    structural_lsh: u64,                // 结构指纹——用于 Creative Leap 跨域匹配
    domain_signature: u64,              // 领域签名——用于 formalize 路由、学科聚类
}
PatternStep {
    atom_class: AtomClass,              // u16 统一原子分类（命名空间+索引）
    relation: CausalRelation,           // Before/Causes/Enables/Prevents/Intensifies
    role: StepRole,                     // Precondition/CoreAction/Effect/Modifier
}
```

- `structural_lsh`：O(1) ≈ pattern 结构相似度。预先哈希，Hamming 距离。图拓扑丢失是已知的——Creative Leap 是概率门，不完美 = 涌现条件。
- `domain_signature`：O(1) ≈ 领域分类。含 atom_class 完整命名空间 + 上下文哈希（诱导记忆中的 EntityKind、MaterialCategory 等）。
- `AtomClass`：u16 编码，高 4 位 = 命名空间（Physical=0, Power=1, …），低 12 位 = 索引。域 crate 实现 `IntoAtomClass` trait。

### 3.2 MentalModelDomain 枚举 → DomainSignature

**裁决**：完全移除 17 个硬编码 MentalModelDomain 变体。领域从 AtomTypeTag 组合中涌现。DomainSignature(u64) 承载领域分类。

影响：
- Creative Leap 跨域采样：Jaccard 在 [0.1, 0.6] 的配对（不基于枚举不等）
- 学科涌现：NPC 群体的 MentalModel 在 DomainSignature 空间聚类 → proto-discipline
- formalize_innovation()：按 ATOM_MASK 注册消费者，未注册的默认 → AcademicWork

### 3.3 深思熟虑：连续涌现量

**裁决**：`deliberation_depth = trait_capacity × state_capacity × stakes`

- `trait_capacity`：reflective_impulsive × (1 − rigid_flexible × 0.3)。稳定特质。
- `state_capacity`：mind_quietude × (1−load) × (1−intoxication) × (1−fatigue) × (1−distress) × (1−|arousal|×0.7)。此刻脑子清不清醒。
- `stakes`：候选路径的 predict_outcome() 方差 → sigmoid 压入 [0,1]。

醉酒 + 情绪激动的人面对人生抉择：capacity≈0.1 → 几乎不思 → 冲动决定。不硬编码。

### 3.4 记忆的四层压缩架构

**裁决**：用信息密度驱动的连续压缩率替代二元冷热分界。

| 层 | 容量 | 大小/NPC | 压缩触发 | 保留内容 |
|----|------|---------|---------|---------|
| L0 Hot Episodic | ≤2000 | ~512KB | compression_rate>0.8 或超上限 | 全字段 |
| L1 Cold Summary | ≤500 | ~100KB | 超上限→提升至 L2 | text+emotion+participants+location |
| L2 Era Digest | ≤20 | ~4KB | 超上限→合并最相邻时代 | theme_tags+情绪曲线+代表事件×3 |
| L3 Life Abstract | ≤5 | ~1KB | 终生存留 | 核心叙事弧+pivotal_moment+人生主题演化 |

总计 ~617KB/NPC。信息密度公式：

```rust
fn information_density(mem: &EventMemory, days_since: f32) -> f32 {
    mem.impact_score * (0.3 + mem.emotional_encoding.arousal * 0.7)
    * (1.0 + mem.access_count as f32).sqrt()
    / (days_since + 1.0).ln_1p()
}
```

`access_count.sqrt()` 是叙事偏置的关键——经常检索的记忆（因为符合 SelfNarrative）密度更高，但有递减收益。

### 3.5 记忆源混淆

**裁决**：episodic memory 不做社交覆写。虚假记忆通过源混淆涌现。

EventMemory 新增：`original_source: MemorySource`（编码时写入，永不变）、`source_confidence: f32`（随时间衰减，<0.3 时概率性 misattribution）。

编码时 crowd_emotional_field 调制 source_confidence → 恐慌人群中传闻被编码为源不可靠 → 数月后源混淆 → "我记得我亲眼看到了"= 虚假记忆涌现。

### 3.6 感官→认知桥梁

**裁决**：woworld_core 中的 `PerceptBatch` extension methods。零新 crate。零新 trait。

```rust
impl PerceptBatch {
    pub fn enrich(&self, cache: &PerceptualCache, observer: Vec3) -> EnrichedPerceptBatch;
    pub fn bind_modalities(&self, cache: &PerceptualCache) -> Vec<BoundEntity>;
    pub fn estimate_gaze(&self, observer: Vec3) -> Vec<GazeEstimate>;
    pub fn read_emotions(&self, cache: &PerceptualCache) -> Vec<EmotionalReading>;
}
```

新增结构体全部在 woworld_core 中定义：`EmotionalCharge`、`GazeEstimate`、`BoundEntity`、`EnrichedPerceptBatch`。

### 3.7 修辞渲染

**裁决**：修辞 = Creative Leap（认知侧跨域类比）+ TextGenerator 修辞化渲染模式（语言侧）。

`rhetorical_ability = f(wisdom, abstract_concrete, rigid_flexible, charisma, language_proficiency)`。不新增技能。

`UtteranceConcept` 新增 `domain_sig: Option<u64>` 字段。TextGenerator 检测相邻概念的 domain_similarity < 0.4 → 触发修辞化渲染（明喻、暗喻、类比解释）。

### 3.8 其他裁决

| 裁决 | 结论 |
|------|------|
| 幻觉/认知错误 | **False positive Creative Leap = 涌现式迷信，不需过滤层。** Bayesian 验证循环自然淘汰。source_confidence 衰减 → 源混淆 → 虚假记忆涌现。CognitiveBreak（4 个 failure mode）是极端认知错误的涌现。7 个 cognitive biases 是系统性扭曲——不是 bug。所有"错误"从正常认知机制的交互中涌现 |
| 群体心理 | `crowd_emotional_field` ∈ `EnvironmentPerception`（感知产出）。情绪跑在信念前——事后归因驱动信念采纳。 |
| 学术研究 | 创新管线应用于知识领域。formalize 默认产物 = AcademicWork（著作）。 |
| 数学 | 领域技能走物理原子→复合原子→GOAP 三层。精度由 skill_level 的 execution_noise 决定。 |
| 实地调查 | MentalModelTest Intent + GOAP 规划的原子序列。零新原子。
| 学科涌现 | DomainSignature 空间事件驱动增量聚类。设计者不规定有哪些学科。 |

## 七、性能影响

- **PatternExpression**：~120B/个 × 20 个/NPC = ~2.4KB ≈ 已有 MentalModel 预算内
- **四层压缩**：~617KB/NPC 总计 → 1000 L1 = ~617MB。L2/L3 在睡眠期间处理，零帧开销
- **感知富化**：`enrich()` 每决策周期每 L1 NPC <0.01ms。1000 L1 <0.1ms/frame
- **deliberation_depth**：每决策周期 3 次 predict_outcome() = 3 次 MentalModel 查询 ≈ <0.001ms。仅当 GOAP top2 权重接近时触发
- **学科聚类**：事件驱动增量更新。新模型插入 O(k) 其中 k<100。不占用帧预算
- **修辞渲染**：TextGenerator 在 Utterance → String 时附加的模式匹配。仅在生成文本时触发，非每帧
