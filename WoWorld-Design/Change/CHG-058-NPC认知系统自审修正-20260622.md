## 二、致命缺陷修正

### 2.1 deliberation 不改变决策 → deliberation override

**原问题**：`deliberation_depth` 产出一个 f32 深度值——触发慢下来、停顿、喃喃自语——但最终选择的行动仍然是 GOAP 效用最大化。

**修正**：两阶段惰性触发。Stage 1（廉价初筛，depth>0.2 才进入 Stage 2）。Stage 2 为所有候选调用 `predict_outcome()`——预测方差大或模型少于 2 的候选不调制。`final_depth > 0.3` 时每候选权重乘以 `1.0 + expected_pleasure × depth × 1.2`（clamp [0.3, 1.7]）。调制可翻转 GOAP 决策——但只在预测低方差 + 多模型支撑时才发生。

### 2.2 玩家→NPC 信念传播缺失 → 桥接函数

**原问题**：六条信念传播路径全是 NPC↔NPC。玩家对 NPC 说话——NPC 的认知系统没有通路。

**修正**：`receive_player_speech()` 从 player Utterance 合成 incoming MentalModel → `assess_and_integrate_mental_model()`。PlayerReputation 已有。玩家被 NPC 认知系统视为 TaughtByNpc(PLAYER_ENTITY_ID)——与任何其他 NPC 一样。零新 trait。

### 2.3 L2 Era Digest 合并黑箱 → 完整算法

**原问题**：`detect_merge_candidates` 被调用但从未定义。"合并最相邻时代"的"相邻"是什么——无定义。

**修正**：`era_similarity() = theme_tags_Jaccard×0.4 + emotion_cosine×0.35 + location_overlap×0.25`。相似度 > 0.5 的配对 → 贪心合并。合并后 representative_events 6→3（按 information_density），主题标签并集×频率排序取前 8，情绪按 span_days 加权平均。时间作为打破平局的第二标准。

## 三、高严重度缺口修正

### 3.1 认知老化路径的两个核心函数从未定义

**原问题**：`derive_aging_path()` 的两个入参 `crystallized_factor()` 和 `cognitive_engagement_score()` 被调用但从未定义。

**修正**：
- `crystallized_factor() = 0.3 + analytic_intuitive×0.25 + rigid_flexible×0.15 + (1−damping)×0.15`。damping 从 life_event_count 重算。典型 0.3–0.7。
- `cognitive_engagement_score()` 聚合 creation_count + skill_xp_diversity + social_density + profession + belief_diversity + creative_leap_count。消费 `&NpcData`。
- 新增 `health_burden()` = health_history×0.3 + head_trauma×0.05 + chronic_intoxication×0.02。高 burden（>0.3）有独立的致病通路。
- `pathological_annual_degradation()` —— severity = years/10。逐年：activation_threshold↑, decay_factor↑, source_confidence↓, planning_horizon↓, cognitive_damping↑。

### 3.2 source_confidence 衰减不分来源类型

**原问题**：所有 MemorySource 类型用同一个 decay_factor。

**修正**：`source_confidence_decay_rate(MemorySource) → f32`。Witnessed×0.97/yr, Inferred×0.95/yr, ReadFrom×0.90/yr, HeardFrom×0.85/yr。Inferred 初始最低（0.40）但衰减最慢——5 年后超越 HeardFrom（0.70→0.31 vs 0.40→0.31）。Python 验证通过。

### 3.3 信念级联 vs 情绪级联

**裁决**：信念的认知级联从已有三个机制的组合中完全涌现（PublicWisdomSharing + crowd_suggestibility + rumor per-hop decay）。不新增系统。文档化。

### 3.4 教育→认知

**裁决**：已有管道涌现（DeepMentorship + ReadingSession + WisdomSharing）。不新增直连。文档化。

## 四、其他修正

### 4.1 领域级遗忘

`domain_atrophy()` —— DomainSignature Hamming < 0.3 分组（简单分组，不依赖 006 的 HDBSCAN*）。10 年 inactivity → atrophy=0.5。activation_threshold↑ + confidence↓。

### 4.2 回顾性原则修正

"回顾性优先"→"基于模式的向前投射"。附带三问判定标准：输入来自已有数据？/ 单查询非链式？/ 输出带不确定性？

### 4.3 判定标准

三问 checklist 写入 003 §十六。用于评估所有未来 NPC 心智功能提议。

### 4.4 deliberation_depth 字段路径修正

`npc.agent_snapshot.intoxication` → `npc.physiology.intoxication`；`npc.agent_snapshot.fatigue` → `npc.physiology.fatigue`。AgentSnapshot 是 transient，不存于 NpcData。

### 4.5 NpcData 新增字段

- `mental_model_creation_count: u32` —— try_induce_pattern() 或 creative_leap() 成功时递增
- `chronic_intoxication_years: f32` —— 年均更新（avg_daily_intox × days/365）

### 4.6 ActionCandidate.prediction

`prediction: Option<OutcomePrediction>` —— deliberation 未触发时 None。零运行时成本。

### 4.7 不作为本 CHG 的修改

| 问题 | 理由 |
|------|------|
| NPC 操纵 NPC 偏误 | 涌现行为——MindAttribution + DeceptionIntent 已支持。不建新系统 |
| 玩家认知 | 范围外——玩家靠自己的脑子 |
| LifeHistory struct 定义 | 生命周期系统需要但不在本次范围——`cognitive_engagement_score()` 消费 `&NpcData` 绕开 |
