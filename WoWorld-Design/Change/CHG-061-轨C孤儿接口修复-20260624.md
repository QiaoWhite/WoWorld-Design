## 修复 1: ProfessionTag 所有权理顺（Phase 1）

### 问题本质
`经济系统/001-经济系统总纲.md` 中定义了 `pub struct ProfessionTagId(u32)`，同时声明"经济系统不拥有职业标签"。CHG-060 宣称应移至 `woworld_core` 但未落实到文件。

### 决策
按 ID 类型所有权契约（[[CLAUDE.md]]），所有 ID 类型统一定义在 `woworld_core`（零依赖 crate）。Economy 拥有 ProfessionTag **概念**（TOML schema 定义、标签目录、incongruity 规则）。

### 修改文件

| 文件 | 修改内容 |
|------|---------|
| `经济系统/001-经济系统总纲.md` §3.4 | 移除 `pub struct ProfessionTagId(u32)` 内联定义 → 替换为迁移标注 "★ 类型定义已提升至 woworld_core"。保留 TOML schema 定义和 ProfessionEngagement。 |
| `00-全局基础设施/001-核心类型注册表.md` | 新增 ProfessionTagId 条目——与 EntityId/ItemDefId/SkillId 并列。记录概念 Owner=经济系统。 |
| `14-经济/001-接口出口.md` ProfessionTag 条目 | 重写——Owner=经济系统（概念/TAG TOML schema），类型=woworld_core，数据为中性 TOML。 |
| `03-世界生成/002-接口入口.md` ProfessionTag 条目 | 更新——标注经济为概念 Owner，来源=woworld_core。 |
| `CLAUDE-INTERFACES.md` ProfessionTag 行 | 更新——Owner=经济系统（概念/TAG TOML schema），类型=woworld_core，消费者=世界生成+NPC+技能。 |
