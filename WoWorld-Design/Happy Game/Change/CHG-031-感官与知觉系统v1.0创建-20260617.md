## 〇、变更概要

创建 WoWorld 感官与知觉系统 v1.0。这是首次全面设计 NPC 如何感知世界——在这之前，视觉感知、嗅觉感知、统一注意引擎、感官噪声模型等均为空白。听觉/天象/内感受/情绪感知/非语言感知已有各自模块的完整设计——感官系统统一编排消费这些已有通道，并新建缺失的视觉和嗅觉通道。

## 二、新建基础设施（woworld_core）

| 组件 | 说明 |
|------|------|
| `SpatialQuery` trait | 空间实体查询、射线、光照、材质、事件buffer、气味源——纯几何，世界空间管理实现 |
| `SpatialEntity` struct | 实体空间表示——id+位置+速度+粗分类+隐蔽度+发光量+透明度 |
| `SpatialEvent` struct | 空间区块事件——各模块写入buffer，感官消费。Chunk粒度16m，ring buffer 64 |
| `CulturalKnowledgeBase` trait | 文化基线知识查询——TOML数据驱动 |
| `VisionQuery` trait | 视觉主动扫描——woworld_core定义，感官crate实现 |
| `ScentQuery` trait | 嗅觉懒采样——woworld_core定义，感官crate实现 |

## 四、核心架构决策

1. **感官 = 无状态纯函数管线** + 小型PerceptualCache
2. **感知节律 = 决策节律**（零新时钟）。事件通道独立——缩短下次决策间隔，不打断动作
3. **感官不辨识、不诊断、不整合跨模态、不定义事件边界**——全部留给消费者
4. **同一份PerceptBatch**供日常决策、战斗情报、审美判断、大日志消费
5. **感知退化是距离的连续函数**——LOD层级只是调度分桶
6. **注意 = 异常检测**（背景/信号二分类），不是显著性排分
7. **玩家和NPC走同一套被感知路径**

## 六、接口契约

| 概念 | 权威 Owner | 关键约定 |
|------|-----------|---------|
| `PerceptualCache` | 感官系统→NPC存储 | 有界(64实体)，LRU淘汰，不持久化 |
| `VisionParams` | 感官系统→Life初始化→NPC存储 | 对标HearingModel。种族+perception+年龄+人格 |
| `VisionQuery` trait | woworld_core定义→感官crate实现 | 纯函数——不依赖NpcData |
| `ScentQuery` trait | woworld_core→感官crate | 懒采样。气味源→各模块管理 |
| `SpatialQuery` trait | woworld_core→世界空间管理 | 纯几何——不含业务语义 |
| `Knowledge` | NPC crate | 仅存超出文化基线的个人知识 |
| `AestheticFrameworks` | 审美系统→NPC存储 | native+personal+adopted，四过程模型，无硬上限 |
| `PerceptBatch` | 感官系统 | 统一知觉产出——不区分战斗/日常 |
| `PerceptualModifiers` | 感官系统 | 状态效果→感知调制。NPC crate做桥接 |
| GOAP事实知识源 | NPC crate prepare_facts() | FactSet每周期临时构建。GOAP永不直接读世界真值 |
| 大日志感官输入 | 感官系统 perceive_lightweight() | 玩家角色轻量感官管线 |
| 战斗情报层级 | 战斗crate combat_intelligence.assess() | 消费PerceptBatch→派生CombatantAssessment |
