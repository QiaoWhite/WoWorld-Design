# 008 — 决策器集成与行为涌现

## 一、NPC 感知循环中的审美集成

### 1.1 流程

```
每 5 秒感知 tick:
  NPC 扫描可见实体
    └── 对每个受注意的实体:
        ├── 1. 如果该实体实现了 HasAestheticSignal:
        │      signal = entity.aesthetic_signal()
        │      ctx = build_aesthetic_context(npc, entity)
        │      judgment = judge(&signal, &npc.taste, &ctx)
        │
        ├── 2. React(judgment) → 情绪 delta + 记忆 + 行为倾向
        │      · 仅内部状态变更
        │      · 可能在下一帧行为树中产生可见行为（靠近/远离）
        │
        └── 3. 如果需要外化（expression_level > 阈值）:
               Articulate(judgment, mode) → 语言/社交/经济信号
```

### 1.2 build_aesthetic_context() —— 调用方组装的逻辑

```rust
fn build_aesthetic_context(npc: &NpcState, target: &AestheticTarget) -> AestheticContext {
    let pressure = npc.vitals.hunger.max(npc.vitals.thirst)
        .max(npc.vitals.fatigue)
        .max(npc.combat_state.threat_level());
    
    AestheticContext {
        self_preservation_pressure: pressure,
        cognitive_load: npc.current_cognitive_load(),
        available_time_ms: npc.idle_time_budget_ms(),
        mood_valence: npc.emotion.current_valence(),
        familiarity: npc.memory.count_similar_aesthetic_encounters(target) as f32 / 100.0,
        prior_expectation: derive_prior_expectation(npc, target),
        stakes: assess_stakes(npc, target),
        social: assess_social_context(npc),
        jitter_seed: hash_combine(npc.id.0, target.hash(), current_hour()),
    }
}
```

## 二、概率决策器中的审美权重

### 2.1 need_action_match 的审美扩展

现有 need_action_match 将需求 urgency → 行为权重。审美判断不产生"审美 urgency"（审美不是需求），但**调制已有需求的权重**：

```rust
// 在 need_action_match 内部（NPC 决策器）
fn aesthetic_modulation(action: &ActionType, recent_judgments: &[AestheticMemory]) -> f32 {
    // 如果 NPC 最近有高 valence 的审美体验 → 情绪 uplift → 
    // 社交意愿上升 → 社交类行为的 urgency 乘数微升
    let avg_valence = recent_judgments.iter()
        .filter(|m| m.timestamp.elapsed_sec() < 300.0)  // 过去 5 分钟内
        .map(|m| m.judgment.valence)
        .sum::<f32>() / recent_judgments.len().max(1) as f32;
    
    1.0 + avg_valence * 0.15  // 最多 ±15%
}
```

## 三、涌现验证：完整的艺术创作链

```
─────────────────────────────────────────────────────────────
NPC: 精灵画家 Elen，高 openness + 高 artistry 文化

1. 动机:
   IntrinsicGoal::CreateSomething(Artwork)
   relevance = 0.7×0.7 + 0.9×0.3 + 0.8×0.2 = 0.92
   commitment = 0.7×0.6 + (1-0)×0.4 = 0.82

2. 准备:
   行为树选择 CreateSomething → 子计划 GatherMaterials
   → 经济系统: 购买画布/颜料

3. 创作:
   Embellish(
     creator: Elen,
     target: NewItem(Painting),
     intention: { target: [0.6, 0.3, 0.7, 0.9, 0.8, 0.7] },
     skill: painting=0.78,
     materials: [canvas, pigment_set],
     duration: 3游戏日
   )
   → output_signal: { fluency:0.72, novelty:0.35, complexity:0.68, 
                       harmony:0.85, expressiveness:0.78, virtuosity:0.70 }
   → 物品系统: create_item → ItemEntId(0x...)

4. 自我评价:
   Elen.judge(自己刚完成的作品, 自己的taste, ctx)
   → { valence:0.55, interest:0.4, respect:0.60, arousal:0.3 }
   → React: 记忆写入 + 情绪轻微满足
   → respect=0.60 × honor_weight_for_domain(ARTISTIC)=0.9
   → esteem_deficit -= 0.54（已有管道——CHG-028）

5. 展示:
   Articulate(mode=Behavioral, DisplayIntent)
   → 挂在自家墙上

6. 他人观看（NPC B 路过）:
   B.Attend → signal → judge(signal, B.taste, ctx)
   → { valence:0.82, interest:0.70, respect:0.75, arousal:0.4 }
   → React: 情绪 uplift + "精灵画家真有天赋" 记忆
   → Articulate: "这幅画真美……" (mode=Exclamation)

7. 二手传播（B 告诉 C）:
   B.Articulate(mode=Social) → "你应该去看看Elen的新画"
   → C 产生兴趣 → C 去看 → C.judge() → C 也喜欢
   → C.Articulate(mode=Behavioral) → CommissionIntent("能给我也画一幅吗？")

8. 经济涌现:
   多人 CommissionIntent → Elen 的作品供不应求
   → 经济系统: Elen 的画的价格在订单簿中上升
   → Elen 可以靠绘画维生
─────────────────────────────────────────────────────────────
```

**全部由三原子 + judge() + 已有系统驱动。零硬编码"艺术市场"或"画家职业"。**

## 四、不修改 GOAP 安全网

审美系统**不修改 GOAP 安全网**。心理需求（esteem/competence/social）不进 GOAP——只有生存需求（hunger/thirst/fatigue/health/combat）+ 配偶压力进。

审美判断产生的是**行为倾向**（approach/avoid/possess），不是 GOAP 目标。行为树在权重链中消费行为倾向，而非 GOAP 的强制目标满足。
