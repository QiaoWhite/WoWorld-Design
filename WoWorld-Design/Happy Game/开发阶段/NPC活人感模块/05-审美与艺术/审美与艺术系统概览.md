# 05 — 审美与艺术系统 v1.0

> **核心设计哲学**：审美系统是 WoWorld 的**无状态引擎**——它不存储持久数据，只定义一套所有模块共享的审美"通用语"：`AestheticSignal`（任何可感知物的 6 维客观美学特征向量）、`AestheticTaste`（每个感知者的主观审美滤镜）和 `AestheticJudgment`（4 维审美判断输出）。通过 4 个不可再分的事件原子（React / Articulate / Adopt / Embellish），艺术创作、艺术商业、艺术表演、时尚传播、审美共识等复杂现象**从系统交互中涌现**，而非硬编码。

> **与需求系统的本质差异**：需求是匮乏驱动的稳态维持（deficit → urgency → action → recovery），审美是吸引驱动的信息评估（perceive → judge → experience → remember）。审美没有"审美 deficit"，不存在"审美饥饿"——体验本身就是终点。

> **设计原则**：
> 1. **无状态引擎**：审美模块零持久存储——对标 `WeatherQuery::sample()` 模式。
> 2. **零反向依赖**：审美模块不 import 任何业务模块——只定义 trait/struct/函数。
> 3. **涌现优先**：所有复杂艺术现象从原子交互涌现，零硬编码。
> 4. **确定性噪声**：jitter seed = hash(npc_id, target_hash, hour_of_day)，存档可复现。
> 5. **三层门控**：注意力层（NPC 感知系统决定是否调用 judge()）→ judge() 层（全精度，无压制）→ 效应层（React/Articulate 内部调制响应强度）。
> 6. **对标已有模式**：ConsumableEffect 的 schema 模式、ReligiousPracticeProfile 的存储模式、PowerAtom 的原子枚举模式、CultureQuery 的无分配 trait 模式。

## §一 核心架构：三层管道

```
AestheticSignal (6维客观信号) → AestheticTaste (主观滤镜) → AestheticJudgment (4维输出)
         ↑                              ↑                            ↓
   各模块通过 trait 提供           NPC 存储，审美模块定义       驱动 React/Articulate/Adopt/Embellish
```

```
React:     判断 → 内部状态（情绪 delta / 记忆写入 / 行为倾向）
Articulate: 内部状态 → 外部世界（语言表达 / 社交信号 / 经济意图）
Adopt:     外部世界 → 内部状态（品味被他人/文化/作品影响）
Embellish: 审美意图 → 持久产出（创造/装饰/表演）
```

四个原子三个方向，完备正交：内省不自主动作（React）· 人际自愿外化（Articulate）· 人际→内省半自愿（Adopt）· 产出（Embellish）。

## §二 AestheticSignal — 6 维客观美学信号

### 2.1 定义

```rust
/// 任何可被审美感知的事物的客观美学特征向量
/// Copy 类型，24 bytes（6 × f32），栈上传递，零分配
/// 
/// ★ 审美模块只定义 struct——各实体所属模块在自己的 impl HasAestheticSignal 中计算
#[derive(Copy, Clone, Default)]
pub struct AestheticSignal {
    /// [0, 1] 加工流畅度 —— 对称性/清晰度/原型符合度/可预测性
    /// 神经对应：感知流畅性 → 眶额皮层正向激活
    pub fluency: f32,

    /// [0, 1] 新颖度 —— 偏离原型/预期/熟悉模式的程度
    /// 神经对应：多巴胺能预测误差 → VTA 阶段性放电
    /// ★ 对 valence 是倒 U 型曲线，非单调
    pub novelty: f32,

    /// [0, 1] 复杂度 —— 信息密度/细节层次/结构层次
    /// 神经对应：前扣带回（ACC）+ dlPFC 认知负荷
    /// ★ complexity ≠ 混乱。高 complexity + 高 harmony = 精妙。高 complexity + 低 harmony = 混乱。
    pub complexity: f32,

    /// [0, 1] 和谐度 —— 比例/色彩/节奏/空间的结构协调性
    /// 神经对应：听觉皮层（和弦协和）+ 视觉皮层（色彩和谐）+ 眶额皮层（整体"对"感）
    /// ★ 唯一在所有感官上以相似方式运作的维度（共同的数学基础：比例/频率比/强度比）
    pub harmony: f32,

    /// [0, 1] 情感表现力 —— 对象承载和传达情感信号的程度
    /// 神经对应：镜像神经元系统 + 岛叶（interoception）+ 杏仁核（情感检测）
    /// ★ 与创作意图关联最强。功能性物品（普通杯子）→ 极低。挽歌 → 极高。
    pub expressiveness: f32,

    /// [0, 1] 技艺可见度 —— 制作/执行难度和技巧的外显程度
    /// 神经对应：内侧前额叶（社会认知）+ 前运动皮层（动作模拟——"我能做到吗？"）
    /// ★ 过程 virtuosity = f(skill_level)。产物 virtuosity = f(Quality, Rarity)。两者可分离。
    pub virtuosity: f32,
}
```

### 2.2 派生属性（不在 Signal 中存储，由 judge() 内部计算）

| 派生属性 | 公式 | 含义 |
|---------|------|------|
| `somatic_impact` | `novelty × expressiveness × (1 + stakes.mortality_salience × 0.5)` | "被美击中"的身体感觉——又新又动情 → 岛叶激活 |
| `symbolic_depth` | `complexity × prior_expectation × (1 - familiarity × 0.5)` | 象征深度——复杂 + 有文化框架 + 不熟悉 = 可被解读 |
| `cultural_fit` | `Σ(signal[dim] × taste.dimension_weights[dim]) / 6` | 对象与个人品味的契合度——用于冒犯检测 |

### 2.3 为何是 6 维——推导依据

来自神经科学的四个可分离审美通路的反推：

| 通路 | 需要的输入维度 |
|------|-------------|
| 多巴胺奖赏通路（伏隔核） | `fluency`（省力=愉悦）、`harmony`（协调=愉悦） |
| 交感神经激活通路 | `novelty`（新=激活）、`expressiveness`（有情感=激活） |
| 认知投入通路（ACC/dlPFC） | `complexity`（信息密度=认知挑战）、`novelty`（新信息=认知动机） |
| 社会评价通路（mPFC/TPJ） | `virtuosity`（技艺=社会评价） |

**被排除的候选维度**：`beauty`（=高 valence 的判断输出，非信号）、`somatic_impact`（=紧急产物）、`symbolic_depth`（=紧急产物）、`originality`（= novelty × virtuosity）、`elegance`（= fluency × harmony + 低 complexity）、`sublimity`（= judgment 组合而非信号）。

### 2.4 跨感官适用性

6 维在所有感官上基本适用。某些维度在特定感官上自然权重偏低（触觉的 expressiveness、嗅觉的 virtuosity）——由 AestheticTaste 的维度权重通过文化和个性自然调节，**不需要为不同感官定义独立的 Signal 结构**。

## §三 AestheticTaste — 主观审美滤镜

### 3.1 定义

```rust
/// 审美品味——每个 NPC 持有的主观审美滤镜
/// Copy 类型，32 bytes（6×f32 + 3×f32 + u32）
/// ★ 审美模块定义 struct 和派生公式，NPC 模块存储
/// 对标 ReligiousPracticeProfile 模式
#[derive(Copy, Clone)]
pub struct AestheticTaste {
    /// 6 个信号维度的个人权重 [0, 1]
    /// 索引: [fluency, novelty, complexity, harmony, expressiveness, virtuosity]
    pub dimension_weights: [f32; 6],

    /// 熟悉度偏好 [-1, 1]
    /// -1 = 追求陌生和新奇（"故地无趣"），0 = 中性，1 = 偏爱熟悉和安全（"经典永不过时"）
    pub familiarity_bias: f32,

    /// 审美开放度 [0, 1]
    /// 0 = 封闭（仅接受本文化审美标准），1 = 极度开放（接受任何审美风格）
    /// 影响：novelty 倒 U 峰值位置、Adopt 阻力、跨文化审美宽容度
    pub aesthetic_openness: f32,

    /// 复杂度容忍度 [0, 1]
    /// 0 = 偏好极度简单，1 = 享受极度复杂
    /// 影响：complexity 倒 U 峰值位置
    pub complexity_tolerance: f32,

    /// 生成时的个人种子（不可变，出生时 roll）
    /// 用于确定性个人偏移和 adolescent jitter 补充
    pub personal_seed: u32,
}

// 维度索引常量
pub const DIM_FLUENCY: usize = 0;
pub const DIM_NOVELTY: usize = 1;
pub const DIM_COMPLEXITY: usize = 2;
pub const DIM_HARMONY: usize = 3;
pub const DIM_EXPRESSIVENESS: usize = 4;
pub const DIM_VIRTUOSITY: usize = 5;
```

### 3.2 初始派生：derive_taste()

NPC 进入青春期时（~12-14 岁）调用一次，对标 `CulturalBeautyStandard` 的吸收过程。

**派生输入**：Big Five 个性 + CultureCoreParams + CulturalBeautyStandard + 父母的 taste（可选）+ personal_seed。

**维度权重派生**（6 个权重从 0.5 起点叠加文化和个性偏移，然后 personal_seed 加随机偏移）：

| 源参数 | → Taste 维度 | 权重 | 方向 |
|--------|------------|------|------|
| culture.artistry | expressiveness | +0.30 | 高艺术文化→重视情感表现 |
| culture.uncertainty_avoidance | fluency | +0.20 | 高不确定性规避→偏好清晰可预测 |
| culture.individualism | novelty | +0.15 | 高个人主义→重视独特性 |
| culture.power_distance | virtuosity | +0.25 | 高权力距离→重视技艺等级 |
| culture.indulgence | harmony | +0.20 | 高放纵→追求感官和谐 |
| personality.openness | novelty / expressiveness / complexity | +0.25 / +0.15 / +0.10 | 开放→创新+情感+复杂 |
| personality.conscientiousness | virtuosity / fluency | +0.20 / +0.10 | 尽责→重视技艺+整洁 |
| personality.agreeableness | harmony | +0.20 | 宜人→追求协调 |
| personality.extraversion | expressiveness | +0.15 | 外向→重视情感外显 |
| personality.neuroticism | fluency | -0.15 | 高神经质→对不流畅更敏感 |

各维度从 0.5 起点累加上表偏移，然后每个维度加 `rng.f32_range(-0.12, 0.12)`（[TUNING] `taste_personal_jitter`），最终 clamp 到 `[0.05, 0.95]`。

**三个派生参数的具体公式**：

```rust
familiarity_bias = (culture_params.uncertainty_avoidance * 0.6
                   - personality.openness * 0.4
                   + rng.f32_range(-0.10, 0.10)).clamp(-1.0, 1.0);

aesthetic_openness = (culture_params.openness_to_outsiders * 0.40
                     + personality.openness * 0.40
                     + (1.0 - beauty_standard.aesthetic_confidence) * 0.20
                     + rng.f32_range(-0.08, 0.08)).clamp(0.0, 1.0);

complexity_tolerance = (personality.openness * 0.60
                       + culture_params.artistry * 0.40
                       + rng.f32_range(-0.10, 0.10)).clamp(0.0, 1.0);
```

**反叛因子**：高 openness + 低 agreeableness 的青少年，taste 主动偏离父母 taste：

```rust
let rebellion = personality.openness * 0.5 + (1.0 - personality.agreeableness) * 0.3;
if rebellion > 0.5 {
    if let Some(parents) = parents_taste {
        for dim in 0..6 {
            let parent_deviation = 1.0 - parents.dimension_weights[dim];
            weights[dim] += parent_deviation * rebellion * 0.2;
            weights[dim] = weights[dim].clamp(0.05, 0.95);
        }
    }
}
```

### 3.3 年更新：mature_taste()

每游戏年调用一次（对标文化 drift σ=0.003/年）：

1. **年龄效应**：`complexity_tolerance` 呈倒 U 型（10 岁 0.7 → 18 岁 1.0 → 45 岁平台 → 95 岁 0.6）
2. **技能成长效应**：随着自身 Artisan/FineArts 技能提升，virtuosity 鉴赏力渐进增强（上限 +0.25）
3. **旅行效应**：去过越多文化区域 → novelty 偏好上升（上限 +0.20）
4. **自然漂移**：每维度 ±0.005/年（对标文化 drift 量级）

### 3.4 创伤冲击：traumatic_taste_shift()

React 原子中，如果 `|valence| > 0.9` 且 `somatic_impact > 0.8`，触发一次性 taste 偏移：极度负面审美体验 → harmony 权重永久上升（"美是脆弱的"）→ familiarity_bias 微降。

### 3.5 与 CulturalBeautyStandard 的关系

| | CulturalBeautyStandard | AestheticTaste |
|---|---|---|
| 所有权 | 文化系统 004 | NPC 模块存储，审美模块定义 |
| 粒度 | 文化级（一个文化的"什么是美"） | 个人级（一个 NPC 的审美偏好） |
| 范围 | 仅人体/外貌 | 所有可审美感知对象 |
| 可变性 | 文化漂移（极慢）或统治者更替 | 年更新 + Adopt 事件（中速） |

CulturalBeautyStandard 影响 AestheticTaste 的初始派生（通过 `aesthetic_confidence` 和 `artistry` 等参数），但不直接限制个人 taste。个人可以通过 Adopt 偏离文化标准——这正是审美创新的微观基础。

## §四 AestheticJudgment — 4 维判断输出

### 4.1 定义

```rust
/// 审美判断——主体对客体的多维审美响应
/// Copy 类型，16 bytes（4 × f32）
/// judge() 纯函数的唯一输出
#[derive(Copy, Clone, Default)]
pub struct AestheticJudgment {
    /// 愉悦度 [-1, 1]
    /// 负=厌恶/排斥，0=中性/无感，正=愉悦/喜欢
    /// 神经对应：多巴胺奖赏通路（伏隔核）
    pub valence: f32,

    /// 激活度 [0, 1]
    /// 0=平静/无波动，1=极度兴奋/激动/紧张
    /// 神经对应：交感神经系统
    pub arousal: f32,

    /// 兴趣度 [0, 1]
    /// 0=无聊/不值得关注，1=极度着迷/移不开视线
    /// 神经对应：前扣带回（ACC）+ 认知投入
    pub interest: f32,

    /// 敬佩度 [0, 1]
    /// 0=轻视/不值一提，1=极度敬佩/肃然起敬
    /// 神经对应：内侧前额叶（mPFC）+ 社会评价通路
    pub respect: f32,
}
```

### 4.2 判断组合的类型涌现

| 体验类型 | valence | arousal | interest | respect | 典型场景 |
|---------|---------|---------|---------|---------|---------|
| **美** (beauty) | >0.6 | <0.4 | >0.3 | 任意 | 看一幅和谐的风景画 |
| **震撼** (stunning) | >0.7 | >0.6 | >0.6 | >0.7 | 第一次看哥特教堂内部 |
| **崇高** (sublime) | 0.2~0.6 | >0.7 | >0.5 | >0.8 | 面对暴风雨/巨大山脉 |
| **有趣** (interesting) | >0.2 | 任意 | >0.7 | 任意 | 看到新颖的艺术风格 |
| **可爱** (cute) | >0.6 | 0.1~0.4 | >0.3 | <0.3 | 看一只小猫 |
| **无聊** (boring) | -0.1~0.1 | <0.1 | <0.15 | <0.2 | 千篇一律的流水线杯子 |
| **廉价** (cheap) | -0.2~0 | <0.2 | <0.2 | <0.15 | 粗制滥造的仿制品 |
| **冒犯** (offensive) | <-0.5 | >0.4 | >0.3 | 任意 | 与自身价值观冲突的强烈表达 |
| **做作** (pretentious) | -0.3~0 | >0.3 | <0.3 | <0.4 | 高情感低技艺的拙劣模仿 |
| **令人敬佩但不愉悦** | <-0.2 | 任意 | >0.4 | >0.7 | "我不喜欢但不得不承认它厉害" |

注意：这些标签**不在代码中**。它们是从 4 维值域的自然分段涌现的——消费方可根据需要定义自己的分类。

## §五 judge() 纯函数

### 5.1 签名

```rust
/// 审美判断——零副作用纯函数
///
/// 任何模块可调用。零 I/O，零分配，零锁。
/// 三步流程：感知抖动 → 维度映射 → 情境调制
pub fn judge(
    signal: &AestheticSignal,
    taste: &AestheticTaste,
    ctx: &AestheticContext,
) -> AestheticJudgment
```

### 5.2 第一步：感知抖动

```rust
fn apply_perceptual_jitter(signal: &AestheticSignal, seed: u64) -> AestheticSignal
```

每维度加 `rng.f32_range(-0.02, 0.02)`（[TUNING] `jitter_range`）。seed 由调用方以 `hash(npc_id, target_hash, hour_of_day)` 生成——确定性，存档可复现。

同一 NPC 同一小时看同一对象 → 相同 jitter。不同时间看 → 微妙不同（"今早光线不同""刚才想到的偶然联想"）。

±2% 意味着大师之作永不跌落，边缘作品偶尔摇摆——正是真实审美体验的数学反映。

### 5.3 第二步：维度映射

#### valence 计算

```rust
fn compute_valence(signal: &AestheticSignal, taste: &AestheticTaste, ctx: &AestheticContext) -> f32 {
    // 基础愉悦：fluency(0.45) + harmony(0.35) + expressiveness(0.20)
    let base = signal.fluency * w(FLUENCY) * 0.45
             + signal.harmony * w(HARMONY) * 0.35
             + signal.expressiveness * w(EXPRESSIVE) * 0.20;
    
    // novelty 倒 U 型：峰值位置由 aesthetic_openness 决定
    let novelty_peak = 0.3 + taste.aesthetic_openness * 0.5;
    let novelty_valence = exp(-(signal.novelty - novelty_peak)² / (2 × 0.15²));
    
    // complexity 倒 U 型：峰值由 complexity_tolerance 决定
    let complexity_peak = 0.3 + taste.complexity_tolerance * 0.5;
    let complexity_valence = exp(-(signal.complexity - complexity_peak)² / (2 × 0.25²));
    
    // virtuosity 线性贡献
    let virtuosity_term = signal.virtuosity * w(VIRTUOSITY) * 0.5;
    
    // 冒犯检测：高 expressiveness + 低 harmony + 低 cultural_fit
    let cultural_fit = compute_cultural_fit(signal, taste);
    let offensive = if signal.expressiveness > 0.6 && signal.harmony < 0.3 && cultural_fit < 0.3 {
        (1.0 - cultural_fit) * 0.6
    } else { 0.0 };
    
    // 合成 → 映射到 [-1, 1]
    let raw = base + novelty_valence * w(NOVELTY) * 0.25 
            + complexity_valence * w(COMPLEXITY) * 0.15 
            + virtuosity_term * 0.15 - offensive;
    (raw * 2.0 - 1.0).clamp(-1.0, 1.0)
}
```

**关键设计点**：
- fluency + harmony 占 valence 的 80%——这是"美"的核心基础
- novelty 是倒 U 型——太新=困惑，适度新=愉悦，完全不新=无聊（但无聊主要影响 interest 而非 valence）
- virtuosity 是线性的——技艺越好越愉悦，但不保证（低 harmony 的大师作品可能 virtuosity 高但 valence 低）
- 冒犯检测是唯一的非线性负向惩罚——承载了"这不应该存在"的文化/价值观判断

#### arousal 计算

```rust
fn compute_arousal(signal: &AestheticSignal, taste: &AestheticTaste, ctx: &AestheticContext) -> f32 {
    let raw = signal.novelty * w(NOVELTY) * 0.40
            + signal.expressiveness * w(EXPRESSIVE) * 0.35
            + signal.complexity * w(COMPLEXITY) * 0.25;
    let activated = sigmoid(raw, steepness=8.0, midpoint=0.5);
    let stakes_amp = 1.0 + ctx.stakes.mortality_salience * 0.4;
    (activated * stakes_amp).min(1.0)
}
```

#### interest 计算

```rust
fn compute_interest(signal: &AestheticSignal, taste: &AestheticTaste, ctx: &AestheticContext) -> f32 {
    // 交互项：又新又复杂 → 兴趣最高
    let interaction = signal.novelty * w(NOVELTY) * signal.complexity * w(COMPLEXITY);
    // harmony_gate：至少需要一定和谐度（否则只是混乱，非有趣）
    let harmony_gate = signal.harmony.max(0.15);
    // fluency_inverse：太流畅=无聊
    let fluency_inverse = 1.0 - signal.fluency;
    let raw = interaction * 0.50 + fluency_inverse * 0.20 + signal.novelty * w(NOVELTY) * 0.30;
    let gated = raw * harmony_gate;
    // prior_expectation 放大 interest（"大师名作"→更想细看）但不放大 valence
    let expectation_amp = 1.0 + ctx.prior_expectation * 0.4;
    (gated * expectation_amp).min(1.0)
}
```

#### respect 计算

```rust
fn compute_respect(signal: &AestheticSignal, taste: &AestheticTaste, ctx: &AestheticContext) -> f32 {
    let raw = signal.virtuosity * w(VIRTUOSITY) * 0.50
            + signal.complexity * w(COMPLEXITY) * 0.25
            + signal.expressiveness * w(EXPRESSIVE) * 0.25;
    let expectation_amp = 1.0 + ctx.prior_expectation * 0.2;
    let stakes_amp = 1.0 + ctx.stakes.mortality_salience * 0.3;
    (raw * expectation_amp * stakes_amp).min(1.0)
}
```

### 5.4 第三步：情境调制

```rust
fn apply_context_modulation(judgment: &mut AestheticJudgment, ctx: &AestheticContext)
```

| 调制项 | 影响 | 公式 |
|--------|------|------|
| 情绪偏置 | valence ± 当前情绪效价的 10-15% | `valence += mood_valence × (mood>0 ? 0.15 : 0.10)` |
| 熟悉度衰减 | valence / interest 衰减 | `valence *= (1 - familiarity × 0.6).max(0.2)` |
| 集体同步放大 | 集体观看时 arousal/valence 放大 | `valence *= 1 + synchrony × 0.3`; `arousal *= 1 + synchrony × 0.4` |
| 认知负载压缩 | 高负载下 interest 分辨率下降 | `interest *= (1 - (load-0.6)×1.5).max(0.2)` |
| 时间预算 | 没时间细看 → interest 压制 | `interest *= (available_time_ms/500).max(0.1)` |

### 5.5 关键设计决策：judge() 不做生存压制

生存危机中的 NPC 看敌对剑客——仍然能完整感知美。压制不在 judge() 内，而在：
- **注意力层**（NPC 感知系统）：饥饿者不注意到墙上的画
- **效应层**（React/Articulate 内部）：决斗中的 NPC 不开口赞美，但内心感受到了

## §七 事件原子

### 7.1 React — 内省 · 不自主 · 静默

**触发**：每次 judge() 后自动执行。不可跳过。

**输入**：`ReactInput { agent, target, judgment, context }`

**产出**：`Vec<AestheticEffect>`

| 效果 | 条件 | 公式 |
|------|------|------|
| 情绪 delta | 总是 | `valence × (1-pressure×0.5) × 0.15` |
| 记忆写入 | `(interest + somatic_impact)/2 × (1-pressure×0.4) ≥ 0.1` | 否则不产生记忆 |
| somatic_impact 标记 | `somatic_impact > 0.35`（somatic_impact 按 §2.2 公式计算，已含 stakes 调制） | 标记为高情感权重记忆 |
| 靠近行为倾向 | `pressure<0.5, valence>0.5 或 interest>0.6` | → 行为树 |
| 远离行为倾向 | `pressure<0.5, valence<-0.4` | → 行为树 |
| 占有意图 | `pressure<0.4, valence>0.6, interest>0.4` | → 行为树→经济系统 |

**输出枚举**：

```rust
pub enum AestheticEffect {
    EmotionDelta(f32),                    // → 情绪引擎
    MemoryWrite(AestheticMemory),         // → 记忆系统
    SomaticImpact { intensity: f32 },     // → 情绪引擎 + 记忆高情感标记
    ApproachBehavior { target, reason },  // → 行为树
    AvoidBehavior { target, reason },     // → 行为树
    PossessionInterest { target, strength }, // → 行为树→经济系统
}
```

### 7.2 Articulate — 人际 · 自愿 · 外化

**触发**：`expression_level > 0.2` 时由行为决策器调用

**expression_level 公式**：

```
judgment_intensity = max(|valence|×0.8, interest×0.6, respect×0.7, arousal×0.4)
× personality: extraversion×0.5 + openness×0.3 + (1-neuroticism)×0.2
× context: (is_alone?-0.6:0) + (being_asked?+2.0:0) - (is_formal?0.4:0)
× culture: comm_norms.emotional_expressiveness
× (1.0 - pressure × 0.95).max(0.02)
```

**五个表达通道**：

| 通道 | expression_level | 语言系统参与 | 成本 |
|------|-----------------|------------|------|
| **Silent** | <0.2 | 无 | 0 |
| **Exclamation** | 0.2~0.4 | 模板碎片 <10 模板 | 极低 |
| **Social** | 0.4~0.7 | 片段组合 + 滤镜 | 低 |
| **Critique** | >0.7 + openness>0.6 | 维度→自然语言映射 | 中 |
| **Behavioral** | >0.5 | 无（直接输出行为意图） | 低 |

**产出**：`Vec<ArticulateEffect>`——ExpressionRef（→语言表达系统）、EconomicIntent（→经济系统）、GiftIntent、CommissionIntent、DisplayIntent、DestroyIntent、ApproachIntent、AvoidIntent、SocialSignal（→周围 NPC）。

### 7.3 Adopt — 人际→内省 · 半自愿 · 品味偏移

**触发**：社交接触 + 对审美源的认可


| 触发源 | intensity | credibility |
|--------|-----------|-------------|
| 反复接触某人穿着（≥5次/30天） | 0.2→0.5 | 0.4 |
| 被某件作品强烈打动（respect>0.8 或 somatic_impact） | 0.6 | 0.9 |
| 文化教化（青春期每年一次） | 0.15 | 0.9 |
| 被同伴口头推荐 | 0.3 | 与关系成正比 |
| 大师/导师的技能教学关系 | 0.4 | 0.85 |
| 节日集体活动参与 | 0.2 | 0.5 |

**偏移公式**：

```rust
pub fn adopt_effect(agent_taste, source_taste, credibility, intensity) -> AestheticTaste {
    let resistance = (1.0 - agent_taste.aesthetic_openness) * 0.5;
    let force = credibility * intensity * (1.0 - resistance);
    for dim in 0..6 {
        let delta = (source_taste[dim] - agent_taste[dim]) * force * 0.25;
        agent_taste[dim] = (agent_taste[dim] + delta).clamp(0.05, 0.95);
    }
    // familiarity_bias / complexity_tolerance 也受影响（force×0.15）
}
```

**涌现效应**：时尚传播、审美共识、艺术流派、文化审美漂移——全部从大量 Adopt 事件的统计中涌现。审美模块不编码"时尚"或"流派"。

每维：`actual[dim] = intent.target[dim] × skill_factor × dim_requirement + material_bonus + noise`
其中 `dim_requirement` 对 virtuosity=0.95、complexity=0.8、fluency=0.7、harmony=0.6、expressiveness=0.3、novelty=0.2——反映各维度对技能的依赖程度。

- 学徒画杰作 → skill_factor=0.20 → 严重偏离意图 → "儿童画"
- 学徒的真诚努力在 expressiveness 得到补偿（技能依赖仅 0.3）→ 父母仍珍视孩子的画
- 大师故意"画烂" → intention.target 设置低 harmony → 实际结果低 harmony

### 9.3 审美模块零反向依赖

审美模块不 import 任何业务模块。所有交互通过：
- 审美模块定义 trait → 消费方在自己的模块中实现
- 审美模块定义 struct → 调用方在感知循环中组装后传入
- 审美模块发出事件枚举 → 调用方消费事件

## §十 决策器集成

### 10.1 NPC 感知循环

```
每 5s 感知 tick:
  NPC 扫描可见实体
    └── 对每个受注意的实体:
        ├── 如果实体现 HasAestheticSignal:
        │      signal = entity.aesthetic_signal()
        │      ctx = build_aesthetic_context(npc, entity)
        │      judgment = judge(&signal, &npc.taste, &ctx)
        │
        ├── React(judgment) → 情绪 delta + 记忆 + 行为倾向
        │
        └── 如 expression_level > 阈值:
               Articulate(judgment, mode) → 社交/经济信号
```

### 10.2 概率决策器

审美不产生"审美 urgency"（审美不是需求），但**调制已有需求的权重**：最近 5 分钟内的平均审美 valence → 情绪 uplift → 社交类行为权重微调（最多 ±15%）。

### 10.3 IntrinsicGoal 的审美驱动

`CreateSomething(Artwork)` 的 relevance 新增 AestheticTaste 因子：

```rust
relevance = openness × 0.7 + culture.artistry × 0.3 + taste.aesthetic_openness × 0.2
```

### 10.4 GOAP 安全网

**审美系统不修改 GOAP 安全网**。审美判断产生的是行为倾向（approach/avoid/possess），不是 GOAP 目标。

## §十一 技能系统扩展：FineArts 独立第 6 大类

### 11.1 为什么独立于 Artisan

工艺（Artisan, 0x03）和艺术（FineArts）有本质差异：

| | 工艺 | 艺术 |
|---|------|------|
| 首要目的 | 功能性产出 | 审美体验本身 |
| 技能进步机制 | 重复生产 → 肌肉记忆 | 不完全由重复决定——有灵感/突破/风格转变 |
| 评价标准 | Quality × 功能性 | 审美判断——virtuosity 只是 6 维之一 |
| 交叉训练基础 | 材料/工具/精细控制 | 感知敏锐度 + 表现力 + 创造力 |

### 11.2 分类定义

```rust
pub enum SkillCategory {
    Combat = 0x01,
    Magic = 0x02,
    Artisan = 0x03,
    Academic = 0x04,
    Survival = 0x05,
    FineArts = 0x06,   // ★ 新增
}
```

| 前缀 | 子组 | 技能 | 子组内交叉 |
|------|------|------|-----------|
| 0x0601 | Visual（视觉艺术） | painting, sculpture, calligraphy | 0.15 |
| 0x0602 | Musical（音乐） | music_composition, music_performance | 0.20 |
| 0x0603 | Literary（语言艺术） | poetry, creative_writing | 0.15 |
| 0x0604 | Performing（表演艺术） | dance, acting | 0.12 |

### 11.3 跨类交叉训练

仅保留有生理/认知基础的，最高 0.08（低于 Artisan 内部跨子组的 0.12-0.15）：

| 艺术技能 | 已有技能 | 基础 | 强度 |
|---------|---------|------|------|
| painting | Artisan.precision.jewelry | 精细手部控制 | 0.05 |
| sculpture | Artisan.woodstone.stonework | 石材知识 | 0.08 |
| sculpture | Artisan.woodstone.carpentry | 3D 空间思维 | 0.05 |
| calligraphy | Academic.linguistics | 文字知识 | 0.05 |
| poetry | Academic.linguistics | 语言掌握 | 0.08 |
| music_performance | Survival.listening_perception | 听觉敏锐度 | 0.05 |
| dance | Combat.body_control | 身体控制 | 0.06 |

### 11.4 TechnologyProfile 天花板

`artistry × 0.50 + writing × 0.50`（文化系统 005 已有公式——将目标从 Artisan 的 textiles/writing 改为 FineArts 类别）。

## §十二 性能预算与存储分析

### 12.1 存储

| 项目 | 大小 | 说明 |
|------|------|------|
| 审美模块自身存储 | **0** | 无状态引擎——对标 WeatherQuery |
| `AestheticTaste` / NPC | 32 bytes | Copy 类型 |
| 10,000 NPC | 320 KB | SoA → L2 友好 |
| 100,000 NPC | 3.2 MB | |
| `AestheticMemory`（记忆条目） | ~48 bytes/条 | 与已有的 2000 条 NPC 记忆共享配额 |

### 12.2 CPU 热路径

| 操作 | 频率 | 每操作 | 总量（10,000 NPC） |
|------|------|--------|-------------------|
| `aesthetic_signal()` | 仅在 Attend 时 lazy | ~100ns | — |
| `judge()` | 每 NPC/5s/1-3 目标 | ~150 FP | **<0.001ms/帧** |
| `React` | 同 judge() | ~50 FP | **<0.001ms/帧** |
| **热路径总计** | | | **<0.002ms/帧** |

远低于天气系统的 0.25ms/帧预算。

### 12.3 最差情况：节日

1000 NPC 同时观看 1 个表演，每 2s 一次 judge()：

```
1000 × 150 FP ÷ 2000ms = 75 FP/ms ≈ 0.075ms/帧
```

仍然远低于天气预算。

### 12.4 VRAM

**零增量**。审美模块不引入纹理/mesh/shader。

### 12.5 可并行性

- `judge()` 纯函数 → rayon 友好
- 每个 NPC 的审美判断彼此独立 → SoA → SIMD 友好
- 各 `HasAestheticSignal` 实现者独立 → 无锁

## §十四 扩展预留

### 14.1 抽象审美对象（第 4 类）

数学/逻辑结构之美（对标欧拉公式）、社会事件的审美质量（政治辩论/葬礼）——预留接口，不优先实现。仅极少数高 intelligence + 高 openness NPC 能产生这类抽象审美，且无可见外部行为。

### 14.2 审美历史记录

长期来看，一个文化的审美变迁可被历史系统记录为 `CulturalShift` 事件。当足够多的 NPC 通过 Adopt 在某个方向收敛时 → `CulturalBeautyStandard` 可更新。

### 14.3 LLM 增强

对标语言表达 011 的 LLM 装饰器模式——深度批评（Critique 通道）可被 LLM 增强，失败/关闭时无缝回退模板。

### 14.4 玩家审美交互

玩家作为人类做自己的审美判断，但可通过以下方式感知 NPC 的审美：
1. NPC 对话中的 Articulate 文本（主通道）
2. NPC 的可见行为反应（靠近看/摇头走开/触摸）
3. NPC 对玩家穿着和行为的反应
4. Debug overlay（开发工具，非正式功能）
