## 一、事件目录

所有生命周期事件由生命模块发布，通过 EventBus（woworld_core 薄分发层）路由到订阅方。

| 事件 | 发布者 | 发布时机 | 负载 |
|------|--------|---------|------|
| `LifeStageTransition` | AgeClock | 实体越过阶段边界 | entity_id, from, to, at_age |
| `Birth` | GestationState | 胎儿足月分娩 | child_id, mother_id, father_id, blueprint, location |
| `Death` | AgeClock / Combat / Life | 任何死因生效 | entity_id, cause, location, spirit_remaining, was_violent, body_intact |
| `GestationEnded` | GestationState | 孕期结束（活产/流产/死产/母死） | mother_id, outcome |

## 二、各模块的完整订阅与响应

### 2.1 NPC 活人感模块

| 事件 | 响应 |
|------|------|
| `Birth` | BigFive 生成、Identity{age:0} 初始化、外观种子解压娩出、prenatal_influences→初始记忆种子 |
| `Death` | 关系中标记 is_dead、记忆注入死亡事件、自我叙事闭合 |
| `LifeStageTransition` | 更新 CognitiveStyle 成熟度参数、AestheticTaste 结晶化条件检查 |
| `GestationEnded(Miscarriage/Stillbirth)` | 母亲认知-情绪管线处理——取决于人格/信念/处境 |

### 2.2 生命模块（自身消费）

| 事件 | 响应 |
|------|------|
| `Death` | DeathSummary 创建、三阶段压缩调度 |
| `GestationEnded(LiveBirth)` | 母亲移除 Pregnant 标签、添加 Nursing 标签 |
| `GestationEnded(Miscarriage/Stillbirth)` | 母亲移除 Pregnant 标签、触发 recovery 过程 |

### 2.3 历史模块

| 事件 | 响应 |
|------|------|
| `Birth` | `HistoricalEvent::NpcBirth` 记录到事件日志 |
| `Death` | LifeTrace 创建（墓碑+传记）、`HistoricalEvent::NpcDeath` 记录 |
| `LifeStageTransition(Adolescent→YoungAdult)` | 若为重大历史人物→可选记录"成年" |

### 2.4 权力模块

| 事件 | 响应 |
|------|------|
| `Birth` | 若父母持有可继承头衔→继承权登记 |
| `Death` | 若逝者持有头衔→SuccessionRule 触发 |
| `LifeStageTransition(Adolescent→YoungAdult)` | 可能触发"成年后继承"延迟生效的头衔 |

### 2.5 信仰模块

| 事件 | 响应 |
|------|------|
| `Birth` | ChildFaithProfile 继承（已设计 CHG-025） |
| `Death` | FuneralCustoms × LifeStage × DeathCause → 葬礼形式判定；亡灵生成条件判定 |
| `LifeStageTransition(Adolescent→YoungAdult)` | 若信仰有成年礼→调度 RitualDef 执行 |
| `GestationEnded` | 若信仰有出生仪式/流产后净化仪式→调度 RitualDef |

### 2.6 经济模块

| 事件 | 响应 |
|------|------|
| `Death` | 遗产按 InheritanceLaw 分配；劳动力池更新；若逝者持有债务→债务转移 |

### 2.7 技能模块

| 事件 | 响应 |
|------|------|
| `Death` | 若逝者为唯一 X 技能导师→该技能的 Apprenticeship 路径标记不可用 |

### 2.8 模型/动画/物理

| 事件 | 响应 |
|------|------|
| `LifeStageTransition` | 骨骼缩放因子更新 |
| `Death` | 骨架松弛（刚度 1.0→0.02 指数衰减） |

### 2.9 空间查询

| 事件 | 响应 |
|------|------|
| `Birth` | `SpatialEntity::register(entity_kind=Creature)` |
| `Death` | velocity→0, entity 保留在 EntityIndex（不注销）, 30 天后 Scent(Rot) |

### 2.10 战斗

| 事件 | 响应 |
|------|------|
| `Death` | 若在战斗中→终止战斗状态 |

## 四、EventBus 设计

```rust
// —— woworld_core ——
// 极薄的类型驱动分发层。不做逻辑，只做路由。

pub struct EventBus {
    // 内部：HashMap<TypeId, Vec<Box<dyn Fn(&dyn Any) + Send + Sync>>>
    // 或编译期单态：
    subscribers: FxHashMap<TypeId, Vec<EventCallback>>,
}

impl EventBus {
    /// 发布事件到所有订阅者
    pub fn publish<E: 'static + Send + Sync>(&self, event: E) {
        if let Some(subscribers) = self.subscribers.get(&TypeId::of::<E>()) {
            let event_ref = &event as &dyn Any;
            for callback in subscribers {
                (callback.func)(event_ref);
            }
        }
    }

    /// 订阅某一类型的事件
    pub fn subscribe<E: 'static + Send + Sync, F: Fn(&E) + Send + Sync + 'static>(
        &mut self, callback: F,
    ) {
        let type_id = TypeId::of::<E>();
        self.subscribers.entry(type_id).or_default().push(EventCallback {
            func: Box::new(move |any: &dyn Any| {
                if let Some(event) = any.downcast_ref::<E>() {
                    callback(event);
                }
            }),
        });
    }
}

// 使用示例：
// event_bus.publish(LifeEvent::Death { entity_id, cause, .. });
// event_bus.subscribe::<LifeEvent>(|event| match event { .. });
```

运行时成本 ≈ 查找订阅者（通常 ≤10 个）→ 遍历调用。编译期单态——非虚函数调用。
