## §一、什么是领域复合原子

领域复合原子 = 物理基元的有向序列 + 领域专属校验。各领域模块的 `execute()` 内部编排第一层物理原子，加上领域规则（技能检定、品质计算、合法性判断）。GOAP 规划器不直接操作物理原子——它操作的是第三层 ActionCandidate，其 execute() 调用领域复合原子。

## §二、完整目录与编排序列

### 2.1 建造与生产

| 原子 | 物理原子序列 | 领域校验 |
|------|-----------|---------|
| **CRAFT** | HEAT→{GRASP+STRIKE×N→OBSERVE}→DIP→COOL→WIPE | SkillRequirement.min_level, MaterialProperties兼容 |
| **HARVEST** | MOVE→GRASP(工具)→CUT/SCOOP→STACK→LIFT | PlantSpecies成熟度, 工具bonus |
| **DIG** | MOVE→GRASP(工具)→STRIKE→BREAK→SCOOP→LIFT | TerrainQuery.surface_material, 工具等级 |
| **FELL** | MOVE→GRASP(斧)→STRIKE×N→OBSERVE(倒下方向)→PUSH | 树种硬度, 树径 |
| **CONSTRUCT** | MOVE→LIFT→ATTACH→STRIKE→SPREAD→STACK | Blueprint约束, 建材承重 |
| **PLANT** | SCOOP(挖坑)→INSERT(种子)→SPREAD(水)→WET | 土壤肥力, 季节窗口 |
| **COOK** | CUT→POUR→HEAT→SCOOP | 食材新鲜度, 配方 |
| **MINE** | MOVE→GRASP(镐)→STRIKE→BREAK→SCOOP→LIFT | 矿石硬度, 矿脉储量 |

### 2.2 战斗

| 原子 | 物理原子序列 | 领域校验 |
|------|-----------|---------|
| **PARRY** | MOVE(微调)→STRIKE(偏转, timing=0.1-0.2s窗口) | skill_proficiency, shield_coverage |
| **GRAPPL** | GRASP(活体)→PULL→PUSH→THROW(全身) | 力量对抗(str+skill vs str+agi) |

| **AIM** | POSTURE→OBSERVE(瞄准)→WAIT | 专注力, 风速 |
| **DODGE** | MOVE(侧移)→JUMP→POSTURE(闪避) | agility×skill, fatigue |

### 2.3 魔法

## §三、模块注册机制

```rust
// 定义在 woworld_core
struct ActionCandidate {
    id: u64,              // 模块内部标识
    domain: u8,           // 领域标签(经济/战斗/社交/魔法/信仰/...)
    precondition_fn: fn(&AgentSnapshot, &WorldCtx) -> Precheck,
    utility_fn: fn(&AgentSnapshot, &WorldCtx) -> f32,
    execute_fn: fn(&mut World, &AgentSnapshot) -> ActionResult,
    spatial_target: Option<Vec3>,
    duration_category: DurationClass, // Instant/Short/Long/Ritual/Indefinite
}

// 各模块在初始化时注册
fn register_module_actions(registry: &mut ActionRegistry) {
    // 经济模块:
    registry.register(ActionCandidate { id: 0x01_0001, domain: ECONOMY, ... }); // CRAFT
    registry.register(ActionCandidate { id: 0x01_0002, domain: ECONOMY, ... }); // TRADE
    
    // 战斗模块:
    registry.register(ActionCandidate { id: 0x02_0001, domain: COMBAT, ... }); // Fight
    
    // ... 所有模块只注册，NPC 模块不拥有也不枚举
}
```

**关键**: NPC 模块不拥有 ActionCandidate 列表。它只提供 GOAP 规划器的搜索管道，通过 `registry.iter_for_agent(&agent_snapshot)` 收集可见候选。新模块=新注册=GOAP自动可见。

## §四、年龄涌现

所有领域复合原子消费 AgentSnapshot——不设门控。CRAFT 是典型的"幼儿可尝试但几乎不可能成功"的复合原子: `agent.strength=0.3` → GRASP工具→force_check失败→CRAFT 的 execute() 在第一步就返回失败。没有"幼儿禁止锻造"的代码。
