## §一、问题: "第一个铁匠"从哪来

世界生成时，NPC 已经有了技艺。开发者/玩家视角: 世界刚生成，这些NPC是第一批存在的人。NPC 世界内视角: "我师父教的，师父的师父是传奇铁匠张三，追溯到500年前发现锻造的李四……"

**核心张力**: 两套时间尺度的协调。

## §二、KnowledgeSeed TOML

KnowledgeSeed 是**技术史的时间线元数据**——不是往 NPC 脑子里灌动作序列。定义每个技术时代×区域: 什么技艺在何时被发现、传播速度、基础技能上限、发现前提。

```toml
[[era]]
name = "StoneAge"
start_year = -3000; end_year = -1500

[[era.tech]]
domain = "FireUse"
discovery = "Inevitable"           # 必然发现——任何人类聚落
base_skill_ceiling = 60
propagation = "Fast"

[[era.tech]]
domain = "FlintKnapping"
discovery = "Inevitable"
base_skill_ceiling = 50

# ---

[[era]]
name = "BronzeAge"
start_year = -1500; end_year = -500

[[era.tech]]
domain = "CopperSmelting"
discovery = "Conditional"          # 条件发现
condition = { biome = "Mountain", resource = "CopperOre" }
base_skill_ceiling = 60
propagation = "Slow"

[[era.tech]]
domain = "IronSmelting"
discovery = "Breakthrough"         # 突破性——不是每个文明都会
condition = { prerequisite = "CopperSmelting", tech_level = 0.6 }
base_skill_ceiling = 40
propagation = "VerySlow"
```

**时代框架和技艺树是手工设计的**（类似 SkillDef TOML，定义了98个技能的分类）。**具体传播路径是历史引擎自动生成的**（根据贸易路线/文化接触/战争计算哪些聚落在什么时间学到了什么）。

## §三、历史引擎传播算法

世界生成 P9 阶段的历史模拟使用以下传播模型:

### 3.1 传播概率公式

```
P(聚落B在年份Y学到技艺T) = 
    base_transmission_rate(T.propagation_speed)  // 基础年传播率
    × trade_route_strength(A, B)                 // 贸易连接强度 0-1
    × cultural_similarity(A, B)                  // 文化相似度 0-1  
    × distance_decay(distance(A, B))             // 距离衰减
    × settlement_B.tech_receptivity              // B的技术接纳度 0-1
```

其中:
- `base_transmission_rate`: Fast=0.30/yr, Slow=0.05/yr, VerySlow=0.01/yr [TUNING]
- `trade_route_strength`: 来自经济/贸易数据，两聚落间贸易量归一化
- `cultural_similarity`: 来自文化系统，CultureCoreParams 的余弦相似度
- `distance_decay`: `exp(-distance / max_propagation_range)` — max_range 因传播媒介而异（陆路 500km, 海路 2000km）
- `tech_receptivity`: 来自聚落的 CultureCoreParams.uncertainty_avoidance(逆) × openness

### 3.2 三类传播事件

1. **RootEvent**: 当 T.discovery_time ≤ Y 且条件满足(资源/前置技艺/生物群系) → 创建发现事件。首次成功的聚落由加权随机选择(人口×tech_receptivity最高的N个候选)
2. **PropagationEvent**: 每年对每个"已掌握T的聚落A"和每个"未掌握T的邻近聚落B"进行概率检查。概率=P(传播)。成功后→生成传播事件+设置B.tech_level[T]=A.tech_level[T]×0.7(衰减传播)
3. **InstitutionEvent**: 当聚落的某项技艺水平≥60 且持续≥100年 → 概率生成行会/学院/寺庙等制度事件 → 传播速率×1.5(制度化加速)

## §四、与技能系统的对接

当代 NPC 的技能由 `generate_starting_skills()` 赋予——这个函数接受 KnowledgeSeed 的传播结果作为 `settlement_tech_level` 参数，根据 NPC 的年龄、职业、天赋计算合理技能水平。**不需要模拟 NPC 逐帧学习2000年。**

NPC 的"我师父教的"信念: 从历史事件链中提取——王五知道"我师父是赵六, 赵六的师父是孙七..." 因为历史引擎已经生成了这条传承链的事件记录。

## §五、对技能系统的影响

- 新增 `KnowledgeSeed` TOML 注册表 (数据, 非代码)
- `generate_starting_skills()` 新增 `tech_level` 输入参数
- World Gen P8/P9 调用链: KnowledgeSeed → 历史引擎传播 → 当代技能分布
- **不修改**: TeachingSession、三层天赋、XP公式、cross-training
