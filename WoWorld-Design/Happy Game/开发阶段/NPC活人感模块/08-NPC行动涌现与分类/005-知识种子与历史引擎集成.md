# 005-知识种子与历史引擎集成

> **日期**: 2026-06-18

## §一、问题: "第一个铁匠"从哪来

世界生成时，NPC 已经有了技艺。开发者/玩家视角: 世界刚生成，这些NPC是第一批存在的人。NPC 世界内视角: "我师父教的，师父的师父是传奇铁匠张三，追溯到500年前发现锻造的李四……"

**核心张力**: 两套时间尺度的协调。

## §二、KnowledgeSeed TOML

KnowledgeSeed 是**技术史的时间线元数据**——不是往 NPC 脑子里灌动作序列。定义每个技术时代×区域: 什么技艺在何时被发现、传播速度、基础技能上限、发现前提。

```toml
[[era]]
name = "StoneAge"
start_year = -3000; end_year = -1500

[[era.tech]]
domain = "FireUse"
discovery = "Inevitable"           # 必然发现——任何人类聚落
base_skill_ceiling = 60
propagation = "Fast"

[[era.tech]]
domain = "FlintKnapping"
discovery = "Inevitable"
base_skill_ceiling = 50

# ---

[[era]]
name = "BronzeAge"
start_year = -1500; end_year = -500

[[era.tech]]
domain = "CopperSmelting"
discovery = "Conditional"          # 条件发现
condition = { biome = "Mountain", resource = "CopperOre" }
base_skill_ceiling = 60
propagation = "Slow"

[[era.tech]]
domain = "IronSmelting"
discovery = "Breakthrough"         # 突破性——不是每个文明都会
condition = { prerequisite = "CopperSmelting", tech_level = 0.6 }
base_skill_ceiling = 40
propagation = "VerySlow"
```

**时代框架和技艺树是手工设计的**（类似 SkillDef TOML，定义了98个技能的分类）。**具体传播路径是历史引擎自动生成的**（根据贸易路线/文化接触/战争计算哪些聚落在什么时间学到了什么）。

## §三、历史引擎传播

世界生成 P9 阶段的历史模拟:

1. KnowledgeSeed 提供初始条件 → "铁器冶炼在 year -1500 至 -500 间在各区可被发现"
2. 历史引擎的事件因果链系统（历史系统 002）自动生成传播事件:
   - "Settlement_A 于 year -847 首次成功炼铁" (RootEvent)
   - "Settlement_A → Settlement_B 通过贸易路线传播, year -720" (PropagationEvent)
   - "Settlement_B 建立铁匠行会, year -650" (InstitutionEvent)
3. 传播的成功率由 KnowledgeSeed.propagation 速度 × 贸易路线强度 × 文化接触度调制

## §四、与技能系统的对接

当代 NPC 的技能由 `generate_starting_skills()` 赋予——这个函数接受 KnowledgeSeed 的传播结果作为 `settlement_tech_level` 参数，根据 NPC 的年龄、职业、天赋计算合理技能水平。**不需要模拟 NPC 逐帧学习2000年。**

NPC 的"我师父教的"信念: 从历史事件链中提取——王五知道"我师父是赵六, 赵六的师父是孙七..." 因为历史引擎已经生成了这条传承链的事件记录。

## §五、对技能系统的影响

- 新增 `KnowledgeSeed` TOML 注册表 (数据, 非代码)
- `generate_starting_skills()` 新增 `tech_level` 输入参数
- World Gen P8/P9 调用链: KnowledgeSeed → 历史引擎传播 → 当代技能分布
- **不修改**: TeachingSession、三层天赋、XP公式、cross-training
