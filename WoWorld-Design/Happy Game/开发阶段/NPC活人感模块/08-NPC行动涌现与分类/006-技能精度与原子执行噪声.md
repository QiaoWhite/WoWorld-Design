# 006-技能精度与原子执行噪声

> **关联**: [[001-NPC行动涌现总纲|001-总纲]] · [[../../技能系统/001-技能系统总纲|技能系统]] · [[011-关键问题回应与设计论证|011-关键问题回应]]
> **日期**: 2026-06-18

## §一、问题

把技艺拆成原子序列后，Lv 10 和 Lv 90 铁匠执行同样的 HEAT→STRIKE×N→DIP→WIPE 序列。差别在哪？还能用数值表示技艺高低吗？

**答**: 技能 = 执行精度，不是序列门控。Lv 1 NPC 可以做所有原子——只是做得差。

## §二、核心函数

```rust
/// 技能等级 → 原子执行噪声标准差
/// skill_level: 0.0-100.0 (来自技能系统的SkillEntry.level)
/// 返回: 0-1 归一化噪声标准差
fn execution_noise_std(skill_level: f32) -> f32 {
    let proficiency = skill_level / 100.0;
    BASE_NOISE * (1.0 - proficiency).powi(2)  // 二次衰减
}
```

**二次衰减的意义**: 新手→熟手的精度提升非常快（前半段陡峭），大师→宗师的精度提升越来越慢（后半段平缓）。符合真实学习曲线。

## §三、各原子的 BASE_NOISE

| 原子参数 | BASE_NOISE | Lv 1 噪声 | Lv 50 噪声 | Lv 90 噪声 | Lv 100 |
|---------|-----------|----------|-----------|----------|--------|
| STRIKE 力度 | 0.40 | ±40% | ±10% | ±0.4% | 0 |
| STRIKE 位置 | 0.15 | ±15cm | ±3.8cm | ±0.15cm | 0 |
| HEAT 温度 | 0.30 | ±30% | ±7.5% | ±0.3% | 0 |
| DIP 角度 | 0.25 | ±25° | ±6.3° | ±0.25° | 0 |
| OBSERVE 阈值 | 0.50 | ±50% | ±12.5% | ±0.5% | 0 |
| POSTURE 稳定 | 0.10 | ±10% | ±2.5% | ±0.1% | 0 |
| GRASP 握力 | 0.20 | ±20% | ±5% | ±0.2% | 0 |
| THROW 精度 | 0.35 | ±35% | ±8.8% | ±0.35% | 0 |

## §四、OBSERVE 精度 — 高技能者"眼更毒"

OBSERVE 不是独立技能——它同样受噪声调制。高技能铁匠能准确判断火候（"亮红色=1050°C±10°C"），新手看不清（"有点红? 800°C? 1200°C? 不知道"）。

```
Lv 10 铁匠看铁坯:
  实际温度: 1050°C (完美淬火温度)
  感知: 980°C ± 150°C → "还不够热" → 继续加热 → 过热 → 晶粒粗大 → 剑脆

Lv 90 铁匠看铁坯:
  实际温度: 1050°C  
  感知: 1045°C ± 10°C → "就是现在" → 淬火 → 完美马氏体 → 极品好剑
```

## §五、与现有技能系统的关系

- 不新增 `SkillEntry` 字段。现有 `proficiency()` = level/100.0 直接传入 `execution_noise_std()`
- 现有三层天赋 (MentalAccess × InnateAptitude × CrossTraining) 不变——它们影响 **XP 获取速率**，噪声函数只消费最终的 **level 数值**
- 现有 `quality_influence` 不变——它决定"技能对产品品质的贡献权重"，噪声决定"原子执行的物理精度"。两者独立
- 现有 `SkillRequirement.min_level` 不变——配方门槛仍然存在（"锻造铁剑需要 Blacksmith ≥ 20"）。只是 Lv 15 也可以试——噪音大会失败

## §六、涌现差异

Lv 10 铁匠和 Lv 90 铁匠执行同一序列 `HEAT→STRIKE×N→DIP→WIPE`:

| 阶段 | Lv 10 表现 | Lv 90 表现 |
|------|-----------|-----------|
| HEAT | 温度噪声 ±120°C → 过热/不够热 | 温度噪声 ±3°C → 精准 |
| STRIKE 力 | 力度变异 ±32% → 有的轻有的重 | 力度变异 ±2% → 均匀一致 |
| STRIKE 位 | 位置偏差 ±12mm → 打偏 | 位置偏差 ±0.6mm → 每次精准 |
| OBSERVE | 判断准确度 60% → "差不多了?" | 判断准确度 99.5% → "还需3锤" |
| DIP | 角度噪声 ±20° → 淬火不均→弯曲 | 角度噪声 ±1° → 均匀硬化 |
| **结果** | Rough品质 (65%概率) 或 失败 (10%) | Refined品质 (70%) 或 Perfect (15%) |

**没有一行代码说 `if level < 25 { cannot_quench() }`**。物理自己算。
