## §四、LODCoordinator 7维联动

现有 LOD 协调器 (技术栈 v4.0 §二十一) 新增 physics_lod 维度:

```
physics_lod: u8 (mapped to existing field in LodPrescription)
  0: 全碰撞扫掠 + 全IK + 全PBD迭代
  1: 简化碰撞(15→5胶囊) + 全IK
  2: 粗碰撞(AABB) + 简化IK + PBD降为1次迭代
  3: AABB only + 无IK
  4: 无碰撞 + 无动画

跨维约束 (新增):
  skeleton_lod ≥ 2 → physics_lod ≥ 2
  ai_lod ≥ 3 → physics_lod ≥ 3
  physics_lod = 4 → animation_lod = 4
  scene_lod N → physics_lod ≥ N-2
```

### 7.1 AgentSnapshot 刷新与 LOD

| LOD | AgentSnapshot 刷新频率 | 使用场景 |
|-----|----------------------|---------|
| L0 | 每帧 | 全字段完整刷新，用于 IK + 碰撞扫掠 |
| L1 | 每 0.3s (18帧) | 全字段完整刷新(与决策周期对齐) |
| L2 | 每 1-5s | 简化: physical字段保持, cognitive/social使用上一次缓存值 |
| L3+ | 不刷新 | 无 AgentSnapshot — 使用统计聚合 |

AgentSnapshot 结构对所有LOD保持不变——只是刷新频率和字段精度不同。低LOD NPC的物理原子使用过期(缓存)的 AgentSnapshot 值，差异对远距离NPC不可见。

### 7.2 MaterialProperties 热缓存失效

GripHandle 中缓存的 MaterialDef指针在以下情况需刷新:
- 被握持实体的温度变化超过 ±100K → `hardness_at_temp()` 需要重算 → 不重查表，只重算硬度
- 被握持实体被其他原子的副作用改变(如被HEAT加热) → 帧末批量刷新标志 + 下次STRIKE前重读
- 被握持实体被DESTROY → GripHandle 整体失效(见002 §十一)

## §八、结论

碰撞箱方案可行。三项零成本: IK (骨骼预算内零增量)、MaterialProperties (L2缓存常驻零增量)、碰撞扫掠 (本可忽略)。唯一新增: 非Rigid武器 PBD ≤0.3ms。总增量 0.45ms，在 7.0ms Rust 预算内留有 0.55ms 余量。
