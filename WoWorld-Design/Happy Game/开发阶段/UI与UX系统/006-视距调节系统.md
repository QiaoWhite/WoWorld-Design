## 一、设计目标

Minecraft 风格的视距滑块——玩家拖一个数字，世界可见范围随之变化。不暴露内部 LOD 结构，不需要理解"clipmap ring"或"Signed Heightfield"。

| 属性 | 值 |
|------|-----|
| 调节范围 | 1-40 km |
| 步长 | 1 km |
| 默认值 | 20 km |
| 交互方式 | HSlider 拖动 + "应用"按钮确认 |
| 生效方式 | 确认后即时生效（非实时预览） |
| 持久化 | 保存到玩家配置文件 |

## 二、ViewDistance 类型

```rust
// woworld_core/src/view_distance.rs — 新文件

/// 视距配置——单一真相源，各子系统独立消费。
///
/// 消费方:
/// - WorldDriver: 决定渲染到第几层 clipmap ring
/// - Camera3D: far plane 距离
/// - Atmosphere: fog 参数微调
#[derive(Debug, Clone, Copy)]
pub struct ViewDistance {
    km: f64,  // clamp [1.0, 40.0]
}

impl ViewDistance {
    pub fn new(km: f64) -> Self {
        Self { km: km.clamp(1.0, 40.0) }
    }

    pub fn km(self) -> f64 { self.km }
    pub fn meters(self) -> f64 { self.km * 1000.0 }

    /// 需要渲染的最远 LOD 层级索引。
    /// 利用 LodLevel::min_range —— min_range < view_distance 的最外层。
    pub fn max_lod_level(self, levels: &[LodLevel]) -> u8 {
        levels.iter()
            .filter(|l| l.min_range < self.meters())
            .map(|l| l.index)
            .max()
            .unwrap_or(0)
    }
}
```

定义于 `woworld_core`（仅 `glam` 依赖）——与所有其他 ID 类型和 trait 在同一 crate。

## 三、运行时传播路径

```
Godot 设置界面
  HSlider (1-40, step 1) + "应用"按钮
       │
       ▼  (用户点击"应用")
WorldDriver.set_view_distance_km(km)    ← GDExtension #[export] property
       │
       ├── Camera3D.far = km * 1000.0   ← WorldDriver 持有 camera_node 引用
       │                                     (从 player_node → get_node("Camera3D") 获取)
       ├── 各 LOD 层可见性更新
       │     LEVELS[i].min_range > view_distance_meters → set_visible(false)
       │     隐藏层跳过 heightmap 更新
       ├── OceanPlane 无变化
       │     far tier 固定 50km —— 始终覆盖最大视距
       │     near tier 固定 5km —— 始终渲染近场
       └── Shader fog 更新
             fog_density = 1.2 / (km * 1000.0)
             fog_color = ResolvedAtmosphere.sky_horizon
```

## 四、性能特征

视距滑块虽然是连续的（1km 步长），实际渲染成本在 LOD 边界处跳变：

```
 4km → 10km:  6 clipmap rings (L1-L6)，成本不变
10km → 11km:  7 rings (+L7 kicks in)      ← GPU 跳变
11km → 20km:  7 rings，成本不变
20km → 21km:  8 rings (+L8 kicks in)      ← GPU 跳变
21km → 30km:  8 rings，成本不变
30km → 31km:  9 rings (+L9 kicks in)      ← GPU 跳变
31km → 40km:  9 rings，成本不变
```

这与 Minecraft 的区块滑块行为一致——连续 UI 背后是阶梯式成本。玩家体验连续，内部实现按需加载。

**L8/L9 mesh**在启动时创建（额外 165K 顶点，可忽略）。**Heightmap**懒生成——首次切换到对应距离时才触发异步 job。

## 五、Godot UI 规格

```
┌──────────────────────────────────────────┐
│  图形设置                                 │
│                                          │
│  视距                    [ 重置 ] 20 km  │
│  ├────┼────┼────┼──o──┼────┼────┼────┤  │
│  1    5   10   15   20   25   30   35  40 │
│                                          │
│  ⚠ 30km 以上可能影响性能                  │
│                                          │
│              [ 应用 ]  [ 取消 ]           │
└──────────────────────────────────────────┘
```

- `HSlider` 范围 1-40，`step = 1.0`，`value = 20.0`（默认）
- 标签实时显示当前值："视距: 23 km"
- "重置"按钮恢复默认 20km
- 拖动滑块时**不**实时生效——点"应用"才提交
- 30km+ 时性能提示文字变为橙色
- "应用"调用 `WorldDriver.set_view_distance_km(value)`
- 值持久化到 `player_settings.toml`

## 六、与现有系统的协调

| 系统 | 影响 | 说明 |
|------|------|------|
| LODCoordinator | 无 | `distance_to_scene_lod()` 的 L8/L9 距离带已包含——自动处理 |
| ECS sensory | 预留 | NPC 视距受限于 `ViewDistance`——sensory system 读取此值 |
| 存档系统 | 无 | 地形是程序化的，视距不影响存档 |
| 世界生成 | 无 | 视距是运行时设置，不影响生成管线 |
| 海洋 | 无 | Far tier (50km) 和 near tier (5km) 固定，不受视距影响 |

## 七、Godot ↔ Rust 通信

```rust
// WorldDriver 中

#[export]
fn get_view_distance_km(&self) -> f64 {
    self.view_distance.km()
}

#[export]
fn set_view_distance_km(&mut self, km: f64) {
    self.view_distance = ViewDistance::new(km);
    // 立即传播：
    // 1. 更新 camera.far
    if let Some(ref cam) = self.camera_node {
        cam.set_far(self.view_distance.meters() as f32);
    }
    // 2. LOD 层可见性在下一帧 process() 中自动更新
    // 3. fog_density 在下一帧 process() 中自动更新
}

// ready() 中获取 camera 引用:
self.camera_node = self.player_node
    .as_ref()
    .and_then(|p| p.get_node_or_null("Camera3D"))
    .and_then(|n| n.try_cast::<Camera3D>().ok());
```
