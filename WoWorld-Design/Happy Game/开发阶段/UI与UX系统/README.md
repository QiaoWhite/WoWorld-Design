## 模块定位

UI/UX 系统是 WoWorld **唯一的玩家 I/O 层**。Rust 模拟核心不包含任何渲染或输入处理——全部 UI 在 Godot 侧以 Control 节点实现。UI 系统不做游戏逻辑决策；它只负责将模拟核心的数据转化为玩家可视/可操作的信息，并将玩家输入转化为模拟核心可消费的指令。

### 核心设计约束

1. **Rust 不碰 UI**: 所有 UI 在 Godot Control 节点层。Rust 只提供数据——不包含字体/颜色/布局。
2. **沉浸式优先**: 默认隐藏一切可隐藏的 UI。信息按需浮现——不是在屏幕上永久驻扎。
3. **手柄=键鼠**: 两个输入设备平等支持，不设"主输入设备"。
4. **信息边界**: UI 不泄露"NPC 不知道的信息"——与感官系统一致。
5. **玩家可自定义**: 允许玩家选择哪些元素默认显示、透明度、位置。

## 关键参数表

| 参数 | 值 | 说明 |
|------|-----|------|
| 帧预算 | < 2ms/帧 | Godot 侧 UI 开销（不含渲染） |
| HUD 刷新率 | 每帧 | 状态条、准心 |
| 小地图刷新 | 1 Hz | 位置/朝向更新 |
| 面板刷新 | 按需 | 打开面板时一次性查询 |
| 最大对话气泡数 | 5 同时 | 超过时最旧的淡出 |
| 对话气泡持续时间 | 2-5 秒 | 按文本长度 |
| 默认准心 | 十字 | 2px 白色 + 1px 黑边 |
| 对话选项数 | ≤4 项/轮 | 通过数字键 1-4 选择 |
| UI 内存 | ~50 MB VRAM | 纹理/图标/字体 |

## 架构速览

```
Rust 模拟核心 (woworld_*)           Godot UI 层
┌─────────────────────┐         ┌──────────────────────┐
│ NpcData             │────────→│ HUD 状态条            │
│ VitalSigns          │  数据流 │ 对话气泡             │
│ ItemRegistry        │────────→│ 物品栏面板            │
│ SkillRegistry       │────────→│ 技能面板              │
│ WorldMapData        │────────→│ 地图面板              │
│ BigLog              │────────→│ 日志面板              │
│ DialogOption[]      │←───────→│ 对话界面(双向)        │
│ PlayerInput         │←────────│ 输入映射              │
└─────────────────────┘         └──────────────────────┘
```

## 性能预算

| 资源 | 预算 |
|------|------|
| CPU (UI 逻辑) | < 2ms/帧 |
| GPU (UI 渲染) | < 1ms/帧（Control 节点合批后） |
| VRAM (UI 纹理) | ~50 MB（图标/字体/面板背景） |
| 输入延迟 | < 16ms（一帧内响应） |
