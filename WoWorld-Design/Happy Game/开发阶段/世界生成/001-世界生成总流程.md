## 一、总览

世界生成是从一个 `uint64` 种子出发，经过 **11 个阶段**逐级细化的确定性管线。同一种子永远生成同一个世界。

v3 重构的核心变化：
- **插入 P0/P1**：全球大陆架构 + 海洋系统优先于陆地生成
- **废弃"玩家中心密度控制"**：改为全局均匀生成 + Clipmap LOD 分层加载
- **海陆 7:3**：海洋从"空地"升级为"内容"
- **全部参数重算**：以 25 万 km² 为基准（约 500×500km）

## 二、各阶段详细设计

### P0：全球大陆架构

**输入**：`uint64` 世界种子
**输出**：海陆掩码（2D）、造山带图、裂谷图、热点岛链图

```rust
/// 全球大陆架构的噪声参数
struct GlobalContinentParams {
    /// 大陆板块噪声频率（波长 1000-10000km）
    continent_frequency: f64,       // 默认: 0.0001
    /// 海陆阈值——低于此值为海洋
    sea_threshold: f64,             // 默认: 0.70（→ 海:陆 ≈ 7:3）
    /// 造山带噪声频率（板块交界处→山脉）
    orogen_frequency: f64,          // 默认: 0.0005
    /// 裂谷噪声频率
    rift_frequency: f64,            // 默认: 0.0003
    /// 热点噪声频率（孤立火山岛链）
    hotspot_frequency: f64,         // 默认: 0.0008
}

/// 海陆掩码生成
fn generate_land_sea_mask(seed: u64, params: &GlobalContinentParams) -> LandSeaMask {
    let continent_noise = layered_noise(
        hash(seed, "continent"),
        params.continent_frequency,
        NoiseType::Perlin,  // 大陆尺度用 Perlin
    );

    // 二值化：大于阈值为陆地
    let land_mask: Grid2D<bool> = continent_noise
        .map(|v| v > params.sea_threshold);

    // 造山带：板块交界处的陆地边缘
    let orogen = detect_plate_boundaries(&continent_noise, params);

    // 热点岛链：在海洋区域随机分布
    let hotspots = generate_hotspot_chains(seed, &land_mask, params);

    LandSeaMask {
        land_mask,      // true = 陆地
        sea_mask: !land_mask,
        orogen_zones: orogen,
        rift_zones: compute_rifts(seed, &land_mask, params),
        hotspots,
        // 海陆比可调：sea_threshold 浮动 ±0.05 → 海陆比在 65:35 ~ 75:25
    }
}
```

**关键参数**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `continent_frequency` | 0.0001 | 波长 1000-10000km，决定大陆数量和大小 |
| `sea_threshold` | 0.70 | 海:陆 ≈ 7:3。下调→更多陆地；上调→更多海洋 |
| `orogen_frequency` | 0.0005 | 造山带宽度和分布 |
| `min_continent_size` | 500 km² | 小于此值的陆地碎片合并到最近大陆 |

**自动校验**：
- 至少存在 1 个面积 ≥ 10,000 km² 的大陆（保证有"旧大陆"）
- 最大海洋距离（任意陆地到最近陆地的距离）≥ 500km（保证跨洋探索有意义）

### P1：海洋系统

**输入**：海陆掩码、全球温度基线
**输出**：海洋深度分级、洋流图、海岸线分类、可航行海域

```rust
/// 海洋系统输出
struct OceanSystem {
    /// 深度分级（栅格 500m×500m）
    depth_map: Grid2D<OceanDepth>,
    /// 洋流（简化的全球温盐环流模型）
    currents: Vec<OceanCurrent>,
    /// 海岸线逐段分类
    coast_segments: Vec<CoastSegment>,
    /// 可航行海域标记
    navigable_zones: NavigabilityMap,
}

enum OceanDepth {
    Shallow { depth: f32 },      // <50m——珊瑚礁、海藻森林
    ContinentalShelf { depth: f32 }, // 50-200m——渔场
    DeepSea,                     // 200-4000m（深层）
    Abyssal,                     // 4000m+（深渊）
    Trench,                      // 海沟——极深
}

struct OceanCurrent {
    direction: Vec2,             // 流向
    speed: f32,                  // 流速 (m/s)
    temperature: f32,            // 相对温度 (暖流/寒流)
    season_modifier: f32,        // 季节性变化幅度
}

struct CoastSegment {
    start: Vec2,
    end: Vec2,
    coast_type: CoastType,
}

enum CoastType {
    Tectonic,        // 构造海岸——悬崖/峡湾（造山带直伸入海）
    Depositional,    // 沉积海岸——沙滩/滩涂（平原入海）
    CoralReef,       // 珊瑚礁——热带浅海
    Mangrove,        // 红树林海岸——热带/亚热带潮间带
    Fjord,           // 峡湾——冰川侵蚀的深窄海湾
}

struct NavigabilityMap {
    // 每格：可航行等级（0=无法航行，1=小型船，2=中型船，3=大型船）
    // 由深度、洋流速度、暗礁/浅滩、季节冰封共同决定
    grid: Grid2D<u8>,
}
```

**海岸线分类规则**：

| 条件 | 海岸类型 |
|------|---------|
| 造山带与海洋交界 | Tectonic（构造海岸——悬崖/峡湾） |
| 平原与海洋交界 | Depositional（沉积海岸——沙滩/滩涂） |
| 热带浅海（纬度<23° + 深度<50m） | CoralReef 或 Mangrove |
| 高纬度（>55°） + 造山带海岸 | Fjord（峡湾） |

### P2：陆地自然基底

**输入**：海陆掩码、造山带图
**输出**：高程图、水文图、气候图、生物群系图

仅在陆地内部运行（包括内陆水域）。

```rust
fn generate_terrain_base(
    seed: u64,
    mask: &LandSeaMask,
) -> TerrainBase {
    // 分层密度场（v3.0 §4.2）——P2 仅生成地形层（7层）
    // 完整密度场含 11 层（见 007-体素设计决策 §1.4）：
    //   后 4 层（文化覆盖/NPC修改/玩家SDF/天气临时）在后续管线阶段叠加
    let density_field = DensityField::new(seed)
        .add_layer(global_land_sea(mask))       // 海陆边界
        .add_layer(continental_skeleton(seed))   // 大陆骨架 → 700m 山
        .add_layer(regional_terrain(seed))       // 山丘群/谷地
        .add_layer(local_detail(seed))           // 脚下起伏
        .add_layer(geological_features(seed))    // 洞穴/悬崖
        .add_layer(hydrology_layer(seed))        // 河流侵蚀
        .add_layer(fantasy_features(seed))       // 奇幻特征——可插拔
        .build();

    let elevation = extract_elevation_map(&density_field);
    let hydrology = simulate_hydrology(&elevation, seed);
    let climate = compute_climate_zones(&elevation, mask);
    let biomes = classify_biomes(&elevation, &climate, &hydrology);

    TerrainBase { elevation, hydrology, climate, biomes }
}
```

**噪声基准参数（v3.0 §4.3）**：

| 层 | 频率 | 振幅 | 用途 |
|----|------|------|------|
| 全球海陆 | 0.00005-0.0002 | 1.0（二值化） | 大陆形状 |
| 大陆骨架 | 0.0005-0.002 | 300-800m | 造山带/大平原骨架 |
| 区域地形 | 0.005-0.02 | 50-200m | 山丘群/谷地系统 |
| 局部细节 | 0.03-0.08 | 5-30m | 脚下的起伏 |

**自动校验**：
```
max_elevation(500m_radius) - min_elevation(500m_radius) ≥ 10m   // 不死亡平板
max_elevation(10km_radius) ≥ 700m  // 保证有"真正的山"
max_elevation(10km_radius) - min_elevation(100km_radius) ≥ 1500m // 完整垂直范围
```

**水文模拟算法**（保留原方案）：
1. 在高程 > 0.6 的区域随机撒河流种子点
2. 从每个种子点沿坡度最陡方向下流（重力驱动）
3. 流量累积：支流汇入处增大河流宽度（Strahler 1-5 级）
4. 局部低洼处形成湖泊；无法入海的形成内陆湖/盐湖
5. 河流侵蚀：水流经过处轻微降低高程，形成河谷

**气候带**（新增——25 万 km² 跨度需要纬度映射）：
- 纬度由世界 Y 坐标映射（热带 0-23° / 温带 23-66° / 寒带 66-90°）
- 温度 = f(纬度, 高程, 洋流接近度)
- 湿度 = f(距海距离, 雨影, 洋流温度)
- 详细生物群系见 [[002-自然景观]]

### P4：聚落选址

详见 [[004-聚落生成]]。

核心变化：
- **承载容量先于选址**：先算每个位置能养活多少人，再决定是否建聚落
- **港口选址**：海岸线分类 + 可航行海域 + 内陆腹地资源
- **游牧选址**：不适用固定选址——标记"迁徙走廊"和"季节性草场"

```rust
fn score_settlement_site(
    pos: Vec2,
    carrying: &CarryingCapacity,
    resources: &ResourceMap,
    neighbors: &[Settlement],
) -> f64 {
    let water_score  = 0.30 * carrying.water_score(pos);
    let food_score   = 0.30 * carrying.food_score(pos);
    let build_score  = 0.20 * carrying.build_material_score(pos);
    let defense      = 0.12 * carrying.defense_score(pos);
    let trade_score  = 0.08 * carrying.trade_score(pos);
    water_score + food_score + build_score + defense + trade_score
}
```

### P7：交通网络

详见 [[006-交通网络]]。

**陆地**（保留原方案 MST 算法）：
- 所有聚落 + 独立建筑为节点
- 边权重 = 欧氏距离 × 地形代价 × 0.7 + 已有道路利用 × 0.3

**海上**（新增）：
- 港口城市为节点
- 边权重 = f(洋流助力, 季风方向, 航行距离, 中途岛屿补给)
- 航线分级：近海航线/跨海航线/远洋航线

## 三、确定性保证

整个管线必须是确定性的。给定相同的 `seed` → 完全相同的世界。

```rust
/// 所有随机数从 seed 派生的 RNG 序列中获取
struct DeterministicRng {
    // 每个生成阶段从 hash(seed, stage_name, chunk_coord?) 派生独立 RNG
}

fn stage_rng(seed: u64, stage: &str) -> StdRng {
    let stage_seed = hash(seed, stage);
    StdRng::seed_from_u64(stage_seed)
}

fn chunk_rng(seed: u64, stage: &str, cx: i64, cy: i64) -> StdRng {
    let chunk_seed = hash_3(seed, stage, cx as u64, cy as u64);
    StdRng::seed_from_u64(chunk_seed)
}
```

原则：
- 不依赖系统时间、外部状态、硬件差异
- 多线程生成时，每个 Chunk 从 `hash(seed, stage, chunk_x, chunk_y)` 派生独立 RNG
- 噪声函数使用 `seed` 作为输入

## 五、关键数据结构

```rust
/// 世界生成的顶层数据结构
struct WorldGenPipeline {
    seed: u64,
    params: WorldGenParams,
}

struct WorldGenParams {
    continent: GlobalContinentParams,
    ocean: OceanParams,
    terrain: TerrainParams,
    /// 海陆比（默认 0.70 → 海:陆 = 7:3）
    sea_ratio: f64,
}

/// 生成管线各阶段的输出
struct WorldGenOutput {
    land_sea_mask: LandSeaMask,           // P0
    ocean_system: OceanSystem,            // P1
    terrain_base: TerrainBase,            // P2
    resource_map: ResourceMap,            // P3
    settlements: Vec<SettlementData>,      // P4-P5
    buildings: Vec<BuildingData>,          // P6
    road_network: RoadNetwork,            // P7
    sea_routes: Vec<SeaRoute>,            // P7 (新)
    npc_population: NpcDistribution,      // P8
    history_result: HistorySimulationResult, // P9 —— 类型定义见 [[006-外部接口与性能预算|历史/006]] §2.1
    boundary_config: WorldBoundary,       // P10
}

/// 生成入口
impl WorldGenPipeline {
    fn generate(&self) -> WorldGenOutput {
```
