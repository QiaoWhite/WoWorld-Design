## §一、编排哲学

### 1.1 世界生成是编排器，不是执行器

```
┌─────────────────────────────────────────────────────────┐
│                  woworld_worldgen                        │
│                                                         │
│  编排器: 15 阶段生成管线，协调所有领域 crate 的初始化       │
│                                                         │
│  它决定"什么数据在哪个阶段被创建、引用什么其他数据"          │
│  它不拥有: NPC 行为逻辑、权力行使逻辑、经济交易逻辑          │
│  它拥有: 地形、资源、聚落、交通、生成编排权                 │
│                                                         │
│  各领域 crate 拥有: 运行时行为（GOAP/权力行使/订单簿撮合）   │
│  各领域 crate 提供: 纯数据结构的构造签名（世界生成调用）      │
└─────────────────────────────────────────────────────────┘
```

**核心区分**：
- **生成时**（world gen）：一次通过，确定性构造初始世界快照。所有数据结构公开构造。
- **运行时**（游戏循环）：领域 crate 消费初始快照，执行涌现行为。价格从交易涌现，权力从行使中波动。

### 1.2 设计原则

1. **种子决定一切**。`uint64 seed` → 完全相同的世界。所有随机数从种子派生，不依赖系统时间/硬件。
2. **体素 = 大地素材**。自建 Transvoxel（Rust 侧）→ Godot ArrayMesh。
3. **海洋是内容，不是空地**。海:陆 ≈ 6:4。航海/岛屿/跨洋发现是核心体验。
4. **涌现优先**。不写故事——只写规则。世界生成设定涌现的初始条件。
5. **结构层全量，细节层渐进**。NPC/权力/经济/历史全量预生成。建筑内饰/植物实例/动物个体按 Chunk 渐进生成。
6. **协同生成 > 顺序生成**。NPC、家族、历史事件在同一个过程中产生——因为它们互相引用。
7. **Bootstrap 不是破坏涌现**。初始订单簿、初始权力边、初始价格——是"已运行千年世界的当前快照"，不是"从零开始"。

## §二、完整生成管线（15 阶段）

```
种子(uint64)
  │
  ├── P0:  全球大陆架构
  │        双层海陆噪声 → 海陆掩码(6:4) → 造山带/裂谷/热点岛链
  │
  ├── P1:  海洋系统
  │        洋流模型 → 深度分级(0-4000m+) → 海岸分类 → 可航行海域
  │
  ├── P2:  陆地自然基底
  │        分层密度场(L0-L6) → 高程图 → 水文模拟 → 气候带 → 连续群系参数场
  │        ★ L6.5植被密度场不再在P2隐式生成——移至P2.25
  │
  ├── P2.25: ★ 植被覆盖（新增 · CHG-046）
  │        BiomeParamField → 优势种筛选(香农熵加权) → 多层噪声群落
  │        → 垂直分层光竞争 → L6.5密度场四通道
  │        → TimberAvailability · SurfaceMaterialMap · FuelLoadMap
  │        详见 [[012-植被覆盖生成]]
  │
  ├── P2.5: 文化与信仰种子
  │        文化起源点 + 障碍Voronoi → CultureCoreParams → 语言分配
  │        信仰种子 → FaithDefinition 目录 → 聚落信仰构成
  │        ★ 消费P2.25植被覆盖 → 知道"森林/草原/荒漠" → 建筑风格偏好
  │
  ├── P3:  资源分布
  │        矿田生成 → 岩料/木材/土沙/液体/有机/魔法资源 → ResourceMap
  │
  ├── P3.5: ★ 生命世界初始化（动物·怪物·亡灵）
  │        ★ 植物迁移至P2.25；动物种群(食物链顺序) → 怪物放置 → 亡灵从历史推导
  │
  ├── P4:  聚落选址
  │        承载容量计算 → 资源达标判断 → 选址评分 → 等级确定 → 港口/游牧特殊选址
  │
  ├── P5:  ★ 社会结构推导
  │        功能需求推导 → BuildingSpec → RoleTemplate → PowerSkeleton → 户外工作场所
  │
  ├── P6:  建筑生成
  │        调用建筑模块: BuildingSpec → BuildingData (含 rooms)
  │
  ├── P7:  交通网络
  │        陆地 MST 道路 + 海上航线 → TransportNetwork
  │
  ├── P8:  ★ 人口投影与家族协同生成
  │        ★ v2.1 五Pass: 祖先创建→配偶池合并→社会关系初始化→已故子女→DAG验证
  │        + Phase C 职业世系(整数规划) + Phase D 实例化(含婚姻审计)
  │
  ├── P9:  ★ 知识种子与历史事件生成
  │        KnowledgeSeed 传播链 → 因果事件图 → SettlementTechState → WorldStateDelta
  │
  ├── P10: ★ 权力 Bootstrap
  │        PowerEdgeTemplate → bootstrap_power_topology() → emerge_polities()
  │
  ├── P11: ★ 经济 Bootstrap
  │        bootstrap_economy() → 初始订单簿/钱包/NpcEconomicState/EconomicRelation
  │
  ├── P11.5: ★ 家具与放置物品生成
  │        P11.5a: Pass A — 附着家具（遍历建筑 Attachment 面 → 壁炉/嵌墙架/墙挂神龛）
  │        P11.5b: Pass B — 自由家具填充（消费 CultureFurnishProfile+NPC属性+经济水平+本地材料）
  │        P11.5c: Pass C — 野外放置实体（神龛/摊位/路标等）
  │        家具全量预生成（种子确定性）。持久化由存档系统接管。渐进的是渲染/物理 LOD。
  ├── P12: 世界边界处理
  │        自然海洋包裹 + 风暴带软边界 → WorldBoundary
  │
  ├── P13: ★ 生成后校验与完整性保证
  │        引用一致性·时间一致性·FamilyTree结构校验·最小可玩性·死因装饰·总预算核算·ModuleInit
  │
  └── 输出: ✅ ValidatedWorldState → [[../存档系统/README|存档系统]] 负责首次持久化（Initial 存档）
```

## §三、各阶段数据合同

### P0：全球大陆架构

| | |
|---|---|
| **输入** | `uint64` 世界种子, `GlobalContinentParams` |
| **输出** | `LandSeaMask { land_mask: Grid2D<bool>, sea_mask, orogen_zones, rift_zones, hotspots }` |
| **保证** | 至少 1 个面积 ≥ 10,000 km² 的大陆；最大海洋穿越距离 ≥ 500km |
| **确定性** | 种子→大陆噪声→二值化掩码。同一种子=同一大陆形状。 |

### P1：海洋系统

| | |
|---|---|
| **输入** | `LandSeaMask`, 全球温度基线 |
| **输出** | `OceanSystem { depth_map, currents: Vec<OceanCurrent>, coast_segments, navigable_zones }` |
| **被消费** | Life（海洋生物栖息地）, Vehicle（航路）, Weather（洋流→气候） |

### P2：陆地自然基底

| | |
|---|---|
| **输入** | `LandSeaMask`, `orogen_zones` |
| **输出** | `TerrainBase { elevation, hydrology, climate, biomes: BiomeParamField, surface_material_grid }` |
| **密度场** | 11 层叠加: 全球海陆→大陆骨架→区域地形→局部细节→地质特征→水文→奇幻→文化覆盖→NPC修改→玩家SDF→天气临时 |
| **群系参数场** | 连续值: temperature_annual / humidity / soil_fertility / soil_type / elevation_zone / water_table_depth。离散标签仅用于日志显示。 |
| **SurfaceMaterial** | 21 变体——从顶层体素 material_id 查表派生（不新增存储） |
| **依赖注入** | P2 后实现 5 个 trait: `ElevationQuery`/`ClimateParamsQuery`/`BiomeMicroclimateQuery`/`OceanCurrentQuery`/`WorldBoundaryQuery` |

### P2.25：植被覆盖（★新增 · CHG-046）

| | |
|---|---|
| **输入** | `BiomeParamField` (P2), `LandSeaMask` (P0), `HabitatProvider` (P2), `FantasyTerrainMap` (P2), `SoilTypeMap` (P2), `PlantSpeciesRegistry` (Life crate 只读) |
| **输出** | `VegetationCoverMap { community_templates, L6.5密度场(4通道), SurfaceMaterialMap, TimberAvailabilityMap, RootZoneMap, FuelLoadMap }` |
| **算法** | 香农熵加权优势种筛选 → 多层噪声群落密度(50m/100m/30m) → 垂直分层光竞争(冠层→林下→灌木→地被) → Poisson盘个体放置(仅出生点50km²) → 推导木材品质·地表覆盖·可燃物密度 |
| **核心接口** | `VegetationProvider` trait (woworld_core) — 建筑/NPC/天气/音频/战斗等模块的统一消费入口 |
| **L6.5密度场** | 冠层/林下/灌木/地被 四通道——注入体素密度场管线(L6幻想与L7文化覆盖之间) |
| **详见** | [[012-植被覆盖生成]] · [[010-植被群落与覆盖|生命/010]] |

### P2.5：文化与信仰种子

| | |
|---|---|
| **输入** | `TerrainBase`, `LandSeaMask`, **`VegetationCoverMap` (P2.25—用于推导建筑风格偏好)**, 文化起源点配置, 信仰目录 TOML |
| **输出** | `CultureCoreParams` (每文化区), `FaithDefinition` 目录, `SettlementFaithComposition` (每聚落) |
| **算法** | 文化: 起源点→障碍 Voronoi 扩散。从植被推导 `LandcoverType` → 建筑风格偏好(森林=木结构/草原=土坯/荒漠=石构)。信仰: 种子驱动 + 文化 tolerance 决定多信仰并存度。 |
| **详见** | [[../文化系统/002-文化核心参数与生成|文化 002]] · [[../信仰系统/001-信仰系统总纲|信仰 001]] |

### P3：资源分布

| | |
|---|---|
| **输入** | `TerrainBase`, `VegetationCoverMap` (P2.25), 资源种子, `ResourceDistribution` 参数 |
| **输出** | `ResourceMap { ore_fields, stone_deposits, wood_zones, liquid_sources, organic_zones, magic_sources }` |
| **木材推导** | ★ wood_zones 从 P2.25 的 `TimberAvailabilityMap` 推导——不再使用独立噪声。木材品质/丰度由植被群落的树种组成决定，非独立生成。 |
| **盈余推导** | 每种资源的年可采量 → P5 社会结构推导 + P11 经济 bootstrap 的基础 |

### P3.5：生命世界初始化 ★植物已迁移至P2.25

| | |
|---|---|
| **输入** | `VegetationCoverMap` (P2.25—消费初级生产力), `BiomeParamField`, `HabitatProvider`, `FantasyTerrainMap`, (亡灵需要) `EventLog`(P9 产出——亡灵部分后移) |
| **输出** | `LifeWorldState { animal_populations, monster_placements, undead_placements, legendary_life }` |
| **内部顺序** | ★ 植物不再在此阶段——已在P2.25完成。草食动物 → 肉食动物 → 顶级捕食者 → 怪物 → 亡灵。每层校验食物链承载力。初级生产力从 VegetationCoverMap 的物种生长率计算。 |
| **存储模型** | 种群模板 (~80B/Chunk 动物, ~200B 怪物/亡灵)。运行时玩家进入 Chunk 时从模板+种子确定性实例化个体。 |
| **注意** | 亡灵部分依赖 P9 产出的大规模死亡地点 → P3.5 分两次调用: 动物+怪物在 P3 后；亡灵在 P9 后 |

### P4：聚落选址

| | |
|---|---|
| **输入** | `TerrainBase`, `ResourceMap`, `CarryingCapacity` |
| **输出** | `Vec<SettlementData> { id, position, grade, population_cap, culture_id, territory, founding_year }` |
| **等级分布** | 村庄 60% · 小镇 25% · 城市 10% · 城堡 5% · 港口城市(城市子类) · 游牧聚落 |
| **选址评分** | `water×0.30 + food×0.30 + build×0.20 + defense×0.12 + trade×0.08` |
| **CarryingCapacity** | 定义在 `woworld_core` ——纯函数。计算后存入 `SettlementData.carrying_capacity`。 |

### P5：社会结构推导 ★新增

| | |
|---|---|
| **输入** | `SettlementData`, `ResourceMap`, `SettlementTechState`(暂为初值), `CultureCoreParams` |
| **输出** | `SettlementStructure { building_specs, role_templates, power_skeleton, outdoor_workplaces, functional_zones }` |

## §四、确定性保证

```rust
/// 所有随机数从 seed 派生的 RNG 序列中获取
struct DeterministicRng {
    // 每阶段从 hash(seed, stage_name) 派生独立 RNG
    // 每 Chunk 从 hash(seed, stage, chunk_x, chunk_y) 派生
}

fn stage_rng(seed: u64, stage: &str) -> StdRng {
    StdRng::seed_from_u64(hash(seed, stage))
}

fn chunk_rng(seed: u64, stage: &str, cx: i64, cy: i64) -> StdRng {
    StdRng::seed_from_u64(hash_3(seed, stage, cx as u64, cy as u64))
}
```

**绝对规则**：
- 不依赖系统时间、外部状态、硬件差异
- 多线程生成时，每个 Chunk 从 `hash(seed, stage, chunk_x, chunk_y)` 派生独立 RNG
- 各 crate 的 bootstrap 函数消费传入的 `&mut DeterministicRng`——不内部创建 RNG
- 生成顺序固定——同一阶段的随机数消费顺序固定
