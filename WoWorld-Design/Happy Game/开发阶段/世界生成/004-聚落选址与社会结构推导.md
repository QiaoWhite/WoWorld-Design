## §一、P4：聚落选址

### 1.1 承载容量先于选址

```rust
fn place_settlements(
    terrain: &TerrainBase,
    resources: &ResourceMap,
    culture_params: &CultureCoreParams,
    rng: &mut DeterministicRng,
) -> Vec<SettlementData> {
    // 1. 对全陆地计算承载容量评分 (每 1km² 采样)
    let capacity_grid = compute_carrying_capacity_grid(terrain, resources);
    
    // 2. 筛选候选点: capacity ≥ 50人 (最小村庄)
    let candidates: Vec<Vec2> = capacity_grid.iter()
        .filter(|(_, cap)| cap.population_cap >= 50)
        .map(|(pos, _)| pos)
        .collect();
    
    // 3. 泊松圆盘采样——最小间距 3km（防止聚落过于密集）
    let sites = poisson_disk_sample(&candidates, 3.0, rng);
    
    // 4. 每站点评分——确定等级和类型
    sites.iter().map(|pos| {
        let score = score_settlement_site(pos, &capacity_grid, terrain, resources);
        let grade = determine_grade(score, rng);
        SettlementData { position: *pos, grade, carrying_capacity: capacity_grid.at(*pos), .. }
    }).collect()
}
```

### 1.2 选址评分

```
score = water_score  × 0.30    // 水源距离+水质+地下水
      + food_score   × 0.30    // 可耕地+猎场+渔场
      + build_score  × 0.20    // 木材/石材可获取性
      + defense_score × 0.12   // 地形防御值
      + trade_score   × 0.08   // 河流交汇/海岸/群域交界/天然港口
```

**`trade_score`** 从地理特征推导——不是从"已知其他聚落"推导（此时聚落尚未生成）。

### 1.3 聚落等级体系

| 属性 | 村庄 | 小镇 | 城市 | 城堡 | 港口城市 | 游牧 |
|------|------|------|------|------|---------|------|
| **生成概率** | 60% | 25% | 10% | 5% | (城市子类) | (种子决定) |
| **核心目的** | 食物+原材料 | 加工+特种贸易 | 统治+贸易+防御 | 军事控制 | 航海贸易+渔业 | 畜牧迁徙 |
| **固定建筑** | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ |
| **选址条件** | 水+食物+材料 | +核心产业资源 | +防御纵深+贸易交汇 | +防御潜力第一 | +天然港湾+可航行海域 | +迁徙走廊+季节性草场 |

## §二、P5：社会结构推导

**核心公式**: `社会结构 = f(资源禀赋, settlement_tech, 文化参数, 承载容量)`

### 2.1 功能需求推导

从聚落周边资源→推导聚落的经济功能权重。

```rust
fn derive_function_profile(
    settlement: &SettlementData,
    resources: &ResourceMap,
    terrain: &TerrainBase,
) -> FunctionProfile {
    // 5km 半径内检查资源存在性
    let has_iron   = resources.ore_within(settlement.position, settlement.grade.resource_radius(), ORE_IRON);
    let has_copper = resources.ore_within(settlement.position, ...);
    let has_coal   = ...;
    let has_clay   = ...;
    let has_magic  = resources.magic_zone_at(settlement.position);
    
    // 群系决定农业/畜牧/渔业/林业倾向
    let arable_score = terrain.biomes.arable_quality(settlement.position);
    let pasture_score = terrain.biomes.grassland_quality(settlement.position);
    let forest_score = terrain.biomes.forest_density(settlement.position);
    let fishery_score = terrain.hydrology.fishing_potential(settlement.position);
    
    // 海岸+可航行海域 → 港口功能
    let harbor_score = if terrain.is_coastal(settlement.position) 
        && terrain.ocean.navigable_at(settlement.position) { 1.0 } else { 0.0 };
    
    FunctionProfile {
        mining:      normalize(has_iron + has_copper + has_coal + ...),
        agriculture: arable_score,
        pastoralism: pasture_score,
        forestry:    forest_score,
        fishing:     fishery_score,
        trade:       harbor_score + trade_node_score(settlement.position, terrain),
        magic:       has_magic as u8 as f32,
    }
}
```

### 2.2 建筑规格推导 (BuildingSpec)

```rust
fn derive_building_specs(
    profile: &FunctionProfile,
    grade: SettlementGrade,
    population: u32,
    culture: &CultureCoreParams,
    tech: &SettlementTechState,
    rng: &mut DeterministicRng,
) -> Vec<BuildingSpec> {
    let mut specs = Vec::new();
    
    // 核心生产建筑 — 从功能权重+技术天花板推导
    if profile.mining > 0.1 {
        let smithy_count = (profile.mining * population as f32 / 200.0).ceil() as u32;
        specs.push(BuildingSpec {
            function: BuildingFunction::Smithy,
            count: smithy_count.max(1),
            area_range: (40.0, 120.0),
            floor_range: (1, 2),
            culture_style: culture.style_id(),
            household_size_hint: None,   // 非居住
            acoustic_tag: AcousticTag::Workshop,
            ..default_for_grade(grade)
        });
    }
    
    // 居住建筑 — 从家庭结构类型推导单栋面积
    let family_type = culture.family_structure_type();
    let household_size = match family_type {
        FamilyStructureType::Nuclear => HouseholdSize::Nuclear,
        FamilyStructureType::Extended => HouseholdSize::Extended,
        FamilyStructureType::ClanBased => HouseholdSize::Compound,
        FamilyStructureType::LooseExtended => HouseholdSize::Nuclear,
    };
    let residence_count = (population as f32 / household_size.avg_persons()).ceil() as u32;
    specs.push(BuildingSpec {
        function: BuildingFunction::Residence,
        count: residence_count,
        area_range: household_size.area_range(),
        household_size_hint: Some(household_size),
        acoustic_tag: AcousticTag::WoodenRoom,
        ..
    });
    
    // 公共建筑 — 从等级决定
    match grade {
        Town | City => {
            specs.push(BuildingSpec { function: TownHall, count: 1, .. });
            specs.push(BuildingSpec { function: Market, count: 1..=3, .. });
        }
        City | Castle => {
            specs.push(BuildingSpec { function: Barracks, count: 1, .. });
            specs.push(BuildingSpec { function: Temple, count: 1..=3, .. });
        }
        _ => {}
    }
    
    // 例外注入: 5-15% 概率在非预期功能区插入异类建筑
    //   (如: 住宅区中有一间小铁匠铺——"巷子里的老王")
    inject_exceptions(&mut specs, rng, EXCEPTION_RATE);
    
    specs
}
```

**`household_size` 如何影响建筑面积**: `HouseholdSize::Nuclear` → 60-100m², 2-3 卧室。`Extended` → 120-250m², 4-6 卧室。`Compound` → 200-500m², 多院落。

### 2.3 角色模板推导 (RoleTemplate)

```rust
fn derive_role_templates(
    building_specs: &[BuildingSpec],
    profile: &FunctionProfile,
    culture: &CultureCoreParams,
) -> Vec<RoleTemplate> {
    let mut roles = Vec::new();
    let mut role_idx = 0;
    
    // 从每类建筑→推导需要的角色
    for spec in building_specs {
        match spec.function {
            Smithy => {
                roles.push(RoleTemplate {
                    role_index: role_idx,
                    occupation_tags: vec![ProfessionTag::Blacksmith],
                    count_range: (spec.count, spec.count + 2),  // 师父+学徒
                    expected_skill_range: (0.3, tech_ceiling(spec.function)),
                    social_standing_tier: 2,  // 工匠阶层
                });
                role_idx += 1;
            }
            Residence => {
                roles.push(RoleTemplate {
                    role_index: role_idx,
                    occupation_tags: vec![ProfessionTag::Farmer, ProfessionTag::Laborer,
                                          ProfessionTag::Artisan, ProfessionTag::Merchant],
                    count_range: (spec.count as u32, (spec.count as f32 * 1.5) as u32),
                    expected_skill_range: (0.0, 1.0),
                    social_standing_tier: 1,  // 平民
                });
                role_idx += 1;
            }
            TownHall => {
                let authority_depth = culture.authority_depth();
                roles.push(RoleTemplate {
                    role_index: role_idx,
                    occupation_tags: vec![ProfessionTag::Noble, ProfessionTag::Magistrate,
                                          ProfessionTag::Steward, ProfessionTag::TaxCollector],
                    count_range: (authority_depth as u32, authority_depth as u32 * 3),
                    expected_skill_range: (0.3, 1.0),
                    social_standing_tier: 4,  // 统治阶层
                });
                role_idx += 1;
            }
            // ... 其余建筑类型
        }
    }
    
    roles
}
```

### 2.4 权力骨架推导 (PowerSkeleton)

从 RoleTemplate → 推导初始权力边模板。P10 实例化为具体 PowerEdge。

```rust
fn derive_power_skeleton(
    roles: &[RoleTemplate],
    culture: &CultureCoreParams,
    settlement: &SettlementData,
) -> Vec<PowerEdgeTemplate> {
    let mut edges = Vec::new();
    let ruler_role = roles.iter().find(|r| r.has_tag(ProfessionTag::Noble)).unwrap();
    let citizen_roles: Vec<_> = roles.iter().filter(|r| r.social_standing_tier < 4).collect();
    let authority_depth = culture.authority_depth();
    
    // 领土控制: 统治者 → 领土范围内所有人
    edges.push(PowerEdgeTemplate {
        holder_role_index: ruler_role.role_index,
        subject_role_index: ALL_CITIZENS,  // 特殊标记→所有居民
        atom_type: AtomTypeTag::Constrain,
        domain: PowerDomain::Territory { bounds: settlement.territory, auto_apply: true },
        source: PowerSource::Inherited,
        legitimacy_initial: culture.power_distance * 0.8,  // <1.0, 留波动空间
        cardinality: EdgeCardinality::OneToMany,
        ..
    });
    
    // 税收: 统治者 → 所有成年居民
    edges.push(PowerEdgeTemplate {
        holder_role_index: ruler_role.role_index,
        subject_role_index: ALL_CITIZENS,
        atom_type: AtomTypeTag::Extract,
        domain: PowerDomain::Behavior { behavior_type: BehaviorType::Tax, scope: Always },
        ..
    });
    
    // 委托链: authority_depth ≥ 2 → 创建中间管理层
    if authority_depth >= 2 {
        let steward_role = roles.iter().find(|r| r.has_tag(ProfessionTag::Steward)).unwrap();
        edges.push(PowerEdgeTemplate {
            holder_role_index: ruler_role.role_index,
            subject_role_index: steward_role.role_index,
            atom_type: AtomTypeTag::Delegate,
            cardinality: EdgeCardinality::SinglePair,
            ..
        });
    }
    
    edges
}
```

### 2.5 户外工作场所推导

从承载力模型的抽象标量→物理地块坐标。

```rust
fn derive_outdoor_workplaces(
    settlement: &SettlementData,
    terrain: &TerrainBase,
    carrying: &CarryingCapacity,
    profile: &FunctionProfile,
    rng: &mut DeterministicRng,
) -> OutdoorWorkplaces {
    // 农田: 5km 半径内 soil_fertility > 0.3 + elevation < 200m + slope < 5°
    //       泊松圆盘采样(间距≥100m) → 每块 ~2ha
    let farmlands: Vec<FarmlandPlot> = poisson_disk_sample_arable(
        settlement.position, 5000.0,
        |pos| terrain.biomes.soil_fertility.at(pos) > 0.3
              && terrain.elevation.at(pos) < 200.0
              && terrain.slope.at(pos) < 5.0,
        100.0,  // 最小间距
        (carrying.arable_land_ha / 2.0).ceil() as usize,  // 需要的地块数
        rng,
    ).iter().map(|center| FarmlandPlot {
        center: *center,
        radius: 80.0 + rng.gen::<f32>() * 120.0,  // 80-200m 半径
        crop_type: derive_crop_type(terrain.climate.at(*center), profile),
        capacity_farmers: 2 + (rng.gen::<f32>() * 4.0) as u8,
        annual_yield_kg: (16000.0 * terrain.biomes.soil_fertility.at(*center) 
                          * rng.gen_range(0.8, 1.2)) as u32,
    }).collect();
    
    // 牧场: grassland + 坡度 < 15°
    let pastures = ...;
    
    // 渔场: 河流/湖泊/海岸
    let fishing_spots = ...;
    
    // 猎场: 森林区 (10km 半径)
    let hunting_grounds = ...;
    
    OutdoorWorkplaces { farmlands, pastures, fishing_spots, hunting_grounds }
}
```

### 2.6 游牧载具与装备（★ 新增 — 载具系统集成）

游牧聚落不生成 BuildingSpec。替代为初始载具和可打包物品。

```rust
fn derive_nomadic_equipment(
    clan: &NomadicStructure,
    culture: &CultureCoreParams,
    tech: &SettlementTechState,
    biome: &BiomeParamField,
    rng: &mut DeterministicRng,
) -> NomadicEquipment {
    // 载具: 从 VehicleArchetype(TOML) × Culture × Tech × Biome → VehicleDef
    let vehicles = vehicle_crate::initialize_nomadic_vehicles(clan, culture, tech, biome, rng);
    
    // 帐篷/穹庐: 作为 ItemEntId 存在于氏族库存
    let tents = generate_nomadic_tents(clan.population, culture, rng);
    
    // 驮兽: 马/骆驼/牦牛——由生命系统生成（坐骑，非载具）
    let livestock = clan.livestock_composition.clone();
    
    NomadicEquipment {
        vehicles,       // Vec<VehicleState> — 马车/篷车/货车
        tents,          // Vec<ItemSpec> — 帐篷/穹庐（可打包物品）
        livestock,      // LivestockComposition — 马/骆驼/牛/羊
    }
}
```

**载具作为移动居所**: 游牧 NPC 的 `NpcAnchors.home = NomadicCamp { current_camp }`。篷车是卧榻——载具具有 `living_quarters.is_mobile_home = true`。牲畜提供拉力（坐骑系统），马车载具定义拉力需求。

**固定聚落的可选载具**（运行时也可建造更多）: 港口城市→渔船×3-10+商船×1-3，沿海村庄→渔船×1-5，河畔聚落→渡船×1-2。这些在 P5 中可选生成。

## §三、跨聚落统筹

**问题**: 两个相邻聚落共享同一矿田 → 每个都会尝试把自己变成矿业镇。需要统筹。

```rust
fn coordinate_cross_settlement_specialization(
    settlements: &mut [SettlementData],
    structures: &mut [SettlementStructure],
    resources: &ResourceMap,
) {
    // 检测共享资源冲突:
    //   如果两个聚落都依赖同一个矿田 → 距离矿田更近的那个获得 mining 功能
    //   另一个退化为加工/服务功能
    
    for (a, b) in settlement_pairs_within(settlements, SHARED_RESOURCE_MAX_DIST) {
        for resource in SHARED_RESOURCE_TYPES {
            if both_depend_on(a, b, resource, resources) {
                let a_dist = distance_to_resource(a.position, resource);
                let b_dist = distance_to_resource(b.position, resource);
                if a_dist < b_dist {
                    // a = 矿镇, b = 加工镇
                    a.structure.function_profile.mining *= 1.2;
                    b.structure.function_profile.mining *= 0.5;
                    b.structure.function_profile.trade *= 1.3;  // 加工品需要贸易出口
                }
            }
        }
    }
}
```
