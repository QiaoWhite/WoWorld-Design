## §〇、设计目标

100K+ NPC，每人拥有:
- 年龄、性别、BigFive 人格、技能等级、职业标签
- 在 FamilyTree（有向无环图）中的位置
- 绑定的物理锚点（`NpcAnchors`: 家/工作场所/产权）
- 初始社会地位和钱包（钱包在 P11 填充）
- 可供查询但**不预存文本**的个人史

**约束**:
- 总人口 = Σ settlement.population（来自 P4 承载容量×等级系数）
- 人口金字塔符合 Gompertz 死亡率 + 文化生育率
- 职业分布满足 P5 `RoleTemplate` 的数量约束
- 家族关系跨代一致（无时间悖论）
- 跨聚落婚姻在小聚落中自动启用

## §一、四阶段生成算法

```
Phase A — 骨架:   按年龄/性别生成占位 NPC。职业=未分配。
Phase B — 家族:   逆向人口投影。婚姻配对→子女→反推祖先。
Phase C — 职业世系: 全局整数规划。满足 RoleTemplate 约束。
Phase D — 实例化: 技能+BigFive+NpcAnchors+初始动作状态。
```

### 1.1 Phase A：骨架生成

对每个聚落，从承载容量→人口总数→年龄分布直方图→按桶采样。

```rust
fn generate_skeletons(
    settlement: &SettlementData,
    culture: &CultureCoreParams,
    rng: &mut DeterministicRng,
) -> Vec<NpcSkeleton> {
    let total_pop = settlement.population;
    
    // 1. 稳态人口金字塔
    let age_pyramid = gompertz_age_distribution(
        total_pop,
        culture.fertility_rate,     // TOML配置 → 年生育率
        culture.marriage_age_avg,   // 平均初婚年龄
        GOMPERTZ_ALPHA,             // ~0.0005 [TUNING]
        GOMPERTZ_BETA,              // ~0.08
        rng,
    );
    // age_pyramid[0..=19] = 每5岁桶的人数

    // 2. 按桶采样——分配年龄和性别
    let mut skeletons = Vec::with_capacity(total_pop as usize);
    for (bucket_idx, count) in age_pyramid.iter().enumerate() {
        let age_min = bucket_idx as u8 * 5;
        let age_max = age_min + 4;
        for _ in 0..*count {
            skeletons.push(NpcSkeleton {
                id: NpcId::from_hash(hash(seed, settlement.id, skeleton_index)),
                age: rng.gen_range(age_min..=age_max),
                gender: if rng.gen::<f32>() < 0.51 { Female } else { Male },
                birth_settlement: settlement.id,
                birth_year: -1 * rng.gen_range(age_min..=age_max) as i32,
                // 职业=未分配——Phase C 填充
                profession_tags: Vec::new(),
            });
        }
    }
    skeletons
}
```

**Gompertz 年龄分布**：`μ(x) = α × exp(β × x)`。在中世纪农业社会参数下，底部宽（高出生率+高婴儿死亡率）、顶部窄（极少活到 80+）。

### 1.2 Phase B：家族结构——逆向人口投影

从活着的 NPC 反推他们的父母。不正向模拟——反向填充 FamilyTree。

```
算法: Reverse Population Projection

INPUT: 每个聚落的 Vec<NpcSkeleton> (按年龄从老到幼排序)
OUTPUT: FamilyTree (FatherOf/MotherOf/MarriedTo 边), Vec<NpcSkeleton> (含祖先)

FOR each settlement:
  1. 按年龄降序排列活人 NPC（老人先配对）

  2. 婚姻配对:
     FOR each NPC (年龄 ≥ marriage_age_min, 未配对):
       候选配偶 = 同龄±15岁, 异性, 同聚落(小聚落扩展到邻近聚落),
                  social_standing ±1 tier, blood_distance ≥ 2
       评分 = age_compatibility × standing_match × 随机噪声
       选择最高分→创建 MarriedTo 边(start_year = min(birth_year)+marriage_age)
       已婚 NPC 标记为 paired

  3. 生成子女:
     FOR each 已婚配对:
       子女数 = 泊松(文化生育率 × 结婚年数 / fertility_window)
       FOR each 子女:
         如果子女年龄 < 0 → 还未出生, 跳过
         如果子女年龄 ≥ 0 且不在活人列表中 → 已死, 加入 dead_ancestors
         如果子女在活人列表中 → 已有, 创建 FatherOf/MotherOf 边

  4. 反推祖先:
     FOR each NPC (已死或活着):
       以概率(1 - mortality_rate[parent_age_at_child_birth]) → 父母还活着
       如果活着 → 创建 FatherOf/MotherOf 边
       如果已死 → 加入 dead_ancestors（仅保留最近 2-3 代）
       递归直到: 父母年龄 < settlement_founding_year 或 递归深度 > 3

  5. 跨聚落婚姻:
     小聚落(<200人) → 30-60% 配偶来自邻近聚落
     P(跨聚落) = exp(-distance / marriage_range) × culture.exogamy_rate
     社会阶层匹配: ±1 tier
```

**跨聚落婚姻**的关键：先完成所有聚落的 Phase B.2（内部配对），然后对剩余未配对的 NPC 做跨聚落匹配。这避免了小聚落的基因瓶颈。

### 1.3 Phase C：职业世系——整数规划

**问题**: 将 N 个 NPC 分配到 M 种职业，满足 P5 RoleTemplate 的数量约束，最大化匹配质量。

```
约束:
  ∀ occupation: Σ_i assigned[i][occ] ∈ [role.count_min, role.count_max]
  ∀ npc: assigned[npc] 恰好为一个职业标签集合 (≤4个标签)
  
目标: 最大化 Σ 匹配质量
  匹配质量[npc][occ] =
    family_inheritance_bonus[npc][occ] × 0.40   // 父亲是铁匠→子女优先
    + big_five_fit[npc][occ] × 0.35              // 人格适配度
    + settlement_need_urgency[occ] × 0.25         // 聚落缺人程度
```

**算法**: 最小费用最大流。

```rust
}

fn big_five_fit(npc: &NpcSkeleton, occupation: ProfessionTagId) -> f32 {
    // 铁匠: conscientiousness↑, openness↓ → 高分
    // 商人: extraversion↑, agreeableness↓ → 高分  
    // 冒险者: openness↑, neuroticism↓ → 高分
    // 映射表在 ProfessionTag TOML 中定义 (big_five_profile 字段)
}

fn family_inheritance_bonus(npc: &NpcSkeleton, occupation: ProfessionTagId) -> f32 {
    if let Some(father) = family_tree.father_of(npc.id) {
        if father.has_tag(occupation) { return 0.8; }  // 强继承
    }
    if let Some(mother) = family_tree.mother_of(npc.id) {
        if mother.has_tag(occupation) { return 0.5; }  // 弱继承（母系）
    }
    0.0
}
```

**继承强度调制**: `actual_inheritance = base_inheritance × culture.occupation_inheritance_strength`。高传统社会→严格世袭。低传统社会→子女自由选择。

### 1.4 Phase D：实例化

```rust
fn instantiate_npc(
    skeleton: NpcSkeleton,
    occupation_tags: Vec<(ProfessionTagId, f32)>,  // PhaseC产出
    family_tree: &FamilyTree,
    buildings: &[BuildingData],
    outdoor_workplaces: &OutdoorWorkplaces,
    settlement_tech: &SettlementTechState,
    culture: &CultureCoreParams,
    rng: &mut DeterministicRng,
) -> NpcData {
    // 1. BigFive: 文化基线 + 个体偏移 + 职业修正
    let big_five = generate_big_five(culture, &occupation_tags, rng);

    // 2. 技能等级: 职业相关技能 = settlement_tech × age_progress × talent
    let skills = generate_skills(&occupation_tags, skeleton.age,
                                  settlement_tech, &big_five, rng);

    // 3. NpcAnchors: 家(从FamilyTree→household_members→分配民居)
    //              工作场所(从occupation→匹配对应BuildingData或户外场所)
    let anchors = generate_anchors(&skeleton, family_tree, buildings,
                                    outdoor_workplaces, rng);

    // 4. 初始动作状态: ProfessionTag daily_schedule × 当前时间
    let initial_action = initial_action_state(&occupation_tags, world_time_zero,
                                               &anchors, rng);

    NpcData {
        id: skeleton.id,
        name: generate_name(culture, skeleton.gender, rng),
        birth_settlement: skeleton.birth_settlement,
        birth_year: skeleton.birth_year,
        age: skeleton.age,

        skills,
        profession_tags: occupation_tags,

        emotion_state: EmotionState::default(), // 首帧 EmotionEngine 计算
        // ... 其余字段由各子系统的初始化函数填充
    }
```

**建筑→NPC 匹配**（`assign_workplaces`）:
1. 将 NPC 按 occupation 分组，将建筑按 BuildingFunction 分组
2. 对每组职业，在匹配功能的建筑+户外场所中评分选择
3. 评分 = `skill_level × 100` + `1000/(distance_to_home + 1)` + `family_history_bonus(500)`
4. 分数最高的 NPC 优先选择——已满建筑跳过

## §四、游牧聚落特殊处理

游牧 NPC 的 `NpcAnchors`：
- `home = NomadicCamp { current_camp: Vec2 }` —— 当前季节营地坐标

## §五、大五人格分配

```rust
fn generate_big_five(
    culture: &CultureCoreParams,
    occupations: &[(ProfessionTagId, f32)],
    rng: &mut DeterministicRng,
) -> BigFive {
    // 1. 文化基线 (CultureCoreParams → 该文化人群的人格均值)
    let mut base = culture.big_five_baseline();

    // 2. 个体偏移 (高斯噪声 σ=0.15)
    for dim in base.iter_mut() {
        *dim = (*dim + rng.gen_normal(0.0, 0.15)).clamp(0.0, 1.0);
    }

    // 3. 职业修正 (从 ProfessionTag TOML 的 big_five_profile 字段)
    for (tag, _) in occupations {
        if let Some(profile) = tag_def(*tag).big_five_profile {
            base.openness            += profile.openness_mod * 0.1;
            base.conscientiousness   += profile.conscientiousness_mod * 0.1;
            base.extraversion        += profile.extraversion_mod * 0.1;
            base.agreeableness       += profile.agreeableness_mod * 0.1;
            base.neuroticism         += profile.neuroticism_mod * 0.1;
        }
    }

    // 4. 年龄修正: young→openness↑, old→conscientiousness↑
    //    所有修正乘法叠加，重归一化到 0-1

    base.normalize()
}
```

**无硬编码**: 职业→人格映射全在 ProfessionTag TOML 中。

## §六、技能分配

```rust
fn generate_skills(
    occupations: &[(ProfessionTagId, f32)],
    age: u8,
    settlement_tech: &SettlementTechState,
    big_five: &BigFive,
    rng: &mut DeterministicRng,
) -> Vec<SkillEntry> {
    // 1. 从 ProfessionTag TOML 的 related_skills → 确定关联技能
    // 2. 技能天花板 = settlement_tech.tech_levels[domain] / 100.0
    //    （该聚落该技艺的最高水平——来自 KnowledgeSeed 传播）
    // 3. 年龄进度曲线:
    //    学徒期(12-18): 0.1 → 0.4
    //    熟练期(18-35): 0.4 → 0.85
    //    维持期(35-55): 0.85 ± 0.05
    //    下降期(55+):   0.85 → 0.7
    // 4. 天赋修正 (三层天赋: 天生×交叉训练×MentalAccess)
    // 5. 天花板裁剪: skill = min(raw_skill, ceiling)

    occupations.iter().flat_map(|(tag, proficiency_base)| {
        let def = tag_def(*tag);
        def.related_skills.iter().map(|(skill_id, weight)| {
            let raw = proficiency_base * weight * age_progress_curve(age, skill_id);
            let ceiling = settlement_tech.tech_levels
                .get(&def.tech_domain).unwrap_or(&30) / 100.0;
            let talent_mod = compute_talent(big_five, skill_id, rng);
            SkillEntry {
                skill_id: *skill_id,
                level: (raw * talent_mod).clamp(0.0, ceiling as f32),
            }
        }).collect::<Vec<_>>()
    }).collect()
}
```

## §八、性能

| 阶段 | 复杂度 | 100K NPC 预估 |
|------|--------|-------------|
| Phase A 骨架 | O(N) per settlement | ~50ms |
| Phase B 家族 (含跨聚落婚姻) | O(N×avg_children) + O(P²) 跨聚落匹配 | ~200ms |
| Phase C 整数规划 | O(N×M×log(N)) 费用流 | ~300ms |
| Phase D 实例化 | O(N) per NPC | ~150ms |
| **总计 P8** | | **~0.7-1.5s** (rayon 按聚落并行) |

**内部并行**: 每个聚落的 Phase A+B+D 完全独立 → rayon `par_iter`。Phase C 在 Phase B 完成后对每个聚落独立运行。跨聚落婚姻匹配在 Phase B 末期集中处理一次（~10ms）。

**内存**: NpcData (100K × ~6KB) = 600MB 构造峰值 → SoA 压缩后 ~400MB。FamilyTree 边 (~300K 条 × ~24B) = 7MB。
