## §一、设计原则

1. **不预生成全部个体** — 存储种群模板（~100B/Chunk）。运行时玩家进入时从模板+种子确定性实例化个体。
2. **食物链顺序** — 草食→肉食→顶级捕食者。初级生产力从 P2.25 VegetationCoverMap 读取。每层校验承载力。
3. **亡灵从历史推导** — 大规模死亡地点(战争/瘟疫) + 灵元素印记场 → 亡灵生成。
4. **传奇生命种子稀有** — 龙/世界树/巨兽。每个种子 0-5 个。

## §二、植物分布 —— ★已迁移至 P2.25

> **CHG-046**: 植物种群分布从此阶段（P3.5）迁移至 P2.25 植被覆盖阶段。
> 详见 [[012-植被覆盖生成]] · [[010-植被群落与覆盖|生命/010]]。
>
> **迁移原因**：
> 1. 植被是自然基底（如地形），不是"生命层"——P2.25在P2地形和P2.5文化之间更合理
> 2. L6.5植被密度场需要早于Signed Heightfield烘焙——P2.25紧接P2确保远山天际线有森林剪影
> 3. 文化(P2.5)需要知道森林/草原分布来塑造建筑风格偏好
> 4. 资源(P3)的木材zone应从植被群落推导，非独立噪声
>
> **植物不再在P3.5放置。** 动物初级生产力从 P2.25的 `VegetationCoverMap` 中读取物种生长率计算。

## §三、动物种群分布（消费 P2.25 植被）

### 3.1 食物链顺序生成

```
Step 1: 每 Chunk 的植物初级生产力 (kcal/m²/yr)
        = 从 P2.25 VegetationCoverMap 读取 canopy_closure
          × species_registry.average_growth_rate(dominant_species)
          × soil_fertility × 365.0
        ★ 植物不再在此阶段生成——直接从 VegetationCoverMap 查询

Step 2: 草食动物
        max_herbivore_count = primary_productivity / species.daily_kcal_need / 365
        实际数 = max × rng(0.5, 0.9)  // 生态位未满

Step 3: 肉食动物
        max_carnivore = Σ(herbivore_count × prey_ratio) / predator_prey_ratio
        实际数 = max × rng(0.3, 0.7)  // 顶级捕食者密度更低

Step 4: 顶级捕食者
        极低密度: 1-2 只 / 100 km²
```

### 3.3 迁徙物种

标记为 migratory → 季节性密度变化 → 运行时沿 migration_route 移动。路线从群系参数场推导——沿绿色走廊（高土壤肥力带）连接夏季和冬季栖息地。

## §四、怪物分布

```rust
fn place_monsters(
    biome_params: &BiomeParamField,
    fantasy_terrains: &FantasyTerrainMap,
    monster_registry: &MonsterSpeciesRegistry,
    rng: &mut DeterministicRng,
) -> Vec<MonsterPlacement> {
    let mut placements = Vec::new();
    
    for chunk in land_chunks() {
        let params = biome_params.sample(chunk.center);
        let magic_conc = fantasy_terrains.magic_concentration_at(chunk.center);
        
        // 一般怪物: 群系危险度 × 魔法浓度 × 与聚落距离
        let danger = params.biome_danger() * (1.0 + magic_conc) 
                     * (1.0 - settlement_proximity_decay(chunk.center));
        
        if rng.gen::<f32>() < danger * MONSTER_SPAWN_RATE {
            let candidates = monster_registry.query(params, magic_conc);
            if let Some(species) = weighted_sample_one(&candidates, rng) {
                placements.push(MonsterPlacement {
                    species_id: species,
                    position: chunk.random_point(rng),
                    tier: if rng.gen::<f32>() < 0.02 { Elite } else { Normal },
                    variant_seed: rng.gen(),
                    spawn_reason: SpawnReason::BiomeNatural,
                });
            }
        }
    }
    
    // 精英/首领: 特殊地形放置
    place_legendary_monsters(fantasy_terrains, &mut placements, rng);
    
    placements
}
```

## §五、亡灵分布（P9 后运行）

亡灵从历史事件 + 灵元素印记场推导。此阶段在 P9 产生 EventLog 后运行。

```rust
fn place_undead(
    event_log: &EventLog,
    aether_field: &AetherImprintField,   // 从 History 004 灵元素印记
    rng: &mut DeterministicRng,
) -> Vec<UndeadPlacement> {
    // 从 event_log 提取大规模死亡事件
    event_log.iter()
        .filter(|e| e.death_toll > UNDEAD_DEATH_THRESHOLD)  // >50 人死亡
        .filter(|e| years_since(e.timestamp, world_time_zero) < UNDEAD_MAX_AGE)  // <500 年
        .map(|event| {
            // 亡灵强度 = f(死亡人数, 时间深度, 灵元素浓度)
            let aether_strength = aether_field.at(event.location);
            let intensity = (event.death_toll as f32 / 1000.0)
                          * (1.0 - years_since(event.timestamp, 0) as f32 / 500.0)
                          * aether_strength;
            
            UndeadPlacement {
                undead_type: derive_undead_type(event.death_cause, aether_strength),
                position: event.location + noise_spread(rng, event.location_radius),
                estimated_count: (intensity * 50.0) as u16,
                source_event_id: event.id,
                death_cause: event.death_cause,
                years_since_death: years_since(event.timestamp, 0),
            }
        })
        .filter(|u| u.estimated_count > 0)
        .collect()
}
```

```rust
fn place_legendary_life(
    seed: u64, fantasy_terrains: &FantasyTerrainMap, rng: &mut DeterministicRng,
) -> Vec<LegendaryLife> {
    let mut legends = Vec::new();
    
    // 龙: 火山/高山 + 极低概率 (0.3% per eligible site)
    for volcano in fantasy_terrains.volcanoes() {
        if rng.gen::<f32>() < 0.003 {
            legends.push(LegendaryLife::Dragon {
                element: derive_dragon_element(volcano),
                age_category: rng.gen_range(DragonAge::Young..=DragonAge::Ancient),
                position: volcano + noise,
                is_deceased: rng.gen::<f32>() < 0.2,  // 20% 已死→留下骸骨
            });
        }
    }
    
    // 世界树: 古森林 + 高土壤肥力 + 极低概率
    // 巨兽: ...
    // 古代巫妖: 极稀有魔法地形 + 大规模历史死亡事件
    
    legends
}
```

## §七、运行时实例化策略

| 细节层 | 存储格式 | 实例化触发 | 实例化后 |
|--------|---------|-----------|---------|
| 植物 MultiMesh | 种群模板 (species+coverage) | Chunk 进入 L1 (0-100m) | MultiMeshInstance3D 批量渲染 |
| 动物个体 | 种群模板 (count+territory) | Chunk 进入 L1 | 确定性抽样 → 创建简化行为树实体 |
| 怪物 | 位置+类型 | 玩家接近 ~200m | 激活 AI |
| 亡灵 | 位置+类型+源事件 | 玩家接近 ~200m | 激活 AI |
| 传奇生命 | 位置 | 世界加载时 | 始终存在（数量极少） |
