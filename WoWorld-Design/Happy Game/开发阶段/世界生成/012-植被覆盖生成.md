## 〇、定位

**P2.25 植被覆盖** 是世界生成管线中 P2（陆地自然基底）和 P2.5（文化种子）之间的新阶段。

**为什么存在**：L6.5植被密度场在P2已隐式计算（因为Signed Heightfield需要树冠顶面烘焙远山天际线），但物种选择和群落组合缺乏正式的管线阶段。P2.25将此正式化为独立阶段，让植被从"地形的附属品"升级为"自然基底的正式层"。

**核心原则**：
- 植被是自然基底，不是"生命"——它和地形一样是画布，动物/NPC/建筑在此之上操作
- 群落从参数场推导，群系标签从群落反推——不是先生成标签再往里面塞树
- 有木本植物的Chunk → 自动推导木材可获得性 → 下游模块不需要知道植物物种

## 二、输入详解

### 2.1 BiomeParamField（P2产出）

7维连续参数场，每像素（~1m²）一个采样点：

| 参数 | 范围 | 影响 |
|------|------|------|
| `temperature_annual` | -30~45°C | 物种温度耐受筛选 |
| `temperature_seasonal_range` | 0~40°C | 常绿/落叶决策 |
| `humidity` | 0.0~1.0 | 水分需求筛选 |
| `soil_fertility` | 0.0~1.0 | 生长率、丰富度 |
| `soil_type` | enum(沙/壤/黏/砾/泥炭/火山灰) | 根系适应性 |
| `elevation_zone` | enum(低地/丘陵/山地/高山/极高) | 高程带物种筛选 |
| `water_table_depth` | 0.0~1.0(归一化) | 湿地/河岸物种 |

### 2.2 PlantSpeciesRegistry（Life crate只读）

```rust
/// P2.25只读消费——不修改物种定义
trait PlantSpeciesRegistry {
    /// 按群系参数查询候选物种
    fn query(
        &self,
        temperature: f32,
        humidity: f32,
        soil_fertility: f32,
        elevation_zone: ElevationZone,
    ) -> Vec<(SpeciesId, f32)>;  // (物种ID, 适应度评分 0-1)

    /// 获取物种完整定义（用于木材品质推导等）
    fn get(&self, id: SpeciesId) -> Option<&PlantSpecies>;
}
```

适应度评分计算（在Life crate中）：
```rust
fn fitness(species: &PlantSpecies, params: &BiomeSample) -> f32 {
    let temp_fit = gaussian(species.ecology.temp_optimum, species.ecology.temp_tolerance, params.temperature);
    let water_fit = gaussian(species.ecology.water_optimum, species.ecology.water_tolerance, params.humidity);
    let soil_fit = if species.ecology.soil_types.contains(&params.soil_type) { 1.0 } else { 0.1 };
    let light_fit = species.ecology.light_optimum; // 开阔地=全光照

    temp_fit * 0.35 + water_fit * 0.35 + soil_fit * 0.20 + light_fit * 0.10
}
```

## 三、生成算法

### Step 1: 每VMC优势种筛选

> **空间粒度**：Vegetation Macro-Chunk (VMC) = 1km × 1km。物种组成在此尺度上才真正变化。
> 渲染Terrain Chunk (TC) = 32m × 32m。运行时从父VMC确定性展开。

```rust
fn generate_vegetation_cover(
    biome_params: &BiomeParamField,
    land_mask: &LandSeaMask,
    plant_registry: &PlantSpeciesRegistry,
    fantasy: &FantasyTerrainMap,
    rng: &mut DeterministicRng,
) -> VegetationCoverMap {
    land_mask.iter_land_vmcs()  // 1km×1km网格
        .map(|vmc_coord| {
            let params = biome_params.sample(vmc_coord.center);
            let magic_conc = fantasy.magic_concentration_at(vmc_coord.center);

            // 查询候选物种
            let mut candidates = plant_registry.query(
                params.temperature_annual,
                params.humidity,
                params.soil_fertility,
                params.elevation_zone,
            );

            // 魔法群系→混入魔法植物候选
            if magic_conc > 0.4 {
                let magic_candidates = plant_registry.query_magic(magic_conc);
                candidates.extend(magic_candidates);
            }

            // 香农熵加权筛选（详见 生命/010 §3.2）
            let dominant = select_dominant_species(&candidates, &params, rng);

            (chunk_coord, dominant)
        })
        .collect()
}
```

### Step 2: 多层噪声群落密度

```
第1层: 大尺度密度噪声 (~50m 波长)
  noise_1 = perlin(hash(seed, "density", chunk_xy / 50m))
  → density_forest: 0.0(空地) ~ 1.0(密不透光)
  → 一个Chunk是"密林"还是"林间空地"还是"过渡带"

第2层: 演替阶段噪声 (~100m 波长)
  noise_2 = perlin(hash(seed, "succession", chunk_xy / 100m))
  → 成熟林 / 再生林 / 林窗
  → years_since_disturbance = f(noise_2, fantasy_terrain)

第3层: 物种混合噪声 (~30m 波长)
  每个优势种产生自己的密度子图:
    species_density[i] = perlin(hash(seed, species_i, chunk_xy / 30m))
  最终该位置的物种覆盖 = weighted_blend(species_density, dominant_weights)
```

### Step 3: 垂直分层光竞争

```
冠层树(canopy, >15m): 先放置
  │  树冠半径投射"光锥阴影"
  │  shadow(x,y) = 1 - exp(-canopy_closure × crown_radius / distance)
  ▼
林下树(understory, 5-15m): 仅在光照 > 40% 的位置放置
  │
  ▼
灌木层(shrub, 1-5m): 仅在光照 > 20% 的位置放置
  │
  ▼
地被层(herb/grass): 密度 = f(光照, 土壤湿度) — 不为个体，为密度图
```

### Step 4: Poisson盘个体放置（仅出生点周围50km²）

```rust
fn poisson_place_trees(
    community: &PlantCommunityTemplate,
    chunk_coord: ChunkCoord,
    rng: &mut DeterministicRng,
) -> Vec<PlantInstance> {
    let mut instances = Vec::new();

    // 冠层树: 间距 3-5m（从dominant_species的max_trunk_diameter推导）
    let canopy_spacing = community.dominant_species.iter()
        .map(|(id, _)| plant_registry.get(*id).unwrap().morphology.stem.max_trunk_diameter * 2.0)
        .fold(3.0, f32::max)
        .clamp(3.0, 8.0);

    let canopy_points = poisson_disk(chunk_coord, canopy_spacing, 30, rng);
    for pt in &canopy_points {
        let species = weighted_choice(&community.dominant_species, rng);
        instances.push(PlantInstance::new(species, pt, rng));
    }

    // 林下树: 间距 1.5-3m（仅光照足够处）
    // 灌木: 间距 1-2m
    // ...

    instances
}
```

> **重要**：远距离Chunk不运行Poisson盘——只存群落模板（~100B）。玩家接近时从模板+种子确定性展开。这是P2.25性能的关键优化。

### Step 5: 推导输出

```rust
impl VegetationCover {
    fn derive_outputs(&mut self, species_registry: &PlantSpeciesRegistry) {
        // 冠层郁闭度 = Σ(冠层树冠面积) / Chunk面积
        self.canopy_closure = compute_canopy_closure(&self.community, species_registry);

        // 木材可获得性 = f(有木本优势种, 冠层郁闭度, 坡度)
        self.timber = derive_timber_availability(
            &self.community, self.canopy_closure, species_registry
        );

        // 地表覆盖 = f(冠层郁闭度, 地被密度, 土壤类型)
        self.surface = derive_surface_material(
            self.canopy_closure, self.community.vertical_structure, self.soil_type
        );

        // 可燃物密度 = f(冠层密度, 地被密度, 枯立木比例)
        self.fuel_load = compute_fuel_load(&self.community);

        // 香农熵
        self.shannon_diversity = compute_shannon(&self.community);

        // L6.5密度场四通道（注入体素密度场管线）
        self.density_field = VegetationDensityField {
            canopy: rasterize_canopy(&self.community),
            understory: rasterize_understory(&self.community),
            shrub: rasterize_shrub(&self.community),
            groundcover: rasterize_groundcover(&self.surface),
        };
    }
}
```

## 五、特殊植被处理

### 5.1 魔法植物

```rust
fn place_magic_plants(
    fantasy: &FantasyTerrainMap,
    community: &mut PlantCommunityTemplate,
    params: &BiomeSample,
    rng: &mut DeterministicRng,
) {
    let magic_conc = fantasy.magic_concentration_at(community.chunk_center);

    if magic_conc > 0.4 {
        // 魔法浓度 > 0.4 → 普通植物有概率灵元素升高
        for (species_id, weight) in &mut community.dominant_species {
            if rng.gen::<f32>() < magic_conc * 0.1 {
                // 该Chunk的该物种出现魔法变异个体
                community.has_magic_variant.insert(*species_id);
            }
        }
    }

    if magic_conc > 0.7 {
        // 地脉节点 → 生成专有魔法植物
        let magic_species = plant_registry.query_magic(magic_conc);
        if let Some(ms) = weighted_choice_one(&magic_species, rng) {
            community.companion_species.push(ms);
        }
    }
}
```

### 5.3 水生植物


## 六、性能策略

### 6.1 双层空间——全量VMC模板 + 近场TC预展开

```
全量VMC（~75K, 1km×1km）:
  ✅ Step 1: 优势种筛选 → 0.75s(串行) → ~0.05s(16线程)
  ✅ Step 2: 群落模板生成 → 37.5s(串行) → ~2.5s(16线程)
  ✅ Step 5: 推导输出 → ~2s(串行) → ~0.15s(16线程)
  ❌ Step 3-4: 跳过（不生成垂直分层+Poisson个体）
  全量VMC小计: ~3s (rayon 16线程)

出生点近场预展开（~200 VMC，覆盖初始半径~10km，~195K TC）:
  ✅ Poisson盘个体放置 → ~195s(串行) → ~12s(16线程)
  ✅ L6.5密度场四通道 → ~19.5s(串行) → ~1.2s(16线程)
  近场预展开小计: ~13s

远距离VMC玩家接近时:
  → 从模板+种子确定性展开TC: 垂直分层+Poisson个体+L6.5密度场
  → 每TC ~1.5ms（后台线程，不在帧预算内）
  → 移动速度~5m/s → 每秒穿越~0.15个TC → 0.2ms/s后台负载

P2.25总计: 全量VMC模板 ~3s + 近场预展开 ~13s ≈ **~16s** (rayon 16线程)
```

```rust
/// P3木材zone从VegetationCover推导——不再使用独立噪声
fn derive_wood_zones(
    vegetation: &VegetationCoverMap,
    plant_registry: &PlantSpeciesRegistry,
) -> WoodResourceMap {
    vegetation.iter()
        .filter(|(_, cover)| cover.timber.available)
        .map(|(chunk, cover)| {
            let quality = cover.timber.quality;
            let abundance = cover.timber.abundance;
            // 木材zone = 有木本优势种的Chunk
            // 品质 = TimberQuality（已推导）
            // 丰度 = 冠层郁闭度 × 优势种木材密度
            (chunk, WoodZone { quality, abundance })
        })
        .collect()
}
```

### P2.25 → P3.5（输出——动物消费）

```rust
/// P3.5动物初级生产力从植被推导
fn primary_productivity(
    vegetation: &VegetationCover,
    species_registry: &PlantSpeciesRegistry,
) -> f32 {  // kcal/m²/year
    let canopy_biomass = vegetation.canopy_closure
        * species_registry.average_growth_rate(&vegetation.community.dominant_species);
    let understory_biomass = vegetation.community.vertical_structure.understory_cover
        * species_registry.average_growth_rate(&vegetation.community.companion_species);
    let herb_biomass = vegetation.community.vertical_structure.herb_density * 0.3;

    (canopy_biomass + understory_biomass + herb_biomass)
        * vegetation.soil_fertility
        * 365.0  // kcal/m²/yr
}
```

```rust
}
```

同一种子 → 同一群落模板 → 同一Poisson盘 → 同一个体 → 同一instance_id → T1/T3 Delta正确匹配。
