## §一、背景与动机

### 1.1 问题起源

2026-06-28 至 2026-07-01 的 6 个冲刺（Sprint 020-026）将地形渲染从 CPU tile-based mesh 生成迁移至 GPU-Driven Clipmap 架构。架构方向正确——50% 死代码清理、性能改善显著。但在随后的 bug 修复中暴露了三个架构层面未对齐的缺陷：

1. **Shader 嵌入 Rust 字符串**（`const CLIPMAP_SHADER: &str`）——改色带需重编译 Rust cdylib + 重启 Godot，违反宪法 §14.1 Godot 纯表现原则
2. **地形零光照**（`render_mode unshaded`）——昼夜循环系统已就绪，但地形完全不消费光照数据
3. **单层噪声采样**——所有高度查询直接调用 `noise.sample_height()`，无密度场层叠抽象，[[007-体素设计决策]] §1.4 规定的 11 层密度场无一实现

这三个问题不是参数调优能解决的——需要从架构层面重新对齐设计文档与代码实现。

## §二、架构设计

### 2.1 新增 trait: `DensityProvider`

**定义位置**：`woworld_core/src/density.rs`（仅 `glam` 依赖——引擎无关，满足宪法 §14.1 Rust 权威原则）

```rust
use woworld_core::prelude::WorldPos;

/// 单层密度场——P2a 层叠体系的基础单元
///
/// 各领域模块实现此 trait 注册自己的密度贡献：
/// - woworld_worldgen: TerrainBaseDensity（地形基底）、CaveDensity（洞穴）
/// - woworld_building: BuildingFoundationDensity（建筑地基切削）
/// - NPC 系统: NpcEditDensity（挖掘/踩踏修改）
/// - 玩家系统: PlayerSdfDensity（SDF 雕刻）
///
/// 正值 = 实体（solid），负值 = 空（air/water）
pub trait DensityProvider: Send + Sync {
    fn density_at(&self, pos: WorldPos) -> f32;
    fn material_at(&self, pos: WorldPos) -> u8;
    fn priority(&self) -> u8;
    fn layer_name(&self) -> &'static str { "unnamed" }
}
```

### 2.2 新增类型: `DensityStack`

**定义位置**：`woworld_worldgen/src/density.rs`

```rust
/// 有序密度层叠——按 priority 升序叠加
pub struct DensityStack {
    layers: Vec<Box<dyn DensityProvider>>,
}

impl DensityStack {
    pub fn new() -> Self { Self { layers: Vec::new() } }
    pub fn push(&mut self, layer: Box<dyn DensityProvider>) { /* 保持 priority 升序 */ }
    pub fn density_at(&self, pos: WorldPos) -> f32 {
        self.layers.iter().fold(0.0, |acc, l| acc + l.density_at(pos))
    }
}
```

### 2.3 基底层实现: `TerrainBaseDensity`

```rust
/// 地形基底密度层——P2 噪声驱动的基础地形
/// 将当前 HeightfieldTerrain 的纯函数噪声采样包装为 DensityProvider
pub struct TerrainBaseDensity {
    noise: Arc<WorldNoise>,
}

impl DensityProvider for TerrainBaseDensity {
    fn density_at(&self, pos: WorldPos) -> f32 {
        pos.y as f32 - self.noise.sample_height(pos.x, pos.z) as f32
    }
    fn material_at(&self, pos: WorldPos) -> u8 {
        // 委托给存量函数：高度 → SurfaceMaterial → u8
        material_from_height(self.noise.sample_height(pos.x, pos.z)) as u8
    }
    fn priority(&self) -> u8 { 0 }
    fn layer_name(&self) -> &'static str { "terrain_base" }
}
```

### 2.4 架构收益

- **各模块解耦**：洞穴系统只需 `impl DensityProvider for CaveDensity`，无需修改 `HeightfieldTerrain` 内部
- **引擎无关**：`DensityProvider` 仅依赖 `glam`——可在纯 Rust 测试中验证密度场组合逻辑
- **增量扩展**：`DensityStack::push()` 即插即用——未来模块上线时注册即可，不破坏现有代码
- **共享 WorldNoise**：`TerrainBaseDensity` 和 `BiomeClassifier` 共享 `Arc<WorldNoise>`——消除当前 10 个 Perlin 实例的冗余

## §三、渲染系统升级

### 3.1 Shader 架构迁移

**当前（有问题）**：shader 代码嵌在 `terrain_chunk.rs` 的 Rust `const &str` 中

**目标**：独立 `.gdshader` 文件——`woworld/godot/shaders/terrain.gdshader`

```
woworld/godot/shaders/
├── terrain.gdshader    # ★ 地形 spatial shader（本冲刺新建）
└── ocean.gdshader       # 海洋 shader（已存在）
```

Rust 侧通过 `ResourceLoader::load("res://shaders/terrain.gdshader")` 加载，然后 `ShaderMaterial::set_shader_parameter()` 设 uniform。改色带→改 `.gdshader`→Godot 热重载生效，零 Rust 编译。

### 3.2 光照：启用 Godot 标准 spatial 管线

**关键变更**：去掉 `render_mode unshaded`。加上 `render_mode cull_back`（恢复背面剔除，节省 50% 光栅化）。

`DirectionalLight3D`（已存在于场景中——当前 sun 节点）+ Godot Forward+ 渲染器自动处理漫反射、环境光、天空散射。地形颜色从 `h` 高度驱动改为材质纹理驱动（见 §3.4）。

### 3.3 梯度法线

**不在 CPU 侧计算、不在纹理中存储**。Vertex shader 中从高度图梯度推法线：

```glsl
// 世界空间梯度 → 曲面法线（微型积分——数学等价于有限差分）
float ts = hm_extent / float(textureSize(heightmap, 0).x);  // 1 texel in world meters
float h_c = texture(heightmap, uv).r;
float h_r = texture(heightmap, uv + vec2(ts/hm_extent, 0.0)).r;
float h_u = texture(heightmap, uv + vec2(0.0, ts/hm_extent)).r;
float gx = (h_r - h_c) / ts;  // ∂h/∂x
float gz = (h_u - h_c) / ts;  // ∂h/∂z
vec3 world_normal = normalize(vec3(-gx, 1.0, -gz));
NORMAL = mat3(VIEW_MATRIX) * world_normal;
```

4 次额外 `texture()` 采样，全部命中 L1 cache（当前顶点已有 1 次采样，邻域采样在缓存行内）。

### 3.4 双重纹理管线

当前 `generate_heightmap()` 仅返回 `Vec<f32>`（高度）。改为返回 `HeightmapData`（高度 + 材质色）：

```rust
pub struct HeightmapData {
    pub heights: Vec<f32>,       // hm_size × hm_size
    pub material_colors: Vec<[f32; 4]>, // hm_size × hm_size, RGBA
}
```

每像素调用 `terrain.sample_vertex()`（4 次 Perlin：高度 + 3 次有限差分法线 + `SurfaceMaterial` 分类），替代当前 `height_at()`（1-3 次 Perlin）。

**材质色表**：TOML 定义 `SurfaceMaterial → RGBA`，`woworld_worldgen` 加载并消费：

```toml
# woworld/assets/material_colors.toml
[materials.Sand]
color = [0.76, 0.70, 0.50, 1.0]
[materials.Grass]
color = [0.42, 0.68, 0.30, 1.0]
# ... 全部 21 变体
```

**GPU 纹理**：每 LOD 层两张纹理（R32F 高度 + RGBA8 材质色），双缓冲交换——创建新 `ImageTexture` 后原子替换 shader 参数，不修改 GPU 正采样的纹理，零管线停顿。

### 3.5 LOD 消缝

**根因**：两个来源——① 邻层高度图在世界空间同一位置的纹理采样可能差 1-2mm（分辨率不同导致双线性插值差异）② 环形网格顶点不完全对齐（T-junction）。

**方案：裙边 + 过渡带混合**

**裙边**（安全网——已实现于 `generate_clipmap_grid()`）：边界处生成 y=-400 垂直几何，`VERTEX.y < -300` 跳过 VTF。Shader 中裙边顶点颜色 = 相邻地表顶点同色（采样同一材质纹理）。

**过渡带混合**（高度一致化——本冲刺新增）：每层外边界 2×spacing 宽的环内顶点设置 `blend_factor`（通过 vertex attribute `CUSTOM0.x` 传入）：
- 最外圈 `blend_factor = 1.0`：高度完全采样邻层高度图
- 向内递减：`0.5` → `0.0`

Shader 侧：
```glsl
float h = texture(heightmap, uv).r;
if (blend_factor > 0.001) {
    float h_neighbor = texture(neighbor_heightmap, uv).r;
    h = mix(h, h_neighbor, blend_factor);
}
world_pos.y = h;
```

**为什么这套组合是最优解**：
| 条件 | 裙边 | 过渡混合 |
|------|------|---------|
| 地面行走 | 被地表几何遮挡——不可见 | 表面无缝——肉眼无 LOD 边界 |
| 高空俯瞰 | 远层裙边被大气雾化掩盖 | 混合区在 >500m 处——不可分辨 |
| 侧面视角 | 裙边面可见——但着色与邻层同色 | 高度连续——无台阶 |

### 3.6 保留：camera-relative fp32 精度修复

当前 vertex shader 中的 `mat3(VIEW_MATRIX) * (world_pos - camera_pos)` 保留不动。这是 Sprint 025 的核心成果——GPU 中间坐标 <500m，fp32 精度 ~0.05mm，消除远距离裂缝。

## §四、明确的偏离与后续计划

### 4.1 Transvoxel 延后

[[007-体素设计决策]] §1.2 规定 LOD 0-5 使用 Transvoxel 3D 等值面提取。当前全部 LOD 使用 Signed Heightfield（2D 网格 + VTF 位移）。

**原因**：Transvoxel 需要 3D 密度场（每体素 `density: u16` + `material_id: u8`）→ 等值面提取 → 三角形输出。当前仅有高度场（单值 f(x,z)→h），无 3D 密度场。**本冲刺的目标正是构建密度场地基**——`DensityProvider` trait + `DensityStack` + `TerrainBaseDensity` 基底层。在此地基上，洞穴层（Worley 3D）、矿脉层（Poisson 体）才能叠加，3D 密度场才有意义。

**计划**：P2a 就位后，LOD 0-3 迁移至 Transvoxel（见 [[007-体素设计决策]] §1.2 目标架构表）。LOD 4-7 永久保留 Signed Heightfield（远距离 3D 等值面无视觉益处）。

### 4.2 群系参数场延后至 P2c

[[002-自然景观]] 要求 7 维连续群系参数场。当前仅 2 维（温度×降水硬盒分类）。本冲刺的双重纹理管线使用 `BiomeClassifier` 现有输出产出 `SurfaceMaterial`——这是正确的数据流。7 维场将在 P2c 冲刺中实现，届时 `sample_vertex()` 的内部逻辑升级，`HeightmapData` 和 shader 纹理管线无需改变。

### 4.3 单层密度→11 层

当前仅实现 1 层（`TerrainBaseDensity`）。`DensityStack` 的扩展接口已就位——其余 10 层依赖其他模块实现（洞穴→`woworld_worldgen` 内部 Worley 噪声、矿脉→资源系统、建筑→建筑模块、NPC→NPC 编辑系统、玩家→SDF 编辑系统）。
