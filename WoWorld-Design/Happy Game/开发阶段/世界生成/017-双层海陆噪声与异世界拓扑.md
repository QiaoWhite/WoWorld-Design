# 017 — 双层海陆噪声与异世界拓扑

> **开发代号**: WoWorld (Wonder World)
> **引擎**: Godot 4.7
> **日期**: 2026-07-10
> **状态**: 设计完成
> **定位**: 定义双层噪声 AND 规则，涌现异世界群岛地理。替代旧单层 continent 海陆判定。
> **关联**: [[018-LOD层级40km扩展]] · [[../../Change/CHG-068-视距扩展与海洋升级-20260710|CHG-068]] · [[001-世界生成总流程|001 总流程]]
> **前身**: 旧单层 continent 噪声海陆判定（`continent_val > sea_threshold`）

---

## 一、动机

旧系统用单层 continent 噪声（100km 波长）+ 单阈值 `sea_threshold` 决定海陆。这在英国尺度（~500km）工作良好——一辈子周期覆盖世界 5 次，产生合理的陆地分布。

但世界尺度从 500km 扩展到 3200km 后，同样的 100km 波长覆盖世界 32 次。结果：陆地过于细碎，没有"汪洋连接几块大陆"的壮阔感——而是数千个小岛均匀散布。

**解法**：加一个更宏观的噪声层（~3333km 波长），两层 AND 判定。这是涌现式设计——不手写"这里有一个大陆"，只定义简单规则，让地理从噪声交互中自然出生。

## 二、两层 AND 规则

```
super_continent 噪声 (~3333km 波长, 全世界约 1 个周期)
        ×
region 噪声 (~333km 波长, 全世界约 10 个周期, 原名 continent)
        ↓
   两者都超过各自阈值 → 陆地
   任一低于阈值 → 海洋
```

```rust
// sample_height() 中的海陆判定（伪代码）
let is_land = super_continent_val > SC_THRESH   // 0.38
           && region_val > REG_THRESH;           // 0.35

if is_land {
    // 陆地高度 = base + detail + mountain
} else {
    // 海洋深度 = weighted_deficit * sea_depth
}
```

## 三、涌现地理类型

两层 Perlin 噪声各有独立 seed，取值独立。交集产生的地理类型完全涌现：

| super_continent | region | 结果 | 异世界角色 |
|-----------------|--------|------|-----------|
| 高值 (>0.6) | 高值 (>0.6) | **大陆腹地** | 王国、帝国、内陆文明 |
| 高值 | 中值 (0.3-0.6) | **沿海/半岛** | 港口城市、海岸聚落 |
| 高值 | 低值 (<0.3) | **大陆内海/大湖** | 地中海式的陆间海 |
| 中值 (0.4-0.6) | 高值 | **大岛** | 独立岛国、大岛屿 |
| 中值 | 中值 | **群岛链** | 跳岛航线、贸易路线 |
| 低值 (<0.4) | 高值 | **远洋孤岛** | 秘境、Boss 巢穴、隐藏岛屿 |
| 低值 | 中值 | **小群岛/礁石** | 补给点、渔场、航海标记 |
| 低值 | 低值 | **汪洋** | 远洋航线、海怪领域 |

super_continent 层每 ~1666km 从高值翻转到低值——所以世界约有 2-3 个"大陆群"（super_continent 正值区），每个大陆群内部被 region 层雕刻成大陆主体 + 群岛 + 内海。

## 四、参数

```toml
# noise_params.toml — 海陆判定部分

super_continent_scale = 0.0000003   # ~3333km 波长（3200km 世界 ≈ 1 周期）
super_continent_threshold = 0.38    # 上层阈值

region_scale = 0.000003             # ~333km 波长（原 continent_scale）
region_threshold = 0.35             # 下层阈值

# 旧 sea_threshold 字段 — 已删除
```

海:陆 = (1 - 0.38) × (1 - 0.35) ≈ **40% 陆地，60% 海洋**。

> **调参指南**：
> - 增加 `super_continent_threshold` → 更少/更小的大陆群，更多汪洋
> - 增加 `region_threshold` → 大陆边缘更光滑，更少小岛和半岛
> - 两者同比例降低 → 更多陆地（如 0.30/0.28 → 海:陆 ≈ 5:5）

## 五、海深计算

旧系统：`sea_factor = (sea_threshold - continent_val) / (sea_threshold + 1.0)`。

新系统没有 `sea_threshold`，改用两个层各自低于阈值的程度——**加权平均 deficit**：

```rust
const SC_THRESH: f64 = 0.38;
const REG_THRESH: f64 = 0.35;

let sc_deficit = (SC_THRESH - super_continent_val).max(0.0);
let rc_deficit = (REG_THRESH - region_val).max(0.0);
let sea_factor = sc_deficit * 0.5 + rc_deficit * 0.5;

let ocean_depth = sea_factor.powf(0.4) * params.sea_depth;  // sea_depth = 400.0m
```

- 近岸（一层刚低于阈值）→ deficit 小 → 浅海
- 大陆架（super 高但 region 低，或反之）→ deficit 中等 → 中等深度
- 远洋（两层都远低于阈值）→ deficit 大 → 深海（最深 ~400m）

> 旧的海床细节噪声（`ocean_detail`）保留——叠加在 deficit 深度上，使海床有自然的起伏纹理。

## 六、噪声层完整表

| 层 | 波长 | 用途 | 消费方 |
|----|------|------|--------|
| super_continent | ~3,333km | 板块尺度大陆群布局 | sample_height(), sample_height_coarse() |
| region | ~333km | 区域内大陆/岛屿/半岛 | sample_height(), sample_height_coarse() |
| detail | ~100m | 地形起伏 | sample_height() only |
| mountain | ~500m | 山脊突出 | sample_height() only |
| temperature | ~2km | 温度场（群系判定） | sample_temperature() |
| precipitation | ~2km | 降水场（群系判定） | sample_precipitation() |

所有 6 层共享同一 master seed 派生的独立 Perlin seed。`sample_height_coarse()` 仅用前 2 层——供 L8/L9 远景 heightmap 使用，省 60% 采样时间。

## 九、涌现特性

从简单的两层 Perlin AND 规则中涌现的 gameplay 相关特性：

- **航海路线**：大陆之间的汪洋 + 群岛链自然形成航线——NPC 商船沿岛链跳岛
- **探索动机**：远洋孤岛（super 低、region 高）成为天然秘境——隐藏的 dungeons、特殊资源
- **地缘政治**：大陆腹地文明 vs 沿海城邦 vs 岛国——文化/经济/军事策略从地理中涌现
- **天气系统**：大陆尺度影响洋流和风带（远期——气候模拟）
- **世界边界**：汪洋延伸至世界边缘 → 风暴带软边界（见 [[009-世界边界与无限延伸]]，需适配 3200km 尺度）
