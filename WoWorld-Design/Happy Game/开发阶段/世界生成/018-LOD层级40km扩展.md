# 018 — LOD 层级 40km 扩展

> **开发代号**: WoWorld (Wonder World)
> **引擎**: Godot 4.7
> **日期**: 2026-07-10
> **状态**: 设计完成
> **定位**: 将 GPU-Driven Clipmap LOD 从 8 层（15km）扩展到 10 层（40km），包括远景粗采样、距离雾、远层简化。
> **关联**: [[017-双层海陆噪声与异世界拓扑|017 噪声]] · [[../../Change/CHG-068-视距扩展与海洋升级-20260710|CHG-068]] · [[015-P2a密度场架构与渲染升级方案-20260701|015 密度场]]

## 一、动机

当前 LOD 7 (10-15km, SH 64m 间距) 是最外层。Camera far plane 20km，海面 22km。LOD 7 以远只有天空和海面——地形在 15km 处被截断。

目标：扩展到 40km，新增 L8 (20-30km) 和 L9 (30-40km)。默认视距 20km（L7 最外层），极限 40km（L9 最外层）。

## 二、新 LOD 层级表

| 层级 | 距离范围 | 算法 | 间距 | Ring 顶点估算 | 说明 |
|------|---------|------|------|-------------|------|
| L0 | 0-30m | Transvoxel | 0.5m | — | VoxelChunk 5×5 grid |
| L1 | 30-80m | Transvoxel | 1m | ~5K | |
| L2 | 80-200m | Transvoxel | 2m | ~30K | |
| L3 | 200-500m | Transvoxel | 4m | ~45K | |
| L4 | 500m-1.5km | Transvoxel | 8m | ~100K | |
| L5 | 1.5-4km | Signed Heightfield | 16m | ~180K | |
| L6 | 4-10km | Signed Heightfield | 32m | ~329K | |
| L7 | 10-20km | Signed Heightfield | 64m | ~293K | 原 L7 (10-15km) 拉伸到 20km |
| **L8** | **20-30km** | **Signed Heightfield** | **128m** | **~122K** | **新增** |
| **L9** | **30-40km** | **Signed Heightfield** | **256m** | **~43K** | **新增** |

总计 clipmap ring 顶点约 1,147K，VRAM ~37 MB。Heightmap 纹理（9 层 × 512² × 8 字节/像素）~18 MB。合计 terrain ~55 MB——GTX 1660 SUPER 6GB 的 0.9%。

> **L7 拉伸说明**：原 L7 覆盖 10-15km（max_range=15000），现拉伸到 10-20km（max_range=20000）。mesh 顶点数增加（从 ~200K 到 ~293K），但仍在可接受范围。这是为了让 L8 从 20km 开始——20km 是一个自然的分界（默认视距上限）。

## 三、远景粗采样

L8 (128m 间距) 和 L9 (256m 间距) 的间距远超 detail 噪声（100m 波长）和 mountain 噪声（500m 波长）的 Nyquist 限——这些高频细节在如此粗的网格上完全不可见，采样也白采。

**方案**：L8/L9 调用 `WorldNoise::sample_height_coarse()`——仅消费 super_continent + region 两层（见 [[017-双层海陆噪声与异世界拓扑|017]]）。跳过 detail 和 mountain Perlin 求值。每像素省 2 次 Perlin + 条件分支，~60% 采样时间缩减。

**异步传递链**：

```
WorldDriver::process()
  → 检测 level.index >= 8
  → HeightmapJob { use_coarse: true, ... }
  → rayon::spawn
  → generate_heightmap_data(use_coarse=true)
  → HeightfieldTerrain::sample_vertex_coarse()
  → WorldNoise::sample_height_coarse()
```

`HeightmapJob` 结构体新增 `use_coarse: bool` 字段。`HeightfieldTerrain` 新增 `sample_vertex_coarse()` 方法。

**性能**：`layer_tex_config()` 的通配分支 `_ => (512u32, 512.0 * spacing)` 自动覆盖 L8/L9。L8 margin = 7.7km，L9 margin = 25.3km。即使以 500 km/h 飞行，L9 有 3 分钟穿越 margin——9 帧最坏延迟 = 0.15s，零 pop-in 风险。

## 四、LEVELS 常量化

当前 `terrain_chunk.rs` 和 `lod.rs` 中有大量硬编码的 `8`、`7`、`.min(7)`。全部改为从 `LEVELS.len()` 派生：

```rust
// clipmap.rs
pub const LEVELS: [LodLevel; 10] = [ /* ... */ ];
pub const NUM_LOD_LEVELS: usize = LEVELS.len();  // = 10
pub const MAX_SCENE_LOD: u8 = (NUM_LOD_LEVELS - 1) as u8;  // = 9
```

**`terrain_chunk.rs` 改动表**（10 处精确位置）：

| 行号区域 | 当前 | 改为 |
|---------|------|------|
| lod_layers 声明 | `[Option<LodLayer>; 8]` | `[Option<LodLayer>; NUM_LOD_LEVELS]` |
| 创建循环 | `1..8u8` | `1..NUM_LOD_LEVELS as u8` |
| coarse 绑定循环 | `1..7u8` | `1..(NUM_LOD_LEVELS-1) as u8` |
| process 循环 | `0..8u8` | `0..NUM_LOD_LEVELS as u8` |
| 最外层引用 | `self.lod_layers[7]` | `self.lod_layers[NUM_LOD_LEVELS-1]` |
| fine 同步循环 | `1..8u8` | `1..NUM_LOD_LEVELS as u8` |
| coarse 同步循环 | `0..7u8` | `0..(NUM_LOD_LEVELS-1) as u8` |
| cull margin | `15000.0` | `level.max_range as f32 * 1.2` |
| L7 自身 coarse | `self.lod_layers[7]` | `self.lod_layers[NUM_LOD_LEVELS-1]` |
| neighbor 引用循环 | `0..7u8` | `0..(NUM_LOD_LEVELS-1) as u8` |

**`lod.rs` 改动**：

- `distance_to_scene_lod()`：追加 L8(20-30km)、L9(30-40km) 距离带
- `clamp_all()`：`self.scene_lod.min(7)` → `.min(MAX_SCENE_LOD)`
- VRAM 压力降级：4 处 `.min(7)` → `.min(MAX_SCENE_LOD)`
- `test_scene_lod_boundaries`：追加 L8/L9 边界测试用例

## 五、交叉采样 Morphing

当前 terrain shader 已经实现了 CDLOD 风格的 fine/coarse smoothstep blend（`terrain.gdshader` 的 `t_inner`/`t_outer` 逻辑）。L7→L8→L9 的过渡**零新 GLSL 代码**——仅需正确配置 shader uniform：

```
L7 的 outer_bound: 99999 →  20000 (L8 的 min_range)
L7 的 coarse_heightmap → L8 的 heightmap_tex
L8 的 outer_bound → 30000 (L9 的 min_range)
L8 的 coarse_heightmap → L9 的 heightmap_tex
L9 的 outer_bound = 99999 (最外层, 无更粗引用)
```

在 `terrain_chunk.rs` 的 coarse 绑定循环中，扩展 `1..7u8` → `1..(NUM_LOD_LEVELS-1) as u8` 即可自动覆盖。

## 六、隐藏层优化

`process()` 中，视距以外的层跳过 heightmap 更新并隐藏 mesh：

```rust
if LEVELS[level_idx].min_range > view_distance.meters() {
    layer.instance.set_visible(false);
    continue;  // 不提交 heightmap job
}
layer.instance.set_visible(true);
```

L8/L9 mesh 在启动时创建——额外 165K 顶点，创建成本可忽略。Heightmap 懒生成：首次切换到 40km 时才触发 L8/L9 的 heightmap job。

## 七、大气：距离指数雾

`terrain.gdshader` 新增：

```glsl
uniform vec3 fog_color = vec3(0.6, 0.7, 0.8);
uniform float fog_density = 0.00003;
varying float frag_dist;

// vertex:  frag_dist = length(world_pos - camera_pos);
// fragment:
float fog = 1.0 - exp(-frag_dist * fog_density);
ALBEDO = mix(ALBEDO, fog_color, fog);
```

`fog_color` = `ResolvedAtmosphere.sky_horizon`（已有 17 参数之一），与天空色一致。`fog_density` = `1.2 / view_distance_meters`（视距越大雾越稀薄）。

L8/L9 额外设置 `is_distant_layer = true`，跳过 per-fragment normal 从 heightmap 的采样（省 3 次 texture sample），直接用 `(0,1,0)` 法线——256m 间距下 normal 细节完全不可见。

## 八、LODCoordinator 适配

`distance_to_scene_lod()` 的新距离带自动生效——LODCoordinator 的 8 步算法不需要改动。VRAM 压力降级中的 `.min(7)` → `.min(MAX_SCENE_LOD)` 确保 L8/L9 也能被降级。

## 九、性能预算

| 指标 | 20km 默认 | 40km 极限 | 说明 |
|------|----------|----------|------|

| Ring 顶点 | ~990K | ~1,147K | +157K verts (+16%) |
| Heightmap 纹理 | 7 × 512² | 9 × 512² | +2 纹理 (~4 MB) |
| Heightmap 更新 | 正常 | 正常 | 粗采样比细采样更快 |
