## 一、ReligiousPracticeProfile——实践画像

### 1.1 数据结构（详见 001 §2.2）

每个 NPC 存储一个 `ReligiousPracticeProfile`（~40 bytes/NPC）。**这不是"信仰立场"——这是"实践记录"**。大多数 NPC 没有神学立场——他们有参与实践（去不去庙、拜不拜神、捐不捐香火）。神学标签从中派生，不驱动实践。

### 1.2 字段详解

#### participation

`Vec<(FaithId, f32)>`——NPC 参与了哪些信仰的实践活动及参与强度。

- 权重范围：0.01（偶尔随家人去一次庙）到 1.0（每天去、捐钱、当义工）
- 多数 NPC：1-3 个条目。见神就拜型可能 3-5 个条目但每个权重都不高。
- 单一信仰深度参与只在 `exclusivity > 0.7` 且 `theological_depth > 0.5` 时自然出现。
- 空 vec → 不参与任何宗教实践（NonPracticing/AntiReligious/Indifferent）。
- 条目不可删除——即使参与降到极低也保留 0.001 的微量（"小时候去过，现在可能还会偶尔路过时进去坐坐"）。降至 <0.01 时视为"不活跃"但不从 vec 中移除以保留历史痕迹。

#### theological_depth

`f32`——NPC 对信仰的神学理解深度。

- 从 theology 技能 + intelligence + 受教育程度（技能系统 linguistics/academic 水平）派生。
- 0.0 = 纯民俗（"我奶奶这么做的"）
- 0.5 = 能解释基本教义（"我们相信轮回报应——做善事有好报"）
- 1.0 = 神学家（能论证抽象的教义命题）
- 初始值在 P8 NPC 生成时分配：`base = intelligence * 0.3 + education * 0.3 + theology_skill * 0.4`，加种子随机抖动 ±0.1。
- 随 theology 技能升级而增长（通过技能系统的 Teaching/DirectInfusion 路径）。

#### motivation

`ReligiousMotivation`——NPC 参与宗教实践的驱动因素。

- 绝大多数 NPC 出厂时 = `Habitual` 或 `SocialCustom`。
- 动机可以随生活事件转变（见 §三）。
- `CrisisDriven` 是临时状态——危机过去后，NPC 可能回到原动机（"神啊救救我！……得救了啊，哦那算了"），也可能转化为 `PersonalDevotion`（罕见："他真的应允了我的祈祷——我从此虔诚"）。

## 二、Prayer——祈祷

### 2.1 祈祷作为行为

祈祷是 `ActionCategory::Pray`（NPC 模块决策引擎新增）——由 NPC 的 GOAP/概率行为树触发。

```rust
/// NPC 决策引擎触发祈祷的条件——信仰系统提供判断函数。
fn should_pray(profile: &ReligiousPracticeProfile, spiritual_need: f32, context: &Context) -> bool {
    // 条件 1：精神需求累积
    if spiritual_need > 0.3 { return true; }
    
    // 条件 2：高虔诚度 + 今天是神圣日
    if profile.participation.iter().any(|(fid, w)| *w > 0.7 && is_holy_day_today(*fid)) {
        return true;
    }
    
    // 条件 3：危机后自发祈祷
    if context.recent_crisis_event() && profile.motivation != ReligiousMotivation::AntiReligious {
        return true;
    }
    
    false
}
```

### 2.2 祈祷地点选择

NPC 选择祈祷地点遵循偏好层级（从信仰系统的 `prayer_location_preference()` 返回）：

1. 神殿/寺庙（该信仰的 sacred_site 或神殿建筑内）——首选
2. 聚落圣龛/路边祠堂（开放空间的神圣标记物）——次选
3. 家中圣坛——如果 NPC 有自定义物品标记为"家庭圣坛"
4. 自然圣地（高 nature_sacredness 信仰特有——圣树/圣石/圣泉）——野外可选
5. 任意安静角落——最低选项

地点选择受距离（太远不走）、可达性（锁门的不去）、和当前情境（战斗中不祈祷）约束。

### 2.3 DivinePresence 调用

```rust
/// 信仰系统调用——模式透明。无论 A/B/C 模式，接口完全相同。
fn perform_prayer(
    npc: &dyn FaithAgent,
    prayer_type: PrayerType,
    faith_id: FaithId,
    divine: &dyn DivinePresence,
    rng: &mut StdRng,
) -> PrayerOutcome {
    let piety = npc.practice_profile().participation.iter()
        .find(|(fid, _)| *fid == faith_id)
        .map(|(_, w)| *w)
        .unwrap_or(0.0);

    let outcome = divine.respond_to_prayer(npc.as_npc_ref(), prayer_type, piety, rng);
    
    // A 模式：outcome 可能是 MinorBlessing / MajorMiracle / DirectCommunication
    // B 模式：永远返回 NoResponse
    // C 模式：低概率返回 Coincidence
    // 信仰系统不关心具体返回值——只负责传递结果给 NPC 记忆系统
    
    outcome
}
```

### 2.4 祈祷结果对 NPC 的影响

| PrayerOutcome | participation 变化 | motivation 影响 | 记忆编码 |
|--------------|-------------------|----------------|---------|
| `NoResponse`（长期） | 缓慢衰减 -0.001/次 | 可能导致 Habitual → SocialCompelled | "神从未回应"——长期不回应触发信仰危机检测 |
| `Coincidence(String)` | +0.005-0.02 | 神经质高→解读为神意，可能→PersonalDevotion | "那天我刚祈祷完就下雨了" |
| `MinorBlessing(Effect)` | +0.02-0.08 | 强化当前动机 | "神恩临身"——温暖/平静的感觉 |
| `MajorMiracle(Miracle)` | +0.1-0.3 | 大幅强化→PersonalDevotion | 强烈宗教体验——终生难忘 |
| `DirectCommunication(String)` | +0.15-0.5 | 几乎必定→PersonalDevotion | "神亲自对我说话"——改变人生的记忆 |

## 三、Participation 权重动力学

### 3.1 自然衰减

```rust
/// 每游戏年的自然衰减——信仰不是消费品，衰减极慢。
/// 200 年无参与 → participation 从 0.5 降到 ~0.18。
const ANNUAL_DECAY_RATE: f32 = 0.005;
```

### 3.2 参与强化

参与宗教活动（节日/仪式/祈祷）→ participation 增加：

| 活动 | participation 增量 |
|------|-------------------|
| 个人祈祷 | +0.002 |
| 参加神圣日仪式 | +0.01 |
| 参加年度盛大节日 | +0.05 |
| 朝圣完成 | +0.10（一次性，上限重置） |
| 目睹神迹（A 模式） | +0.1-0.3 |

### 3.3 生活事件驱动的骤变

| 事件 | participation 变化 | 原因 |
|------|-------------------|------|
| 被神职背叛 | -0.1-0.3 | 信任崩塌 |
| 见证神学矛盾 | -0.05-0.15 | 认知失调 |
| 跨信仰婚姻 | 配偶信仰的 participation +0.1-0.3 | 社交传染 |
| 濒死体验 | 可能向任一方向（种子随机 × 人格修正） | 危机重塑信仰 |
| 子女出生 | 若 motivation=IdentityBased → +0.05 | 延续家族信仰的责任感 |

### 3.4 高 exclusivity 信仰的排他效应

当 NPC 参与一个 exclusivity > 0.7 的信仰，且 theological_depth > 0.5（真正理解了教义）：
- 其他信仰的 participation 权重每游戏年衰减 0.02（比自然衰减快 4 倍）
- 约 50 年内其他信仰退出"活跃"范围
- 不强制清零——历史上的多信仰痕迹保留在 profile 中（"少年时我也去过那个庙……"）

## 四、SpiritualNeed——精神需求

### 4.1 定义

对标 Physiology 中的 hunger/thirst——NPC 有一个精神维度的需求：

```rust
/// NPC 模块 Physiology 新增字段。
/// 由信仰系统提供增长/满足公式。NPC 模块执行更新。
pub spiritual_need: f32,  // 0.0 = 满足, 1.0 = 极度精神饥渴
```

### 4.2 增长公式

```rust
/// 每游戏日自动增长。
/// 越虔诚 → 越需要宗教实践来维持——"食粮"需求随信仰深度而增加。
fn daily_spiritual_decay(profile: &ReligiousPracticeProfile) -> f32 {
    let total_participation: f32 = profile.participation.iter().map(|(_, w)| w).sum().min(1.0);
    // 不参与 → 无需求（精神需求为 0）
    // 深度参与 → 增长快（不祈祷就难受）
    0.03 * total_participation.clamp(0.1, 1.0)
}
```

### 4.3 满足公式

| 行为 | spiritual_need 降低 |
|------|-------------------|
| 个人祈祷 | -0.15（每日最多一次有效减免） |
| 参加仪式/聚会 | -0.30 |
| 读经/禅修 | -0.10（每日可多次，递减——第二次 -0.05，第三次 -0.03） |
| 朝圣完成 | -0.50（上限重置——朝圣后长期满足） |

### 4.4 阈值效果

```rust
fn spiritual_need_effects(need: f32, mood: &mut f32, triggers_faith_crisis: &mut bool) {
    if need > 0.6 { *mood -= 0.1; }        // 轻度的精神空虚感
    if need > 0.8 { *mood -= 0.2; }        // 明显的精神饥渴
    if need > 0.9 { *triggers_faith_crisis = true; }  // 触发信仰危机检测
}
```

## 五、Motivation 转变

### 5.1 转变条件

```rust
/// 信仰系统定义——NPC 模块在每游戏日的 NPC 更新中调用。
fn check_motivation_transition(profile: &mut ReligiousPracticeProfile, events: &[LifeEvent]) {
    match profile.motivation {
        ReligiousMotivation::Habitual => {
            // 习惯型 → 个人虔诚：经历了重大神迹/宗教体验
            // 习惯型 → 社会强制：长期不参与导致 social pressure 累积
            // 习惯型 → 反宗教：被宗教机构/神职严重伤害
        }
        ReligiousMotivation::CrisisDriven { since, intensity } => {
            // 危机过去后（30 天后检测）：
            //   参与度下降了 → 回到原动机（"神啊救救我……哦得救了——算了"）
            //   参与度保持且强化 → PersonalDevotion（罕见："他真的应允了我的祈祷"）
        }
        ReligiousMotivation::SocialCompelled => {
            // 社会强制 → 习惯：离开强制环境（搬家到世俗城市）→ 可能转为 Habitual 或 NonPracticing
        }
        // ... 其他转变路径
    }
}
```

### 5.2 转变矩阵

| 从 \ 到 | Habitual | SocialCustom | PersonalDevotion | CrisisDriven | Pragmatic | SpiritualSeeker | AntiReligious |
|---------|----------|-------------|-----------------|-------------|-----------|----------------|---------------|
| **Habitual** | — | 社区压力↑ | 神迹/宗教体验 | 危机事件 | 接触多信仰 | theology 学习 | 被宗教机构伤害 |
| **SocialCustom** | 搬离 | — | 神迹 | 危机事件 | 贸易城市 | theology 学习 | 被宗教机构伤害 |
| **CrisisDriven** | 危机后冷却 | 危机后冷却 | 祈祷"应验" | — | 危机后冷却 | 危机引发追寻 | — |
| **PersonalDevotion** | — | — | — | 新危机 | — | theology 深化 | 神学矛盾/背叛 |
| **Pragmatic** | 长期定居一地 | 社区压力 | 祈祷"应验" | 危机事件 | — | theology 学习 | — |

## 六、改信过程

### 6.1 不设"改信"二进制事件

NPC 的 participation 权重随时间自然漂移——一个信仰的权重降到 <0.01 时自然"脱离"，一个新信仰的权重上升到最高时自然"归属"。不存在"我现在正式宣布改信X教！"的仪式性事件（除非信仰本身的 ritual 定义了改信仪式）。

### 6.2 加速因素

| 因素 | 旧信仰衰减加速 | 新信仰增长加速 |
|------|-------------|-------------|
| 社交网络中多数人信另一宗教 | 1.5× | 2.0× |
| 婚姻到另一信仰家庭 | 2.0× | 3.0× |
| 神职人员/传教士主动传教 | 1.5× | 3.0× |
| 征服者强制改信（Compel 行为） | 5.0× | 5.0×（但抵抗 ∝ faith_as_identity × old_participation） |
| 见证神迹（A 模式） | — | 巨大——直接 +0.1-0.3 |

### 6.3 信仰危机检测

```rust
/// 每游戏年检测一次。
fn check_faith_crisis(profile: &ReligiousPracticeProfile, memory: &[EventMemory]) -> Option<FaithCrisis> {
    let active = active_participations(profile);
    if active.is_empty() { return None; }
    
    for (faith_id, weight) in &active {
        // 条件 1：长期祈祷无回应（B/C 模式）——participation > 0.5 但 spiritual_need 持续 > 0.8
        // 条件 2：神学矛盾——两个同时参与的信仰有冲突的教义
        // 条件 3：道德创伤——该信仰的神职做了与该信仰教义严重冲突的事
        // 条件 4：深度理解后的觉醒——theological_depth 提升后意识到"我信的东西说不通"
        
        if crisis_condition_met {
            return Some(FaithCrisis {
                faith_id: *faith_id,
                crisis_type,
                severity,
            });
        }
    }
    None
}
```

信仰危机的结果（种子随机 × 人格修正）：
- 30%：participation 下降 0.2-0.5（"也许我不那么确定了"）
- 25%：转向 SpiritualSeeker 动机（"我要去寻找真理"）
- 20%：转向另一个已有低参与的信仰
- 15%：保留信仰但动机变为 SocialCompelled（"我不信了但还是得去——面子"）
- 10%：信仰反而强化（"考验——我应该更坚定"）——最常见的危机应对

## 七、信仰情感调制与记忆显著性

### 7.1 信仰对情感的调制

```rust
/// 信仰系统提供映射表。NPC 情感系统在计算情感时调用。
pub fn faith_emotion_modulation(
    profile: &ReligiousPracticeProfile,
    base_emotion: EmotionType,
    event: &WorldEvent,
    divine_mode: DivineMode,
) -> Option<(EmotionType, f32)> {
    // 仅对有一定参与度的 NPC 生效
    let total_participation: f32 = profile.participation.iter().map(|(_, w)| w).sum();
    if total_participation < 0.2 { return None; }

    match base_emotion {
        EmotionType::Fear => {
            // 信仰缓冲恐惧——"神与我同在"
            let reduction = total_participation * 0.2;
            Some((EmotionType::Fear, -reduction))
        }
        EmotionType::Guilt => {
            // 如果是宗教违规事件→罪咎感放大
            if event.is_religious_violation() {
                let amplification = total_participation * 0.3;
                Some((EmotionType::Guilt， amplification))
            } else { None }
        }
        EmotionType::Awe => {
            // 宗教体验→敬畏感放大
            if event.is_religious_experience() {
                let amplification = 0.3 + total_participation * 0.2;
                Some((EmotionType::Awe， amplification))
            } else { None }
        }
        EmotionType::Grief => {
            // 信仰缓冲丧亲之痛——"他去了更好的地方"（如果教义包含正面 afterlife）
            // 具体系数取决于该 NPC 主导信仰的 afterlife 教义
            let comfort = total_participation * 0.1;
            Some((EmotionType::Grief， -comfort))
        }
        _ => None,
    }
}
```

### 7.2 信仰对记忆显著性的放大

```rust
/// 信仰系统提供。NPC 记忆系统在 calculate_impact() 中调用。
pub fn faith_memory_amplifier(
    profile: &ReligiousPracticeProfile,
    event_type: &EventType,
) -> f32 {
    let total_participation: f32 = profile.participation.iter().map(|(_, w)| w).sum();
    if total_participation < 0.1 { return 0.0; }

    match event_type {
        // 宗教仪式/节日——高参与 NPC 记忆更深
        EventType::Ceremony if is_religious_ceremony(event_type) => total_participation * 0.25,
        // 神迹/宗教体验——极深记忆
        EventType::Revelation if is_religious_revelation(event_type) => total_participation * 0.40,
        // 道德违规——信仰提供道德框架，强化记忆
        EventType::MoralViolation { .. } => {
            let orthodoxy_weight = dominant_theology(profile).map(|t| t.orthodoxy_vs_orthopraxy).unwrap_or(0.5);
            total_participation * orthodoxy_weight * 0.20
        }
        // 一般事件——轻微放大
        _ => total_participation * 0.05,
    }
}
```

## 八、信仰继承——子女的 ReligiousPracticeProfile

生命系统 012（繁衍）在 NPC 出生时调用信仰系统：

```rust
/// 信仰系统提供——由生命系统 012 在 NPC 出生时调用。
pub fn inherit_faith_profile(
    parents: &[(ReligiousPracticeProfile, f32)],  // (parent_profile, care_time_weight)
    community_faiths: &[(FaithId, f32)],          // 社区信仰分布——次要影响
    rng: &mut StdRng,
) -> ReligiousPracticeProfile {
    // 1. 父母信仰混合
    let mut participation: Vec<(FaithId, f32)> = Vec::new();
    let mut weight_map: HashMap<FaithId, f32> = HashMap::new();
    
    for (profile, care_weight) in parents {
        for (fid, p_weight) in profile.participation.iter() {
            *weight_map.entry(*fid).or_insert(0.0) += p_weight * care_weight * 0.7;
            // 0.7 因子——子女被动参与，不如父母投入
        }
    }
    
    // 2. 社区温和引力（同 settlement 主要信仰给予增量）
    for (fid, community_weight) in community_faiths {
        *weight_map.entry(*fid).or_insert(0.0) += community_weight * 0.05;
    }
    
    // 3. 正则化到 participation vec（cap 每个权重 ≤ 0.8——年幼时不会极度虔诚）
    for (fid, weight) in weight_map {
        if weight > 0.01 {
            participation.push((fid, weight.min(0.8)));
        }
    }
    
    ReligiousPracticeProfile {
        participation,
        theological_depth: 0.05 + rng.gen::<f32>() * 0.10,  // 0.05-0.15——家庭熏陶的浅层理解
        motivation: ReligiousMotivation::Habitual,            // 年幼被动参与
    }
}
```

子女不自动"信仰"父母信的东西——他们自动**参与**父母的宗教实践。成年后（coming-of-age 仪式或约 16 岁），motivation 可能转变为其他类型 → participation 权重可能大幅变化。
