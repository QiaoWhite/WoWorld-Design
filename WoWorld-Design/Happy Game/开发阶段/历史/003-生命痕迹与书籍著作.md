## 〇、总览

历史不仅仅是重大事件的时间线。它也是墓碑上的一句话、一本三代人写的日记、一张发黄的欠条、一面墙上模糊的刻字。**这些"小历史"是玩家触摸历史中具体的人的唯一方式。**

本规格覆盖两个相关联但性质不同的系统：
- **生命痕迹**（上篇）——个体层面留下的物理痕迹。概率驱动 + 需求驱动，七种触发情境
- **书籍著作**（下篇）——系统化的文字记录。Work→PhysicalBook 两层模型，五条生成路径

## 上篇：生命痕迹

### 一、什么是生命痕迹

```rust
/// 生命痕迹——个体 NPC 在世界上留下的物理文字痕迹
/// 它不是 event_log 的一部分。
/// 它不描述"世界发生了什么"——它让玩家触摸到历史中具体的人。
struct LifeTrace {
    id: TraceId,
    trace_type: LifeTraceType,
    owner: NpcId,                      // 谁留下的痕迹
    owner_name_at_time: String,        // 留下痕迹时这个人叫什么名字
    location: LocationId,              // 在世界的哪里
    content: TraceContent,             // 痕迹的内容
    created_at: i32,                   // 留下痕迹的时间戳
    
    /// 物理耐久
    durability: f32,                   // 0.0(化为尘土)-1.0(崭新)
    decay_rate: f32,                   // 每年衰减量
    
    /// 可发现性
    discoverability: f32,              // 0.0(深埋地下)-1.0(广场中央)
    accessibility: f32,                // 0.0(无法接触)-1.0(伸手可及)
    
    /// 触发这条痕迹生成的原始情境
    trigger: WriteTrigger,
    
    /// 关联的 event_log 事件（如果此痕迹是因某个事件而产生的）
    related_events: Vec<EventId>,
}

enum LifeTraceType {
    Grave,              // 墓碑/墓穴
    DiaryPage,          // 日记页
    WallCarving,        // 墙/树/石上的刻字
    Contract,           // 契约/欠条
    Letter,             // 信件
    LostItem,           // 遗落的有铭文的物品
    VotiveOffering,     // 供品/还愿物（神殿/古树下）
    HiddenStash,        // 藏起来的储物（含留言）
    BuildingInscription, // 建筑上的铭文（建造完成时）
    NameCarving,        // "XX到此一游"
}

struct TraceContent {
    /// 文字内容——参数化生成，不包含完整文本
    /// 完整的叙事文本在玩家交互时根据参数即时拼装
    text_params: HashMap<String, String>,
    
    /// 内容摘要（用于索引和搜索）
    summary: String,
    
    /// 文字是否完整（随时间可能部分脱落/腐蚀）
    text_completeness: f32,            // 1.0 = 完整可读, 0.0 = 完全不可读
}
```

### 二、双重驱动模型

#### 2.1 类型 A：副作用记录（概率驱动）

适用于**没有"写作"作为日常工作的 NPC**。记录是某件事已经发生后的概率性副作用。不是 NPC 有意主动记录的。

```
触发模型:
  1. NPC 的某项需求/活动完成（死亡、交易、情绪爆发、家族变动等）
  2. 完成后 → 概率性触发记录行为
  3. 概率由多个因子调制（识字率、文化习俗、事件强度、是否有书写工具）
  4. 记录与否不影响已完成的行为

大多数 NPC 一生产出的痕迹数量: 0-5 条
```

#### 2.2 类型 B：职业记录（需求驱动）

适用于**以写作为日常工作的 NPC**。写作是 GOAP 目标之一。产出是规律性的、系统化的。

```
触发模型:
  1. NPC 的 GOAP 目标或概率行为树中包含写作
  2. 按职业节奏产出——编年史官每年写、书记官每天写
  3. 写作的内容 > 概率性痕迹的内容（更完整、更系统）
  4. 产出可能是独立的 PhysicalBook
```

#### 2.3 类型 C：爱好记录（半需求半概率）

适用于**高开放性 + 识字的 NPC**。写日记是日常行为树中的概率分支，不是 GOAP 目标。

#### 2.4 类型 D：口头记录（不产生文字）

适用于**所有 NPC（不需要识字）**。在社交互动中分享记忆。不进痕迹系统——走信息传播系统的口头通道。

### 三、七种触发情境

#### 情境 1: 死亡——墓碑

```
触发: NPC 死亡
行为: 由在世的亲属/战友/邻居处理遗体时概率触发

基础概率: 0.05（大多数死者没有墓碑）
调制因子:
  × 有存活的亲属 → 0.3
  × 与主要亲属的关系强度（0.0-1.0）→ (0.3 + strength * 0.7)
  × 文化中有立碑传统 → 2.0
  × 死于事件（非自然死亡）→ 3.0（死于战争、瘟疫、刺杀 → 更可能被纪念）
  × 附近有石材资源 → 1.5
  × trigger_by_death

最终概率 = base × product(调制因子)

内容参数:
  { "name": NPC姓名, "birth_year": 生年, "death_year": 卒年,
    "relation": 立碑者与死者的关系称谓,
    "death_cause_short": 死因简述（死于战争/死于饥荒/寿终正寝）,
    "most_important_life_event": 该NPC记忆系统中 importance 最高的事件简述 }

碑文模板 (参数填充):
  "{name} · {relation} · {birth_year}-{death_year}年 · {death_cause_short}"
  如果 NPC 有重要人生事件: "——{most_important_life_event}"
  
例如:
  "刘氏 · 爱妻·母 · 1386-1423年 · 死于饥荒第三年"
  "王大锤 · 铁匠·父 · 1350-1420年 · ——锻造过一把永不生锈的剑"
```

#### 情境 2: 经济交易——契约/欠条

```
触发: NPC 完成一笔金额 > 阈值的交易

基础概率: 0.0002/交易（绝大多数交易不留下书面记录）
调制因子:
  × 交易金额（以本地月均收入为单位）→ 越高越可能留字据
  × 识字率（交易双方任一方识字即可）
  × 该文化中有书面契约传统 → 3.0
  × 交易涉及土地/房屋/奴隶/婚姻 → 5.0
  × 交易是借款 → 2.0

内容参数:
  { "party_a": 甲方名, "party_b": 乙方名,
    "amount": 金额+单位, "item": 交易物,
    "date": 交易日期, "witness": 见证人名 }

契约文本模板:
  "兹有{party_a}与{party_b}，于{date}约定：{item}，作价{amount}。见证人：{witness}。"
```

#### 情境 3: 情绪峰值——日记/信件

```
触发: NPC 的某一种情绪强度 > 0.85（大五人格情绪引擎的峰值）

基础概率: 0.01（大多数情绪峰值不产生日记）
调制因子:
  × 识字率
  × 大五人格开放性 → 0.2 + openness * 0.8
  × 情绪强度 → intensity * 1.5
  × 有书写工具在身 → 1.0，无 → 0.02
  × 情绪类型: 悲伤 → 1.5（悲伤时更倾向于写下来）
              喜悦 → 0.8
              愤怒 → 1.2
              恐惧 → 1.0

内容参数:
  { "emotion": 主要情绪, "intensity": 强度值,
    "subject": 情绪的对象（人名/事件名/"这个世界"）,
    "day_summary": 当天的主要事件简述,
    "key_phrase": 日记的最核心句子——从 NPC 对当天事件的记忆中提取 }

日记文本模板:
  "{date}。{key_phrase}。{day_summary}"
  
例如:
  "春第7天。他回来了。我以为他在战争中死了。十七年。今天一只鸽子落在窗台上。"
```

#### 情境 4: 社会地位变动——家谱更新

```
触发: 家族中发生婚姻/出生/死亡/驱逐/继承事件

基础概率: 0.05
调制因子:
  × 识字率
  × 家族规模 → 越大越可能记录（≥10 人 → 2.0）
  × 社会地位 → 贵族→5.0, 富商→3.0, 平民→1.0, 农夫→0.2
  × 文化中的宗族传统权重 → 0.5 + tradition * 2.0
  × 有书写工具 → 1.0, 无 → 0.01

内容:
  { "event_type": 婚姻/出生/死亡/驱逐/继承,
    "person": 当事人名,
    "relation_changes": [关系变更列表] }
```

#### 情境 5: 重大事件目击——第一手记录

```
触发: NPC 在 event_log 事件的 N km 范围内，且事件 intensity > 0.5

基础概率: 0.02
调制因子:
  × 识字率
  × NPC 靠近事件中心的距离 → distance_km < 1 → 3.0, < 10 → 1.0, < 50 → 0.3
  × 事件强度 → intensity * 4.0
  × NPC 是否在这件事中有亲属卷入 → 3.0 for Yes, 1.0 for No
  × 有书写工具 → 1.0, 无 → 0.01

关键设计:
  100 个目击同一场战役的人 → 只有 2-3 人会写日记
  这三人的日记记载各不相同——每个人看到了事件的不同角落
  一个人的日记:"国王冲锋时摔倒了吗？"（他看到了国王的镜头）
  另一个人的日记:"什么都没发生"（他没看到关键画面）

内容参数:
  { "event_summary": 从该 NPC 视角看到的部分（不完整）,
    "npc_reaction": NPC 对所见的情感反应，
    "npc_assessment": NPC 对事件原因的理解（可能错误） }
```

#### 情境 6: 建造完成——建筑铭文/奠基石

```
触发: 重要建筑（神殿/城墙/王宫/桥梁/图书馆）建造完成

基础概率: 0.3
调制因子:
  × 建筑等级 (1-5) → 等级越高越可能留铭文
  × 文化中有建筑铭文传统 → 3.0, 无 → 0.1
  × 建筑类型: 神殿 → 5.0, 图书馆 → 4.0, 桥梁 → 2.0, 民居 → 0.01

内容:
  { "building_name": 建筑名,
    "builder": 主持建造者,
    "funded_by": 出资者（如有）,
    "completion_year": 完工年份,
    "purpose": 建筑目的简述 }
```

#### 情境 7: 临终整理——晚年回顾

```
触发: NPC 年龄 > 该种族预期寿命 × 0.8

基础概率: 0.02
调制因子:
  × 识字率
  × 大五人格开放性 → openness * 1.5
  × 有子女 → 1.5（想给后代留点什么）
  × 没有子女 → 2.0（没有人可以口述 → 更倾向于写下来）
  × 一生中经历了 > 2 个重大事件 → 2.0

内容:
  从 NPC 记忆系统中提取一生中最重要的事件（取 importance 最高的 top 3-5）
  不是自传——只是几条关键事件的简要记录
  "我活了七十二年。经历过两次战争、三次饥荒。最重要的事是嫁给了对的人。"
```

### 四、痕迹的耐久与衰减

```rust
/// 痕迹的物理耐久管理
fn tick_trace_durability(trace: &mut LifeTrace, dt_years: f32, world: &WorldState) {
    // 基础衰减
    let base_decay = trace.decay_rate * dt_years;
    
    // 材质修正
    let material_modifier = match trace.trace_type {
        LifeTraceType::Grave => {
            // 石刻 = 极慢衰减
            0.1
        }
        LifeTraceType::WallCarving | LifeTraceType::BuildingInscription | LifeTraceType::NameCarving => {
            // 刻在硬面上 = 慢衰减
            0.3
        }
        LifeTraceType::DiaryPage | LifeTraceType::Contract | LifeTraceType::Letter => {
            // 纸/羊皮 = 快衰减（除非在极干燥环境）
            let humidity = world.climate.humidity_at(trace.location);
            if humidity < 0.2 { 0.3 } else if humidity > 0.7 { 3.0 } else { 1.0 }
        }
        LifeTraceType::LostItem => {
            // 金属物品 = 极慢；木/纸 = 快
            0.5
        }
        _ => 1.0,
    };
    
    // 保护条件修正
    let protection_modifier = if trace.accessibility < 0.3 {
        // 难以接触 = 被保护（深埋/密封/在洞穴深处）
        0.3
    } else if trace.accessibility > 0.8 {
        // 极易接触 = 暴露（风雨侵蚀）
        2.0
    } else {
        1.0
    };
    
    let total_decay = base_decay * material_modifier * protection_modifier;
    trace.durability -= total_decay;
    trace.durability = trace.durability.max(0.0);
    
    // 耐久影响文字可读性
    trace.content.text_completeness = trace.durability;
    
    // 如果耐久降到 0——痕迹彻底消失
    if trace.durability <= 0.0 {
        // 标记为不可读——不删除（可能还有考古价值？）
        // 但玩家交互时只看到"模糊的痕迹"
    }
}

/// 每年 tick（历史模拟中）
fn tick_all_traces(traces: &mut Vec<LifeTrace>, world: &WorldState, dt_years: f32) {
    for trace in traces.iter_mut() {
        tick_trace_durability(trace, dt_years, world);
    }
    
    // 清理耐久为 0 的痕迹（只保留最近 100 年的——给玩家"刚刚模糊"的体验）
    traces.retain(|t| t.durability > 0.01 || world.current_timestamp - t.created_at < 1200);
}
```

### 五、生成规模与密度

```
预生成（世界创建时的一次性生成）:

  墓碑:
    全图 50K-150K 条 × 80B = 4-12 MB
    
  情绪日记:
    全图 80K-300K 条 × 100B = 8-30 MB
    
  墙上刻字/建筑铭文:
    全图 20K-80K 条 × 60B = 1.2-4.8 MB
    
  契约/信件/供品/秘藏:
    全图 30K-100K 条 × 60B = 1.8-6 MB

  总计: ~200K-500K 条 × 平均 80B = 16-40 MB

运行时新增:
  玩家 100 小时 ≈ 游戏内 3-10 个月（取决于时间流速）
  NPC 死亡 → 墓碑: ~500-2,000 条 × 80B = 40-160 KB
  情绪日记: ~2,000-5,000 条 × 100B = 200-500 KB
  总计: < 1 MB
```

## 下篇：书籍著作

### 六、书籍的三层模型

```
三层: PhysicalBook  —— 物理副本。有位置、有耐久、有批注。玩家捡到的"东西"
      ↓ 指向
二层: Work           —— 著作。抽象的文字内容。一个 Work 可能有多个 PhysicalBook
      ↓ 由...组成
一层: TextSegment    —— 文本段落。每个段落标注作者、写作时间、写作情境
```

```rust
/// 著作——抽象的文字内容
struct Work {
    id: WorkId,
    title: String,                    // 标题
    work_type: WorkType,
    primary_author: Option<NpcId>,    // 主要作者（可能匿名/佚名）
    creation_date: i32,
    creation_location: LocationId,
    language: LanguageId,
    
    /// 文本内容的段落序列
    contents: Vec<TextSegment>,
    
    /// 偏见配置文件
    bias_profile: WorkBias,
    
    /// 引用的来源
    cited_sources: Vec<SourceReference>,
    
    /// 是否是某个更早 Work 的编纂/汇总
    compiled_from: Vec<WorkId>,
    
    /// 此 Work 关联的 event_log 事件
    related_events: Vec<EventId>,
}

struct TextSegment {
    segment_type: SegmentType,
    author: Option<NpcId>,
    date: i32,
    content: SegmentContent,
    writing_trigger: WriteTrigger,
}

enum SegmentType {
    OriginalComposition,    // 原创
    Annotation,             // 批注——后世读者写的（v1.2+）
    Continuation,           // 续写——后世作者接着写（v1.2+）
    CopyistColophon,        // 抄写者的跋（v1.1+）
    OwnershipMark,          // 藏书印/签名
    Censorship,             // 审查标记——有内容被移除
    CompilationNote,        // 编纂者说明——"以下五章来自某某的日记"
}

struct SegmentContent {
    /// 文本的参数化表示——不存储完整叙事文本
    /// 完整的可读文本在玩家阅读时即时生成
    text_params: HashMap<String, String>,
    
    /// 字数估算（用于存储预算和文本生成预算）
    estimated_chars: u32,
}

enum WorkType {
    Chronicle,             // 编年史——每年若干条目
    Treatise,              // 学术论著/分析
    ReligiousScripture,    // 宗教经文/奇迹录
    Genealogy,             // 家谱/族谱
    CompiledRecords,       // 编纂物——多来源记录汇集
    PoetryCollection,      // 诗集/歌集（吟游诗人产出）
    Biography,             // 传记
    Travelogue,            // 游记
    Manual,                // 技术手册/指南
    PersonalDiaryCollection, // 个人日记汇集（一生日记的合订）
    LegalCodex,            // 法典
    TradeLedger,           // 商会账本
    Atlas,                 // 地图集
}

struct WorkBias {
    polity_allegiance: f32,        // 对自己政权的美化 0.0-1.0
    religious_lens: Option<ReligionId>,
    class_perspective: f32,        // 偏向哪个阶级（0=平民, 1=贵族）
    completeness: f32,             // 已知事件中的覆盖比例（0.0-1.0）
    accuracy: f32,                 // 已知事件中描述的正确比例
}

/// 书的物理副本——世界的具体位置中存在的物体
struct PhysicalBook {
    id: PhysicalBookId,
    work_id: WorkId,                // 指向哪个著作
    copy_generation: u32,           // 第几代抄本（0 = 原稿/孤本）
    
    /// 当前位置
    location: LocationId,
    container: BookContainer,
    condition: f32,                 // 0.0(化为尘土)-1.0(崭新)
    
    /// 这个物理副本特有的附加文本
    unique_annotations: Vec<TextSegment>,
    
    /// 物理属性
    material: BookMaterial,
    binding_quality: f32,
    is_complete: bool,
    missing_sections: Vec<u32>,
    
    /// 流转历史——被谁拥有过
    provenance: Vec<OwnershipRecord>,
    
    /// 可读性——是否需要特殊条件
    readability: ReadabilityRequirement,
}

enum BookContainer {
    InLibrary { shelf: String },
    InPrivateCollection { owner: NpcId },
    InInventory { owner: NpcId },
    OnGround,
    InRuins,
    Hidden { location_hint: String },
}

enum BookMaterial {
    Paper,              // 纸——不耐久
    Parchment,          // 羊皮纸——中等耐久
    Vellum,             // 精制犊皮纸——高耐久
    BambooSlips,        // 竹简
    StoneTablet,        // 石板
    ClayTablet,         // 泥板
    MetalPlate,         // 金属板
    MagicScroll,        // 魔法卷轴（耐久极高）
}

struct OwnershipRecord {
    owner: ParticipantId,
    acquired_date: i32,
    acquisition_method: AcquisitionMethod,
    disposed_date: Option<i32>,
    disposition_method: Option<DispositionMethod>,
}

enum AcquisitionMethod {
    Purchased, Inherited, Gifted, Looted, Found, Commissioned, Copied,
}

enum DispositionMethod {
    Sold, Lost, Destroyed, Donated, Bequeathed, Stolen, Confiscated,
}

enum ReadabilityRequirement {
    Any,               // 任何人都能看
    RequiresLiteracy,  // 需要识字
    RequiresSkill { skill: String, min_level: f32 },
    Damaged { text_completeness: f32 },  // 部分字迹模糊
}

/// 写作触发——什么驱动了这段文字的写作
struct WriteTrigger {
    trigger_type: WriteTriggerType,
    timestamp: i32,
    context: WriteContext,
}

enum WriteTriggerType {
    /// 副作用：做完某事后概率性记录
    SideEffect { completed_action: String },
    /// 职业需求：GOAP 目标驱动
    OccupationalGoal { goal_type: String },
    /// 爱好：日常行为树的概率分支
    Hobby,
    /// 委托：被他人委托写作
    Commissioned { commissioner: ParticipantId },
    /// 强制：命令/法律要求
    Mandated { mandating_authority: ParticipantId },
}

struct WriteContext {
    /// 写作时的世界状态简要快照
    location: LocationId,
    world_events_active: Vec<EventId>,  // 当时的活跃事件
```

```rust
fn generate_chronicle(chronicler: &Npc, polity: &Polity, year: u32, event_log: &HistoryGraph) -> Work {
    let relevant_events = event_log.query_events(
        since: year_start_timestamp(year),

    for event in &relevant_events {
        // 编年史官不访问 event_log——他们只能用自己的信息渠道
        // 但历史模拟中我们直接模拟"编年史官能获取到的信息"
        let accessible = chronicler.can_access_information_about(event);

            segments.push(TextSegment {

                author: Some(chronicler.id),

            });
        }
    }

        creation_date: year_to_timestamp(year),

/// 路径 2: 编纂——多记录汇集
fn compile_work(
    compiler: &Npc,
    source_works: Vec<WorkId>,
    subject: String,
) -> Work {
    // 从已有 Work 中选择相关段落
    let mut segments = Vec::new();
    for work_id in &source_works {
        let work = load_work(*work_id);
        for segment in &work.contents {
            if segment_is_relevant_to_subject(segment, &subject) {
                segments.push(segment.clone());
            }
        }
    }
    
    // 编纂者加序言
    segments.insert(0, TextSegment {
        segment_type: CompilationNote,
        author: Some(compiler.id),
        date: current_timestamp(),
        content: SegmentContent {
            text_params: compilation_preface_params(compiler, &subject, &source_works),
            estimated_chars: 500,
        },
        writing_trigger: WriteTrigger {
            trigger_type: OccupationalGoal { goal_type: "Compilation".into() },
            timestamp: current_timestamp(),
            context: WriteContext {
                location: compiler.location,
                world_events_active: vec![],
            },
        },
    });
    
    Work {
        id: WorkId::new(),
        title: format!("{}——关于{}的汇集", compiler.name, subject),
        work_type: WorkType::CompiledRecords,
        primary_author: Some(compiler.id),
        creation_date: current_timestamp(),
        creation_location: compiler.location,
        language: compiler.primary_language,
        contents: segments,
        bias_profile: compiler.as_work_bias(),
        cited_sources: source_works.iter().map(|w| SourceReference { work_id: *w, segment_indices: vec![] }).collect(),
        compiled_from: source_works,
        related_events: vec![],
    }
}

/// 路径 3: 抄写——已有书的复制 (v1.1+)
fn copy_book(
    original: &PhysicalBook,
    copyist: &Npc,
    location: LocationId,
) -> PhysicalBook {
    let work = load_work(original.work_id);
    let mut new_segments = Vec::new();
    
    // 对每个段落应用抄写失真
    for segment in &work.contents {
        let copied = apply_copy_errors(segment, original.copy_generation + 1, copyist);
        new_segments.push(copied);
    }
    
    // 抄写员可能加一个 colophon
    if rand::random::<f32>() < 0.3 {
        new_segments.push(TextSegment {
            segment_type: CopyistColophon,
            author: Some(copyist.id),
            date: current_timestamp(),
            content: SegmentContent {
                text_params: colophon_params(copyist, location),
                estimated_chars: 100,
            },
            writing_trigger: WriteTrigger {
                trigger_type: OccupationalGoal { goal_type: "CopyWork".into() },
                timestamp: current_timestamp(),
                context: WriteContext { location, world_events_active: vec![] },
            },
        });
    }
    
    // 创建新 Work（如果抄写引入了足够的错误——可能需要 fork 一个新 Work）
    // 大多数情况——指向同一个 Work
    let new_work_id = if segments_diverge_significantly(&new_segments, &work.contents) {
        let new_work = Work {
            id: WorkId::new(),
            contents: new_segments,
            ..work.clone()  // 继承大部分属性
        };
        save_work(new_work);
        new_work.id
    } else {
        original.work_id
    };
    
    PhysicalBook {
        id: PhysicalBookId::new(),
        work_id: new_work_id,
        copy_generation: original.copy_generation + 1,
        location,
        container: BookContainer::InInventory { owner: copyist.id },
        condition: 1.0,  // 新抄本——崭新
        unique_annotations: vec![],
        material: BookMaterial::Paper,  // 取决于可用材料
        binding_quality: copyist.bookbinding_skill,
        is_complete: true,
        missing_sections: vec![],
        provenance: vec![OwnershipRecord {
            owner: ParticipantId::Npc(copyist.id),
            acquired_date: current_timestamp(),
            acquisition_method: AcquisitionMethod::Copied,
            disposed_date: None,
            disposition_method: None,
        }],
        readability: ReadabilityRequirement::RequiresLiteracy,
    }
}
```

### 八、抄写错误 (v1.1+)

```rust
/// 抄写过程中可能引入的错误
enum CopyError {
    CharacterSwap,          // 形近字——"己已巳"
    HomophoneSwap,          // 听写时同音字——"清"变"轻"
    Omission,               // 漏抄一行
    Duplication,            // 重抄一行
    Paraphrase,             // 用自己的话改写（低技能抄写员）
    Misattribution,         // 把一段话的作者抄错
    MarginalNoteMigration,  // 前人的批注抄成正文
    DateError,              // 日期抄错——"1273"变"1373"
    TitleError,             // 地名/人名抄错
    IntentionalEdit,        // 故意的修改——审查/美化
}

/// 应用抄写失真
fn apply_copy_errors(
    segment: &TextSegment,
    copy_generation: u32,
    copyist: &Npc,
) -> TextSegment {
    // 基础错误率——每次抄写引入错误的概率
    let base_error_rate = 0.02;  // 2% 的段落会有某种错误
    
    // 代际修正——越后的抄本越容易累积错误
    let generation_modifier = 1.0 + (copy_generation as f32 - 1.0) * 0.15;
    
    // 抄写员技能修正
    let skill_modifier = 1.0 - copyist.copying_skill * 0.8;  // 0.2-1.0
    
    let error_probability = base_error_rate * generation_modifier * skill_modifier;
    
    if rand::random::<f32>() < error_probability {
        // 根据抄写员的特征选择最可能的错误类型
        let error = select_copy_error(copyist);
        let mut new_segment = segment.clone();
        apply_error(&mut new_segment, error);
        new_segment
    } else {
        segment.clone()
    }
}

fn select_copy_error(copyist: &Npc) -> CopyError {
    // 低技能 → 容易 Paraphrase/Omission
    // 高技能但有政治倾向 → 容易 IntentionalEdit
    // 文化背景不同 → 容易 TitleError/Misattribution
    
    if copyist.copying_skill < 0.3 {
        weighted_select(&[
            (Paraphrase, 0.3),
            (Omission, 0.25),
            (CharacterSwap, 0.2),
            (Duplication, 0.15),
            (Misattribution, 0.1),
        ])
    } else {
        weighted_select(&[
            (CharacterSwap, 0.4),
            (DateError, 0.2),
            (MarginalNoteMigration, 0.15),
            (IntentionalEdit, 0.15),
            (TitleError, 0.1),
        ])
    }
}
```
