## 〇、总览

历史存在于世界中的方式不是"查看历史界面"。它通过**物理管道嵌入环境**——玩家在行走、探索、交互时自然地感知到历史的重量。本规格覆盖三条环境历史管道：

- **灵元素印记**（读树）——历史事件的情绪+魔力空间残留，通过老树读取
- **文物痕迹**——魔法物品的历史包装层，物品参与历史的记录
- **关系遗产**——世仇/联盟/债务/附庸的因果起源，社会格局本身就是历史

## 上篇：灵元素印记

### 一、什么是灵元素印记

灵元素印记是重大历史事件在空间中留下的**灵元素层面的残留**。它不是物理痕迹（不是废墟、不是碑文），而是情绪和魔力的空间烙印——像土地"记得"发生过的事。

关键设计原则：
- **印记归历史模块拥有**——它是历史事件的残留
- **印记在空间（Chunk）上存储**——不是树的属性。树只是读取器
- **读取归植物模块**——读树是植物的"能力"，不是历史模块的功能
- **印记被后来的事件叠加覆盖**——同一片地发生多次事件 → 多层印记

### 二、数据结构

```rust
/// 灵元素印记——历史事件在空间中的灵元素残留
/// 归历史模块拥有
struct AetherImprint {
    id: ImprintId,
    
    /// 来源——哪个事件产生了这个印记
    source_event: EventId,
    
    /// 印记在空间中的范围
    footprint: Footprint,
    
    /// 强度热图——中心最强，径向衰减
    intensity_map: IntensityField,
    
    /// 核心情绪——事件残留的情绪基调
    dominant_emotion: Vec<(Emotion, f32)>,
    
    /// 灵元素偏向——事件中活跃的元素类型
    element_bias: Vec<(Element, f32)>,
    
    /// 事件发生的时间
    event_timestamp: i32,
    
    /// 印记清晰度——随时间衰减
    clarity: f32,  // 1.0(清晰如昨) - 0.0(完全消散)
    
    /// 基础年衰减率
    decay_rate: f32,  // 默认 0.0005/年 → 2000年衰减约63%
    
    /// 是否被后来的印记覆盖
    overlaid_by: Vec<ImprintId>,
    /// 是否覆盖了更早的印记
    overlays: Vec<ImprintId>,
}

/// 印记的空间范围
struct Footprint {
    /// 几何中心
    center: LocationId,
    /// 覆盖的形状（简化：椭圆）
    shape: FootprintShape,
    /// 半径（米）——由事件 intensity 决定
    radius_m: f32,
}

enum FootprintShape {
    Circle { radius: f32 },
    Ellipse { semi_major: f32, semi_minor: f32, angle: f32 },
    Polygon { vertices: Vec<Vec2> },
}

struct IntensityField {
    /// 稀疏强度采样点
    /// 中心点 intensity = 1.0，边缘衰减至 0.0
    samples: Vec<(Vec2, f32)>,  // (相对坐标, 强度)
    /// 插值方法
    interpolation: InterpolationMethod,
}

/// 印记与事件的关联
/// 事件 intensity → 印记范围映射:
///   0.5-0.6: radius ~500m
///   0.6-0.7: radius ~1km
///   0.7-0.8: radius ~2km
///   0.8-0.9: radius ~5km
///   0.9+:     radius ~10km
fn imprint_radius_from_event_intensity(intensity: f32) -> f32 {
    if intensity < 0.5 {
        return 0.0;  // 不够强——不产生印记
    }
    let base = 500.0;  // 500m 基准
    let exponent = (intensity - 0.5) * 8.0;  // 0.5→0, 0.9→3.2
    base * (2.0_f32).powf(exponent)
}
```

### 三、印记的写入

```rust
/// 事件发生后——为其生成灵元素印记
impl AetherImprint {
    fn from_event(event: &HistoricalEvent, world: &WorldState) -> Option<Self> {
        if event.intensity < 0.5 {
            return None;  // 不够强——不产生印记
        }
        
        let center_location = event.location.as_point().map(|(loc, _)| loc)
            .unwrap_or(world.default_location());
        
        let radius = imprint_radius_from_event_intensity(event.intensity);
        
        Some(AetherImprint {
            id: ImprintId::new(),
            source_event: event.id,
            footprint: Footprint {
                center: center_location,
                shape: FootprintShape::Circle { radius },
                radius_m: radius,
            },
            intensity_map: IntensityField::generate(radius),
            dominant_emotion: emotion_from_event(event),
            element_bias: element_from_event(event),
            event_timestamp: event.timestamp,
            clarity: 0.9,  // 初始清晰度——不完全清晰（有短暂的信息沉淀期）
            decay_rate: 0.0005,  // 默认年衰减
            overlaid_by: vec![],
            overlays: vec![],
        })
    }
    
    /// 从事件类型 + 叙事参数推导核心情绪
    fn emotion_from_event(event: &HistoricalEvent) -> Vec<(Emotion, f32)> {
        match event.event_type {
            TerritorialWar | Conquest | CivilWar | Rebellion => {
                vec![(Emotion::Fear, 0.9), (Emotion::Anger, 0.7), (Emotion::Sadness, 0.5)]
            }
            Plague | Famine => {
                vec![(Emotion::Sadness, 0.85), (Emotion::Fear, 0.7)]
            }
            MagicDisaster | MagicalCreatureRampage => {
                vec![(Emotion::Fear, 0.95), (Emotion::Surprise, 0.6)]
            }
            CulturalRenaissance | MagicBreakthrough | TechnologicalBreakthrough => {
                vec![(Emotion::Joy, 0.7), (Emotion::Curiosity, 0.8)]
            }
            TreatyPeace | TreatyAlliance => {
                vec![(Emotion::Relief, 0.6), (Emotion::Hope, 0.5)]
            }
            SettlementAbandoned => {
                vec![(Emotion::Sadness, 0.7), (Emotion::Loss, 0.8)]
            }
            ReligiousCrusade | ReligiousPersecution => {
                vec![(Emotion::Fear, 0.8), (Emotion::Anger, 0.6), (Emotion::Zeal, 0.7)]
            }
            _ => {
                vec![(Emotion::Surprise, 0.5)]
            }
        }
    }
    
    fn element_from_event(event: &HistoricalEvent) -> Vec<(Element, f32)> {
        match event.event_type {
            MagicDisaster | MagicalCreatureRampage => {
                event.narrative_params.params.get("element")
                    .map(|e| vec![(parse_element(e), 0.9)])
                    .unwrap_or(vec![(Element::Ling, 0.8)])
            }
            Plague => vec![(Element::Xue, 0.6)],
            Earthquake => vec![(Element::Tu, 0.7)],
            Flood => vec![(Element::Shui, 0.8)],
            Wildfire => vec![(Element::Huo, 0.8)],
            _ => vec![(Element::Ling, 0.3)],  // 大多数事件带微弱的灵元素残留
        }
    }
}
```

### 四、印记的衰减与叠加

```rust
/// 每年 tick——印记衰减
fn tick_imprints(imprints: &mut Vec<AetherImprint>, dt_years: f32) {
    for imprint in imprints.iter_mut() {
        // 基础衰减
        imprint.clarity *= (1.0 - imprint.decay_rate).powf(dt_years);
        
        // 事件强度越高 → 衰减越慢（更"深刻"的事件留得更久）
        // 通过事件反查（此处简化为直接存储相关参数）
        let intensity_retention = 1.0;  // 默认
        
        // 如果印记 clarity < 0.05 → 读取时返回"极模糊的残影"
        // clarity = 0.0 → 印记完全消散（从活跃索引中移除）
    }
    
    // 移除已完全消散的印记
    imprints.retain(|i| i.clarity > 0.01);
}

/// 新印记写入时——检查与已有印记的重叠

            // 这个关系用于"读树"时的印记排序
        }
    }
}
```

### 五、读树——消费端

```rust
/// 读树 trait——由植物模块实现
/// 历史模块只提供 AetherQuery 接口
trait AetherQuery {
    /// 查询指定位置的灵元素印记列表
    /// max_age: 只查此时间之后的印记（树龄限制——年轻树读不到古老事件）
    fn query_imprints(
        &self,
        location: LocationId,
        max_age: i32,            // 树龄月数（只查树龄以内的印记）
        min_clarity: f32,         // 最小清晰度
    ) -> Vec<AetherImprint>;
}

/// 植物模块的读树实现（伪代码——不在历史模块中）
fn read_tree(tree: &Tree, location: LocationId, aether_query: &dyn AetherQuery) -> ReadResult {
    let imprints = aether_query.query_imprints(
        location,
        tree.age_months,
        0.1,  // 最小清晰度——过于模糊的印记读不到
    );
    
    if imprints.is_empty() {
        return ReadResult::Nothing;
    }
    
    // 按树本身的灵元素亲和力过滤
    let accessible: Vec<_> = imprints.iter()
        .filter(|i| i.clarity * tree.spirit_affinity > 0.2)
        .collect();
    
    if accessible.is_empty() {
        return ReadResult::Nothing;
    }
    
    // 按清晰度排序（最清晰的排前面——最近的或最强烈的事件）
    // 返回意象碎片列表——植物模块负责翻译为文本
    ReadResult::Impressions(accessible)
}
```

**读树 ≠ 读书——一个关键区分：**

- **读书**：书是某个作者刻意写下的**事实陈述**。它可能有偏见、有遗漏、有错误——但它**试图陈述事实**。编年史写着"1423年冰冠宣战"——它是一个关于过去的声明。
- **读树**：树不陈述任何事实。树只是**情绪和意象的残留空间**。触碰一棵老树感受到的不是"1423年发生了战争"——感受到的是恐惧的余波、火烧的气味、一个女人哭泣声的片段。这些感受**不构成事实**——它们只是那个地点曾经发生过强烈情绪事件的证据。

对大日志的影响：读书获得的内容进入大日志的知识条目（可能被验证/证伪）。读树获得的内容进入大日志的感受条目——永远标为 `Unverified`，因为感受不是可验证的事实。读树告诉玩家"这里发生过什么"，但具体"发生了什么"——需要书、碑文、NPC 证言来验证。

## 下篇：历史嵌入世界

### 六、六条历史嵌入管道

```
管道 1: 废墟/建筑老化 — 视觉管道。玩家走进一座废弃神殿 = 不需要任何文字
管道 2: 文物 —        物品管道。捡到一把古剑 = 追溯它的历史
管道 3: 书本 —        阅读管道。读编年史/日记/碑文 = 深度了解
管道 4: 读树 —        灵元素管道。触碰老树 = 感受意象碎片
管道 5: 关系遗产 —    社会管道。两个世仇家族 = 历史活在当下的恨意中
管道 6: NPC 口头传说 — 对话管道。铁匠闲聊提到往事
```

### 七、管道 1: 废墟与建筑老化

核心洞察：**建筑不只是"旧了"——建筑是"旧了，因为XX事件"。** 废墟不只是物理衰减的结果，是历史事件的物理余波。神殿东翼不是自然塌的——是被 300 年前那场战争烧毁的。这堵墙上的裂缝不是年久失修——是 150 年前那场地震留下的。一座废弃的村庄不是"没人住了"——是"1423 年的饥荒导致人口全部迁移或死亡，之后再也没人回来"。

这是从"世界看起来旧"到"世界看起来有故事"的质变。不需要玩家读任何文字——眼睛看到废墟，就能感觉到"这里出过事"。

```rust
/// 建筑的历史状态——由事件修改
struct BuildingHistoryState {
    building_id: BuildingId,
    
    /// 建造信息
    constructed_at: i32,
    constructed_by: Option<NpcId>,
    
    /// 被历史事件修改的记录
    damage_history: Vec<DamageRecord>,
    renovation_history: Vec<RenovationRecord>,
    
    /// 当前状态
    current_condition: f32,        // 0.0(完全废墟) - 1.0(完好)
    is_abandoned: bool,
    abandoned_at: Option<i32>,
    
    /// 关联的历史事件
    related_events: Vec<EventId>,
}

struct DamageRecord {
    event_id: EventId,
    damage_ratio: f32,
    cause: DamageCause,
    timestamp: i32,
}

enum DamageCause {
    War,
    NaturalDisaster,
    Fire,
    MagicalAccident,
    Neglect,             // 年久失修
    Looting,
}

/// 建筑老化的历史感知
/// 玩家交互时获取：
fn inspect_building(building_id: BuildingId, history: &BuildingHistoryState) -> BuildingInspection {
    BuildingInspection {
        building_id,
        condition: history.current_condition,
        age: current_timestamp() - history.constructed_at,
        visible_scars: history.damage_history.iter()
            .filter(|d| d.cause.is_visible())  // 火烧的焦痕可见，年久失修不可见
            .map(|d| (d.cause, age_of(d.timestamp)))
            .collect(),
        history_summary: generate_building_history_summary(history),
    }
}
```

```rust
    Seal,              // 印章/契约物

    event_id: EventId,
```

### 十一、管道 5: 关系遗产

这是最微妙的历史管道——它不是物理物体，而是**社会关系格局本身就是历史的痕迹**。

```rust
/// 关系遗产——由历史事件产生的、持续到"现在"的团体间关系
struct RelationalLegacy {
    id: LegacyId,
    
    /// 关系的两端
    from_group: ParticipantId,
    to_group: ParticipantId,
    
    /// 关系类型
    legacy_type: LegacyType,
    
    /// 起源事件——是什么历史事件导致了这种关系
    originating_events: Vec<(EventId, f32)>,  // (事件, 贡献权重)
    
    /// 当前强度
    current_intensity: f32,   // 0.0-1.0
    
    /// 持续了多少代
    generations_active: u32,
    
    /// 方向（单向还是双向）
    direction: LegacyDirection,
}

enum LegacyType {
    Feud,               // 世仇
    Alliance,           // 世代联盟
    Debt,               // 世代债务（一方欠另一方）
    Vassalage,          // 附庸关系
    Protection,         // 保护关系
    TradePartnership,   // 世代贸易伙伴
    BloodDebt,          // 血债
    Grudge,             // 怨仇（比世仇轻）
    MutualRespect,      // 互相尊重（过去的战争打出了敬意）
    CulturalKinship,    // 同文化源——虽然政治上分裂但认为对方是"远亲"
}

enum LegacyDirection {
    Unilateral,   // 单向——A恨B但B不在乎A
    Bilateral,    // 双向——互相恨
}

/// 关系遗产的生成——历史模拟中每次事件后检查
fn update_relational_legacies(
    legacies: &mut Vec<RelationalLegacy>,
    event: &HistoricalEvent,
    timestamp: i32,
) {
    match event.event_type {
        TerritorialWar | Conquest => {
            // 战争在参战方之间创建或强化世仇
            if let EventOutcome::Decisive { winner, loser } = &event.narrative_params.outcome {
                let existing = find_legacy_between(*winner, *loser, legacies);
                if let Some(legacy) = existing {
                    // 已有的世仇——强化
                    legacy.current_intensity = (legacy.current_intensity + event.intensity * 0.4).min(1.0);
                    legacy.generations_active += 1;
                    legacy.originating_events.push((event.id, event.intensity));
                } else {
                    // 新的世仇
                    legacies.push(RelationalLegacy {
                        id: LegacyId::new(),
                        from_group: *loser,   // 战败方更记仇
                        to_group: *winner,
                        legacy_type: LegacyType::Feud,
                        originating_events: vec![(event.id, event.intensity)],
                        current_intensity: event.intensity * 0.5,
                        generations_active: 0,
                        direction: LegacyDirection::Unilateral,
                    });
                }
            }
        }
        TreatyAlliance => {
            // 盟约创建/强化联盟
            for pair in event.participants.iter().combinations(2) {
                let existing = find_legacy_between(*pair[0], *pair[1], legacies);
                if let Some(legacy) = existing {
                    if matches!(legacy.legacy_type, LegacyType::Feud | LegacyType::Grudge) {
                        // 前仇敌现在结盟——世仇变为"曾经的敌人现在的盟友"（复杂关系）
                        legacy.legacy_type = LegacyType::MutualRespect;
                        legacy.current_intensity *= 0.3;
                    }
                } else {
                    legacies.push(RelationalLegacy {
                        id: LegacyId::new(),
                        from_group: *pair[0],
                        to_group: *pair[1],
                        legacy_type: LegacyType::Alliance,
                        originating_events: vec![(event.id, event.intensity)],
                        current_intensity: 0.4,
                        generations_active: 0,
                        direction: LegacyDirection::Bilateral,
                    });
                }
            }
        }
        _ => {}
    }
}

/// 玩家感知关系遗产
/// 不需要"打开历史界面"——在正常游戏中自然感知:
///   1. NPC 对话中流露:"我们和他们打了三百年了"
///   2. 行为上:NPC 对某族群的成员态度明显不同
```

### 十三、统一的 Trace 枚举

历史模块向外暴露的所有痕迹类型的统一枚举：

```rust
enum Trace {
    /// 生命痕迹
    LifeTrace { trace: LifeTraceData, owner: NpcId },
    
    /// 建筑痕迹
    ArchitecturalTrace {
        building_id: BuildingId,
        damage_ratio: f32,
        historical_context: Vec<EventId>,
    },
    
    /// 灵元素印记
    AetherImprint { imprint: AetherImprintData },
    
    /// 文物痕迹
    ArtifactTrace {
        magic_item_id: MagicItemId,
        role: ArtifactRole,
        provenance: Vec<ArtifactProvenance>,
    },
    
    /// 书本痕迹
    BookTrace { physical_book_id: PhysicalBookId },
    
    /// 关系遗产
    RelationalLegacy {
        from_group: ParticipantId,
        to_group: ParticipantId,
        legacy_type: LegacyType,
        originating_events: Vec<EventId>,
        current_intensity: f32,
    },
}
```
