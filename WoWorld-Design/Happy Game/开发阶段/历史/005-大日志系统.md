## 〇、总览

大日志是一个**纯功能性 UI 工具**——玩家的外置大脑，角色的知识库。它不叙事、不灌输、不评价。它只客观地记录角色所见所闻的一切，用搜索、双向链接和关系图谱让玩家能随时回溯和管理自己的知识。

**关键定位**：
- 大日志不是 event_log — 它不展示世界真相
- 大日志不是任务日志 — WoWorld 没有任务
- 大日志不是成就系统 — 不评判玩家的行为
- 大日志不是叙事工具 — 不做任何感情色彩的文字加工
- 大日志是纯功能性工具 — 现代 UI 风格，搜索框，平实语言

**关键约束——大日志记录的是角色知道的，不是玩家知道的。** 屏幕前的玩家可能在论坛上查了攻略、在 wiki 上读到了霜铃兰的学名、在视频里看到了某个事件的真相——但这些信息如果角色在游戏内从未从任何来源获知，大日志就不会更新。大日志不是玩家的记事本，是角色的知识库。这个区分决定了所有自动记录的触发条件：必须是角色在游戏世界中通过正常渠道（亲眼所见、亲耳所闻、亲手触碰、亲自阅读）获得的信息。

## 一、核心数据模型

### 1.1 条目结构

```rust
/// 大日志中的一条记录
struct JournalEntry {
    id: EntryId,
    entry_type: EntryType,
    
    /// 创建时间（角色何时第一次获知）
    created_at: GameTimestamp,
    
    /// 条目的显示名称
    /// 已知名称 → 名称
    /// 未知 → "?"
    display_name: String,
    
    /// 知识块列表——每条知识独立验证
    knowledge_blocks: Vec<KnowledgeBlock>,
    
    /// 关联的其他条目（双向链接）
    related_entries: Vec<(EntryId, RelationType)>,
    
    /// 更新历史
    updates: Vec<EntryUpdate>,
    
    /// 玩家手动标注
    player_tags: Vec<String>,
    player_notes: Vec<PlayerNote>,
}

enum EntryType {
    Person,         // 人物
    Event,          // 事件
    Animal,         // 动物
    Plant,          // 植物
    Monster,        // 怪物
    Magic,          // 魔法
    CombatSkill,    // 战斗技能
    Item,           // 物品
    Group,          // 团体
    Polity,         // 政权
    Settlement,     // 聚落
    Location,       // 地点
    Knowledge,      // 知识/传说——模糊的不确定信息
}

struct EntryUpdate {
    timestamp: GameTimestamp,
    description: String,       // "确认为岩鹫"、"名称经编年史验证"
    updated_blocks: Vec<usize>, // 更新了哪些知识块的索引
}
```

### 1.2 知识块——渐进验证模型

```rust
/// 知识块——条目中的一项具体知识
/// 每条知识有独立的验证状态
struct KnowledgeBlock {
    /// 知识字段类型
    field: KnowledgeField,
    
    /// 当前值
    value: KnowledgeValue,
    
    /// 验证状态
    verification: VerificationStatus,
    
    /// 获知历史——此知识怎么来的
    acquisition_history: Vec<KnowledgeAcquisition>,
}

enum KnowledgeField {
    Name,               // 名称（"霜铃兰" / "?"）
    Appearance,         // 外观描述
    Location,           // 位置/发现地
    Properties,         // 属性/特性
    Relations,          // 与其他条目的关系
    HistoricalContext,  // 历史背景
    Abilities,          // 能力/技能
    Behavior,           // 行为模式
    Habitat,            // 栖息地/生长环境
    Uses,               // 用途
    Origin,             // 来源/出身
    Affiliation,        // 所属（政权/团体/家族）
    Role,               // 角色/职位
    Population,         // 人口规模
    Territory,          // 领地范围
    Custom(String),     // 其他自定义字段
}

enum VerificationStatus {
    /// 未验证——单一来源或仅直接观察
    Unverified,
    /// 已验证——直接观察或多来源印证
    Verified,
    /// 已证伪——曾被记载为 X，现确认错误
    Falsified {
        previous_value: KnowledgeValue,
        falsified_by: CorrectionSource,
        falsified_at: GameTimestamp,
    },
    /// 有争议——多来源说法不一，证据不足
    Contested {
        competing_versions: Vec<(KnowledgeValue, Vec<KnowledgeAcquisition>)>,
    },
    /// 未知——此字段存在但无任何信息
    Unknown,
}

enum KnowledgeValue {
    Text(String),
    Numeric(f64),
    Boolean(bool),
    Reference(EntryId),    // 引用大日志中的另一条目
    List(Vec<KnowledgeValue>),
    Range { min: f64, max: f64 },
}

struct KnowledgeAcquisition {
    /// 怎么获知的
    method: AcquisitionMethod,
    /// 来源
    source: Option<SourceId>,
    /// 时间
    timestamp: GameTimestamp,
}

enum AcquisitionMethod {
    DirectObservation,     // 亲眼所见——对物理属性的最高权威
    DirectConversation,    // 与 NPC 直接对话
    Overheard,             // 旁听 NPC 之间的对话
    ReadFromBook,          // 从书中读到
    ReadFromInscription,   // 从碑文/刻字中读到
    FeltViaAether,         // 读树感受到
    CombatEncounter,       // 战斗中发现的
    CraftingResult,        // 制作/炼金过程中发现的
    PlayerDeduction,       // 玩家手动推断标注
}

enum CorrectionSource {
    DirectObservation,
    MultipleSources { source_count: u32 },
    ExpertTestimony { expert: NpcId },
}
```

### 1.3 显示名称规则

```
人物:    NPC 自己说了名字 → 名字。没说 → "?"
         如果 NPC 给了假名 → "?"（Name 字段 Unverified，另有 "自称" 字段）

动物/植物/怪物:
         从 NPC/书中学到名称 → 名称。否则 → "?"
         例: "?" (blue, bell-shaped, cold-glow-on-touch)

地点/聚落:
         从 NPC/书/路牌/地图学到 → 名称。否则 → "?"
         例: "?" (峡谷, 铁炉镇以北2km, 南北走向)

物品:    鉴定后 → 名称。未鉴定 → "?"
         普通物品（铁剑、面包）→ 角色天生知道名称（常识）

魔法/技能:
         看到/经历但不知名 → "?"
         从书中学到/NPC告知 → 名称

事件:    无"正式名称"——display_name 始终是系统自动摘要
         格式: "{地点}·{类型}·{时间}"
         例: "北山·动物袭击事件·春第5日"

知识/传说:
         同事件——display_name 是系统自动摘要
         格式: "{主题}·{来源类型}·{来源}"
         例: "河谷战争·传闻·来源:酒保老陈"
```

## 二、自动记录逻辑

### 2.1 全量记录原则

**所有角色见过的东西都记录。** 不设过滤——玩家经过一片草原时看到的 20 种植物全记。90 小时后回到这里，搜索"蓝色花的植物"定位到霜铃兰——这就是全量记录的价值。

### 2.2 触发条件——按条目类型

```rust
/// 每帧 / 每游戏分钟的检查
fn tick_journal(journal: &mut GrandJournal, player: &Player, world: &WorldState) {
    // ── 人物 ──
    // 触发: 与该 NPC 有过直接对话
    //       在战斗中交手
    //       在近距离（~10m）观察超过 5 秒
    //       从其他 NPC 处听到了关于此人的描述
    for npc in player.nearby_npcs(10.0) {
        if player.has_interacted_with(npc) || player.has_observed_for(npc, 5.0) {
            journal.ensure_entry(EntryType::Person, npc.id);
        }
    }
    
    // ── 地点 ──
    // 触发: 玩家进入该地点（聚落/自然地理点/人工地标/遗迹/特殊地点）
    for location in player.nearby_locations() {
        if player.entered_location(location) {
            journal.ensure_entry(EntryType::Location, location.id);
        }
    }
    
    // ── 聚落 ──
    // 触发: 进入聚落
    if let Some(settlement) = player.current_settlement() {
        journal.ensure_entry(EntryType::Settlement, settlement.id);
    }
    
    // ── 动物/植物 ──
    // 触发: 首次在视野中看到该物种
    for creature in player.visible_creatures() {
        journal.ensure_entry(EntryType::Animal, creature.species_id);
    }
    for plant in player.visible_plants() {
        journal.ensure_entry(EntryType::Plant, plant.species_id);
    }
    
    // ── 怪物 ──
    // 触发: 遭遇/战斗/击败/被击败
    for monster in player.combat_targets() {
        journal.ensure_entry(EntryType::Monster, monster.species_id);
    }
    
    // ── 魔法 ──
    // 触发: 亲眼看到该法术被施放/被该法术影响/从书中学到/亲自施放
    for spell in player.witnessed_spells() {
        journal.ensure_entry(EntryType::Magic, spell.id);
    }
    
    // ── 战斗技能 ──
    // 触发: 学会/使用/看到敌人使用
    for skill in player.observed_skills() {
        journal.ensure_entry(EntryType::CombatSkill, skill.id);
    }
    
    // ── 物品 ──
    // 触发: 获得/鉴定/使用/在他人身上看到重要魔法物品
    for item in player.inventory_items() {
        journal.ensure_entry(EntryType::Item, item.id);
    }
    
    // ── 团体 ──
    // 触发: 与团体成员互动/进入团体控制领地/从 NPC 口中听说
    for group in player.known_groups() {
        journal.ensure_entry(EntryType::Group, group.id);
    }
    
    // ── 政权 ──
    // 触发: 进入政权领土/从 NPC 口中听说/从地图或书中获知
    for polity in player.known_polities() {
        journal.ensure_entry(EntryType::Polity, polity.id);
    }
    
    // ── 事件 ──
    // 触发: 亲身经历/在事件发生后到达现场/从 NPC 或书本中获知
    for event in player.known_events() {
        journal.ensure_entry(EntryType::Event, event.id);
    }
    
    // ── 知识/传说 ──
    // 触发: 从 NPC 口中听到/从书本中读到/从读树中感受到/从碑文中读到
    for knowledge in player.acquired_knowledge() {
        journal.ensure_entry(EntryType::Knowledge, knowledge.id);
    }
}

/// 确保条目存在——不存在则创建
fn ensure_entry(&mut self, entry_type: EntryType, subject_id: WorldObjectId) -> EntryId {
    if let Some(existing) = self.find_entry(entry_type, subject_id) {
        return existing;
    }
    
    let entry = JournalEntry {
        id: EntryId::new(),
        entry_type,
        created_at: current_timestamp(),
        display_name: "?".to_string(),
        knowledge_blocks: self.initial_blocks(entry_type, subject_id),
        related_entries: vec![],
        updates: vec![],
        player_tags: vec![],
        player_notes: vec![],
    };
    
    let id = entry.id;
    self.entries.push(entry);
    id
}
```

### 2.3 信息获取——更新知识块

```rust
/// 角色获取了新信息——更新对应条目的知识块
fn acquire_knowledge(
    journal: &mut GrandJournal,
    entry_id: EntryId,
    field: KnowledgeField,
    value: KnowledgeValue,
    method: AcquisitionMethod,
    source: Option<SourceId>,
) {
    let entry = journal.get_entry_mut(entry_id);
    
    // 查找现有知识块
    if let Some(existing_block) = entry.knowledge_blocks.iter_mut()
        .find(|b| b.field == field)
    {
        // 已有此字段的知识——根据验证规则处理
        let new_acquisition = KnowledgeAcquisition {
            method, source,
            timestamp: current_timestamp(),
        };
        
        match &existing_block.verification {
            VerificationStatus::Unknown => {
                // 首次获知——标记为 Unverified
                existing_block.value = value;
                existing_block.verification = VerificationStatus::Unverified;
                existing_block.acquisition_history.push(new_acquisition);
            }
            VerificationStatus::Unverified => {
                // 已有未验证信息——检查是否印证
                if existing_block.value == value {
                    // 同值——看来源是否独立
                    if is_independent_source(&existing_block.acquisition_history, &new_acquisition) {
                        existing_block.verification = VerificationStatus::Verified;
                    }
                } else {
                    // 不同值——根据新来源的权威性
                    if method == AcquisitionMethod::DirectObservation {
                        // 亲眼所见 → 旧信息证伪
                        existing_block.verification = VerificationStatus::Falsified {
                            previous_value: existing_block.value.clone(),
                            falsified_by: CorrectionSource::DirectObservation,
                            falsified_at: current_timestamp(),
                        };
                        existing_block.value = value;
                    } else {
                        // 两个非直接观察来源矛盾 → 争议
                        let competing = vec![
                            (existing_block.value.clone(), existing_block.acquisition_history.clone()),
                            (value.clone(), vec![new_acquisition.clone()]),
                        ];
                        existing_block.verification = VerificationStatus::Contested {
                            competing_versions: competing,
                        };
                    }
                }
                existing_block.acquisition_history.push(new_acquisition);
            }
            VerificationStatus::Verified => {
                // 已验证——如果新来源和验证值矛盾但来自 DirectObservation
                if value != existing_block.value && method == AcquisitionMethod::DirectObservation {
                    // 亲眼所见优先——即使之前被"验证"过
                    existing_block.verification = VerificationStatus::Falsified {
                        previous_value: existing_block.value.clone(),
                        falsified_by: CorrectionSource::DirectObservation,
                        falsified_at: current_timestamp(),
                    };
                    existing_block.value = value;
                }
                existing_block.acquisition_history.push(new_acquisition);
            }
            VerificationStatus::Falsified { .. } => {
                // 已经被证伪——新信息如果和当前值不同且更权威
                if value != existing_block.value && method == AcquisitionMethod::DirectObservation {
                    // 继续更新
                    existing_block.value = value;
                }
                existing_block.acquisition_history.push(new_acquisition);
            }
            VerificationStatus::Contested { competing_versions } => {
                // 有争议——新信息可能解决争议
                if method == AcquisitionMethod::DirectObservation {
                    existing_block.verification = VerificationStatus::Verified;
                    existing_block.value = value;
                } else {
                    // 增加到竞争版本中
                    competing_versions.push((value, vec![new_acquisition]));
                }
                existing_block.acquisition_history.push(new_acquisition);
            }
        }
    } else {
        // 此字段没有现有知识块——创建新的
        entry.knowledge_blocks.push(KnowledgeBlock {
            field,
            value,
            verification: match method {
                AcquisitionMethod::DirectObservation => VerificationStatus::Verified,
                _ => VerificationStatus::Unverified,
            },
            acquisition_history: vec![KnowledgeAcquisition {
                method, source,
                timestamp: current_timestamp(),
            }],
        });
    }
    
    // 更新 display_name（如果获知了名称）
    if field == KnowledgeField::Name {
        entry.display_name = match &entry.knowledge_blocks.iter()
            .find(|b| b.field == KnowledgeField::Name)
            .unwrap().verification
        {
            VerificationStatus::Verified => value.as_text(),
            VerificationStatus::Unverified => format!("{}?", value.as_text()),
            _ => "?".to_string(),
        };
    }
}
```

## 三、纠错机制

### 3.1 三级纠错模型

```rust
enum CorrectionLevel {
    /// 玩家亲眼所见——最高权威
    /// 触发行为：旧信息 → 已证伪，新信息覆盖
    DirectObservation,
    
    /// 多个可靠来源交叉印证
    /// 触发行为：旧信息 → 存疑，新信息标注多来源印证
    MultipleReliableSources { source_count: u32 },
    
    /// 单一来源 vs 单一来源
    /// 触发行为：两个版本并存，标注"待验证"
    SingleVsSingle,
}

/// 确定纠错级别
fn determine_correction_level(
    new_method: &AcquisitionMethod,
    existing_history: &[KnowledgeAcquisition],
) -> CorrectionLevel {
    match new_method {
        AcquisitionMethod::DirectObservation => {
            CorrectionLevel::DirectObservation
        }
        AcquisitionMethod::ReadFromBook | AcquisitionMethod::DirectConversation => {
            // 计算独立可靠来源数
            let reliable_count = existing_history.iter()
                .filter(|a| a.is_reliable_source())
                .count() + 1;  // +1 包括新来源
            
            if reliable_count >= 3 {
                CorrectionLevel::MultipleReliableSources { source_count: reliable_count as u32 }
            } else if reliable_count >= 1 {
                // 有至少一个可靠来源和旧信息存在——仍不足以完全解决
                CorrectionLevel::SingleVsSingle
            } else {
                CorrectionLevel::SingleVsSingle
            }
        }
        _ => {
            // 旁听/读树/推测——始终需要第三方印证
            CorrectionLevel::SingleVsSingle
        }
    }
}
```

```
  显示为 "?"
```

## 四、搜索

### 4.1 全字段索引

```rust
struct JournalSearchIndex {
    /// 全文本搜索索引——对条目的所有文本字段建立倒排索引
    inverted_index: HashMap<String, Vec<EntryId>>,
    /// 按条目类型索引
    type_index: HashMap<EntryType, Vec<EntryId>>,
    /// 按验证状态索引
    verification_index: HashMap<VerificationStatus, Vec<EntryId>>,
    /// 按标签索引（玩家自己加的标签）
    tag_index: HashMap<String, Vec<EntryId>>,
    /// 按获知方法索引
    acquisition_index: HashMap<AcquisitionMethod, Vec<EntryId>>,
    /// 按位置索引
    location_index: HashMap<LocationId, Vec<EntryId>>,
}

/// 搜索
fn search_journal(
    journal: &GrandJournal,
    query: &str,
    filters: SearchFilters,
) -> Vec<EntryId> {
    let tokens: Vec<String> = query.to_lowercase()
        .split_whitespace()
        .map(|t| t.to_string())
        .collect();
    
    // 从倒排索引中查询
    let mut results: HashMap<EntryId, u32> = HashMap::new();  // entry_id → 命中数
    
    for token in &tokens {
        if let Some(entries) = journal.index.inverted_index.get(token) {
            for entry_id in entries {
                *results.entry(*entry_id).or_insert(0) += 1;
            }
        }
    }
    
    // 按命中数排序
    let mut sorted: Vec<(EntryId, u32)> = results.into_iter().collect();
    sorted.sort_by_key(|(_, hits)| std::cmp::Reverse(*hits));
    
    // 应用过滤器
    sorted.into_iter()
        .map(|(id, _)| id)
        .filter(|id| filters.apply(journal.get_entry(*id)))
        .collect()
}

struct SearchFilters {
    entry_types: Option<Vec<EntryType>>,
    verification: Option<Vec<VerificationStatus>>,
    discovered_after: Option<GameTimestamp>,
    discovered_before: Option<GameTimestamp>,
    has_tag: Option<String>,
    acquisition_method: Option<AcquisitionMethod>,
    /// 仅显示有 ? 未知字段的条目
    has_unknown_fields: bool,
    /// 仅显示有争议的条目
    has_contested: bool,
}
```

### 4.2 搜索响应时间

条目的知识块全文存储 → 文本极少（每条目几百字节）。即使 2000 条目全量扫描，在 2000 × 500B = 1MB 数据上的文本搜索耗时 < 1ms。倒排索引下 < 0.3ms。

### 5.2 系统自动关联

```rust
/// 系统自动发现关联
fn auto_discover_connections(journal: &mut GrandJournal, world: &WorldState) {
    let entries = &journal.entries;
    
    for i in 0..entries.len() {
        for j in (i+1)..entries.len() {
            if let Some(relation) = detect_relation(&entries[i], &entries[j], world) {
                journal.add_bidirectional_edge(entries[i].id, entries[j].id, relation);
            }
        }
    }
}

fn detect_relation(a: &JournalEntry, b: &JournalEntry, world: &WorldState) -> Option<RelationType> {
    // 同一地点的条目 → 关联
    if let (Some(loc_a), Some(loc_b)) = (a.primary_location(), b.primary_location()) {
        if loc_a == loc_b {
            return Some(RelationType::SocialRelation(SocialRelationType::Colocated));
        }
    }
    
    // 同一事件的参与者 → 关联
    if let (Some(event_a), Some(event_b)) = (a.related_event_id(), b.related_event_id()) {
        if event_a == event_b {
            return Some(RelationType::SocialRelation(SocialRelationType::CoParticipant));
        }
    }
    
    // 物品被某人持有 → 关联
    if a.entry_type == EntryType::Item && b.entry_type == EntryType::Person {
        if a.is_owned_by(b) {
            return Some(RelationType::OwnedBy);
        }
    }
    
    // NPC 提及了另一个条目 → 关联
    // 来自 information_propagation 系统的数据
    
    None
}
```

### 5.3 玩家手动关联

```rust
/// 玩家手动连接两条条目
fn player_connect(
    journal: &mut GrandJournal,
    from: EntryId,
    to: EntryId,
    relation_type: RelationType,
    note: Option<String>,
) {
    journal.add_edge(GraphEdge {
        from,
        to,
        relation_type,
        edge_source: EdgeSource::PlayerManual,
        verification: EdgeVerification::PlayerGuess,  // 默认是猜测——直到系统验证
        weight: 0.3,  // 手动关联默认较低权重
    });
    
    if let Some(note) = note {
        journal.get_entry_mut(from).player_notes.push(PlayerNote {
            timestamp: current_timestamp(),
            content: note,
            related_to: Some(to),
        });
    }
}

/// 系统验证玩家的手动关联
fn verify_player_connections(journal: &mut GrandJournal, world: &WorldState) {
    for edge in journal.edges.iter_mut() {
        if edge.edge_source == EdgeSource::PlayerManual 
            && edge.verification == EdgeVerification::PlayerGuess
        {
            // 检查世界状态中是否存在此关联的证据
            if world.has_evidence_for_relation(edge.from, edge.to, &edge.relation_type) {
                edge.verification = EdgeVerification::Verified;
                edge.weight = 0.8;
            }
        }
    }
}
```

### 7.1 条目数量预估

```
100 小时游戏:
  人物:    100-500 条目
  聚落:    10-100 条目
  地点:    50-200 条目
  动物:    30-80 条目
  植物:    50-150 条目
  怪物:    20-50 条目
  魔法:    10-50 条目
  战斗技能: 5-30 条目
  事件:    5-30 条目
  物品:    200-1000 条目（含大量普通物品）
  知识:    50-200 条目
  团体:    10-50 条目
  政权:    5-20 条目
────────────────────
  总计:   ~550-2500 条目

每条目: ~500B（知识块 + 验证状态 + 获知历史 + 关联引用）
2500 × 500B = 1.25 MB

图谱关联边: 平均每条目 3-5 条 × 2500 = 7,500-12,500 条
每条边 ~80B × 12,500 = 1 MB

总计: < 3 MB（运行时内存）
存档: < 2 MB（压缩后）
```

### 7.2 运行时性能

```
搜索: 倒排索引 → < 0.3 ms
图谱展开（一度关系）: < 1 ms
条目详情渲染: < 0.1 ms
自动记录 tick: < 0.1 ms（纯事件驱动——获得信息时触发，非每帧扫描）

总帧预算影响: < 1 ms（仅在大日志打开时有 UI 渲染开销）
```
