## 〇、定位

本文档是对 [[005-大日志系统]] 的 UI 扩展——在已有搜索/图谱基础上新增**五个浏览镜头**。每个镜头是确定性的查询——筛选、排序、聚类。不变动已有数据模型。零新存储、零新 trait。纯 UI 组织层。

**核心原则**：系统只做组织，玩家自己做叙事。不放任何情感词汇，不做任何"故事化"加工。

---

## 一、镜头一：时间线

### 1.1 布局

顶部一条水平时间轴——从角色出生日到当前时间。下方针状滚动区域放条目卡片。

### 1.2 密度纵览

时间轴上方显示灰色柱状图——每天/每月的条目密度。用半透明灰色柱状条，高度 = log(条目数+1)。密度高峰 = 生命的事件密集期。点击高峰区域 → 时间轴跳转到该时段。

### 1.3 间隙分离

连续 30 天以上无新条目 → 时间轴上自动插入一条细灰线分隔。"间隙"不是系统命名的——只是视觉空白。和 SelfNarrative 的 `life_chapters` 对齐——每个 chapter 的起止时间在时间轴上是一段淡彩色背景带。

### 1.4 双角色混排

两个角色（如果双角色存在）的条目可用不同颜色来源标记：
- 角色 A = 暖色系文字，角色 B = 冷色系文字
- 可切换显示：仅 A / 仅 B / 混合
- 时间轴上的间隙只看"两人都无条目"的区间

### 1.5 跳转与快捷

- 双击时间轴上的点 → 条目视图跳到该时间点
- 拖动时间轴缩放条 → 从"十年粒度"缩到"天粒度"
- 搜索框和已有时序视图无缝联动

## 二、镜头二：地图

### 2.1 布局

左上角世界地图（和世界生成的群系/地形地图共享底图——但只有玩家去过的区域有透明度覆盖）。右侧为选中位置的条目列表。

### 2.2 密度热力图

- 玩家去过多次的区域 → 暖色热区（从群系地图底图上叠加一层热力图）
- 只经过一次的 → 冷色散点
- 一眼看出"北山区域探索过多次"vs"西大陆只去过一次"

### 2.3 点击定位点

点击地图上某个位置 → 弹窗显示该位置的所有条目：
- 首次发现：`"?" (峡谷)` → 第二次：`"铁炉峡谷"` → 第三次：`"铁炉峡谷·洪水后"` —— 同一位置的三条条目是时间断层扫描
- 从弹窗可跳转到条目详情

### 2.4 路线连线

相邻时间的两个有位置条目 → 地图上自动连线（灰色细线）。线的粗细 = log(走过次数)。形成旅行路线图。和 NPC 五级道路模型对比——你走过的路，有些被后来者走成了硬化路，有些仍是独属于你的记忆路线。

### 2.5 和已有位置索引的关系

已有 `location_index`（大日志 §4.1）——每条目的位置已索引。此镜头只是换一种呈现方式——从列表变成地图。

## 三、镜头三：人物

### 3.1 布局

左侧 NPC 列表（可滚动）。右侧为选中 NPC 的交互总览卡片。

### 3.2 NPC 交互卡片

每个 NPC 一张卡片：

| 卡片区域 | 内容 | 来源 |
|---------|------|------|
| 顶部 | 名字（或 "?" —— 如果 Name 未验证）+ NPC 类型标签 | `related_entries` × `EntryType::Person` |
| 关键时间 | 首次遇到时间 · 最近一次互动时间 | `created_at` |
| 统计 | 总互动次数 | `related_entries` 计数 |
| 关系 | 一度展开——此 NPC 还与谁关联 | `related_entries` × `RelationType` |
| 关键事件 | 战斗/合作/背叛/婚礼/死亡 —— 高 impact 条目列表 | `impact_score` 排序 |
| 最后状态 | 已故 NPC → 灰色，仍保留。信息不衰减 | NPC 系统死亡数据 |

### 3.3 排序

- **最近互动**（默认）：降序
- **互动次数**：降序
- **认识时长**：降序
- **影响程度**：降序——系统自动算：该 NPC 参与了多少条高 impact 条目

### 3.4 已故牌

NPC 死了 → 卡片变灰，但保留全部内容。大日志不会忘记任何人。你知道这个人死了——但他和你的过往仍然完整可查。

## 四、镜头四：主题聚类

### 4.1 算法

每个条目已有 `tags: Vec<String>`（系统自动生成—— `"dragon"`, `"combat"`, `"trade"`, `"betrayal"`...）+ `player_tags`（玩家手动标签）。

```
全量条目 → TF-IDF 词频-逆文档频率计算
         → K-means 或层次聚类（聚类数自动从肘部法确定）
         → 每个簇选出现频率最高的 3-5 个标签作为簇标签
```

计算量：2500 条目 × ~5 标签 = 12500 个标签。TF-IDF + 聚类 < 1ms。每次打开镜头时惰性计算（条目变更后缓存失效）。

### 4.2 展示

每个主题簇一张卡片：

```
┌──────────────────────────────────────────┐
│ dragon · fire · flight · north_mountain  │  ← 系统标签簇，不做命名
│ ───────────────────────────────          │
│ 12 条条目  ·  春第5天 → 春第347天        │
│ [展开 ▶]                                 │
└──────────────────────────────────────────┘
```

**不做的事**：系统不会把这组命名为"屠龙传奇"、"师徒反目"、"航海时代"。只给标签簇。玩家自己脑子里完成叙事。

### 4.3 跨角色关联

角色 A 的 `"屠龙"` 簇和角色 B 的 `"龙鳞护甲制作"` 簇——如果共享 `"dragon"` 标签，在主题聚类中自动有距离接近，可能被归入同一簇。两个独立条目 + 一个共享标签 = 玩家自己拼出关联。

## 五、镜头五：生命章节

### 5.1 数据来源

SelfNarrative 已缓存 `life_chapters: Vec<LifeChapter>`。每个 chapter 有：
- `title`——NPC/玩家自我认知的归纳（"铁匠岁月"、"流浪者"、"失去一切之后"）
- `start_time` / `end_time`
- `summary`——该阶段的概括

### 5.2 章节卡片

```
┌──────────────────────────────────────────────┐
│ 铁匠岁月                                     │
│ 春第102天 — 秋第1234天  ·  1132 天           │
│ ──────────────────────────────────────       │
│ 条目: 287 条                                 │
│ 人物: 34 人  ·  地点: 12 处                  │
│ 条目类型分布: ████ 事件  ██ 人物  █ 地点      │
│ 重要 NPC: [头像] 老王  [头像] 学徒小刘        │
│ 地理位置: [热力图缩略]                        │
└──────────────────────────────────────────────┘
```

- 章节之间的"间隙"里发生了什么——那条跨越间隙的事件（NPC 的死亡/新地点的到达/某场战斗）自动高亮显示。

### 5.3 双角色模式

两个角色的 `life_chapters` 在同一时间轴上上下并排。你能看到"她在海上流浪的时候，我在苍山城开铁匠铺"——而不需要任何人告诉你这个对比。

## 六、和已有 UI 的关系

已有大日志 UI（搜索框 + 条目列表 + 条目详情 + 图谱）保持不变。五个镜头是**新增的顶部 Tab**：

```
[搜索框: ___________________ 🔍]  [分类▼] [过滤▼]
┌────────────────────────────────────────────────────────────┐
│ [时间线] [地图] [人物] [主题] [生命章节] [搜索] [图谱]      │ ← 新增 Tab 行
├────────────────────────────────────────────────────────────┤
│                                                            │
│   (选中镜头的内容)                                          │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

"搜索" Tab = 原来的条目列表视图。"图谱" Tab = 原来的关系图谱视图。

## 七、性能

| 镜头 | 计算 | 耗时 |
|------|------|------|
| 时间线 | `created_at` 排序（已是主键）+ 密度图采样 | < 0.1ms |
| 地图 | `location_index` 已有 → 热力图渲染 | < 0.2ms |
| 人物 | `related_entries` 遍历 + 分组聚合 | < 0.1ms |
| 主题 | `inverted_index` + TF-IDF + 聚类 | < 1ms per 2500 条目 |
| 生命章节 | `life_chapters` 已缓存 → 直接渲染 | < 0.05ms |

纯 UI 渲染开销 = Godot 侧 8 个 Panel 控件 → < 0.5ms。
