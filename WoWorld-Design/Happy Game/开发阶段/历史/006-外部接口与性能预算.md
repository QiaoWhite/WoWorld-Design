## 一、与信息传播系统的接口

### 1.1 接口总览

```
历史模块 ← 信息传播系统 （单向消费）
  信息传播系统: 查询可传播的历史事件 + 查询物理书籍 + 查询可读痕迹

信息传播系统 → 历史模块 （不反向引用）
  传播触发新事件 → 新事件由历史模块的运行时触发器捕获，
  信息传播系统不直接写入 event_log
```

### 1.2 PropagableEventSource

```rust
/// 历史模块暴露给信息传播系统的 trait
/// 信息传播系统调用——拉取模式
trait PropagableEventSource {
    /// 查询某个时间点之后、指定区域内、满足传播条件的事件
    fn query_propagable_events(
        &self,
        since: GameTimestamp,
        region: RegionId,
        min_intensity: f32,
    ) -> Vec<PropagableEvent>;
}

struct PropagableEvent {
    event_id: EventId,
    event_type: EventType,
    
    /// 传播数据包——纯参数，无叙事
    propagation_payload: PropagationPayload,
    
    /// 事件位置——决定传播起点
    location: LocationId,
    
    /// 事件强度——决定初始传播速度和人数
    intensity: f32,
    
    /// 参与者 NPC ID 列表——信息传播系统用做初始传播者
    participant_npc_ids: Vec<NpcId>,
}

struct PropagationPayload {
    /// 核心事实（参数化的键值对）
    /// "war_declaration": "冰冠王国→七河联盟"
    /// "date": "1423年6月"
    /// "location": "河谷地区"
    core_facts: HashMap<String, String>,
    
    /// 定量数据（信息传播系统应用 ScaleDistortion 的基础）
    /// "casualties": 3000.0
    /// "area_affected_km2": 500.0
    /// "duration_years": 3.0
    quantitative_data: HashMap<String, f64>,
    
    /// 参与者角色标签
    /// (NpcId, "aggressor"/"defender"/"victim"/"instigator")
    participant_roles: Vec<(NpcId, String)>,
}
```

/// 世界生成 P9 调用历史模块的主入口

trait HistorySimulator {

    /// world: 已完成 P0-P8 初始化的世界状态（NPC 已放置、聚落已生成）

### 2.3 世界生成 v2.0 初始化接口 ★ CHG-045 新增

> **详细规格**: [[../../../参考文档/037-世界生成重构跨模块补充需求-20260620/006-历史系统补充需求|037-006 历史系统补充需求]]

v2.0 重构后，历史模块新增以下世界生成可调用的接口：

**KnowledgeSeed TOML 注册表**: 手工设计的技术时代元数据。`[[era]]` 定义时代，`[[era.tech]]` 定义技艺（discovery: Inevitable/Conditional/Breakthrough, propagation: Fast/Slow/VerySlow）。TOML 数据驱动——新增时代无需改代码。

**generate_pre_history()**: `generate_pre_history(knowledge_seeds, settlements, npcs, families, power_topology, culture, years, rng) → (EventLog, SettlementTechState, WorldStateDelta)` — 两步走: P9a KnowledgeSeed 传播→SettlementTechState(P8 技能天花板)；P9b 因果事件图+参与者分配→EventLog+WorldStateDelta。

**SettlementTechState**: 传播最终产物——每聚落每技艺的当前最高水平(0-100)，含传入年份和传播来源。P8 消费此类型作为 NPC 技能天花板。

**WorldStateDelta**: P9b 产出的副作用报告——`buildings_damaged`, `settlements_abandoned`, `mass_death_sites`, `ruin_sites`。P13 终组阶段应用。

### 8.5 风险点与缓解

| 风险 | 概率 | 影响 | 缓解策略 |
|------|------|------|---------|
| 因果链爆炸性分支（超过 50K 事件） | 低 | 生成时间超限 + LMDB 存储超预算 | 硬上限 50K 事件 + 递归深度限制 + 种子参数调节 |
| 生命痕迹生成峰值内存 > 50MB | 中 | 预生成期间 OOM | 分批生成，每 10K 批次写入 LMDB 后释放内存 |
| LMDB 写入放大（大量痕迹写入） | 中 | 磁盘 I/O 瓶颈 | 批量事务提交（每 1000 条一个事务）|
| 力量检测复杂度 O(P²) 膨胀（政权数 > 100） | 低 | 月 tick 耗时 > 1ms | v1.0 假设 ≤50 政权，v1.1 加入空间哈希优化 |
| 大日志条目爆炸（玩家 500+ 小时） | 低 | > 5000 条目，搜索变慢 | 搜索仍然是 < 1ms（倒排索引对 10K 条目无压力）|
