## 定位

本文件夹存放 WoWorld 历史系统的**正式开发规格**。历史系统不是一套手写的叙事内容——它是**事件记录与因果生成引擎、物理化历史痕迹系统、以及玩家知识管理工具**的组合。它回答三个核心问题：

1. **世界过去发生了什么**——预生成历史（种子确定性的因果链事件），玩家进入后新事件的实时记录
2. **历史怎么嵌入世界**——通过废墟、文物、书籍、碑文、灵元素印记、关系遗产等管道让玩家感知
3. **玩家怎么记住和理解**——大日志作为纯功能性外置大脑，全量记录、渐进验证、关系图谱

**核心原则**：
- **只创造规则，不创造故事**。事件由因果链涌现，叙事由 NPC 和玩家拼凑
- **历史是分层的**。event_log 是世界真相，书籍/NPC 传说/读树是它的不完全视图
- **大日志纯粹**。不做叙事加工，不做世界观灌输——只客观记录玩家角色知道什么

## 关键概念速览

```
┌─────────────────────────────────────────────────────────────┐
│                      历史系统架构                            │
│                                                              │
│  event_log (世界真相, 30K-40K 事件)                           │
│    ├── 预生成: 种子→因果链→0-2000年历史模拟                    │
│    ├── 运行时: 冲击型触发器 + 累积型触发器 + 命运种子          │
│    └── 因果图: 趋势层→力量层→事件层 (有环有向)                │
│                                                              │
│  痕迹系统 (玩家可感知的历史)                                  │
│    ├── 生命痕迹: 墓碑/日记/刻字/遗物/契约/供品/秘藏          │
│    ├── 团体档案: 家族/商会/佣兵团半形式化记录                 │
│    ├── 书籍著作: Work→PhysicalBook·抄写链·流转历史           │
│    ├── 灵元素印记: 空间残留·读树                              │
│    ├── 文物痕迹: 魔法物品的历史包装层                          │
│    └── 关系遗产: 世仇/联盟/债务/附庸的因果起源                │
│                                                              │
│  大日志 (玩家的外置大脑)                                      │
│    ├── 全量自动记录 (所见所闻→条目)                            │
│    ├── 渐进验证模型 (? → 验证 → 证伪/争议)                   │
│    ├── 关系图谱 (双向链接·手动关联·自动发现)                   │
│    └── 不直接暴露 event_log                                   │
│                                                              │
│  历史嵌入世界 (玩家用眼睛感知)                                │
│    ├── 废墟/建筑老化 (视觉——直接看到)                         │
│    ├── 文物 (物品——拾取/查看)                                 │
│    ├── 书本 (阅读——深度了解)                                  │
│    ├── 读树 (交互——意象碎片)                                  │
│    ├── 关系遗产 (间接——观察社会格局)                          │
│    └── NPC 口头传说 (对话——闲聊/打听) → 走信息传播系统         │
└─────────────────────────────────────────────────────────────┘
```

## 关键参数速查

| 参数 | 值 | 说明 |
|------|-----|------|
| **预生成事件数** | 30K-40K | event_log 总条目，由因果链生成 |
| **根事件数** | 5-20（种子决定） | 因果链的起点——世界的"历史骨架" |
| **历史模拟跨度** | 0-2000 年（种子决定） | 世界生成 P9 |
| **模拟步长** | 1 月 | 历史模拟 tick 粒度 |
| **因果递归深度上限** | 12 层 | 概率衰减自然截断，硬上限 |
| **事件总数硬上限** | 50K | 防止因果链爆炸性分支 |
| **生命痕迹（预生成）** | 200K-500K 条 | 全图散布的物理痕迹 |
| **书籍（预生成）** | v1.0：~147 个 Work，~150-700 个 PhysicalBook（单副本为主）；v1.1：500-2000 Work（多副本链+抄写传播） |
| **灵元素印记** | ~9,000 个 | 仅 intensity > 0.5 的事件产生 |
| **事件存储（LMDB）** | ~25 MB | event_log + 因果边 + 索引 |
| **痕迹存储（LMDB）** | ~30-60 MB | 生命痕迹 |
| **总存储（LMDB）** | ~100-120 MB | 全部历史数据 |
| **预生成总时间** | ~3-7 秒 | 一次性（玩家点"新游戏"时） |
| **运行时月 tick** | < 1 ms | 独立后台线程 |
| **运行时热数据内存** | < 5 MB | 玩家附近热点缓存 |

## 与其他模块的关系

| 模块 | 历史模块消费什么 | 历史模块提供什么 |
|------|----------------|----------------|
| **世界生成** | 世界状态（地形/聚落/人口/资源） | 历史模拟（P9）→ 输出带历史痕迹的世界 |
| **NPC 活人感** | NPC 属性（人格/技能/记忆/关系） | 事件参与者信息、团体档案 |
| **植物** | 植物灵元素亲和力、读树交互 | AetherImprint 查询接口 |
| **魔法** | MagicItem（物品引用） | ArtifactTrace（物品的历史包装） |
| **信息传播** | — | PropagableEventSource / PhysicalBookSource / ReadableTraceSource |
| **大日志** | — | event_log（间接——通过正常游戏渠道转译） |
