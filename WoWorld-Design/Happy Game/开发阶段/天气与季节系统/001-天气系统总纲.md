# 001 — 天气系统总纲

## 一、设计哲学

### 1.1 只创造规则，不创造故事

天气系统是 WoWorld 底层物理规则的一部分——与地形、海洋、重力同级。它不定义"暴风雨那天 NPC 做了什么"，只定义"这里现在下着中雨，温度12°C，风速4m/s"。NPC 的情绪反应、植物的生长加速、战斗的视野缩减——全都是其他模块基于天气数据**自行涌现**的结果。

```
天气系统的职责边界:

✅ 天气系统做:
├─ 回答"某时某地的天气是什么？"——提供客观物理事实
├─ 管理季节循环——全球统一时钟
├─ 生成极端天气——从气候参数自然涌现
├─ 输出视觉渲染参数——云/降水/雾/天空色调
└─ 提供历史气象数据——供历史模块转化为事件

❌ 天气系统不做:
├─ NPC 对天气的主观感知 → NPC 模块（SkyPerception）
├─ 植物生长/枯萎/开花的具体逻辑 → 植物模块
├─ 动物迁徙的触发决策 → 动物模块
├─ 战斗中的元素环境联动 → 战斗模块 + 魔法模块
├─ 天气粒子的渲染 → Godot 渲染层
├─ 天气音效的播放 → Godot 音频层
├─ 天气事件的叙事转化 → 历史模块
└─ 体素密度场的写入 → 世界生成模块
```

### 1.2 统一轮询，零事件总线

所有消费方通过唯一的 `WeatherQuery` trait 同步查询天气数据——不发布事件、不维护订阅。理由：

- 天气变化速度（分钟级）远慢于消费方轮询频率（帧级到秒级）
- 天气过渡是连续的（30-60秒 blend）——轮询天然匹配连续过渡
- 轮询结果 = 纯函数 `(pos, time) → WeatherSample` ——完全确定性
- 无事件基础设施 → 代码简洁、调试清晰、无时序耦合

唯一的特殊处理是历史模块——每月 tick 一次，可能错过月内发生-结束的短期极端天气。通过 `extreme_events_since()` 批量查询补足。

### 1.3 基础设施，不是玩法

与物品系统、技能系统相同：天气系统只定义**机制**（天气怎么生成、季节怎么循环），不定义**内容**（雨天能做什么是其他模块的事）。

## 三、天气空间结构

25万km² 的世界无法用单一天气状态覆盖。采用**两层**模型：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
              天气空间结构
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ClimateRegion (~50km grid)
├─ 数量: ~100 (25万km² / 2500km²)
├─ 从世界生成气候参数场降采样
├─ 产出: 区域天气基线 (温度/湿度/盛行风向/季节性幅度)
├─ 更新频率: 每游戏日
└─ 存储: 预计算查找表 (~100区域 × 64 bytes = 6.4KB)

LocalWeather (~1km)
├─ 从所在 ClimateRegion 双线性插值
├─ 叠加: TerrainModifier + BiomeMicroclimateModifier
├─ 运行: Markov 天气状态机（每游戏分钟 tick）
├─ 产出: WeatherSample (栈上 Copy 类型，~120 bytes)
└─ 更新频率: 每游戏分钟 (Markov tick) / 每次 WeatherQuery::sample() (插值+修正)

砍掉的设计:
└─ 全球气压系统层 — 玩家永远感知不到"高压中心向东移动"。
   物理准确性 ≠ 游戏价值。"山那边在下雨，这边晴天"就是玩家需要的一切。
```

`WeatherSample` 是栈上的 Copy 类型（~120 bytes），每次轮询就是一次内存读取——130个消费方（NPC×100 + 植物×10 + …）每秒轮询数次，全部命中 L2 cache。零分配、零锁。

## 四、核心数据模型

### 4.1 WeatherSample — 天气系统唯一输出

```rust
/// 某时某地的完整天气样本
/// 天气系统对外的唯一数据合同
/// 所有模块通过 WeatherQuery::sample(pos, time) 获取
struct WeatherSample {
    // ═══════════════════════════════════════════════
    // 温度（双层）
    // ═══════════════════════════════════════════════
    /// 大气温度 °C — 区域基准，不考虑地表修正
    /// 用途: NPC 在高程位置（塔楼/飞行）自选此温度
    regional_temperature: f32,
    /// 地面温度 °C — 考虑冠层遮阳/雪面反照/沙地辐射/水体缓冲/城市热岛
    /// 用途: 植物生长/动物体温/NPC 地面活动
    ground_temperature: f32,

    // ═══════════════════════════════════════════════
    // 湿度（双层）
    // ═══════════════════════════════════════════════
    /// 区域大气相对湿度 0-1
    regional_humidity: f32,
    /// 地面局地湿度 0-1 — 考虑蒸腾/沼泽/水体邻近
    ground_humidity: f32,

    // ═══════════════════════════════════════════════
    // 降水
    // ═══════════════════════════════════════════════
    precipitation_type: PrecipitationType,
    /// 降水强度 0-1 — 连续谱，非离散档位
    /// 消费方自行映射到自己的档位: 战斗 visibility = f(intensity)
    precipitation_intensity: f32,
    /// 降水变化趋势 -1.0~+1.0
    /// 负值 = 趋向转晴，正值 = 趋向降水加剧
    /// NPC 用此值产生"感觉要下雨了"的直觉
    precipitation_trend: f32,

    // ═══════════════════════════════════════════════
    // 风（双层）
    // ═══════════════════════════════════════════════
    /// 区域大气风速 m/s — 海洋/载具/风暴生成 消费
    regional_wind_speed: f32,
    /// 地表有效风速 m/s — 考虑冠层/建筑遮挡
    /// NPC/植物/技能检定 消费
    ground_wind_speed: f32,
    /// 风向单位向量 (x, z)
    wind_direction: Vec2,
    /// 阵风因子 0-1 — 叠加在基础风速上的瞬时波动
    gust_factor: f32,

    // ═══════════════════════════════════════════════
    // 天空
    // ═══════════════════════════════════════════════
    /// 云覆盖率 0-1
    cloud_cover: f32,
    /// 云类型 — 积云/层云/积雨云/卷云
    cloud_type: CloudType,
    /// 有效水平视距 m — 考虑降水/雾/沙尘
    visibility_m: f32,

    // ═══════════════════════════════════════════════
    // 地面状态
    // ═══════════════════════════════════════════════
    surface_state: SurfaceState,

    // ═══════════════════════════════════════════════
    // 雷电
    // ═══════════════════════════════════════════════
    /// 当前天气下的雷击基准概率 0-1
    lightning_risk: f32,
    /// 最近一次雷击的精确时间和位置
    /// Godot 渲染/音频 每帧检查此字段是否变化 → 触发闪光/雷声
    lightning_last_strike: Option<(GameTime, Vec3)>,

    // ═══════════════════════════════════════════════
    // 极端天气
    // ═══════════════════════════════════════════════
    is_extreme_weather: bool,
    extreme_weather_type: Option<ExtremeWeatherType>,
    /// 暴风雨锋面距离 km — 无暴风雨时 = f32::MAX
    /// NPC: distance < 5km → "暴风雨要来了"感知
    storm_front_distance: f32,

    // ═══════════════════════════════════════════════
    // 稳定性
    // ═══════════════════════════════════════════════
    /// 天气稳定性 0-1
    /// 1.0 = 当前状态极稳定（如持续晴天）
    /// 0.0 = 极不稳定（即将变化）
    /// NPC: stability < 0.3 + precipitation_trend > 0.5 → "天要变了"
    weather_stability: f32,

    // ═══════════════════════════════════════════════
    // 天象（来自 SeasonClock）
    // ═══════════════════════════════════════════════
    /// 雨后彩虹是否可见
    rainbow_visible: bool,
    /// 太阳高度角 rad — 纬度+时间函数
    solar_elevation: f32,
}
```

**大小**: ~120 bytes。栈上 Copy。零分配轮询。

### 4.1.1 WindSample — 风场采样 (CHG-033 权威定义)

> **[CHG-033 迁移 — 2026-06-18]**: `wind_at()` 从 SpatialQuery (模型动作与物理系统) 迁移至 WeatherQuery。天气系统是风场的权威 Owner——风是物理大气现象，空间查询模块不应拥有风场定义。模型动作与物理模块通过 WeatherQuery::wind_at() 消费风场数据。

```rust
/// 风采样 — WeatherQuery::wind_at(pos, time) 返回
/// 消费方: 模型动画(布料/头发摆动)、载具(帆船)、魔法(气流效果)、音频(风噪)
struct WindSample {
    /// 风向单位向量 (3D)
    direction: Vec3,
    /// 风速 m/s
    speed: f32,
    /// 阵风因子 0-1 — 叠加在基础风速上的瞬时波动
    gust_factor: f32,
}
```

### 4.2 枚举定义

```rust
/// 降水类型 — 由温度和天气状态决定
enum PrecipitationType {
    None,           // 无降水
    Rain,           // 雨（温度 > 2°C）
    FreezingRain,   // 冻雨（温度 -2~2°C）
    Snow,           // 雪（温度 < 2°C）
    Hail,           // 冰雹（强对流，夏季）
    Dust,           // 沙尘（干旱 + 强风）
}

/// 地面状态 — 由降水历史 + 温度推导
enum SurfaceState {
    Dry,            // 干燥
    Wet,            // 潮湿 — 雨后/露水
    Ice,            // 冰面 — Wet + 温度<0°C
    Snow,           // 雪盖 — 积雪累积
    Mud,            // 泥泞 — Wet + 土壤 + 持续降水
}

/// 云类型 — 对应技术栈 v3 的 CloudType
enum CloudType {
    Cumulus,        // 积云 — 晴天散云
    Stratus,        // 层云 — 阴天均云
    Cumulonimbus,   // 积雨云 — 暴风雨前兆
    Cirrus,         // 卷云 — 高空
}

/// 极端天气类型 — 不新增天气状态，参数组合自然区分
/// 历史模块可用此标签做叙事命名（如"大台风"）
enum ExtremeWeatherType {
    Storm,          // 暴风雨 — HeavyStorm + 风速>20m/s
    Blizzard,       // 暴风雪 — HeavyStorm + 雪 + 风速>15m/s + 温度<-5°C
    Typhoon,        // 台风 — HeavyStorm + 风速>30m/s + 海岸区域
    DustStorm,      // 沙尘暴 — 沙尘 + 风速>12m/s + 干旱带
    Hailstorm,      // 冰雹暴 — 强对流 + 温度梯度大
    Heatwave,       // 热浪 — 温度>夏季95%分位 + 持续>36游戏时
    ColdSnap,       // 寒流 — 温度<冬季5%分位 + 持续>24游戏时
}
```

### 4.3 天气状态机状态

```rust
/// Markov 链的 6 个核心状态
/// 渐变——不突然从晴跳到暴风雨
enum WeatherState {
    Clear,              // 晴 — 云量<0.2
    PartlyCloudy,       // 多云 — 云量0.2-0.5
    Overcast,           // 阴 — 云量>0.5，无降水或微量
    LightPrecip,        // 小雨/小雪 — 降水强度<0.3
    ModeratePrecip,     // 中雨/中雪 — 降水强度0.3-0.6
    HeavyStorm,         // 暴风雨/暴风雪 — 降水强度>0.6
}

/// 雾 — 独立布尔维度，可与任意 WeatherState 共存
/// 雾不影响 Markov 转移，由湿度+温差独立触发
struct FogState {
    is_foggy: bool,
    fog_density: f32,       // 0-1
    fog_type: FogType,
}

enum FogType {
    Radiation,      // 辐射雾 — 晴天清晨，低洼地带
    Advection,      // 平流雾 — 暖湿空气经过冷水面
    Upslope,        // 上坡雾 — 山脉迎风坡
    SeaFog,         // 海雾 — 海岸区域
}
```

## 五、SeasonClock 架构

`SeasonClock` 是纯时间函数——输入 `GameTime`，输出 `SeasonState`。无状态、无存储。

```rust
struct SeasonClock {
    /// 游戏天时长（实时秒） [TUNING]
    day_duration_seconds: f32,      // 2880 (48分钟)
    /// 一年天数 [TUNING]
    days_per_year: u32,             // 120
    /// 每季天数（均匀四季）
    days_per_season: u32,           // 30
    /// 世界诞生的年号（种子决定——虚数）
    world_birth_year: u64,
    /// 初始季节偏移（0=春分开局）
    initial_season_day: u32,        // 0（春分第1天）
}

/// 季节状态 — 纯时间函数输出
struct SeasonState {
    season: Season,                 // Spring/Summer/Autumn/Winter
    day_of_season: u32,             // 1-30
    day_of_year: u32,               // 1-120
    year_number: u64,               // 世界诞生至今（虚数 + 玩家进入后递增）
    season_progress: f32,           // 0-1 当前季节内的进度
    year_progress: f32,             // 0-1 年进度
}

enum Season {
    Spring,     // 春 — 万物复苏
    Summer,     // 夏 — 对流活跃
    Autumn,     // 秋 — 收获季节
    Winter,     // 冬 — 低温休眠
}
```
