## 一、核心概念

### 1.1 组件族（ComponentFamily）

**组件族**是一个参数化的建筑组件的抽象模板。一个 `ComponentFamily` 定义「这是什么类型的组件」，而 `ComponentParams` 定义「这个具体实例的参数」。

关键设计决策：**组件族使用关联类型（associated type）**，每个族定义自己的 `Params` 类型。这支持 Mod 扩展——新族不需要修改核心 crate 的 enum。

```rust
/// 组件族的 trait —— 所有组件类型的抽象
pub trait ComponentFamily: Send + Sync + 'static {
    /// 此族的全局唯一标识
    fn family_id(&self) -> ComponentFamilyId;

    /// 此族的显示名（用于 Blueprint TOML 引用："Wall"、"Door"）
    fn family_name(&self) -> &str;

    /// 根据 BuildContext 派生此族在此语境下的默认参数
    fn default_params(&self, ctx: &BuildContext) -> Self::Params;

    /// 声明此组件的连接面（只声明实际使用的面，非六面全必须）
    fn connection_faces(&self, params: &Self::Params) -> Vec<ConnectionFace>;

    /// 此组件在此语境下是否可用
    fn is_available(&self, ctx: &BuildContext) -> bool;

    /// 参数 schema（供 Blueprint 验证和 TOML 反序列化用）
    fn params_schema(&self) -> ParamSchema;

    /// 参数验证
    fn validate_params(&self, params: &Self::Params) -> Result<(), Vec<ParamError>>;

    /// WFC tile 映射——此族在 WFC 格中的 tile 表示
    fn wfc_tiles(&self, ctx: &BuildContext) -> Vec<WfcTileVariant>;
}

/// 组件参数的 marker trait
pub trait ComponentParams: Serialize + DeserializeOwned + Clone + Send + Sync + 'static {
    /// 从此参数派生空间尺寸
    fn dimensions(&self) -> Dimensions;

    /// 此组件使用的材质类别
    fn material_classes(&self) -> Vec<MaterialClass>;

    /// 渲染 mesh 标识
    fn mesh_key(&self) -> MeshKey;
}
```

### 1.2 网格系统

**网格定义对齐精度，不限制组件尺寸。**

| 层级 | 网格尺寸 | 用途 |
|------|----------|------|
| **主网格** | 1.0m | 墙、地板、天花板、柱子、门框——结构组件 |
| **子网格** | 0.25m | 薄墙、窗框、栏杆、楼梯台阶——精细结构组件 |
| **自由层** | 无网格 | 家具旋转、装饰品、灯具——非结构装饰 |

组件占位必须是主网格或子网格的整数倍。一个「石墙组件」可以是 1m×3m×0.5m（宽×高×厚），占据 4×12 个子网格。

**为什么参数化族而非离散变体**：核心结构组件（墙、地板、门、窗、柱、梁、楼梯）用参数化族——尺寸变化是连续的（受种族体型、文化规范影响），不可能枚举所有文化×种族组合。装饰/特色组件用离散定义。

### 1.3 连接面（ConnectionFace）

连接面是组件之间的物理连接点。不是六面全必须——每个族只声明实际使用的面。

```rust
pub struct ConnectionFace {
    /// 此面在组件局部坐标中的方位
    pub face: FaceDir,

    /// 面的尺寸
    pub area: (f32, f32),

    /// 连接类型
    pub connection_type: ConnectionType,

    /// 此面在局部坐标中的偏移
    pub offset: Vec3,
}

pub enum ConnectionType {
    /// 承重连接——墙接地板、梁接柱。载荷必须通过此面传播。
    Structural { load_bearing: f32 },

    /// 非承重连接——窗嵌入墙、装饰挂墙。不传播载荷。
    Attachment,

    /// 穿越连接——门在墙上、楼梯穿过楼层。
    PassThrough,
}

pub enum FaceDir {
    PosX, NegX,
    PosY, NegY,  // PosY = 顶面, NegY = 底面
    PosZ, NegZ,
}
```

**兼容性规则**：两个 ConnectionFace 能连接当且仅当：
1. `face_a == opposite(face_b)`（面相对）
2. `connection_type` 匹配（Structural ↔ Structural, Attachment ↔ Attachment, PassThrough 一个提供面一个接受）
3. 面积偏差 ≤ 20%
4. 位置偏差 ≤ 0.25m（子网格 snap 容差）

连接兼容性从 ComponentRegistry 自动推导为兼容矩阵——无需手写。

## 二、核心组件族

### 2.1 Wall（墙）——最基础的围护结构

```rust
pub struct WallParams {
    pub width: f32,        // 沿墙面的水平长度
    pub height: f32,       // 垂直高度
    pub thickness: f32,    // 墙面法线方向的厚度
    pub material: MaterialClass,
    pub wall_type: WallType,
    pub style: WallStyle,
}

pub enum WallType {
    Exterior,     // 外墙——一面朝外(Empty)，一面朝内(Floor)
    Interior,     // 内墙——两面都朝室内
    Retaining,    // 挡土墙——嵌入地形
}

pub enum WallStyle {
    PlankHorizontal, PlankVertical,
    BlockRough, BlockSmooth,
    BrickRunning, BrickFlemish,
    AdobeSmooth, WattleAndDaub,
    LogHorizontal,
}
```

**连接面（以 ExteriorWall 为例）**：
- 底面（NegY）：Structural → 接地板/地基
- 顶面（PosY）：Structural → 接天花板/上层墙/屋顶梁
- 左面（NegX）：Structural → 接相邻墙或柱
- 右面（PosX）：Structural → 接相邻墙或柱
- 前面（NegZ 或 PosZ，取决于朝向）：无声明连接面（此面是外立面或内立面——可供 Attachment 嵌入窗/门/装饰）

**尺寸派生**（从 BuildContext）：
```rust
fn default_params(ctx: &BuildContext) -> Self::Params {
    WallParams {
        thickness: ctx.climate.wall_thickness(0, 1),  // 单层建筑壁厚
        material: ctx.culture.preferred_structural_material(),
        wall_type: WallType::Exterior,
        style: ctx.culture.wall_style_for(material),
        width: 1.0,   // WFC 后处理时合并相邻 cell 决定实际宽度
        height: ctx.race.ceiling_height(),  // 人类2.5m, 兽人3.5m, 矮人2.1m
    }
}
```

### 2.2 Floor（地板/楼板）

```rust
pub struct FloorParams {
    pub area: (f32, f32),    // 长×宽
    pub thickness: f32,
    pub material: MaterialClass,
    pub floor_type: FloorType,
}

pub enum FloorType {
    GroundFloor,    // 接触地面——需要防潮
    UpperFloor,     // 上层楼板——需要梁支撑
    Ceiling,        // 天花板——仅装饰
}
```

**连接面**：
- 底面（NegY）：Per FloorType——GroundFloor → Structural（接地基），UpperFloor → Structural（接下层墙/梁）
- 顶面（PosY）：Structural → 接上层墙/家具（提供 PlacementSurface）

### 2.3 Door（门）

```rust
pub struct DoorParams {
    pub width: f32,
    pub height: f32,
    pub material: MaterialClass,
    pub door_type: DoorType,
    pub swing: SwingDir,
}

pub enum DoorType {
    Single,        // 单扇门
    Double,        // 双扇门（宫殿/豪宅）
    Arch,          // 拱门（无门扇）
    Portcullis,    // 吊闸门（城堡）
    Trapdoor,      // 地板门（通往地下室）
}

pub enum SwingDir { Inward, Outward, Both, None }
```

**关键**：门的 `default_params` 中 `height` 由 `BuildContext.race.avg_height` 派生——兽人门 3.0m+，矮人门 1.8m。`width` 同理。
门不独立存在——在 WFC 后处理中嵌入相邻的 Wall。ConnectionFace 类型 = PassThrough。

### 2.4 Window（窗）

```rust
pub struct WindowParams {
    pub width: f32,
    pub height: f32,
    pub material: MaterialClass,
    pub window_type: WindowType,
}

pub enum WindowType {
    NarrowSlit,     // 防御性窄缝（城堡）
    Standard,       // 标准窗
    Wide,           // 大窗（开放文化）
    Bay,            // 凸窗
    Round,          // 圆窗（精灵/魔法风格）
    ArrowSlit,      // 箭缝（军事建筑）
}
```

窗的尺寸和密度受文化隐私规范和气候双重影响——沙漠文化+防御文化 → 窗极小且少；开放文化+温带 → 大窗多。

### 2.5 Beam（梁）

```rust
pub struct BeamParams {
    pub length: f32,
    pub cross_section: (f32, f32),  // 宽×高
    pub material: MaterialClass,
    pub beam_type: BeamType,
}

pub enum BeamType {
    FloorBeam,      // 支撑楼板
    RoofBeam,       // 支撑屋顶
    Lintel,         // 门窗过梁
    TieBeam,        // 拉梁——连接相对墙
}
```

跨度最大 = material.strength × cross_section.1² / (load × safety_factor)。跨度超限时 WFC 自动插入 Column。

### 2.7 Stairs（楼梯）

```rust
pub struct StairsParams {
    pub floor_from: i8,
    pub floor_to: i8,
    pub width: f32,
    pub material: MaterialClass,
    pub stair_type: StairType,
}

pub enum StairType {
    Straight,       // 直梯
    LShaped,        // L 形
    Spiral,         // 螺旋
    Ladder,         // 简易梯子
}
```

楼梯是垂直连接件——ConnectionFace::PassThrough 连接 floor N 和 floor N+1。`step_height` 从 `BuildContext.race.avg_height` 派生（矮人步高 0.13m，兽人步高 0.28m）。

### 2.8 Foundation（地基）

```rust
pub struct FoundationParams {
    pub width: f32,
    pub length: f32,
    pub depth: f32,
    pub foundation_type: FoundationType,
    pub material: MaterialClass,
}

pub enum FoundationType {
    Strip,          // 条形基础（普通建筑）
    Slab,           // 板式基础（重型建筑）
    Pile,           // 桩基（软土/永冻土/水上）
    Stilts,         // 吊脚（洪水区/红树林）
    TreePlatform,   // 树冠平台（精灵书屋）
    Caisson,        // 沉井——桥墩

}
```

FoundationType 由 `BuildContext.climate` 和 `SiteConstraints` 共同决定。

### 2.9 Roof（屋顶）

屋顶不在 WFC 格中生成——是后处理添加。但它仍是 ComponentFamily——有自己的参数。

```rust
pub struct RoofParams {
    pub roof_type: RoofStyle,    // Gable, Hip, Flat, Dome, Spire, Mansard
    pub pitch: f32,              // 坡度（度）——雪荷载驱动：雪大→≥30°，沙漠→≤5°
    pub material: MaterialClass,
    pub overhang: f32,           // 屋檐深度——降水量驱动
}
```

## 三、一族一例——WallFamily 完整实现

```rust
pub struct WallFamily {
    id: ComponentFamilyId,
}

impl ComponentFamily for WallFamily {
    type Params = WallParams;

    fn family_id(&self) -> ComponentFamilyId { self.id }

    fn family_name(&self) -> &str { "Wall" }

    fn default_params(&self, ctx: &BuildContext) -> Self::Params {
        let thickness = wall_thickness(0, 1, ctx);  // (floor, total_floors, ctx)
        let material = ctx.culture
            .map(|c| c.preferred_structural_material())
            .unwrap_or(MaterialClass::Stone);

        WallParams {
            width: 1.0,          // WFC 合并后填充
            height: ctx.race.ceiling_height(),
            thickness,
            material,
            wall_type: WallType::Exterior,
            style: ctx.culture.map(|c| c.wall_style_for(material))
                .unwrap_or(WallStyle::BlockRough),
        }
    }

    fn connection_faces(&self, params: &Self::Params) -> Vec<ConnectionFace> {
        let half_w = params.width / 2.0;
        let half_t = params.thickness / 2.0;
        let h = params.height;
        vec![
            ConnectionFace {
                face: FaceDir::NegY,
                area: (params.width, params.thickness),
                connection_type: ConnectionType::Structural {
                    load_bearing: params.material.compressive_strength() * params.thickness * params.width,
                },
                offset: Vec3::new(0.0, 0.0, 0.0),
            },
            ConnectionFace {
                face: FaceDir::PosY,
                area: (params.width, params.thickness),
                connection_type: ConnectionType::Structural {
                    load_bearing: params.material.compressive_strength() * params.thickness * params.width,
                },
                offset: Vec3::new(0.0, h, 0.0),
            },
            ConnectionFace {
                face: FaceDir::NegX,
                area: (params.height, params.thickness),
                connection_type: ConnectionType::Structural {
                    load_bearing: params.material.shear_strength() * params.height * params.thickness,
                },
                offset: Vec3::new(-half_w, 0.0, 0.0),
            },
            ConnectionFace {
                face: FaceDir::PosX,
                area: (params.height, params.thickness),
                connection_type: ConnectionType::Structural {
                    load_bearing: params.material.shear_strength() * params.height * params.thickness,
                },
                offset: Vec3::new(half_w, 0.0, 0.0),
            },
        ]
    }

    fn is_available(&self, ctx: &BuildContext) -> bool {
        // Wall 几乎总是可用——但某些文化可能偏爱其他围护结构
        true
    }

    fn params_schema(&self) -> ParamSchema { /* 字段定义 */ todo!() }

    fn validate_params(&self, params: &Self::Params) -> Result<(), Vec<ParamError>> {
        let mut errors = Vec::new();
        if params.thickness < params.material.min_thickness() {
            errors.push(ParamError::too_thin("thickness", params.material.min_thickness()));
        }
        if params.height > params.material.max_wall_height(params.thickness) {
            errors.push(ParamError::too_high("height", params.material.max_wall_height(params.thickness)));
        }
        if errors.is_empty() { Ok(()) } else { Err(errors) }
    }

    fn wfc_tiles(&self, ctx: &BuildContext) -> Vec<WfcTileVariant> {
        vec![
            WfcTileVariant {
                kind: WfcTileKind::ExteriorWall,
                material: MaterialClass::Stone,
                subtype: WallStyle::BlockRough as u8,
                weight: 1.0,
            },
            WfcTileVariant {
                kind: WfcTileKind::ExteriorWall,
                material: MaterialClass::Wood,
                subtype: WallStyle::PlankVertical as u8,
                weight: 0.8,
            },
            // ... 更多材质 + 样式变体
        ]
    }
}
```

## 四、ComponentRegistry

```rust
/// 组件族注册中心 —— 编译期构建，依赖注入
pub struct ComponentRegistry {
    families: HashMap<ComponentFamilyId, Box<dyn ComponentFamily>>,
    name_index: HashMap<String, ComponentFamilyId>,
    /// 预计算的连接面兼容矩阵
    face_compatibility: FaceCompatibilityMatrix,
}

impl ComponentRegistry {
    pub fn builder() -> ComponentRegistryBuilder { ... }

    pub fn resolve(&self, id: ComponentFamilyId) -> Option<&dyn ComponentFamily> { ... }
    pub fn resolve_by_name(&self, name: &str) -> Option<&dyn ComponentFamily> { ... }
    pub fn compatible(&self, face_a: &ConnectionFace, face_b: &ConnectionFace) -> bool { ... }
    pub fn all_families(&self) -> impl Iterator<Item = &dyn ComponentFamily> { ... }
}

pub struct ComponentRegistryBuilder { ... }

impl ComponentRegistryBuilder {
    pub fn register<F: ComponentFamily>(mut self, family: F) -> Self { ... }
    pub fn register_with_mod(mut self, mod_toml: &str) -> Result<Self, ModError> { ... }
    pub fn build(self) -> ComponentRegistry { ... }
}
```

### 4.1 启动时构建

```rust
let registry = ComponentRegistry::builder()
    .register(WallFamily::new())
    .register(FloorFamily::new())
    .register(DoorFamily::new())
    .register(WindowFamily::new())
    .register(BeamFamily::new())
    .register(ColumnFamily::new())
    .register(StairsFamily::new())
    .register(FoundationFamily::new())
    .register(RoofFamily::new())
    // 可选文化扩展
    .register(ButtressFamily::new())         // 飞扶壁（哥特）
    .register(ArchFamily::new())             // 拱门（罗马/哥特）
    .register(WindCatcherFamily::new())      // 风塔（沙漠）
    .register(TatamiMatFamily::new())        // 榻榻米（和风）
    .register(LivingWoodFamily::new())       // 活木（精灵）
    // Mod 扩展
    .register_with_mod("dwarven_masonry.toml")?
    .register_with_mod("elven_sylvan.toml")?
    .build();
```

**不是全局单例**——`ComponentRegistry` 是普通 struct，通过引用传入所有需要它的地方。Mod 通过 TOML 声明注册，不是运行时 register()。

### 4.2 Mod 扩展格式

```toml
# dwarven_masonry.toml
[family]
name = "DwarvenVaultCeiling"
display_name = "矮人拱顶"

[family.params]
fields = [
    { name = "span", type = "f32", min = 2.0, max = 8.0 },
    { name = "rise", type = "f32", min = 0.5, max = 3.0 },
    { name = "material", type = "MaterialClass", default = "Stone" },
]

[family.connection_faces]
# 底部 Structural，顶部 Structural，拱侧无连接

[family.wfc_tiles]
# 在 WFC 中表示为特殊 tile——拱顶 cell
```
