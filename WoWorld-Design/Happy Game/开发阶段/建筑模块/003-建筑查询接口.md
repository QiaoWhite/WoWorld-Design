## 一、BuildingQuery trait — 唯一对外只读接口

`BuildingQuery` 是建筑模块对外暴露的**唯一数据读取通道**。消费者无需知道 Building 的内部存储结构。

```rust
/// 建筑世界数据的只读查询接口
pub trait BuildingQuery: Send + Sync {
    // ━━ 基础查询 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    /// 给定坐标，返回该位置的建筑（如果有）
    fn building_at(&self, pos: WorldCoord) -> Option<BuildingId>;

    /// 获取建筑的完整数据
    fn building(&self, id: BuildingId) -> Option<&Building>;

    // ━━ 组件查询 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    /// 获取单个组件
    fn component(&self, id: ComponentInstanceId) -> Option<&ComponentInstance>;

    /// 某建筑的所有组件 ID 列表
    fn components_of(&self, id: BuildingId) -> &[ComponentInstanceId];

    /// 区域内所有组件（跨建筑）
    fn components_in_region(&self, min: WorldCoord, max: WorldCoord) -> Vec<&ComponentInstance>;

    /// 以某点为中心、指定半径内的所有组件
    fn components_in_radius(&self, center: WorldCoord, radius: f32) -> Vec<&ComponentInstance>;

    // ━━ 空间拓扑 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    /// 建筑内的空间分区
    fn regions_of(&self, id: BuildingId) -> &[Region];

    /// 区域之间的连接关系（Portal 列表）
    fn portals_of(&self, id: BuildingId) -> Vec<Portal>;

    /// 区域连接图（用于导航）
    /// ⚠️ 建筑模块不提供寻路——NPC 行动系统消费 `regions_of()+portals_of()`
    ///    自行构建邻接图并执行 A*。建筑模块只提供拓扑数据。
    fn region_connections(&self, id: BuildingId) -> Vec<(RegionId, RegionId, Portal)>;

    /// 给定坐标的天花板高度（None = 不在建筑内）
    fn ceiling_height_at(&self, pos: WorldCoord) -> Option<f32>;

    /// 给定坐标是否在室内
    /// ⚠️ 室内微气候（温度）由 [[天气与季节系统]] 负责——
    ///    建筑模块只提供 `is_indoor()+building_geometry()+BuildingMaterialProps.thermal_insulation`。
    fn is_indoor(&self, pos: WorldCoord) -> bool;

    // ━━ 表面查询 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    /// 以某点为中心、指定半径内的可放置表面
    /// ⚠️ 返回的引用绑定到查询的借用生命周期——不可缓存跨帧。
    /// 家具放置系统应在放置那一刻重新查询表面可用性（处理并发放置冲突）。
    fn surfaces_in_region(&self, pos: WorldCoord, radius: f32) -> Vec<&dyn Surface>;

    // ━━ 结构与状态 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    /// 结构完整性评分 0-1（运行时快速查询）
    /// ⚠️ 当组件 health 降低但尚未移除时，渲染层在现有 mesh 上叠加 decal（裂纹/焦痕）
    ///    ——不需要等 mesh 重生成。mesh 重生成仅在组件被完全移除或建筑倒塌后触发。
    fn structural_integrity(&self, id: BuildingId) -> f32;

    /// 结构诊断——翻修/破坏时的具体问题列表
    fn structural_diagnosis(&self, id: BuildingId) -> Vec<StructuralIssue>;

    /// 单个组件的健康度
    fn component_health(&self, id: ComponentInstanceId) -> f32;

    // ━━ 所有权 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    /// 建筑所有权状态
    fn ownership(&self, id: BuildingId) -> OwnershipState;

    // ━━ 几何简化 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    /// 建筑的外壳几何（供 Weather/Audio 做射线检测/遮蔽计算）
    fn building_geometry(&self, id: BuildingId) -> BuildingGeometry;

    /// 建筑占地面积和高度
    fn building_footprint(&self, id: BuildingId) -> Option<(Aabb2D, f32)>;

    // ━━ 历史 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    /// 建筑的历史记录
    fn history(&self, id: BuildingId) -> Option<&BuildingHistory>;

    /// 经济估值因子（供 [[经济系统]] 房产估值）
    fn structure_value_factors(&self, id: BuildingId) -> StructureValueFactors;

    // ━━ 风格 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    /// 此建筑建时的风格快照
    fn style_at_build(&self, id: BuildingId) -> Option<&StyleSnapshot>;
}
```

## 二、Surface trait — 可放置表面

`Surface` trait 定义在建筑模块内。Items 模块消费它来放置家具。自然地形的表面通过 `TerrainQuery` 扩展提供，不实现此 trait。

```rust
/// 可放置物品的表面——由建筑组件暴露
pub trait Surface: Send + Sync {
    /// 表面在世界空间中的平面
    fn plane(&self) -> SurfacePlane;

    /// 表面类型
    fn surface_kind(&self) -> SurfaceKind;

    /// 可用面积 (m²)
    fn available_area(&self) -> f32;

    /// 负载能力 (kg)——None = 无限（地面）/ 未计算
    fn load_capacity(&self) -> Option<f32>;

    /// 表面材质类别（决定可钉/可粘/只能放）
    fn material(&self) -> Option<MaterialSurfaceClass>;

    /// 表面来源
    fn source(&self) -> SurfaceSource;
}

pub enum SurfaceKind {
    Floor,       // 水平面朝上——放家具
    Wall,        // 垂直面——挂画、武器、灯具
    Ceiling,     // 水平面朝下——吊灯
    Counter,     // 半高水平面——柜台、桌面
}

pub struct SurfacePlane {
    pub origin: Vec3,
    pub normal: Vec3,    // 指向可用方向（Floor = +Y, Wall = -Z/+Z/-X/+X, Ceiling = -Y）
    pub u_axis: Vec3,    // 表面 U 方向
    pub v_axis: Vec3,    // 表面 V 方向
    pub width: f32,      // U 方向尺寸
    pub height: f32,     // V 方向尺寸
}

pub enum MaterialSurfaceClass {
    Rigid(MaterialHardness),    // 石、硬木——可钉可钻
    Soft(MaterialHardness),     // 软木、泥土——可钉但承重低
    Friable,                    // 砂岩、灰泥——不能钉，只能放
    Magical,                    // 魔力表面——特殊规则
}

pub enum SurfaceSource {
    BuildingComponent(ComponentInstanceId),
    BuildingFloor(BuildingId),
}
```

### 2.1 PlacementSurface 从 Building 的推导

```rust
impl Building {
    pub fn collect_surfaces(&self) -> Vec<impl Surface> {
        let mut surfaces = Vec::new();

        for component in &self.components {
            let params = &component.params;
            let dim = params.dimensions();

            match component.cold.family {
                // Floor 组件 → 顶面是 PlacementSurface (Floor 类型)
                FLOOR_FAMILY_ID => {
                    surfaces.push(FloorSurface { component, dim });
                }
                // Wall 组件 → 内立面和外立面是 PlacementSurface (Wall 类型)
                WALL_FAMILY_ID => {
                    surfaces.push(WallInnerSurface { component, dim });
                    surfaces.push(WallOuterSurface { component, dim });
                }
                // Beam 组件 → 底面可以挂东西 (Ceiling 类型)
                BEAM_FAMILY_ID => {
                    surfaces.push(BeamBottomSurface { component, dim });
                }
                // Column 组件 → 侧面可以挂东西
                COLUMN_FAMILY_ID => {
                    for face in &[FaceDir::PosX, FaceDir::NegX, FaceDir::PosZ, FaceDir::NegZ] {
                        surfaces.push(ColumnFaceSurface { component, face: *face });
                    }
                }
                // ... 其他族
                _ => {}
            }
        }

        surfaces
    }
}
```

## 三、BuildingGeometry — 给 Weather/Audio 的简化外壳

建筑内部的完整组件图对 Weather（风场、雨遮蔽）和 Audio（声音反射）来说太重。需要简化的几何壳：

```rust
/// 建筑简化外壳——用于环境物理计算
pub struct BuildingGeometry {
    /// 封闭的凸壳顶点列表（世界坐标）
    pub hull_vertices: Vec<Vec3>,
    /// 外壳的面（三角形索引）
    pub hull_faces: Vec<(u16, u16, u16)>,
    /// 建筑 AABB
    pub bounds: Aabb,
    /// 屋顶类型（用于雨水遮蔽计算）
    pub roof_type: RoofStyle,
    /// 主要材质（用于声音反射/吸收）
    pub primary_material: MaterialClass,
    /// 开口位置——门、大窗（风可以通过）
    pub openings: Vec<Opening>,
}

pub struct Opening {
    pub position: Vec3,
    pub area: f32,
    pub normal: Vec3,
    pub is_door: bool,  // 门可以开关 → 遮蔽可变
}
```

## 三-B、LOD1RenderPacket — 远景渲染数据包

建筑模块不生成 GPU mesh——那是 Godot 的事。建筑模块产出此数据包，Godot 的 `BuildingRenderer` 消费它生成简化 mesh：

```rust
/// 传给 Godot 渲染层的 LOD1 渲染数据包
/// 32 字节对齐，批量传入 GPU buffer
#[repr(C)]
pub struct LOD1RenderPacket {
    pub aabb_min: [f32; 3],
    pub aabb_max: [f32; 3],
    pub style_hash: u32,       // 编码屋顶类型+材质+颜色（见 002 §4.1）
    pub floor_count: u8,
    pub building_type: u8,     // BuildingTypeTag → u8（特殊形状——塔楼/穹顶/尖顶）
    pub orientation: f32,      // 建筑朝向（弧度）
    pub _pad: [u8; 2],
}
// 32 字节 × 50,000 栋 = 1.6MB —— 可一次性上传到 GPU buffer
// Godot 侧: BuildingRenderer 持 style_hash → LOD1MeshPrototype 查找表
//           同一 style_hash 的建筑合并为一次 GPU instancing draw call
```

## 四、StructureValueFactors — 经济估值因子

经济系统进行房产估值时消费此数据。建筑模块不参与估价计算。

```rust
/// 建筑结构估值因子——供 [[经济系统]] 房产市场使用
pub struct StructureValueFactors {
    /// 总建筑面积 (m²)
    pub total_area: f32,
    /// 组件数量
    pub component_count: u32,
    /// 平均组件健康度
    pub avg_component_health: f32,
    /// 结构完整性评分
    pub structural_integrity: f32,
    /// 建筑年龄（游戏时间）
    pub build_age: GameDuration,
    /// 是否有完整屋顶
    pub has_roof: bool,
    /// 可辨识区域数
    pub rooms: u8,
    /// 楼层数
    pub floor_count: u8,
    /// 地基类型
    pub foundation_type: FoundationType,
    /// 建筑材料类别（影响耐久度和维护成本）
    pub primary_material: MaterialClass,
    /// 所有权状态摘要
    pub ownership_summary: OwnershipSummary,
}

pub enum OwnershipSummary {
    Private, Shared, Abandoned, Public, Contested,
}
```

## 五、BuildingMaterialProps — 建筑物理属性

建筑模块定义自己的物理属性 struct——不直接依赖 [[NPC活人感模块/08-NPC行动涌现与分类/001-NPC行动涌现总纲|NPC 物理原子层]] 的 `MaterialProperties`。

```rust
/// 建筑模块定义的材质物理属性
/// 从 MaterialProperties（NPC 物理原子层）转换而来——胶水代码做转换
#[derive(Copy, Clone)]
pub struct BuildingMaterialProps {
    pub compressive_strength: f32,  // Pa——抗压
    pub shear_strength: f32,        // Pa——抗剪
    pub density: f32,               // kg/m³
    pub durability: f32,            // 风化抗性 0-1
    pub nailability: f32,           // 可钉性 0-1
    pub min_thickness: f32,         // 最小结构厚度 (m)
    pub max_span: f32,              // 最大无支撑跨度 (m)
    pub thermal_insulation: f32,    // 隔热系数 0-1
}
```

胶水代码（World Gen 或运行时的材质管理器）负责从 `MaterialProperties` 转换为 `BuildingMaterialProps`。建筑模块不 import NPC 物理原子层的类型。

## 六、调用频率与性能约束

| 方法 | 频率 | 单次预算 | 备注 |
|------|------|---------|------|
| `building_at(pos)` | 每帧 ~2000 次 | ≤0.5μs | 渲染裁剪 + NPC 空间查询 |
| `is_indoor(pos)` | 每帧 ~200 次 | ≤1μs | NPC 天气响应 |
| `ceiling_height_at(pos)` | 每帧 ~200 次 | ≤0.5μs | NPC 运动 |
| `components_in_radius(pos, r)` | 每帧 ~50 次 | ≤5μs | 渲染 LOD |
| `structural_integrity(id)` | 每帧 ~50 次 | ≤0.5μs | 渲染稳定性标记 |
| `surfaces_in_region(pos, r)` | 低频（家具放置时） | ≤1ms | NPC 拿新家具/玩家编辑器 |
| `structural_diagnosis(id)` | 极低频（翻修/破坏） | ≤1ms | |
| `history(id)` | 偶发查询 | ≤0.5ms | NPC 认知/考古 |

热路径合计 ≤ 1.6ms/帧——留有充足余量。
