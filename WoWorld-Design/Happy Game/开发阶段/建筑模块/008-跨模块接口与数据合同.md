## 二、入口——本模块消费的接口

### 2.1 BuildContext（参数聚合 struct）

`BuildContext` 是建筑模块定义的 struct，由调用方（World Gen / 玩家编辑器 / NPC）填充后传入。建筑模块不 import 任何外部模块的 trait。

```rust
/// 建筑模块定义——外部模块的各路参数在此聚合
#[derive(Clone)]
pub struct BuildContext {
    // ━━ 种族体型 → [[生命系统]] 提供 ━━
    pub race: RaceBodyProfile,

    // ━━ 气候参数 → [[天气与季节系统]] 提供 ━━
    pub climate: ClimateBuildProfile,

    // ━━ 本地材料可用性 ━━
    // · 木材 → VegetationProvider::timber_availability()（CHG-046）
    // · 非木材(石/黏土/冰等) → [[世界生成]] P3 ResourceMap
    // · 胶水代码合并两个来源
    pub materials: MaterialAvailability,

    // ━━ 文化建筑偏好 → [[文化系统]] 提供 ━━
    pub culture: Option<CultureBuildProfile>,

    // ━━ 信仰建筑要求 → [[信仰系统]] 提供 ━━
    pub faith: Option<FaithBuildProfile>,

    // ━━ 业主个体偏好 → [[NPC活人感模块]] 提供 ━━
    pub owner: Option<OwnerBuildPrefs>,

    // ━━ 场地约束 → [[世界生成]] / TerrainQuery 提供 ━━
    pub site: SiteConstraints,
}

/// 建筑场地的物理约束
pub struct SiteConstraints {
    pub ground_height: f32,          // 地面海拔 (m)——地基深度计算基准
    pub slope: f32,                  // 地面坡度 (度)
    pub soil_class: SoilClass,       // 来自 TerrainQuery
    pub flood_risk: f32,             // 0-1
    pub water_depth: f32,            // 水上建筑 > 0
    pub is_subterranean: bool,       // 地下建筑
    pub must_attach_to: Option<FeatureId>,  // 树冠平台、崖壁等
}
```

### 木材材料来源（★ CHG-046）

木材材料的完整路径：
```
PlantSpecies (Life 009) → derive_timber_quality() (Life 010, 纯函数)
  → TimberQuality (woworld_core) → glue code → MaterialClass::Wood
```

建筑模块不 import 任何 Life/植物类型。构建时胶水代码消费
`VegetationProvider::timber_availability()`（woworld_core trait），
将 `TimberQuality` 映射为 `Availability` 后插入 `BuildContext.materials`。

见 [[../生命/010-植被群落与覆盖|生命 010 §五]] WoodMaterialContract。

**调用方（World Gen 胶水代码）范例**：

```rust
fn build_context_at(location: WorldCoord, culture_id: CultureId) -> BuildContext {
    // 非木材材料——来自 World Gen P3
    let mut materials = world_gen.material_availability(location);

    // ★ CHG-046: 木材材料——来自 VegetationProvider (woworld_core trait)
    let timber = vegetation_provider.timber_availability(location.as_vec2());
    if timber.available {
        // TimberQuality → Availability 映射（胶水代码——不 import Life 类型）
        let availability = match timber.quality {
            TimberQuality::Softwood | TimberQuality::Hardwood =>
                Availability::Available(timber.abundance),
            TimberQuality::TropicalHardwood | TimberQuality::GiantWood =>
                Availability::Scarce(timber.abundance * 0.3),
            TimberQuality::MagicWood =>
                Availability::Scarce(timber.abundance * 0.1),
        };
        materials.insert(MaterialClass::Wood, availability);
    }

    BuildContext {
        race: life_module.query_race_profile(location).into(),
        climate: weather.query_climate_profile(location).into(),
        materials,
        culture: culture.query_building_profile(culture_id),
        faith: faith.query_building_profile(location),
        owner: None,  // 初始生成时无业主
    }
}
```

每个 `.into()` = 外部模块数据 → 建筑模块定义格式的转换。胶水代码负责。

### 2.2 建筑模块定义的自有类型

**这些 struct 全部由建筑模块定义。** 外部模块的数据通过胶水代码的 `.into()` 转换为这些格式——建筑模块不 import 外部类型。

```rust
/// ━━ RaceBodyProfile: 种族体型参数 ━━
pub struct RaceBodyProfile {
    pub avg_height: f32,      // 平均身高 (m)
    pub avg_width: f32,       // 平均体宽 (m)
    pub avg_weight: f32,      // 平均体重 (kg)
    pub ceiling_height(&self) -> f32 { self.avg_height * 1.4 }  // 舒适天花板 = 身高×1.4
    pub door_height(&self) -> f32 { self.avg_height * 1.25 }
    pub step_height(&self) -> f32 { self.avg_height * 0.1 }     // 舒适步高
}

/// ━━ ClimateBuildProfile: 气候建筑参数 ━━
pub struct ClimateBuildProfile {
    pub temp_range: (f32, f32),    // 年均低温~高温 (°C)
    pub precipitation: f32,         // 年降水量 (mm)
    pub wind_speed_max: f32,        // 年最大风速 (m/s)
    pub snow_load: f32,             // 雪荷载 (kN/m²)
    pub flood_risk: f32,            // 洪水风险 0-1
    pub wall_thickness(&self, floor: u8, total: u8) -> f32 {
        let base = 0.25;  // 常温基础厚度
        let thermal = if self.temp_range.0 < -10.0 { 0.2 } else { 0.0 };  // 严寒→加厚
        let load_factor = (total - floor) as f32 / total.max(1) as f32;
        base + thermal + load_factor * 0.3  // 高层→底层加厚
    }
    pub fn roof_pitch(&self) -> f32 {
        if self.snow_load > 1.0 { 35.0 } else if self.precipitation > 1000.0 { 25.0 }
        else { 10.0 }
    }
}

/// ━━ MaterialAvailability: 材料可用性 ━━
/// ★ CHG-046: Wood键由 VegetationProvider::timber_availability() 填充（胶水代码）
///    非木材键由 World Gen P3 ResourceMap 填充
pub type MaterialAvailability = HashMap<MaterialClass, Availability>;

pub enum Availability {
    Abundant(f32),   // 权重乘数 > 1
    Available(f32),  // ≈ 1
    Scarce(f32),     // < 1
    Unavailable,     // 0
}

/// ━━ CultureBuildProfile: 文化建筑偏好 ━━
pub struct CultureBuildProfile {
    pub roof_style: Option<RoofStyle>,
    pub wall_style_preference: Vec<(MaterialClass, f32)>,  // 材质权重列表
    pub color_palette: ColorPalette,
    pub openness: f32,                // 0=全隔间 1=全开放
    pub visual_privacy: f32,         // 窗墙比偏好
    pub decoration_density: f32,     // 0=极简 1=繁复
    pub symmetry: f32,               // 0=不对称 1=严格对称
    pub quality_threshold: f32,      // 施工验收最低品质 0-1
    pub preferred_structural_material(&self) -> MaterialClass {
        self.wall_style_preference.first().map(|(m, _)| *m).unwrap_or(MaterialClass::Stone)
    }
    pub fn wall_style_for(&self, mat: MaterialClass) -> Option<WallStyle> { ... }
}

/// ━━ FaithBuildProfile: 信仰建筑要求 ━━
/// 从 [[信仰系统]] SacredArchitectureParams（7维）派生
pub struct FaithBuildProfile {
    pub sacred_geometry: SacredGeometry,      // Axial/Centripetal/Organic
    pub orientation: SacredOrientation,        // 朝向
    pub height_profile: HeightProfile,         // 高耸/宽阔/地下/朴素
    pub light_quality: LightQuality,           // 暗/亮/彩色/自然
    pub iconography: IconographyApproach,      // 具象/符号/禁止/自然元素
    pub material_palette: Vec<(String, f32)>,  // 偏好材质名→权重
    pub spatial_layout: TempleLayout,          // 中殿+侧廊/庭院/圆形/岩窟/多栋
}
// 以上枚举( SacredGeometry, SacredOrientation, ... )与信仰系统保持同名同变体——
// 建筑模块不需要 import 信仰系统，胶水代码做值映射。

/// ━━ OwnerBuildPrefs: 业主个体偏好 ━━
pub struct OwnerBuildPrefs {
    pub wealth_level: f32,           // 0-1 财富水平 → 影响材质/规模
    pub aesthetic_taste: Vec<StyleTag>,  // 审美偏好标签
    pub expansion_ambition: f32,     // 扩建意愿 0-1
    pub maintenance_diligence: f32,  // 维护勤勉 0-1 → 影响耐久衰减
}
```

### 2.3 不依赖的模块

建筑模块**不 import** 以下模块的任何类型：
- ❌ Culture（文化参数通过 `CultureBuildProfile` 传入）
- ❌ Faith（信仰参数通过 `FaithBuildProfile` 传入）
- ❌ Weather（气候参数通过 `ClimateBuildProfile` 传入）
- ❌ Life（体型参数通过 `RaceBodyProfile` 传入）
- ❌ Economy（不碰 Market/价格——只产出估值因子）
- ❌ Items（不碰物品定义——只通过 `MaterialClass` 引用材料类别）
- ❌ NPC（不碰 NPC 具体类型——只通过 `EntityId` 引用业主）

### 三-C、TerrainModificationRequest

```rust
/// 建筑模块声明的场地修改需求——建筑模块不执行 SDF 编辑
/// World Gen / Terrain 消费此声明，在施工前完成地形修改
pub struct TerrainModificationRequest {
    pub area: Aabb2D,
    pub operation: TerrainOp,
}

pub enum TerrainOp {
    /// 下挖至指定深度（地基坑）
    Excavate { depth: f32 },
    /// 平整至平均高度
    Flatten,
}
```

`BuildingGenerator::generate()` 返回 `(Building, Vec<TerrainModificationRequest>)`。
