## 二、入口——本模块消费的接口

### 2.1 BuildContext（参数聚合 struct）

`BuildContext` 是建筑模块定义的 struct，由调用方（World Gen / 玩家编辑器 / NPC）填充后传入。建筑模块不 import 任何外部模块的 trait。

```rust
/// 建筑模块定义——外部模块的各路参数在此聚合
#[derive(Clone)]
pub struct BuildContext {
    // ━━ 种族体型 → [[生命系统]] 提供 ━━
    pub race: RaceBodyProfile,

    // ━━ 气候参数 → [[天气与季节系统]] 提供 ━━
    pub climate: ClimateBuildProfile,

    // ━━ 本地材料可用性 → [[世界生成]] 提供 ━━
    pub materials: MaterialAvailability,

    // ━━ 文化建筑偏好 → [[文化系统]] 提供 ━━
    pub culture: Option<CultureBuildProfile>,

    // ━━ 信仰建筑要求 → [[信仰系统]] 提供 ━━
    pub faith: Option<FaithBuildProfile>,

    // ━━ 业主个体偏好 → [[NPC活人感模块]] 提供 ━━
    pub owner: Option<OwnerBuildPrefs>,
}
```

**调用方（World Gen 胶水代码）范例**：

```rust
fn build_context_at(location: WorldCoord, culture_id: CultureId) -> BuildContext {
    BuildContext {
        race: life_module.query_race_profile(location).into(),
        climate: weather.query_climate_profile(location).into(),
        materials: world_gen.material_availability(location),
        culture: culture.query_building_profile(culture_id),
        faith: faith.query_building_profile(location),
        owner: None,  // 初始生成时无业主
    }
}
```

每个 `.into()` = 外部模块数据 → 建筑模块定义格式的转换。胶水代码负责。

### 2.2 建筑模块定义的自有类型

| 类型 | 来源模块 | 转换方式 |
|------|---------|---------|
| `RaceBodyProfile` | [[生命系统]] `RaceBodyPlan` | `.into()` — 提取 avg_height, avg_width, avg_weight |
| `ClimateBuildProfile` | [[天气与季节系统]] `ClimateParams` | `.into()` — 提取温度范围、降水、风速、雪荷载、洪水风险 |
| `MaterialAvailability` | [[世界生成]] 资源分布 | 直接传入——`HashMap<MaterialClass, Availability>` |
| `CultureBuildProfile` | [[文化系统]] `CultureCoreParams` | `.into()` — 提取空间规范、材质偏好、颜色、隐私、屋顶、装饰密度、施工质量阈值 |
| `FaithBuildProfile` | [[信仰系统]] `FaithTheology` | `.into()` — 提取 SacredArchitectureParams（7维）|
| `OwnerBuildPrefs` | [[NPC活人感模块]] personality/wealth/aesthetic_taste | `.into()` — 提取个体建筑偏好 |
| `BuildingMaterialProps` | [[NPC活人感模块/08-NPC行动涌现与分类/001-NPC行动涌现总纲\|NPC 物理原子层]] `MaterialProperties` | `.into()` — 提取抗压、抗剪、密度、耐久、可钉性 |

### 2.3 不依赖的模块

建筑模块**不 import** 以下模块的任何类型：
- ❌ Culture（文化参数通过 `CultureBuildProfile` 传入）
- ❌ Faith（信仰参数通过 `FaithBuildProfile` 传入）
- ❌ Weather（气候参数通过 `ClimateBuildProfile` 传入）
- ❌ Life（体型参数通过 `RaceBodyProfile` 传入）
- ❌ Economy（不碰 Market/价格——只产出估值因子）
- ❌ Items（不碰物品定义——只通过 `MaterialClass` 引用材料类别）
- ❌ NPC（不碰 NPC 具体类型——只通过 `EntityId` 引用业主）

## 四、所有权裁决

| 概念 | 权威 Owner | 消费方 | 说明 |
|------|-----------|--------|------|
| `ComponentFamily` / `ComponentRegistry` | **建筑模块** | 所有生成器 | Mod 通过 TOML 扩展 |
| `ComponentInstance` / `Building` | **建筑模块** | World Gen, NPC, Combat, Senses | 权威数据源 |
| `BuildingQuery` trait | **建筑模块** | 10 个消费方 | 唯一对外读接口 |
| `Surface` trait | **建筑模块** | Items（PlacedItem）| 放置表面查询 |
| `Blueprint`（TOML 格式） | **建筑模块** | 玩家, Items (item 0x52) | 跨存档可分享 |
| `BuildContext` | **建筑模块定义** | World Gen, 玩家编辑器 | 参数聚合 struct |
| `BuildingGenerator` trait | **建筑模块** | World Gen P5/P6 | 多策略生成 |
| `WfcBuildingSolver` | **建筑模块** | World Gen P5/P6, 玩家编辑器 | 建筑级 WFC |
| `ConstructionScheduler` | **建筑模块** | NPC 行动, 玩家 | 工期编排 |
| `ConstructionModifier` trait | **建筑模块定义** | Magic（实现）| 魔法预留 |
| `MaterialRequirementList` | **建筑模块产出** | NPC EconomicBehavior | 声明式 |
| `BuildingHistory` | **建筑模块** | History (AetherImprint), NPC 认知 | 建筑考古 |
| `StructureValueFactors` | **建筑模块产出** | Economy（房产估值）| 不参与估价计算 |
| `BuildingMaterialProps` | **建筑模块定义** | 胶水代码转换 | 不依赖 MaterialProperties |
| `BuildingStylePreferences` | **文化系统**（已有）| 建筑模块（通过 BuildContext） | 不重复定义 |
| `SacredArchitectureParams` | **信仰系统**（已有）| 建筑模块 CathedraGenerator | 不重复定义 |
| `MaterialProperties` | **NPC 物理原子层**（已有）| 建筑模块（通过胶水转换）| 不直接依赖 |
| `FurnitureItemDef` / `PlacedItem` | **物品系统** | 建筑模块不碰——这是 Items 的事 | 家具不属于建筑模块 |
