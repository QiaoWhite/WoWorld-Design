## 一、战斗在 WoWorld 中的定位

战斗不是"切出去打一架再切回来"的独立模块。它是 NPC 行为在高压情境下的自然延续——**同一套人格、情绪、记忆、决策引擎**，只是行为类别的权重偏向战斗相关。

**核心原则**：开发者只创造规则（招式积木、武器属性、魔法方程式、环境交互规则），不创造故事（不写"这场战斗应该怎么打"）。具体的战斗风格、战术选择、流派演化——由 **NPC 人格 × 知识 × 处境 × 武器 × 魔法 × 环境** 实时搅拌产生。

## 二、核心设计决策

| # | 决策 | 说明 |
|---|------|------|
| 1 | **玩家 AI = NPC AI** | 同一套 Rust 代码。玩家只是选择战斗风格（等同于选择人格偏移）—角色自动执行 |
| 2 | **三层模型** | 本能逐帧(自动闪避/格挡) → 节奏0.5-1s(前压/稳守/周旋/猛攻/脱战) → 战略5-30s+事件驱动 |
| 3 | **信息不对称** | 没有上帝视角。敌方 HP/体力/魔力不可见——通过动画和行为推断。雾/夜/烟幕中信息不足 |
| 4 | **招式积木程序化生成** | 不是手写技能列表——招式由部件拼装（攻击轨迹+伤害类型+附加效果+消耗），流派从 NPC 群体的招式偏好中涌现 |
| 5 | **魔法融入战斗** | 十元素+七系魔法→战斗中的施法。法术熟练度→本能化（500+次→身体自动施放） |
| 6 | **半自动** | 玩家选战斗风格+关键指令，日常战斗全自动。不弹窗/不暂停/不减 |
| 7 | **人格驱动行为** | 大五人格决定冲突风格：保护者/问题解决者/破坏者/实验者。知识覆盖人格默认倾向 |
| 8 | **战后过渡** | 战斗不"突然结束"。情绪残留/惊跳反应/记忆编码/伤势恢复/声誉传播——战后的社会后果与战斗本身同样重要 |

## 三、与 NPC 活人感系统的深度绑定

战斗全程和 NPC 活人感模块交织——没有独立"战斗 AI"：

| NPC 系统组件 | 战斗中的角色 |
|-------------|------------|
| **DecisionEngine** | 日常模式（概率决策器）+ 战斗模式（同一引擎的高压版本——权重向战斗行动倾斜） |
| **EmotionState** | 战斗中实时情绪变化（愤怒↑→攻击性↑防御↓、恐惧↑→脱战倾向↑、兴奋→专注↑）—影响三层各层的决策权重 |
| **MemoryStore** | CombatNarrativeDetector 检测值得记忆的战斗时刻（被重创/反杀/队友牺牲/极限闪避）→ 写入事件记忆 + 情感标签 |
| **BigFive** | 人格决定冲突风格——不直接选招式，但决定"倾向成为什么样的战士" |
| **Knowledge** | 战斗经验覆盖人格默认倾向——"急，但我知道雾里不能乱冲" |
| **PostCombatTransition** | 战后人格×创伤×记忆→情绪残留/惊跳反应/社交影响/长期行为标签 |
| **Life.Vitals** | 读取身体状态——HP/体力/灵元素→影响战斗可行动作和决策 |

## 四、框架总图

```
战斗发起 (感知驱动——遭遇响应四选一)
    │
    ├─ 逃离 ──→ 不进入战斗状态（速度检定/障碍物/追逃AI）
    ├─ 妥协 ──→ 外交解决（谈判/屈服/投降/交易——避免战斗）
    ├─ 回避 ──→ 绕路/潜行绕过——同样不进入战斗
    │
    └─ 战斗 ──→ 三层模型全开
                  │
    ┌─────────────┼─────────────┐
    │             │             │
    ▼             ▼             ▼
 战略层        节奏层        本能层
 5-30s        0.5-1s       每帧(16ms)
 +事件驱动    +触发事件    +条件触发
    │             │             │
    ├─ 总目标     ├─ 倾向选择   ├─ 自动闪避
    ├─ 阶段目标   ├─ 前压/稳守  ├─ 自动格挡
    ├─ 限定条件   ├─ 周旋/猛攻  ├─ 自动反击
    └─ 自察       └─ 脱战       ├─ 本能法术
    输入:                      ├─ 掩体本能
    ·人格 ·情绪                └─ 身体反应
    ·记忆 ·知识                输入:
    ·处境 ·外部输入             ·技能熟练度
                                 ·身体状态
                                 ·疲劳度
                  │
                  ▼
              战斗结束 ──→ PostCombatTransition
                            ├─ 情绪残留(数分钟-数小时)
                            ├─ 记忆编码(关键事件→长期记忆)
                            ├─ 伤势(恢复天数由伤势严重度决定)
                            ├─ 声誉(围观NPC→传播→影响社交)
                            └─ 创伤/成长(长期人格微调)
```

## 五、性能预算

战斗系统运行在 Rust 模拟核心中——Godot 侧仅渲染+播放动画+处理玩家输入。

| 子系统 | 帧预算 | 说明 |
|--------|--------|------|
| **本能层** (每帧) | ≤0.5ms | 碰撞检测+闪避/格挡判定——纯计算，无 AI |
| **节奏层** (每0.5-1s) | ≤1.0ms | 倾向决策——分摊到多帧，每帧仅 1-2 个 NPC 做节奏决策 |
| **战略层** (每5-30s) | ≤2.0ms | GOAP 规划——仅目标变化时运行，分摊后几乎无感 |
| **战斗AI总计** (每帧分摊) | **≤1.5ms** | v3.0 技术栈方案 §11.2 的预算 |
| **骨骼矩阵计算** | ≤0.5ms | Rust CPU批量计算→Godot GPU skinning |
| **魔法效果计算** | ≤0.3ms | 十元素交互+法术效果 |

## 六、战斗系统关键 Rust trait

```rust
/// 战斗决策引擎——日常和战斗共用
trait CombatDecisionEngine {
    /// 给定感知输入和内部状态→输出战斗行动
    fn decide(&self, perception: &CombatPerception, state: &CombatState) -> CombatAction;
    /// 人格驱动的冲突风格
    fn conflict_style(&self, personality: &BigFive) -> ConflictStyle;
    /// 知识覆盖人格倾向
    fn knowledge_override(&self, knowledge: &Knowledge, default: CombatAction) -> CombatAction;
}

/// 战斗感知——信息不对称的核心
struct CombatPerception {
    /// 可见的敌人（视线内+光照+距离决定）
    visible_enemies: Vec<PerceivedEnemy>,
    /// 感知到的队友（视线或交流）
    perceived_allies: Vec<PerceivedAlly>,
    /// 环境感知（掩体/危险/脱战路径）
    environment: EnvironmentalPerception,
    /// 自身状态感知（可能不准确——疲劳时低估自身能力/恐惧时高估敌人）
    self_assessment: SelfAssessment,
}

/// 敌人——不是"数据"，是"感知到的信息"（可能错误）
struct PerceivedEnemy {
    entity_id: EntityId,
    estimated_hp: Range<f32>,           // "看起来伤得不轻" vs 精确HP数字
    estimated_stamina: Option<f32>,     // "他喘得厉害"
    observed_style: Option<FightStyle>, // 观察期积累的风格判断
    visible_weapon: Option<WeaponType>,
    casting_spell: Option<SpellType>,   // "他在施法!"
    threat_level: f32,                  // 主观威胁评估——可能被恐惧放大
}
```

## 七、与魔法系统的接口

| 魔法系统 | 战斗中的使用 |
|---------|------------|
| **十元素** | 火→持续燃烧伤害+环境点燃、冰→减速+冻结、雷→麻痹+水域传导、金→武器强化、木→HP恢复、水→灭火+湿润debuff、风→推飞+空中战斗、土→掩体创造+地震、血→生命汲取、灵→灵魂攻击 |
| **法术熟练度** | 500+次→本能化——条件触发时自动施放。300-500次→快速施法(无需完整吟唱)。<100次→完整施法流程+高失败率 |
| **魔力体系** | 魔力消耗→施法有限制。魔力恢复速度=环境魔力浓度×自身恢复率。低魔力→无法施法→只能物理攻击 |
| **施法媒介** | 不同媒介在战斗中的差异——法杖(远程+慢)、符文(即时+近距)、手势(最快+最小范围)、仪式(极慢+极强——只能在战斗外准备) |

## 八、与生命系统的接口

| 生命系统 | 战斗中的关联 |
|---------|------------|
| **BodyState** | HP/体力/灵元素→实时消耗。部位伤害→影响可用动作(右臂重伤→不能持武器) |
| **SpiritState** | 灵元素消耗→魔力枯竭→只能物理攻击。灵元素高的种族→施法更持久 |
| **CreatureTraits** | 种族特质影响战斗——精灵的敏捷→闪避倾向↑、矮人的耐久→格挡倾向↑ |
| **灵兽** | 驯服的灵兽参与战斗——有自己的战斗AI(InstinctiveMind+)——玩家可发出指令 |
| **死亡** | HP归零→濒死→死亡原因记录。亡灵可通过战斗杀死活人汲取灵元素延缓自身消散 |

---

## 九、与生命系统的接口

战斗系统和生命系统在以下层面交叉：

- **BodyState** 直接驱动战斗可行动作——右臂重伤则不能持武器
- **SpiritState** 限制施法——灵元素消耗即魔力枯竭
- **CreatureTraits** 影响战斗倾向——精灵偏敏捷闪避、矮人偏力量格挡
- **灵兽** 参与战斗——有独立战斗AI——玩家可下达指令
- **亡灵** 在战斗中攻击活人以汲取灵元素延缓消散
