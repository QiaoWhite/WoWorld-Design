# 002-战斗发起与遭遇响应

> **关联**: [[001-战斗系统总览]] · [[003-三层战斗模型]] · [[004-战斗信息与感知]]

---

## 一、核心原则：感知驱动，非距离驱动

系统知道附近在战斗 ≠ 每个 NPC 都知道。NPC 必须先通过感官外壳检测到威胁信号，才能进入"遭遇评估"。**没有上帝视角。**

```rust
enum EncounterResponse {
    Fight,      // 进入战斗——三层模型全开
    Flee,       // 逃跑——速度检定/障碍物/追逃AI
    Negotiate,  // 妥协——谈判/屈服/投降/交易——避免战斗
    Avoid,      // 回避——绕路/潜行——不进入战斗
}

fn evaluate_encounter(npc: &NpcRef, perceived: &PerceivedEntity, context: &Context) -> EncounterResponse {
    let threat = assess_threat(perceived, npc.personality, npc.emotion, npc.memory);
    let capability = assess_own_capability(npc, perceived);
    let cost = assess_cost(threat, capability, context, npc.personality);
    decide_response(threat, capability, cost, npc.personality, npc.emotion, npc.memory)
}
```

## 二、战斗触发的五种条件

任一条满足 → 该 NPC 进入"遭遇评估"：

### 2.1 蓄意攻击
任何实体对 NPC 执行了"攻击"行动。先手判定：攻击者的敏捷 × 隐蔽度 vs 防御者的感知能力。成功→防御者提前察觉（获得先手信息）。失败→攻击者获得先手优势（防御者的信息池在攻击命中后才更新）。

### 2.2 威胁升级
日常摩擦（争夺资源/社交对峙）持续 >15s 且双方情绪愤怒>0.6，或一方做出"攻击前兆"（拔剑/蓄力魔法/战吼/威胁性靠近）。

### 2.3 被卷入——感知驱动
必须通过感官外壳检测到战斗信号：
- **听到**：武器碰撞声/爆炸声/呼救声/战吼——距离衰减。戴耳罩/被沉默→听不到→不触发
- **看到**：视野内队友/同派系成员正在战斗——遮挡检测。在建筑物背面/被烟幕遮挡→看不到→不触发
- **直觉/魔法感知**：高阶能力——灵元素敏感者可感知魔力波动/杀气/隐形单位——非所有人都有
- **被交流告知**：队友呼唤——需要队友主动传递 + 自身接收到

### 2.4 主动宣战
对话中选择战斗触发选项。双方同时进入遭遇评估。先发起方有轻微先手（约0.3s——相当于一句话的时间）。

### 2.5 伏击
一方从潜行/隐藏状态发起→持续性先手检定→可能被提前发现。详见 [[011-潜行伏击与骑乘战斗]]。

## 三、遭遇评估——人格×处境×情绪×记忆的搅拌

评估在 2-5 秒内完成。没有固定权重——所有因素同时在场，它们的"声音"或大或小地混在一起。

```
评估 = f(
    实力对比    (我的战斗力 vs 感知到的对方实力——可能不准),
    代价收益    (打赢得到什么？打输失去什么？不打有什么后果？),
    限定条件    (信仰/道德/法律/职业职责——"我是守卫——不能跑"),
    人格偏移    (神经质→逃离↑, 宜人性→妥协↑, 尽责性→职责履行↑),
    当前情绪    (愤怒→战斗, 恐惧→逃离, 信任→谈判, 悲伤→冷漠/回避),
    已知支援    (附近有我认识的人吗？——只基于"我知道的"),
    记忆        (上次和这种人打的结果——"上次差点死了"),
)
```

### 人格对遭遇响应的塑造

| 人格组合 | 倾向 |
|----------|------|
| 高神经质+低宜人性(破坏者) | 有七成把握→开战。"你是不是在找死" |
| 高宜人性(保护者) | "能不流血就别流血"→先尝试谈判 |
| 高尽责性(问题解决者) | 精确评估→"三成以下跑，七成以上打，中间谈判" |
| 高神经质+高恐惧 | 高估威胁→"他肯定有埋伏"→容易逃跑 |
| 高开放性(实验者) | "这局面有意思——让我看看会怎样"→行为不可预测 |

**经验覆盖**：老兵（多次经历类似场景）的威胁评估更准确——"我看得出来他是在虚张声势"——不被恐惧放大也不被自信缩小。

## 四、四种遭遇响应的详细展开

### 4.1 战斗 (Fight)
评估后认为自己有胜算/有职责义务/情绪压倒了理智。

子目标：击杀/致残/击退/拖延(等援军)/活捉。选择 Fight 后→立即进入三层战斗模型（战略层在进入同时首决策）。

### 4.2 逃离 (Flee)
实力悬殊/高神经质/不值得拼命。

- **速度检定**：对方速度 ≥ 我的速度 ×1.3→几乎跑不掉
- **障碍物利用**：我可以翻墙他不能→脱战概率↑
- **队友掩护**：有人殿后→脱战概率显著上升
- **恐惧加剧**：逃跑中被追上的恐惧→应激反应——回身拼命/继续跑/分头跑/躲起来

对方不追击→不进入战斗状态→只是"逃离状态"。对方追击→可能被迫进入战斗。

### 4.3 妥协/谈判 (Negotiate)
实力悬殊但逃不掉/高宜人性/有谈判筹码。

- **劝降**："你已经输了——放下武器我不杀你"
- **屈服/投降**："我认输——随便你处置"
- **交易**：花钱买路/用物品交换安全通行/提供情报换命
- **威吓/虚张声势**："我身后有一整队守卫马上来!"——可能被识破

人格决定谈判策略——高魅力偏向真诚谈判、低宜人性偏向虚张声势、高智力更懂得"何时说什么话"。对方接受→不进入战斗。对方拒绝→重新评估（再评估中实力对比更悲观——"谈判失败=我逃不掉也打不过"）。

### 4.4 回避 (Avoid)
还没被发现/可以躲起来/让其他人去处理。

- **绕路**：绕开而不被发现——潜行技能决定
- **等待**：原地不动等对方离开——需要耐心(尽责性)
- **静默**：停止一切行动屏息——"别动——他可能没看到我们"
- **伪装**：假装在做别的——"我只是过路的农民"

---

## 五、进入战斗状态后

一旦选择 Fight → 三层模型全开。首次战略决策在进入战斗的同时发生——不是"先进入战斗再思考"——是"决定战斗的同时就已经在战略层"。

战略层立即设定总目标和初始阶段目标→节奏层开始输出→本能层持续运行。

**可中途改变**——"打着打着发现打不过"→战略层重新评估→改变总目标为"安全脱战"→节奏层切换为脱战倾向。**打完一波敌人≠战斗结束**——还有后续威胁时继续战斗状态。最后一个威胁消失后 ≥10s（由人格决定）→进入战后过渡。
