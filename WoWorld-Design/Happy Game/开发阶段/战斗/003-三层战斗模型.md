# 003-三层战斗模型

> **关联**：[[001-战斗系统总览]] · [[004-战斗信息与感知]] · [[007-半自动战斗与HUD]]

## 〇、三层对比

| 维度 | 本能层 | 节奏层 | 战略层 |
|------|--------|--------|--------|
| **频率** | 每帧(16ms)——持续 | 0.5-1s一次——持续 | 5-30s一次 + 事件驱动 |
| **内容** | 自动闪避/格挡/反击/本能法术 | 选择节奏倾向 | 设定总目标+阶段目标+限定条件 |
| **意识** | 完全无意识——身体自动 | 半意识——"感觉"层面 | 完全意识——"思考"层面 |
| **人格** | 不受人格影响 | 被人格染色 | 被人格深度塑造 |
| **知识** | 技能熟练度 | 经验积累→倾向微调 | 知识覆盖人格默认倾向 |
| **玩家干预** | 不可干预(本能=自动) | 可干预(手动移动/闪避) | 可干预(换目标/撤退/交流) |
| **CPU** | ≤0.5ms/帧 | ≤1.0ms/帧(分摊) | ≤2.0ms/帧(分摊——极稀疏) |

## 一、本能层——逐帧自动执行

不需要"决定"格挡——就像人不需要"决定"眨眼。只要当前姿态允许+来袭攻击在感知范围内→自动执行。

### 1.1 本能反应清单

| 本能 | 触发条件 | 成功率公式 |
|------|---------|-----------|
| **自动闪避** | 检测到来袭攻击+当前姿态允许 | `skill_proficiency × (1 - fatigue × 0.5) × (1 - flanked × 0.3)` |
| **自动格挡** | 防御姿态下+来袭攻击在格挡范围内 | `skill_proficiency × shield_coverage × (1 - fatigue × 0.3)` |
| **精准弹反** | 完美格挡窗口(攻击命中前0.1-0.2s) | `格挡成功率 × (proficiency - 0.5) × timing_precision` |
| **自动反击** | 完美格挡成功→对方出现后摇 | `skill_proficiency × 0.8`——几乎必定触发 |
| **本能化法术** | 熟练度500+次的法术+触发条件满足 | 自动施放——成功率=熟练度/1000 |
| **掩体本能** | 被远程攻击+附近有掩体 | 自动微调位置进入掩体阴影——不消耗专注 |
| **被围攻警觉** | 被2+敌人同时攻击 | 自动扩大闪避幅度+格挡面扩大——但体力消耗加快 |
| **濒死本能** | HP<15% | 闪避/格挡倾向↑↑、攻击完全交给本能——"拼命求活" |

本能层的表现上限 = 技能熟练度 + 当前身体状态。一个老练剑客的手会自动找到剑柄——不需要人格，不需要知识。

### 1.2 本能消耗

- 闪避：消耗 stamina × (1 - skill_proficiency × 0.5)——越熟练越省力
- 格挡：消耗 stamina × 盾牌重量系数 × (1 - strength × 0.3)
- 本能法术：消耗 spirit(灵元素)——与正常施法消耗相同，但不消耗专注
- 疲劳度：fatigue > 0.7 后，本能成功率开始下降

### 1.3 本能层 × 节奏层的冲突解决

本能化法术（500+次）自动触发时——需遵循节奏倾向的约束：
- **前压/猛攻** → 攻击性本能法术自动激活
- **稳守/周旋** → 攻击性本能法术被抑制（防御性本能法术仍激活）
- **脱战** → 所有本能法术抑制，仅保留闪避/格挡/濒死本能

**节奏约束的优先级高于本能**——哪怕你的身体"想"放火球反击，如果你的节奏是稳守，它就会被压住。

## 二、节奏层——0.5-1s 一次

不是"选一个动作"——是"选一个节奏"。输出不是具体招式——是一个**倾向**。节奏持续数秒，在这数秒内不重新决策。

### 2.1 五种节奏倾向

| 倾向 | 行为 | 适用场景 | 体力消耗 | 风险 |
|------|------|---------|---------|------|
| **前压** | 主动接近+攻击频率↑+防守靠本能层 | 想给压力/创造机会 | 高 | 中 |
| **稳守** | 保持距离+攻击频率↓+等对方先出手+观察 | 信息不足/等队友支援/等技能冷却 | 低 | 低 |
| **周旋** | 横向移动+寻找破绽+不冒进+时刻准备躲 | 试探阶段/实力相当/不想受伤 | 中 | 中 |
| **猛攻** | 连续出招+不管防守+体力消耗极快+追求速杀 | 抓住窗口/对方失衡/不想被拖入持久战 | 极高 | 高 |
| **脱战** | 拉开距离+优先逃跑路径+只防不攻 | 战略层决定撤退/HP太低/魔力枯竭 | 低 | 低(逃得掉) |

### 2.2 节奏切换触发条件

节奏持续期间，以下事件可**立即**触发节奏重新评估：
- 被重创（HP 骤降 >20%）→ 倾向立刻转向稳守或脱战
- 对方节奏明显改变（前压→猛攻 or 前压→脱战）
- 战略层新命令（新目标/限定条件改变）
- 关键技能冷却完毕
- 队友濒死或阵亡
- 环境剧变（火蔓延/建筑坍塌/浓雾突然降临）

### 2.3 节奏由什么决定

```
节奏选择 = f(
    人格(神经质高→偏好稳守, 外向高→偏好前压),
    处境(HP/体力/魔力剩余, 敌我人数比, 武器射程),
    情绪(愤怒→猛攻倾向↑, 恐惧→稳守/脱战↑),
    知识(经验: "这种敌人压制太强——不能前压"),
    外部(战略层目标, 队友状态, 环境)
)
```

**关键**：节奏不是"选动作"——是"选意图"。前压不意味着连续攻击——意味着"我想缩短距离并增加出手频率"。具体怎么实现交给行动生成器。

## 三、战略层——5-30s + 事件驱动

真正的"思考"。自察 → 总目标 → 阶段目标 → 限定条件。

### 3.1 战略决策流程

```
1. 自察
   我的HP/体力/魔力还剩多少？
   我受伤了吗——伤在哪——影响什么？
   我的队友在哪——状态如何？
   当前阶段目标达成了吗？

2. 评估总目标
   这场战斗我想要什么结果？
     - 击杀/摧毁对方
     - 迫使对方撤退
     - 自己安全脱战
     - 拖延时间(等援军/等关门/等技能CD)
     - 活捉/俘虏
     - 保护某人/某物

3. 设定阶段目标
   达成总目标的第一步是什么？
     - "先干掉对方的施法者"
     - "拉开距离——远程耗他"
     - "撑住——援军还有30秒"

4. 设定限定条件（约束——"我不能做什么"）
     - "不能让她受伤"（保护目标）
     - "不能在神殿里用火——会烧了圣物"
     - "不能杀她——得活着带回去"
     - "我答应了妻子不冒不必要的险"

5. 输出 → 节奏层
   战略层不直接控制招式——它告诉节奏层"要达成什么"。
   节奏层根据战略目标 + 自身倾向 + 处境 → 选择节奏。
```

### 3.2 战略决策的输入权重

```
战略决策 = 人格(40%) × 处境(25%) × 情绪(15%) × 记忆(10%) × 知识(10%)

人格主导总目标和保护优先级——不同人格的冲突风格:
  ·高宜人性 → 保护者。保护队友和弱者的优先级 > 击杀敌人
  ·高尽责性+低神经质 → 问题解决者。最高效方案,目标精确,成本可控
  ·高神经质+低宜人性 → 破坏者。最大伤害,可能过度杀戮
  ·高开放性 → 实验者。不固定,看这场战斗有什么"有趣"的事

处境修正(25%)——超出人格控制的现实:
  ·1v3 → 哪怕你是破坏者,也得考虑活命
  ·魔力10% → 哪怕你是高开放实验者,也得节约

情绪染色(15%)——情绪剧烈时可以主导:
  ·愤怒→保护者可能短暂变成破坏者

记忆10%——"上次这样做结果很惨" → 压制人格倾向
知识10%——经验积累 → 覆盖人格默认倾向
```

### 3.3 战略决策的触发事件

日常周期 5-30s——但以下事件 → 立即推送到战略层 → 强制重新评估：
- 阶段目标完成 → "打掉了施法者——下一步？"
- 总目标变得不可能 → "跑吧"
- 被重创（HP骤降 20%+）
- 队友濒死或阵亡
- 敌方士气崩溃/逃跑
- 环境剧变（火蔓延到脚下/天花板塌了/地面裂开）
- 外部输入（援军到达/新敌人出现/NPC喊话）

## 四、层级关系——同时发生，彼此染色

不是"一层传给另一层"的数据流。三层同时运行，互相染色：

```
本能感到吃力 (雾中看不清 → 闪避成功率↓)
  → 不直接导致节奏改变
  → 但它"染了"节奏的底色: 一种身体层面的不安
  → 节奏没有"选"稳守——节奏只是"不稳"了(倾向权重被染色但未切换)

节奏的犹豫 (本来前压被雾压住——前压权重自动降低)
  → 不直接导致战略改变
  → 但它"染了"战略的判断: "我现在没有在推进——为什么？"

战略层同时感受到:
  - 身体的不安 (本能层染色——来自上行的紧张感)
  - 节奏的犹豫 (节奏层染色——来自上行的不确定性)
  - "他也在雾里" (战斗信息的推理——不是我处于劣势)
  - "上次在雾里我差点死了" (记忆——不是现在,是过去)
  - "但我答应过妻子不冒没必要的险" (限定条件——不是"能不能"是"应不应该")
```

**上行染色**(本能→节奏→战略)：身体状态和行动感受向上渗透，影响高层的"信心"和"不安"。

**下行约束**(战略→节奏→本能)：战略目标约束节奏倾向，节奏倾向约束本能反应——最高层的决策最终决定低层的行为范围。

## 五、信息不足 = 放大人格 × 知识

在信息充足时（晴天、开阔地、对方无伪装）——不同人格的战术差异比较小，因为事实很清楚。大家都知道：对方比我多、我是远程他是近战→我要保持距离。事实主导决策。

在信息不足时（雾/夜/烟幕/沙尘/水下/密林/对方刻意隐藏）——不同人格的战术差异**急剧放大**，因为没有清楚的"正确选择"——每个人按自己的天性去填补空白：

- 高神经质 → "雾里一定有人埋伏" → 稳守甚至脱战
- 高开放性 → "雾里是什么样的战斗？让我试试" → 偏向前压探
- 高尽责性 → "不能冒险" → 周旋等雾散
- 低宜人性+高外向 → "管他有没有人——冲进去再说" → 猛攻

**但知识可以覆盖人格的默认倾向**：一个急躁的老兵（高神经质+低宜人性），因为经历过十几次烟雾战——"急，但我知道雾里不能乱冲"。人格仍然急躁，知识压住了它。

## 六、三层模型的性能实现

```rust
/// 三层战斗模型的每帧更新
fn tick_combat_entity(entity: &mut CombatEntity, delta: f32, world: &CombatWorld) {
    // 本能层——每帧、所有实体
    entity.instinct.tick(delta, world);

    // 节奏层——按需（0.5-1s间隔——用定时器分摊）
    if entity.rhythm_timer.should_tick(delta) {
        entity.rhythm.evaluate(entity, world);
    }

    // 战略层——按需（5-30s间隔或事件触发）
    if entity.strategy_timer.should_tick(delta) || entity.has_strategic_event() {
        entity.strategy.reassess(entity, world);
    }

    // 行动生成——由节奏倾向+武器+魔法可用→具体招式/移动
    entity.action_generator.generate(entity.current_rhythm(), entity.state, world);
}
```
