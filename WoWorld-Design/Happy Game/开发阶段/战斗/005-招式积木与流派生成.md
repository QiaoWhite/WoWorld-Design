## 一、核心概念：招式积木——不是手写技能列表

传统RPG：开发者手写"重击""旋风斩""火球术"等技能。WoWorld**不这样**。

招式 = 若干积木的组合。每块积木是一个小的功能单元——若干积木拼在一起=一个完整招式。积木的拼装由武器属性+使用者的技能+战斗处境驱动——不是固定招式表。

---

## 二、三层积木体系

### 2.1 最底层：动作基元（开发者定义，~15-20个）

刺 / 斩 / 撩 / 劈 / 扫 / 点 / 崩 / 缠 / 引 / 化

每个基元是一个基础的**攻击轨迹**——决定了这个动作的物理形态和力学特性。基元是"身体可以做什么"的最小单位——不由武器决定，但武器限制了可使用的基元集合。

### 2.2 中间层：程序化招式（程序化组合）

招式 = 1-3个动作基元 + 属性修饰(快/慢/轻/重) + 特效(破防/震退/出血/眩晕/位移) + 消耗(体力/魔力)

```rust
struct MoveTemplate {
    trajectory: AttackTrajectory,  // 由1-3个基元组成
    modifiers: Vec<MoveModifier>,  // 快/慢/轻/重
    effects: Vec<MoveEffect>,      // 破防/震退/出血/眩晕/位移……
    cost: MoveCost,                // 体力+魔力消耗
    conditions: Vec<MoveCondition>,// 使用条件(距离/姿态/武器/技能)
}

enum AttackTrajectory {
    OverheadSlash { angle: f32, speed: f32 },  // 上段斩——破防强/速度慢
    Thrust { range_bonus: f32 },               // 刺击——快/范围小/精确
    Sweep { arc: f32 },                         // 横扫——大范围/速度中
    UpwardStrike { launch_power: f32 },         // 挑击——浮空目标
    DownwardSmash { crush_bonus: f32 },         // 下砸——对倒地目标
    FeintAndRedirect { redirect_angle: f32 },  // 佯攻变向——欺骗格挡
}
```

### 2.3 上层：流派（从招式中涌现）

流派不是"被设计出来的"——是"被训练出来的"。大师的自然常用招式→教给弟子→弟子改进派生→几代后形成特色。同一个武器在不同文化中因物理属性×审美×战争需求×历史偶然长出不同的用武方式。

## 四、武器模块化

> 📘 **物品系统 v1.0 已建立通用装配框架**：`ItemAssembly::Composite { root, components, connections }`。Combat 通过注册 slot_type 定义武器组件（MeleeGrip/StrikingHead/BowLimb/BowString 等）。以下近战公式是装配框架的子集。详见 [[../物品系统/001-物品系统总纲|物品系统 001 §三]]。

武器 = 柄 + 刃/攻击部 + 卡槽(0-3个) + 饰品(可选)

所有参数为**滑块式**（非选项式）：
- **柄长**：0.3m-2.5m。短→单持快。中→可单可双。长→仅双持
- **刃**：材质/长度/宽度/厚度→决定伤害量+可用动作基元+速度
- **卡槽**：火焰/冰霜/雷/电/吸血等符文。可拆卸(卡槽)或永久(直接附魔)

**使用门槛**：重量→需要力量≥门槛。长度→需要力量+敏捷。材质→需要技能≥门槛。属性不足→effectiveness×0.5。远远不足→拿不动。

> ⚠️ **待补充**：
> 1. **远程武器**：弓/弩/投掷的模块化参数和动作基元（瞄准/拉弓/松弦/上弦/装填）有待补充。物品系统已预留 `BowRiser`/`BowLimb`/`BowString`/`CrossbowStock`/`CrossbowProd`/`CrossbowString`/`CrossbowTrigger`/`AmmunitionSlot` 等 slot_type。
> 2. **非刚性武器**：当前动作基元表仅覆盖 Rigid 装配（直剑/弯刀/斧/锤/矛/法杖）。四种 JointType 中的三种——[[../物品系统/001-物品系统总纲|Hinge(双节棍)/Chain(连枷/锁镰/九节鞭/流星锤)/Flexible(鞭子/绳镖/飞爪)]]——尚无动作基元映射。这些武器的动作（甩/缠/卷/锁/绞/抡）需要从 JointType × 组件类型派生，而非单纯的刃部形态派生。
> 3. **法器**：魔杖/法杖/宝珠/魔导书的 Combat 侧动作（引导/屏障/增强）和 Magic 侧法力计算待对接。
> 4. **拳套/手部武器**：不占 Grasp 槽——覆盖手部。装备拳套后空手攻击使用拳套属性，持握武器时拳套打击属性暂失效。

## 五、生成约束

- **力学约束**：重·慢·弧线不能紧接短·快·直线（违反动量——除非中间有缓冲动作）
- **武器约束**：匕首不能有"劈"(刀身太短)、长矛不能有"短·快"组合(惯量太大)
- **文化约束**：沙漠雇佣兵不会发展出仪式感很重的慢招式(缺水+高温——需要高效)
- **人格约束**：高神经质→招式偏向快速/多次/短后摇(即使武器更适合慢节奏)

## 六、流派涌现机制

### 6.1 流派如何产生

```
1. NPC_A 发现: 在沼泽地用横扫→把敌人打进泥里→接着下砸→击杀
2. NPC_A 靠这招赢了——记住了——"继续这样打"
3. NPC_A 教给学徒——"打沼泽里的敌人——先扫腿再砸头"
4. 该招式组合在一个聚落/派系内传播——更多人使用
5. 招式组合 + 特定处境 + 文化偏好 → 形成可识别的"流派"
```

### 6.2 流派要素

| 要素 | 说明 |
|------|------|
| **偏好招式组合** | 该流派"喜欢"的3-5个积木组合——不是固定列表而是加权偏好 |
| **适用处境** | "沼泽/泥地""狭窄室内""开阔平原1v多"——流派有擅长场景 |
| **人格倾向** | 该流派吸引什么样的战士——"急躁的人适合这个流派"——人格匹配加速学习 |
| **文化特性** | 精灵偏木/灵元素、矮人偏金/火元素、沙漠族偏高效短快 |
| **传承方式** | 师徒传承/血族传承/书/自学发现/战场上学会/灵兽教的 |

### 6.3 流派的生命周期

- **诞生**：某NPC发现有效招式组合（可能偶然）→开始使用
- **成长**：该NPC战胜对手→"这招有效"的信念强化→持续使用→被认可为流派
- **传播**：教授学徒→扩大使用者群体→地域扩张
- **演变**：不同地区的同一流派可能出现变体——"北方派"vs"南方派"
- **消亡**：无人再使用→被遗忘→可能被后人从古代记载中重新发现

## 七、战斗中的物品与装备

### 装备损坏
武器/盾牌连续承受重击→耐久下降。耐久<20%→effectiveness×0.7。耐久=0→需切换备用武器。高尽责性 NPC 定期维护装备，低尽责性用坏了才修。

### 武器掉落/缴械
特殊招式或投技可导致脱手。掉落→地面→可拾取。失去武器→自动切体术/备用武器。

### 武器切换
切换时间 = 收当前武器 + 取新武器。例：弓→剑 = 0.4s+0.4s = 0.8s。切换期间不能攻击/格挡，只能闪避。切换滞后规则：新行动比当前好 >0.3 + 敌人位置稳定 >1s → 才切换——防止边界跳变。

### 有效射程——连续有效性曲线
不是三段硬边界。是一条连续曲线：`effectiveness = f(distance, height_angle, line_of_sight, target_attention, self_positioning, environment) × skill × personality`。任何因子为0→总effectiveness=0。
