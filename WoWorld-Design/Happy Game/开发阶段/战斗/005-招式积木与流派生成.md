# 005-招式积木与流派生成

## 一、三层积木体系

### 1.1 最底层：动作基元（开发者定义，约15-20个）

刺 / 斩 / 撩 / 劈 / 扫 / 点 / 崩 / 缠 / 引 / 化

### 1.2 中间层：程序化招式（程序化组合）

招式 = 1-3 个动作基元 + 属性修饰（快/慢/轻/重）+ 特效（破防/震退/出血/眩晕/位移）

### 1.3 上层：流派（从招式中涌现）

流派不是"被设计出来的"——是"被训练出来的"。大师的自然常用招式 → 教给弟子 → 弟子改进派生 → 几代后形成特色。同一个武器在不同文化中因物理属性×审美×战争需求×历史偶然长出不同的用武方式。

| >1.2m | ❌ | ✅ | ×0.8 |

### 战斗中的物品与装备

**快速使用物品**：按 [5] 打开快速物品栏 → 选择药水/道具 → 使用动作(0.5-1s, 期间不能攻击/格挡)。AI 在 HP<30% 或队友濒死时可能自动使用(取决于资源策略滑块)。

**装备损坏**：武器/盾牌连续承受重击 → 耐久下降。耐久<20%→effectiveness×0.7。耐久=0→需切换备用武器。高尽责性 NPC 定期维护装备，低尽责性用坏了才修。

**武器掉落/缴械**：特殊招式或投技可导致脱手。掉落→地面→可拾取。失去武器→自动切体术/备用武器。

## 二、生成约束

- **力学约束**：重·慢·弧线不能紧接短·快·直线（违反动量）
- **武器约束**：匕首不能有"劈"（刀身太短），长矛不能有"短·快"组合
- **文化约束**：沙漠雇佣兵不会发展出仪式感很重的慢招式
- **人格约束**：高神经质→招式偏向快速、多次、短后摇（即使武器更适合慢节奏）

## 三、武器模块化

武器 = 柄 + 刃/攻击部 + 卡槽(0-3个) + 饰品(可选)

所有参数为滑块式（非选项式）：
- 柄长：0.3m-2.5m。短→单持快。中→可单可双。长→仅双持。
- 刃：材质/长度/宽度/厚度→决定伤害量+可用动作基元+速度
- 卡槽：火焰/冰霜/雷电/吸血等符文。可拆卸（卡槽）或永久（直接附魔）

**武器使用门槛**：重量→需要力量≥门槛。长度→需要力量+敏捷。材质→需要技能≥门槛。属性不足→effectiveness×0.5。远远不足→拿不动。

**武器×招式积木**：刃部决定可用的动作基元（直剑刃→刺斩劈撩点，锤头→劈崩），柄部决定范围+速度修饰，卡槽叠加特殊效果。

## 四、有效射程——连续有效性曲线

不是三段硬边界。是一条连续曲线：effectiveness = f(distance, height_angle, line_of_sight, target_attention, self_positioning, environment) × skill × personality。

任何因子为0 → 总 effectiveness = 0。不按距离分段——按连续曲线+物理现实。

## 五、武器切换

切换时间 = 收当前武器 + 取新武器。例：弓→剑 = 0.4s + 0.4s = 0.8s。切换期间不能攻击/格挡，只能闪避。

切换滞后规则：新行动比当前行动好 >0.3 + 敌人位置稳定 >1s → 才切换。防止边界跳变。
