# 008-团队战斗

> **关联**：[[001-战斗系统总览]] · [[003-三层战斗模型]] · [[004-战斗信息与感知]] · [[007-半自动战斗与HUD]]

---

## 一、两种团队模式——共用同一套AI

### 1.1 扁平团队（冒险小队/临时雇佣兵）

各自决策，信息共享有限，配合靠默契。没有谁"指挥"谁——每个人都是平等的决策者。协调通过团队信息共享自然涌现——不是硬编码的"坦在前/奶在后"。

### 1.2 层级团队（军队/卫兵队/有明确上下级）

上级命令覆盖下级的战略层自主决策：`command_override: Option<CombatCommand>`。有命令时服从——没命令时自主。服从质量取决于：下级的尽责性(高→主动执行、低→敷衍)、下级的忠诚/信任(对上级的态度)、命令的合理性(与下级自身战略判断冲突→执行犹豫)。

两种模式共用同一套 AI——只是战略层多了一个可选的命令输入。

## 二、团队信息共享——感知驱动

团队成员**不自动获得**彼此信息。所有信息必须通过感官外壳获取：

| 距离 | 能感知到什么 | 延迟 |
|------|-------------|------|
| <10m | 能看到+能听到——队友位置/节奏倾向/HP估计(三级)/交流内容 | ~0s |
| 10-30m | 能看到但听不清——队友位置/节奏倾向/只能看到大概HP | ~1-2s |
| >30m | 只能看到大概——队友位置/是否在战斗(是否/否)/HP完全看不清 | ~3-5s |
| 被遮挡/远距离 | 不更新——仅靠战斗交流消息(队友主动呼喊/打信号) | — |

不看队友 = 不知道他的状态。这和 [[004-战斗信息与感知]] 的信息不对称原则一致。

## 三、人格驱动的自然配合

协调行为不靠硬编码——每个 NPC 的战略层看到队友状态后，自己的三层模型自然决定做什么：

- **支援型(高宜人性)**：看到队友濒死 → "我要保护他" → 倾向转向队友 + 准备治疗/掩护
- **战术型(高尽责性)**：看到队友正面吸引火力 + 自己在侧面 → "他在创造机会" → 猛攻侧翼
- **狂暴型(高神经质+低宜人性)**：看到队友濒死 → "他快不行了——但我停不下来" → 节奏不变。队友只能自求多福
- **实验型(高开放性)**：看到队友在用奇怪战术 → "有意思——我也试试" → 配合队友的不可预测行动

---

## 四、交流系统

- 战斗中交流通过战斗信息池中的"交流传递源"
- NPC之间可发出：请求支援/报告敌情/警告("盗贼在你后面!")/建议("追!他快不行了!")
- 信任度影响：队友关系好 → 优先相信警告。队友平时大惊小怪 → 可能忽略
- 无领导的扁平团队：交流是建议。层级团队：交流可以是命令

---

## 五、玩家在团队中

- **扁平团队**：玩家是平等的成员。队友有自己的判断——可能不听取玩家的建议(取决于关系值+玩家的声望/战斗记录)
- **层级团队**：玩家的命令覆盖队友的战略层。但队友的人格仍影响执行质量——高尽责性队友精确执行命令；低尽责性队友在压力下可能违令

---

## 六、士气——团队层面的情绪

团队士气不只是个人情绪的加总——它是团队共享的心理场：

- HP<40% → 个人士气下降
- 队长/领导者阵亡 → 团队士气崩塌——所有人的脱战倾向急剧上升
- 看到队友逃跑 → 连锁逃跑——但低神经质+高尽责性者可能顶住
- 反杀/关键击杀 → 士气飙升——"我们还有机会!"
- 寡不敌众感 → 人数差>3:1→士气持续下降——除非坚守阵地(职责约束压过士气)

```rust
struct TeamMorale {
    individual_morale: f32,    // 个人的战斗意志——由恐惧/愤怒/信任/职责共同决定
    perceived_team_morale: f32,// "我感觉队友还有多少斗志"——可能不准
    cohesion: f32,             // 团队凝聚力——高凝聚力团队=个人士气下降时互相支撑
    leader_alive: bool,        // 领导者是否存活——层级团队中此因子极重
}
```
