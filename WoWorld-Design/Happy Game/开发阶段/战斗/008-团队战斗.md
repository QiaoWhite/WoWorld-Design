# 008-团队战斗

## 一、两种团队模式

### 1.1 扁平团队（冒险小队/临时雇佣兵）

各自决策，信息共享有限，配合靠默契。没有谁"指挥"谁。

### 1.2 层级团队（军队/卫兵队/有明确上下级）

上级命令覆盖下级的战略层自主决策：`command_override: Option<CombatCommand>`。有命令时服从，没命令时自主。

两种模式共用同一套 AI——只是战略层多了一个可选的命令输入。

## 二、团队信息共享

每个成员自动获得（但有延迟）：

- 队友位置（实时，可信度确定——除非被遮蔽）
- 队友当前节奏标签（前压/稳守/周旋/猛攻/脱战）
- 队友 HP 估计（健康/受伤/濒死，三级而非精确数字）
- 队友当前目标

延迟取决于距离：<10m（~0s）→ 10-30m（~1-2s）→ >30m（~3-5s）→ 远距离（仅靠战斗交流消息更新）

## 三、人格驱动的配合

协调行为不靠硬编码——每个 NPC 的战略层看到队友状态后，自己的三层模型自然决定做什么。

- **支援型**：看到队友濒死 → "我要保护他" → 倾向转向队友 + 准备治疗
- **战术型**：看到队友正面吸引火力 + 自己在侧面 → "他在创造机会" → 猛攻侧翼
- **狂暴型**：看到队友濒死 → "他快不行了，但我停不下来" → 节奏不变。队友只能自求多福

## 四、交流系统

- 战斗中的交流通过"战斗信息池"中的交流传递源
- NPC 之间可能发出：请求支援、报告敌情、警告（"盗贼在你后面!"）、建议（"追！他快不行了!"）
- 信任度影响：队友关系好→优先相信警告；队友平时大惊小怪→可能忽略

## 五、玩家在团队中

扁平团队：玩家是平等的成员。队友有自己的判断，可能不听取玩家的建议。
层级团队：玩家的命令覆盖队友的战略层。但队友的人格仍然影响执行质量。
