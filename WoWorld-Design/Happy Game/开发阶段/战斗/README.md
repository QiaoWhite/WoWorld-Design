## 核心设计原则

1. **玩家AI = NPC AI** — 同一套Rust代码。玩家不操作时完全自主战斗。
2. **三层模型** — 本能(逐帧)→节奏(0.5-1s)→战略(5-30s)。同时运行，互相染色。
3. **信息不对称** — 没有上帝视角。感知驱动，非距离驱动。信息不足放大人格×知识。
4. **只创造规则，不创造故事** — 招式积木+武器模块化+人格→自然涌现千百种战斗风格。
5. **战斗 = NPC活人感的高压延续** — 不是独立模块，是同一套人格/情绪/记忆在高压下的自然表达。

## 性能预算

| 子系统 | 每帧预算 | 说明 |
|--------|---------|------|
| 本能层 | ≤0.5ms | 逐帧——所有战斗实体 |
| 节奏层 | ≤1.0ms(分摊) | 0.5-1s周期——轮询分布 |
| 战略层 | ≤2.0ms(分摊) | 5-30s周期——极稀疏分布 |
| 魔法效果计算 | ≤0.3ms | 元素交互/环境联动计算 |
| 战斗AI总计 | ≤1.5ms | 三层+魔法分摊后的实际并发帧预算 |
| 骨骼矩阵 | ≤0.5ms | GPU skinning前计算 |

## 架构集成

```
Rust 战斗引擎 (woworld_combat)
├── CombatAI trait — 玩家和 NPC 同一实现
├── 本能层 (~0.5ms/帧) — 所有战斗实体的逐帧反应
├── 节奏层 (~1.0ms/帧分摊) — 0.5-1s周期的战术决策
└── 战略层 (~2.0ms/帧分摊) — 5-30s周期的全局策略
        ↓ CombatEvent
EventBus → 感官/音频/历史/NPC记忆模块独立订阅
```

## 涌现战斗风格

```
招式积木 (动作基元) × 武器模块化参数 × 人格(BigFive) × 情绪(PAD) × 身体状态(AgentSnapshot)
    → 自然涌现千百种独特战斗风格
    没有"战士/盗贼/法师"职业——只有参数组合的统计聚类
```

## 关键参数

| 参数 | 范围 | 说明 |
|------|------|------|
| 战斗并发上限 | ≤20 组 | GTX 1660 SUPER |
| 本能层周期 | 逐帧 | 16.7ms |
| 节奏层周期 | 0.5-1s | 人格/情绪驱动变异 |
| 战略层周期 | 5-30s | 含GOAP重规划 |
| 信息衰减 | 距离+遮挡 | 感知驱动，非距离驱动 |
