# 001 — 技能系统总纲

## 一、设计哲学

### 1.1 "用进废退"——退取消

在 WoWorld 中，技能**只升不降**。用剑→剑术提升。做饭→烹饪提升。十年不碰剑→不代表你忘了怎么握剑。肌肉记忆是永久的。

这个选择不仅符合真实世界的神经科学（程序性记忆几乎不衰减），也简化了数据模型——无需追踪 each skill 的 last_used 时间，无需每日衰减计算。技能成为 NPC 的**永久累积资产**。

### 1.2 技能系统是基础设施，不是玩法

与物品系统相同：技能系统只定义**机制**（怎么计量、怎么提升、怎么传授），不定义**内容**（剑术80级能做什么——那是 Combat 的事）。

| 技能系统管 | 技能系统不管 |
|----------|------------|
| SkillId / SkillEntry 数据模型 | 挥剑的伤害公式 → **Combat** |
| 累积XP曲线 / 天赋修正 / 交叉训练 | 锻造铁剑需要多少材料 → **物品系统 006** |
| TeachingSession / 四种学习路径 | 法术的魔力消耗 → **Magic** |
| MentalAccess trait（心智→学习效率） | NPC 的初始技能分配 → **World Gen** |
| add_xp / add_xp_direct 管线 | "剑术80级解锁旋风斩" → **不存在。不做** |

### 1.3 故事生成器的人才根基

WoWorld 是故事生成器。技能系统提供的是"人有多擅长什么"这个基本答案——从它涌现出：

- 为什么这个 NPC 是镇上最好的铁匠？（锻造 Lv87，天生1.2倍率，跟过大师学徒十年）
- 为什么那个村庄被兽人屠了？（守卫最高剑术 Lv12 —— 穷村雇不起好教官）
- 为什么这把剑值 200 金币？（制作者锻造 Lv92 → 完美品质 + 直接附魔）

**技能差异 = NPC 命运差异 = 故事涌现的引擎。**

## 二、核心数据模型

### 2.1 SkillId —— 全局恒定技能标识

```rust
/// 技能标识符 —— 全局恒定，每个技能一个 ID
/// 布局: [8-bit category][8-bit subgroup][16-bit reserved][32-bit skill_index]
///
/// category:
///   1 = Combat（战斗）
///   2 = Magic（魔法）
///   3 = Artisan（工艺）
///   4 = Academic（学术）
///   5 = Survival（生存）
///   0 = 未分类（预留）
///   6-255 = 预留
///
/// 前 16 bit 已足够确定归属 —— 不查表即可判断类别和子组
pub struct SkillId(u64);

impl SkillId {
    pub fn category(&self) -> u8 {
        ((self.0 >> 56) & 0xFF) as u8
    }
    pub fn subgroup(&self) -> u8 {
        ((self.0 >> 48) & 0xFF) as u8
    }
    pub fn skill_index(&self) -> u32 {
        (self.0 & 0xFFFF_FFFF) as u32
    }
}
```

- **全局恒定**：剑术的 SkillId 在所有存档、所有机器、所有版本中相同
- **Mod 安全**：category 6-255 预留 Mod
- **TOML 数据驱动**：技能定义（名称/分类/子组/交叉组）从 TOML 加载——不是枚举

### 2.2 SkillEntry —— 单技能实例数据

```rust
/// NPC/玩家身上一个技能的完整状态
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SkillEntry {
    /// 当前等级 0.0–100.0（显示值——由 xp 通过累积公式推导）
    pub level: f32,

    /// 累积经验值（隐藏——驱动 level 的真实数字）
    pub xp: f32,

    /// 天生经验倍率 0.7–1.3
    /// NPC 出生时每个技能独立 roll 一次（正态分布 N(1.0, 0.15), clamped）
    /// 此后不可变
    pub innate_aptitude: f32,

    /// 累计获得的总经验（仅统计用途——"他这辈子打了多少铁"）
    /// 用于里程碑检测和成就
    pub total_xp_earned: f32,

    /// 使用总次数
    /// Magic 016 本能化阈值: times_used ≥ 500 → 法术可进入本能层
    pub times_used: u64,
}
```

**稀疏存储**：`BTreeMap<SkillId, SkillEntry>`——只有获得过 XP 的技能才占用存储。不存在于 map 中的 skill = untrained（从未接触过此技能）。

每 NPC ~10-20 活跃技能。100K NPC × 15 技能 × ~40 bytes ≈ 60MB。

### 2.3 累积 XP 公式

```
累积 total_xp_for_level(L) = base × (e^(k×L) ― 1) / k
其中 base = 100, k = 0.04

逆向: level_from_xp(xp) = ln(xp × k / base + 1) / k

各级所需累积 XP:
    L=10  →     491    (新手→学徒  ~2周)
    L=25  →   1,718    (学徒→熟练  ~1.5月)
    L=50  →   6,389    (熟练→专家  ~6月)
    L=75  →  20,085    (专家→大师  ~2年)
    L=90  →  36,598    (大师→宗师  ~4年)
    L=100 →  53,607    (宗师→传奇  ~6年持续苦练)

base=100, k=0.04 为默认值——[TUNING] TOML 可全局调节
```

**解释**：从 0 到 10 级只需要 ~491 XP（约 2 周日常使用），但从 90 到 100 需要额外 ~17,000 XP（约半年持续苦练）。精度保留在 XP 层（f32），level 是派生显示值。

### 2.4 基础 XP 值的归属

**技能系统不定义具体行动的基础 XP**。各消费模块自行定义：

| 模块 | 基础 XP 示例 | 定义位置 |
|------|------------|---------|
| **Combat** | 格挡成功=3XP, 命中=5XP, 完美弹反=10XP, 击杀=15XP | Combat 内部 |
| **Magic** | 施法成功=5-20XP（取决于法术复杂度）, 消耗1刻魔力=1XP | Magic 内部 |
| **Artisan** | 锻造铁剑=20XP, 烹饪=3XP, 熔炼铁锭=5XP | 物品系统 006 配方数据 |
| **Academic** | 读完一章书=10XP, 鉴定遗迹=25XP | 未来书籍系统 |
| **Survival** | 成功采集=2XP, 追踪猎物=5XP, 钓鱼=3XP | Life 模块 |

技能系统只提供 `add_xp(skill, base_xp, source, mental, physical, all_skills, skill_def, time)` 的**加工管线**——基础 XP 进去，经过天赋修正+交叉训练出来。

## 三、天赋三层模型

### 3.1 第一层：心智属性 × 技能类别

**技能系统定义 trait，NPC 模块实现**——技能系统不知道 NPC 类型，不依赖 NPC 数据结构。

```rust
/// 技能系统定义 —— 心智属性访问接口
/// NPC活人感模块和玩家各自实现此 trait
pub trait MentalAccess {
    fn intelligence(&self) -> f32;   // 0.0–1.0 智力
    fn wisdom(&self) -> f32;         // 0.0–1.0 智慧
    fn willpower(&self) -> f32;      // 0.0–1.0 意志力
    fn charisma(&self) -> f32;       // 0.0–1.0 魅力——用于教学能力修正，非社交技能
}

/// 技能系统定义 —— 身体属性访问接口
/// Life模块实现此 trait
pub trait PhysicalAccess {
    fn strength(&self) -> f32;       // 0.0–1.0 力量
    fn dexterity(&self) -> f32;      // 0.0–1.0 灵巧
    fn agility(&self) -> f32;        // 0.0–1.0 敏捷
    fn endurance(&self) -> f32;      // 0.0–1.0 耐久
}

/// 技能系统内部 —— 心智因子计算
/// 返回 0.7–1.0。好心智加速最多 30%，差心智减速最多 30%
fn mental_factor(mental: &dyn MentalAccess, physical: &dyn PhysicalAccess, category: SkillCategory) -> f32 {
    let raw = match category {
        SkillCategory::Combat => {
            mental.willpower() * 0.5 + physical.strength() * 0.15 + physical.dexterity() * 0.15 + physical.agility() * 0.2
        }
        SkillCategory::Magic => {
            mental.intelligence() * 0.5 + mental.willpower() * 0.5
        }
        SkillCategory::Artisan => {
            mental.intelligence() * 0.4 + mental.willpower() * 0.3 + physical.endurance() * 0.3
        }
        SkillCategory::Academic => {
            mental.intelligence() * 0.6 + mental.wisdom() * 0.4
        }
        SkillCategory::Survival => {
            mental.wisdom() * 0.4 + mental.willpower() * 0.3 + physical.endurance() * 0.15 + physical.agility() * 0.15
        }
    };
    // 映射到 0.7–1.0
    0.7 + raw * 0.3
}
```

**为什么用 trait 而不是直接引用 NpcData？**
- 技能系统是地基模块——不应依赖上层模块的数据结构
- 玩家没有 NpcData 但需要同样的 skill 升级机制
- 测试和未来扩展（如动物也可能有技能？）不依赖特定 NPC 类型

**玩家实现**：MentalAccess 和 PhysicalAccess 均返回全 1.0。玩家的技能进步完全取决于实际做了什么——不受虚拟属性削弱。

> **charisma 的特殊性**：charisma 参与教学能力公式（003 §1.3），但不参与 mental_factor（学习速度）。社交技能不在技能系统管辖——但 charisma 作为心智基础属性，对教学效率的修正属于技能系统管辖的教学机制。

### 3.2 第二层：天生经验倍率

```rust
/// NPC 出生时为每个技能独立 roll 一次——此后不可变
///
/// 分布：正态 N(1.0, 0.15), clamped [0.7, 1.3]
///   0.7   = 天生迟钝（学习慢 30%）—— ~2.3% NPC
///   1.0   = 平均—— ~68% NPC
///   1.3   = 天生奇才（学习快 30%）—— ~2.3% NPC
///
/// 这是"天生我才"的部分——
/// 铁匠世家哥哥 roll 到 1.22（学锻造快 22%），弟弟 roll 到 0.83（慢 17%）
/// 开发者不决定谁更适合锻造，随机性+心智属性分布决定
///
/// 存储为 SkillEntry.innate_aptitude —— 永不改变
```

**这个设计的关键约束**：
- 每个 NPC 每个技能独立 roll——一个 NPC 可能剑术 1.25 但魔法 0.78
- 不由种族决定——不由开发者硬编码
- 不由人格决定——人格通过 mental_factor（第一层）影响，不是通过 innate_aptitude
- 一次性确定——不可训练、不可改变

### 3.3 第三层：交叉训练 + 同组共享

见 [[002-技能分类体系|002 分类体系]] 中的完整交叉规则。

三层叠加的最终 XP 获得：

```
effective_xp_gain = base_action_xp
    × mental_factor(mental, physical, skill.category)    // 第一层: 0.7–1.0
    × innate_aptitude                                    // 第二层: 0.7–1.3
    × (1.0 + sum of cross_boosts)                        // 第三层: 已有同组高技能的加速
```

## 四、add_xp 管线

### 4.1 XpSource —— 打破递归的关键

```rust
/// 经验来源标记 —— 防止交叉训练的无限循环
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum XpSource {
    /// 实际使用技能（挥剑/施法/锻造）—— 触发交叉训练
    DirectAction,

    /// 交叉训练产生 —— 不可再触发交叉训练（打破递归！）
    CrossTraining,

    /// 教学获得（师承/学院）—— 类同 DirectAction, 触发交叉训练
    Teaching,

    /// 秘传注入（Magic 模块调用 add_xp_direct）—— 触发交叉训练
    DirectInfusion,
}

impl XpSource {
    pub fn triggers_cross_training(&self) -> bool {
        match self {
            XpSource::DirectAction | XpSource::Teaching | XpSource::DirectInfusion => true,
            XpSource::CrossTraining => false,  // ← 这是打破递归的关键
        }
    }
}
```

### 4.2 add_xp —— 核心管线

```rust
/// 向指定技能添加经验
///
/// # Arguments
/// * `skill_id` - 目标技能
/// * `base_xp` - 基础经验值（由消费模块定义——见§2.4）
/// * `source` - 经验来源（决定是否触发交叉训练）
/// * `mental` - 心智属性访问（NPC或玩家实现）
/// * `physical` - 身体属性访问（Life实现）
/// * `all_skills` - NPC全部技能的可变引用（用于交叉训练写入）
/// * `skill_def` - 技能定义（含 category/subgroup/cross_group）
/// * `world_time` - 当前游戏时间
///
/// # Returns
/// * 交叉训练事件列表（供 History 记录）—— 如果 source 不触发交叉训练则为空
fn add_xp(
    skill_id: SkillId,
    base_xp: f32,
    source: XpSource,
    mental: &dyn MentalAccess,
    physical: &dyn PhysicalAccess,
    all_skills: &mut BTreeMap<SkillId, SkillEntry>,
    skill_def: &SkillDef,
    world_time: GameTimestamp,
) -> Vec<CrossTrainingEvent> {
    let skill = all_skills.entry(skill_id).or_default();

    // 1. 心智因子
    let mf = mental_factor(mental, physical, skill_def.category);

    // 2. 同组加速（已有高技能→学同组新技能更快）
    //    上限 MAX_CROSS_BOOST = 0.5——防止极端情况（5个Lv100同组=1.0）
    let cross_boost = if source.triggers_cross_training() {
        compute_cross_boost(skill_id, all_skills, skill_def).min(0.5)
    } else {
        0.0
    };

    // 3. 最终 XP
    let effective = base_xp * mf * skill.innate_aptitude * (1.0 + cross_boost);
    skill.xp += effective;
    skill.total_xp_earned += effective;
    skill.times_used += 1;
    skill.level = level_from_xp(skill.xp).min(100.0);

    // 4. 交叉训练（仅 DirectAction/Teaching/DirectInfusion 触发）
    let mut cross_events = Vec::new();
    if source.triggers_cross_training() {
        cross_events = apply_cross_training(skill_id, base_xp, mental, physical, all_skills, skill_def);
    }

    cross_events
}
```

### 4.3 add_xp_direct —— Magic 秘传入口

```rust
/// 直接注入经验——供 Magic 模块的秘传路径调用
///
/// 与 add_xp 的区别：
///   - 不需要 "base_xp" —— 注入量由 Magic 模块的仪式机制决定
///   - 不经过 mental_factor 修正 —— 仪式效果不由学生心智决定
///   - 仍受 innate_aptitude 影响（天赋好的学生从注入中获得更多）
///   - source = DirectInfusion → 触发交叉训练
///
/// 此函数只是安全写入入口——秘传的契约条款、仪式风险、
/// 注入知识的类型等全部由 Magic 模块管辖。
fn add_xp_direct(
    skill_id: SkillId,
    xp_amount: f32,
    innate_aptitude: f32,
    all_skills: &mut BTreeMap<SkillId, SkillEntry>,
    skill_def: &SkillDef,
    world_time: GameTimestamp,
    mental: &dyn MentalAccess,
    physical: &dyn PhysicalAccess,
) -> Vec<CrossTrainingEvent> {
    let skill = all_skills.entry(skill_id).or_default();
    let effective = xp_amount * innate_aptitude;
    skill.xp += effective;
    skill.total_xp_earned += effective;
    skill.level = level_from_xp(skill.xp).min(100.0);

    // 秘传注入触发交叉训练
    apply_cross_training(skill_id, xp_amount, mental, physical, all_skills, skill_def)
}
```

## 五、技能等级与 proficiency 转换

Combat 和 Magic 需要 `proficiency`（0.0–1.0）而非 `level`（0–100）：

```rust
impl SkillEntry {
    /// 熟练度 —— 用于 combat/magic 公式
    /// 线性映射: level 0→0.0, level 100→1.0
    pub fn proficiency(&self) -> f32 {
        (self.level / 100.0).clamp(0.0, 1.0)
    }
}
```

- Combat 003 公式：`skill_proficiency × (1 - fatigue × 0.5)` —— 直接 = proficiency()
- Magic 016 本能化：`times_used ≥ 500` —— 读 times_used 字段

## 六、初始技能生成

World Gen P8（NPC 初始化）和运行时 NPC 出生（婴儿长大/移民进城）需要 `generate_starting_skills()`：

```rust
/// 为 NPC 生成初始技能集
///
/// # Arguments
/// * `age_years` - NPC 当前年龄
/// * `profession_skills` - 职业主/副技能列表（来自 SkillProfile TOML 或 Profession 系统）
/// * `mental` - 心智属性
/// * `physical` - 身体属性
/// * `settlement_tech_level` - 定居点科技水平（影响可接触的技能上限）
/// * `rng` - 随机数生成器
///
/// # Algorithm
/// 1. 分配职业主技能到目标范围（如铁匠: 锻造 30-70，取决于年龄+天赋+科技）
/// 2. 模拟交叉训练效果（主技能→副技能，基于 XP 反推）
/// 3. 加入随机噪声 ±5 级
/// 4. 添加出生环境自然获得的生存技能（采集/攀爬/游泳——取决于地形）
fn generate_starting_skills(
    age_years: u32,
    profession_skills: &[(SkillId, f32, f32)],  // (skill, min_level, max_level)
    mental: &dyn MentalAccess,
    physical: &dyn PhysicalAccess,
    settlement_tech_level: TechEra,
    starting_biome: BiomeParams,
    rng: &mut impl Rng,
) -> BTreeMap<SkillId, SkillEntry>;
```

详细算法见 [[003-教学与跨模块接口|003 §七]]。

## 七、技能里程碑 —— History 接口

技能系统**不发射事件**（遵循 CHG-013 接口模式——History 主动拉取）：

```rust
/// History 005（大日志）或 History 002（因果链）定期调用
fn skills_snapshot(all_skills: &BTreeMap<SkillId, SkillEntry>) -> Vec<SkillSnapshot> {
    all_skills.iter()
        .filter(|(_, entry)| entry.level >= 50.0)  // 仅记录≥50的技能
        .map(|(id, entry)| SkillSnapshot {
            skill_id: *id,
            level: entry.level,
            total_xp: entry.total_xp_earned,
            times_used: entry.times_used,
        })
        .collect()
}

struct SkillSnapshot {
    skill_id: SkillId,
    level: f32,
    total_xp: f32,
    times_used: u64,
}
```

History 模块对比快照检测里程碑（50→70→90→100），生成相应历史事件。

## 八、明确不做

| 项目 | 理由 |
|------|------|
| **社交技能** | 不在管辖范围——由对话系统的实际表现和关系网络反推 |
| **经济技能**（交易/讨价还价） | 本质是社交行为——不在管辖范围 |
| **技能衰减/废退** | 技能只升不降——肌肉记忆不消退 |
| **种族固定天赋** | 不由开发者硬编码——世界生成规则决定种族属性分布 |
| **认知错乱** | 删除——秘传副作用仅剩契约条款 |
| **显式前置条件** | 世界规则自然产生依赖——不需要硬编码"需要 Lv5 剑术才能学斧术" |
| **技能树/天赋树** | 无"升级点技能树"——用进废退。不做 |
| **技能等级解锁招式** | "剑术80级解锁旋风斩"——不存在。招式 = 积木组合（Combat 005），不由等级锁定 |
