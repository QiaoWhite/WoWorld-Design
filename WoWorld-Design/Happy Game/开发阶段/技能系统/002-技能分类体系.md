# 002 — 技能分类体系

## 一、五大类别总览

| 类别 ID | 类别名 | 管辖内容 | 不在管辖 |
|---------|--------|---------|---------|
| 1 | **Combat**（战斗） | 近战武器 / 远程武器 / 徒手 / 防御 / 战术 | 招式效果、伤害公式 → Combat 模块 |
| 2 | **Magic**（魔法） | 十元素 / 施法技艺 / 法器运用 / 魔法工程 | 法术威力、魔力消耗 → Magic 模块 |
| 3 | **Artisan**（工艺） | 金属 / 纺织 / 木石 / 精密 / 日用 | 配方输入输出 → 物品系统 006 |

| 5 | **Survival**（生存） | 采集 / 移动 / 感知 / 野外工艺 | 采集产物 → Life 009（植物） |

## 二、SkillDef —— TOML 数据格式

技能定义是**数据**，不是代码：

```toml
# skills/combat.toml

[[skill]]
id = 0x0101_0000_0000_0001  # combat.melee.sword
name_key = "skill.combat.sword"
category = "Combat"
subgroup = "MeleeWeapon"
cross_group = "MeleeWeapon"
description_key = "skill.combat.sword.desc"

[[skill]]
id = 0x0101_0000_0000_0002  # combat.melee.axe
name_key = "skill.combat.axe"
category = "Combat"
subgroup = "MeleeWeapon"
cross_group = "MeleeWeapon"

[[skill]]
id = 0x0102_0000_0000_0001  # combat.ranged.bow
name_key = "skill.combat.bow"
category = "Combat"
subgroup = "RangedWeapon"
cross_group = "RangedWeapon"
```

```toml
# skills/magic.toml

[[skill]]
id = 0x0200_0000_0000_0001  # magic.fire
name_key = "skill.magic.fire"
category = "Magic"
subgroup = "Element"
cross_group = "Fire"

[[skill]]
id = 0x0200_0000_0000_0002  # magic.water
name_key = "skill.magic.water"
category = "Magic"
subgroup = "Element"
cross_group = "Water"
```

**为什么 TOML 而不是枚举？**
- 新增技能不需要修改 Rust 代码
- Mod 可以在 category 6-255 自由添加技能
- 交叉训练组可以数据配置——调整 cross_ratio 不需要重新编译

#### cross_pair TOML — 偏离默认值的例外登记

```toml
# config/cross_pair.toml
# 仅登记偏离默认层级的技能对——ID 参照 002 §六 SkillId 快速参考表

[[cross_pair]]
skill_a = 0x0200_0000_0000_0001  # magic.fire
skill_b = 0x0200_0000_0000_0003  # magic.wind
ratio = 0.15                      # 覆盖默认 0.10 → 0.15（亲和）

[[cross_pair]]
skill_a = 0x0200_0000_0000_0002  # magic.water
skill_b = 0x0200_0000_0000_0001  # magic.fire
ratio = 0.04                      # 覆盖默认 0.10 → 0.04（排斥）

[[cross_pair]]
skill_a = 0x0301_0000_0000_0001  # artisan.metalwork.blacksmith
skill_b = 0x0101_0000_0000_0001  # combat.melee.sword
ratio = 0.05                      # 跨类别逻辑关联（默认0.0→0.05）
```

**谁维护**：元素对由 Magic 003 权威定义。工艺/学术/生存的跨类别对由各自模块维护。技能系统只提供配置能力 + 查询管线。

#### 交叉训练：抽象 Tier + TOML 数据 = 权威关系

技能系统定义**三个抽象比率 Tier**——不定义哪些元素对属于哪个 Tier：

| Tier | 比率 | 语义 | 数据来源 |
|------|------|------|---------|
| 亲和 (Affinity) | 0.15 | 元素间自然协同——训练A对B有显著帮助 | Magic 003 "亲和关系" 表 |
| 中性 (Neutral) | 0.10 | 默认——同大类无特殊关系 | 未定义亲和/排斥的所有元素对 |
| 排斥 (Opposition) | 0.04 | 元素间天然对抗——训练A对B帮助极小 | Magic 003 "排斥关系" 表 |
| 灵↔任何 | 0.10 | 灵是中立桥梁——对所有元素基础共通 | Magic 003 "灵不参与亲和/排斥" |
| 血↔任何 | 0.10 | 血是八元素预调和——对所有元素中立 | Magic 003 "血是中立" |

**交叉查询逻辑**：

```
cross_ratio(skill_a, skill_b):
    if cross_pair_table.contains((a, b)) → use cross_pair value
    elif a.subgroup == b.subgroup → same_subgroup_ratio (同子组)
    elif a.category == b.category → same_category_ratio (0.10, 默认中性)
    elif cross_category_logic(a, b) → 0.05
    else → 0.0
```

## 四、交叉训练完整规则

### 4.1 触发条件

只有 `XpSource::DirectAction | Teaching | DirectInfusion` 触发交叉训练。`CrossTraining` 不传播。

### 4.2 交叉训练比率查询

`cross_training_ratio(skill_a, skill_b, def_a, def_b)` 的优先级：

```
1. cross_pair TOML 显式定义的比率（如有）
     → 使用 cross_pair.ratio
2. 同 cross_group（def_a.cross_group == def_b.cross_group）
     → 使用子组默认比率（来自 def.subgroup_ratio: 0.20/0.15/0.12）
3. 同 category 不同 cross_group（def_a.category == def_b.category）
     → 使用类别默认比率（来自 def.category_default_ratio: 通常 0.10）
4. 跨类别但 cross_pair 已登记
     → 见 §3.2-§3.5 的 cross_pair 示例（0.05）
5. 其他
     → 0.0（无交叉训练）
```

**默认比率取值**：

| 层级 | 默认值 | 来源 |
|------|--------|------|
| 同子组 | 0.20 / 0.15 / 0.12 | SkillDef.subgroup_ratio（每组可不同） |
| 同类别 | 0.10 | SkillDef.category_default_ratio |
| Tactics → 其他 Combat | 0.05（覆盖类别默认） | SkillDef.category_default_ratio = 0.05 |
| 跨类别逻辑关联 | 0.05（必须在 cross_pair 中显式登记） | cross_pair TOML |
| 元素亲和/排斥 | 0.15 / 0.04（必须在 cross_pair 中显式登记，Magic 003 权威） | cross_pair TOML |
| 无关系 | 0.0 | 默认 |

> **关键约定**：0.10 的 same_category 默认值、0.05 的跨类别默认值、Tactics 的降级 0.05——这些是在 SkillDef 子组定义中配置的参数，不需要 cross_pair TOML。cross_pair 只登记**偏离这些默认值**的个别技能对。

### 4.3 交叉训练公式

```
主技能A 获得 DirectAction XP = base_xp × mental_factor(A) × innate_aptitude(A) × (1 + cross_boost)
→ 同组技能B 被动获得:
    cross_xp = base_xp × mental_factor(B) × innate_aptitude(B) × cross_training_ratio
```

交叉训练获得的 XP 仅影响 B 的 xp/level——不影响 B 的 `total_xp_earned`（那个只记录直接获得）和 `times_used`。

#### `compute_cross_boost` — 已有高技能加速主技能学习

```rust
/// 计算同组已有高技能对学习主技能的加速
///
/// 反向于交叉训练：剑术 Lv90 的人学斧术比零基础快——
/// 因为他已经理解近战武器的重量、平衡、节奏。
///
/// 上限 MAX_CROSS_BOOST = 0.5（防止极端情况）
fn compute_cross_boost(
    skill_id: SkillId,
    all_skills: &BTreeMap<SkillId, SkillEntry>,
    skill_def: &SkillDef,
) -> f32 {
    let mut sum = 0.0;
    let cross_group = &skill_def.cross_group;
    let category = skill_def.category;

    for (other_id, other_entry) in all_skills.iter() {
        if *other_id == skill_id { continue; }
        let other_def = /* lookup SkillDef for other_id */;

        let ratio = cross_training_ratio(skill_id, *other_id, skill_def, other_def);
        // cross_training_ratio 优先级见 §4.2

        if ratio > 0.0 {
            sum += other_entry.proficiency() * ratio;
        }
    }

    sum  // 调用方负责 .min(MAX_CROSS_BOOST)
}
```

#### `apply_cross_training` — 主技能使用→向其他技能传播 XP

```rust
/// 使用技能 A 后，向同组/关联技能 B 传播交叉 XP
///
/// 不变式：
///   1. 不传播到自身
///   2. 目标已达天花板(cross_training_cap) → 跳过
///   3. 目标等级 > 源等级 → 跳过（不反向交叉）
///   4. 交叉 XP 不影响 target.total_xp_earned 或 target.times_used
fn apply_cross_training(
    source_skill: SkillId,
    base_xp: f32,                    // 源技能的基础 XP（未修正）
    mental: &dyn MentalAccess,
    physical: &dyn PhysicalAccess,
    all_skills: &mut BTreeMap<SkillId, SkillEntry>,
    skill_def: &SkillDef,
) -> Vec<CrossTrainingEvent> {
    let mut events = Vec::new();
    let source_level = all_skills.get(&source_skill).map(|e| e.level).unwrap_or(0.0);
    let ceiling = cross_training_cap(source_level);

    for (&target_id, target_entry) in all_skills.iter_mut() {
        if target_id == source_skill { continue; }
        if target_entry.level >= ceiling { continue; }
        if target_entry.level > source_level { continue; }

        let target_def = /* lookup SkillDef for target_id */;
        let ratio = cross_training_ratio(source_skill, target_id, skill_def, target_def);
        if ratio <= 0.0 { continue; }

        let cross_xp = base_xp
            * mental_factor(mental, physical, target_def.category)
            * target_entry.innate_aptitude
            * ratio;

        target_entry.xp += cross_xp;
        target_entry.level = level_from_xp(target_entry.xp).min(ceiling);

        events.push(CrossTrainingEvent {
            source_skill,
            target_skill: target_id,
            cross_xp,
        });
    }

    events
}

struct CrossTrainingEvent {
    source_skill: SkillId,
    target_skill: SkillId,
    cross_xp: f32,
}
```

### 4.4 天花板限制

交叉训练**单独**作用的上限：

```
cross_training_cap(source_level: f32) -> f32 {
    min(40.0, source_level * 0.5)
}
```

| 主技能等级 | 交叉训练天花板 |
|-----------|--------------|
| 10 | min(40, 5) = 5 |
| 30 | min(40, 15) = 15 |
| 60 | min(40, 30) = 30 |
| 80 | min(40, 40) = 40 |
| 95 | min(40, 47.5) = 40 |

**这意味着**：
- 一个剑术 Lv95 的剑圣，通过交叉训练最多让斧术到 Lv40——"会用但不算专家"
- 要从 Lv40 以上继续提升斧术，**必须亲自练斧子**
- Lv40 是交叉训练的绝对天花板——任何技能不能仅通过"相关技能好"而超过这个水平

### 4.5 交叉训练不触发的边界情况

- 目标技能已有等级 ≥ cross_training_cap(source_level) → 不触发交叉（已达天花板）
- 目标技能等级 > source_level → 不触发（不能从低技能"反向交叉"）

## 五、SkillId 布局细则

```
u64 bits:
[63:56] category    (8 bits) — 1=Combat, 2=Magic, 3=Artisan, 4=Academic, 5=Survival
[55:48] subgroup     (8 bits) — 子组编号（每类别最多 256 子组）
[47:32] reserved     (16 bits) — 保留为 0
[31:0]  skill_index  (32 bits) — 技能序号（每子组 42 亿技能——远超需求）

示例：
0x0101_0000_0000_0001 = Combat(01) / MeleeWeapon(01) / 剑术(#1)
0x0200_0000_0000_0001 = Magic(02) / Element(00) / 火(#1)
0x0301_0000_0000_0001 = Artisan(03) / Metalwork(01) / 锻造(#1)
```

通过查看前 16 bit 即可不查表判断技能的类别和子组——在交叉训练查询等热路径上节省查表开销。

## 八、技能定义的重复使用

同一 SkillId 可被多个消费模块以不同方式使用——技能系统不限制：

```
锻造（SkillId 0x0301_0000_0000_0001）:
  → 物品系统 006: SkillRequirement { skill: 锻造, min_level: 20 } → 制造铁剑配方
  → Combat: 不查询（锻造不是战斗技能）
  → NPC: IntrinsicGoal::MasterSkill { skill: 锻造, target: 50 }
  → World Gen: generate_starting_skills { 锻造: (30, 70) } → 铁匠职业
```

一个技能可以同时是制造门槛、NPC人生目标、职业初始值、交叉训练源——互不干扰。
