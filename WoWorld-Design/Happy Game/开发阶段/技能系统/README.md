# 技能系统（Skill System）— README

## 模块定位

技能系统是 WoWorld 四大**地基层模块**（物品/技能/天气/对话）之二——定义游戏中一切的"熟练度"：一个NPC（或玩家）有多擅长做什么。

**核心原则**：技能系统是**基础设施**，不是"玩法模块"——它只定义"熟练度怎么计量、怎么提升、怎么传授"，不定义"剑术80级能放什么招式"（那是 Combat 的职责）。

技能系统**不管辖的范围**：
- **社交能力** — 不由技能数值衡量。由 NPC 对话系统的实际表现、关系网络、charisma 属性反推
- **经济能力**（交易/讨价还价） — 本质是社交行为

## 关键设计决策

| 决策 | 结论 |
|------|------|
| **管辖范围** | 5大类：Combat（战斗）/ Magic（魔法）/ Artisan（工艺）/ Academic（学术）/ Survival（生存）。社交/经济**不在管辖内** |
| **升级模型** | 指数减速累积XP（base=100, k=0.04）——初期快（0→30约2周游戏时间），后期极慢（90→100约半年）。**用进不退**——技能只升不降 |
| **硬上限** | level = 100（绝对人体极限）。99.9% NPC一辈子到不了任何技能的90+ |
| **天赋** | **三层叠加**：①心智属性（MentalAccess trait——从NPC活人感的MentalAttributes派生，range 0.7-1.0） ②天生经验倍率（每NPC每技能独立roll，0.7-1.3，出生时锁定） ③交叉训练（同组技能被动获得微量XP，上限=min(40, source_level×0.5)） |
| **种族天赋** | **不由开发者硬编码**。种族对技能的影响通过心智属性分布间接体现——世界生成规则决定 |
| **玩家** | 完全相同的SkillEntry和add_xp机制。MentalAccess+PhysicalAccess返回全1.0——不受虚拟属性削弱 |
| **交叉训练** | 非递归。天花板=min(40, source_level×0.5)。四种比率：同子组0.20 / 同大类0.10 / 跨类逻辑关联0.05 / 对立元素0.04 |
| **教学** | 统一在技能系统。四种路径：自学（1.0×）/ 师承（1.2-3.0×，上限=导师95%）/ 学院（1.3×）/ 秘传（→Magic模块调用 add_xp_direct） |
| **认知错乱** | **删除**——不在此系统。秘传副作用仅剩契约条款 |
| **教学风险** | TeachingRisk trait——默认空实现。Combat可选（对战轻伤），Magic可选（契约条款）。大多数技能教学无特殊风险 |
| **基础XP值** | **不由技能系统定义**——各消费模块各自决定（Combat: 命中5XP/击杀15XP, Magic: 施法5-20XP, Artisan: 配方数据定义） |
| **技能定义** | TOML数据驱动——不做枚举。增删技能不需要修改代码 |
| **前置条件** | **不做**——世界规则自然产生依赖（没人教+没熔炉=学不了锻造）。不硬编码 |
