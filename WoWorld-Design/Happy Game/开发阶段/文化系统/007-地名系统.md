## 一、总览

### 1.1 问题

在 WoWorld 中，同一座山在不同文化中有不同的名字。一条被称为"白山"的山峰，在邻文化中可能被连着周围山脉一起统称为"彩色山脉"——甚至没有独立名称。一片海域，一个文明视其为"广洋"的一部分，另一个文明则认为它是"东海"和"南洋"两片独立的海。聚落、政体、森林、河流——都存在同样的命名差异。

**这不是 bug。这是世界真实感的来源。**

### 1.2 核心设计

地名系统（Toponymy System）是文化系统的一个子模块——它处理**同一客观地理实体在不同文化视角下的不同命名**。

```
客观层（世界生成）： GeographicEntity —— 山脉、河流、湖泊、海洋、森林……
                         每个实体有客观几何边界和物理属性

主观层（文化系统）：    GeographicName —— 每个文化对该实体的称呼
                         包括：endonym（本地名）、exonym（他称）、历史名
```

**设计原则**：
- 地理实体是客观的——世界生成确定"存在什么"
- 地名是主观的——每个文化独立生成对该实体的称呼
- 命名是涌现的——种子驱动模板生成，不手写任何地名
- 地名是历史的——实体改变后旧名不消失，成为历史层积
- `CultureQuery` trait 提供统一查询接口——消费方不知道地名如何存储

## 二、GeographicEntity —— 客观地理实体

### 2.1 实体类型

世界生成管线负责识别物理世界中显著的地理特征并创建 `GeographicEntity` 记录。地名系统不定义实体——只消费它们。

```rust
/// 地理实体标识符——全局唯一，世界生成时分配
/// 
/// 定义位置：世界生成模块（待新增）
/// 消费位置：文化系统（命名）、历史系统（事件地点引用）、NPC（记忆中的地点）
#[derive(Copy, Clone, PartialEq, Eq, Hash, Serialize, Deserialize)]
pub struct GeographicEntityId(pub u32);

/// 地理实体类型——由世界生成模块定义
/// 
/// 地名系统消费此枚举以决定命名模板选择。
/// 
/// 注意：此枚举覆盖**自然地理实体**。聚落(SettlementId)和政体(PolityId)是独立实体类型——
/// 它们的命名在各自的模块中处理，但名称也通过 CultureQuery 查询。
/// SettlementId 和 PolityId 可用 From<> 转换为 GeographicEntityId（如果该实体被注册为地理实体）。
pub enum GeographicEntityType {
    // ━━━ 山岳类 ━━━
    /// 独立山峰——相对高差大、可独立识别的单峰
    MountainPeak,
    /// 山脉——连续的山脊线，包含多座山峰
    MountainRange,
    /// 火山——锥形山体、火山口（无论死活）
    Volcano,
    /// 高原/台地——顶部平坦、边缘陡峭
    Plateau,
    /// 峡谷——深度切割的狭窄谷地
    Canyon,

    // ━━━ 水域类 ━━━
    /// 河流——Strahler ≥ 3 级的命名河流
    River { strahler_order: u8 },
    /// 湖泊——面积 > 识别阈值的静止水体
    Lake,
    /// 海洋——广阔咸水体
    Ocean,
    /// 海域——被陆地半包围的海洋子区域
    Sea,
    /// 海峡——连接两片海域的狭窄水道
    Strait,
    /// 海湾——海岸线凹陷形成的大型水域
    Bay,
    /// 瀑布——落差显著的跌水
    Waterfall { height_m: f32 },

    // ━━━ 陆地类 ━━━
    /// 森林——连续林冠覆盖区域
    Forest,
    /// 沙漠——干旱区域
    Desert,
    /// 平原/草原——广阔平坦区域
    Plain,
    /// 湿地/沼泽——常年或季节性积水区域
    Wetland,
    /// 冰原/冰川——永久冰覆盖区域
    Glacier,

    // ━━━ 海岸与岛屿类 ━━━
    /// 岛屿——被水完全包围的陆地
    Island,
    /// 群岛——紧密聚集的岛屿组
    Archipelago,
    /// 半岛——三面环水的陆地突出部
    Peninsula,
    /// 海角/岬角——伸入海中的陆地尖端
    Cape,
    /// 礁石/暗礁——近水面或水下的岩石结构（航海危险）
    Reef,

    // ━━━ 特殊类 ━━━
    /// 绿洲——沙漠中的水源和植被区域
    Oasis,
    /// 洞穴/洞穴系统入口
    Cave,
    /// 温泉/间歇泉
    HotSpring,
    /// 山谷/谷地——山间的可居住低地
    Valley,
    /// 山口/山隘——穿越山脉的可通行最低点
    MountainPass,
}

/// 地理实体上下文——用于查询实体周边的同类实体密度
/// 
/// 定义位置：世界生成模块
/// 消费位置：地名系统（nameworthiness 独特性因子）
pub struct GeographicContext {
    /// 空间索引——按实体类型分组的四叉树
    type_spatial_indices: FxHashMap<GeographicEntityType, SpatialIndex>,
}
impl GeographicContext {
    /// 在指定位置附近、指定半径（km）内计算同类实体数量
    pub fn count_similar_entities(&self, entity_type: GeographicEntityType, center: WorldPos, radius_km: f32) -> u32;
}

/// 地理实体存储——世界生成维护
/// 
/// 定义位置：世界生成模块（共享 WorldState 的一部分）
/// 消费位置：地名系统（实体查询）、历史系统（事件地点）、NPC（记忆中的位置）
pub struct GeographicEntityStore {
    entities: Vec<GeographicEntity>,
    context: GeographicContext,
}
impl GeographicEntityStore {
    pub fn get(&self, id: GeographicEntityId) -> &GeographicEntity;
    pub fn get_mut(&mut self, id: GeographicEntityId) -> &mut GeographicEntity;
    pub fn spatial_context(&self) -> &GeographicContext;
}

/// 地理实体——由世界生成管线创建
/// 
/// 定义位置：世界生成模块
/// 存储位置：共享世界状态（WorldState）
pub struct GeographicEntity {
    pub id: GeographicEntityId,
    pub entity_type: GeographicEntityType,
    
    /// 客观几何边界（轴对齐包围盒）
    pub bounding_region: Aabb2D,
    /// 代表点——用于距离计算和地图标注
    pub representative_point: WorldPos,
    
    /// 物理属性——用于命名特征提取
    pub physical_attrs: GeographicPhysicalAttrs,
    
    /// 创建时间（游戏时间）
    pub created_at: GameTime,
    
    /// 实体状态
    pub status: EntityStatus,
    
    /// 前身实体——此实体取代了什么
    pub preceded_by: Option<GeographicEntityId>,
    /// 后继实体——什么取代了此实体
    pub succeeded_by: Option<GeographicEntityId>,
    /// 状态变更的时间
    pub status_changed_at: Option<GameTime>,
}

/// 地理实体的物理属性——命名模板的输入
pub struct GeographicPhysicalAttrs {
    /// 最高点海拔（m）
    pub max_elevation: f32,
    /// 相对高差（m）——从基底到顶点
    pub prominence: f32,
    /// 面积（km²）
    pub area_km2: f32,
    /// 长度（km）——河流长度、山脉长度、海岸线长度
    pub length_km: f32,
    /// 主要颜色——基于 satellite-pass 的地表推测
    /// 白(雪盖/白岩)、黑(玄武岩/暗密林)、红(铁锈/红岩)、绿(植被)、蓝(深水)、灰(裸岩)
    pub dominant_color: SurfaceColor,
    /// 植被覆盖率 0-1
    pub vegetation_cover: f32,
    /// 水体类型——实体包含的水体
    pub water_presence: WaterPresence,
}

pub enum SurfaceColor { White, Black, Red, Green, Blue, Grey, Brown, Gold }

pub enum WaterPresence { None, Seasonal, Permanent, IsWater }

pub enum EntityStatus {
    /// 活跃——如其类型定义般存在
    Active,
    /// 已被显著改变但实体仍可识别
    Altered { description: String },
    /// 已消亡——原始实体不再以原形式存在
    Defunct,
    /// 已被其他实体取代
    Superseded { by: GeographicEntityId },
}
```

### 2.2 命名价值评分——哪些实体值得被命名

不是所有物理特征都值得专名。一个直径 20m 的池塘不需要名字——NPC 说"那边那个水塘"就足够了。命名价值评分决定一个实体是否进入地名系统：

```rust
/// 命名价值评分——实体"值得被命名的程度"
/// 
/// 范围 0.0-1.0。≥ 0.4 → 可能有专名。≥ 0.7 → 一定有专名。
/// 此为四个因子的乘积模型——任何一项接近零则整体接近零。
pub fn nameworthiness(
    entity: &GeographicEntity,
    culture: &CultureCoreParams,
    distance_from_culture_center_km: f32,
    context: &GeographicContext,
) -> f32 {
    let physical = physical_prominence(entity);
    let cultural = cultural_relevance(&entity.entity_type, culture);
    let proximity = proximity_factor(distance_from_culture_center_km, culture);
    let uniqueness = uniqueness_factor(entity, context);
    
    physical * cultural * proximity * uniqueness
}
```

**因子一：物理显著性**

```rust
fn physical_prominence(entity: &GeographicEntity) -> f32 {
    let attrs = &entity.physical_attrs;
    match entity.entity_type {
        MountainPeak | Volcano => {
            // 相对高差 / 500m——500m 相对高差 = 1.0
            (attrs.prominence / 500.0).clamp(0.0, 1.0)
        }
        MountainRange => {
            // 长度 / 100km 且 最高点 > 区域平均 + 300m
            let length_score = (attrs.length_km / 100.0).clamp(0.0, 1.0);
            length_score
        }
        River { strahler_order } => {
            // 3 级=0.5, 5 级=1.0
            (strahler_order as f32 - 2.0).max(0.0) / 3.0
        }
        Lake => {
            // 对数缩放：1km²=0.2, 10km²=0.5, 100km²=0.8, 1000km²=1.0
            (attrs.area_km2.ln() / 7.0).clamp(0.0, 1.0)
        }
        Forest | Desert | Plain => {
            // 面积对数 + 该类型为该区域最大时 bonus
            let area_score = (attrs.area_km2.ln() / 8.0).clamp(0.0, 1.0);
            area_score
        }
        Ocean => 0.95,  // 海洋几乎总是被命名
        Sea => {
            (attrs.area_km2.ln() / 8.0).clamp(0.0, 1.0)
        }
        Waterfall { height_m } => {
            (height_m / 50.0).clamp(0.0, 1.0) // 尼亚加拉≈50m
        }
        Canyon => {
            (attrs.prominence / 300.0).clamp(0.0, 1.0) // 深度>300m=显著
        }
        Volcano => {
            // 火山总是高显著性——独特的锥形+文化意义
            (0.6 + attrs.prominence / 1000.0).clamp(0.0, 1.0)
        }
        Island => {
            // 对数：1km²=0.2, 100km²=0.6, 10000km²=1.0
            (attrs.area_km2.ln() / 9.0).clamp(0.0, 1.0)
        }
        Glacier => {
            (attrs.area_km2.ln() / 7.0).clamp(0.0, 1.0)
        }
        Oasis => 0.55, // 绿洲在沙漠中天然显著
        Reef => {
            (attrs.area_km2.ln() / 5.0).clamp(0.3, 1.0) // 对航海文化更重要——但在 physical 面保持中性
        }
        HotSpring | Cave => 0.45, // 独特但较小
        Valley | MountainPass | Peninsula | Cape | Bay | Strait | Plateau | Wetland | Archipelago => {
            // 通用：面积+长度的混合
            let area_score = (attrs.area_km2.ln() / 6.0).clamp(0.0, 1.0);
            let length_score = (attrs.length_km / 30.0).clamp(0.0, 1.0);
            (area_score + length_score) / 2.0
        }
    }
}
```

**因子二：文化相关性——不同文化关心不同的地貌**

```rust
fn cultural_relevance(entity_type: &GeographicEntityType, culture: &CultureCoreParams) -> f32 {
    let base = 0.5;
    
    let modifier = match entity_type {
        // 航海文化：海岸和水域特征
        GeographicEntityType::Ocean | GeographicEntityType::Sea
            => culture.openness_to_outsiders * 0.35 + 0.40,
        GeographicEntityType::Strait | GeographicEntityType::Bay
            => culture.openness_to_outsiders * 0.40 + 0.35,
        GeographicEntityType::Reef
            => culture.openness_to_outsiders * 0.35 + 0.25, // 航海危险——对航海文化极重要
        GeographicEntityType::Island | GeographicEntityType::Archipelago
            => culture.openness_to_outsiders * 0.30 + 0.40,
        GeographicEntityType::Peninsula | GeographicEntityType::Cape
            => culture.openness_to_outsiders * 0.25 + 0.40,
        
        // 军事文化：战略地形
        GeographicEntityType::MountainPass
            => culture.militarism * 0.40 + 0.35, // 山隘=军事要地
        GeographicEntityType::MountainPeak | GeographicEntityType::MountainRange
            => culture.militarism * 0.20 + culture.religiosity * 0.15 + 0.45,
        GeographicEntityType::River { .. }
            => culture.militarism * 0.15 + culture.long_term_orientation * 0.20 + 0.40, // 河流=天然防线+贸易路线
        
        // 农业文化：水和沃土
        GeographicEntityType::Lake
            => culture.long_term_orientation * 0.20 + 0.45,
        GeographicEntityType::Valley
            => culture.long_term_orientation * 0.25 + 0.40, // 山谷=可耕可居
        GeographicEntityType::Plain
            => culture.long_term_orientation * 0.20 + 0.45,
        
        // 宗教/艺术文化：奇观
        GeographicEntityType::Volcano
            => culture.religiosity * 0.30 + 0.50, // 火山=神圣/敬畏
        GeographicEntityType::Waterfall { .. }
            => culture.artistry * 0.30 + culture.religiosity * 0.15 + 0.35,
        GeographicEntityType::HotSpring
            => culture.indulgence * 0.25 + culture.religiosity * 0.15 + 0.35,
        GeographicEntityType::Cave
            => culture.religiosity * 0.20 + 0.30, // 洞穴=神秘/冥界入口
        
        // 荒漠文化：稀缺资源
        GeographicEntityType::Oasis
            => 0.85, // 对任何在沙漠中生存的文化都极重要
        GeographicEntityType::Desert
            => 0.55, // 沙漠本身就是显著的存在
        
        // 通用
        GeographicEntityType::Forest
            => culture.artistry * 0.15 + (1.0 - culture.militarism) * 0.10 + 0.45,
        GeographicEntityType::Glacier
            => 0.45, // 只对寒带文化重要
        
        _ => 0.45,
    };
    
    (base + modifier).clamp(0.15, 1.0)
}
```

**因子三：邻近性——越近越值得命名**

```rust
fn proximity_factor(distance_km: f32, culture: &CultureCoreParams) -> f32 {
    // 文化规模决定衰减距离
    let decay_distance = 50.0; // 默认 50km 衰减距离 [TUNING]
    
    // 指数衰减
    (-distance_km / decay_distance).exp().clamp(0.01, 1.0)
}
```

**因子四：独特性——在同区域中是否独一无二**

```rust
fn uniqueness_factor(entity: &GeographicEntity, context: &GeographicContext) -> f32 {
    let center = entity.representative_point;
    let radius_km = 50.0; // [TUNING]
    
    let similar_nearby = context.count_similar_entities(
        entity.entity_type, center, radius_km
    );
    
    // 如果周边 50km 内有 N 个同类实体——每个的"独特性"降低
    // 1 个=独一→1.0, 5个=0.5, 10个=0.33
    1.0 / (1.0 + similar_nearby as f32 * 0.2)
}
```

### 2.3 命名阈值

对于每个文化，只为 `nameworthiness >= 0.4` 的实体生成名称：

| 评分 | 处理 |
|------|------|
| **0.7+** | **一定有专名**——该文化核心地理认知 |
| **0.4-0.7** | **可能有专名**——取决于文化细节度（`artistry > 0.4` → 更倾向于命名） |
| **< 0.4** | **无专名**——NPC 使用方位+地貌类型的描述短语（"北边那条大河"），由语言表达模块在运行时生成 |

## 三、GeographicName —— 文化地名

### 3.1 数据结构

```rust
/// 一个文化对某个地理实体的称呼
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct GeographicName {
    /// 该文化语言中的地名字符串
    /// 例："白山"、"Storm Cape"、"龙牙山脉"
    pub name: String,
    
    /// 字面义——名称中的描述性含义
    /// 例："White Mountain"、"the cape of storms"、"Dragon's Teeth Range"
    pub literal_meaning: String,
    
    /// 名称类型
    pub name_type: GeographicNameType,
    
    /// 该名称是否已废弃（被新名替换）
    pub is_obsolete: bool,
    
    /// 该名称首次被使用的游戏时间
    pub first_used: GameTime,
}

pub enum GeographicNameType {
    /// 描述型：描述性特征 + 地貌通用词
    /// 例："白山" = "白" + "山"
    Descriptive {
        descriptor: String,          // 描述词（"白"、"长"、"黑"、"大"）
        feature_word: String,        // 该文化语言中的地貌词（"山"、"河"、"海"）
    },
    
    /// 关联型：与某物关联
    /// 例："King's Forest" = 与国王关联
    Associative {
        associated_with: String,     // 关联对象（"国王"、"龙"、"赫拉人"）
        association_type: AssociationType,
    },
    
    /// 音译型：从另一文化的地名通过本文化音系扭曲
    PhoneticAdaptation {
        source_culture: CultureId,
        source_name: String,         // 源文化中的原名
    },
    
    /// 原生型：不可分解的独特名称
    /// 例："Elenari"（星辰之子——直接是文化自名，不作地名的描述分解）
    Indigenous,
    
    /// 派生型：从另一实体的名称派生
    Derived {
        from_entity: GeographicEntityId,
        derivation: String,          // 派生方式描述
    },
}

pub enum AssociationType {
    /// 与统治者/名人关联
    Ruler,
    /// 与神话/传说中的生物关联
    MythicalCreature,
```

## 四、命名生成

### 4.1 生成管线

```
GeographicEntity (物理属性 + 类型)
        │
        ▼
[特征提取] → descriptor_candidates = [颜色, 大小, 形状, 方位, 关联资源, 关联生物]
        │
        ▼
[模板选择] → 基于实体类型 + 文化命名习惯选择命名模板

        ▼

        │
        ▼
```

### 4.4 命名模板渲染

```rust
fn generate_geographic_name(
    entity: &GeographicEntity,
    culture: &CultureCoreParams,
    lexicon: &CultureToponymyLexicon,
    rng: &mut impl Rng,
) -> GeographicName {
    let templates = naming_templates_for(&entity.entity_type);
    let template = pick_weighted(&templates, rng);
    
    let feature_word = lexicon.feature_words.get(&entity.entity_type)
        .cloned()
        .unwrap_or_else(|| generic_feature_word(entity.entity_type));
    
    let descriptor = pick_descriptor(entity, lexicon, rng);
    
    let (name, literal_meaning, name_type) = template.render(
        &descriptor, &feature_word, entity, culture, lexicon, rng
    );
    
    GeographicName {
        name,
        literal_meaning,
        name_type,
        is_obsolete: false,
        first_used: entity.created_at,
    }
}

/// 从实体物理属性中提取候选描述词
fn pick_descriptor(
    entity: &GeographicEntity,
    lexicon: &CultureToponymyLexicon,
    rng: &mut impl Rng,
) -> String {
    let attrs = &entity.physical_attrs;
    let mut candidates: Vec<(&str, f32)> = Vec::new();
    
    // 颜色——权重由 dominant_color 决定
    candidates.push((&lexicon.colors[&attrs.dominant_color], 0.35));
    
    // 大小
    if attrs.length_km > 50.0 || attrs.area_km2 > 200.0 {
        candidates.push((&lexicon.size_large, 0.25));
    }
    if attrs.length_km < 5.0 && attrs.area_km2 < 10.0 {
        candidates.push((&lexicon.size_small, 0.15));
    }
    
    // 方位（相对于文化中心）
    let dir = direction_from_culture_center(entity.representative_point, culture);
    candidates.push((dir.name_in(lexicon), 0.20));
    
    // 气象相关——海拔 > 1500m → 常被云雾覆盖 → "雾"前缀
    if attrs.max_elevation > 1500.0 {
        candidates.push((&lexicon.weather_misty, 0.10));
    }
    
    // 独特性相关——如果实体是独一无二的 → "大"前缀
    // （在 nameworthiness 中 uniqueness 已评估）
    
    weighted_pick(&candidates, rng).to_string()
}
```

## 五、Exonym 生成——跨文化他称

当一个文化首次"知道"另一文化区域的地理实体时——懒生成他称：

```rust
fn get_or_generate_exonym(
    entity_id: GeographicEntityId,
    observer_culture_id: CultureId,
    toponymy: &mut ToponymySystem,
    culture_query: &dyn CultureQuery,
    entity_store: &GeographicEntityStore,
) -> GeographicName {
    let names = toponymy.get_names(entity_id);
    
    // 已缓存 → 直接返回
    if let Some(exonym) = names.current.get(&observer_culture_id) {
        return exonym.clone();
    }
    
    // 懒生成
    let entity = entity_store.get(entity_id);
    let observer = culture_query.core_params(observer_culture_id);
    let observer_lexicon = toponymy.get_lexicon(observer_culture_id);
    let local_name = names.current.get(&names.primary_culture);
    let mut rng = toponymy.exonym_rng(entity_id, observer_culture_id);
    
    let exonym = generate_exonym_inner(
        entity, observer, observer_lexicon, local_name, &mut rng
    );
    
    names.current.insert(observer_culture_id, exonym.clone());
    exonym
}

fn generate_exonym_inner(
    entity: &GeographicEntity,
    observer: &CultureCoreParams,
    observer_lexicon: &CultureToponymyLexicon,
    local_name: Option<&GeographicName>,
    rng: &mut impl Rng,
) -> GeographicName {
    let roll: f32 = rng.gen_range(0.0..1.0);
    
    if roll < 0.60 {
        // 60%：音变——从本地名经由观察者音系扭曲
        if let Some(local) = local_name {
            let corrupted = phonetic_adapt(local, observer_lexicon);
            return GeographicName {
                name: corrupted.clone(),
                literal_meaning: format!("phonetic adaptation of '{}'", local.name),
                name_type: GeographicNameType::PhoneticAdaptation {
                    source_culture: local_name.unwrap().source_culture(),
                    source_name: local.name.clone(),
                },
                is_obsolete: false,
                first_used: GameTime::now(),
            };
        }
        // 无本地名→降级为描述型
    }
    
    if roll < 0.75 {
        // 15%：地理描述——从观察者视角重新描述该实体
        return generate_geographic_name(entity, observer, observer_lexicon, rng);
    }
    
    if roll < 0.85 {
        // 10%：关联历史事件——如果两个文化有共享历史
        // （v1.0 简化：降级为描述型）
        return generate_geographic_name(entity, observer, observer_lexicon, rng);
    }
    
    // 10%：观察者文化不知道该实体或觉得不值得命名
    // → 返回空标记——NPC 使用描述短语而非专名
    if observer.openness_to_outsiders < 0.2 || rng.gen_bool(0.5) {
        return GeographicName::unnamed();
    }
    
    // 5%：独立命名——观察者文化有自己完全不同的名字
    generate_geographic_name(entity, observer, observer_lexicon, rng)
}

impl GeographicName {
    /// 空名称——该文化不使用专名来指称此实体
    pub fn unnamed() -> Self {
        Self {
            name: String::new(), // 空字符串标记"无专名"
            literal_meaning: String::new(),
            name_type: GeographicNameType::Indigenous, // 不相关
            is_obsolete: false,
            first_used: GameTime::now(),
        }
    }
    
    pub fn is_unnamed(&self) -> bool { self.name.is_empty() }
}
```

### 5.1 音系扭曲函数

```rust
/// 将源文化的名称经由观察者文化的音系规则扭曲
/// 
/// 例：源文化叫 "Kaelimdor"，观察者文化没有 "ae" 元音和 "dor" 音节的习惯
///   → "Kalimtor" 或 "Kelimdur"
fn phonetic_adapt(
    source_name: &GeographicName,
    observer_lexicon: &CultureToponymyLexicon,
) -> String {
    let phonology = &observer_lexicon.phonology;
    let mut result = source_name.name.clone();
    
    // 1. 替换观察者音系中不存在的音素为最接近的存在音素
    // 2. 调整音节结构以符合观察者的音节模板
    // 3. 调整重音位置
    // （具体实现依赖文化系统的音系引擎——此处置于地名系统的音系规则表中）
    
    phonology.apply_to(&mut result);
    result
}
```

## 六、感知分组——不同文化对同一片区域的不同划分

### 6.0 问题描述

你描述的场景——一座山峰在一个文化中被独立称为白山，在另一个文化中被纳为彩色山脉的一部分——是**感知分组差异**：客观物理世界是一样的，但不同文化把世界切成不同大小和边界的命名单位。

### v1.0 方案：客观实体 + 层级引用

在 v1.0 中，每个 GeographicEntity 有**固定的客观边界**（世界生成时确定）。一个山峰就是一个山峰——文化 A 称它为白山，文化 B 可能没有它的独立专名，但可以用以下方式引用它：



每个 GeographicEntity 有一个可选的  字段——记录该实体属于哪个更大的地理实体。这允许世界生成管线在 P3.0 构建层级结构。



**文化感知差异的处理**：
- 世界生成确定一个标准的实体层级（山脉包含山峰，海域包含海湾……）
- 文化 A 如果 nameworthiness >= 0.7，则为子实体（如白山）生成独立名称
- 文化 B 如果 nameworthiness < 0.4 对子实体，则不给子实体专名——但其 parent（如彩色山脉）可能 nameworthiness 很高
- 文化 B 的 NPC 引用子实体时使用描述短语

### v1.1 愿景（不在此版本实现）

未来每个文化可以拥有独立的  模型——文化 B 可以将白山+黑崖+龙脊三座独立山峰感知为一个实体彩色山脉，而文化 A 将它们感知为三个独立实体。这是从客观层级向**主观层级**的进阶。

## 七、地名变更——当实体改变时

### 7.1 五类变更场景

| # | 场景 | 触发方式 | 名称行为 |
|---|------|---------|---------|
| A | 地貌物理改变 | 采矿、SDF 编辑、自然灾害 | 旧名保留为历史名。可能涌现新描述名。实体 status→Altered |
| B | 实体替换 | 森林→聚落、湖泊→农田 | 旧实体标记 Superseded。新实体继承旧名（大概率） |
| C | 实体消亡 | 湖干涸、岛沉没 | 实体标记 Defunct。名称全部进入历史层 |
| D | 政治变更 | 征服、殖民 | 征服者可能赋予新名。旧名进入历史层（ConquestRenamed） |
| E | 文化视野变化 | 首次"发现"、首次贸易接触 | 懒生成 exonym（见 §五） |

### 7.2 场景 A——地貌物理改变

```rust
/// 当地理实体的物理属性发生显著变化时调用
/// 
/// 触发器：世界生成运行时检测到 density field 在实体范围内有重大修改
///        或历史系统发出 TerrainAltered 事件
fn on_entity_terrain_significantly_altered(
    entity_id: GeographicEntityId,
    change_magnitude: f32,       // 0-1，物理变化的程度
    toponymy: &mut ToponymySystem,
    culture_query: &dyn CultureQuery,
    entity_store: &GeographicEntityStore,
) {
    let entity = entity_store.get_mut(entity_id);
    let names = toponymy.get_names_mut(entity_id);
    
    if change_magnitude < 0.3 { return; } // 小变化——不值得更名
    
    entity.status = EntityStatus::Altered {
        description: describe_change(entity, change_magnitude),
    };
    
    // 高 artistry 文化可能创造新描述名
    for &culture_id in names.current.keys().collect::<Vec<_>>() {
        let culture = culture_query.core_params(culture_id);
        if culture.artistry > 0.5 && change_magnitude > 0.5 {
            let new_name = generate_geographic_name(
                entity, culture, toponymy.get_lexicon(culture_id), 
                &mut toponymy.name_rng(entity_id, culture_id)
            );
            
            // 旧名 → 历史层
            if let Some(old_name) = names.current.get(&culture_id) {
                names.historical.push(HistoricalName {
                    name: old_name.clone(),
                    used_by: culture_id,
                    used_from: old_name.first_used,
                    used_until: Some(GameTime::now()),
                    reason_obsoleted: NameChangeReason::TerrainAltered,
                });
            }
            
            // 新名 → 当前（与旧名共存——年轻 NPC 用新名，老年 NPC 保留旧名）
            names.current.insert(culture_id, new_name);
        }
    }
}
```

### 7.3 场景 B——实体替换

```rust
/// 当一个地理实体被另一个实体取代时调用
/// 
/// 例：森林被清除，聚落在此建立
/// 
/// 触发器：世界生成运行时检测，或历史系统 SettlementFounded 事件
fn on_entity_superseded(
    old_entity_id: GeographicEntityId,
    new_entity_id: GeographicEntityId,
    toponymy: &mut ToponymySystem,
    entity_store: &mut GeographicEntityStore,
) {
    // 1. 旧实体标记为被取代
    let old_entity = entity_store.get_mut(old_entity_id);
    old_entity.status = EntityStatus::Superseded { by: new_entity_id };
    old_entity.succeeded_by = Some(new_entity_id);
    old_entity.status_changed_at = Some(GameTime::now());
    
    // 2. 新实体继承旧实体的命名链
    let new_entity = entity_store.get_mut(new_entity_id);
    new_entity.preceded_by = Some(old_entity_id);
    
    // 3. 旧实体的当前名 → 新实体的名称基础
    let old_names = toponymy.get_names(old_entity_id);
    let new_names = toponymy.get_or_create_names(new_entity_id);
    
    for (culture_id, old_name) in &old_names.current {
        if old_name.is_unnamed() { continue; }
        
        // 大概率：新实体继承旧名 → "暗林" → "暗林村"
        let derived_name = GeographicName {
            name: derive_settlement_name(&old_name.name, culture_id, toponymy),
            literal_meaning: format!("settlement of {}", old_name.literal_meaning),
            name_type: GeographicNameType::Derived {
                from_entity: old_entity_id,
                derivation: format!("settlement built on former {}", old_entity.entity_type.name()),
            },
            is_obsolete: false,
            first_used: GameTime::now(),
        };
        new_names.current.insert(*culture_id, derived_name);
    }
    
    // 4. 旧实体的所有当前名 → 历史层
    for (culture_id, name) in old_names.current.drain() {
        if !name.is_unnamed() {
            old_names.historical.push(HistoricalName {
                name,
                used_by: culture_id,
                used_from: name.first_used,
                used_until: Some(GameTime::now()),
                reason_obsoleted: NameChangeReason::EntitySuperseded,
            });
        }
    }
}
```

### 7.4 场景 C——实体消亡

```rust
fn on_entity_defunct(
    entity_id: GeographicEntityId,
    toponymy: &mut ToponymySystem,
    entity_store: &mut GeographicEntityStore,
) {
    let entity = entity_store.get_mut(entity_id);
    entity.status = EntityStatus::Defunct;
    entity.status_changed_at = Some(GameTime::now());
    
    let names = toponymy.get_names_mut(entity_id);
    
    // 所有当前名 → 历史层
    for (culture_id, name) in names.current.drain() {
        if !name.is_unnamed() {
            names.historical.push(HistoricalName {
                name,
                used_by: culture_id,
                used_from: name.first_used,
                used_until: Some(GameTime::now()),
                reason_obsoleted: NameChangeReason::EntityDefunct,
            });
        }
    }
    // 注意：current 被 drained 后为空——任何文化都不再有"当前名"
    // NPC 引用此实体时使用："曾经的黑水湖"——由语言表达模块的时态模板生成
}
```

### 7.5 场景 D——政治变更

```rust
fn on_conquest_rename(
    entity_id: GeographicEntityId,    // 被征服的政体或聚落
    conqueror_culture_id: CultureId,
    toponymy: &mut ToponymySystem,
    culture_query: &dyn CultureQuery,
    entity_store: &GeographicEntityStore,
) {
    let entity = entity_store.get(entity_id);
    let conqueror = culture_query.core_params(conqueror_culture_id);
    let names = toponymy.get_names_mut(entity_id);
    
    // 征服者的 militarism + power_distance 决定强制更名概率
    let rename_probability = conqueror.militarism * 0.5 + conqueror.power_distance * 0.5;
    
    if rename_probability < 0.4 { return; } // 低强制→保留旧名
    
    // 旧名 → 历史层（被征服的旧文化名）
    let conquered_culture = names.primary_culture;
    if let Some(old_name) = names.current.remove(&conquered_culture) {
        names.historical.push(HistoricalName {
            name: old_name,
            used_by: conquered_culture,
            used_from: names.historical.last().map(|h| h.used_from).unwrap_or(entity.created_at),
            used_until: Some(GameTime::now()),
            reason_obsoleted: NameChangeReason::ConquestRenamed,
        });
    }
    
    // 征服者赋予新名
    let new_name = generate_geographic_name(
        entity, conqueror, toponymy.get_lexicon(conqueror_culture_id),
        &mut toponymy.name_rng(entity_id, conqueror_culture_id)
    );
    names.current.insert(conqueror_culture_id, new_name);
}
```

### 7.6 地名历史在 NPC 对话中的涌现

不需要硬编码"老年 NPC 使用旧名"。NPC 记忆系统自然产生这种效果：

```
NPC 在实体变更前出生 + 在变更前形成关于该实体的 SpatialMemory
→ NPC 的 memory 中地名 = 旧名
→ NPC 对话中使用旧名

NPC 在实体变更后出生
→ NPC 只知道当前名
→ NPC 对话中使用新名

年长 NPC 同时有旧 memory 和新 knowledge
→ 对话中可能产生注释："暗林村——我们年轻时还叫暗林呢"
→ 这是 EventMemory + SpatialMemory 的自然涌现——无需地名系统特殊处理
```

### 9.3 政体命名（P5/P9）

政体名不由地名系统单独生成——它在 [[../权力系统/007-政治实体与外交涌现|权力系统 007]] 中通过文化名 + 地理 + 政体类型的组合模板生成。但地名系统提供政体名中的"地理"部分：

```rust
// 权力系统 007 中 generate_polity_name() 的实现框架：
fn generate_polity_name(
    primary_culture: CultureId,
    dominant_government: &GovernmentForm,
    core_region_entity: Option<GeographicEntityId>,  // ← 从地名系统获取
    toponymy: &dyn CultureQuery,
) -> String {
    let culture_name = toponymy.culture_name(primary_culture).endonym;
    let geo_name = core_region_entity
        .and_then(|eid| toponymy.geographic_name(eid, primary_culture))
        .map(|n| n.name.clone())
        .unwrap_or_default();
    
    // 模板："[地理] + [文化名] + [政体类型]"
    // 例："河湾地赫拉王国"、"北山矮人酋邦"、"碧海城邦联盟"
    format!("{}{}{}", geo_name, culture_name, government_form_name(dominant_government))
}
```
