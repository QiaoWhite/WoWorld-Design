## 一、节日的定义

在 WoWorld 中，一个**节日（Festival）**是一个满足以下全部条件的文化事件：

1. **时间固定的集体活动**——在特定日期或季节重复发生（不一定是每年——可以是每季、每十年）
2. **参与者超越单一家庭**——最少覆盖一个聚落的大部分居民
3. **具有文化或宗教意义**——不是纯功能性的聚集（如"赶集"不是节日）
4. **包含非常规行为**——节日期间的行为与日常 Routine 显著不同
5. **被该文化赋予专名**——"收获节"、"龙王祭"、"春分庆典"

**节日 ≠ 仪式**：仪式是节日的**子组件**——一个节日包含一个或多个仪式。仪式也可以在非节日语境中发生（婚礼、葬礼、加冕）。

**节日 ≠ 集市**：集市是经济驱动的定期聚集——功能性的。节日是文化驱动的——有意义的。但节日可以**包含**集市。

**节日 ≠ 竞技赛事**：竞技可以在非节日发生（常规的剑术比赛）。大型竞技**可以**成为节日的一部分（如奥林匹克原为宗教节日的一部分）。

## 二、四类节日

### 2.1 日历节日（Calendar Festival）

**来源**：天文/季节周期——非文化选择，是物理事实。

**产生条件**：文化参数 × 季节锚点 → 日历节日自动涌现。

| 锚点 | 季节 | 典型节日方向 | 涌现条件 |
|------|------|------------|---------|
| 春分 (day 1) | 春 | 新年、播种祭、春耕祝福 | `long_term > 0.4`（农业文化）→ 播种祭 |
| day 15 | 春 | 春季节庆、花祭 | `artistry > 0.5` + `indulgence > 0.4` |
| 夏至 (day 30) | 夏 | 仲夏节、太阳崇拜、户外竞技 | `indulgence > 0.5` → 狂欢型；`militarism > 0.5` → 竞技型 |
| day 45 | 夏 | 收获前祈福 | `religiosity > 0.4` |
| 秋分 (day 60) | 秋 | 收获节、感恩祭 | `long_term > 0.3` → 农业型收获节（几乎所有农耕文化都有） |
| day 75 | 秋 | 秋收庆功、储粮祭 | `competition > 0.5` → 比谁收成多 |
| 冬至 (day 90) | 冬 | 追思节、光明节、团圆节 | `uncertainty_avoidance > 0.5` → 追思型（"一年中最长的夜——缅怀逝者"）；`indulgence > 0.5` → 狂欢型（"用灯火驱散黑暗"） |
| day 105 | 冬 | 年终祭、新年准备 | 几乎所有文化都有某种形式的"年关"仪式 |

**关键设计决策**：不是所有锚点都产生节日——只有锚点的文化相关性超过阈值的才产生节日。一个航海文化可能忽略春分而庆祝"首次顺风日"（非天文锚点——见 §2.4 涌现型）。

**日历节日的命名**：
```
模板：
  "[季节/天文特征] + [活动/意义] + [节/祭/日/庆典]"
  
示例：
  春分 + 播种 + 祭 → "春播种祭"
  秋分 + 收获 + 节 → "秋收感恩节"
  冬至 + 光明 + 庆典 → "光明节庆典"
  
由文化音系规则处理最终名称（与 [[../../Happy Game/开发阶段/文化系统/007-地名系统|地名系统 007]] 的命名模板机制一致）
```

### 2.2 宗教节日（Religious Festival）

**来源**：信仰系统生成的神祇/教义/神圣日期。

**产生条件**：文化 `religiosity > 0.3`（低于此值 → 没有宗教节日）。信仰系统提供神圣日期锚点，文化系统决定日期如何被庆祝。

**信仰系统需要提供的数据**：
```rust
/// 信仰系统为节日系统提供的接口
trait FaithCalendarQuery {
    /// 查询该信仰的神圣日期列表
    /// 返回：(day_of_year, 神圣意义, 重要性 0-1)
    fn holy_days(&self, faith_id: FaithId) -> Vec<(u32, String, f32)>;
    
    /// 查询该信仰的主要神祇列表——用于节日命名
    fn primary_deities(&self, faith_id: FaithId) -> Vec<DeityInfo>;
}
```

**宗教节日的命名**：
```
模板：
  "[神祇名] + [活动] + [祭/日]"
  "[神圣事件] + [纪念]"
  
示例：
  "火神诞辰祭"、"创世纪念日"、"龙神升天祭"
  "七日斋戒"、"朝圣月"、"万灵节"
```

**一个文化有多种信仰时的处理**：
- 文化的 `religiosity` 决定"所有信仰的总节日密度"
- 每种信仰按信徒占比分配节日名额
- 主流信仰（>60% 人口）：2-4 个节日
- 少数信仰（10-40%）：1-2 个节日
- 微小信仰（<10%）：0-1 个节日

### 2.3 纪念性节日（Commemorative Festival）

**来源**：重大历史事件——全然涌现，不预设。

**产生条件**（全部满足）：
1. 历史事件的 `impact_score >= 0.7`（全局级影响）
2. 事件发生在至少 10 年前（即时创立的节日没有意义——时间沉淀是必要的）
3. 与该事件关联的文化仍存在
4. 连续 3 年在事件的周年日有至少 5% 的当地 NPC 自发进行纪念行为（→ 系统检测并正式化）

**纪念性节日的类型**：
- 胜利纪念日（重大战争胜利的周年）
- 建国/建城纪念日
- 伟人/英雄纪念日（传奇 NPC 逝世周年）
- 灾难纪念日（大地震、大瘟疫——"警示后人的日子"）
- 解放纪念日（被征服后重获独立的周年）

**命名模板**：
```
"[事件名] + 纪念[日/节]"
"[英雄名] + 日"
"[地名] + 之日"
```

**关键设计**：纪念性节日是唯一**不由种子生成**的节日类型——它们完全由运行时历史事件驱动。种子可以生成的 P9 历史模拟中也能产生足够影响力的"第一代"纪念性节日（"王国的第一场胜利纪念日"）。

### 2.4 涌现型节日（Emergent Festival）

**来源**：自然现象 + NPC 自发行为 → 被系统检测并正式化。

**这不是预设的节日**——系统不知道"鲑鱼洄游"是节日。系统只检测一种模式：

```
检测条件（全部满足且持续）：
1. 某地点在每年同一时段（±5天）出现 NPC 密度 ≥ 该地平均密度 × 3
2. 该聚集持续 ≥ 2天
3. NPC 在聚集期间的行为偏离日常 Routine（购物/饮酒/社交行为频率 ×2+）
4. 该模式连续重复 ≥ 3年

→ 系统自动创建 EmergentFestival 记录
→ 赋予名称（基于聚集原因 + 文化的词汇）
→ 加入该文化的节日日历
→ 历史系统记录："鲑鱼洄游的临时市集被正式命名为鲑鱼节"
```

**涌现型节日的物理来源**：
- 动物迁徙/聚集（鲑鱼洄游、候鸟过境、野兽繁殖季）
- 天文现象（流星雨、日食/月食、极光高峰、特殊星象）
- 植物周期（樱花盛开、萤火虫季、菌菇爆发）
- 天气现象（季风到达日、初雪日、解冻日）
- 魔法现象（魔力潮汐高峰、元素裂隙周期性活跃）

## 三、混合生成模型

### 3.1 种子生成部分（P2.5 文化生成期间）

世界生成时为每个文化生成**节日框架**（不是完整节日）：

```rust
/// 节日框架——种子生成，运行时填血肉
struct FestivalFramework {
    /// 节日在一年中的日期
    day_of_year: u16,          // 1-120
    /// 节日持续天数（默认 1）
    duration_days: u8,         // 1-7
    /// 节日类型
    festival_type: FestivalType,
    /// 文化内部的名称
    name: GeographicName,       // 使用地名系统相同的命名机制
    /// 情绪基调
    emotional_tone: EmotionalTone,
    /// 该节日包含的活动类型（种子选择）
    activities: Vec<FestivalActivityType>,
    /// 该节日绑定的宗教（如果是宗教节日）
    bound_faith: Option<FaithId>,
    /// 该节日是否强制性
    mandatory_attendance: bool,
}

enum FestivalType {
    Calendar { anchor: CalendarAnchor },    // 日历节日
    Religious { holy_day_id: HolyDayId },   // 宗教节日
    // 纪念性和涌现型不在种子生成——运行时创建
}

enum CalendarAnchor {
    SpringEquinox,       // day 1
    MidSpring,           // day 15
    SummerSolstice,      // day 30
    MidSummer,           // day 45
    AutumnEquinox,       // day 60
    MidAutumn,           // day 75
    WinterSolstice,      // day 90
    MidWinter,           // day 105
}

enum EmotionalTone {
    Solemn,        // 庄严肃穆——祭祀、追思
    Joyful,        // 欢乐庆祝——宴饮、舞蹈
    Competitive,   // 竞技——比赛、争夺
    Contemplative, // 沉思——祈福、冥想
    Ecstatic,      // 狂喜——狂欢、放纵
}
```

**每文化的节日数量**：

```rust
fn festival_count_for_culture(core: &CultureCoreParams, rng: &mut impl Rng) -> u32 {
    let base = 3.0; // 最少 2-4 个（随机波动）
    
    // 各参数对节日数量都有温和正向贡献——节日越多的文化越"热闹"
    let modifiers = 
        core.religiosity * 1.5       // 宗教性 → 更多宗教节日
        + core.artistry * 1.0        // 艺术性 → 更多文化节日
        + core.indulgence * 0.8      // 放纵性 → 更愿意过节
        + core.competition_orientation * 0.5 // 竞争性 → 更多竞技节日
        + (1.0 - core.uncertainty_avoidance) * 0.3; // 灵活性 → 更多新节日
    
    (base + modifiers).round().clamp(2, 7) as u32
}
```

**节日类型的文化驱动权重**：

```rust
fn festival_type_weights(core: &CultureCoreParams) -> Vec<(FestivalTypeTag, f32)> {
    let mut weights = Vec::new();
    
    // 日历节日权重——所有文化都有
    weights.push((CalendarTag, 1.0));
    
    // 宗教节日权重——由 religiosity 驱动
    if core.religiosity > 0.3 {
        weights.push((ReligiousTag, core.religiosity * 1.5));
    }
    
    // 竞技型节日——由 militarism + competition 驱动
    if core.militarism > 0.4 || core.competition_orientation > 0.5 {
        weights.push((CompetitiveTag, (core.militarism + core.competition_orientation) / 2.0));
    }
    
    // 艺术型节日——由 artistry 驱动
    if core.artistry > 0.5 {
        weights.push((ArtisticTag, core.artistry));
    }
    
    weights
}
```

### 3.2 运行时填血肉部分

种子生成的框架只有骨架——节日日期、名称、类型、情绪基调。以下由运行时决定：

| 运行时因素 | 影响 |
|-----------|------|
| 当年丰收/饥荒 | 收获节规模——丰年盛大 ×1.5，荒年冷清 ×0.3 或取消 |
| 当前战争状态 | 军事阅兵 ×2.0，狂欢型节日被压制 |
| 瘟疫流行 | 大型聚集节日被取消 |
| 统治者合法性 | 统治者主持节日的合法性获取得分 |
| 天气当天 | 雨/雪 → 户外活动取消 → 节日规模降低 |
| 近期历史事件 | 如果最近有重大胜利 → 本季节日额外盛大 ×1.3 |
| NPC 参与热情 | 个体 NPC 的情绪/人格/关系 → 实际参加率 |

## 四、FestivalId 与核心类型

```rust
/// 节日标识符——全局唯一
/// 
/// 种子生成的节日：u32 高 16bit = CultureId, 低 16bit = 节日序号
/// 运行时创建的节日（纪念/涌现）：最高位置 1 标记
#[derive(Copy, Clone, PartialEq, Eq, Hash, Serialize, Deserialize)]
pub struct FestivalId(pub u32);

impl FestivalId {
    pub fn is_seed_generated(&self) -> bool { self.0 >> 31 == 0 }
    pub fn is_runtime_created(&self) -> bool { self.0 >> 31 == 1 }
    pub fn source_culture(&self) -> Option<CultureId> {
        if self.is_seed_generated() { Some(CultureId((self.0 >> 16) as u32 & 0x7FFF)) }
        else { None } // 涌现型节日可能不属于单一文化
    }
}

/// 节日实例——一个节日在特定年份的一次实际发生
pub struct FestivalInstance {
    pub festival_id: FestivalId,
    /// 此次发生的年份
    pub year: u64,
    /// 当前所处的阶段
    pub phase: FestivalPhase,
    /// 实际规模（0-1，相对于该节日的"标准"规模）
    pub scale: f32,
    /// 实际参加人数（动态更新）
    pub participant_count: u32,
    /// 节日期间的临时经济数据
    pub economic_impact: Option<FestivalEconomicImpact>,
}
```
