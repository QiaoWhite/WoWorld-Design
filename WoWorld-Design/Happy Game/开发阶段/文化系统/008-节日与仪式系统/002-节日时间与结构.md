## 一、日期系统

### 1.1 日期锚定方式

节日日期使用 `day_of_year`（1-120）作为全局锚定。但**不同文化对同一日期的感知不同**——全局 120 天太阳历是底层物理事实，文化视角是上层解释。

```
客观层：GlobalCalendar { day_of_year: 60 }  → 秋分
文化A视角：这天是"收获感恩节"（秋分农业节日）
文化B视角：这天是"???"（该文化没有庆祝秋分——不是农业文化）
文化C视角：这天是"海神安息日"（该文化的航海季节结束日恰好与秋分同日）
```

### 1.2 天文锚点 vs 固定日期

**优先使用天文锚点**。原因：天文锚点对玩家是可感知的——"昼夜等长的那天"是一个物理上可见的事件。固定日期（"第 60 天"）只有查日历才知道。

```rust
enum DateAnchor {
    /// 天文锚点——基于季节循环的物理事实
    Celestial {
        event: CelestialEvent,   // Equinox/Solstice
        offset_days: i8,         // ±天数偏移
    },
    /// 固定日期——基于 day_of_year
    Fixed { day_of_year: u16 },
}

enum CelestialEvent {
    SpringEquinox,     // 春分——昼夜等长
    SummerSolstice,    // 夏至——最长日
    AutumnEquinox,     // 秋分——昼夜等长
    WinterSolstice,    // 冬至——最短日
}
```

**种子生成偏好**：
- 日历节日 → 优先天文锚点（70%概率绑定到 Equinox/Solstice ±3天）
- 宗教节日 → 优先固定日期（宗教的神圣日不需要天文解释——虽然经常与之巧合）
- 纪念性节日 → 固定日期（历史事件发生在特定日期）

### 1.3 节日在一年中的分布

**避免节日聚集**。如果所有节日都在春季——冬季没有任何节日——日历不自然。

```rust
/// 节日日期生成——确保合理的季节分布
fn assign_festival_dates(
    count: u32,
    festival_types: &[FestivalTypeTag],
    culture: &CultureCoreParams,
    rng: &mut impl Rng,
) -> Vec<u16> {
    let mut dates = Vec::new();
    
    // 按季节分区——每季至少尝试分配一个节日
    let per_season = count.max(4); // 至少考虑每季一个
    let season_quotas = distribute_quotas(count, per_season, rng);
    
    for (season, quota) in season_quotas {
        let range = season.day_range(); // 春季: 1-30, 夏季: 31-60, ...
        
        // 日历节日 → 优先绑定到季节的天文事件
        // 宗教节日 → 随机在该季范围内
        // 竞技节日 → 偏好温和季节（春/秋——户外舒适）
        
        for _ in 0..quota {
            let date = pick_date_in_season(season, &festival_types, rng);
            if !dates.iter().any(|&d| (d as i32 - date as i32).abs() < 5) {
                dates.push(date); // 与已有节日至少间隔 5 天
            }
        }
    }
    
    dates
}
```

### 1.4 节日持续时长

| 类型 | 典型时长 | 决定因素 |
|------|---------|---------|
| 日历节日 | 1-3 天 | `indulgence` 高 → 更长（"多庆祝几天"） |
| 宗教节日 | 1-7 天 | 信仰系统定义——七日斋戒、三日庆典 |
| 纪念性节日 | 1 天 | 通常仅周年日当天 |
| 涌现型节日 | 由自然现象持续时间决定 | 鲑鱼洄游持续 5 天 → 节日持续 5 天 |

## 二、节日的四季内容约束

季节不仅决定节日日期——也温和约束节日内容。这不是硬性规则（文化可以创新），而是权重偏移：

| 季节 | 自然特征 | 偏好的节日活动 | 文化参数修正 |
|------|---------|--------------|------------|
| **春** | 万物复苏、播种、新生 | 祈福、播种仪式、新生祭、婚礼集中 | `long_term` 高 → 更重视春季节日 |
| **夏** | 长日照、温暖、户外 | 竞技、户外宴饮、游行、水上活动 | `indulgence` 高 → 夏季节日更长更狂欢 |
| **秋** | 收获、储粮、准备过冬 | 收获祭、感恩、评比（最大南瓜）、集市 | 几乎所有农耕文化都重视秋季节日 |
| **冬** | 短日照、寒冷、室内 | 追思、团圆、灯火、室内宴饮、占卜新年 | `uncertainty_avoidance` 高 → 冬季节日更庄重（"一年的尽头——反思"） |

## 三、节日的四阶段生命周期

每个 Festival 在每年有一个 `FestivalInstance`——该年的具体发生。Instance 经历四个阶段：

```
宣告期 → 预备期 → 庆祝期 → 余波期
 (3-7天)  (1-3天)  (1-7天)   (1-2天)
```

### 3.1 宣告期（Announcement Phase）

**触发**：`current_day_of_year == festival.day_of_year - announcement_lead_days`

**期间发生的事**：
- `FestivalInstance` 被创建，phase = Announcement
- 信息传播系统发出"节日宣告"——通过多通道传播到受影响区域的 NPC
  - 官方公告（`InformationChannel::Official`）—— 聚落布告板、城镇公告员
  - 口头传播（`InformationChannel::Oral`）—— NPC 间对话中提及
  - 钟声/号角（环境音频）—— 玩家可直接听到
- NPC 的心理模型更新：`known_festivals` 中标记此节日为"即将到来"
- NPC 的概率行为树开始增加"节日准备"行为的权重

**宣告期的长度**：
- 大型节日（全聚落参与）：7 天宣告期
- 中型节日（社区级）：3-5 天
- 小型节日（家庭级或特定群体）：1-3 天

### 3.2 预备期（Preparation Phase）

**触发**：`current_day_of_year == festival.day_of_year - preparation_lead_days`（通常 1-3 天前）

**期间发生的事**：
- 经济系统：节日物品需求激增——食物、酒、礼物、装饰品的 bid 量和价格上升
- NPC 行为：购买节日物品、准备节日服装（`OutfitMode::Ceremonial`）、装饰住宅
- 聚落视觉：公共空间的节日装饰逐渐出现（灯笼/旗帜/花环——临时贴花）
- 外地 NPC 开始旅行：附近聚落的 NPC 如果决定参加 → 开始沿道路旅行

### 3.3 庆祝期（Celebration Phase）

**触发**：`current_day_of_year == festival.day_of_year`

**期间发生的事**——节日的核心：
- 日常 Routine 被覆盖：NPC 不去工作（除非是必要工种——医生、守卫轮值）
- 商店根据 `closed_on_holidays` 标记决定是否关闭
- 节日活动按 `activities` 列表依次或并行触发：
  - 开幕式（`TurnMode::Ritual`）—— 祭司/统治者的开场仪式
  - 竞技比赛（如果有）—— NPC 参与或观赛
  - 宴饮—— 酒馆消费激增、公共长桌宴
  - 舞蹈/音乐—— `SocialField` 情绪传染放大
  - 赠礼—— NPC 间互赠节日物品
  - 祭祀—— 宗教仪式（`TurnMode::Ritual`）
  - 夜市/灯会—— 持续至深夜（`day_length` 被灯光延长）
- 情绪放大：`SocialField` 的惯性降低（更容易被带动），极化加速（共同情绪快速形成）

**庆祝期的每一天**：
```
每天有不同的"主题"活动（种子预设或文化参数派生）：
  第1天：开幕式 + 竞技初赛
  第2天：宴饮日 + 赠礼
  第3天：竞技决赛 + 闭幕式 + 烟火/灯火
```

### 3.4 余波期（Aftermath Phase）

**触发**：庆祝期结束后 1-2 天

**期间发生的事**：
- 经济结算：商人的额外收入、NPC 的额外支出（挥霍了多少？）
- 装饰拆除：公共空间的节日装饰逐渐消失
- 外地 NPC 返程
- 情绪回落：`SocialField` 的高涨情绪在 1-2 天内回归基线
- 历史记录：如果节日规模显著（`scale > 0.7` 或发生特殊事件）→ 写入历史事件
- 文化偏移：节日完成 → 对文化参数的温和加固或偏移

### 4.2 仪式组件与魔法仪式的区别

| | 社会仪式（节日中的仪式） | 魔法仪式（魔法系统 010） |
|---|---|---|
| 目的 | 文化维护、权力展示、社会凝聚 | 产生魔法效果 |
| 主祭 | 统治者/祭司/长者 | 法师/祭司（视文化魔法-宗教关系） |
| 是否有魔力参与 | 可选——如果有则从魔法系统调用 | 必需——核心目的 |
| 失败后果 | 社会不满、合法性下降、尴尬 | 魔法反噬、元素泄漏、契约惩罚 |
| 由谁管辖 | 节日系统 | 魔法系统 |

**重叠处理**：如果节日中的仪式同时也是魔法仪式（如"祈求丰收的魔法阵"），则两个系统协作——节日系统管理社会层面（参与、时间、情绪），魔法系统管理魔力层面（法力注入、法阵激活、风险）。

**活动组合的文化派生**：

```rust
fn derive_activities(
    festival_type: &FestivalTypeTag,
    emotional_tone: &EmotionalTone,
    core: &CultureCoreParams,
    rng: &mut impl Rng,
) -> Vec<FestivalActivityType> {
    let mut pool = Vec::new();
    
    // 来自节日类型的基线活动
    match festival_type {
        CalendarTag => {
            pool.extend(&[Feast, Dance, Music]);
            if core.competition_orientation > 0.4 { pool.push(Tournament); }
            if core.artistry > 0.5 { pool.extend(&[Theatre, Decoration]); }
        }
        ReligiousTag => {
            pool.extend(&[Prayer, Procession]);
            if core.religiosity > 0.6 { pool.push(Sacrifice); }
            if core.indulgence > 0.4 { pool.push(Feast); } // 宗教节日也可以有宴饮
        }
        CompetitiveTag => {
            pool.extend(&[Tournament, Race, Contest]);
        }
        ArtisticTag => {
            pool.extend(&[Dance, Music, Theatre, Fireworks, Decoration]);
        }
    }
    
    // 来自情绪基调的调节
    match emotional_tone {
        Solemn => { pool.retain(|a| matches!(a, Prayer | Sacrifice | Memorial | RitualCleansing | Procession)); }
        Joyful => { pool.retain(|a| matches!(a, Feast | Dance | Music | GiftGiving | Fireworks | Decoration | Reunion)); }
        Competitive => { pool.retain(|a| matches!(a, Tournament | Race | Game | Contest)); }
        Ecstatic => { pool.extend(&[Dance, Feast, Fireworks]); /* 所有活动都更疯狂 */ }
        Contemplative => { pool.retain(|a| matches!(a, Prayer | RitualCleansing | Memorial | SpecialFood)); }
    }
    
    // 来自文化参数的修正
    if core.indulgence > 0.6 { pool.push(Feast); pool.push(GiftGiving); }
    if core.militarism > 0.5 { pool.push(Tournament); }
    if (1.0 - core.individualism) > 0.6 { pool.push(Reunion); } // 集体文化 → 团圆
    if core.competition_orientation > 0.5 { pool.push(Contest); }
    
    // 随机选择 2-5 个活动
    let count = rng.gen_range(2..=5).min(pool.len());
    pool.shuffle(rng);
    pool.truncate(count);
    pool
}
```

## 六、节日的情绪基调

```rust
impl EmotionalTone {
    pub fn derive(festival_type: &FestivalTypeTag, core: &CultureCoreParams, rng: &mut impl Rng) -> Self {
        match festival_type {
            CalendarTag => {
                // 日历节日的情绪由 indulgence 和 uncertainty_avoidance 决定
                if core.indulgence > 0.6 { return if rng.gen_bool(0.3) { Ecstatic } else { Joyful }; }
                if core.uncertainty_avoidance > 0.6 { return Solemn; }
                Joyful
            }
            ReligiousTag => {
                // 宗教节日的情绪由 religiosity 的具体性质决定
                if core.religiosity > 0.7 { Solemn }  // 高度宗教 → 庄严
                else if core.indulgence > 0.5 { Joyful } // 中度宗教+享受 → 欢乐型宗教节日
                else { Contemplative }
            }
            CompetitiveTag => Competitive,
            ArtisticTag => {
                if core.indulgence > 0.6 { Ecstatic } else { Joyful }
            }
        }
    }
}
```

## 七、节日的强制性与排他性

### 7.1 强制性

节日的 `mandatory_attendance` 字段决定 NPC 是否被强制参加：

- **true**（义务型节日）：NPC 缺席 → 触发 `Duty` 违约 → 社会制裁。适用于：国教节日、统治者加冕周年、重大祭祀
- **false**（自愿型节日）：NPC 按个人意愿决定。适用于：大多数节日

**谁有权利强制**：
- 统治者通过 `PowerDomain::Behavior` 的 `PrescribeRule` 原子创建"必须参加节日"的义务
- 该义务对统治者的所有 `subject` 生效
- `ImmunitySet` 可以豁免特定群体（如异教徒不被强制参加国教节日）

### 7.2 排他性

```rust
enum FestivalExclusivity {
    /// 完全开放——任何人（包括外来者、异教徒）都可以参加
    Open,
    /// 温和限制——外来者可以旁观但不能参与核心仪式
    Restricted { restricted_activities: Vec<FestivalActivityType> },
    /// 严格封闭——只有特定群体可以参加
    Closed { eligible_groups: Vec<EligibilityCriterion> },
}
```

排他性由 `openness_to_outsiders` 决定：

```
openness > 0.6 → Open（"客人来参加我们的节日是荣幸"）
openness 0.3-0.6 → Restricted（核心宗教仪式仅信徒可参与，宴饮开放）
openness < 0.3 → Closed（"外人不得参加我们的神圣节日"）
```

**但即使是 Closed 节日——玩家仍可能通过特殊途径参与**（伪装成本地人、获得特别邀请、完成相关任务）。
