## 一、模块定位

权力系统是 WoWorld 的 **控制关系物理引擎**——定义人与人、人与团体、团体与团体之间"谁对谁能做什么"的底层规则。正如经济系统是"价值交换的物理引擎"，权力系统是"控制关系的物理引擎"。

**核心洞察**：权力不是国家的专利——它是二人以上社会关系的普遍属性。一个族长限制族人自由，和一个国王限制国民自由，是**同一个权力结构在不同规模上的实例化**。

### 一句定调

> 经济系统回答"什么东西值多少钱"。  
> 权力系统回答"谁能让谁做什么，凭什么，以及被管的人认不认"。

### 为什么独立于经济系统

经济系统定义了"对市场的权力"（PowerAtom × MarketAuthority）。权力系统定义的是**普适权力**——对人、对领土、对暴力、对规则本身的控制能力。经济权力（设定税率、发放许可、铸币） = 普适权力在经济域的特化。

## 二、核心设计哲学

1. **权力第一，政治实体第二**。`Polity`（王国/共和国/部落联盟）不是被"创建"的对象——它是权力拓扑图中稳定连通分量的**涌现标签**。权力关系是过程，政治实体是结果。

2. **17 个普适原子覆盖全部尺度**。从亲子关系到国际条约——同一套原子，同一套代码路径。家庭 = 轻型权力群体，王国 = 重型权力实体，结构相同。

3. **玩家 AI = NPC AI**。玩家行使权力和 NPC 行使权力走**同一套 `exercise_power()` 代码路径**。玩家获得权力的 8 条路径 = NPC 获得权力的 8 条路径。

4. **权力 = 稀疏图，非全局状态**。10 万 NPC 中，99% 的人不持有对他人的直接权力。只有少数掌权者持有出边。大部分 NPC 仅有入边（是被统治者）。

5. **合法性是主观的、动态的、可崩溃的**。每个 subject 对每个 holder 的合法性认可度独立计算，随时间变化，受治理结果、文化契合、信息失真、外部冲击影响。合法性归零 → 权力事实崩溃 → 革命或分裂。

6. **制裁不是自动的**。Duty 违约 → 找有权制裁的人 → 找有意愿制裁的人 → 执行或无人执行。无人制裁 → legitimacy 下降 → 积累到临界 → 权力结构重组。

## 四、关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 第一性概念 | **Power（权力关系）** | Polity 是涌现标签——见 007。权力先于国家存在 |
| 权力原子数 | **17 个** | 16 个单边 + 1 个双边（Contract）。Pledge 为独立原子（自指） |
| Contract 模型 | **双边原子——创建一对对称受限边** | 不降格为复合事件。`ContractRecord` 关联两边，支持联动失效 |
| Immunity | **subject 属性（ImmunitySet），不是原子** | 比增加原子更干净。覆盖外交豁免、议会免责、贵族特权 |
| Polity 锁定 | **不锁定。惰性快照 + 弱惯性反馈（≤0.15 legitimacy 加成）** | 防止循环因果。权力边变化 → Polity 可能在下次重算时变化/消失 |
| 制裁执行 | **塌缩链——不自动。有权者→有意愿者→执行或 legitimacy 下降** | "无人能制裁国王"和"人民起义"是同一套机制的两种结果 |
| 经济桥接 | **调度层 `PowerToEconomicBridge`** | 两模块互不导入。桥接层单独知道映射关系 |
| 存储布局 | **分表 SoA**（每种原子专属参数列） | 避免 tagged union 的内存浪费和类型安全缺失 |
| 事件发射 | **exercise_power 返回事件列表，调度层分发** | 保持权力系统零依赖其他模块 |
| 性能目标 | **<0.1ms/帧 CPU，~250MB 内存**（优化后可降至 ~180MB） | 低频社会事件——非性能瓶颈 |

## 五、核心数据结构速览

### UniversalPowerAtom（17 个）

| 类别 | 原子 | 含义 |
|------|------|------|
| **结构** | `Constitute` `DefineMembership` `Delegate` `Relinquish` `Contract` | 操作权力拓扑图的节点和边 |
| **自指** | `Pledge` | 自我约束（holder=subject） |
| **关系** | `Constrain` `Compel` `Extract` `Access` `ConferRank` `Represent` | 单向对 subject 行使 |
| **规范** | `PrescribeRule` `Derogate` | 创建或豁免通用规则 |
| **裁决** | `Adjudicate` `Sanction` `Remit` | 处理争议与结果 |

### PowerTopology

有向多重图——节点 = Entity（NPC/Player/Group），边 = PowerEdge。分表 SoA 存储（17 种原子各占专属参数列），4 重索引（出边/入边/空间/原子类型）。

### PowerEdge

```rust
holder → subject: atom(domain)
  × legitimacy: 0-1
  × enforcement: 0-1
  × source: 8 条获取路径之一
  × established_at / valid_until / last_exercised_at
  × succession_rule
```

见各自文档。
