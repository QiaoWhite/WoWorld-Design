## VegetationProvider trait ★新增 CHG-046

- **Owner**: woworld_core 定义, woworld_vegetation 实现 | CHG-046 权威
- **方法** (7):
  - `query_community(pos, radius) → PlantCommunitySnapshot` — 植被群落快照
  - `canopy_closure(pos) → f32` — 冠层郁闭度 0-1
  - `timber_availability(pos) → TimberAvailability` — 木材可获得性
  - `ground_cover(pos) → GroundCoverMap` — 地表覆盖四通道
  - `query_harvestable(pos, radius) → Vec<HarvestableInfo>` — 可采集物
  - `fuel_load(pos) → f32` — 可燃物密度
  - `root_interference(pos) → f32` — 根系干扰
- **消费者** (三级): ✅已验证: [[../04-天气与季节/002-接口入口|天气]](冠层截留) · [[../01-生命/002-接口入口|生命(动物)]](初级生产力) | 🔗设计合理: [[../23-建筑模块/002-接口入口|建筑]](木材) · [[../14-经济/002-接口入口|经济]](木材供给) · [[../09-文化/002-接口入口|文化]](LandcoverType) · [[../16-音频/002-接口入口|音频]](脚步声) · [[../02-NPC活人感/002-接口入口|NPC]](采集) | ⚠️接口预留: 战斗(掩体)/NPC(森林恐惧)/历史(读树)
- **模式**: Pattern D — trait inversion。消费方依赖trait，不依赖实现crate。
- **详见**: [[../../../生命/010-植被群落与覆盖|生命 010]]

## LodPrescription ★新增 CHG-049

- **Owner**: 技术栈方案 v4.0 §二十一 — woworld_core | CHG-049 权威
- **字段** (7维, ~8B):
  - `scene_lod: u8` (0-7) — 地形/建筑/海洋/云/植被统一
  - `skeleton_lod: u8` (0-4) — 35→33→28→15→0骨
  - `animation_lod: u8` (0-4) — 9层全栈→6核心→4行为→2层→无
  - `render_lod: u8` (0-4) — 1500面→800→300→impostor→无
  - `physics_lod: u8` (0-4) — 全碰撞+IK→简化胶囊→AABB→AABB仅→无
  - `audio_lod: u8` (0-4) — 全传播→简化衰减→事件声→检测无内容→静默 ★CHG-049: 4→5档
  - `ai_lod: u8` (0-4) — 全GOAP→轻量决策→统计模板→统计聚合→仅存在
- **消费者**: 全部模块 — 各模块按 LodPrescription 分配精度/频率
- **跨维约束**: 7 条硬约束见 CLAUDE-INTERFACES.md CHG-049 节
- **详见**: [[../../../../Change/CHG-049-LOD架构全面深化-20260620|CHG-049 §三]]

## LODCoordinator ★新增 CHG-049

- **Owner**: 技术栈方案 v4.0 §二十一 — woworld_core trait | CHG-049 权威
- **签名**: `fn compute_lod(input: &LodCoordinatorInput, prev: &HashMap<EntityId, LodPrescription>) -> HashMap<EntityId, LodPrescription>`
- **输入** (7项): CameraState + PlayerAttention + FrameBudget + VramPressure + Vec\<EntityLodInput\> + Vec\<AudioBroadcast\> + Vec\<InteractionIntent\>
- **算法**: 8步冲突解决 (基础分配→硬约束→级联拉升→Attention加成→VRAM降级→帧预算降级→跨维Clamp→迟滞)
- **性能**: <0.05ms/帧
- **消费者**: 全部模块 — 每帧调用一次，输出驱动所有模块的 LOD 行为
- **详见**: [[../../../../Change/CHG-049-LOD架构全面深化-20260620|CHG-049 §五]]
