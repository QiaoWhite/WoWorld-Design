# 音频系统 — 变更影响链

> **最后验证**: 2026-06-18 | **源文档版本**: 音频系统 v1.0 (CHG-030)
>
> 当音频系统发生变更时, 以下模块和概念会受影响.

## AudioQuery trait 签名变更
→ **NPC感知**: perceived_sounds()/perceived_speech() — 每决策周期热路径
→ **战斗情报**: 声音情报(位置/类型/距离) — 每帧可能查询
→ **感官系统**: 听觉通道 PerceptBatch 组装
→ **语言表达**: TTS播放 VoiceManager 优先级查询
→ **Godot渲染**: AudioRenderPacket 格式

## VoiceProfile 变更 ⚠️ 已从语言表达迁入
→ **NPC**: SensoryState 中 VoiceProfile 存储字段
→ **语言表达**: TTS消费 VoiceProfile 派生语音
→ **社交系统**: NPC间语音识别
→ **记忆系统**: 声音记忆(谁的声音)
→ **⚠️ CHG-030 迁移的残留**: 语言表达012 旧 VoiceProfile 定义需删除

## VoiceEmotionModulation 变更
→ **语言表达**: TTS渲染的情绪调制参数
→ **NPC情绪引擎**: EmotionState→调制映射

## VoiceManager / VoicePriority 变更
→ **语言表达**: TTS播放队列行为
→ **Godot**: 多NPC同时说话时的音频混合

## SoundFootprint 结构变更
→ **战斗**: 战斗动作写入格式
→ **NPC**: ActionRingBuffer 条目格式
→ **物品**: 物品交互声音数据
→ **环境**: 脚步声/环境音数据

## 传播引擎参数变更
→ **天气**: 风矢量调制/温度梯度影响
→ **世界生成**: 地形遮挡/介质边界
→ **所有听者**: perceived_sounds() 结果

## 话语清晰度5档变更
→ **NPC记忆**: 话语清晰度→文本模糊化映射
→ **语言表达**: resolve_partial() 返回质量
→ **战斗**: 战斗中的话语清晰度判定

## AudioMaterial / SurfaceMaterial 枚举变更
→ **物品**: AudioMaterial 存储字段 (15变体)
→ **世界生成**: SurfaceMaterial 赋值 (21变体)
→ **所有声音映射**: 材质→声音映射表

## HearingModel 变更
→ **Life**: 初始化派生公式
→ **NPC**: 存储于 SensoryState
→ **感官系统**: 听觉查询阈值

## 音乐系统变更
→ **文化**: FestivalMusicPreset 预设
→ **技能**: FineArts → NPC演奏 (Layer2)

## 提示音系统变更
→ **全部模块**: PlayerNotificationEvent (52变体) — 各模块发射事件. 音频订阅 world_core 事件总线

## 跨L层声音传播变更
→ **所有模块**: L1/L2/L3 分桶策略 → 性能与准确度平衡

## 跨CHG影响
- **CHG-030**: VoiceProfile/VoiceEmotionModulation/VoiceManager 从语言表达012迁入 — 语言表达需删除旧定义
- **CHG-033**: 空间查询改为四trait. 音频遮挡/传播依赖 VisibilityQuery+TerrainQuery+EntityIndex
