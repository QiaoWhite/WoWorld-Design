## v1.0 完整导出

### PlacementStore
- **概念**: 世界中所有摆放物品的空间存储。分块 HashMap + per-chunk RwLock。实现 EntityIndex trait。
- **Owner**: [[../../物品系统/家具与放置物品/014-放置物品数据模型|物品系统/家具与放置物品 014]]
- **消费者**: 世界生成胶水代码 · NPC 行动系统 · History · Economy · Audio
- **关键方法**: `place()` / `pick_up()` / `relocate()` / `batch_place()` / `surfaces_in_radius()` / `weather_exposure()`

### validate_furniture_placement() — 纯函数放置验证
- **概念**: 七步放置验证管线（方向匹配 → 坡度 → 承载 → 碰撞 → 净空 → 通行 → Snap）。纯函数，零隐藏状态。
- **Owner**: [[../../物品系统/家具与放置物品/013-放置规则引擎|物品系统/家具与放置物品 013]]
- **消费者**: 世界生成 · NPC 行动 · 玩家编辑器 · PlacementStore

### ItemPlacementProps
- **概念**: 附加在 ItemProperties 上的可选放置属性。None = 不可摆放。定义足迹/质量/方向/表面供给/AffordanceSet 等。
- **Owner**: [[../../物品系统/家具与放置物品/012-家具族参数化定义|物品系统/家具与放置物品 012]]
- **消费者**: PlacementStore · 世界生成 Pass B · 制造系统 · NPC GOAP

### PlacedEntry / PlacedCold (32B+40B)
- **概念**: 单个摆放物品的紧凑数据。热冷分离——热数据 Copy type，L1 友好；冷数据仅低频历史查询时碰。
- **Owner**: [[../../物品系统/家具与放置物品/014-放置物品数据模型|物品系统/家具与放置物品 014]]
- **消费者**: PlacementStore · EntityIndex impl

### AffordanceSet (u64 bitflags)
- **概念**: 64-bit 物理供给标记——CAN_SIT/CAN_GRASP/IS_WORKSTATION/IS_LIGHT_SOURCE 等。NPC GOAP 用位运算过滤（< 5ns）。
- **Owner**: woworld_core — 物品系统填充值
- **消费者**: NPC 行动系统 (GOAP 规划器) · 物理原子层

### FurnishParams + 9 FurnitureFamily
- **概念**: 家具参数化族的完整参数空间。9 族 × ~27 mesh 模板 × 连续参数 × morph target = ~12,000+ 视觉变体。
- **Owner**: [[../../物品系统/家具与放置物品/012-家具族参数化定义|物品系统/家具与放置物品 012]]
- **消费者**: 制造系统 · 世界生成 Pass B · CultureFurnishProfile

### CultureFurnishProfile
- **概念**: 从 CultureCoreParams 派生的家具文化参数。14 个字段，各有显式派生公式。
- **Owner**: [[../../文化系统/004-文化审美与物质|文化 004（定义）]] — [[../../物品系统/家具与放置物品/015-家具文化派生|015（完整公式）]]
- **消费者**: 世界生成 Pass B · FurnishEnsemble

### WorkstationRegistry (u16 注册制)
- **概念**: 工作站类型的 u16 标签注册表。物品系统只存 u16——领域模块注册并解释。烹饪/锻造/纺织/炼金/文书各注册自己的 tag。
- **Owner**: 启动胶水代码 — 各领域模块注册
- **消费者**: NPC 行动系统 · 物理原子层
