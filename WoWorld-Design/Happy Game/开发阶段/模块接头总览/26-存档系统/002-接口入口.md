## 消费其他模块的接口

存档系统**不消费任何模块的数据接口**。存档系统只知道 `SaveableModule` trait——不知道任何模块的具体类型。

存档系统通过模板匹配分裂借用获取各模块的 `&mut dyn SaveableModule` 引用：

```rust
let World { terrain, npc, items, ... } = world;
let modules: Vec<&mut dyn SaveableModule> = vec![terrain, npc, items, ...];
save_system.save(world_dir, &mut modules, header)?;
```

存档系统不持有任何模块的所有权。模块归 World 结构体所有。

### ★ CHG-057: NPC 认知模块新增持久化类型

NPC 认知系统 (CHG-057) 新增以下需序列化类型，由认知模块的 `SaveableModule` impl 自行处理：

- **四层记忆压缩序列化** (L0 Hot → L1 Cold → L2 Era → L3 Life)：跨层级压缩状态需持久化——包括各层容量/年龄边界/压缩触发阈值
- **PatternExpression + PatternStep**：NPC 认知模式库，~120B/条，需完整序列化
- **DecisionPoint**：决策快照（含 deliberation_depth / counterfactual_regret / source_confidence），用于跨会话决策审计
- **MentalModelRecord**：心智模型记录（含 DomainSignature u64），存档系统不感知内部结构

存档系统本身无需修改——以上类型均由 NPC 认知模块在其 `snapshot_dirty()` / `snapshot_full()` 中自行序列化到 named_db `"npc_cognition"`。

## 依赖的外部系统

| 系统 | 用途 |
|------|------|
| LMDB crate | 存储引擎 |
| bincode / rkyv (模块自选) | 序列化（模块内部使用，存档系统不感知） |
| uuid crate | UUID v4 生成 |
| chrono crate | 时间戳管理 |
| serde_json | world_meta.json 读写 |
| walkdir | 目录扫描和 .tmp 清理 |
| std::fs | 文件原子重命名、session.lock 管理 |
