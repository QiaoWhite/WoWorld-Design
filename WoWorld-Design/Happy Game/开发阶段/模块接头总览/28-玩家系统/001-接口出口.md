## 玩家系统导出的接口概念

### ControlMode

```rust
enum ControlMode {
    Auto,
    Manual { action_override: Action },
    DomainDelegated { manual_domains: HashSet<ActionDomain> },
}

enum ActionDomain {
    Movement, Combat, Speech, ItemUse, Interaction, MagicUse,
}
```

**消费方**: NPC生命周期 008、输入系统

### PlayerGoal（长期计划槽）

```rust
struct PlayerGoal {
    id: GoalId,
    description: String,
    priority: f32,              // 0-1
    state: GoalState,           // Draft/Active/Paused/Completed/Abandoned
    sub_goals: Vec<GoapGoal>,   // GOAP 自动展开
}
```

**消费方**: NPC活人感 GOAP 规划器

**消费方**: NPC生命周期 007、存档系统

### 输入动作清单

~40 动作（移动/战斗/交互/对话/角色与视角/系统），手柄+键鼠平等，可全改键。

**消费方**: UI/UX 系统

### 相机视点与朝向（007，CHG-069）

- **`woworld_core::lod::CameraState`**（复用，非新造）：`{ position: DVec3, forward: DVec3, fov_radians: f32 }`。相机为**生产者**，碰撞夹紧后的真实相机位/朝向经 `WorldDriver.publish_camera_state()` 供 LOD/剔除/音频消费。
- **`camera_transform`（yaw basis）**：CameraRig 世界 transform → `InputState.camera_transform` → `CMoveIntent.camera_transform`（移动相对相机 + RotationLock::CameraForward）。
- **相机 yaw**：供 UI/UX 小地图"镜头朝上" + 玩家箭头指向。
- **相机 forward**：为 `PlayerAttention.聚焦方向` 提供数据源（不拥有 PlayerAttention）。

**消费方**: LOD 系统（LodCoordinator）、角色控制器 004（player_input_system）、UI/UX 002（小地图）

### EntityVisual.controlled（007，CHG-069）

`EntityVisual` 新增 `controlled: bool`——被控角色抑制头顶名字/气泡、近距/FP 隐藏胶囊。`entity_visual_system` 从"排除玩家"改为"打标玩家"。

**消费方**: EntityRenderer（模型动作与物理 007）
