## 关键架构决策

1. **物理**: 放弃 Godot PhysicsServer3D 作为模拟侧物理依赖。仅玩家 CharacterBody3D 保留。其余全部 Rust 侧空间查询。
2. **面部**: 表情从骨骼驱动切换到 512² 共享纹理图集。6 字节/NPC。
3. **骨骼**: 人形 33 骨(L1) / 35 骨(L0)。双骨蒙皮。
4. **动画**: 模块化骨架 + 9 层可叠加动画栈 + 连续参数涌现（步态风格×人格，战斗流派×参数）。
5. **管线**: Blender→GLB→Godot ArrayMesh。Rust CPU→骨骼矩阵→GDExtension→GPU skinning。
6. **死亡**: 不做布娃娃——做骨架松弛物理。

## 性能 ✓

1000 NPC 同屏 60fps。CPU 动画 ≤0.5ms，GPU ≤0.7ms。GTX 1660 SUPER 6GB 可行。

## 9层动画栈速览

```
Layer 0: 基础姿态      — 站立/坐下/躺卧等静态姿态
Layer 1: 情绪偏移      — 13参数: 7身体(低头/蜷缩/挺胸/握拳…) + 6面部
Layer 2: 过渡混合      — 姿态间平滑过渡
Layer 3: 根运动        — 位移/旋转驱动
Layer 4: 注视          — 头部+眼球追踪目标
Layer 5: 脚部IK        — 地形适应
Layer 6: 武器手IK      — 武器持握/瞄准
Layer 7: 脊柱桥接      — 上下半身协调
Layer 8: 中断恢复      — 受击/闪避的打断与恢复
```

## 38模块姿态 + 15基元轨迹

- **38 模块姿态**: 涵盖战斗(8)/生活(10)/社交(6)/情绪(8)/特殊(6)
- **15 基元轨迹**: 步态/跑姿/转向/攀爬/游泳/飞行/潜行/闪避/格挡/装填/采集/建造/施法/演奏/骑乘
- **涌现式步态**: 9参数从 BigFive 派生——神经质高→步态紧张, 外向高→步态轻快
