## 〇、三类动作对比

| | 离散动作 | 持续动作 | 充能动作 |
|---|---|---|---|
| **触发** | 按键瞬间 | 按住 | 按住（充能） |
| **结束** | 计时器到期 | 松键 | 松键（触发子动作）或充能到顶自动释放 |
| **active_ms** | > 0 | 0（无限） | 0（无限，等释放） |
| **Recovery** | 有 | 有（松键后短暂收尾） | 无（由子动作接管） |
| **取消窗口** | Recovery 末尾 | Active 中始终开放 | Active 中始终开放 |
| **例子** | light_attack, dodge, jump | block, hold_breath, meditate | aim_bow, charge_heavy, channel_spell |

## 一、ReleaseBehavior

```rust
pub enum ReleaseBehavior {
    /// 纯结束——释放即完成
    Complete,
    /// 释放时触发固定子动作
    Trigger { action_id: ActionId },
    /// 根据充能时长选择子动作
    Charged {
        stages: &'static [ChargeStage],
        on_overcharge: OverchargeBehavior,
    },
}
```

### ChargeStage

```rust
pub struct ChargeStage {
    pub threshold_ms: u32,       // 需要充能多少毫秒
    pub action_id: ActionId,     // 触发的子动作
    pub power_multiplier: f32,   // 伤害/效果乘数
}
```

### OverchargeBehavior

```rust
pub enum OverchargeBehavior {
    AutoRelease,                        // 充能到顶自动释放
    Penalize { accuracy_loss_per_sec: f32 },  // 允许继续充能但精度惩罚
    ForceCancel,                        // 超时动作失败
}
```

## 二、SustainPhase + SustainDrain

```rust
pub enum SustainPhase {
    Normal,                          // 正常——资源按基础速率消耗
    Overextended { penalty: f32 },   // 过久——消耗加速
    Critical { forced_release_in: f32 },  // 即将强制结束
}

pub struct SustainDrain {
    pub resource: ResourceType,
    pub rate_per_sec: f32,
    pub overextend_multiplier: f32,  // Overextended 时的消耗乘数
}
```

> **注（三段语义·Sprint-065 审计）**：两阈值 `overextend_threshold_secs` / `critical_threshold_secs` 划分三段——`Normal`（正常速率）→ `Overextended`（消耗 ×`overextend_multiplier`，即**渐进警告区**）→ `Critical`（硬停，立即强制释放）。**`Critical{forced_release_in}` 当前恒 `0.0`**：数据仅一个 critical 阈值，无"进入 Critical→倒计时→强制"的第三段窗口，到阈值即强制。f32 字段作**预留**——若未来 playtest 需要"读秒警告"手感，可加 `critical_grace_secs` 数据填充倒计时（延后）。渐进紧张感由 Overextended 区间承担。

## 三、Block——完整防御示例

```toml
[action.block]
name = "防御"
category = "Combat"
kind = "Continuous"
priority = 20
commitment = "Soft"
windup_ms = 50
active_ms = 0                          # 无限
recovery_ms = 100
cancel_set = ["dodge", "parry", "light_attack", "heavy_attack"]
cancel_window_ms = 999
physics_req = "Grounded"
movement_lock = "Partial"
movement_lock_speed = 1.0
release_behavior = "Complete"

sustain_drain = { resource = "Stamina", rate_per_sec = 3.0, overextend_multiplier = 2.0 }
overextend_threshold_secs = 8.0
critical_threshold_secs = 12.0

[action.block.combat_params]
stability = 50.0                       # 格挡稳定性——被击中消耗
stability_regen_per_sec = 10.0         # 不被打时回复
damage_reduction = 0.85
```

**双资源**——Stamina 消耗（控制能防多久），Stability 消耗（控制每次被击中的承伤上限）。被重击 stability→0 → 破防 → Stagger。

> **注（域归属·Sprint-065 审计）**：`[action.block.combat_params]`（stability / stability_regen_per_sec / damage_reduction）是**战斗域数据，由战斗系统消费，不入 `ActionDef`**。角色控制器只负责阶段机 + `sustain_drain`（Stamina）。ActionDef 无 combat_params 字段属**有意的域分离**，非遗漏。破防/承伤模型待战斗冲刺实现。

## 四、AimBow——完整充能示例

```toml
[action.aim_bow]
name = "瞄准拉弓"
category = "Combat"
kind = "Charge"
priority = 12
commitment = "Hard"
windup_ms = 200
active_ms = 0
recovery_ms = 0
cancel_set = ["dodge"]
cancel_window_ms = 999
physics_req = "NotInWater"             # 地面和空中都能拉弓
movement_lock = "Partial"
movement_lock_speed = 0.5              # ⚠️ 尚未接线，见下方 Q3 注
sustain_drain = { resource = "Stamina", rate_per_sec = 5.0, overextend_multiplier = 1.5 }
overextend_threshold_secs = 2.0
critical_threshold_secs = 4.0

# ReleaseBehavior/OverchargeBehavior 走 serde 默认外部标签（变体名作 key）
[action.aim_bow.release_behavior.Charged]
stages = [
    { threshold_ms = 0,   action_id = "quick_shot", power_multiplier = 0.5 },
    { threshold_ms = 400, action_id = "aimed_shot", power_multiplier = 1.0 },
    { threshold_ms = 800, action_id = "full_draw",  power_multiplier = 1.3 },
]
on_overcharge = { Penalize = { accuracy_loss_per_sec = 0.15 } }
```

> **注（序列化表示·Sprint-065）**：`release_behavior` 用 **serde 默认外部标签**（`[action.aim_bow.release_behavior.Charged]`，变体名作 TOML key），非早期示例的内部标签 `{ kind = "Charged" }`——数据表示按实现，设计语义不变。`action_id`（`quick_shot` 等）是**可读字符串键**，由 `ActionId::from_key`（FNV-1a）哈希为 `u32 ActionId`；registry 委托同一 `from_key` 保证子动作引用与 map key 同源。

**涌现记忆点**：拉弓超过 2s→手抖（准星扩大），超过 4s→几乎瞄不准。玩家自己能感受到"极限时刻"。

> **注（实现状态·Sprint-065 审计）**：`movement_lock_speed=0.5`（半速慢走）**尚未接线**——当前 `MovementLockDef` 为裸枚举，所有 `Partial` 在运行时映射为 `MovementLock::Partial{speed_cap:1.0}`（硬编码）。运行时类型本身支持任意 speed_cap，缺的仅是 TOML→运行时映射。**目标设计**：给 `ActionDef` 加 `movement_lock_speed: Option<f32>`（serde default 1.0），在 3 个 Partial 映射点读配置值替换硬编码。延后至 aim_bow 绑键冲刺（届时可实机验证半速手感）。aim_bow 现无键位、不可触发，零现时影响。

## 五、充能子动作的帧间传递

```rust
// ActionController 释放充能动作时:
self.finish_current(FinishReason::Released);
self.pending_follow_up = self.charge_follow_up(registry);  // 充能→选子动作

// 下一帧:
ActionRequestBuf 注入 pending_follow_up → ActionController 接受子动作
```

一帧间隙——由 Recovery 帧或 id 帧覆盖，玩家无感。

## 六、NPC 持续动作——GOAP 控制释放

```
GOAP: "我需要防御到敌人死亡"
→ 写入 ActionRequest { Block } + 维护计时器/条件检测
→ 条件满足 → 写入 ActionRequest { Block, priority=RELEASE }
→ ActionController 收到 release → Completed(Block)

与玩家释放走完全相同路径。没有玩家/NPC 分支。
```
