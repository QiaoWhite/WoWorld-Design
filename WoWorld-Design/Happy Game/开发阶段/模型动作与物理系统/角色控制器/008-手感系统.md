## 二、CInputBuffer——环形缓冲区

```rust
pub struct CInputBuffer {
    entries: SmallVec<[BufferedInput; 4]>,  // 固定容量 4
    pub prev_frame_inputs: u64,              // 位集——检测"刚按下"vs"持续按住"
}

pub struct BufferedInput {
    pub action_request: ActionRequest,
    pub pressed_at: f32, pub expires_at: f32,
    pub buffer_priority: BufferPriority,
}

pub enum BufferPriority {
    Emergency = 100,   // 系统紧急
    Instinct = 80,     // 本能反应
    Defensive = 60,    // 闪避/招架
    Movement = 40,     // 跳跃
    Combat = 30,       // 攻击/技能
    Interaction = 10,  // 交互
}
```

**满容量时**——淘汰最低优先级的旧条目。新输入优先级更低→丢弃。

**消费时**——`pop_if(|req| is_physically_possible(req))` 重新检查物理合法性。缓冲了一个 Grounded 动作但消费时在 PhysicsBody 中→不弹出→过期后自然丢弃。

## 三、缓冲消费优先级链

ActionController 空闲时，从缓冲选择下一个动作的顺序：

```
1. 同一 cancel_set 内的最高优先级动作（连招）
2. 防御性动作（闪避/招架）
3. 移动动作（跳跃）
4. 战斗动作（攻击/技能）
5. 交互动作
```

## 四、土狼时间

```rust
pub struct CCoyoteTime {
    pub remaining: f32,        // > 0 = 仍在窗口内
    pub left_ground_at: Vec3,  // 离地位置
}
```

- **触发**：MovementSystem 检测到 `was_grounded → not_grounded` 且不是主动跳跃
- **消费**：ActionResolver 中——`coyote.remaining > 0` 允许跳跃
- **过期**：MovementSystem 每帧减 dt，归零后不可用
- **着地**：归零

---

## 五、落地预输入（Jump Buffer）

空中按跳跃（已过土狼时间）→ 缓冲 `Jump`（优先级 Movement，过期 0.2s）。着地瞬间——MovementSystem 检测 `!was_grounded → is_grounded` → 从缓冲 `pop_if(|req| req == Jump)` → 注入 ActionRequestBuf 前端 → 落地即跳。

**与土狼时间的协作**——落地帧同时存在两种触发可能：土狼时间跳跃 + 缓冲跳跃。MovementSystem 的优先顺序确保两者不冲突。

## 六、Combo Buffer——动作链预输入

轻攻击的 `cancel_window_ms = 120`。在此窗口内按下轻攻击→缓冲→当前动作完成时自动开始下一个轻攻击。

```rust
// ActionController 中
if current.phase == Recovery && current.cancel_window_open {
    if let Some(next) = buffer.pop_if(|req| def.cancel_set.contains(&req.action_id)) {
        self.pending_request = Some(next);  // 下一帧注入
    }
}
```

两动作之间有一帧间隙——角色有正常的移动（无缝，不是"卡住"）。

## 七、边缘吸附

```rust
fn apply_ledge_snap(transform, terrain, feel) {
    let pos = transform.position;
    if let Some(hit) = terrain.terrain_raycast(
        pos + Vec3::Y * 0.1, Vec3::NEG_Y, feel.max_snap_distance + 0.1
    ) {
        if slope_angle(hit.normal) <= feel.max_snap_angle_deg {
            transform.position.y = hit.point.y;  // 吸附到下方地面
        }
    }
}
```

**只处理"角色刚好踩在边缘"**（0.3m 内）——不处理真悬崖（>0.3m 向下无地面）。

## 八、帧级时序示例

```
冲刺→踩空→土狼时间跳跃→空中→落地→闪避：
帧0: Sprinting+Grounded
帧1: 踩空→loco=PhysicsBody, Airborne(Falling{coyote:0.15})
帧2: 按空格→coyote>0→Jump→Windup(50ms)
帧3: Windup→Active→Airborne(Jumping{control_ratio:0.7})
帧4-12: 空中; 帧10: 再按空格→coyote=0→Jump入缓冲
帧13: 着地→loco=Grounded; MovementSystem检测Jump缓冲→pop→起跳
帧17: 落地→缓冲中Dodge→消费→闪避
✅ 全程无缝
```

## 九、NPC 不需要缓冲

NPC 的 GOAP 产出 ActionRequest 时已考虑时机——算法决定"何时起跳"，不是玩家手感。`CInputBuffer` 只在有 `CPlayerTag` 的实体上激活。
