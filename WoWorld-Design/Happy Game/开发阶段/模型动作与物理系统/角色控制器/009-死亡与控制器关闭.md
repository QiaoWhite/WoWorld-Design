## 〇、定位

死亡不是"一个状态"。死亡是**控制器的终结**——不是"角色在做死亡这个动作"，而是"没有角色了，只有一个受物理支配的尸体"。处理方式不是加一个分支，是**在所有 Gameplay System 之前加全局门控**。

---

## 一、CDead——死亡标记

```rust
pub struct CDead {
    pub died_at: GameTime,
    pub cause: DeathCause,
    pub died_in_locomotion: LocomotionMode,
    pub died_in_movement: MovementState,
}

pub enum DeathCause {
    VitalDepleted { vital, source },  // HP/Spirit/Breath 归零
    OldAge, SpiritExhaustion,
    InstantDeath { source, spell_id },
    Environmental, Executed { authority, method },
}
```

---

## 二、死亡时间线

```
濒死（HP≤0） → 瘫软（Action中断，MovementLock=Full）
     ↓ 濒死结束
死亡帧 → CDead 设置
     │  ActionController: Interrupted(Death) → current=None, buffer.clear()
     │  MovementState: pace=Still, special=None, recovery.clear()
     │  CGoapIntent + CNpcMovementBehavior: 移除
     │  LocomotionMode: 保留（地面死的=Grounded，空中死的=PhysicsBody）
     │
死亡后 → SkeletalRelaxationSystem（仅 CDead 实体处理）
     │  Phase: Primary → Collapsing → FullyRelaxed
     │  可选: RigorMortis（死僵——特定种族/条件，肌肉硬化而非松弛）
     │  muscle_stiffness 指数衰减（e^(-decay_rate × elapsed)）
     │  不同死因→不同衰减速率
     │
完全松弛 → CorpsePhysicsSystem（仅 CFullyRelaxed 实体）
     │  静态 COM 物理——贴地 + 停止移动
     └  尸体作为静态障碍物——EntityIndex 注册 AABB
```

## 三、受影响的 System——完整清单

| System | 死亡后行为 | 门控方式 |
|--------|----------|---------|
| ActionResolverSystem | 跳过——玩家死后按键无响应 | `Without<CDead>` |
| ActionController | 停止——不产出新动作 | `Without<CDead>` |
| MovementModeSystem | 状态冻结 | `Without<CDead>` |
| StaminaGateSystem | 体力不再消耗 | `Without<CDead>` |
| GoapIntentDispatchSystem | 停止规划 | `Without<CDead>` |
| InstinctLayerSystem | 本能层关闭 | `Without<CDead>` |
| Combat Systems | 不能被攻击（伤害管线跳过） | `Without<CDead>` |
| Climbing Systems | 握力停止消耗，攀爬状态清除 | `Without<CDead>` |
| GripRegenerationSystem | 握力不再恢复 | `Without<CDead>` |
| **AnimationSystem** | **不跳过**——播放死亡动画 + 松弛 | 读 `CSkeletalRelaxation` |
| **SkeletalRelaxationSystem** | **仅 CDead**——处理尸体 | `With<CDead>` |
| **CorpsePhysicsSystem** | **仅 CDead + CFullyRelaxed** | `With<CFullyRelaxed>` |
| **SensorySystem** | **不跳过**——尸体可被感知 | 读 `CDead` 位置 |

## 四、死因与松弛参数

```toml
# death_physics.toml
[death.relaxation.vital_depleted]  decay_rate = 0.5, primary_phase_secs = 0.3
[death.relaxation.old_age]         decay_rate = 0.3, primary_phase_secs = 0.5  # 更慢——"静静走了"
[death.relaxation.spirit_exhaustion] decay_rate = 2.0, primary_phase_secs = 0.1  # 极快崩塌
[death.relaxation.instant_death]   decay_rate = 5.0, primary_phase_secs = 0.0  # 立即
[death.relaxation.executed]        decay_rate = 0.4, primary_phase_secs = 0.4  # 略慢——"仪式感"
```

## 五、玩家死亡特殊处理

- **按键被门控**——所有 Gameplay System 的 `Without<CDead>` 阻止响应。系统菜单（Esc）仍在 Godot 侧可用。
- **双角色**——角色 A 死亡，角色 B 继续 Auto 模式。玩家可选择立刻切换到 B。
- **继承 UI**——1.5s 后自动进入"死亡后的选择"（继承链/大日志/存档）。见 [[../../玩家系统/005-死亡继承与衰老|死亡继承与衰老]]。
- **死亡时各机制的清理**——MovementRecovery.clear(), buffer.clear(), action_buf.clear(), grip/climb/mounted 清除。

---

## 六、死亡后感知

SensorySystem 不跳过 CDead 实体。NPC 发现尸体→`NpcObservedDeathEvent`→情绪响应（恐惧/悲伤/愤怒）+ 行动响应（报警/逃跑/搜尸）+ 社会系统（声誉传播/法律调查）+ 大日志（见证记录）。

---

## 七、复活恢复

复活（濒死被救活）→ CDead 移除 → MovementState 恢复为虚弱状态（`Pace: Walking`, `exhaustion_cooldown: 2.0`）。ActionController 和 InputBuffer 已空——从零开始。
