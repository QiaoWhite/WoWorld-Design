## 〇、定位

NPC 的移动与玩家共享同一套 MovementSystem + ActionController。差异只在**意图来源**——NPC 的 `MoveIntent` 来自 GOAP 路径规划，而非玩家摇杆。

## 一、12 种移动行为模式

| 行为 | MovementState | MoveIntent.direction | 触发 |
|------|--------------|---------------------|------|
| **闲逛/徘徊** | Walk, Standing | 随机方向（~5s 换） | GOAP: Idle |
| **日常出行** | Walk/Run | 沿路径→目的地 | GOAP: TravelTo(normal) |
| **赶往某事** | Run/Sprint | 沿路径→目的地 | GOAP: TravelTo(urgent) |
| **跟踪/尾随** | Sneak, Crouch | 朝向目标+维持距离 | GOAP: Follow |
| **逃跑** | Sprint | 远离威胁源 | GOAP: FleeFrom |
| **战斗接近** | Run | 朝向目标 | 战斗三层模型 |
| **战斗周旋** | Walk | 切向目标（圆弧） | 战斗三层模型 |
| **巡逻/守卫** | Walk | 沿路径点循环 | GOAP: Patrol |
| **搜索** | Walk | 不规则方向（~2s 换） | GOAP: Search |
| **社交接近** | Walk | 朝向对话对象 | GOAP: InteractWith |
| **回家/回巢** | Run | 沿路径→家 | GOAP: ReturnHome |
| **水中/攀爬/骑乘** | 对应 SpecialMode | 朝向目标 | 介质自动切换 |

## 三、GoapIntentDispatchSystem——GOAP→意图翻译

```rust
fn goap_intent_dispatch_system(
    intent: &CGoapIntent, mode: &CControlMode,
    buf: &mut CActionRequestBuf, move_intent: &mut CMoveIntent,
    behavior: &mut CNpcMovementBehavior, move_state: &mut CMovementState,
) {
    let domains = mode.manual_domains();

    // GOAP 只在自己控制的域发离散动作——接管域不发
    for req in &intent.action_requests {
        if !domains.contains(&req.action_id.domain()) {
            buf.push(req.clone());
        }
    }

    // 移动域——同理。接管时保留 MovementState 不变。
    if !domains.contains(&ActionDomain::Movement) {
        if let Some(desired) = intent.desired_movement {
            move_intent.desired_state = Some(desired);
        }
    }

    // 设置移动行为——由 npc_movement_dispatch_system 进一步翻译
    if let Some(goal) = intent.current_goal {
        behavior.behavior = goal.to_move_behavior();
        move_state.pace = goal.urgency().to_pace();
    }
}
```

## 四、PathFollowingSystem——目标→每帧方向

```rust
fn path_following_system(
    behavior: &CNpcMovementBehavior, transform: &CTransform,
    path_state: &mut CPathFollowState, move_intent: &mut CMoveIntent,
    terrain: &dyn TerrainQuery, entity_index: &dyn EntityIndex,
) -> Option<ActionRequest> {  // 到达→返回 MovementArrived
    match &behavior.behavior {
        Navigate { destination, path, arrival } => {
            // 沿路径点前进→到达路径点→下一个→路径完成→检查到达判据
            let target_dir = (next_waypoint - pos).normalize();
            move_intent.direction = target_dir;
            if dist_to_dest < arrival.threshold() {
                return Some(ActionRequest::MovementArrived);
            }
        }
        Flee { threat_last_position, calm_down_at } => {
            move_intent.direction = (pos - threat_last_position).normalize();
            if now >= calm_down_at { return Some(ActionRequest::MovementArrived); }
        }
        Follow { target, desired_distance, max_distance } => {
            // 太远→接近，太近→后退，理想距离内→面朝不动
        }
        Loiter { next_change_at, .. } => {
            if now >= next_change_at { move_intent.direction = random_dir(); }
        }
        // ...
    }
    None
}
```

## 五、情绪/人格移动调制

```rust
fn emotion_movement_modulation(state: &mut CMovementState, emotion: &CEmotionState, big_five: &CBigFive) {
    match emotion.dominant() {
        Fear => { if state.pace == Walking { state.pace = Running; } }
        Sadness | Grief => { if state.pace == Running { state.pace = Walking; } }
        Anger => { if state.pace == Walking { state.pace = Running; } }
        _ => {}
    }
    // 高神经质→倾向于走快
    if big_five.neuroticism > 0.7 && state.pace == Walking { state.pace = Running; }
}
```

**步态涌现**——这些调制最终体现为每个人的独特走路动画，通过 `MovementState::gait_params()` 流入动画系统。

## 六、路径缓存策略

- 路径缓存 30 秒——大部分 NPC 不需要每帧重新寻路
- 重新寻路触发：路径过期 / 路径为空 / 被障碍物阻挡（`is_stale`）
- 分摊—每帧只对 ~5 个 NPC 运行 A*
- 远场 L2/L3 NPC——用统计移动，不走 A*
