## 〇、核心原则

夺舍不是创建新实体、不是复制状态、不是切换执行器。它是**在一个已有实体上装两个 Component，拆两个 Component**。

交接瞬间的原则：
1. 身体状态不受影响——MovementState/LocomotionMode/物理量全保留
2. 当前动作尽量保留——打铁的继续打铁，走路的继续走路
3. AI 的"思维"立即停止——GOAP 不再产出新意图
4. 输入缓冲清空——玩家还没开始按键，不应有残留的 NPC 意图
5. 不产生"切换感"——角色不应该弹跳、静止、改变姿势

## 一、三种夺舍场景

```rust
enum PossessionScenario {
    Quiet,       // NPC 空闲/走路/日常活动——保留当前动作，平滑接管
    SemiUrgent { current_action },  // NPC 在执行长期动作（打铁/做饭）——保留，标记可被取消
    Urgent { current_action, reason: InCombat | Dying },  // 强制清空——玩家需要立即完全控制
}
```

**判定逻辑**：
- 濒死 → `Urgent(Dying)`
- 在战斗中（动作类别=Combat 或状态=被击飞） → `Urgent(InCombat)`
- 在执行长期动作（Craft/Cook/Smith/Sleep/Eat/ChannelSpell/AimBow） → `SemiUrgent`
- 其他 → `Quiet`

## 二、PossessionTransitionSystem——七步交接

```rust
fn possession_transition_system(
    newly_possessed: Query<(Entity, &mut ActionBuf, &mut InputBuffer, &mut MovementState,
                             Option<&ActiveAction>, Option<&GoapIntent>, Option<&NpcMoveBehavior>),
                          Added<PlayerTag>>,
) {
    for (entity, buf, buffer, move_state, action, goap, behavior) in newly_possessed {
        // 1. 判定场景
        let scenario = classify(action, move_state, goap);

        // 2. 处理当前动作（按场景）
        match scenario {
            Quiet => { /* 保留——不做任何事 */ }
            SemiUrgent => {
                commands.add(CInterruptibleByPlayer { action_instance, cancel_on: AnyInput });
            }
            Urgent(InCombat) => {
                emit(Interrupted { by: PossessionTakeover });
                commands.remove::<ActiveAction>();
            }
            Urgent(Dying) => { /* 角色已瘫软——无动作需清理 */ }
        }

        // 3. 清空输入缓冲——NPC 残留意图不应影响玩家
        buffer.clear();

        // 4. 清空 ActionRequestBuf——GOAP 的待处理请求全部丢弃
        buf.clear();

        // 5. 保留物理状态，移除 AI 状态
        //   保留: MovementState, LocomotionMode, MovementRecovery, CoyoteTime,
        //         ClimbingState, Grip, MountedState, Vitals, EmotionState
        //   移除: GoapIntent, NpcMovementBehavior

        // 6. 设置初始玩家状态——Quiet+空闲→Walking；Urgent→Still+Standing
        if scenario == Quiet && action.is_none() { /* keep move_state */ }
        else if move_state.special.is_none() { move_state.pace = Still; }

        // 7. 通知 UI——状态摘要
        emit(PossessionSummary { location, current_activity, recent_key_event, ... });
    }
}
```

## 三、CInterruptibleByPlayer——安静取消

```rust
pub struct CInterruptibleByPlayer {
    pub action_instance: ActionInstanceId,
    pub cancel_on: PlayerCancelTrigger,  // AnyInput / MoveInput / CombatInput
}
```

NPC 在打铁→玩家夺舍→`CInterruptibleByPlayer` 标记。玩家按任何键→`PlayerInterruptSystem` 检测→发出当前动作的取消信号→ActionController 中断打铁→玩家按键被正常响应。不按键→打铁正常完成→标记随 Action 完成自动移除。

## 四、释放——玩家放弃控制权

```rust
fn release_possession_system(
    released: Query<(Entity, &mut InputBuffer, &mut ActionBuf), Removed<PlayerTag>>,
) {
    // 清空玩家输入残留
    buffer.clear();
    // 保留移动相关动作（身体已经在做），清空战斗/交互
    buf.retain(|req| req.action_id.category() == Movement);
    // MovementState 保留——GOAP 下一帧读当前状态重新规划
    // GOAP 看到"我在冲刺但没有紧急目标→改为 Walking"
}
```

## 五、DomainDelegated 特殊处理

夺舍时指定 `DomainDelegated { Combat, Movement }` → 仅这些域被接管。GoapIntentDispatchSystem **不**移除 `CGoapIntent`——GOAP 继续在非接管域（Speech/ItemUse/Interaction/MagicUse）产出。但接管域的动作被过滤。

## 七、边界情况全表（11 个场景）

| 场景 | 处理 |
|------|------|
| **NPC 在睡觉** | SemiUrgent——按任何键起床 |
| **NPC 在水中** | MovementState 保留 Swimming——玩家接管游泳方向 |
| **NPC 在攀爬** | CClimbingState 保留——玩家接管攀爬方向 |
| **NPC 在骑乘** | CMountedState 保留——玩家接管坐骑方向控制 |
| **NPC 在濒死** | Urgent(Dying)——瘫软，MovementLock=Full。只能等待救援或死亡 |
| **NPC 刚完成动作** | Quiet——完全无感。玩家推摇杆→角色走向新方向 |
| **NPC ActionBuf 中有 [Greet]** | 清空——"我正要走近铁匠……嗯，本来要说什么来着？"（合理化） |
| **NPC Buffer 中有 Jump** | 清空——NPC 的跳跃缓冲是 GOAP 意图，不是物理事实 |
| **两个角色切换** | A: release→buffer 清空→GOAP 接管。B: possession→buffer 清空→玩家接管。两边无缝 |
| **同 L1 范围内切换** | 无延迟——两者都已完全加载。瞬时切换 |
| **远处角色切换** | 加载目标区域→游戏暂停→加载完摄像机切过去（玩家系统 003 定义） |

## 八、完整帧时序示例

```
帧 T: 玩家按 CharacterSwitch
  [铁匠 A]: remove(PlayerTag) → release_possession_system → GOAP 接管
  [法师 B]: add(PlayerTag) → possession_transition_system:
    scenario = Urgent(InCombat)（法师在击飞中）
    → buffer.clear(), action_buf.clear()
    → remove(GoapIntent), remove(NpcMovementBehavior)
    → MovementState 保留: Airborne(KnockedBack{recoverable_at:+0.3s})
    → LocomotionMode 保留: PhysicsBody
    → CMovementRecovery 保留: [{Standing,Sprinting}]
    → UI: "法师·苍山战场" "正在被击飞"

帧 T+10: 法师着地→loco=Grounded→pop recovery→{Standing,Sprinting}
  玩家推摇杆→方向立即响应。✅
```
