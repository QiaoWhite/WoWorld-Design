## 核心定位

角色控制器是 WoWorld 所有"物理存在"的**行动执行层**——它接收意图（无论来自玩家按键还是 NPC 的 GOAP），仲裁优先级，驱动身体执行。玩家和 NPC 走**同一套控制器代码**——差异只在意图来源（摇杆/WASD vs GOAP 规划器），不在身体执行。

**本模块不定义**：
- 物理运动学（`LocomotionMode` 三态机、`ImpulseQueue`、COM 积分 → [[../运动学地基/001-质量冲量与涌现运动学|CHG-067]]）
- 动画生成（9 层动画栈、骨骼矩阵 → [[../动画系统/001-动画系统总纲|动画系统]]）
- 空间查询（`TerrainQuery`、`EntityIndex` → [[../空间查询与物理/001-空间查询与物理系统|空间查询与物理]]）
- 战斗 AI（三层模型、本能层/节奏层/战略层 → [[../../战斗/003-三层战斗模型|三层战斗模型]]）
- AI 决策（GOAP、记忆、情绪 → [[../../NPC活人感模块/NPC活人感开发文档ver2.0|NPC活人感]]）

**本模块定义**：意图如何进入、动作如何仲裁、状态如何推进、结果如何通知——即控制器全部。

## 架构速览

```
                        意图来源（可插拔）
              ┌────────────┼────────────┐
              │            │            │
         PlayerInput   GOAPPlanner   BehaviorTree
         (按键→Intent)  (目标→Intent)  (动物AI→Intent)
              │            │            │
              └────────────┼────────────┘
                           ▼
              ┌────────────────────────┐
              │   ActionResolver       │  ← Phase 0
              │   · 六层输入映射        │     意图生成层
              │   · 上下文解析 + 轮盘    │
              └───────────┬────────────┘
                          │ ActionRequest
                          ▼
              ┌────────────────────────┐
              │   ActionController     │  ← Phase 1 中段
              │   · 承诺中断系统        │     行动仲裁层
              │   · 取消窗口 + 优先级    │
              │   输出: MovementLock    │
              └───────────┬────────────┘
                          │
                          ▼
              ┌────────────────────────┐
              │   MovementSystem       │  ← Phase 1 晚段
              │   · LocomotionMode 三态 │     连续执行层
              │   · 加速度/摩擦/斜坡    │
              │   · 碰撞响应            │
              └───────────┬────────────┘
                          │ (位移, 速度, 骨骼矩阵)
                          ▼
              ┌────────────────────────┐
              │   Rust → Godot 桥接    │  ← Phase 2
              │   set_transform +       │     纯渲染
              │   GPU skinning 推送     │
              └────────────────────────┘
```

**核心设计哲学**：
1. **控制器统一，意图分叉**——同一套执行代码，"谁在控制"是可插拔的
2. **承诺中断，非状态图**——复杂度在数据（TOML）中，不在代码中
3. **涌现式物理**——定义质量+冲量+积分器，让击退/体型压制/坠落涌现
4. **玩家 = NPC + I/O 适配层**——玩家只是"一个 NPC 被接管了控制权"

## 性能预算

| 子系统 | 每帧成本 | 占 16.6ms | 说明 |
|--------|---------|-----------|------|
| ActionResolver（玩家 1人） | < 0.01ms | < 0.1% | 分支 + HashMap 查找 |
| ActionController（全部实体） | ~0.01ms/实体 | — | 仅当前有 ActiveAction 的实体 |
| MovementSystem（全部实体） | ~0.05ms（SoA 批处理） | ~0.3% | rayon 并行，与实体数解耦 |
| 攀爬（近场 150人） | 2.20µs | 0.013% | 仅攀爬中实体 |
| 输入缓冲 | 忽略不计 | — | 环形缓冲区——O(1) 操作 |
| 事件系统（双缓冲） | 忽略不计 | — | 每帧 swap + Vec push |
| **合计（保守估计）** | **< 0.2ms** | **< 1.2%** | 在 7.0ms Rust 模拟预算内充裕 |
| NPC 寻路（分摊） | 1-2ms/帧（全 L1） | 6-12% | 分摊到多帧，每帧 ~5 个 NPC |
