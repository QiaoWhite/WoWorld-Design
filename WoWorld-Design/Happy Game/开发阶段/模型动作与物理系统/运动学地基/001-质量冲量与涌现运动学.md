## 二、冲量队列与单体应用（Q2/Q4）

冲量是**定向、一次性的命令**（目标实体 + 冲量向量），不是组件，也不是感知事件。它走**专用冲量队列**（`ImpulseQueue` Resource），一个系统每 tick drain 一次转成速度变化：

```rust
// woworld_ecs::resources::ImpulseQueue —— push-once / drain-once
pub struct Impulse { pub target: EntityId, pub impulse: Vec3 }  // J（N·s），世界空间

// woworld_ecs::systems::physics::impulse_apply —— Phase 1 早段
// 确定性：drain 前按 target EntityId 排序
fn impulse_apply_system(world, queue: &mut ImpulseQueue) {
    for ev in queue.drain_sorted_by_target() {
        let (mass, vel, mode) = world.query_mut(ev.target);   // Mass, Velocity, LocomotionMode
        *vel += ev.impulse / mass.0;                          // Δv = J / m —— 体型差自动涌现
        *mode = LocomotionMode::PhysicsBody;                  // 进入 COM 慢路
    }
}
```

> ⚠️ **与现有 `SpatialEventBus` 的分工（本 CHG 明确）**：`空间查询与物理/001 §1.3` 的 `SpatialEvent::WeaponImpact{force, weapon_type}` 只带**标量力 + 位置**，是给感官/音频用的**感知通知**（NPC 听见/看见这一击），**不带目标实体也不带定向冲量向量**。因此定向冲量另走 `ImpulseQueue`——命中时战斗/魔法**同时**：① 向 `ImpulseQueue` push 定向冲量（打谁、多大、朝哪）；② 向 `SpatialEventBus` push `WeaponImpact/Explosion` 感知通知。两条通道正交，不复用。

- **体型差免费涌现**：同一冲量打小妖（小 mass）飞老远、打巨魔（大 mass）几乎不动。场景 2"体型质量差异影响碰撞"无需单独写。
- **确定性**：drain 顺序按 `target EntityId` 排序，纯函数，同 seed 可复玩。

## 三、运动学三态机（Q2/Q3/Q5）

每个物理存在的实体处于三态之一。**常态零成本**是设计锋线：99% NPC 常驻 `Grounded` 快路，只有正在飞的少数进 `PhysicsBody` 慢路。

```rust
// woworld_ecs::components::physics
#[derive(Debug, Clone, Copy, PartialEq)]
pub enum LocomotionMode {
    Grounded,                          // 贴地/贴面：height_at 或攀爬切平面。无 COM 积分
    PhysicsBody,                       // 空中：COM 抛物体积分（空间查询 §2.2）
    Attached { anchor: EntityId },     // 挂在移动锚上：local→world（见 §六）
}
```

| 态 | 每帧 | 进入 | 退出 |
|----|------|------|------|
| **Grounded** | `height_at` 贴地 + 坡度门控（或攀爬切平面 §八） | 着地/重挂 | 冲量/走出地面/跳跃/死亡 |
| **PhysicsBody** | COM 积分（空间查询 §2.2）+ 飞行中离散接触（§五） | `impulse_apply` / 失足 / detach | 三级着地（空间查询 §2.3）→ Grounded 或 Attached |
| **Attached** | `world = anchor.transform × local`（§六） | 登船/骑乘/爬上巨兽 / detach 后落到锚 | 跳跃/被击飞（→ PhysicsBody，带速度交接） |

**玩家（Q3）**：被击退 → 统一进 `PhysicsBody`（与 NPC 同一 COM 积分），落地即还控。日常移动仍走玩家专属 Godot 路（见 §九）。

> ⚠️ **集成留白（首冲刺敲定）**：现有 COM 积分器在 `AnimationBodyState.com_velocity` 上跑（空间查询 §2.2），而 ECS 有独立 `Velocity` 组件（现为死代码）。二者须定**单一真相源**——建议 `Velocity` 组件为权威、`com_velocity` 为其每帧镜像；具体桥接在 Q2-Q5 首冲刺敲定。记入 [[../../../../参考文档/045-物理地基设计讨论-20260709/003-待验证与开放问题|045/003]]。

> ✅ **着地阈值单一真相源（Q-A1 已审批 2026-07-09）**：三级着地/坠落伤害以**速度制为唯一真相源**——用 COM 积分原生输出的命中速度（同时兼容水平撞墙冲击），阈值定义见空间查询 §2.3（>8/>20 m/s 量级，具体数值待重调 N4）。战斗 009 的"落差 >10m/>30m"距离阈值**降级为速度制的涌现推导**，不再独立定义（已在战斗 009 §三加一致性指针）。

## 四、冲量分流（Q4）——手感涌现

命中瞬间，武器动量按武器属性分流。武器动量 = `weapon_mass × 挥击末端速度`（末端速度是 capsule 扫掠副产品，零额外成本）：

```rust
// woworld_core::kinematics —— 纯函数
pub struct ImpulseSplit { pub knockback_coeff: f32, pub penetration_coeff: f32 } // TOML/武器
pub fn split_impact(momentum: f32, split: ImpulseSplit, hit_dir: Vec3)
    -> (Vec3 /*knockback impulse*/, f32 /*penetration*/)
{
    ( hit_dir * momentum * split.knockback_coeff,
      momentum * split.penetration_coeff )
}
```

- **一次读数喂两消费者**：同一 `momentum` → 一路给伤害（场景 2"加速度加成"= 伤害∝动量）、一路给击退冲量（场景 4"飞多远"）。
- **手感涌现**：钝锤（高 knockback_coeff）→ 人飞出去；锐剑（高 penetration_coeff）→ 扎入不飞。零硬编"某招击退力=X"。
- **不删旧设计**：`AttackTrajectory::UpwardStrike{launch_power}`（战斗 005 已存在）重新解释为"上挑传递系数"（决定动量里多少转竖直冲量），非绝对高度。
- 系数具体值待战斗调参（N1，provisional）。

## 五、飞行中离散接触（Q5）——非递归

`PhysicsBody` 每步除对地形 raycast 外，对扫掠 AABB 内实体做一次 `entities_in_aabb`：

```rust
// PhysicsBody 每步（LOD 0-1 才开；远场跳过）
if let Some(other) = index.entities_in_aabb(swept_aabb, filter).first() {
    stop_or_tumble(self, at_contact);                       // 飞行体停/翻滚
    if self.impact_depth == 0 {                             // 递归护栏：深度上限 1
        queue.push(Impulse { target: other.id,              // 二次小冲量入冲量队列（封顶）
                             impulse: capped_secondary(self.velocity, other) });
        // 可选：另发一个 SpatialEvent 感知通知供旁观 NPC 反应
    }
}
```

- **非递归**：被撞旁观者最多吃一个封顶二次冲量进入自己的 PhysicsBody；他再撞第三者只触发踉跄、不再传冲量。拿到"多米诺一下"观感，**永不掉进 N 体动量守恒求解**。
- **一套盖三场景**：生物正常对撞走避让快路（空间查询 §0 迭代分离）；生物高速撞 / 被击飞 → 本节；载具相撞 → 本节 + 接建筑 `component_health` 结构伤害。
- **明确放弃**：真实回弹/动量守恒（台球物理）。二次冲量封顶值 N5 待定。

## 六、SurfaceAnchor 移动参考系（Q6）——甲板/坐骑/巨兽

**薄契约，非超类型**：可被站立的移动锚实现一个 trait，载具与生物**平等实现**，不触碰"载具≠坐骑"的意志分界（那是按意志分的正交关注点）。

```rust
// woworld_core —— trait + 值类型
pub trait SurfaceAnchor: Send + Sync {
    fn anchor_transform(&self, id: EntityId) -> Mat4;             // 本帧世界位姿
    fn velocity_at(&self, id: EntityId, world_pt: Vec3) -> Vec3;  // 接触点瞬时速度 = v + ω×r
    fn walkable_local(&self, id: EntityId) -> &WalkableArea;      // 局部可行走区
}
// woworld_ecs::components —— 两个小组件
pub struct AttachedTo   { pub anchor: EntityId, pub local_pos: Vec3, pub local_yaw: f32 }
pub struct WalkableAnchor { pub area: WalkableAreaId }
```

- 载具给船实现（`anchor_transform` = 粗位姿 ∘ Gerstner 俯仰横摇）；Life 给坐骑/巨兽实现（= 鞍骨/背骨世界 transform）。这是 §4.2 骑乘"骨盆=马鞍 transform"的泛化。
- **挂载时**：`world = anchor.transform × local`，走路只更新 `local_pos`（波动跟随由 transform 合成免费得到）。
- **脱离（跳跃/被击飞）**：detach → `PhysicsBody`，世界速度 = `velocity_at(接触点) + 主动速度`（"跳下行进船"）→ 落到任意 `SurfaceAnchor` → 重挂。**这条把本系统焊回 §三三态机**。
- **双层位姿（频率坑）**：载具导航 `world_transform` 每 1-10s 粗更新（载具 003 L131）；但载人的船需每帧俯仰横摇——由**每帧物理放置系统**算瞬时位姿（波数据向 `OceanProvider` 采样船体 4 点），**Rust 单一权威源**同喂 Godot 渲染与甲板挂载（防"人浮在视觉甲板上方"）。位姿对所有可见锚算（便宜），挂载解算只对载人的锚跑。
- **涌现记忆点**：被巨浪从甲板掀飞 → COM 抛物线 → 落在掠过的海蛇背上（另一 `SurfaceAnchor`）→ 自动重挂。两者实现同一 trait，零特判。

## 七、投射物内核与 CCD（Q7）——三空白一并填

**复用 core 纯运动学内核，不复用 PhysicsBody 重表示**（数量/结构/碰撞/生命周期全不同）：

```rust
// woworld_ecs::components —— 轻量，SoA 批处理
pub struct Projectile {
    pub vel: Vec3, pub gravity_scale: f32, pub drag: f32,
    pub kind: ProjectileKind, pub shooter: EntityId, pub payload: HitPayload,
}
```

- `ProjectileKind → (gravity_scale, drag, radius, hit_behavior)` 全 TOML。`gravity_scale` 让"抛物 vs 直线"涌现：箭=1.0（弧线）、火球=0.0（直线）、投矛=1.0。
- **CCD 逐步扫掠**（防快箭穿薄目标）：每步把 `pos→pos+vel*dt` 一段线段测 ① 地形 `terrain_raycast`（DDA 逐体素不穿）② 实体 `segment_vs_capsule`（`entities_in_aabb` broadphase + 解析求交，glam 手写）。
- **命中喂回**：箭伤 = 箭动量（同 §四读数，直接结算 payload）；火球命中 → 对半径内每个实体向 `ImpulseQueue` push 一发径向冲量（按距离衰减）+ 向 `SpatialEventBus` push `Explosion` 感知通知 → **炸飞周围（填魔法 AOE 击退空白）**。
- **三空白一并填**：投射物(#3) + 火球 AOE 击退 + 掉落物积分(#4：`DroppedItem` = `gravity_scale=1, hit=Rest` 的投射物，落地静止转 `PlacedItem`)。
- **Godot**：箭用 `MultiMeshInstance3D` GPU 实例化，火球用 `GPUParticles3D` 做 VFX，**模拟在 Rust**。
- **LOD**：屏外大战走方阵统计模型，逐箭只在近场。
- **否决**：hitscan（给不了弧线/风偏/可闪避/可拾回/AOE 击退）；预建对象池（先 ECS 实体，profiling 显示 spawn 抖动再迁）。

## 八、攀爬泛化运动学（Q8）

攀爬 = 把 `movement_system` 从"假设法线朝上"**泛化到任意表面法线**，不是新系统。

```rust
// woworld_ecs::components —— 常驻能力组件（不增删，防 archetype 抖动）
pub struct Climbing {
    pub max_grip_angle: f32,           // 可通行的最大面角（人45°→攀爬90°→蜘蛛任意）
    pub grip_strength: f32,            // 握力/体力衰减率
    pub gravity_frame: GravityFrame,   // World（攀爬，会掉）| Surface（壁面行走，不掉）
}
```

| | 贴地（现有） | 泛化后（贴任意面） |
|---|---|---|
| 贴面查询 | `height_at`（竖直） | `terrain_raycast(向面)` + `normal_at`（任意向） |
| 门槛 | 全局常量 0.707 | 每实体 `Climbing.max_grip_angle` |
| 移动 | XZ 平面 | 切平面投影 `move - n*(move·n)`（glam） |
| 掉落 | 无 | 握力/体力失效 → detach → COM 坠落（§三） |

- **握力涌现**：`surface_material_at`（湿岩/冰面低、干岩高，连天气雨后变滑）× `vitals.fatigue`（体力）→ 握力。失手 → detach → 空间查询 §2 坠落。**无脚本序列**。
- **体素 SDF 天生适配**：任意面有真实法线，"攀悬垂下方/洞顶"是地形表示的自然能力（高度场做不到）。差异化记忆点。
- **壁面重力重定向**（蜘蛛，`gravity_frame=Surface` + 身体 up 对齐法线 `Quat::from_rotation_arc(Vec3::Y, n)`）= 同系统参数极值，**动画/相机更费工，延后**。

### 八·补 — "大量 NPC 都具备攀爬"的规模化（Q8 修正）

算力非瓶颈（150 近场逐帧=2.2µs、3000 远场粗粒度=1.8µs）。规模化的**承重墙是导航 + LOD，不是物理**：

1. **导航图必须编码"可攀爬边" + 按握力过滤**（🟡 **属导航/NPC 行动系统，非本模块**）——A* 加能力掩码 `entity.max_grip_angle ≥ edge.required_grip`；按需/局部/随 chunk 流式生成，非全局预计算。**本模块只提供逐帧执行内核，"能不能规划出爬墙路径"由那边负责**（见跨模块插口 §十一 / Q-A4）。
2. **LOD 攀爬遍历状态**——远场粗粒度边遍历（无 raycast + 种子失手概率，统计涌现）；近场提升逐帧；粗↔细提升锚同一攀爬边保连续。
3. **ECS**——`Climbing` 常驻不增删，"当前是否在爬"用 `LocomotionMode`/状态位。
