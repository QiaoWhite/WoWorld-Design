# 001 — 物品系统总纲

## 一、设计哲学

### 1.1 "万物有 ID"

WoWorld 中一切可持有、交换、装备、消耗、丢弃的东西——从一把传世名剑到一块路边的鹅卵石——都有两个身份：

```
ItemDefId: 这是"什么类型"的东西——在全部存档中恒定不变
ItemEntId: 这是"哪一个具体"的东西——在此存档中独一无二
```

铁矿石永远是同一个 ItemDefId。但王五铁匠铺里那块特定的铁矿石有一个唯一的 ItemEntId——History 系统可以追踪它被谁开采、卖给谁、锻造成了什么。

### 1.2 物品系统是基础设施，不是玩法

物品系统只定义：
- 东西怎么标识（ID）
- 东西有哪些属性（Properties）
- 东西怎么组装（Assembly）
- 东西怎么存放（Inventory/Container）

物品系统**不定义**：
- 武器怎么挥砍 → Combat 管
- 药水怎么起效 → Magic 管
- NPC 为什么选这把剑 → NPC 活人感管
- 这把剑有什么历史 → History 管

### 1.3 故事生成器的物质根基

物品不只是属性载体。它们有 provenance——出生、转手、磨损、附魔、被偷、被埋、被重新发现。一把剑可以活一千年，见证王朝兴衰，最终被玩家从海底废墟中打捞出来。**物品的生命周期是 WoWorld 历史的最小叙事单元。**

## 二、两层 ID 模型

### 2.1 ItemDefId — 模板/原型 ID（全局恒定）

```rust
/// 全局恒定的物品定义标识符
/// 布局: [8-bit category][56-bit def_index]
/// category 0x00-0x7F: 核心游戏（128 类别）
/// category 0x80-0xFF: Mod 命名空间（128 类别）
pub struct ItemDefId(u64);

impl ItemDefId {
    pub fn category(&self) -> u8 {
        (self.0 >> 56) as u8
    }
    pub fn def_index(&self) -> u64 {
        self.0 & 0x00FF_FFFF_FFFF_FFFF
    }
}
```

- **全局恒定**：铁矿石的 ItemDefId 在所有存档、所有机器、所有版本中完全相同
- **Mod 安全**：核心游戏 0-127 类，Mod 128-255 类——不会冲突
- **蓝图/配方引用它**：`.blueprint` TOML 文件中写的是 ItemDefId——跨存档可分享
- **56-bit 实例空间**：每类别 72 千万亿个可分配 ID——足够

### 2.2 ItemEntId — 实例/实体 ID（存档内唯一）

```rust
/// 存档内唯一的物品实体标识符
/// 出生时分配，消耗/销毁后不复用（56-bit 空间足够永不回收）
/// 具名物品（有 provenance）的 ItemEntId 永生——
/// 即使物品沉入海底一千年，History 仍可引用
pub struct ItemEntId(u64);

/// 可变状态——不存于 ID 中
pub struct ItemState {
    pub durability: f32,           // 0.0-1.0
    pub quality: Quality,          // 当前品质
    pub enchantments: Vec<EnchantmentId>,
    pub custom_name: Option<String>,  // "杀龙剑"——玩家/NPC 命名
    pub inscription: Option<String>,  // 铭文
    pub owner_history: Vec<(NpcId, GameTimestamp)>,  // 所有权链
    pub continuous_params: ItemParams,  // 连续参数（尺寸/材质等）
}
```

- **存档内唯一**：每个存档的"杀龙剑"是不同的剑——History 不跨存档
- **出生分配**：所有物品（包括矿堆里的每一块铁矿石）出生时获得 ItemEntId
- **永不回收**：56-bit 每存档 = 72 千万亿 ID。每秒生成 100 万物品，可用 2,300 年
- **Provenance 锚点**：History 系统只追踪 ItemEntId——不管它变成什么形态

### 2.3 两层之间的映射

```rust
/// 世界中的所有物品实例
pub struct ItemRegistry {
    /// ItemEntId → (ItemDefId, ItemState)
    instances: HashMap<ItemEntId, (ItemDefId, ItemState)>,
    /// ItemDefId → ItemProperties（模板数据，全局加载一次）
    definitions: HashMap<ItemDefId, ItemProperties>,
}
```

## 三、通用组件装配框架

### 3.1 组装的不同形态

WoWorld 的武器不是属性表中的一行——它们是多个参数化组件的装配结果。物品系统提供**通用装配框架**，Combat 和 Magic 注册各自的组件类型：

```rust
/// 物品可以是单件或组合件
pub enum ItemAssembly {
    /// 单件物品（大多数物品——矿石/药水/书籍/货币）
    Simple { def: ItemDefId },

    /// 多组件装配（武器/工具/复杂物品）
    Composite {
        root_ent_id: ItemEntId,        // 装配体的身份（History 追踪此 ID）
        components: Vec<ComponentSlot>, // 子组件列表
        connections: Vec<ComponentConnection>,  // 组件之间的连接方式
    },
}

/// 组件槽——装配体中的一个位置
pub struct ComponentSlot {
    pub slot_type: u16,               // 组件类型标签（Combat/Magic 注册）
    pub occupant: Option<ItemEntId>,  // 占用的组件（自身也是物品）
    pub params: ItemParams,           // 连续参数（长/宽/厚/曲率/密度等）
}

/// 组件之间的连接方式
pub struct ComponentConnection {
    pub from_slot: usize,
    pub to_slot: usize,
    pub joint: JointType,
}

pub enum JointType {
    Rigid,                                          // 刚性——99% 的武器（柄+刃=一根棍子）
    Hinge { axis: Vec3, limits_deg: (f32, f32) },   // 铰链——双节棍
    Chain { links: u16, link_length_mm: f32 },      // 链连接——连枷/锁镰
    Flexible { stiffness: f32, taper_ratio: f32 },  // 柔性——鞭子/绳镖
}

/// 连续参数——由 Combat/Magic 的组件类型定义其 schema
pub struct ItemParams {
    pub floats: BTreeMap<u16, f32>,  // 参数ID → 值
    pub ints: BTreeMap<u16, u32>,
}
```

### 3.2 近战武器示例

```

    components: [
        Slot { type: MeleeGrip(Combat注册), occupant: 钢柄, params: { length: 0.8m } },
        Slot { type: StrikingHead(Combat注册), occupant: 铁刃, params: { L:1.1m, W:4cm, T:8mm } },
        Slot { type: Guard(Combat注册), occupant: 铁护手, params: {} },
        Slot { type: Pommel(Combat注册), occupant: 钢配重, params: {} },
        Slot { type: EnchantmentSlot(Combat注册), occupant: 火焰符文, params: {} },

```

```

    root: ItemEntId(0xC017),
    components: [
        Slot { type: ImplementBody(Magic注册), occupant: 黑檀杆, params: { length: 1.8m } },
        Slot { type: ImplementCore(Magic注册), occupant: 龙晶核心, params: { purity: 0.95 } },
        Slot { type: ImplementFocus(Magic注册), occupant: 蓝宝石, params: { diameter: 3cm } },

    ],
}
```

### 3.5 为什么用注册制而不是枚举

Item System 不知道"BowLimb"是什么。它只知道"slot_type=0x0201"。Combat 模块注册 `0x0201 = BowLimb`，并提供：
- 参数 schema（可调 float 有哪些？可调 int 有哪些？）
- 兼容性检查（这个 slot_type 能与哪些父 DefId 搭配？）
- 动作基元派生（从 slot_type → 可用 combat actions）

**新增武器类型不需要修改物品系统。** 加火器？Combat 注册新 slot_type → 物品系统直接支持。魔法引擎？Magic 注册新 slot_type → 同上。
