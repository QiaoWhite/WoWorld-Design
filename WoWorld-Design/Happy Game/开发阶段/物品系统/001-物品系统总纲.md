# 001 — 物品系统总纲

## 一、设计哲学

### 1.1 "万物有 ID"

WoWorld 中一切可持有、交换、装备、消耗、丢弃的东西——从一把传世名剑到一块路边的鹅卵石——都有两个身份：

```
ItemDefId: 这是"什么类型"的东西——在全部存档中恒定不变
ItemEntId: 这是"哪一个具体"的东西——在此存档中独一无二
```

铁矿石永远是同一个 ItemDefId。但王五铁匠铺里那块特定的铁矿石有一个唯一的 ItemEntId——History 系统可以追踪它被谁开采、卖给谁、锻造成了什么。

### 1.2 物品系统是基础设施，不是玩法

物品系统只定义：
- 东西怎么标识（ID）
- 东西有哪些属性（Properties）
- 东西怎么组装（Assembly）
- 东西怎么存放（Inventory/Container）

物品系统**不定义**：
- 武器怎么挥砍 → Combat 管
- 药水怎么起效 → Magic 管
- NPC 为什么选这把剑 → NPC 活人感管
- 这把剑有什么历史 → History 管

### 1.3 故事生成器的物质根基

物品不只是属性载体。它们有 provenance——出生、转手、磨损、附魔、被偷、被埋、被重新发现。一把剑可以活一千年，见证王朝兴衰，最终被玩家从海底废墟中打捞出来。**物品的生命周期是 WoWorld 历史的最小叙事单元。**

## 二、两层 ID 模型

### 2.1 ItemDefId — 模板/原型 ID（全局恒定）

```rust
/// 全局恒定的物品定义标识符
/// 布局: [8-bit category][56-bit def_index]
/// category 0x00-0x7F: 核心游戏（128 类别）
/// category 0x80-0xFF: Mod 命名空间（128 类别）
pub struct ItemDefId(u64);

impl ItemDefId {
    pub fn category(&self) -> u8 {
        (self.0 >> 56) as u8
    }
    pub fn def_index(&self) -> u64 {
        self.0 & 0x00FF_FFFF_FFFF_FFFF
    }
}
```

- **全局恒定**：铁矿石的 ItemDefId 在所有存档、所有机器、所有版本中完全相同
- **Mod 安全**：核心游戏 0-127 类，Mod 128-255 类——不会冲突
- **蓝图/配方引用它**：`.blueprint` TOML 文件中写的是 ItemDefId——跨存档可分享
- **56-bit 实例空间**：每类别 72 千万亿个可分配 ID——足够

### 2.2 ItemEntId — 实例/实体 ID（存档内唯一）

```rust
/// 存档内唯一的物品实体标识符
/// 出生时分配，消耗/销毁后不复用（56-bit 空间足够永不回收）
/// 具名物品（有 provenance）的 ItemEntId 永生——
/// 即使物品沉入海底一千年，History 仍可引用
pub struct ItemEntId(u64);

/// 可变状态——不存于 ID 中
pub struct ItemState {
    pub durability: f32,           // 0.0-1.0
    pub quality: Quality,          // 当前品质
    pub enchantments: Vec<EnchantmentId>,
    pub custom_name: Option<String>,  // "杀龙剑"——玩家/NPC 命名
    pub inscription: Option<String>,  // 铭文
    pub owner_history: Vec<(NpcId, GameTimestamp)>,  // 所有权链
    pub continuous_params: ItemParams,  // 连续参数（尺寸/材质等）
}
```

- **存档内唯一**：每个存档的"杀龙剑"是不同的剑——History 不跨存档
- **出生分配**：所有物品（包括矿堆里的每一块铁矿石）出生时获得 ItemEntId
- **永不回收**：56-bit 每存档 = 72 千万亿 ID。每秒生成 100 万物品，可用 2,300 年
- **Provenance 锚点**：History 系统只追踪 ItemEntId——不管它变成什么形态

### 2.3 SlotInstanceId ——装配体内的位置身份

```rust
/// 装配体内部——每个组件槽位有独立身份
/// ItemEntId = 装配体根节点的存档内身份（History 追踪此 ID）
/// SlotInstanceId = 槽位在装配体内的位置身份（仅在该装配体内唯一）
pub struct SlotInstanceId(u16);  // 槽位索引——0..components.len()

// 举例：
// 一把剑的 ItemEntId = 0xA001（History 追踪"杀龙剑"的所有事件）
//   槽位 0 (SlotInstanceId(0)) = 钢柄（换了三次柄——每次 occupant ItemEntId 不同）
//   槽位 1 (SlotInstanceId(1)) = 铁刃（原件，从未换过）
//   槽位 2 (SlotInstanceId(2)) = 火焰符文（上周新换的）
//
// 换了柄 → SlotInstanceId(0).occupant 变了 → 但 root ItemEntId 不变
// → History 不受影响 → "杀龙剑"仍然是"杀龙剑"
```

### 2.4 两层之间的映射

```rust
/// 世界中的所有物品实例
pub struct ItemRegistry {
    /// ItemEntId → (ItemDefId, ItemState)
    instances: HashMap<ItemEntId, (ItemDefId, ItemState)>,
    /// ItemDefId → ItemProperties（模板数据，全局加载一次）
    definitions: HashMap<ItemDefId, ItemProperties>,
}
```

## 三、通用组件装配框架

### 3.1 组装的不同形态

WoWorld 的武器不是属性表中的一行——它们是多个参数化组件的装配结果。物品系统提供**通用装配框架**，Combat 和 Magic 注册各自的组件类型：

```rust
/// 物品可以是单件或组合件
pub enum ItemAssembly {
    /// 单件物品（大多数物品——矿石/药水/书籍/货币）
    Simple { def: ItemDefId },

    /// 多组件装配（武器/工具/复杂物品）
    Composite {
        root_ent_id: ItemEntId,        // 装配体的身份（History 追踪此 ID）
        components: Vec<ComponentSlot>, // 子组件列表
        connections: Vec<ComponentConnection>,  // 组件之间的连接方式
    },
}

/// 组件槽——装配体中的一个位置
pub struct ComponentSlot {
    pub slot_id: SlotInstanceId,      // 此槽位的身份（换了 occupant 后槽位身份不变）
    pub slot_type: u16,               // 组件类型标签（Combat/Magic 注册）
    pub occupant: Option<ItemEntId>,  // 占用的组件（自身也是物品）
    pub params: ItemParams,           // 连续参数值——由 ParamSchema 验证（见 003 §四）
    pub param_schema_id: u16,         // 指向 registered ParamSchema 的 ID
}

/// 组件之间的连接方式
pub struct ComponentConnection {
    pub from_slot: usize,
    pub to_slot: usize,
    pub joint: JointType,
}

pub enum JointType {
    Rigid,                                          // 刚性——99% 的武器（柄+刃=一根棍子）
    Hinge { axis: Vec3, limits_deg: (f32, f32) },   // 铰链——双节棍
    Chain { links: u16, link_length_mm: f32 },      // 链连接——连枷/锁镰
    Flexible { stiffness: f32, taper_ratio: f32 },  // 柔性——鞭子/绳镖
}

/// 连续参数——由 Combat/Magic 的组件类型定义其 schema
pub struct ItemParams {
    pub floats: BTreeMap<u16, f32>,  // 参数ID → 值
    pub ints: BTreeMap<u16, u32>,
}
```

### 3.3 投掷武器示例（单件——无装配）

```
手里剑: Simple { def: ItemDefId(ThrownWeapon.Shuriken) }
飞斧:   Simple { def: ItemDefId(ThrownWeapon.ThrowingAxe) }
标枪:   Simple { def: ItemDefId(ThrownWeapon.Javelin) }
```

投掷武器是 `ItemAssembly::Simple`——不需要装配框架。它们的飞行/命中行为由 Combat 模块定义。投掷后物品实体转为 projectile，落地后可被回收（与弹药同理——见 §3.4 弹药生命周期）。

### 3.4 拳套/手部武器（穿戴而非持握）

```
拳套: Composite {
    root: ItemEntId(0xA200),
    components: [
        Slot { slot_type: HandWornBody(Combat注册), occupant: 皮革手套基体, params: { size: M } },
        Slot { slot_type: KnucklePlate(Combat注册), occupant: 铁指节, params: { hardness: 0.8 } },
        Slot { slot_type: EnchantmentSlot(Combat注册), occupant: 雷电符文, params: {} },
    ],
}
```

拳套不占用 Grasp 槽位——它**覆盖**握持槽位对应的手部，与手套/护手共享身体部位。穿戴拳套后仍可握持武器（握持武器时拳套的打击属性暂失效），但空手时拳套的打击效果生效。这为体术战斗提供装备维度。

```
        Slot { slot_type: CrossbowProd(Combat注册), occupant: 钢制弩臂, params: { draw_weight: 350N } },
        Slot { slot_type: CrossbowString(Combat注册), occupant: 钢丝弦, params: { tension: 400N } },
        Slot { slot_type: CrossbowTrigger(Combat注册), occupant: 铁扳机组件, params: { pull_force: 5N } },
        Slot { slot_type: AmmunitionSlot(Combat注册), occupant: None, params: { capacity: 1 } },
    ],

```

**弹药生命周期**——弹药不是"无限箭袋"，是独立物品实体：

```
1. 库存中:   箭 = ItemEntId(0xD001), ItemDefId(Ammunition.Arrow.Broadhead)
2. 装填:     弓.AmmunitionSlot.occupant = Some(0xD001)      // 从库存移到武器槽位
3. 发射:     let arrow = 弓.AmmunitionSlot.take()           // 离开槽位
            spawn_projectile(arrow)                          // 转为飞行实体
4. 命中:     arrow.durability -= 1
            if arrow.durability > 0 { drop_on_ground(arrow) } // 落在地上→可回收
            else { mark_destroyed(arrow) }                    // 折断→无法回收
5. 回收:     玩家/NPC 从地面捡起 → arrow ItemEntId 回到库存
```

投掷武器同理——`Simple` 物品被投掷后转为 projectile，落地后可回收。

> ⚠️ **重要**：弓/弩的 slot_type（BowRiser/CrossbowStock 等）已在物品系统注册，但 [[../战斗/005-招式积木与流派生成|Combat 005]] **尚未补充远程武器的动作基元映射和模块化参数**。远程武器的瞄准/拉弓/松弦/上弦/装填等动作基元、以及弓臂拉重/弓弦张力对射程和伤害的影响公式，均为后续 CHG 待办。

### 3.7 法器示例

**法杖（混种武器）**: 同一 ItemEntId 既是 Combat 武器也是 Magic 法器：

```
法杖: Composite {
    root: ItemEntId(0xC017),  // ← 唯一身份——Combat 和 Magic 都通过此 ID 查询
    components: [
        Slot { slot_type: ImplementBody(Magic注册), occupant: 黑檀杆, params: { length: 1.8m } },
        Slot { slot_type: ImplementCore(Magic注册), occupant: 龙晶核心, params: { purity: 0.95 } },
        Slot { slot_type: ImplementFocus(Magic注册), occupant: 蓝宝石, params: { diameter: 3cm } },
        Slot { slot_type: StrikingHead(Combat注册), occupant: None, params: {} },  // 可选打击部
    ],
}
// Combat 查询: 组件树中有 StrikingHead? → 可近战打击（劈/扫/格挡）
// Magic 查询:   组件树中有 ImplementCore? → 魔力放大系数/元素亲和
// 同一个 ItemEntId, 双模块各自查询自己关心的组件——互不干扰
```

**宝珠（法器，无物理打击）**:

```
宝珠: Composite {
    root: ItemEntId(0xC050),
    components: [
        Slot { slot_type: ImplementFocus(Magic注册), occupant: 完美水晶球, params: { diameter: 10cm, clarity: 0.98 } },
        Slot { slot_type: ImplementCore(Magic注册), occupant: 星辉精华, params: { purity: 0.99 } },
    ],
}
// 宝珠无 ImplementBody 无 StrikingHead——纯法器，不可近战击打
// 但可作为投掷媒介（施法时将魔力投射出去）
```

> ⚠️ **重要**：法器组件类型（ImplementBody/Core/Focus/GrimoireBinding/GrimoirePages）已在物品系统注册，但 [[../战斗/005-招式积木与流派生成|Combat 005]] **尚未补充法器的战斗动作（引导/屏障/增强）**，且 Magic 模块的法器属性计算（魔力放大/元素亲和/施法速度修正）待完全对接。物品系统已预留完整接口——见 [[009-跨模块接口与数据合同|009 §三]]。

### 3.8 为什么用注册制而不是枚举

Item System 不知道"BowLimb"是什么。它只知道"slot_type=0x0201"。Combat 模块注册 `0x0201 = BowLimb`，并提供：
- 参数 schema（可调 float 有哪些？可调 int 有哪些？——见 [[003-物品属性与品质|003 §四 ParamSchema]]）
- 兼容性检查（这个 slot_type 能与哪些父 DefId 搭配？）
- 动作基元派生（从 slot_type → 可用 combat actions）

**新增武器类型不需要修改物品系统。** 加火器？Combat 注册新 slot_type → 物品系统直接支持。魔法引擎？Magic 注册新 slot_type → 同上。框架天然支持连锁武器（九节鞭=8个Chain连接）、柔性武器（流星锤/飞爪=Flexible+StrikingHead）等任何排列组合。
