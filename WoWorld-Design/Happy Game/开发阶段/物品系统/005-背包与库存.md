# 005 — 背包与库存

## 一、设计原则

**槽位为主，负重为安全网。槽位稀缺 → 驱动仓储行为。**

**玩家与 NPC 共用同一套库存代码**——`PersonalInventory`、`CategorizedInventory`、`CharacterWallet` 对玩家和所有 NPC 完全相同。WoWorld 的核心原则"玩家 AI = NPC AI = 同一套代码"在库存层面同样适用。

30 个基础槽位够日常冒险——但要囤积材料、收藏战利品、经营生意，你需要容器、箱子、仓库、马车。这就是经济行为的物质根基。

## 二、随身库存

### 2.1 基础数据模型

```rust
pub struct PersonalInventory {
    /// 物理槽位——统一槽位池
    slots: Vec<InventorySlot>,
    /// 分类索引——类别 → slot 位置（展示层，不影响存储）
    category_index: BTreeMap<ItemCategory, Vec<usize>>,
    /// 当前总重量（克）——缓存，增删时更新
    total_weight_grams: u32,
}

pub struct InventorySlot {
    pub item: ItemEntId,
    pub quantity: u32,          // 堆叠数量
    pub is_favorite: bool,      // 玩家标记为"收藏"——防止误卖/误丢
}

impl PersonalInventory {
    /// 可用槽位总数
    pub fn total_slots(&self) -> u16 {
        self.slots.len() as u16  // 槽位池动态增长（装备容器时增加）
    }

    /// 已用槽位数
    pub fn used_slots(&self) -> u16 {
        self.slots.iter().filter(|s| s.quantity > 0).count() as u16
    }

    /// 添加物品
    pub fn add(&mut self, item: ItemEntId, quantity: u32, 
               props: &ItemProperties, max_weight: f32) -> Result<(), InventoryError> {
        // 1. 尝试堆叠到已有槽位
        if props.stack_size > 1 {
            for slot in &mut self.slots {
                if slot.item == item && slot.quantity < props.stack_size {
                    let space = props.stack_size - slot.quantity;
                    let to_add = quantity.min(space);
                    slot.quantity += to_add;
                    // 如果全部堆完，返回 Ok
                    // 否则继续处理剩余 quantity
                }
            }
        }
        // 2. 需要新槽位
        if self.used_slots() >= self.total_slots() {
            return Err(InventoryError::NoSlots);  // "背包满了"
        }
        // 3. 检查重量
        let added_weight = props.weight_grams * quantity;
        if self.total_weight_grams + added_weight > max_weight {
            return Err(InventoryError::TooHeavy);  // "太重了"
        }
        // 4. 占用空槽位
        // ...
        Ok(())
    }
}
```

```
基础槽位 (30)
+ 军用背囊 (60, 背槽)
+ 皮质信使包 (25, 肩槽)
+ 采集腰包 (15, 左腰)
+ 药剂腰包 (15, 右腰)
+ 麻袋 (40, 左手——非战斗)
+ 工具箱 (30, 右手——非战斗)
─────────────────
= 215 槽位
```

这个人走得很慢（四个移速惩罚叠加），但他**在搬家**——他有 215 槽位是因为他带了六个容器。卸下容器就是普通的 30+60+25+15+15 = 145（日常冒险配置），再卸下背包就是 30 基础。

NPC 同样使用分类——但目的是**加速搜索**而非 UI 展示：
- "饿了" → 搜 Consumable → 找食物
- "要开工" → 搜 Tool → 找锤子
- "给朋友送礼" → 搜 Weapon/Armor/Accessory → 选价值最高的

## 五、容器非战斗规则

- 家具容器只能在家/店里访问——不能随身携带
- 移动仓储可以跟随你——但需要驾驶员/马车夫
- 容器之间转移物品需要时间（体积/重量越大，NPC 搬运越慢）
- 从随身库存放入/取出家具容器——需要 NPC 走到容器旁边

## 六、货币独立于库存

```rust
pub struct CharacterWallet {
    copper: u32,  // u32 最大 ≈ 42 亿铜 ≈ 42 万金——远超任何玩家自然持有量
                   // 不占任何库存槽位——货币是记账单位
}

impl CharacterWallet {
    pub fn to_display(&self) -> String {
        let g = self.copper / 10_000;
        let s = (self.copper % 10_000) / 100;
        let c = self.copper % 100;
        match (g, s, c) {
            (0, 0, _) => format!("{c}铜"),
            (0, _, 0) => format!("{s}银"),
            (_, 0, 0) => format!("{g}金"),
            (0, s, c) => format!("{s}银{c}铜"),
            (g, s, c) => format!("{g}金{s}银{c}铜"),
        }
    }
}
```

## 七、体积折重

每个物品有 `bulk_factor`（见 [[003-物品属性与品质|003]]）。有效负担 = 实际重量 × bulk_factor：

```
铁锭 5kg, bulk=1.0 → 5kg 有效负担 ✓
空木桶 5kg, bulk=8.0 → 40kg 有效负担 ✓（一个人只能扛两三个）
1000根羽毛 10kg, bulk=15.0 → 150kg 有效负担 ✓（体积爆炸）
长剑 2kg, bulk=0.8 → 1.6kg 有效负担 ✓（绑在腰上）
```

不是网格系统——是 O(1) 计算的体积修正。NPC 端完全兼容，无需空间推理。

## 八、性能

- 库存槽位 = Vec 连续内存——搜索 O(n)（n ≤ 215，可忽略）
- 分类索引 = BTreeMap——按类别过滤 O(log n)
- 重量缓存——增删时更新，查询 O(1)
- 100K NPCs 库存全量序列化 ~128MB（平均 80 槽 × 16 bytes = 1,280 bytes/NPC）
- 容器网格也是 Vec——同样高效
