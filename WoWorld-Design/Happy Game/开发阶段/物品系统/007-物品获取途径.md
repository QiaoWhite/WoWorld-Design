# 007 — 物品获取途径

## 一、八条获取路径总览

```
采集 (Gather)    — 植物/矿/猎/渔——Life 模块定义产出规则
掉落 (Loot)      — 击败敌人/翻找尸体——Combat 模块定义掉落表
购买 (Purchase)  — 从 NPC 商人购买——经济系统定义价格
偷窃 (Steal)     — 从 NPC 库存中盗取——犯罪系统定义风险
制作 (Craft)     — 原料→成品——见 006 制造系统
任务 (Quest)     — 任务奖励——任务系统定义
发现 (Discover)  — 箱子/废墟/隐藏宝藏——World Gen 定义放置规则
礼物 (Gift)      — NPC 赠送——NPC 模块定义送礼动机
```

## 二、采集——Life 模块接口

[[../生命/009-植物|植物系统]] 定义了 `HarvestableProduct`：

```rust
// Life 模块的数据合同
struct HarvestableProduct {
    product_type: ProductCategory,   // Wood/Fiber/Food/Medicine/Dye
    yield_base: f32,                 // 基础产量
    harvest_seasons: Vec<Season>,
    regen_days: u16,
    tool_bonus: Option<ToolType>,
}

// 物品系统映射
// ProductCategory → ItemCategory
// Wood → WoodMat, Fiber → FiberMat, Food → Food
// Medicine → OrganicMat (Medicine sub), Dye → OrganicMat (Dye sub)
```

采集流程：Life 触发 harvest → 调用物品系统 `ItemEntIdAllocator.allocate()` 分配实例 ID → 产物进入采集者库存。

**产量公式**：`base_yield × skill_multiplier(0.5-1.5) × tool_bonus`

## 三、战利品掉落——Combat 模块接口

[[../战斗/012-战后过渡与伤势|Combat 012]] 定义战后掉落：

```rust
// Combat 模块调用物品系统
struct CombatLoot {
    items: Vec<(ItemDefId, u32)>,  // 物品模板 + 数量
    drop_probability: f32,         // 0-1
    condition_factor: f32,         // 战斗中消耗对耐久的影响
}
```

掉落物品可能已有 provenance——被击杀的 NPC 的武器，带着它自己的 ItemEntId 和历史。物品系统不关心掉落规则——只负责为每件掉落物品分配/保留 ItemEntId 并放入胜者库存。

## 五、偷窃——犯罪系统接口

```rust
trait TheftTarget {
    fn visible_items(&self, observer: NpcId) -> Vec<ItemEntId>;  // 能被看到的物品
    fn steal_attempt(&mut self, thief: NpcId, target_item: ItemEntId) 
        -> Result<ItemEntId, TheftError>;  // 成功返回物品，失败返回原因

```

物品系统不定义偷窃成功率和风险——那是犯罪系统的职责。物品系统只负责将物品从一个 inventory 移到另一个。

## 七、礼物——NPC 模块接口

NPC 之间送礼（见 [[../NPC活人感模块/NPC活人感开发文档ver2.0|NPC ver2.0]] —— `GiveGift(target_id)` 行动）：

```rust
// NPC 模块调用
fn give_gift(giver: &mut Npc, receiver: &mut Npc, item: ItemEntId) -> Result<(), GiftError> {
    // 1. 从 giver 库存移除
    giver.inventory.remove(item);
    // 2. 放入 receiver 库存
    receiver.inventory.add(item)?;
    // 3. 关系变化由 NPC 模块计算——物品系统不管
    Ok(())
}
```

礼物的审美属性和价值影响收礼者的情绪反应——由 NPC 模块计算。见 [[004-装备系统|004 §五 审美判断]]。

## 九、交互配方表——物理获取的统一入口

### 9.1 设计定位

采集、拆解、屠宰——这些"通过物理原子从世界实体获取物品"的路径，共享同一套 TOML 数据驱动的配方表。配方表定义三件事：

1. **什么实体 × 什么工具 → 执行什么复合原子**
2. **产物由谁解析**（Life / Item / WorldGen）
3. **事后 XP 归属**（可选——不影响物理执行）

**配方表不定义**：技能门槛（物理从 MaterialProperties × AgentSnapshot 涌现）、成功概率（物理原子自然决定）、品质公式（execution_noise 自然调制）。

**和 `CraftingRecipe`（006 制造系统）的关系**：两张独立表。CraftingRecipe 定义材料→产品的制造规则（有 SkillRequirement）。AcquisitionRecipe 定义实体→产物的采集/拆解规则（无硬门槛——物理自然过滤）。

### 9.3 AcquisitionRegistry — 运行时查询

```rust
/// 物品获取配方注册表——启动时从 config/acquisition/*.toml 加载
/// 放在物品系统 crate 内。不新建 crate。不定义新 trait。
pub struct AcquisitionRegistry {
    /// key = (EntityKind, composite_atom) —— O(1) HashMap 查询
    recipes: HashMap<(EntityKind, String), AcquisitionRecipe>,
}

#[derive(Deserialize)]
struct AcquisitionRecipe {
    id: String,
    description: String,
    filter: EntityFilter,
    requirement: ToolRequirement,
    execution: ExecutionRef,
    xp: Option<XpAward>,
    yield_resolver: YieldResolverRef,
}

#[derive(Deserialize)]
struct EntityFilter {
    /// 对于非Item实体: 必填，匹配 entity.entity_kind()
    /// 对于Item实体(PlacedItem/DroppedItem): 应为 None——走 entity.is_item() 分支 + state/assembly 匹配
    kind: Option<EntityKind>,
    tags: Vec<String>,              // EntityIndex::has_tag() 匹配
    /// 仅 entity.is_item()=true 时使用: "Placed" / "Dropped"
    state: Option<String>,
    /// 仅 entity.is_item()=true 时使用: "Composite" / "Simple"
    assembly: Option<String>,
}

#[derive(Deserialize)]
struct ToolRequirement {
    tool_tags: Vec<String>,         // 匹配 ItemProperties.tool_tags
}

#[derive(Deserialize)]
struct ExecutionRef {
    composite_atom: String,         // 引用已注册的复合原子名
}

#[derive(Deserialize)]
struct XpAward {
    skill: Option<String>,          // 固定技能 ID
    skill_derived_from: Option<String>,  // 或从目标派生（如 "material"）
    base: f32,
}

#[derive(Deserialize)]
struct YieldResolverRef {
    resolver: String,               // YieldResolverRegistry 注册名
    params: HashMap<String, String>,
}

impl AcquisitionRegistry {
    /// 从 config/acquisition/ 目录加载全部 TOML 文件
    pub fn load_from_dir(path: &str) -> Result<Self, LoadError> { /* 启动时一次性 */ }

    /// 查询匹配的配方
    /// 返回第一个匹配的配方——或 None（此实体不可被采集/拆解）
    pub fn find(
        &self,
        entity_kind: EntityKind,
        entity_tags: &[String],
        tool_tags: Option<&[String]>,
    ) -> Option<&AcquisitionRecipe> {
        // 1. 按 EntityKind 过滤候选
        // 2. 按 entity_tags 匹配（filter.tags ⊆ entity_tags）
        // 3. 按 tool_tags 匹配（requirement.tool_tags ⊆ tool_tags，或 requirement 为空）
        // 4. 返回第一个完全匹配的配方
        // O(n) where n = 同 EntityKind 的配方数（~10-20）——线性扫描可接受
    }
}
```

### 9.4 YieldResolverRegistry — 产物解析注册制

```rust
/// 产物解析函数注册表——各模块在启动时注册自己的解析器
/// 放在物品系统 crate 内。依赖方向: 注册模块 → 物品系统（不反向）
pub struct YieldResolverRegistry {
    resolvers: HashMap<String, YieldResolverFn>,
}

type YieldResolverFn = fn(
    entity: &EntityId,
    params: &HashMap<String, String>,
    entity_index: &dyn EntityIndex,
    item_registry: &ItemRegistry,
) -> Vec<YieldEntry>;

struct YieldEntry {
    item_def: ItemDefId,
    quality: f32,       // 0-1 基础品质（受 execution_noise 调制后为最终品质）
    quantity: u32,
}

impl YieldResolverRegistry {
    pub fn register(&mut self, name: &str, resolver: YieldResolverFn) {
        self.resolvers.insert(name.to_string(), resolver);
    }

    pub fn resolve(&self, name: &str, /* ... */) -> Option<Vec<YieldEntry>> {
        self.resolvers.get(name).map(|f| f(/* ... */))
    }
}

// === 各模块在启动时注册 ===

// Life 模块注册植物采集解析:
YieldResolverRegistry::register("plant_harvest", |entity, params, entity_index, item_registry| {
    let species = entity_index.plant_species(entity)?;
    let products = species.harvestable.iter()
        .filter(|h| params.get("product_category")
            .map_or(true, |cat| h.product_type.as_str() == cat))
        .map(|h| YieldEntry {
            item_def: item_registry.map_product_category(h.product_type),
            quality: 0.5,  // 基础品质——受 execution_noise 调制
            quantity: (h.yield_base * 100.0) as u32,
        })
        .collect();
    Ok(products)
});

// Item 系统注册装配体分解解析:
YieldResolverRegistry::register("assembly_decompose", |entity, params, entity_index, item_registry| {
    let assembly = item_registry.get_assembly(entity)?;
    let recovery = params.get("recovery_base")
        .and_then(|v| v.parse::<f32>().ok())
        .unwrap_or(0.5);
    let products = assembly.components.iter()
        .filter_map(|slot| slot.occupant)
        .map(|component_id| YieldEntry {
            item_def: item_registry.get_def(component_id)?.def_id,
            quality: 0.5 * recovery,
            quantity: 1,
        })
        .collect();
    Ok(products)
});

// Life 模块注册动物尸体产物解析:
YieldResolverRegistry::register("creature_product", |entity, params, entity_index, item_registry| {
    let species = entity_index.creature_species(entity)?;
    let products = species.products.iter()
        .map(|p| YieldEntry {
            item_def: item_registry.map_product_category(p.product_type),
            quality: 0.5,
            quantity: p.yield_base as u32,
        })
        .collect();
    Ok(products)
});

// World Gen 模块注册矿物/石料/土沙资源解析:
YieldResolverRegistry::register("resource_extract", |entity, params, entity_index, item_registry| {
    let feature = entity_index.terrain_feature(entity)?;
    let resource_type = params.get("resource")
        .ok_or("missing resource param")?;
    let provider = RESOURCE_REGISTRY.get_provider(resource_type)?;
    // ResourceProvider trait (World Gen 003 §六) 定义每种资源的 base_yield_per_break
    Ok(vec![YieldEntry {
        item_def: item_registry.map_resource_to_item(resource_type),
        quality: 0.5,
        quantity: provider.base_yield_per_break(),
    }])
});
```

### 9.5 完整入口汇总——九条获取路径

| # | 入口 | 数据通道 | 物理原子 | 配方表条目 |
|---|------|---------|---------|----------|
| 1 | **采集植物** | Life::HarvestableProduct → ItemEntId | HARVEST / FELL | fell_tree, pick_fruit 等 |
| 2 | **采矿** | WorldGen 矿脉数据 → ItemEntId | MINE | mine_ore |
| 3 | **屠宰动物** | Life::CreatureProduct → ItemEntId | BUTCHER | butcher_carcass |
| 4 | **拆解放置物品** | Item::ComponentTree → ItemEntId | DISASSEMBLE | disassemble_placed_furniture |
| 5 | **剥离建筑附着物** | BuildingComponent.item_id → ItemEntId | DETACH → GRASP → LIFT | salvage_builtin |
| 6 | **战利品掉落** | Combat::LootTable → ItemEntId | —（不走配方表） | — |
| 7 | **购买** | 经济系统 → ItemEntId | —（不走配方表） | — |
| 8 | **偷窃** | 犯罪系统 → ItemEntId | —（不走配方表） | — |
| 9 | **发现/礼物/任务** | 各自模块接口 → ItemEntId | —（不走配方表） | — |

**前 5 条走交互配方表**——有物理原子序列。**后 4 条走原有模块接口**——纯数据操作或非物理获取。

### 9.6 配方表的调用链

```
NPC GOAP: "GatherResources"
  ↓
1. EntityIndex::entities_in_radius(agent.pos, 5m) → 附近实体列表
  ↓
2. 对每个实体: AcquisitionRegistry::find(entity.kind, entity.tags, tool.tags)
  ↓  返回匹配的 AcquisitionRecipe
3. GOAP utility_fn 评估: 预期产物 × 需求权重 → 选择最优实体
  ↓
4. GOAP 执行: 调用 recipe.execution.composite_atom → execute()
  ↓  复合原子内部:
  ├─ AgentSnapshot.force_check → 物理可行性
  ├─ execution_noise_std(proficiency) → 物理精度
  ├─ MaterialProperties 对比 → 涌现行为
  └─ DETACH/STRIKE/CUT... × N → 物理结果
  ↓
5. 复合原子末尾:
  ├─ YieldResolverRegistry::resolve(recipe.yield_resolver) → 产物 ItemEntId(s)
  ├─ Inventory::add(item, quantity)
  └─ Skills::add_xp(recipe.xp.skill, recipe.xp.base, DirectAction)
```

**不经过**：SkillRequirement 检查、手动成功率判定、品质门控。所有门槛从物理涌现。

## 十、所有途径的统一入口

```rust
impl ItemRegistry {
    /// 任何途径创建新物品实例的统一入口
    pub fn create_instance(&self, def_id: ItemDefId, source: AcquisitionSource) -> ItemEntId {
        let ent_id = self.allocator.allocate();
        let state = ItemState::default_for(def_id, &self.definitions);
        self.instances.insert(ent_id, (def_id, state));

        // History 系统注册创建事件（可选——仅对有意义物品）
        if self.definitions.get(def_id).unwrap().rarity >= Rarity::Rare {
            history::record_item_birth(ent_id, def_id, source, current_timestamp());
        }

        ent_id
    }
}

enum AcquisitionSource {
    Gathered { plant_id: EntityId, location: WorldPos },
    Looted { victim: NpcId, battle_id: EventId },
    Purchased { merchant: NpcId, price_copper: u32 },
    Stolen { victim: NpcId, success: bool },
    Crafted { recipe: RecipeId, crafter: NpcId },
    QuestReward { quest_id: QuestId },
    Discovered { container_id: ItemEntId, location: WorldPos },
    Gifted { giver: NpcId, occasion: String },
}
```
