## 五、NPC 模块接口

### 5.1 MaterialId → ItemDefId 迁移

NPC ver2.0 的 `material_inventory: BTreeMap<MaterialId, u32>` 迁移为：

```rust
// 旧
material_inventory: BTreeMap<MaterialId, u32>,
// 新
material_inventory: BTreeMap<ItemDefId, u32>,
```

MaterialId 废弃——直接使用 ItemDefId（Material 类别）。

### 5.2 物品相关行动

```rust
// NPC 行动系统调用物品系统
enum NpcItemAction {
    UseItem(ItemEntId),       // 使用物品（喝药水/读卷轴）
    EquipItem(SlotId, ItemEntId),  // 装备物品
    UnequipItem(SlotId),      // 卸下装备
    GiveGift(NpcId, ItemEntId),   // 送礼
    MoveItem(ItemEntId, ContainerId),  // 物品入库/出库
    CraftItem(RecipeId),      // 制造
    BuyItem(ItemDefId, u32),  // 购买
    SellItem(ItemEntId),      // 出售
}
```

### 5.3 Outfit 自主切换

NPC 模块决定切换时机——物品系统只提供切换的数据操作（见 [[004-装备系统|004 §二]]）。服饰切换见 [[../NPC活人感模块/NPC活人感开发文档ver2.0|NPC ver2.0（决策引擎）]]。

### 5.4 holding_item_type 渲染参数

```rust
// 物品系统提供
fn get_holding_visual(item: Option<ItemEntId>) -> HoldingItemType {
    match item.and_then(|id| registry.get_def(id)) {
        Some(props) => match props.category {
            ItemCategory::Weapon | ItemCategory::Tool => {
                if props.tags.contains(TwoHanded) { HoldingItemType::TwoHanded }
                else { HoldingItemType::OneHanded }
            }
            ItemCategory::Consumable(Food) => HoldingItemType::Food,
            _ => HoldingItemType::Generic,
        },
        None => HoldingItemType::None,
    }
}
```

## 八、性能预算

| 项目 | 数值 | 备注 |
|------|------|------|
| ItemDefId 加载 | ~1MB (~5000 defs) | 启动时一次性 |
| ItemState 每实例 | ~128 bytes | 不含 provenance 链 |
| 库存每 NPC | ~1.6KB | 含装备数据 |
| 100K NPCs 装备+库存存档 | ~160MB | 32GB 内存中 0.5% |
| ItemEntId 分配 | AtomicU64, O(1) | 无锁 |
| 品质/稀有度查询 | O(1) 枚举映射 | 无哈希 |
| 武器属性计算 | 遍历组件树(≤~10 节点) | <1μs |
| 审美判断 | ~200 FP 指令 | 每 NPC 每 5s 周期 |

## 九、与 CHG-013 契约兼容

- 不修改 Physiology/Vitals 派生关系
- 不修改魔力单位（u32 刻）
- 不修改十元素顺序
- 不修改部位伤害渐进模型
- ItemDefId/ItemEntId 是**新增**命名空间——不是替代
- MaterialId/MagicItemId/resource_type 通过映射表桥接——渐进迁移
