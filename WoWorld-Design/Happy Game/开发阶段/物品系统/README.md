# 物品系统（Item System）— README

## 模块定位

物品系统是 WoWorld 四大**地基层模块**（物品/技能/天气/对话）之首——定义游戏中一切可持有、可交换、可装备、可消耗的"东西"的 ID 命名空间、属性规范、装配规则和跨模块接口。

**核心原则**：物品系统是**基础设施**，不是"玩法模块"——它只定义"东西是什么"，不定义"东西怎么用"（那是 Combat/Magic/NPC 的职责）。

## 关键设计决策

| 决策 | 结论 |
|------|------|
| ID 模型 | 两层：ItemDefId（全局模板）+ ItemEntId（存档内实例） |
| 品质/稀有度 | **分离**：Quality（4 档工艺精度）+ Rarity（5 档自然稀缺），独立维度 |
| 装备槽位 | BodyPlan 自动派生——不预定义物种类型 |
| 双套服饰 | Combat Outfit / Civilian Outfit —— NPC 自主决定何时穿哪套，BG3 式 0 成本即时切换 |
| 饰品 | 最多 10 戒指/1 项链/4 手镯/2 耳环/1 头环——不随 Outfit 模式切换 |
| 库存 | 槽位为主（30 基础 + 容器），重量为安全网 |
| 附魔 | 卡槽（可撤换，日常经济）+ 直接（永久，历史锚点），两者可共存 |
| 货币 | 铜/银/金 100:1 进制——独立于库存（CharacterWallet），不占槽位 |
