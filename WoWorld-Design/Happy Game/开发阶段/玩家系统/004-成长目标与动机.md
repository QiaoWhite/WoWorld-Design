## 〇、定位

WoWorld 没有"通关"，没有主线任务。玩家的目标由玩家自己定义。本文档定义**PlayerGoal 系统**——玩家如何在意识层面定方向，角色的 GOAP 如何自主展开执行，以及**无主线哲学下的引导体系**。

---

## 一、PlayerGoal 系统

### 1.1 核心概念

玩家设定一个自然语言目标（"我想去北山看看"），角色在 Auto 模式下自主将其展开为可执行的 GOAP 子目标链。玩家可以随时接管控制权干预或改变方向。

**这是"玩家角色变成可编程 NPC"**——玩家在意识层面定方向，角色的全套心智系统自主执行。他想去北山，于是自己做好计划、筹备粮食和资金、锻造武器和铠甲、招募雇佣兵或邀请伙伴、骑马寻路——这一切在玩家眼皮底下发生。玩家可随时接管、终止、或改定新目标。

### 1.2 数据模型

```rust
struct PlayerGoal {
    id: GoalId,
    description: String,           // 玩家写的自然语言描述
    priority: f32,                 // 0-1，玩家设定
    state: GoalState,              // Draft → Active → Paused / Completed / Abandoned
    created_at: GameTimestamp,
    completed_at: Option<GameTimestamp>,
    // GOAP 可理解的子目标（由角色 GOAP 自动展开，非玩家填写）
    sub_goals: Vec<GoapGoal>,
    // 展开上下文
    expansion_context: GoalExpansionContext,
}

enum GoalState {
    Draft,       // 玩家写了但未激活
    Active,      // 角色的 GOAP 正在推进
    Paused,      // 玩家暂停
    Completed,   // 角色判断已达成
    Abandoned,   // 玩家放弃
}

struct GoalExpansionContext {
    expanded_by: EntityId,           // 哪个角色做的展开
    expanded_at: GameTimestamp,
    expansion_version: u32,          // 重新展开时递增
    // 展开时角色的心智快照（用于解释"他为什么这样理解这个目标"）
    personality_snapshot: BigFiveSnapshot,
    mental_snapshot: MentalAttributesSnapshot,
}
```

### 1.3 PlayerGoal 注入 GOAP

PlayerGoal **不直接覆盖** GOAP 的目标优先级。它进入一个**长期计划槽**：

```
PlayerGoal.priority
       ↓
长期计划槽（LongTermGoalSlot）—— 与需求驱动目标（饥饿/疲劳/安全）平等竞争
       ↓
GOAP 规划器：每轮评估所有 goal 的 (priority × urgency)
       ↓
如果 PlayerGoal 胜出 → 执行其 sub_goals 中的当前项
如果需求目标胜出 → 先吃饱再继续推进 PlayerGoal
```

- **priority 高** ≠ "立刻去做"——角色仍会确保基本生存
- **priority 极高**（如 0.95+）→ 角色可能饿着肚子先赶路——这是玩家意图的体现
- **priority 低**（如 0.2）→ 角色在闲暇时才推进——"有空再去"

### 1.4 Sub-goals 自动展开

**Sub-goals 不是玩家填的——是角色 GOAP 自动展开的。**

同一个 PlayerGoal "去北山看看"，不同角色展开成不同的 sub-goal 链：

| 角色 | 展开结果 |
|------|----------|
| **谨慎的矮人铁匠** | 1.评估距离和路线 2.检查库存食物（不够）→ 3.去市场买三周干粮 4.检查装备耐久（胸甲快坏了）→ 5.打铁修装备 6.去酒馆招募一个护卫 7.出发 |
| **鲁莽的年轻猎人** | 1.评估方向 2.检查箭袋（还有箭）→ 3.直接出发 |
| **城里的商人** | 1.评估路线——选大路不选林间小径 2.筹备商品（顺路做生意）3.雇佣保镖 4.出发 |
| **老练的精灵游侠** | 1.选林间最短路径 2.检查装备（没问题）→ 3.出发。sub_goals 共 3 项——经验丰富，不需多想 |

**展开依据**：目标描述 + 角色的人格（BigFive + CognitiveStyle）+ 角色的当前状态（Vitals/装备/库存/位置）+ 角色对目标区域的了解（大日志）。

**重新展开**：当角色状态发生重大变化（受伤/装备损毁/资金耗尽/得知新信息），GOAP 重新展开 sub_goals。旧版保留在 expansion_version 中。

## 二、技能成长路径

### 2.1 用进废退

技能提升靠反复使用——没有经验值和等级以外的独立成长系统。详见 [[../技能系统/README|技能系统]]。

### 2.2 透明度

技能的成长不为玩家"黑箱"——信息通过以下通道透明化：

| 信息 | 获取方式 |
|------|----------|
| 这个角色擅长什么 | 穿越者：创建时自选。原住民：大日志 + 小精灵（"你似乎对火元素特别敏感"——从 MentalAccess 读取） |
| 练什么能变强 | 大日志自动记录——"今天打了 3 小时铁，感觉手臂更有力了"（力量 +0.002） |
| 技能间的交叉影响 | 菜单中的游戏玩法百科（SkillGroup 跨训练映射说明） |
| 为什么某技能涨得慢 | 小精灵可解释——"你的身体不太适应这个"（PhysicalAccess 低）或"这需要魔力天赋" |

**不设独立"技能树"UI 面板**——技能列表和等级可见（[[006-玩家专属I-O适配层]]），但"怎么练"靠大日志 + 小精灵 + 百科，不是靠点天赋点。

## 三、引导体系

### 3.1 设计哲学：Minecraft 式

玩家进入世界后——世界在运转、NPC 在生活。**没有新手教程关卡、没有任务箭头、没有"下一步该做什么"的弹窗。** 引导从三个来源涌现：

### 3.2 引导来源

| 来源 | 触发方式 | 例 |
|------|----------|------|
| **NPC 的日常对话** | 被动（玩家走路时听到） | "这口井快干了，得找人挖深点" → 玩家可以去帮忙 |
| **角色自身的需求** | Vitals 驱动 | 饿了 → 需要食物 → 采集/购买/偷 |
| **大日志的"可能感兴趣"** | 可选查阅 | 附近有未标记的遗迹、最近有人在招铁匠学徒 |
| **小精灵** | 玩家询问 或 触发式提醒 | "你该找点吃的了" / "这个对手太强了"（详见小精灵系统） |
| **菜单百科** | 玩家主动查阅 | 游戏玩法概念解释（魔法体系/经济原理/技能分类），类似 Minecraft 的配方书 |

### 3.3 小精灵的引导边界

> 小精灵系统完整设计见 [[../小精灵系统/001-小精灵系统总纲]]

小精灵是引导的核心——但她**不给目标、不给任务、不给奖励**：

- ✅ 生存提醒（饥饿/口渴/疲劳/极端天气/危险敌人）——仅触发式
- ✅ 回答玩家主动问的问题（"这附近有铁匠吗"→ 查大日志）
- ✅ 解释世界运作机制（"为什么我的火魔法变弱了"→ "你的灵元素枯竭了"）
- ❌ 不给方向建议（"我该去哪里"→ 不答）
- ❌ 不评价玩家行为
- ❌ 不透露 meta 信息（"按 X 键打开地图"——这是菜单百科的职责）

### 3.4 原住民模式的特有引导

原住民玩家的引导天然存在——角色本来就有工作、有家庭、有义务。铁匠该打铁、农夫该收麦、卫兵该巡逻。玩家切入时角色已经在做某事——这本身就是引导。

### 3.5 穿越者模式的初始体验

穿越者开局——野外随机出生点、一无所有、没有社会关系。最初几分钟到几十分钟的引导：
- **Vitals 驱动**：饿了/冷了 → 这是第一个"任务"
- **最近聚落的物理可达性**：向地平线走——看到炊烟/灯光/路标 → 找到第一个聚落
- **第一个 NPC 对话**：问路/讨水/找工作——NPC 的反应是玩家对这个世界的第一次试探
- **小精灵的初始引导**：角色创建完成后 → 小精灵出现，简短自我介绍，解释基本操作查阅方式

## 四、目标来源全景

玩家目标从四个来源涌现：

| 来源 | 例 | 如何成为 PlayerGoal |
|------|-----|------|
| **物理环境** | 看到远处的塔→想去、发现矿脉→想挖 | 大日志自动记录"发现XX"→ 玩家可将其转为 PlayerGoal |
| **NPC 交互** | 铁匠说"矿里闹怪物"、有人聘请护卫 | 大日志记录对话要点 → 玩家可转为 PlayerGoal |
| **大日志** | 翻到古老记录→想去遗迹验证 | 大日志条目 → 转为 PlayerGoal |
| **自身需求/成长** | 想变强→需要练、想学魔法→找导师 | 玩家自己构思 → 写入 PlayerGoal |
