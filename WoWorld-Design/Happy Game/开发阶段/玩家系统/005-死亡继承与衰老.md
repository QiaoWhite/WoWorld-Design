## 〇、定位

玩家角色和 NPC 走完全相同的死亡流程。本文档定义**玩家专属的死亡后处理**——继承链、技能/关系的不可继承性、大日志的跨代延续、衰老规则。尸体处理、死因记录、死后世界变化——全走 [[../NPC活人感模块/07-生命周期系统/007-死亡与死后|NPC 同一路径]]。

---

## 一、死亡瞬间

### 1.1 和 NPC 相同的死亡流程

- 角色 HP 归零 → 进入濒死窗口（抢救/魔法干预）
- 没人救 → 死亡
- 死因：30 种 DeathCause——和 NPC 完全一致

### 1.2 玩家体验

**如果玩家正在操控该角色**：
- 身体倒地。摄像机留在原地约 3-5 秒（告别间隙）
- 屏幕边缘微暗，一声低沉的安静音效
- 屏幕底部文字：`[角色名] 死亡 · 死因：XX · 地点：XX · 享年：XX 岁`
- 如果另一个角色活着 → 底部提示：`[另一个角色名] 正在 XX。切换？`
- 如果单角色或另一个也已死 → 进入继承流程

**如果玩家正在操控另一个角色**：
- 大日志里多了一条死亡条目——点开日志就看到了
- 会不会悲伤——取决于角色的大五人格 × 情绪 × 两者之间的关系
- 强硬的战士可能沉默，高神经质的恋人可能崩溃——全是涌现，不是系统推送

### 1.3 NPC 的反应

全由 NPC 自主：
- 熟人 → 哀悼、传播消息、遗忘
- 路人 → "又死了一个"
- 债主 → 按继承规则追债
- 朋友 → 可能参加葬礼（如果有）

---

## 二、尸体

和 NPC 的尸体完全一致：

- 物理存在——可搬运/掩埋/焚烧/抢夺
- 按 [[../生命/004-身体状态与生命过程|生命模块规则]] 腐烂/风化
- 多年后，该地点的 AetherImprint 多了一条"一个人死在这里"
- 读树的人可能感受到模糊碎片——不完整、有诗意

---

## 三、继承规则

### 3.1 继承顺序

```
角色死亡
  │
  ├─► 有一代以内血亲？
  │     ├─ 是 → 继承。玩家控制切换到该血亲。
  │     │        被继承的 NPC 保持自己的 Identity/Memory/Relationship。
  │     │        玩家现在用这个人的眼睛看世界。
  │     │
  │     └─ 否 → 继续
  │
  ├─► 两个角色 + 所有血亲都不可用？
  │     ├─ 是 → 弹出大日志选择
  │     │        显示候选人：年龄、生命阶段、职业、社会地位、位置、简要生平
  │     │        玩家选择一个或两个候选人继续游戏
  │     │
  │     └─ 用户拒绝 → 结算
  │                   存档保存
  │                   下次读取时仍弹出选择提示
```

### 3.2 一代以内血亲过滤

```rust
fn one_degree_blood_relatives(deceased: EntityId, world: &World) -> Vec<EntityId> {
    world.relationships_of(deceased)
        .filter(|r| matches!(r.kind,
            RelationshipKind::Parent |
            RelationshipKind::Child |
            RelationshipKind::Sibling))
        .filter(|r| r.is_blood_relation)
        .filter(|r| !r.target.is_dead)
        .map(|r| r.target)
        .collect()
}
```

### 3.3 玩家不扮演儿童的边界

如果候选继承人是婴幼儿——他的 `action_repertoire` 极小（≈ ["cry", "sleep", "eat"]），`planning_horizon = 1`。玩家操控一个婴儿几乎不能做任何事——但这本身就是一种游戏内容。玩家可以在大日志选择中看到"2 岁婴儿"的选项并跳过。

### 3.4 物品继承

和 NPC 继承一致——走 [[../生命/001-生命总纲|生命模块]] 的继承管道：
- 物品 → 血亲 / 债权人 / 同僚 → 无人则留在尸体上
- 货币 → 同上

## 四、技能与关系的不可继承性

### 4.1 核心原则

**新角色是新的人。**

- 旧角色四十年的技能——没了。新角色有自己的 Skills
- 旧角色的人际关系——新角色和那些人**不认识**
- 大日志知道你旧角色的人生——但你现在是另一个人。你记得一切，但你不会那些手艺、朋友不再是你的朋友
- 你可以选择告诉旧角色的朋友"我是他的兄弟"——也可以沉默。系统不强推"你长得像他"

### 4.2 大日志的继承选项

| 选项 | 效果 |
|------|------|
| **继承全部** | 旧角色条目保留，新角色延续同一本日志 |
| **封印** | 旧条目标灰但仍可查——"前世"的记忆还在但被标记为过去 |
| **清除** | 全部移除，新角色从空白大日志开始 |

## 五、双角色全死

- 两个角色都死了 + 都无血亲 → 弹出结算
- 结算：查看大日志回顾一生 → 选新角色 → 或退出
- 存档保留——下次加载时再次弹出继承提示

## 六、孩子的定位

### 6.1 孩子 = Auto NPC

- 玩家角色如果是智能种族 + 满足繁衍条件 → 可以生育
- 孩子是 **Auto NPC（SapientMind）**——有自己独立的人格、技能、属性
- 孩子的 traits = 父母 traits 加权 + 种子随机变异（遗传 × 变异）
- **不因"父母是玩家的角色"而自动成为可控角色**

### 6.2 孩子成为继承候选

- 当父母之一死亡时 → 孩子自然进入继承候选池（一级血亲）
- 玩家可以选择接管，也可以不接管
- 不接管 → 孩子继续做 NPC，过自己的人生

### 6.3 待决定

- 如果两个可控角色都是玩家控制的——两人生的孩子：由谁控制？孩子的成长速度 vs 时间流速？
- 这些留待后代系统正式设计时裁决（生命模块繁衍系统已有框架）

## 七、衰老

### 7.1 和 NPC 一致的衰老

- 玩家角色和 NPC 一样——每年长一岁。`age_years` 递增
- `life_stage` 按种族 `max_lifespan` 自动推进：Young → Adult → MiddleAge → Old → Ancient
- 精灵活 500 人类活 80——年龄体验取决于种族选择

### 7.2 衰老的物理影响（预留框架）

```rust
/// 衰老修正——从 life_stage 派生（预留）
struct AgingModifiers {
    strength_mod: f32,       // Old: ~0.7-0.8, Ancient: ~0.5-0.6
    endurance_mod: f32,      // 耐力下降最明显
    perception_mod: f32,     // 感官迟钝——但 wisdom 可能上升
    spirit_mod: f32,         // 魔力——精灵等长寿种族可能越老越强
    learning_rate_mod: f32,  // 老了学新东西慢
}
```

**设计原则**：
- 衰老不是惩罚——是人生阶段。老铁匠可能体力不如从前，但他的剑因为几十年经验仍然是最好的
- "我练了 40 年手艺因为老了握不稳工具"——这不是 Bug，是人世的一部分
- 不同种族的选择对游戏体验有深远影响——选精灵意味着看几代人类朋友老去

## 八、离线时间处理（待正式设计）

### 8.1 问题

WoWorld 是单机游戏。玩家关闭游戏 → 世界停止。NPC 的"40 年人生"只在玩家在线时推进——实际游戏时间如何定义？

### 8.2 待选方案

| 方案 | 描述 |
|------|------|
| **暂停** | 关闭 = 世界冻结。下次打开时从关闭时刻继续 |
| **摘要推演** | 关闭 = 低精度模拟。关键事件统计（谁死了/谁出生/战争结果）→ 玩家回来时看到变化 |
| **混合** | 短离线暂停（< 一周现实时间）→ 长离线的摘要推演 |
