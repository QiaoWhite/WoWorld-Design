## 核心定位

玩家系统是独立一级模块。WoWorld 的核心哲学是**玩家 = NPC + I/O 适配层**——玩家操控的角色和 NPC 共享同一套 BodyState、RaceTraits、部位伤害、死亡流程、GOAP 心智引擎。玩家系统的职责是定义**玩家如何与这些共享系统交互**。

**本模块不定义**：
- 新实体类型（Player ≠ 独立结构体——Player = SapientMind + ControlMode）
- BodyState/Vitals/RaceTraits → [[../生命/001-生命总纲|生命模块]]
- GOAP/记忆/情绪/关系/BigFive → [[../NPC活人感模块/NPC活人感开发文档ver2.0|NPC活人感]]
- 技能成长（用进废退）→ [[../技能系统/README|技能系统]]
- 魔法/战斗/交易——全部走 NPC 同一代码路径
- UI 具体布局/控件 → [[../UI与UX系统/README|UI/UX系统]]（轨B.1）

## 架构速览

```
玩家系统（本模块）
│
├── 角色创建 ── 7步向导 → LifeEntity + SapientMind 字段直写
│   └── 外观动态 ── 着装可见性 / 伪装 / 身份隐藏（无头衔标签）
│
├── 双角色系统 ── 1-2 个可控角色，随时切换、无距离上限
│   ├── ControlMode: Auto / Manual / DomainDelegated
│   ├── 6 ActionDomain: Movement / Combat / Speech / ItemUse / Interaction / MagicUse
│   ├── 非操控角色 = Auto 模式（完全自主，和任何 NPC 一样）
│   └── 两个角色交互走 NPC-NPC 同一代码路径
│
├── 成长与目标 ── PlayerGoal → GOAP 自动展开 sub_goals（不同程度角色不同展开）
│   ├── 引导: NPC 日常 + Vitals + 大日志 + 小精灵（不给目标）+ 菜单百科
│   └── 技能: 用进废退——透明度由大日志 + 小精灵 + 百科提供
│
├── 死亡与继承 ── 血亲继承 → 大日志选择 → 全新角色
│   ├── 技能/关系不可继承（新角色是新的人）
│   └── 孩子 = Auto NPC（不自动成为可控角色）
│
└── I/O 适配层 ── 输入映射 + 信息展示边界 + 11 系统差异清单
    ├── 输入: ~40 动作分类——手柄+键鼠平等、可全改键
    └── 信息边界: 自身数据✅ / 他人内心❌ / 头衔标签❌
```

## 性能预算

玩家系统不引入计算开销——它定义的是适配层+流程：
- ControlMode 切换：O(1) 枚举赋值
- PlayerGoal sub_goals 展开：复用 GOAP 规划器（已预算 ~0.5ms/帧）
- 输入映射：Godot InputMap 原生——< 0.01ms
- 角色创建：一次性操作——不在帧预算内
- 双角色切换：正常 LOD/加载开销（LODCoordinator 已预算 ≤ 0.05ms）

**玩家系统自身不增加帧级计算**——所有计算由已有模块承担。
