## 二、组装规则

```rust
fn generate_creature(
    biome: &BiomeType,
    history: &BiomeHistory,
    rng: &mut StdRng,
) -> Creature {
    // 1. 群系筛选合法积木候选集
    let candidates = biome.filter_legal_parts(&PARTS_CATALOG);
    // 2. 历史微调权重
    let weighted = history.adjust_weights(candidates);
    // 3. 选择身体模式模板
    let template = select_template(biome, rng);
    // 4. 在模板约束范围内随机采样
    let parts = template.sample_parts(&weighted, rng);
    // 5. 硬约束检查
    validate_hard_constraints(&parts)?;
    // 6. 生态学过滤
    if !ecology_filter(&parts, biome) { return generate_creature(biome, history, rng); }
    // 7. 美学评分→稀有度
    let score = aesthetic_score(&parts);
    // 8. 输出
    Creature::assemble(parts, score_to_rarity(score))
}
```

## 三、四层质量防线

### 第一层：硬约束
类型矛盾（膜翼+羽毛 ✗）、力学矛盾（巨型+细腿 ✗）、功能矛盾（滤食口+掠食者 ✗）——拒绝后重新采样。

### 第二层：生态学过滤
食物检查→防御检查→繁殖可行性→气候适应→生态位重叠。淘汰率 60-80%。

### 第三层：模式模板
~15-20 个身体模式模板（四足兽/鸟/蛇/鱼/昆虫/类人/……）。模板约束比例范围——保证"看起来像生物"。

### 第四层：美学评分
评分维度：比例和谐度 + 色彩一致性 + 功能一致性 + 轮廓可识别性 + 生态位契合。
```
0.8-1.0 → 常见（自然优美）
0.5-0.8 → 常见到不常见（合理）
0.3-0.5 → 不常见（奇特但真实）
0.1-0.3 → 稀有（诡异但存在）
0.0-0.1 → 极稀有（魔法实验失败的产物感）
```

struct PlantSpecies {
    morphology: PlantMorphology,
    biome_affinity: BiomeAffinity,

## 五、关键设计原则

- 积木是**参数化模板**——不是固定3D模型
- 环境约束**收缩空间**——不是所有积木在所有群系合法
- 历史**微调方向**——同一群系在不同种子下产生不同但合理的结果
- 低分≠拒绝——低分=稀有。鸭嘴兽也是真实的
