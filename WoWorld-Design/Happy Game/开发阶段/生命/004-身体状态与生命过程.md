## 一、核心生命值 (Vitals)

所有生命共享——植物除外（植物无 Vitals）。

| 数值 | 范围 | 说明 | 归零后果 | 恢复方式 |
|------|------|------|---------|---------|
| `hp` | 0 ~ max_hp | 生命值 | 濒死状态→短暂抢救窗口→死亡 | 休息/治疗/魔法/药草/食物 |
| `max_hp` | 种族+体质决定 | 上限 | — | 无法超过(除魔法临时提升) |
| `stamina` | 0 ~ max_stamina | 体力——奔跑/战斗/劳作消耗 | 无法行动，需强制休息 | 休息/睡眠 |
| `max_stamina` | 种族+耐久决定 | 上限 | — | — |
| `hunger` | 0.0(极度饥饿) ~ 1.0(饱腹) | 饱腹度 | <0.1→体力恢复停止→<0→开始扣HP→死亡 | 吃食物 |
| `thirst` | 0.0(脱水) ~ 1.0(不渴) | 饮水度 | <0.1→比饥饿更快→持续扣HP→死亡 | 喝水 |
| `fatigue` | 0.0(精力充沛) ~ 1.0(极其疲劳) | 疲劳度 | >0.9→全属性-20%、>1.0→强制昏睡 | 睡眠 |
| `body_temperature` | 种族正常值±耐受范围 | 体温 | 偏离>80%耐受范围→开始扣HP→过低=冻死/过高=热死 | 环境调节/衣物/魔法/躲入建筑 |
| `spirit` | 0 ~ spirit_capacity | 当前灵元素量 | 0→不可逆死亡（无法成亡灵） | 自然恢复/魔力环境/食物/魔力药水 |
| `element_surplus[8]` | 0.0 ~ 1.0 per element | 八元素过剩累积（金木水火土风雷电——不含血和灵） | 无。不处理→微弱不适（情绪引擎 pull） | 主动释放(全清) / 睡眠被动释放(5%/晚) |
| `libido` | 0.0 ~ 1.0 | 性驱力——bio-signal, not survival vital | 无 | 周期自然波动——不能"恢复"，只能等周期 |

**新增字段说明**：
- `element_surplus`：进食后食物元素谱被吸收→血元素合成消耗掉种族配比需要的部分→**多余元素累积**。索引 0-7 对应 金/木/水/火/土/风/雷/电（对齐十元素系统，排除血和灵——两者不参与消化-过剩循环）。详见 [[#十一、元素平衡 (Elemental Balance)|§十一]]
- `libido`：性驱力的周期信号（非生存生命值）。归零不死亡。周期模式由 `RaceTraits.libido_type` 决定——Continuous（持续型）/ Seasonal（季节型）/ EventTriggered（触发型）。详见 [[#十二、性驱力 (Libido)|§十二]]

**消耗速度**：
- 饥饿消耗：种族基础代谢率 × 活动量 × 环境温度修正(寒冷→消耗加快)
- 口渴消耗：高温环境→加速；湿润食物→减缓
- 疲劳累积：体力消耗速度 × (1 - 耐久)
- 体温变化：环境温度 - 体温 = 驱动力 × 外皮隔热性
- 元素过剩累积：随进食事件触发（非时间驱动）——偏科饮食→累积快；均衡饮食→几乎不累积
- libido 周期：绝对时间函数（正弦或 Seasonal 窗口）——非累积衰减，按游戏日查正弦位置

## 二、身体属性 (Physical Attributes)

由种族+体型+历史涌现。全部 0.0-1.0。

| 属性 | 说明 | 影响 |
|------|------|------|
| `strength` | 力量 | 物理伤害/负重上限/推拉物体/破坏障碍 |
| `agility` | 敏捷 | 移动速度/闪避概率/攻击速度/跳跃距离 |
| `constitution` | 体质 | HP上限/耐力恢复速度/疾病抵抗力/温度耐受范围/毒抗 |
| `perception` | 感知 | 视野范围/听觉灵敏度/发现隐藏物体/远程攻击精度/陷阱侦测 |
| `dexterity` | 灵巧 | 精细操作/手工效率/开锁/武器暴击率/施法手势精度 |
| `endurance` | 耐久 | 体力上限/疲劳累积速度减慢/持续劳作/长途旅行耐力/长时间战斗耐力 |

**属性来源**：
```
属性 = 种族基础值(从种族历史涌现)
     + 个体变异(种子随机 ±0.05-0.15)
     + 年龄调整(不同生命阶段有乘数)
     + 伤病/状态效果临时修正
     + 训练(重复使用=缓慢提升——有上限)
```

## 四、魔法/灵元素属性 (Spirit & Magic)

除亡灵外所有生命都有灵元素——含量决定了魔力天赋。

| 属性 | 范围 | 说明 |
|------|------|------|
| `spirit_capacity` | 0.0-1.0 | 灵元素总容量——0=无生命/亡灵、1=神明级 |
| `elemental_affinity[10]` | 每元素 0.0-1.0 | 十元素(金木水火土风雷电血灵)各向亲和力——决定学该元素魔法的效率+威力加成 |
| `magic_power` | 0.0-1.0 | 魔力强度——影响法术伤害/治疗效果/魔法效果幅度 |
| `magic_control` | 0.0-1.0 | 魔力控制——影响法术精度/失败率/魔法失控概率(失控→反噬) |
| `magic_resistance` | 0.0-1.0 | 魔法抗性——减免魔法伤害/抵抗负面魔法效果 |
| `mana_regen_rate` | 0.0-1.0 | 魔力自然恢复速度——在高魔力浓度环境中翻倍 |
| **`max_mana_ke`** | **u32** | **魔力储量上限（刻）——规范整数单位。由 spirit_capacity × magic_power × 种族基数计算** |

**max_mana_ke 的派生公式**：
```
max_mana_ke = (spirit_capacity × magic_power × race_base_mana) as u32
```

- `race_base_mana`：种族魔力基数（人类≈500, 精灵≈800, 矮人≈200, 灵兽≈300-1000, 动物≈0-50）
- **运行时所有魔力运算使用 u32 刻值**——避免浮点精度误差。spirit_capacity/magic_power 仅用于角色生成时的属性计算
- 个体 max_mana_ke 随训练增长（通过缓慢提升 spirit_capacity 和 magic_power 间接提升）
- Magic 模块中的"刻"即为此 u32 整数值——1 刻 ≈ 维持一个基础法术一瞬所需的能量
- 具体储量范围与 Magic 模块一致：学徒 10-50 刻，成熟法师 100-500 刻，传奇数千刻

**灵元素的重要性**：
- spirit_capacity ≥ 0.4 → 才能成为灵兽(可驯服协作)
- spirit_capacity 高 → 施法消耗小、魔力恢复快
- spirit_capacity 低 ≠ 不能施法——只是学得慢、消耗大、效率低
- **灵元素枯竭(spirit=0)→唯一的不可逆死亡——无法成为亡灵——灵魂完全消散。这是 WoWorld 中最彻底的死亡形式。**

### Spirit 消耗模型：常规 vs 过载

**常规消耗**：施法消耗 Mana（从 spirit+blood 提炼）。Mana 耗尽 → 无法施法，但不危及生命。Mana 通过休息/冥想/药剂恢复。

**过载消耗**：Mana=0 后仍强行施法 → 消耗 raw spirit。过载不会立即死亡——先出渐进症状，到 0 才死：

| spirit 剩余比例 | 症状 | 可否施法 |
|----------------|------|---------|
| >50% | 轻微眩晕、灵元素波动感知异常 | 可（但每次施法加速spirit消耗） |
| 30-50% | 视觉模糊、肢体震颤、魔法失控概率+15% | ⚠️ 系统警告——强烈建议停止 |
| 10-30% | 意识模糊、元素反噬（自伤）、记忆暂时混乱 | 🚫 **禁止施法**（系统强制阻断） |
| 0% | **不可逆死亡**——灵魂消散，不能成为亡灵 | — |

过载后 spirit 恢复极慢（比 Mana 恢复慢 10 倍以上）。每次过载在灵-血界面上留下痕迹——累积到一定程度导致 spirit_capacity 永久下降（魔法资质受损）。魔力药剂可恢复 Mana，但**没有任何手段能快速恢复过载消耗的 spirit**——只能靠长期休养。

## 五、身体部位伤害 (Body Part Damage)

由形态模板决定具体部位集。**所有伤害削减数值，所有数值可恢复。无永久性伤害。**

部位伤害仅影响属性数值，不影响战斗动作模组——NPC/玩家不会因为手臂受伤而被迫切换单手动画。这是为了开发可行性的刻意选择：肢体受伤=数值惩罚，而非动作逻辑改变。

### 部位伤害的数值影响

攻击方向+闪避方向→击中部位（O(1)几何查表）。损伤程度分三级，与 `[[012-战后过渡与伤势]]` 的渐进 debuff 模型一致：

| 部位 | 轻伤 (damage<30%) | 中度伤 (30-60%) | 重伤 (>60%) |
|------|-------------------|-----------------|-------------|
| **持械臂** | 精度-3% | 精度-8%/攻速-5% | 精度-15%/攻速-10% |
| **非持械臂** | 几乎无影响 | 格挡稳定性-10% | 格挡稳定性-25% |
| **躯干** | HP恢复速度-10% | HP恢复速度-25% | HP恢复速度-40%/耐力消耗+20% |
| **腿** | 闪避成功率-3% | 闪避成功率-8%/冲刺距离-15% | 闪避成功率-15%/冲刺距离-30% |
| **头** | 专注值衰减+20% | 专注值衰减+40%/视觉感知距离-15% | 专注值衰减+60%/视觉-30%/听觉-15% |
| **尾**（如有） | 平衡微调-5% | 平衡-15%/游泳速度-20% | 平衡-30%/游泳速度-40% |
| **翼**（如有） | 飞行速度-10% | 飞行速度-25%/机动性-20% | 飞行速度-50%/无法精确悬停 |
| **额外肢体** | 同持械臂规则 | 同持械臂规则 | 同持械臂规则 |

均为软数值影响——不阻止行动。中度伤在同一部位累积→可能转为重度伤（需治疗）。不同部位的伤独立计算。

### 部位损伤值存储

每个部位独立存储 `damage: f32` (0.0-1.0，0=完好)。`Physiology.body_part_damage` 持有当前各部位损伤值。战斗结束后损伤持久化到 NPC 数据中——不会自动清零。

### 部位伤害的恢复

自然恢复极慢（数日-数周）/ 药草加速 / 魔法可快速治愈（需魔力消耗）。

### 针对不同身体架构

- 四足兽：左前肢/右前肢/左后肢/右后肢 + 头/躯干/尾
- 二足：左臂/右臂/左腿/右腿 + 头/躯干
- 蛇形：头/前段/中段/后段/尾尖
- 软体：无部位区分——所有伤害打在 pool 上

## 六、生命阶段 (Life Stage)

| 阶段 | 年龄占比 | 属性比例 | 说明 |
|------|---------|---------|------|
| **幼体** (Juvenile) | 0-10% | 15-30% | 需亲代抚育——不能独立生存 |
| **少年** (Adolescent) | 10-25% | 50-70% | 可做简单工作(学徒/帮工)——学习+人格形成 |
| **青年** (YoungAdult) | 25-40% | 90-100% | 可独立——可结婚/生育——职业选择 |
| **成体** (Adult) | 40-70% | 100% | 生产力巅峰——多数繁衍发生在此 |
| **中年** (MiddleAge) | 70-85% | 90-100% | 身体属性微降 -0~10%——经验/智慧继续增长 |
| **老年** (Elder) | 85-100% | 60-90% | 属性持续下降——无法生育——由子女/社区照护 |

## 八、可叠加标签——完整展开

标签不互斥——一个生命可同时有多个标签。

### 8.1 来源/本质标签

| 标签 | 说明 |
|------|------|
| `Natural` | 自然演化产生 |
| `Magical` | 魔法相关——天生魔力/魔法改造/魔力浸染 |
| `Corrupted` | 被扭曲——曾是正常生物→异常特征叠加 |
| `Undead` | 亡灵——灵元素异常残留 |
| `Extraplanar` | 异界/位面来源 |
| `Created` | 人工创造——炼金/魔法实验/神明造物 |
| `Ancient` | 远古幸存物种——比现有生态系统古老 |
| `Legendary` | 传说级——独特个体——世界唯一 |

### 8.2 智能等级标签

| 标签 | 说明 | 适用生命类型 |
|------|------|-------------|
| `Mindless` | 无智能——纯被动/反应式 | 植物/低等亡灵 |
| `Instinctual` | 纯本能——无学习能力 | 低等动物/昆虫型 |
| `Trainable` | 可训练——条件反射+简单学习 | 普通动物 |
| `Intelligent` | 一般智能——理解指令+工具使用+简单问题解决 | 灵兽 |
| `Sapient` | 完全智能——语言+抽象思维+文化+自我认知 | 智能种族 |
| `Divine` | 超越常规认知的智能——全知或近似全知 | 神明(A)/龙 |

| 标签 | 说明 |

## 十一、元素平衡 (Elemental Balance)

> **关联模块**：[[../../NPC活人感模块/03-基本需求系统|NPC/03 基本需求系统]] · [[../../魔法/01-基础层/002-十元素|Magic/002 十元素]] · [[../../物品系统/003-物品属性与品质|Items/003 物品属性]]

### 11.1 设计公理

来自魔法系统十元素理论：**血元素 = 金木水火土风雷电八元素的复合材料**（灵元素不参与消化代谢）。

进食 → 食物携带的元素谱被吸收 → 血元素合成消耗掉"需要的部分"（按种族血元素配比）→ 多余元素累积到 `Vitals.element_surplus[8]`。累积到一定程度 → 有意识地释放（几秒，瞬间完成，无声无形）。睡眠时缓慢被动释放（5%/游戏日）。

### 11.2 瓶颈模型——血元素合成

```
给定:
  food_element_yield[i]  for i in 0..7  (食物携带的元素谱)
  race_blood_ratio[i]    for i in 0..7  (种族血元素的八元素配比, Σratio = 1.0)
  portion: f32                           (进食份量)

步骤 1: 各元素合成血元素的需求量
  needed[i] = race_blood_ratio[i] × portion

步骤 2: 瓶颈计算——最稀缺元素限制合成效率
  bottleneck = min(1.0, min_over_i(absorbed[i] / needed[i]))
  // bottleneck ≤ 1.0
  // =1.0: 所有元素充足，血元素完整合成
  // <1.0: 至少一个元素不足，血元素合成受限

步骤 3: 各元素去向
  used[i]    = needed[i] × bottleneck    // 实际用于合成血元素的
  surplus[i] = absorbed[i] - used[i]     // 正=过剩累积，负=某元素不足
  element_surplus[i] += max(0.0, surplus[i])

步骤 4: 合成效率反哺饥饿
  血元素合成效率 = bottleneck
  效率<1.0 → 同等食物量恢复的饱腹度(Δhunger)打了折扣
  效率=1.0 → 饱腹度正常恢复
```

**涌现意义**：连着吃偏科食物 → 某元素积累过剩 → 同时瓶颈可能来自其他元素不足 → 饥饿恢复效率下降 → 饥饿和元素平衡两种需求在"吃什么"上形成张力。NPC 不存在"连着吃三天肉就腻了"的脚本——是瓶颈模型算出来的。

### 11.3 食物元素谱的三层默认值

```
Level 1 — 手写值 (ItemProperties.consumable.element_profile)
  特殊/魔法/异域食物。例：龙肉{火:0.7, 土:0.1, ...}

Level 2 — 群系默认 (biome_default_element_profile(BiomeId))
  沙漠→偏火土, 海岛→偏水风, 密林→偏木土

Level 3 — 全局默认
  [0.125; 8] — 完全均衡。面包、普通水果、寻常肉类不产生元素过剩
```

### 11.4 释放与睡眠代谢

```rust
// ===== 进食处理 =====
pub fn ingest_food(vitals: &mut Vitals, race: &RaceTraits, food: &ConsumableEffect, portion: f32) {
    // 1. 饱腹恢复 (已有效率乘子)
    let bottleneck = calc_element_bottleneck(food, race, portion);
    vitals.hunger += food.satiation * bottleneck;
    vitals.thirst += food.moisture;
    vitals.hp     += food.vitality_restore;
    vitals.stamina += food.stamina_restore;
    vitals.body_temperature += food.temp_adjust;

    // 2. 元素消化——瓶颈模型
    if let Some(profile) = food.element_profile {
        let mut needed = [0.0f32; 8];
        for i in 0..8 {
            needed[i] = race.blood_element_ratio[i] * portion;
        }
        let bottleneck = calc_bottleneck(&profile, &needed);
        for i in 0..8 {
            let used = needed[i] * bottleneck;
            let surplus = profile[i] - used;
            if surplus > 0.0 {
                vitals.element_surplus[i] += surplus;
                vitals.element_surplus[i] = vitals.element_surplus[i].min(1.0);
            }
        }
    }
    // element_profile == None → 默认均衡 → 不累积过剩
}

// ===== 瓶颈计算 =====
fn calc_bottleneck(absorbed: &[f32; 8], needed: &[f32; 8]) -> f32 {
    let mut min_ratio = 1.0f32;
    for i in 0..8 {
        if needed[i] > 0.0 {
            let ratio = absorbed[i] / needed[i];
            min_ratio = min_ratio.min(ratio);
        }
    }
    min_ratio.min(1.0)
}

// ===== 主动释放 =====
pub fn vent_surplus(vitals: &mut Vitals) {
    vitals.element_surplus = [0.0; 8];
    // v1.0: 不修改环境元素浓度场
}

// ===== 睡眠被动释放 =====
pub fn tick_passive_vent(vitals: &mut Vitals, dt_days: f32) {
    let decay = 1.0 - 0.05 * dt_days;  // 5%/游戏日
    for s in &mut vitals.element_surplus {
        *s *= decay;
    }
}
```

### 11.5 公式与[TUNING]参数

| 参数 | 默认值 | 含义 |
|------|-------|------|
| `passive_vent_rate_per_night` | 0.05 | 睡眠一晚被动释放的比例 |
| `element_surplus_cap` | 1.0 | 单元素过剩累积上限（per-element clamp）|
| `bottleneck_satiation_penalty` | 1.0 × (1-bottleneck) | 瓶颈对饱腹恢复的惩罚——线性 |

### 11.6 边缘情况

| 情况 | 处理 |
|------|------|
| 亡灵 | 无 Vitals——不参与元素平衡 |
| 植物 | 无 Vitals——不参与 |
| 食魔者(Manavore) | 无物理进食——element_surplus 恒为 0 |
| Lithovore | 矿物元素谱极偏——频繁主动释放 |
| NPC 死亡 | `element_surplus = [0.0; 8]` — 归零，不需释放 |
| 时间跳跃 | 若睡眠期间跳时→tick_passive_vent(dt_days)批量处理。清醒期间→不变化 |

### 12.1 三种周期模式

```rust
/// libido 的周期模式——每物种(race)一个, 种子生成时确定
pub enum LibidoType {
    /// 持续型——全年有性欲, 温和正弦波动。基准 0.3 ± 0.3 → 范围 0.0~0.6
    /// 示例: 人类、矮人、精灵
    Continuous,

    /// 季节型——每年仅窗口期内有性欲, 非窗口期归零。窗口期内剧烈波动 0.0~1.0
    /// 示例: 鹿(~60天/年)、某些灵兽
    Seasonal {
        onset_day_of_year: u16,   // 发情期开始的日历日 (0-364)
        duration_days: u16,       // 发情期持续天数
    },

    /// 触发型——默认 0.0。特定天象/魔法浓度/生命阶段跳变
    /// 示例: 龙(特定天象+高魔力浓度)、某些神明
    /// v1.0: 未实现触发逻辑, 恒为 0.0
    EventTriggered,
}
```

### 12.2 周期计算

```rust
/// 计算当前 libido 值 (0.0-1.0)
/// game_day: 当前游戏日数
/// age_factor/hp_factor: 来自 LifeEntity
pub fn compute_libido(
    libido_type: &LibidoType,
    cycle_days: u32,
    game_day: f64,
    age_factor: f32,
    hp_factor: f32,
) -> f32 {
    let raw = match libido_type {
        LibidoType::Continuous => {
            // 温和正弦: 0.3 + 0.3 × sin(2π × day/cycle)
            let phase = (game_day % cycle_days as f64) / cycle_days as f64;
            0.3 + 0.3 * (std::f64::consts::TAU * phase).sin() as f32
        }
        LibidoType::Seasonal { onset_day_of_year, duration_days } => {
            let day_of_year = game_day as u32 % 365;
            if is_in_window(day_of_year, *onset_day_of_year, *duration_days) {
                let progress = (day_of_year - *onset_day_of_year) as f32 / *duration_days as f32;
                0.5 + 0.5 * (std::f32::consts::TAU * progress * 3.0).sin()
                // sin 频率 3× 使窗口期内有多个起伏, 避免单峰
            } else {
                0.0  // 非窗口期——归零
            }
        }
        LibidoType::EventTriggered => 0.0,  // v1.0 恒为 0.0
    };
    (raw * age_factor * hp_factor).clamp(0.0, 1.0)
}
```

### 12.3 各生命阶段的年龄修正（age_factor）

| 生命阶段 | age_factor | 说明 |
|---------|-----------|------|
| 幼体+少年 | 0.0 | 完全不产生 libido |
| 青年 | 0.5-0.8 (阶段内线性) | 初次生成 |
| 成体 | 1.0 | 峰值 |
| 中年 | 0.6-0.9 (阶段内线性) | 缓慢下降 |
| 老年 | 0.0-0.3 (阶段内线性) | 接近零 |

### 12.4 边缘情况

| 情况 | 处理 |
|------|------|
| 亡灵/植物 | 无 Vitals——libido 恒为 0.0 |
| NPC 死亡 | libido = 0.0 |
| 时间跳跃 | 按绝对时间重新计算正弦位置——不向前积分（避免浮点精度累积）|
| age_factor/hp_factor | 由 LifeEntity 提供——不在 libido 模块内部计算 |

## 十三、消耗品效果数据合同 (ConsumableEffect)

> **关联模块**：[[../../物品系统/003-物品属性与品质|Items/003 物品属性]] · [[../../物品系统/009-跨模块接口与数据合同|Items/009 跨模块接口]]

### 13.1 ConsumableEffect —— Life 定义，物品模块存储

```rust
/// Life 模块定义——消费模块定义 schema, 物品模块存储
/// 对齐已有模式: Magic → EnchantmentSchema, Assembly → AssemblySchema
/// 符合 CHG-014 契约: "各模块在此之上叠加"
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ConsumableEffect {
    /// 饱腹恢复量 (0.0 ~ 1.0). 吃掉后 hunger += satiation × bottleneck
    pub satiation: f32,

    /// 水分恢复量 (0.0 ~ 1.0). 吃掉后 thirst += moisture
    pub moisture: f32,

    /// ★ 食物元素谱——None = 默认均衡
    /// 仅包含金木水火土风雷电八元素——血和灵不参与消化-过剩循环
    /// Some([f32; 8]): 手写值
    /// None: 群系默认 → 全局均衡（三层回退, 见 §11.3）
    pub element_profile: Option<[f32; 8]>,

    /// HP 恢复量（绝对值, 如 5.0 HP 恢复）
    pub vitality_restore: f32,

    /// 体力恢复量（绝对值, 如 10.0 体力恢复）
    pub stamina_restore: f32,

    /// 消化时长（游戏秒）. 影响 NPC "多久后再吃"的间隔
    /// 饱腹恢复不是瞬间的——在 duration_seconds 内线性生效
    pub duration_seconds: f32,

    /// 体温瞬时调整（热汤 +0.05, 冰水 -0.03, 常温 0.0）
    pub temp_adjust: f32,
}
```

### 13.2 物品系统注册

```rust
// 物品系统 ItemProperties 新增
pub struct ItemProperties {
    // ... 已有字段 (weight, quality, rarity, durability, 等) ...
    pub consumable: Option<ConsumableEffect>,  // ★ None = 不可食用
}

// 物品系统 ItemRegistry（具体结构体——非 trait）新增固有方法
impl ItemRegistry {
    // ... 已有方法 ...
    /// ★ v1.0 基本需求系统 — 查询消耗品效果
    pub fn get_consumable(&self, id: ItemDefId) -> Option<&ConsumableEffect> {
        self.definitions.get(&id)
            .and_then(|props| props.consumable.as_ref())
    }
}
```

### 13.3 ConsumableEffect 可被任何模块注册扩展

遵循 CHG-014 契约——"各模块在此之上叠加"。当前 Life 模块独占 ConsumableEffect 的定义权（作为生命过程的权威）。未来其他模块可注册新的消耗品效果组件（如 Magic 模块注册 ManaRestoreComponent）。

## 十四、进食事件处理管道

### 14.1 完整管道

```
玩家/NPC 使用食物 ItemDefId
    │
    ▼
物品系统: ItemRegistry::get_consumable(item_id)
    │ → Option<&ConsumableEffect>
    │
    ▼
Life 模块: ingest_food(vitals, race, consumable, portion)
    ├── hunger    += satiation × bottleneck
    ├── thirst    += moisture
    ├── hp        += vitality_restore
    ├── stamina   += stamina_restore
    ├── body_temperature += temp_adjust
    └── element_surplus[i] += max(0, food[i] - race.ratio[i] × portion × bottleneck)

其中 bottleneck = 血元素合成的瓶颈率 (见 §11.2)
```

### 14.2 NPC 决策侧消费

进食后 Physilogy::from_vitals() 自动映射:
- `element_balance_urgency = max(surplus)×0.5 + avg(surplus)×0.5`
- `libido` 直通 (绝对值, 已为 0-1)

NPC 概率决策器通过 `need_action_match()` 消费 urgency (详见 NPC/03 §7)。

### 14.3 向外的信号

| 信号 | 消费方 | 用途 |
|------|--------|------|
| `Vitals.hunger < 0.3` | NPC GOAP | 紧急找食物 |
| `Vitals.element_surplus` | NPC 概率决策器 | VentElements 行动权重 |
| `Vitals.libido` | NPC 概率决策器 | SeekPartner/Groom 行动权重 |
| `ConsumableEffect` (通过 ItemRegistry) | NPC 决策 | 选择吃什么时比较效果 |
| `RaceTraits.blood_element_ratio` | (预留) 魔法模块 | 血元素偏向×法术增益 |
