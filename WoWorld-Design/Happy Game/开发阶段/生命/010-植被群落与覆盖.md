## 〇、核心定位

**植被是植物个体的涌现层。** `[[009-植物]]` 定义了一棵橡树是什么（8维生成、WoodType、PlantSpecies、产物）——那是"原子"。本文档定义千万棵树如何组成一片森林——那是"统计力学"。

关键区分：

| | 009-植物（个体层） | 010-植被群落（群落层） |
|---|---|---|
| **关注点** | 单株植物的生物学属性 | 多物种共存的生态学规律 |
| **核心问题** | "这棵橡树长什么样？" | "这片森林是橡树主导还是松树主导？" |
| **数据结构** | `PlantSpecies`, `PlantInstance` | `PlantCommunitySnapshot`, `VegetationCover` |
| **生成方式** | 8维积木 → 物种定义 | 多层噪声 + 光竞争 + Poisson盘 → 群落快照 |
| **持久化** | T0零存储 / T1-T3稀疏Delta | 群落模板 ~100B/Chunk（LMDB） |
| **对外接口** | `PlantSpeciesRegistry`（Life内部） | `VegetationProvider` trait（woworld_core——跨模块消费） |

## 一、VegetationProvider trait —— 植被的统一对外接口

### 1.1 设计原则：trait inversion（Pattern D）

植被是所有跨模块查询中最频繁的之一。NPC每5秒查20m半径可采集物；天气每降水周期查冠层截留；音频每帧查地面材质——不能让这些消费者依赖Life crate的完整物种表。

遵循 `TerrainQuery`/`SpatialQuery`/`VisibilityQuery`/`EntityIndex` 四trait的先例，`VegetationProvider` 定义在 `woworld_core`，由 `woworld_vegetation` crate实现。

```rust
// woworld_core/src/vegetation.rs

/// 植被世界状态的统一查询接口。
/// 由 woworld_vegetation crate 实现。
/// 消费方：建筑模块、NPC系统、经济系统、天气系统、音频系统、战斗系统、生命(动物)。
pub trait VegetationProvider: Send + Sync {
    /// 查询指定位置的植被群落快照（含物种组成+个体实例列表）
    fn query_community(&self, pos: Vec2, radius: f32) -> PlantCommunitySnapshot;

    /// 冠层郁闭度 0-1
    /// 消费方：天气(冠层截留)、战斗(掩体/隐身)、NPC(森林恐惧)、渲染(阴影)
    fn canopy_closure(&self, pos: Vec2) -> f32;

    /// 木材可获得性
    /// 消费方：建筑模块(BuildContext.materials)、经济系统(市场木材供给)
    fn timber_availability(&self, pos: Vec2) -> TimberAvailability;

    /// 地表覆盖（草/苔藓/落叶/裸土 四通道，和=1）
    /// 消费方：音频(脚步声选择)、NPC(移动速度修正)、渲染(地表纹理)
    fn ground_cover(&self, pos: Vec2) -> GroundCoverMap;

    /// 可采集物列表（T1药草/浆果/蘑菇等）
    /// 消费方：NPC采集行为(NPC每5s查询20m半径)
    fn query_harvestable(&self, pos: Vec2, radius: f32) -> Vec<HarvestableInfo>;

    /// 可燃物密度 0-1（火灾蔓延接口——预留）
    fn fuel_load(&self, pos: Vec2) -> f32;

    /// 根系深度影响因子 0-1（挖掘/采矿难度修正）
    fn root_interference(&self, pos: Vec3) -> f32;
}
```

## 二、VegetationCover —— P2.25 的核心产出

### 2.1 数据结构

`VegetationCover` 是 P2.25 植被覆盖阶段的输出，以 Chunk 为粒度存入 LMDB。

```rust
/// P2.25 产出——每 Chunk 的植被覆盖完整状态
struct VegetationCover {
    /// 群落快照（瞬态生成，但缓存模板以加速）
    community_template: PlantCommunityTemplate,

    /// L6.5 密度场四通道
    density: VegetationDensityField,

    /// 地表覆盖（从群落推导）
    surface: SurfaceMaterialMap,

    /// 木材可获得性（从树种+密度推导）
    timber: TimberAvailability,

    /// 根系密度 0-1（采矿/挖掘难度修正）
    root_density: f32,

    /// 可燃物总密度 0-1
    fuel_load: f32,

    /// 距上次重大干扰的年数（演替阶段纯函数输入）
    years_since_disturbance: f32,

    /// 该Chunk的香农熵（物种多样性）
    shannon_diversity: f32,
}

/// 群落模板——不存储个体实例，存储统计参数
/// 运行时查询时从模板+种子确定性展开为 PlantCommunitySnapshot
struct PlantCommunityTemplate {
    /// 优势种（1-3个，按覆盖度排序）
    dominant_species: Vec<(SpeciesId, f32)>,  // (物种ID, 覆盖权重)

    /// 伴生种（最多10个，低覆盖度）
    companion_species: Vec<SpeciesId>,

    /// 冠层郁闭度 0-1
    canopy_closure: f32,

    /// 垂直分层覆盖
    /// canopy_layer: >15m树的覆盖度
    /// understory_layer: 5-15m树的覆盖度
    /// shrub_layer: 1-5m灌木覆盖度
    /// herb_layer: <1m草本覆盖度（非个体，密度）
    vertical_structure: VerticalLayers,

    /// 物种混合种子——确定性展开为Poisson盘
    mixture_seed: u64,

    /// 演替相关
    years_since_disturbance: f32,
    disturbance_history: Vec<DisturbanceRecord>,
}

struct VerticalLayers {
    canopy_cover: f32,       // 0-1
    understory_cover: f32,   // 0-1
    shrub_cover: f32,        // 0-1
    herb_density: f32,       // 0-1（密度，非个体覆盖）
}

/// L6.5 植被密度场四通道
struct VegetationDensityField {
    canopy_density: Grid2D<f32>,       // 冠层(>15m)
    understory_density: Grid2D<f32>,   // 林下(5-15m)
    shrub_density: Grid2D<f32>,        // 灌木(1-5m)
    groundcover_density: Grid2D<f32>,  // 地被(草/苔藓/落叶)
    // 分辨率: 1m² per cell（与体素密度场对齐）
}
```

### 2.2 双层空间架构：Vegetation Macro-Chunk (VMC)

**核心洞察**：物种组成在32m尺度上几乎不变——一片橡树林跨越数百米。用32m分辨率的渲染Chunk存储物种组成是过度精度。但密度场（冠层/灌木/地被）确实需要高分辨率来驱动渲染/阴影/音频。

**双层架构**：

```
┌─────────────────────────────────────────────────────┐
│  Vegetation Macro-Chunk (VMC): 1km × 1km             │
│                                                       │
│  数量: 75,000 km² 陆地 ÷ 1 km² = 75,000 个            │
│  存储: PlantCommunityTemplate (~100B/VMC)              │
│  内容: 优势种·伴生种·冠层郁闭度·木材品质·演替年数         │
│  生成: P2.25 世界生成时全量预生成（75K VMC）            │
│                                                       │
│  ┌───────────────────────────────────────────────┐   │
│  │  渲染Terrain Chunk (TC): 32m × 32m              │   │
│  │                                                 │   │
│  │  每个VMC含 ~976 个TC                             │   │
│  │  运行时: 查父VMC → 确定性展开                     │   │
│  │    → PlantCommunitySnapshot                     │   │
│  │    → Poisson盘个体 · 垂直分层 · 地面覆盖           │   │
│  │    → L6.5密度场(1m²分辨率, 仅TC内)               │   │
│  │  T0实例: 零存储——从VMC种子+TC坐标推导              │   │
│  └───────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

**为什么是1km**：
- 物种组成在1km尺度上才真正变化（生态学事实——森林斑块的最小可辨识尺度 ~0.5-2km）
- 75,000 km² ÷ 1 km² = 恰好 75,000 个VMC——与文档中"75K"数字一致
- 1km² ≈ 1km×1km正方形——VMC中心间距1km，噪声波长自然匹配

### 2.3 存储预算（修正）

| 数据 | 大小/VMC | × 75K VMC | 总计 |
|------|---------|----------|------|
| `PlantCommunityTemplate` | ~100B/VMC | 75K | **~7.5MB** |
| `TimberAvailability` | ~64B/VMC | 75K | **~4.8MB** |
| 香农熵+演替年数+其他标量 | ~32B/VMC | 75K | **~2.4MB** |
| **P2.25 LMDB总计（结构层）** | | | **~15MB** |

| 数据 | 大小/TC | × 预展开TC | 策略 |
|------|---------|-----------|------|
| `VegetationDensityField` (4×1024 f32) | ~16KB/TC | 仅出生点周围展开 | **不预存储——运行时从VMC生成** |
| `GroundCoverMap` (4×f32) | ~16B/TC | 同上 | **不预存储** |

> **关键**：L6.5密度场和地表覆盖在运行时从VMC模板确定性展开——不持久化存储。VMC模板 ~15MB 是世界生成时写入LMDB的全部植被数据。远距离VMC仅存模板；玩家接近时惰性展开TC。
>
> 出生点周围预展开策略：P2.25 阶段对初始玩家位置周围半径~22km（~500 VMC, 覆盖约1500 km²）预计算Poisson盘个体和L6.5密度场，供初始加载即时渲染。远距离VMC在玩家移动时后台线程展开，每TC ~1.5ms，不在帧预算内。

## 三、群落生成算法

### 3.1 总流程

```
世界种子
  │
  ▼
P2 群系参数场（温度/湿度/土壤/高程/…）
  │
  ▼
┌─────────────────────────────────────────────────┐
│ P2.25 植被覆盖生成                                │
│                                                   │
│ Step 1: 每Chunk优势种筛选                          │
│   PlantSpeciesRegistry.query(params) → candidates  │
│   香农熵加权采样 → 1-3个优势种                       │
│                                                   │
│ Step 2: 多层噪声群落密度                            │
│   第1层: 大尺度密度噪声(~50m) → 密林/空地/过渡      │
│   第2层: 演替阶段噪声(~100m) → 成熟/再生/林窗       │
│   第3层: 物种混合噪声(~30m) → 优势种空间分布         │
│                                                   │
│ Step 3: 垂直分层光竞争                              │
│   冠层→林下→灌木→地被 依序放置                      │
│   每层光照 = f(上层覆盖度)                          │
│                                                   │
│ Step 4: Poisson盘个体放置（仅出生点周围50km²）       │
│   冠层树: 3-5m间距 / 灌木: 1-2m                    │
│   采样域扩展5m → 跨Chunk边界一致                    │
│                                                   │
│ Step 5: 推导输出                                    │
│   冠层郁闭度 → TimberAvailability                  │
│   地被覆盖 → SurfaceMaterialMap                    │
│   垂直结构 → VegetationDensityField                │
│   物种比例 → shannon_diversity                     │
└─────────────────────────────────────────────────┘
```

### 3.2 优势种筛选——香农熵加权

不硬编码"这个群系有3个树种"。用香农熵从候选集中自然决定：

```rust
fn select_dominant_species(
    candidates: &[(SpeciesId, f32)],  // (物种, 适应度评分)
    params: &BiomeSample,
    rng: &mut DeterministicRng,
) -> Vec<(SpeciesId, f32)> {
    // 按适应度排序
    let mut ranked: Vec<_> = candidates.iter()
        .filter(|(_, fitness)| *fitness > 0.3)  // 低于阈值的淘汰
        .collect();
    ranked.sort_by(|a, b| b.1.partial_cmp(&a.1).unwrap());

    // 香农熵决定丰富度
    // H = -Σ(p_i × ln(p_i)) 其中 p_i = fitness_i / Σfitness
    let total_fitness: f32 = ranked.iter().map(|(_, f)| f).sum();
    let entropy: f32 = -ranked.iter()
        .map(|(_, f)| {
            let p = f / total_fitness;
            if p > 0.0 { p * p.ln() } else { 0.0 }
        })
        .sum();

    // 丰富度 = f(熵, 土壤肥力) —— 连续函数，非硬编码整数
    // 高熵+高肥力 → 许多物种共存（热带雨林）
    // 低熵+低肥力 → 单一优势种（泰加林）
    let richness = (entropy * params.soil_fertility * 8.0).ceil() as usize;
    let n = richness.clamp(1, 5);  // 1-5个优势种

    ranked.truncate(n);

    // 权重从适应度归一化
    let weight_sum: f32 = ranked.iter().map(|(_, f)| f).sum();
    ranked.iter()
        .map(|(id, f)| (*id, f / weight_sum))
        .collect()
}
```

**涌现效果**：
- 热带雨林：温度高+湿度高+肥力中高 → 适应度接近的候选多 → 熵高 → 5个优势种共存
- 泰加林：温度低+湿度中 → 只有针叶候选适应度高 → 熵低 → 1-2个优势种
- 过渡带：参数在临界值 → 适应度接近的不同群系候选都有 → 熵高 → 斑块镶嵌

### 3.3 香农熵的涌现意义

```
每个Chunk的 shannon_diversity 不是装饰性统计值——
它直接驱动多个下游系统的行为：

  高熵Chunk (H>1.5):
    · 采集行为: 可采集物种多 → NPC更愿来此采集
    · 生态韧性: 单种病害不致命 → 灾害恢复快
    · 音频: 多种叶声混合 → 环境音复杂度高

  低熵Chunk (H<0.5):
    · 采集行为: 单一物种 → 专业化采集(如果那一种有价值)
    · 生态脆弱: 虫害/火灾 → 整个Chunk可能被摧毁
    · 音频: 单一叶声 → 环境音单调
```

## 四、演替动态——纯函数，非状态标签

### 4.1 核心原则

不在 `PlantCommunitySnapshot` 中存储 `succession_stage: SuccessionStage` 枚举。改为存储 `years_since_disturbance`，演替阶段是纯函数：

```rust
/// 演替阶段 = f(距上次干扰年数, 土壤肥力, 气候)
/// 不是状态标签——是纯函数推导
fn succession_phase(
    years_since_disturbance: f32,
    soil_fertility: f32,
    temperature: f32,
    humidity: f32,
) -> SuccessionPhase {
    // 生长速率因子（年/单位进度）
    let growth_rate = soil_fertility * 0.5
                    + temperature_factor(temperature) * 0.3
                    + humidity * 0.2;

    let effective_years = years_since_disturbance * growth_rate;

    // sigmoid过渡——不是硬阈值
    if effective_years < 5.0 {
        SuccessionPhase::Pioneer       // 灌丛+速生草+幼苗
    } else if effective_years < 30.0 {
        SuccessionPhase::YoungForest   // 幼林——冠层开始闭合
    } else if effective_years < 100.0 {
        SuccessionPhase::MatureForest  // 成熟林——冠层封闭
    } else if effective_years < 300.0 {
        SuccessionPhase::OldGrowth     // 老熟林——大树+林窗+枯立木
    } else {
        SuccessionPhase::AncientWood   // 古老林地——巨树+深层腐殖土+灵元素富集
    }
}

enum SuccessionPhase {
    Pioneer,
    YoungForest,
    MatureForest,
    OldGrowth,
    AncientWood,
}
```

**涌现效果**：
- 同一片森林，玩家50年后重访 → `years_since_disturbance + 50` → 自动从YoungForest过渡到MatureForest
- 不同气候带同一时间 → 热带(growth_rate高)比寒带更早进入OldGrowth
- NPC砍伐一片林 → T3写入disturbance → `years_since_disturbance`重置 → 该Chunk回到Pioneer阶段

## 五、木材形式化合同（WoodMaterialContract）

### 5.1 问题

当前木材从植物生物学到建筑结构经过5个模块的隐式链路，没有单一可验证合同：
```
Life::ProductCategory::Wood → Items::WoodMat(0x24)
→ World Gen::MaterialAvailability → NPC物理原子::MaterialProperties
→ Building::MaterialClass::Wood
```

### 5.2 形式化

**合同权威**：本文档（010-植被群落与覆盖）是木材可获得性的权威Owner。

**合同枢纽**：`woworld_core::VegetationProvider::timber_availability()` 是唯一的木材查询入口。

**合同条款**：

| 条款 | 内容 | 验证方式 |
|------|------|---------|
| **TQ-推导** | `TimberQuality` 不从PlantSpecies直接标注，从形态参数推导：`f(wood_type, max_trunk_diameter, growth_rate, spirit_capacity)` | 单元测试：给定PlantSpecies参数→断言输出的TimberQuality |
| **TA-可获得性** | `TimberAvailability.available = true` 当且仅当 `canopy_closure > 0.1 && 至少一个优势种的wood_type==木本` | 集成测试 |
| **TA-消费方隔离** | Building模块不import `PlantSpecies`、`WoodType`、`StemSpec` | 编译检查（Building crate的Cargo.toml无life依赖） |
| **TA-扩展性** | 新增魔法木材种→只需在woworld_vegetation内添加映射规则；Building、Items、Economy零改动 | 回归测试 |
| **TA-性能** | `timber_availability()` 查询 < 0.001ms（哈希查表，不实时计算） | Benchmark |

### 5.3 TimberQuality 推导函数

```rust
/// 从植物形态参数推导木材品质——零硬编码标注
/// 原则：不在PlantSpecies中加 timber_quality 字段。
///       品质是形态参数的涌现属性。
fn derive_timber_quality(species: &PlantSpecies, density: f32) -> TimberQuality {
    use WoodType::*;

    match species.morphology.stem.wood_type {
        // 草本/藤本/多肉/匍匐 → 不产木材
        Herbaceous | Vine | Succulent | Creeping => {
            return TimberQuality::Softwood; // 降级——只能做纤维
        }

        // 空心(竹型) → 轻质木材
        Hollow => TimberQuality::Softwood,

        // 木本 → 从干径和密度推导
        Woody => {
            let diameter = species.morphology.stem.max_trunk_diameter;
            let spirit = species.spirit.spirit_capacity;

            if spirit > 0.15 {
                // 魔法植物 → 魔法木
                TimberQuality::MagicWood
            } else if diameter > 3.0 {
                // 胸径 > 3m → 巨木级
                TimberQuality::GiantWood
            } else if diameter > 1.5 && density > 0.7 {
                // 粗干+高密度 → 硬木中的顶级（热带硬木级）
                TimberQuality::TropicalHardwood
            } else if density > 0.55 {
                // 中等密度 → 硬木
                TimberQuality::Hardwood
            } else {
                // 低密度 → 软木
                TimberQuality::Softwood
            }
        }
    }
}
```

### 6.2 并查集连通分量分析

```rust
/// 从 VegetationCover 的冠层密度图计算森林斑块连通性
/// 运行时查询时动态计算——不存储预定义区域
fn compute_forest_patches(
    canopy_density: &Grid2D<f32>,
    forest_threshold: f32,  // 冠层密度 > 此值视为"森林"
) -> Vec<ForestPatch> {
    let mut uf = UnionFind::new(canopy_density.width() * canopy_density.height());

    // 4邻域连通（上下左右）
    for y in 0..canopy_density.height() {
        for x in 0..canopy_density.width() {
            if canopy_density[(x, y)] < forest_threshold { continue; }
            let idx = y * canopy_density.width() + x;

            // 右邻
            if x + 1 < canopy_density.width()
                && canopy_density[(x + 1, y)] >= forest_threshold {
                uf.union(idx, idx + 1);
            }
            // 下邻
            if y + 1 < canopy_density.height()
                && canopy_density[(x, y + 1)] >= forest_threshold {
                uf.union(idx, idx + canopy_density.width());
            }
        }
    }

    // 聚合连通分量
    let mut patches: HashMap<usize, Vec<(usize, usize)>> = HashMap::new();
    for y in 0..canopy_density.height() {
        for x in 0..canopy_density.width() {
            if canopy_density[(x, y)] >= forest_threshold {
                let root = uf.find(y * canopy_density.width() + x);
                patches.entry(root).or_default().push((x, y));
            }
        }
    }

    patches.into_iter()
        .map(|(_, cells)| ForestPatch {
            area_cells: cells.len() as u32,
            centroid: compute_centroid(&cells),
            // 不存边界——查询时从连通分量即时计算
        })
        .filter(|p| p.area_cells >= MIN_PATCH_SIZE)  // 过滤过小碎片
        .collect()
}

struct ForestPatch {
    area_cells: u32,         // Chunk格数
    centroid: Vec2,          // 斑块中心
}
```

### 6.3 Voronoi林窗检测

```
每棵树 = Voronoi cell的中心
Voronoi cell面积 = 该树的"势力范围"（光+水+养分竞争半径）

正常密林: Voronoi面积 ≈ 物种典型间距²（σ小）
林窗:      Voronoi面积 >> 典型间距² → 此处应有树但缺失
密丛:      Voronoi面积 << 典型间距² → 竞争过密（自疏将发生）

涌现:
  · 林窗自动检测 → NPC: "林子那边有片空地，野花应该很多"
  · 自疏预测 → 未来数十年该处树木死亡率高于周边
  · 大树周围Voronoi面积大 → 自动识别"这棵是林中之王"
```
