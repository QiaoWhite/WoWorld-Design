## 一、三种存在模式

玩家在创建世界时选择：

| 模式 | 含义 | 游戏表现 |
|------|------|---------|
| **A. 神真实存在** | 神是实体——拥有 DivineMind，可交互 | 神迹是神的主动行为、可与神对话/交易/战斗 |
| **B. 神是信仰建构** | 神只在 NPC 信仰中存在 | 祈祷从无回应——"巧合"由自然/魔法解释 |
| **C. 模棱两可** | 存在无法解释的现象但不下结论 | 随机巧合事件打上"可能是神意"标签 |

## 二、统一 trait

```rust
trait DivinePresence {
    fn respond_to_prayer(&self, npc: &NpcRef, prayer: PrayerType, piety: f32, rng: &mut StdRng) -> PrayerOutcome;
    fn manifest(&self, location: Vec3, observers: &[NpcRef]) -> Option<DivineManifestation>;
    fn influence_world(&self, world: &mut WorldState, rng: &mut StdRng);
    fn mode(&self) -> DivineMode;
}
```

三种实现：`RealGods`(A) / `NoGods`(B) / `AmbiguousGods`(C)。trait 从设计之初就定义，A 实现可最晚交付。

## 三、NPC 信仰系统

完整保留 `NPC信仰系统.md` 的 7 种信仰类型：

单神信仰 / 多神信仰 / 萨满教 / 泛灵论 / 祖先崇拜 / 无神论 / 奇幻特有类型(登神/龙神/魔法即神/元素信仰/世界机)。

每个 NPC 有**虔诚度**(0-1)：狂热→虔诚→普通→冷淡→无信仰。信仰影响 NPC 日常行为、社交、道德判断、战斗意志。

## 四、A 模式：神作为 Life 实体

在 A 模式下，神是 Life 基类的特殊实例：

```rust
struct GodEntity {
    life: LifeEntity,       // DivineMind, 灵元素极高
    domains: Vec<DivineDomain>,  // 概念执掌(战争/农业/死亡/爱情/…)
    pantheon: PantheonId,   // 所属万神殿
    worshippers: Vec<NpcId>,
    manifestation_limit: ManifestationRule,  // 神不能随便降世——有限制
}
```
