## 一、三种存在模式——玩家在世界创建时选择

| 模式 | 含义 | 游戏表现 |
|------|------|---------|
| **A. 神真实存在** | 神是实体——拥有 DivineMind——存在于世界中，可交互 | 神迹是神的主动行为、可与神对话/交易/战斗/侍奉、凡人可登神 |
| **B. 神是信仰建构** | 神只存在于 NPC 信仰中——不存在实体 | 祈祷从无回应、神迹=自然现象被解读为神意、神的问题永远不能被证实或证伪 |
| **C. 模棱两可** | 存在无法解释的现象但不下结论——神的存在与否留给玩家和 NPC 各自判断 | 随机"巧合"事件(刚好下雨/刚好痊愈/刚好躲过危险)可被解读为神意——也可以解释为运气/魔法/自然 |

**三种模式在 NPC 信仰系统中完全相同**——NPC 的信仰行为（祈祷/朝圣/祭祀/节日/禁忌）不受模式影响。区别仅在于**是否真的有实体神在回应**。

## 二、统一 trait——从设计之初定义

```rust
trait DivinePresence: Send + Sync {
    /// NPC 祈祷后是否有回应？
    fn respond_to_prayer(&self, npc: &NpcRef, prayer: PrayerType, piety: f32, rng: &mut StdRng) -> PrayerOutcome;

    /// 神是否在世界中显现？
    fn manifest(&self, location: Vec3, observers: &[NpcRef], rng: &mut StdRng) -> Option<DivineManifestation>;

    /// 神是否干预世界(影响天气/治愈疾病/降下灾祸……)?
    fn influence_world(&self, world: &mut WorldState, rng: &mut StdRng);

    /// 当前模式
    fn mode(&self) -> DivineMode;
}

enum PrayerOutcome {
    NoResponse,                             // B模式——始终返回此
    Coincidence(String),                    // C模式——随机巧合——"刚好下雨了"
    MinorBlessing(StatusEffect),            // A模式——轻微神恩
    MajorMiracle(DivineMiracle),            // A模式——重大神迹(改变天气/治愈绝症/死者短暂回魂)
    DirectCommunication(String),            // A模式——神直接说话/显灵
}
```

**三种实现——同一个 trait**：
- `RealGods`: A模式——神作为 Life 实体存在。有位置、有AI、有自己的目标。回应取决于神的性格+信徒虔诚度+祈祷内容是否符合神的执掌领域
- `NoGods`: B模式——`respond_to_prayer` 始终返回 `NoResponse`。NPC 的信仰行为仅提供心理安慰（+心情/+士气等）
- `AmbiguousGods`: C模式——`respond_to_prayer` 以低概率返回 `Coincidence`。永远不被证实或证伪

## 三、A模式：神作为Life实体

```rust
struct GodEntity {
    life: LifeEntity,           // DivineMind + 灵元素极高(>0.9)
    domains: Vec<DivineDomain>, // 概念执掌——战争/农业/死亡/爱情/知识/海洋/……
    pantheon: PantheonId,       // 所属万神殿
    worshippers: Vec<NpcId>,    // 信徒列表
    manifestation_limit: ManifestationRule,
    // 神不能随便降世——有限制:
    //   - 消耗信仰之力(信徒的虔诚总和)
    //   - 在圣所/神殿/圣地更易显现
    //   - 显现频率低——数年-数十年一次
}

enum DivineDomain {
    War, Harvest, Death, Love, Knowledge, Ocean,
    Smithing, Healing, Justice, Trickery, Nature, Magic,
    // …… 更多 —— 由世界生成时从文化+种子决定
}

enum ManifestationRule {
    FaithPowered,     // 显现消耗信仰之力——信徒越多/越虔诚→越容易显现
    AvatarRequired,   // 需要人类/生物的躯体作为"容器"
    LocationBound,    // 仅在特定圣地/神殿/神域可显现
    RitualRequired,   // 需要特定仪式召唤
    Free,             // 无限制——但古老的神通常不屑于显现
}
```

**A模式关键——凡人登神路径**：
- 后天登神：凡人通过伟大功绩/魔法仪式/神之试炼→升格为神
- 登神不等于立即"无敌"——新神可能比其他神弱得多
- 登神条件极苛刻——可能是世界中的终极目标之一

## 四、NPC信仰系统

完整保留 `想法/WoW World/生命/神明/NPC信仰系统.md` 的完整设计——这是现有文档中唯一详尽的部分：

### 7种信仰类型（基础分配100%）
1. **单神信仰**（创世独神/干预独神/二元独神/泛神倾向独神）——25%
2. **多神信仰**（万神殿型/职能分工型/地域守护型/自然力量型）——30%
3. **萨满教**（万物有灵+灵界旅程+祖灵崇拜+口传传统）——15%
4. **泛灵论**（万物皆有灵+互惠关系+禁忌系统）——10%
5. **混合型**（两种或以上信仰元素的个人化融合——常见于边境/贸易枢纽）——15%
6. **无神论/不可知论**（理性/漠然/反神/实用世俗）——5%

**不占基础比例的叠加类型**：
- **祖先崇拜**——可独立或与其他信仰混合（以家族/氏族为单位），与上述任何类型叠加
- **奇幻特有类型**（登神信仰/龙神信仰/魔法即神/元素信仰/世界机信仰）——额外叠加层，可覆盖在任何基础类型上

### 虔诚度(Piety)——每个NPC的属性
| 虔诚度 | 值 | 表现 |
|--------|-----|------|
| 狂热 | 0.9-1.0 | 每天多次祈祷、严格遵循教规、可能主动传教或迫害异教徒 |
| 虔诚 | 0.7-0.9 | 定期参加仪式、教规是行为的重要参考 |
| 普通 | 0.4-0.7 | 参加主要节日、教规偶尔被考虑 |
| 冷淡 | 0.1-0.4 | 名义上属于该信仰、几乎不参与 |
| 无信仰 | 0.0 | 无宗教信仰 |

### 信仰分配规则
- 政治实体可有国教(官方推行——享受政策优惠)
- 聚落有主流信仰(60-90%居民)——大城市信仰最多元
- 少数信仰共存(10-40%)——偏远村庄信仰最单一
- 边境区域信仰混合——两种信仰的交界处自然形成融合

### 信仰对NPC行为的影响
- 日常行为(祈祷/仪式/饮食禁忌/节律)
- 社交行为(同信仰→好感度加成、异信仰→取决于宽容度)
- 道德判断(信仰是道德共识的重要来源、神权政体的法律从教义派生)
- 战斗意志(A模式——"为神而战"+狂热加成)
- 生育决策(见 [[012-繁衍系统]] §3.1)

## 五、信仰的动态变化

- **传播**: 贸易路线是信仰传播的主渠道、征服/殖民→信仰强制改变、传教士NPC主动传播
- **演变**: 代际漂移(教义微变)、大分裂(神学分歧→分裂为两派)、融合(两种信仰接触→混合)、消亡(信徒过少→消失)
- **与神明的关系**: A模式——神的行为影响信仰变化。B/C模式——信仰变化纯粹由社会动力学驱动
