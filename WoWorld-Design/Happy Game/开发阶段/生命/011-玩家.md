## 一、定位

玩家是智能种族的特例——共享完全相同的生物学基础，但心智不由 GOAP AI 驱动。

```rust
struct Player {
    body: BodyState,          // ← 与同种族 NPC 相同的身体系统
    race: RaceTraits,         // ← 相同的种族参数
    mind: PlayerMind,         // ← 由人类玩家输入驱动
}
```

| 系统 | 玩家 | NPC |
|------|------|-----|
| BodyState (HP/饥饿/疲劳/体温……) | ✅ 共用 | ✅ 共用 |
| RaceTraits (种族参数) | ✅ 共用 | ✅ 共用 |
| 心智层 (GOAP/记忆/情绪) | ❌ 人类直接控制 | ✅ AI 驱动 |
| 战斗 AI | ❌ 半自动——玩家选风格+关键指令 | ✅ 同套 Rust AI |
| 灵兽协作 | ✅ 可捕获/训练/指挥 | ✅ NPC 也可拥有灵兽 |
| 死亡 | 可死亡——死亡后？(见下) | 同系统 |

## 二、玩家死亡

玩家死亡不意味着"读档"——由游戏设计决定（重生/继承者系统/永久死亡模式等——本次不对死亡后处理做最终决定）。

但身体层面：玩家和 NPC 走完全相同的死亡流程——HP归零/濒死/死亡原因记录——无区别。

## 三、半自动战斗中的玩家

战斗系统已定义（`开发阶段/战斗/`）：玩家的战斗 AI = NPC 的战斗 AI = 同一套 Rust 代码。

玩家手动干预：战斗风格切换、特殊技能释放、队伍命令（集火/保护/撤退/散开）、关键决策（投降/谈判/逃跑/追击）、**灵兽指令**。

## 四、灵兽与玩家

玩家可以：捕获灵兽(5种驯服路径)、训练协作技能、建立纽带、灵兽死亡=永久损失。

NPC 同理——NPC 也会拥有和训练自己的灵兽伙伴。

## 五、玩家种族选择

创建角色时可选种族：人类(基准) / 精灵 / 矮人 / 程序化涌现的种族(如果你的世界有)。

所有可选种族共享同一套底层，差异仅在于 RaceTraits——而这些特质**不是手写表**，是从世界生成中该种族的历史涌现的。
