# 004 — 交易主体与角色涌现

## 概述

经济系统提供**客观事实**——订单簿、价差、盈余量。NPC 用自己的心智模型在这些事实上做**主观决策**。

角色不是被"指定"的。农民不是"农民角色"——他是拥有农田技能+生产粮食的 NPC。如果他攒够资本+低神经质+听说了邻镇铁价，他可以成为商人。如果他亏损破产，他不再经商。

这就是**角色涌现**：经济行为从条件中自发产生，没有预设的职业路径。

本文档描述四个核心涌现路径：供给侧（盈余→卖单）、需求侧（匮乏→买单）、中间商（四条件→套利）、以及借贷与店铺经营的特化涌现。

## 一、供给侧涌现——盈余驱动

### 1.1 核心原理

每个 NPC 在每一轮经济心跳中评估自己的库存。如果**持有量 > 自身需求量**，自动产生卖单。不需要"指定农民是粮食供应者"——盈余自动产生供给。

```
供给涌现 = fn(持有量, 自身需求, 个性)
```

### 1.2 盈余计算公式

```rust
/// 计算某个物品的盈余量（可卖出数量）
/// 返回 None 表示无盈余
fn calculate_surplus(
    npc: &NpcState,
    item_id: ItemDefId,
    inventory: &Inventory,
) -> Option<SurplusAssessment> {
    let current_holding = inventory.count(item_id);

    // 日均消耗 = 自身日常使用 + 职业消耗 + 家庭消耗
    let daily_consumption = estimate_daily_consumption(npc, item_id);

    // 储备天数 = 神经质驱动的安全边际
    // neuroticism ∈ [0.0, 1.0]
    // 储备天数 ∈ [3, 17] 天
    let reserve_days = npc.personality.neuroticism * 14.0 + 3.0;

    // 自身需求 = 日均消耗 × 储备天数
    let self_need = daily_consumption * reserve_days;

    // 盈余 = 当前持有 - 自身需求
    let surplus = current_holding as f32 - self_need;

    if surplus <= 0.0 {
        return None;
    }

    // 可卖出数量（取整，至少保留自身需求）
    let sellable_qty = surplus.floor() as u32;
    if sellable_qty == 0 {
        return None;
    }

    Some(SurplusAssessment {
        item_id,
        current_holding,
        daily_consumption,
        reserve_days,
        self_need,
        surplus_quantity: sellable_qty,
        // 卖价由个性+市场参考价共同决定（见 1.4）
        ask_price: determine_ask_price(npc, item_id, sellable_qty),
    })
}
```

### 1.3 储备天数的个性驱动

| neuroticism | 储备天数 | 行为特征 |
|-------------|----------|----------|
| 0.0（极低神经质） | 3 天 | "够用就行，卖了吧" |
| 0.3 | ~7 天 | 适度储备 |
| 0.5（中等） | 10 天 | 标准安全边际 |
| 0.7 | ~13 天 | 偏保守 |
| 1.0（极高神经质） | 17 天 | "万一灾荒呢？再多囤点" |

高神经质 NPC 不仅囤更多——他们挂的卖价也更高（因为卖出意味着降低安全边际，需要更高的价格来补偿心理不适）。

### 1.4 卖价决定

```rust
fn determine_ask_price(
    npc: &NpcState,
    item_id: ItemDefId,
    quantity: u32,
) -> CopperAmount {
    // 基准：取市场参考价（最近成交价中位数）
    let market_ref = get_market_reference_price(item_id, npc.current_region);

    // 个性修正
    let neuroticism_premium = npc.personality.neuroticism * 0.20;  // 最多+20%
    let agreeableness_discount = npc.personality.agreeableness * 0.10; // 最多-10%

    // 稀缺性修正：如果市场上该物品买单多、卖单少，加价
    let scarcity_bonus = calculate_scarcity_bonus(item_id, npc.current_region);

    // 关系修正：对高好感度买家可能给折扣（在撮合阶段体现，此处为挂牌价）
    let modifier = 1.0 + neuroticism_premium - agreeableness_discount + scarcity_bonus;

    (market_ref.as_f32() * modifier.max(0.5).min(2.0)) as CopperAmount
}
```

### 1.5 全局盈余评估

```rust
/// 心跳中调用：评估 NPC 所有物品的盈余，自动挂单
fn assess_all_surpluses(npc: &NpcState, inventory: &Inventory, orderbook: &mut OrderBook) {
    for (&item_id, &qty) in inventory.iter_items() {
        if let Some(surplus) = calculate_surplus(npc, item_id, inventory) {
            // 已有同物品卖单？更新而非重复创建
            if let Some(existing) = orderbook.find_ask(npc.id, item_id) {
                orderbook.update_ask_quantity(existing.order_id, surplus.surplus_quantity);
            } else {
                orderbook.place_order(Order {
                    order_id: OrderId::new(),
                    npc_id: npc.id,
                    item_id,
                    order_type: OrderType::Ask,
                    listing_type: ListingType::Normal,
                    quantity: surplus.surplus_quantity,
                    price_per_unit: surplus.ask_price,
                    placed_at: current_game_time(),
                    expires_at: current_game_time() + ORDER_LIFETIME,
                    region_id: npc.current_region,
                });
            }
        } else {
            // 无盈余 → 撤销已有卖单（如果持有量已低于自身需求）
            orderbook.cancel_asks_for_item(npc.id, item_id);
        }
    }
}
```

## 二、需求侧涌现——匮乏驱动

### 2.1 需求来源分类

匮乏来自三个方向：

| 需求类别 | 触发条件 | 示例 |
|----------|----------|------|
| **生理需求** | 饥饿<0.6、口渴<0.5、健康<0.4 | "饿了→找食物买单" |
| **职业需求** | 工匠缺原料、农民缺种子 | "铁匠缺铁矿石→挂铁矿石买单" |
| **社交需求** | 送礼、奢侈品、地位消费 | "想送花给心上人→挂玫瑰花买单" |

### 2.2 需求评估

```rust
struct NeedAssessment {
    item_id: ItemDefId,
    need_category: NeedCategory,   // Physiological | Occupational | Social
    desired_quantity: u32,
    urgency: Urgency,              // Critical | High | Normal | Low
    max_acceptable_price: CopperAmount,
    reason: NeedReason,
}

enum NeedCategory {

    Occupational,    // 职业驱动

    Critical,   // 生命威胁（饥饿=0→立即以任何价格买入食物）
    High,       // 严重影响（原料耗尽→停产）
    Normal,     // 日常补货
    Low,        // 愿望清单
}
```

```rust
fn assess_physiological_needs(
    npc: &NpcState,
    inventory: &Inventory,
    orderbook: &mut OrderBook,
) {
    // 饥饿检查
    if npc.vitals.hunger < 0.6 {
        let urgency = if npc.vitals.hunger < 0.2 {
            Urgency::Critical
        } else if npc.vitals.hunger < 0.4 {
            Urgency::High
        } else {
            Urgency::Normal
        };

        // 查找可食用物品
        let food_items = inventory.find_missing_edible(npc);
        for food in food_items {
            place_buy_order(npc, food, urgency, orderbook);
        }
    }

    // 口渴、健康同理...
}

fn assess_occupational_needs(
    npc: &NpcState,
    inventory: &Inventory,
    orderbook: &mut OrderBook,
) {
    // 获取 NPC 当前正在进行的生产计划
    let pending_recipes = npc.get_pending_crafting_recipes();

    for recipe in pending_recipes {
        for ingredient in recipe.required_ingredients {
            let current = inventory.count(ingredient.item_id);
            let needed = ingredient.quantity;

            if current < needed {
                let shortage = needed - current;
                let urgency = if current == 0 {
                    Urgency::High  // 完全缺料→停产
                } else {
                    Urgency::Normal
                };

                place_buy_order(npc, NeedAssessment {
                    item_id: ingredient.item_id,
                    need_category: NeedCategory::Occupational,
                    desired_quantity: shortage,
                    urgency,
                    max_acceptable_price: calculate_max_buy_price(npc, ingredient),
                    reason: NeedReason::CraftingIngredient {
                        recipe_id: recipe.id,
                        product: recipe.output_item_id,
                    },
                }, orderbook);
            }
        }
    }
}

fn assess_social_needs(
    npc: &NpcState,
    inventory: &Inventory,
    orderbook: &mut OrderBook,
) {
    // 检查送礼计划
    for gift_intent in npc.social.get_pending_gift_intents() {
        if inventory.count(gift_intent.item_id) < 1 {
            place_buy_order(npc, NeedAssessment {
                item_id: gift_intent.item_id,
                need_category: NeedCategory::Social,
                desired_quantity: 1,
                urgency: Urgency::Low,
                max_acceptable_price: gift_intent.budget,
                reason: NeedReason::Gift {
                    recipient_id: gift_intent.target_id,
                    occasion: gift_intent.occasion.clone(),
                },
            }, orderbook);
        }
    }

    // 奢侈品/地位消费：由 openness + extraversion 驱动
    if npc.personality.openness > 0.6 && npc.wallet.spare_copper() > LUXURY_THRESHOLD {
        let luxury_item = select_luxury_desire(npc);
        if let Some(item) = luxury_item {
            place_buy_order(npc, NeedAssessment {
                item_id: item,
                need_category: NeedCategory::Social,
                desired_quantity: 1,
                urgency: Urgency::Low,
                max_acceptable_price: npc.wallet.spare_copper() / 4,
                reason: NeedReason::LuxuryDesire,
            }, orderbook);
        }
    }
}
```

## 三、中间商涌现——四条件模型

### 3.1 核心原理

商人不是预设职业。任何一个 NPC 在满足四个条件后，可能**自发开始套利**。套利成功→trader_state 积累→身份自我强化→更频繁地寻找套利机会。

```
商人涌现 = fn(信息, 资本, 心理, 物流) → 套利行为 → 角色强化
```

### 3.2 四个涌现条件

```rust
/// 判断一个 NPC 是否具备成为商人的条件
fn can_become_trader(npc: &NpcState, economy: &EconomyState) -> TraderEligibility {
    let mut reasons: Vec<TraderBlockReason> = Vec::new();

    // ---- 条件 1：信息条件 ----
    // 至少知道两个市场的价格（直接知道或通过社交网络间接知道）
    let known_markets = economy.get_markets_known_by(npc.id);
    if known_markets.len() < 2 {
        reasons.push(TraderBlockReason::InsufficientMarketKnowledge {
            known: known_markets.len(),
            required: 2,
        });
    }

    // ---- 条件 2：资本条件 ----
    // 闲钱 ≥ 5000 铜 [TUNING]
    let spare_copper = npc.wallet.spare_copper();
    const MIN_TRADER_CAPITAL: CopperAmount = 5000;
    if spare_copper < MIN_TRADER_CAPITAL {
        reasons.push(TraderBlockReason::InsufficientCapital {
            have: spare_copper,
            need: MIN_TRADER_CAPITAL,
        });
    }

    // ---- 条件 3：心理条件 ----
    // trader_score = (1-neuroticism)×0.6 + openness×0.4
    // neuroticism 低 → 敢于冒险；openness 高 → 愿意尝试新事物
    let trader_score = (1.0 - npc.personality.neuroticism) * 0.6
                     + npc.personality.openness * 0.4;
    const MIN_TRADER_SCORE: f32 = 0.4;
    if trader_score < MIN_TRADER_SCORE {
        reasons.push(TraderBlockReason::PersonalityUnsuitable {
            trader_score,
            required: MIN_TRADER_SCORE,
        });
    }

    // ---- 条件 4：物流条件 ----
    // 旅行能力 ≥ 0.3（能安全往返两个市场）
    let travel_capability = assess_travel_capability(npc);
    const MIN_TRAVEL_CAPABILITY: f32 = 0.3;
    if travel_capability < MIN_TRAVEL_CAPABILITY {
        reasons.push(TraderBlockReason::InsufficientTravelCapability {
            capability: travel_capability,
            required: MIN_TRAVEL_CAPABILITY,
        });
    }

    if reasons.is_empty() {
        TraderEligibility::Eligible {
            trader_score,
            known_markets,
            spare_capital: spare_copper,
            travel_capability,
        }
    } else {
        TraderEligibility::NotEligible { reasons }
    }
}
```

### 3.3 条件阈值说明

| 条件 | 阈值 | 来源 | [TUNING] |
|------|------|------|----------|
| 信息 | ≥2 个已知市场 | 社交网络+旅行经历+传言 | 可调 |
| 资本 | ≥5000 铜闲钱 | 个人积蓄+继承+偶然财富 | 可调 |
| 心理 | trader_score ≥ 0.4 | 大五人格（neuroticism, openness） | 可调 |

### 3.4 套利扫描

```rust
/// 扫描所有已知市场之间的价差，寻找套利机会
fn scan_arbitrage_opportunities(
    npc: &NpcState,
    economy: &EconomyState,
) -> Vec<ArbitrageOpportunity> {
    let known_markets = economy.get_markets_known_by(npc.id);
    let mut opportunities = Vec::new();

    // 两两市场比较
    for i in 0..known_markets.len() {
        for j in (i + 1)..known_markets.len() {
            let market_a = known_markets[i];
            let market_b = known_markets[j];

            // 获取两个市场的价格快照
            let prices_a = economy.get_price_snapshot(market_a);
            let prices_b = economy.get_price_snapshot(market_b);

            for (&item_id, &price_a) in prices_a.iter() {
                if let Some(&price_b) = prices_b.get(&item_id) {
                    // 价差
                    let spread = price_a as i64 - price_b as i64;
                    let abs_spread_pct = (spread.abs() as f32)
                        / (price_a.min(price_b) as f32).max(1.0);

                    // 估算往返运费
                    let travel_cost = estimate_round_trip_cost(
                        npc, market_a, market_b
                    );

                    // 套利利润 = |价差| - 运费
                    let profit_per_unit = abs_spread_pct * price_a.min(price_b) as f32
                        - travel_cost.per_unit;

                    if profit_per_unit > 0.0 {
                        // 有套利空间
                        let (buy_from, sell_to) = if price_a < price_b {
                            (market_a, market_b)
                        } else {
                            (market_b, market_a)
                        };

                        // 考虑可运输量和资本限制
                        let max_affordable_qty = npc.wallet.spare_copper()
                            / price_a.min(price_b).max(1);
                        let transportable_qty = assess_transport_capacity(npc, item_id);

                        let trade_qty = max_affordable_qty
                            .min(transportable_qty)
                            .min(MAX_SINGLE_TRADE_QTY);

                        if trade_qty > 0 {
                            opportunities.push(ArbitrageOpportunity {
                                item_id,
                                buy_from,
                                sell_to,
                                buy_price: price_a.min(price_b),
                                sell_price: price_a.max(price_b),
                                spread_pct: abs_spread_pct,
                                profit_per_unit,
                                max_trade_quantity: trade_qty,
                                estimated_travel_cost: travel_cost,
                                estimated_total_profit: profit_per_unit
                                    * trade_qty as f32,
                            });
                        }
                    }
                }
            }
        }
    }

    // 按总利润排序，选最优机会
    opportunities.sort_by(|a, b| {
        b.estimated_total_profit
            .partial_cmp(&a.estimated_total_profit)
            .unwrap_or(std::cmp::Ordering::Equal)
    });

    opportunities
}
```

### 3.5 执行套利计划

```rust
fn execute_arbitrage_plan(
    npc: &NpcState,
    opportunity: &ArbitrageOpportunity,
    economy: &mut EconomyState,
) -> TradeResult {
    // 1. 在低价市场挂买单
    let buy_order_id = economy.place_order(Order {
        order_id: OrderId::new(),
        npc_id: npc.id,
        item_id: opportunity.item_id,
        order_type: OrderType::Bid,
        listing_type: ListingType::Normal,
        quantity: opportunity.max_trade_quantity,
        price_per_unit: opportunity.buy_price,
        placed_at: current_game_time(),
        expires_at: current_game_time() + ORDER_LIFETIME,
        region_id: opportunity.buy_from,
    });

    // 2. 等待撮合（在后续心跳中完成）
    //    → 买单成交 → 物品进入 NPC 库存

    // 3. NPC 旅行到目标市场
    let travel_plan = TravelPlan {
        npc_id: npc.id,
        from_region: opportunity.buy_from,
        to_region: opportunity.sell_to,
        departure_time: current_game_time(),
        estimated_arrival: current_game_time()
            + opportunity.estimated_travel_cost.duration,
        cargo: vec![(opportunity.item_id, opportunity.max_trade_quantity)],
        purpose: TravelPurpose::Trade,
    };
    economy.register_travel_plan(travel_plan);

    // 4. 到达后在目标市场挂卖单
    //    （在到达时触发，非立即）

    TradeResult::PlanCreated {
        buy_order_id,
        travel_plan_id: travel_plan.id,
        estimated_profit: opportunity.estimated_total_profit,
    }
}
```

### 3.6 商人身份强化

```rust
/// 商人状态的积累与退化
struct TraderState {
    /// 累计成功套利次数
    successful_trades: u32,
    /// 累计利润（铜）
    total_profit: CopperAmount,
    /// 套利活跃度（每次成功+0.05，每天不交易-0.02）
    activity_score: f32,
    /// 已知的有利路线
    known_profitable_routes: Vec<ProfitableRoute>,
    /// 是否自我认同为"商人"
    self_identifies_as_trader: bool,
}

impl TraderState {
    /// 每次成功套利后调用
    fn on_successful_trade(&mut self, profit: CopperAmount, route: ProfitableRoute) {
        self.successful_trades += 1;
        self.total_profit += profit;
        self.activity_score = (self.activity_score + 0.05).min(1.0);

        // 更新或添加有利路线
        self.update_route(route);

        // 当 activity_score > 0.5 时，NPC 开始自认为是"商人"
        if self.activity_score > 0.5 && !self.self_identifies_as_trader {
            self.self_identifies_as_trader = true;
            // 这可能影响 NPC 的自我描述、社交互动中的自我介绍等
            // 例：NPC 可能在对话中说"我是个生意人"
        }
    }

    /// 每日衰减
    fn daily_decay(&mut self) {
        self.activity_score = (self.activity_score - 0.02).max(0.0);

        // 长期不交易 → 失去商人身份
        if self.activity_score < 0.2 {
            self.self_identifies_as_trader = false;
        }
    }
```

## 四、借贷——特化交易

### 4.1 借贷的经济学本质

贷款 = 用欠条+抵押物 赊购铜币。还贷 = 买回欠条。

借贷不需要"银行系统"。任何有铜币+愿意承担风险的 NPC 都可以放贷。借贷从交易中自然涌现——当买方缺铜币但有信誉/抵押时，卖方可能接受"先拿货、后付钱"。

### 4.2 IOU（欠条）结构

IOU 是物品系统中的一个 ItemDefId，属于 `ItemCategory::Financial`。

```rust
/// IOU（I Owe You）—— 欠条
/// 这是一个可交易的物品实体
#[derive(Debug, Clone)]
struct IouContract {
    /// 物品实体 ID（在物品系统中注册）
    entity_id: ItemEntId,
    /// 指向 IOU 模板的 ItemDefId
    def_id: ItemDefId,  // ItemCategory::Financial

    // --- 借贷双方 ---
    /// 借款人
    borrower_id: NpcId,
    /// 放贷人
    lender_id: NpcId,

    // --- 财务条款 ---
    /// 本金（铜）
    principal: CopperAmount,
    /// 年化利率（小数，0.05 = 5%）
    annual_interest_rate: f32,
    /// 还款期限（游戏内天数）
    due_date: GameTime,
    /// 已还金额
    repaid_amount: CopperAmount,

    // --- 抵押物 ---
    /// 抵押物列表（物品实体 ID）
    collateral: Vec<CollateralEntry>,

    // --- 状态 ---
    status: LoanStatus,
    created_at: GameTime,
}

#[derive(Debug, Clone)]
struct CollateralEntry {
    item_entity_id: ItemEntId,
    /// 抵押物估值（放贷时双方协商/评估）
    estimated_value: CopperAmount,
    /// 是否已被放贷人没收（违约后）
    forfeited: bool,
}

#[derive(Debug, Clone, PartialEq, Eq)]
enum LoanStatus {
    Active,          // 还款中
    Repaid,          // 已还清
    Defaulted,       // 违约
    Forgiven,        // 放贷人免除
    Sold,            // 欠条已转卖
```

### 4.3 放贷决策——完全由放贷人心智驱动

```rust
/// 放贷人评估是否接受借款请求
fn evaluate_loan_request(
    lender: &NpcState,
    borrower: &NpcState,
    request: &LoanRequest,
    economy: &EconomyState,
) -> LoanDecision {
    // --- 宜人性影响利率和抵押要求 ---
    // 高宜人性 → 低利率、对熟人免抵押
    let agreeableness_factor = lender.personality.agreeableness;

    // 利率调整：宜人性高 → 0.03（3%），宜人性低 → 0.20（20%）
    let base_rate = 0.10; // 基础 10% 年化
    let interest_rate = base_rate * (2.0 - agreeableness_factor).max(0.8);

    // 抵押要求：对熟人+高宜人性 → 可能免抵押
    let relationship = economy.get_relationship(lender.id, borrower.id);
    let requires_collateral = if relationship.friendship > 0.7 && agreeableness_factor > 0.7 {
        false  // "老朋友了，不用抵押"
    } else {
        true
    };

    // --- 神经质影响风险承受 ---
    // 高神经质 → 要求更多抵押（抵押物估值 ≥ 本金 × 1.5）
    let collateral_ratio = 1.0 + lender.personality.neuroticism * 1.5;
    // neuroticism=0.3 → ratio=1.45（适度抵押）
    // neuroticism=0.9 → ratio=2.35（极高抵押要求）

    // --- 智慧影响借款人评估 ---
    let intelligence_factor = (lender.intelligence().unwrap_or(0.5) as f32);

    // 评估借款人还款能力
    let repayment_capacity = assess_repayment_capacity(borrower, request.amount);
    let borrower_reliability = estimate_borrower_reliability(
        borrower, lender, economy
    );

    // 靠谱程度 = 还款能力×0.5 + 历史信用×0.3 + 智慧修正×0.2
    let trustworthiness = repayment_capacity.score * 0.5
        + borrower_reliability.credit_score * 0.3
        + intelligence_factor * 0.2;

    // --- 决策 ---
    if trustworthiness < 0.3 {
        return LoanDecision::Reject {
            reason: format!("借款人还款能力不足（评估分 {:.2}）", trustworthiness),
        };
    }

    let max_lendable = (lender.wallet.spare_copper() as f32 * 0.5) as CopperAmount;
    // 不放超过闲钱的 50%

    let approved_amount = request.amount.min(max_lendable);

    if requires_collateral {
        let min_collateral_value = (approved_amount as f32 * collateral_ratio) as CopperAmount;
        let offered_value = request.collateral.iter()
            .map(|c| c.estimated_value)
            .sum::<CopperAmount>();

        if offered_value < min_collateral_value {
            return LoanDecision::Reject {
                reason: format!(
                    "抵押物不足：需要 ≥{} 铜，实际 {} 铜",
                    min_collateral_value, offered_value
                ),
            };
        }
    }

    LoanDecision::Approve {
        approved_amount,
        interest_rate,
        due_date: current_game_time() + request.proposed_duration,
        requires_collateral,
        collateral_entries: request.collateral.clone(),
    }
}
```

### 4.4 发放贷款

```rust
fn issue_loan(
    economy: &mut EconomyState,
    lender_id: NpcId,
    borrower_id: NpcId,
    decision: LoanDecision,
) -> Result<IouContract, LoanError> {
    let LoanDecision::Approve { approved_amount, interest_rate, due_date, .. } = &decision else {
        return Err(LoanError::NotApproved);
    };

    // 1. 放贷人出铜币
    let lender = economy.get_npc_mut(lender_id)?;
    lender.wallet.withdraw(*approved_amount)?;

    // 2. 借款人得铜币
    let borrower = economy.get_npc_mut(borrower_id)?;
    borrower.wallet.deposit(*approved_amount);

    // 3. 创建 IOU 物品实体
    let iou = IouContract {
        entity_id: economy.item_registry.allocate_entity_id(),
        def_id: IOU_ITEM_DEF_ID,  // 物品系统中预定义的 IOU 模板
        borrower_id,
        lender_id,
        principal: *approved_amount,
        annual_interest_rate: *interest_rate,
        due_date: *due_date,
        repaid_amount: 0,
        collateral: decision.collateral_entries.clone(),
        status: LoanStatus::Active,
        created_at: current_game_time(),
    };

    // 4. IOU 存入放贷人库存（放贷人"持有"欠条）
    let lender = economy.get_npc_mut(lender_id)?;
    lender.inventory.add_item(iou.entity_id, 1);

    // 5. 记录到经济系统
    economy.iou_registry.register(iou.clone());

    Ok(iou)
}
```

### 4.5 还款

```rust
fn repay_loan(
    economy: &mut EconomyState,
    iou_entity_id: ItemEntId,
    amount: CopperAmount,
) -> Result<RepayResult, LoanError> {
    let iou = economy.iou_registry.get_mut(iou_entity_id)?;

    if iou.status != LoanStatus::Active {
        return Err(LoanError::LoanNotActive);
    }

    // 计算应收（本金+已产生利息）
    let elapsed_days = days_between(iou.created_at, current_game_time());
    let accrued_interest = calculate_interest(
        iou.principal - iou.repaid_amount,
        iou.annual_interest_rate,
        elapsed_days,
    );
    let total_owed = (iou.principal - iou.repaid_amount) + accrued_interest;

    // 还款
    let borrower = economy.get_npc_mut(iou.borrower_id)?;
    let actual_payment = amount.min(total_owed);
    borrower.wallet.withdraw(actual_payment)?;

    // 铜币转给放贷人
    let lender = economy.get_npc_mut(iou.lender_id)?;
    lender.wallet.deposit(actual_payment);

    iou.repaid_amount += actual_payment;

    // 是否还清？
    if iou.repaid_amount >= iou.principal + accrued_interest {
        iou.status = LoanStatus::Repaid;

        // 归还抵押物
        for collat in &iou.collateral {
            // 从放贷人库存归还
            let lender = economy.get_npc_mut(iou.lender_id)?;
            lender.inventory.remove_item(collat.item_entity_id, 1);
            let borrower = economy.get_npc_mut(iou.borrower_id)?;
            borrower.inventory.add_item(collat.item_entity_id, 1);
        }

        // IOU 物品销毁（已还清，欠条失效）
        economy.item_registry.deallocate_entity(iou_entity_id);

        return Ok(RepayResult::FullyRepaid {
            total_paid: iou.repaid_amount,
            interest_paid: accrued_interest,
        });
    }

    Ok(RepayResult::PartiallyRepaid {
        paid: actual_payment,
        remaining: total_owed - actual_payment,
    })
}
```

### 4.6 违约处理——完全由放贷人心智驱动

```rust
/// 逾期未还款 → 由放贷人决定如何处理
fn handle_default(
    economy: &mut EconomyState,
    iou_entity_id: ItemEntId,
) -> Result<DefaultResolution, LoanError> {
    let iou = economy.iou_registry.get(iou_entity_id)?;
    let lender = economy.get_npc(iou.lender_id)?;
    let borrower = economy.get_npc(iou.borrower_id)?;

    iou.status = LoanStatus::Defaulted;

    // 放贷人根据自己的人格决定处理方式
    let resolution = decide_default_resolution(lender, borrower, iou);

    match &resolution {
        DefaultResolution::Forgive => {
            // 高宜人性："算了，谁都有难处"
            iou.status = LoanStatus::Forgiven;
            return_collateral(economy, iou);
            // 关系可能提升（感恩）或降低（被利用感）
            adjust_relationship(economy, lender.id, borrower.id, 0.05);
        }
        DefaultResolution::Negotiate { new_terms } => {
            // 高智慧："我们重新谈一下还款计划"
            let new_iou = restructure_loan(iou, new_terms);
            economy.iou_registry.register(new_iou);
        }
        DefaultResolution::ViolentCollection { method } => {
            // 低宜人性+有暴力资源："不还钱？你知道后果"
            // 触发战斗系统或威胁事件
            // 可能的结果：NPC 受伤、物品被强夺、关系永久破裂
            economy.dispatch_event(EconomyEvent::DebtCollection {
                lender_id: lender.id,
                borrower_id: borrower.id,
                iou_id: iou_entity_id,
                method: method.clone(),
            });
        }
        DefaultResolution::SellIou { price } => {
            // 高开放性："把欠条卖掉，让别人去收"
            // IOU 作为可交易物品在市场上挂牌
            economy.orderbook.place_order(Order {
                order_id: OrderId::new(),
                npc_id: lender.id,
                item_id: iou.entity_id.into(),
                order_type: OrderType::Ask,
                listing_type: ListingType::Normal,
                quantity: 1,
                price_per_unit: *price,
                placed_at: current_game_time(),
                expires_at: current_game_time() + ORDER_LIFETIME,
                region_id: lender.current_region,
            });
            iou.status = LoanStatus::Sold;
        }
        DefaultResolution::DoNothing => {
            // 中低智慧+中宜人性："唉……等着吧"
            // 关系微降，无进一步行动
            adjust_relationship(economy, lender.id, borrower.id, -0.05);
        }
    }

    Ok(resolution)
}

fn decide_default_resolution(
    lender: &NpcState,
    borrower: &NpcState,
    iou: &IouContract,
) -> DefaultResolution {
    let p = &lender.personality;

    // 高宜人性 → 倾向原谅
    if p.agreeableness > 0.8 && iou.principal < FORGIVE_THRESHOLD {
        return DefaultResolution::Forgive;
    }

    // 高智慧 → 倾向协商
    if lender.intelligence().unwrap_or(0.5) > 0.7 {
        return DefaultResolution::Negotiate {
            new_terms: NegotiatedTerms {
                extended_due_date: current_game_time() + 30,
                reduced_interest: iou.annual_interest_rate * 0.5,
                installment_plan: Some(generate_installment_plan(iou)),
            },
        };
    }

    // 低宜人性+有暴力资源 → 倾向暴力追债
    let has_violent_resources = assess_violent_capability(lender);
    if p.agreeableness < 0.3 && has_violent_resources {
        return DefaultResolution::ViolentCollection {
            method: DebtCollectionMethod::Threaten,
        };
    }

    // 高开放性 → 转卖欠条
    if p.openness > 0.7 {
        let sell_price = (iou.principal as f32 * 0.6) as CopperAmount; // 打折出售
        return DefaultResolution::SellIou { price: sell_price };
    }

    // 默认：什么都不做
    DefaultResolution::DoNothing
}
```

```rust
/// 评估 NPC 是否有开店意愿
fn assess_storefront_motivation(npc: &NpcState, economy: &EconomyState) -> Option<StoreMotivation> {
    // ---- 动机 1：有盈余想卖 ----
    // 如果 NPC 持续有盈余但市场买单不足（卖不掉），开店可以增加曝光
    let surplus_history = get_surplus_history(npc.id, 7); // 最近 7 天
    if surplus_history.has_persistent_surplus()
        && surplus_history.sell_through_rate() < 0.5  // 卖单成交率 < 50%
    {
        return Some(StoreMotivation::SurplusOutlet {
            items: surplus_history.get_surplus_items(),
            frustration: surplus_history.days_with_unsold_surplus,
        });
    }

    // ---- 动机 2：发现某地缺某物 ----
    // 旅途中发现某区域缺某种物品 → 开个店专门供应
    if let Some(gap) = detect_market_gap(npc, economy) {
        return Some(StoreMotivation::MarketGap {
            item_id: gap.item_id,
            region_id: gap.region_id,
            estimated_monthly_profit: gap.estimated_profit,
        });
    }

    // ---- 动机 3：习惯/传承 ----
    // "我爷爷就在这儿开了铁匠铺"
    if npc.family.has_storefront_tradition() {
        return Some(StoreMotivation::FamilyTradition {
            location: npc.family.ancestral_location(),
            item_category: npc.family.traditional_trade(),
        });
    }

    // ---- 动机 4：社交需求 ----
    // 高外向性 NPC 可能开店为了社交（"铺子是交朋友的地方"）
    if npc.personality.extraversion > 0.75 {
        return Some(StoreMotivation::SocialHub {
            emphasis: StoreEmphasis::CommunitySpace,
        });
    }

    None
}
```

```rust
/// 每日店铺经营心跳
fn storefront_daily_tick(
    store: &mut Storefront,
    owner: &NpcState,
    economy: &mut EconomyState,
) {
    if store.status != StoreStatus::Open {
        return;
    }

    store.operating_days += 1;

    // 1. 盘点库存
    let inventory_snapshot = owner.inventory.snapshot();

    // 2. 更新店内卖单（基于当前库存和owner需求）
    //    店主自己的盈余自动转为店内卖单
    update_store_listings(store, owner, economy);

    // 3. 评估是否需要进货
    //    如果某物品卖完了但需求还在→进货补货
    assess_restock_needs(store, owner, economy);

    // 4. 计算盈亏
    let daily_revenue = calculate_daily_revenue(store, economy);
    let daily_costs = calculate_daily_costs(store);
    let daily_profit = daily_revenue as i64 - daily_costs as i64;

    store.total_profit += daily_profit.max(0) as CopperAmount;

    // 5. 评估是否应该关店
    if should_close_store(store, daily_profit, owner) {
        store.status = StoreStatus::Closed {
            reason: ClosureReason::Unprofitable,
        };
    }
}

/// 判断是否应该关店
fn should_close_store(
    store: &Storefront,
    daily_profit: i64,
    owner: &NpcState,
) -> bool {
    // 连续 30 天亏损 + 总利润为负
    let consecutive_loss_days = store.get_consecutive_loss_days();
    if consecutive_loss_days > 30 && store.total_profit == 0 {
        return true;
    }

    // 店主的 perseverance (毅力) 可以延长坚持时间
    // 低神经质（抗压）+ 高尽责性（坚持）→ 更不容易放弃
    let grit = (1.0 - owner.personality.neuroticism) * 0.5
             + owner.personality.conscientiousness * 0.5;

    if consecutive_loss_days > (30.0 + grit * 60.0) as u32 {
        return true;  // 即使有毅力也熬不住了
    }

    false
}
```

| 无利可图 | 连续亏损 > 30+天 | 店主关闭铺子，回归原有生计 |
| 破产 | 债务 > 资产 | 强制关闭，可能失去建筑所有权 |
| 强制关闭 | 违反地方法规/被暴力威胁 | 短期或永久关闭 |
| 店主死亡无继承 | owner 死亡 + 无继承人 | 店铺废弃，物品随时间散失 |
| 店主转行 | 发现更好的谋生方式 | 主动关闭，可能转卖给他人 |

## 六、角色涌现全景

### 6.1 从条件到角色

```
条件组合                      涌现角色           自我认知
─────────────────────────────────────────────────────
盈余粮食 + 每日挂卖单     →  粮食供应者     "我是个种地的"
盈余武器 + 每日挂卖单     →  武器供应商     "我造的家伙什还行"
经常挂食物买单           →  食物消费者     （无特殊认知）
四条件满足 + 开始套利     →  兼职商人       "偶尔跑跑买卖"
套利成功 10+次            →  职业商人       "我是个生意人"
开店 + 经营稳定           →  店主           "老XXX的铺子"
放贷 5+笔                →  放贷人         "需要用钱来找我"
频繁借贷                 →  负债者         "唉，欠了一屁股债"
```

### 6.2 角色可以重叠

一个 NPC 可以同时是多个角色：
- 铁匠+店主+偶尔商人："我打铁为生。有铺子。听说邻镇铁矿石便宜时会跑一趟。"
- 农民+放贷人："种地攒了些钱。村东头小伙子借了些——收点利息。"

### 6.3 角色会退化

- 商人连续 30 天不交易 → activity_score 衰减 → 不再自认商人
- 店主关店后 → 回归原有生计
- 放贷人遭遇一次大额违约 → neuroticism 驱动下可能再也不放贷

## 七、性能考量

### 7.1 心跳频率

| 操作 | 频率 | 说明 |
|------|------|------|
| 盈余评估 | 每日一次 | 对 L1/L2 NPC 全量；L3 抽样 10%；L4 统计近似 |
| 需求评估 | 每日一次 | 同上 |
| 套利扫描 | 每日一次 | 仅对满足四条件的 NPC |
| 店铺经营循环 | 每日一次 | 仅对活跃店铺 |
| 贷款逾期检查 | 每日一次 | 全量（IOU 数量远小于 NPC 数量） |
| 市场集市检测 | 每日一次 | 仅对有 N≥3 个摊贩的区域 |

### 7.2 复杂度

| 操作 | 复杂度 | 说明 |
|------|--------|------|
| 盈余评估 | O(N_ITEMS × N_NPC) | 每个 NPC 每个物品类型评估 |
| 需求评估 | O(N_NPC × 需求来源数) | 三个需求来源，常数因子 |
| 套利扫描 | O(MARKETS² × ITEMS_OVERLAP) | 已知市场间物品交集 |
| 借贷评估 | O(1) per 请求 | 单个贷款请求的评估 |
| 店铺循环 | O(STORES × ITEMS) | 只遍历活跃店铺 |

### 7.3 分层近似

- **L1**（玩家附近 < 500m）：全部 NPC 全量评估
- **L2**（500m - 2km）：全量评估，跳过套利扫描（需要跨区域才有套利）
- **L3**（2km - 10km）：抽样 10% NPC 评估供需；套利仅扫描主要贸易路线
- **L4**（> 10km）：统计近似——不追踪个体，使用供需曲线+随机扰动
