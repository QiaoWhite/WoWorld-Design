## 一、PhaticLayer——应酬交流层

### 1.1 什么是 Phatic 交流

**Phatic communication（应酬性交流）**——信息密度为零，社交功能为全部。语言学中：不是为了传递信息，而是为了建立/维持社会联系。

在 WoWorld 中，Phatic 层是 NPC 从"任务看板"变成"活人"的关键。一个铁匠抱怨"湿度高碳烧得慢"——不推进任何任务、不传递重要信息。但玩家知道了**这个世界的铁匠关心天气对火候的影响**。这就是活人感。

### 1.2 PhaticFragment 定义

```rust
/// Phatic 片段——短文本，高社交功能，低信息量
pub struct PhaticFragment {
    pub id: u32,
    /// 触发条件
    pub conditions: PhaticCondition,
    /// 内容——通常 10-50 字
    pub text: String,
    /// 对关系的微小影响
    pub social_effect: SocialMicroEffect,
    /// 预期不会引发深入讨论——但可以作为话题跳板
    pub can_lead_to_topic: bool,
}

pub enum PhaticCondition {
    /// 对话开场——先寒暄，再进入正题
    Opening {
        formality: FormalityLevel,
        time_of_day: bool,       // "早上好" / "晚上好"
        weather_trigger: bool,   // "今天真热啊"
    },
    /// 对话进行中——在两个正题之间随机穿插
    Interstitial {
        emotion: Option<Vec<MoodLabel>>,
        personality: Option<Vec<PersonalityTraitFilter>>,
        min_trust: Option<f32>,
        probability: f32,        // 触发概率——默认 0.3
    },
    /// 对话结束——告别前的收尾
    Closing {
        tone: ClosingTone,
    },
    /// NPC 反应性吐槽——观察到什么就说出来
    Reactive {
        trigger: ReactiveTrigger,
    },
    /// 无特定目的——单纯想说句话
    Idle {
        personality_filters: Vec<PersonalityTraitFilter>,
        frequency_modifier: f32,  // 外向=高频率, 内向=低频率
    },
}

pub enum FormalityLevel { VeryFormal, Formal, Casual, Intimate }
pub enum ClosingTone { Warm, Neutral, Cold, Urgent }

pub struct PersonalityTraitFilter {
    pub trait_name: String,    // "extraversion" / "agreeableness" / "openness"
    pub min: f32,
    pub max: f32,
}

pub enum ReactiveTrigger {
    /// 观察到对方的物品——"你这剑保养得不错"
    SawItem(ItemId),
    /// 天气变化——"天阴下来了"
    WeatherChanged,
    /// 对方受伤——"你流血了！"
    SawInjury,
    /// 对方穿着——"这件披风是北地风格吧？"
    SawClothing,
    /// 对方的行为——"你刚才那一剑真漂亮"
    SawAction { action_desc: String, valence: f32 },
    /// 刚有人进入/离开
    SomeoneEntered { who: EntityId },
    SomeoneLeft { who: EntityId },
}

pub struct SocialMicroEffect {
    /// 信任变化——通常 ±0.01 ~ ±0.03
    pub trust_delta: f32,
    /// 熟悉度变化
    pub familiarity_delta: f32,
    /// 对当前 SocialField 的瞬间影响
    pub field_impulse: Option<Emotion>,
}
```

### 1.3 PhaticLayer 实现

```rust
pub struct PhaticLayer {
    /// 所有 Phatic 片段——按分类索引
    openings: Vec<PhaticFragment>,
    interstitials: Vec<PhaticFragment>,
    closings: Vec<PhaticFragment>,
    reactives: HashMap<ReactiveTriggerType, Vec<PhaticFragment>>,
    idles: Vec<PhaticFragment>,
}

impl PhaticLayer {
    /// 对话开场——在第一个正题之前
    pub fn opening(
        &self,
        speaker: &Participant,
        ctx: &DialogueContext,
        generator: &TextGenerator,
    ) -> Option<NpcUtterance> {
        let formality = self.compute_formality(speaker, ctx);
        self.openings.iter()
            .filter(|f| f.conditions.matches_opening(formality, ctx))
            .choose_deterministic(speaker.entity_id, ctx.current_time)
            .map(|f| self.render_fragment(f, speaker, ctx, generator))
    }

    /// 对话间歇——在两个正题之间概率触发
    pub fn interstitial(
        &self,
        speaker: &Participant,
        ctx: &DialogueContext,
        generator: &TextGenerator,
    ) -> Option<NpcUtterance> {
        self.interstitials.iter()
            .filter(|f| f.conditions.matches_interstitial(speaker, ctx))
            .filter(|_| fast_rand() < f.probability)
            .choose_deterministic(speaker.entity_id, ctx.current_time)
            .map(|f| self.render_fragment(f, speaker, ctx, generator))
    }

    /// 对话收尾
    pub fn closing(
        &self,
        speaker: &Participant,
        tone: ClosingTone,
        ctx: &DialogueContext,
        generator: &TextGenerator,
    ) -> Option<NpcUtterance> {
        self.closings.iter()
            .filter(|f| f.conditions.matches_closing(tone, ctx))
            .choose_deterministic(speaker.entity_id, ctx.current_time)
            .map(|f| self.render_fragment(f, speaker, ctx, generator))
    }

    /// 反应性吐槽——观察到触发条件时
    pub fn reactive(
        &self,
        trigger: &ReactiveTrigger,
        speaker: &Participant,
        ctx: &DialogueContext,
        generator: &TextGenerator,
    ) -> Option<NpcUtterance> {
        let trigger_type = ReactiveTriggerType::from_trigger(trigger);
        self.reactives.get(&trigger_type)?
            .iter()
            .filter(|f| f.conditions.matches_reactive(trigger, speaker, ctx))
            .choose_deterministic(speaker.entity_id, ctx.current_time)
            .map(|f| self.render_fragment(f, speaker, ctx, generator))
    }

    /// 计算对话的正式度
    fn compute_formality(&self, speaker: &Participant, ctx: &DialogueContext) -> FormalityLevel {
        let avg_trust = ctx.relationships.iter()
            .map(|r| r.trust)
            .fold(0.0, |a, b| a + b) / ctx.relationships.len().max(1) as f32;

        if avg_trust > 0.7 { FormalityLevel::Intimate }
        else if avg_trust > 0.4 { FormalityLevel::Casual }
        else if avg_trust > 0.15 { FormalityLevel::Formal }
        else { FormalityLevel::VeryFormal }
    }
}

pub struct NpcUtterance {
    pub text: String,
    pub emotion: Emotion,
    pub social_effect: SocialMicroEffect,
    pub is_phatic: bool,  // true → "这不要求玩家回应"
}
```

### 1.4 Phatic 片段数量

| 分类 | 片段数 | 示例 |
|------|--------|------|
| **Opening** | ~40 | 不同 formality × 时间 × 个性 |
| **Interstitial** | ~50 | 不同情绪 × 个性组合 × 职业 |
| **Closing** | ~30 | 不同 tone × 关系层级 |
| **Reactive** | ~40 | 观察物品/天气/NPC/行为 |
| **Idle** | ~50 | 不同个性 × 职业的自发吐槽 |
| **总计** | **~210** | 全部短文本 (10-50 字) |

### 1.5 Phatic 示例

```
// Opening——低友善度 + 下午 + 雨天
"[Opening:formality=Casual, time_of_day=afternoon, weather=rain]"
→ 老王(agreeableness=0.4): "下雨天还出来？"

// Opening——高友善度 + 下午 + 晴天
→ 开朗学徒(agreeableness=0.8): "下午好！今天天气真好——适合打铁，火候刚好！"

// Interstitial——铁匠职业视角
"[Interstitial:profession=blacksmith, personality=low_agreeableness, probability=0.3]"
→ "这种天——炉子反而好烧。湿度高，碳烧得慢。"

// Reactive——看到玩家的剑
"[Reactive:trigger=SawItem(weapon), condition=item_quality>Good]"
→ "你这剑——保养得不错。刃口还亮。"

// Closing——低信任 + 中性语调
"[Closing:tone=Neutral, trust<0.4]"
→ "行，你忙你的。"

// Closing——高信任 + 温暖语调
"[Closing:tone=Warm, trust>0.7]"
→ "下次来再聊。给你留壶凉茶。"

// Idle——低友善度 + 高外向性 → 吐槽
"[Idle:personality=low_agreeableness+high_extraversion, profession=blacksmith]"
→ "上次那个说自己会打铁的客人，打出来的东西跟狗啃的似的。"

// Idle——高友善度 + 高外向性 → 热情分享
"[Idle:personality=high_agreeableness+high_extraversion]"
→ "我跟你说——昨天我徒弟第一次独立打出了一把好剑！"
```

## 二、SocialField——群体社会动力学

### 2.1 数学模型

```rust
/// 社会场——群体层面的情感/态度/注意力状态
/// 不是个体状态的简单平均——有群体特有的涌现属性
pub struct SocialField {
    /// 主流情绪方向——大多数人趋同后的结果
    pub dominant_emotion: Emotion,

    /// 情绪分歧程度——0 = 所有人感受一致, 1 = 极度分裂
    pub emotional_polarization: f32,

    /// 对当前发言者/话题的认可比例 0-1
    pub approval_ratio: f32,

    /// 听众躁动程度——0 = 全神贯注, 1 = 快要散场
    pub restlessness: f32,

    /// 子群体——在主流之外的意见/情绪集群
    pub factions: Vec<SocialFaction>,

    /// 群体对个体的从众压力——"别人都鼓掌了，我也得鼓掌"
    pub conformity_pressure: f32,

    /// 当前激活的群体身份——"我们是苍山城的市民"
    pub activated_identity: Option<GroupIdentity>,

    /// 群体规模——影响惯性
    pub group_size: u32,
}

pub struct SocialFaction {
    pub label: String,
    /// 该派系在群体中的比例
    pub size: f32,         // 0-1
    /// 该派系的主导情绪
    pub emotion: Emotion,
    /// 该派系对当前话题的立场（-1 反对 → +1 支持）
    pub stance: f32,
}

pub struct GroupIdentity {
    pub label: String,           // "苍山城市民" / "守军" / "冒险者行会"
    pub cohesion: f32,           // 群体凝聚力 0-1
    pub outgroup_present: bool,  // 是否有外群体在场（激活内群体偏袒）
}
```

### 2.2 群体情绪惯性

```rust
impl SocialField {
    /// 群体规模越大，情绪变化越慢
    pub fn emotional_inertia(&self) -> f32 {
        1.0 / (1.0 + self.group_size as f32 * 0.02)
    }
    // group_size=2:  inertia ≈ 0.96  → 变化很快
    // group_size=10: inertia ≈ 0.83  → 中等
    // group_size=100:inertia ≈ 0.33  → 变化缓慢
    // group_size=1000:inertia≈0.048  → 几乎不变

    /// 吸收一轮发言的影响
    pub fn absorb_utterance(
        &mut self,
        utterance: &ConversationTurn,
        speaker_authority: f32,
        speaker_charisma: f32,
    ) {
        let influence = speaker_authority * speaker_charisma;
        let inertia = self.emotional_inertia();

        // ① 情绪更新
        self.dominant_emotion = self.dominant_emotion.lerp(
            utterance.emotion,
            influence * inertia * 0.3,
        );

        // ② 认可比例更新——已有认可产生从众惯性
        let conformity_weight = self.approval_ratio * self.conformity_pressure;
        self.approval_ratio =
            self.approval_ratio * (0.7 + conformity_weight * 0.3)
            + utterance.persuasiveness * influence * (1.0 - conformity_weight) * 0.3;

        // ③ 躁动更新——单调→躁动增加，高潮/冲突→归零
        if utterance.is_climactic() {
            self.restlessness = 0.0;
        } else {
            self.restlessness = (self.restlessness + 0.02 * (1.0 - utterance.engagement_value)).min(1.0);
        }

        // ④ 极化——争议性话题 + 高唤醒 → 加剧分裂
        if utterance.is_controversial() && self.dominant_emotion.arousal > 0.6 {
            self.emotional_polarization = (self.emotional_polarization + 0.05).min(1.0);
        }
    }

    /// 群体解散风险
    pub fn risk_of_dissolution(&self) -> f32 {
        (self.restlessness * 0.6 + self.emotional_polarization * 0.4)
            .min(1.0)
    }
}
```

### 2.3 SensoryMapping——SocialField → Godot 感官反馈

```rust
/// 确定性映射——不需要 LLM，不需要复杂 AI
/// SocialField 的数字 → 人群动画 + 音效
pub struct SensoryMapping;

impl SensoryMapping {
    /// 基于 approval_ratio 的群体听觉反馈
    pub fn crowd_audio(&self, field: &SocialField) -> CrowdAudio {
        match field.approval_ratio {
            r if r < 0.2 => CrowdAudio::Silence,          // 死寂
            r if r < 0.35 => CrowdAudio::Murmurs,          // 低声交谈
            r if r < 0.5 => CrowdAudio::ScatteredResponse, // 零星回应
            r if r < 0.7 => CrowdAudio::Approval,          // 欢呼/鼓掌
            r if r < 0.85 => CrowdAudio::Enthusiastic,     // "对！""守卫家园！"
            _ => CrowdAudio::Thunderous,                   // 全场沸腾
        }
    }

    /// 基于 restlessness 的群体视觉反馈
    pub fn crowd_animation(&self, field: &SocialField) -> CrowdAnim {
        match field.restlessness {
            r if r < 0.1 => CrowdAnim::Attentive,         // 专注——目光跟随演讲者
            r if r < 0.3 => CrowdAnim::Fidgeting,          // 小动作——有人换脚重心
            r if r < 0.5 => CrowdAnim::Restless,           // 躁动——交头接耳
            r if r < 0.7 => CrowdAnim::Agitated,           // 骚动——有人起身
            _ => CrowdAnim::Dispersing,                    // 散场——人们离场
        }
    }

    /// 少数派可视化
    pub fn faction_visuals(&self, field: &SocialField) -> Vec<FactionVisual> {
        field.factions.iter()
            .filter(|f| f.size > 0.05)  // 占比 > 5% 才可见
            .map(|f| FactionVisual {
                description: match (f.stance, f.emotion.arousal) {
                    (s, a) if s < -0.5 && a > 0.5 => "有几个人愤怒地摇头",
                    (s, _) if s < -0.3 => "前排几个贵族面有不悦",
                    (s, a) if s > 0.5 && a > 0.5 => "一群年轻人用力鼓掌",
                    _ => "少数人表情复杂",
                }.to_string(),
                estimated_count: (field.group_size as f32 * f.size) as u32,
            })
            .collect()
    }
}

pub enum CrowdAudio { Silence, Murmurs, ScatteredResponse, Approval, Enthusiastic, Thunderous }
pub enum CrowdAnim { Attentive, Fidgeting, Restless, Agitated, Dispersing }
pub struct FactionVisual { pub description: String, pub estimated_count: u32 }
```

## 三、完整的对话轮次处理

```rust
impl DialogueSystem {
    /// 处理一轮对话（玩家输入 → NPC 回应）
    pub fn process_turn(
        &mut self,
        conversation: &mut Conversation,
        player_input: PlayerInput,
        ctx: &DialogueContext,
    ) -> DialogueResult {
        // ① 首轮？→ Opening Phatic
        let phatic_opening = if conversation.history.is_empty() {
            self.phatic_layer.opening(&conversation.current_speaker_participant(), ctx, &self.generator)
        } else { None };

        // ② 解析玩家输入 → 话题
        let topic = match player_input {
            PlayerInput::SelectOption { option_index } => {
                ConversationTopic::Preset { /* 来自当前选项列表 */ .. }
            }
            PlayerInput::JournalReference { entry_id } => {
                let entry = /* 从 ctx.player_journal_entries 找到对应条目 */;
                self.topic_selector.from_journal(&entry, &ctx.npc_snapshot)
            }
            PlayerInput::FreeText { text, .. } => {
                let intent = self.input_interpreter.interpret(&text, ctx);
                self.topic_selector.from_intent(&intent, &ctx.npc_snapshot)
                    .first()
                    .cloned()
                    .unwrap_or(ConversationTopic::FreeTextTopic { /* fallback */ })
            }
            PlayerInput::MatchedOption { option_index, confidence, .. } => {
                // 走预设话题管道
                ConversationTopic::Preset { /* 匹配到的选项对应的话题 */ .. }
            }
        };

        // ③ NPC 解析话题 → 回应
        let npc_response = self.topic_resolver.resolve(
            &topic,
            &ctx.npc_snapshot,
            ctx,
            &self.generator,
        );

        // ④ Phatic 穿插——30% 概率在正题之间插入应酬片段
        let phatic_interstitial = if fast_rand() < 0.3 {
            self.phatic_layer.interstitial(&conversation.current_speaker_participant(), ctx, &self.generator)
        } else { None };

        // ⑤ 组装最终文本
        let final_text = self.assemble_response_text(
            phatic_opening,
            npc_response,
            phatic_interstitial,
        );

        // ⑥ 更新 SocialField
        conversation.social_field.absorb_utterance(
            &ConversationTurn { text: final_text.clone(), emotion: npc_response.emotion, .. },
            conversation.current_speaker_authority(),
            conversation.current_speaker_charisma(),
        );

        // ⑦ 记录对话历史
        conversation.history.push(TurnRecord {
            speaker: ctx.npc_snapshot.id,
            text: final_text.clone(),
            emotion: npc_response.emotion,
            timestamp: ctx.current_time,
        });

        DialogueResult {
            npc_text: final_text,
            npc_emotion: npc_response.emotion,
            player_options: npc_response.player_options,
            social_field_update: conversation.social_field.clone(),
        }
    }
}
```
