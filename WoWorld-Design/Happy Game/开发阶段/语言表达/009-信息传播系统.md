## 一、系统定位

### 1.1 与语言表达模块的关系

信息传播系统是语言表达模块的**延伸**，而非独立模块。它消费语言表达系统提供的 `Conversation`/`DialogueIntent`/`ExpressionRef` 基础设施，在此基础上建立跨 NPC 的信息流动管道。

```
语言表达模块（对话基础设施）
        │
        ├── Conversation · TurnMode · TopicSelector
        ├── ExpressionRef · ContentResolver
        └── PhaticLayer · SocialField
                │
                ▼
信息传播系统（本文件）——建立在对话基础设施之上
        │
        ├── 传播通道（口头/书信/信鸽/魔法/公告）
        ├── 失真算子（每跳衰减 + 内容扭曲）
        ├── 谣言生命周期（诞生→传播→变异→消亡）
        ├── NPC 间对话渲染（玩家可见的 NPC 交谈）
        └── 悄悄话/密谋（私密对话模式）
```

### 1.2 核心认识

WoWorld 的"故事"由 NPC 之间的信息流动自然涌现。开发者不写故事——开发者定义信息如何传播、如何失真、如何被记忆消化。历史模块提供事件的**源头**（`PropagableEventSource`），信息传播系统定义这些事件**如何在 NPC 口中变形和扩散**。

#### 口头传播触发条件

> ⚠️ **[CHG-030 权威注解]**: effective_audible_radius() 的声学公式定义位于音频模块。ConversationMode 中的 audible_radius_m 参数为消费侧引用。

```rust
/// NPC 行为树每 5-30 秒评估是否发起口头传播
fn should_propagate_orally(npc: &NpcData, memory: &EventMemory) -> bool {
    // ① 记忆必须是可分享的
    if !memory.is_public { return false; }

    // ② 记忆的 impact 必须高于传播阈值
    if memory.impact_score < 0.15 { return false; }

    // ③ 必须有可用的对话对象在社交距离内
    //    或者在 NPC 的行为计划中安排了社交活动

    // ④ 概率 = base × extraversion_mod × urgency_mod
    let base_prob = 0.02;  // 每次评估的基础概率
    let extraversion_mod = 0.5 + npc.personality.extraversion * 1.0;  // 0.5-1.5
    let urgency = memory.impact_score * memory.recency_factor();
    let prob = base_prob * extraversion_mod * urgency;

    fast_rand() < prob
}
```

### 2.3 玩家可见的 NPC 间对话

当两个 NPC 在玩家附近交谈时，玩家应该能看到和听到。

```rust
/// 环境对话——发生在玩家附近但不涉及玩家的对话
pub struct AmbientConversation {
    /// 底层的 Conversation 实例
    pub conversation: Conversation,
    /// 对话的可见性级别

pub enum ConversationVisibility {
    /// 完全可见——玩家在 Normal 对话的 3m 范围内
    Full,
    /// 部分可见——玩家在 Loud 对话的 5-8m 范围内，听到只言片语
    Partial { clarity: f32 },
    /// 仅能察觉——看到两个 NPC 在交谈，但听不清
    BarelyPerceptible,
    /// 不可见——对话以悄悄话/密谋模式进行，玩家完全不知道
    Invisible,
}

pub struct PlayerPerception {
    /// 玩家能听到的文本片段（距离衰减 → 部分文字丢失）
    pub heard_text: Option<String>,
    /// 看到的表情/姿势变化——比听到文字更容易
    pub visible_gestures: Vec<GestureObservation>,
    /// 说话者的身份（玩家认识的话）
    pub speaker_identity: Option<String>,
    /// 对话的情绪基调——即使听不清也能感知
    pub emotional_tone: Option<Emotion>,
}

/// Godot 渲染环境对话:
/// 1. 检查玩家附近（L1 ≤50m）所有活跃 Conversation
/// 2. 对每个 Conversation，计算 PlayerPerception:
///    - 距离 < 3m + Normal 模式 → Full → 完整字幕
///    - 距离 3-8m + Loud 模式 → Partial → 模糊字幕 + "..." 替代
///    - 距离 < 0.5m + Whisper 模式 → BarelyPerceptible → "两个人在低声交谈"
///    - 距离 > audible_radius → 看不到文字，但能看到肢体语言
/// 3. 渲染：
///    - NPC 头顶气泡/字幕（L1 only）
///    - 非语言表达（PoseOffset 动画——已有的 §4.4 系统）
///    - 环境音效（模糊的对话声）
```

#### NPC 间对话渲染流程

```
① NPC A 的 DialogueIntent::SpreadInformation 被激活
② TurnAllocator 分配发言权 → Conversation 创建
③ TopicSelector 选择话题 → TopicResolver 生成对话文本
④ 对话在 Rust 侧正常进行（TurnMode::FreeForm）
⑤ 每轮发言后，检查玩家距离:
   ├── 玩家在 50m 内 (L1) → 计算 PlayerPerception
   │   ├── 距离 < audible_radius → 渲染完整/模糊文本
   │   └── 距离 > audible_radius → 渲染肢体语言 + 情绪
   └── 玩家 > 50m → 不渲染文本，仅记录传播结果
⑥ propagate_memory() 在后台执行——无论玩家是否看到
⑦ 对话结束后，双方 NPC 的 EventMemory 更新
```

```rust
pub enum CourierType {
    /// 旅行商人顺路带信——慢，便宜，丢失风险中等
    TravelingMerchant { speed_km_per_day: f32 },
    /// 专业信使——快，贵，丢失风险低
    ProfessionalCourier { speed_km_per_day: f32 },
    /// 信鸽/信使动物——只适用于固定路线
    Animal { speed_km_per_day: f32, route_specific: bool },
    /// 魔法使魔——即时但昂贵
    MagicFamiliar { mana_cost: u32 },
}

impl LetterPropagation {
    /// 计算到达时间
    pub fn travel_days(&self, network: &TransportNetwork) -> f32 {
        let distance_km = network.distance(self.sender_location(), self.recipient_location());
        distance_km / self.courier_type.speed()
    }

    /// 丢失判定——在信件到达时 roll
    pub fn check_loss(&self, world_state: &WorldState) -> bool {
        let base_risk = self.loss_probability;
        // 经过危险区域 → +20% 丢失风险
        // 暴风雨/暴雪天气 → +15% 丢失风险
        // 专业信使 → -30% 丢失风险
        // 魔法使魔 → 0% 丢失风险
        fast_rand() < base_risk
    }
}
```

## 三、失真算子

### 3.1 基础衰减模型

```rust
/// 信息传播的核心失真模型——每次传播 (hop) 都应用
pub struct PropagationDistortion {
    /// 基础冲击力衰减——已存在 (propagate_memory)
    pub impact_decay: f32,          // 默认 0.7
    /// 信任修正
    pub trust_modifier: f32,        // decay = impact_decay * (0.5 + trust * 0.5)
}

/// 每次传播应用的失真算子
pub enum DistortionOperator {
    /// 数量膨胀/缩小——"死了一百人" → "死了上千人"
    ScaleDistortion {
        param_name: String,         // 要扭曲的参数名
        factor_range: (f32, f32),   // 扭曲因子范围
    },
    /// 细节丢失——"蓝眼睛、金发、脸上有疤的北方人" → "一个北方人"
    DetailLoss {
        fraction_lost: f32,         // 丢失比例 0-1
    },
    /// 情感放大——中性事件 → 更戏剧化
    EmotionalAmplification {
        emotion_type: Emotion,
        amplification: f32,         // 1.0=不变，2.0=双倍
    },
    /// 归因替换——"是兽人干的" → "是北方蛮族干的"（符合听众偏见）
    AttributionShift {
        from_tag: String,
        to_tag: String,
    },
    /// 道德评价附加——中性事实 + 道德判断
    Moralization {
        valence: f32,               // -1 负面道德化 / +1 正面道德化
    },
}

impl DistortionOperator {
    /// 对传播内容应用失真
    pub fn apply(
        &self,
        payload: &mut PropagationPayload,
        teller: &NpcData,
        listener: &NpcData,
        propagation_count: u32,     // 第几次传播
    ) {
        match self {
            DistortionOperator::ScaleDistortion { param_name, factor_range } => {
                if let Some(value) = payload.quantitative_data.get_mut(param_name) {
                    let factor = lerp(factor_range.0, factor_range.1, fast_rand());
                    // 越多次传播，扭曲因子越极端
                    let amplified = factor * (1.0 + propagation_count as f32 * 0.1);
                    *value *= amplified;
                }
            }
            DistortionOperator::DetailLoss { fraction_lost } => {
                // 从 core_facts 中随机删除 key
                let keys: Vec<String> = payload.core_facts.keys().cloned().collect();
                let lose_count = (keys.len() as f32 * *fraction_lost) as usize;
                for _ in 0..lose_count {
                    if let Some(key) = keys.choose_random() {
                        payload.core_facts.remove(key);
                    }
                }
            }
            DistortionOperator::EmotionalAmplification { emotion_type, amplification } => {
                // 情感编码放大——影响听众的 emotional_encoding
                // 在第 propagate_memory 的 EmotionalEncoding 阶段处理
            }
            DistortionOperator::AttributionShift { from_tag, to_tag } => {
                // 替换 tags 中的归因标签
                for tag in &mut payload.tags {
                    if *tag == *from_tag { *tag = to_tag.clone(); }
                }
            }
            DistortionOperator::Moralization { valence } => {
                // 添加道德评价标签
                if *valence < 0.0 {
                    payload.tags.push("immoral".into());
                } else if *valence > 0.0 {
                    payload.tags.push("righteous".into());
                }
            }
        }
    }
}
```

### 3.2 说谎与欺骗

```rust
/// NPC 可以在传播中故意注入虚假信息
/// 这与"无意的失真"不同——是主动行为
pub enum DeceptionIntent {
    /// 无欺骗——正常传播
    Honest,
    /// 夸大——把 impact_score 放大（吹牛）
    Exaggeration { boost: f32 },
    /// 隐瞒——故意略去某些关键事实
    Omission { omitted_facts: Vec<String> },
    /// 完全捏造——传播一个完全虚假的记忆
    Fabrication { fabricated_summary: String },
}

/// 欺骗检测——听众评估说话者是否在说谎
pub struct DeceptionDetection {
    /// 听众的 intelligence → 更可能识破漏洞
    intelligence_factor: f32,
    /// 听众对说话者的 trust → 高信任 = 放松警惕
    trust_blind_spot: f32,       // trust > 0.8 → 大幅降低检测率
    /// 听众是否拥有矛盾信息
    has_contradicting_info: bool,
    /// 听众的 agreeableness → 高友善 = 倾向相信他人
    agreeableness_bias: f32,
}

impl DeceptionDetection {
    /// 计算检测概率
    pub fn detect_probability(&self, teller: &NpcData, listener: &NpcData) -> f32 {
        let base = 0.2;  // 基础检测率 20%
        let intel_bonus = (listener.intelligence_factor() - 0.5) * 0.3;
        let trust_penalty = listener.trust_toward(teller) * 0.4;
        let contradiction_bonus: f32 = if self.has_contradicting_info { 0.3 } else { 0.0 };
        (base + intel_bonus - trust_penalty + contradiction_bonus).clamp(0.0, 1.0)
    }
}
```

### 4.1 传播速度模型

```rust
/// 信息在不同社会网络中的传播速度
pub struct PropagationSpeed {
    /// 口头传播速度——受人口密度影响
    /// 城市: ~50 NPC/day, 村庄: ~5 NPC/day
    pub oral_speed: f32,     // NPC/天
    /// 书信传播速度——受交通网络影响
    pub letter_speed: f32,   // km/天
    /// 魔法传播速度——即时
    pub magical_speed: f32,  // 无限
    /// 公告传播速度——受人口密度影响
    pub official_speed: f32, // NPC/天
}

impl PropagationSpeed {
    /// 口头信息到达指定人数的天数
    pub fn oral_penetration_days(
        &self,
        target_reached: u32,
        initial_tellers: u32,
        location_population: u32,
    ) -> f32 {
        // 逻辑斯谛增长模型——开始时慢，中间快，饱和时慢
        // dR/dt = k * R * (1 - R/N)
        // 其中 R = reached, N = population, k = oral_speed
        // 解析解: R(t) = N / (1 + (N/R0 - 1) * e^(-k*t))
        // 求 t 使 R(t) >= target_reached
        let n = location_population as f32;
        let r0 = initial_tellers as f32;
        let target = target_reached as f32;
        let k = self.oral_speed;
        // t = -(1/k) * ln((N/target - 1) / (N/r0 - 1))
        -(1.0 / k) * ((n / target - 1.0) / (n / r0 - 1.0)).ln()
    }
}
```

## 五、悄悄话与密谋

### 5.1 私密对话模式

```rust
/// 悄悄话/密谋——Conversation 的特殊 TurnMode
/// 不是独立的 TurnMode 变体——是 Conversation 的 mode 参数
impl Conversation {
    /// 切换到私密模式
    pub fn set_private_mode(&mut self, mode: PrivateMode) {
        self.private_mode = Some(mode);
        // 私密对话的参与者必须靠近——物理约束
        self.enforce_proximity_constraint();
    }

    /// 检测是否被偷听
    pub fn check_eavesdropping(&self, world: &WorldState) -> Vec<Eavesdropper> {
        let radius = self.private_mode.as_ref()
            .map(|m| m.audible_radius())
            .unwrap_or(3.0);

        // 查找 audible_radius 范围内的所有其他实体
        world.query_nearby_entities(self.location(), radius)
            .into_iter()
            .filter(|e| !self.is_participant(e.id))
            .map(|e| Eavesdropper {
                entity_id: e.id,
                distance: e.distance,
                clarity: self.eavesdropping_clarity(e.distance),
                is_intentionally_listening: self.is_suspicious_behavior(e),
            })
            .collect()
    }
}

pub enum PrivateMode {
    /// 悄悄话——压低声音，但旁边的人能听到
    Whisper {
        audible_radius: f32,     // 默认 0.5m
    },
    /// 密谋——有意识地确认周围无人
    Conspiracy {
        audible_radius: f32,     // 默认 0.3m
        /// 参与者会主动扫描周围环境
        participants_scanning: bool,
        /// 发现偷听者后的反应
        reaction_on_discovery: DiscoveryReaction,
    },
}

pub enum DiscoveryReaction {
```

### 5.2 偷听检测

```rust
impl Conversation {
    /// 计算玩家/NPC 偷听到的内容
    pub fn eavesdropping_clarity(&self, distance: f32) -> f32 {
        let radius = self.private_mode.as_ref()
            .map(|m| m.audible_radius())
            .unwrap_or(3.0);

        if distance <= radius {
            1.0  // 在范围内——完全清晰
        } else {
            // 指数衰减——每超出 1m 清晰度减半
            let excess = distance - radius;
            (0.5_f32).powf(excess)
            // 0.5m 超出 → 0.71 清晰度
            // 1m 超出 → 0.5 清晰度
            // 2m 超出 → 0.25 清晰度
            // 3m+ 超出 → 几乎听不到
        }
    }

    /// 应用清晰度衰减到文本
    pub fn degrade_text_by_clarity(text: &str, clarity: f32) -> String {
        if clarity >= 0.8 { return text.to_string(); }
        text.chars()
            .map(|c| {
                if c.is_whitespace() || c.is_ascii_punctuation() { return c; }
                if fast_rand() < clarity { c } else { '…' }
            })
            .collect()
    }
}
```
