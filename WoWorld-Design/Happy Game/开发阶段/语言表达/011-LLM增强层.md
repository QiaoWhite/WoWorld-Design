## 一、定位

LLM 增强层是语言表达模块的**可选装饰层**——包裹现有模板管道，在玩家启用时提供 LLM 增强的输出，在未启用或不可用时无缝回退到模板。它不修改模板管道的任何代码，不绑定任何特定 LLM 来源。

核心原则：
- **独立于模板系统**——模板管道永远是 fallback，LLM 是锦上花
- **场景粒度开关**——玩家按 19 个场景自由选择是否使用 LLM
- **后端无关**——本地部署（ollama/llama.cpp/vLLM）和云端 API（OpenAI/Anthropic/Groq）统一抽象
- **NPC 自主决定深度**——不由系统设信任硬阈值，NPC 个性/情绪/文化自主决定对话意愿

## 五、NPC 自主对话意愿

### 5.1 NpcDialogueWillingness

系统不设信任硬阈值。NPC 通过六维模型自主决定是否愿意深度对话：

```rust
pub struct NpcDialogueWillingness {
    pub overall: f32,  // 0-1
    pub breakdown: WillingnessBreakdown,
}
pub struct WillingnessBreakdown {
    pub personality_contribution: f32,    // openness×0.30 + extraversion×0.20 + agreeableness×0.10
    pub mood_contribution: f32,           // 平静=高意愿，极端情绪=只想发泄
    pub relationship_contribution: f32,   // trust×0.20——连续值，非阈值
    pub topic_contribution: f32,          // 亲身经历/专业领域=高兴趣
    pub situation_contribution: f32,      // 旅途中+0.10，危险中-0.20
    pub culture_contribution: f32,        // 直接性+表达性
}
```

willingness 不决定"是否调用 LLM"——它决定 LLM 生成什么样的回应：
- `>0.7`: "Elaborate freely, share personal stories."
- `>0.4`: "Engage but don't overshare."
- `>0.15`: "Keep responses short and guarded."
- `<0.15`: 不调用 LLM——模板足够

### 6.2 话题来源: 景色 + 时事

- **景色**: 地标(个人/历史/美学/实用 4级连接)、天象(日落/彩虹/风暴)、野生动物(猎人/博物学者高兴趣)、季节变化

## 八、沉默模型

沉默是有效的交流。`SilenceTracker` 基于文化沉默容忍度 + 上一轮话语类型（问题/陈述/情感/应酬/命令/自言自语）决定沉默含义：

| 状态 | NPC 反应 |
|------|---------|
| Normal | 不说话——沉默正常 |
| Prolonged | 外向 NPC 可能主动打破 |
| Awkward | 高友善→温和打破；低友善→质问 |
| ConversationEnded | 自然结束→Closing Phatic |

## 十、对话取消/中断全时机

12 种结束原因：NaturalExhaustion / Farewell / SilencedToEnd / CombatStarted / UrgentThirdParty / EnvironmentalEmergency / DistanceDrifted / ActivitySilenced / PlayerCancelled / NpcGoalOverride / NpcEmotionalExit / LlmTimeout(fallback)

中断后：CombatStarted→保存快照("刚才说到一半——")；DistanceDrifted(>50m)→保存快照可能过期；ActivitySilenced(Sneaking)→立即静音

## 十一、安全边界

- LLM 永远不能直接修改游戏状态——只能产出文本
- ActionLibrary 封闭——LLM 只能选预定义行动 ID
- EntityValidator——引用必须存在
- PersonalityValidator——不违反核心人格
- ContentFilter——拦截不当内容
- 全部 LLM 输出记录日志
- 模板永远作为 fallback
