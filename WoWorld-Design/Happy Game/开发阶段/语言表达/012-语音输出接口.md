## 一、定位

语音输出（TTS）是语言表达管道的**输出端**——与 STT 输入端（[[007-自然语言理解与输入]]）对称。它不依赖 LLM：模板管道生成文本→TTS 朗读；LLM 生成文本→TTS 朗读。两条管道共用同一语音输出接口。

TTS 是 **Godot 侧的渲染层**（类似文字渲染），不在 Rust 模拟核心中执行音频合成。Rust 侧提供语音**元数据**（音色/语速/情绪修饰/优先级），Godot 侧负责实际播放。

## 二、VoiceProfile——NPC 语音身份

```rust
/// 每个 NPC 有一个 VoiceProfile——由种族/性别/年龄/个性派生
/// 存储在 NPC 的 LifeEntity 数据中，生成时确定
pub struct VoiceProfile {
    pub profile_id: VoiceProfileId,

    // ── 物理决定的音色 ──
    pub base_pitch: f32,           // 基础音高(Hz)——种族+性别+年龄
    pub pitch_variance: f32,       // 音高自然波动 0-1
    pub timbre: TimbrePreset,      // 音色预设
    pub base_speed: f32,           // 基础语速 词/分钟

    // ── 个性决定的说话风格 ──
    pub expressiveness: f32,       // 0=单调, 1=丰富语调
    pub breathiness: f32,          // 0=干净, 1=气声
    pub roughness: f32,            // 0=平滑, 1=粗砺

    // ── 来源选择 ──
    pub tts_source: TtsSource,
}

pub enum TimbrePreset { Deep, Bright, Husky, Crisp, Warm, Nasal, Breathy }
```

### 2.1 语音身份生成

```rust
impl VoiceProfile {
    pub fn generate(npc: &NpcData) -> Self {
        let race = npc.life_entity.race();
        let gender = npc.life_entity.gender();
        let age = npc.life_entity.age_years();

        // 物理基线
        let base_pitch = race.base_pitch_hz()
            * gender.pitch_modifier()       // 男×0.7, 女×1.0
            * age.pitch_modifier();          // 儿童×1.3, 老年×0.85

        // 个性修饰
        let expressiveness = npc.personality.extraversion * 0.7
            + npc.personality.openness * 0.3;
        let roughness = (1.0 - npc.personality.agreeableness) * 0.5
            + npc.life_tenacity_factor();  // 生活经历→声音粗砺
        let breathiness = npc.personality.neuroticism * 0.4;

        VoiceProfile {
            profile_id: VoiceProfileId::new(),
            base_pitch, pitch_variance: 0.1,
            timbre: race.default_timbre().merge(gender.timbre_bias()),
            base_speed: 150.0,  // 词/分钟基线
            expressiveness, breathiness, roughness,
            tts_source: TtsSource::default_for(race, gender),
        }
    }
}
```

## 三、情绪驱动的语音修饰

```rust
/// 每轮发言时，基于 NPC 当前情绪动态调整语音参数
pub struct VoiceEmotionModulation {
    pub pitch_shift: f32,          // 半音偏移——高兴+2, 悲伤-3
    pub speed_multiplier: f32,     // 语速——愤怒1.3, 恐惧1.5, 悲伤0.7
    pub volume_multiplier: f32,    // 音量——愤怒1.4, 恐惧0.6
    pub tremor: f32,               // 颤抖 0-1——恐惧0.6, 悲伤0.3
    pub breathiness_add: f32,      // 额外气声——疲惫0.5
}

impl VoiceEmotionModulation {
    pub fn from_emotion(emotion: &EmotionState) -> Self {
        VoiceEmotionModulation {
            pitch_shift: emotion.joy * 2.0 - emotion.sadness * 3.0 + emotion.anger * 1.5,
            speed_multiplier: 1.0 + emotion.anger * 0.3 + emotion.fear * 0.5 - emotion.sadness * 0.3,
            volume_multiplier: 1.0 + emotion.anger * 0.4 - emotion.fear * 0.4 - emotion.sadness * 0.2,
            tremor: emotion.fear * 0.6 + emotion.sadness * 0.3,
            breathiness_add: emotion.sadness * 0.3,
        }
    }
}
```

## 四、VoicePacket——Rust→Godot 统一语音包

```rust
#[derive(Clone)]
pub struct VoicePacket {
    pub text: String,                          // 要朗读的文本
    pub voice_profile_id: VoiceProfileId,      // 音色身份
    pub emotion_mod: VoiceEmotionModulation,   // 情绪修饰
    pub delivery: VoiceDelivery,               // 说话模式
    pub priority: VoicePriority,               // 优先级
}

pub enum VoiceDelivery {
    Normal,       // 正常朗读
    Whispered,    // 耳语——降音量+气声
    Shouted,      // 喊叫——提音量+可能失真
    Internal,     // 内心独白——仅玩家可听
}

pub enum VoicePriority {
    Critical,     // 战斗警告/求救——打断当前语音
    High,         // 回应玩家的对话
    Normal,       // 一般对话
    Ambient,      // NPC 间闲聊——排队
    Background,   // 背景旁白
}
```

### 4.1 集成位置

```
任何管道(模板/LLM) → NpcResponse { text, voice_packet, ... }
  │
  ▼
Godot DialogueRenderer:
  ① 渲染文字(头顶气泡/对话UI)
  ② 如果 tts_config.enabled:
      VoiceManager::enqueue(packet)
        ├→ 检查 priority→高优先打断当前
        ├→ 选 TTS 引擎→基于 voice_profile_id
        └→ engine.speak(packet)→音频播放
  ③ 字幕模式检查
```

pub trait TtsEngine {
    fn name(&self) -> &str;
    fn is_available(&self) -> bool;

## 六、VoiceManager——语音播放管理

```rust
pub struct VoiceManager {
    engine: Box<dyn TtsEngine>,
    config: TtsConfig,
    queue: VecDeque<VoicePacket>,
    current: Option<VoicePacket>,
}

impl VoiceManager {
    /// 入队播放——高优先级打断当前
    pub fn enqueue(&mut self, packet: VoicePacket) {
        if packet.priority == VoicePriority::Critical {
            self.engine.stop();
            self.queue.push_front(packet);
        } else {
            self.queue.push_back(packet);
        }
        self.try_play_next();
    }

    /// 旅伴对话专用——低音量快速播放，不打断游戏
    pub fn enqueue_travel(&mut self, packet: VoicePacket) {
        let mut p = packet;
        p.priority = VoicePriority::Ambient;
        self.queue.push_back(p);
    }

    fn try_play_next(&mut self) { /* 消耗队列播放 */ }
}
```

## 七、与语言表达模块各管道的集成

| 管道 | VoicePacket 来源 | 优先级 |
|------|-----------------|--------|
| 正式对话(模板) | NpcResponse.voice_packet | High/Normal |
| 正式对话(LLM) | NpcResponse.voice_packet | High/Normal |
| 旅伴对话 | TravelUtterance → VoicePacket(Ambient) | Ambient |
| NPC 间对话 | 渲染时生成 VoicePacket | Ambient |
| 战斗通讯 | CombatComms → VoicePacket(Critical) | Critical |
| 快速问候 | PhaticFragment → VoicePacket | Normal |
| 环境字幕 | 文本→VoicePacket | Background |
