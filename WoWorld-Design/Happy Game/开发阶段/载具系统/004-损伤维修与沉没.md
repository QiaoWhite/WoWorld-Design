## 一、三通道损伤模型

对标战斗系统 012 的部位伤害（轻/中/重伤三档渐进），载具损伤也是**连续值 + 非线性软阈值**。

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct VehicleDamageState {
    /// 船体完整性 1.0(完好) → 0.0(解体/沉没)
    pub hull: f32,
    /// 推进系统 1.0 → 0.0(完全无法移动)
    pub propulsion: f32,
    /// 操控系统 1.0 → 0.0(舵卡死/失控漂移)
    pub steering: f32,
    /// 紧急修补——临时修复随时间衰退
    pub emergency_patches: Vec<EmergencyPatch>,
}
```

### 1.1 为什么是三个独立通道

| 通道 | 损伤来源 | 维修方式 | 对航行的影响 |
|------|---------|---------|------------|
| **hull** | 风暴巨浪、触礁、炮击、冰封 | 船体修补/抽水（紧急）；船坞大修（港口） | 进水→减速→沉没 |
| **propulsion** | 引擎过载、桅杆折断、桨损坏、炮击 | 引擎修复/换帆/补桨（紧急）；港口全面检修 | 速度降低→无法移动 |
| **steering** | 舵被击中、风暴冲击舵链、触礁 | 临时舵/舵链修复（紧急）；港口换舵 | 偏航→失控漂移 |

三个通道同时受损的场景——暴风雨中触礁：hull(礁石) + propulsion(浪打坏帆) + steering(舵卡住)。千疮百孔地爬向港口——这是航海故事的黄金时刻。

### 1.2 损伤效果函数

```rust
impl VehicleDamageState {
    /// 速度惩罚——从 hull 和 propulsion 取最差情况
    pub fn speed_penalty(&self) -> f32 {
        soft_threshold(self.hull).min(soft_threshold(self.propulsion))
    }

    /// 操控惩罚——仅从 steering
    pub fn steering_penalty(&self) -> f32 {
        soft_threshold(self.steering)
    }
}

/// 软阈值——连续值到惩罚因子
/// 1.0 → 1.0, 0.7 → ~0.85, 0.4 → ~0.45, 0.1 → ~0.08, 0.0 → 0.0
fn soft_threshold(value: f32) -> f32 {
    0.1 * smoothstep(0.1, 0.4, value)
        + 0.4 * smoothstep(0.4, 0.7, value)
        + 0.5 * value
}

/// 平滑阶梯函数——在 edge0 和 edge1 之间三次 Hermite 插值
fn smoothstep(edge0: f32, edge1: f32, x: f32) -> f32 {
    let t = ((x - edge0) / (edge1 - edge0)).clamp(0.0, 1.0);
    t * t * (3.0 - 2.0 * t)
}
```

## 二、损伤来源

```rust
/// 对载具施加损伤
/// 无论损伤来源是什么——风暴/战斗/触礁——统一走此方法
fn apply_damage(
    vehicle: &mut VehicleState,
    amount: f32,          // 0-1 的损伤量（如 0.05 = 5%）
    channel: DamageChannel,
    source: DamageSource,
) {
    let resistance = match channel {
        DamageChannel::Hull => {
            // 风暴抗性或船体本身的耐受力
            match &source {
                DamageSource::Storm => vehicle.def.damage_profile.storm_resistance,
                DamageSource::Ice => vehicle.def.damage_profile.ice_resistance,
                _ => 0.0,  // 炮击/触礁——无抗性
            }
        }
        _ => 0.0,
    };

    let effective_damage = amount * (1.0 - resistance);

    match channel {
        DamageChannel::Hull => vehicle.damage.hull = (vehicle.damage.hull - effective_damage).max(0.0),
        DamageChannel::Propulsion => vehicle.damage.propulsion = (vehicle.damage.propulsion - effective_damage).max(0.0),
        DamageChannel::Steering => vehicle.damage.steering = (vehicle.damage.steering - effective_damage).max(0.0),
    }
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum DamageSource {
    Storm,
    Ice,
    ReefGrounding,
    CombatArtillery,
    CombatBoarding,
    Fire,
    WearAndTear,     // 正常磨损
    Sabotage,        // 人为破坏
}
```

## 三、维修

### 3.1 紧急修补——海上/野外

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct EmergencyPatch {
    /// 修补时间
    pub applied_at: GameTimestamp,
    /// 修补了哪个通道
    pub channel: DamageChannel,
    /// 临时恢复的量（最多 0.3）
    pub restored_amount: f32,
    /// 衰退速率（每小时）
    pub decay_rate_per_hour: f32,
    /// 谁修的
    pub applied_by: NpcId,
    /// 修补时使用的技能
    pub skill_used: f32,
    /// 消耗的材料
    pub materials_used: Vec<(ItemDefId, u32)>,
}

/// 执行紧急修补
fn perform_emergency_patch(
    vehicle: &mut VehicleState,
    channel: DamageChannel,
    npc: NpcId,
    materials: &[ItemEntId],
) -> Result<EmergencyPatch, PatchError> {
    let skill = query_npc_skill(npc, relevant_skill(channel));
    let vitals = query_npc_vitals(npc);

    // 技能 → 恢复量（0.05-0.3）
    let restored = 0.05 + 0.25 * (skill / 100.0).min(1.0);

    // 技能 → 衰退速率（高技能修的持久）
    let base_decay = 0.05;  // 每小时衰退 5%（最差）
    let decay = base_decay * (1.0 - 0.7 * (skill / 100.0).min(1.0));

    // 疲劳影响品质
    let fatigue_mod = 1.0 - 0.3 * vitals.fatigue;
    let restored = restored * fatigue_mod;

    // 应用
    let patch = EmergencyPatch {
        applied_at: current_game_time(),
        channel,
        restored_amount: restored,
        decay_rate_per_hour: decay,
        applied_by: npc,
        skill_used: skill,
        materials_used: materials.iter().map(|m| (m.def_id(), 1)).collect(),
    };

    vehicle.damage.emergency_patches.push(patch.clone());

    // 临时恢复——不改变原始损伤值，只是叠加补偿
    // 实际有效值 = damage[channel] + Σ active_patches.restored

    Ok(patch)
}
```

特点：
- 任何船员都可以尝试——不需要特殊场所
- 恢复量有限：0.05（菜鸟）~ 0.30（大师）
- 会随时间衰退：12-72 小时后回到原状
- 重复修补效果递减——同一个洞补三次已经不行了
- 技能检定：木工/工程/魔法工程（对应通道不同）

### 3.2 港口维修

- 需要：有船坞设施（P6 WFC 建筑模板中的 Shipyard）+ 有 Shipwright 技能的 NPC
- 全量恢复：三通道全部修复到 1.0
- 时间：与损伤程度 × 载具规模成正比（数小时到数天）
- 成本：材料费用 + 工匠工资——走经济系统
- 对标物品系统的铁匠维修——同一套模式

## 四、沉没与搁浅

### 4.1 流程

```
hull = 0.0
  │
  ├─ 检查水深
  │   ├─ depth ≤ vehicle.def.min_water_depth_m → 搁浅
  │   │   └─ 载具不会消失——成为可探索的残骸
  │   │       在海岸/浅滩/岩礁上
  │   │
  │   └─ depth > min_water_depth_m → 开始沉没
  │       ├─ 沉没速度 = f(size_class, 水深)
  │       │   大船沉得慢（数十分钟-数小时），小船快（数分钟）
  │       │
  │       ├─ 船员/乘客行为
  │       │   ├─ GOAP: AbandonShip（船长下令）
  │       │   ├─ GOAP: SavePassengers（船员协助乘客）
  │       │   ├─ GOAP: SalvageCargo（抢救货物）
  │       │   └─ NPC 跳入救生艇（如果配备了）或游泳逃生
  │       │
  │       └─ 最终: VehicleStatus::Wrecked
  │           ├─ VehicleId 保留但标记为 destroyed
  │           ├─ 位置记录在 Wrecked.location
  │           ├─ 历史系统记录事件
  │           └─ 如果是深水 → 灵元素印记(AetherImprint)
  │               → 沉船遗址 → 后世可探索/打捞
  │
  └─ 契书 → 变成废纸（契书持有者失去了载具——权力系统记录）
```

### 4.2 历史印记

```
沉船是一个"历史锚点":
  1. 历史系统记录事件: "沧海号在暴风雨中沉没于东海深处"
  2. 灵元素印记: 沉船位置留下 AetherImprint
     → 后世施法者/冒险者可以感知到"这里发生了什么"
  3. 物品遗产: 船上货物、船员遗物——全部留在海底
     → 后世的打捞者可能找到这些物品
     → ItemEntId 的 provenance 链条上添加 "沉于东海, 打捞于XX年"
  4. 船员命运:
     → 幸存者: 记忆中留下创伤 + "那天失去了船"
     → 遇难者: 墓碑（如果有遗体）/ 海中失踪
```
