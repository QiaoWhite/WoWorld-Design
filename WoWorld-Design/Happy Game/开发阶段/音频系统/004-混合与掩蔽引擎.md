# 004 — 混合与掩蔽引擎 v1.0

> **核心职责**：一堆声音到达听者的耳朵后——哪些能分辨出来、哪些糊成一片、哪个占据主导、最终的声景是什么。掩蔽引擎回答"NPC A 能听清 NPC B 说什么吗？"——直接影响 NPC 的对话理解和记忆写入。

> **性能关键**：广播式声场计算——每个有听者的 cell 计算一次，非 per-listener 全量。

## §一 频段分割

所有声音按频率分三频段独立处理掩蔽：

| 频段 | 范围 | 典型声音 | 被什么掩蔽 |
|------|------|---------|-----------|
| Low | <250Hz | 引擎/脚步低频/打雷/大鼓 | 低频环境（河水/瀑布） |
| Mid | 250-4000Hz | **人声**/大部分乐器/动物鸣叫 | 中频噪声（人群嗡嗡/雨声） |
| High | >4000Hz | 金属碰撞/箭矢呼啸/虫鸣 | 高频噪声（风/树叶沙沙） |

**掩蔽是 per-band 的**：低频噪声不影响中频声音的清晰度。

## §二 掩蔽模型

    SimpleThreshold { threshold_db: f32 },     // 单阈值

### 2.2 默认：频段对比

```rust
fn compute_clarity(
    source_per_band: [f32; 3],   // 声源在每个频段的 SPL
    ambient_per_band: [f32; 3],  // 环境噪声在每个频段的 SPL
    config: &MaskingConfig,
) -> SoundClarity {
    let mut band_clarity = [1.0; 3];

    for band in 0..3 {
        let diff = source_per_band[band] - ambient_per_band[band];

        if diff < config.full_mask_threshold_db {      // 默认 -15dB
            band_clarity[band] = 0.0;  // 完全掩蔽
        } else if diff < config.partial_mask_threshold_db {  // 默认 -5dB
            // 部分掩蔽——线性插值 (0.0 ~ 1.0)
            band_clarity[band] = (diff - config.full_mask_threshold_db)
                / (config.partial_mask_threshold_db - config.full_mask_threshold_db);
        } else {
            band_clarity[band] = 1.0;  // 完全清晰
        }
    }

    // 总体清晰度——中频权重最大（人声理解的关键频段）
    let overall = band_clarity[0] * 0.2  // 低频
                + band_clarity[1] * 0.6  // 中频
                + band_clarity[2] * 0.2; // 高频

    SoundClarity { overall, per_band: band_clarity }
}
```

## §三 话语清晰度

### 3.1 五档映射

```rust
fn speech_clarity_level(clarity: &SoundClarity) -> SpeechClarityLevel {
    let score = clarity.per_band[1] * 0.8 + clarity.overall * 0.2;
    match score {
        s if s > 0.9 => SpeechClarityLevel::Perfect,
        s if s > 0.7 => SpeechClarityLevel::Clear,
        s if s > 0.4 => SpeechClarityLevel::Garbled,
        s if s > 0.1 => SpeechClarityLevel::Fragment,
        _ => SpeechClarityLevel::Inaudible,
    }
}
```

### 3.2 清晰度→文本模糊化

| 档位 | clarity 范围 | 语言表达 resolve_partial 返回 | 听者体验 |
|------|-------------|------------------------------|---------|
| Perfect | ≥0.9 | 完整文本截取到 fraction | "我昨天在森林里发现了一..." |
| Clear | 0.7-0.9 | 完整文本，偶尔边缘模糊 | "我昨-天在森林里发现了..." |
| Garbled | 0.4-0.7 | 关键词保留，连接词丢失 | "我...森林...发现...废弃...神殿" |
| Fragment | 0.1-0.4 | 仅主题方向 | "好像说了关于森林的什么" |
| Inaudible | <0.1 | 无内容——仅方位 | "隐约听到说话声（东边）" |

**关键**：语言表达模块的 `resolve_partial(fraction, clarity_level)` 根据清晰度级别返回不同质量的文本。这不只是"截断到 fraction"——低清晰度时人工分词和"填空"会出错。

### 4.2 人群噪声模型

```rust
/// 人群噪声 = f(附近 NPC 数量, 距离, 活动类型)
/// 对数密度缩放——不是简单求和（避免 50 人 = 震耳欲聋）
fn crowd_noise(nearby_npcs: &[(EntityId, Vec3, f32)]) -> [f32; 3] {
    let mut total_mid = 0.0;
    for (_, _, dist) in nearby_npcs {
        if *dist > 20.0 { continue; }
        let spl = 40.0 - 20.0 * dist.log10().max(0.1);  // 每人 ~40dB @ 1m
        total_mid += db_to_linear(spl);
    }
    let n = nearby_npcs.len().max(1) as f32;
    let density_db = (n.log10() / 2.0_f32.log10()) * 3.0;  // 每 10 倍人 +3dB
    [
        linear_to_db(total_mid * 0.03) + density_db,  // 低频：脚步声
        linear_to_db(total_mid) + density_db,          // 中频：人声
        linear_to_db(total_mid * 0.1) + density_db,   // 高频：偶尔的金属碰撞
    ]
}
```

### 4.3 昼夜差异

| 时段 | 野外 | 村庄 | 大城市 |
|------|------|------|--------|
| 深夜 (0-2h) | -10dB | -8dB | -5dB |
| 其余夜间 | -8dB | -5dB | -3dB |
| 清晨 | -5dB | -3dB | -1dB |
| 白天 | -3dB | 0dB | +0dB |
| 黄昏 | -3dB | -2dB | -1dB |

## §五 声场快照（广播式计算）

### 5.1 关键优化

**不是每个听者独立全量计算。** 先对存在听者的 cell 构建声场快照 → 听者只在快照中查询。

```rust
struct SpatialSoundField {
    cells: HashMap<CellCoord, CellNoise>,
    loudest_per_cell: HashMap<CellCoord, Vec<SoundEventRef>>,
}

struct CellNoise {
    ambient_per_band: [f32; 3],
    dominant_source: Option<SoundEventRef>,
    source_count: u16,
}

impl SpatialSoundField {
    fn build(events: &[SoundEvent], listener_positions: &[Vec3]) -> Self {
        let mut field = Self::default();
        for listener in listener_positions {
            let cell = CellCoord::from_pos(*listener);
            if field.cells.contains_key(&cell) { continue; }
            let nearby = events.iter().filter(|e| distance(e.pos, cell.center()) < 200.0);
            let ambient = compute_ambient(&nearby, cell.center());
            let loudest = nearby.sorted_by_spl().take(10);
            field.cells.insert(cell, CellNoise { ambient, dominant: loudest.first(), source_count: nearby.count() });
            field.loudest_per_cell.insert(cell, loudest);
        }
        field
    }
}
```

## §六 AcousticMix 输出

```rust
pub struct AcousticMix {
    /// 清晰可辨的独立声音（最多 5-7 个——认知负荷上限）
    pub distinct: SmallVec<[PerceivedSound; 7]>,

    /// 不可分辨的背景噪声——被掩蔽声音坍缩为统计摘要
    pub ambient: AmbientNoiseLevel,

    /// 当前主导声音
    pub dominant: Option<SoundEventId>,

    /// 当前声学空间
    pub acoustic_space: AcousticSpace,
}

pub struct AcousticSpace {
    pub reverb_profile: ReverbProfile,
    pub occlusion: f32,            // 0-1
    pub room_size_m3: f32,
    pub surface_absorption: f32,   // 0-1
}

pub enum ReverbProfile {
    Outdoors,      // 无混响
    SmallRoom,     // 0.2-0.5s
    MediumHall,    // 0.5-1.0s
    LargeHall,     // 1.0-2.0s（石制教堂）
    Cathedral,     // 2.0-4.0s
    Cave,          // 不规则
    Underwater,    // 完全不同
}
```

**AcousticSpace 来源**：世界生成在建筑上设置 `AcousticTag`（SmallRoom/StoneCathedral/WoodTavern），音频模块在运行时展开为 `AcousticSpace`（叠加当前人数、天气等动态因素）。

### 主导声音选举

```rust
fn elect_dominant(sounds: &[PerceivedSound], attention: Option<EntityId>) -> Option<SoundEventId> {
    // 1. 当前注意焦点（正在对话的NPC）→ 即使不是最响
    if let Some(focus) = attention {
        if let Some(focused) = sounds.iter().find(|s| s.source_entity == Some(focus)) {
            return Some(focused.event_id);
        }
    }
    // 2. 最高 SPL
    sounds.iter().max_by_key(|s| (s.perceived_spl * 100.0) as u32).map(|s| s.event_id)
}
```
