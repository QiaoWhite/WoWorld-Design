## §一 原则与模式

### 1.1 权威原则

音频模块是本模块所有概念的唯一权威来源。当其他模块中存在与音频相关的设计时，**以音频模块的设计为准**——不删除原有设计（除非是冲突修正），而是建立正确的引用/派生/映射关系。

### 1.2 四个标准模式

| 模式 | 说明 | 例子 |
|------|------|------|
| **Schema 存储** | 音频定义结构 → 其他模块存储不解 | `AudioMaterial`（音频定义枚举，物品存储） → 对标 `ConsumableEffect` |
| **Trait 定义在消费方** | 音频定义 trait → 其他模块 impl | `HasSoundEmitter`（音频定义，各模块实现） → 对标 `HasAestheticSignal` |
| **查询 trait** | 音频消费已有模块的数据 | `WeatherQuery::sample()` / 物品查询 / Vitals 查询 |
| **事件总线** | 各模块发射事件 → 音频订阅 | `PlayerNotificationEvent` → `world_core` → 音频消费 |

## §三 音频模块 OWN 的概念清单

以下概念以音频模块为**唯一权威 Owner**。其他模块中若有涉及，应标注引用音频模块为权威：

| 概念 | 说明 | 其他模块如有自制版本→处理方式 |
|------|------|------------------------------|
| `AudioMaterial` 枚举 | 物品/装备的声学材质分类 | 物品系统存 `Option<AudioMaterial>` 字段 |
| `SurfaceMaterial` 枚举 | 地表的声学材质分类 | 世界生成定义具体值 |
| `HearingModel` | 听觉感知参数（灵敏度/频率范围/方向性） | Life 004 已有的 perception→标注引用音频模块 |
| `VoiceProfile`（声学参数） | NPC 语音的声学身份 | 语言表达 012 → 迁至音频模块，语言表达保留 TTS 渲染 |
| `VoiceEmotionModulation` | 情绪→声学参数调制 | 迁至音频模块 |
| `audible_radius()` | 声音传播距离计算公式 | 语言表达 011 → 迁至音频模块 |
| `SoundEvent` 结构 | 世界中声音事件的完整定义 | 唯一权威 |
| 传播模型（衰减/吸收/遮挡） | 声音的物理传播全部规则 | 唯一权威 |
| 掩蔽模型 | 声音叠加/覆盖/清晰度规则 | 唯一权威 |
| `AudioQuery` trait | 全部模块获取声音数据的唯一入口 | 唯一权威 |
| `AudioRenderPacket` | 跨 GDExtension 的播放指令格式 | 唯一权威 |
| `AudioAssetId` | 音频资产的全局标识 | 唯一权威 |
| 音乐系统规则（情境→预设映射） | BGM 的动态切换规则 | 唯一权威——文化模块定义节日音乐预设，但映射/过渡/分层在音频 |
| 提示音规则表 | 事件→提示音映射 | 唯一权威 |

## §四 各模块详细接口（17 模块逐个枚举）

### 4.1 NPC 模块

**NPC 模块存储/写入（woworld_types 类型）**：
- `ActionRingBuffer`：每 NPC 一个，记录最近 64 个瞬时动作（NPC 本来就需要——决策器"刚才在干什么"、记忆系统"发生了什么"）
- `HearingModel`：NPC 的听觉参数（从种族+大五人格派生——对标 Physiology 从 Vitals 派生）
- `CurrentSpeech`：当 NPC 在说话时设置（语言表达模块通过 NPC 模块写入）
- `SilenceIntent`：当 NPC 在潜行/躲藏/装死时设置
- `VoiceProfile`：NPC 的语音声学身份（从种族×性别×年龄×大五人格生成）
- `HasSoundFootprint` trait impl：暴露 `SoundFootprint`

**NPC 模块消费（通过 AudioQuery）**：
- `perceived_sounds()`：感知决策——"我听到了威胁/熟人/有趣的事"
- `perceived_speech()`：获取 `fraction_heard` → 调语言表达 `resolve_partial()` → 写入记忆
- `ambient_noise_level()`：影响 `effective_audible_radius` 判断

**需要修改的 NPC 文档**：
- `NPC活人感开发文档ver2.0.md`：`NpcData` 新增字段 `action_log: ActionRingBuffer`、`current_speech: Option<CurrentSpeech>`、`silence: Option<SilenceIntent>`、`hearing: HearingModel`、`voice_profile: VoiceProfile`
- 感官外壳部分的"听觉"标注以音频模块为权威

### 4.2 Life 模块（动物）

**Life 模块存储/写入（woworld_types 类型）**：
- `SoundFootprint`：每生物维护（表面接触+运动状态）
- `CreatureSoundProfile`：程序化生物声音参数（archetype/body_mass/vocal_organ/freq_range/temporal_pattern）
- `ContinuousSound`（通过 `HasSoundEmitter`）：持续发声（心跳/呼吸/翅膀嗡嗡）
- `HasSoundEmitter` trait impl

**Life 模块消费（通过 AudioQuery）**：
- `perceived_sounds()`：动物听到威胁→逃跑/警戒
- `echo_query()`（预留）：蝙蝠/海豚回声定位

**已有并保持的设计**：
- Life 003 Dimension 4（头部与感官·听觉类型）：生命模块保持听觉解剖学定义——"有什么耳朵结构"。音频模块定义 `HearingModel` 的参数如何从解剖学派生
- Life 003 Dimension 12（声音与通信·发声方式+声音类型）：生命模块保持生物发声**能力**的定义——"能发出什么类型的声音"。音频模块消费这些参数映射到具体 `VocalType`
- Life 004：`perception` 属性影响听觉灵敏度→音频模块消费

**需要修改的 Life 文档**：
- Life 003：标注"声学传播/感知规则以音频模块为权威"
- Life 009：植物 `LeafSoundProfile` → 标注"被音频模块消费为 EnvironmentSoundSource"

### 4.3 战斗模块

**战斗模块存储/写入（woworld_types 类型）**：
- `SoundFootprint`：挥砍/格挡/受击/箭矢飞行
- `SilenceIntent`：沉默法术设置
- `ActionRingBuffer` 条目：战斗相关 ActionAtom

**战斗模块消费（通过 AudioQuery）**：
- `perceived_sounds()`：战术信息——"敌人的位置被脚步声暴露了"、"有法术在吟唱"
- `PerceivedSound.threat_hint`：加速威胁评估

**需要修改的战斗文档**：
- 战斗 004（战斗信息与感知）：`heard_sounds: Vec<PerceivedSound>` → 改为通过 `AudioQuery::perceived_sounds()` 获取。标注"听觉信息格式以音频模块为权威"

### 4.4 魔法模块

**魔法模块存储/写入（woworld_types 类型）**：
- `SoundFootprint`：施法咏唱/爆炸/传送/召唤
- `HasSoundEmitter` impl（魔法结界持续嗡嗡声/附魔光芒声）
- `SilenceIntent::MagicallySilenced`：沉默法术设置此状态在目标实体上
- 雷元素魔法：调节 `SoundFootprint` 的传播参数（通过音频模块的特化 API——`AudioMut::set_region_silence()`）

**魔法模块消费（通过 AudioQuery）**：
- `perceived_sounds()`：检测远处魔法活动
- `speed_of_sound()`：声波魔法利用介质速度差异

**需要修改的魔法文档**：
- 魔法 002（十元素）：雷元素 §声音/振动 → 标注"声音传播规则以音频模块为权威，魔法模块只定义雷元素如何影响传播"
- 沉默法术 → 标注"沉默效果的声学实现以音频模块为权威"

### 4.5 语言表达模块

**语言表达模块存储/写入（woworld_types 类型）**：
- `CurrentSpeech`：写入 `NpcData.current_speech`（语言表达不知道音频在读）
- `ExpressionQuery::resolve_partial(fraction, clarity)`：提供部分文本解析

**语言表达模块消费（通过 AudioQuery）**：
- `perceived_speech()`：获取 `fraction_heard` + `clarity` → 决定调 `resolve_partial` 的参数

**所有权变更（从语言表达迁至音频）**：
- `VoiceProfile`（声学参数）→ 迁至音频模块
- `VoiceEmotionModulation` → 迁至音频模块
- `VoiceManager`（播放队列+打断）→ 迁至音频模块
- `audible_radius` 六因子公式 → 迁至音频模块
- `effective_audible_radius()` → 迁至音频模块

**保留在语言表达**：
- `TtsEngine` trait（TTS 渲染——非模拟层）
- `TtsConfig`（玩家偏好设置）
- 对话系统本身（话题/流转/NLU/文本生成/社交层）
- `ExpressionRef` 与 `ContentResolver`

**需要修改的语言表达文档**：
- 012-语音输出接口：标注 `VoiceProfile`/`VoiceEmotionModulation`/`VoiceManager`/`audible_radius` → 迁至音频模块。保留 TTS 渲染部分
- 011-LLM增强层：`effective_audible_radius()` → 标注以音频模块为权威
- 009-信息传播系统：`ConversationMode` 的 `audible_radius_m` → 标注以音频模块为权威
- 005-对话系统：打断机制 → 标注以音频模块的 CancelReason 为权威

## §七 AudioQuery trait 完整方法清单

```rust
pub trait AudioQuery: Send + Sync {
    // === 感知查询（per-listener, per-frame） ===

    /// 听者感知到的所有声音（瞬时+持续+环境——统一处理）
    /// time_window: 过去 N 秒内的声音（默认 1s）
    fn perceived_sounds(&self, listener_pos: Vec3, hearing: &HearingModel,
                        time_window: Duration, attention: &AttentionFilter,
                        head_medium: Medium) -> Vec<PerceivedSound>;

    /// 对特定说话者的话语感知状态（含时间维度——fraction_heard + clarity）
    fn perceived_speech(&self, listener_pos: Vec3, speaker: EntityId,
                        now: GameInstant, hearing: &HearingModel) -> Option<SpeechPerception>;

    /// 当前位置的环境噪声水平（per-band: [low, mid, high]）
    fn ambient_noise_level(&self, pos: Vec3, time: GameInstant) -> AmbientNoiseLevel;

    /// 当前位置的声学空间（混响/遮挡/吸收）
    fn acoustic_space(&self, pos: Vec3) -> AcousticSpace;

    // === 传播查询 ===

    /// 声音在给定介质+温度下的速度 (m/s)
    fn speed_of_sound(&self, medium: Medium, temperature: f32) -> f32;

    /// 给定强度/频率/介质的声音可听半径 (m)
    fn audible_radius(&self, intensity_db: f32, freq: &FrequencyProfile,
                      medium: Medium, weather: &WeatherSample) -> f32;

    /// 介质边界衰减 (dB) —— 空气↔水，空气↔固体等
    fn medium_boundary_loss(&self, source_medium: Medium, head_medium: Medium) -> f32;

    /// 频率选择性透过率（听者头部介质过滤）
    fn head_medium_filter(&self, freq: &FrequencyProfile, head_medium: Medium) -> f32;

    // === 资产/映射查询 ===

    /// 材质+交互+表面 → 声音资产 ID + 播放参数
    fn material_sound(&self, material: AudioMaterial, interaction: InteractionType,
                      surface: SurfaceMaterial) -> SoundDef;

    /// 地表材质+步态 → 脚步声音
    fn footstep_sound(&self, surface: SurfaceMaterial, gait: GaitStyle,
                      is_wet: bool) -> FootstepSoundDef;

    /// 当前位置的环境音效定义
    fn ambient_profile(&self, pos: Vec3, time: GameInstant, indoor: bool) -> AmbientProfile;

    /// 程序化生物声音参数 → 变体播放参数（pitch/EQ/包络）
    fn creature_sound_variant(&self, profile: &CreatureSoundProfile) -> SoundVariantParams;

    /// 环境声源查询（河流/瀑布/海洋/火山——静态放置）
    fn environment_sound_sources(&self, pos: Vec3, radius: f32) -> Vec<EnvironmentSoundSource>;

    // === 音乐查询 ===

    /// 当前位置+情境的音乐状态
    fn music_state(&self, pos: Vec3, context: MusicContext) -> MusicState;

    /// 当前主导音乐源
    fn dominant_music_source(&self) -> MusicSource;

    /// 玩家音乐配置
    fn player_music_config(&self) -> PlayerMusicConfig;

    /// 世内演奏者列表（当前位置可闻的）
    fn nearby_inworld_performers(&self, pos: Vec3, radius: f32) -> Vec<InWorldPerformer>;

    // === 元数据查询 ===

    /// 给定实体当前是否在说话
    fn is_speaking(&self, entity: EntityId) -> bool;

    /// 给定实体的语音 Profile
    fn voice_profile(&self, entity: EntityId) -> Option<&VoiceProfile>;

    /// 实体是否处于主动沉默状态
    fn is_silenced(&self, entity: EntityId) -> Option<SilenceIntent>;

    /// 活动 SoundEvent 计数（调试/监控）
    fn active_event_count(&self) -> u32;

    /// 全局环境噪声基准（调试/监控）
    fn global_noise_baseline(&self) -> f32;

    // === 无障碍查询 ===

    /// 当前位置应显示的声源方向指示器
    fn direction_indicators(&self, listener_pos: Vec3, camera_forward: Vec3) -> Vec<DirectionIndicator>;

    /// 当前位置的字幕行
    fn subtitle_lines(&self, listener_pos: Vec3, hearing: &HearingModel) -> Vec<SubtitleLine>;

    // === 预留 ===

    /// 回声定位查询（蝙蝠/海豚）—— 返回"感知到"的空间形状
    fn echo_query(&self, emitter_pos: Vec3, direction: Vec3, frequency: f32,
                  max_range: f32) -> EchoResponse;  // 预留——v1.0 返回空

    // === 便捷/聚合查询 ===

    /// 快速检查：给定实体当前发出的声音是否能到达听者位置
    fn is_audible(&self, entity: EntityId, listener_pos: Vec3) -> bool;

    /// 传播延迟查询——不执行完整传播计算，仅返回延迟
    fn propagation_delay(&self, source_pos: Vec3, listener_pos: Vec3,
                         medium: Medium, temperature: f32) -> Duration;

    /// 当前位置的当前主导声音（单一最显著声源）
    fn dominant_sound(&self, listener_pos: Vec3) -> Option<SoundEventId>;

    /// 当前活跃说话者数量（调试/监控）
    fn ongoing_speech_count(&self) -> u32;

    /// L2 聚落音频聚合查询——对标 SettlementFaithSnapshot
    fn settlement_audio_aggregate(&self, settlement_id: u32) -> Option<SettlementAudioAggregate>;
}
```

**共 30 个方法。零分配。高频方法（perceived_sounds/speech/noise_level/dominant_sound）→ O(1) 缓存优化。低频方法（settlement_audio_aggregate/echo_query）→ 惰性计算。**

### AudioQuery 辅助类型

```rust
/// 注意力过滤器——由 NPC 感知/认知系统在调用 perceived_sounds() 前填入。
/// ★ 音频模块不管理注意力——选择性注意力（鸡尾酒会效应）属于 NPC 认知系统。
/// 音频模块只根据传入的 focus 做信号增强（主导声音选举时优先当前注意目标）。
#[derive(Clone, Default)]
pub struct AttentionFilter {
    /// 当前注意焦点实体——"我正在跟这个人说话"
    pub focus_entity: Option<EntityId>,
    /// 最低 SPL 阈值——低于此直接丢弃（注意力过滤，非物理阈值）
    pub min_spl_override: Option<f32>,
    /// 优先级：威胁 > 社交 > 新颖 > 其他
    pub threat_bias: f32,        // 0-1，威胁敏感度（恐惧高的 NPC 更高）
    pub novelty_bias: f32,       // 0-1，新颖敏感度（开放性高的 NPC 更高）
    pub ignore_self: bool,       // 忽略自己的声音
}

/// L2 聚落音频聚合——对标 SettlementFaithSnapshot。
/// 每游戏日惰性更新。当 L1 玩家进入该区域时展开为 per-NPC 数据。
#[derive(Clone)]
pub struct SettlementAudioAggregate {
    pub settlement_id: u32,
    pub updated_at: GameInstant,
    pub dominant_distant_sounds: Vec<DistantSoundReport>,
    pub ambient_disturbance: f32,  // 0-1 "有多吵"
}

#[derive(Clone)]
pub struct DistantSoundReport {
    pub event_type: SoundType,
    pub estimated_distance_km: f32,
    pub intensity_at_source_db: f32,
    pub perceived_count: u8,       // 今天听到了几次
    pub first_heard_at: GameInstant,
}
```
