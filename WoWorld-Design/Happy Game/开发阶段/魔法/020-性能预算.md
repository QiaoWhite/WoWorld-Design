# 020 — 魔法系统性能预算

> **开发代号**: WoWorld (Wonder World)
> **引擎**: Godot 4.6 LTS
> **模块**: 魔法系统 | **版本**: v1.0 | **CHG-048**: 交叉审计后追溯建立

## 一、计算模型

魔法系统的计算负载集中在 L1 NPC（~1000 人，含玩家）的施法行为上。L2-L3 NPC 通过统计模型批量处理，不触发逐人施法管线。L4 为纯统计无实例。

## 二、CPU 预算

### 2.1 每帧固定开销

| 操作 | 频率 | 每操作成本 | 总 CPU/帧 |
|------|------|-----------|----------|
| Mana 自然回复 tick | ~1Hz/魔法NPC (决策周期，非每帧) | ~3 f32 乘法 | <0.001ms |
| 元素亲和衰减 | 同决策周期 | 1 f32 指数衰减 | <0.001ms |
| 法术冷却递减 | 同决策周期 | BTreeMap 查询 + 1 f32 减法 | <0.001ms |
| **固定开销小计** | | | **<0.003ms** |

### 2.2 施法峰值开销

| 操作 | 频率 | 每操作成本 | 峰值 CPU/帧 |
|------|------|-----------|------------|
| 法术施放（完整管线） | 战斗突发: ~20 法术/帧 | <0.01ms | <0.2ms |
| 元素浓度采样 | 每施法 1 次 | Texture2D sample (GPU) | ~免费 |
| 元素交互矩阵查询 | 每施法（仅多元素法术） | 10×10 矩阵 f32 查表 | <0.001ms |
| 法术方程式求值 | 每施法 1 次 | 7 f32 乘法 + 条件分支 | <0.001ms |
| 过载检测 (spirit 安全边界) | 每施法 1 次 | 1 f32 比较 | <0.001ms |
| 施法自动化升级检查 | 每 100/300/500 次施法 | 1 u32 比较 | <0.001ms |
| 法阵碰撞检测 (AoE) | 每活跃法阵 | EntityIndex 查询 | <0.01ms (10 活跃法阵) |
| **施法峰值小计** | | | **<0.21ms** |

### 2.3 非帧级操作（加载/后台）

| 操作 | 频率 | 成本 | 备注 |
|------|------|------|------|
| 法术定义 TOML 加载 | 启动时 1 次 | <1ms | ~200 法术定义 |
| 魔导书生成 | 世界生成 P9 | 已计入世界生成预算 | — |
| 魔法物品附魔 | 物品实例化时 | 已计入物品系统预算 | — |
| 契约魔法结算 | 游戏日边界 (48min/日) | <1ms | 批量处理 |

**总 Rust CPU 预算**: **<0.25ms/帧**（含峰值）。正常帧 <0.01ms。

## 三、RAM 预算

| 数据 | 大小 | 备注 |
|------|------|------|
| 法术定义 TOML | ~100KB | ~200 法术 × ~500B |
| 元素交互矩阵 | 400B | 10×10 f32 |
| 法阵模板库 | ~50KB | ~50 模板 × ~1KB |
| 每 NPC Mana 状态 | 8B/NPC | `current_mana: f32` + `max_mana: f32` |
| 每 NPC 法术冷却 | ~200B/魔法NPC | BTreeMap (5-20 条目) |
| 每 NPC 已知法术 | ~500B/魔法NPC | ~10 法术 × ~50B |
| L2 NPC 魔法统计 | ~5MB | 100K L2 × ~50B 聚合 |
| 元素浓度缓存 | ~10MB | 预计算纹理，在世界数据预算中 |
| **RAM 总计** | **<20MB** | 100K NPC 全为魔法用户的极端情况 |

## 四、VRAM/GPU 预算

魔法系统无独立 VRAM 分配。所有视觉产出在以下预算中：

| 视觉产出 | 所属预算 | 备注 |
|---------|---------|------|
| 法术粒子 VFX | Godot 粒子系统 (技术栈方案 VRAM) | 约 30-50 种法术特效模板 |
| 法阵纹理 | 贴花/叠加层预算 | 法阵为 512² 贴花 |
| 元素环境着色 | 地形着色器 | 预计算纹理，无额外 VRAM |

## 五、峰值互斥

| 互斥对 | 说明 |
|--------|------|
| 战斗施法峰值 (0.2ms) × GOAP 全量规划 (1.5ms) | 战斗中使用高压版 GOAP (0.3ms)，非全量——互斥 |
| 法阵 AoE 碰撞 × 物理射线 | 法阵使用 EntityIndex 粗查询，不发射射线——非互斥但低重叠 |
| 契约魔法结算 × 经济批量 | 游戏日边界触发，与帧级操作时间分片 |

## 六、参数标注

- `[TUNING]` mana_regen_base: f32 = 1.0 — 基础回复速率乘数
- `[TUNING]` mana_regen_meditation_mult: f32 = 3.0 — 冥想回复倍率
- `[TUNING]` max_active_spells_per_npc: u8 = 5 — NPC 同时维持法术上限
- `[TUNING]` spell_automation_thresholds: [u32; 3] = [100, 300, 500] — 自动化升级阈值
