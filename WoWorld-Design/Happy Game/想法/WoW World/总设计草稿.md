## 〇、写在前面：雄心与务实

### 与其说这是一个游戏，不如说我们在创建另一个"人世"

驱动这个项目的根本冲动，不是"做一个好玩的游戏"——而是"创建另一个能够像现实世界那样运转的人世"。

启发我们的那些东西——龙之信条里的冒险小队、D&D 中的地下城探索、Kenshi 里那个不围着你转的世界、奇幻小说中广袤的未知大陆、日式异世界幻想番剧中"来到另一个世界开始新人生"的浪漫——它们的共同点不是某种玩法机制，而是一种感觉：**这个世界在你到来之前已经存在了很久，在你离开之后它还会继续存在。**

但 WoWorld 不止于冒险。

冒险当然是重要的——组建一支冒险小队，去无穷的远方见识各种各样的文明、遗迹、物种、奇迹，去经历波谲云诡的由人组成的"社会"——这是驱动这个项目的最初火花，也永远会是游戏中最激动人心的体验之一。

但"冒险者"只是玩家在这个世界中可以选择的无数身份之一。

玩家可以做铁匠。可以经商。可以插足一个地区乃至一个国家的政治。可以当警卫。可以做游侠。可以当情圣。可以当法外狂徒。可以几乎做任何事。

**我们的目标不是做一个"冒险游戏"。我们的目标是创建一个完整的、鲜活的"人世"——在这里，冒险与其他所有生活方式拥有平等的地位。** 就像在现实中，有人选择成为探险家，有人选择开一家面包店，有人选择从政，有人选择隐居山林——每一种选择都是人生的一种完整形态，没有哪种是"主线"，没有哪种是"支线"。

### 这个世界有历史——很多很多年的历史

玩家创建并进入一个世界时，这个世界已经运转了很多很多年。这个数字可以是 2，可以是 52，可以是 2026，可以是 2312313 或者随便什么其他的数字。这意味着：

- 这个世界的人们对于过去有着相同的、不同的记忆和传说。
- 这个世界的书本、遗迹、墓碑、纪念碑、古老建筑……不同程度地保留了以往的先民们遗留写下来的**活过**的证据。
- 玩家踏入这个世界，就像一个人第一次从母胎来到现实中那样——世界早就在这里了，你需要自己去理解它。

同时，玩家也可以选择从零开始——从一无所有、刀耕火种的蛮荒时代起步，就像猴子刚学会直立那样，一个人或是与其他 NPC 一起建立全新的文明。这有点像 RimWorld、Minecraft、矮人要塞的开局。但**有相当历史的、有相当程度文明的世界才是我们的重点**——因为我们想做的，是那个已经运转了千百万年的、厚重的人世。

### 关于"人"：NPC 是我们最高的上限

本游戏关于**人**（NPC）的设计极为重要。NPC 越是活灵活现，越是能自由地行动和思考，越是在自由和丰富的程度上贴近乃至超出现实中的人类，我们这款游戏的上限就越高。

以往的游戏，尤其是生存建造类，大多是与"物"打交道——采集、建造、制造。而与"人"打交道才是人类的天性，是人类整日离不开的、又爱又恨的、充满张力、故事拓展空间和自由度的事务。

我们将尽可能地将每个人变得**完整**。一个完整的人，有记忆、预测、情感、思想、人际关系、欲望、习惯、偏好——这些表现和特质，要尽可能地重现。这是一项没有终点的追求，但每逼近一步，这个世界的可信度就增加一分。

### 这不是一个"有故事的游戏"，这是一个"会自己写故事的游戏"

传统的RPG——从《博德之门》到《上古卷轴》——本质上都是开发者把自己写好的故事放进代码里。故事总有尽头。设计师的剧本写到哪一页，游戏的寿命就基本上终止在哪一页。即使有分支剧情，分支的数量也是有限的——因为每一个分支都是人手工写出来的。

WoWorld 选择了另一条路。

**我们不写故事。我们创造现实。**

就像我们所在的人世间——没有人给地球写剧本。物理定律、生物本能、社会规则、心理机制这些东西构成了现实的"底层规则"，而每天发生的无数故事——一场争吵、一次邂逅、一段恩怨、一个帝国的兴衰——都是从这些规则的互动中自然涌现的。

WoWorld 做的就是这件事。我们定义这个世界的"底层规则"：NPC的人格参数如何运作、情绪如何在事件中波动、记忆如何在社交中传播、经济如何在供需中平衡、文化如何在人群中演化。然后我们放手——让世界自己长出故事。

> **核心设计哲学：只创造规则，不创造故事。我们是世界的立法者，不是世界的编剧。**

这意味着：
- 游戏的寿命不由"开发者写了多少剧情"决定——因为根本没有预设剧情。世界自己会源源不断地产生新故事。
- 每个存档都是一个完全不同的世界——不是"山的位置不同"，而是文明形态、文化格局、势力版图可能天差地别。
- 即使是开发者本人，也无法预测一个给定种子的世界100年后会变成什么样——涌现的复杂性超出了任何人的预判能力。

### 风格：西方奇幻为基底，全球多元文化自由融入

这是一个剑与魔法的世界——但不是中世纪欧洲的翻版。WoWorld 是一个**属于它自己的异世界**。

它以西方奇幻（剑、魔法、骑士、龙）为基底，但世界各地的文化元素都可以在这里找到自己的位置：东方（道袍法师、麒麟、东方楼阁、武术、东方哲学），中东与波斯（清真寺与几何花纹、沙漠商队、新月骑兵），非洲（部落图腾、萨满仪式、草原帝国），北欧与凯尔特（符文石碑、迷雾森林、德鲁伊），美洲原住民（自然崇拜、羽蛇神、图腾柱），南亚与东南亚（印度教神祇、浮屠塔、象骑兵）……这个列表不是排他性的——任何人类文明中诞生的文化形态，都可以在这个世界中找到自己的表达。

更重要的是：**这些文化不是被我们"放置"在世界中的——它们是从文明的种子中自己生长出来的。** 一个世界如果演化出了某种风格的文明，那是因为它的文化种子、地理条件、历史事件共同导向了这个结果。不同存档中的文明形态可能天差地别——这个世界中的各个文明也可以自行发展、融合、演变出自己独一无二的文化特色，而非开发者预设的模板。

> 我们不是在做一个"欧洲奇幻 + 日本元素 + 中国元素"的拼盘。我们是在创造一个文化能够自然演化的生态系统。

## 一、游戏定义

### 一句话描述

一个**没有剧本**、**有历史**的异世界——以西方剑与魔法为基底，全球多元文化自由融入。NPC有完整的记忆、情感和欲望。你可以组建冒险小队去无穷远方探索失落的文明与奇迹，也可以做一个铁匠、商人、政客、法外之徒度过一生。每条路都是"主线"，因为这个世界本身就是内容。

### 核心体验

| 体验 | 说明 |
|------|------|
| **人世感** | 这不是一个"游戏地图"——这是一个已经运转了千百万年的世界。它有历史、有记忆、有传说。你踏入它，就像第一次从母胎来到现实中 |
| **探索感** | 组建冒险小队，去无穷的远方见识各种各样的文明、遗迹、物种、奇迹，以及波谲云诡的由人组成的"社会" |
| **活着感** | NPC不是在等你对话的任务板——他们在种田、交易、恋爱、争斗、死亡。他们有记忆、预测、情感、思想、欲望、习惯 |
| **自由感** | 冒险者、铁匠、商人、农夫、佣兵队长、游侠、情圣、政客、土匪、警卫、法外狂徒——游戏不告诉你该做什么，每条路都是一生的完整形态 |
| **影响感** | 你的行动（或不行动）改变世界，NPC记得你做的事，社会关系因你而变 |
| **叙事感** | 没有预设的主线。你的游玩历程本身就是一部独一无二的故事——而且这个故事不会被"通关" |
| **无限性** | 每个存档都是不同的世界。文明形态、文化格局、历史轨迹可能天差地别。世界不会"完结"——因为内容是自己长出来的 |

### 不是什么

- ❌ 不是"另一个老滚"——我们不追求电影级演出和手写剧情。我们根本不写剧情
- ❌ 不是D&D或传统CRPG——没有预设的主线任务链，没有"通关"的概念。但D&D式的冒险小队精神和奇幻世界的广度是我们的重要灵感
- ❌ 不是"另一个环世界"——玩家是世界中的人，不是上帝视角的管理者
- ❌ 不是MMO——单机优先，联机后议
- ❌ 不是AI聊天机器人——LLM是可选的锦上添花，不是核心玩法
- ❌ 不是"中世纪欧洲模拟器"——这是一个全球多元文化融合的原创异世界，不是任何现实历史时期的复制
- ❌ 不是生存建造游戏——与"物"打交道是手段，与"人"打交道才是核心

### 参考与对标

| 参考游戏 | 借鉴什么 |
|----------|---------|
| **Minecraft** | 每个存档都是全新世界；世界本身的多样性即是内容的无限性；种子决定一切的哲学 |
| **龙之信条** | 冒险小队（Pawn系统）的精神——与性格各异的同伴一起踏上征途的感觉 |
| **D&D（龙与地下城）** | 奇幻冒险的广度、组队探索地下城与未知领域的核心体验、世界设定的深度 |
| **日式异世界幻想番剧** | "来到另一个世界开始新人生"的浪漫感；剑与魔法日常的生动刻画 |
| **Kenshi** | 自由度的精神、世界不围着你转的感觉、残酷而真实的生存感 |
| **RimWorld** | NPC需求驱动行为的系统设计、涌现叙事的威力；从蛮荒起步建立文明的体验 |
| **Elona / Elin** | "什么都能做"的广度、东方元素自然融入西幻世界的手法 |
| **Mount & Blade** | NPC团体政治、势力动态、从无名小卒到一方霸主 |
| **Valheim** | 体素世界的连续感、低多边形美学的潜力、探索的rewarding感 |
| **艾瑞亚之歌**（未发布）| NPC心理描摹系统的参考方向 |
| **Dwarf Fortress** | 世界生成深度、历史模拟、"法则→涌现"的设计哲学；从零开始建立文明的可能性 |

## 二、核心支柱

这5根柱子撑起整个游戏。任何系统设计都要回到这些支柱上。

### 支柱1：活的NPC ⭐最重要

这是游戏最核心的差异化竞争力。

NPC不是脚本驱动的木偶。每个NPC拥有：
- **大五人格** — 开放性、尽责性、外向性、宜人性、神经质（0-1连续值）
- **情绪引擎** — 基于普拉奇克情绪轮的完整情绪模拟，含30种复合情绪
- **记忆系统** — 200条事件记忆，含遗忘曲线和记忆重构（人会记错、美化、自我辩护）
- **行为决策** — 90%概率驱动日常 + 10% GOAP安全网（生存本能）
- **社会关系** — 好感度、信任度、熟悉度、支配/服从地位
- **文化属性** — 习俗、规范、禁忌、价值观，随代际漂移

详见《NPC活人感开发文档》（开发阶段/NPC活人感模块/）

### 支柱2：程序化世界

每次新游戏 = 一个新世界。种子决定一切。

世界生成包含：
- **自然地形** — 多层噪声函数→山川河流海洋高原沙漠空岛
- **资源分布** — 矿产/植被/水源按地理逻辑分布
- **人文景观** — 城市>城堡>小镇>村庄四级聚落体系
- **建筑内部** — 建筑可进入，有完整内饰
- **NPC初始状态** — 包括初始家庭、团体、社会关系

世界不是白纸——生成时就"已经活了几百年"，有历史痕迹、有势力格局。

### 支柱3：高自由度

玩家可以做的事（非穷举）：

| 维度 | 活动 |
|------|------|
| 战斗 | 近战、远程、魔法、空战、信息战 |
| 社交 | 交友、恋爱、结盟、背叛、经商、谈判 |
| 经营 | 开店、种田、养殖、锻造、炼药 |
| 建筑 | 建造/改造房屋、布置内饰、建立聚落 |
| 探索 | 登山、航海、洞穴探险、遗迹探索 |
| 政治 | 加入势力、建立势力、发动战争、外交 |
| 生活 | 烹饪、钓鱼、采集、读书、祈祷、参加节日 |

### 支柱4：涌现式玩法

不写故事，只写规则。故事自然发生。

例子：
- 铁匠因饥饿抢了面包师→面包师记恨→面包师传播铁匠的坏话→铁匠生意变差→铁匠更加暴躁→形成恶性循环
- 两个势力因资源竞争→年轻NPC被情绪感染走向极端→引发小规模冲突→被著名度系统记录→成为地区"历史事件"

### 支柱5：可选LLM增强

- 玩家自备API Key
- LLM让NPC对话更自然、生成游玩日志
- 无LLM时游戏完全正常运行
- 安全网关：LLM只能从预定义行动库选动作，不能自由发挥
- 成本控制：仅L1 NPC（离玩家50m内）可调用，异步队列，速率限制

## 三、世界系统

### 3.1 体素世界

### 3.1 体素世界

**关键决策：连续体素地形 + 网格物体混合世界。**

> 详细体素设计见 `想法/世界生成/06-体素设计决策.md`

- **体素只做地形**——山、谷、河、洞、矿脉。使用 SDF（有符号距离场）+ Marching Cubes/Transvoxel 提取等值面
- **网格做一切在地形之上的东西**——建筑、家具、角色、植被、物品
- 体素单位 0.5m³，密度场连续（0=空气，1=实心），非方块状
- 地形修改通过 SDF 雕刻（加减密度），不是"放置/移除方块"
- 远景 LOD：Transvoxel 算法，远处降低体素精度但保持轮廓
- 参考视觉效果：Valheim（连续地面+网格建筑）、雾锁王国（体素地形+自由建造）

**为什么不用纯 Mesh 地形？**
- 玩家/NPC 可以改造地形（挖矿、挖地基、挖战壕）→ 体素天然支持 SDF 操作
### 3.2 世界规模（渐进式）

- **初始可玩区域**：4km × 4km，约 1.6 亿地表体素格
- **可扩展至**：最大 16km × 16km（后期，通过世界边界扩展机制）
- 区块系统：超区块 500m×500m，内部分为 32×32 块 chunk（每 chunk 约 16m×16m）
- 动态加载：玩家周围 3 个超区块半径全加载，远处统计模拟
- 存档总大小目标：< 3GB（含体素+NPC+事件数据）
- 实际游玩中，玩家在单个存档内活跃范围极少超过 4km²
- 建筑需要细节精度（门把手、窗框、雕刻）远超 0.5m 体素
- 模块化网格更省面数（200-500面 vs 体素等值面2000+面）
- 纹理 UV 在网格上更好控制
- 详见 `06-体素设计决策.md`

### 3.2 世界规模

- 世界有限但巨大（类似Minecraft：边界存在但对单人游玩几乎无限）
- 预估：16km × 16km 初始世界区域，可扩展
- 区块系统：32×32个区块，每个区块约500m×500m
- 动态加载：玩家周围的区块全加载，远处统计模拟

### 3.3 世界生成管线

```
种子(seed)
  │
  ├─→ 大陆形状噪声（低频率Perlin/Simplex）
  │     └─→ 海陆分布
  │
  ├─→ 高程图（多层噪声叠加）
  │     ├─→ 山脉（高频率+高振幅）
  │     ├─→ 丘陵（中频率+低振幅）
  │     ├─→ 平原（低频率+极低振幅）
  │     └─→ 高原（中等高程+低频率）
  │
  ├─→ 水文模拟（简化）
  │     ├─→ 河流从高山流向海洋
  │     ├─→ 湖泊在低洼处形成
  │     └─→ 湿地在水源丰富的平坦区域
  │
  ├─→ 气候与生物群系
  │     ├─→ 纬度+高程→温度
  │     ├─→ 海洋距离+风向→湿度
  │     └─→ 温度×湿度→生物群系
  │
  ├─→ 资源分布
  │     ├─→ 矿产按地质逻辑（山区多矿、河床有金）
  │     ├─→ 植被按生物群系
  │     └─→ 特殊资源按奇幻规则
  │
  └─→ 人文景观生成
        ├─→ 选址评估（淡水+农田+防御+资源）
        ├─→ 聚落放置（城市>城堡>小镇>村庄）
        ├─→ 建筑生成（按聚落类型+文化风格）
        ├─→ 道路连接（城市间→小镇间→村庄支路）
        └─→ NPC/NPC团体/NPC关系初始化
```

| 丘陵 | 草丘、石丘、森林丘陵 |

### 3.5 人文景观层级

```
聚落体系（四级）
  ├─ 城市（一级）— 区域统治+贸易加工+军事防御
  │   └─ 规模：50-200栋建筑，含城墙、市政厅、大市场、教堂
  ├─ 城堡（二级）— 军事控制+要地防御+权威象征

  ├─ 小镇（三级）— 资源精加工+特种贸易+区域服务

  └─ 村庄（四级）— 食物与基础原材料生产
      └─ 规模：5-15栋建筑

独立建筑：哨站、神龛、磨坊、桥梁、驿站、隐修院
遗迹：任何类型的废弃版本，无人维护，自然衰败
```

## 四、NPC系统（核心中的核心）

> 📘 详细规格见 `开发阶段/NPC活人感模块/NPC活人感开发文档ver1.01short.md`  
> 本节为概要，不做重复。完整数据结构、参数阈值、算法细节请查阅该文档。

### 4.1 系统概述

NPC系统是WoWorld最关键的差异化竞争力。设计目标是：**玩家无法轻易区分"这个NPC是活的还是脚本"。**

核心指标：
- 世界常驻目标 100,000 NPC（全部尽数模拟），其中L1全模拟目标约1,000个
- 每个NPC有独立的人格、记忆、情绪、关系
- NPC之间、NPC与玩家之间产生涌现式叙事
- 随着技术迭代与优化，成功模拟的NPC数量将持续提升

### 4.2 五层模型

```
     ┌──────────────────────────┐
     │  5. 社会投影              │  ← 其他NPC记忆中的"他"
     │  ┌──────────────────────┐ │
     │  │ 4. 物理躯体          │ │  ← CharacterBody3D
     │  │ ┌──────────────────┐ │ │
     │  │ │ 3. 感官外壳      │ │ │  ← 视觉/听觉/情绪感知
     │  │ │ ┌──────────────┐ │ │ │
     │  │ │ │ 2. 心智核心  │ │ │ │  ← 情绪/记忆/决策
     │  │ │ │ ┌──────────┐ │ │ │ │
     │  │ │ │ │1.数据本体│ │ │ │ │  ← NPCData Resource
     │  │ │ │ └──────────┘ │ │ │ │
     │  │ │ └──────────────┘ │ │ │
     │  │ └──────────────────┘ │ │
     │  └──────────────────────┘ │
     └──────────────────────────┘
```

### 4.3 三层模拟（性能核心）

独立开发者实现100,000 NPC的唯一可行路径。

| 维度 | L1 全模拟 | L2 轻量 | L3 统计 |
|------|----------|---------|---------|
| 数量上限 | 目标~1,000 | 目标~10,000 | 目标~100,000 |
| 距离玩家 | ≤50m | 50-150m | >150m |
| 3D节点 | 完整 | 简化 | 无(纯数据) |
| 情绪模拟 | 完整引擎 | 仅衰减收敛 | 无 |
| 记忆系统 | 完整读写 | 只读 | 无 |
| 行为决策 | 概率+GOAP | 简化状态机 | 无(宏观推演) |
| GPU开销 | 完整渲染 | 简化渲染 | 零 |
| 更新频率 | 每0.2s | 每秒2-3次 | 每日/周 |

**升降级机制**：
- 玩家接近→升级，滞后区间防止抖动（升50m/降55m）
- 紧急升级：战斗触发/高冲击力事件目击/玩家交互
- 关键角色可永久锁定L1（override机制）

### 4.4 三大决策环

```
日常行为 (90%)
  └─ 本地概率决策
       └─ 文化习俗权重 × 个人习惯权重 × 人格偏移 × 情绪修正
       └─ 加权随机采样 → 坚持执行直到完成/中断

生存危机 (9%)
  └─ GOAP安全网
       └─ 饥饿>0.85 / 口渴>0.9 / 疲劳>0.9 / 生命<0.3
       └─ 确定性规划，确保合理行为

复杂社交 (1%)
  └─ LLM增强(可选)
       └─ 仅从预定义行动库选取
       └─ 安全网关过滤
```

### 4.5 情绪引擎速览

- **三维底层轴**：愉悦(-1~1) × 觉醒(0~1) × 掌控(-1~1)
- **8种基本情绪**：喜、信、惧、惊、悲、恶、怒、期
- **30种复合情绪**：喜爱、敬畏、内疚、嫉妒、怀旧、尴尬……
- **情绪感染**：8重调节因子（亲密度、地位差、群体相似性、人格……）
- **集体情绪**：级联→共振→极化→免疫

### 4.6 文化系统三层

```
表层：行为习俗 ← 行为频率统计，每游戏月更新
  ↓
中层：规范与禁忌 ← 社会反馈积累（赞许/惩戒计数）
  ↓
深层：价值观与信念 ← 缓慢结晶/代际漂移
```

文化动态：群体隔离→分化 / 群体接触→融合

## 五、玩家系统

### 5.1 角色创建

- 种族选择（人类/精灵/矮人/兽人/……可扩展）
- 初始出身（影响初始属性、技能、社会关系、初始地点）
- 大五人格滑动条（影响可选行为倾向，非职业锁定）
- 外观定制（受AI生成资产限制，预设+参数化）

### 5.2 技能系统

**设计原则：用进废退。** 没有"升级点技能树"。你用剑→剑术提升。你做饭→烹饪提升。

```
技能 = {
  当前等级(0-100),
  经验值(隐藏),
  天赋加成(种族+人格由来，恒定),
  最近使用时间(决定衰减速度)
}
```

技能分类（示例，非穷举）：
- **战斗**：剑术、斧术、弓术、格挡、闪避、魔法各系
- **生产**：锻造、炼药、烹饪、纺织、木工、建筑
- **社交**：说服、欺骗、恐吓、魅力、交易
- **知识**：医术、博物学、历史学、魔法理论
- **生存**：采集、钓鱼、追踪、潜行、攀爬

### 5.3 物品与装备

- **武器**：模块化组装——柄+刃/锤+符文卡槽，材料决定属性和锻造要求
- **防具**：头/身/手/腿/鞋五槽，轻甲/中甲/重甲
### 6.1 设计目标

- 即时制（非回合制），但有足够的战术深度
- 碰撞判定（hitbox + 惯性考虑），提供打击感
- 与环境的交互丰富
- NPC的心理模块在战斗中持续运作（战斗意志/士气）
- 包含信息战元素（渐进实现）

> ⚠️ **渐进实现**：战斗系统分三步走——
> 1. MVP：碰撞盒+轻重攻击+格挡+血条
> 2. 进阶：元素魔法(火冰雷先行)+战斗风格+基础信息战
> 3. 完整：10元素+完整信息战+空战
> 详见 `想法/世界生成/07-设计整合分析.md` §一·冲突5
- 房屋有所有权和使用权属性
- 内饰：家具、装饰、储存容器的摆放

## 六、战斗系统

### 6.1 设计目标

- 即时制（非回合制），但有足够的战术深度
- 物理碰撞决定命中判定（类似Kenshi/龙腾世纪：起源）
- 与环境的交互丰富
- NPC的心理模块在战斗中持续运作
- 包含信息战元素

### 6.2 战斗风格

NPC的战斗风格从人格自动派生，玩家可自由选择方向：

| 人格组合 | 战斗风格 | 特点 |
|----------|---------|------|
| 高神经质+低宜人性 | 狂暴型 | 高伤害、低防御、不收手 |
| 高尽责性+低外向性 | 战术型 | 利用掩体、精确打击、等待时机 |
| 高宜人性+高外向性 | 支援型 | 辅助队友、控制敌人、治疗 |
| 高开放性+高魔法 | 施法者 | 远程魔法、范围攻击、环境操纵 |

### 6.3 武器模块化

```
武器 = 柄(材料/长度/重量) + 攻击部(刃/锤/刺,材料/形状) + 卡槽(0-3个)

卡槽可嵌入：
  - 符文：赋予元素属性（火/冰/雷……）
  - 宝石：增加特殊效果（吸血/破甲/眩晕……）
  - 附魔卷轴：一次性强化
```

### 6.4 魔法系统

- **10元素**：金、木、水、火、土、风、雷、电、血、灵
- **施法方式**：仪式+物质+语言+意志 → 操纵元素
- **灵元素**是生物连接其他元素的桥梁
- 魔法与物理战斗并存——战士用魔法附魔武器，法师用法术直接攻击

### 6.5 信息战

战斗不只是血条对砍：
- NPC对敌方实力的认知来自**过往信息+当前观察**
- 承受高伤害后"强装镇定"可以挫伤对方战斗意志
- 房屋有"隔断指数"——决定室内战斗是否影响室外
- "战斗显眼度"决定多少人关注
- "战斗事件度"决定是否被社会记忆

### 6.6 空战

- 飞行坐骑+空对空/空对地战斗
- 高空强风、能见度等环境因素影响

## 七、经济与社会

### 7.1 动态经济

> ⚠️ **实施警告**：完全动态经济极难做好。建议采用"受控动态"方案。

**受控动态经济方案**：
- 世界生成时建立稳定的区域经济体系（以NPC团体为单位）
- 日常交易在预设价格范围内波动（±15%）
- 重大事件（战争、灾害、贸易路线切断）触发价格重调整
- "神秘力量"兜底——当经济出现异常时，系统悄悄注入/回收资源

### 7.2 团体与势力

NPC基于共同利益/性格/文化自发组成团体：

| 层级 | 例子 | NPC数 |
|------|------|-------|
| 家庭 | 夫妻+子女+亲属 | 2-15 |
| 小队 | 冒险队/佣兵团/商队 | 3-12 |
| 行会 | 铁匠行会/商人工会/冒险者公会 | 20-200 |
| 党派 | 政治派系/宗教教派 | 50-500 |
| 国家 | 王国/帝国/城邦联盟 | 500-5000 |

团体类型：经济、政治、文化、宗教、师徒、从属、契约

### 7.3 事件系统

事件是游戏的基本要素——不是"支线任务"，而是世界运转的自然产物。

```
事件生命周期：
  发起（有智能的个体/团体产生意图）
  → 扩散（影响力传播，其他单位介入）
  → 推进（各参与方的行为改变事件走向）
  → 结局（产生后果，写入记忆，影响后续）
```

事件分类：
- 一级事件（交易、闲聊、纠纷）
- 二级事件（婚礼、葬礼、战斗、交易大单）
- 三级事件（影响大量单位的重大事件：战争爆发、自然灾害、英雄诞生）
- 四级事件（待开发。目前设计为开发者单独设计的事件）

### 7.4 道德与法律

- **道德**：描述一个人多大可能性认可/施行某行为（个人层面的善恶观）
- **道德共识**：一个团体内共享的"该做什么/不该做什么"
- **法律**：由统治团体强制执行的道德共识
- **执法**：违反法律→被执法者发现→追责（罚款/劳动/囚禁/放逐/处决）

### 7.5 仪式与节日

- 由文化系统自动生成节日日期和仪式内容
- 节日：大量NPC聚集、特殊行为、情绪感染放大器
- 仪式：婚礼、葬礼、成人礼、入会式——都是社交事件的高发场景

## 八、LLM集成方案

### 8.1 设计原则

> LLM是锦上添花，不是地基。游戏在无LLM时必须完整可玩。

### 8.2 架构

```
玩家自备API Key
      │
      ▼
LLM客户端管理器
  ├─ 速率限制器
  ├─ 异步队列
  ├─ 成本计算器（预估token消耗）
  └─ 缓存层（相似请求复用）
      │
      ▼
安全网关
  ├─ 实体过滤器（LLM只能引用存在的NPC/地点/物品）
  ├─ 行动边界检查（只能选预定义行动库中的动作）
  └─ 输出模板化（确保返回值可被解析）
      │
      ├─→ NPC对话生成
      ├─→ 玩家日志生成
      ├─→ 人际关系/国际关系参考
      └─→ 事件描述润色
```

### 8.3 调用时机

- ✅ NPC复杂社交对话（L1 NPC，玩家主动交互时）
- ✅ 生成游玩日志（按游戏天/重大事件后）
- ✅ NPC复杂决策参考（极少数情况，如多方势力谈判）
- ❌ 常规日常行为（概率驱动已足够）
- ❌ L2/L3 NPC（完全程序化）
- ❌ 战斗决策（延迟不可接受）

### 8.4 成本控制

- 仅L1 NPC可调用（目标最多约1,000个/全地图，场景内按L1范围限制）
- 每个NPC有调用冷却（如每游戏日1次）
- 异步队列，同时最多1-2个LLM请求
- 玩家可设置每日token预算上限
- 缓存层复用相似上下文

## 九、美术与音频

### 9.1 美术方向

**低多边形（Low-Poly）+ 像素纹理。**

为什么：
- 没有美术支持→AI生成+手动调整是唯一可行路径
- 低多边形风格对AI生成友好，容错率高
- 艾尔登法环证明了美术>画质——风格统一>面数多
- 独立开发者可以独立完成

### 9.2 AI美术管线

```
概念阶段：Midjourney/Stable Diffusion → 风格探索 → 确定视觉方向
资产阶段：
  ├─ 角色：Stable Diffusion + Character Sheet + 转3D（Rodin/TripoSR）
  ├─ 建筑：Stable Diffusion → 参考图 → Blender手工/几何节点建模
  ├─ 物品：SD → 小物件生成 → 手动调整
  ├─ 植被：几何节点+简单着色
  └─ UI：手动绘制图标+AI辅助
后处理：
  ├─ 统一调色板
  ├─ 减面到目标多边形数
  └─ 烘焙法线/环境光遮蔽
```

### 9.3 音频

- 环境音：AI生成+免费音效库，按生物群系/天气/时间动态混音
- NPC声音：优先无配音（文本展示），后期可考虑TTS
- 音乐：AI生成背景音乐（Suno/Udio）+ 免费版权音乐
- 战斗音效：合成音效+免费素材

## 十二、风险与对策

| 风险 | 严重度 | 对策 |
|------|--------|------|
| 100,000 NPC的性能 | 🔴高 | 三层LOD模拟；L3纯统计；分帧更新；渐进式规模扩展 |
| NPC行为看起来假 | 🔴高 | 概率驱动为主（不可预测=看起来聪明）；充分测试涌现边界 |
| 世界生成太单调 | 🟡中 | 多层噪声叠加；大量文化种子变体；手工添加特殊地标模板 |
| 动态经济崩溃 | 🟡中 | 受控动态而非完全自由；神秘力量兜底 |
| 开发周期过长 | 🔴高 | MVP思维；尽早可玩；砍掉非核心功能 |
| AI美术风格不统一 | 🟡中 | 严格的调色板；后处理统一；接受"风格化不完美" |
| 物理交互复杂度 | 🟡中 | 限制可交互物体类型；分帧处理物理 |
| LLM成本失控 | 🟢低 | 玩家自备Key；默认关闭；充分缓存 |

## 十三、设计原则速查

开发过程中遇到设计决策时，按以下优先级判断：

1. **这会让NPC看起来更像活人吗？** — 是→做 / 否→再想想
2. **这会让玩家有"我想做这件事"的冲动吗？** — 是→做 / 否→再想想
3. **这可以作为后续系统的地基吗？** — 是→设计时留接口 / 否→可以硬编码
4. **这会显著增加开发时间吗？** — 是→找更简单的方案 / 否→做
5. **这能用规则涌现而不是手写内容吗？** — 是→写规则 / 否→再想想
