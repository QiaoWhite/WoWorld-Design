## 二、各阶段详细设计

### 阶段一：自然基底

**输入**：`uint64` 世界种子  
**输出**：高程图、水文图、气候图（均为 2D 数组）

```mermaid
flowchart LR
    Seed["种子"] --> H1["hash_to_noise_seeds()\n派生 8 个子种子"]
    H1 --> N1["Perlin 噪声\n大尺度地形轮廓"]
    H1 --> N2["Simplex 噪声\n中尺度丘陵/盆地"]
    H1 --> N3["Worley 噪声\n小尺度破碎地形"]
    N1 --> Blend["加权混合\nP:50% S:35% W:15%"]
    N2 --> Blend
    N3 --> Blend
    Blend --> Elev["高程图\n0.0(海床) ~ 1.0(山巅)"]
    Elev --> Hydro["水文模拟"]
    Hydro --> Climate["气候带计算"]
```

**关键参数**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `sea_level` | 0.35 | 海平面阈值，低于此值为水域 |
| `mountain_threshold` | 0.72 | 山脉阈值 |
| `river_density` | 0.003 | 河流起始点密度 |
| `lake_probability` | 0.01 | 湖泊生成概率 |

**水文模拟算法**：
1. 在高程 > 0.6 的区域随机撒河流种子点
2. 从每个种子点沿坡度最陡方向下流
3. 流量累积：支流汇入处增大河流宽度
4. 局部低洼处形成湖泊
5. 河流最终汇入海洋

**资源分布规则**：

| 资源类型 | 决定因素 | 示例 |
|----------|---------|------|
| 食物 | 生物群系 + 水源距离 | 河流旁农田肥沃 |
| 木材 | 森林密度 | 针叶林 > 雨林 > 草原 |
| 石材 | 高程 + 地质 | 山区多花岗岩 |
| 金属矿 | 地质层（Perlin） | 铁矿浅层，金矿深层 |
| 魔法材料 | 特殊群系 + 低概率 | 水晶森林产魔力水晶 |
| 水源 | 河流/湖泊距离 | 聚落必需 ≤ 200m |

### 阶段三：自然景观布局

**核心概念**：围绕玩家加载范围进行跨区块自然景观分布规划。这是 Canvas `ver0.5` 的独特设计——世界生成不是全域均匀的，而是围绕玩家活动范围进行密度控制。

```mermaid
flowchart TD
    Player["玩家当前位置"] --> Range["确定加载范围\n半径 3 区块"]
    Range --> DistPlan["跨区块自然景观分布规划"]
    
    DistPlan --> Check1{"关键地貌是否\n在加载范围内？"}
    Check1 -->|"缺失"| ForceGen["强制生成至少 1 个\n该地貌类型的区域"]
    Check1 -->|"存在"| DensityCheck["密度检查"]
    
    ForceGen --> DensityCheck
    
    DensityCheck --> Check2{"景观密度是否\n在目标范围内？"}
    Check2 -->|"过高"| Thin["稀疏化"]
    Check2 -->|"过低"| Enrich["增补生成"]
    Check2 -->|"合适"| Done["✅ 完成"]
    
    Thin --> Done
    Enrich --> Done
```

**每种地貌的资源规划**（对应 Canvas 中的 `各地貌资源规划` 节点）：

每个地貌区块确定后，单独计算其内部资源分布：
- 食物产出能力（kcal/年）
- 木材储量（吨）
- 矿产种类与储量
- 水资源容量（升/秒）
- 可建造面积（m²）

这些数值决定后续聚落能否在此选址。

### 阶段四：聚落选址

详见 [`02-聚落生成规则.md`](./02-聚落生成规则.md)

简版流程：

```mermaid
flowchart TD
    Input["地貌资源数据"] --> Judge{"资源种类 ≥ 3？\n(食物/水/建材 至少各一)"}
    
    Judge -->|"✅ 达标"| Select["聚落选址\n确定等级"]
    Judge -->|"❌ 不达标"| Nearby["查询临近地貌"]
    
    Nearby --> CanSupplement{"临近地貌资源\n可补足？"}
    CanSupplement -->|"✅ 可补足"| MultiSite["多地貌聚落群选址\n（跨区块聚落）"]
    CanSupplement -->|"❌ 不可补足"| NoSettlement["不生成聚落\n概率终止"]
    
    Select --> Hierarchy["聚落等级链\n村庄→小镇→城市→城堡"]
    MultiSite --> Hierarchy
```

**资源种类达标判断**（`聚落生成地貌资源判断`）：

| 必要条件 | 最低阈值 |
|----------|---------|
| 淡水源 | 河流/湖泊 ≤ 200m 或地下水 |
| 食物来源 | 可耕地/狩猎区/渔场 |
| 建筑材料 | 木材或石材 |

三项全部满足 → 达标。缺一项 → 检查临近地貌（≤ 500m）是否可补足。

## 三、多地貌聚落群选址

来自 Canvas `ver0.5` 的独特设计（`ver0.6` 移除了此逻辑——保留它作为深度选项）。

当一个地貌内部资源不足时，尝试与临近地貌组成聚落群：

```mermaid
sequenceDiagram
    participant M1 as 地貌A（缺木材）
    participant Judge as 资源判断器
    participant M2 as 地貌B（多木材）
    participant Gen as 生成器
    
    M1->>Judge: 资源不达标（缺木材）
    Judge->>M2: 查询临近地貌（≤500m）
    M2-->>Judge: 木材丰富 + 未生成聚落
    Judge->>Gen: 多地貌聚落群选址
    Gen->>Gen: 聚落主体在地貌A<br/>林场/伐木站在地貌B
    Note over Gen: 生成连接道路<br/>标记为"跨地貌聚落"
```

## 四、确定性保证

整个管线必须是**确定性**的：

```
给定 seed S → 总是生成完全相同的世界
```

实现方式：
- 所有随机数从 `seed` 派生的 RNG 序列中获取
- 噪声函数使用 `seed` 作为输入
- 不依赖系统时间或外部状态
- 多线程生成时，每个区块从 `hash(seed, chunk_x, chunk_y)` 派生独立 RNG

```mermaid
flowchart LR
    Seed --> Hash1["hash(seed, 'noise')"] --> RNG1["RNG1: 噪声层"]
    Seed --> Hash2["hash(seed, 'resource')"] --> RNG2["RNG2: 资源分布"]
    Seed --> Hash3["hash(seed, 'settlement')"] --> RNG3["RNG3: 聚落生成"]
    Seed --> Hash4["hash(seed, 'npc')"] --> RNG4["RNG4: NPC 生成"]
```

## 五、ver0.5 vs ver0.6 差异

| 特性 | ver0.5 | ver0.6 |
|------|--------|--------|
| 跨地貌聚落群 | ✅ 支持 | ❌ 移除 |
| 临近地貌资源补足 | ✅ 多步判断 | ❌ 简化 |
| 聚落间道路规划 | ✅ 详细 | ❌ 未涉及 |
| 独立建筑选址 | ✅ 有 | ✅ 有 |

**建议**：以 ver0.5 为蓝本实现完整版，ver0.6 的简化逻辑可作为性能优化手段（在极低端硬件上使用）。
