## 一、总览

聚落生成是世界管线第四至第五阶段。它决定：在什么位置、生成什么等级的聚落、聚落内部如何布局。

```mermaid
flowchart TD
    subgraph Input["输入端"]
        I1["地貌资源数据\n(食物/水/建材/矿产)"]
        I2["地形数据\n(高程/坡度/平坦度)"]
        I3["文化参数\n(从种子派生)"]
    end

    Input --> SiteSelection

    subgraph SiteSelection["第一阶段：选址"]
        S1{"资源判断\n必要条件全满足？"}
        S2["跨地貌补足判断"]
        S3["聚落等级确定"]
    end

    SiteSelection --> Layout

    subgraph Layout["第二阶段：内部布局"]
        L1["政体/文化/类型确认"]
        L2["核心建筑定位"]
        L3["公民分区"]
        L4["功能区布置"]
        L5["城防体系"]
        L6["交通基础设施"]
    end

    Layout --> Output

    subgraph Output["输出端"]
        O1["聚落数据\n位置/等级/建筑列表"]
        O2["道路连接图"]
        O3["NPC分配计划"]
    end

    style Input fill:#0f3460,color:#fff
    style SiteSelection fill:#533483,color:#fff
    style Layout fill:#e94560,color:#fff
    style Output fill:#2d5a27,color:#fff
```

## 二、选址逻辑

### 2.1 资源判断

```mermaid
flowchart TD
    Start["地貌资源数据输入"] --> Check1{"淡水源 ≤ 200m？"}
    Check1 -->|"❌"| CheckWater{"地下水层存在？"}
    CheckWater -->|"❌"| Fail["不满足水源条件"]
    CheckWater -->|"✅"| Check2
    Check1 -->|"✅"| Check2{"可耕作/狩猎/渔场\n≥ 最低阈值？"}
    
    Check2 -->|"❌"| CheckFood{"临近地貌 ≤ 500m\n有食物补充？"}
    CheckFood -->|"❌"| Fail
    CheckFood -->|"✅"| Check3
    Check2 -->|"✅"| Check3{"建材（木/石）\n≥ 最低阈值？"}
    
    Check3 -->|"❌"| CheckMaterial{"临近地貌 ≤ 500m\n有建材补充？"}
    CheckMaterial -->|"❌"| Fail
    CheckMaterial -->|"✅"| Pass
    Check3 -->|"✅"| Pass
    
    Fail --> Retry{"重试概率\n(由种子决定)"}
    Retry -->|"30%"| NearbyMulti["尝试跨地貌聚落群"]
    Retry -->|"70%"| Abandon["不生成聚落"]
    
    Pass --> Grade["聚落等级确定"]
    NearbyMulti --> Grade
```

### 2.2 聚落等级链

等级从低到高，高级聚落包含低级聚落的全部功能，并增加额外功能建筑：

```mermaid
flowchart LR
    Village["🏘️ 村庄\n人口: 20-200"] -->|"农业集中区\n满足条件升级"| Town["🏠 小镇\n人口: 200-2000"]
    Town -->|"非农业资源集中\n满足条件升级"| City["🏙️ 城市\n人口: 2000-20000"]
    City -->|"战略/政治需求\n低概率升级"| Castle["🏰 城堡\n人口: 500-5000\n+驻军"]
```

**等级决定因素**：

| 因素 | 村庄 | 小镇 | 城市 | 城堡 |
|------|------|------|------|------|
| 资源种类 | 1-2 种 | 2-3 种 | 3-4 种 | 3-4 种 + 战略位置 |
| 农业资源 | 必须集中 | 可有可无 | 可有可无 | 可有可无 |
| 非农业资源 | 不需要 | 必须集中 | 必须集中 | 必须集中 |
| 生成概率 | 60% | 25% | 10% | 5% |
| 水资源 | 井/溪流 | 河流/湖泊 | 河流 | 河流 + 护城河水源 |

### 3.3 功能建筑选址逻辑

每种功能建筑根据聚落等级决定是否存在，根据文化/政体决定具体类型：

| 建筑类型 | 村庄 | 小镇 | 城市 | 城堡 | 选址规则 |
|----------|------|------|------|------|---------|
| 领袖宅邸 | 村长屋 | 镇长官邸 | 市政厅/王宫 | 要塞 | 最高处/中心 |
| 宗教建筑 | 小祠堂 | 庙宇 | 大教堂/寺院 | 城堡教堂 | 一等公民区 |
| 商业建筑 | 集市摊 | 商铺街 | 商会/市场 | 军需仓库 | 二等公民区 |
| 手工业 | 铁匠铺 | 作坊区 | 工厂/工会 | 军工作坊 | 靠近资源 |
| 教育建筑 | — | 私塾 | 学院/大学 | 军事学院 | 一等公民区 |

## 七、关键算法：聚落选址评分

对于每个候选位置，评分函数：

```
score = water_score × 0.35
      + food_score  × 0.30
      + build_score × 0.20
      + defense_score × 0.15
```

| 评分项 | 计算方式 |
|--------|---------|
| `water_score` | `1.0 - clamp(distance_to_water / 200, 0, 1)` |
| `food_score` | `min(food_production / min_threshold, 1.5) / 1.5` |
| `build_score` | `flatness × material_availability` |
| `defense_score` | `elevation_bonus + water_barrier_bonus` |

选择得分最高的候选位置。
