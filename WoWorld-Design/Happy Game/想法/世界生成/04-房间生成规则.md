## 一、总览

房间生成是建筑内部装修的最后阶段。每个房间从功能和文化确认开始，经过核心用具→辅助用具→装修的三层细化。

```mermaid
flowchart TD
    Input["输入：房间数据\n功能类型 + 文化类型 + 面积 + 位置"] --> P1

    subgraph P1["阶段一：功能与文化确认"]
        F1["解析房间核心功能\n卧室/厨房/工坊/神殿..."]
        F2["解析文化装饰风格\n简约/雕刻/彩绘/华丽"]
        F3["是否有预设模板？"]
    end

    P1 --> P2

    subgraph P2["阶段二：核心功能用具"]
        C1["核心用具 = 房间存在理由\n厨房→灶台 卧室→床 工坊→工作台"]
        C2["位置：房间中心/靠墙"]
        C3["尺寸：占房间面积 10-30%"]
    end

    P2 --> P3

    subgraph P3["阶段三：其余功能用具"]
        O1["辅助用具\n厨房→水缸/案板 卧室→衣柜/床头柜"]
        O2["装饰用具\n花瓶/挂画/地毯/帘幔"]
        O3["填充规则：面积越大\n装饰比例越高"]
    end

    P3 --> P4

    subgraph P4["阶段四：装修风格应用"]
        D1["墙面\n石灰/壁纸/木板/石面"]
        D2["地面\n土/木/石/地毯"]
        D3["天花板\n裸露/木板/彩绘"]
        D4["照明\n窗户/蜡烛/油灯/魔法灯"]
    end

    P4 --> Output["输出：完整房间数据\n可渲染 + 可交互"]

    style Input fill:#2d5a27,color:#fff
    style Output fill:#2d5a27,color:#fff
```

## 二、模板 vs 个性化

与建筑生成一致，房间也有模板和个性化两条路径：

```mermaid
flowchart TD
    RoomType["房间功能 + 文化"] --> Check{"常见组合？"}
    
    Check -->|"✅ 常见组合"| Template
    
    subgraph Template["模板路径（快速）"]
        T1["直接摆放预设用具组"]
        T2["微调：文化材质替换"]
        T3["随机偏移：±15% 位置"]
        T4["适用：普通民居的\n厨房/卧室/客厅"]
    end
    
    Check -->|"❌ 特殊组合"| Custom
    
    subgraph Custom["个性化路径（详细）"]
        C1["逐件确定用具"]
        C2["考虑 NPC 职业/财富/性格"]
        C3["适用：重要 NPC 住所\n神殿/王宫/特殊建筑"]
    end
```

**模板覆盖率**：

| 房间类型 | 模板化比例 |
|----------|-----------|
| 普通卧室 | 95% |
| 普通厨房 | 90% |
| 普通客厅 | 85% |
| 铁匠铺工坊 | 80% |
| 神殿内室 | 20% |
| NPC 专属房间 | 0%（全部个性化） |

## 三、核心功能用具

**核心用具 = 定义房间用途的那件物品。**

```mermaid
flowchart LR
    subgraph Bedroom["卧室"]
        BB["床\n尺寸: 单人/双人/大床\n质量: 草垫→木板→弹簧→丝绸"]
    end
    
    subgraph Kitchen["厨房"]
        KC["灶台/炉灶\n类型: 明火→石灶→砖炉→魔法炉\n附属: 烟囱/排烟口"]
    end
    
    subgraph Workshop["工坊"]
        WW["工作台/铁砧/织机\n类型由 NPC 职业决定"]
    end
    
    subgraph Temple["神殿"]
        TA["神像/祭坛\n文化宗教类型决定"]
    end
    
    subgraph Study["书房"]
```

**核心用具质量等级**：

| 等级 | 材料 | 耐久度 | 使用效果 | 所属 NPC 财富 |
|------|------|--------|---------|-------------|
| 0 | 粗制木材/石头 | 100 | 基础功能 | 贫困 |
| 1 | 普通木材 | 200 | 基础功能 | 普通 |
| 2 | 优质木材/铁件 | 400 | +10% 效率 | 小康 |
| 3 | 硬木/钢件 | 800 | +20% 效率 | 富裕 |
| 4 | 稀有材料/魔法 | 1600 | +35% 效率 | 富豪/贵族 |

## 四、其余功能用具

```mermaid
flowchart TD
    Area["房间面积 m²"] --> Density["用具密度计算\n每 4m² = 1 个用具槽位"]
    
    Density --> Priority
    
    subgraph Priority["用具优先级排序"]
        P1["🔴 必须 — 房间功能的物理需求\n厨房→水缸(取水距离>50m时)"]
        P2["🟡 重要 — 大幅提升功能效率\n厨房→案板/刀具架/调料架"]
        P3["🟢 锦上添花 — 装饰品\n花瓶/挂画/地毯"]

    Priority --> Fill["按优先级填充槽位\n直到所有槽位填满"]
```

## 七、房间连通性

房间不是孤立的——通过门、走廊、楼梯连接：

```mermaid
flowchart LR
    Room1["房间 A"] -->|"门\n宽度 1m"| Corridor["走廊"]
    Room2["房间 B"] -->|"门"| Corridor
    Corridor -->|"楼梯\n宽度 1.2m"| Floor2["上层走廊"]
    Corridor -->|"大门\n宽度 1.5-3m"| Outside["室外"]
```

**连接规则**：
- 同一楼层：相邻房间直接开门，非相邻房间通过走廊连接
- 跨楼层：楼梯连接（位置在建筑生成阶段已确定）
- 门类型：普通木门/铁门/拱门/屏风（由文化决定）

## 八、性能考量

房间生成的数量级分析：

| 聚落等级 | 建筑数 | 每建筑平均房间数 | 总房间数 |
|----------|--------|-----------------|---------|
| 村庄 | 15-40 | 2-4 | 30-160 |
| 小镇 | 50-150 | 3-6 | 150-900 |
| 城市 | 200-1500 | 4-10 | 800-15000 |
| 城堡 | 30-80 | 5-15 | 150-1200 |

**优化策略**：
- 模板房间 O(1) 生成，仅个性化房间需要详细计算
- 未进入玩家视野的房间不生成内饰（LOD 延迟）
- 房间数据使用紧凑二进制格式，非交互距离用统计摘要

## 九、关键数据结构

```gdscript
# room_data.gd (概念)
class RoomData:
    var function_type: int          # 房间功能 ID
    var culture_style: int          # 文化风格 ID
    var area: float                 # 面积 m²
    var position: Vector3           # 房间中心位置
    var core_furniture: FurnitureData    # 核心用具
    var other_furniture: Array[FurnitureData]  # 其余用具
    var wall_style: int             # 墙面风格
    var floor_style: int            # 地面风格
    var ceiling_style: int          # 天花板风格
    var lighting: Array[LightData]  # 照明源

class FurnitureData:
    var type: int                   # 用具类型 ID
    var quality: int                # 0-4
    var position: Vector3           # 相对位置
    var rotation: float             # 朝向
    var is_interactive: bool        # 是否可交互
    var storage_inventory: Array    # 容器内容（如果是容器）
```
