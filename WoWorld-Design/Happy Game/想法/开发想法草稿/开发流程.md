## 阶段〇：最小生存循环（3-6月）

### 目标

创造一个体素世界，放入一个会饿的胶囊体，让它找食物。这是整个游戏的第一行可运行代码。

### 完成标准

- [ ] Godot 4.6.2 项目初始化，导入 Voxel Tools 插件
- [ ] 体素地形生成（至少3层噪声叠加 → 基本高低起伏）
- [ ] 体素网格 → 等值面Mesh渲染（Marching Cubes）
- [ ] 体素地形生成（至少3层噪声叠加 → 基本高低起伏）
- [ ] 体素密度场 → Marching Cubes等值面Mesh渲染
- [ ] 玩家角色：可移动的第三人称胶囊体 + 基础摄像机
- [ ] 1个NPC胶囊体：
  - 有最简单的 NPCData（饥饿值 + 位置）
  - GOAP 安全网已接通：饿了 → 找食物 → 吃 → 饱了
  - 能找到场景中预设的食物
- [ ] 世界交互：玩家能用工具雕刻地形（SDF操作验证）
- [ ] 基础光照、天空、简单的环境

### 技术要点

| 项目 | 方案 |
|------|------|
| 体素引擎 | Voxel Tools (Zylann) 或 Voxel-Core |
| 等值面 | Transvoxel（Marching Cubes的LOD变体） |
| NPC移动 | Godot NavigationServer + 体素地形导航网格烘焙 |
| NPC数据 | Resource (.tres) 或 Dictionary 直接读写 |

### 不要做的事

- ❌ 不要做美观的UI
- ❌ 不要做多个NPC
- ❌ 不要做世界生成细节
- ❌ 不要做战斗

### 里程碑：一个会饿的胶囊体在体素世界里找苹果吃。

## 阶段一：心智地基（6-9月）

### 目标

NPC拥有完整的数据本体、感官、GOAP安全网和基础情绪引擎。这时候的NPC是一个"有感觉的生存者"。

### 完成标准

- [ ] **NPCData 完整结构**
  - identity、personality（大五人格）、physiology、emotion 字段全部就位
  - 全局NPC字典（Dictionary存储，按ID索引）
  - 序列化/反序列化通过
- [ ] **感官外壳**
  - 视觉：射线检测视线遮挡 + 距离限制
  - 听觉：声音事件半径 + 衰减
  - 嗅觉（预留）：气味标记点
- [ ] **GOAP安全网完整**
  - 4大生存目标：进食、饮水、睡眠、治疗
  - 规划器能从当前状态找到行动链达成目标
  - 效用评估 → 选择最优规划
- [ ] **基础情绪引擎**
  - 三维轴（愉悦/觉醒/掌控）实时更新
  - 8种基本情绪强度
  - 情绪惯性 + 冲击力覆盖
  - 生理拉扯（饿→愉悦↓）
- [ ] **分层模拟 L1/L2/L3 框架**
  - 距离触发升降级
  - L1 全模拟（半径50m内）
  - L2 轻量（衰减+简化状态机）
  - L3 纯数据
- [ ] **多个NPC（10-20个）同时在场景中运行**

### 技术要点

| 项目 | 方案 |
|------|------|
| GOAP规划器 | 经典GOAP：世界状态→规划器→行动队列 |
| 情绪更新 | 分帧更新（每0.2s一批NPC，而非每帧全部） |
| NPC管理 | NPCManager 单例，按区块分组管理 |
| 性能 | 确保20个L1 NPC + 200个L2 NPC不掉帧 |

### 里程碑：20个NPC在场景中各自为生存奔波，你能看到他们饿了去吃、困了去睡。

## 阶段二：社会织网（6-12月）

### 目标

NPC之间建立关系、有记忆、记忆可以传播、可以自发组成团体。这是"涌现叙事"的根基。

### 完成标准

- [ ] **关系系统**
  - 好感度、信任度、熟悉度三个维度
  - 关系随互动自动更新
  - 社交互动：打招呼、闲聊、交易、送礼、辱骂、威胁
- [ ] **记忆系统完整**
  - EventMemory 完整结构（时间、地点、参与者、情绪编码、冲击力）
  - 编码时机：社交互动 / 目标完成/失败 / 阈值触发
  - 冲击力计算：影响范围+后果深度+道德权重+反转程度
  - 检索：按情绪一致性/时间/冲击力排序
  - 遗忘：衰减因子驱动的遗忘曲线
- [ ] **社交传播（B5b）**
  - NPC之间可以闲聊传播记忆
  - 每次传播冲击力衰减30%
  - 传闻路线（空间记忆，航点逐次丢失）
- [ ] **团体系统**
  - 团体数据结构（成员、类型、规范、共识场）
  - 团体形成：共同利益+性格相似性+地理邻近 → 概率结合
  - 团体解散：成员流失+共同目标消失 → 解体
- [ ] **文化习俗初现**
  - 行为频率统计 → 形成习俗权重
  - 文化种子系统：不同地区有不同习俗
- [ ] **玩家交互系统基础**
  - 对话选项（预制选项+关键词匹配）
  - 与NPC的基本互动（交易、赠送、攻击）
- [ ] **100-500个NPC运行**（大部分L2/L3，少量L1）

### 技术要点

| 项目 | 方案 |
|------|------|
| 记忆检索 | 加权排序 + 缓存热点记忆 |
| 关系更新 | 事件驱动，而非轮询 |
| 团体管理 | FactionManager 单例，团体间关系图 |
| 性能 | L2 NPC仅做关系衰减，不主动社交 |

### ⚡ 这是第一个"可以玩"的版本。开始内部测试。

## 阶段三：内在深度（6-9月）

### 目标

NPC有长线规划、记忆会被重构、具备意图识别能力。NPC从"生存者"变成"有故事的人"。

### 完成标准

- [ ] **长线目标栈**
  - 志向系统：每个NPC有1-3个长期志向（致富/掌权/复仇/知识/艺术）
  - 志向→子目标自动分解（致富→攒钱→开店→吸引顾客……）
  - 目标优先级动态调整（生存>志向）
- [ ] **记忆重构过滤器**
  - 自我辩护：将负面行为合理化
  - 效价扭曲：当前情绪影响回忆
  - 掌控感扭曲：成功归己/失败归外
  - 情绪一致性：更容易记起与当前情绪一致的事
- [ ] **社会认知 B4 完整**
  - 意图识别：推测他人行为的目的
  - 计划预测：推测他人接下来的行动
  - 偏见：内群体偏好/外群体歧视
  - 对玩家离谱行为的解释滤镜
- [ ] **注意力系统**
  - 微中断：被蝴蝶吸引、被声音分散
  - 注意力资源有限（L1 NPC约3-5个同时关注对象）
- [ ] **造物系统**
  - 龙、小精灵等特殊生物的初始模板
  - 特殊生物的行为规则

### 里程碑：观察一个NPC的一周，你能看到他因为被轻视而怀恨在心，计划复仇——而他记得的那件事其实是他自己选择性记忆的版本。

## 阶段四：身体与痕迹（3-6月）

### 目标

NPC的情绪可视化、行为在世界上留下物理痕迹、动画系统有LOD。游戏开始"好看"。

### 完成标准

- [ ] **非语言表达**
  - 情绪驱动微动作（开心时轻快步伐、愤怒时紧握拳头）
  - 姿态变化（自信/疲惫/戒备）
  - 注视方向（关注感兴趣的目标）
- [ ] **环境痕迹**
  - 脚印系统（常走路→土径形成，对应五级道路的2级）
  - 刻痕/纪念物（重要事件→NPC可能留下标记）
  - 建筑老化（无人维护→外观衰败）
- [ ] **动画LOD**
  - L1：完整动画（行走、跑步、工作动画、情绪微动作）
  - L2：简化动画（仅行走、空闲）
  - L3：无动画
- [ ] **基础美术资产**
  - NPC基础模型（人类男性/女性，不同体型）
  - 基础服装（农民、商人、贵族、士兵）
  - 基础建筑模型（木屋、石屋、商铺）

### 技术要点

| 项目 | 方案 |
|------|------|
| 动画 | Godot AnimationTree + BlendSpace |
| 痕迹 | 贴花系统或简单体素修改 |
| 模型 | AI生成 → Blender清理 → Godot导入 |

## 阶段五：宏大叙事（6-9月）

### 目标

英雄从社会传播中涌现、社会共识形成、道路网络有机生长、玩家能看到自己的"传奇"。

### 完成标准

- [ ] **著名度统计（B5c）**
  - 被提及次数累积 → 著名度
  - 著名度阈值触发"英雄"标签
  - 英雄的行为会成为社会记忆的锚点
- [ ] **共识场**
  - 团体层面的共享记忆/信念
  - 共识场生命周期：生（事件触发）→老（稳定传播）→病（争议）→死（遗忘）
- [ ] **五级道路系统完整**
  - 1级：世界生成时的硬化路
  - 2级：高频使用自然踩出的土径
  - 3级：个人记忆路线
  - 4级：传闻路线
  - 5级：魔法/特殊路径
- [ ] **玩家日志系统**
  - 自动记录重要事件
  - LLM可选：将事件转化为叙事性描述
  - 时间线视图
- [ ] **世界记录**
  - NPC生平摘要
  - 大事件编年史
  - 可翻阅的历史

战斗系统完整、LLM可选集成、模组接口预留。

### 完成标准

  - LLM客户端管理器

## 阶段七：打磨与测试（6-12月）

### 目标

性能达标、涌现行为合理、可以发布。

### 完成标准

- [ ] **离线时间模拟**
  - L3 NPC宏观推演（生死/婚育/迁徙/技能变化）
  - 玩家回到某区域时，世界已自然变化
- [ ] **涌现行为调试器**
  - 世界暂停+单NPC检视
  - 决策链回溯
  - 控制台命令
- [ ] **性能优化**
  - 10,000 NPC 内存 < 200MB
  - 存档序列化 < 500ms
  - 单NPC概率决策 < 0.1ms
  - L1场景 50个NPC 稳定60fps（最低配置）
- [ ] **涌现测试**
  - 长时间运行测试（游戏内数年）
  - 边界情况检测
  - 经济/社会不崩溃验证
- [ ] **数据老化**
  - 低冲击力记忆压缩为摘要
  - 未使用空间记忆删除
  - 存档版本迁移（自愈+熔断）
- [ ] **UI/UX完善**
  - 主菜单、存档管理
  - 对话界面
  - 物品栏、技能面板
  - 地图
- [ ] **本地化**（至少中文、英文）

## 开发原则

### 每个阶段的铁律

1. **产出可运行的 demo** — 不是文档、不是设计图
2. **阶段末代码冻结 + 重构评估** — 别让技术债累积
3. **性能基线测试** — 别等到阶段七才发现帧率崩了
4. **涌现行为观察日志** — 记录"惊喜时刻"和"尴尬时刻"

### 技术债管理

- 阶段〇~二：允许快糙猛，但关键接口要对
- 阶段三~五：逐步偿还技术债
- 阶段六~七：禁止新增技术债

### 砍功能原则

每个阶段结束时问自己：
1. 这个系统实际产生了好玩的涌现行为了吗？
2. 如果没有，是参数问题还是设计问题？
3. 设计问题→简化或砍掉，不要留恋
