### 核心原则

1. **数据与逻辑分离** — NPCData是纯数据字典，各引擎只读/写数据，不持有状态
2. **事件驱动** — 系统间通信通过EventBus，避免直接耦合
3. **分帧计算** — 所有重计算分散到多帧执行，绝不阻塞主线程
4. **单一真相来源** — 全局NPC字典是NPC数据的唯一存储位置

## 六、关键算法

### 6.1 情绪感染算法

```gdscript
func calculate_contagion(source: NPCData, observer: NPCData) -> float:
    var base = source.label_intensity
    
    # 8重调节因子
    var intimacy = source.relationships[observer.npc_id].get("affection", 0.0)
    var intimacy_factor = lerp(0.05, 1.0, intimacy)  # 0.05~1.0
    
    var status_gradient = get_status_diff(source, observer)
    var status_factor = 1.5 if source_higher else 0.5
    
    var similarity = get_group_similarity(source, observer)
    var similarity_factor = 1.0 + similarity * 0.3  # 1.0~1.3
    
    var personality_factor = 1.0
    if observer.agreeableness > 0.7:
        personality_factor *= 1.4
    if observer.neuroticism > 0.7 and source.emotion_pleasure < 0:
        personality_factor *= 1.3
    
    var prior_emotion_factor = 1.0
    if observer.emotion_pleasure < -0.5:
        prior_emotion_factor *= 0.5
    
    return base * intimacy_factor * status_factor * similarity_factor * personality_factor * prior_emotion_factor
```

### 6.2 记忆冲击力计算

```gdscript
func calculate_impact(event: Dictionary) -> float:
    var scope = event.get("scope", 0.0)        # 0~0.3 影响范围
    var consequence = event.get("consequence", 0.0)  # 0~0.3 后果深度
    var moral_weight = event.get("moral_weight", 0.0) # 0~0.2 道德权重
    var reversal = event.get("reversal", 0.0)    # 0~0.2 反转程度
    return scope + consequence + moral_weight + reversal
```

### 6.3 GOAP规划器（简化版）

```gdscript
func plan(npc: NPCData, goal: String) -> Array[String]:
    var world_state = get_current_world_state(npc)
    var available_actions = get_available_actions(npc)
    
    # A* 搜索行动序列
    var open_set = PriorityQueue.new()
    open_set.push(world_state, 0)
    
    while not open_set.empty():
        var current = open_set.pop()
        if goal_satisfied(current, goal):
            return reconstruct_plan(current)
        
        for action in available_actions:
            if action.preconditions_met(current):
                var new_state = apply_effects(current, action)
                var cost = current.cost + action.cost
                open_set.push(new_state, cost + heuristic(new_state, goal))
    
    return []  # 无可行计划
```

## 七、性能策略

### 7.1 核心约束

| 约束 | 值 | 策略 |
|------|-----|------|
| L1 NPC数 | ≤50 | 距离+紧急升级控制 |
| L2 NPC数 | ≤500 | 远距离自动降级 |
| L3 NPC数 | 无上限 | 纯数据，每日推演 |
| 单帧NPC感官更新 | ≤2条视觉射线 | 分摊到多帧，事件驱动听觉 |
| 情绪更新频率 | 每1-2秒/NPC | L1 50个分摊到60帧≈1个/帧 |
| GOAP搜索深度 | ≤8 | 仅在目标变化时运行(30-120s/NPC) |
| 概率决策触发间隔 | 10-30秒/NPC | 行为一旦选定持续执行数分钟 |
| 记忆检索 | ≤最近50条热缓存 | 加权排序+懒加载旧记忆 |
| 存档大小 | <3GB | JSON压缩+二进制体素 |

### 7.2 每帧 CPU 预算

| 项目 | 耗时 | 说明 |
|------|------|------|
| 感官射线(视觉) | ≤4ms | 每帧≤2条射线 |
| 情绪更新 | ≤1ms | 摊分后每帧约1个NPC |
| 决策(GOAP/概率) | ≤1ms | 决策间隔长，摊分后极低 |
| 记忆写入 | ≤1ms | 事件驱动，非轮询 |
| NPC移动+动画 | ≤8ms | L1 50个×简单位移 |
| **总计** | **≤15ms** | 在60fps(16.7ms)预算内 |

### 7.3 分帧策略

- NPCData使用Dictionary而非Resource存储（避免引用计数开销）
- 全局字典使用 `npc_registry: Dictionary = {}`（Godot Dictionary比多个Resource文件快）
- 存档使用JSON Lines格式（增量写入，不一次性序列化全部数据）
- 体素数据使用自定义二进制格式（远小于JSON）

### 8.2 版本迁移

```gdscript
func migrate_save(save_data: Dictionary) -> Dictionary:
    var version = save_data.get("data_version", [0, 0])
    
    if version[0] != CURRENT_MAJOR:
        # 主版本不兼容→熔断，提示玩家
        return {"error": " incompatible_version"}
    
    # 次版本→逐版迁移
    while version[1] < CURRENT_MINOR:
        save_data = apply_migration(save_data, version[1])
        version[1] += 1
    
    return save_data
```

## 十、避坑指南

### Godot特有问题

| 坑 | 解决方案 |
|----|---------|
| GDScript Dictionary大内存 | 用Dictionary而非Resource存储NPCData |
| NavigationServer在大地图卡顿 | 分区块烘焙导航网格，L3不参与 |
| 体素插件API不稳定 | 封装抽象层，换插件时只改封装 |
| Node _process太多 | 统一由Manager驱动，NPC节点不独立_process |
| 信号过多导致GC压力 | 同类型事件合并，批量emit |

### 设计陷阱

| 陷阱 | 警告 |
|------|------|
| NPC太聪明反而出戏 | 随机性>完美逻辑。人是不完美的。 |
| 经济完全自由 | 必崩溃。留\"神秘力量\"后门。 |
| 过早优化 | 阶段〇~二允许性能差，算法对更重要 |
| 追求完美才发布 | 阶段二就可以小范围测试了 |
