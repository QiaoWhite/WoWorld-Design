### 核心原则

1. **Rust 模拟，Godot 渲染** — Rust 侧处理所有逻辑（NPC/世界生成/战斗AI/经济/载具/魔法），Godot 侧处理渲染/UI/音频/输入。通过 GDExtension 传递 `PackedByteArray`。 ✅
2. **玩家 AI = NPC AI** — 同一套 Rust `DecisionEngine`。玩家不操作时角色完全自主战斗。 ✅
3. **数据与逻辑分离** — SoA 布局（`NpcStore` 而非逐个 `NPCData` 对象）。rayon 并行。 ✅
4. **事件驱动 + 分帧** — 系统间通信通过事件总线。重计算分散到多帧。 ✅
5. **种子确定性** — 全球种子 → 大陆形状 → 地形 → 文化 → NPC。同一种子 = 同一世界。 ✅

## 三、语言分工边界（Rust ↔ Godot）

> ✅ 已确定。这是 v3.0 最核心的架构决策。

```
Rust 侧（模拟核心）:
  ✓ 所有 NPC 逻辑（GOAP/概率决策/情绪/记忆/关系/社交）
  ✓ 世界生成全管线（噪声→密度场→Transvoxel→Mesh数据）
  ✓ 战斗 AI（玩家 + NPC 共用一套代码）
  ✓ 经济模拟（供需/价格/交易验证）
  ✓ 魔法方程式计算
  ✓ 时间/天气/天象逻辑
  ✓ 载具逻辑（移动参考系/船员GOAP/航线规划）
  ✓ 蓝图解析 + 施工调度
  ✓ Mod TOML 解析
  ✓ 骨骼矩阵批量计算（CPU side）
  ✓ 导航/寻路（含载具局部 NavMesh）

Godot 侧（渲染 + I/O）:
  ✓ 场景树管理
  ✓ 体素 Mesh 渲染（ArrayMesh）
  ✓ Gerstner 海洋 shader
  ✓ 体积云 compute shader (RenderingDevice)
  ✓ GPU skinning shader
  ✓ 光照/阴影/后处理
  ✓ UI 渲染（Control 节点）
  ✓ 音频播放
  ✓ 玩家输入采集
  ✓ 物理模拟（PhysicsServer3D RID）
```

### GDExtension 数据通道

```rust
// Rust → Godot: 每帧打包传输
// - NPC 位置/旋转/动画骨骼矩阵 → PackedByteArray
// - 体素 Mesh 数据（顶点+索引） → PackedByteArray
// - UI 状态更新（事件驱动，非逐帧）

// Godot → Rust: 输入+物理结果
// - 玩家输入意图 → 战斗指令层
// - 物理碰撞事件 → NPC 感知
// - Chunk 加载请求 (玩家位置)
```

> ⚠️ FFI 边界的具体数据格式、序列化协议、错误处理策略——框架确定但**详细设计尚未展开**。

  ├─→ 10. 道路网络生成
  ├─→ 11. NPC 初始化（家庭/团体/关系）

  └─→ 12. 序列化至 LMDB

## 六、战斗系统

> ✅ 权威规格 —— 见 `开发阶段/战斗/`（14 篇开发文档）

### 6.1 核心原则

**半自动战斗**。玩家 AI = NPC AI = 同一套 Rust 代码。玩家选择战斗风格（=人格偏移）→ AI 自动执行。玩家随时可手动干预关键决策。

### 6.2 三层模型

```
战略层 (5-30s + 事件驱动)
  ├─ 总目标 + 阶段目标 + 限定条件
  ├─ GOAP 规划（≤8 并发）
  └─ 人格×情绪×记忆×知识×处境 搅拌

节奏层 (0.5-1s)
  ├─ 倾向：前压/稳守/周旋/猛攻/脱战
  └─ 武器射程×魔力储备×掩体评估×队友状态

本能层 (逐帧)
  ├─ 自动闪避/格挡/反击
  └─ 本能化法术（高度熟练后自动化）
```

## 八、经济系统

> ⚠️ 设计方向确定，详细规格尚未展开。

### 8.1 核心原则

- **受控动态经济**——完全自由的模拟经济必崩溃。保留"神秘力量"后门（基础价格锚点）。
- **供需驱动**——局部供需决定价格。贸易路线连接区域市场。
- **NPC 全参与**——每个 NPC 都是经济参与者（生产者/消费者/商人）。
- **铁匠和冒险者同一条经济系统**——不区分"玩家经济"和"NPC 经济"。

## 十一、关键算法（Rust 侧）

### 11.1 情绪感染

```rust
fn calculate_contagion(
    source_emotion: &EmotionState,
    observer: &NpcRef,
    relationship: &RelationData,
    observer_personality: &BigFive,
) -> f32 {
    let base = source_emotion.label_intensity;
    
    // 亲密度因子 (0.05~1.0)
    let intimacy = relationship.affection;
    let intimacy_factor = lerp(0.05, 1.0, intimacy);
    
    // 地位梯度
    let status_factor = if source_has_higher_status { 1.5 } else { 0.5 };
    
    // 群体相似度 (1.0~1.3)
    let similarity = compute_group_similarity(source, observer);
    let similarity_factor = 1.0 + similarity * 0.3;
    
    // 人格调节
    let mut personality_factor = 1.0;
    if observer_personality.agreeableness > 0.7 { personality_factor *= 1.4; }
    if observer_personality.neuroticism > 0.7 && source_emotion.pleasure < 0.0 {
        personality_factor *= 1.3;
    }
    
    // 先验情绪
    let prior_factor = if observer.emotion.pleasure < -0.5 { 0.5 } else { 1.0 };
    
    base * intimacy_factor * status_factor * similarity_factor * personality_factor * prior_factor
}
```

### 11.2 记忆冲击力

```rust
fn calculate_impact(event: &Event) -> f32 {
    // 四维度评估：范围(0-0.3) + 后果(0-0.3) + 道德(0-0.2) + 反转(0-0.2)
    event.scope + event.consequence + event.moral_weight + event.reversal
}
```
