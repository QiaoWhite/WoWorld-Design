“战斗信息”是你整个战斗系统的感知层。它连接着**世界状态**与**决策行为**，是所有 IF…THEN… 规则的条件来源，也是目标管理、交流、锁定的依据。以下从定义、属性、分类到系统设计四个层次深入展开。

---

## 一、什么是“战斗信息”

战斗信息不是简单的“敌人血量”，而是**角色在战斗情境下所能感知、推断、记忆的一切战场要素的结构化表示**。

在 WoWorld 中，战斗信息有两条根本原则：

1. **信息不对称**：没有“全局可知”。一个角色能获取的战斗信息，受限于其感知能力、朝向、视线遮挡、精神状态。铁匠背对着你，就不知道你拔出了剑。
2. **信息有代价**：获取高质量战斗信息可能需要专注值、时间或特定技能。不付出代价，你只能得到模糊印象（“那边有动静”），而非精确数据（“一个持斧的兽人，正在蓄力重击”）。

---

## 二、战斗信息的属性

每一则战斗信息都有以下元属性，这些属性决定了它的使用方式与有效范围。

### 2.1 来源
信息从哪里来？

- **视觉**：可见的敌人、动作前摇、特效、环境变化。
- **听觉**：武器出鞘声、脚步声、吟唱声、呼救声、远处爆炸。
- **感知/直觉**：非五感的高阶能力——感知魔力波动、杀气、隐身单位。
- **记忆**：战斗前已知的敌人情报、之前识别的信息残留（“我记得那个弓箭手躲在那棵树后”）。
- **推理**：从已知信息推导出的新信息（“他往左边躲了两次，第三次可能还是左边”）。
- **交流传递**：战友告知、敌人嘲讽中暴露的信息。

### 2.2 可信度
信息不一定准确。

- **确定**：直接观测，无误。（“我看到他在吟唱”）
- **模糊**：感知受限，只有范围。（“那边有人影，大概三到五个”）
- **猜测**：推理所得，可能错误。（“他的法力应该快耗尽了”）
- **过时**：曾经验证，但可能已变化。（“三秒前他在柱子后面”）

### 2.3 时效性
信息随时间衰减。

- **实时**：当前帧有效。
- **短期有效**：数秒内有效，过期标记为过时。
- **长期有效**：整场战斗有效（如敌人的种族、已知弱点）。
- **永久**：写入 NPC 记忆，战后仍存在（“那个戴红围巾的佣兵很擅长反制魔法”）。

### 2.4 共享范围
- **仅自身**：未传达给任何人。
- **小队共享**：队伍内部可获取。
- **公开**：战斗中所有人都能感知（如巨龙的咆哮）。

## 三、战斗信息的分类

### 3.1 自身信息
角色对自己的实时认知。

| 子类 | 内容示例 |
|------|----------|
| 资源状态 | 血量、法力、耐力、架势值、专注值 |
| 异常状态 | 中毒、燃烧、眩晕、恐惧、被标记 |
| 装备状态 | 武器耐久、盾牌损毁、箭矢余量 |
| 可用技能 | 冷却完毕的技能列表、因状态被锁的技能 |
| 姿势与硬直 | 是否处于后摇、硬直、倒地、浮空 |
| 位置信息 | 绝对坐标、朝向、所处地形标签 |

自身信息是 **可信度=确定、时效=实时** 的特殊信息，不需要感知检定。

---

### 3.2 敌人信息
对当前敌对目标的一切感知。

| 子类 | 内容示例 |
|------|----------|
| 基本属性 | 种族、体型、大致装备类型 |
| 状态推测 | 血量估计（健康/受伤/濒死）、魔力估计、耐力估计 |
| 当前动作 | 空闲、蓄力、吟唱、格挡、闪避、硬直 |
| 攻击前摇 | 识别到的攻击动作阶段（起手/持续/收招） |
| 锁定目标 | 敌人当前正在看/打谁 |
| 位移意图 | 正在接近、正在拉开、正在绕后 |
| 弱点/抗性 | 已知或推测的部位弱点、属性弱点 |
| 心情/士气 | 愤怒（出招更快更乱）、恐惧（倾向防御）、绝望（可能逃跑或求饶） |
| 个性表现 | 鲁莽者频繁突进、谨慎者保持距离、狡猾者佯攻 |

---

### 3.3 环境信息
战场本身的信息。

| 子类 | 内容示例 |
|------|----------|
| 地形 | 高地、狭窄通道、可破坏物、不可通行区域 |
| 掩体 | 可阻挡远程/视线的物体及其位置 |
| 介质 | 可遁地的土壤、可踏浪的水域、可攀爬的墙面 |
| 中立单位 | 平民、牲畜、无关NPC的位置 |
| 战场边界 | 撤离点、陷阱区、毒雾区 |
| 可交互物 | 可引爆的桶、可触发的落石、可拾取的武器 |

---

### 3.4 友军信息
对战友的感知。

| 子类 | 内容示例 |
|------|----------|
| 位置与动作 | 友军在哪、正在做什么 |
| 资源状态 | 友军血量、法力估计 |
| 当前目标 | 友军锁定的是谁 |
| 意图与需求 | 友军在交流中表达的请求、警告 |
| 士气状态 | 是否恐惧、动摇、兴奋 |

---

### 3.5 战斗全局信息
对整个战局的综合评估。

| 子类 | 内容示例 |
|------|----------|
| 人数对比 | 敌我数量 |
| 战局态势 | 优势/均势/劣势 |
| 时间压力 | 战斗已持续多久、是否有增援倒计时 |
| 关键事件 | 谁刚刚阵亡、谁使用了决定性技能 |

---

### 3.6 元信息
关于信息本身的信息。

| 子类 | 内容示例 |
|------|----------|
| 注意力分配 | 当前关注的对象、被忽略的威胁 |
| 信息缺口 | “我不知道那个方向有什么”——影响谨慎程度 |
| 误导状态 | 是否被佯攻、幻象欺骗 |

## 四、战斗信息的系统运作

### 4.1 获取：识别

识别是战斗信息系统的入口，分两种模式：

**触发式识别**
事件驱动，立即执行：
- 视野内出现新敌人 → 生成敌人信息条目
- 听到武器出鞘声 → 生成听觉信息（来源=听觉，可信度=模糊）
- 受到伤害 → 更新自身信息、生成攻击来源信息

**周期式识别**
固定间隔刷新：
- 更新所有已识别敌人的位置、动作、状态推测
- 检查已过期的信息是否需要丢弃或标记
- 重新评估战局态势

周期识别的频率受 **专注值** 影响。专注值高时识别精细（更多信息、更高可信度）；专注值低时识别粗糙（可能漏掉侧面威胁，或误判敌人意图）。

---

### 4.2 存储：战场信息池

每个角色的战斗信息独立存储为 **战场信息池**，是一个随时间动态变化的结构。

信息池中的条目遵循生命周期：
```
生成 → 更新 → 过期 → 丢弃（或被记忆保留）
```

**信息冲突处理**：当新信息与旧信息矛盾时（如“他在左边”vs 新的视觉信息“他在右边”），以可信度高的为准。若可信度相当，则进入 **疑惑状态**，可能触发谨慎行为（如放弃当前攻击，重新识别）。

---

### 4.3 消费：谁在用战斗信息

战斗信息的消费者遍布整个战斗流程：

| 消费者 | 消费方式 |
|--------|----------|
| 自察模板 | 读取自身信息与敌人信息，激活对应的 IF…THEN… 规则 |
| 限定条件 | 读取攻击行动的范围与环境中立单位信息，判断是否否决 |
| 目标管理 | 读取全局信息，评估总目标是否需要更改 |
| 交流生成 | 读取敌人状态、友军状态，生成合适的交流内容 |
| 锁定系统 | 读取敌人信息（威胁度、距离、弱点暴露），决定锁定对象与部位 |
| 战术决策 | 读取所有信息，选择行动类别（攻击/防御/位移/道具等） |
| 技能派生 | 读取距离、硬直、资源等信息，决定具体招式 |
| 表现层 | 读取信息池，生成对应的动作、音效、特效 |

---

### 4.4 不对称性的设计意义

战斗信息的不对称性和可信度系统，自然地产生了以下涌现玩法：

- **侧击与偷袭有效**：不是因为技能说明里写了“背后伤害+50%”，而是因为被袭击者的信息池里根本没有你这个敌人条目，先手判定时感知属性几乎无效。
- **佯攻与欺骗**：你可以故意让敌人“识别”到一个虚假信息（夸张的动作前摇、假装的逃跑），诱导对方错误决策。
- **信息战**：烟雾弹不只是挡住视线，它在敌人信息池里制造了大量模糊条目，迫使其进入谨慎模式。沉默法术不只是禁止吟唱，它剥夺了敌人通过声音获取信息的能力。
- **老兵的优势**：一个经历过多次同类型战斗的 NPC，其记忆中长期存有该类敌人的弱点信息，这比一个初次遇敌的新手有更高的初始信息质量——这就是“经验”的系统化表达。

## 五、战斗信息与 WoWorld 涌现叙事的衔接

战斗信息不仅驱动当次战斗，还可以写入 **NPC 的长期记忆**：

- 战斗结束后，信息池中标记为“永久”的条目（如你的战斗风格、你使用过的标志性技能、你的残忍或仁慈行为）会被写入 NPC 记忆。
- 下次相遇时，这些记忆作为初始信息载入，影响 NPC 的自察、总目标与交流内容。
- 如果记忆被传播（NPC 之间的日常交流），你会在这个世界获得 **声誉**——“那个从不杀倒地敌人的剑客”或“那个战斗中会用恐惧法术的法师”，而这些声誉又会影响未来战斗中其他 NPC 对你的初始识别信息。
