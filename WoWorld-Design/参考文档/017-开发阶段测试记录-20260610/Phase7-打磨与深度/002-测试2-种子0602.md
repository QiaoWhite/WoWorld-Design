# Phase 7 测试 #2

> **种子**: 0x0602 | **时长**: 42分钟 | **日期**: 2026-06-10
> **测试员视角**: 玩家（A面）→ 开发者（B面）
> **特殊测试**: LLM 增强接入——对比 LLM 对话 vs 模板对话的自然度和沉浸感影响

## A面：玩家所见所闻所感

### LLM 对话测试
与 NPC_22（铁匠，好感度 +0.35——之前测试中多次互动的 NPC）对话。LLM 模式开启。

> **NPC_22**："嘿，你来了。上次你帮我赶走那个家伙之后，我一直在想——你为什么要帮我？我们那时候几乎不认识。"  
> （LLM 生成——参考了记忆中的"被帮助"事件 + 当前好感度 + NPC 人格：尽责性高→会对"欠人情"感到在意）

> **我（自由文本输入）**："因为看起来你需要帮助。而且我不喜欢欺负人的人。"

> **NPC_22**："（点了点头）嗯。我也是。……你知道吗，我年轻的时候被人欺负过。那时候没人帮我。所以后来我学了打铁——至少可以靠自己。"  
> （LLM 生成——人格：尽责+宜人→会对"被欺负"主题产生共情反应，同时带出了"为什么当铁匠"的背景故事）

**A面**：**这不是模板。这是对话。** 他说了关于他过去的事——这是模板系统中不存在的"角色深度"。

### 对比：同一 NPC，模板模式
> **NPC_22**："啊，是你。最近还好吗？"  
> （模板：greeting_familiar）

> "挺好的。你在做什么？"

> **NPC_22**："这几天在赶一批农具。播种季快到了，村子里的农民都需要新的锄头和犁。"  
> （模板：topic_work_blacksmith）

**A面**：正确。合理。但和 LLM 对话相比——**模板对话是"功能性的"，LLM 对话是"人性的"。**

### LLM 延迟感知
LLM 对话有明显的响应延迟——提问后约 1.5-3 秒 NPC 才"回答"。这 3 秒的停顿——在 A 面感觉是"他在思考怎么回答"。**延迟被拟人化为"思考时间"。**

### 总体感受
- **惊喜的**：LLM 让 NPC 有了"过去"和"为什么"——这是模板永远无法提供的深度。
- **困惑的**：LLM 的成本和延迟——在短暂的测试中无所谓，但在长时间游玩中可能累积（token 消耗+等待疲劳）。
- **一句话总结**：**LLM 不是替代模板——它是模板的"情感深度扩展"。** 模板处理 95% 的日常对话（"你好"、"我买东西"），LLM 在 5% 的深度对话中释放 NPC 的"灵魂"。

## 三：差距分析

### 核心发现：LLM = 情感深度引擎
LLM 对话的最大价值不是"生成更多文本"——是**让 NPC 有了"为什么"**。"我年轻时候被人欺负过——所以学了打铁"——这句话揭示了 NPC 的"动机起源"，使他的行为（打铁）有了情感根基。模板只能描述"他在打铁"，LLM 能描述"他为什么打铁"。

**这指向了一个更深的 AI 集成方向**：不是让 LLM 决定 NPC 的行为（那会破坏涌现），而是让 LLM 在玩家与 NPC 深度对话时**揭示 NPC 行为背后的"人格叙事"**——NPC 的行为始终由程序化系统驱动，但 NPC 对这些行为的"解释"可以由 LLM 润色。

### 改进建议
- **Phase 7 打磨**：LLM 调用的"触发条件"——不是每次对话都调用 LLM（太贵+太慢）。而是"深度对话"触发（好感度 >0.3 + 对话超过 3 轮 + 对话涉及记忆事件）。
- **成本优化**：缓存 LLM 生成的"角色背景故事"——一次生成后存入 NPC 的永久记忆，后续对话直接引用缓存而非重新生成。
