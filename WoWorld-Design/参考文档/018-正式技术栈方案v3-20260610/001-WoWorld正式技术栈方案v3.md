## 一、设计哲学的技术映射

| 哲学原则 | 技术映射（v3 增强） |
|---------|-----------------|
| **只创造规则，不创造故事** | 参数驱动的涌现引擎。天气/地形/载具/城市——全部是规则的产物 |
| **创建另一个"人世"** | 100K+ NPC × 25 万 km² = 密度约 0.4 人/km²（对标中世纪末开发地区） |
| **每个存档都是新世界** | 全球种子→大陆形状→海陆分布→地形→文化种子→全部涌现 |
| **全球多元文化自然演化** | 不同大陆/岛屿的隔离 → 文化分化；贸易路线连接 → 文化融合 |
| **冒险与生活平等** | 铁匠和冒险者共用战斗 AI、经济系统、关系网络——同一套底层 |
| **无限探索** | 种子→Chunk 无限生成。海洋 70% → 航海本身就是"发现新大陆"的探索 |
| **载具是社会的延伸** | 火车/帆船上的 NPC 继续过日子——载具是"移动的微型聚落" |

## 四、世界生成——v3 核心变化

### 4.1 全球尺度噪声（大陆形状）

```
全球噪声层 (频率 0.0001-0.001, 波长 1000-10000km)
  ├─ 大陆板块噪声 → 海陆掩码 (7:3)
  ├─ 造山带噪声（板块交界处→山脉）
  ├─ 裂谷噪声（板块内部拉伸→低地/海洋裂谷）
  └─ 热点噪声（孤立火山岛链）

海陆掩码 → 陆地: 继续展开地形生成管线
             海洋: 跳过，直到海岸线附近才展开 Chunk
```

### 4.2 分层密度场（v3 扩展）

```
最终密度 = 全球海陆掩码层(陆地=继续, 海洋=跳过或水面)
         + 基础地形层(大陆尺度骨架 → 700m 山 → 谷地 → 平原)
         + 地质特征层(洞穴/悬崖/矿脉)
         + 水文层(河流从高山流向海洋——侵蚀模拟)
         + 奇幻特征层(DensityProvider trait 可插拔)
         + 文化覆盖层(建筑地基/道路/农田/城市)
         + NPC修改层(挖掘/踩踏——批量提交)
         + 玩家修改层(SDF 雕刻)
         + 天气临时层(积雪/积水——每chunk每30s)
```

### 4.3 噪声基准参数（v3——波澜壮阔）

| 层 | 频率 | 振幅 | 用途 |
|----|------|------|------|
| 全球海陆 | 0.00005-0.0002 | 1.0（二值化→海/陆） | 大陆形状 |
| 大陆骨架 | 0.0005-0.002 | 300-800m | 造山带/大平原骨架 |
| 区域地形 | 0.005-0.02 | 50-200m | 山丘群/谷地系统 |
| 局部细节 | 0.03-0.08 | 5-30m | 脚下的起伏 |

**自动校验**：
```
max_elevation(500m_radius) - min_elevation(500m_radius) ≥ 10m  // 不死亡平板
max_elevation(10km_radius) ≥ 700m  // 保证有"真正的山"
max_elevation(10km_radius) - min_elevation(100km_radius) ≥ 1500m  // 完整垂直范围
```

### 4.4 垂直稀疏 Chunk

替代 v2.0 的均匀 16m×16m×32m Chunk：

```
Chunk 水平尺寸: 32m × 32m（不变）
Chunk 垂直范围: 动态——仅覆盖 [地表-50m, 地表+地表厚度]
  地表以上: 不存储体素（空气）, 仅地表附近存植被/建筑标记
  地下浅层: 0-50m（洞穴/矿脉/地基）
  地下深层: 按需展开（靠近特殊地质特征时）
 
活跃 Chunk 的垂直层数: ~100（vs v2.0 的 64）
  但这 100 层只覆盖 [地表-50m, 地表+5m]——大部分是稀疏的
```

### 4.5 LOD 体系（v3——含 Clipmap）

```
LOD 0-1: Transvoxel 全精度 (0.5m-1m/voxel, 0-100m)
          → 含悬挑/石桥/拱门——完整 3D 细节
LOD 2-3: Transvoxel 中等 (2m-4m/voxel, 100-500m)
          → 悬挑细节开始简化，但主要特征保留
LOD 4-5: Transvoxel 低精度 (8m-16m/voxel, 500m-3km)
          → 大型岩架保留，细小悬挑丢失
LOD 6-7: Signed Heightfield + 法线贴图烘焙 (32m-64m, 3km-10km)
          → 悬挑→法线贴图的光影欺骗。山体轮廓和雪线正确。
LOD 8:   Billboard + 大气散射融合 (10km+)
          → 天际线、远山轮廓、浮空岛剪影
```

### 4.6 水文——河流与海岸

```
全球 → 区域 → 局部 的河流网络:
  高山降水 → 径流累积 → 小溪 → 支流 → 大河 → 入海口
  河流侵蚀两岸 → 峡谷/河谷 → 冲积平原（下游）
  
海岸线:
  构造海岸（悬崖/峡湾——造山带直伸入海）
  沉积海岸（沙滩/滩涂——平原入海）
  珊瑚礁/红树林——热带浅海
```

### 4.7 海洋——Gerstner 波

```rust
trait OceanProvider {
    fn wave_height(&self, x: f64, z: f64, time: f64) -> f64;
    fn wave_normal(&self, x: f64, z: f64, time: f64) -> Vec3;
    fn foam_density(&self, x: f64, z: f64, time: f64) -> f32;
}
```

- Godot 侧：Gerstner shader——~8-12 个波叠加，GPU 成本 ~0.5-1ms
- Rust 侧：船只浮力采样——`wave_height()` 在船体下方 4-6 个样点
- 浪高 = f(风速, 水深, 离岸距离)
- 暴风雨 → 浪高 ×3-5 → 航海危险

**未来升级接口**：`OceanProvider` trait——Gerstner 实现可替换为 FFT 实现，不影响上层逻辑。

### 4.8 空间语义标签层

不变——同 v2.0 §4.4。但新增"海洋标签"（海峡、海湾、洋流）和"山岳标签"（山峰、山脊线、山坳/垭口）。

### 4.9 世界边界设计

初步版（25 万 km²）：自然海洋边界包裹整个大陆。海洋延伸到约 500km 后——进入"无尽的深海"——航行到此的船只遇到"无法航行的风暴带"（软边界，v2.0 §11.5 的软边界逻辑）。

最终版（Minecraft 级）：种子无限延伸——无硬边界。

## 五、天空——混合体积云

### 5.1 三层架构

| 层 | 技术 | 距离 | GPU 成本 |
|----|------|------|---------|
| 地平线层云 (>5km) | 2D impostor（噪声纹理+大气散射） | 远 | <0.2ms |
| 可穿越云 (0.3-5km) | 3D 体积密度场（Rust compute → Godot `RenderingDevice` ray march） | 中 | ~1.0ms |
| 高空卷云 (>8km) | 噪声纹理 + 简单 alpha 混合 | 任意 | <0.1ms |

**云 = f(天气, 季节, 时间, 纬度)**：
- 晴天 → 少量散云
- 多云 → 中低体积云覆盖 60-80%
- 暴风雨 → 厚积雨云（3D 体积，可穿越——飞行坐骑穿入暴风云中是可能的！）
- 冬季 → 低云更多，高云减少
- 热带纬度 → 积雨云更多（对流强）

### 5.2 NPC 天象感知

```
SkyPerception {
    cloud_cover: f32,        // 0-1 云覆盖率
    cloud_type: CloudType,   // 积云/层云/积雨云/卷云
    rainbow_visible: bool,   // 雨后彩虹
    storm_approaching: bool, // 暴风雨临近
    visibility_quality: f32, // 能见度
}

NPC 概率决策器读取 SkyPerception → 温和偏移:
  乌云密布 → anxiety_modifier +0.12
  彩虹出现 → mood_boost +0.15
  暴风雨前兆 → "加速当前户外任务"权重 +30%
  晴空万里 → outdoor_activity_modifier +0.08
```

**关键原则**：天象是 NPC 活动的**考量和影响因素，不是决定因素**。高神经质 NPC 对天气变化更敏感（偏移系数×1.5）；低神经质 NPC 几乎不受影响。

## 七、战斗系统——半自动 v3

### 7.1 核心原则

**玩家的战斗 AI = NPC 的战斗 AI = 同一套 Rust 代码。**

NPC 的战斗由人格参数驱动行为权重（狂暴型/战术型/支援型/施法者）。玩家选择战斗风格 → 等同于选择了一组人格偏移 → 战斗 AI 自动执行。

### 7.2 玩家指令层

```
自动执行（无需玩家操作）:
  ✓ 目标选择（最近/最弱/最威胁——取决于战斗风格）
  ✓ 移动（接近/保持距离/利用掩体）
  ✓ 普通攻击（轻/重攻击节奏由风格决定）
  ✓ 防御（格挡/躲避时机由风格决定）
  ✓ 体力管理

玩家手动干预:
  ○ 战斗风格切换（一键——狂暴↔战术↔支援↔施法）
  ○ 特殊技能释放（魔法大招/必杀技/物品使用）
  ○ 队伍命令（集火 X / 保护 Y / 撤退 / 散开）
  ○ 关键决策（投降/谈判/逃跑/追击）
```

### 7.3 战斗风格对应的 AI 行为

同 v2.0 §5.2 战斗风格参数（狂暴型/战术型/支援型/施法者）。玩家选择风格 → 底层 AI 参数调整同 NPC。

### 7.4 F3 完整战斗

- 10 元素 + 环境交互（火→森林/草原燃烧，冰→减速+水面冻结，雷→水域导电）
- 信息战（强装镇定/敌方实力误判/心理博弈）
- 空战（飞行坐骑 + 空中对决）
- 战斗士气（个人+团队层面——HP<40% 士气下降，队长阵亡→团队士气崩塌）
- 社会后果（围观 NPC 反应 → 战斗著名度记录 → 影响后续社交）

## 八、NPC 系统——v3 扩展

### 8.1 分层模拟（扩展 L4）

| | L1 | L2 | L3 | **L4（新增）** |
|---|-----|-----|-----|--------------|
| 数量 | ≤1,000 | ≤10,000 | ~89,000 | **~数百万** |
| 距离玩家 | ≤50m | 50-150m | >150m | **>5km** |
| 更新频率 | 每 0.2s | 每 1-2s | 每日/周 | **每周/月（统计批量）** |
| 心智 | 完整 | 轻量 | 统计 | **宏观统计（人口/经济/文化指标）** |
| 用途 | — | — | — | **远方城市人口基数/文明存在感** |

### 8.2 载具上的 NPC 行为

载具状态是 NPC 行为权重的新输入：
```
if on_vehicle:
    if role == CREW:
        增加 GOAP 目标: "操控载具"、"瞭望"、"维护"
    elif role == PASSENGER:
        日常行为继续（吃/喝/社交/睡眠）
        户外活动权重 × 0.3（受限于甲板/车厢空间）
        新增: "观景"（概率性驻足望向载具外）
```

### 8.3 长途旅行中的 NPC 分布

主干道（城市间道路）上动态生成"旅行者 NPC"：
- 商队（3-8 NPC + 马车/驮兽）
- 信使（1-2 NPC，快速移动）
- 流浪者/朝圣者/冒险小队
- 盗贼（由当地经济崩溃率+治理缺失涌现）

旅行者密度 = f(道路等级, 当地治安, 经济繁荣度)。

## 九、城市规划与建造

### 9.1 规划层（先于建筑生成）

```
城市结构类型:
  单中心: 教堂/城堡/市场在中心 → 环状路网 → 城墙
  多中心: 多个广场/集市分散 → 网格+放射路网
  线性: 沿河流/海岸线延伸 → 带状路网
  地形适应: 山地城市（梯田式）、港口城市（半圆形沿海）

规划步骤:
  1. 选址评估（淡水+防御+贸易+资源——已有）
  2. 确定城市结构类型（由地形+文化种子决定）
  3. 路网生成（主干道+次级街道+小巷）
  4. 功能区划分（市场区/工匠区/富人区/宗教区/贫民区/港口区）
  5. 重要建筑定位（市政厅/大市场/教堂/城墙/城门）
  6. WFC 按区域填充建筑（建筑风格受文化种子+区域功能双重约束）
```

### 9.2 城市规划类型

| 类型 | 路网形态 | 代表 | 适合 |
|------|---------|------|------|
| 单中心放射 | 环状+放射 | 中世纪巴黎 | 平原大城市 |
| 网格 | 正交路网 | 罗马军营城市 | 新建城市/殖民地 |
| 线性 | 平行于河流/海岸 | 港口城市 | 河流/海岸地形 |
| 多中心 | 多个广场+不规则路网 | 中世纪伦敦 | 自然生长的大城市 |
| 梯田式 | 等高线平行 | 山城 | 山地地形 |

### 9.3 蓝图系统

```
三途径设计 ──→ .blueprint TOML 标准化 ──→ 导入游戏世界
                                               │
                    ┌──────────────────────────┘
                    ▼
            NPC 团体获得施工任务
                    │
        ┌───────────┼───────────┐
        ▼           ▼           ▼
   收集材料    施工NPC调度   逐步建造
        │           │           │
        └───────────┴───────────┘
                    │
                    ▼
          建筑/载具 完成（历时数日-数月游戏时间）
```

### 9.4 建筑设计的参数化

基础建筑（参数化滑块）——大部分玩家的需求：
```
建筑类型: 铁匠铺/民居/酒馆/教堂/仓库/商店...
面宽: 4-12m
层数: 1-5
屋顶风格: 尖顶(西) / 曲线(东) / 平顶(沙漠) / 长屋(北欧)
立面材质: 石材/木构/砖/土坯
```

进阶建筑（模块化拼装）——追求独特的玩家：
- 从预制构件库选择（石墙/木梁/拱门/楼梯/柱式/窗框/门/阳台/壁炉）
- 在网格上拼装
- 构件受文化种子约束（东方构件库 vs 西方构件库）

## 十、载具系统

### 10.1 载具类型

| 载具 | 动力 | 轨道/航道需求 | 容量 | 建造者 |
|------|------|------------|------|--------|
| 小帆船/渔船 | 魔力 | 海洋/大湖泊 | 2-5 人 | 玩家+少数 NPC |
| 大帆船（商船/战舰） | 魔力 | 海洋 | 20-80 人 | NPC 团体 |
| 火车（客运/货运） | 魔力 | 轨道 | 50-200 人 | NPC 铁路公司 |
| 飞行船（远期） | 魔力 | 空中 | 10-50 人 | NPC 团体 |

### 10.2 移动参考系

```
载具在世界中的位置 + 旋转 = 全局变换矩阵 T_V
NPC 在载具上的位置 = 载具本地坐标 P_local
NPC 在世界中的位置 = T_V × P_local（GPU 每帧更新）

载具上的 NavMesh: 载具本地空间的局部寻路网格
  乘客在甲板/车厢内寻路 → NavigationServer 的局部 NavMesh
  船员在操控位置之间移动 → 特定的航点路径
```

### 10.3 载具上的 NPC 行为

- **船员角色**（新 GOAP 目标）：
  - `NavigateVehicle`（保持航向/轨道）
  - `Lookout`（瞭望前方——陆地/礁石/其他船只/盗贼）
  - `MaintainVehicle`（暴风雨后维修、补给燃料/魔力）
  - `AssistPassengers`（暴风雨时安抚/引导乘客）
- **乘客角色**：
  - 常规需求循环继续（简化版）
  - 新增"观景"行为——概率性驻足望向窗外/甲板外
  - 社交互动（乘客之间的闲聊、交易、赌博——海上数日的事件涌现）

### 10.4 旅行节奏调节

**海上**：魔力驱动 → 航行本身不受风向影响。但：
- 暴风雨 → 浪高 ×3-5 → 船只可能受损 → 需要靠岸维修
- 海盗 → 战斗可能发生 → 船体可能受损
- 礁石/浅滩 → 需要熟练瞭望 → 船员技能影响航线安全
- 这些因素**不阻止旅行——但调节旅行的节奏和风险**

## 十三、风险矩阵

| 风险 | 等级 | 缓解 |
|------|------|------|
| **1500m 垂直稀疏 Chunk 性能** | 🔴 高 | Clipmap LOD + 3km 切换高度场。先做 500m 山验证再扩展到 700m+ |
| **6GB VRAM 超预算** | 🔴 高 | 精确到 MB 的预算表。纹理压缩+Mesh LOD+流式加载。如超标→降低体积云分辨率 |
| **Gerstner 波+体积云+体素+城市 同时渲染** | 🟡 中 | 分帧策略——云每 2 帧更新一次，海洋 LOD 随距离降级 |
| **无限世界存档** | 🟡 中 | 种子确定性的 Chunk 流式存储。初步版（25 万 km²）LMDB 足够 |
| **半自动战斗"没有参与感"** | 🟡 中 | 关键技能手动释放+队伍命令+战斗风格切换 → 保留足够的决策参与 |
| **蓝图系统的施工调度复杂度** | 🟡 中 | 先实现参数化（滑块），模块化拼装和自然语言委托远期实现 |
| **载具移动参考系 NavMesh** | 🟡 中 | Godot NavigationServer 支持局部 NavMesh。在移动平台上验证 |
| Rust 学习曲线 | 🟢 低 | LLM 按需辅助 |
| Godot 5.0 GDExtension 不兼容 | 🟢 低 | 锁定 4.6 LTS |
