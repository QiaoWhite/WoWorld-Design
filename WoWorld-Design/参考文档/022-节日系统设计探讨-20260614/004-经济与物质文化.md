## 一、节日消费激增

### 1.1 需求激增的物品类别

| 类别 | 节日期间需求倍数 | 示例物品 |
|------|----------------|---------|
| 食物（宴会用） | ×2-4 | 肉类、酒、面包、甜品、节日限定食物 |
| 礼物 | ×3-5 | 饰品、玩具、鲜花、甜点 |
| 装饰品 | ×5-10 | 灯笼、旗帜、花环、彩带——节日特有物品 |
| 祭祀用品 | ×3-8 | 香、蜡烛、祭品（由信仰系统决定） |
| 仪式服饰 | ×2-3 | Ceremonial 服装、面具、头饰 |
| 竞技装备 | ×1.5-2 | 武器维护、箭矢、护甲——用于竞技赛事 |

**需求激增的时间线**：

```
宣告期：需求开始上升——有远见的 NPC 提前购买
预备期（节前1-3天）：需求达到峰值——最后一刻的准备
庆祝期：食物和酒持续消费——"再来一轮！"
余波期：需求回落至正常——剩余节日物品可能打折销售
```

### 1.2 市场影响

```rust
/// 节日对当地市场的经济冲击
struct FestivalEconomicImpact {
    /// 该节日期间本地市场的额外交易量（铜币）
    additional_trade_volume: u64,
    
    /// 价格波动——各物品类别的临时价格乘数
    price_multipliers: FxHashMap<ItemCategory, f32>,
    // 食物 ×1.3, 酒 ×1.5, 礼物 ×1.2, 装饰品 ×1.8
    
    /// 外地商人带来的额外供给（平衡了部分需求激增）
    external_supply_influx: f32, // 0-1，0=无外地商人，1=充足
    
    /// NPC 的平均额外支出（铜币/人）
    avg_per_capita_spending: f32,
}
```

**稳定器机制**：经济系统已有五大自动稳定器（见经济系统 008）。其中之一是"跨区流动性"——如果本地节日导致食物价格暴涨 → 邻镇商人运食物过来 → 价格回落。节日不会导致持续通货膨胀——只是 5-7 天的临时波动。

### 1.3 商店关闭

`closed_on_holidays` 标记（经济系统 002 中 `OpeningHours` 的字段）：

| 商店类型 | 节日期间行为 |
|---------|------------|
| 神殿/宗教建筑 | **开放**——节日是神殿最忙碌的时候 |
| 酒馆/客栈 | **开放**——消费激增，但店员可能减少 |
| 食品店/面包房 | **开放**（上午）→ 下午关闭去参加节日 |
| 普通商铺 | 根据 `closed_on_holidays` 和店主个人决策 |
| 铁匠铺/工坊 | **关闭**——非紧急的制造暂停 |
| 政府/行政建筑 | **关闭**——官员也过节 |

## 二、节日限定物品

### 2.1 程序化生成 vs 数据预设

**不做数据预设**。不手写"元宵灯笼 ItemDefId=0x..."。节日限定物品从节日参数程序化生成：

```rust
/// 节日限定物品——从节日参数自动派生
fn generate_festival_items(
    festival: &FestivalFramework,
    culture: &CultureCoreParams,
) -> Vec<FestivalItemTemplate> {
    let mut items = Vec::new();
    
    // 根据节日活动派生需要的物品
    for activity in &festival.activities {
        match activity {
            FestivalActivityType::Decoration => {
                // 装饰品 → 灯笼/旗帜/花环
                items.push(FestivalItemTemplate {
                    base_category: ItemCategory::Decoration,
                    festival_flavor: "lantern", // 或 "banner", "garland"
                    material: derive_material(culture),  // 纸/布/木/金属——由文化技术决定
                    color: derive_festival_color(culture, festival.emotional_tone),
                    is_consumable: true, // 节日后丢弃或储存
                });
            }
            FestivalActivityType::SpecialFood => {
                items.push(FestivalItemTemplate {
                    base_category: ItemCategory::Food,
                    festival_flavor: "festival_dish",
                    material: derive_food_material(culture, festival.bound_faith),
                    color: ColorPalette::default(),
                    is_consumable: true,
                });
            }
            FestivalActivityType::GiftGiving => {
                items.push(FestivalItemTemplate {
                    base_category: ItemCategory::Trinket,
                    festival_flavor: "festival_gift",
                    material: derive_gift_material(culture),
                    color: derive_festival_color(culture, festival.emotional_tone),
                    is_consumable: false,
                });
            }
            FestivalActivityType::Fireworks => {
                items.push(FestivalItemTemplate {
                    base_category: ItemCategory::Consumable,
                    festival_flavor: "firework",
                    material: MaterialType::Gunpowder,
                    color: ColorPalette::vibrant(),
                    is_consumable: true,
                });
            }
            FestivalActivityType::Mask => { /* 面具生成 */ }
            _ => {}
        }
    }
    
    items
}
```

### 2.2 节日限定物品的生命周期

```
制造（节前2-5天，工匠 NPC 制作）
  → 销售（节前1天至节日期间，市场/临时摊贩）
  → 消费（节日期间使用——灯笼挂起、食物吃掉、烟火点燃）
  → 残留（节后——部分物品留存为纪念品/收藏品）
```

玩家可以在节日后保留节日物品作为旅行的物质记忆。"这是北山赫拉人收获节的灯笼——我在那年秋天经过他们的村子。"

## 三、仪式服饰

### 3.1 节日服装切换

NPC 的 `OutfitSystem`（见物品系统 004）在节日期间自动切换到 `OutfitMode::Ceremonial`：

```rust
// 物品系统 004-装备系统.md 中已有 OutfitMode::Ceremonial 定义
// 节日系统提供切换触发器

fn on_festival_celebration_phase_begin(
    festival: &FestivalInstance,
    npc_registry: &mut NpcRegistry,
) {
    for npc_id in festival.attending_npcs() {
        if let Some(npc) = npc_registry.get_mut(npc_id) {
            // 如果 NPC 有 Ceremonial 服装 → 切换
            if npc.outfits.has_ceremonial() {
                npc.outfits.set_mode(OutfitMode::Ceremonial);
            }
            // 如果没有 → NPC 穿自己最好的衣服（quality 最高的 outfit）
            // 如果连最好衣服都没有 → NPC 可能在节日前制作/购买（由经济驱动）
        }
    }
}
```

### 3.2 节日服装的文化差异

| 文化参数 | 节日服装特征 |
|---------|------------|
| `artistry` 高 | 精致的节日服装——刺绣、珠宝、复杂编织 |
| `artistry` 低 | 实用性的节日服装——干净但朴素（"最好的日常衣服"） |
| `power_distance` 高 | 分层——统治者的节日服装极尽奢华，平民的简单 |
| `power_distance` 低 | 相对平等——所有人穿类似级别的节日服装 |
| `religiosity` 高 | 宗教符号在节日服装中显著——圣徽、特定颜色 |
| `indulgence` 高 | 大胆的颜色和暴露度——"节日是展示自己的时刻" |
| `indulgence` 低 | 克制但整洁——"节日是庄重的场合" |

## 四、节日装饰

### 4.1 聚落视觉变化

节日装饰不是永久的体素修改——是**临时贴花**层，叠加在建筑表面上：

```
预备期（节前1-3天）：
  → 公共建筑出现装饰贴花（神殿/市政厅/广场周边）
  → 主要街道出现旗帜/灯笼
  → 市场区域出现临时摊位结构

庆祝期：
  → 全聚落装饰达到峰值
  → 夜间有灯火效果（如果节日包含 Fireworks/Lights 活动）

余波期（节后1-2天）：
  → 装饰逐渐减少
  → 节后3天全部清除
```

### 4.2 性能

临时贴花 = 堆叠在建筑材质上的额外纹理层。不需要新 mesh，不需要碰撞体。VRAM 增量 ≤ 2MB（临时纹理 × 聚落数）。GPU 成本 ≤ 0.1ms（额外纹理采样）。

## 五、节日食物

### 5.1 与饮食文化的关联

节日食物不是随机生成的——它从文化的 `DietaryBasePreferences`（文化 004 §三）派生：

| 饮食参数 | 节日食物派生 |
|---------|------------|
| `staple: GrainBread` → | 节日面包——加果干/蜂蜜/香料的升级版日常面包 |
| `staple: Maritime` → | 节日鱼宴——特定品种的鱼仅节日捕捞 |
| `alcohol_stance: Ritualistic` → | 节日是少数允许饮酒的场合——特殊的仪式酒 |
| `spice_preference` 高 → | 节日食物用更多香料——日常不用的贵重香料 |
| `meat_role: Occasional` → | 节日是少数允许吃肉的场合——肉=节日 |

### 5.2 宗教饮食禁忌的节日联动

信仰系统可能给特定的宗教节日附带禁食或特殊饮食规则：

- 斋戒日：节日期间禁止特定食物（如肉、酒）
- 仪式食物：只有节日期间才能吃的食物（"这面包只在冬至日烤"）
- 圣餐：仪式性地分食特定食物（所有参与者共享一块饼/一杯酒）

**关键设计决策**：饮食规则由信仰系统定义，节日系统不定义禁忌。节日系统只提供"今天是宗教节日——查询该信仰的饮食规则"的接口。
