## 核心设计原则

1. **涌现优于预设**：不预设"龙舟节""圣诞节"——节日从文化参数 × 季节周期 × 历史事件耦合中产生
2. **框架 + 血肉分离**：种子上生成节日的框架（日期/名称/类型），运行时填上血肉（规模/情绪/市场效应）
3. **玩家 AI = NPC AI**：玩家参加节日的方式与 NPC 一致——同一套参与规则
4. **节日是文化的维护机制**：节日反复举行 → 文化参数被温和加固或缓慢偏移
5. **Rust 解耦**：`FestivalQuery` trait（pub）供消费方只读查询，`FestivalMut` trait（pub(crate)）仅文化系统和历史系统可修改

## 关键跨模块依赖速览

```
文化系统 (CultureCoreParams)
    │
    ├─→ 节日类型权重 (religiosity → 宗教节日, militarism → 阅兵, artistry → 艺术节)
    ├─→ 节日情绪基调 (indulgence → 放纵度, uncertainty_avoidance → 仪式严谨度)
    └─→ 节日反馈 (节日反复举行 → apply_culture_shift)
    
天气/季节 (SeasonClock)
    │
    └─→ 节日日期锚点 (day_of_year, equinox/solstice)
    
NPC 系统
    │
    ├─→ 参加决策 (GOAP 目标 + 概率权重覆盖)
    ├─→ 群体聚集 (SocialField 情绪传染)
    └─→ 节日行为 (赠礼/舞蹈/醉酒/祭祀)
    
语言表达
    │
    ├─→ TurnMode::Ritual (仪式对话序列)
    ├─→ TurnMode::Speech (公开演讲)
    └─→ 节日文本片段 (问候/祝福/回忆)
    
权力系统
    │
    ├─→ legitimacy.ritual 因子 (统治者主持节日 → 合法性增加)
    ├─→ 节日义务 (Duty 创建)
    └─→ 征服后节日变更
    
经济系统
    │
    ├─→ 消费激增 (节日物品需求 ×3-5)
    ├─→ closed_on_holidays
    └─→ 临时商人涌入

历史系统
    │
    ├─→ 节日事件记录
    └─→ 纪念性节日的触发条件
    
魔法系统 (仪式魔法 010)
    │
    └─→ 仪式组件模型 (主祭/参与者/地点/物品/时间/祭品)
```
