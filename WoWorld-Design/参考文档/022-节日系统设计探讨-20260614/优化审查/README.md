## 发现的关键问题速览

### 🔴 错误

1. **FestivalId 位布局有逻辑漏洞**——运行时创建的节日不属于单一文化，高 16bit=CultureId 编码失效
2. **`festival_count_for_culture` 的公式总和可能超过上限**——修改器直接相加未归一化
3. **FestivalInstance 未处理跨年边界**——3 天节日从 day 119 到 day 2 会出 bug
4. **`will_attend_festival` 使用"当天天气"做决策**——NPC 在宣告期（7 天前）做决定时无法预测当天天气
5. **文化反馈 0.002/年太小**——500 年不断才能累积 1.0，远弱于代际漂移，形同虚设
6. **`mandatory_attendance` 冗余**——应该从 Duty 是否存在派生，不应存为节日字段
7. **`ConquerorFestivalPolicy::Syncretize` 语义不明确**——`mapping` 字段用途含混
8. **经济冲击中的 `price_multipliers` 硬编码**——应作为经济系统的内生价格发现，不应由节日系统预定义乘数
9. **仪式组件（RitualComponents）与仪式魔法（魔法 010）的关系未定义**——两个独立系统定义了类似的"仪式组件"，接口重叠

### 🟡 遗漏

1. **节日取消机制**——战争/瘟疫/天灾进行中时节日如何被取消或中止
2. **室内/户外活动的天气敏感性区分**——不同活动类型对天气的敏感度不同
3. **节日音频设计**——钟声、音乐、人群喧哗的音频层
4. **节日竞技的具体规则**——NPC 如何比赛、如何判定胜者
5. **聚落间节日协调**——相邻聚落节日同日时的 NPC 选择
6. **节日相关的犯罪类型**——"亵渎神圣仪式""节日盗窃"等
7. **玩家日历/日志集成**——如何记录已发现的节日

### 🟡 架构

1. **`FaithCalendarQuery` trait 的归属**——信仰系统尚未正式设计，却要求信仰系统提供此 trait
2. **`FestivalInstance` 与 `HistoricalEvent` 的重叠**——实例同时存在于两个系统中
3. **`TopicCategory::Festival` 添加时机**——语言表达模块是否应该为了节日而修改自己的枚举
4. **`OutfitMode::Ceremonial` 的切换触发器耦合**——节日系统直接调用 NPC 的 outfit 切换
5. **地名系统与节日命名的关系**——是否复用 GeographicName 基础设施
6. **`FestivalQuery` 的方法粒度过细**——12 个方法可能应该分离为高频/低频 trait

### 🟢 性能

1. **群体流场缺乏具体算法描述**——仅提及概念但无实现路径
2. **涌现型节日检测的复杂度未分析**——每游戏日扫描所有 NPC 位置的开销
3. **`attending_npcs` 返回 Vec 可能产生分配**——高频调用路径上的堆分配
