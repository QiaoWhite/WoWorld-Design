# 006 — 跨模块依赖与接口全面清单

> ⚠️ **本文档是本模块设计的关键依据**。按照用户要求：本模块作为专门讨论进阶需求内容的地方，其他模块涉及本模块相关方面的，应以本模块的设计为权威。

---

## 一、依赖方向总览

```
                     ┌──────────────────────┐
                     │  进阶需求系统 (NPC)    │ ← 本模块（权威 Owner）
                     │  - esteem_deficit     │
                     │  - competence_frust   │
                     │  - need_action_match  │
                     │  - survival_suppress  │
                     │  - frustration_regr   │
                     └──────┬───────────────┘
                            │ 通过已有 trait 查询（拉取，不推送）
        ┌───────┬───────────┼───────────┬───────────┬──────────┐
        ▼       ▼           ▼           ▼           ▼          ▼
    Culture  Skill      Memory      Relation    Language    Power
    Query    Query      Query       Query       (对话)     Query
   (honor_  (get_      (recent_    (get_       (SpeechAct (legiti-
    weight  level)     mentions)   status)     ::Praise)   macy)
    _for_                      
    domain) ← 唯一新方法
```

**关键原则**：
- 本模块不从其他模块"导入类型"来定义自己的核心结构
- 其他模块可以不导入本模块的任何类型（零反向依赖）
- 唯一的跨模块新增：`CultureQuery::honor_weight_for_domain()`
- 其他所有数据通过**已有 trait** 查询——本模块只做内部计算

### 2.3 文化系统

**涉及文档**：
- `002-CultureCoreParams.md`
- `004-文化对NPC的影响.md`
- `006-文化演变.md`

**已有相关内容**：
| 内容 | 位置 | 与本模块关系 | 处理方式 |
|------|------|------------|---------|
| honor_weight (单标量) | 004 §? | 目前是一个从 7 核心参数派生的 f32 | 需扩展：增加 `honor_weight_for_domain()` 方法 |
| CultureCoreParams 10参数 | 002 | militarism/artistry/religiosity 等可直接映射 | 不需要新参数——只用已有参数派生 |
| CultureQuery trait | 006 | 需增加 1 个方法 | 新增 `fn honor_weight_for_domain()` |

**新增接口详细设计**：

```rust
// 在 CultureQuery trait 上新增（文化系统实现）：
pub trait CultureQuery: Send + Sync {
    // ... 已有方法 ...

    /// 该文化对特定认可领域的重视度
    /// 纯派生函数——从 CultureCoreParams 已有参数计算
    /// 调用频率：每次 esteem 恢复计算（低频，非热路径）
    fn honor_weight_for_domain(
        &self,
        culture_id: CultureId,
        domain: RecognitionDomain, // 由 NPC 模块定义
    ) -> f32;
}
```

**派生公式**（文化系统提供实现）：
| RecognitionDomain | 派生来源 | 理由 |
|-------------------|---------|------|
| Combat | `militarism` | 直接映射 |
| Artistic | `artistry` | 直接映射 |
| Scholarly | `long_term_orientation × 0.6 + openness_to_outsiders × 0.4` | 学术重视长期+开放 |
| Faith | `religiosity` | 直接映射 |
| Power | `power_distance` | 权力距离→对权威的认可 |
| Wealth | `indulgence × 0.5 + competition_orientation × 0.5` | 放纵+竞争→财富展示 |
| Crafting | `long_term_orientation × 0.5 + artistry × 0.3 + 0.2` | 工艺=长期+艺术性 |
| Social | `(1-individualism) × 0.7 + 0.3` | 集体主义→社交能力受尊重 |

**潜在冲突与修正**：
- ⚠️ `honor_weight` 目前是单个 f32（已有 CHG-024 契约）。保持不变——`honor_weight_for_domain` 是**新增**方法，不替代 `honor_weight`。`honor_weight` 继续作为通用的"荣誉文化重视度"标量。

## 六、本模块的权威声明

以下概念以**本模块（进阶需求系统）为权威 Owner**：

| 概念 | 说明 | 其他模块不得 |
|------|------|------------|
| esteem_deficit | 尊重/认可匮乏——心理状态 | 不得在 Life.Vitals 中定义 |
| competence_frustration | 胜任挫折——心理状态 | 不得在技能系统中重复建模 |
| RecognitionDomain | 认可领域的 8 分类 | 不得在其他模块中重复定义 |
| survival_suppression | sigmoid 软衰减 | 不得在 GOAP 中硬编码替代 |
| frustration_regression | ERG 挫折回归 | 不得在其他模块中重复建模 |
| intrinsic_motivation_weight | 形式化公式 | 不得在 SelfNarrative 中重复定义 |
| NeedSensitivity 的完整 8 字段 | 大五派生 | 其他模块不得自行定义 sensitivity |
