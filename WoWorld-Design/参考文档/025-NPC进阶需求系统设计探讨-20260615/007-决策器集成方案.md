## 二、need_action_match 扩展（9维）

### 2.1 扩展后的 urgency 计算

```
urgency_i = deviation_i × sensitivity_i

i ∈ { hunger, thirst, fatigue, element_balance, libido, social, 
      esteem, competence }
```

**不变的部分**（6维保持原样）：
- hunger, thirst, fatigue, element_balance, libido, social

**新增的 deviation**：

| 维度 | deviation | 说明 |
|------|-----------|------|
| **esteem** | `esteem_deficit`（直接） | 0.0 = 被充分认可，1.0 = 极度渴求 |
| **competence** | `competence_frustration`（直接） | 0.0 = 无挫败，1.0 = 严重挫败 |

**新增的 sensitivity**（已在 002/003 中定义）：
- `esteem`: `0.2 + extraversion × 0.5 + (1.0 - agreeableness) × 0.3`
- `competence`: `0.2 + conscientiousness × 0.5 + neuroticism × 0.3`

### 2.2 扩展后的 need_action_match 算法

```rust
fn need_action_match(
    action: &ActionType,
    physiology: &Physiology,
    npc_data: &NpcData,
    sensitivity: &NeedSensitivity,
) -> f32 {
    let satisfied = action.satisfied_needs();
    if satisfied.is_empty() {
        return 1.0; // 非需求驱动的行为
    }

    let mut total_urgency = 0.0;
    for need in satisfied {
        let deviation = match need {
            NeedTag::Hunger         => 1.0 - physiology.hunger,
            NeedTag::Thirst         => 1.0 - physiology.thirst,
            NeedTag::Fatigue        => physiology.fatigue,
            NeedTag::ElementBalance => physiology.element_balance_urgency,
            NeedTag::Libido         => physiology.libido,
            NeedTag::Social         => npc_data.social_deficit,
            NeedTag::Esteem         => npc_data.esteem_deficit,       // ← 新增
            NeedTag::Competence     => npc_data.competence_frustration, // ← 新增
        };
        let s = match need {
            NeedTag::Hunger         => sensitivity.hunger,
            NeedTag::Thirst         => sensitivity.thirst,
            NeedTag::Fatigue        => sensitivity.fatigue,
            NeedTag::ElementBalance => sensitivity.element_balance,
            NeedTag::Libido         => sensitivity.libido,
            NeedTag::Social         => sensitivity.social,
            NeedTag::Esteem         => sensitivity.esteem,       // ← 新增
            NeedTag::Competence     => sensitivity.competence,   // ← 新增
        };
        total_urgency += deviation * s;
    }

    let avg_urgency = total_urgency / satisfied.len() as f32;
    (1.0 + avg_urgency).clamp(1.0, 2.0)
}
```

## 三、intrinsic_motivation_weight 集成

### 3.1 调用时机

每次概率决策器运行时调用——与 need_action_match 在同一帧。

```rust
fn compute_total_weight(
    action: &ActionType,
    npc: &NpcData,
    physiology: &Physiology,
    goals: &[IntrinsicGoal],
    // ... 其他参数
) -> f32 {
    let base = culture_weight × habit_weight × personality_modifier
             × emotion_modifier × mood_modifier;

    let need_weight = need_action_match(action, physiology, npc, &npc.need_sensitivity);

    let intrinsic_weight = intrinsic_motivation_weight(goals, action);

    let suppression = survival_suppression(physiology);

    let env_weight = time_modifier × weather_modifier × vehicle_modifier;

    base * need_weight * intrinsic_weight * suppression * env_weight
}
```

### 3.2 goal 不活跃时的行为

```rust
fn intrinsic_motivation_weight(goals: &[IntrinsicGoal], action: &ActionType) -> f32 {
    let active_goals: Vec<_> = goals.iter()
        .filter(|g| g.state == GoalState::Active)
        .collect();

    if active_goals.is_empty() {
        return 1.0; // 无目标 → 权重不变
    }

    let mut weight = 1.0;
    for goal in active_goals {
        let relevance = action_relevance(&goal.category, action);
        if relevance < 0.1 { continue; }
        let commitment = goal.commitment_strength();
        weight *= 1.0 + relevance * commitment * 0.5;
    }

    weight.clamp(1.0, 3.0)
}
```

## 四、frustration_regression 集成

### 4.1 作为"预处理"步骤

frustration_regression **不持久化修改** NeedSensitivity——它在每次决策时创建临时副本：

```rust
fn prepare_decision_context(npc: &NpcData, goals: &[IntrinsicGoal]) -> DecisionContext {
    let mut sensitivity = npc.need_sensitivity.clone(); // Copy

    // 可能触发挫折回归——修改临时 sensitivity
    apply_frustration_regression(goals, &mut sensitivity);

    DecisionContext {
        sensitivity, // 可能是调制后的，不影响 NpcData 中的永久值
        // ...
    }
}
```

### 4.2 为什么是临时的？

- frustration_regression 反映的是**当前情境的心理状态**
- 当 goal progress 改善时，回归应自然解除
- 持久化 modulation 会留下"后遗症"——NPC 的人格表现会漂移
- 这与 weather_modifier 模式一致：下雨影响今天的行为权重，不影响明天的

## 五、情绪引擎扩展

### 5.1 新增心理牵引

```rust
fn apply_psychological_pull(
    pleasure: &mut f32,
    arousal: &mut f32,
    control: &mut f32,
    npc: &NpcData,
    dt: f32,
) {
    // 已有：社会赤字牵引
    if npc.social_deficit > 0.5 {
        *pleasure -= npc.social_deficit * 0.001 * dt;
    }

    // 新增 1：esteem 牵引
    if npc.esteem_deficit > 0.4 {
        *pleasure -= npc.esteem_deficit * 0.002 * dt;  // 不被认可 → 不愉快
        *control -= npc.esteem_deficit * 0.003 * dt;   // 不被认可 → 失控感
    }

    // 新增 2：competence 牵引
    if npc.competence_frustration > 0.3 {
        *pleasure -= npc.competence_frustration * 0.003 * dt;  // 挫败 → 不愉快(比esteem强)
        *arousal  += npc.competence_frustration * 0.002 * dt;  // 挫败 → 焦虑/烦躁
        *control  -= npc.competence_frustration * 0.005 * dt;  // 挫败 → 最强失控感
    }
}
```

**系数对比**：

| 信号 | pleasure | arousal | control |
|------|----------|---------|---------|
| social_deficit>0.5 | -0.001 | — | — |
| esteem_deficit>0.4 | -0.002 | — | -0.003 |
| competence_frust>0.3 | **-0.003** | **+0.002** | **-0.005** |

competence_frustration 对情绪的牵引最强——因为挫败感是一种尖锐的心理状态。

### 5.2 与已有牵引的叠加

这些新牵引与已有的 `apply_physiological_pull` 叠加（平行作用）。一个既饥饿又被忽视又被挫败的 NPC 会受到所有三个来源的 pleasure 衰减——这会在情绪上显著体现。

## 六、GOAP 边界——明确不变

```rust
fn should_activate(npc: &NpcData, physiology: &Physiology) -> Option<GoapGoal> {
    // ✓ 以下不变
    if physiology.hunger < 0.3     { return Some(GoapGoal::SurvivalEat); }
    if physiology.thirst < 0.25    { return Some(GoapGoal::SurvivalDrink); }
    if physiology.fatigue > 0.9    { return Some(GoapGoal::SurvivalSleep); }
    if physiology.health < 0.3     { return Some(GoapGoal::SurvivalHeal); }
    if combat_threat > 0.7         { return Some(GoapGoal::SurvivalDefend); }

    if courtship_pressure > 0.3    { return Some(GoapGoal::SeekPartner); }

    // if competence_frustration > X { ... }  ← 不添加！

}
```

## 七、完整决策流程（伪代码）

```rust
fn decide_action(npc: &mut NpcData, world: &WorldState) -> Action {
    // 1. 状态同步
    let physiology = Physiology::from_vitals(&world.get_vitals(npc.id));
    let goals = npc.self_narrative.active_intrinsic_goals();

    // 2. GOAP 检查（最先，最短路径）
    if let Some(goal) = GoapPlanner::should_activate(&physiology, &world) {
        return GoapPlanner::plan(goal, npc, world);
    }

    // 3. 准备决策上下文
    let mut ctx = DecisionContext::default();
    ctx.sensitivity = npc.need_sensitivity.clone();
    apply_frustration_regression(&goals, &mut ctx.sensitivity);

    // 4. 计算所有可用 action 的权重
    let available_actions = world.get_available_actions(npc);
    let mut weighted_actions = Vec::new();

    for action in &available_actions {
        let weight = compute_action_weight(
            action, npc, &physiology, &goals, &ctx, world
        );
        weighted_actions.push((action, weight));
    }

    // 5. 加权随机采样（带迟滞）
    weighted_random_with_hysteresis(weighted_actions, npc.last_action)
}
```
