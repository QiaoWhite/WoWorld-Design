# 005-语音管道与对话时间维度（大纲）

> **开发代号**: WoWorld (Wonder World)
> **状态**: 设计草稿（大纲·要点）

---

## 一、核心设计：内容×声音分离

- 对标"书的内容(语言表达) vs 书的物理(物品系统)"
- 语言表达模块：拥有 ExpressionRef + 文本内容
- 音频模块：拥有传播 + 时间 + 清晰度
- 语言表达不知音频存在；音频不解析文本

## 二、CurrentSpeech（语言表达写入 NpcData）

- expression_ref / started_at / word_count / words_per_second / delivery
- 语言表达不知道音频在读这个字段

## 三、音频模块检测与追踪

- 每帧轮询 CurrentSpeech → 检测 开始/继续/停止
- OngoingSpeech 追踪：elapsed/fraction/is_complete
- 停止检测：正常说完 vs 被打断（查实体状态→CancelReason）

## 四、听者查询时间维度

- AudioQuery::perceived_speech() → SpeechPerception
  - fraction_heard：距离延迟+语速×时间→传到了多少
  - clarity：受掩蔽/遮挡/介质影响
  - is_ongoing / cancel_reason / delivery / propagation_delay
- 听者用 fraction×clarity 调 lang.resolve_partial() → 部分文本

## 五、打断链路

- NPC A 说话中→桌子飞来→昏迷→语言表达设 current_speech=None→音频检测消失→cancel=CombatImpact→听者 B 只记得前半句
- 涌现：事后追问/误解/0.5信息进传播通道

## 六、清晰度→文本模糊化

- Perfect(≥0.9) → "我昨天在森林里发现了一..."
- Garbled(0.4-0.7) → "我...森林...发现..." 
- Fragment(0.1-0.4) → "好像说了关于森林的什么"

## 七、VoiceProfile——迁至音频模块

- 声学参数(base_pitch/timbre/speed/expressiveness/breathiness/roughness)
- 生成公式：种族×性别×年龄×大五人格
- VoiceEmotionModulation：情绪→声学调制(pitch_shift/speed/volume/tremor/breathiness)

## 八、TTS——留在语言表达模块

- TtsEngine trait（渲染层——不是模拟层）
- TtsConfig（玩家音量/字幕设置）
- VoiceManager 迁至音频模块（队列+打断）
