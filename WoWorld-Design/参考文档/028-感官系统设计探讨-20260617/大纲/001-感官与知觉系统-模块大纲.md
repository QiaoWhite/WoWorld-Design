## 〇、模块定位

感官与知觉系统是 WoWorld NPC 感知世界的**统一入口**。它不是一组独立的感觉器官——它是介于世界物理真值与 NPC 心智之间的**信息压缩与变形层**。

**一句话概括**：感官系统不做决策、不存记忆、不辨识实体。它只做一件事——**把世界真值转化为每个 NPC 独一无二的知觉流（PerceptBatch）。**

---

## 一、核心职责边界

### 感官系统做什么

| 职责 | 说明 |
|------|------|
| **编排已有感知通道** | 听觉（AudioQuery）、天象（perceive_sky）、内感受（Physiology）、非语言信号（NonVerbalPerception）、情绪传染——这些已有通道的输出被统一汇聚到 PerceptBatch |
| **提供缺失的感知通道** | 视觉（VisionQuery）——主动扫描，分级可见性，射线分配。嗅觉（ScentQuery）——懒采样，风场调制，时间衰减 |
| **注意筛选** | 显著性引擎——背景/信号二分类。只让值得注意的知觉进入意识 |
| **噪声注入** | 所有感官信道注入受控的、可复现的伪随机噪声 |
| **场景感知** | 输出空间上下文（开放度、光照）、声音景观背景、气味景观、天体物理事实 |

### 感官系统不做什么

| 非职责 | 实际负责方 |
|--------|-----------|
| ❌ 辨识实体（"这是龙"） | `prepare_facts()` — NPC 决策引擎的信念推导层 |
| ❌ 诊断伤势（"他受了重伤"） | 战斗情报层（CombatantAssessment）或日常信念推导 |
| ❌ 整合跨模态知觉（"听到脚步声+看到影子=同一个人"） | `prepare_facts()` 的共现推断 |
| ❌ 定义事件边界（"被攻击了"） | 记忆系统的编码逻辑 |
| ❌ 存储空间知识（"那条巷子很危险"） | 记忆系统 |
| ❌ 做审美判断（"夕阳很美"） | 审美系统 + AestheticFrameworks |
| ❌ 维护信念（"邻居偷了我的工具"） | 记忆系统 + 社会认知 |

## 三、关键设计决策

| # | 决策 | 说明 |
|---|------|------|
| 1 | **无状态纯函数管线** | `perceive(npc_params, world_snapshot, time, rng) → PerceptBatch`。感官不持有世界状态 |
| 2 | **感知节律 = 决策节律** | 不新建时钟。事件通道独立：显著事件→缩短下次决策间隔，不打断动作 |
| 3 | **感官不定义美** | 感官只报告物理事实（色温、亮度、变化趋势）。审美意义由文化框架+个人经历赋予 |
| 4 | **注意 = 异常检测，非显著性排分** | 背景/信号二分类。大多数实体标记为Background→零射线→零处理 |
| 5 | **感知退化是距离的连续函数** | LOD层级只是调度分桶。参数连续变化——可随意增减层数 |
| 6 | **PerceptBatch = 统一产出** | 同一份知觉同时供给日常决策、战斗情报、审美判断、大日志 |
| 7 | **玩家和NPC走同一套被感知路径** | 玩家在空间中也是SpatialEntity。NPC感知玩家的路径和感知其他NPC完全相同 |

## 六、PerceptBatch——统一知觉产出

```rust
pub struct PerceptBatch {
    pub entities: SmallVec<[PerceptEntry; 24]>,   // 实体知觉（离散条目）
    pub visual_scene: VisualScene,                 // 空间+审美+天体+氛围
    pub auditory_scene: AuditoryScene,             // 声音景观背景
    pub olfactory_landscape: OlfactoryLandscape,   // 气味景观
    pub body_state: BodyPerception,                // 内感受
    pub environment: EnvironmentPerception,        // 天气/温度
    pub timestamp: GameInstant,
    pub boundary_signals: PerceptionDelta,         // 相对上一批知觉的变化
}
```

## 七、视觉核心算法概要

1. **实体分类**（O(n)，无射线）：Locked / Motion / Threat / GoalRelevant / SociallySalient / Novel → 其他全部为 Background 忽略
2. **自适应射线分配**（≤8条动态预算）：环境复杂度 + 状态（战斗→翻倍，睡眠→0）+ 注意力负载 → 动态确定
3. **可见性判定**：视线检测 + 四档细节等级（FullRecognition / BlurredIdentity / MotionAndShape / BarelyVisible）
4. **运动检测**（双路径）：`SpatialEntity.velocity` + PerceptualCache 位置比较
5. **暗适应**：指数松弛模型——亮→暗 30min 适应，暗→亮 30s 适应
6. **天际线采样**：`SpatialQuery::sample_horizon()` → 自然奇观从高度图涌现

## 八、审美框架——文化×个人×跨文化

- **四独立过程**：获取（intelligence+openness门槛）、深化（当前openness）、衰减（仅受memory影响，永不完全归零）、激活（当前cognitive_clarity+社会语境）
- **无硬上限**：能学几个学几个——数量 = learn_rate / decay_rate 的自然均衡
- **多框架加权混合**：不是选一个框架——是所有相关框架在 NPC 心中竞争
