## 核心讨论：战斗感知和日常感知是什么关系？

### 问题

战斗 004 已经定义了 `CombatPerception`（含 `visible_enemies`、`HpEstimate`、`threat_level`）。感官系统也要做视觉感知。两套系统——是否重复？如何衔接？

### 关键决策：两层分离——感官做感知，战斗做情报

**采纳的方案**：

```
感官系统 → PerceptBatch（原始知觉，不区分战斗/日常）
    "实体A在5m外，人形，持金属物体，自信姿态，左臂有红色液体流出"
            │
    战斗情报层 → CombatantAssessment（战斗专用解释）
    "敌人#123: HP轻微伤，武器长剑，威胁度0.3"
```

感官不做战斗评估。战斗不重复视觉感知。层与层之间通过 PerceptBatch 传递信息。

### 关键决策：HpEstimate 从通用身体特征派生

**保留** `HpEstimate` 六档枚举（Uninjured/MinorWounds/ModeratelyWounded/SeverelyWounded/NearDeath/Unknown）——因为战斗决策需要这些离散档位。

**改变**：HpEstimate 的获取方式。从"战斗系统自己做视觉判定"→"读取 PerceptEntry.body_state 的特征，用 combat_experience 做解释"。

```
感官输出: PerceivedBodyState {
    visible_fluids: [Blood(中量)],
    posture_anomalies: [Limping],
    motion_anomalies: [Slowed],
}

战斗情报解释 (combat_experience=0.9的老兵):
    → "左腿受伤，失血中等，动作变慢" → SeverelyWounded (准确)
    
战斗情报解释 (combat_experience=0.1的新手):
    → "他在流血，走路有点瘸" → ModeratelyWounded (低估了严重度)
```

### 关键决策：信息战机制完全保留

战斗 004 §六 的七种信息战战术（示弱/威慑/隐蔽实力/误导/信息封锁/偷袭/沉默法术）——全部保留。它们的底层实现改变了：

**之前**：战斗系统直接修改 `CombatPerception` 中的信息。
**现在**：NPC A 改变自己的可观测属性（姿态、声音、运动模式）→ 感官系统产生相应的 PerceptEntry → B 的战斗情报层据此推断 → 可能被误导。

```
示弱（假装受伤）:
    A 改变自己的运动模式: 故意跛行+速度降低
    → A的 SpatialEntity.velocity降低, posture_anomalies=[Limping]
    → B的感官系统: PerceptEntry { posture_anomalies: [Limping], motion: [Slowed] }
    → B的战斗情报: HpEstimate=SeverelyWounded (可能错误!)
    → B的GOAP: "追击濒死敌人" → 中计了
    
    如果B是老兵(combat_experience>0.7):
    → 经验修正: "SeverelyWounded但不太对劲...他的眼神还很锐利"
    → 记忆提供: "上次那个人也装伤" → HpEstimate 调回 Uncertain
    → 老兵不上当——涌现的"见过这招"
```

信息战不需要新机制——它就是"改变可观测属性+让对方推断+可能出现错误"。

### 关键决策：玩家和NPC走同一套被感知路径

玩家在空间中是 `SpatialEntity`（`entity_kind=Creature`，`id=PLAYER_ENTITY_ID`）。NPC 感知玩家的路径和感知其他 NPC 完全一样——视线判定、运动检测、声音检测。玩家没有特殊待遇。这是"玩家不是特殊存在"的核心设计承诺。
