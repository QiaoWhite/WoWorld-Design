# 001-理论框架与维度论证

## 待讨论的核心问题

### 1. 为什么NPC需要"思考"而不仅仅是"反应"？

已有系统（决策引擎、GOAP、概率权重链）已经让NPC看起来有智能。为什么还需要一个独立的认知层？

### 2. WoWorld预设的"思考"哲学

- 不是形式逻辑——是关联链模型（记忆→共享元素跳转→推理）
- 不是符号AI——是统计涌现（MentalModel从记忆中归纳，置信度从证据中更新）
- 不是二值理性——是偏误调制理性（确认偏误、负面偏误等从人格派生）
- 不是上帝视角——是信息受限推理（NPC只知道ta感知到的、记得的、他人告诉ta的）

### 3. 维度论证

| 维度 | 问题 | 对应机制 |
|------|------|---------|
| 认知风格 | NPC的思维方式有差异吗？ | CognitiveStyle 4维 |
| 信念形成 | NPC如何从经历中形成关于世界的信念？ | try_induce_pattern → MentalModel |
| 信念消化 | 外来信念如何变成"自己的"？ | micro_digest + macro_digest + source转移 |
| 思考涌现 | 什么触发NPC"想"某件事？ | 6类ThoughtTrigger |
| 思考浮现 | 玩家如何感知NPC在思考？ | 3级清晰度ThoughtFragment → 6种SurfacingType |
| 智慧传递 | 信念如何在NPC之间流动？ | 已有通道 + WisdomSharing/SeekClarification |
| 睡眠认知 | NPC睡觉时心智在做什么？ | 记忆巩固 + 情绪复位 + 正则化 + 梦境 |
| 创新 | NPC能产生全新的想法吗？ | CreativeLeap → 验证 → 形式化 |
| 社会认知 | NPC能理解其他NPC的心智吗？ | MindAttribution (Theory of Mind) |

### 4. 与已有"人感"系统的关系

- 本模块是"人感"拼图的最后一块
- 已有：记忆（记住）、情绪（感受）、需求（驱动）、自我叙事（反思）
- 本模块：思考（加工）——把以上所有东西串起来的**认知管线**

### 5. 设计原则

- 零新trait：全部在NPC crate内部
- 零新调参旋钮：一切从已有维度派生
- 全部惰性：不主动轮询、不per-frame计算
- 可见行为的涌现：不弹标签、不显示UI面板
