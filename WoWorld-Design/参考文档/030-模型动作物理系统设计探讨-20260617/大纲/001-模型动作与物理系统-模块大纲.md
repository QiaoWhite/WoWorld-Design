## 〇、模块定位

模型动作与物理系统是 WoWorld 中**所有"物理存在"的完整表达层**。它定义三件事：

1. **模型** — 东西长什么样。骨架结构、网格变体、外观参数、面部表情机制、挂载系统。
2. **动作** — 东西怎么动。关键帧姿态数据库、程序化步态生成、战斗轨迹求解、多层级动画叠加、过渡衔接、受击/格挡/死亡响应。
3. **物理** — 东西怎么和世界互动。空间查询（地形/视线/实体索引）、移动解析、抛物飞行、水下浮力、双人交互约束、骨架松弛死亡物理。

**一句话概括**：模型定义"是什么"，动作定义"怎么动"，物理定义"在哪、撞到什么、怎么响应"。三者统一在同一个 Rust 模拟核心→GDExtension→Godot 渲染客户端架构中。

## 一、核心职责边界

### 本模块做什么

| 职责 | 说明 |
|------|------|
| **定义所有有物理存在的实体** | 骨架（SkeletonDef）、身体计划（BodyPlan）、网格变体（ModelVariant）、外观参数（PhysicalAppearance） |
| **驱动所有骨骼动画** | 关键帧姿态数据库（PoseDatabase）、程序化步态生成（GaitGenerator）、战斗基元轨迹（PrimitiveTrajectory）、9 层可叠加动画层（AnimationLayer） |
| **提供空间查询基础设施** | 地形查询（TerrainQuery）、实体索引（EntityIndex）、空间事件总线（SpatialEventBus）、视线检测（VisibilityQuery） |
| **处理动画衔接与过渡** | 接触锁定、指数收敛、相位匹配、动量继承、循环周期渐变 |
| **响应物理事件** | COM 抛物体飞行、崖壁碰撞、着地分级响应、骨架松弛死亡、水下浮力/阻力 |
| **管理装备挂载** | 装备替换身体部件（ReplacementMode）、附件绑定骨骼（AttachmentTransform） |
| **提供全 TOML 自定义接口** | 步态风格、战斗风格、过渡参数、面部图集、双人交互模板——全部数据驱动 |

### 本模块不做什么

| 非职责 | 实际负责方 |
|--------|-----------|
| ❌ 渲染网格/执行 GPU draw call | Godot 渲染器——本模块只输出骨骼矩阵和 per-instance 数据 |
| ❌ 加载 GLTF 文件/管理纹理 | Godot GLTFDocument——本模块只通过 GDExtension 查询已加载的 RID |
| ❌ NPC 决策（"走到哪里""攻击谁"） | NPC AI（GOAP + 概率决策器）——本模块只执行决策结果的动作 |
| ❌ 计算攻击伤害/命中判定 | 战斗系统——本模块只提供打击段的武器轨迹 |
| ❌ 管理物品属性/耐久度 | 物品系统——本模块只提供装备挂载和武器物理参数读取 |
| ❌ 生成地形/密度场 | 世界生成系统——本模块通过 TerrainQuery trait 消费地形数据 |
| ❌ 识别实体/语义理解 | 感官系统 → 信念推导层——本模块只提供空间查询原始数据 |

## 三、关键子模块

| 子模块 | 核心内容 | 关键决策 |
|--------|---------|---------|
| **空间查询系统** | TerrainQuery/EntityIndex/SpatialEventBus/VisibilityQuery 四个 trait。纯 Rust 实现。仅玩家 CharacterBody3D 保留 PhysicsServer3D | 放弃 PhysicsServer3D 作为模拟侧物理依赖 |
| **模型定义系统** | 人形骨架 35 骨(L0)/33 骨(L1)。TABS 式 512² 面部图集驱动表情。双骨蒙皮。非人形骨架从 BodyPlan 程序化生成 | 面部表情用纹理图集替代骨骼驱动 |
| **关键帧姿态系统** | ~38 个模块姿态。日常/社交/劳作/施法/战斗。四阶段动作包络(Anticipation→Strike→Follow-through→Recovery) | 模块化骨架——Lower/Upper 独立关键帧，脊柱桥接组合 |
| **程序化动画系统** | 9 参数步态风格从大五人格派生。15 条武器/空手基元轨迹。程序化非人形步态(四足/蛇形/鸟飞/鱼游) | Keyframe 只用在必须的地方——人体微妙姿态 |
| **动画衔接系统** | 接触锁定、指数收敛、相位匹配、动量继承、分区过渡 | 五项自然衔接法则 |
| **物理响应系统** | COM 抛物体飞行、崖壁碰撞、三级着地响应、骨架松弛死亡、水下浮力 | 不做布娃娃——做骨架松弛物理 |
| **装备挂载系统** | 替换式装备（非叠加）。AttachmentTransform 骨骼绑定。WeaponPhysicalParams→轨迹适配 | — |
| **性能与LOD** | 1000 L1 NPC CPU 0.50ms / GPU 0.72ms。多级骨骼LOD+Mesh LOD。相位偏移调度 | GTX 1660 SUPER 6GB 60fps 可行 |

## 四、跨模块关键接口

| 消费方 | 接口 | 方向 |
|--------|------|------|
| 感官系统 | TerrainQuery, EntityIndex, SpatialEventBus, VisibilityQuery | 感官消费本模块提供的空间查询 |
| NPC AI | AnimationBodyState（读取当前动作）, animation tick 输出 | NPC 发出动作指令 → 动画执行 |
| 战斗系统 | PrimitiveTrajectory, CombatStyleParams, 基元链求解 | 战斗发出基元序列 → 动画生成轨迹 |
| 物品系统 | WeaponPhysicalParams, EquipmentMounting, JointType | 物品定义物理参数 → 动画适配 |
| 生命系统 | BodyPlan, SkeletonVariant, Vitals→GaitStyle 派生 | 生命定义身体计划 → 动画选择骨架 |
| 音频系统 | AnimationSoundMarker → SpatialEventBus | 动画推送音效事件 → 音频消费 |
| 世界生成 | TerrainQuery 的具体实现 | 世界生成提供密度场 → 空间查询读取 |
| Godot 渲染 | bone_matrices: [Mat4; N] + PerInstanceData, 双缓冲共享内存 | Rust 推送 → Godot 渲染 |

## 五、自定义与扩展接口

| 扩展点 | 格式 | 说明 |
|--------|------|------|
| 步态风格 | TOML `gait_styles` | 参数覆盖或新步态定义 |
| 战斗风格 | TOML `combat_styles` | 速度/力量/机动性/佯攻偏好 + 基元加权表 |
| 过渡参数 | TOML `transitions` | 每对动作的 duration/ease_curve/lock_bones |
| 面部图集 | TOML `facial_atlas` | 嘴型/眉型/眼态变体数 + 文化默认映射 |
| 双人交互模板 | TOML `interactions` | 锚点+约束定义 |
| 自定义动画层 | Rust `CustomAnimationLayer` trait | 注册到动画层栈 |
| 自定义步态生成器 | Rust `CustomGaitGenerator` trait | 适用于特定身体计划 |
| 自定义武器轨迹 | Rust `CustomPrimitiveTrajectory` trait | 新增基元或覆盖已有 |

---

## 六、不在本模块范围内

- 渲染管线（Godot 侧 MultiMesh + GPU skinning shader + cel shading）
- 植被动画（vertex shader 顶点偏移，不经过骨骼系统）
- NPC AI 决策（GOAP/概率决策器/情绪引擎——属于 NPC 模块）
- 战斗伤害计算/命中判定（属于战斗模块）
- 技能系统（属于技能模块）
- 音乐系统（属于音频模块）
- UI/HUD（属于 Godot 侧）
