## 概述

系统性地评估 WoWorld 将游戏引擎从 Godot 切换至 Bevy 的三种可能路径：

- **场景 A**：完全切换至纯 Bevy 引擎
- **场景 B**：Godot + Bevy ECS 混合方案（通过 godot-bevy 集成）
- **场景 C**：保持现状 — Godot 4.6 LTS + Rust GDExtension（基线对照）

## 核心结论

**推荐保持 Godot + Rust GDExtension 现状，持续观察 Bevy 演进。**

- Godot 在 10/12 维度中持平或领先 Bevy
- 纯 Bevy 方案技术上可行，但需 10-18 个月纯迁移工时
- 混合方案对 Solo 开发者增加了三层架构维护负担
- Bevy 是未来——但不是 2026 年的未来
- Rust 核心已被设计为引擎无关——迁移保险已买入

**重新评估 Bevy 的触发条件**：Bevy 1.0 + 官方编辑器可用 + UI/动画/音频生态成熟 + 体素方案开箱可用（预计 2027-2028）。
