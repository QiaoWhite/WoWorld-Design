## 一、本模块是什么

**NPC 行动涌现与物理原子层** 定义 WoWorld 中 NPC 与世界交互的**通用物理语法**——极少数物理基元 × 材料属性 × 连续参数 = 无限行为涌现。

它不是一份"NPC 能做什么"的清单，而是一套**三层架构**：

```
第三层: 社会抽象原子 (~25) — GOAP 规划器操作
        TRADE, PRAY, JUDGE, TEACH, MARRY, CELEBRATE...
        ↑
第二层: 领域复合原子 (~40) — 各模块实现 execute()
        CRAFT, CAST, HARVEST, PARRY, DANCE, SIGN...
        ↑
第一层: 物理基元 (35) — woworld_core, SoA 批量执行
        MOVE, GRASP, STRIKE, ATTACH, IGNITE, OBSERVE...
```

---

## 二、模块边界

### 本模块负责
- 定义 35 个物理基元的完整签名和行为语义
- 定义 AgentSnapshot（连续能力参数快照）的结构和派生规则
- 定义 MaterialProperties 的数据模型和注册表
- 定义三层原子之间的消费/组合关系
- 定义年龄/状态如何连续影响行为（非门控）
- 定义原子执行→音频/感官/历史事件的自动产出规则

### 本模块不负责
- 各领域模块的具体实现（经济模块自己写 TRADE 的 execute()）
- GOAP 规划器的决策算法（NPC 模块负责）
- 具体的动画资产或视觉表现（模型/动画模块负责）
- 技能教学系统的具体公式（技能系统负责）
- 物品制造的具体配方内容（物品系统负责）

## 三、核心设计原则

1. **零年龄门控** — 不写 `if age < X { cannot_do(Y) }`。所有行为差异从 AgentSnapshot 连续参数涌现
2. **零硬编码禁止** — "中毒不能劳动"不是规则，是 health_penalty → 全原子效用下降 → GOAP 自然选择买药 > 打工
3. **动画是物理的输出** — IK 解算从武器轨迹生成动画，零预设战斗动画
4. **材料属性是数据** — 新增材料 = 新增一行 TOML，不新增代码
5. **玩家=NPC** — 同一套原子引擎，同一套 AgentSnapshot 参数
6. **只创造规则，不创造故事** — 原子定义交互的可能性空间，具体故事由模拟涌现

## 四、与其他模块的关系

### 消费本模块的模块
- **NPC 活人感**: 使用 AgentSnapshot 做 GOAP 决策，注册 ActionCandidate
- **战斗**: STRIKE/THROW 的轨迹参数化 + 碰撞检测
- **魔法**: CAST/CHANT/CHANNEL 等复合原子的实现
- **经济**: TRADE/CRAFT/EXCHANGE 等复合原子的实现
- **文化/信仰**: DANCE/PRAY/SACRIFICE 等复合原子的实现
- **生命**: SLEEP/NURSE/INGEST 等复合原子的实现
- **模型/动画**: 物理原子驱动 IK 解算和骨骼矩阵

### 被本模块消费的模块
- **物品系统**: MaterialProperties 注册表（MaterialDef）
- **技能系统**: execution_noise() 函数
- **生命周期系统**: AgentSnapshot 的 lifecycle 参数来源
- **天气系统**: AgentSnapshot 的 environment 参数来源

| 概念 | 一句话定义 | 详细位置 |
|------|-----------|---------|
| **物理基元** | 身体与世界的最小可交互单元（MOVE/GRASP/STRIKE...） | 开发阶段文档 001 |
| **AgentSnapshot** | NPC 的连续能力参数快照（~96 bytes SoA），贯穿全三层 | 开发阶段文档 001 |

| **execution_noise** | 技能等级→原子执行噪声的连续映射函数 | 开发阶段文档 001 + 技能系统 004 |
| **KnowledgeSeed** | 技术时代×区域的初始知识条件 TOML 数据 | 开发阶段文档 001 + 技能系统 004 |
| **IK 动画管线** | 从武器轨迹反推骨骼姿态的解算器 | 开发阶段文档 001 + 战斗 014 |
| **碰撞扫掠** | 武器碰撞体沿轨迹扫掠→与身体碰撞体求交→命中判定 | 开发阶段文档 001 + 战斗 014 |
