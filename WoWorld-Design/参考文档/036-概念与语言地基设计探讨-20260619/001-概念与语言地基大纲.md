## 一、本模块是什么

**概念与语言地基系统** 定义 WoWorld 中 **NPC 如何思考、理解、表达和传承概念** 的底层规则。

它回答五个核心问题：

| # | 问题 | 答案位置 |
|---|------|---------|
| 1 | NPC 记忆中的词汇和概念从哪里来、如何构成？ | [[../../Happy Game/开发阶段/概念与语言地基/002-概念空间与模式识别|002-概念空间与模式识别]] |
| 2 | NPC 的思考和对话同语言有关的侧面如何实现？ | [[../../Happy Game/开发阶段/概念与语言地基/003-Utterance与渲染管道|003-Utterance与渲染管道]] |
| 3 | NPC 如何理解逻辑关系、培养理性或感性思维？ | [[../../Happy Game/开发阶段/概念与语言地基/005-推理系统|005-推理系统]] |
| 4 | NPC 的知识如何代际传递？ | [[../../Happy Game/开发阶段/概念与语言地基/006-代际知识传递|006-代际知识传递]] |
| 5 | 不同文化和语言之间的翻译如何实现？ | [[../../Happy Game/开发阶段/概念与语言地基/004-语言与文字系统|004-语言与文字系统]] |

**核心架构**：三层模型

```
Layer 0: 地面真相（客观、全球统一）
  PatternSignature(u64) — 从权力拓扑/经济/关系中数学提取的模式指纹
  不存在"雇佣"概念——只存在"Contract+Extract(labor)+Extract(wage)"模式

Layer 1: 文化概念空间（CultureId 相对）
  每个文化用自己的方式切分世界
  "在中原文化中，模式#0x3F7A被识别为'雇佣'"
  "在兽人文化中，同一模式是'强者权利'的子集"

Layer 2: 语言词汇（LanguageId × ConceptId 相对）
  概念在特定语言中的词语投影 + 每种语言独立的片段库和构词规则
```

## 二、模块边界

### 本模块负责
- 定义 PatternSignature（客观模式指纹）和 classify_pattern()（概念识别）
- 定义文化概念空间模型（CultureConceptDef）和跨文化翻译机制
- 定义 Utterance（结构化话语）作为 NPC 语言产出的统一数据格式
- 定义 TextGenerator 的统一渲染管道（对话/自语/书籍三个入口）
- 定义语言 proficiency 和文字系统 proficiency 的习得与衰减模型
- 定义概念习得模型（从暴露累积中涌现）
- 定义推理系统（归纳/类比/演绎三种推理模式的完整覆盖）
- 定义代际知识传递的保真度数学模型
- 定义概念空间的跨代漂移模型
- 混合概念空间的加权派生规则

### 本模块不负责
- 书籍的物理存储和历史记录（历史系统负责）
- NPC 的记忆编码和检索算法（NPC/认知系统负责）
- GOAP 和概率行为树的决策（NPC 行动涌现负责）
- 具体的 TTS 音频合成（音频系统负责）
- 语音的物理传播和掩蔽（音频系统负责）
- 语言 proficiency 对技能教学的影响公式（技能系统负责）
- 权力系统的合法性计算（权力系统负责——本模块只提供语言障碍影响 legitimacy 的纯函数）
- 信仰仪式的具体内容（信仰系统负责——本模块只提供仪式语言的翻译管道）
- LLM 模型的调用和后端管理（语言表达/LLM增强层负责——本模块定义 LLM 在概念层的操作接口）

## 三、核心设计原则

1. **思维是结构化数据，语言是惰性渲染** — NPC 内心以概念（ConceptLocalId + PatternSignature）思考，自然语言文本仅在需要外显时生成
2. **概念先于词语** — 概念是语言无关的，词语是概念在特定语言中的投影
3. **文化相对概念空间** — 同一个客观模式，不同文化可能识别为零个、一个或多个独立概念
4. **翻译 = 模式重叠搜索** — 将源概念的模式覆盖投影到目标文化概念空间，找最大重叠
5. **语言 proficiency 从原子日志涌现** — LISTEN/SPEAK/READ/WRITE/EXAMINE 物理原子的执行记录驱动 proficiency 增长
6. **概念习得从暴露累积涌现** — 新概念的暴露累积超过阈值 → 习得，初始低置信
7. **零新门控，零硬编码路径** — 语言学习、概念习得、认知风格演变全部从连续参数的数学交互中涌现
8. **代际衰减是数学函数** — 每种传递路径有独立的保真度衰减率 f(hops)
9. **玩家 = NPC（贯彻到语言层）** — 同一套 Utterance、proficiency、概念空间。UI 语言是独立的渲染目标
10. **概念识别在集成层，不在业务模块** — 各业务模块不知道"概念"的存在。它们只产出客观数据

## 四、与其他模块的关系

### 消费本模块的模块
- **语言表达**: TextGenerator 消费 Utterance，渲染为自然语言文本、语音输出、书籍文字
- **NPC 活人感**: NpcData 消费 CompactConceptEncoding 写入 EventMemory；AgentSnapshot 消费语言/文字 proficiency
- **认知与智慧系统**: 消费推理系统（deductive_chain / creative_leap / wisdom_effective）；概念空间版本检测
- **NPC 行动涌现**: AgentSnapshot 包含语言 proficiency；原子日志驱动语言学习
- **信息传播系统**: 消费 CompactConceptEncoding 做结构化失真传播
- **LLM增强层**: 在概念层操作 Utterance（增强/丰富概念），不直接产出文本
- **音频系统**: 消费 UtteranceId → VoicePacket → TTS 播放
- **信仰系统**: 仪式语言翻译管道
- **权力系统**: 政令语言 → legitimacy 影响（语言不通 → 统治合法性下降）
- **历史系统**: PhysicalBook 携带 source_concepts（概念引用）

### 被本模块消费的模块
- **权力系统**: PowerTopologyQuery（用于 compute_pattern_signature）
- **经济系统**: EconomyQuery（同上）
- **NPC 活人感**: EntityIndex::close_relations()（用于社会事件感知路由）
- **技能系统**: linguistics 技能等级（用于语言学习速率调制）
- **天气系统**: WeatherQuery（环境影响认知/语言学习）

## 五、关键概念速览

| 概念 | 一句话定义 | 详细位置 |

| `CompactConceptEncoding([u8;64])` | 压缩概念编码——存储在 EventMemory、PhysicalBook、InformationPayload 中 | 002 |
| `Utterance` | 结构化话语 { concepts, speech_act, language, delivery } | 003 |
| `UtteranceId(u64)` | 瞬时话语的标识——与 ExpressionRef（持久可读物句柄）分离 | 003 |
| `RenderContext` | 渲染上下文：utterance + speaker_culture + listener_culture + proficiency + ui_lang | 003 |
| `effective_concept_space()` | 纯函数——从 birth_culture + residences + acquired_concepts 加权派生混合概念空间 | 002 |
| `translate_concepts()` | 纯函数——源概念 × 目标文化概念空间 × proficiency → 翻译结果 | 003 |
| `classify_pattern()` | 纯函数——PatternSignature × ConceptSpace → Vec<(ConceptLocalId, confidence)> | 002 |
| `deductive_chain()` | 纯函数——事实 + MentalModel × 链式应用 → 演绎结论 | 005 |
| `transmission_fidelity()` | 纯函数——源模型 × 路径 × 代数 × 接收者 → 保真度 | 006 |
| `wisdom_effective()` | 纯函数——从反思/经验内化/认知平衡/年龄派生——不存储 | 005 |
| `knowledge_preservation_impulse()` | 从 wisdom × 年龄 × 有价值知识 × 紧迫感派生——是否写下来 | 006 |
| `concept_space_drift()` | 使用频率 × 文化接触 × 代际 × 概念漂移 | 006 |

## 六、新 crate 架构

```
woworld_types/          ← 纯类型，零依赖
  + PatternSignature, ConceptLocalId, CultureConceptDef
  + CompactConceptEncoding, Utterance, UtteranceId
  + RenderContext, ResidenceRecord
  + ConceptFact, MentalModelRef, DeductiveConclusion
  + EntityIndex::close_relations()  — 新增方法

woworld_core/           ← 纯数学函数，仅依赖 woworld_types
  + compute_pattern_signature()
  + classify_pattern()
  + pattern_overlap()
  + deductive_chain()
  + transmission_fidelity()
  + format_number()

woworld_lang/           ← 依赖 woworld_types + TOML，不依赖任何业务 crate
  + TextGenerator (三模式: render_utterance / generate / render_thought)
  + translate_concepts()
  + effective_concept_space()
  + compute_language_exposure()
  + concept_space_drift()
  + FragmentLibrary, MorphologyRegistry
```

**依赖方向**：`types ← core ← lang`，业务模块 `→ types`，集成层 `→ 全部`。零循环。

## 八、与已有文档的关系

本模块**不取代**任何已有模块。本模块是**新增地基层**——它填补了语言表达模块（文本管道）和 NPC 认知模块（思维系统）之间的**概念表示层**空白。

已有模块的修改仅限于：
- 语言表达模块：TextGenerator 新增概念渲染入口
- NPC 模块：NpcData/EventMemory/AgentSnapshot 新增字段
- 历史模块：PhysicalBook 新增概念引用字段
- 技能系统：linguistics 技能与 language proficiency 的桥接说明
