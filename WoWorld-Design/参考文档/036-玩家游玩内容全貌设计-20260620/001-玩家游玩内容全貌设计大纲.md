# 玩家游玩内容全貌设计——大纲

> **创建日期**: 2026-06-20
> **状态**: 大纲 — 阶段一输出
> **范围**: WoWorld 玩家实际游玩内容的全貌设计——涵盖进入世界到死亡继承的完整玩家旅程

---

## 〇、总体结论

**不需要"玩家旅程系统"独立模块。** 玩家游玩内容从已有 24 个模块的交汇中涌现。本次设计只做：
1. 确认各模块交汇点已覆盖全部玩家游玩场景
2. 填补识别出的真实缺口（小精灵系统、方阵统计模型、复合原子注册、接口补充）
3. 输出参考文档记录全貌分析

---

## 第一编：进入世界

### 一、两种进入模式
- 原住民：接管既有 NPC 的人生 → `Change/hand/010-玩家进入游戏.md`
- 穿越者：日式异世界开局 → 自选参数 → 随机出生点

### 二、角色创建（详见 `开发阶段/生命/012-玩家角色管理与继承.md`）
- 身份层：名字、种族、性别、出生地
- 身体层：PhysicalAttributes（6维）、PhysicalAppearance（8维面部+体型+肤色+发色+发型+声线）
- 心智层：MentalAttributes（5维）、BigFive（5维，创建后锁死）、CognitiveStyle（4维）、AestheticTaste、AttractionTemplate
- 技能层：任意 SkillId 初始 xp/level/innate_aptitude
- 魔法层：十元素亲和、魔力强度/控制/抗性/恢复速度、初始已知法术
- 语言层：第一语言 + 额外语言及其 proficiency
- 物品层：初始衣服、初始货币
- 人格锁定：创建后和 NPC 一样随经历微变——仅极高冲击力创伤(>0.9)或长期环境冲突(>100日)时Δt≤0.05

### 三、第一分钟体验
- 穿越者：随机野外 → 第一个交互可能是树/兔子/石头 → 最近聚落可能要走两天
- 原住民：继承既有生活的瞬间 → NPC 会自然跟你说话
- 小精灵可选辅助（详见 `开发阶段/小精灵系统/README.md`）

### 四、语言障碍
- 玩家 UI 语言独立于角色语言 → 概念与语言地基/011
- 角色语言 proficiency 决定"能听懂/说多少"
- 跨文化手势误解 → 文化/003 GestureCultureMapping

## 第二编：社会交互

### 五、与人打交道——核心体验
- NPC 有 2000 条记忆、情绪引擎（33 种复合情绪）、BigFive、MentalModel、MindAttribution、SelfNarrative
- 没有"任务看板"——对话本身就是内容

### 六、对话发起（详见 `开发阶段/语言表达/005-对话系统-核心机制.md`）
- 方式一：屏幕中心指针 + 按键 → 预设选项（滚轮选择）+ 自由输入（键盘/语音）+ 大日志引用 + 托管
- 方式二：按住说话键 + 滚轮控制声量 → 语音转游戏内可理解内容 → 广播 → NPC 自主决定回应
- 方式三：托管模式下角色按自己意愿发起对话

### 七、对话内容（详见 `开发阶段/语言表达/`）
- 210 个 Phatic 片段（应酬交流——开场/间歇/告别/反应/闲聊）
- 三种话题来源：预设（NPC 记忆/个性/关系自动生成）、大日志引用、自由文本
- 四种发言权模式：自由交谈/轮流发言/演讲/仪式
- 多参与者容器（2→1000+人）
- SocialField 群体动力学（情绪惯性×规模、极化、从众压力）
- 打断与恢复（战斗/闯入/环境事件/走远/LLM超时）
- 对话后记忆消化（PostConversationDigestion → EventMemory）

### 八、社交持续性
- 关系数据：affection（短期波动）、trust（长期积累）、familiarity（不衰减）、attraction
- NPC 的态度由人格×情绪×关系×文化规范×场合×玩家名声 综合涌现
- 撒谎与识破：从 MentalModel + 记忆比对 + 非语言信号 + 多来源印证中自然涌现——不需要"谎言系统"

### 九、伙伴/团体（详见 `开发阶段/权力系统/`）
- 不存在"队伍系统"——伙伴是独立的人
- 17 个普适权力原子覆盖从亲子到帝国的全部权力关系
- 玩家可以是小组首脑、可以是普通成员、可以不属于任何组织
- 同伴 AI 完全自主——不因"加入玩家队伍"而变成任由摆布

### 十、名声（详见 `参考文档/035-法律涌现与执法机制分析-20260620/` §3.4）
- 分散式——存在于每个 NPC 的 EventMemory + trust/fear 数据中
- 不聚合为"声望分数"——玩家像现实中一样感受
- 不同文化对同一行为有不同评价——从 CultureCoreParams 自然涌现

## 第三编：法律与秩序

### 十一、法律涌现（详见 `参考文档/035-法律涌现与执法机制分析-20260620/`）
- 不需要独立法律模块
- 完整链路：PrescribeRule → Duty → 目击 → 记忆 → 推理 → 告发 → Adjudicate → 制裁塌缩链 → 惩罚
- 17 个权力原子覆盖从亲子到帝国刑法的全部尺度
- 程序正义 = 元规则（约束 holder 的 PrescribeRule）
- 记忆衰减 = 自然追诉时效（但严重犯罪因 important_event_ids 永久可追诉）
- 识别出 10 个接口缺口并给出修复方案

## 第四编：谋生与经济

### 十二、全部谋生路径
- 铁匠/工匠：采矿配方表 → 熔炼 → 锻造（制造/006）
- 商人：跨区价差（经济系统）+ 货运（载具系统）+ 信息不对称
- 政客：权力系统 17 原子
- 雇佣兵/冒险者：战斗 + 分散式名声
- 农夫/牧民：植物/009 §六 农业 + 生命/006 灵兽
- 学者/抄写员：语言表达 + 概念与语言地基 + 技能系统人文学科
- 全部路径由已有系统覆盖——不需要新模块

### 十三、制造哲学
- 玩家是策略者，角色是执行者——和战斗同一哲学
- 决策点穿插在 AI 自动执行的物理原子序列中（淬火时机/合金比例/加入顺序/火候）
- 质量从技能精度 noise × MaterialProperties 中涌现
- 制造没有进度条——玩家看角色干活，想干预就干预
- 生活制造走配方表（如木筏 ASSEMBLE），不走物理逐帧模拟——降低开发负担
- 技能成长可视化：Lv1→Lv60 锤声从杂乱→均匀→有"签名"节奏

### 十四、农业与养殖
- 不需要独立系统——已有模块延伸
- 开垦：TILL/PLANT/HARVEST 复合原子（注册于 08/003）
- 作物生长：植物/009 §六 growth_per_day 公式 × 气候匹配检查
- 牲畜获得：经济购买 / Combat Apprehend（非致命捕获）
- 日常打理：FEED/MILK/SHEAR 复合原子 + AI 自动（小型手动，大型周期性统计）
- 动物繁衍：生命模块繁衍系统 + traits 遗传变异

### 十五、家
- 不需要"家系统"——四个模块交汇涌现：
- 物理空间：建筑 Blueprint TOML + ConstructionScheduler + 家具容器（L1）
- 存储：物品系统五层容器体系（随身→家具→仓储→移动→城镇）
- 财产权：权力系统 possession 边 + Territory + Duty 保护
- 社会节点：文化系统 openness_to_outsiders + NPC 邻里决策
- 离开期间：邻居可能照看/偷盗/占用——全是 NPC 决策涌现
- 房子可能自然损耗/被破坏/被风吹坏——同物理规律

## 第五编：旅行与探索

### 十六、旅行
- 不跳过时间——旅行本身就是内容
- 路上遇到的都是世界上本来就有的动态实体（商人/雇佣兵/猎人/怪物/失踪小孩）
- 时间尺度：中世纪两城之间真实距离
- 载具：火车（蒸汽时代速度）、帆船（比游泳快）
- 长途时玩家可切到另一个可控角色活动

### 十七、探索的回报
- 稀有自然配置（世界树、巨树、魔法林地、隔离文化）
- 独一份的历史残留（老树 AetherImprint）
- 隔离演化 500 年的文化孤本
- 信息跨空间搬运——玩家是唯一不受 GOAP 空间约束的实体
- 命名权——未名之物被玩家命名后进入世界语言层
- 纯美学瞬间——天气×渲染×生态×季节的不可复现输出

### 十八、海洋（详见各已有模块的海上延伸）
- 游泳：SWIM 物理原子 × 体力 × 水温 × Gerstner 波
- 第一艘船：木筏=配方表 ASSEMBLE，独木舟=SCOOP复合原子，帆船=载具+建筑 Blueprint
- 船上生活：钓鱼(FISH) + 集水(SCOOP) + 导航(技能 noise)
- 风暴：天气系统 × 波 × 船体耐久 × NPC 自主恐慌反应
- 岛屿：世界生成 P1 海陆掩码 + 群系参数场 + 文化隔离
- 潜水：憋气(endurance) × 压力(health) × 光源(光衰减深度)
- 沉船：真沉真残骸 + 真物品 + AetherImprint
- 零新模块——全是已有系统在"湿的那部分"上的延伸

## 第六编：战斗与战争

### 十九、战斗体验
- 半自动——像龙之信条/Kenshi：角色自己战斗，玩家可插手干预
- 三层模型（本能→节奏→战略）
- 节奏由交战者自己的攻速决定（双刀快/大剑慢）
- 信息来自：HUD元素、屏幕效果、交战日志、提示音效、角色战斗中的话
- 同伴/敌人状态不在 HUD 上——需自己观察和交流

### 二十、信息不对等与计谋
- 不需要任何新系统
- 欺诈 = NPC 自主决策（GoalDriven × personality × 关系 × 文化 × 记忆）
- 识破 = 一致性检查（MentalModel）+ 非语言信号（微表情/视线）+ 多来源印证
- 玩家也能撒谎——后果从 trust 逐条下降中涌现
- 策略深度 = 信息流动理解 + 玩家信息优势（大日志不衰减 + 自由移动）

### 二十一、大规模战争
- 不需要"战争系统"——是已有系统在"大规模"上的并发运行
- 宣战：统治者 GOAP 决策
- 动员：Compel 原子 + 每个 NPC 自主响应
- 后勤：经济系统 + Extract 原子
- 组织链：Delegate 链自动形成指挥结构
- **唯一新设计**：方阵统计模型 + 结晶/溶解算法（详见 `开发阶段/战斗/015-大规模战争方阵统计模型.md`）
  - L1: 全 Combat AI + 全 3D（<100m, ~20-50人）
  - L2: 简化 AI + LOD1 模型（100-500m, ~200-1000人）
  - L3: 方阵级统计（500m+，每方阵~100-300人聚合，每游戏分钟更新）
  - L4: 纯统计军队级别胜率/伤亡率（>3km）
  - 方阵在 L2/L3 边界结晶化/溶解
- 战后：战场遗址（AetherImprint + 特殊植物生态）、老兵 PTSD、社会重建（婴儿潮/孤儿/文化偏移）

## 第七编：信仰与神明

### 二十二、玩家与信仰
- 可以信神、可以不信、可以亵渎
- 信仰行为：PRAY/MEDITATE/SACRIFICE/CHANT 复合原子——已有 POSTURE/CHANT/GRASP 物理原子的组合
- 托管给角色 AI：角色按人格+信仰内在化程度+情绪自主决定信仰行为
- A 模式：神明是 DivineMind 实体——神的回应来自神的自主决策（非概率骰子）
- 普通人被回应的概率极低——因为神的注意力带宽有限
- 渎神后果：第一层=信徒愤怒，第二层=神自主决定是否降灾

### 二十三、登神
- 登神 = 从 SapientMind 转变为 DivineMind——是生命模块 MindLayer 的边界跨越
- 三条可能路径：死后登神（需要信仰规模+spirit残留）、生前登神（极端魔法操作）、继承神位（旧神死亡→essence散逸→凡人吸收）
- 不是"规划目标"——是极低概率 × 极特殊条件 × 种子允许的涌现结果

## 第八编：时间、死亡与传承

### 二十四、双角色系统
- 两个可控角色，随时切换
- 双角色都睡/不可活动 → 可选时间跳过
- 两人关系：和 NPC 间关系一样涌现——不是玩家手动设定
- 大日志共享——两个角色看到同一本日志
- 可一起活动也可分开——未被操控的角色自己做事

### 二十五、死亡与继承
- 死亡瞬间：角色倒下 → 3-5秒告别间隙 → 死亡信息（角色名/死因/地点/年龄）→ 切换/继承选项
- 尸体：和 NPC 完全一致——可被搬运/掩埋/抢夺/腐烂/读树（AetherImprint 自动生成）
- 继承：血亲（一代以内）→ 大日志上的任何人 → 全新角色
- 大日志可选择：继承/封印（灰显可查）/清除
- NPC 对旧角色死亡的反应：全部涌现——哀悼/传播/遗忘
- NPC 对新角色：不认识——旧人是旧人，新人是新人
- 另一个角色获知死亡：通过共享大日志——条目自动出现。是否悲伤由其人格×情绪×关系涌现

### 二十六、衰老与后代（待正式设计确认）
- 玩家角色和 NPC 一样会老——age_years + max_lifespan
- 老了之后 PhysicalAttributes 是否下降？
- 玩家能不能有孩子？如果能——孩子是 NPC，可成为继任可控角色
- 孩子继承父母的 traits（遗传）+ 种子决定的变异

### 二十七、玩家离线
- 需要正式设计：玩家关闭游戏时世界暂停还是继续？
- 若暂停——NPC 世代更替仅在玩家在线时推进，"40年人生"需要等效游戏时间定义
- 若继续——需时间跳跃摘要推演（离线期间关键事件统计摘要）

## 第九编：知识与记忆

### 二十八、大日志——多镜头架构（详见 `开发阶段/历史/005-大日志系统.md` 扩展）
- 定位：冷功能工具——角色知识库，非任务日志，非成就系统，非叙事工具
- 条目结构：EntryType(13种) + KnowledgeBlock(渐进验证) + related_entries(图谱) + player_tags + player_notes
- 已有能力：全字段搜索、倒排索引、图谱可视化（节点×边×验证状态）、2500条目<3MB
- **新增多镜头**：
  - 镜头一：时间线（密度纵览+间隙分离+双角色混排+life_chapters背景带）
  - 镜头二：地图（密度热力图+点位弹窗+路线连线）
  - 镜头三：人物（NPC交互卡片+排序维度+已故灰色处理）
  - 镜头四：主题聚类（TF-IDF标签簇——不做叙事命名）
  - 镜头五：生命章节（SelfNarrative life_chapters+条目分布+章节过渡事件）
- 核心原则：系统只做组织，玩家自己做叙事
- 零新存储、零新 trait、纯 UI 组织层

### 二十九、SelfNarrative
- 玩家角色有 SelfNarrative——和大日志一样写在大日志里
- 自我标签、生命章节、核心价值观——从行为和记忆中自动归纳
- 50小时后"我做了什么"→ 大日志五镜头回顾 + 世界中的物理痕迹 + NPC 记忆碎片

## 第十编：魔法

### 三十、魔法×物理——统一原子层
- CHG-042：魔法施法从独立系统改为消费物理原子层
- 六个魔法复合原子（CAST/CHANNEL/TRANSMUTE/INFUSE/BIND）全分解为物理原子序列
- 火球烧木门 = IGNITE 原子 × MaterialProperties.flammability——和火把烧木门是同一行代码
- 水系魔法灭火 = WET 原子修改 flammability——和下雨灭火同一计算
- 过程接触（有轨迹，可挡）vs 凭空接触（绕过空间，×1.5-2x魔力）——两种物理传导模式
- 元素浓度环境动态：从群系+天气+历史事件派生，L3周期更新

### 三十一、学法与施法
- 第一法术：法术书/拜师/自学/自然觉醒——都可
- 施法操作：先选对象再选法术，或先选法术再选对象
- 半自动战斗中角色自己按战斗规划施法，玩家可中途介入

## 第十一编：创造

### 三十二、玩家原创内容
- 写书：语言表达 ExpressionRef + TextGenerator 管道 → 留在图书馆/书架 → 五百年后 NPC 可读
- 创造法术：10 元素体系 + 方程式七要素 × 元素亲和矩阵 → 新法术 ID 写入魔法模块
- 命名：未名之物被玩家命名 → 信息传播系统扩散 → 世界语言层永久改变
- 种树：种下去 → 百年后是巨树 → NPC 在其下乘凉
- 留下传统：有意义的行为 × 重复 × 系统检测 → 节日涌现。"没有'创建节日'按钮"
