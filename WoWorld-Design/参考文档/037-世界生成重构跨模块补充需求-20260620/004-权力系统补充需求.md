> **目标模块**: [[../../Happy Game/开发阶段/权力系统/001-权力系统总纲|权力系统]]（CHG-023）
> **紧迫度**: 🔴 高 — P10 权力 Bootstrap 阶段必须消费
> **日期**: 2026-06-20

---

## 〇、总因：为什么权力系统需要补充

> **核心矛盾**: 权力系统的 17 个普适原子和 PowerTopology 是为**运行时**设计的——holder 行使权力，subject 承受权力，legitimacy 随治理效果波动，Polity 每年重算涌现。但权力系统**没有定义"初始权力边从哪来"**。如果世界生成不创建初始边，玩家进入时 PowerTopology 是空的——没有统治者、没有税收关系、没有任何政治实体。
>
> 权力系统的涌入哲学是对的——Polity 不应该被"指定"。但世界生成必须为涌入提供初始种子边。这些边不从涌出产生——它们是从 P5 的社会结构骨架（谁在这个聚落中扮演什么角色）中**推导**的。
>
> **解耦要求**: 权力系统定义 `PowerEdgeTemplate`（"我需要什么参数来创建一条边"），并提供 `bootstrap_power_topology()` 函数（"给我模板+NPC列表，我返回填充好的 PowerTopology"）。世界生成不直接创建 PowerEdge 的内部字段。

---

## 一、新增类型：PowerEdgeTemplate

> **★ 为什么需要模板分离**
>
> P5 产出的权力骨架是"角色级"的——它说"统治者（role_index=0）应该对臣民（role_index=1..N）拥有 Extract(tax) 权力"。但 P5 阶段 NPC 还不存在——只有 RoleTemplate，没有 NpcId。
>
> `PowerEdgeTemplate` 是"角色→角色"的边模式，不引用具体 EntityId。它使用 role_index 作为占位符——P10 实例化时映射到具体 NPC。这种两阶段设计（模板→实例）避免了"先有权力边还是先有 NPC"的鸡生蛋问题。

```rust
pub struct PowerEdgeTemplate {
    pub template_id: PowerTemplateId,
    pub holder_role_index: usize,       // ★ 角色索引——不是 NpcId
    pub subject_role_index: usize,      // ★ 角色索引——不是 NpcId
    pub atom_type: AtomTypeTag,
    pub domain: PowerDomain,
    pub source: PowerSource,
    pub legitimacy_initial: f32,        // ★ < 1.0——留空间给运行时波动
    pub revocable: bool,
    pub succession_rule: SuccessionRule,
    pub cardinality: EdgeCardinality,   // SinglePair / OneToMany / ManyToOne
    pub selection_preference: SelectionPreference,
}
```

## 二、新增接口：bootstrap_power_topology()

> **★ 为什么需要**
>
> 权力系统没有"从零创建初始拓扑"的接口。现有 API 是运行时的：`exercise_power(holder, subject, atom)` 创建/修改边。世界生成需要一次性批量创建 100K-500K 条初始权力边——然后运行 `emerge_polities()` 识别出初始政治实体。
>
> 这个接口的设计约束：必须从 PowerEdgeTemplate（不引用 NpcId）+ 已生成的 NpcData 列表 → 填出完整的 PowerTopology。模板→实例的选择算法由权力系统提供（它知道什么属性对应什么权力角色），但选择的目标（具体哪个 NPC）由世界生成的种子决定。

```rust
pub fn bootstrap_power_topology(
    skeleton: &[PowerEdgeTemplate],
    roles: &[RoleTemplate],
    npcs: &[NpcData],
    families: &[FamilyTree],
    settlements: &[SettlementData],
    culture_params: &CultureCoreParams,
    rng: &mut DeterministicRng,
) -> PowerTopology;
```

## 三、边模板实例化的选择算法

> **★ 为什么需要在权力系统内定义选择算法**
>
> `select_npc_for_role()` 的评分权重（social_standing × 0.4 + age_midpoint × 0.3 + conscientiousness × 0.2 + noise × 0.1）反映了"谁适合当统治者"的涌现逻辑——它不是硬编码"选贵族"或"选最年长者"。BigFive 责任心（conscientiousness）驱动选择 → 高责任心的人更可能被选为统治者。但噪声分量（0.1）保证了不完全确定性——不需要每次都是同一个人。
>
> 这个算法属于权力系统——因为它定义了"什么属性使一个人适合拥有某种权力"。世界生成只调用它。
