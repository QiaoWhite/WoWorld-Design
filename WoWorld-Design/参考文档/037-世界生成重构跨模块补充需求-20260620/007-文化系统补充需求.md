# 007 — 文化系统补充需求

> **目标模块**: [[../../Happy Game/开发阶段/文化系统/001-文化系统总纲|文化系统]]（CHG-024）
> **紧迫度**: 🟡 中 — P5 社会结构推导消费文化参数
> **日期**: 2026-06-20

---

## 〇、总因：为什么文化系统需要补充

> **核心矛盾**: 文化系统定义了 `CultureCoreParams` 的 10 个核心参数——但世界生成 P5 需要将文化参数**转化为具体的生成决策**：`individualism=0.3` 意味着扩展家族还是核心家庭？`power_distance=0.8` 意味着几层统治？答案是：这些映射规则是文化系统对"文化参数如何影响社会结构"的权威知识——不属于世界生成。
>
> 如果世界生成硬编码"individualism < 0.4 → 扩展家族"，那么当文化系统的 individualism 语义发生变化（例如 0→1 的含义反转），世界生成也会断裂。映射规则由文化系统定义和拥有。

---

## 一、individualism → 家庭结构

> **★ 为什么需要**
>
> P5 的 `BuildingSpec.household_size_hint`（Small/Nuclear/Extended/Compound）决定了民居的面积和房间数。这个值从文化 individualism 推导。硬编码映射 → 世界生成对文化系统的内部语义做了假设。将此映射定义为文化系统的导出函数 → 文化系统的参数变化自动传播到建筑规模。

```rust
pub fn family_structure_from_culture(params: &CultureCoreParams) -> FamilyStructureType;
// FamilyStructureType: Nuclear | LooseExtended | Extended | ClanBased
```

---

## 二、power_distance → 统治层级深度

> **★ 为什么需要**
>
> P5 的 PowerSkeleton 需要知道"这个聚落的统治有几层"。`power_distance=0.2` 的平等社会可能只有 1 层（头人→所有人），`power_distance=0.9` 的极权社会可能有 4 层（皇帝→总督→领主→基层官员→臣民）。这个推导不但影响权力边的数量，还影响中间管理角色的 RoleTemplate 数量。映射规则属于文化系统——它定义了 power_distance 和权力层级之间的语义。

```rust
pub fn authority_depth_from_culture(params: &CultureCoreParams, social_complexity: f32) -> u8;
```

---

## 三、文化参数 → 建筑风格映射

> **★ 为什么需要**
>
> P5 为每个 `BuildingSpec` 填入 `culture_style_id`。这个 ID 对应的参数（屋顶样式、墙体材质、装饰级别、颜色调色板）需要从文化参数 + 本地可用材料 + 气候中推导。当前文化系统定义了文化审美参数，但没有将它们映射为建筑模块可消费的 `BuildingStylePreferences`。

```rust
pub fn building_style_from_culture(
    params: &CultureCoreParams,
    local_materials: &[MaterialId],
    climate: &ClimateZone,
) -> BuildingStylePreferences;
```

---

## 四、cultural traditionalism → 职业世系继承强度

> **★ 为什么需要**
>
> P8 的职业世系传播中，"铁匠的儿子有多大概率也是铁匠"由 `occupation_inheritance_strength` 参数决定。这个参数从文化的 `traditionalism` 推导——高传统社会，子承父业是规范；低传统社会，子女自由选择职业。这是文化对生成逻辑的直接调制——映射规则属于文化系统。

```rust
pub fn occupation_inheritance_strength(params: &CultureCoreParams) -> f32;
```
