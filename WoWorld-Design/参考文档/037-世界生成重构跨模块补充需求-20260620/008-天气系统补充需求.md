> **日期**: 2026-06-20

## 〇、总因：为什么天气系统需要补充（几乎不需要）

> **核心判断**: 天气系统是运行时系统——天气每游戏分钟从 WeatherQuery 采样，受 Markov 链 + 季节 + 气候参数场驱动。世界生成**不需要初始化天气**——玩家进入首帧，天气系统自动从 SeasonClock(world_time=0) + 种子开始运转。
>
> **唯一需要确认的事项**: SeasonClock 的初始季节是否从种子派生——即 `world_time = Year(0) + Day(0)` 时，SeasonClock 给出的季节是确定性的（种子→Day 0 在一年中的位置）。
>
> 如果 SeasonClock 不从种子派生 → 相同种子的世界可能在春天或冬天开始 → 玩家体验不一致。需要确认。

---

## 一、确认：world_time=0 时的 SeasonClock 行为

> **★ 为什么需要确认**
>
> 相同种子的世界必须在相同的季节开始。如果初始季节是随机的（不受种子控制），同一个种子可以产生春天和冬天两个版本——确定性保证失效。

```rust
// 期望行为: Day 0 在一年中的位置 = f(seed) ——确定性
pub fn season_at(world_time: GameTimestamp, seed: u64) -> Season;
```

---

## 二、确认：AgentSnapshot 的环境字段在首帧可用

> **★ 为什么需要确认**
>
> `AgentSnapshot.wetness` 和 `AgentSnapshot.temperature_exposure` 从 `WeatherQuery::sample()` 派生。如果 WeatherQuery 在世界生成阶段不可调用（因为 weather 是运行时系统），首帧 AgentSnapshot 的环境字段将为空→NPC 无法正确评估 wetness/temperature 相关的物理原子。
>
> **预期**: 首帧 WeatherQuery 正常工作——它依赖的群系参数场（来自世界生成 P2，存储在 LMDB 中）在首帧已可用。不需要世界生成特殊处理。

---

## 三、依赖注入 trait 实现清单（★ 新增）

> **★ 为什么需要**
>
> 天气系统通过 5 个 trait 消费世界生成数据（CHG-016 契约）。这些 trait 定义在 `woworld_core`，世界生成实现，天气系统通过 trait 对象消费。我们的设计未明确列出这些 trait 及其实现位置。

| Trait | 消费方 | 世界生成实现点 | 实现方式 |
|-------|--------|-------------|---------|
| `ElevationQuery` | 天气温度递减率 | P2 后 | 查询已产出的 `TerrainBase.elevation` |
| `ClimateParamsQuery` | 天气 Markov 链基线 | P2 后 | 查询 `TerrainBase.climate` |
| `BiomeMicroclimateQuery` | 局部天气修正 | P2 后 | 查询群系参数场 + 地表 material_id |
| `OceanCurrentQuery` | 海洋天气影响 | P1 后 | 查询 `OceanSystem.currents` |
| `WorldBoundaryQuery` | 边界风暴带 | P10 后 | 查询 `WorldBoundary` |

**不需要新数据**——5 个 trait 的 impl 只是对已有世界生成产出的查询封装。世界生成 P13 阶段注册这些 impl 到类型系统。

## 四、SurfaceMaterial 查询澄清（★ 新增 — 解决缺口 1）

> **★ 为什么需要澄清**
>
> 音频系统（CHG-030）要求世界生成提供 `SurfaceMaterial`（21 变体枚举）。但这不是一个新数据层——它是从密度场的最顶层体素 `material_id` 查表派生。
>
> `MATERIAL_TO_SURFACE: [SurfaceMaterial; 256]` 常量表（~512B），在 `woworld_core` 中定义:
> - material_id=2(草皮) → Grass
> - material_id=3(沙) → Sand
> - material_id=6(石头) → Stone
> - material_id=11(雪) → Snow
> - material_id=30(水) → Water
> - ...
>
> 世界生成 P2 产出的体素密度场已包含 material_id——`SurfaceMaterial` 只是对已有数据的查表视图。**不需要新增任何世界生成产出。**
