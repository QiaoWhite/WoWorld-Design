> **日期**: 2026-06-20

## 〇、总因：为什么信仰系统需要补充

> **核心矛盾**: 信仰系统（CHG-025）定义了实践优先模型、10 参数 FaithTheology、30 方法 FaithQuery，以及运行时的 NPC→NPC 接触传染渠道。但信仰系统**没有定义"NPC 的初始信仰从何而来"**——信仰的运行时传播需要初始条件。
>
> 如果世界生成不分配初始信仰 → 所有 NPC 都是无神论者 → 信仰相关的 GOAP 行动（PRAY, SACRIFICE, PILGRIMAGE）没有人执行 → 宗教建筑无人使用 → 信仰系统在运行时永远无法启动。
>
> **解耦要求**: 信仰系统定义 `derive_individual_variant()`（从主流信仰+个体 openness 派生个体变体）和 `FaithProfile` 类型。世界生成 P8 在创建 NPC 时调用这些函数分配初始信仰。运行时信仰系统接手——NPC 间接触传染自然演变。

---

## 一、FaithTheology::derive_individual_variant()

> **★ 为什么需要**
>
> 一个聚落的主导信仰（如"太阳神教"）不是所有居民以相同强度、相同神学解释信——每个 NPC 基于其 BigFive 开放性和责任心有不同的神学偏差。`derive_individual_variant()` 从主流信仰 + 个体差异 → 个人的 FaithTheology 变体。这防止了"全聚落信仰完全相同"的硬编码均匀性——信仰差异从人格参数涌现。
>
> 不在信仰系统中定义此函数 → 世界生成只能硬编码"所有人信同样的神学参数"。

```rust
impl FaithTheology {
    pub fn derive_individual_variant(&self, openness: f32, rng: &mut DeterministicRng) -> FaithTheology;
}
```

## 二、FaithProfile 类型与初始化

> **★ 为什么需要**
>
> NPC 的宗教信仰状态需要一个类型承载——当前信仰系统文档没有 `FaithProfile`。它需要包含：
> - 神学参数（从主流信仰派生或从父母继承）
> - 虔诚度（0-1）——从 BigFive + 文化传统主义推导
> - 实践频率——虔诚度 → 每月礼拜次数
> - 宗教角色——祭司/僧侣/萨满从 P5 RoleTemplate 分配
>
> 虔诚度不是随机数——它从人格推导：高传统主义 + 高责任心 → 高虔诚度；高开放性 → 可能低虔诚度或异端倾向。

```rust
pub struct FaithProfile {
    pub theology: FaithTheology,
    pub religiosity: f32,                            // 0-1
    pub practice_frequency: u8,                     // 次/月
    pub religious_role: Option<ReligiousRole>,
    pub conversion_history: Vec<ConversionRecord>,  // 运行时填充
}

pub fn initialize_npc_faith(
    settlement_faith: &FaithTheology,
    parents_faith: Option<&FaithTheology>,
    big_five: &BigFive,
    culture_traditionalism: f32,
    rng: &mut DeterministicRng,
) -> FaithProfile;
```

## 三、聚落级信仰构成

> **★ 为什么需要**
>
> 一个聚落可能不止一种信仰。文化 tolerance 参数决定少数派信仰的存在。高 tolerance→多种信仰共存（≥ 3 种），低 tolerance→主导信仰几乎覆盖全部居民（>90%）。这从文化系统的 uncertainty_avoidance(逆) 推导——不是硬编码。

```rust
pub struct SettlementFaithComposition {
    pub dominant: FaithTheology,
    pub minorities: Vec<(FaithTheology, f32)>,  // (信仰, 信众比例)
    pub tolerance: f32,                          // 0-1
}
```
