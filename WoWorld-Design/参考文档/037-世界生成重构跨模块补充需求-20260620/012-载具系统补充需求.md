## 〇、总因：为什么载具系统需要补充

> **核心矛盾**: 载具系统定义 VehicleDef 从 VehicleArchetype(TOML) × Culture × Technology × Biome 涌现，VehicleState 是运行时实体。但游牧民族的马车/篷车是其生活方式的物理基础——没有载具，游牧聚落无法迁徙。这些初始载具不能等运行时 NPC 建造——世界生成时需要预置。
>
> **需要补充的能力**: 世界生成→载具系统的初始化调用。游牧聚落生成初始 VehicleState（马车/篷车/驮兽），固定聚落可选择性生成渔船/渡船。

## 一、新增接口：initialize_nomadic_vehicles()

```rust
/// 定义在 woworld_vehicle crate 中
/// 世界生成 P5 调用——为游牧聚落生成初始载具
pub fn initialize_nomadic_vehicles(
    nomadic_group: &NomadicStructure,
    culture: &CultureCoreParams,
    tech: &SettlementTechState,
    biome: &BiomeParamField,
    rng: &mut DeterministicRng,
) -> Vec<VehicleState> {
    // 1. 从 VehicleArchetype 中筛选 domain=Land 的原型
    // 2. 适用文化参数→变形 VehicleDef
    // 3. 按氏族人口推算所需载具数:
    //    - 每 5-10 人 1 辆运输马车
    //    - 氏族首领 1 辆篷车
    //    - 每 2-3 个家庭 1 辆公用货车
    // 4. 载具初始位置 = 当前季节营地坐标 + 噪声排列
    // 5. 载具所有权 = 氏族集体或氏族首领 NpcId (P8 填充)
}

/// 载具初始规格
pub struct NomadicVehicleSpec {
    pub archetype_id: VehicleArchetypeId,  // 如 "core.land.nomad_wagon"
    pub count: u32,
    pub owner_type: VehicleOwnerType,
}

pub enum VehicleOwnerType {
    ClanCollective,     // 氏族集体所有
    ClanLeader,         // 首领个人
    Family(u32),        // 特定家庭索引
}
```

## 二、游牧载具的 VehicleArchetype 示例

```toml
# archetypes/land/nomad_wagon.toml

[archetype]
id = "core.land.nomad_wagon"
name_key = "vehicle.archetype.nomad_wagon"
domain = "Land"

[size_class]
base = "Medium"

[capacity]
cargo_kg_range = [500, 3000]
passengers_range = [2, 8]
# 篷车也是居所——有睡眠空间

[propulsion]
allowed_kinds = ["AnimalDrawn"]
# 马/牛/骆驼拉力——坐骑系统管辖
# 载具定义拉力需求(kg)，坐骑提供拉力
draft_force_required_kg = [200, 800]

[living_quarters]
is_mobile_home = true
sleeping_capacity = [2, 6]
storage_capacity_kg = [200, 1000]
```

## 三、固定聚落的初始载具（可选）

非游牧聚落也可以预置少量初始载具：

| 聚落类型 | 初始载具 | 用途 |
|---------|---------|------|
| 港口城市 | 渔船 ×3-10, 商船 ×1-3 | 渔业+贸易 |
| 沿海村庄 | 渔船 ×1-5 | 渔业 |
| 河畔聚落 | 渡船 ×1-2 | 渡河 |
| 商路枢纽镇 | 货车 ×2-5 | 本地运输 |

这些是**可选生成**——运行时 NPC 可以建造更多。游牧载具是**必须生成**——游牧生活依赖它们。

---

## 四、载具→物品锚点

游牧 NPC 的 NpcAnchors.home = `NomadicCamp { current_camp }`。载具本身作为 `ItemEntId` 存在于氏族库存中（可打包/运输），同时在世界中作为 VehicleState 存在。两者通过 VehicleId ↔ ItemEntId 双向引用关联。
