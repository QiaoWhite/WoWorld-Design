## 〇、修订总览

| 维度 | v4.0（当前） | v4.7（修订） |
|------|------------|------------|
| **基础着色** | 单 diffuse pass · flat/cel | **Toon 2-tone 硬切 + Fresnel rim light** |
| **材质** | 纯 Albedo 贴图 | 纯 Albedo 贴图——不引入 Roughness/Metallic/Normal |
| **后处理** | 基础后处理+UI | **SSAO + Bloom + AgX 色调映射 + 大气透视** |
| **氛围** | — | **BiomeAtmosphere 17 参数 × 时间 × 天气 × 季节 × SkyEvent** |
| **水下** | — | **深度分层氛围覆盖 + 过渡 lerp** |
| **室内** | — | **光源派生氛围 + 门口过渡** |
| **做旧** | — | **wear shader uniform + 共享脏污贴图 triplanar** |
| **面部** | 512² 图集（未详述） | **图集 + shader 合成 + MultiMesh per-instance data** |
| **伤口** | — | **shader blend——不破坏 MultiMesh 批量** |
| **Godot 渲染预算** | ~8.0ms | **~9.1ms（+1.1ms）——仍在 16.7ms 内** |

## 一、渲染管线全景

```
══════ Godot 渲染管线 ══════

基础着色（一份共享 ShaderMaterial）:
  Toon 2-tone 硬切（Godot DIFFUSE_TOON 内置）
  + Fresnel rim light (Fresnel term · <0.05ms)
  + wear_level uniform (saturation×factor, brightness×factor, triplanar dirt blend)
  + 伤口 shader blend (per-instance data → scar overlay)
  + 面部表情合成 (per-instance brow/mouth/eye → 512² atlas lookup)

全局后处理（Godot WorldEnvironment）:
  SSAO                      ~0.3ms
  Bloom                     ~0.1ms
  AgX 色调映射              零开销（Godot 4.7 内置——已为 HDR 重写）
  大气透视（shader）         ~0.1ms

可选:
  体积雾                     ~0.4ms
  体积光（God rays）         ~0.2ms

总计: ~9.1ms（基础） / ~8.7ms（无体积雾） / ~9.5ms（全开）
仍远在 16.7ms 帧预算内
```

## 二、Toon 2-Tone + Fresnel Rim

### 2.1 设计决策——为什么用一份共享 ShaderMaterial

- 所有模型（地形、建筑、植被、NPC、物品）共用**一份** `res://shaders/toon_shared.gdshader`
- 材质差异仅在 albedo 贴图——不需要不同的 shader
- Solo 维护一份 shader 文件——改一处，全场景自动更新

### 2.2 Shader 结构

```glsl
// toon_shared.gdshader
shader_type spatial;
render_mode diffuse_toon;  // Godot 4.x 内置 toon 硬切

// ── 全局参数（Godot 每帧设置一次 global_shader_parameter）──
global uniform vec3 atmo_shadow_tint;      // 暗面色调
global uniform vec3 atmo_ambient_tint;     // 环境光色调
global uniform float atmo_saturation;      // 饱和度乘数
global uniform float atmo_contrast;        // toon 对比度
global uniform vec3 atmo_aerial_tint;      // 大气透视图层
global uniform float atmo_aerial_blur;     // 大气模糊量
global uniform float atmo_rim_strength;    // Fresnel rim 基础强度
global uniform vec3 atmo_rim_color;        // rim 颜色（主光源色/固定暖/固定冷）

// ── per-instance data (MultiMesh·从 Rust 每情绪变化更新) ──
instance uniform vec4 custom_data;  // [skeleton_var, brow+mouth, eye+wound_type, wound_age]

// ── per-material uniform ──
uniform float wear_level;           // 0=新, 1=破败
uniform sampler2D dirt_override;    // 256² 共享万能脏污贴图

void fragment() {
    // Godot toon 自动处理亮/暗硬切
    vec3 shaded = DIFFUSE_TOON_LIGHT;

    // Fresnel rim
    float rim = pow(1.0 - abs(dot(NORMAL, VIEW)), 4.0);
    shaded += rim * atmo_rim_strength * atmo_rim_color;

    // 磨损
    float sat = 1.0 - wear_level * 0.4;
    float bright = 1.0 - wear_level * 0.15;
    ALBEDO = ALBEDO * sat * bright;
    ALBEDO += texture(dirt_override, WORLD_POS * 0.5).rgb * wear_level * 0.6;

    // 伤口（per-instance）
    float wound_type = fract(custom_data.z);
    float wound_age = custom_data.w;
    if (wound_type > 0.0) {
        ALBEDO = blend_scar(ALBEDO, wound_type, wound_age);
    }

    // 面部表情合成（per-instance——仅面部 mesh 调用此路径）
    // brow/mouth/eye 索引从 custom_data 解包 → 查 512² atlas → 合成到 ALBEDO
}
```

### 2.3 为什么弃用 Roughness

Roughness 服务于 PBR 的 GGX 高光——软高光。Toon 2-tone 的阴影是硬切的——高光要么有要么没有。一个 GGX 软高光出现在 cel 渲染里，像水墨画里嵌了一张照片。**它属于另一个美术体系。** 材质区分改用 Fresnel rim 的宽度和强度 + 脏污贴图的 triplanar 投影。

## 三、BiomeAtmosphere —— 氛围参数系统

### 3.1 核心结构

```rust
/// 完整氛围描述——Rust 计算, Godot 消费
/// 每帧计算一次·<0.02ms
pub struct ResolvedAtmosphere {
    // 地面层
    pub shadow_tint: [f32; 3],
    pub ambient_tint: [f32; 3],
    pub saturation_mod: f32,
    pub contrast_mod: f32,
    pub night_sat_r: f32, night_sat_g: f32, night_sat_b: f32,
    pub floor_brightness: f32,

    // 尘土·雾层
    pub dust_color: [f32; 3], pub dust_density: f32, pub dust_height: f32,
    pub fog_color: [f32; 3], pub fog_density_base: f32,
    pub fog_height_floor: f32, pub fog_height_ceiling: f32,
    pub fog_vertical_gradient: f32,

    // 大气透视
    pub aerial_tint: [f32; 3], pub aerial_blur: f32,

    // 天空层
    pub sky_zenith_tint: [f32; 3], pub sky_horizon_tint: [f32; 3],
    pub night_sky_brightness: f32, pub night_sky_tint: [f32; 3],

    // 发光·轮廓
    pub bloom_threshold: f32,
    pub rim_strength: f32, pub rim_color_source: RimColorSource,

    // 水面（远期）
    pub water_clarity: f32, pub water_reflection_strength: f32,
}

pub enum RimColorSource { PrimaryLight, FixedWarm, FixedCool }
```

### 3.2 四层调制

```
ResolvedAtmosphere = biome_base × time_curve(hour) × weather_curve(weather) × season_curve(season)

不是替换——是累乘。每层调制前一层的结果。
```

**biome_base**（biome_atmosphere.toml）——每个群系一组 17 参数基线。边界双线性插值。12 个群系 × 17 参数。

**time_curve**（time_curve.toml）——6 个时间锚点（midnight/dawn/morning/noon/dusk/evening）。锚点间线性插值。控制 sun_color、sun_intensity、ambient_color、ambient_intensity、shadow_strength、saturation_mult、contrast_mult。

**weather_curve**（weather_curve.toml）——天气→调制乘数。晴朗/多云/雨/暴风雨/雾/雪各一行。每行乘数独立于群系和时间。

**season_curve**（season_modulation.toml）——四季→调制乘数。春（微暖+高饱）、夏（基准）、秋（暖金+低饱）、冬（冷白+低饱+可选雪覆盖）。

### 3.3 TOML 示例

```toml
# biome_atmosphere.toml
[biome.temperate_forest]
label = "温带阔叶林"
shadow_tint = [0.12, 0.18, 0.22]
ambient_tint = [0.10, 0.18, 0.10]
fog_color = [0.35, 0.48, 0.55]
fog_density = 0.15
saturation_mod = 1.05
contrast_mod = 0.95
aerial_tint = [0.38, 0.48, 0.62]
aerial_blur = 0.08
sky_zenith_tint = [0.30, 0.42, 0.65]
sky_horizon_tint = [0.48, 0.55, 0.68]
night_sky_brightness = 0.12
floor_brightness = 0.015
bloom_threshold = 0.7
rim_strength = 0.12
rim_color_source = "primary_light"
```

### 3.4 Rust → Godot 传输

```rust
// 每帧一次——不单独传 17 个参数
fn get_atmosphere() -> PackedFloat32Array {
    let atm = resolve_atmosphere(player_pos, time, weather, season, sky_event);
    PackedFloat32Array::from(atm.as_float_array())  // 17 floats, 68 bytes
}
```

```gdscript

# ... 其余参数

```

## 四、水下深度分层

消费已有的 `OceanProvider::wave_height()` 和 `depth_query()`：

```rust
fn resolve_atmosphere(pos: Vec3, ocean: &dyn OceanProvider, time: f64, ...) -> ResolvedAtmosphere {
    let surface = resolve_surface(pos, ...);  // biome × time × weather × season

    let water_height = ocean.wave_height(pos.x, pos.z, time);
    if pos.y >= water_height { return surface; }

    let depth = water_height - pos.y;
    let underwater = match depth {
        d if d < 0.5  => surface.lerp(&underwater_params(0.0), d / 0.5),
        d if d < 200.0 => underwater_params(0.0),  // epipelagic
        d if d < 1000.0 => lerp(epipelagic, mesopelagic, (d-200.0)/800.0),
        d if d < 4000.0 => lerp(mesopelagic, bathypelagic, (d-1000.0)/3000.0),
        _ => underwater_params(3.0),  // abyssopelagic
    };

    surface.lerp(&underwater, transition_t)  // 0.5m 过渡带平滑切换
}
```

水下参数由 `underwater.toml` 提供——4 个深度带，每带独立的 fog_density、saturation_mod、ambient_tint、bloom_threshold。和 BiomeAtmosphere 同结构，只是数据源不同。

## 五、室内氛围

室内氛围不由 TOML 定义——由 Godot 侧活跃光源数据自动派生：

```
Godot 每帧:
  camera_pos → Rust BuildingQuery::building_at(camera_pos) → Option<BuildingId>
  
  if BuildingId 且 camera_pos 在 building.bounds 内:
    active_lights = get_interior_lights(building)  // AreaLight3D + OmniLight3D
    indoor_atm = derive_from_lights(active_lights)  // 光源色温·强度·数量 → ambient_tint, shadow_tint
    atmosphere = lerp(outdoor_atm, indoor_atm, transition_t)
    // transition_t = 门口距离 / 2.0m → 从室外走入的平滑过渡
```

**Rust 仅做几何判定（已有 BuildingQuery）。Godot 做光照派生和过渡。** 室内不需要新 TOML——光源数据已在 Godot 的 AreaLight3D 和 OmniLight3D 节点中。

## 六、Wear 渲染

```rust
// Rust — wear shader uniform 值
// per 建筑: BuildingRuntime.wear_level (每游戏日更新)
// per 物品: ItemState.durability (已有)
// per 装备: 同物品——装备是物品的子类
```

```glsl
// Godot toon shader — 已有 wear_level uniform
float wear = wear_level;  // 0=新, 1=破败
ALBEDO *= (1.0 - wear * 0.4);           // 褪色
ALBEDO *= (1.0 - wear * 0.15);          // 变暗
ALBEDO += dirt_triplanar * wear * 0.6;   // 脏污贴图投影
```

脏污贴图：一张 256² RGBA8 万能做旧 overlay——底部暗绿/灰色渐变（模拟近地苔藓）+ 上半随机暗斑（水渍）。triplanar 世界坐标投影——**不需要朝向感知**，重力方向是全局下方向。

VRAM: 256KB。整个游戏共用一张。

## 七、面部表情

### 7.1 512² 图集 + per-instance data

```
512² RGBA8 图集布局:
  row 0: 8 种眉形 (y=0-32px)
  row 1: 8 种嘴型 (y=33-64px)
  row 2: 4 种眼睛 (y=65-96px)
  row 3: 4 种肤色基底 (y=97-128px)

MultiMesh per-instance custom data (4×f32):
  [0]: skeleton_variant
  [1]: brow(0-7) + mouth(0-7) packed u8
  [2]: eye(0-3) + wound_type(0-15) packed u8
  [3]: wound_normalized_age (0-1)
```

### 7.2 情绪→表情映射

```toml
# face_expression.toml
[expression.neutral]
brow = 3; mouth = 3; eye = 2
blink_interval_ms = 3500

[expression.joy.low]    brow = 2; mouth = 4; eye = 1
[expression.joy.medium] brow = 2; mouth = 5; eye = 2
[expression.joy.high]   brow = 3; mouth = 7; eye = 3
# ... sadness/anger/fear/surprise 同理
```

```rust
fn resolve_face(emotion: &EmotionState, config: &FaceExprConfig) -> FaceExpression {
    let (dominant, intensity) = emotion.dominant();  // 最高强度的情绪
    if intensity < 0.25 { return config.neutral(); }
    let level = intensity_to_level(intensity);  // low/medium/high
    config.lookup(dominant, level)
}
```

**Rust 不知道 brow/mouth/eye 的视觉含义。** 它只查 TOML——产出 3 个 u8 离散索引。Godot shader 读索引 → 查图集 → 合并。零额外纹理。MultiMesh 单次 draw call 渲染所有 LOD 0 NPC 的不同表情。

## 八、伤口渲染

伤口不生成额外纹理。在同一个 toon shader 中 blend：

```glsl
// per-instance: wound_type > 0 → blend scar overlay
if (wound_type > 0.0) {
    vec4 scar = texture(scar_atlas, uv_scar);
    ALBEDO = mix(ALBEDO, scar.rgb, wound_age * 0.3 * wound_severity);
}
```

Rust 物理原子 STRIKE 伤害 → WoundState → per-instance custom data → shader 自动 blend。不需要独立材质实例——和 MultiMesh 完全兼容。

## 九、性能预算（修订）

| 阶段 | v4.0 | v4.7 修订 |
|------|------|----------|
| Toon 2-tone（替换 flat diffuse） | ~0ms 增量 | 替換——成本相近 |
| Fresnel rim | — | +<0.05ms |
| SSAO | — | +~0.3ms |
| Bloom | — | +~0.1ms |
| 大气透视 | — | +~0.1ms |
| AreaLight3D（≤15 盏 LOD 0-1） | — | +<0.15ms |
| 体积雾（可选） | — | +~0.4ms |
| **Godot 渲染总计** | **~8.0ms** | **~9.1ms** |
| + 体积雾 | — | **~9.5ms** |

VRAM 增量：
- 共享脏污贴图（256²）: +256KB
- 面部图集（512²·已有）: ±0
- DrawableTexture2D 纹理: +~6MB（地图 4MB + 告示牌 ≤2MB）
- 全局 shader uniform: ±0

**VRAM 总计: ~4.21GB（+<10MB）——仍在 6GB 预算内。**
