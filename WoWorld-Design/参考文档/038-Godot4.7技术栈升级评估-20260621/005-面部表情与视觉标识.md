## 一、面部表情——图集 + shader 合成

### 1.1 架构

```
Rust 情绪引擎 → EmotionState（已有）
  → face_expression.toml 查表 → FaceExpression { brow: u8, mouth: u8, eye: u8 }
  → per-instance custom data [brow+mouth packed, eye+wound packed]
  → Godot toon shader: 读索引 → 512² 图集采样 → 合成到面部 mesh

零独立纹理。零 DrawableTexture2D。MultiMesh 单次 draw call。
```

### 1.2 512² 图集布局

```
row 0: 8 种眉形 (每格 64×32px)
row 1: 8 种嘴型 (每格 64×32px)
row 2: 4 种眼睛 (每格 128×32px)
row 3: 4 种肤色基底 (可选)
```

### 1.3 情绪→表情映射

```toml
# face_expression.toml
[expression.neutral]
brow = 3; mouth = 3; eye = 2
blink_interval_ms = 3500

[expression.joy.low]    brow = 2; mouth = 4; eye = 1
[expression.joy.medium] brow = 2; mouth = 5; eye = 2
[expression.joy.high]   brow = 3; mouth = 7; eye = 3
```

3 级强度——低/中/高。Rust 产出 `dominant_emotion + intensity_level → TOML 查表`。

### 1.4 中性 + 眨眼

无主导情绪（所有 emotion < 0.25）→ neutral。Godot shader 周期性执行眨眼（100ms 闭眼 → 3500ms 间隔 ±1500ms 方差）。

---

## 二、伤口——shader blend

```
Rust 物理原子 STRIKE 伤害 → WoundState { type, severity, normalized_age }
  → per-instance custom data [wound_type + wound_age packed]
  → Godot toon shader: wound_type > 0 → blend scar overlay
  → MultiMesh 完全兼容——零独立纹理
```

和面部表情共享同一套 per-instance data 管道——Custom data [2] 的 4 bits 用于 wound_type，Custom data [3] 用于 wound_age。

---

## 三、MultiMesh per-instance custom data（4×f32）

```
[0]: skeleton_variant (已有)
[1]: brow(0-7) bits 0-3 | mouth(0-7) bits 4-7
[2]: eye(0-3) bits 0-2 | wound_type(0-15) bits 3-7
[3]: wound_normalized_age (0-1)
```

批量更新: `multimesh.set_buffer()` ——单次 0.1ms 内完成 1000 实例。Godot 侧每游戏日更新一次。

---

## 四、决策要点

| 决策 | 理由 |
|------|------|
| Rust 不产生 brow/mouth/eye ID 映射公式 | TOML 数据驱动——可 Mod。Rust 只查表 |
| 图集 + shader 合成——不生成独例纹理 | MultiMesh 兼容。零 VRAM 增量 |
| 伤口不生成纹理——shader blend | 不破坏 MultiMesh 批量 |
| 3 级强度（不连续混合）| 低多边形 cel/toon——8 种眉形粒度不足以表达 0.6×joy+0.3×sadness 混合 |
