## 一、这是什么模块

**家具与放置物品系统**是[[../../Happy Game/开发阶段/物品系统/001-物品系统总纲|物品系统]]的子模块——处理"物品从背包中被拿出来、放在世界中的某个表面上"的全部语义。

它覆盖的不是"物品是什么"，而是**"物品被摆放在世界中之后的行为"**——从一把椅子到桌上的烛台，从壁炉到雕塑，从地毯到野外神龛。

---

## 二、核心设计决策

### 2.1 统一放置实体连续谱

不区分"建筑组件"和"家具实体"。一切在世界上占据空间、能被感知和交互的人造物，共享同一套空间系统（`EntityIndex` trait）。差异不在类型上，而在属性梯度上：

```
承重墙 → 门/窗 → 固定长椅 → 可移动桌椅 → 装饰花瓶
  structural_role: 1.0 ─────────────────────────→ 0.0
```

### 2.2 一切皆物品，一切皆可交互

世界上每一个被摆放的实体都是 `ItemEntId`（→ `EntityId`）。物理原子层（[[../../Happy Game/开发阶段/NPC活人感模块/08-NPC行动涌现与分类/002-物理原子层定义与签名|物理原子层]]）的 35 个原子——`GRASP`/`SIT`/`BREAK`/`IGNITE`/`PUSH`——统一作用于 EntityId，不区分家具和武器和建筑组件。

### 2.3 参数化家族而非穷举枚举

~9 个家具族 × 平均 3 个 mesh 模板 × 连续参数（文化/材料/品质） × morph target = ~12,000+ 视觉变体。新增文化风格只需调整参数映射表。

### 2.4 双 Pass 生成 + 种子确定性

- **Pass A (P11.5a)**: 附着家具——壁炉、嵌墙架——遍历建筑 Attachment 连接面
- **Pass B (P11.5b)**: 自由家具——消费 `CultureFurnishProfile` + NPC 属性 + 经济水平
- **Pass C (P11.5c)**: 野外放置实体——神龛/摊位/路标
- 所有家具全量预生成，写入 LMDB。渐进是渲染和物理 LOD 的事。

## 三、模块边界

| | 家具系统（Owner） | 建筑模块（配合方） | 魔法系统（消费方） | NPC 行动（消费方） |
|---|---|---|---|---|
| **提供** | PlacedItem 数据模型、放置验证、PlacementStore | `Surface` trait、`Chimney` 组件族 | — | — |
| **消费** | `Surface` trait（woworld_core）、`EntityIndex` trait | — | `MaterialProperties::elemental_affinity` | `EntityIndex::affordances_of()` |
| **不碰** | 建筑结构计算、NPC AI 决策、魔法效果计算 | 物品 ID、物品 provenance | — | — |
