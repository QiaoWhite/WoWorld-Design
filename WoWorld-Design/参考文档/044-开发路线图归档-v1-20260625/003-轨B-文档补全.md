## B.1 致命缺失：UI/UX 系统

**当前状态**：零设计。58 个文件、326 处交叉引用全部假设了一个不存在的 UI 层。

**需覆盖的核心问题**：

| 界面 | 说明 | 关键设计问题 |
|------|------|-------------|
| **HUD** | 常驻抬头显示 | 生命/魔力/ stamina 显示方式、小地图、快捷物品栏、当前任务/Bark |
| **对话界面** | NPC 对话 | 选项轮盘（手柄兼容）/ 数字键选择 / 语音输入选项 / 自由文本输入框。按 `hand/011 玩家进入游戏.md` 裁决 |
| **物品栏与装备** | 物品管理 | 30格基础槽位 + 容器展开、装备槽位（BodyPlan派生）、拖拽/右键操作 |
| **地图与导航** | 世界地图 | 知识迷雾（未探索区域灰色）、已发现标记、快速旅行条件 |
| **蓝图编辑器** | 建筑 DIY | 参考建筑模块 Blueprint TOML 格式，设计可视化编辑界面 |
| **世界日志浏览** | 大日志查阅 | 搜索/过滤/关系图谱可视化/时间线导航。参考历史系统 005-大日志 |

**推荐文档结构**（5篇）：

```
UI与UX系统/
├── README.md                    # 模块定位 + 文档索引 + 关键参数表
├── 001-信息架构与设计哲学.md     # 信息层级、沉浸式原则、手柄+键鼠双输入、无障碍
├── 002-HUD与常驻界面.md         # 状态栏、小地图、快捷栏、Bark气泡、日志提示
├── 003-对话与交互界面.md         # NPC对话、交易、交互轮盘、自由文本输入
├── 004-面板与管理系统.md         # 物品栏、装备、技能、地图、日志、设置
└── 005-跨模块接口与性能预算.md   # UI消费各模块的数据合同、渲染预算（<2ms/帧）
```

**关键设计约束**：
- UI 仅在 Godot 侧（Control 节点），Rust 侧只提供数据
- 手柄 + 键鼠平等支持（用户决策 `hand/011`）
- 沉浸式优先——默认最小 HUD，需要时展开
- 与感官系统一致：UI 不泄露"NPC 不知道的信息"

## B.3 高度缺失：名声系统

**当前状态**：零设计。6 文件 17 次交叉引用。

**设计核心**（从现有模块引用推断）：
- 名声是**分散式 NPC 记忆 × 信息传播 × 共识**的涌现结果，不是全局变量
- 每个 NPC 对同一个玩家的"名声认知"可能不同（取决于该 NPC 听到了什么）
- 名声维度：英勇/残忍/诚实/狡诈/慷慨/吝啬/忠诚/背叛…（多维，非单一善恶值）
- 头衔：从名声+权力+文化中涌现（"XXX的征服者"/"YYY的铁匠大师"）

**推荐文档结构**（3篇）：

```
名声系统/
├── README.md
├── 001-名声系统总纲.md           # 涌现式名声模型、多维声誉向量、个体认知 vs 公共共识
├── 002-名声传播与衰减.md         # 信息传播通道（口头/书面/公告）、失真、遗忘曲线
└── 003-头衔荣誉与社会反馈.md     # 头衔涌现规则、NPC对玩家名声的行为响应
```

## B.4 高度缺失：法律与秩序

**当前状态**：零独立设计。但参考文档 035b（27.7KB）已做深入分析——**结论是不需要独立法律模块**。

**035b 分析核心发现**：
- 17 个权力原子已覆盖完整执法链：`PrescribeRule → Duty → Witness → Memory → Reason → Report → Adjudicate → Sanction`
- 每个环节由不同模块负责：权力（规范/制裁）、文化（规范内容）、感官（目击）、NPC认知（记忆/推理）、语言表达（举报）
- 仅需在 10 个模块中修补接口缺口，不需要新建模块

**需修补的接口**（4篇，轻量）：

```
法律与秩序（接口修补）/
├── README.md
├── 001-法律涌现分析.md           # 核心论点：为何不需要独立模块（基于035b）
├── 002-权力模块接口修补.md       # Duty/Sanction/Adjudicate 法律接口
├── 003-感官NPC语言模块修补.md    # 目击→记忆→推理→举报 数据链路
└── 004-文化与信仰模块修补.md     # 规范内容来源、禁忌、宗教法
```

**融合**：`想法/WoW World/道德/道德.md`（146 bytes）——"道德描述一个人有多大概率认可他人执行甚至亲自执行某个行为。道德分类。道德共识：什么该做、什么不该做"——融入文化系统的 CommunicationNorms + 信仰系统的 FaithTheology。

## 补全工作流

每启动一个新模块：

1. **/grill-me 访谈** — 与用户逐项走通设计树
2. **创建模块目录** — `/woworldidea-design gate` 脚手架
3. **编写开发规格** — 所有子文档（目标≥60行/篇）
4. **跨模块审计** — 检查与已有模块的一致性
5. **CHG 文档** — 在 `WoWorld-Design/Change/` 创建变更记录
6. **保鲜** — 更新接头总览 + CLAUDE-INTERFACES.md
