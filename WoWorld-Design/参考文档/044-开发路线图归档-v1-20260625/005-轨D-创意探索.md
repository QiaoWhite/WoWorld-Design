## D.2 冒险小队

**来源**：`想法/WoW World/冒险小队.md`（347 bytes）

**核心概念**：
- 龙之信条式同伴系统：玩家 + 若干 NPC 组成冒险队伍
- 队伍成员是世界中真实存在的 NPC——有自己的记忆、性格、动机
- LLM 驱动的队伍对话：旅行时闲聊、战斗中喊话、酒馆里复盘
- Kenshi 式自主行为：同伴可以离队办事、自己买东西、与他人社交
- 队伍关系动态：信任/摩擦/依赖 从实际互动中涌现

**与现有模块的交汇**：
- NPC 活人感模块（同伴 = 标准 NPC，无特殊代码路径）
- 语言表达系统（队伍对话走标准 Conversation 管道）
- 战斗系统（队伍战斗 = 多个 NPC 协同战斗 AI）
- 名声系统（冒险者队伍的公共名声）

**何时展开**：UI/UX + 玩家系统完成后。需要对话界面和队伍管理面板。

**潜力评估**：⭐⭐⭐ 核心幻想。"和一群有血有肉的同伴一起冒险"是 WoWorld 区别于 Minecraft/Terraria 的关键体验。

## D.3 Storyteller / 世界机

**来源**：`想法/WoW World/世界机.md`（437 bytes）

**核心概念**：
- 三种神格（类似 RimWorld 的 Storyteller）：
  - 仁慈者：倾向于有趣的事件、温和的挑战、喜剧性转折
  - 好战者：倾向于冲突、危机、生存压力
  - 冷漠者：世界自行运转，事件自然涌现，不干预
- 远近状态机：全局趋势监控（战争/和平/繁荣/衰败）→ 调整事件概率权重
- 动态难度：根据玩家当前实力和资源调整挑战强度
- 不是"任务生成器"——它决定的是"世界往哪个方向倾斜"，不直接生成具体事件

**与现有模块的交汇**：
- 历史系统（事件因果链）
- 权力系统（战争/和平由权力拓扑的紧张度涌现）
- 经济系统（繁荣/衰退由订单簿和供给需求涌现）
- 天气系统（气象异常——风暴/干旱/丰收——本身就是"世界机"的自然表达）

**何时展开**：NPC 系统 + 社会系统基本可运行后。Storyteller 是元层协调器——在底层涌现出足够多的数据之后才有意义。

**潜力评估**：⭐⭐⭐ 元叙事。World 的"灵魂"——决定了玩家感受到的世界是温暖的、残酷的、还是冷漠的。

## D.4 魔法场论

**来源**：`WoWorld-Design/Change/hand/012 魔法场论.md`（知乎文章链接）

**核心概念**：
- 将魔法建模为**张量场**（类似电磁场的麦克斯韦方程）
- 魔法粒子（mana quanta）在空间中流动、聚集、消散
- 不同元素对应不同的场的张量结构
- 施法 = 在局部扭曲魔法场，产生可观测效应
- 魔法场的"天气"——某些区域天然富集某些元素，随时间变化

**与现有魔法系统的关系**：
- 现有魔法系统（十元素/七系/五施法媒介）是**实践层**——描述"怎么施法"
- 魔法场论是**物理层**——描述"魔法到底是什么"
- 场论可以解释：为什么某些地点魔力充沛（元素节点）、为什么魔法有射程限制（场衰减）、为什么死灵魔法在战场遗址更强（残留场印记）

**何时展开**：魔法系统性能预算建立后。场论会显著影响魔力体系的数值模型（从"魔力池"变成"场强度×施法效率"），需要先验证基础性能可行。

**潜力评估**：⭐⭐ 魔法系统 v2.0 的理论地基。非常优雅——但数学复杂度和计算开销需要仔细评估。当前是灵感，非规范。

## D.5 环境心理学

**来源**：`想法/WoW World/环境.md`（325 bytes）

**核心概念**：
- 环境不仅提供物理约束（温度/光照/声音），还影响心理状态
- 狭窄空间 → 幽闭倾向（情绪修正）、开阔视野 → 心旷神怡
- 自然景观 → 敬畏感（awe）、废墟/战场 → 忧郁/恐惧
- 熟悉的环境 → 安全感、陌生环境 → 警觉/焦虑
- 文化差异：沙漠民族对沙漠的情感不同于外来者

**与现有模块的交汇**：
- 感官与知觉系统（视觉天际线/听觉回响/嗅觉记忆触发）
- NPC 认知系统（CognitiveTide 潮汐 × 环境心理修正）
- 大气与氛围系统（BiomeAtmosphere 不仅影响渲染，还影响情绪基线）
- 审美与艺术系统（AestheticSignal 的 judge() 可消费环境心理参数）

**何时展开**：感官系统 + 认知系统运行后。环境心理是锦上添花——世界已经运转，加这一层让 NPC 的行为更细腻。

**潜力评估**：⭐⭐ 感官系统的自然扩展。不是新模块——更像是现有系统的参数注入。

## 创意探索工作流

与轨 B（文档补全）不同，轨 D 不追求结构化产出：

1. **灵感捕捉**：看到有趣的概念/论文/游戏机制 → 写入 `想法/` 或 `参考文档/`
2. **快速原型**：用伪代码/草图/数学公式探索可行性
3. **用户讨论**：/grill-me 访谈走通设计树
4. **决定**：值得做 → 排入轨 B 队列；不值得 → 归档到参考文档；太早 → 保留在想法/等待时机
