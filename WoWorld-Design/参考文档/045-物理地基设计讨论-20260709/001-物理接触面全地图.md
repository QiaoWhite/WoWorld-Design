## 一、盘点总纲

WoWorld 全项目的"物理"高度一致地表达为 **"确定性数据源 + 涌现属性 + 启发式乘数"**——凡能这么表达的就不写硬核刚体仿真。真正的连续物理求解只集中在两处：① 音频传播引擎（波的数学，全项目最完整）；② 模型动作与物理系统的 COM 质点物理 + 骨架松弛。**全项目唯一带 `velocity` 的物理体是 `DroppedItemRecord`（家具 014）。** 无任何 Rapier/Avian/Jolt 依赖。

代码现状：Rust 侧空间查询 trait 齐全（TerrainQuery/EntityIndex/SpatialEventBus/VisibilityQuery）+ 地形高度/法线/AABB 索引可用，但**动力学（重力/碰撞/刚体/击退/下落）几乎全空**——NPC 靠每帧 `height_at` 吸附移动，玩家靠 Godot GDScript 手写重力，`Velocity` 组件是死代码。

## 二、按"物理种类"分组的接触面

状态图例：✅ 有明确规格 · 🟡 草稿/浅 · ⚠️ 零散提及或悬空 · ❌ 完全空白

### A. 单体运动学（质点/COM 积分）
| 接触面 | 状态 | 技术方案 / 出处 |
|--------|------|----------------|
| 飞行/坠落/击飞 | ✅ | Rust COM 抛物体积分 + 重力（空间查询与物理 §2） |
| 水下浮力 | ✅ | 同一 COM 积分 + 浮力项 + `medium_at`（§4.1） |
| 动物飞行路径 | ✅ | bezier+perlin，**明确"不模拟空气动力学"**（生命 005 §4.1） |
| 掉落物落点/滚动/静止 | ⚠️ | 仅 `DroppedItemRecord{pos,velocity}` 字段，**无积分/静止算法**（家具 014） |
| 投射物弹道（火球/飞刀/弓箭） | ❌ | 魔法只有"过程接触有路径"概念；战斗/物品明标"待补充" |

### B. 碰撞检测（相交查询，不含动量）
| 接触面 | 状态 | 技术方案 |
|--------|------|---------|
| 武器命中 | ✅ 详尽 | capsule 扫掠相交（物理原子层 007，~5ns/test） |
| 生物避让 | ✅ 轻量 | `EntityIndex::entities_in_aabb` + 迭代分离 |
| 家具放置/占位 | ✅ 详尽 | 纯函数七步（足迹 AABB + 净空 + A* 可达）+ 表面链射线（家具 013/014） |
| 格挡 | ✅ | 攻防武器 capsule 几何求交判成败 |
| 建筑阻挡通行 | ⚠️ | 仅提供 AABB/网格/Portal 尺寸；解算外包给本模块 |

### C. 碰撞响应 / 动量传递（真正的求解器领域）
| 接触面 | 状态 | 备注 |
|--------|------|------|
| 击退力度（impulse→距离） | ❌ | `HitResult.impulse` 有，转位移无公式 |
| 多体连锁碰撞 | ❌ | 击飞者撞第三者——无 |
| 盾牌反弹力传导 | ❌ | 格挡只是成功率检定 |
| 载具-载具动量碰撞 | ❌ | 只有 steering 避让 |
| 生物基于质量的碰撞/踩踏 | ❌ | 无 mass→动量链路 |
| 爆炸推开物体 | ❌ | AOE 只做 EntityIndex 粗查，无冲击波 |

### D. 移动参考系（非惯性系）
| 接触面 | 状态 | 技术方案 |
|--------|------|---------|
| 骑乘 | ✅ | 骨盆 world transform = 马鞍 world transform（空间查询 §4.2） |
| 载具甲板行走 | 🟡 | `walkable_area: Option<VehicleWalkableArea>` + `world_transform: Mat4`（载具 001，草稿 v0.1） |
| 扒在大型生物身上 | ❌ | 完全空白 |

### E. 结构/约束物理（准静态）
| 接触面 | 状态 | 技术方案 |
|--------|------|---------|
| 建筑承重 | ✅ | 载荷图迭代求解（非 FEM），`ConnectionFace.load_bearing`（建筑 004） |
| 建筑倒塌 | ✅ | 状态切换 `BuildingState::Ruins` + decal/fallback mesh，**非碎块物理** |
| 物品堆叠稳定 | ⚠️ | friction 参数存在，无堆积动力学 |

### F. 连续场 / 介质（波、扩散、流体）
| 接触面 | 状态 | 技术方案 |
|--------|------|---------|
| 声音传播 | ✅ 最完整 | SPL 衰减+吸收+遮挡+介质边界+风/温梯度（音频 003） |
| 气味扩散 | ✅ | 懒采样解析式（高斯核×风×时间衰减）（嗅觉 001） |
| 视线遮挡 | ✅ | DDA march（VisibilityQuery） |
| 海洋波 | ✅ | Gerstner 程序化波（渲染侧） |
| 流体/液体（倒水/血液/泼洒） | ❌ | 仅 LiquidMat 类别 + porosity 字段 |
| 火焰蔓延 | ❌ | 下放魔法，无模型 |
| 积雪深度累积/融雪 | ⚠️ | 下放世界生成 L10，无公式 |
| 地面摩擦系数 μ | ❌ | "冰面打滑"只是闪避-50% 乘数，物理上不滑 |
| 材料热胀冷缩 | ❌ | 全模块未提及 |

### G. 角色-地形 & 动画-物理交接
| 接触面 | 状态 | 技术方案 |
|--------|------|---------|
| NPC 地形吸附 | ✅ | `height_at` 贴地 + 坡度门控（**无重力**，movement_system） |
| 玩家重力/跳跃 | ✅ | Godot GDScript 手写（player.gd） |
| 骨架松弛死亡 | ✅ | 否决布娃娃（空间查询 §2.4） |
| 脚/手 IK + look-at | ✅ | 9 层动画栈 Layer 5/6/4（无命名求解器） |
| Root motion 叠加 + 位移中断 | ✅ | 中断优先级 0-4 |
| 爬墙/攀爬 | ❌ | 仅动画基元轨迹 + 技能条目，**无位移/碰撞物理** |

## 三、盲区排名（用户担心的"遗漏"——按"卡地基"程度排）

| # | 盲区 | 为什么危险 | 关联场景 |
|---|------|-----------|---------|
| 1 | **`mass`/质量字段在生命模块根本不存在** | 场景 2/4 地基缺失——无质量则无动量、无击退距离、无承载 | 2, 4 |
| 2 | **击退距离公式空白** | 用户明确问"应飞多远"，恰是空的 | 4 |
| 3 | **投射物弹道横跨 3 模块全空白** | 火球/飞刀/弓箭——公共缺口，无人 own | 2, 4 |
| 4 | **DroppedItem.velocity 悬空** | 唯一物理体却无人积分 | 1, 2 |
| 5 | **扒在大型生物身上 = 0** | 最难的动态参考系，想要但无设计 | 4 |
| 6 | **爬墙 = 0** | 重力重定向，想要但无设计 | 1 |
| 7 | **地面摩擦 μ 不存在** | "打滑"是伪物理（只改命中率，人不真滑） | 5 |
| 8 | **多体连锁碰撞 = 0** | 求解器诱惑最强处——**决策为明确放弃** | 2, 4 |

> 决策树如何逐一处置这 8 个盲区，见 [[002-决策树Q1-Q8]]。
