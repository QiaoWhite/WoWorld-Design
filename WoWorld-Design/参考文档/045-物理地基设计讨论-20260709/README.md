## 三条贯穿性结论（先说结论）

1. **不引入通用刚体物理引擎，扩展现有 Rust 运动学 + 空间查询架构。** 这不只是重申 CHG-033——全项目从头到尾都假设"没有求解器"：建筑倒塌是状态切换、战争是方阵统计、动物飞行"砍掉空气动力学"、死亡是骨架松弛不是布娃娃、帆船风驱动被显式删除。引擎是逆着全项目取向走。

2. **WoWorld 的涌现预算花在"心智"上，不花在"物质"上。** 大规模刚体动力学（成百上千物体堆叠/翻滚、碎块崩落、货物乱滚）与三根支柱正面冲突：① 持久化——存不下的碎块落点不是涌现是装饰；② 确定性——刚体求解器跨帧率/平台不可复现；③ LOD 预算——没人看的地方不模拟，也就谈不上涌现。滑坡走 CHG-065 地形编辑（密度场），倒塌走视觉 VFX，货物抽象成库存。

3. **一条统一的「质量→冲量→单体 COM」力学地基能让击退/坠落/碰撞/投射/水浮全部涌现自 2 个物理量（质量、冲量）+ 1 个积分器。** 这是"故事生成器"哲学在物理层的体现——定义底层规则，让宏观结果涌现，而非逐招硬编"飞多远"。
