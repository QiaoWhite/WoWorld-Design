# 003-程序化世界生成参考

> 对应设计：WoWorld 世界生成管线——多层噪声→生物群系→聚落→建筑(WFC)→道路→NPC初始化→历史模拟

---

## 一、世界生成的哲学分层

WoWorld 的世界生成不是一次性完成的。它分为两个宏大的阶段：

```
阶段A：物理世界生成（种子→地形→生物群系）
阶段B：人文世界生成（聚落→建筑→道路→NPC→历史）
```

A 决定了"这个世界的物理形态是什么样的"。  
B 决定了"在这个物理形态上，人是怎么生活的"。

两者同样重要。一个只有物理没有人文的世界是死寂的；一个只有人文没有物理支撑的世界是不可信的。

## 二、地形生成参考

### 2.1 噪声函数的选择与组合

| 噪声 | 特点 | 在 WoWorld 中的用途 |
|------|------|-------------------|
| **Perlin** | 平滑渐变、自然感 | 大陆轮廓、基本高低起伏 |
| **Simplex** | 比Perlin计算更快、方向性伪影更少 | 中型地形特征 |
| **Worley (Cellular)** | 细胞状图案 | 山脊、峡谷、特殊地貌 |
| **Fractal (fBm)** | 多层叠加、细节丰富 | 微地形细节 |
| **Domain Warping** | 用噪声扭曲噪声的输入坐标 | 奇幻地形（浮空岛、扭曲山脉） |

**多层叠加策略**：
1. 第一层（大尺度）：决定大陆和海洋的基本形态
2. 第二层（中尺度）：叠加山脉、高原、盆地
3. 第三层（小尺度）：叠加丘陵、河谷、平原微起伏
4. 特殊层：奇幻地形特征（魔力污染区、陨石坑、古战场痕迹）

### 2.2 水文系统

河流是文明的血管——聚落沿河而建，道路沿河延伸。

**生成流程**：
1. 根据高程图确定分水岭
2. 从高山向低处追踪水流路径
3. 支流汇入干流，形成树状水系
4. 低洼处形成湖泊、湿地

**设计注记**：河流必须"物理正确"——水向低处流。如果山和水的位置不合理，整个世界会失去可信度。

### 2.3 生物群系

生物群系不应仅依赖纬度，还应考虑：
- **高程**：海拔越高越冷（类似温带→寒带过渡）
- **降水**：山脉的雨影效应（迎风坡湿润、背风坡干燥）
- **洋流**：暖流带来温暖湿润
- **魔法影响**：特殊区域可以有非自然的群系（被魔法扭曲的森林、元素位面渗漏区）

## 三、聚落与建筑生成参考

### 3.1 聚落选址的逻辑

人类为什么在这里建立聚落？这个问题必须是可回答的。

| 选址因素 | 权重 | 说明 |
|---------|------|------|
| 水源 | 🔴 必需 | 河流、湖泊、地下水 |
| 可耕种土地 | 🔴 高 | 平原、河谷 |
| 防御性 | 🟡 中 | 高地、半岛、河湾 |
| 资源 | 🟡 中 | 矿产、森林、渔业 |
| 交通节点 | 🟡 中 | 河流交汇处、山口、天然港口 |
| 特殊原因 | 🟢 低 | 宗教圣地、魔法资源点、古代遗迹旁 |

### 3.2 聚落等级体系

```
帝国都城 (>10,000 NPC)
  └─ 王国都城 (3,000-10,000)
      ├─ 公国首府 (1,000-5,000)
      │   ├─ 城镇 (500-2,000)
      │   │   ├─ 村庄 (50-500)
      │   │   │   └─ 独立农庄/猎户小屋 (<50)
```

**聚落间关系**：
- 政治隶属（B是A的封臣）
- 经济依赖（C向B提供粮食）
- 竞争关系（D和E争夺同一资源）
- 文化亲缘（F和G共享同一文化种子）

### 3.3 建筑生成：Wave Function Collapse (WFC)

WFC 是 WoWorld 建筑布局生成的核心算法。

**基本原理**：
- 将建筑平面分割为网格单元
- 每个单元有"可能的状态"（走廊、房间、墙壁、空地）
- 相邻单元的状态必须符合约束规则（房间旁不能是墙壁除非有门）
- 逐步坍缩：每次选择熵最小的单元，随机确定其状态，传播约束

**WFC 在 WoWorld 中的分层应用**：
- **城市级 WFC**：街区和道路的布局
- **建筑级 WFC**：建筑内部房间和走廊的布局
- **房间级 WFC**：房间内家具和装饰的摆放

**参考**：Maxim Gumin 的原始 WFC 实现；Oskar Stålberg 的 Townscaper 和 Bad North

## 四、道路系统参考

WoWorld 的五级道路系统是世界"活着"的重要标志：

| 级别 | 类型 | 生成方式 |
|------|------|---------|
| 1级 | 硬化路 | 世界生成时的主要交通干线 |
| 2级 | 土径 | NPC高频行走自然踩出 |
| 3级 | 个人记忆路线 | 单个NPC经常走的路线 |
| 4级 | 传闻路线 | 从他人那里听来的路径 |
| 5级 | 魔法/特殊路径 | 传送门、虚空通道、古老秘道 |

**设计灵感**：
- **Desire Path**（意愿路径）——城市规划中的概念：人们在草地上踩出的捷径就是最自然的道路。WoWorld 的2级道路正是这个概念的数字化
- **动物踪迹→人类小径→马车道→石板路**——道路的自然演化链

## 五、历史模拟参考

### 5.1 Dwarf Fortress 的世界生成历史

DF 在世界生成时模拟了数百年甚至上千年的历史：
- 文明建立城市
- 战争、外交、贸易
- 重要人物（英雄、国王、叛徒）的兴衰
- 神器被锻造、巨龙被杀死、诅咒被降下

WoWorld 的历史模拟应该借鉴这种"预生成历史+持续推演"的模式。

### 5.2 历史模拟的关键事件类型

| 事件类型 | 对世界的影响 |
|---------|------------|
| 文明建立/灭亡 | 城市、道路、文化、语言的诞生与消亡 |
| 大规模战争 | 边界移动、人口减少、仇恨记忆、英雄诞生 |
| 瘟疫/自然灾害 | 人口剧减、废墟增加、社会重组 |
| 宗教运动 | 新信仰传播、旧信仰消亡、圣地建立 |
| 技术/魔法突破 | 新物品、新建筑、新职业 |
| 重要人物的生死 | 关系网断裂与重组、权力真空 |

### 5.3 历史记录的"多版本"特性

不同文化/势力对同一历史事件的记载可能完全不同。一场战争——胜方称之为"光荣征服"，败方称之为"野蛮入侵"。这不是需要修正的bug，而是世界深度的来源。

> **对 WoWorld 而言**：世界生成不是"做一张好看的随机地图"——而是创造一个让10万NPC能够安居、迁徙、争斗、繁衍的物理和人文空间。每一座山、每一条河、每一座城都必须经得起"为什么在这里"的追问。
