## CH-1 · 要素数量标题与内容不符

### 涉及位置

- `01-魔法.md` 第13行标题：`*组成一个魔法的六个要素*`
- `01-魔法.md` 第13行公式：`魔法 = 元素 + 范围 + 对象 + 接触 + 影响 + 时间 + 条件`（7项）
- `01-魔法.md` 第15-95行：实际展开为 1.范围 2.对象 3.接触 4.影响 5.时间 6.条件，另加第94行 `7.魔法元素`
- `12-魔法设计.md` 第7行：`魔法方程式的七大要素`

### 冲突描述

`01-魔法.md` 的章节标题明确写的是"六个要素"，但：
- 公式中列出了7个加号连接的项
- 内容区编号为1-6，但多出一个独立编号的"7.魔法元素"（第94行）
- 同文件夹下的 `12-魔法设计.md` 一致称之为"七大要素"

最可能的原因是"元素"最初不被计入"要素"（被视为基底而非变量），但后续设计演进中将其纳入公式，标题却未同步更新。

### 修改建议

**方案A（推荐）**：将标题改为"七个要素"，删除第94行孤立的"7.魔法元素"条目，将元素定义自然融入前文公理部分已有的十元素说明。公式已清晰表达七个要素的关系，无需单独赘述。

**方案B**：保留六要素框架，将"元素"降级为"基底参数"（而非独立要素），在数学表达式中以脚注或括号注明其特殊地位。但此方案与 `12-魔法设计.md` 的"七大要素"表述产生新冲突，不推荐。（废除）

选择：A

## CH-2 · 基本操作列表不一致

### 涉及位置

- `01-魔法.md` 第9行（公理二）：`增加、减少、吸收、释放、转化、转移`（6项，含转化/转移，不含排斥）
- `01-魔法.md` 第54-56行（影响·基本操作）：`增加、减少、排斥、吸收、释放`（5项，含排斥，不含转化/转移）
- `01-魔法.md` 第58-64行（影响·复合影响）：`转移`、`转化`、`链接` 被归类为复合操作

### 冲突描述

同一文档的两处对"基本操作"的界定出现分歧：

| 操作 | 公理中的地位 | 影响节中的地位 |
|------|-------------|---------------|
| 增加 | 基本 | 基本 ✅ |
| 减少 | 基本 | 基本 ✅ |
| 吸收 | 基本 | 基本 ✅ |
| 释放 | 基本 | 基本 ✅ |
| 排斥 | **未提及** | 基本 ❌ |
| 转化 | **列为基本** | 复合 ❌ |
| 转移 | **列为基本** | 复合 ❌ |

公理将"转化""转移"提升至基本操作层级，暗示它们在宇宙法则中具有原始性；而影响节则从工程视角将它们拆解为更小操作的组合。两种视角各有道理，但并列出现且无说明，构成事实上的矛盾。

### 修改建议

**方案A（推荐）**：在公理部分保留高层的六项（增加、减少、吸收、释放、转化、转移），在影响节开头增加一句过渡说明：

> "在公理层面，增加、减少、吸收、释放、转化、转移是六种基本变化。而从施法工程的角度，转化与转移可进一步拆解为基本操作的组合，另引入排斥以满足力场类魔法的描述需求。因此实际施法方程式中，基本操作池为：增加、减少、排斥、吸收、释放；转化、转移、链接等为复合操作。"

即将公理的"基本"与工程的"基本"区分为两个层级，消除表面的矛盾。

**方案B**：统一为同一套列表。以公理为准，将"排斥"归入"释放"的子类，将"转化""转移"保留为基本操作。后果是需要在魔法类型表中调整各类型的"核心影响"字段。不推荐——当前影响节的分类（基本5+复合3）工程可用性更强。

**无论如何，至少应**：在公理段落的"转化""转移"旁加注释指向影响节，或在影响节开头注明与公理的视角差异。

（选择：A和B都不要。按我的定义的精神重构“魔法影响”部分，包含“基本操作”和“复合影响”）

## CH-3 · 接触方式分类在类型表中被模糊化

### 涉及位置

- `01-魔法.md` 第44-48行（接触分类）：明确定义两种接触——**过程接触** 与 **凭空接触**，二分互斥
- `01-魔法.md` 第118-126行（魔法类型表）：

| 类型  | 典型接触方式   |
| --- | -------- |
| 变幻系 | 过程/凭空    |
| 召唤系 | 过程       |
| 破坏系 | **过程为主** |
| 守护系 | 过程/凭空    |
| 惑心系 | **凭空为主** |
| 疗愈系 | **凭空为主** |
| 死灵系 | 过程/凭空    |

### 冲突描述

分类体系只定义了"过程"和"凭空"两种离散类别，但类型表中出现了"过程为主"和"凭空为主"——这些修饰语暗示一个魔法可以同时具备两种接触特征，或介于两者之间。这在当前二元分类中无法表达，属于类型表引用了尚未定义的分类粒度。

### 修改建议

**方案A（推荐）**：在接触分类中增加"混合接触"概念——一个魔法可以同时指定过程接触和凭空接触两种路径，分别在方程式中占有不同权重。"为主"意味着该魔法以某种接触为主导，辅以另一种。需定义权重标注方式（如接触=过程:0.8 + 凭空:0.2）。

**方案B**：将类型表中的"为主"全部替换为确定的单一分类，待混合接触概念成熟后再修订。破坏系→过程，惑心系→凭空，疗愈系→凭空。简单但损失了设计精度。


（选择：都不选。你没有理解这张表格的含义。变幻系、召唤系等魔法类型不是具体的魔法，他们是某些魔法的大类。比如魔法A和魔法B都是变化系魔法，但魔法A是过程的，魔法B是凭空的。因而我原本写的 典型接触方式 一栏是正确的。我要做的更改是，接触方式 都既有凭空又有过程，某一种接触方式不是某一系魔法的专属）

## CH-4 · 施法媒介内容完全重复

### 涉及位置

- `01-魔法.md` 第99-112行（"施法媒介与方式"章节）
- `07-施法媒介与方式.md` 全文

### 冲突描述

两份内容**逐字完全相同**（语言、物质、仪式、意志、契约五条）。这不是观点冲突，而是维护性风险：任何一方的更新若未同步，将立即产生事实性矛盾。

### 修改建议

**方案A（推荐）**：`01-魔法.md` 中保留章节标题和一句概述，用 wikilink `[[07-施法媒介与方式]]` 指向独立文档，正文仅保留独立文档中不便包含的上下文衔接句。

**方案B**：删除 `07-施法媒介与方式.md`，将全部媒介内容归入 `01-魔法.md`。但考虑到当前文档结构的模块化趋势（魔导书、魔法卷轴均已独立），不推荐此方向。


(选择：方案A)

## CH-5 · 七要素排列顺序不一致

### 涉及位置

- `01-魔法.md` 第13行：`魔法 = **元素** + 范围 + 对象 + 接触 + 影响 + 时间 + 条件`（元素在前）
- `12-魔法设计.md` 第7行：`范围、对象、接触、元素、影响、时间、条件`（范围在前，元素第四位）

### 冲突描述

两处对七大要素的排列顺序不同。魔法.md 以"元素"为首（暗示元素是基底），魔法设计.md 以"范围"为首（暗示从空间维度开始设计）。在严格的形式化系统中，顺序可能影响方程式的解析方式。

### 修改建议

统一为同一顺序。建议以 `12-魔法设计.md` 的工程视角为准（范围→对象→接触→元素→影响→时间→条件），因为魔法设计流程天然遵循"先确定作用范围、再确定目标、再确定传递路径"的思维顺序。`01-魔法.md` 的公式改为：

```
魔法 = 范围 + 对象 + 接触 + 元素 + 影响 + 时间 + 条件
```

(赞同建议)

## CH-6 · 灵元素本质的表述差异

### 涉及位置

- `01-魔法.md` 第8行：`灵是寄宿于血、连接精神与物质的"灵魂元素"`（陈述句，确定）
- `02-元素.md` 第3-4行：`关于灵元素的本质尚无定论，一说是寄宿在血元素之中，一说是血元素本身`（承认不确定）

### 冲突描述

`01-魔法.md` 将"灵寄宿于血"作为确定事实陈述；`02-元素.md` 则认为这是学术争议（寄宿说 vs 同一说）。两份文档的认知论层级不同：魔法.md 偏向应用层面（不管本质如何，实用模型就是灵寄宿于血），元素.md 偏向本体论层面（事物的终极本质仍未被完全理解）。

这不一定需要"解决"——在游戏中保持这种张力可能是刻意的世界构建（学者之间也有争议）。但当前问题是两文档的作者语气不一致：一个以确定语气陈述，一个以不确定语气陈述，但都没有声明自己的认知论层级。

### 修改建议

在 `01-魔法.md` 公理部分增加一句限定：

> "在实用魔法学的框架下，**灵**被视为寄宿于**血**、连接精神与物质的'灵魂元素'。关于灵的本质是否为血的某种形式，学术界仍有争论，详见[[02-元素]]。"

`02-元素.md` 无需修改——它已经正确地呈现了争议。

（赞同建议）

## CH-7 · 法阵术语跨文档不一致

### 涉及位置

- `12-魔法设计.md` 第41行：`法阵被绘制在卷轴上，用导魔墨水精确描摹`
- `14-魔法卷轴.md` 第5行：`持用者用手或触媒划过卷轴上绘制的**特殊长条纹路**`

### 冲突描述

`12-魔法设计.md` 明确指出卷轴上的图形就是"法阵"，而 `14-魔法卷轴.md` 用"特殊长条纹路"这一描述性用语替代了"法阵"这一学术术语。从世界构建的角度，普通使用者可能确实称之为"条纹"——但 `14-魔法卷轴.md` 是设计文档而非游戏内文本，应从设计者视角使用统一术语。卷轴制作门槛一节也提到"卷轴专用的绘图与附魔技法"，更应在此处明确这就是一种法阵绘制技术。

### 修改建议

`14-魔法卷轴.md` 修改为：

```
持用者用手或触媒划过卷轴上绘制的法阵纹路（外观上呈现为特殊的长条纹路），即可静默释放该法术。
```

保留"长条纹路"作为外观描述，但明确其学术身份为"法阵"。在制作门槛节中补充："卷轴专用的绘图与附魔技法本质上是法阵绘制的一个特殊分支。"


（反对建议。我要澄清：魔法卷轴上的**长条纹路**是特制的，区别于**法阵**）
