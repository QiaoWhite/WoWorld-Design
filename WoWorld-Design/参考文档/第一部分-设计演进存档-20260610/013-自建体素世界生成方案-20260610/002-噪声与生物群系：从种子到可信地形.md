# 002-噪声与生物群系：从种子到可信地形

> 对应 `01-世界生成总流程.md` 的阶段一~三  
> 从种子出发，经过多层噪声叠加→水文模拟→生物群系分类→地质特征，产出"可信的"基础地形密度场

---

## 〇、设计原则

程序化地形生成的核心挑战不是"让地形看起来随机"——而是"让地形看起来**合理**"。

一个合理的地形是：山在它应该在的地方（板块碰撞边缘），河向低处流（水文正确性），沙漠在山脉的雨影区（降水梯度），聚落沿河而建。这些约束不来自噪声函数——来自对自然过程的理解。

WoWorld 的地形生成分四个步骤，每一步都增加一层"合理性"：

```
种子 → 大尺度地形（大陆轮廓、山脉走向）
     → 中尺度地形（丘陵、河谷、盆地）
     → 水文系统（河流、湖泊、海岸线）
     → 生物群系（温度、降水 → 植被、地表类型）
     → 地质特征（洞穴、悬崖、矿脉）
```

## 一、多层噪声叠加

### 1.1 噪声分工

| 层级 | 噪声类型 | 频率 | 振幅 | 用途 |
|------|---------|------|------|------|
| 大陆层级 | Perlin | 极低（1/5000m） | 极高（±200m） | 大陆和海洋的基本轮廓 |
| 山脉层级 | Ridged Perlin | 低（1/1000m） | 高（±100m） | 山脉走向和高度 |
| 丘陵层级 | Fractal Brownian Motion (fBm) | 中（1/200m） | 中（±30m） | 丘陵、高原、谷地 |
| 细节层级 | Worley (Cellular) | 高（1/50m） | 低（±5m） | 微地形变化（岩石露头、小凹陷） |
| 奇幻层级 | Domain Warping | 中 | 变化 | 魔法扭曲的地形、浮空岛（可选） |

### 1.2 噪声组合公式

```rust
fn sample_base_density(world_pos: Vec3, seed: u64) -> f32 {
    // 1. 大陆层级——决定"这里是陆地还是海洋"
    let continental = perlin_2d(world_pos.x / 5000.0, world_pos.z / 5000.0, seed);
    
    // 2. 高度基值（由大陆轮廓决定）
    let base_height = continental * 200.0;  // -200m (海盆) ~ +200m (大陆)
    
    // 3. 山脉叠加——只在陆地上有效
    let mountain = ridged_perlin_2d(world_pos.x / 1000.0, world_pos.z / 1000.0, seed + 1);
    let mountain_height = if continental > 0.3 {
        mountain * 100.0 * (continental - 0.3).max(0.0)
    } else {
        0.0
    };
    
    // 4. 丘陵叠加
    let hills = fbm_2d(world_pos.x / 200.0, world_pos.z / 200.0, seed + 2, octaves=4);
    let hill_height = hills * 30.0;
    
    // 5. 微地形
    let micro = worley_2d(world_pos.x / 50.0, world_pos.z / 50.0, seed + 3);
    let micro_height = micro * 5.0;
    
    // 6. 最终地表高度（y=0 为海平面）
    let surface_height = base_height + mountain_height + hill_height + micro_height;
    
    // 密度值 = 高度 - 当前采样点的 y 坐标
    // y < surface_height → 密度 > 0（固体内部）
    // y > surface_height → 密度 < 0（空气）
    surface_height - world_pos.y
}
```

## 二、水文系统

### 2.1 河流生成

```rust
struct RiverSystem {
    rivers: Vec<River>,
}

struct River {
    source: Vec2,          // 源头位置（高山/高地）
    mouth: Vec2,           // 入海口/汇入湖泊
    path: Vec<Vec2>,       // 河道路径
    width: f32,            // 河宽（源头窄→入海口宽）
    flow_rate: f32,        // 流速
}
```

**生成算法**：
1. 在高降水 + 高海拔区域随机选取源头点
2. 从源头出发，沿最陡下坡方向迭代前进——每一步向相邻单元格中最低的点移动
3. 支流汇入：当两条河流的路径在 3 格内交叉时，支流汇入干流
4. 河道对地形的影响：河流经过的体素，密度值降低（河道凹陷），宽度从源头到入海口递增

### 2.2 湖泊

- 局部低洼区域，如果降水足够 → 积水形成湖泊
- 湖泊面积 = 低洼区域的汇水面积 × 降水系数
- 湖面高度 = 湖水溢出点的海拔

### 2.3 海岸线

- 海平面 = 0（设计选择）
- 海洋深度 = -continental × 200m（大陆架较浅，海盆较深）
- 海岸线附近的密度值有一个过渡带（沙滩——从固体到液体的平滑过渡）

## 三、生物群系

### 3.1 决定因素

| 因素 | 来源 | 影响 |
|------|------|------|
| **温度** | 纬度 + 海拔 | 热带/温带/寒带 |
| **降水** | 纬度 + 洋流 + 山脉雨影 | 湿润/半湿润/半干旱/干旱 |
| **高程** | 噪声叠加结果 | 极高海拔 = 冰雪，即使是赤道 |
| **纬度** | z 坐标映射 | 热带(±30°)/ 温带(±60°)/ 寒带 |

### 3.2 基于温度-降水二维空间的群系分类

```
降水 →
          干旱      半干旱     湿润      极湿
温度   ┌─────────┬─────────┬─────────┬─────────┐
  ↓ 热带 │ 沙漠    │ 稀树草原 │ 季风林   │ 热带雨林 │
     温带 │ 荒漠    │ 草原     │ 阔叶林   │ 温带雨林 │
     寒带 │ 冻原    │ 苔原     │ 针叶林   │ 沼泽    │
     极寒 │ 冰原    │ 冰雪覆盖  │ 冰雪覆盖  │ 冰雪覆盖 │
      └─────────┴─────────┴─────────┴─────────┘
```

### 3.3 群系对密度场的影响

不同的生物群系不只是"贴图不同"——它们影响体素密度场：

- 沙漠：表面有沙丘（低频低振幅噪声叠加）
- 森林：表面有轻微的随机凹凸（树根、倒木）
- 沼泽：表面有小型水坑（局部密度降低）
- 冰雪：高海拔/高纬度的固体表面被积雪覆盖

## 四、洞穴与地质特征

### 4.1 洞穴生成

```
洞穴使用 3D 噪声（而非 2D）——因为洞穴存在于地下三维空间中：

cave_noise = perlin_3d(pos.x / 100.0, pos.y / 30.0, pos.z / 100.0, seed + 6)

规则：
  - 如果 cave_noise > 0.6 且 地表以下 → 挖空（密度设为负值）
  - 如果 0.4 < cave_noise < 0.6 → 半挖空（产生钟乳石/石笋效果）
  - 如果 cave_noise < 0.4 → 保持固体

洞穴只在陆地表面以下生效——不挖空海底
洞穴密度随深度递减——极深处更少洞穴（地幔压力）
```

### 4.2 悬崖与陡坡

- 当相邻体素的表面高度差 > 阈值 → 产生悬崖
- 悬崖处的密度场需要特殊的"削平"处理——避免 Marching Cubes 产生锯齿状边缘
- 参考：`dust` 项目中的 dual contouring 方法——保留尖锐特征的备选方案

### 4.3 矿脉分布

```
矿脉 × 生物群系 × 深度的三维分布：

  coal:    温带 + 深度 > 20m, 密度 0.01
  iron:    全群系 + 深度 > 30m, 密度 0.005
  copper:  热带 + 深度 > 25m, 密度 0.005
  gold:    山脉 + 深度 > 40m, 密度 0.001
  mithril: 魔法区 + 深度 > 50m, 密度 0.0005
  
矿脉形状: 片状噪声——不是均匀分布，而是集中在噪声值 > 阈值的地方
```

## 五、确定性保证

整个基础地形生成必须是**确定的**——

```
给定:
  - 相同的世界种子 (u64)
  - 相同的世界坐标 (x, y, z)
  - 相同的生成参数版本

→ 永远返回相同的密度值
```

这意味着：
- 噪声函数使用种子派生的确定性种子
- 不使用系统时间或任何外部随机源
- 生成参数（噪声频率、振幅、生物群系阈值）在存档 header 中记录——如果未来改变参数，旧存档仍用旧参数
- 存档只保存"与种子的差异"（chunk 修改层）

## 六、性能策略

| 策略 | 说明 |
|------|------|
| **逐 Chunk 评估** | 噪声不在世界坐标系中全局采样——每个 chunk 独立评估其覆盖的体素范围 |
| **GPU 加速噪声** | 远期将噪声生成迁移到 compute shader。对于 32×32×32 体素的 chunk（32768 个采样点），GPU 可以一次 dispatch 完成——CPU 需要逐点循环 |
| **噪声缓存** | 同一个 chunk 的噪声值在生成后缓存——修改只影响"修改层"，基础地形层不变 |
| **LOD 降采样** | LOD 1/2 的密度值通过 LOD 0 的降采样获得——不需要重新运行噪声 |
