## 概要

当前完成 12 个 Phase 的设计产出——11 个模块 ~23,750 行开发规格。但全面审计发现：

- **5 个致命缺失模块**（被大量交叉引用）：文化、UI/UX、载具系统、蓝图与建造、经济与社会
- **5 个高度缺失**：存档、政治实体、采矿深度、名声、玩家系统
- **7 个中度缺失**：音频、疾病/医疗、派系、节日/仪式、教程、冒险小队、世界机
- **12+ 个低度缺失**
- **现有模块薄弱点**：NPC 缺感官系统、魔法缺性能预算和 AI、战斗缺远程武器、家具仅 v0.1

本规划以 6 个设计产出 Phase（13-18）+ 1 个验证 Phase（19），覆盖所有缺失和薄弱模块。

## 总体规划概览

```
Phase 13: 社会运转层 → 经济·文化·法律·政治·名声 (5大模块)
Phase 14: 世界交互层 → 载具·建造·采矿·农业·地下城 (5大模块)
Phase 15: 玩家体验层 → UI/UX·玩家扩展·教程·死亡传承·日志 (5大模块)
Phase 16: 深度涌现层 → 节日·娱乐·时尚·饮食·NPC生命周期 (5大模块)
Phase 17: 现有模块增强 → NPC感官系统★·魔法深扩·战斗远程·NPC v2.1·家具 v2.0·疾病医疗·音频 (7项)
Phase 18: 技术支撑层 → 存档·MOD·无障碍·本地化·调试工具·难度系统 (5大模块)
Phase 19: 全面跨模块理念走通验证
```

## 关键决策

- **NPC 必须通过模拟感官感知世界**——不能"查询数据库"。视听嗅触+内感觉五通道，信息不完整→决策不确定→行为涌现→故事生成
- **文化系统独立模块**——非魔法附属（当前仅 `魔法/018` 86 行）。文化参数场连续非离散，与群系参数场同级
- **先设计后开发**——Phase 13-19 全部为纯设计文档工作。在理念走通前不写代码
- **每个 Phase 含跨模块接口契约**——延续 CHG-013 契约表传统，新模块接口必须与已有模块对齐
- **每个 Phase 含理念走通验证场景**——端到端场景逐链检查，确保系统相互独立又相互配合
