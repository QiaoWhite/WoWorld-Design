## 一、NPC 如何知道节日

### 1.1 多通道宣告

`FestivalInstance` 进入宣告期（Announcement Phase）时，信息传播系统发射节日宣告到所有受影响的 NPC。五种传播通道各有角色：

| 通道 | 在节日宣告中的角色 | 优先级 |
|------|------------------|--------|
| **Official** | 聚落布告板（贴出节日通告）、公告员喊话、钟声/号角信号 | 高——NPC 在聚落内必然收到 |
| **Oral** | NPC 间对话传播——"下周就是收获节了，你家准备好了吗？" | 中——社交网络中传播 |
| **Letter** | 远距离通知——远方亲属收到节日邀请信 | 低——仅对已有社交关系的 NPC |
| **Magical** | 魔法传讯——法师/神殿向远方信徒发送 | 极低——仅适用于魔法亲和的文化 |
| **Courier** | 信使传递——官方信使通知周边聚落 | 低——仅对大型跨聚落节日 |

### 1.2 NPC 的"节日意识"状态

```rust
/// NPC 对即将到来的节日的认知
struct NpcFestivalAwareness {
    /// 已知的即将到来的节日列表
    upcoming: Vec<UpcomingFestival>,
    /// 是否打算参加
    will_attend: FxHashMap<FestivalId, bool>,
    /// 准备状态
    preparation: FxHashMap<FestivalId, FestivalPreparation>,
}

enum FestivalPreparation {
    Unaware,          // 还不知道
    Aware,            // 知道但还没行动
    Preparing,        // 在准备（购物/做衣服/练习竞技）
    Travelling,       // 在旅途中（外地NPC）
    Attending,        // 正在参加
    Done,             // 已经参加了
    Skipped,          // 选择不参加
}
```

NPC 从 Unaware → Aware 的转变由信息传播系统的宣告抵达触发。抵达时间取决于 NPC 的位置和使用的传播通道——聚落内的 NPC 几乎即刻知道，偏远农田的 NPC 可能通过口传在 1-2 天后才知道，远方的 NPC 可能完全不知道（除非有个人关系纽带）。

## 二、NPC 如何决定参加

### 2.1 参加决策因子

```rust
fn will_attend_festival(
    npc: &NpcData,
    festival: &FestivalInstance,
    culture: &dyn CultureQuery,
    faith: &dyn FaithQuery,
    current_weather: &WeatherSample,
) -> f32 {
    // 返回 0-1 参加概率
    
    // 1. 义务因子——如果节日是强制性的且 NPC 在义务范围内
    let obligation = if festival.framework.mandatory_attendance 
        && npc.is_subject_to(festival.organizer_power_edge) {
        if npc.has_immunity_for("festival_attendance") { 0.0 }
        else { 0.95 } // 几乎一定参加——否则有制裁
    } else { 0.0 };
    
    // 2. 虔诚度——宗教节日的核心因子
    let piety = if let Some(faith_id) = festival.bound_faith {
        npc.piety(faith_id) * 0.4
    } else { 0.0 };
    
    // 3. 人格因子
    let personality = 
        npc.big_five.extraversion * 0.20      // 外向→喜欢节日人群
        + npc.big_five.openness * 0.15        // 开放→对新体验好奇
        - npc.big_five.neuroticism * 0.10;     // 神经质→人群焦虑（轻微负影响）
    
    // 4. 关系因子——有亲密关系的人已经决定参加？
    let social = if npc.has_close_relations_attending(festival.id) {
        0.25 // "我最好的朋友都去——我也去"
    } else { 0.0 };
    
    // 5. 距离因子
    let distance_km = npc.position.distance_km(festival.location);
    let distance = (-distance_km / 10.0).exp().clamp(0.05, 1.0); // 指数衰减，10km 特征距离
    
    // 6. 天气因子
    let weather = match current_weather.condition {
        WeatherCondition::Clear | WeatherCondition::PartlyCloudy => 1.0,
        WeatherCondition::Overcast => 0.8,
        WeatherCondition::LightPrecip => 0.5,
        WeatherCondition::HeavyStorm => 0.1, // 暴雨/暴雪→几乎不外出
        _ => 0.3,
    };
    
    // 7. 文化因子——参加节日是该文化的"正常"行为
    let cultural_norm = culture.communication_norms(npc.culture_id)
        .map(|n| n.emotional_expressiveness * 0.1 + 0.1)
        .unwrap_or(0.1);
    
    // 综合
    let raw = obligation.max( // obligation 可以覆盖其他因子
        piety + personality * 0.25 + social + distance * 0.20 + weather * 0.10 + cultural_norm
    );
    
    raw.clamp(0.0, 1.0)
}
```

### 2.2 参加决策在 NPC 三层决策系统中的位置

NPC 的三层决策系统（见 NPC ver2.0）：
1. **概率行为树（~90%决策）**：节日的参加权重覆盖正常 Routine 的权重
2. **GOAP 安全网（~9%）**：不触发——节日不是生存威胁，除非 NPC 角色是组织者（强制 GOAP）
3. **LLM 增强（~1%）**：L1 NPC 的节日特殊对话

**概率行为树的节日覆盖**：

```
宣告期：
  正常 Routine 权重保持不变
  + "节日准备"行为获得温和权重加成（0.1-0.2）
  
预备期：
  "节日准备"行为权重继续增加
  "购物（节日物品）"权重 ×1.5
  "社交（讨论节日）"权重 ×1.3
  
庆祝期：
  如果 NPC 决定参加：
    正常 Routine 权重 ×0.1（接近暂停——除了必要行为）
    + 节日活动权重 ×3.0（占主导）
  如果 NPC 决定不参加：
    正常 Routine 权重保持不变（但聚落少了很多人）
```

### 2.3 角色化参加——不同 NPC 在节日中的不同角色

| NPC 角色 | 节日行为 |
|---------|---------|
| **祭司/宗教人员** | 主持宗教仪式——`TurnMode::Ritual` 主导 |
| **统治者/官员** | 主持开场式、颁奖、发表演讲——合法性获取 |
| **商人/店主** | 加班营业（节日消费激增）或关闭店铺参加 |
| **艺人**（吟游诗人/舞者） | 表演——在节日中获得最大收入 |
| **守卫/士兵** | 维持秩序——节日期间的犯罪率 ×1.3 |
| **普通居民** | 参加活动、宴饮、社交——正常 Routine 暂停 |
| **外来 NPC** | 旅行参加——住在客栈或亲友家 |
| **儿童 NPC** | 专门的儿童游戏活动——人格可塑性偏移 |

## 三、人群聚集机制

### 3.1 分层人群模拟（对齐 L1/L2/L3/L4 分层系统）

| 层次 | 距离 | 节日中的人群处理 |
|------|------|----------------|
| **L1**（≤50m） | 玩家近范围 | 每个 NPC 完整模拟——个体行为+对话+动画。`SocialField` 实时情绪传染 |
| **L2**（50-200m） | 中等距离 | 关键 NPC 完整模拟；普通 NPC 行为由统计模型驱动——"广场上有约 200 人，正在跳舞" |
| **L3**（200m-3km） | 远距离 | 人群密度贴图——广场区域密度高，街道密度低。不模拟个体 |
| **L4**（3km+） | 超远距 | 聚落的夜间灯光增多（如果有灯火节日）——烟雾密度/噪声水平反映节日活跃度 |

### 3.2 群体寻路优化

如果 500 个 NPC 同时从各自家中向广场聚集——不应 500 次独立 A* 寻路。

```
方案：
1. 分帧错开：宣告期→预备期的 3-7 天中，NPC 逐渐调整 Routine——
   不是在庆祝期第 1 天早晨全部同时冲向广场
2. L1 内 (< 50m)：个体 A* 寻路（玩家可见——需要完整性）
3. L2 外 (> 50m)：群体流场——预计算从各居住区到广场的最优路径
   → NPC 沿流场移动 → 只在遇到障碍时做局部避让
4. NPC 到达后：在广场中使用简化的社会力模型（Social Force Model）
   避免物理穿透——但不需要完整碰撞检测
```

### 3.3 情绪传染放大

`SocialField`（语言表达 006 中定义）已建模群体情绪动力学：

```
节日中的 SocialField 修正：
  - inertia × 0.5（更容易被带动——"节日氛围"）
  - polarization × 1.5（情绪更快趋于一致——"大家一起开心"）
  - conformity × 1.3（从众压力增大——"所有人都跳舞了，我也跳吧"）
```

**情绪传染链**：
```
祭司/统治者的开场仪式 → 参与者情绪 ↑
→ SocialField 传播到围观者 → 围观者情绪 ↑
→ 更多人加入 → 正反馈循环
→ 节日气氛达到高峰（第 2-3 天通常情绪最高）

余波期：
→ SocialField 修正恢复为正常值
→ 情绪在 1-2 天内回归基线
→ 部分 NPC 可能经历"节后失落"（mood 暂时低于基线 0.05-0.1）
```

## 四、NPC 的特殊节日行为

### 4.1 行为数据模型

```rust
/// NPC 在节日期间可执行的额外行为
enum FestivalNpcBehavior {
    /// 赠礼——NPC 向另一个 NPC 赠送节日物品
    GiveGift { recipient: NpcId, item: ItemEntId },
    
    /// 舞蹈——NPC 参与集体舞蹈
    /// 高外向性 + 低神经质 → 更可能参与
    Dance { dance_type: DanceType },
    
    /// 醉酒——节日宴饮中过度饮酒
    /// 效果：临时人格修正（尽责性↓神经质↓外向性↑）
    GetDrunk { intensity: f32 },
    
    /// 参加竞技
    Compete { competition_id: CompetitionId },
    
    /// 祈福/许愿——在宗教节日中
    Pray { prayer_type: PrayerType, target_deity: Option<DeityId> },
    
    /// 社交——与平时不常接触的人交谈
    Socialize { target: NpcId, topic: ConversationTopic },
    
    /// 观赏——作为观众看表演/竞技
    Spectate { event: SpectacleId },
}
```

### 4.2 赠礼行为

节日赠礼是重要的社交行为——它影响 NPC 间的关系：

```
赠礼触发条件：
  - NPC A 与 NPC B 的 relationship > 0.3（熟人以上）
  - 或 A 正在追求 B（courtship）
  - 或 B 是 A 的上级（权力距离高→向上赠礼是义务）
  
赠礼对关系的影响：
  - 礼物价值匹配期望 → 关系 +0.05
  - 礼物超出期望 → 关系 +0.10
  - 礼物远低于期望 → 关系 -0.02
  - 未收到回礼（如果文化要求互赠）→ 关系 -0.03
```

### 4.3 醉酒行为

醉酒 = 临时人格修正（已在 NPC ver2.0 中提到，但没有数据模型）：

```rust
/// 醉酒状态——临时叠加在 NPC 的 BigFive 和决策参数上
struct IntoxicationEffect {
    /// 尽责性：醉酒 → ↓
    conscientiousness_delta: f32,  // -0.15 per 酒的单位
    /// 神经质：初期↓，严重↑
    neuroticism_delta: f32,        // -0.10 initially, +0.10 if very drunk
    /// 外向性：醉酒 → ↑
    extraversion_delta: f32,       // +0.10
    /// 决策噪声：醉酒 → 行为更随机
    decision_noise: f32,           // +0.15
    /// 持续时间（游戏小时）
    duration_hours: f32,
}
```

**醉酒在节日中的涌现行为**：
- 高尽责性 NPC → 醉酒后可能"说漏嘴"（透露秘密——信息传播的意外通道）
- 高竞争取向 NPC → 醉酒后更容易挑衅 → 节日斗殴
- 高浪漫倾向 NPC → 醉酒后表白 → 意外关系形成

## 五、玩家作为节日参与者

### 5.1 玩家如何发现节日

| 方式 | 触发条件 |
|------|---------|
| **视觉线索** | 进入聚落→街道装饰（灯笼/旗帜）、NPC 穿 Ceremonial 服装、广场人群聚集 |
| **音频线索** | 钟声/号角、人群喧哗、音乐、NPC 节日问候（如果开启了 TTS） |
| **NPC 对话** | NPC 在对话中提及——"你来得正好！明天就是收获节了！" |
| **布告板** | 如果玩家阅读聚落布告板——节日通告（已通过 Official 通道发布） |
| **日志提醒** | 如果玩家之前与该聚落建立了关系（"你的朋友 XX 邀请你参加"） |
| **地图标注** | 玩家已探索区域的地图上标记即将到来的节日（如果玩家购买了"节日历"道具） |

### 5.2 玩家能做什么

| 角色 | 行为 |
|------|------|
| **旁观者** | 观看仪式/竞技/表演——不主动参与 |
| **参与者** | 加入宴饮/舞蹈/赠礼——与 NPC 相同的参与规则。玩家行为影响 NPC 对玩家的评价 |
| **竞技者** | 报名参加竞技——与 NPC 同台比赛。获胜 → 名声事件（+荣誉/勇武） |
| **商人** | 在节日集市摆摊——利用消费激增赚钱 |
| **捣乱者** | 破坏仪式、偷窃、斗殴——导致名声受损、被守卫追捕 |
| **演讲者** | 如果玩家有足够的声望和关系——可以发表公开演讲（TurnMode::Speech） |

### 5.3 玩家参与是否受节日排他性限制？

如果节日是 `Closed`（严格封闭）——玩家不能直接参加核心仪式。但可以：
- 在聚落外围感受节日氛围（装饰、音乐、NPC 的兴奋）
- 如果伪装/易装（`disguise_system`）→ 可能混入
- 如果获得本地权威的特别邀请 → 例外准入
- 如果强闯 → 立即产生法律后果（`CrimeRecord`）
