## 一、核心原则

> **载具系统只定义物理事实。运费从交易中涌现。成本从 NPC 心智中涌现。**

载具系统不定义：
- 运费公式 → 船长 NPC 自行定价
- 工资标准 → 船长和水手的雇佣谈判
- 燃料价格 → 经济系统订单簿撮合

载具系统只定义：
- 这个载具物理上能装多少货
- 这个载具每跑 100km 消耗多少什么燃料
- 这个载具物理上需要多少操作人员

## 二、VehiclePhysicalProfile —— 只定义物理事实

```rust
/// 载具的物理/运维参数——由 VehicleDef 定义
/// 这些是物理事实，不是经济参数
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct VehiclePhysicalProfile {
    /// 最大载货重量（kg）
    pub cargo_capacity_kg: f32,
    /// 最大载货容积（升）
    pub cargo_capacity_liters: f32,
    /// 乘客容量（不含船员）
    pub passenger_capacity: u32,

    /// 燃料类型（None = 人力/畜力/风力——不需要燃料）
    pub fuel_item: Option<ItemDefId>,
    /// 燃料消耗率（每 100km 消耗的燃料量）
    pub fuel_consumption_per_100km: f32,

    /// 正常磨损率（每 100km 的 hull 磨损——如 0.001 = 1000km 后磨损 1%）
    pub wear_rate_per_100km: f32,

    /// 建议维护间隔（km）——到了之后"建议维护"标记出现
    pub maintenance_interval_km: f32,
    /// 每次标准维护消耗的材料
    pub maintenance_materials: Vec<(ItemDefId, f32)>,

    /// 最低操作人数
    pub min_crew: u32,

    /// ★ 参考日薪——仅用于世界生成和 NPC 初始估值
    /// 不是"工资公式"——实际合同金额是两个 NPC 之间谈判的结果
    /// 对标现实中的"行业平均薪资"
    pub crew_salary_reference_copper_per_day: u32,
}
```

## 四、运费定价——船长 NPC 的心智涌现

```
场景: 船长的商船"飞鱼号"要从东海镇开到南海城（600km，航程约4天）

船长计算:
  成本:
    ├─ 魔力引擎: 600km ÷ 100 × 8.0 水晶/100km = 48 颗魔力水晶
    │   水晶市价(东海镇): ~5 铜/颗 → 48×5 = 240 铜
    ├─ 船员薪资: 12 船员 × 20 铜/天 × 4 天 = 960 铜
    ├─ 磨损: 600km × 0.001/100km = 0.006 hull 损伤
    │   港口维修 ~200 铜（摊销）
    ├─ 港口停泊费(南海城): 50 铜
    └─ 成本合计: ~1450 铜

  利润目标:
    ├─ 船长的人格: 高尽责性+中等贪婪 → 利润加成 30%
    └─ 利润: 1450 × 0.3 = 435 铜

  运力:
    └─ 剩余货舱: 800kg

  定价:
    每 100kg 运费 = (1450+435) ÷ 8 = ~235 铜/100kg

  如果船长发现竞争激烈:
    港口的另一条船报价 200 铜/100kg
    → 船长可能降到 190 铜/100kg（薄利多销）
    → 或者强调"我更快"（走外洋航线3天到——加急费）

  如果船长拥有垄断地位（唯一的船）:
    可能报价 400 铜/100kg
    → 但货主可能觉得太贵——走陆路（马车）反而便宜
    → 或者货主接受——因为货物必须海运

全部是 NPC 心智的自发行为——载具系统不写一行定价代码。
```

## 五、经济行为的全链路——一次委托货运

```
商人李四有 500kg 铁锭要从东海镇运到南海城

Step 1: 李四的 GOAP 目标 = "把铁锭卖了赚钱"
        → 查询自己的 EconomicCognition: 南海城铁锭市价更高
        → 决定运过去

Step 2: 李四找船
        → 调用 VehicleQuery::available_transport(东海镇, 南海城, 500kg)
        → 返回三条候选:
           飞鱼号: 4天, 235铜/100kg, 商船, 船长=张三
           海鸥号: 6天, 150铜/100kg, 渔船兼运, 船长=李七
           旋风号: 2天, 500铜/100kg, 快船, 船长=周九

Step 3: 李四的决策（NPC 心智——GOAP）
        权重:
          - 成本（重要: 0.4）
          - 速度（铁锭不是生鲜，不急: 0.1）
          - 安全性（听说飞鱼号船长经验丰富: 0.3）
          - 关系（李四认识张三——信任: 0.2）
        → 综合: 选飞鱼号（信任+安全 压倒 价格微差）

Step 4: 李四→张三（语言表达系统对话）
        李四: "张三船长，500kg铁锭到南海城，多少钱？"
        张三: "235/100kg——1175铜"
        李四: "成。三天后出发？"
        张三: "对。货先搬上船。"
        → Contract 创建: (李四, 张三, Extract=1175铜, Compel=将500kg铁锭安全送达)

Step 5: 装载
        李四.仓库 → 飞鱼号.cargo_hold: 500kg 铁锭
        （走物品系统 Container.move_item()）

Step 6: 航行
        4天后到达南海城

Step 7: 交付
        飞鱼号.cargo_hold → 南海城.李四的仓库: 500kg 铁锭
        李四.钱包 -= 1175 铜
        张三.钱包 += 1175 铜
        Contract 标记完成
        双方记忆更新
```

载具系统在整个流程中做了**两件事**：
1. `VehicleQuery::available_transport()` —— 告诉李四有哪些船可选
2. 物理移动——把船从东海镇飞到了南海城

其他全部是经济系统 + NPC 心智 + 权力系统 + 语言表达系统 + 物品系统的协作。

## 六、玩家体验

| 玩家角色 | 操作 | 底层 |
|---------|------|------|
| **货主** | 在码头搜索"去南海城的船"→ 比价 → 委托运输 | NPC 船长提供的运力 + 玩家询价 + Contract |
| **船长** | 设定航线 → 设定运费（手动定价或自动建议）→ 接货 → 装载 → 起航 | 玩家的心智替换 NPC 心智——同一套决策接口 |
| **乘客** | 买船票 → 上船 → 在船上度过数日 → 下船 | 票价=船长定价；船上生活 = 完全的 NPC 日常（自动涌现） |
| **投机者** | 在 A 港低价买货 → 雇船运到 B 港 → 高价卖出 | 全部走已有经济系统——玩家和 NPC 走同一路径 |
