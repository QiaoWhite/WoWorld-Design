### 3.2 新代码

```rust
/// 每个 Action 声明它满足的需求标签
pub enum NeedTag {
    Hunger,
    Thirst,
    Fatigue,
    ElementBalance,
    Libido,
    Social,
    Safety,
}

/// Action 定义中声明
pub struct ActionDef {
    pub action_type: ActionType,
    /// ★ 新增：此行动满足的需求
    /// 编译期常量切片, 零分配
    pub satisfied_needs: &'static [NeedTag],
    // ... 其他字段
}

/// 核心算法：计算行动对当前需求的匹配度
///
/// 对 action 声明的所有需求, 取各自的 urgency, 算术平均 → 权重乘数
fn need_action_match(
    action: &ActionDef,
    phys: &Physiology,
    social_deficit: f32,
) -> f32 {
    let needs = action.satisfied_needs;
    if needs.is_empty() { return 1.0; }  // 该行动不满足任何需求, 不受需求影响

    let mut total_urgency: f32 = 0.0;
    for need in needs {
        total_urgency += match need {
            NeedTag::Hunger          => (1.0 - phys.hunger)   * hunger_sensitivity(phys),
            NeedTag::Thirst          => (1.0 - phys.thirst)   * thirst_sensitivity(phys),
            NeedTag::Fatigue         => phys.fatigue           * fatigue_sensitivity(phys),
            NeedTag::ElementBalance  => phys.element_balance_urgency,
            NeedTag::Libido         => phys.libido,
            NeedTag::Social         => social_deficit          * social_sensitivity(phys),
            NeedTag::Safety         => 1.0 - phys.health,
        };
    }
    let avg_urgency = total_urgency / needs.len() as f32;

    // 需求越迫切, 权重越高。1.0 + urgency = 范围 1.0~2.0
    (1.0 + avg_urgency).clamp(1.0, 2.0)
}
```

### 3.3 行动-需求映射表

```rust
impl ActionType {
    pub const fn satisfied_needs(&self) -> &'static [NeedTag] {
        match self {
            // === 生存 ===
            Eat(_)           => &[NeedTag::Hunger],
            Drink(_)         => &[NeedTag::Thirst],
            Sleep(_)         => &[NeedTag::Fatigue],
            Rest(_)          => &[NeedTag::Fatigue],  // 休息缓解疲劳,但不如睡眠

            // === 元素平衡 ===
            VentElements(_)  => &[NeedTag::ElementBalance],

            // === 社交 ===
            Socialize(_)     => &[NeedTag::Social],
            VisitTavern(_)   => &[NeedTag::Hunger, NeedTag::Thirst, NeedTag::Social],
            AttendFestival(_)=> &[NeedTag::Social, NeedTag::Libido],
            VisitMarket(_)   => &[NeedTag::Social],         // 逛市场有社交成分(碰到熟人)

            // === 性/libido ===
            SeekPartner(_)   => &[NeedTag::Libido, NeedTag::Social],
            Groom(_)         => &[NeedTag::Libido],          // 梳妆打扮
            AttendGathering(_) => &[NeedTag::Social, NeedTag::Libido],

            // === 无需求驱动的 ===
            Work(_)          => &[],
            Travel(_)        => &[],
            TrainSkill(_)    => &[],
            Explore(_)       => &[],
            // ...
        }
    }
}
```

### 3.4 多需求聚合的意义

"去酒馆吃饭" = Eat + Drink + Social 三个需求同时满足 → `avg_urgency` 三个需求的平均。

- NPC 同时饿+渴+孤独 → 酒馆行动的权重极高 → 自然涌现"矿工下班必去酒馆"
- NPC 只饿不孤独 → 酒馆权重和在家吃差不多 → 看心情选

这就是涌现：不需要写 "if 矿工 then 去酒馆"——是需求聚合算出来的。

### 4.2 新增 3 行

```rust
fn apply_physiological_pull(
    emotion: &mut EmotionState,
    phys: &Physiology,
    social_deficit: f32,  // ★ 新增参数
) {
    // === 已有, 不改 ===
    if phys.hunger < 0.4 {
        emotion.pleasure -= 0.001 * (0.4 - phys.hunger);
    }
    if phys.fatigue > 0.7 {
        emotion.arousal -= 0.001 * phys.fatigue;
    }
    if phys.health < 0.5 {
        emotion.control -= 0.001 * (0.5 - phys.health);
    }

    // === ★ 新增 ===
    // 元素不平衡 → 微弱不适
    if phys.element_balance_urgency > 0.5 {
        emotion.pleasure -= 0.0005 * (phys.element_balance_urgency - 0.5);
    }
    // 高 libido → 轻度焦躁（arousal 上升）
    if phys.libido > 0.7 {
        emotion.arousal += 0.001 * (phys.libido - 0.7);
    }
    // 孤独 → 快乐下降
    if social_deficit > 0.5 {
        emotion.pleasure -= 0.001 * social_deficit;
    }
}
```

系数极低——数值经过数十游戏日的累积才会产生可感知的情绪变化。这是**长期背景色调**，不是"几分钟不社交就抑郁"。

## 五、L1-L4 分层更新策略

| 层级 | NPC 数量 | 决策频率 | 需求更新策略 |
|------|---------|---------|------------|
| **L1** | ≤1,000 | 每 0.2s | 完整计算：所有 7 个需求的 urgency + need_action_match |
| **L2** | ≤10,000 | 每 1-2s | 简化：仅 hunger/thirst/fatigue 完整；element_surplus/libido/social 用上次缓存值（每游戏日刷新一次） |
| **L3** | ~89,000 | 每日/每周 | 仅统计层面：长期饥饿→人口统计消耗；元素平衡/libido/social 不影响 L3 统计 |
| **L4** | 数百万 | 每周/每月 | 无个体数据——遥远城市有"食物价格"和"人口变动"的统计关联 |
