# 005 — 跨层桥接机制

## 一、为什么需要桥接？

三层需求模型在架构上是独立的，但心理上不是。脱离现实的后果：

| 没有桥接的情况 | 为什么荒谬 |
|---------------|-----------|
| 濒死的 NPC 在写诗 | 生存危机不抑制高层需求 |
| 连续失败 100 天的学徒精神饱满 | 成长受挫不影响心理状态 |
| 饥饿不影响社交选择 | 生理状态与心理需求完全隔离 |
| 长期孤独不影响进食行为 | 心理需求不反馈到生理行为 |

桥接机制的目的不是建立硬性规则（"饥饿时禁止自我实现"），而是**让心理真实的动态从数学中涌现**。

## 二、桥接 1：生存抑制 (Survival Suppression)

### 2.1 数学选择：为什么是 sigmoid？

比较三种可能的衰减函数：

```
线性:    suppression = 1.0 - max_urgency
  → max_urgency=0.5 → suppression=0.5
  问题: 中度饥饿就砍掉一半高层需求——太剧烈

硬阈值:  suppression = if max_urgency > 0.7 { 0.2 } else { 1.0 }
  → 0.69→1.0, 0.71→0.2
  问题: 二值化违背涌现原则——0.01的差距产生5倍的行为差异

Sigmoid: suppression = 1/(1+e^(-k(x-x0)))
  → 在拐点附近平滑过渡，远端平坦
  优势: 正常状态几乎无抑制，危机状态急剧抑制，中间平滑衔接
```

### 2.2 公式

```
survival_suppression(physiology) = 1.0 / (1.0 + e^(-10.0 × (max_urgency - 0.7)))

其中:
    max_urgency = max(
        1.0 - physiology.hunger,    // hunger deviation
        1.0 - physiology.thirst,    // thirst deviation
        physiology.fatigue,          // fatigue (0 already = energetic)
        1.0 - physiology.health,     // health deviation
    )

参数:
    k = 10.0  [TUNING] — 斜率。越大过渡越尖锐
    x0 = 0.7  [TUNING] — 拐点。max_urgency 在此处 suppression=0.5
```

### 2.3 数值表现

| max_urgency | suppression | 含义 |
|-------------|-------------|------|
| 0.0 | 0.999 | 完美状态——几乎无抑制 |
| 0.2 | 0.993 | 轻微饥饿——无影响 |
| 0.3 | 0.982 | GOAP-threshold(hunger)——概率引擎继续运行 |
| 0.4 | 0.953 | 中度饥饿——高层需求轻微受影响 |
| 0.5 | 0.881 | 明显饥饿——高层需求开始让步 |
| 0.6 | 0.731 | 严重饥饿——高层需求显著减弱 |
| **0.7** | **0.500** | **拐点——高层需求减半** |
| 0.8 | 0.269 | 非常严重——高层需求大幅抑制 |
| 0.9 | 0.119 | 危险状态——几乎只考虑基本需求 |
| 0.95 | 0.076 | 濒死——只有生存本能 |

### 2.4 为什么 survival_suppression 只影响概率引擎？

GOAP 是紧急响应系统——它在 survival 阈值触发时**硬接管**行为选择。survival_suppression 只作用于概率引擎的权重链。

当 GOAP 触发时（如 hunger < 0.3），行为选择已经完全由 GOAP 控制——概率引擎不再主导行为。survival_suppression 在此时几乎没有影响（因为概率引擎已经退居二线）。

但当 survival 状态在 GOAP 阈值**以上**时（如 hunger = 0.35，不足够触发 GOAP），survival_suppression 确保高层需求的权重被**温和但明确地**压低。

这就是"软抑制"的含义：GOAP 是硬安全网，survival_suppression 是软调控。

### 2.5 权重链中的位置

```
need_action_match(9维)  → [1.0, 2.0]  ← 基本/心理需求 urgency
    ×
intrinsic_motivation     → [1.0, 3.0]  ← 成长驱力
    ×
survival_suppression     → [0.08, 1.0] ← 在这里：统一衰减
    ×
time/weather/vehicle     → ...         ← 环境调制
```

放在 need_action_match 和 intrinsic 之后、环境调制之前：
- 先算出"你想做什么"（需求+志向）
- 再用生存状态统一衰减
- 最后用环境（时间/天气/载具）调整

这个顺序保证了：一个创作欲极强的 NPC（intrinsic=3.0）在 medium 饥饿时（suppression=0.73）仍然可能去创作——但概率大幅降低。**这正是涌现。**

## 三、桥接 2：挫折回归 (Frustration-Regression)

### 3.1 理论基础：ERG 的挫折-回归假说

Alderfer 的 ERG 理论认为：
- 当**成长需求**（Growth）持续受挫时，人会向下回归到**关系需求**（Relatedness）寻求补偿
- 这不是倒退——这是**心理自我保护**
- 回归的强度与挫折的持续时间和严重程度成正比

在 WoWorld 中的体现：
- IntrinsicGoal 长期进展缓慢 → "我是不是不够好？"
- 转而寻求**社交认可**（esteem）和**社交连接**（social）作为补偿
- 或通过**舒适行为**（吃/性）来缓解心理不适

### 3.2 触发条件

```
if active_intrinsic_goals.is_empty():
    return  // 没有内在目标 = 没有挫折来源

avg_frustration = mean(1.0 - goal.progress for goal in active_intrinsic_goals)

if avg_frustration < 0.6:
    return  // 进展还算正常，不需要回归

// 触发回归
regression_intensity = (avg_frustration - 0.6) / 0.4  // 归一化: 0.0 → 1.0
```

### 3.3 敏感度调制

```rust
fn apply_frustration_regression(
    avg_frustration: f32,
    sensitivities: &mut NeedSensitivity,  // 临时副本，不持久化
) {
    let r = ((avg_frustration - 0.6) / 0.4).clamp(0.0, 1.0);

    // 社交补偿 — 最强的回归路径
    sensitivities.social  *= 1.0 + r * 0.50;   // +50% max
    // "我的创作不顺利，我需要朋友"

    // 认可替代 — 用外部认可填补内在成就感的缺失
    sensitivities.esteem  *= 1.0 + r * 0.40;   // +40% max
    // "虽然我成不了大师，但至少我可以被尊重"

    // 舒适进食 — 经典的情绪调节
    sensitivities.hunger  *= 1.0 + r * 0.30;   // +30% max
    // "化悲痛为食欲"

    // 性作为情绪出口
    sensitivities.libido  *= 1.0 + r * 0.25;   // +25% max

    // 以下不变：
    // thirst — 不是舒适型需求，不受挫折回归影响
    // fatigue — 受情绪影响但不由挫折直接调制
    // element_balance — 纯粹生理
    // competence — 已经在计算 frustration，不自指
}
```

### 3.4 为什么这些特定的放大系数？

| 需求 | 放大 | 理由 |
|------|------|------|
| social | +50% | ERG 核心路径——关系需求是成长受挫后最自然的回归方向 |
| esteem | +40% | 认可可以替代成就感（"至少人们知道我很努力"） |
| hunger | +30% | 情绪性进食是跨文化的心理调节行为 |
| libido | +25% | 性作为情绪出口——幅度最低因为 libido 受生物因素强约束 |

### 3.5 临时性

```
重要：敏感度调制是临时的 — 每次决策重新计算，不持久化到 NpcData。
这与其他 modifier（如 weather_modifier）一致 —
都是"当前状态"的反映，不是"永久改变"。
```

这意味着：
- 当 intrinsic goal 的 progress 改善 → avg_frustration ↓ → regression 自然解除
- 当 intrinsic goal 被放弃 → 无活跃 goal → regression 不再触发
- NPC 的行为变化是**情境性的**，不是永久性的"人格改变"

## 四、双桥接的协同效果

### 4.1 情景推演：同时触发双桥接

```
NPC 状态:
  - hunger=0.25 (严重饥饿, dev=0.75)
  - 3个 intrinsic goals, avg_progress=0.2 (avg_frustration=0.8)
  
survival_suppression(0.75) = 0.378
  → 高层需求权重 × 0.378

frustration_regression(0.8):
  r = (0.8-0.6)/0.4 = 0.5
  social_sensitivity *= 1.25
  esteem_sensitivity *= 1.20
  hunger_sensitivity *= 1.15
  libido_sensitivity *= 1.125

效果:
  1. 生存抑制大幅降低高层需求权重
  2. 但挫折回归提升了基本需求敏感度
  3. 结果是: NPC 的 hunger 行为权重被生存抑制+敏感度提升双重放大
     → 优先找食物，但吃的时候更"讲究"（舒适进食）
     → 吃完后仍然有较强的社交驱力（挫折回归提升了 social sensitivity）
```

### 4.2 涌现的故事

这些不是脚本写出的故事，而是从数学中涌现的行为模式：

- **中年危机**：stagnation_sense↑ + competence_frustration↑ + LifeTheme→Crisis → frustration_regression 触发 → 突然买奢侈品（esteem↑）/去酒馆酗酒（social↑+libido↑）
- **绝境中的坚持**：survival_suppression→0.1 + intrinsic→3.0 × 0.1 = 0.3——创作欲被极度压制但依然不为零 → "即使在死亡边缘，我仍然在构思我的诗"
- **舒适创作**：survival_suppression→0.95 + intrinsic→1.5 → NPC 吃饱喝足后自然地转向创作 → "衣食足而知荣辱"

## 五、参数校准

所有参数标记为 [TUNING]：

| 参数 | 默认值 | 调高效果 | 调低效果 |
|------|--------|---------|---------|
| `survival_k` | 10.0 | 抑制过渡更尖锐（接近硬阈值） | 抑制过渡更平滑 |
| `survival_x0` | 0.7 | 更晚开始抑制（更宽容） | 更早开始抑制（更保守） |
| `regression_threshold` | 0.6 | 更难触发回归 | 更容易触发回归 |
| `social_regression_amp` | 0.50 | 受挫后社交需求更强烈 | 社交补偿更弱 |
| `esteem_regression_amp` | 0.40 | 更依赖认可补偿 | 认可补偿更弱 |
| `hunger_regression_amp` | 0.30 | 更多情绪性进食 | 较少 |
| `libido_regression_amp` | 0.25 | — | — |

## 六、开放问题

1. **是否应该记录"回归历史"？** — v2.0 可记录 NPC 经历过多少次挫折回归，用于 LifeChapter 叙事
2. **慢性回归是否有累积效应？** — 连续多年挫折回归是否会导致永久性敏感度改变？
3. **文化差异？** — 不同文化对挫折回归的"允许度"不同（如 stoic 文化抑制情绪性进食）
4. **survival_suppression 的非对称性？** — 生存危机对 intrinsic 的抑制是否应该强于对 social 的抑制？
