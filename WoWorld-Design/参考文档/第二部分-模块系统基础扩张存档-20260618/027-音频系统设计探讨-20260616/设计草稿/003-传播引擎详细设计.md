# 003-传播引擎详细设计（大纲）

> **开发代号**: WoWorld (Wonder World)
> **状态**: 设计草稿（大纲·要点）

---

## 一、核心职责

给定 SoundEvent + 听者位置 + 环境 → 回答：传到了多少、什么频率畸变、过了多久

## 二、传播模型组件

### 几何衰减
- 反平方律：SPL - 20×log₁₀(distance)
- 可配置 AttenuationModel（InverseSquare/WithAbsorption/Custom）

### 介质吸收
- 空气：高频衰减快（8kHz = 7.5dB/100m，125Hz = 0.03dB/100m）
- 水：低频几乎不衰减（鲸歌传百公里），高频瞬间消失
- 固体：介于中间

### 风矢量调制
- 顺风→传更远(×0.7)，逆风→传更近(×1.3)
- 湍流断续：强风+高频→声音断断续续

### 温度梯度
- 冷空气密度高→声波向冷侧弯曲
- 冬夜逆温层→声音困在地表传3-4倍远

### 障碍遮挡
- 三种算法可选（simple_sphere/raycast_sample/voxel_grid）
- 默认 4 条射线采样

### 多普勒效应
- Rust 透传 velocity → Godot AudioStreamPlayer3D 原生支持

### 介质边界
- 空气↔水：99.9%反射（30dB损失）
- 听者头部介质过滤：水下高频全砍

### 传播延迟
- delay = distance / speed_of_sound(medium, temperature)
- 涌现：钟声先见后闻/龙吼比动作晚/闪电→雷声

## 三、可听半径

传入 source_spl + freq + medium + weather → 二分查找衰减到 min_audible_spl 的距离
用于 L2/L3 声音的强度门控判定

## 四、性能

- 几何衰减 ~5ns/对
- 遮挡射线 ~200ns×4 仅近距离
- 总计 ~180µs L1 全量
