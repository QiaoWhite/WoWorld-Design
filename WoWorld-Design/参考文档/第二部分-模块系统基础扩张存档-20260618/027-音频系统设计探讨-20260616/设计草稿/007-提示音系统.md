# 007-提示音系统（大纲）

> **开发代号**: WoWorld (Wonder World)
> **状态**: 设计草稿（大纲·要点）

---

## 一、本质区别

- NPC 听不到，不进物理模拟，不受掩蔽/衰减影响
- 纯 2D 播放，始终清晰
- 玩家独立音量滑条

## 二、触发机制

- PlayerNotificationEvent 枚举（~30 变体）→ world_core 事件总线
- 各模块发射事件——不知道音频存在
- 音频模块订阅事件→查规则表→播放
- 大日志也订阅（写历史）+ UI 也订阅（浮动文字）

## 三、事件分类

- 战斗：ThreatDetected / SkillCooldownReady / EquipmentBroken / LowHealth
- 任务：QuestAccepted/Completed/Failed / ObjectiveUpdated
- 物品：ItemAcquired(rarity分级) / InventoryFull / CraftingComplete
- 生存：Hunger/Thirst/Temperature/Drowning 警告
- 技能：SkillLevelUp / SkillUnlocked
- 社交：NpcGreeting / LetterDelivered
- 经济：TradeComplete / AuctionWon / DebtDue
- 载具：VehicleDamaged / FuelLow / DestinationNear
- 天气：ExtremeWeatherWarning
- UI：MenuNavigate / SaveSuccess
- 其他：Achievement / Discovery / MusicTuning(乐器调音/乐谱评分)

## 四、规则表——notification_sounds.toml

- 事件→声音映射（同一事件按参数选不同音效，如ItemAcquired按Rarity）
- 优先级 / max_overlap / min_interval / volume
- loop_while_active（持续型如低血量心跳）
- pitch/volume 随 severity 自动变化

## 五、冷却/打盹

- min_interval（默认 200ms）
- 连续跳过 5 次→强制播放一次
- 音量闪避：0.3s 内密集提示→自动降低至 60%

## 六、音量控制——六滑条

- Master / World SFX / Voice / Music(Game) / Music(Player) / Notification / Ambient
