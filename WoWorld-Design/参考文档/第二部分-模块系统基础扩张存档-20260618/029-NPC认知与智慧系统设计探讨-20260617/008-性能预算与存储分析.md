# 008-性能预算与存储分析

## 待讨论的核心问题

### 1. 每NPC存储增量

| 字段 | 大小 | 持久化？ |
|------|------|---------|
| CognitiveStyle | 16B (4×f32) | LMDB |
| CognitiveTide | 16B (4×f32) | LMDB |
| MentalModel[] | ≤1.6KB (20×~80B) | LMDB |
| MindAttribution[] | ≤960B (16×~60B) | LMDB |
| BeliefHistory | ≤768B (16×~48B) | LMDB |
| cognitive_distress | 4B (f32) | LMDB |
| ReadingSession | ~200B | LMDB（仅读书中） |
| SleepState | ~64B | LMDB（仅睡觉中） |
| ThoughtFragment seeds | ≤200B (10×seed) | 不持久化 |

**总计**：<4KB/L1 NPC

对标已有存储（MemoryStore ~30KB, Knowledge ~2.5KB, AestheticFrameworks ~2KB），在预算内。

### 2. 每帧CPU增量

| 操作 | 频率 | 成本 | 1000L1帧均 |
|------|------|------|-----------|
| CognitiveTide更新 | 每决策周期 | 6次浮点乘加 | <0.001ms |
| CognitiveBiases惰性计算 | 每次调用 | ~15次浮点 | 惰性，几乎0 |
| ThoughtTrigger检查 | 惰性，被携带 | 大多数None | <0.005ms |
| ThoughtFragment生成 | 仅玩家15m内(~5 NPC) | 片段组合<0.02ms | <0.1ms |
| MentalModel权重调制 | 每决策周期 | ≤20次浮点/周期 | <0.005ms |
| EmbodiedCognition | 每决策周期 | 从已有数据派生 | <0.001ms |

**帧均增量：<0.15ms**。对标感官系统管线约1ms，在预算内。

### 3. 每游戏日惰性加工

| 操作 | 频率 | 成本 | 日均总 |
|------|------|------|--------|
| micro_digest | ~3次/NPC/天 | scan≤20模型×5次比较 | ~100K浮点 |
| macro_digest | 每7天 | 全模型扫描+融合检测 | ~200K浮点/7天 |
| CognitiveStyle重派生 | 每7天 | ~12次浮点+4次sigmoid | ~12K/7天 |
| 睡眠认知处理 | 1次/天 | ~0.1ms/NPC | ~100ms/天(分摊=0ms) |

**日均增量：分摊后≈0**。

### 4. LOD区分策略

| LOD | 运行内容 | 存储内容 |
|-----|---------|---------|
| L1（全模拟） | 全部认知功能 | 全部字段 |
| L2（降频） | CognitiveTide + 低频MentalModel归纳 | 全文存储，L1 promote时补算消化 |
| L3（统计抽象） | 不跑个体 | 不存（个体不存在） |
| L4（纯统计） | 不跑个体 | 不存 |

### 5. 惰性快照

- SettlementWisdomAggregate：对标已有SettlementFaithSnapshot
- 每游戏日惰性聚合，仅dirty settlement
- 存储成本<100B/settlement

### 6. LMDB分区策略

- 对标已有MemoryPartition（热/重要/冷）
- 认知数据属于"热"区——决策周期内频繁访问
- 序列化使用bincode（对标已有保存系统NPC ver2.0 §6.3）

### 7. 存档大小核算

- 100K NPC × 4KB = ~400MB 认知数据
- 对比已有：MemoryStore 30KB×100K=3GB，Knowledge 2.5KB×100K=250MB
- 认知数据在总NPC存档中占比~8%
