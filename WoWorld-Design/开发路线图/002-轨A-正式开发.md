## A.3 阶段三：woworld_spatial 空间索引

**预计**: 3-5 天

### 实现方案

| trait | 实现策略 | 复杂度 |
|-------|---------|--------|
| `EntityIndex` | 稀疏网格 + HashMap<ChunkKey, Vec> | 中 |
| `VisibilityQuery` | DDA（3D Digital Differential Analyzer）射线 | 低 |
| `SpatialEventBus` | 环形缓冲区 + 时间窗口过滤 | 低 |

### EntityIndex 核心结构

```rust
pub struct GridEntityIndex {
    // 每个 Chunk（32m×32m×32m）中的实体列表
    chunks: HashMap<ChunkKey, Vec<EntityId>>,
    // 实体详细数据
    entities: HashMap<EntityId, SpatialEntity>,
    // 脏标记（用于局部更新）
    dirty_chunks: HashSet<ChunkKey>,
}
```

### VisibilityQuery 核心逻辑

DDA 步进算法：
1. 从 `origin` 沿 `direction` 以固定步长（0.5m = 体素尺寸）推进
2. 每步调用 `TerrainQuery::density_at()` 检查是否命中固体
3. 命中则返回 `TerrainHit`，超过 `max_dist` 返回 `None`
4. 性能目标：~10μs/射线

## A.4 阶段四：world_gen 世界生成 v0.1

**预计**: 5-8 天

### A.4a 双层噪声地形

**噪声层**：

| 层 | 频率 | 用途 | 算法 |
|----|------|------|------|
| 大陆形状 | ~100km 波长 | 海陆分布 | 大尺度 Perlin + 阈值 |
| 地形细节 | ~100m 波长 | 山脉/丘陵/平原 | 多倍频 Perlin（1-6 octaves） |

**海洋掩码**：
- 海:陆 ≈ 7:3（可调参数）
- 大陆噪声 > `sea_threshold` → 陆地，否则 → 海洋
- 过渡带：海滩/浅水区（基于高度插值）

**高度范围**：
- 最低点：-200m（浅海床）
- 海平面：0m
- 最高点：~700m（山脉顶峰）
- 使用 `f64` 世界坐标避免大坐标的浮点精度问题

### A.4b 垂直稀疏 Chunk

```
Chunk 规格:
- 水平: 32×32 voxels（16m×16m @ 0.5m/voxel）
- 垂直: 可变（由局部地形高度决定，最低 1，最高 ~100）
- 存储: 3D 密度场数组（u8，0=空气 255=固体）
```

**Transvoxel 网格生成**（Rust 侧）：
1. 遍历 Chunk 内所有体素边
2. 查找密度场过零点（Marching Cubes 变体）
3. 生成顶点 + 法线 + 索引缓冲区
4. → `PackedByteArray` 传输到 Godot
5. Godot 侧：`ArrayMesh` 创建 + `MeshInstance3D` 渲染

### A.4c 简单群系

**5 种基础群系**：

| 群系 | 温度范围 | 降水范围 | 地表材质 |
|------|---------|---------|----------|
| 草原 | 0.3-0.7 | 0.2-0.6 | Grass + 稀疏 Dirt |
| 森林 | 0.3-0.7 | 0.6-0.9 | Moss + LeafLitter |
| 沙漠 | 0.7-1.0 | 0.0-0.2 | Sand + 稀疏 Rock |
| 雪原 | 0.0-0.3 | 0.0-0.5 | Snow + Ice |
| 沼泽 | 0.3-0.7 | 0.7-1.0 | Mud + Moss |

**群系映射**：温度 + 降水二维噪声 → 最近群系中心（Voronoi 或硬分类）

**TOML 配置** (`assets/biomes.toml`)：
```toml
[biomes.grassland]
temp_range = [0.3, 0.7]
precip_range = [0.2, 0.6]
surface = "Grass"
```

### 性能预算（原型阶段）

| 指标 | 目标 | 说明 |
|------|------|------|
| 地形生成 | <5s 初始世界 | ~25万 km² → 按需 Chunk 生成，非全量 |
| Chunk 网格生成 | <2ms/Chunk | Transvoxel + 并行 |
| 内存 | <2GB | 视距 500m，~200 Chunks 常驻 |
| 视野内 Chunk 数 | ~200 | 500m 视距 / 16m Chunk ≈ 31²×π ≈ 3,000 → LOD 后 ~200 |
