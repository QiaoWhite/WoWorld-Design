## 这是什么？

本文件夹是 WoWorld 从**纯设计阶段**过渡到**设计+开发并行阶段**的路线图。它回答了：在已经有了 ~107,000 行设计规格、24 个完成模块的当下，如何同时推进写代码、补文档、修问题、想创意。

## 四轨总览

```
轨 A: 正式开发 ── 从零搭建 Rust + Godot 项目，实现世界生成 + 玩家移动
轨 B: 文档补全 ── 优先补 UI/UX·玩家扩展·名声·法律分析，其余按优先级排队
轨 C: 问题修复 ── 孤儿接口·CHG-047延期·性能预算·模块保鲜
轨 D: 创意探索 ── 属性注册表·冒险小队·Storyteller·魔法场论
```

**并行策略**：四轨可同时推进。每轮对话聚焦 1-2 轨，按优先级交替推进。

## 关键决策

| 决策 | 依据 |
|------|------|
| **不追求设计"冻结"再写代码** | 编码会反向暴露设计缺陷。设计→编码→反馈→修正，循环推进 |
| Rust workspace monorepo 结构 | 所有 crate 共享 trait 定义，单一 Cargo.lock 简化依赖管理 |
| 先做 core + spatial + world_gen | 最小可行原型依赖链：一切从地形开始 |
| 只实现 TerrainQuery，不做 OceanProvider | Phase 1 不需要海洋物理——海面用平面+shader 作弊 |
| 暂不接入 LMDB | 第一个原型不需要存档。SaveableModule trait 已定义，后期插入 |
| UI/UX 文档优先于 UI 代码 | 先理清信息架构，再写 Godot Control 节点——避免返工 |
