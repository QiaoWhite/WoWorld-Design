# 002 — 信号维度详细设计

## 一、AestheticSignal 完整定义

```rust
/// 任何可被审美感知的事物的客观美学特征向量
/// Copy 类型，24 bytes（6 × f32），栈上传递，零分配
/// 
/// 审美模块只定义 struct——各实体所属模块在自己的 impl HasAestheticSignal 中计算
#[derive(Copy, Clone, Default)]
pub struct AestheticSignal {
    pub fluency: f32,        // [0, 1] 加工流畅度
    pub novelty: f32,        // [0, 1] 新颖度
    pub complexity: f32,     // [0, 1] 复杂度
    pub harmony: f32,        // [0, 1] 和谐度
    pub expressiveness: f32, // [0, 1] 情感表现力
    pub virtuosity: f32,     // [0, 1] 技艺可见度
}
```

## 二、逐维度详细分析

### 2.1 fluency — 加工流畅度

**定义**：感知者处理该对象信息时的"省力"程度。高 fluency 意味着对称、清晰、符合原型——大脑不需要费力就能理解。

**神经对应**：感知流畅性（processing fluency）→ 眶额皮层正向激活

**组成因子**：

| 子因子 | 权重 | 说明 |
|--------|------|------|
| 对称性 | 0.30 | 左右/上下对称程度。完全对称→1.0 |
| 清晰度 | 0.25 | 轮廓/边界/信号-噪声比的清晰程度 |
| 原型符合度 | 0.25 | 与记忆中该类事物的典型样本的匹配程度 |
| 可预测性 | 0.20 | 下一笔/下一个音符/下一个动作的可预测性 |

**不同对象类型的典型 fluency**：

| 对象 | high fluency | low fluency |
|------|-------------|-------------|
| 面部 | 对称的脸 | 不对称/变形的脸 |
| 建筑 | 古典对称建筑 | 有机/不规则建筑 |
| 音乐 | 简单旋律 | 无调性音乐 |
| 动作 | 行云流水的舞姿 | 磕磕绊绊的动作 |
| 战斗 | 流畅的连招 | 生硬的挥砍 |

**跨感官适用性**：
- 视觉：对称+清晰轮廓+熟悉构图 → 高 fluency
- 听觉：流畅旋律+清晰节拍+熟悉和声进行 → 高 fluency
- 触觉：光滑均匀的材质 → 高 fluency
- 嗅觉：纯净不刺鼻的气味 → 高 fluency

### 2.2 novelty — 新颖度

**定义**：对象偏离原型/预期/熟悉模式的程度。高 novelty 意味着"从没见过这样的"——触发好奇心。

**神经对应**：多巴胺能预测误差（reward prediction error）→ 腹侧被盖区（VTA）阶段性放电

**组成因子**：

| 子因子 | 权重 | 说明 |
|--------|------|------|
| 原型偏离度 | 0.40 | 偏离同类事物典型样本的程度 |
| 元素组合意外性 | 0.35 | 将不常在一起出现的元素组合在一起 |
| 统计稀有度 | 0.25 | 在世界上该类型的客观稀有程度 |

**注意**：novelty 不是"好不好"——它对 valence 是倒 U 型曲线。太新 → 困惑（低 valence），适度新 → 有趣（高 valence），完全不新 → 无聊（低 interest）。

### 2.3 complexity — 复杂度

**定义**：对象的信息密度、细节层次、结构层次。高 complexity 意味着有"很多可以看/听/感受的东西"。

**神经对应**：前扣带回（ACC）+ 背外侧前额叶（dlPFC）认知负荷

**组成因子**：

| 子因子 | 权重 | 说明 |
|--------|------|------|
| 元素数量 | 0.20 | 视觉元素/声部/层次的绝对数量 |
| 结构层次深度 | 0.30 | 嵌套/层级/组织结构的深度 |
| 信息熵 | 0.25 | 信息论意义上的不可压缩度 |
| 细节密度 | 0.25 | 单位空间/时间内的细节量 |

**关键**：complexity ≠ 混乱。高 complexity + 高 harmony = 精妙复杂的作品（赋格曲/哥特教堂）。高 complexity + 低 harmony = 混乱（噪音/涂鸦）。

### 2.4 harmony — 和谐度

**定义**：元素间的结构协调性——比例、色彩、节奏、空间的"合在一起感觉对"的程度。

**神经对应**：听觉皮层（和弦协和性）+ 视觉皮层（色彩和谐）+ 眶额皮层（整体的"对"感）

**组成因子**：

| 子因子 | 权重 | 说明 |
|--------|------|------|
| 比例协调 | 0.30 | 黄金分割/整数比/视觉重量平衡 |
| 色彩/音色协调 | 0.25 | 互补色/类似色/音色匹配 |
| 节奏协调 | 0.25 | 时间结构的规律性和韵律感 |
| 空间协调 | 0.20 | 元素的空间/声场分布平衡 |

**跨感官统一性**：harmony 是唯一在所有感官上都以相似方式运作的维度——"协调"在视觉（色彩和谐）、听觉（和弦协和）、嗅觉（香气融合）、触觉（材质触感平衡）中有共同的数学基础（比例/频率比/强度比）。

### 2.5 expressiveness — 情感表现力

**定义**：对象承载和传达情感信号的程度。高 expressiveness 意味着"你能感觉到创作者/表演者的情感"。

**神经对应**：镜像神经元系统 + 岛叶（interoception）+ 杏仁核（情感检测）

**组成因子**：

| 子因子 | 权重 | 说明 |
|--------|------|------|
| 情感清晰度 | 0.35 | 传达的情感是否明确（悲伤就是悲伤，不模糊） |
| 情感强度 | 0.35 | 情感信号的幅度/浓度 |
| 创作者可感知性 | 0.30 | 能否"感觉到创作者的存在"——看到笔触/听到呼吸/感受到用心 |

**特殊性**：expressiveness 是所有维度中与创作意图关联最强的。一件功能性物品（一个普通杯子）的 expressiveness 极低（"这只是一个杯子"），而一件有意表达情感的作品（一首挽歌）的 expressiveness 极高。

**但注意**：expressiveness 高的对象不一定"美"——一首充满愤怒的嘶吼歌曲 expressiveness 极高，但 harmony 可能很低 → valence 可能是负的 → "有力但不愉悦"。

### 2.6 virtuosity — 技艺可见度

**定义**：制作/执行难度和技巧的外显程度。高 virtuosity 意味着"一看就是高手做的"。

**神经对应**：内侧前额叶（社会认知）+ 前运动皮层（动作模拟——"我能做到吗？"）

**组成因子**：

| 子因子 | 权重 | 说明 |
|--------|------|------|
| 难度可见性 | 0.35 | 制作/执行的难度有多容易被感知 |
| 精确度 | 0.30 | 执行的精密度——线条直不直、音准不准、边缘齐不齐 |
| 稀有技能标志 | 0.20 | 是否使用了罕见的技术/材料/手法 |
| 完成度 | 0.15 | 作品的完成程度——抛光/润色/收尾 |

**关键区分**：

- 过程的 virtuosity = f(skill_level, 动作流畅度, 节奏精准度) → 看的是"做的人有多厉害"
- 产物的 virtuosity = f(Quality, Rarity, 精确度, 完成度) → 看的是"做出来的东西有多精良"

两者通常正相关，但可以分离：大师随手打一根钉子 → 过程 virtuosity=0.95, 产物 virtuosity=0.05。

## 三、派生属性（不在 Signal 中存储，由 judge() 内部计算）

### 3.1 somatic_impact — 身体冲击度

```rust
fn somatic_impact(signal: &AestheticSignal, stakes: &StakesContext) -> f32 {
    let base = signal.novelty * signal.expressiveness;
    let stakes_amplification = 1.0 + stakes.mortality_salience * 0.5;
    (base * stakes_amplification).min(1.0)
}
```

又新又动情 → 身体被"击中"。如果还牵涉生死（stakes 高），冲击更强。

触发阈值：`somatic_impact > 0.35` → "被美击中的瞬间"（心跳漏一拍/脊背发凉/起鸡皮疙瘩）→ 记忆标记为高情感权重。

### 3.2 symbolic_depth — 象征深度

```rust
fn symbolic_depth(signal: &AestheticSignal, context: &AestheticContext) -> f32 {
    let base = signal.complexity * context.prior_expectation;
    let familiarity_mod = 1.0 - context.familiarity * 0.5; // 熟悉降低象征神秘感
    (base * familiarity_mod).min(1.0)
}
```

complexity 提供了"可以被解读"的土壤，prior_expectation 提供了"应该去解读"的文化框架。

### 3.3 cultural_fit — 文化契合度

```rust
fn cultural_fit(signal: &AestheticSignal, taste: &AestheticTaste) -> f32 {
    // 对象的信号与主体taste的维度权重是否同向
    let mut fit = 0.0;
    for dim in 0..6 {
        fit += signal[dim] * taste.dimension_weights[dim];
    }
    (fit / 6.0).clamp(0.0, 1.0)
}
```

用于判断是否"冒犯"——高 expressiveness + 高 novelty + 极低 cultural_fit → 可能是文化冲突。

## 四、跨感官的维度适用性

| 维度 | 视觉 | 听觉 | 触觉 | 嗅觉/味觉 |
|------|------|------|------|----------|
| fluency | ✅ 对称/清晰 | ✅ 旋律流畅 | ✅ 光滑/均匀 | ✅ 纯净/不刺鼻 |
| novelty | ✅ 新风格 | ✅ 新音色 | ✅ 没摸过的质感 | ✅ 没闻过的香料 |
| complexity | ✅ 细节层次 | ✅ 声部层次 | ✅ 纹理层次 | ✅ 复合香气 |
| harmony | ✅ 色彩协调 | ✅ 和弦协和 | ✅ 冷/暖/硬/软平衡 | ✅ 气味融合度 |
| expressiveness | ✅ 笔触/色彩情感 | ✅ 旋律情感 | ⚠️ 触觉情感承载弱 | ⚠️ 嗅觉情感承载弱 |
| virtuosity | ✅ 可见技艺 | ✅ 演奏技巧 | ✅ 触感反映手艺 | ⚠️ 调香技艺不可见 |

> ⚠️ 标记的维度在特定感官上自然权重偏低。这不需要不同感官的独立 Signal 结构——AestheticTaste 的维度权重会由文化和个性自然调节。一个香料商文化的 NPC 自然会对嗅觉的 complexity 和 harmony 权重更高。

## 五、Signal 的值域和分布假设

所有 6 维 ∈ [0, 1]。但实际分布不是均匀的：

| 维度 | 典型分布 | 说明 |
|------|---------|------|
| fluency | 聚集在 0.4-0.8 | 极低和极高都罕见（杰作 or 完全混乱） |
| novelty | 偏态（大多数日常物品 <0.2） | 真正新颖的东西是稀有的 |
| complexity | 取决于对象类型 | 建筑 0.5-0.9, 杯子 0.05-0.2 |
| harmony | 聚集在 0.3-0.8 | 自然物体>人造物体 |
| expressiveness | 偏态（大多数 <0.3） | 除非创作者有意表达 |
| virtuosity | 取决于创作者技能 | 学徒 0.1-0.3, 大师 0.7-0.95 |
