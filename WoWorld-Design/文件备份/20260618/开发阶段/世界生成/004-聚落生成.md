## 一、总览

聚落生成涵盖 P4（选址）和 P5（内部布局）。v3 核心变化：
- **承载容量先于选址**——先算能养活多少人，再决定是否建聚落
- **港口城市**——70% 海洋 → 港口是主要聚落类型之一
- **游牧/部落聚落**——不适用固定四级链
- **资源禀赋→经济关系自动推导**——依赖方向双向，由禀赋决定不论等级

## 二、聚落等级体系

### 2.1 四级固定聚落链

从低到高，高级聚落包含低级聚落的全部功能并增加额外功能。

| 属性 | 村庄 | 小镇 | 城市 | 城堡 |
|------|------|------|------|------|
| **人口** | 由承载容量决定（非硬编码范围） | ← | ← | 500-5,000 + 驻军 |
| **核心目的** | 食物与基础原材料生产 | 资源精加工、特种贸易 | 区域统治、贸易加工、军事防御 | 军事控制、要地防御 |
| **资源需求** | 淡水+可耕地/猎场/渔场+建材 | 核心产业资源+运输路线 | 防御纵深+贸易路线交汇 | 防御潜力第一、扼守要地 |
| **生成概率** | 60% | 25% | 10% | 5% |
| **水资源** | 井/溪流 | 河流/湖泊 | 河流 | 河流+护城河水源 |

> **重要变更**：人口不再硬编码（原 20-200、200-2000 等），改为由承载容量模型计算上限后乘以等级系数。

### 2.2 港口城市（新增独立子类型）

港口城市是"城市"等级的特殊子类，有独特的生成逻辑：

| 属性 | 港口城市 |
|------|---------|
| **选址条件** | 海岸线（沉积/峡湾/构造） + 可航行海域 + 内陆腹地资源 |
| **核心功能区** | 码头区、仓库区、海关、灯塔、造船厂、渔市 |
| **路网形态** | 半圆形沿海布局（线性变体）——沿海岸线延伸 |
| **海防体系** | 防波堤、海墙（替代陆墙）、灯塔/瞭望塔、封锁链 |
| **经济特点** | 跨海贸易+本地渔业双驱动——承载力不完全依赖周边农业 |

```rust
struct PortSettlement {
    base: SettlementData,
    // 港口特有属性
    harbor_size: HarborSize,
    dock_capacity: u32,          // 可同时停靠船只数
    sea_defense_level: u8,       // 0-4 海防等级
    trade_route_count: u32,      // 连接的海上航线数
}

enum HarborSize {
    FishingVillage,   // 小渔港——几个简易码头
    TradingPort,      // 贸易港——多个码头+仓库区
    NavalBase,        // 军港——要塞化+大型干船坞
    GrandHarbor,      // 大港——以上全部+海关+灯塔+市场
}
```

### 2.3 游牧/部落聚落（新增类型）

不适用固定四级链——有完全不同的空间组织方式。

| 特征 | 固定聚落 | 游牧聚落 |
|------|---------|---------|
| **空间组织** | 固定建筑群 | 移动营地——随季节迁徙 |
| **选址逻辑** | 一次决定，永久 | 标记"迁徙走廊"和"季节性草场" |
| **建筑类型** | 固定建筑（木/石） | 可拆装结构（帐篷/穹庐/篷车） |
| **承载容量** | 周边农田产出 | 草场载畜量 × 迁徙周期 |
| **社会结构** | 四级等级链 | 氏族/部落——血缘+军事领袖 |
| **政治组织** | 政体明确 | 部落联盟/大汗国/酋邦 |
| **与固定聚落的关系** | — | 贸易（畜产品↔谷物/工具）+ 可能的冲突（争夺草场/水源） |

```rust
struct NomadicGroup {

}
```

## 三、承载容量模型（新增）

### 3.1 模型

承载容量在选址之前独立计算，决定每个位置的**理论上限人口**。

```rust
struct CarryingCapacity;

impl CarryingCapacity {
    /// 计算指定位置的最大可承载人口
    fn population_cap(&self, pos: Vec2, terrain: &TerrainBase) -> u32 {
        let food_cap = self.food_capacity(pos, terrain);
        let water_cap = self.water_capacity(pos, terrain);
        let material_cap = self.build_material_capacity(pos, terrain);
        let defense_cap = self.defense_capacity(pos, terrain);

        // 瓶颈原则——最稀缺的资源决定上限
        *[food_cap, water_cap, material_cap, defense_cap]
            .iter()
            .min_by(|a, b| a.partial_cmp(b).unwrap())
            .unwrap()
    }

    /// 粮食承载 = 可耕地×亩产 + 猎场×猎物密度 + 渔场×渔获量
    fn food_capacity(&self, pos: Vec2, terrain: &TerrainBase) -> u32 {
        let arable = self.arable_land_in_radius(pos, 5.0);  // 5km 半径
        let hunting = self.hunting_yield_in_radius(pos, 10.0);
        let fishing = self.fishing_yield_in_radius(pos, terrain);

        // 每人每年需要约 300kg 粮食当量
        let annual_output_kg = arable * CROP_YIELD_PER_HA
                             + hunting * HUNTING_YIELD_PER_KM2
                             + fishing * FISHING_YIELD_PER_KM2;
        (annual_output_kg / 300.0) as u32
    }

    /// 水源承载 = 河流流量/井水补给率 ÷ 人均日用水
    fn water_capacity(&self, pos: Vec2, terrain: &TerrainBase) -> u32 { /* ... */ }

    /// 建材承载 = 周边木材/石材年可采量 ÷ 人均建材消耗
    fn build_material_capacity(&self, pos: Vec2, terrain: &TerrainBase) -> u32 { /* ... */ }

    /// 防御承载 = 地形防御值 × 城防体系能保护的最大范围
    fn defense_capacity(&self, pos: Vec2, terrain: &TerrainBase) -> u32 { /* ... */ }
}
```

### 3.2 承载容量与聚落等级

```
实际人口 = 承载容量上限 × 聚落等级系数 × 波动系数

聚落等级系数:
  村庄: 0.3 - 0.6   （实际人口远低于理论容量——留有余地）
  小镇: 0.5 - 0.8
  城市: 0.7 - 0.95  （接近容量上限——城市更"拥挤"）
  城堡: 0.2 - 0.5   （人口主要由驻军驱动，非农业承载）

波动系数: 由种子决定，范围 0.85 - 1.15
```

## 四、选址逻辑

### 4.1 资源达标判断

原方案的选址逻辑框架保留，但增加了承载容量作为前置条件：

```rust
fn can_place_settlement(pos: Vec2, terrain: &TerrainBase, resources: &ResourceMap) -> bool {
    let carrying = CarryingCapacity::population_cap(pos, terrain);

    // 至少能养活 50 人（最小村庄）
    if carrying < 50 { return false; }

    // 原有三项必要条件（保留）
    let has_water = nearest_water_distance(pos) <= 200.0 || has_groundwater(pos);
    let has_food = food_score(pos) >= MIN_FOOD_THRESHOLD;
    let has_material = build_material_score(pos) >= MIN_MATERIAL_THRESHOLD;

    if !has_water || !has_food || !has_material {
        // 尝试临近地貌补足（≤ 2km）
        return can_supplement_from_neighbor(pos, terrain, resources);
    }

    true
}
```

### 4.2 选址评分（保留+扩展）

```
score = water_score × 0.30    // 微调：水源从 0.35→0.30
      + food_score  × 0.30
      + build_score × 0.20
      + defense_score × 0.12
      + trade_score  × 0.08   // 新增：贸易潜力
```

**新增 `trade_score`**：位置在交通节点上的潜力——河流交汇/海岸线/两群域交界/天然港口。

### 4.3 港口选址（新增）

```rust
fn score_port_site(pos: Vec2, coast: &CoastSegment, ocean: &OceanSystem) -> f64 {
    let harbor_quality = match coast.coast_type {
        CoastType::Depositional => 0.7,  // 沙滩——需要人工港
        CoastType::Fjord => 1.0,          // 峡湾——天然深水港
        CoastType::Tectonic => 0.6,       // 悬崖——难以登陆
        CoastType::CoralReef => 0.4,      // 暗礁危险
        CoastType::Mangrove => 0.3,       // 浅水泥沼
    };

    let navigability = ocean.navigable_zones.at(pos);
    let inland_access = assess_hinterland_access(pos);

    harbor_quality * 0.50
        + navigability as f64 * 0.30
        + inland_access * 0.20
}
```

## 六、聚落群域 → 群落 → 政治实体

保留原方案的三级组织体系（逻辑框架不变，参数按 25 万 km² 重算）。

### 6.1 聚落群域（Settlement Cluster）

一定范围内集齐四级聚落所需全部资源时，将此区域作为一个整体。

| 规模 | 聚落数量 | 覆盖面积 |
|------|---------|---------|
| 小型群域 | 3-8 个 | ~15-50 km² |
| 中型群域 | 8-20 个 | ~50-200 km² |
| 大型群域 | 20-50 个 | ~200-800 km² |

### 6.2 群落（Cluster Confederation）

数个相邻聚落群域因地理邻近+贸易+文化同源+政治联盟组成。

- 相邻群域间距 ≤ 50km（25 万 km² 下增大——原 30km）
- 存在至少一条可通行道路连接
- 共享至少一种文化元素

### 6.3 政治实体（Polity）

## 七、资源禀赋→经济关系推导（新增）

### 7.1 核心原则

聚落间的经济关系不是手写的——从**资源禀赋差异**自动推导。依赖方向**双向、不论等级**。

```
禀赋重叠 → 竞争
禀赋互补 → 贸易
禀赋不足 → 依赖能补足的聚落（城堡依赖村庄粮食 = 正常）
```

### 7.2 推导算法

```rust
fn derive_economic_relations(
    settlements: &[SettlementData],
    resources: &ResourceMap,
    terrain: &TerrainBase,
) -> Vec<EconomicRelation> {
    let mut relations = Vec::new();

    for (i, a) in settlements.iter().enumerate() {
        for b in settlements.iter().skip(i + 1) {
            let distance = a.pos.distance(b.pos);
            if distance > MAX_ECONOMIC_RANGE { continue; }

            let a_endowment = resource_endowment(a, resources);
            let b_endowment = resource_endowment(b, resources);

            let overlap = a_endowment.overlap_score(&b_endowment);
            let complement = a_endowment.complement_score(&b_endowment);

            if complement > COMPLEMENT_THRESHOLD {
                // 禀赋互补 → 建立贸易关系
                relations.push(EconomicRelation {
                    from: a.id,
                    to: b.id,
                    relation_type: EconomicRelationType::Trade,
                    goods: identify_trade_goods(&a_endowment, &b_endowment),
                    volume: estimate_trade_volume(distance, complement),
                });
            } else if overlap > OVERLAP_THRESHOLD && distance < COMPETITION_RANGE {
                // 禀赋重叠+近距 → 竞争关系
                relations.push(EconomicRelation {
                    from: a.id,
                    to: b.id,
                    relation_type: EconomicRelationType::Competition,
                    contested_resource: identify_contested_resource(&a_endowment, &b_endowment),
                    ..Default::default()
                });
            }

            // 依赖方向：检查每个聚落的"不足"是否有邻居能补足
            // 方向由禀赋决定，与等级无关
            resolve_dependencies(a, b, &a_endowment, &b_endowment, &mut relations);
        }
    }

    relations
}

enum EconomicRelationType {
    Trade,           // 互补→贸易
    Competition,     // 重叠→竞争
    Dependency,      // A依赖B（提供某资源）
    OneWaySupply,    // 单向供养（如村庄→城堡粮食）
    MutualDependency,// 双向依赖
}
```

### 7.3 承载容量与资源关系的联动

```
如果一座城市的人口超过了承载容量：
  → 必须从周边聚落进口粮食（依赖关系增强）
  → 如果周边聚落也无法满足
    → 城市人口衰减（饥荒/移民）
    → 城市衰落或发动战争夺取新资源

聚落人口 ＞ 承载容量 → 依赖加深（或危机）
聚落人口 ＜ 承载容量 → 有出口盈余 → 贸易优势
```
