## 〇、总览

显著性引擎决定"此刻什么值得上传到工作记忆与GOAP规划器"。它不是对所有实体排绝对分——而是**异常检测**：过滤掉正常的背景，只让偏离期望的信号进入意识。对标音频系统的 `AttentionFilter`（音频声明"选择性注意属于NPC认知系统"——本引擎是实现）。

---

## 一、核心原则：背景/信号二分类

```
不对所有实体计算显著性 → 排序 → 取 Top N
而是：先分类 → 绝大多数标记为Background → 跳过 →
      只有触发规则的实体进入信号处理
```

---

## 二、实体分类规则

| 优先级 | 分类 | 触发条件 | 可视化 |
|--------|------|---------|--------|
| Critical | Locked | 当前注意焦点实体，仍在注视锁中 | 👁 |
| High | Motion | `SpatialEntity.velocity` magnitude > 阈值 | 🏃 |
| High | Threat | 实体已知具有威胁标签（捕食者/敌方战斗员） | ⚔ |
| Medium | GoalRelevant | 闭包返回——"这个实体和当前GOAP目标相关" | 🎯 |
| Low | SociallySalient | 高地位/仇恨/亲密对象——从Relationship查 | 👑 |
| Low | Novel | PerceptualCache 中没有此实体+非明显背景 | ❓ |
| — | Background | 以上全不满足→跳过 | — |

## 三、注意切换事件

当注意力发生转移时，感官系统向 `event_bus` 广播：

```rust
struct AttentionShiftEvent {
    npc_id: NpcId,
    old_focus: Option<EntityId>,
    new_focus: Option<EntityId>,
    reason: AttentionShiftReason,
    timestamp: GameInstant,
}

enum AttentionShiftReason {
    StartlingStimulus,     // 突发巨响/强光
    MotionDetected,        // 外周运动
    GoalRelevantFound,     // "我找的东西出现了"
    SociallyDrawn,         // 高地位/熟人进入视野
    NoveltyDriven,         // 完全没见过的东西
    RandomSaccade,         // 无意识扫视
    SocialGazeFollowing,   // 跟随他人视线方向
}
```

## 四、习惯化

重复出现的刺激 → 响应递减 → 释放注意力资源：

```rust
struct HabituationState {
    consecutive_occurrences: u32,
    last_response_intensity: f32,
    decay_since_last: TimeDelta,
}

fn habituated_salience(raw_salience: f32, habituation: &HabituationState) -> f32 {
    let habituation_factor = (-0.1 * habituation.consecutive_occurrences as f32).exp();
    raw_salience * habituation_factor
}
```

## 五、社会性共同注意（涌现——不特殊编码）

```
NPC A 抬头看天。
NPC B 的感官: NonVerbalPerception 检测 A 的视线向上。
NPC B 的注意引擎: "A 在看什么？" → GoalRelevant(好奇心) → 射线分配给上方。
NPC B 的视觉: 发现天上有一只龙。
NPC B 的注意: 注意力转移→龙。
```

从现有机制涌现——不需要"共同注意系统"。
