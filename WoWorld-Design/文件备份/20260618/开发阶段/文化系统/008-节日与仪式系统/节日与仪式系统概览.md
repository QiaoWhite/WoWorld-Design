## 一、设计哲学

### 1.1 定义

- **仪式（Ritual）**：结构化的集体行为——有主祭、参与者、地点、言语和身体动作。仪式是原子——不可再分的仪式行为单元。
- **节日（Festival）**：一个或多个仪式按日历循环重复的大规模文化活动。节日是仪式在时间轴上的组织框架。
- **个人仪式（Personal Ritual）**：由生命事件触发的单次仪式（婚礼、葬礼、成年礼）——使用同一套 `RitualDef` 结构，但不由日历调度。

### 1.2 核心主张

**仪式是统一的。节日是仪式在日历上的循环。权力的强制是权力系统的事——不耦合到节日系统。**

```
RitualDef —— 所有仪式的通用原子（文化系统 008）
    │
    ├─── + 日历调度 → FestivalFramework（节日）
    │
    ├─── + 生命事件触发 → PersonalRitual（个人仪式——由 NPC 生命模块调度）
    │
    └─── + 魔法成分 → 魔法仪式（MagicRitualRef 引用魔法系统 010）
```

## 二、RitualDef —— 仪式原子

### 2.1 数据结构

```rust
/// 仪式定义——所有仪式的通用原子结构
/// 
/// 无论是一个国王的加冕、铁匠的婚礼、还是收获节的开场祭祀——
/// 都使用同一套 RitualDef。区别仅在于参与者的规模和仪式的调度方式。
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct RitualDef {
    /// 仪式在文化中的名称（如"加冕礼"、"婚誓"、"丰收祭"）
    pub name: GeographicName,
    
    /// 主祭——谁主持仪式
    pub celebrant: CelebrantRole,
    
    /// 参与者角色及数量要求
    pub participants: Vec<ParticipantRequirement>,
    
    /// 仪式地点类型
    pub location: RitualLocation,
    
    /// 预计持续时长（实际分钟）
    pub duration_minutes: f32,
    
    /// 言语部分——与 TurnMode::Ritual 对接
    pub speech: Vec<RitualSpeechStep>,
    
    /// 身体动作部分
    pub actions: Vec<PhysicalAction>,
    
    /// 可选的物品要求
    pub required_items: Vec<ItemRequirement>,
    
    /// 可选的超自然成分——引用魔法系统
    pub magic: Option<MagicRitualRef>,
}

/// 主祭角色
pub enum CelebrantRole {
    /// 统治者主持——权力合法性的展示
    Ruler { power_edge_id: Option<PowerEdgeId> },
    /// 宗教人员——祭司/萨满/僧侣
    Clergy { npc_id: Option<NpcId>, faith_id: FaithId },
    /// 长者——宗族/氏族传统
    Elder { npc_id: Option<NpcId> },
    /// 轮流——每年由不同人担任
    Rotating { selection: SelectionRule },
    /// 无主祭——全体参与者共同完成
    Collective,
    /// 仪式主体自身——婚礼中新郎/新娘自己宣誓
    Subject,
}

/// 参与者要求
pub struct ParticipantRequirement {
    pub role: ParticipantRole,
    pub count: Range<u32>,
    pub eligibility: Vec<EligibilityCriterion>,
}

pub enum ParticipantRole {
    Assistant,      // 助祭
    OfferingBearer,  // 祭品携带
    Choir,           // 诵经/唱诗
    Witness,         // 见证人
    Audience,        // 观礼群众
}

/// 仪式地点
pub enum RitualLocation {
    /// 建筑物内——神殿/王宫大厅
    Building { building_id: BuildingId },
    /// 露天——广场/圣泉/古树下
    OpenAir { geographic_entity_id: GeographicEntityId },
    /// 游行——沿特定路线移动
    Procession { waypoints: Vec<GeographicEntityId> },
    /// 由当时上下文决定——在节日举办地举行
    FestivalDefault,
}

/// 言语步骤——与 语言表达 005 TurnMode::Ritual 的 RitualStep 对齐
pub struct RitualSpeechStep {
    pub speaker_role: SpeakerRole,
    pub text_template_id: u32,
    pub params: FxHashMap<String, String>,
    pub duration_seconds: f32,
}

pub enum SpeakerRole {
    Celebrant,
    SpecificParticipant { role: ParticipantRole, index: u32 },
    AllParticipants,    // 群众齐声
    CallAndResponse,    // 主祭↔参与者 应答
}

/// 身体动作
pub enum PhysicalAction {
    Kneel,
    Prostrate,
    RaiseHands,
    OfferItem { item_slot: u8 },
    Pour { substance: String },
    Light { target: String },  // 点燃火炬/蜡烛
    ProcessionStep { from: u8, to: u8 },
    Dance { dance_type: String },
    Silence { duration_seconds: f32 },
}

/// 魔法成分——引用魔法系统，不定义魔法逻辑
pub struct MagicRitualRef {
    /// 魔法系统 010 中定义的仪式规格 ID
    pub magic_spec_id: u32,
    /// 预期魔力消耗
    pub expected_mana_cost: u32,
    /// 风险等级
    pub risk_level: MagicRiskLevel,
}

pub enum MagicRiskLevel { Safe, LowRisk, ModerateRisk, HighRisk, Catastrophic }
```

### 2.3 RitualScope —— 仪式规模

```rust
/// 仪式规模——从 subject 社会地位 + 文化参数 + 事件历史重要性自然涌现
/// 不由代码预设——不由类型（婚礼 vs 加冕）决定
pub enum RitualScope {
    Family,           // 仅家族成员——普通人的婚礼
    Community,        // 邻里+朋友——受尊敬的铁匠的葬礼
    Settlement,       // 全聚落——镇长的就职仪式
    CrossSettlement,  // 跨聚落——国王加冕
    Cultural,         // 整个文化区域——国教最高祭司的葬礼
}

impl RitualScope {
    pub fn determine(
        subject_social_status: f32,          // 0-1
        culture_power_distance: f32,         // 等级制文化→大人物的仪式更重要
        event_historical_significance: f32,  // 0-1
    ) -> Self {
        let scale = subject_social_status * 0.40
                  + culture_power_distance * 0.35
                  + event_historical_significance * 0.25;
        if scale > 0.8 { RitualScope::Cultural }
        else if scale > 0.6 { RitualScope::CrossSettlement }
        else if scale > 0.4 { RitualScope::Settlement }
        else if scale > 0.2 { RitualScope::Community }
        else { RitualScope::Family }
    }
}
```

### 2.4 RitualDef 的三种使用语境

| 语境 | 调度方式 | 规模决定 | 示例 |
|------|---------|---------|------|
| **节日中的仪式** | FestivalFramework.schedule 绑定到 day_of_year | 节日规模 × 文化参数 | 收获节开场祭祀、新年祈福 |
| **个人生命仪式** | NPC GOAP 目标触发（Goal::GetMarried → 婚礼仪式） | RitualScope——从 subject 社会地位 + power_distance 派生 | 铁匠婚礼(家族级)、国王加冕(全文化级) |
| **魔法仪式** | 魔法系统调度（法师发起） | 魔法规格决定 | 传送门开启、祈雨术 |

## 三、FestivalFramework —— 节日

### 3.1 数据结构

```rust
/// 节日框架——种子生成或运行时创建
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct FestivalFramework {
    pub id: FestivalId,
    /// 该节日的"本地文化"
    pub primary_culture: CultureId,
    
    /// 节日名称（使用地名系统 GeographicName 基础设施）
    pub name: GeographicName,
    
    /// 日期——day_of_year (1-120)
    pub day_of_year: u16,
    /// 持续天数 (1-7)
    pub duration_days: u8,
    /// 宣告期提前天数
    pub announcement_lead_days: u8,
    /// 预备期提前天数
    pub preparation_lead_days: u8,
    
    /// 节日类型
    pub festival_type: FestivalType,
    /// 情绪基调
    pub emotional_tone: EmotionalTone,
    /// 包含的核心仪式（至少一个）
    pub rituals: Vec<RitualDef>,
    /// 附加活动（宴饮、竞技、舞蹈……）
    pub activities: Vec<FestivalActivityType>,
    
    /// 宗教绑定（如果是宗教节日）
    pub bound_faith: Option<FaithId>,
    /// 排他性
    pub exclusivity: FestivalExclusivity,

    /// 组织者类型——种子生成时可为 None（运行时动态决定）
    /// 组织者决定了节日的权力来源和谁从节日中获益
    pub organizer: Option<OrganizerType>,
}

/// 节日组织者——谁负责组织和主持节日
pub enum OrganizerType {
    Ruler,                  // 统治者主持——与权力系统桥接（合法性ritual因子）
    ReligiousInstitution,   // 神殿/宗教机构主持
    Guild,                  // 行会/商会主持
    Grassroots,             // 自发群众——涌现型节日的常见组织者
    Clan,                   // 特定家族/氏族主持——高集体主义文化
}

/// 组织者从节日中获得的利益——由运行时检测，不存为字段
pub enum OrganizerBenefit {
    LegitimacyGain,       // 统治者→合法性ritual因子更新
    InfluenceGain,        // 神殿/行会→在聚落中的影响力
    ReputationGain,       // 个人→名声提升
    FinancialProfit,      // 商人→从节日消费中获利
}

/// FestivalId —— 全局唯一，32位
/// 布局：[31]=source(0=种子,1=运行时), [30:24]=reserved, [23:0]=index
#[derive(Copy, Clone, PartialEq, Eq, Hash, Serialize, Deserialize)]
pub struct FestivalId(pub u32);

/// 节日类型
pub enum FestivalType {
    Calendar {
        anchor: CalendarAnchor,
    },
    Religious {
        holy_day_id: u32,  // 信仰系统分配的神圣日 ID
    },
    Commemorative {
        event_id: EventId,  // 被纪念的历史事件
    },
    Emergent {
        source: EmergentSource,
    },
}

/// 天文锚点
pub enum CalendarAnchor {
    SpringEquinox,     // day 1
    MidSpring,         // day 15
    SummerSolstice,    // day 30
    MidSummer,         // day 45
    AutumnEquinox,     // day 60
    MidAutumn,         // day 75
    WinterSolstice,    // day 90
    MidWinter,         // day 105
}
impl CalendarAnchor {
    pub fn season(&self) -> Season {
        match self {
            SpringEquinox | MidSpring => Season::Spring,
            SummerSolstice | MidSummer => Season::Summer,
            AutumnEquinox | MidAutumn => Season::Autumn,
            WinterSolstice | MidWinter => Season::Winter,
        }
    }
}

/// 涌现型节日的来源
pub struct EmergentSource {
    pub detected_pattern: String,  // "annual_salmon_run" / "meteor_shower" / "spontaneous_market"
    pub geographic_entity: Option<GeographicEntityId>,
    pub first_detected_year: u64,
}

/// 情绪基调
pub enum EmotionalTone {
    Solemn,         // 庄严肃穆——祭祀、追思
    Joyful,         // 欢乐庆祝——宴饮、舞蹈
    Competitive,    // 竞技——比赛
    Contemplative,  // 沉思——祈福、冥想
    Ecstatic,       // 狂喜——狂欢、放纵
}

/// 排他性
pub enum FestivalExclusivity {
    Open,                                            // 完全开放
    Restricted { restricted_activities: Vec<u8> },   // 部分活动仅限特定群体
    Closed { eligible_criteria: Vec<EligibilityCriterion> },
}
```

```rust
/// 节日在某年份的具体发生——运行时动态数据
pub struct FestivalInstance {
    pub festival_id: FestivalId,
    pub year: u64,
    pub phase: FestivalPhase,
    
    /// 实际规模 0-1（受当年条件调节）

    Dormant,         // 尚未激活（年份未到）

    Celebration,     // 庆祝期——核心活动进行中
    Aftermath,       // 余波期——清理/情绪回落
    Concluded,       // 已结束（实例归档）
}

pub enum FestivalCancellation {
    /// 提前取消——在庆祝期开始前
    Cancelled {
        reason: CancellationReason,
        cancelled_at: GameTime,
        will_resume_next_year: bool,
    },
    /// 进行中被中止——庆祝期中被中断
    Aborted {
        reason: AbortReason,
        aborted_at: GameTime,
        completion_ratio: f32,  // 已完成/总天数
    },
}

pub enum CancellationReason {
    War, Plague, Famine, RulerDeath, NaturalDisaster,
    ConquerorSuppression, LackOfResources,
}

pub enum AbortReason {
    EnemyAtGates, FireOrDisaster, RitualFailure,
    ViolenceOutbreak, WeatherCatastrophe,
}

/// 取消/中止的后果
pub struct CancellationAftermath {
    /// 集体情绪 penalty——取消的节日导致 mood 下降
    pub mood_penalty: f32,              // 0.05(天灾) ~ 0.20(统治者无故取消)
    /// 如果是 RitualFailure → 可能触发信仰危机
    pub faith_crisis_probability: f32,  // 0-1
    /// 如果是 ConquerorSuppression → 增加抵抗情绪
    pub resistance_boost: f32,
    /// 取消是否值得写入历史（规模越大越值得）
    pub historical_significance: f32,
}
```

```rust
    pub heavy_rain_cancels: bool,
    pub snow_cancels: bool,
    pub wind_sensitive: bool,
    pub heat_sensitive: bool,
    pub cold_sensitive: bool,
}
impl WeatherSensitivity {
    pub fn indoor() -> Self {
        Self { heavy_rain_cancels: false, snow_cancels: false,
               wind_sensitive: false, heat_sensitive: false, cold_sensitive: false }
    }
    pub fn is_cancelled_by(&self, weather: &WeatherSample) -> bool {
        if self.heavy_rain_cancels && weather.is_heavy_precip() { return true; }
        if self.snow_cancels && weather.is_snow() { return true; }
        if self.wind_sensitive && weather.wind_speed_ms > 15.0 { return true; }
        false
    }
}
```

## 四、节日生命周期

### 4.1 阶段流转

```
Dormant → Announcement → Preparation → Celebration → Aftermath → Concluded
  ↑                                                                    │
  └──────────────────── 下一年重新开始 ───────────────────────────────┘
```

| 阶段 | 触发 | 持续 | 关键行为 |
|------|------|------|---------|
| **Announcement** | `day_of_year - announcement_lead_days` | 3-7天 | 信息传播系统发出节日宣告；NPC 获知 |
| **Preparation** | `day_of_year - preparation_lead_days` | 1-3天 | 购物激增、服饰准备、装饰出现、外地 NPC 旅行 |
| **Celebration** | `day_of_year` | 1-7天 | 仪式执行、竞技、宴饮、舞蹈——日常 Routine 暂停 |
| **Aftermath** | 庆祝期结束后 | 1-2天 | 装饰拆除、经济结算、情绪回落、历史记录 |

### 4.2 跨年边界处理

```rust
/// 判断给定 day_of_year 是否在节日的指定阶段内
fn is_day_in_phase(framework: &FestivalFramework, phase: FestivalPhase, day: u16) -> bool {
    let (start_offset, end_offset) = match phase {
        FestivalPhase::Announcement => (-(framework.announcement_lead_days as i32), 0),
        FestivalPhase::Preparation => (-(framework.preparation_lead_days as i32), 0),
        FestivalPhase::Celebration => (0, framework.duration_days as i32),
        FestivalPhase::Aftermath => (framework.duration_days as i32, framework.duration_days as i32 + 2),
        _ => return false,
    };
    
    let start = modulo_120(framework.day_of_year as i32 + start_offset);
    let end = modulo_120(framework.day_of_year as i32 + end_offset);
    
    if end >= start { day >= start && day < end }
    else { day >= start || day < end } // 跨越年边界
}

fn modulo_120(n: i32) -> u16 {
    ((n % 120 + 120) % 120) as u16
}
```

```rust
fn weather_sensitivity(activity: &FestivalActivityType) -> WeatherSensitivity {
    match activity {
        Procession | Tournament | Race | Fireworks | Dance | Market
            => WeatherSensitivity { heavy_rain_cancels: true, wind_sensitive: true, ..indoor(false) },
        Feast | Music | Theatre | Contest | GiftGiving
            => WeatherSensitivity { heavy_rain_cancels: false, wind_sensitive: true, ..indoor(false) },
        Prayer | Sacrifice | RitualCleansing | Matchmaking | Auction
            => WeatherSensitivity::indoor(),
        Fireworks | Decoration
            => WeatherSensitivity { heavy_rain_cancels: true, wind_sensitive: true, ..indoor(false) },
        _ => WeatherSensitivity { heavy_rain_cancels: false, snow_cancels: false, wind_sensitive: false, heat_sensitive: false, cold_sensitive: false },
    }
}

/// 将天气敏感活动在恶天气下标记为取消——降低节日 scale
fn apply_weather_to_festival(instance: &mut FestivalInstance, weather: &WeatherSample, framework: &FestivalFramework) {
    let mut cancelled_count = 0u32;
    let total = framework.activities.len() as u32;
    
    for activity in &framework.activities {
        let sens = weather_sensitivity(activity);
        if sens.is_cancelled_by(weather) { cancelled_count += 1; }
    }
    
    instance.weather_impact = 1.0 - (cancelled_count as f32 / total.max(1) as f32) * 0.7;
    instance.scale *= instance.weather_impact;
}
```

```rust
/// 每文化节日数量——归一化密度指数
fn festival_count_for_culture(core: &CultureCoreParams, rng: &mut impl Rng) -> u32 {
    let density = (
        core.religiosity * 0.30
        + core.artistry * 0.25
        + core.indulgence * 0.20
        + core.competition_orientation * 0.15
        + (1.0 - core.uncertainty_avoidance) * 0.10
    ).clamp(0.0, 1.0);
    
    let count = 2.0 + density * 5.0;
    let jitter = rng.gen_range(-0.5..0.5);
    (count + jitter).round().clamp(2.0, 7.0) as u32
}

/// 节日持续时长——从文化参数 + 节日类型派生
fn derive_duration(
    festival_type: &FestivalType,
    core: &CultureCoreParams,
    rng: &mut impl Rng,
) -> u8 {
    let base = match festival_type {
        FestivalType::Calendar { .. } => 1.0,          // 日历节日：至少1天
        FestivalType::Religious { .. } => 1.0 + core.religiosity * 2.0,  // 1-3天
        FestivalType::Commemorative { .. } => 1.0,      // 纪念日：通常1天
        FestivalType::Emergent { .. } => 2.0,            // 涌现型：通常持续自然现象时长
    };
    let indulgence_bonus = core.indulgence * 2.0;        // 放纵度高→多庆祝几天
    let artistry_bonus = core.artistry * 1.0;            // 艺术性高→活动丰富→需要更长时间
    let raw = base + indulgence_bonus + artistry_bonus;
    (raw + rng.gen_range(-0.5..0.5)).round().clamp(1.0, 7.0) as u8
}

/// 节日类型的文化驱动权重
fn festival_type_weights(core: &CultureCoreParams) -> Vec<(FestivalTypeTag, f32)> {
    let mut w = vec![(CalendarTag, 1.0)]; // 所有文化都有日历节日
    
    if core.religiosity > 0.3 {
        w.push((ReligiousTag, core.religiosity * 1.5));
    }
    if core.militarism > 0.4 || core.competition_orientation > 0.5 {
        w.push((CompetitiveTag, (core.militarism + core.competition_orientation) / 2.0));
    }
    if core.artistry > 0.5 {
        w.push((ArtisticTag, core.artistry));
    }
    w
}
```

### 5.2 日历节日——天文锚点 + 文化相关性

```rust
fn will_attend_festival(
    npc: &NpcData, festival: &FestivalInstance, framework: &FestivalFramework,
    culture: &dyn CultureQuery, power: &dyn PowerTopologyQuery,
    season: &SeasonState, piety: f32,

    let obligation = if power.has_duty_for(npc.id, DutyType::Obligation, 
        &DutyContent::MandatoryObservance { festival_id: festival.festival_id }) 
        { 0.95 } else { 0.0 };
    
    // 2. 虔诚度——宗教节日的核心因子
    let piety_factor = if framework.bound_faith.is_some() { piety * 0.4 } else { 0.0 };
    
    // 3. 人格因子
    let personality = npc.big_five.extraversion * 0.20
        + npc.big_five.openness * 0.15
        - npc.big_five.neuroticism * 0.10;
    
    // 4. 社交因子——有亲密关系的人决定去？
    let social = if npc.has_close_relations_attending(festival.festival_id) { 0.25 } else { 0.0 };
    
    // 5. 距离衰减——指数
    let dist_km = 5.0; // 简化：NPC已知自己与节日地点的距离（从SpatialMemory获取）
    let distance = (-dist_km / 10.0).exp().clamp(0.05, 1.0);

    let weather_expect = match season.season {
        Season::Summer => 0.95, Season::Spring => 0.85,
        Season::Autumn => 0.80, Season::Winter => 0.60,
    };
    
    let raw = obligation.max(piety_factor + personality * 0.25 + social 
        + distance * 0.20 + weather_expect * 0.10 + 0.1);
    raw.clamp(0.0, 1.0)
}
```

```rust
/// 节日竞技——节日中的比赛活动实例
struct FestivalCompetition {
    competition_type: CompetitionType,
    contestants: Vec<Contestant>,
    format: CompetitionFormat,  // 淘汰赛/循环赛/一次性决赛
    judges: Vec<Judge>,
}

enum CompetitionType {
    /// 武艺竞技——使用技能系统 proficiency 判定
    Martial { skill_id: SkillId },
    /// 速度竞技——基于 PhysicalAccess
    Race { race_type: RaceType },
    /// 评分类——裁判打分
    Judged { skill_id: SkillId, criteria: Vec<JudgingCriterion> },
    /// 力量竞技
    Strength { event: StrengthEvent },
}

enum CompetitionFormat { SingleElimination, RoundRobin, OneShot }

/// 判定胜者——离线计算（比赛开始一次性运行，不逐帧）
fn determine_winner(comp: &FestivalCompetition, rng: &mut impl Rng) -> Vec<Contestant> {
    for contestant in &comp.contestants {
        contestant.score = match comp.competition_type {
            Martial { skill_id } => {
                let prof = contestant.npc_skills.get(&skill_id).map(|s| s.proficiency()).unwrap_or(0.1);
                let apt = contestant.npc_skills.get(&skill_id).map(|s| s.innate_aptitude).unwrap_or(1.0);
                prof * (1.0 + (apt - 1.0) * 0.5) + rng.gen_range(-0.15..0.15)
            }
            Race { .. } => {
                let speed = contestant.physical.agility * 0.6 + contestant.physical.endurance * 0.4;
                1.0 / (1.0 + speed * rng.gen_range(0.8..1.2))
            }
            Judged { .. } => { /* 每位裁判独立打分 → 去高低取平均 */ 0.5 }
            Strength { .. } => {
                contestant.physical.strength * (1.0 - rng.gen_range(0.0..0.3))
            }
        };
    }
    // 按得分降序
    comp.contestants.sort_by(|a, b| b.score.partial_cmp(&a.score).unwrap());
    comp.contestants.clone()
}
```

### 6.5 聚落间节日协调

相邻聚落同日有节日时——NPC 只能选一个参加。节日日期分配时考虑周边已有节日：

```rust
/// 120天循环日历的距离
fn circular_distance_120(a: u16, b: u16) -> u16 {
    let diff = (a as i32 - b as i32).abs() as u16;
    diff.min(120 - diff)
}

/// 分配节日日期——避开周边已有节日（≥3天间距优先）
fn assign_festival_date_with_coordination(
    target_range: Range<u16>,
    nearby_festival_days: &[u16],
    rng: &mut impl Rng,
) -> u16 {
    let candidates: Vec<(u16, f32)> = target_range
        .map(|day| {
            let min_dist = nearby_festival_days.iter()
                .map(|&d| circular_distance_120(day, d))
                .min().unwrap_or(120);
            let weight = if min_dist >= 3 { 1.0 } else { 0.3 };
            (day, weight)
        })
        .collect();
    weighted_pick(&candidates, rng)
}
```

NPC 面对同日两个节日时：优先本聚落 → 有亲密关系的人去的 → 距离近的。

```rust
// → NPC 在每日 Duty 检查中知道自己有义务
// → NPC 的节日参加决策中 obligation_factor = 0.95
```

### 9.3 PowerSource::Divine 仪式参照

`PowerSource::Divine { ritual: String }` —— 此处的 `ritual` 字符串是 `RitualRegistry` 中已注册仪式的名称。权力系统通过 `RitualQuery` 验证该仪式确实被执行过。仪式定义的权威在文化系统——权力系统不定义仪式。

## 十一、节日演变

### 11.1 文化反馈

节日每年完成后对文化参数的温和推力：

```rust
fn festival_cultural_feedback(festival: &FestivalFramework, instance: &FestivalInstance, 
                                culture: &mut CultureCoreParams) {
    let novelty = |param: f32| 1.0 - param; // 已很高的参数→再推更难
    let strength = instance.scale * (instance.actual_participants as f32 
        / instance.planned_participants.max(1) as f32) * 0.003;
    
    match festival.emotional_tone {
        EmotionalTone::Ecstatic => culture.indulgence += strength * 0.5 * novelty(culture.indulgence),
        EmotionalTone::Solemn => culture.religiosity += strength * 0.3 * novelty(culture.religiosity),
        EmotionalTone::Competitive => culture.competition_orientation += strength * 0.4 * novelty(culture.competition_orientation),
        _ => {}
    }
    
    culture.clamp_all();
}
```

### 11.2 节日诞生/衰落/融合/漂移

| 路径 | 机制 |
|------|------|
| 诞生 | 种子生成 / 历史事件结晶 / 涌现检测 / 文化接触引入 |
| 衰落 | 连续 5 年参与率 <20% → Declining → 5 年观察 → Defunct |
| 融合 | 两文化日期接近+类型相似+长期接触 → 四种融合模式（见下） |
| 漂移 | 外围元素（食物/舞蹈/装饰）逐渐变化——核心仪式保持稳定（见下） |

**节日融合——四种模式与检测算法**：

| 融合模式 | 条件 | 结果 |
|---------|------|------|
| 日期借用 | 两节日日期距离 ≤15天 + openness≥0.4 | 合并为单一日期——通常是优势文化的日期 |
| 内容混合 | 贸易/通婚 ≥50年 | A 节日中出现 B 文化的活动类型 |
| 名称融合 | 混合区聚落 ≥100年 | 节日名称变为混合——A词根+B文化后缀（复用 GeographicName exonym 机制） |
| 全新结晶 | 混合文化区 + 300年 | 两节日完全融合为新的第三个节日——克里奥尔化 |

```rust
fn detect_festival_fusion(
    culture_a: CultureId, culture_b: CultureId, contact_years: u32,
    festivals_a: &[FestivalFramework], festivals_b: &[FestivalFramework],
    openness_a: f32, openness_b: f32,
) -> Vec<FusionCandidate> {
    let mut candidates = Vec::new();
    for fa in festivals_a {
        for fb in festivals_b {
            let date_dist = circular_distance_120(fa.day_of_year, fb.day_of_year) as u32;
            if date_dist > 15 { continue; }
            let type_sim = festival_type_similarity(&fa.festival_type, &fb.festival_type);
            if type_sim < 0.3 { continue; }
            let prob = (contact_years as f32 / 100.0).clamp(0.0, 1.0)
                * openness_a.min(openness_b) * type_sim * (1.0 - date_dist as f32 / 15.0);
            if prob > 0.5 {
                candidates.push(FusionCandidate {
                    festival_a: fa.id, festival_b: fb.id,
                    suggested_date: ((fa.day_of_year as u32 + fb.day_of_year as u32) / 2) as u16,
                    fusion_probability: prob,
                });
            }
        }
    }
    candidates
}

struct FusionCandidate {
    festival_a: FestivalId,
    festival_b: FestivalId,
    suggested_date: u16,
    fusion_probability: f32,
}
```

**节日内容代际漂移**——外围元素随代际缓慢变化，核心仪式保持稳定：

```rust
fn apply_festival_drift(
    festival: &mut FestivalFramework,
    core: &CultureCoreParams,
    years: u32,
    rng: &mut impl Rng,
) {
    // 漂移速率：低不确定性规避→快速创新；高→几乎不变
    let drift_rate = (1.0 - core.uncertainty_avoidance) * 0.02 * years as f32;
    // 每 10 年：开放文化有 ~10% 概率替换一个外围活动
    
    // 只漂移 activities（外围活动），不漂移 rituals（核心仪式）、emotional_tone、festival_type
    if rng.gen_bool((drift_rate as f64).min(0.8)) {
        let compatible = activities_compatible_with_tone(festival.emotional_tone);
        if let Some(activity) = festival.activities.iter_mut().find(|a| is_peripheral(*a)) {
            *activity = compatible[rng.gen_range(0..compatible.len())];
        }
    }
}
```

## 十二、历史记录与记忆

### 12.1 节日事件的历史记录

不是每次节日都写入 HistoricalEvent——选择性记录以防止历史数据膨胀：

| 条件 | 记录方式 | 理由 |
|------|---------|------|
| **首届节日**（节日被创建时） | 完整的 HistoricalEvent（`EventType::FestivalInaugurated`） | 节日的起源本身就是历史事件 |
| **年度 scale ≥ 0.8** | 轻量 HistoricalEvent（`EventType::FestivalHeld`，仅含 scale+参与人数） | 异常盛大——值得单独记录 |
| **节日被中断/取消** | 完整的 HistoricalEvent + 因果链（`FestivalAborted`/`FestivalCancelled`） | 中断比正常举办更值得记录——反映历史动荡 |
| **节日中发生特殊事件**（竞技死亡/神显/火灾） | 完整的 HistoricalEvent + 关联到该节日的 causal chain | 特殊事件本身就是历史 |
| **年度 scale 0.4-0.8** | 合并到该聚落当年的"年度社会生活摘要"中——不单独记录 | 正常规模——聚合足够 |
| **年度 scale < 0.4** | 不记录 | 太冷清——不值得记录 |

轻量的年度摘要统一通过 `RecurringEventSeries` 跟踪——所有节日（无论规模）至少有一个年度条目用于演变检测。

```rust
/// 周期性事件系列——轻量记录所有节日的年度摘要
pub struct RecurringEventSeries {
    pub series_id: EventSeriesId,
    pub festival_id: Option<FestivalId>,
    pub first_year: u64,
    pub latest_year: u64,
    pub skipped_years: Vec<u64>,
    /// year → scale（仅存规模——不存完整 HistoricalEvent）
    pub annual_summaries: BTreeMap<u64, f32>,
}
```

### 12.2 NPC 对节日的记忆

NPC 的 `EventMemory` 在每年节日后更新：`FestivalMemory` 记录参加次数、最有印象的一届、关联个人事件（"在那年节日上遇见了妻子"）。节日作为时间锚点——NPC 用节日标记其他事件的时间（"收获节前三天"）——由语言表达的时间描述模板自然处理。

## 十四、玩家体验

### 13.1 参与层次

| 层次 | 行为 | 后果 |
|------|------|------|
| **旁观** | 观看仪式/竞技/表演 | 无直接后果 |
| **参与** | 宴饮、舞蹈、赠礼、逛集市 | NPC 关系提升、经济消费 |

| **干预** | 做演讲、救场、捣乱 | 名声显著改变 / 法律后果 |
| **创立** | 史诗任务 → 当地自发纪念 3 年 → 系统检测并正式化 | 间接——玩家不会直接点击"创立节日" |

### 13.2 发现线索

玩家通过五层线索发现节日：旅行者对话 → 远距离灯火(L3视觉) → 街道装饰 → NPC 服装 → 直接与 NPC 对话询问。跨文化节日体验受文化距离(similarity)调制——文化距离大 → 新奇或困惑。
