# 007 — 物品获取途径

## 一、八条获取路径总览

```
采集 (Gather)    — 植物/矿/猎/渔——Life 模块定义产出规则
掉落 (Loot)      — 击败敌人/翻找尸体——Combat 模块定义掉落表
购买 (Purchase)  — 从 NPC 商人购买——经济系统定义价格
偷窃 (Steal)     — 从 NPC 库存中盗取——犯罪系统定义风险
制作 (Craft)     — 原料→成品——见 006 制造系统
任务 (Quest)     — 任务奖励——任务系统定义
发现 (Discover)  — 箱子/废墟/隐藏宝藏——World Gen 定义放置规则
礼物 (Gift)      — NPC 赠送——NPC 模块定义送礼动机
```

## 二、采集——Life 模块接口

[[../生命/009-植物|植物系统]] 定义了 `HarvestableProduct`：

```rust
// Life 模块的数据合同
struct HarvestableProduct {
    product_type: ProductCategory,   // Wood/Fiber/Food/Medicine/Dye
    yield_base: f32,                 // 基础产量
    harvest_seasons: Vec<Season>,
    regen_days: u16,
    tool_bonus: Option<ToolType>,
}

// 物品系统映射
// ProductCategory → ItemCategory
// Wood → WoodMat, Fiber → FiberMat, Food → Food
// Medicine → OrganicMat (Medicine sub), Dye → OrganicMat (Dye sub)
```

采集流程：Life 触发 harvest → 调用物品系统 `ItemEntIdAllocator.allocate()` 分配实例 ID → 产物进入采集者库存。

**产量公式**：`base_yield × skill_multiplier(0.5-1.5) × tool_bonus`

## 三、战利品掉落——Combat 模块接口

[[../战斗/012-战后过渡与伤势|Combat 012]] 定义战后掉落：

```rust
// Combat 模块调用物品系统
struct CombatLoot {
    items: Vec<(ItemDefId, u32)>,  // 物品模板 + 数量
    drop_probability: f32,         // 0-1
    condition_factor: f32,         // 战斗中消耗对耐久的影响
}
```

掉落物品可能已有 provenance——被击杀的 NPC 的武器，带着它自己的 ItemEntId 和历史。物品系统不关心掉落规则——只负责为每件掉落物品分配/保留 ItemEntId 并放入胜者库存。

trait MerchantInventory {

## 五、偷窃——犯罪系统接口

```rust
trait TheftTarget {
    fn visible_items(&self, observer: NpcId) -> Vec<ItemEntId>;  // 能被看到的物品
    fn steal_attempt(&mut self, thief: NpcId, target_item: ItemEntId) 
        -> Result<ItemEntId, TheftError>;  // 成功返回物品，失败返回原因
}
```

物品系统不定义偷窃成功率和风险——那是犯罪系统的职责。物品系统只负责将物品从一个 inventory 移到另一个。

## 七、礼物——NPC 模块接口

NPC 之间送礼（见 [[../NPC活人感模块/NPC活人感开发文档ver2.0|NPC ver2.0]] —— `GiveGift(target_id)` 行动）：

```rust
// NPC 模块调用
fn give_gift(giver: &mut Npc, receiver: &mut Npc, item: ItemEntId) -> Result<(), GiftError> {
    // 1. 从 giver 库存移除
    giver.inventory.remove(item);
    // 2. 放入 receiver 库存
    receiver.inventory.add(item)?;
    // 3. 关系变化由 NPC 模块计算——物品系统不管
    Ok(())
}
```

礼物的审美属性和价值影响收礼者的情绪反应——由 NPC 模块计算。见 [[004-装备系统|004 §五 审美判断]]。

## 九、所有途径的统一入口

```rust
impl ItemRegistry {
    /// 任何途径创建新物品实例的统一入口
    pub fn create_instance(&self, def_id: ItemDefId, source: AcquisitionSource) -> ItemEntId {
        let ent_id = self.allocator.allocate();
        let state = ItemState::default_for(def_id, &self.definitions);
        self.instances.insert(ent_id, (def_id, state));

        // History 系统注册创建事件（可选——仅对有意义物品）
        if self.definitions.get(def_id).unwrap().rarity >= Rarity::Rare {
            history::record_item_birth(ent_id, def_id, source, current_timestamp());
        }

        ent_id
    }
}

enum AcquisitionSource {
    Gathered { plant_id: EntityId, location: WorldPos },
    Looted { victim: NpcId, battle_id: EventId },
    Purchased { merchant: NpcId, price_copper: u32 },
    Stolen { victim: NpcId, success: bool },
    Crafted { recipe: RecipeId, crafter: NpcId },
    QuestReward { quest_id: QuestId },
    Discovered { container_id: ItemEntId, location: WorldPos },
    Gifted { giver: NpcId, occasion: String },
}
```

History 系统（[[../历史/004-灵元素印记与历史嵌入世界|History 004]]）可选择地追踪稀有以上物品的 provenance——从出生到每一次转手。
