# 001 — 经济系统总纲

## 一、设计哲学

### 1.1 经济系统是"物理引擎"，不是"玩法脚本"

经济系统只定义价值交换的底层规则——不定义"商人该卖什么"、"玩家该怎么赚钱"、"什么商品值多少钱"。

| 经济系统管 | 经济系统不管 |
|----------|------------|
| 供需匹配的订单簿撮合引擎 | NPC该买什么 → **NPC心智**管（人格×情绪×记忆×认知） |
| 价格从交易中涌现的机制 | 物品的"公允价值" → **物品系统**的 base_value_copper 管 |
| 货币的创造与回收管道 | NPC的职业收入 → **NPC心智**×**生产设施**×**技能**决定 |
| 交易主体的涌现条件（谁成为商人） | 哪些NPC被"指定"为商人 → **没有指定——只有涌现** |
| 市场权力的行使接口 | 权力怎么获得/继承/合法化 → **政治系统**管 |
| 信息不对称下的价格知识传播 | 价格信息的失真修正 → **语言表达系统**的信息传播通道管 |

### 1.2 核心原则：客观事实层 × 主观决策层

```
经济系统 = 客观物理事实的提供者
  - 这个市场有什么在卖，什么价格
  - 你的钱包有多少铜币
  - 你的职业每天产生多少收入
  - 每个NPC的所知价格可以不同（信息不对称）

NPC心智 = 主观消费决策的驱动者
  - 人格(BigFive) → 冲动vs耐心、激进vs保守
  - 情绪(EmotionState) → 开心→多花、忧郁→少花
  - 记忆(MemoryStore) → 锚定效应、"上次买贵了"
  - 认知(EconomicCognition) → 金融素养、搜索广度、满意阈值
  - 渴望(SelfNarrative) → "我想成为富商" vs "钱够用就行"
```

**经济系统从不告诉NPC该买什么。** NPC用自己的全部心智模型做消费决策。两个有同样铜币的NPC，因为人格不同，买的东西完全不同——这就是涌现。

### 1.3 故事生成器的经济根基

经济系统的设计目标不是"做一个好玩的交易小游戏"——而是让经济事件成为故事涌现的土壤：

- 饥荒中农妇典当祖传银镯 → 三十年后孙子从当铺赎回 → 家族记忆的一环
- 铁匠技艺精进但缺本钱 → 向放贷人借债 → 第一年亏、第二年平、第三年赚 → "白手起家"的生命章节
- 商人发现了两个城市间的价差 → 白手起家经营商队 → 一代后成为行会会长 → "商业王朝"涌现
- 国王昏庸加税 → 商人逃离 → 市场凋敝 → 饥荒 → 农民起义 → "王朝覆灭"的因果链

**开发者不写这些故事。只定义供需规则、交易机制、权力接口。故事自己长出来。**

## 二、模块边界

### 2.1 经济系统 OWN 什么

| 概念 | 说明 |
|------|------|
| `Market` — 供需匹配机制 | 订单簿撮合引擎、交易模式、市场范围。Market ≠ 物理地点——可以是LocalBazaar/TradeNetwork/Underground |
| `Storefront` — 交易原子单位 | 个体店铺/摊位——铁匠铺、家门口菜摊、流动摊贩。任何NPC在任何地点都可以开 |
| `Price` — 从交易中涌现的价格 | 不预设价格表。价格是最近成交价的统计概括。每个NPC的所知价格可以不同 |
| `Transaction` — 原子交易 | 两阶段提交（钱包±库存）。玩家和NPC走同一代码路径 |
| `MoneySupply` — 货币总量 | 铸币管道、消费回收管道、跨区流动 |
| `MarketRegulations` — 市场规则参数 | 价格管制/配额/关税/准入。参数化——同一套引擎适应自由市场到军国经济 |
| `MarketAuthority` — 权力原子集合 | 谁在什么市场能做什么（设定价格/征税/禁运/征用）|
| `EconomicEvent` — 经济事件 | 价格飙升/市场崩溃/垄断检测 → 写入历史系统 |

### 2.2 经济系统 NOT OWN

| 概念 | 归属 | 理由 |
|------|------|------|
| NPC的消费决策（买什么/买多少） | **NPC心智** | 人格×情绪×记忆驱动，经济系统只提供价格+供给信息 |
| 物品的基准价值 | **物品系统** | `base_value_copper` + Quality×Rarity矩阵 |
| 制造配方和材料需求 | **物品系统** | TOML数据驱动 |
| 职业→收入的具体映射 | **NPC模块 + 技能系统** | 经济系统只提供收入来源类型（工资/自雇/贸易利润/租金） |
| 社会阶级/地位 | **社会阶级系统（独立模块）** | 财富是输入之一，但阶级由出身+教育+文化+头衔综合决定 |
| 权力的继承/选举/政变 | **政治系统（待设计）** | 经济系统只记录"谁当前拥有什么PowerAtom" |
| 关税/商业法的制定 | **法律系统（待设计）** | 经济系统消费关税数据 |
| 消费偏好（喜欢吃什么/穿什么） | **文化系统（待设计）** | 需求结构修正 |
| 商队物理移动 | **载具系统（待设计）** | 经济系统只管商路网络+贸易决策 |
| 价格信息的传播和失真 | **语言表达系统** | 复用五通道+失真算子 |
| 偷窃/抢劫的经济后果 | **法律系统** | 经济系统接收"物品被盗"事件，更新持股 |

### 2.3 职业标签——经济系统的消费方

职业标签（`ProfessionTag`）是跨模块的NPC属性——由世界生成初始分配，随技能积累和人生转变而涌现。经济系统**不拥有**职业标签，但消费它们：

- 职业标签的排列组合 → 决定 NPC 的收入来源类型
- 标签转变 → 可能触发收入来源切换
- 职业标签的 proficiency → 从技能系统派生

职业标签本身是"基础职业原子"（~80-100个），任何NPC可以持有多个标签。`[锻造0.9 + 交易0.6]` 就是"铁器商"——不需要系统预先定义"铁器商"这个职业。

## 三、核心数据模型概要

### 3.1 NpcEconomicState — NPC经济子状态

```rust
/// 注入到每个NPC的经济子状态
/// 通过trait注入NpcData，不修改NpcData本体
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct NpcEconomicState {
    /// 钱包（已在物品系统005定义——此处为NPC绑定实例）
    pub wallet: CharacterWallet,

    /// NPC已知的各市场价格（可能过期、错误——信息不对称的根源！）
    /// 键: (MarketId, ItemDefId) → 值: (已知价格, 得知时间, 来源, 信心)
    pub known_prices: BTreeMap<(MarketId, ItemDefId), PriceKnowledge>,

    /// 盈余评估缓存（每日更新一次）
    pub surplus_assessment: BTreeMap<ItemDefId, SurplusAssessment>,
    /// 需求评估缓存
    pub need_assessment: BTreeMap<ItemDefId, NeedAssessment>,

    /// 经济认知能力（从人格×技能×经验派生）
    pub cognition: EconomicCognition,

    /// 贸易偏好
    pub trade_preferences: TradePreferences,

    /// 商人特有状态（满足四条件涌现的NPC才有——None for 非商人）
    pub trader_state: Option<TraderState>,

    /// 职业标签→投入度（引用——权威来源为NPC身份系统）
    pub profession_tags: BTreeMap<ProfessionTagId, ProfessionEngagement>,
}

/// NPC经济认知能力——从人格×技能×经验派生的缓存
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct EconomicCognition {
    /// 金融素养 0-1：理解储蓄/投资/利息/风险
    /// 来源：education + 相关技能 + 经商经验
    pub financial_literacy: f32,

    /// 市场理解 0-1：对供需决定价格的理解深度
    /// 0=完全不懂，1=深刻理解（"矿山减产→铁矿会涨→先囤一些"）
    pub market_understanding: f32,

    /// 价格记忆精度 0-1
    pub price_memory_accuracy: f32,

    /// 时间偏好率（年化）：0.05=极耐心, 0.50=极短视
    /// 从 conscientiousness + wisdom 派生
    pub time_preference_rate: f32,

    /// 搜索广度：购买决策时会搜索几个市场？（1-5）
    /// 从 intelligence + extraversion 派生
    pub market_search_breadth: u8,

    /// 满意阈值 0-1：多接近"最低价"才算满意
    /// 0.3=比均价低就买, 0.9=必须接近最低价
    pub satisficing_threshold: f32,
}
```

### 3.4 ProfessionTag — 职业标签原子

职业标签是"基础职业原子"——约90个基础标签，任意2-4个排列组合穷举可达数十万种职业描述。`[锻造0.9 + 交易0.6]` 就是"铁器商"——不需要系统预先定义"铁器商"这个职业。标签组合本身就是新职业。

```rust

/// TOML数据驱动——不是枚举。Mod可在category 128-255自由新增
#[derive(Debug, Clone, PartialEq, Eq, Hash, Serialize, Deserialize)]
pub struct ProfessionTagId(u32);

/// 职业标签的投入度
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ProfessionEngagement {

    pub time_share: f32,        // 0-1 时间投入
    pub income_share: f32,      // 0-1 收入来源占比
    pub is_primary: bool,       // 是否主要职业
    pub self_identifies: bool,  // NPC自己是否认同此身份
}
```

#### 3.4.4 职业标签生命周期

```
┌─ 涌现 ─┐  技能达门槛 + time_share超阈值 + 外部事件(任命/继承)
│          │  或经济驱动(长期从事→形成职业认同)
│          │  或SelfNarrative(aspiration达成→新自我标签)
├─ 活跃 ─┤  time_share > 0.1 + income_share > 0 + self_identifies
│          │  正常参与此职业——标签影响收入来源类型和行为权重
├─ 休眠 ─┤  time_share持续<0.1超过N天(技能仍在——"用进不退")
│          │  self_identifies可能仍为true("退休了但骨子里还是铁匠")
│          │  标签保留在NPC职业历史中——不删除
├─ 历史化─┤  NPC死亡→所有标签固化。或NPC完全失去某能力(伤残)
│          │  保留在SelfNarrative.life_chapters中
│          │  "他曾是镇上最好的铁匠，后来手废了"——这是故事
```

#### 3.4.5 Proficiency → 技能系统映射

```rust
/// 职业proficiency从技能系统派生——不独立存储
impl ProfessionEngagement {
    pub fn derive_proficiency(npc: &NpcData, tag: ProfessionTagId) -> f32 {
        let def = load_profession_tag_def(tag);  // 从TOML读取related_skills
        let skills = &def.related_skills;

        if skills.is_empty() {
            // 非技能型职业（乞丐/国王/无业者/冒险者）
            // 从心智属性+行为记录派生
            return derive_non_skill_proficiency(npc, tag);
        }

        // 技能型职业：加权平均
        let mut total_w = 0.0;
        let mut weighted = 0.0;
        for (skill_id, weight) in skills {
            if let Some(entry) = npc.skills.get(skill_id) {
                weighted += entry.level * weight;
                total_w += weight;
            }
        }
        if total_w == 0.0 { return 0.0; }
        weighted / total_w
    }
}
```
- **铁匠** → Artisan.Metalwork.Blacksmith(0.8) + Artisan.Metalwork.Smelting(0.2)
- **猎人** → Combat.Ranged.Bow(0.5) + Survival.Perception.Tracking(0.3) + Survival.FieldCraft.Skinning(0.2)
- **雇佣兵** → Combat.Melee.Sword(0.4) + Combat.Defense.Shield(0.2) + Combat.Tactics(0.2) + Survival.Mobility(0.2)
- **冒险者** → 无特定技能——从心智属性+行为记录派生
- **商人/乞丐/国王** → 无特定技能——经济/社交能力不在技能系统管辖

## 四、关键机制概览

### 4.1 价格形成：从交易中涌现

价格不从每日公式计算。每个Market维护限价订单簿：
- 卖方挂Ask（最低愿意卖的价格），买方挂Bid（最高愿意出的价格）
- Bid ≥ Ask → 自动撮合，成交
- 最近成交价 = "市场价格"——不存在中央定价
- **分层定价**：正常中间价 / 紧急偏移 / 薄市场协商 / 政府强制价
- 详见 [[002-市场与订单簿]]

### 4.2 交易主体：从条件中涌现

- **供给侧**：任何NPC，持股 > 自身需求 → 自动挂卖单
- **需求侧**：任何NPC，生理/职业/社交需求未满足 → 自动挂买单
- **中间商**：满足四条件（信息+资本+心理+物流）→ 自动扫描价差 → 发起套利
- **不需要在world gen时"指定商人NPC"**
- 详见 [[004-交易主体与角色涌现]]

### 4.3 非自由市场：同一引擎 × 参数

`MarketRegulations` 的全部参数是连续值。一套订单簿引擎 + 参数差异 = 自由市场/计划经济/军国经济/自给自足：
- 价格上限 + 配额 → 公开市场有价无市 → **黑市自然涌现**
- 政府托底收购 + 大宗军需 → 军工景气
- 极高进口关税 → 自给自足
- 详见 [[005-经济体制参数化]]

### 4.4 权力与市场：PowerAtom集合

掌权者不是"国王能改规则"这种硬编码——每个人拥有的是一组 `PowerAtom`（单项市场操作能力）：
- PowerAtom来源：继承/任命（正式）、购买（贿赂）、事实产生（垄断）、暴力
- 同一套规则：贤明治理者 vs 高效寻租者 vs 昏庸掠夺者——人格×能力×信息决定权力效果
- 玩家通过任命/行会/财富/政变获得PowerAtom
- 详见 [[006-权力与市场]]

### 4.5 借贷：特化交易，非独立系统

贷款就是用"欠条+抵押物"赊购铜币。还贷就是买回欠条。不需要银行系统——欠条是物品系统中的一个 `ItemDefId`（`ItemCategory::Financial`）。放贷人不是特殊NPC——任何有铜币+愿意承担风险的人都是放贷人。违约处理由放贷人心智驱动。

### 4.6 NPC经济心智：行为经济学全覆盖

详见 [[007-NPC经济心智]]。BigFive+EmotionState+MemoryStore+EconomicCognition覆盖14个行为经济学核心概念中的10个（损失厌恶/心理账户/锚定/禀赋效应/双曲贴现/羊群效应/有限理性/公平偏好/过度自信/现状偏误）。

### 4.7 货币与稳定器

货币三管道（铸币/消费回收/跨区流动）+ 五大自动稳定器（供给弹性/累进税/必需品刚性/运费壁垒/货币商品对齐）。详见 [[008-货币、通胀与经济稳定器]]。

## 六、性能预算

| 子系统 | CPU预算 | 说明 |
|--------|---------|------|
| 订单簿撮合 | <0.1ms/笔 | 每次交易触发（事件驱动，不等tick）。O(n)线性扫描同类物品队列 |
| NPC日常消费tick | 每日 ~15K次 × ~50μs = 0.75ms（分摊到48分钟 → 可忽略） | L1-L2 NPC，每日1-2次 |
| NPC价格扫描 | 每4h ~10K次 × ~100μs（分摊到20秒 → <0.05ms/帧） | 更新已知价格 |
| 套利扫描 | <500次 × ~500μs（分摊到数分钟 → <0.01ms/帧） | 仅满足trader条件的NPC |
| L3统计消费 | 每日批量 1ms（分摊 → 可忽略） | 统计公式，不生成具体交易 |
| L4宏观指标 | 每周 2ms（分摊 → <0.001ms/帧） | 聚合指标更新 |
| 价格知识传播 | <0.1ms/帧 | 复用语言表达系统的传播管道 |
| **总计** | **<2ms/帧（分摊后）** | vs 技术栈方案 Rust CPU预算 ~7ms/帧 |

| 资源 | 预算 |
|------|------|
| 活跃Market数量 | 800-3000（~25万km²） |
| 每Market活跃挂单 | 200-2000条 |
| 订单簿内存 | ~5KB/Market平均 → ~15MB总计 |
| NPC经济状态 | ~200 bytes/NPC → ~20MB（100K NPC） |
| LMDB存储（挂单+交易历史） | ~50MB（含冗余） |

## 七、明确不做

| 条目 | 理由 |
|------|------|
| 独立的"银行系统"或金融机构 | 借贷=特化交易（赊购铜币），不需要银行。中世纪奇幻语境，金融体系违和 |
| 债券/股票/衍生品 | 中世纪奇幻超前——留给MOD |
| 信用评分/信用货币 | 货币是铜/银/金实物币。信用=某个NPC记忆中"这人可不可靠" |
| 中央定价公式 | 价格从交易涌现，不由函数计算 |
| "指定商人NPC" | 商人身份从四条件涌现 |
| NPC消费的"财富档位规定" | NPC用心智模型自己决定——不按外部标签消费 |
| 全球经济同步价格 | 每个NPC有自己的所知价格（信息不对称） |
| 全局拍卖行 | 交易需要物理/信息可达性——没有"一键买卖全球"的便利 |
