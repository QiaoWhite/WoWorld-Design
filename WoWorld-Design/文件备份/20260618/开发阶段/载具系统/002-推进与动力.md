## 一、核心决策

> **动力来源多样性（魔力/人力/畜力/风力/混合），日常航行不受自然力影响，仅灾害级天气施加硬约束。**

动力来源决定三件事：
1. **消耗什么**（燃料/体力/免费）
2. **灾害时什么会出问题**（桅杆折断/引擎过载/动物受惊）
3. **文化外观**（有帆的船 vs 有烟囱的船）

动力来源**不影响**日常航线选择——NPC 想走什么路线就走什么路线。

## 二、VehiclePropulsion —— 五种动力类型

```rust
/// 载具推进系统——不是枚举，是参数组合
/// 五种基础类型 + Hybrid 混合
#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum VehiclePropulsion {
    /// 人力驱动——划桨/脚踏/摇杆
    MusclePowered(MuscleConfig),

    /// 畜力牵引——马/牛/驮兽拉车
    AnimalTowed(AnimalTowConfig),

    /// 风力驱动——帆
    WindPowered(SailConfig),

    /// 魔力引擎——魔法物品
    MagicEngine(MagicEngineConfig),

    /// 混合动力——主+辅
    Hybrid {
        primary: Box<VehiclePropulsion>,
        secondary: Box<VehiclePropulsion>,
    },
}
```

### 2.1 MusclePowered —— 人力

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct MuscleConfig {
    /// 最少需要多少划桨/脚踏的人
    pub rowers_needed: u32,
    /// 最优人数——超出后边际收益递减
    pub optimal_rowers: u32,
    /// 每人产生的推力基准（N）
    pub thrust_per_person_n: f32,
    /// 消耗——每游戏小时每人消耗的 Stamina
    pub stamina_per_hour: f32,
}
```

- **消耗**：走 Life 系统的 Vitals.stamina
- **灾害影响**：无特殊——人力船在暴风雨中和其他船一样减速/受损
- **特点**：速度受限于人体，无法长途远洋（不是不能去——是太累太慢——NPC 自己会判断）

### 2.2 AnimalTowed —— 畜力

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct AnimalTowConfig {
    /// 最少需要多少牵引动物
    pub min_animals: u32,
    /// 最优动物数
    pub optimal_animals: u32,
    /// 每头动物产生的拉力基准（N）
    pub pull_force_per_animal_n: f32,
    /// 需要的套具 ItemDefId（可选的装备）
    pub harness_item: Option<ItemDefId>,
}
```

- **消耗**：走 Life 系统的动物 Vitals.stamina——动物也需要休息、进食
- **灾害影响**：动物可能在暴风雨/火灾/战斗中受惊→失控
- **关键边界**：动物本身**不是载具的一部分**——马走 Life 基类的完整 AI。载具只定义"需要多少拉力"，Life 系统提供"有多少拉力可用"。解耦干净。

### 2.3 WindPowered —— 风力

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SailConfig {
    /// 帆的数量
    pub sail_count: u32,
    /// 帆的总面积（m²）
    pub total_sail_area_m2: f32,
    /// 帆装类型（文化决定外观——不影响模拟）
    pub rig_type: RigType,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
pub enum RigType {
    SquareRig,       // 横帆——顺风最快
    LateenRig,       // 三角帆——逆风也能走
    LugRig,          // 中式帆装
    JunkRig,         // 中国帆船帆装
    GaffRig,         // 斜桁帆
}
```

- **消耗**：无——帆是免费午餐。风是免费的。
- **日常航行**：NPC 想走什么路线就走什么路线——风力不作为航线计算的约束条件
- **灾害影响**：暴风雨→桅杆折断→propulsion 通道损伤
- **装饰性参数**：`rig_type` 和 `sail_count` 由文化造船传统决定——仅影响外观和 lore，不影响模拟核心
- **为什么日常不受风向影响**：游戏性上，让 NPC 计算风向决定航线→每条船多 10+ 浮点运算/决策周期，且对玩家体验无增益（"这艘船今天逆风不走"对玩家只有挫折感）。风向只在灾害级有意义（"暴风雨——浪高×5——必须靠岸"）

## 三、MagicEngine —— 魔力引擎（走魔法系统 + 物品系统）

### 3.1 核心洞察

魔力引擎不是载具系统发明的概念——它是 **魔法物品系统（015-魔法物品与法器）** 中的一个特化应用。对标关系：

| 魔法系统概念 | 魔力引擎中的对应 |
|------------|---------------|
| 直接附魔（永久→不可撤销） | 引擎本体——建造时一次性完成 |
| 魔力池（内置魔力储存） | `mana_pool_capacity_ke` |
| 卡槽附魔（可替换符文） | 引擎符文槽——效率/抗性符文 |
| 充能方式 | 水晶（消耗品）/ 施法者注入 / 港口法阵 |
| 魔力消耗 | 巡航消耗率（刻/100km） |

### 3.2 数据结构

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct MagicEngineConfig {
    // ── 引擎本体（永久——建造时确定） ──
    /// 魔力池容量（刻）
    pub mana_pool_capacity_ke: f32,
    /// 巡航消耗率（刻/100km）
    pub mana_per_100km: f32,
    /// 转换效率 0-1 —— 1 刻魔力 → 多少有效推力
    /// 品质决定（由建造者的 MagicEnchanting + Engineering 技能决定）
    pub conversion_efficiency: f32,
    /// 最大充能速率（刻/小时）——限制施法者注入速度
    pub max_charge_rate_ke_per_hour: f32,

    // ── 符文槽（可替换——走物品系统卡槽模型） ──
    /// 引擎上的符文槽——对标 Item 008 的 EnchantmentSlot
    /// 可插入效率符文/抗性符文/风暴符文等
    pub rune_slots: Vec<EngineRuneSlot>,
}

/// 引擎符文槽——对标物品系统 ComponentSlot
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct EngineRuneSlot {
    pub slot_id: u8,
    /// 当前插入的符文 ItemEntId（None = 空槽）
    pub occupant: Option<ItemEntId>,
    /// 此槽支持的符文类型
    pub accepted_rune_types: Vec<EngineRuneType>,
}

/// 引擎符文类型
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
pub enum EngineRuneType {
    /// 效率符文——降低 mana_per_100km
    Efficiency,
    /// 冰洋符文——提高 ice_resistance
    IceResistance,
    /// 暴风符文——提高 storm_resistance
    StormResistance,
    /// 速度符文——提高 base_speed_ms（但消耗增加）
    Speed,
    /// 静默符文——降低引擎噪音（对潜行/海盗规避有用）
    Silent,
}
```

### 3.3 运行时状态

pub struct MagicEngineState {

    CasterChanneled { caster: EntityId, amount_ke: f32 },

### 3.4 充能方式——三种路径

```
路径 A: 魔力水晶（常规）
  船长从市场购买魔力水晶(ItemEntId)
  → 消耗水晶 → 引擎 mana_pool += 水晶蕴含的魔力
  → 水晶是消耗品——对标符文经济（工匠制作→市场出售→消耗→再买）
  → 走经济系统订单簿——价格从交易中涌现

路径 B: 施法者注入（紧急/辅助）
  船上或港口有施法者（NPC或玩家）
  → 施法者调用 MagicSystem::expend_mana(caster, amount_ke)
  → 魔法系统处理施法者的储量、过载风险、回复
  → 载具系统收到返回的实际消耗量 → 加入引擎 mana_pool
  → 接口: ManaConsumer trait

路径 C: 港口法阵充能（大规模/昂贵）
  大港口有魔力充能法阵（对标魔法系统 008 大型法阵）
  → 船长付费 → 港口法阵向引擎注入大量魔力
  → 走经济系统——一项付费服务
  → 仅少数大港口有此设施
```

### 3.5 与魔法系统的接口——仅两条

```rust
/// 载具系统 → 魔法系统的唯一接口
/// 对标 WeatherQuery / CultureQuery 的 trait 模式
pub trait ManaConsumer: Send + Sync {
    /// 一个实体向载具引擎充能
    ///
    /// # 参数
    /// * `caster` - 谁在充能（NPC 或 玩家）
    /// * `vehicle_id` - 目标载具
    /// * `amount_ke` - 想充多少刻
    ///
    /// # 返回
    /// * `Ok(actual)` - 实际充入的刻数（可能 < amount_ke——施法者不够、引擎满、速率限制）
    /// * `Err(...)` - 充能失败（施法者无魔力、引擎不在充能范围内、引擎已损坏）
    ///
    /// # 魔法系统内部处理
    /// * 施法者的 mana 储量检查和扣减
    /// * 过载风险（燃灵/透支/借力）
    /// * 灵-血界面永久损伤累积
    /// ——载具系统对这些一无所知，也不需要知道
    fn channel_mana_to_engine(
        &self,
        caster: EntityId,
        vehicle_id: VehicleId,
        amount_ke: f32,
    ) -> Result<f32, ManaChannelError>;
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum ManaChannelError {
    CasterHasNoMana,
    CasterInsufficientMana { available: f32 },
    EngineFull,
    EngineDamaged,
    EngineNotInRange,
    RateLimited { retry_after_seconds: f32 },
}
```

## 四、灾害模型——自然力仅在极端条件下起效

### 4.1 日常航行

```
航速 = base_speed_ms
     × crew_efficiency_factor   (在岗 × 技能 × 疲劳)
     × damage_speed_penalty     (hull/propulsion 损伤)
     × (1.0)                    (日常——不受风向洋流影响)
```

### 4.2 灾害级天气

当 `WeatherQuery::is_extreme_weather(pos)` 返回 `true` 时：

| 灾害类型 | 触发条件（天气系统提供） | 对载具的影响 |
|---------|----------------------|------------|
| 暴风雨/台风 | wind_speed > 阈值 | speed×0.3-0.5，hull 持续损伤，steering 偏移，船长 L3 决策 |
| 浓雾 | visibility < 阈值 | speed×0.3，触礁风险 ×5（瞭望技能→降低风险），自动减速 |
| 冰封 | surface_state=Ice | speed×0.1-0.3，hull 慢速持续损伤（冰挤压船体），航道可能完全阻断 |
| 魔力风暴 | is_magic_anomaly=true（魔法系统判定） | MagicEngine: 魔力池可能过载→propulsion 损伤，符文可能临时失效 |

### 4.3 灾害响应——L3 战略决策

```rust
/// 灾害响应选项——由 VehicleController trait 实现
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum HazardResponse {
    /// 继续航行——船长判断能扛过去（高技能低神经质倾向于此）
    Continue,
    /// 减速通过——降速 50%
    SlowDown,
    /// 转向最近港口（中等风险偏好）
    DivertToNearestPort,
    /// 原地下锚等风暴过去
    DropAnchor,
    /// 弃船——极端情况（hull<0.2 + 无港口在范围 + 船上有救生艇）
    AbandonVehicle,
}
```

NPC 的决策由人格驱动：
- 高神经质 → 容易 Divert/DropAnchor
- 高开放性 → 可能赌 Continue
- 高尽责性+高技能 → 冷静评估→做出最合理的选择（既不是鲁莽也不是怯懦）
- 老兵船长（低神经质+高技能） → "这风暴我见过——能过"

## 五、手动操控——仅小型载具+玩家

```rust
/// 载具操控模式
#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum VehicleControlMode {
    /// 全自动——NPC 和大多数玩家情况
    /// "去东海镇" → 自动沿航线航行
    Autonomous {
        route: Route,
        /// 操控者——NPC 或 玩家（同一 trait）
        controller: EntityId,
    },
    /// 手动——仅 size_class ≤ Small 且 controller=Player 时可用
    /// 对标 Minecraft 划船体验
    Manual {
        throttle: f32,    // -1.0(全速倒退) ~ 1.0(全速前进)
        steer_input: f32, // -1.0(左满舵) ~ 1.0(右满舵)
    },
}
```

| 载具 | 操控模式 |
|------|---------|
| 独木舟 | 玩家可选 Manual；NPC 总是 Autonomous |
| 小渔船 | 同上 |
| 马车 | 玩家可选 Manual（驭马）；NPC 总是 Autonomous |
| 中大型商船 | 仅 Autonomous——一个人操作不了 |
| 火车 | 仅 Autonomous——沿轨道自动 |
| 飞行船 | 仅 Autonomous |
