## 一、核心原则

> **命名不是从名字列表里随机抽——是 NPC 心智在命名那一刻从自己的认知中提取素材，由人格和情绪加权选择。**

载具系统不拥有命名素材——素材从各模块注入。载具系统只提供命名框架和选择机制。

## 二、命名素材的十个来源

```rust
/// NPC 命名载具时的认知素材池
struct VehicleNamingContext {
    namer: NpcId,
    vehicle_def: VehicleDefId,
    time: GameTimestamp,
    location: WorldPos,
}

/// 每个候选素材
struct NamingIngredient {
    /// 素材文本
    text: String,
    /// 来源模块
    source: IngredientSource,
    /// ★ NPC 对该素材的个人情感权重 0-1
    /// 决定选中概率的核心因子
    personal_significance: f32,
}

enum IngredientSource {
    PersonalMemory,   // NPC 的个人记忆
    Faith,            // 信仰系统
    Culture,          // 史诗/传说/图腾
    History,          // 历史人物/战役/传奇
    Life,             // 动物/植物/灵兽
    Magic,            // 元素/法术/魔力现象
    Weather,          // 天象/季节/风雨
    Geography,        // 母港/出生地/向往之地
    Power,            // 头衔/王朝/权力象征
    Economy,          // 财富/贸易品
}
```

| 来源 | NPC 可能使用的素材 | 例子 |
|------|------------------|------|

| **信仰** | 信仰的神祇名、圣兽、神迹事件、圣日、来世概念、经文片段 | "海神之息"、"圣光号"、"轮回号"、"妈祖保佑号" |

## 三、命名模板——文化音系规则提供骨架

```rust
/// 命名模板——从文化系统获取
struct VehicleNameTemplate {
    /// 命名模式
    /// "[修饰词] + [核心词] + [后缀]" 或 "[人名] + [后缀]" 或 自由
    pattern: NamePattern,
    /// 后缀——"号"、"丸"、"船"、"舰"等（文化决定）
    suffix: String,
    /// 前缀——"圣"、"新"等（可选）
    prefix: Option<String>,
}

enum NamePattern {
    /// 修饰词+核心词+后缀: "海燕号"、"铁鲸号"
    ModifierCoreSuffix,
    /// 人名+后缀: "玛丽号"、"哥伦布号"
    PersonSuffix,
    /// 地名+后缀: "东海号"、"伦敦号"
    PlaceSuffix,
    /// 抽象词+后缀: "自由号"、"希望号"
    AbstractSuffix,
    /// 自由文本（玩家命名模式）
    FreeText,
}
```

## 四、命名决策流程

```
船长王五完成了他的第一艘商船的建造
船安静地浮在东海镇码头的滑道尽头

Step 1: GOAP 产生目标 "NameVehicle"
        触发条件: vehicle.current_name 为空 && owner == self

Step 2: 从各模块收集素材
        记忆:
          ├─ "父亲是个老渔夫，说过'要像海燕一样不怕浪'" → "海燕", weight=0.9
          ├─ "女儿小翠昨天在码头上向我挥手" → "翠", weight=0.7
          └─ "我在东海镇出生" → "东海", weight=0.5
        信仰:
          └─ "我信妈祖——妈祖保佑出海人" → "妈祖", weight=0.6
        生命:
          └─ "见过虎鲸在船边游过——一辈子忘不了" → "虎鲸", weight=0.4
        (其他模块返回的素材 weight 低于阈值 0.3 → 被过滤)

Step 3: 生成候选组合
        候选A: "海燕" + "号" → "海燕号"
              素材: 个人记忆（父亲的教诲）
              personal_significance: 0.9
        候选B: "翠翠" + "号" → "翠翠号"
              素材: 个人记忆（女儿小翠）
              personal_significance: 0.7
        候选C: "妈祖" + "保佑" + "号" → "妈祖保佑号"
              素材: 信仰
              personal_significance: 0.6
              但——太长，可能不适合船名（文化音系规则过滤）
        候选D: "东海" + "海燕" + "号" → "东海海燕号"
              复合素材
              personal_significance: 0.5+0.9/2 = 0.7

Step 4: 人格过滤
        王五: 中等开放性(0.5), 高宜人性(0.8), 低神经质(0.2)
        → 偏见: 温暖、稳重、有人情味的名字
        → 浮夸的名字(如"无敌号"、"毁灭者号")被自动排除
        → 过于矫情的名字(如"永远的爱号")不太可能
        → 最终: "海燕号"——简单、温暖、有父亲的影子

Step 5: 命名
        VehicleState.current_name = "海燕号"
        VehicleState.original_name = "海燕号"
        历史系统记录: "王五将新建的商船命名为'海燕号'"
        王五的记忆新增: "我的船叫海燕号——取自父亲的话"
```

## 五、改名与命名历史

```rust
/// 载具改名——VehicleState 中有完整追踪
fn rename_vehicle(
    vehicle: &mut VehicleState,
    new_name: String,
    renamed_by: EntityId,
    reason: Option<String>,
) {
    let now = current_game_time();

    // 记录旧名
    vehicle.name_history.push((
        vehicle.current_name.clone(),
        now,
        renamed_by,
    ));

    // 更新当前名
    vehicle.current_name = new_name;

    // 历史系统记录
    record_historical_event(
        format!("船'{}'被{}改名为'{}'",
            vehicle.name_history.last().unwrap().0,
            renamed_by,
            vehicle.current_name,
        )
    );

    // 改名者的记忆
    add_memory(renamed_by, format!("将船改名为'{}'", vehicle.current_name));
}
```

改名场景：
- 玩家购买二手船 → 改名（"老李的破船"→"飞鱼号"）
- 载具被俘虏 → 海盗船长改名（"皇家海军坚毅号"→"黑珍珠号"）
- 政治变动 → 官方改名（"国王号"→"自由号"——革命后）
- 纪念某人 → 以恩人命名

老年 NPC 可能仍然使用旧名（"——那艘船以前叫海燕号，现在年轻人叫她飞鱼号"）。走记忆系统——旧名不删除。
