## 一、VehicleQuery trait —— 所有模块的唯一只读入口

```rust
/// 载具系统只读查询接口
/// 对标 CultureQuery / FaithQuery / WeatherQuery 模式
/// 零分配——所有返回值 Copy 或引用
pub trait VehicleQuery: Send + Sync {
    // ═══════════════════════════════════════════════════════
    // 基础查询（O(1)，缓存友好，最热路径）
    // ═══════════════════════════════════════════════════════

    /// 获取载具的类型定义
    fn vehicle_def(&self, id: VehicleId) -> Option<&VehicleDef>;

    /// 获取载具的完整只读快照
    fn vehicle_state(&self, id: VehicleId) -> Option<&VehicleState>;

    /// 载具当前世界位置——渲染 + 天气用
    fn position(&self, id: VehicleId) -> Option<WorldPos>;

    /// 载具损伤状态——战斗系统热路径（每帧可能查询）
    fn damage_state(&self, id: VehicleId) -> Option<&VehicleDamageState>;

    /// 载具运动向量——战斗系统（瞄准修正用）
    fn motion_vector(&self, id: VehicleId) -> MotionVector;

    // ═══════════════════════════════════════════════════════
    // 战斗环境查询（每帧——仅参与战斗的少数载具）
    // ═══════════════════════════════════════════════════════

    /// 载具上所有碰撞体（在载具本地空间）
    fn collision_bodies(&self, id: VehicleId) -> &[CollisionShape];

    /// 可作为掩体的表面（窗框/桅杆/护栏——载具本地空间）
    fn cover_surfaces(&self, id: VehicleId) -> &[CoverSurface];

    // ═══════════════════════════════════════════════════════
    // 空间查询
    // ═══════════════════════════════════════════════════════

    /// 某位置半径内的所有载具
    fn vehicles_in_radius(&self, center: WorldPos, radius_m: f32) -> Vec<VehicleId>;

    /// 某港口停泊的所有载具（Docked 状态）
    fn vehicles_at_port(&self, settlement: SettlementId) -> Vec<VehicleId>;

    // ═══════════════════════════════════════════════════════
    // 所有权查询
    // ═══════════════════════════════════════════════════════

    /// 某实体拥有的所有载具
    fn vehicles_owned_by(&self, owner: EntityId) -> Vec<VehicleId>;

    /// 某 NPC 是否是指定载具的船员（在岗）
    fn is_crew_on_duty(&self, npc: NpcId, vehicle: VehicleId) -> bool;

    /// 载具的所有权完整记录
    fn ownership_record(&self, id: VehicleId) -> Option<&VehicleOwnershipRecord>;

    // ═══════════════════════════════════════════════════════
    // 航线/运输查询
    // ═══════════════════════════════════════════════════════

    /// 查询从 A 到 B 的可用运输选项（有舱位 + 航线经过/愿意接单）
    fn available_transport(
        &self,
        from_settlement: SettlementId,
        to_settlement: SettlementId,
        cargo_kg: f32,
    ) -> Vec<TransportOption>;

    // ═══════════════════════════════════════════════════════
    // 经济查询
    // ═══════════════════════════════════════════════════════

    /// 载具的物理参数——NPC 经济心智用
    fn physical_profile(&self, id: VehicleId) -> Option<&VehiclePhysicalProfile>;

    /// 载具当前可用货舱（已用kg, 最大kg, 已用升, 最大升）
    fn cargo_space(&self, id: VehicleId) -> (f32, f32, f32, f32);

    // ═══════════════════════════════════════════════════════
    // 港口条件查询——世界生成用
    // ═══════════════════════════════════════════════════════

    /// 给定自然条件，此位置适合建什么规模的港口设施
    fn port_suitability(
        &self,
        pos: WorldPos,
        water_depth_m: f32,
        coastal_features: &CoastalFeatures,
    ) -> PortSuitability;
}

/// 运输选项——available_transport() 的返回
#[derive(Debug, Clone)]
pub struct TransportOption {
    pub vehicle_id: VehicleId,
    /// 载具名——"飞鱼号"
    pub vehicle_name: String,
    /// 载具类型标签——"中型商船"
    pub vehicle_type_label: String,
    /// 所有者
    pub owner: EntityId,
    /// 可用货舱（kg）
    pub available_cargo_kg: f32,
    /// 可用客舱
    pub available_passenger_slots: u32,
    /// 航线——"东海镇→北海港→南海城"
    pub route_description: String,
    /// 预计航行时间（游戏天数）
    pub estimated_days: f32,
    /// 船长报价（铜/100kg——如果有标准价）
    pub reference_freight_rate_copper_per_100kg: Option<u32>,
    /// 出发时间
    pub departure_time: Option<GameTimestamp>,
}
```

## 二、VehicleMut trait —— pub(crate) 仅授权模块写入

```rust
/// 载具修改接口——仅限授权模块
/// 对标 CultureMut / FaithMut 模式
pub(crate) trait VehicleMut {
    // ── 导航 ──
    /// 更新载具导航（每决策周期——1-10s）
    fn update_navigation(&mut self, id: VehicleId, dt: GameDuration, weather: &dyn WeatherQuery);

    // ── 损伤 ──
    /// 对载具本体造成伤害（战斗系统/天气系统调用）
    fn apply_damage(&mut self, id: VehicleId, amount: f32, channel: DamageChannel, source: DamageSource);

    /// 执行紧急修补（船员 NPC 调用）
    fn perform_emergency_patch(&mut self, id: VehicleId, channel: DamageChannel, npc: NpcId) -> Result<(), PatchError>;

    /// 港口全量维修
    fn port_repair(&mut self, id: VehicleId, settlement: SettlementId) -> Result<(), RepairError>;

    // ── 船员 ──
    /// 设置船员岗位（船长/玩家调用）
    fn assign_crew(&mut self, id: VehicleId, slot: CrewSlotId, npc: Option<NpcId>);

    // ── 货舱 ──
    /// 装入货物（走物品系统 Container 接口）
    fn load_cargo(&mut self, id: VehicleId, items: Vec<ItemEntId>) -> Result<(), CargoError>;

    /// 卸出货物
    fn unload_cargo(&mut self, id: VehicleId, items: Vec<ItemEntId>) -> Result<(), CargoError>;

    // ── 生命周期 ──
    /// 注册新载具（建造完成时）
    fn register_vehicle(&mut self, vehicle: VehicleState);

    /// 注销载具（沉没/彻底拆解）
    fn deregister_vehicle(&mut self, id: VehicleId) -> Result<(), VehicleError>;

    // ── 状态变更 ──
    /// 设置载具状态（停泊/维修/废弃等）
    fn set_status(&mut self, id: VehicleId, status: VehicleStatus);

    /// 设置所有权——仅 Adjudicate 结果调用
    fn set_owner(&mut self, id: VehicleId, new_owner: EntityId, event: OwnershipEventKind);

    /// 设置载具名称
    fn set_name(&mut self, id: VehicleId, new_name: String, renamed_by: EntityId);
}
```

## 三、消费者矩阵

| 消费者 | 使用的方法 | 频率 | 热度 |
|--------|----------|------|------|
| **NPC GOAP** | `vehicles_at_port`, `available_transport`, `vehicles_owned_by`, `is_crew_on_duty`, `physical_profile` | 每决策周期 (1-10s) | 中 |
| **战斗系统** | `damage_state`, `motion_vector`, `collision_bodies`, `cover_surfaces` | 每帧 (16ms) | 🔴 最高 |
| **天气系统** | `position`（判断载具是否在灾害区域内） | 每天气更新 (1-10s) | 中 |
| **经济系统** | `physical_profile`, `cargo_space`, `available_transport` | NPC 商业决策时 | 低 |
| **权力系统** | `ownership_record`, `vehicles_owned_by`（产权争议裁决时） | 偶尔 | 极低 |
| **历史系统** | `vehicle_state`（全量快照→记录事件） | 事件发生时 | 极低 |
| **世界生成** | `register_vehicle`（P8.5 初始分布） | 一次性 | — |
| **技能系统** | `CrewSlotDef.required_skill`（被动引用——不调用任何方法） | — | — |
| **语言表达** | `CrewSlotDef.label_key`（船员岗位名称本地化） | — | — |
| **文化系统** | `CulturalShipbuildingTradition::derive()` 消费 CultureCoreParams（只读） | 世界生成时 | — |
| **魔法系统** | `ManaConsumer::channel_mana_to_engine()` 由载具系统调用 | 充能时 | 低 |
| **物品系统** | `cargo_hold: ContainerId` → 走物品系统的 Container 接口 | 装卸时 | 低 |
| **建造系统** | `VehicleConstructionSpec`（消费——建造执行方） | 建造时 | 低 |
| **Godot 渲染** | `position` + `world_transform`——通过共享内存直接读（不走 trait） | 每帧 | 🔴 最高 |

## 四、与其他模块的契约变更——精确操作

| # | 现有内容 | 问题 | 解决方案 | 需改动 |
|---|---------|------|---------|--------|
| 1 | **天气 004 §4.9**: `wind→帆船航速`、`风角修正`、`魔力驱动船/帆船区分` | 与技术栈方案+载具设计矛盾——日常航行不应受自然力影响 | **删除** wind→航速计算。仅保留灾害级: `is_extreme_weather→减速`, `storm_front_distance→航行危险`, `visibility→触礁风险`, `surface_state(Ice)→冰封航线`, `gust_factor→倾覆风险`。删除"魔力船vs帆船"的细化区分——所有载具统一灾害响应 | `天气与季节系统/004-跨模块接口与数据合同.md` §4.9 |
| 2 | **世界生成 006 §3.2**: `compute_sailing_time()` 用洋流/季风修正航速 | 航速应由载具系统 `VehicleDef.base_speed_ms` 定义 | 改为 `time = distance_km / VehicleQuery::physical_profile(id).base_speed_ms / 3600.0`。删除 `OceanCurrents`/`wind_factor`/`season` 参数 | `世界生成/006-交通网络.md` §3.2 |
| 3 | **NPC ver2.0**: `CrewRank`、`CrewDuty`、`VehicleRole`、`CrewSlot` 为未定义幽灵类型 | 这些类型应由载具系统权威定义 | NPC 文档中的引用改为 `use vehicle_system::CrewSlotDef` 等。载具系统定义 `CrewSlotDef`、`CrewDuty`、`CrewRank`、`VehicleRole` | `NPC活人感模块/NPC活人感开发文档ver2.0.md` |
| 4 | **战斗 011**: "马不是载具"——边界确认正确。但"载具上战斗"接口未定义 | 载具应提供战场环境数据 | 载具系统提供 `VehicleQuery::collision_bodies/cover_surfaces/motion_vector/damage_state`。战斗系统新增引用 | `战斗/011-潜行伏击与骑乘战斗.md` |
| 5 | **技术栈方案 §十**: "魔力驱动——不考虑风力和洋流" | 表述狭隘——现有5种动力类型+灾害约束设计 | 更新为: "动力来源多样（魔力/人力/畜力/风力/混合）。日常航行不受自然力影响。灾害级天气对所有动力类型施加硬约束" | `技术栈方案/001-WoWorld正式技术栈方案v3.md` §十 |
| 6 | **物品系统 002**: 契书 ItemCategory | 契书对标 Financial(欠条→Category 0x50) ——可复用或新增 VehicleDeed 子类 | 建议: 复用 `ItemCategory::Financial`——VehicleDeed 和 IOU 同属金融文书。如后续有特殊需求可新增子类 | `物品系统/002-物品分类与ID体系.md` |
| 7 | **经济系统**: 无载具运营接口 | 不需改动——运营成本由 NPC 心智自行计算。运费从对话+Contract 涌现。燃料/维修走市场订单簿。 | — | 无需改动 |
