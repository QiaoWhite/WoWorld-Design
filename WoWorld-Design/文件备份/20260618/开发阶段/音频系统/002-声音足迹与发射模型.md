# 002 — 声音足迹与发射模型 v1.0

> **核心设计**：声音不是"脚步声"或"挥剑声"——它是**表面接触 × 运动 × 发声器官振动 × 物品交互 × 身体状态**的物理组合。各业务模块描述物理现实，音频模块完全拥有"物理→声音"的映射逻辑。加一种新声音 ≠ 改业务模块代码。

> **对标模式**：`ConsumableEffect` schema（Life 定义结构，物品存储不解）· `HasAestheticSignal` trait（消费方定义 trait，提供方 impl）· `ReligiousPracticeProfile` 存储模式（小 Copy 类型存储于实体，32B 级）。

## §一 SoundFootprint：实体物理声学快照

### 1.1 定义（woworld_types）

```rust
/// 实体的当前物理声学状态。由各业务模块维护——音频模块每帧轮询。
/// Copy 类型（？待定——取决于 SmallVec 是否实现 Copy。不实现 Copy 则用 Arc 或每帧 clone）
/// 每实体 ~128 bytes，栈上或 inline 存储。
#[derive(Clone)]
pub struct SoundFootprint {
    /// 当前表面接触（双脚/多足踩在不同材质上，可 0-4 个）
    pub surface_contacts: SmallVec<[SurfaceContact; 4]>,

    /// 当前运动状态
    pub motion: MotionSnapshot,

    /// 当前主动发声（说话/喊叫/咆哮/唱歌/鸣叫/呻吟）
    pub vocalization: Option<VocalizationState>,

    /// 当前物品交互（挥动/使用/操作/掉落）
    pub item_interaction: Option<ItemInteraction>,

    /// 是否在主动抑制声音
    pub silence: Option<SilenceIntent>,
}
```

## §二 ActionRingBuffer：瞬时动作追踪

### 2.1 设计

```rust
/// 小型环形缓冲区——记录最近 64 个瞬时动作（~1s @ 60fps）。
/// 各模块本来就要记录动作日志（给决策器回顾"刚才干嘛了"、记忆系统、大日志）。
/// 音频模块只是多了一个消费者——零额外开销。
#[derive(Clone)]
pub struct ActionRingBuffer {
    entries: [ActionEntry; 64],
    head: u8,
}

#[derive(Clone, Copy)]
pub struct ActionEntry {
    pub action_type: ActionAtom,  // 动作类型（~40 变体）
    pub frame: u16,               // 发生帧号
    pub metadata: u32,            // 模块自定义（步态编码/力度/法术ID）
}
```

## §四 身体声音（直接读 Vitals）

身体内部声音（心跳/呼吸/肚子叫）**不需要 HasSoundEmitter**。音频模块直接通过已有查询读取 Vitals：

```rust
// 音频模块内部——ingestion/body_sounds.rs
fn compute_body_sounds(
    vitals: &Vitals,          // 来自 Life 已有查询——不新增接口
    emotion: &EmotionState,   // 来自 NPC 已有数据
    race_heart_rate: f32,     // 种族基础心率
    rng: &mut DeterministicRng,
) -> SmallVec<[ContinuousSound; 3]> {
    let mut sounds = SmallVec::new();

    // 心跳——持续存在
    let heart_rate_mult = 1.0 + emotion.fear * 0.5
        + vitals.exertion() * 0.3
        + (1.0 - vitals.health()) * 0.4;
    let heart_intensity = if vitals.health() < 0.3 { 2.0 } else { 0.3 };
    sounds.push(ContinuousSound {
        sound_type: ContinuousSoundType::Heartbeat,
        intensity_db: 20.0 + heart_intensity * 15.0,
        frequency: FrequencyProfile::heartbeat(race_heart_rate * heart_rate_mult),
        metadata: 0,
    });

    // 粗重呼吸——体力耗尽
    if vitals.stamina() < 0.3 {
        sounds.push(ContinuousSound {
            sound_type: ContinuousSoundType::Breathing,
            intensity_db: 15.0 + (1.0 - vitals.stamina()) * 30.0,
            frequency: FrequencyProfile::breath(),
            metadata: 0,
        });
    }

    // 肚子叫——概率性（不每帧：hunger>0.6 → 2-10%/帧）
    if vitals.hunger() > 0.6 && rng.chance(0.02 * vitals.hunger()) {
        // 瞬时声音——直接写入环形缓冲区（音频模块内部路径）
        // 这是音频模块内部行为——不对任何业务模块暴露
    }

    sounds
}
```

**关键**：音频模块通过 Life 查询获取 Vitals——**不修改 Life 模块，不新增接口**。

### 6.6 HearingModel

```rust
/// 实体的听觉感知参数。由 Life 模块在创建实体时初始化。
/// 对标 Physiology 从 Vitals 派生的模式。
#[derive(Clone, Copy)]
pub struct HearingModel {
    /// 听觉阈值 (dB SPL) —— 低于此不感知
    pub threshold_db: f32,         // 健康人类 ~10dB

    pub max_range_m: f32,          // 人类 ~200m
```
