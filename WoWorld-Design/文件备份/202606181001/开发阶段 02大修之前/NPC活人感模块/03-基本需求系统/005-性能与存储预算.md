## 二、运行时计算预算

### 2.1 热路径（每决策周期, L1 NPC）

| 操作 | 频率 | 成本 | 注 |
|------|------|------|-----|
| `from_vitals()` 新增字段 | 每次决策 | ~5ns | 2 个字段 copy + 1 个 max/min 运算 |
| `need_action_match()` | 每决策,每次 action 评分 | ~50ns | 平均 3 个 need × 查表(5ns/次)+ 除法 |
| `apply_physiological_pull()` 新增 | 每次决策 | ~6ns | 3 个条件判断 + 3 个 f32 乘加 |
| **热路径新增总计** | | **~60ns/NPC** | |
| **占 L1 预算 (12.5μs)** | | **0.48%** | |

总: 对 1000 L1 NPC, 每 0.2s → 60ns × 1000 = 60μs → 在 200ms 帧时间中占 0.03%。

### 2.2 冷路径（事件驱动 / 每日 tick）

| 操作 | 频率 | 成本 | 注 |
|------|------|------|-----|
| `ingest_food()` — 瓶颈模型 | 进食事件 (~2次/NPC/日) | ~100ns | 8 × f32 乘加 + 7 次 min 比较 |
| `vent_surplus()` — 主动释放 | ~1次/3-5日 (偏科 NPC) | ~5ns | 8 次 f32 归零 (memset) |
| `tick_passive_vent()` — 睡眠释放 | 每游戏日 (仅睡眠 NPC) | ~20ns | 8 次 f32 乘法 |
| `tick_libido_cycle()` | 每游戏日 | ~50ns | 1 次 sin 查表 + 2 次 f32 乘法 |
| `tick_social_deficit()` | 每游戏日 | ~10ns | 1 次 f32 乘加 |

**所有冷路径操作**：分摊到每游戏日的不同帧执行。100K NPC × (100+50+20+10)ns = 18ms → 在 1 个游戏日（默认 60min，玩家可调）内分摊 → 每帧 <<1ns。完全可忽略。

### 2.3 时间跳跃批量处理

```
游戏跳时间 (如睡觉 8h 或快速旅行几天):

1. 体力值批量更新 (已有逻辑):
   hunger -= rate × dt (保护: 不低于 0, 不致死)
   thirst -= rate × dt
   fatigue += rate × dt

2. 新增批量更新:
   element_surplus (睡眠时): surplus[i] *= (1 - 0.05 × dt_days)
   libido: 按绝对时间重新计算正弦位置 (不积分——避免精度累积)
   social_deficit: 每 24h 无互动 → +0.02

   批量操作成本: ~200ns/NPC × 100K = 20ms 一次性
   在加载画面/黑屏过渡中执行, 对玩家无感。
```

## 三、数据局部性 (SoA 友好度)

### 3.1 访问模式

```
NPC 决策时触发的访问:

Life.vitals[hunger]       → contiguous f32 array, cache-line friendly
Life.vitals[thirst]       → contiguous f32 array
Life.vitals[fatigue]      → contiguous f32 array
Life.vitals[element_surplus_0..7] → 8 contiguous f32 arrays ★ 新增
Life.vitals[libido]        → contiguous f32 array ★ 新增

NPC.social_deficit         → contiguous f32 array ★ 新增

全部顺序访问——rayon par_iter 天然受益于 cache prefetch。
```

### 3.2 向量化友好度

全部 f32 运算——编译器可自动向量化（SSE/AVX）。element_surplus 的 8 个元素同步处理最适合 SIMD。

## 四、年度/多年度模拟性能

### 场景：模拟 50 年世界历史

```
50 年 = 50 × 120 游戏日 = 6,000 游戏日
NPC 进食: ~2次/日 × 100K NPC × 6,000 = 12 亿次进食事件

每次进食的元素消化计算: ~100ns
总: 12亿 × 100ns = 120s → 2 分钟

但历史模拟期间可以离线批量处理：
- 使用简化公式 (不实际计算瓶颈模型)
- 用统计分布近似: 给定群系×文化→直接输出平均 surplus
- <1s 即可完成 50 年模拟
```
