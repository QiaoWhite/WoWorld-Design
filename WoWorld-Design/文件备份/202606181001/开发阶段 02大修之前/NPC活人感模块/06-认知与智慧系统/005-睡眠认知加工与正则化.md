## 一、理论框架：过拟合大脑假说

### 1.1 核心类比

**大脑≈深度神经网络。日常经验≈训练数据。梦境≈正则化。**

如果神经网络只训练而没有正则化→过度拟合——死记训练数据的表面细节，丧失对未见数据的泛化能力。

如果NPC只积累记忆而不做正则化→MentalModel过度拟合——信念太具体、无法应用于新情境。

**睡眠是NPC心智的"泛化维护"时段。**

### 1.2 两种互补正则化

| 正则化类型 | 对应睡眠阶段 | 神经机制类比 | NPC机制 |
|-----------|------------|------------|---------|
| **L1正则化**（稀疏化） | 深睡期（NREM Deep） | 突触修剪、弱连接消除 | 弱MentalModel→abandoned、低impact记忆→加速衰减 |
| **Dropout正则化**（噪声注入） | REM/做梦期 | 随机激活、合成经验生成 | 记忆碎片缝合+噪声注入→荒诞梦境→泛化检验 |

### 1.3 适应性与非适应性

梦境荒诞度不是随机的——它**自适应调节**：

```
今日经验窄度↑ → 正则化需求↑ → 梦境荒诞度↑
今日经验多样 → 正则化需求↓ → 梦境平淡或不做梦
```

### 1.4 对NPC设计的直接推论

- **不睡觉的NPC不只是"累"——他们变得认知僵化。** MentalModel无法泛化，在面对新情境时失败率飙升。
- **梦境是MentalModel泛化的必要维护操作。** 不是可有可无的有趣附加。
- **经验窄度（重复工作）→更强的梦境噪声。** 铁匠每晚做荒诞梦是健康的——ta的锻造MentalModel被正则化保护。
- **经验丰富（冒险者）→弱梦境或无梦。** 不需要额外正则化。

## 二、睡眠阶段数学抽象

### 2.1 四阶段连续模型

```rust
/// 四个连续函数的叠加干涉——不对标离散状态机

pub struct SleepState {
    pub phase: SleepPhase,
    pub progress: f32,          // 0-1 over each ~90min cycle
    pub elapsed: Duration,
    pub quality: f32,           // 综合睡眠质量
}

pub enum SleepPhase {
    /// 入睡期（0-30min）：认知负载指数衰减，思绪→荒诞
    Transition { hypnotic_imagery: bool },
    
    /// 浅睡期（~60%）：记忆巩固主战场
    NremLight { consolidation_progress: f32 },
    
    /// 深睡期（~20%）：记忆筛选 / L1正则化
    NremDeep { pruning_progress: f32 },

    Rem { dream_vividness: f32, cycle_number: u8 },
}
```

### 2.2 周期循环

- 每个睡眠周期 ≈ 90分钟游戏时间
- 完整睡眠 = 4-6个周期
- REM在后期周期中占比增大

## 三、经验窄度追踪

### 3.1 定义

```rust
/// 今天NPC的经验有多"窄"（重复性高）？
/// 每游戏日结束时计算一次（或睡眠开始时计算）
pub fn compute_experience_narrowness(
    today_percepts: &[PerceptEntry],     // 今天的感知序列
    today_actions: &[ActionRecord],      // 今天的行动序列
    today_interactions: &[SocialInteraction], // 今天的社交
) -> f32 {
    let percept_diversity = compute_shannon_entropy(today_percepts, |p| p.entity_type_tag());
    let action_diversity = compute_shannon_entropy(today_actions, |a| a.category());
    let social_diversity = (today_interactions.iter()
        .map(|i| i.partner_id)
        .unique()
        .count() as f32 / 10.0).min(1.0);
    
    // 窄度=1-多样性
    1.0 - (percept_diversity * 0.4 + action_diversity * 0.3 + social_diversity * 0.3)
}

fn compute_shannon_entropy<T, F>(items: &[T], tagger: F) -> f32 
where F: Fn(&T) -> u32 
{
    // 信息熵标准化到[0,1]
    let mut counts = BTreeMap::new();
    for item in items { *counts.entry(tagger(item)).or_insert(0) += 1; }
    let total = items.len() as f32;
    let entropy: f32 = counts.values()
        .map(|&c| { let p = c as f32 / total; if p > 0.0 { -p * p.log2() } else { 0.0 } })
        .sum();
    // 最大熵=log(n_categories)，标准化
    let max_entropy = (counts.len() as f32).max(2.0).log2();
    (entropy / max_entropy).clamp(0.0, 1.0)
}
```

### 3.2 窄度→正则化需求


## 四、记忆三阶段处理

### 4.1 阶段1：标签（入睡期）

```rust
fn sleep_memory_tagging(
    memory_store: &MemoryStore,
    recent_days: u32,  // 扫描最近几天的记忆
) -> SleepMemoryClassification {
    let mut candidates_for_consolidation = Vec::new();
    let mut candidates_for_discard = Vec::new();
    
    for memory in memory_store.scan_recent(recent_days) {
        let relevance = memory.impact_score * 0.5
                      + memory.emotional_encoding.intensity() * 0.3
                      + (memory.access_count as f32 / 10.0).min(1.0) * 0.2;
        
        if relevance > 0.2 {
            candidates_for_consolidation.push(memory.id);
        } else {
            candidates_for_discard.push(memory.id);
        }
    }
    
    SleepMemoryClassification { candidates_for_consolidation, candidates_for_discard }
}
```

### 4.2 阶段2：巩固（浅睡期）

```rust
fn sleep_memory_consolidation(
    memory_store: &mut MemoryStore,
    candidates: &[MemoryId],
    quality: f32,
    neuroticism: f32,
) {
    for &mem_id in candidates {
        if let Some(memory) = memory_store.get_mut(mem_id) {
            // 巩固强度
            let strength = sleep_consolidation_strength(memory, neuroticism) * quality;
            
            // 强化decay_factor（减缓遗忘）
            memory.decay_factor = (memory.decay_factor * (1.0 - strength * 0.3)).max(0.0);
            
            // 高impact+高巩固→可能标记为important
            if memory.impact_score > 0.7 && strength > 0.8 {
                memory_store.mark_important(mem_id);
            }
        }
    }
}

fn sleep_consolidation_strength(memory: &EventMemory, neuroticism: f32) -> f32 {
    let arousal = memory.emotional_encoding.arousal;
    let replay = (memory.access_count as f32 / 10.0).min(1.0);
    
    // 负面情绪记忆→在深睡中得到优先巩固（演化安全优先）
    let negativity = if memory.emotional_encoding.valence() < 0.0 {
        1.0 + neuroticism * 0.5
    } else {
        1.0
    };
    
    (arousal * 0.3 + replay * 0.25 + negativity * 0.2 + memory.impact_score * 0.25)
        .clamp(0.0, 1.0)
}
```

### 4.3 阶段3：筛选（深睡期）

```rust
fn sleep_memory_pruning(
    memory_store: &mut MemoryStore,
    candidates: &[MemoryId],
    quality: f32,
) {
    // 极低impact+无情绪附着+长时间未被访问→加速衰减
    for &mem_id in candidates {
        if let Some(memory) = memory_store.get_mut(mem_id) {
            let pruning_factor = 1.0 - memory.impact_score * quality;
            memory.decay_factor = (memory.decay_factor * (1.0 + pruning_factor * 0.02)).min(1.0);
        }
    }
}
```

## 五、情绪复位

### 5.1 机制

```rust
/// 对标情绪系统已有基线漂移（emotion → personality baseline）
/// 睡眠中回归速度是清醒时的 3-5×
pub fn sleep_emotional_reset(
    emotion: &mut EmotionState,
    quality: f32,
    personality: &BigFive,
) {
    let reset_strength = quality * 0.3;  // 0.15-0.3 range
    
    // Pleasure向人格基线回归
    emotion.pleasure += (emotion.baseline_pleasure - emotion.pleasure) * reset_strength;
    
    // Arousal降低
    emotion.arousal *= 1.0 - reset_strength * 0.5;
    
    // Control向基线回归
    emotion.control += (emotion.baseline_control - emotion.control) * reset_strength * 0.5;
    
    // 保留：高impact记忆的情绪编码（已经在记忆中持久化）——不修改
    // 清除：低强度日常负面情绪摩擦（对标已有情绪惯性）
}
```

## 六、MentalModel正则化

### 6.1 深睡期 L1 正则化

```rust
fn sleep_l1_regularization(
    models: &mut [MentalModel],
    quality: f32,
    style: &CognitiveStyle,
) -> Vec<MentalModelEvent> {
    let mut events = Vec::new();
    
    for model in models.iter_mut().filter(|m| !m.abandoned) {
        // 过拟合检测：高confidence + 低diversity
        let overfit = model.confidence > 0.7 && model.supporting_diversity < 0.3;
        
        if overfit {
            // L1正则化：削弱过度拟合的模型
            let penalty = quality * 0.15 * (1.0 - model.supporting_diversity);
            model.confidence = (model.confidence - penalty).max(0.1);
            
            if penalty > 0.05 {
                events.push(MentalModelEvent::Regularized { model_id: model.id });
            }
        }
        
        // 弱模型剪枝
        if model.confidence < 0.15 && model.supporting_count < 3 && model.emotional_charge < 0.3 {
            model.abandoned = true;
            events.push(MentalModelEvent::Pruned { model_id: model.id });
        }
    }
    
    events
}
```

### 6.2 REM期 Dropout 正则化

```rust
fn rem_dropout_regularization(
    models: &[MentalModel],
    memory: &MemoryStore,
    quality: f32,
    style: &CognitiveStyle,
    narrowness: f32,
    rng: &mut Pcg64,
) -> Vec<SleepInsight> {
    let mut insights = Vec::new();
    
    // 荒诞度 = 自适应正则化强度
    let distortion = narrowness * 0.7 
                   + style.abstract_concrete * 0.2 
                   + (1.0 - style.rigid_flexible) * 0.1;
    
    // 跨界关联：随机采样不同domain的模型对
    let pairs = sample_cross_domain_model_pairs(models, 5, rng);
    
    for (model_a, model_b) in &pairs {
        let similarity = compute_pattern_similarity(&model_a.pattern, &model_b.pattern);
        
        // 睡眠中跨界关联门槛更低（对标：清醒思维的有序 vs 睡眠思维的松弛）
        let threshold = 0.4 - (1.0 - quality) * 0.15;  // 睡眠质量好→更低门槛
        
        if similarity > threshold && rng.gen_bool((similarity * 0.4) as f64) {
            let insight = SleepInsight {
                source_models: [model_a.id, model_b.id],
                new_synthesis: synthesize_model(model_a, model_b, similarity, distortion),
                confidence: 0.15 + similarity * 0.10,  // 0.15-0.25
                clarity: quality * style.abstract_concrete,
            };
            insights.push(insight);
        }
    }
    
    insights
}
```

### 7.2 梦境生成与惰性文本

```rust
pub fn generate_dream(
    npc: &NpcData,
    quality: f32,
    narrowness: f32,
    rng: &mut Pcg64,
) -> Option<Dream> {
    if quality < 0.3 || !rng.gen_bool(0.85) { return None; }
    
    // 选择当天最高impact的记忆作为梦的核心素材
    let core_memory = npc.memory.scan_today().max_by_key(|m| m.impact_score)?;
    
    // 荒诞度 = f(窄度, 开放性, 年龄)
    let distortion = (narrowness * 0.5 
                    + npc.cognitive_style.abstract_concrete * 0.3
                    + (1.0 - npc.mental_age()) * 0.2)  // 年轻人梦更荒诞
                    .clamp(0.0, 1.0);
    
    // 情绪色调
    let emotional_tone = core_memory.emotional_encoding.valence() 
                       * (1.0 - distortion * 0.5);  // 荒诞→弱化原始情绪
    
    // 鲜明度
    let vividness = (quality * 0.5 
                   + npc.cognitive_style.abstract_concrete * 0.3
                   + npc.cognitive_style.reflective_impulsive * 0.2)
                   .clamp(0.0, 1.0);
    
    // 只存储seed——文本惰性生成
    let dream = Dream {
        seed: rng.gen(),
        emotional_tone,
        vividness,
        distortion_level: distortion,
        key_elements: extract_dream_elements(core_memory, npc, distortion, rng),
    };
    
    // 梦境记忆化：vividness>0.6→醒来后编码Dream记忆
    // （在sleep_cognitive_processing结束时执行）
    
    Some(dream)
}
```

## 八、睡眠质量派生

### 8.1 六因子派生

```rust
/// 纯函数——从已有系统全部获取
/// 对标已有模式的综合性派生
pub fn compute_sleep_quality(
    sleep_duration: Duration,
    sleep_continuity: f32,       // 0=多次中断, 1=连续
    health: f32,                 // 从Life.Vitals.health派生
    pain_level: f32,             // 从Physiology.pain派生
    environmental_noise: f32,    // 从AudioQuery.sample()获取
    environmental_comfort: f32,  // 床+温度+地面（从物品系统+WeatherQuery）
    neuroticism: f32,            // 从BigFive派生
) -> f32 {
    // 1. 时长质量
    let duration_quality = sigmoid((sleep_duration.as_hours() - 6.0) * 2.0);
    
    // 2. 生理因素
    let physiological = (health * 0.3 + (1.0 - pain_level) * 0.2 + environmental_comfort * 0.2).clamp(0.0, 1.0);
    
    // 3. 环境因素
    let environmental = ((1.0 - environmental_noise) * 0.15 + sleep_continuity * 0.15).clamp(0.0, 1.0);
    
    // 综合
    let raw = duration_quality * 0.4 + physiological * 0.3 + environmental * 0.3;
    
    // 人格调制：neuroticism→降低主观睡眠质量
    (raw * (1.0 - neuroticism * 0.3)).clamp(0.0, 1.0)
}
```

### 8.2 睡眠质量→次日认知表现

| 睡眠质量 | cognitive_load基线 | quietude初始值 | meta-cognition可用 | 情绪基线回归 |
|---------|-------------------|---------------|-------------------|------------|
| >0.8（好） | -0.05 | 0.64 | 完全可用 | 充分回归(0.3) |
| 0.5-0.8（一般） | ±0 | 0.4 | 可用 | 正常回归(0.2) |
| 0.3-0.5（差） | +0.05 | 0.24 | 受限 | 部分回归(0.1) |
| <0.3（极差） | +0.1 | 0.12 | 几乎不触发 | 很少回归(0.05) |

```rust
pub fn sleep_parameters_for_species(race: &RaceTraits, mental_age: f32) -> SleepParameters {
    let base = race.base_sleep_requirements;
    
    // 学习快→需要更多REM正则化
    let learning_rate = if mental_age < 0.3 { 1.5 } else if mental_age < 0.7 { 1.0 } else { 0.6 };
    
    SleepParameters {
        ideal_duration_hours: base.duration_hours * learning_rate,
        rem_ratio: base.rem_ratio * learning_rate,
        deep_sleep_ratio: base.deep_sleep_ratio,  // 深睡不随年龄大幅变化
        dream_vividness_boost: (learning_rate - 1.0) * 0.5,
        cross_domain_association_boost: (learning_rate - 1.0),
    }
}
```
