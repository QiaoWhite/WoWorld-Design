# 003 — 季节循环与时间

## 一、游戏时间参数

所有时间参数标记为 [TUNING]，方便后续调整。**开发者提供默认值，玩家可在设置中自定义**（见 §1.2）。

```rust
/// 游戏时间常量 — 全部 [TUNING]
struct TimeConstants {
    /// 游戏天时长（实时秒）— 默认 3600 (60 分钟)，玩家可调 [15, 120]
    day_duration_seconds: f32,      // 3600 (60 分钟) [TUNING · 玩家可调]
    /// 游戏天内分钟数
    minutes_per_day: u32,           // 1440 (24 游戏时)
    /// 一年天数
    days_per_year: u32,             // 120
    /// 每季天数（均匀四季）
    days_per_season: u32,           // 30
    /// 游戏时 = 游戏分钟 × 60
    seconds_per_game_hour: f32,     // 150.0 (2.5 实时分钟/游戏时 @60min/天)
}

/// 时间进制（默认值下的换算）
/// 1 游戏日 = 60 实时分钟（玩家可调）
/// 1 游戏年 = 120 游戏日 = 120 实时小时
/// 每天玩 4 小时 → 约 30 天现实时间经历一个完整游戏年
```

### 1.1 替代方案对比

| 方案 | 年长度 | 天长度 | 利弊 |
|------|--------|--------|------|
| A: 快周期 | 40天 | 24min | 一周经历四季——但植物还没长出来就冬天了 |
| **B: 中周期（采纳）** | **120天** | **60min** | 一天够做完整冒险循环（探索·交易·战斗·社交）——从容不迫 |
| C: 慢周期 | 360天 | 60min | 接近现实——但玩家可能永远卡在一个季节里 |
| D: 不均匀 | 春夏90天/秋冬30天 | — | 增长季节长——但复杂度高，游戏不必要 |

### 1.2 玩家自定义

```rust
/// settings.json — 玩家可调
/// "gameplay.day_length_minutes": 60  // 默认 60, 范围 [15, 120]
///
/// 值越小 → 昼夜更替越快 → 季节循环越快
/// 值越大 → 每天时间更"宽敞" → 可以做更多事
///
/// 修改立即生效，不绑定存档（纯显示/节奏参数）
```

| 边界 | 值 | 理由 |
|------|-----|------|
| 下限 | 15 min/天 | 更短则昼夜快到不自然，季节循环失真 |
| 上限 | 120 min/天 | 更长则一季需 60 现实时——大多数玩家永远见不到季节更替 |
| 默认 | 60 min/天 | 一天够做完整冒险循环，4h 会话 ≈ 4 游戏天 |

**关键架构约束**：区域分层模拟（Aggregate/Skeletal tick）使用**固定的现实时间间隔**（90s / 30min），不绑定游戏天长度。详见 [[../技术栈方案/001-WoWorld正式技术栈方案v3|技术栈方案]]。`SeasonClock` 是纯函数 `GameTime → SeasonState`——`day_duration_seconds` 仅影响现实时间↔游戏时间换算，不影响确定性。

## 二、SeasonClock

`SeasonClock` 是纯时间函数——输入 `GameTime`，输出 `SeasonState`。无状态、无存储、完全确定性。

```rust
/// 季节时钟——纯时间函数
/// 所有季节信息都从 GameTime 推导，零内部状态
struct SeasonClock {
    time_constants: TimeConstants,
    /// 世界诞生的年号（种子决定——虚数，仅用于显示）
    world_birth_year: u64,
    /// 初始季节（春分第1天 = 0）
    initial_day_of_year: u32,       // 0
}

impl SeasonClock {
    /// 核心函数——时间→季节状态
    fn get_state(&self, time: GameTime) -> SeasonState {
        let total_days = time.total_game_days();
        let day_of_year = (total_days + self.initial_day_of_year) % self.time_constants.days_per_year;
        let year_number = self.world_birth_year + (total_days / self.time_constants.days_per_year as u32) as u64;
        
        let season_index = day_of_year / self.time_constants.days_per_season;
        let day_of_season = (day_of_year % self.time_constants.days_per_season) + 1; // 1-indexed
        
        let season = match season_index {
            0 => Season::Spring,
            1 => Season::Summer,
            2 => Season::Autumn,
            3 => Season::Winter,
            _ => unreachable!(),
        };
        
        SeasonState {
            season,
            day_of_season,
            day_of_year: day_of_year + 1, // 1-indexed
            year_number,
            season_progress: day_of_season as f32 / self.time_constants.days_per_season as f32,
            year_progress: day_of_year as f32 / self.time_constants.days_per_year as f32,
        }
    }
}

/// 季节状态——纯时间函数输出
struct SeasonState {
    season: Season,
    day_of_season: u32,         // 1-30
    day_of_year: u32,           // 1-120
    year_number: u64,           // 世界诞生至今的年号
    season_progress: f32,       // 0-1 当前季节内的进度
    year_progress: f32,         // 0-1 年进度
}

enum Season {
    Spring,     // 春 — 万物复苏，农业翻土
    Summer,     // 夏 — 对流活跃，暴风雨频发
    Autumn,     // 秋 — 收获季节，风暴再次活跃
    Winter,     // 冬 — 低温休眠，雪盖大地
}
```

### 2.1 虚数年号

```
世界诞生年号 = 种子决定的大数字（如 12,847,293）

"千百万年" 是叙事概念——不是模拟概念。

实际发生的:
├─ 虚数年号: 种子决定 → 用于书籍/墓碑/纪念碑的年份标注
├─ 玩家进入时的实际年号: 种子年号 + player_entry_year_offset
│   例: 12,847,293 + 0 (进入时刻)
├─ 玩家每度过120游戏天 → year_number += 1
└─ 历史模拟: 仅模拟最近 years_to_simulate (500-2000) 年
                    ↑ 不是从年号0开始模拟的百万年

地质时间尺度的效果（土壤厚度、河流侵蚀深度）:
→ 在世界生成管线 P1-P3 一次性产出（噪声函数 + 侵蚀模拟）
→ 不需要季节系统回溯百万年
```

## 三、日长与太阳

### 3.1 日长公式

```rust
/// 日长（小时）——纬度与季节的函数
fn day_length_hours(latitude_deg: f32, season: Season, season_progress: f32) -> f32 {
    let lat_rad = latitude_deg.to_radians();
    
    // 太阳赤纬——地球近似
    // 夏至: +23.5°, 冬至: -23.5°, 春分/秋分: 0°
    let declination = match season {
        Season::Spring => lerp(0.0, 23.5, season_progress),
        Season::Summer => lerp(23.5, 0.0, season_progress),
        Season::Autumn => lerp(0.0, -23.5, season_progress),
        Season::Winter => lerp(-23.5, 0.0, season_progress),
    };
    let decl_rad = declination.to_radians();
    
    // 半日弧长
    let hour_angle = (-lat_rad.tan() * decl_rad.tan()).acos();
    
    // 日长 = 2 × 半日弧长 × 24h / 2π
    let day_hours = 2.0 * hour_angle * 24.0 / (2.0 * std::f32::consts::PI);
    
    // 极昼/极夜处理
    if lat_rad.tan() * decl_rad.tan() > 1.0 {
        24.0  // 极昼
    } else if lat_rad.tan() * decl_rad.tan() < -1.0 {
        0.0   // 极夜
    } else {
        day_hours
    }
}
```

### 3.2 不同纬度的日长变化

| 纬度 | 春分 | 夏至 | 秋分 | 冬至 |
|------|------|------|------|------|
| 赤道 (0°) | 12.0h | 12.0h | 12.0h | 12.0h |
| 热带 (15°) | 12.0h | 13.2h | 12.0h | 10.8h |
| 亚热带 (30°) | 12.0h | 14.0h | 12.0h | 10.0h |
| 温带 (45°) | 12.0h | 15.5h | 12.0h | 8.5h |
| 亚寒带 (60°) | 12.0h | 18.0h | 12.0h | 6.0h |
| 极地 (75°) | 12.0h | 24.0h | 12.0h | 0.0h |

### 3.3 太阳高度角

```rust
/// 太阳高度角 (rad) — 纬度+时间函数
fn solar_elevation(latitude_deg: f32, time: GameTime, season: Season, season_progress: f32) -> f32 {
    let lat_rad = latitude_deg.to_radians();
    
    // 太阳赤纬 (同 day_length 公式)
    let decl_rad = solar_declination(season, season_progress);
    
    // 时角——正午=0，日出=-90°，日落=+90°
    let hour_angle = time.hour_angle(); // 0 到 2π
    
    // sin(高度角) = sin(纬度)×sin(赤纬) + cos(纬度)×cos(赤纬)×cos(时角)
    let sin_elevation = lat_rad.sin() * decl_rad.sin()
        + lat_rad.cos() * decl_rad.cos() * hour_angle.cos();
    
    sin_elevation.asin()
}
```

NPC 活动时长 = day_length × (1.0 - siesta_factor)，其中：
- 热带夏季：`siesta_factor = 0.2`（午休避开正午高温）
- 其他地区：`siesta_factor = 0`

## 四、天象生成

天象分属两个系统——按物理本质分类：

### 4.1 归天气系统：彩虹

```rust
/// 彩虹——降水的光学后果
fn update_rainbow(
    precipitation_type: PrecipitationType,
    precipitation_intensity: f32,
    cloud_cover: f32,
    solar_elevation: f32,
    time_since_rain_stopped_minutes: f32,
) -> bool {
    // 条件: 雨刚停 + 低云量 + 太阳低角度
    precipitation_type == PrecipitationType::None
        && time_since_rain_stopped_minutes < 30.0  // 30分钟内下过雨
        && cloud_cover < 0.4                       // 天空较晴
        && solar_elevation > 0.05                   // 太阳已升起
        && solar_elevation < 0.6                    // 但不是正午（低角度彩虹更明显）
}
```

彩虹出现在 `WeatherSample.rainbow_visible` 字段中。NPC 文档 5.3 中已有"彩虹→15%概率触发天象对话"机制。

### 4.2 归 SeasonClock：极光、日食/月食

```rust
/// 天象事件——纯天文函数（归 SeasonClock）
struct SkyEvents {
    /// 极光
    aurora_visible: bool,
    aurora_intensity: f32,  // 0-1
    /// 日食/月食
    eclipse_type: EclipseType,
    /// 流星雨
    meteor_shower: MeteorShower,
}

enum EclipseType {
    None,
    Partial,    // 偏食
    Total,      // 全食
}

/// 极光——高纬度+夜间+季节
fn aurora_visibility(latitude_deg: f32, time_of_day: f32, season: Season) -> (bool, f32) {
    // 仅在极地附近可见
    if latitude_deg.abs() < 55.0 {
        return (false, 0.0);
    }
    
    // 仅在夜间
    if time_of_day > 0.15 && time_of_day < 0.85 {
        return (false, 0.0); // 非夜间
    }
    
    // 季节影响——冬季最强
    let season_mult = match season {
        Season::Winter => 1.0,
        Season::Autumn | Season::Spring => 0.6,
        Season::Summer => 0.2, // 夏季高纬几乎全是白天，极光不可见
    };
    
    // 纬度越靠近磁极越强
    let lat_intensity = ((latitude_deg.abs() - 55.0) / 35.0).clamp(0.0, 1.0);
    
    let intensity = lat_intensity * season_mult;
    let visible = intensity > 0.2;
    
    (visible, intensity)
}

/// 日食/月食——轨道力学(v1.2+)，v1.0 为种子决定的稀有事件时间
fn eclipse_at_time(time: GameTime, world_seed: u64) -> EclipseType {
    // v1.0: 纯种子决定——整个游戏过程中仅1-3次日食
    // v1.2: 升级为轨道力学精确计算
    EclipseType::None
}

/// 流星雨——v1.2+
fn meteor_shower_at_time(time: GameTime, world_seed: u64) -> MeteorShower {
    // v1.2+ 实现
    MeteorShower { active: false, intensity: 0.0 }
}
```

## 五、初始季节与存档

### 5.1 新存档——春分开局

```
新世界 → SeasonClock 初始化为:
├─ day_of_year = 1 (春分第1天)
├─ season = Spring
├─ year_number = world_birth_year (虚数，种子决定)
└─ 理由:
    ├─ 模拟最自然的起点——万物复苏
    ├─ 植物生长周期从春季起步
    ├─ NPC 农业日历从翻土开始
    ├─ 动物从繁殖季开始
    └─ 历史模拟 P8 中季节自由选择（几千年跨度中季节点无关紧要）
```

### 5.2 已有存档

```
存档加载 → 恢复 SeasonClock:
├─ 存档存储: total_game_days (自玩家进入起)
├─ SeasonClock::get_state(total_game_days) → 当前季节
└─ 完全确定性——无需存储任何季节状态
```

## 六、季节对游戏系统的影响总览

SeasonClock 的数据流方向——季节通过天气系统间接影响消费方，部分消费方直接消费 SeasonState：

```
SeasonClock
├─ → 天气系统 (ClimateModulator —— 温度/湿度/流行/风暴的季节调制)
├─ → 植物模块 (直接消费 season —— 开花季节/结果季节/落叶触发)
├─ → 动物模块 (直接消费 season —— 繁殖季/迁徙触发/冬眠触发)
├─ → NPC 模块 (直接消费 season —— 农业日历/季节活动)
├─ → 世界生成 (直接消费 season —— 游牧迁徙走廊的季节性草场)
├─ → 历史模块 (直接消费 year_number —— 年份标注)
└─ → Godot 渲染 (日长/太阳高度 → Sky shader)
```
