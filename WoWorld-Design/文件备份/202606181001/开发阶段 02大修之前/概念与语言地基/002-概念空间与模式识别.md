## 一、PatternSignature——全球客观的模式指纹

### 1.1 定义

```rust
/// 模式签名——从客观数据中确定性哈希的模式指纹
/// 不含任何概念标签。全球统一。客观可验证。
///
/// 哈希输入: 排序后的 AtomTypeTag 集合 + 排序后的资源类型集合
///           + 边的方向性标记。确定性——同输入永远同输出。
///
/// 碰撞分析: WoWorld 估计有 ~2000 种独特的权力/经济/关系模式。
/// u64 碰撞概率 < 10⁻¹⁶。扩展到 >10⁶ 种模式时需升级为 u128。
#[derive(Copy, Clone, Hash, Eq, PartialEq, Serialize, Deserialize)]
pub struct PatternSignature(u64);
```

### 1.2 生成函数

```rust
// woworld_core/src/pattern.rs
// 纯函数——仅依赖 woworld_types

pub fn compute_pattern_signature(
    edges_ab: &[PowerEdgeRef],  // A 对 B 的权力边
    edges_ba: &[PowerEdgeRef],  // B 对 A 的权力边
) -> PatternSignature {
    let mut hasher = FxHasher::default();

    // ① 排序原子类型标签（确保确定性）
    let mut atoms_ab: Vec<u8> = edges_ab.iter()
        .filter(|e| e.active)
        .map(|e| e.atom_type as u8)
        .collect();
    atoms_ab.sort();
    for a in &atoms_ab { hasher.write_u8(*a); }

    // ② B→A 方向同理
    let mut atoms_ba: Vec<u8> = edges_ba.iter()
        .filter(|e| e.active)
        .map(|e| e.atom_type as u8)
        .collect();
    atoms_ba.sort();
    hasher.write_u8(0xFF);  // 分隔标记
    for a in &atoms_ba { hasher.write_u8(*a); }

    // ③ 资源类型（如果适用）
    // 从 Extract 边的 resource_type 参数中提取
    let mut resources: Vec<u8> = edges_ab.iter()
        .filter(|e| e.atom_type == AtomTypeTag::Extract)
        .filter_map(|e| e.resource_type.map(|r| r as u8))
        .collect();
    resources.sort();
    hasher.write_u8(0xFE);
    for r in &resources { hasher.write_u8(*r); }

    PatternSignature(hasher.finish())
}
```

**关键**：`compute_pattern_signature()` 不依赖任何概念数据。它是纯数学的。任何模块或集成层都可以调用它。

## 二、文化概念空间——文化相对的切分方式

### 2.1 核心定义

```rust
// woworld_types/concepts.rs

/// 文化内的概念 ID——必须搭配 CultureId 才有意义
/// 高 4 位: 命名空间 (0=核心, 1-15=Mod)
/// 低 12 位: 文化内序号 (0-4095)
#[derive(Copy, Clone, Hash, Eq, PartialEq, Serialize, Deserialize)]
pub struct ConceptLocalId(u16);
pub const CONCEPT_NONE: ConceptLocalId = ConceptLocalId(0xFFFF);

/// 文化概念定义——TOML 数据驱动
pub struct CultureConceptDef {
    pub local_id: ConceptLocalId,
    pub culture_id: CultureId,
    /// 概念名称（文化内的称谓）
    pub name: String,
    /// 该概念覆盖的模式空间——一个概念可覆盖多个模式
    pub coverage: Vec<PatternCover>,
    /// 概念粒度——切分得多细（0=非常粗糙的大概念，1=极其精细的小概念）
    pub granularity: f32,
    /// 概念的涌现代（用于漂移计算）
    pub emergence_generation: f32,
    /// 区别于哪些概念
    pub distinct_from: Vec<ConceptLocalId>,
}

/// 单个模式覆盖——"什么模式被识别为什么概念"
pub struct PatternCover {
    /// 必须出现的原子类型集合
    pub required_atoms: Vec<AtomTypeTag>,
    /// 禁止出现的原子类型（用于区分——如雇佣必须不含 Compel，以区别于师徒）
    pub forbidden_atoms: Vec<AtomTypeTag>,
    /// 模式特定参数约束
    pub params: PatternParams,
}

pub struct PatternParams {
    pub extract_resource_a_to_b: Option<ResourceType>,  // Extract 的资源类型
    pub extract_resource_b_to_a: Option<ResourceType>,
    pub has_contract: bool,                             // 是否必须有 Contract
    pub has_pledge: bool,                               // 是否必须有 Pledge
    // 可扩展——更多约束
}
```

### 2.2 TOML 数据格式

```toml
# cultures/zhongyuan/concepts.toml

[culture]
id = "zhongyuan"
name = "中原文化"

[concept.employment]
local_id = 0x0001
name = "雇佣"

distinct_from = ["apprenticeship", "slavery"]

[[concept.employment.coverage]]
required_atoms = ["Contract", "Extract"]
required_atoms_b_to_a = ["Extract"]
forbidden_atoms = ["Compel"]          # 不含强制行为——区别于师徒
params = { has_contract = true, extract_resource_a_to_b = "labor", extract_resource_b_to_a = "wage" }

[concept.apprenticeship]
local_id = 0x0002
name = "师徒"
granularity = 0.8
distinct_from = ["employment"]

[[concept.apprenticeship.coverage]]
required_atoms = ["Contract", "Extract", "Compel"]
required_atoms_b_to_a = ["Extract"]
params = { has_contract = true, extract_resource_a_to_b = "labor",
          compel_behavior = "training", extract_resource_b_to_a = "wage" }

[concept.coerced_labor]
local_id = 0x0003
name = "劳役"
granularity = 0.3  # 粗粒度——涵盖很广
distinct_from = []

[[concept.coerced_labor.coverage]]

params = { extract_resource_a_to_b = "labor" }

```

**兽人文化的概念空间——概念切分完全不同**：

```toml
# cultures/orcish/concepts.toml
```

## 三、classify_pattern()——概念识别

### 3.1 核心函数

```rust
// woworld_core/src/concept.rs
// 纯函数——仅依赖 woworld_types

/// 将客观模式签名分类为文化概念空间中的概念
pub fn classify_pattern(
    signature: PatternSignature,
    concept_space: &[CultureConceptDef],
) -> Vec<(ConceptLocalId, f32)> {
    let mut matches: Vec<(ConceptLocalId, f32)> = concept_space.iter()
        .filter_map(|def| {
            let overlap = pattern_overlap(signature, &def.coverage);
            if overlap > 0.1 {
                Some((def.local_id, overlap))
            } else {
                None
            }
        })
        .collect();

    matches.sort_by(|a, b| b.1.partial_cmp(&a.1).unwrap());
    matches.truncate(3);  // 最多返回 3 个最佳匹配
    matches
}

/// 计算模式签名与概念覆盖的匹配度
pub fn pattern_overlap(
    signature: PatternSignature,
    coverage: &[PatternCover],
) -> f32 {
    // 解码签名中的原子集合
    let sig_atoms = decode_atom_set(signature);
    let sig_resources = decode_resources(signature);

    let mut best_overlap = 0.0;
    for cover in coverage {
        // ① 必须原子的覆盖率
        let atom_overlap = cover.required_atoms.iter()
            .filter(|a| sig_atoms.contains(a))
            .count() as f32 / cover.required_atoms.len().max(1) as f32;

        // ② 禁止原子检查——如果签名包含禁止原子 → 严重降权
        let forbidden_penalty = if cover.forbidden_atoms.iter()
            .any(|a| sig_atoms.contains(a))
        { 0.4 } else { 1.0 };

        // ③ 参数约束匹配（资源类型等）
        let param_match = match_params(&sig_resources, &cover.params);

        let overlap = atom_overlap * forbidden_penalty * param_match;
        if overlap > best_overlap { best_overlap = overlap; }
    }

    best_overlap.clamp(0.0, 1.0)
}
```

### 3.2 性能优化——桶索引

```rust
// ConceptSpaceRegistry 内部——按 PatternSignature 高字节分桶
struct ConceptSpaceIndex {
    buckets: [Vec<ConceptLocalId>; 256],  // 256 桶
}

fn classify_pattern_fast(
    sig: PatternSignature,
    index: &ConceptSpaceIndex,
    concept_defs: &[CultureConceptDef],
) -> Vec<(ConceptLocalId, f32)> {
    let bucket = (sig.0 >> 56) as u8;
    let candidates = &index.buckets[bucket as usize];
    // 期望: ~2 概念/桶 (500 概念 / 256 桶)
    // 遍历成本: <1µs
    candidates.iter()
        .filter_map(|id| {
            let def = &concept_defs[id.0 as usize];
            let overlap = pattern_overlap_sig(sig, &def.coverage);
            if overlap > 0.1 { Some((*id, overlap)) } else { None }
        })
        .sorted_by(|a, b| b.1.partial_cmp(&a.1).unwrap())
        .take(3)
        .collect()
}
```

## 四、混合概念空间——跨文化生活

```rust
// woworld_lang —— 不依赖 NPC crate
pub fn effective_concept_space(
    birth_culture: CultureId,
    residences: &[ResidenceRecord],
    acquired_concept_ids: &[ConceptLocalId],
    registry: &ConceptSpaceRegistry,
) -> Vec<CultureConceptDef> {
    let mut weights: HashMap<CultureId, f32> = HashMap::new();

    // 母文化 = 基础权重 0.6
    weights.insert(birth_culture, 0.6);

    // 居住经历 = 每年 +0.05，上限 0.3
    for residence in residences {
        let years = residence.duration_days as f32 / 360.0;
        let w = (years * 0.05).min(0.3);
        *weights.entry(residence.culture_id).or_default() += w;
    }

    // 归一化
    let total: f32 = weights.values().sum();
    let mut space = Vec::new();
    for (culture_id, weight) in &weights {
        for concept in registry.load(culture_id) {
            let mut weighted = concept.clone();
            weighted.granularity *= *weight / total;
            space.push(weighted);
        }
    }
    // 加上习得的外来概念
    for id in acquired_concept_ids {
        if let Some(concept) = registry.find_concept(*id) {
            if !space.iter().any(|c: &CultureConceptDef| c.local_id == *id) {
                space.push(concept.clone());
            }
        }
    }
    space
}
```

## 五、CompactConceptEncoding——64 字节压缩编码

### 5.1 完整布局

```rust
/// 压缩概念编码 —— 64 字节
/// 存储在 EventMemory、PhysicalBook、InformationPayload 中
#[derive(Copy, Clone, Serialize, Deserialize)]
pub struct CompactConceptEncoding {
    pub pattern_sig: PatternSignature,         // bytes 0-7   (u64)
    pub concept_ids: [ConceptLocalId; 3],      // bytes 8-13  (3 × u16)
    pub confidences: [f16; 3],                 // bytes 14-19 (3 × half)
    pub flags: u8,                             // byte 20
    pub detail_level: PerceptionDetailLevel,   // byte 21
    pub culture_id: CultureId,                 // bytes 22-23
    pub concept_space_version: u16,            // bytes 24-25
    pub recognized_at_day: u16,                // bytes 26-27
    pub reserved: [u8; 36],                    // bytes 28-63
}

#[derive(Copy, Clone, PartialEq, Eq)]
#[repr(u8)]
pub enum PerceptionDetailLevel {
    Full = 0,           // 直接当事人——3 概念全存储
    Partial = 1,        // 物理旁观——2 概念，salience 折扣
    Vague = 2,          // 社交间接——1 概念，标记为道听途说
    Unrecognized = 3,   // 无法识别——concept_ids 全部为 CONCEPT_NONE
}

// Flags
const FLAG_HEARSAY: u8             = 0b0000_0001;
const FLAG_RECONSTRUCTED: u8       = 0b0000_0010;
const FLAG_EMOTIONAL_OVERRIDE: u8  = 0b0000_0100;
const FLAG_CROSS_CULTURAL: u8      = 0b0000_1000;
```

### 5.2 构造器

```rust
impl CompactConceptEncoding {
    pub fn full(sig: PatternSignature, matches: &[(ConceptLocalId, f32)],
                culture: CultureId, version: u16) -> Self {
        let mut enc = Self::zeroed();
        enc.pattern_sig = sig;
        enc.detail_level = PerceptionDetailLevel::Full;
        enc.culture_id = culture;
        enc.concept_space_version = version;
        for (i, (id, conf)) in matches.iter().take(3).enumerate() {
            enc.concept_ids[i] = *id;
            enc.confidences[i] = f16::from_f32(*conf);
        }
        enc
    }

    pub fn partial(sig: PatternSignature, matches: &[(ConceptLocalId, f32)],
                   salience: f32, culture: CultureId, version: u16) -> Self {
        let mut enc = Self::full(sig, matches, culture, version);
        enc.detail_level = PerceptionDetailLevel::Partial;
        enc.concept_ids[2] = CONCEPT_NONE;  // 第三概念丢失
        enc.confidences[2] = f16::ZERO;
        enc.confidences[0] = f16::from_f32(enc.confidences[0].to_f32() * salience);
        enc.confidences[1] = f16::from_f32(enc.confidences[1].to_f32() * salience);
        enc
    }

    pub fn vague(sig: PatternSignature, matches: &[(ConceptLocalId, f32)],
                 fidelity: f32, culture: CultureId, version: u16) -> Self {
        let mut enc = Self::full(sig, matches, culture, version);
        enc.detail_level = PerceptionDetailLevel::Vague;
        enc.flags |= FLAG_HEARSAY;
        enc.concept_ids[1] = CONCEPT_NONE;
        enc.concept_ids[2] = CONCEPT_NONE;
        enc.confidences[1] = f16::ZERO;
        enc.confidences[2] = f16::ZERO;
        enc.confidences[0] = f16::from_f32(enc.confidences[0].to_f32() * fidelity);
        enc
    }

    pub fn unrecognized(sig: PatternSignature) -> Self {
        let mut enc = Self::zeroed();
        enc.pattern_sig = sig;
        enc.detail_level = PerceptionDetailLevel::Unrecognized;
        enc.concept_ids = [CONCEPT_NONE; 3];
        enc
    }

    /// 展开——自包含，不需要外部概念空间
    pub fn expand(&self) -> ExpandedConceptEncoding {
        ExpandedConceptEncoding {
            pattern_sig: self.pattern_sig,
            matches: self.concept_ids.iter()
                .zip(self.confidences.iter())
                .filter(|(id, _)| **id != CONCEPT_NONE)
                .map(|(id, conf)| ConceptMatch { concept_id: *id, confidence: conf.to_f32() })
                .collect(),
            culture_id: self.culture_id,
            detail_level: self.detail_level,
            flags: self.flags,
        }
    }
}
```

## 六、概念识别在集成层——不在业务模块


**关键**：NPC crate 不知道 `classify_pattern()` 的存在。它只接收已识别的 `CompactConceptEncoding`。概念识别完全在集成层完成。

## 七、与已有类型的关系

| 已有类型 | 新类型 | 关系 |
|---------|--------|------|
| `EventType`（NPC ver2.0） | `PatternSignature` | PatternSignature 是更精细的数学指纹——EventType 是粗粒度人读标签。EventMemory 中两者**共存**：`event_type` 保留用于快速筛选（"这是战斗事件还是社交事件"），`compact_concepts` 用于概念级推理和文本渲染。两者不互相替代——EventType 做粗分类，PatternSignature 做精细概念识别 |
| `Knowledge`（感官系统） | `CompactConceptEncoding` | Knowledge 存细粒度事实(~64条)，Compact 存事件的概念编码 |
| `ExpressionRef`（语言表达） | `UtteranceId` | ExpressionRef 用于持久可读物，UtteranceId 用于瞬时话语 |
