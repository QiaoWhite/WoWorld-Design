## 一、实质设计问题（经深度审计确认）

### 1.1 ProfessionTagId 类型定义自相矛盾 ★ 真问题

**矛盾本质**: `ProfessionTagId(u32)` 的 Rust struct 定义在经济系统设计文档中（`经济系统/001-经济系统总纲.md` 行196），但同一文档行88-91明确说"经济系统**不拥有**职业标签，但消费它们"。WorldGen入口将来源标为"经济"。NPC身份系统运行时维护。

**Rust解耦分析**: 如果在代码中按文档实现，`ProfessionTagId` 定义在 `woworld_economy` crate → WorldGen 需依赖 `woworld_economy` 做 P8 人口初始化 → 但 WorldGen 在经济模块初始化**之前**运行 → 启动顺序依赖。NPC 身份系统也需依赖 `woworld_economy` 仅为了一个类型定义 → 不必要的耦合。

| 冲突方 | 声明 | 出处 |
|--------|------|------|
| Economy 设计文档 | `pub struct ProfessionTagId(u32);` — 类型定义在经济系统文档中 | `经济系统/001-经济系统总纲.md` 行196 |
| Economy 设计文档 | "经济系统**不拥有**职业标签，但消费它们" | `经济系统/001-经济系统总纲.md` 行88-91 |
| WorldGen 入口 | ProfessionTag 来源模块 = "经济" | `03-世界生成/002-接口入口.md` 行58 |
| WorldGen 008 | 约15处消费 ProfessionTag（职业分配·BigFive修正·技能生成·初始动作） | `世界生成/008-人口投影与家族协同生成.md` |

**修复**: 
1. `ProfessionTagId(u32)` 应移至 `woworld_core`（与 ItemDefId、SkillId、EntityId 等所有ID类型统一位置——零依赖crate）
2. ProfessionTag TOML 为中性数据文件——各模块通过 `include_str!()` 平等加载
3. WorldGen入口修正：来源="woworld_core 类型定义 · TOML 数据驱动 · WorldGen P8 初始分配"
4. Economy出口修正：标注"消费但不拥有——类型定义在 woworld_core"

### 1.2 CognitiveTide 未进入 AgentSnapshot ★ 设计空白

**发现**: AgentSnapshot（`NPC活人感模块/08-NPC行动涌现与分类/004-AgentSnapshot与连续参数涌现.md`）认知段仅含 `cognitive_damping` 和 `planning_horizon`（均从 CognitiveStyle 派生）。**不含 CognitiveTide**——3维认知潮汐（cognitive_load / rumination_pressure / mind_quietude）每决策周期更新，但未进入贯穿全三层的快照。

**设计影响**: 战斗/魔法/社交/经济所有消费 AgentSnapshot 的系统无法感知 NPC 的实时认知状态。具体：
- 战斗: 疲劳NPC的战略决策精度无法降低
- 魔法: 分心NPC的施法精度无法受影响
- 社交: 认知负载高的NPC对话质量无法下降
- 经济: 疲惫NPC的交易判断无法受损

**出处在源设计文档中确认**:
- AgentSnapshot 完整结构见 `004-AgentSnapshot与连续参数涌现.md` 行13-47——认知段5字段不含 CognitiveTide
- CognitiveTide 定义见 `NPC活人感开发文档ver2.0.md` 行185/212——3维（负载/反刍压力/安静度）
- Combat入口已正确列出 CognitiveStyle/MindAttribution（`06-战斗/002-接口入口.md` 行50-53），但缺失 CognitiveTide

**修复方向**（已执行）: ✅ AgentSnapshot 认知段正式新增 3 字段（cognitive_load / rumination_pressure / mind_quietude）——从 CognitiveTide 每决策周期直接映射。认知段从5字段扩展至8字段·总量~108 bytes·仍在 cache line。`from_sources()` 新增 `cognition_tide: &CognitiveTide` 参数。详见 `004-AgentSnapshot与连续参数涌现.md`。
