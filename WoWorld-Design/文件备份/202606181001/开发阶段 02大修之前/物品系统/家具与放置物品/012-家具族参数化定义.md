## 一、核心原则

### 1.1 不穷举——参数化

武器系统的组件装配框架已经证明了参数化模式：5 种柄 × 8 种刃 × 4 种护手 × 3 种配重 = 480 种变体——不需要手工建模 480 把剑。

家具同理。一个椅子族通过连续参数（seat_height, backrest_angle, leg_style...）覆盖从矮人石凳到精灵王座的全部形态。

### 1.2 三层变体产生机制

```
第一层: mesh 模板 — 决定拓扑结构 (Blender 手工制作，每族 ~3 个)
第二层: 连续参数 — morph target 驱动形态变化 (从 CultureFurnishProfile + wealth 派生)
第三层: 材质覆盖 — 木纹/金属/石材 + 品质/磨损 decal (从 MaterialDef + quality 派生)
```

### 1.3 文化影响参数，不创造新模板

文化不改变 mesh 模板——文化改变**参数落在参数空间的哪个区域**。和风把 `seat_height` 推到 0.05m → 座布団。矮人把 `material` 推到 Stone + `decoration_density` 推到 0.7 → 雕花石凳。同一个模板，参数不同。

## 二、通用参数结构 (FurnitureParams)

所有家具族共享一个通用参数框架：

```rust
/// 家具族通用参数——所有族共享
/// 每个族定义自己的参数范围（min/max/default）
pub struct FurnishParams {
    /// 功能尺寸
    pub primary_height: f32,     // 座高/桌高/床面高/柜高...
    pub primary_width: f32,      // 座宽/桌面宽/床宽/柜宽...
    pub primary_depth: f32,      // 座深/桌面深/床长/柜深...
    
    /// 结构参数（族特定——离散）
    pub leg_style: LegStyle,
    pub backrest_style: Option<BackrestStyle>,
    pub armrest_style: Option<ArmrestStyle>,
    
    /// 材料
    pub dominant_material: MaterialClass,
    pub secondary_material: Option<MaterialClass>,  // 座垫/桌面覆层/装饰嵌件
    
    /// 装饰
    pub decoration_density: f32,          // 0(纯功能) — 1(极尽装饰)
    pub ornament_motif: OrnamentMotif,
    pub edge_profile: EdgeProfile,
    
    /// 品质
    pub quality_override: Option<f32>,    // None = 从 NPC 技能和材料派生
}
```

## 三、九族完整定义

### 3.1 座椅族 (Seat)

**覆盖范围**: 凳 → 靠背椅 → 扶手椅 → 沙发/长椅 → 王座 → 座布団

**Mesh 模板 (4)**:
| 模板 | 拓扑结构 | 覆盖场景 |
|------|---------|---------|
| `four_leg_chair.blend` | 四腿 + 可选的靠背 + 可选的扶手 | 普通椅/扶手椅/王座 |
| `x_frame_stool.blend` | X 交叉腿 + 折叠铰链 | 折叠凳/野外椅 |
| `pedestal_seat.blend` | 单柱底座 + 圆形座面 | 吧台凳/钢琴凳 |
| `floor_cushion.blend` | 无腿——地毯式座面 | 座布団/地板垫/蒲团 |

**核心参数**:
```rust
pub struct SeatParams {
    pub seat_height: f32,         // 0.03m(座布団) — 0.55m(王座)
    pub seat_width: f32,          // 0.3m — 0.8m
    pub seat_depth: f32,          // 0.3m — 0.7m
    pub backrest_height: f32,     // 0(无靠背=凳) — 1.2m(王座高靠背)
    pub backrest_angle_deg: f32,  // 85°(直立) — 115°(后仰)
    pub armrest_height: Option<f32>,
    pub leg_style: LegStyle,
    pub has_upholstery: bool,
    pub upholstery_thickness: f32,
}
```

**参数极端化涌现**:
| 文化 | seat_height | backrest_angle | leg_style | material | → 视觉结果 |
|------|------------|---------------|-----------|----------|-----------|
| 和风 | 0.05 | 95° | None | Tatami | 座布団 |
| 矮人 | 0.30 | 90° | Blocky | Stone | 厚重石凳 |
| 精灵 | 0.50 | 100° | TaperedCurve | LivingWood | 自然曲线木椅 |
| 贵族 | 0.48 | 105° | TurnedLion | Walnut+Velvet | 雕花扶手椅 |
| 国王 | 0.55 | 100° | CarvedCrest | Ebony+Gold | 王座 |

## 五、参数→Morph Target 映射

每个模板配备预定义的 morph target。`FurnishParams` 的连续参数自动计算 morph weight：

```rust
impl SeatParams {
    pub fn to_morph_weights(&self, template: &MeshTemplate) -> [f32; 16] {
        let mut weights = [0.0; 16];
        
        // seat_height → morph[0] "leg_length"
        weights[0] = (self.seat_height - template.default_seat_height) 
            / template.seat_height_range;
        
        // backrest_angle → morph[1] "backrest_tilt"  
        weights[1] = (self.backrest_angle_deg - 90.0) / 25.0;
        
        // seat_width → morph[2] "seat_width"
        weights[2] = (self.seat_width - template.default_seat_width)
            / template.seat_width_range;
        
        // decoration → morph[3..6] "ornament_depth_*" (4个装饰区)
        for i in 0..4 {
            weights[3 + i] = self.decoration_density;
        }
        
        // 材料硬度 → morph[7] "edge_wear_softening"
        // 品质 → morph[8] "warp_amount"
        // ...
        
        weights
    }
}
```

### 6.2 家具变体预设 TOML

## 七、家具工作站的子类别与注册

`FurnitureItem`(0x54) 的子类别 0x54_06 = 工作站族。具体的工作站类型不通过物品系统枚举——通过 **u16 标签注册制**：

```rust
// 启动时——烹饪领域模块注册
WorkstationRegistry::register(0x0001, "cooking_stove",
    "灶台——需要烟道接口 + 燃料");
WorkstationRegistry::register(0x0002, "cooking_pot",
    "烹饪锅——可放在任何火源上");
WorkstationRegistry::register(0x0003, "oven",
    "烤箱——大型固定式");

// 锻造领域模块注册
WorkstationRegistry::register(0x0010, "forge",
    "锻炉——需要烟道 + 淬火水槽");
WorkstationRegistry::register(0x0011, "anvil",
    "铁砧——放在锻炉旁");

// 纺织领域模块注册
WorkstationRegistry::register(0x0020, "loom",
    "织布机——大型，需要室内空间");
WorkstationRegistry::register(0x0021, "spinning_wheel",
    "纺车——可移动，无需建筑依赖");

// 炼金/魔法
WorkstationRegistry::register(0x0030, "alchemy_bench",
    "炼金台——需要通风");
WorkstationRegistry::register(0x0031, "enchanting_altar",
    "附魔台——消费 mana_conductivity 高的材料");

// 文书
WorkstationRegistry::register(0x0040, "writing_desk",
    "书桌——需要光源以提高效率");
```

**物品系统只存 register_tag: u16——不知道"这是灶台"**。NPC 行为系统查询 `WorkstationRegistry::query(tag)` 获取具体属性。完全解耦。

## 八、雕塑族——两层架构

雕塑不能用 morph target 覆盖全部形态——"站立武将"和"抽象球体"在拓扑上是断裂的。

### 8.1 底座（参数化）

与桌台族共享参数系统。`Pedestal` = 桌台族的参数特化（`height > width`）。从简朴石台到雕花大理石基座——连续覆盖。

### 8.2 主体（标记化目录）

```rust
struct SculptureSubject {
    tags: Vec<SubjectTag>,
    mesh: MeshKey,
    bounds: Aabb,
    material_fit: MaterialFit,
    complexity: u8,
}

enum SubjectTag {
    // 主体类别
    Humanoid, Animal, Monster, Abstract, Botanical, Geometric,
    // 语义——游戏系统用这些选择雕塑
    Deity, Ruler, Hero, Ancestor, Saint, MythicalBeast, Guardian,
    HistoricalFigure, Allegorical, Decorative,
    // 姿势（仅具象雕塑）
    Standing, Seated, Mounted, Bust, Relief,
    // 信仰适配
    FaithNeutral, FaithSpecific(FaithId),
}
```

## 九、item_id 的双重归宿

不是所有被摆放的人造物都在 PlacementStore：

| 生成方式 | 数据归宿 | 例子 |
|---------|---------|------|
| Pass A/B/C 自由放置 | PlacementStore | 椅子/桌子/地毯 |
| WFC 嵌入式 (P6 建筑生成) | BuildingStore (ComponentInstance + item_id) | 石雕祭坛/固定长椅/嵌墙书架 |
| 玩家/NPC 运行时放置 | PlacementStore | 玩家手工摆放的火把/家具 |
| 掉落物 | EntityIndex namespace=DroppedItem | 被丢弃的剑/碎片 |

**区别只在数据归宿——物理原子无感知。** 石雕祭坛 (`ComponentInstance.item_id = Some(...)`) 和玩家摆放的石雕 (`PlacedEntry.item_id`) 都通过 `EntityIndex` 被同样地查询和交互。

## 十、与武器装配框架的对称性

| 系统 | 基元 | 组合方式 | 视觉空间 |
|------|------|---------|---------|
| 武器 | ~15 组件槽位类型 | Compose(MeleeGrip + StrikingHead + Guard + Pommel) | ~5,000 |
| 建筑 | 9 ComponentFamily | WFC + 连续参数 | ~50,000 |
| NPC 面部 | 38 模块姿态 | 512² 图集驱动 | ~∞ |
| 动画 | 15 基元轨迹 | 9 层叠加 | ~∞ |
| **家具** | **9 FurnitureFamily × 3 模板** | **连续参数 + morph target** | **~12,000** |

**家具的参数化复杂度是中等**——比武器高（因为有文化维度），比建筑低（因为不需要 WFC 求解）。
