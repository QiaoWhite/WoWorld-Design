## 一、现状分析

### 1.1 当前 NPC 系统的需求建模

NPC ver2.0 中，生理需求通过三个独立路径处理：

**路径 1 — GOAP 安全网（硬阈值）**：
```
hunger < 0.3  → Goal::SurvivalEat
thirst < 0.25 → Goal::SurvivalDrink
fatigue > 0.9 → Goal::SurvivalSleep
health < 0.3  → Goal::SurvivalHeal
```

**路径 2 — 概率决策器（physiology_modifier）**：
一个模糊的权重乘数，文档未展开。意在"饿时吃饭权重略高"。

**路径 3 — 情绪引擎（physiological_pull）**：
```
hunger < 0.4  → pleasure -= 0.001 × (0.4 - hunger)
fatigue > 0.7 → arousal  -= 0.001 × fatigue
health < 0.5  → control  -= 0.001 × (0.5 - health)
```

### 1.2 四大缺陷

1. **悬崖效应**：hunger=0.31 和 hunger=0.29 差异微乎其微，但前者 NPC 正常社交，后者如被电击般冲向食物
2. **无个体差异**：所有 NPC 在同一 hunger 值反应完全一致——抹平了人格/文化/种族多样性
3. **需求间无耦合**：真实生理中睡眠不足→饥饿素上升→更饿。当前模型各需求完全独立
4. **无预测性**：有智慧的 NPC 应预判"下午会饿"，而非等到血糖骤降才反应

### 1.3 已有分散的相关机制

| 机制 | 当前所在 | 与本系统的关系 |
|------|---------|-------------|
| `diet_type` (7种饮食类型) | Life 004 §七 | 部分物种不吃物理食物→不参与饿/渴/元素平衡 |
| `FertilityNorms` (婚姻压力等) | 文化 004 §四 | libido 的文化乘数 |
| `ReligiousReproductionNorms` | 信仰 002（从 Life 012 迁移） | libido 的宗教乘数 |
| `DietaryBasePreferences` (饮食偏好) | 文化 004 §三 | 食物选择的文化过滤器 |
| `AttractionTemplate` (吸引力模板) | NPC 02-性别与吸引力系统 | 性/libido 的行为目标筛选 |
| `stagnation_sense` (停滞感) | NPC SelfNarrative | 与社交匮乏不同的概念——停滞感是综合性"人生无意义" |
| `IntrinsicGoal` (内在目标) | NPC SelfNarrative | 志向驱动,非匮乏驱动——不进需求框架 |
| 饮食→血元素健康 | 魔法 016 | "血元素健康度可通过饮食和锻炼改善"——连接饮食与魔力 |

## 二、需求维度清单（7 维）

### 2.1 需求分类总览

```
┌─ 匮乏驱动（不满足→死亡/崩溃）─── GOAP 安全网 ───┐
│ ① 饥饿 (Hunger)         已有, 机制升级         │
│ ② 口渴 (Thirst)         已有, 机制升级         │
│ ③ 疲劳/睡眠 (Fatigue)   已有, 机制升级         │
│ ④ 个人安全 (Safety)     已有, 不改             │
├───────────────────────────────────────────────┤
├─ 累积驱动（不满足→不适/情绪下降）── 仅进概率 ──┤
│ ⑤ 元素平衡 (Element Balance)  ★ 新增           │
├───────────────────────────────────────────────┤
├─ 周期驱动（不满足→无直接惩罚）── 仅进概率 ──┤
│ ⑥ 性/libido              ★ 新增                │
├───────────────────────────────────────────────┤
├─ 心理匮乏（不满足→情绪长期下降）── 仅进概率 ──┤
│ ⑦ 社交/归属 (Social)     ★ 新增                │
└───────────────────────────────────────────────┘
```

### 2.2 各需求详细说明

#### ① 饥饿 (Hunger)

- **数据来源**：Life.Vitals.hunger (0.0=极度饥饿, 1.0=饱腹)
- **消耗**：种族代谢率 × 活动量 × 环境温度修正
- **归零后果**：<0.1 体力恢复停止 → <0 扣 HP → 死亡
- **决策层**：GOAP (<0.3) + 概率 (0.3~1.0)
- **当前机制问题**：硬阈值,无个体差异
- **升级方案**：urgency = (1.0 - hunger) × personality_sensitivity

#### ② 口渴 (Thirst)

- **数据来源**：Life.Vitals.thirst (0.0=脱水, 1.0=不渴)
- **消耗**：高温加速,湿润食物减缓
- **归零后果**：比饥饿更快扣HP→死亡
- **决策层**：GOAP (<0.25) + 概率 (0.25~1.0)
- **升级方案**：同 hunger

#### ③ 疲劳/睡眠 (Fatigue/Sleep)

- **数据来源**：Life.Vitals.fatigue (0.0=精力充沛, 1.0=极其疲劳)
- **消耗**：体力消耗 × (1 - 耐久)
- **归零后果**：>0.9 全属性-20%, >1.0 强制昏睡
- **决策层**：GOAP (>0.9) + 概率 + 昼夜节律调制
- **升级方案**：增加昼夜节律正弦调制——夜晚 fatigue 自然上升

#### ④ 个人安全 (Personal Safety)

- **数据来源**：Life.Vitals.health + CurrentState.combat_threat
- **决策层**：GOAP (health<0.3) + GOAP (combat_threat>0.7)
- **当前机制**：已有, **不修改**。仅在本系统中列为参考维度

#### ⑤ ★ 元素平衡 (Element Balance)

- **设计理念**：血元素 = 八元素按种族固定配比合成。食物携带元素谱。吸收后,合成血元素用掉"需要的部分",多余元素累积。累积到一定程度→需有意识释放。
- **释放方式**：主动释放(全清,几秒,有意识,瞬间完成) / 睡眠被动释放(5%/晚,慢速)
- **释放观感**：无声无形。NPC 短暂停顿一瞬(与发呆难以区分)。无粒子/无动画/无额外感知。
- **归零后果**：无。不处理→微弱不适(情绪引擎 pull)
- **决策层**：仅概率决策器
- **数据来源**：Life.Vitals.element_surplus: [f32; 8]
- **urgency 公式**：`max(surplus) × 0.5 + avg(surplus) × 0.5`
- **周期**：均衡饮食→几乎不累积；偏科饮食(NPC 吃肉多/吃素多)→3-5天需主动释放
- **详见**：[[文件备份/202606181001/开发阶段 技术栈一致性审查/NPC活人感模块/03-基本需求系统/002-元素平衡需求深化设计]]

#### ⑥ ★ 性/libido

- **设计理念**：NPC 的性驱力。周期波动——非持续恒定,也非二值"发情/不发情"。三种物种类型覆盖全部生命形式。
- **数据来源**：Life.Vitals.libido (bio-signal, not survival vital)
- **三种类型**：
  - Continuous (持续型,如人类/矮人)：`libido = 0.3 + 0.3 × sin(2πt/T)` 温和波动
  - Seasonal (季节型,如鹿/某些灵兽)：每年~60天窗口内波动,非窗口期 0.0
  - EventTriggered (触发型,如某些奇幻物种)：默认 0.0,特定天象/魔法浓度/生命阶段跳变
- **归零后果**：无
- **决策层**：仅概率决策器。不进 GOAP——不是紧急事件
- **reproductive_urge 合成**：`libido × culture.marriage_pressure × faith.reproduction_modifier × personality.openness`
- 进概率决策器,不进 GOAP

#### ⑦ ★ 社交/归属 (Social)

- **设计理念**：NPC 的社交匮乏。和其他需求不同——它不是生物性的,是心理状态。独居一个月的 NPC 身体完全健康,但心理的社交匮乏很高。
- **数据来源**：NPC NpcData.social_deficit (0.0=社交充足, 1.0=极度孤独)
- **累积**：无社交互动 +0.02/游戏日 → 50天独居才满
- **恢复**：每次社交互动 -0.03~-0.15（取决于互动质量×亲密度×情绪）
- **互动定义**：参与对话/共同活动/在公共空间旁观(轻度)
- **归零后果**：无直接。长期→情绪 pleasure 下降
- **决策层**：仅概率决策器
- **与 stagnation_sense 的区别**：stagnation_sense 是综合性"人生意义匮乏"，social_deficit 是具体的"社交不足"。social_deficit 是 stagnation_sense 的输入之一

## 三、统一数学框架

### 3.1 核心公式：统一匮乏→紧迫映射

所有生理需求共享同一公式：

```
urgency_i = (1.0 - current_i) × personality_sensitivity_i
```

- `(1.0 - current_i)`：当前缺乏程度。hunger=0.3 → 缺乏=0.7
- `personality_sensitivity_i`：该 NPC 对此需求的个体敏感度,从大五人格派生

**为什么选择线性而非 sigmoid/幂函数**：
- 线性对调参者最透明——urgency=0.37 的意义直接可读
- 个体差异通过 sensitive_i 体现——两个 NPC 对同一 hunger 不同 response,来自人格差异,而非数学曲线差异
- 对玩家最友好——行为的"可解释性"是活人感的关键。玩家观察 NPC:"这人神经质高,所以饿得快"——一句话解释完
- 极简——一个 f32 乘法,0.5ns。对性能最优

### 3.2 personality_sensitivity 派生表

从大五人格直接查表——不新增 NPC 字段：

| 需求 | sensitivity 公式 | 多样性示例 |
|------|-----------------|-----------|
| hunger | `0.5 + neuroticism × 0.5` | 神经质 0.9→sensitivity 0.95; 神经质 0.1→0.55 |
| thirst | `0.5 + conscientiousness × 0.2 + neuroticism × 0.25` | 自律+焦虑者定时喝水 |
| fatigue | `0.5 + (1.0 - extraversion) × 0.3` | 外向者 0.9→0.53; 内向者 0.2→0.74 |
| element_balance | `0.4 + conscientiousness × 0.45 + neuroticism × 0.15` | 自律者更关注身体平衡 |
| libido | `0.3 + extraversion × 0.4 + openness × 0.3` | 外向开放者 libido 行为更活跃 |
| social | `0.2 + extraversion × 0.8` | 外向 0.9→sensitivity 0.92; 内向 0.1→0.28 |

### 3.3 跨需求耦合（极简版）

只保留真实生理学支持的耦合——稀疏矩阵,非 O(N²)：

```
// 睡眠债务 → 放大饥饿压力 (ghrelin/leptin 调节)
urgency_hunger *= 1.0 + fatigue × 0.25;

// 脱水 → 放大疲劳压力
urgency_fatigue *= 1.0 + (1.0 - thirst) × 0.2;

// 社交匮乏 → 放大各类需求 (孤独→暴食/失眠)
urgency_all *= 1.0 + social_deficit × 0.1;
```

三个乘法,总成本 <1ns。种族差异通过耦合系数实现——但 v1.0 先用全局值,后续可通过 `RaceTraits` 扩展。

### 3.4 昼夜节律

```
circadian_modifier(hour_of_day, chronotype) = 
    1.0 + 0.1 × sin(2π × (hour - chronotype.phase) / 24)

// chronotype: "早鸟型"phase=6am峰, "夜猫型"phase=1am峰
// 影响 fatigue 和 hunger 的 urgency
```

## 四、与现有架构的嵌入

```
Life.Vitals (客观生物事实: hunger/thirst/fatigue/hp/stamina/spirit
                          + element_surplus[8]/libido)
    │
    │  Physiology::from_vitals(v, perception_context)
    │
    ▼
NPC.Physiology (主观感知, 0-1 归一化: + element_balance_urgency/libido)
    │
    ├──→ GOAP 安全网 (仅 ①-④, 阈值不变)
    ├──→ 概率决策器 (need_action_match, 取代 physiology_modifier)
    └──→ 情绪引擎 (apply_physiological_pull, 加 3 行)

NPC.NpcData.social_deficit (心理状态, 不在 Physiology 中)
    │
    └──→ 概率决策器 + 情绪引擎
```

**关键原则**：
- Life 是全部生物数据的唯一权威——不改
- Physiology 是 NPC 的主观感知层——不改角色,只加字段
- GOAP 安全网不改——只有生存需求有资格进
- 概率决策器的 physiology_modifier → need_action_match 重构

## 五、数据存储边界

| 字段 | 模块 | 存储类型 | 理由 |
|------|------|---------|------|
| element_surplus: [f32; 8] | Life Vitals | SoA 列 ×9 | 身体事实,和 hunger 同级 |
| libido: f32 | Life Vitals | SoA 列 ×1 | 身体信号,和 hunger 同级 |
| blood_element_ratio: [f32; 8] | Life RaceTraits | 种族级常量 | 种族参数 |
| libido_type + libido_cycle_days | Life RaceTraits | 种族级常量 | 种族参数 |
| social_deficit: f32 | NPC NpcData | SoA 列 ×1 | 心理状态,非生物性 |
| element_balance_urgency: f32 | NPC Physiology | 每决策周期计算 | 从 Vitals 派生 |
| reproductive_urge: f32 | 不存储 | 每决策周期计算 | libido × culture × faith × personality |

**新增存储总量**：~44 bytes/NPC (Life: 36 + NPC: 4 + Physiology: 4)。
100K NPC ≈ 4.4 MB。可忽略。

## 六、未解决的问题（待讨论）

1. 玩家的 social_deficit？——玩家角色技术上也走同一代码路径,因持续交互自然不累积。荒野独居 50 天→一些累积→沉浸感非惩罚。需确认。
2. 亡灵/植物的需求？——亡灵无 hunger/thirst,也无 element_surplus。植物无 Vitals。当前已有 diet_type 分支覆盖。需确认无遗漏。
3. 儿童/老年的特殊处理？——年龄已经在 LifeEntity 中,libido 的 age_factor 在 Life 模块计算。social_deficit 不受年龄影响。
