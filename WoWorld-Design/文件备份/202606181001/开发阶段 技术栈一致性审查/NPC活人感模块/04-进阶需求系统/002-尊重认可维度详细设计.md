# 002 — 尊重/认可维度详细设计

## 一、核心模型

### 1.1 基本定义

```
esteem_deficit: f32 ∈ [0.0, 1.0]
  0.0 = "我被充分看见、充分认可"
  1.0 = "没有人知道我做了什么，没有人在乎我"

存储位置: NpcData（非 Life.Vitals — 这是心理状态，不是生物信号）
```

### 1.2 与其他需求的区分

| 需求 | 描述 | 本质 |
|------|------|------|
| social_deficit | "我身边有人吗？有人跟我说话吗？" | 社交**连接的量** |
| **esteem_deficit** | "人们尊重我吗？我的成就被认可吗？" | 社交**地位的质** |
| competence_frustration | "我达到自己的标准了吗？" | 内在**能力评价** |

**三者独立**：
- 一个被很多人围绕着但不受尊重的弄臣 → social↓, esteem↑
- 一个在远方被传颂的传奇英雄 → social↑, esteem↓
- 一个自我怀疑的大师 → competence↑, esteem↓
- 一个自信的隐士 → competence↓, social↑（但不一定 esteem↑——取决于他是否在乎外界认可）

## 二、累积与代谢

### 2.1 被动累积（认可的自然褪色）

```
daily_passive_accumulation = 0.01 / 游戏日

→ 100 天从 0.0 到 1.0（如果完全没有任何认可）
→ 对比 social_deficit 的 0.02/天——社会连接比社会认可流失更快
```

**设计理由**：
- 被认可一次（比如造了一把名剑）的"心理保质期"比一次社交互动长
- 但认可确实会褪色——"你昨天是英雄，今天只是普通人"
- 0.01 意味着 NPC 不需要每天都被夸——每 10 天左右有一些认可就够了

### 2.2 主动恢复（钦佩检测）

esteem_deficit 的恢复**不由其他模块推送**——由 NPC 模块从已有社交管道**拉取**。

```rust
// NPC 模块内部函数 — 每次社交互动后调用
fn process_social_esteem_impact(
    interaction: &SocialInteraction,
    npc: &NpcData,
    culture: &dyn CultureQuery,
    relationship: &dyn RelationshipQuery,
) -> f32 {
    let mut recovery = 0.0;

    // 检测 1: 直接的赞美/钦佩
    if interaction.has_speech_act(SpeechAct::Praise)
       || interaction.has_speech_act(SpeechAct::Compliment)
       || interaction.has_speech_act(SpeechAct::Gratitude) {

        // 赞美者的"分量"影响恢复量
        let praiser_status = relationship.get_perceived_status(
            interaction.from, npc.id
        ).unwrap_or(0.5);

        // 赞美者的诚意（来自 relationship.trust × agreeableness）
        let sincerity = estimate_sincerity(interaction, relationship);

        recovery += 0.02                          // 基础: 被夸就是好的
                  + praiser_status * 0.05         // 高地位者的赞美更重
                  + sincerity * 0.03;             // 真诚的赞美更重
        // 范围: 0.02 ~ 0.10
    }

    // 检测 2: 服从/屈服信号（负面认可——被恐惧也是一种"认可"）
    if interaction.has_speech_act(SpeechAct::Submission)
       || interaction.has_speech_act(SpeechAct::Deference) {
        // 对低宜人性 NPC，被畏惧也能降低 esteem_deficit
        let fear_as_esteem = (1.0 - npc.big_five.agreeableness) * 0.5;
        recovery += 0.01 + fear_as_esteem * 0.05;
        // 范围: 0.01 ~ 0.06
    }

    // 检测 3: 被谈论（间接认可）
    // NPC 从记忆中检测到近期有人提到自己
    // 这种"间接认可"幅度小但持续
    // 在 SelfNarrative 反思时处理（7天周期）——不在每次互动中处理

    recovery
}
```

### 2.3 自我认可（Self-Esteem from Achievement）

除了社交钦佩，NPC 也可以从自己的成就中获得自尊：

```rust
// SelfNarrative.reflect() 中处理 — 每 7 游戏天
fn self_evaluate_achievements(
    recent_memories: &[EventMemory],
    self_concept: &SelfConcept,
) -> f32 {
    let mut self_esteem_gain = 0.0;

    for memory in recent_memories {
        // 成就性记忆（锻造杰作、战斗胜利、技能突破等）
        if memory.event_type.is_achievement() {
            // 自我评价: 这件事符合我的自我概念吗？
            let self_concept_alignment = self_concept.alignment_with(memory);
            // 如果符合"我是战士"的自我概念 → 战斗胜利给予更多自尊
            // 如果不符合 → 给予较少

            self_esteem_gain += memory.impact * 0.15
                              * (0.5 + self_concept_alignment * 0.5);
        }
    }

    self_esteem_gain.clamp(0.0, 0.5) // 自我认可有上限——社交认可不可完全替代
}
```

**关键设计**：自我认可最多恢复 0.5（50%）。完全的 esteem 满足需要社会认可。这避免了"隐士完全不在乎他人"的极端——即使最独立的 NPC，如果 100 天没有任何人认可，esteem_deficit 依然会上升到 0.5（0.01×100 - 0.5 自我恢复 → 仍然有 0.5 的 deficit）。

但这个设计有一个微妙的问题：如果 NPC 是真正的隐士（低外向、低宜人），也许他们真的不在乎？这引出了一个重要讨论——

### 2.4 Personality Modulation of Esteem Importance

不是所有 NPC 都同等在乎认可。NeedSensitivity 已通过 sensitivity 处理了"多急迫"，但还有一个维度：**认可恢复的效率是否也受人格影响？**

```
方案 A（简单）：social 管道检测 + self_evaluate → 统一恢复量
  人格差异仅通过 sensitivity 影响 urgency

方案 B（深度）：人格也影响 recovery 效率
  高外向者从社交赞美中恢复更多（×1.3）
  低外向者从自我成就中恢复更多（×1.2）
  高神经质者对赞美半信半疑（×0.7）

方案 C（最简）：只用 sensitivity，recovery 公式统一
```

**推荐方案 A**——保持 recovery 公式统一，让人格通过 sensitivity 产生行为差异。这更简洁，且避免了"外向者被夸恢复快→更少寻求认可→行为反而更不外向"的反直觉结果。

实际的行为差异应该来自：
- 高外向者 → sensitivity 高 → urgency 信号更强 → 更频繁地做寻求认可的事
- 低外向者 → sensitivity 低 → 即使 deficit 高也不会很急迫 → 行为上更"淡定"

## 三、文化调制：honor_weight_for_domain

### 3.1 问题

不同文化对不同类型的认可有不同的权重。在军事主义文化中，战斗胜利获得尊重。在艺术文化中，创作杰作获得尊重。

### 3.2 从 CultureCoreParams 派生

```rust
// 文化系统 — CultureQuery 新增方法
fn honor_weight_for_domain(&self, culture_id: CultureId, domain: RecognitionDomain) -> f32 {
    let params = self.core_params(culture_id);

    match domain {
        RecognitionDomain::Combat => {
            // militarism 直接映射
            params.militarism
        }
        RecognitionDomain::Artistic => {
            // artistry 直接映射
            params.artistry
        }
        RecognitionDomain::Scholarly => {
            // 长期导向 + 开放性 → 学术重视度
            (params.long_term_orientation * 0.6 + params.openness_to_outsiders * 0.4)
        }
        RecognitionDomain::Faith => {
            // religiosity 直接映射
            params.religiosity
        }
        RecognitionDomain::Power => {
            // 权力距离 → 对权威的重视度
            params.power_distance
        }
        RecognitionDomain::Wealth => {
            // 放纵 + 竞争导向 → 对财富展示的认可
            (params.indulgence * 0.5 + params.competition_orientation * 0.5)
        }
        RecognitionDomain::Crafting => {
            // 长期导向 + 艺术性 → 对工艺的尊重
            (params.long_term_orientation * 0.5 + params.artistry * 0.3 + 0.2)
            // +0.2 基础: 任何文化都尊重好的工匠
        }
        RecognitionDomain::Social => {
            // 集体主义(低个人主义) → 社交能力更受尊重
            (1.0 - params.individualism) * 0.7 + 0.3
            // +0.3 基础: 任何文化都尊重善于社交的人
        }
    }
    // 所有值 clamp 到 [0.05, 1.0] — 没有任何文化对某领域的认可权重为 0
}
```

### 3.3 效果举例

| 文化 | militarism | artistry | Combat honor | Artistic honor |
|------|-----------|----------|-------------|----------------|
| 兽人战士文化 | 0.9 | 0.1 | 0.9 | 0.1 |
| 精灵艺术文化 | 0.1 | 0.9 | 0.1 | 0.9 |
| 矮人工匠文化 | 0.5 | 0.7 | 0.5 | 0.7 |
| 人类商业文化 | 0.3 | 0.4 | 0.3 | 0.4 (Wealth=0.7) |

涌现效果：
- 兽人铁匠锻造传说级战斧 → 在兽人文化中获得大量认可（Combat相关）
- 同一个兽人铁匠锻造了精美雕像 → 在兽人文化中几乎无人赞赏，esteem 恢复极少
- **NPC 自然倾向于追求本文化认可的成就路径**——不是因为脚本，而是因为只有这些路径能有效缓解 esteem_deficit

## 四、边界情况与极端场景

### 4.1 完全孤立

NPC 被困在荒岛上——无人可见：
- 被动累积：+0.01/天
- 社交恢复：0（无人可夸）
- 自我恢复：最多 0.5（self_evaluate）
- 结果：50 天后 esteem_deficit 稳定在 ~0.5——"我不需要别人认可，但我仍想知道我的存在有意义"

### 4.2 全民公敌

NPC 被整个社群憎恨：
- 社交互动中充满 insult/condemnation 而非 praise
- 负面社交信号**增加** esteem_deficit（被蔑视比被忽视更糟？）
- 是否应该让 esteem_deficit 超过 1.0 进入"负面尊重"？

**推荐**：不。esteem_deficit 的上限是 1.0。被憎恨和被忽视的不一样——但我们的模型只用 deficit（缺失认可），不建模"负面认可"。被憎恨的 NPC 可能需要其他系统的信号（关系负值、社交互动中的敌意）来表达。**单一维度不做多重概念承载。**

### 4.3 死后认可

NPC 死后才被认可（梵高效应）：
- WoWorld 不建模死后心理状态
- 但这是历史系统的范畴——PhysicalBook 中记载的认可、Legacy 的传承
- 不需要在 esteem_deficit 中处理

### 4.4 假认可（奉承/阿谀）

NPC 被虚假的奉承包围：
- sincerity 检测限制了假奉承的效果（真诚度低 → recovery 小）
- 高智力的 NPC 可能更好地检测 insincerity
- 但这是一个次要的 [TUNING] 细节，v1.0 不做

## 五、设计余量：未来可能的扩展

### 5.1 地位锚定

```
v2.0 可能: "esteem_baseline" — NPC 习以为常的认可水平
  - 长期被尊重的 NPC 的 esteem "设定点"上移
  - 突然失去地位 → 比一直不受尊重的人感到更严重的 deficit
  - 这解释了"曾经的大人物落魄后比普通人更痛苦"
```

### 5.2 比较性认可

```
v2.0 可能: "relative_esteem" — 与同辈比较的认可
  - NPC 不只看绝对认可，也看相对认可
  - "我今年被夸了10次" vs "我弟弟被夸了30次"
  - 需要 social comparison 机制，暂不设计
```

### 5.3 领域特定认可

```
v2.0 可能: 不同领域的认可独立追踪
  - 一个战士可能在战斗中饱受尊重，但在社交中感到不被认可
  - 需要 [RecognitionDomain; 8] 而非单 f32
  - v1.0 用标量——简化但保留了领域区分（通过 domain-specific recovery）
```
