# 003 — 判断维度详细设计

## 一、AestheticJudgment 完整定义

```rust
/// 审美判断——主体对客体的多维审美响应
/// Copy 类型，16 bytes（4 × f32）
#[derive(Copy, Clone, Default)]
pub struct AestheticJudgment {
    /// 愉悦度 [-1, 1]
    /// 负=厌恶/排斥，0=中性/无感，正=愉悦/喜欢
    pub valence: f32,
    
    /// 激活度 [0, 1]
    /// 0=平静/无波动，1=极度兴奋/激动/紧张
    pub arousal: f32,
    
    /// 兴趣度 [0, 1]
    /// 0=无聊/不值得关注，1=极度着迷/移不开视线
    pub interest: f32,
    
    /// 敬佩度 [0, 1]
    /// 0=轻视/不值一提，1=极度敬佩/肃然起敬
    pub respect: f32,
}
```

## 二、judge() 函数：完整数学形式

### 2.1 概述

```rust
fn judge(
    signal: &AestheticSignal,
    taste: &AestheticTaste,
    ctx: &AestheticContext,
) -> AestheticJudgment
```

纯函数。零副作用。任何模块可调用。

三步流程：
1. **感知抖动** (perceptual jitter)：对 signal 每维加确定性噪声
2. **维度映射** (dimension mapping)：signal ⊗ taste → 4 维判断
3. **情境调制** (context modulation)：stakes/social/mood → 修正判断

### 2.2 第一步：感知抖动

```rust
fn apply_perceptual_jitter(signal: &AestheticSignal, seed: u64) -> AestheticSignal {
    let mut rng = DeterministicRng::from_seed(seed);
    AestheticSignal {
        fluency:        (signal.fluency        + rng.f32_range(-0.02, 0.02)).clamp(0.0, 1.0),
        novelty:        (signal.novelty        + rng.f32_range(-0.02, 0.02)).clamp(0.0, 1.0),
        complexity:     (signal.complexity     + rng.f32_range(-0.02, 0.02)).clamp(0.0, 1.0),
        harmony:        (signal.harmony        + rng.f32_range(-0.02, 0.02)).clamp(0.0, 1.0),
        expressiveness: (signal.expressiveness + rng.f32_range(-0.02, 0.02)).clamp(0.0, 1.0),
        virtuosity:     (signal.virtuosity     + rng.f32_range(-0.02, 0.02)).clamp(0.0, 1.0),
    }
}
```

- seed = hash(npc_id, target_hash, hour_of_day) —— 确定性，存档可复现
- 同一 NPC 同一小时看同一对象 → 相同 jitter
- ±2% 意味着大师之作（全部 0.85+）→ 永远是大师之作；边缘作品可能在"还行"和"不太行"之间摇摆

### 2.3 第二步：维度映射

#### valence 计算

```rust
fn compute_valence(signal: &AestheticSignal, taste: &AestheticTaste, ctx: &AestheticContext) -> f32 {
    // ── 基础愉悦：从 fluency/harmony/expressiveness 加权 ──
    let fluency_term  = signal.fluency * taste.dimension_weights[0];
    let harmony_term  = signal.harmony * taste.dimension_weights[3];
    let expressive_term = signal.expressiveness * taste.dimension_weights[4];
    let base_valence = (fluency_term * 0.45 + harmony_term * 0.35 + expressive_term * 0.20);
    
    // ── novelty 对 valence 的倒 U 型效应 ──
    // 峰值位置由 taste.aesthetic_openness 决定
    let novelty_peak = 0.3 + taste.aesthetic_openness * 0.5;
    // 高斯型：exp(-(x-peak)² / (2σ²))
    let novelty_valence = (-(signal.novelty - novelty_peak).powi(2) 
                          / (2.0 * 0.15_f32.powi(2))).exp();
    let novelty_weight = taste.dimension_weights[1];
    let novelty_term = novelty_valence * novelty_weight;
    
    // ── complexity 对 valence 的调制（适度的复杂增加愉悦，过度则减少）──
    let complexity_optimum = 0.3 + taste.complexity_tolerance * 0.5;
    let complexity_valence = (-(signal.complexity - complexity_optimum).powi(2)
                             / (2.0 * 0.25_f32.powi(2))).exp();
    let complexity_term = complexity_valence * taste.dimension_weights[2] * 0.3;
    
    // ── virtuosity 对 valence 的贡献（技艺好→愉悦，但线性，无倒U）──
    let virtuosity_term = signal.virtuosity * taste.dimension_weights[5] * 0.5;
    
    // ── 文化冲突检测（高 expressiveness + 低 harmony + 低 cultural_fit → 冒犯）──
    let cultural_fit = compute_cultural_fit(signal, taste);
    let offensive_penalty = if signal.expressiveness > 0.6 
                            && signal.harmony < 0.3 
                            && cultural_fit < 0.3 {
        (1.0 - cultural_fit) * 0.6  // 最大 -0.6 的惩罚
    } else { 0.0 };
    
    // ── 合成 ──
    let raw = base_valence + novelty_term * 0.25 + complexity_term * 0.15 
            + virtuosity_term * 0.15 - offensive_penalty;
    
    // 映射到 [-1, 1]
    (raw * 2.0 - 1.0).clamp(-1.0, 1.0)
}
```

**设计要点**：
- fluency 和 harmony 是 valence 最强的正向贡献（0.45 + 0.35 = 0.80）
- novelty 是倒 U 型——太新=困惑，适度新=愉悦
- complexity 也是倒 U 型——太简单=乏味，太复杂=困惑
- virtuosity 是线性的——技艺越好越愉悦（但不保证——低 harmony 的大师作品可能 virtuosity 高但 valence 低）
- 冒犯检测是唯一的非线性惩罚项

#### arousal 计算

```rust
fn compute_arousal(signal: &AestheticSignal, taste: &AestheticTaste, ctx: &AestheticContext) -> f32 {
    let novelty_term  = signal.novelty  * taste.dimension_weights[1] * 0.40;
    let expressive_term = signal.expressiveness * taste.dimension_weights[4] * 0.35;
    let complexity_term = signal.complexity * taste.dimension_weights[2] * 0.25;
    
    let raw = novelty_term + expressive_term + complexity_term;
    
    // sigmoid 激活（温和的非线性）
    let activated = 1.0 / (1.0 + (-8.0 * (raw - 0.5)).exp());
    
    // stakes 调制
    let stakes_amp = 1.0 + ctx.stakes.mortality_salience * 0.4;
    
    (activated * stakes_amp).min(1.0)
}
```

**设计要点**：
- arousal 的主要驱动是 novelty + expressiveness + complexity——新+有情感+复杂 → 激活
- stakes（生死攸关）显著放大 arousal
- sigmoid 确保低 arousal 事件（日常物品）的 arousal 确实低，但高质量事件快速攀升

#### interest 计算

```rust
fn compute_interest(signal: &AestheticSignal, taste: &AestheticTaste, ctx: &AestheticContext) -> f32 {
    // ── interest 来自新奇度 × 复杂度 的交互 ──
    let novelty_weight = taste.dimension_weights[1];
    let complexity_weight = taste.dimension_weights[2];
    
    // 交互项：又新又复杂 → 兴趣最高
    let interaction = signal.novelty * novelty_weight * signal.complexity * complexity_weight;
    
    // 但需要至少一定的 harmony——否则只是混乱而非有趣
    let harmony_gate = signal.harmony.max(0.15);  // 地板 0.15
    
    // fluency 对 interest 的影响：太流畅=无聊，适度不流畅=激发好奇心
    let fluency_inverse = 1.0 - signal.fluency;
    
    let raw = interaction * 0.50 + fluency_inverse * 0.20 + signal.novelty * novelty_weight * 0.30;
    let gated = raw * harmony_gate;
    
    // prior_expectation 放大 interest——"这是大师名作"→更仔细看
    let expectation_amp = 1.0 + ctx.prior_expectation * 0.4;
    
    (gated * expectation_amp).min(1.0)
}
```

**设计要点**：
- interest 的关键是 novelty × complexity 交互——又新又复杂才有东西可探索
- harmony_gate 确保兴趣不被混乱淹没
- fluency_inverse：太完美的东西反而无聊（"太熟悉了"）
- prior_expectation 放大 interest 但不放大 valence——"名作"让人更想细看，但不一定让人更喜欢

#### respect 计算

```rust
fn compute_respect(signal: &AestheticSignal, taste: &AestheticTaste, ctx: &AestheticContext) -> f32 {
    let virtuosity_term = signal.virtuosity * taste.dimension_weights[5] * 0.50;
    let complexity_term  = signal.complexity  * taste.dimension_weights[2] * 0.25;
    let expressive_term  = signal.expressiveness * taste.dimension_weights[4] * 0.25;
    
    let raw = virtuosity_term + complexity_term + expressive_term;
    
    // prior_expectation 放大 respect——"这是传说中的XX大师的作品"→更敬佩
    let expectation_amp = 1.0 + ctx.prior_expectation * 0.2;
    
    // stakes 对 respect 的放大——"他在生死决斗中使出这招"→更敬佩
    let stakes_amp = 1.0 + ctx.stakes.mortality_salience * 0.3;
    
    (raw * expectation_amp * stakes_amp).min(1.0)
}
```

**设计要点**：
- virtuosity 是 respect 的最强驱动（0.50）
- complexity + expressiveness 辅助支撑——复杂 + 有情感 = 深度
- prior_expectation 影响 respect（"名气效应"）——这是合理的：知道是大师之作后确实更敬佩
- stakes 放大 respect——生死关头的技艺更令人敬佩

### 2.4 第三步：情境调制

```rust
fn apply_context_modulation(
    judgment: &mut AestheticJudgment,
    ctx: &AestheticContext,
    taste: &AestheticTaste,
) {
    // ── 1. 情绪偏置 ──
    // 心情好 → 看什么都更顺眼一点（但有上限）
    if ctx.mood_valence > 0.2 {
        judgment.valence += ctx.mood_valence * 0.15;
    } else if ctx.mood_valence < -0.2 {
        judgment.valence += ctx.mood_valence * 0.10;  // 负面偏置更小（不对称）
    }
    judgment.valence = judgment.valence.clamp(-1.0, 1.0);
    
    // ── 2. 熟悉度衰减 ──
    let familiarity_penalty = 1.0 - ctx.familiarity * 0.6;
    judgment.valence *= familiarity_penalty.max(0.2);  // 至少保留 20%
    judgment.interest *= familiarity_penalty.max(0.3); // 兴趣衰减更多
    
    // ── 3. 集体同步放大 ──
    if let Some(synchrony) = ctx.social.audience_synchrony {
        judgment.valence *= 1.0 + synchrony * 0.3;
        judgment.arousal *= 1.0 + synchrony * 0.4;
    }
    
    // ── 4. 认知负载压缩 ──
    // 高负载下精细维度分辨率下降
    if ctx.cognitive_load > 0.6 {
        let resolution = 1.0 - (ctx.cognitive_load - 0.6) * 1.5;
        judgment.interest *= resolution.max(0.2);
        // valence 不受认知负载影响——gut feeling 不需要认知资源
    }
    
    // ── 5. 时间预算 ──
    // 没时间细看 → interest 被压制
    if ctx.available_time_ms < 500 {
        judgment.interest *= (ctx.available_time_ms as f32 / 500.0).max(0.1);
    }
}
```

### 2.5 完整 judge() 实现

```rust
pub fn judge(
    signal: &AestheticSignal,
    taste: &AestheticTaste,
    ctx: &AestheticContext,
) -> AestheticJudgment {
    // Step 1: 感知抖动
    let jittered = apply_perceptual_jitter(signal, ctx.jitter_seed);
    
    // Step 2: 维度映射
    let mut judgment = AestheticJudgment {
        valence:  compute_valence(&jittered, taste, ctx),
        arousal:  compute_arousal(&jittered, taste, ctx),
        interest: compute_interest(&jittered, taste, ctx),
        respect:  compute_respect(&jittered, taste, ctx),
    };
    
    // Step 3: 情境调制
    apply_context_modulation(&mut judgment, ctx, taste);
    
    judgment
}
```

## 三、判断组合的类型涌现

不同 4 维组合涌现出不同的审美体验标签（标签不存储在判断中——是消费方根据值域自行推断）：

| 体验类型 | valence | arousal | interest | respect | 典型场景 |
|---------|---------|---------|---------|---------|---------|
| **美** (beauty) | >0.6 | <0.4 | >0.3 | 任意 | 看一幅和谐的风景画 |
| **震撼** (stunning) | >0.7 | >0.6 | >0.6 | >0.7 | 第一次看哥特教堂内部 |
| **崇高** (sublime) | 0.2~0.6 | >0.7 | >0.5 | >0.8 | 面对暴风雨/巨大山脉 |
| **有趣** (interesting) | >0.2 | 任意 | >0.7 | 任意 | 看到新颖的艺术风格 |
| **可爱** (cute) | >0.6 | 0.1~0.4 | >0.3 | <0.3 | 看一只小猫 |
| **无聊** (boring) | -0.1~0.1 | <0.1 | <0.15 | <0.2 | 流水线杯子 |
| **廉价** (cheap) | -0.2~0 | <0.2 | <0.2 | <0.15 | 粗制滥造的东西 |
| **冒犯** (offensive) | <-0.5 | >0.4 | >0.3 | 任意 | 与自己价值观冲突的强烈表达 |
| **做作** (pretentious) | -0.3~0 | >0.3 | <0.3 | <0.4 | 高情感低技艺的拙劣模仿 |
| **令人敬佩但不愉悦** | <-0.2 | 任意 | >0.4 | >0.7 | "我不喜欢但承认厉害" |
