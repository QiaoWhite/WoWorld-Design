## 一、创新不等于新系统

**创新的机制在NPC crate（MentalModel产生），但创新的产物在领域crate（战斗/魔法/艺术/建筑）。**

NPC crate的认知架构不知道"这是战斗创新还是艺术创新"——它只知道"我产生了一个新MentalModel，它在某个domain上，confidence足够高，我需要把它转化成可用的形式。"

**formalize_innovation()返回的是参数数据——不是领域crate的类型。** 调用方将参数传给领域crate的已有构造API。

## 二、统一创新管线6阶段

```
阶段1：前驱条件积累
  技能等级>70 + CognitiveStyle适配 + 跨domain经验 + 低负载 + Zeigarnik张力
  
阶段2：CreativeLeap（创造性飞跃）
  跨界结构模板匹配 → 新MentalModel(confidence=0.15-0.3, hypothesis=true)
  
阶段3：主动验证（MentalModelTest Intent）
  产生测试意图 → 实际行动 → 结果→记忆 → micro_digest更新confidence
  可能需要多轮验证：实验→失败→调整模型→再实验
  
阶段4：形式化（formalize_innovation）
  confidence>0.7 + supporting_count>3 → MentalModel.domain→InnovationArtifact参数
  该参数传递给领域crate的已有构造API
  
阶段5：传播
  WisdomSharing / 书籍 / DeepMentorship / 演示→他人观察
  领域产物本身（新武器/新建筑/新画作）成为传播载体
  
阶段6：采用与演化
  其他NPC assess_and_integrate → 形成新流派/新技术传统
  长期→可能改变文化参数（对标Event-driven cultural change）
```

## 三、创新前驱条件：can_innovate()

```rust
pub struct InnovationPotential {
    pub level: InnovationLevel,  // None / Low / Medium / High
    pub strongest_domain: MentalModelDomain,
    pub readiness: f32,          // 0-1
}

pub fn can_innovate(npc: &NpcData) -> InnovationPotential {
    // 1. 专家水平
    let best_skill = npc.skills.highest_skill_level();
    let skill_check = best_skill > 70.0;
    
    // 2. 认知风格
    let cognitive_check = npc.cognitive_style.abstract_concrete > 0.6
                       && npc.cognitive_style.rigid_flexible > 0.5
                       && npc.cognitive_style.reflective_impulsive > 0.4;
    
    // 3. 跨domain经验
    let experience_check = npc.mental_models.iter()
        .filter(|m| !m.abandoned)
        .map(|m| m.supporting_diversity)
        .max_by(|a, b| a.partial_cmp(b).unwrap())
        .unwrap_or(0.0) > 0.5;
    
    // 4. 认知潮汐
    let tide_check = npc.cognitive_tide.mind_wander_drive() > 0.7
                  || npc.cognitive_tide.mental_quietude > 0.8;  // 或有深度安静
    
    // 5. 认知压力不能太高
    let distress_check = npc.cognitive_distress < 0.5;
    
    // 6. 需要一点"求变"的感觉
    let stagnation_check = npc.self_narrative.stagnation_sense > 0.3;
    
    // 综合判断
    let checks = [skill_check, cognitive_check, experience_check, tide_check, distress_check, stagnation_check];
    let passed = checks.iter().filter(|&&c| c).count();
    
    let level = match passed {
        6 => InnovationLevel::High,
        5 => InnovationLevel::Medium,
        4 => InnovationLevel::Low,
        _ => InnovationLevel::None,
    };
    
    InnovationPotential {
        level,
        strongest_domain: npc.skills.highest_skill_domain().into(),
        readiness: passed as f32 / 6.0,
    }
}
```

**在1000个L1 NPC中，通常5-10个处于High/Medium创新潜力。大多数NPC一生中从未创新——他们维持和传递已有知识。这是历史真实的涌现。**

## 四、形式化：MentalModel→领域产物

### 4.1 统一形式化函数

```rust
/// 纯函数——MentalModel → InnovationArtifact参数
/// 返回的不是领域crate的类型，而是领域crate构造API消费的参数
pub fn formalize_innovation(
    model: &MentalModel,
    innovator: &NpcData,
) -> Option<InnovationArtifact> {
    // 前提：confidence>0.7, supporting_count>3
    if model.confidence < 0.7 || model.supporting_count < 3 { return None; }
    
    match model.domain {
        MentalModelDomain::Combat => {
            Some(InnovationArtifact::CombatTechnique(
                CombatInnovationParams::from_mental_model(model, innovator)
            ))
        },
        MentalModelDomain::Magic => {
            Some(InnovationArtifact::SpellPrototype(
                MagicInnovationParams::from_mental_model(model, innovator)
            ))
        },
        MentalModelDomain::Aesthetics => {
            Some(InnovationArtifact::AestheticInnovation(
                AestheticInnovationParams::from_mental_model(model, innovator)
            ))
        },
        MentalModelDomain::Crafting => {
            Some(InnovationArtifact::CraftingTechnique(
                CraftingInnovationParams::from_mental_model(model, innovator)
            ))
        },
        MentalModelDomain::Construction => {
            Some(InnovationArtifact::ConstructionTechnique(
                ConstructionInnovationParams::from_mental_model(model, innovator)
            ))
        },
        MentalModelDomain::Politics => {
            Some(InnovationArtifact::SocialInnovation(
                SocialInnovationParams::from_mental_model(model, innovator)
            ))
        },
        _ => None,  // 其他domain暂不形式化
    }
}
```

### 5.1 已有基础

### 5.2 创新管线示例

```
战士注意到：
  "对手用长矛时，左手低位格挡+右手高位刺击→成功率异常高"
  → try_induce_pattern → MentalModel(Combat, confidence=0.45)
  
  某战后反思 + CognitiveTide空闲 + high reflective
  → CreativeLeap:
    "低位格挡的原理=改变对手武器轨迹"
    × "匕首格斗中也有类似的轨迹改变原理(来自cross-training 0.05)"
    → "匕首低位格挡+高位刺击组合？"
  → 新MentalModel(Combat, confidence=0.2, hypothesis=true)
  
  展开 MentalModelTest Intent
  → 故意找用长矛的对手试招 → 成功3次
  → supporting_count+3, confidence→0.55 → 继续验证
  → confidence>0.7 + supporting>3
  
  形式化：
  → CombatInnovationParams {
      move_template: 左手匕首格挡+右手刺击的组合参数
      parent_style: 战士的战斗风格参数
    }
  → 传给战斗系统 CombatMoveBuilder
  → 新CombatMoveTemplate → MoveId → MoveLibrary
  
  传播：
  → Apprenticeship→DeepMentorship→新招式传给弟子
  → 多个战士采用→新匕首格斗子流派萌芽
```

## 十一、CognitiveExceptionality：天才涌现

### 11.1 定义

**天才不是类型——是罕见的参数组合。** 对标已有涌现设计原则——"只定义底层参数，让高层现象涌现。"

```rust
/// 完全从已有数据派生——零新存储
pub fn detect_exceptionality(npc: &NpcData) -> Option<CognitiveExceptionality> {
    let domain = npc.skills.highest_skill_domain();
    let skill_level = npc.get_skill_level(domain);
    let domain_models = npc.mental_models.iter()
        .filter(|m| m.domain == domain.into())
        .collect::<Vec<_>>();
    
    // 天才的三个条件AND：
    // 1. 技能>85
    // 2. 该domain的model diversity>0.7（泛化好）
    // 3. 高抽象+高反思
    let skill_check = skill_level > 85.0;
    let diversity_check = domain_models.iter()
        .map(|m| m.supporting_diversity)
        .max_by(|a, b| a.partial_cmp(b).unwrap())
        .unwrap_or(0.0) > 0.7;
    let cognitive_check = npc.cognitive_style.abstract_concrete > 0.7
                       && npc.cognitive_style.reflective_impulsive > 0.7;
    
    if skill_check && diversity_check && cognitive_check {
        Some(CognitiveExceptionality {
            domain: domain.into(),
            pattern_recognition_boost: 0.5,      // →更快的归纳(×1.5速度)
            creative_leap_boost: 0.4,            // →更高创新概率(×1.4)
            teaching_insight_boost: 0.3,         // →教学时可表达更深理解
        })
    } else {
        None
    }
}
```

### 11.2 天才的涌现频率

- 所有条件同时满足的概率：~0.001（千分之一）
- 1000个L1 NPC中约1个
- 大多数世界可能几百年才出一个

## 十二、创新与传统：动态张力

### 12.1 张力公式

```
太灵活(rigid_flexible>0.8) → 创新不断 → 传统无法积累 → 每一代重新发明轮子
太顽固(rigid_flexible<0.2) → 零创新 → 传统成死教条 → 无法适应变化
黄金平衡(0.4-0.6) → 创新+积累并存
```

### 12.2 文化层面的涌现

| 文化特征 | 创新倾向 | 历史涌现 |
|---------|---------|---------|
| 低power_distance+高openness+高individualism | 高创新+快传播 | "黄金时代" |
| 高power_distance+高uncertainty_avoidance | 创新被压制+传播慢 | "保守时期" |
| 中等所有参数+跨域贸易接触 | 创新频率适中+传统积累 | "稳步发展" |

**这不是脚本——是`CognitiveNorms`×`CognitiveStyle`×文化参数的数学涌现。**
