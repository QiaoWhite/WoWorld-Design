## 一、总览

资源分布是 P3 阶段。世界生成管线至此，地形和水文已经确定，现在填充分布其中的资源。

**v3 核心变化**：
- 从"矿物中心"扩展为**完整可开采资源分类体系**（7 大类）
- `ResourceProvider` trait 预留未来新资源类型扩展
- 矿田生成逻辑保留（大范围集中分布），按 25 万 km² 重算数量
- 新增海洋底质资源

## 三、矿田生成系统

### 3.1 核心原则

**矿物不零星散布——大范围集中分布**，可建立矿道、露天开采。

```rust
/// 矿田生成的参数
struct OreFieldParams {
    /// 矿种 ID
    ore_id: u8,
    /// 矿田中心之间的最小距离（km）
    min_distance: f64,
    /// 矿田覆盖范围（km²）
    field_size_range: (f64, f64),
    /// 矿体密度（矿田内部含矿体素的比例）
    body_density: (f32, f32),  // 0.3-0.8
    /// 生成深度范围（m）
    depth_range: (f64, f64),
    /// 储量等级分布权重
    reserve_weights: (f32, f32, f32),  // (一级%, 二级%, 三级%)
}

struct OreField {
    center: Vec2,
    radius: f64,              // 矿田覆盖半径
    ore_type: OreType,
    reserve_grade: ReserveGrade,
    body_boundary: Vec<Vec2>, // 矿体边界多边形
    depth_range: (f64, f64),
    estimated_total: u64,     // 可开采总体素格数估算
}

enum ReserveGrade {
    Small,    // 500-5,000 体素格——个人/小团体开采
    Medium,   // 5,000-50,000——聚落长期运营
    Large,    // 50,000-500,000+——政治实体级经济支柱
}
```

### 3.2 矿田生成算法

```
for each 矿种 in 矿种列表:
    1. 从 world_seed 派生矿种子种子 → hash(seed, "ore_{id}")
    2. 用低频 Perlin 噪声（频率 0.0005-0.002）确定矿田中心候选
    3. 过滤：任意两个矿田中心距离 ≥ min_distance
    4. 围绕每个矿田中心，用中频噪声确定矿体边界
       - 边界内：矿物体素密度 0.3-0.8（非 100%，留有无矿夹层）
       - 边界过渡带：矿化减弱区
    5. 储量等级 = 从 reserve_weights 随机抽样
    6. depth_range = 矿种特定深度（见 §2.1）
```

## 四、资源分布与景观的关联

```rust
/// 每种资源有其"偏好"的生成条件
struct ResourceDistribution {
    /// 偏好群系
    preferred_biomes: Vec<BiomeType>,
    /// 偏好地质层
    preferred_geology: Vec<GeologyType>,
    /// 偏好高程范围
    elevation_range: (f64, f64),
    /// 与水源的距离约束
    water_distance: WaterDistanceConstraint,
}

/// 从景观参数查询可能的资源
fn query_resources_at(pos: Vec2, terrain: &TerrainBase) -> Vec<ResourceLikelihood> {
    let biome = terrain.biomes.at(pos);
    let geology = terrain.geology_layer.at(pos);
    let elevation = terrain.elevation.at(pos);

    ALL_RESOURCES.iter()
        .filter(|r| r.distribution.matches(biome, geology, elevation))
        .map(|r| ResourceLikelihood {
            resource: r,
            probability: r.compute_probability(biome, geology, elevation),
        })
        .collect()
}
```

## 六、ResourceProvider trait（预留扩展）

```rust
/// 所有资源类型的统一接口
trait ResourceProvider: Send + Sync {
    /// 资源唯一标识
    fn resource_id(&self) -> ResourceId;

    /// 资源名称（可本地化 key）
    fn display_name_key(&self) -> &str;

    /// 资源分类
    fn category(&self) -> ResourceCategory;

    /// 查询指定位置该资源的生成概率
    fn spawn_probability(&self, pos: Vec2, terrain: &TerrainBase) -> f32;

    /// 生成该资源在指定位置的具体实例
    fn generate_at(&self, pos: Vec3, rng: &mut StdRng) -> Option<ResourceInstance>;

    /// 该资源是否可耗尽
    fn is_exhaustible(&self) -> bool;

    /// 如果可耗尽，总储量估算
    fn estimated_reserve(&self, field: &OreField) -> u64;
}

enum ResourceCategory {
    Mineral,     // 矿物
    Stone,       // 岩料
    Wood,        // 木材
    Earth,       // 土沙
    Liquid,      // 液体
    Organic,     // 有机
    Magic,       // 魔法
}
```

新资源类型只需实现 `ResourceProvider` trait 并注册到 `ResourceRegistry`。

## 七、采矿场与采石场的预生成

保留原方案的采矿场预生成逻辑——世界生成时，距聚落 ≤ 2km 的矿田自动分配并生成采矿场建筑群。

**采矿场附属设施**（保留+扩展）：

| 设施 | 功能 | 出现条件 |
|------|------|---------|
| 采掘面/矿坑 | 实际开采区域（体素修改区） | 全部 |
| 选矿场 | 矿石筛选、初步加工 | 小镇级以上 |
| 矿工营房 | 矿工休息/居住建筑 | 小镇级以上 |
| 工具库/铁匠铺 | 工具维修和制造 | 全部 |
| 运输道路 | 连接采矿场到最近聚落 | 全部 |
| 废石堆/排土场 | 废弃土石堆放区 | 全部 |
| 管理机构 | 矿务官员/矿主办公处 | 城市级 |
| 警卫哨站 | 防盗矿/保护矿工 | 城市级/高价值矿种 |
| **新增：冶炼厂** | 矿石就地冶炼→运输金属锭 | 城市级/中大型矿田 |
| **新增：矿工家属区** | 大型采矿场常形成独立小聚落 | 二级储量以上 |

**采石场**（岩料资源）同理——但通常更靠近聚落（石料运输成本高）。
