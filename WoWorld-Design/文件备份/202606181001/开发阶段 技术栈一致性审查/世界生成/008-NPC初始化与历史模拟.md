## 一、NPC 初始化（P8）

### 1.1 人口分配

NPC 按聚落的**承载容量上限 × 等级系数**分配，不是硬编码范围。

```rust
fn assign_population(
    settlement: &SettlementData,
    carrying: &CarryingCapacity,
    rng: &mut StdRng,
) -> u32 {
    let max_pop = carrying.population_cap(settlement.position);

    let grade_coefficient = match settlement.grade {
        SettlementGrade::Village => rng.gen_range(0.3..0.6),
        SettlementGrade::Town    => rng.gen_range(0.5..0.8),
        SettlementGrade::City    => rng.gen_range(0.7..0.95),
        SettlementGrade::Castle  => rng.gen_range(0.2..0.5),
        SettlementGrade::PortCity => rng.gen_range(0.6..0.9),
    };

    let fluctuation = rng.gen_range(0.85..1.15);
    (max_pop as f64 * grade_coefficient * fluctuation) as u32
}
```

### 1.3 NPC 属性初始化

每个 NPC 的属性生成（与 NPC 活人感模块 ver2.0 接口一致）：

- **大五人格**：从文化种子的基础人格分布中抽样 → 个体随机偏移
- **技能**：由职业决定基础技能表 → 年龄/经历修正
- **初始记忆**：少量"生成前"的虚拟记忆（"我在这个村子出生长大"、"父亲是铁匠"）
- **初始关系**：家庭关系 + 邻里关系 + 职业关系
- **住宅分配**：按社会阶层→对应居住区→分配建筑/房间

## 二、历史模拟（P9）

### 2.1 概述

世界生成后不是"全新"的。快进 N 年游戏时间（N 由种子决定，0-2000 年），让世界产生历史痕迹。

```rust
struct HistorySimulation {
    /// 快进年数
    years_to_simulate: u32,
    /// 模拟步长（月）
    step_size_months: u32,
    /// 事件概率
    event_probabilities: EventProbabilities,
}

fn simulate_history(
    world: &mut WorldState,
    config: &HistorySimulation,
    rng: &mut StdRng,
) -> HistoryResult {
    let total_steps = config.years_to_simulate * 12 / config.step_size_months;

    for step in 0..total_steps {
        // 人口变化
        tick_population(world, rng);
        // 聚落兴衰
        tick_settlements(world, rng);
        // 建筑老化
        tick_buildings(world, rng);
        // 关系演变
        tick_relations(world, rng);
        // 随机事件
        maybe_trigger_event(world, &config.event_probabilities, rng);
    }

    // 记录重要历史事件
    world.event_log.clone()
}
```

### 2.2 人口动态

```
人口变化 = 出生 - 死亡 + 移入 - 移出

出生率 = f(食物充足度, 和平度, 文化生育偏好)
死亡率 = f(食物充足度, 疾病, 战争, 年龄结构)
移入 = f(经济繁荣度, 邻近聚落衰退度)
移出 = f(资源枯竭, 战争, 压迫)

如果人口 > 承载容量:
  → 粮食短缺 → 死亡率上升
  → 移民加速流出
  → 最终人口回落到承载容量附近
```

### 2.3 聚落兴衰

```
繁荣: 资源充足 + 贸易繁荣 + 人口增长
稳定: 人口 ≈ 承载容量，缓慢波动
衰退: 资源枯竭 + 贸易中断 + 人口下降
废弃: 人口 < 最低阈值 → 聚落消失 → 留下废墟
```

### 2.4 建筑老化

应用 `005-建筑与房间生成` 中定义的质量衰减模型：
- 有主人的建筑定期维护（条件衰减 50% 速度）
- 废弃建筑加速恶化
- 重要建筑（神殿/王宫）可能被继承者维护

### 2.5 文化分化与碰撞

```rust
/// 文化隔离→分化
fn culture_divergence(
    culture_a: &CultureSeed,
    culture_b: &CultureSeed,
    isolation_years: u32,
) -> f32 {
    // 两个文化共享源头但被隔离的年数越长 → 差异越大
    let base_similarity = culture_a.similarity(culture_b);
    let divergence_rate = 0.001;  // 每年 0.1% 的分化
    base_similarity * (1.0 - divergence_rate * isolation_years as f32)
}

/// 跨洋发现→文化碰撞事件
fn culture_contact_event(
    discoverer: &CultureSeed,
    discovered: &CultureSeed,
    rng: &mut StdRng,
) -> CultureContactOutcome {
    let tech_gap = discoverer.tech_level - discovered.tech_level;
    let cultural_distance = discoverer.similarity(discovered);

    // 可能的后果：贸易/征服/和平共存/文化同化/文化融合
    match (tech_gap, cultural_distance) {
        (gap, dist) if gap > 30.0 && dist < 0.3 => CultureContactOutcome::Conquest,   // 技术碾压+文化陌生→征服
        (gap, _) if gap.abs() < 10.0 => CultureContactOutcome::Trade,                  // 技术接近→贸易
        (_, dist) if dist > 0.7 => CultureContactOutcome::PeacefulCoexistence,        // 文化高度相似→和平共存
        _ => CultureContactOutcome::CautiousExchange,                                  // 谨慎试探
    }
}
```

### 2.6 历史事件类型

| 事件类别 | 示例 | 影响 |
|----------|------|------|
| **自然灾害** | 地震、火山喷发、洪水、旱灾、瘟疫 | 人口减少、建筑损毁、饥荒 |
| **战争/冲突** | 领土战争、内战、宗教战争、海盗侵袭 | 人口锐减、建筑破坏、疆域重划 |
| **繁荣/发现** | 发现矿脉、贸易繁荣、技术突破 | 人口增长、建筑升级、文化扩散 |
| **文化/宗教** | 宗教改革、文化复兴、异教传入 | 文化参数变化、新建筑类型 |
| **政治** | 王权更替、政体变革、联盟破裂 | 统治结构变化、外交重排 |
| **探索** | 发现新大陆、首次环球航行 | 跨洋接触、文化碰撞（见上） |

## 四、与 NPC 系统的接口

世界生成产出的 NPC 初始数据 → 对接 NPC 活人感模块 ver2.0：

- NPC 人格/技能/记忆/关系 → 对应 `NpcStore` SoA 布局
- NPC 住宅/工作地点 → 对应 NPC 日常行为树的物理位置
- 初始经济状态（库存/价格/货币） → 对应经济系统初始化
