## 一、生命周期总览

```
出生（P2.5 生成 FaithDefinition）
  → 扩散（5 种渠道——全部 NPC→NPC 接触）
    → 发展（FaithTheology 代际漂移 + 民间分歧累积）
      → 改变（4 种路径——分裂/融合/改革/强制改变）
        → 消亡（信徒<临界值 → extinct）
          → 复活（极少见——考古+学者+新信徒 → 宗教复兴）
```

**"有人的地方才有信仰"贯穿全程**——信仰传播不是"区域参数变化"，而是 NPC→NPC 的个体接触传染。

## 二、扩散——五种传播渠道

### 2.1 日常接触（同聚落内缓慢传染）

```rust
/// 每游戏月 batch 处理。两个 NPC 社交时自动检测。
/// 传播公式：transmitter 的 participation × 传染系数 → receiver 的 participation 微增。
fn daily_contact_contagion(
    transmitter: &ReligiousPracticeProfile,
    receiver: &mut ReligiousPracticeProfile,
    relationship_strength: f32,  // affection + trust 综合
) {
    for (faith_id, t_weight) in &transmitter.participation {
        if *t_weight < 0.3 { continue; }  // 太冷淡的不传染
        
        let theology = faith_def(*faith_id).theology;
        // 传播力 = 传染者投入 × 信仰传教倾向 × 关系强度
        let contagion = t_weight * theology.proselytizing() * relationship_strength * 0.01;
        
        // 更新 receiver 的 participation
        if let Some((_, r_weight)) = receiver.participation.iter_mut().find(|(f, _)| *f == *faith_id) {
            *r_weight = (*r_weight + contagion).min(1.0);
        } else {
            if contagion > 0.01 {  // 门槛——太弱的接触不创建新条目
                receiver.participation.push((*faith_id, contagion.min(0.1)));
            }
        }
    }
}
```

**关键**：利用 NPC 模块已有的社交关系图——只沿有 relationship 的 NPC 对传播。不被街上陌生人影响信仰。

### 2.2 贸易传播（跨文化扩散的主要渠道）

```rust
/// 商人 NPC 携带信仰到目的地——与当地商人/客户社交时触发。
/// 这是跨文化信仰传播的最重要渠道。
struct TradeFaithContagion {
    /// 传染系数——取决于贸易量 × 信仰的 proselytizing 倾向
    /// 范围：0.005-0.05/每程（比日常接触强——商人与当地人深度互动）
    contagion_per_trip: f32,
    /// 距离衰减——极远程贸易中，商人可能停留更久 → 传染更深
    distance_modifier: f32,  // ln(distance_km) × 0.01
}
```

### 2.3 征服传播（爆发式强制改信）

```rust
/// 征服者对征服地 NPC 施加的改信压力。
/// 注意：这不是信仰系统的主动行为——征服者的 NPC 决策引擎选择 Compel 行为，
/// 目标为"改信"——信仰系统只提供抵抗公式。
fn forced_conversion_resistance(
    target: &ReligiousPracticeProfile,
    imposed_faith: FaithId,
    theology: &FaithTheology,
) -> f32 {
    // 抵抗 = piety × faith_as_identity × 0.8
    // 如果该信仰是身份的一部分 → 极难强制改变
    // 如果是 Habitual → 相对容易（"改就改吧，反正我也不怎么在乎"）
    let piety = target.participation.iter()
        .find(|(f, _)| *f == imposed_faith)
        .map(|(_, w)| *w)
        .unwrap_or(0.0);  // 0 = 没信过 → 无抵抗
    
    let identity_resistance = theology.faith_as_identity;
    let motivation_factor = match target.motivation {
        ReligiousMotivation::IdentityBased => 1.5,   // 极强抵抗
        ReligiousMotivation::PersonalDevotion => 1.2,
        ReligiousMotivation::Habitual => 0.3,        // "改就改吧"
        ReligiousMotivation::AntiReligious => 2.0,   // 极强抵抗——"我死也不信"
        _ => 1.0,
    };
    
    (piety * 0.5 + identity_resistance * 0.5) * motivation_factor
}
```

**注意**：征服强制改信触发 [[../历史/002-事件因果链与历史模拟|历史系统]] 的 `ReligiousPersecution` 事件。宗教清洗导致信仰转入地下（participation 不下调——改为隐藏标记）而非真正消失。

### 2.4 婚姻传播（代际——慢但稳）

跨信仰家庭中，配偶之间的 participation 缓慢互相影响（年均 +0.01-0.03）。主要信仰继承通过子女——见 [[003-NPC信仰状态与虔诚度|003]] §七。

### 2.5 传教（主动传播——TravelReason::Religious）

高 participation + 高 depth + 传教倾向的 NPC 主动旅行到异教区域传教。使用 NPC 决策引擎已有的 `TravelReason::Religious`。

## 五、消亡

### 5.1 消亡条件

```rust
/// 每游戏年检测。
fn check_extinction(faith_id: FaithId, all_snapshots: &[SettlementFaithSnapshot]) -> Option<ExtinctionEvent> {
    // 条件：活跃信徒（participation > 0.01）总数 < 50
    let total_believers: u32 = all_snapshots.iter()
        .filter(|s| s.distribution.iter().any(|(f, _)| *f == faith_id))
        .map(|s| s.total_believers_for(faith_id))
        .sum();
    
    if total_believers < 50 && extinction_duration > 100_years {
        return Some(ExtinctionEvent {
            faith_id,
            cause: if total_believers == 0 {
                ExtinctionCause::BelieverExtinction
            } else {
                ExtinctionCause::BelieverExtinction  // 或 AbsorbedBy(dominant_faith)
            },
        });
    }
    None
}
```

### 5.2 消亡后果

1. FaithGenealogy 标记 extinct
2. 神殿失去功能——世界生成标记为"废墟"或允许其他信仰接管
3. HolyDay 停止生成年度 FestivalInstance
4. 经文成为考古物品——通过历史系统的 ArtifactTrace 可被发掘
5. 最后一个信徒死亡 → 信仰真正死去

### 5.3 复活（罕见涌现）

考古发现 + 学者解读经文 + 新信徒重建实践 → 宗教复兴事件。这不是系统设计的目标——而是涌现的可能性。FaithGenealogy 支持 `Revival` 边。

## 六、教际关系

### 6.1 宽容度计算

```rust
/// 信仰 A 对信仰 B 的宽容度——0.0 = 积极敌对, 1.0 = 完全接受。
fn tolerance_toward(subject: FaithId, target: FaithId) -> f32 {
    let sub = faith_def(subject);
    let tar = faith_def(target);
    
    // 基础：1.0 - 排他性
    let base = (1.0 - sub.theology.exclusivity).max(0.0);
    
    // 教义距离惩罚——参数越接近越宽容
    let distance = theology_distance(&sub.theology, &tar.theology);
    let distance_penalty = distance * 0.3;
    
    // 历史关系修饰——查 FaithGenealogy 和 历史系统的 inter-faith 事件
    // 如果双方有长期的和平共处历史 → +0.1
    // 如果双方有过冲突/圣战 → -0.2
    
    (base - distance_penalty + history_modifier).clamp(0.0, 1.0)
}
```

### 6.2 敌对度计算

```rust
/// 0 = 友好, 1 = 圣战级别敌对。
fn inter_faith_hostility(a: FaithId, b: FaithId) -> f32 {
    if a == b { return 0.0; }
    
    // 双向取最大值——一方的敌意足够引发冲突
    let a_to_b = 1.0 - tolerance_toward(a, b);
    let b_to_a = 1.0 - tolerance_toward(b, a);
    a_to_b.max(b_to_a)
}
```

### 6.3 圣战条件

当 `inter_faith_hostility > 0.8` 且涉及军事化护教团体（MilitantOrder）或神权政体 → 战斗系统的战争状态触发。信仰系统只提供 hostility 值——战斗系统和权力系统决定是否开战。

## 七、FaithGenealogy——信仰谱系

```rust
/// 信仰系统的内部数据——不暴露给一般消费方。
/// 对标 CultureGenealogy（文化 001）。
pub struct FaithGenealogy {
    entries: Vec<FaithGenealogyEntry>,
    edges: Vec<GenealogyEdge>,
    // 索引
    by_faith_id: HashMap<FaithId, usize>,
    children_of: HashMap<FaithId, Vec<usize>>,
    parents_of: HashMap<FaithId, Vec<usize>>,
}

struct FaithGenealogyEntry {
    faith_id: FaithId,
    origin_culture: CultureId,
    creation_game_year: i32,         // 可为负值（世界历史起点之前——公元前）
    extinction_year: Option<i32>,
    extinction_cause: Option<ExtinctionCause>,
}

enum ExtinctionCause {
    BelieverExtinction,              // 最后一个信徒死亡
    ForcedErasure,                   // 被征服者强制清除
    AbsorbedBy(FaithId),             // 被融合吸收——标记吸收方
    SchismInto(Vec<FaithId>),        // 分裂后原信仰消亡——分割成的子信仰列表
}

enum GenealogyEdge {
    Schism {
        parent: FaithId, child: FaithId,
        split_year: i32,
        doctrinal_divergence: f32,
        schism_cause: SchismCause,
    },
    Fusion {
        parents: [FaithId; 2], child: FaithId,
        fusion_year: i32,
        fusion_weight: [f32; 2],     // 两个 parent 在 child 中的权重
    },
    Reformation {
        original: FaithId, reformed: FaithId,
        reform_year: i32,
        reformed_params: FaithTheology,
    },
    Revival {
        extinct: FaithId, revived: FaithId,  // 新 FaithId——不是原 ID
        revival_year: i32,
    },
}

enum SchismCause {
    DoctrinalDispute { topic: String },
    LeadershipConflict { claimants: [String; 2] },
    RegionalIsolation { isolated_region: RegionId },
    FolkDivergence { divergence_gap: f32 },
}
```

## 八、初始信徒分配（世界生成 P8）+ 管线流程

### 8.1 初始分配公式

P9 宏观历史模拟完成后，每个 settlement 有信仰分布快照。P8 NPC 初始化时消费此快照：

```rust
/// 信仰系统提供。NPC 初始化时调用——P8 世界生成管线。
fn assign_initial_faith_profile(
    settlement_snapshot: &SettlementFaithSnapshot,
    npc_intelligence: f32,
    npc_theology_aptitude: f32,
    rng: &mut StdRng,
) -> ReligiousPracticeProfile {
    let mut participation = Vec::new();
    
    // 从 settlement 的信仰分布中加权随机选择 1-3 个信仰参与
    for (faith_id, proportion) in &settlement_snapshot.distribution {
        if rng.gen::<f32>() < *proportion {  // 概率 = 该信仰的人口比例
            let initial_weight = rng.gen_range(0.3..0.9);  // 初始权重——种子随机
            participation.push((*faith_id, initial_weight));
        }
    }
    // 上限 3 个信仰（多数 NPC 只有 1-2 个）
    participation.truncate(3);
    
    // 如果 settlement 完全世俗化（无信仰分布）→ participation 为空 → NonPracticing
    
    ReligiousPracticeProfile {
        participation,
        theological_depth: (npc_intelligence * 0.3 + npc_theology_aptitude * 0.4 + rng.gen::<f32>() * 0.1).clamp(0.0, 0.3),
        // 初始 depth 低——出厂 NPC 不是神学家
        motivation: if participation.is_empty() {
            ReligiousMotivation::Habitual  // 无参与 → 不在乎但也不反宗教
        } else {
            // 90% 的 NPC 初始为 Habitual/SocialCustom
            match rng.gen_range(0..100) {
                0..=55 => ReligiousMotivation::SocialCustom,
                56..=90 => ReligiousMotivation::Habitual,
                91..=95 => ReligiousMotivation::Pragmatic,
                96..=98 => ReligiousMotivation::IdentityBased,
                99 => ReligiousMotivation::SpiritualSeeker,
                _ => ReligiousMotivation::Habitual,
            }
        },
    }
}
```

### 8.3 宏观传播——Logistic 扩散方程

P9 历史模拟使用 logistic 扩散方程（对标语言表达 009 的谣言传播模型）：

```rust
/// 信仰 X 在某 settlement 的信徒比例 R 随时间变化：
/// dR/dt = k × R × (1 - R/N) × contagion_coeff
///
/// 其中：
///   k = 传播速率——信仰的 proselytizing 倾向 × 0.01
///   R = 当前信徒比例
///   N = 承载容量——min(1.0, 该 settlement 人口中可被转化者比例)
///   contagion_coeff = 与邻近 settlement 的信仰距离加权和
///
/// P9 每游戏年执行一次——× 数千年历史 → 稳定的信仰分布格局。
```
