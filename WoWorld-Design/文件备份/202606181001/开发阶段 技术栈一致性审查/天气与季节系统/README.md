# 天气与季节系统（Weather & Season System）— README

## 模块定位

天气与季节系统是 WoWorld 四大**地基层模块**（物品/技能/天气/对话）之三——定义世界中的气象事实和季节循环。它是纯物理模拟层：产出某时某地的客观天气数据，不参与任何模块的主观解释。

**核心原则**：天气系统只提供**客观物理事实**（温度、降水、风速、云量、能见度、地面状态）。每个消费模块通过自己的主观解释层来解释这些事实——天气系统不知道 NPC 有情绪、不知道植物有生长公式、不知道战斗有元素加成。

## 关键设计决策

| 决策 | 结论 |
|------|------|
| **数据流模式** | **统一轮询（WeatherQuery trait）——零事件总线**。所有消费方同步调用 `sample(pos, time) → WeatherSample`。天气变化速度（分钟级）远慢于消费方轮询频率（帧级到秒级）——事件总线的"即时性"无实际价值 |
| **解耦原则** | 天气系统只提供**客观物理事实**。NPC 的 SkyPerception（主观感知）、魔法的元素浓度推导（多因子聚合）、战斗的环境判定——全由各自模块自行消费 WeatherSample 推导。天气系统不知道 NPC 有情绪、不知道植物有生长公式 |
| **温度双层** | `regional_temperature`（大气温度——不考虑地表）+ `ground_temperature`（群系修正后地面温度——冠层遮阳/雪面反照/沙地辐射/水体缓冲/城市热岛）。NPC 根据自身高程上下文自选基准温度 |
| **空间粒度** | **两层**：ClimateRegion (~50km，每游戏日更新) + LocalWeather (~1km，每游戏分钟更新)。砍掉全球气压系统层——物理准确性不等于游戏价值 |
| **状态机** | **6 状态 Markov 链**（Clear→PartlyCloudy→Overcast→LightPrecip→ModeratePrecip→HeavyStorm）+ 雾独立布尔维度（可与任意状态共存）。转移矩阵参数化生成——非硬编码 |
| **极端天气** | **不命名枚举类型**。参数组合自然区分——暴风雨+风速>30m/s+海岸=台风效果，雪+强度>0.7+风速>15m/s=暴风雪效果。NPC三层响应：温和偏移→警告强制偏移→灾难级硬约束 |
| **季节** | 120 游戏天/年，48 分钟/天，均匀四季（各30天），春分开局，虚数年号。全部 [TUNING] |
| **历史气象** | 天气系统产出气象异常数据（~5,000条，其中~500灾难级），历史模块消费并转化为历史事件。天气系统不独立生成"事件" |
| **视觉** | WeatherVisualPacket（~200 bytes/帧）→ Godot 渲染层消费。调制已有体积云/天空/海洋 shader——不新增渲染系统 |
| **性能** | Rust CPU ≤0.25ms/帧（含全部轮询），降水粒子 ≤0.4ms GPU，VRAM 增量 ≤2.5MB。全部在技术栈 v3 预算内 |
