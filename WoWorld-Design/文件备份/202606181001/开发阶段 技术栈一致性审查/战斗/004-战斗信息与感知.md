# 004-战斗信息与感知

> **关联**：[[001-战斗系统总览]] · [[003-三层战斗模型]] · [[007-半自动战斗与HUD]]

## 一、核心原则：信息不对称——没有上帝视角

玩家和 NPC 共享同一套感知限制。对方 HP/体力/魔力**不可见**——只能通过视觉/听觉/经验推断。这是 WoWorld 战斗的一个关键设计——不是"公平竞技"——是"在不确定中决策"。

| 信息类别 | 可见性 | 如何推断 |
|----------|--------|---------|
| 对方 HP | ❌ 不可见 | 看伤势动画——瘸腿=腿部重伤/流血=HP在下降/气喘=体力低 |
| 对方体力 | ❌ 不可见 | 动作变慢/姿势崩坏/呼吸急促/挥击变重(累了控制不住力度) |
| 对方魔力 | ❌ 不可见 | 施法次数+施法强度/元素泄漏(发光/火焰鬃)——可能伪装 |
| 对方情绪 | 部分可见 | 表情/身体姿态——可能伪装("强装镇定") |
| 对方技能/法术 | ❌ 不可见 | 战斗中观察到:"他会火球!"——首次使用前不可知 |
| 队友位置/状态 | ✅ 可见(视线内)或通过交流 | 喊话/手势——但被遮挡时信息变为过时/模糊 |

## 二、六种信息来源

| 来源 | 示例 | 可信度默认值 | 更新频率 |
|------|------|------------|---------|
| **视觉** | 可见的敌人/动作前摇/特效/环境变化 | 高 | 每帧(<10m) / 0.2-0.5s(10-50m) / 1-2s(>50m) |
| **听觉** | 武器出鞘声/脚步声/吟唱声/呼救声/爆炸 | 中 | 事件驱动——听到→立即处理 |
| **魔法感知** | 灵元素亲和者感知魔力波动/杀气/隐身单位/附近的施法 | 低 | 每0.5-1s |
| **记忆** | 战斗前已知的敌人情报/上次识别残留的信息 | 取决于来源 | 惰性——需要时检索 |
| **推理** | 从已知信息推导——"他的法力应该快耗尽了" | 低 | 节奏层评估时 |
| **交流传递** | 战友告知/敌人嘲讽中暴露/群众呼喊 | 取决于告知者 | 事件驱动 |

## 三、信息的可信度与时效性

### 3.1 可信度层级

- **确定**：直接观测，无误——"我看到他在吟唱"
- **模糊**：感知受限，只有范围——"那边有人影，大概三到五个"
- **猜测**：推理所得，可能错误——"他的法力应该快耗尽了"
- **过时**：曾经验证但可能已变化——"三秒前他在柱子后面"

### 3.2 时效性

- **实时**：当前帧有效
- **短期有效**：数秒内有效——过期标记为过时
- **长期有效**：整场战斗有效——如敌人的种族/已知弱点/已使用的法术
- **永久**：写入 NPC 记忆——战后仍存在——影响下次遭遇

### 3.3 信息冲突处理

新信息与旧信息矛盾时 → 以可信度高的为准。可信度相当 → 进入"疑惑状态" → 可能触发谨慎行为(节奏偏稳守/周旋)。

## 四、战斗信息池——消费感官系统产出（★ v1.0 重构）

> **v1.0 重构**: 战斗不再自行感知——消费 [[../感官与知觉系统/001-感官系统总纲|感官系统的 PerceptBatch]] 统一产出，战斗情报层在此基础上做战斗专用解释。
> 感官系统不做战斗评估。战斗情报层不做原始感知。层与层之间通过 PerceptBatch 传递信息。

```rust
/// 战斗感知——消费感官系统的原始知觉 + 派生战斗专用评估
struct CombatPerception {
    /// ★ 原始输入——来自感官系统的统一知觉产出（不属于战斗模块）
    /// 战斗不修改此数据——只读消费
    percept_batch: PerceptBatch,

    /// 战斗实体情报评估——从 PerceptBatch 派生
    combatants: Vec<CombatantAssessment>,
    
    /// 空间威胁地图
    threat_map: ThreatMap,
    
    /// 环境战斗评估
    combat_environment: CombatEnvironment,
    
    /// 自我评估
    self_assessment: SelfAssessment,
    
    /// 魔法感知——独立通道（v1.0 保持现有设计，不从感官系统走）
    magic_signatures: Vec<MagicSignature>,
}

/// ★ 战斗实体评估——替代旧的 PerceivedEnemy/PerceivedAlly
/// 所有字段从 PerceptEntry 的通用特征派生——战斗不做自己的"视力检查"
struct CombatantAssessment {
    entity_id: EntityId,
    allegiance: Allegiance,              // Enemy/Ally/Neutral/Unknown
    
    // —— 从 PerceptEntry.body_state 派生 ——
    estimated_hp: HpEstimate,            // 从可见伤势+姿态异常+运动异常 派生
    estimated_stamina: StaminaEstimate,  // 从呼吸+运动+姿态 派生
    
    // —— 从 PerceptEntry.held_object / body_covering 派生 ——
    visible_weapon: Option<WeaponType>,   // 从 HeldObjectFeature 辨识
    visible_armor: ArmorEstimate,         // 从 BodyCoveringFeature 辨识
    
    // —— 从连续观察派生 ——
    observed_style: Option<FightStyle>,   // 需多次观察攻击模式后涌现
    
    // —— 从 PerceptEntry + magic_signatures 派生 ——
    casting_detected: Option<SpellType>,
    
    // —— 战斗情报计算 ——
    threat_level: f32,                    // 从武器+距离+角度+已知战斗力综合
    positional_advantage: PositionalAdvantage,
    last_known_position: Vec3,            // 来自 PerceptualCache
    
    // —— 信息质量 ——
    perceptual_confidence: f32,           // 来自 PerceptEntry.confidence
    last_updated: GameInstant,
}

/// HP 估计——从 PerceivedBodyState 通用特征派生（★ 保留原有枚举不变）
/// 感官系统输出的是"左臂流血+跛行+运动减慢"——战斗情报层解释为"受重伤"
/// 有经验的老兵从同样特征中提取更精确的判断
enum HpEstimate {
    Uninjured, MinorWounds, ModeratelyWounded, SeverelyWounded, NearDeath, Unknown,
}
```

信息池条目的生命周期：生成→更新→过期→丢弃（或被记忆保留）。

## 五、感知的不准确性

| 条件 | 影响 |
|------|------|
| **黑暗/雾/烟幕** | 视觉距离大幅缩短→只能听到声音→威胁可能被高估或低估 |
| **距离远** | 细节丢失——"那边有个人——但看不清是谁或有没武器" |
| **对方刻意伪装** | 假装受伤(诱敌)→假HP估计。假装不累→假体力估计 |
| **自身疲劳/恐惧** | 疲劳→视觉/听觉灵敏↓。恐惧→威胁放大(对方看起来更强) |
| **高经验(老兵)** | 更难被伪装欺骗——"我见过假装受伤——他不会那样装" |

## 六、信息战——主动塑造对方的感知

战斗不只是物理交锋——还是信息交锋。可以主动影响对方对你的感知：

| 战术 | 机制 | 效果 |
|------|------|------|
| **示弱** | 故意露出破绽/假装受伤/缩小实力表现 | 引诱对方冒进——落入陷阱 |
| **威慑** | 展示强大武器/魔法/夸张的战斗姿态/威吓喊叫 | 让对方高估你→可能畏缩/撤退 |
| **隐蔽实力** | 不使用最强技能——保留到关键时刻 | 对方以为你很弱→突然爆发→摧毁士气 |
| **误导** | 伪装施法——让元素泄漏暗示你在准备火球但其实在准备冰墙 | 对方准备错防御 |
| **信息封锁** | 雾/烟幕/沙尘——双向剥夺信息——但如果你有魔法感知=单向优势 |
| **侧击与偷袭** | 对方的信息池里根本没有你→攻击命中后才更新→伤害加成+无法格挡 |
| **烟幕的信息效果** | 不只是挡住视线——在敌人信息池里制造大量模糊条目→迫使其谨慎 |
| **沉默法术的信息效果** | 剥夺敌人通过声音获取信息的能力——"他喊不来帮手了" |

NPC的信息战能力由智力+魅力+战斗经验决定。高智力NPC会主动使用信息战——不只是"站桩输出伤害"。

---

## 七、信息不足放大人格×知识

- **信息充足** → 不同人格的战术差异较小（事实清楚，大家差不多）
- **信息不足** → 人格差异急剧放大（没有清楚的正确选择→每个人按天性填补空白）

但知识可以覆盖：一个急躁的老兵知道"雾里不能乱冲"——人格仍然急躁，但经验压住了它。
