## 一、数据结构总览

`PowerTopology` 是一个**有向多重图**：
- **节点** = `EntityId`（NPC / Player / Group）
- **边** = `PowerEdge`——从 holder 到 subject 的带类型、带参数、带属性的有向边
- **多重** ——同一 (holder, subject) 对可以有多个不同原子类型（或不同 domain）的边

### 设计约束

1. **SoA 存储**：边以列式（Structure of Arrays）存储——利于缓存、利于 rayon 并行
2. **分表参数**：每种原子类型的专属参数独立列——避免 tagged union 的内存浪费
3. **4 重索引**：出边/入边/空间（四叉树）/原子类型——覆盖所有查询模式
4. **软删除**：边不物理移除——`active = false`。历史系统可通过死边追溯"曾经的权力关系"

## 二、PowerEdge —— 单条权力边

```rust
/// 单条权力边的逻辑视图——从 SoA 各列拼装
/// 实际内存中每条边是各列中同一索引位置的值
#[derive(Debug, Clone)]
pub struct PowerEdge {
    pub edge_id: EdgeId,
    pub holder: EntityId,
    pub subject: EntityId,
    pub atom_type: AtomTypeTag,
    pub domain: PowerDomain,
    
    // === 合法性 & 执行力 ===
    pub legitimacy: f32,            // 0-1，subject 对 holder 此权力的认可
    pub enforcement: f32,           // 0-1，实际执行能力
    
    // === 来源 & 时间 ===
    pub source: PowerSource,
    pub established_at: GameTimestamp,
    pub valid_until: Option<GameTimestamp>,
    pub last_exercised_at: Option<GameTimestamp>,
    
    // === 继承 ===
    pub succession_rule: SuccessionRule,
    
    // === 状态 ===
    pub active: bool,
    
    // === 原子专属参数（按 atom_type 取值） ===
    pub params: AtomParams,  // 从分表列组装
}

/// EdgeId = u32，支持 ~40 亿条边
pub type EdgeId = u32;
```

## 四、边的生命周期

### 4.1 状态机

```
                create_edge()
                     │
                     ▼
              ┌──────────┐
              │  ACTIVE  │ ← active=true
              └────┬─────┘
                   │
        ┌──────────┼──────────┐
        │          │          │
   expire()  relinquish()  remove_by_superior()
        │          │          │
        ▼          ▼          ▼
  ┌────────────────────────────────┐
  │         INACTIVE                │
  │  active=false, 保留全部数据     │
  └────────────────────────────────┘
        │
        │  decay_cleanup()  ← 每年一次
        ▼
  [物理删除]（可选——默认保留为历史记录）
```

### 4.2 创建边

```rust
fn create_edge(&mut self, data: PowerEdgeData) -> Result<EdgeId, PowerError> {
    // 1. 验证
    self.validate_edge(&data)?;
    
    // 2. 分配到下一个 edge_id
    let edge_id = self.next_edge_id();
    
    // 3. 写入各列（SoA——all columns grow by one row）
    self.holder_ids.push(data.holder);
    self.subject_ids.push(data.subject);
    self.atom_tags.push(data.atom_type);
    // ... 写入共有列
    self.write_atom_params(edge_id, &data.params);  // 仅在对应原子列写入
    
    // 4. 更新索引
    self.outgoing.entry(data.holder).or_default().push(edge_id);
    self.incoming.entry(data.subject).or_default().push(edge_id);
    if data.domain.is_territorial() {
        self.spatial_index.insert(data.domain.territory_bounds(), edge_id);
    }
    self.atom_type_index[data.atom_type as usize].push(edge_id);
    
    // 5. 如果是 PrescribeRule → 为 scope 内的 subject 批量创建 Duty
    if data.atom_type == AtomTypeTag::PrescribeRule {
        self.create_duties_for_rule(edge_id, &data)?;
    }
    
    // 6. 如果是 Contract → 创建 ContractRecord
    if data.atom_type == AtomTypeTag::Contract {
        self.create_contract_record(data)?;
    }
    
    self.total_edges_created += 1;
    Ok(edge_id)
}
```

### 4.3 软删除边

```rust
fn deactivate_edge(&mut self, edge_id: EdgeId) -> Result<(), PowerError> {
    let idx = self.edge_index(edge_id)?;
    self.active[idx] = false;
    
    // 级联：Deactivate 所有从这条边 Delegate 出去的边
    let delegated = self.find_delegated_edges(edge_id);
    for d_edge in delegated {
        self.deactivate_edge(d_edge)?;
    }
    
    // 如果是 Contract 的边 → 联动另一条边
    if let Some(contract) = self.find_contract_by_edge(edge_id) {
        if contract.interdependent {
            self.deactivate_edge(contract.other_edge(edge_id))?;
        }
    }
    
    // 释放关联的 Duty
    self.release_duties_for_edge(edge_id);
    
    Ok(())
}
```

### 4.4 过期检查

```rust
/// 每游戏日调用一次——检查所有到期边
fn check_expirations(&mut self) -> Vec<PowerEvent> {
    let now = current_time();
    let mut events = Vec::new();
    
    for idx in 0..self.holder_ids.len() {
        if !self.active[idx] { continue; }
        if let Some(expiry) = self.valid_until[idx] {
            if now >= expiry {
                self.active[idx] = false;
                events.push(PowerEvent::EdgeExpired {
                    edge_id: self.edge_id_from_index(idx),
                    holder: self.holder_ids[idx],
                    subject: self.subject_ids[idx],
                    atom_type: self.atom_tags[idx],
                });
            }
        }
    }
    events
}
```

## 五、索引与查询

### 5.1 出边索引 → 某人拥有的所有权力

```rust
/// O(K)，K = holder 的出边数（通常 <50，国王可能 ~200）
fn powers_of(&self, holder: EntityId) -> Vec<&PowerEdge> {
    self.outgoing.get(&holder)
        .map(|ids| ids.iter()
            .filter_map(|&id| self.get_edge(id))
            .filter(|e| e.active)
            .collect())
        .unwrap_or_default()
}
```

### 5.3 空间索引 → 某位置适用的领土权力

```rust
fn territorial_powers_at(&self, pos: WorldPos) -> Vec<&PowerEdge> {
    self.spatial_index.query_point(pos)
        .iter()
        .flat_map(|&edge_id| self.get_edge(edge_id))
        .filter(|e| e.active && matches!(e.domain, PowerDomain::Territory { .. }))
        .collect()
}
```

**NPC 移动优化**：每个 16m×16m Chunk 在首次查询时缓存结果：
```rust
struct ChunkPowerCache {
    cached_territorial_edges: LruCache<ChunkCoord, Vec<EdgeId>>,
}
```
NPC 在 Chunk 内移动 → 零四叉树查询。跨 Chunk 边界 → 查缓存（命中率 ~90%——领土域变化不频繁）→ 未命中才查四叉树。

## 六、委托链查询

```rust
/// 沿委托链向上追溯——"此人权力的最终来源是谁？"
fn ultimate_authority(&self, holder: EntityId, atom_type: AtomTypeTag) -> Option<EntityId> {
    let mut visited = HashSet::new();
    let mut current = holder;
    while let Some(source_holder) = self.delegation_source(current, atom_type) {
        if !visited.insert(source_holder) { break; }  // 循环防护
        current = source_holder;
    }
    Some(current)
}

/// 沿委托链向下追溯——"此人的权力被委托给了谁？"
fn delegated_to(&self, delegator: EntityId, atom_type: AtomTypeTag) -> Vec<EntityId> {
    self.outgoing.get(&delegator)
        .map(|ids| ids.iter()
            .filter_map(|&id| self.get_edge(id))
            .filter(|e| e.atom_type == AtomTypeTag::Delegate 
                && e.delegate_atoms() == Some(&vec![atom_type]))
            .map(|e| e.subject)
            .collect())
        .unwrap_or_default()
}
```

## 七、规范冲突查询

```rust
/// 查询对此 subject 适用的所有 PrescribeRule——按优先级排序
fn applicable_rules(
    &self, 
    subject: EntityId, 
    domain: &PowerDomain,
) -> Vec<&PowerEdge> {
    let mut rules: Vec<_> = self.incoming.get(&subject)
        .map(|ids| ids.iter()
            .filter_map(|&id| self.get_edge(id))
            .filter(|e| e.active && e.atom_type == AtomTypeTag::PrescribeRule)
            .filter(|e| e.domain.overlaps(domain))
            .collect())
        .unwrap_or_default();
    
    // 按规范层级排序：
    // 1. 委托链距离（近 > 远）
    // 2. 领域专属（有 domain 约束 > 通用域）
    // 3. legitimacy（高 > 低）
    rules.sort_by_key(|e| {
        (self.delegation_distance(e.holder, subject),
         if matches!(e.domain, PowerDomain::Universal) { 1 } else { 0 },
         -(e.legitimacy * 100.0) as i32)
    });
    
    rules
}
```

## 十、边创建 / 修改的安全性约束

| 约束 | 检查位置 | 效果 |
|------|---------|------|
| 禁止自我权力（除 Pledge 外） | `create_edge` | `holder == subject && atom != Pledge` → `Err(SelfReferentialEdge)` |
| 循环委托检测 | `create_edge` (Delegate) | DFS 检查 → `Err(CircularDelegation)` |
| Contract 双方必须不同 | `create_edge` (Contract) | `Err(SelfContract)` |
| ConferRank 必须在 group 内 | `create_edge` | subject 必须已是该 group 成员 → `Err(NotGroupMember)` |
| Delegate 必须有可委托的原边 | `create_edge` (Delegate) | delegator 必须持有要委托的原子 → `Err(NothingToDelegate)` |
| PrescribeRule 不能与已有规则完全冲突 | `create_edge` | 警告级（不阻塞——规范层级规则处理冲突） |

## 十一、与 Duty 的双向同步

当边被创建/修改/失效时，关联的 Duty 自动同步：

| 边的变化 | Duty 同步 |
|---------|----------|
| `Extract` 边创建 | 创建持续型 Duty（subject 负缴付义务） |
| `Constrain` 边创建 | 创建持续型 Duty（subject 负不作为义务） |
| `Compel` 边创建 | 创建条件型/持续型 Duty（subject 负作为义务） |
| `PrescribeRule` 边创建 | 为 scope 内所有 subject 批量创建 Duty |
| 边软删除 | 相关 Duty 全部标记为 inactive |
| `Derogate` 边创建 | 为 beneficiary 创建豁免该规则 Duty 的记录 |
