# 信仰模块 — 接口出口

> **最后验证**: 2026-06-18 | **源文档版本**: 信仰系统 v1.0 (CHG-025)

## FaithId
- **概念**: FaithId = u32 扁平全局标识——统一历史系统的 ReligionId。FaithGenealogy 独立存储谱系关系——不编码在 ID 中。
- **Owner**: [[../../信仰系统/001-信仰系统总纲|信仰系统 001]] | **类型**: Rust type alias, woworld_faith
- **消费者**: [[../02-NPC活人感/002-接口入口|NPC]] 信仰归属 · [[../12-历史/002-接口入口|历史]] 宗教事件溯源 · [[../09-文化/002-接口入口|文化]] 文化-信仰关联 · [[../11-权力/002-接口入口|权力]] 神权政体
- **契约**: CHG-025 | **待处理变更**: 无

## FaithTheology — 10 神学参数
- **概念**: `FaithTheology { deity_count, ancestor_importance, nature_sacredness, hierarchy_degree, scripture_centrality, ritual_formality, exclusivity, mysticism, orthodoxy_vs_orthopraxy, faith_as_identity }` — 10 个 f32 连续参数。"信仰类型"标签从此派生，仅用于 UI。
- **Owner**: [[../../信仰系统/002-信仰定义与神学|信仰系统 002]]
- **关键约定**: 无"神学立场"预设——标签从实践派生。废除 5 档 piety 标量。
- **消费者**: [[../09-文化/002-接口入口|文化 005]] 技术派生 · UI 信仰类型展示
- **契约**: CHG-025

## FaithDefinition — 信仰完整定义
- **概念**: 一个具体宗教实例的完整定义——FaithTheology + 教义 + 禁忌 + 万神殿 + HolyDay + FastingRule。~500B/faith，种子驱动生成 (P2.5)。
- **Owner**: [[../../信仰系统/002-信仰定义与神学|信仰系统 002]]
- **消费者**: [[../03-世界生成/002-接口入口|世界生成]] P2.5 信仰生成 · [[../09-文化/002-接口入口|文化]] 文化-信仰配对
- **契约**: CHG-025

## ReligiousPracticeProfile — 宗教实践档案
- **概念**: `ReligiousPracticeProfile { participation: ParticipationLevel, motivation: MotivationType, theological_depth: f32, ... }` — ~40B/NPC。实践优先模型——NPC 无"神学立场"，有参与实践。
- **Owner**: [[../../信仰系统/003-信仰实践|信仰系统 003]]（定义）→ [[../02-NPC活人感/001-接口出口|NPC 模块]]（存储）
- **三维度**: behavioral_intensity / theological_depth / motivational_autonomy——消费者按需取用
- **消费者**: [[../02-NPC活人感/002-接口入口|NPC]] 热路径本地读取 · [[../06-战斗/002-接口入口|战斗]] 士气 · [[../11-权力/002-接口入口|权力]] 神职层级
- **契约**: CHG-025 | **原则**: "有人的地方才有信仰"——信仰数据以 NPC 的 ReligiousPracticeProfile 为存储单元

## FaithQuery trait — 信仰数据统一入口
- **概念**: `pub trait FaithQuery: Send + Sync` — 30 个只读方法。零分配。高频方法 O(1) 缓存。对标 CultureQuery。
- **Owner**: [[../../信仰系统/010-信仰查询与缓存|信仰系统 010]]
- **关键方法**: `faith_def(faith_id)` / `npc_profile(npc_id)` / `settlement_snapshot(settlement_id)` / `faith_morale_bonus(context)` / `active_faiths_in(region)`
- **消费者**: **全部模块** — 任何需要信仰数据的消费方
- **契约**: CHG-025

## FaithMut trait — 信仰修改入口
- **概念**: `pub(crate) trait FaithMut` — 信仰修改的唯一入口。消费模块不可调用。
- **Owner**: [[../../信仰系统/010-信仰查询与缓存|信仰系统 010]]
- **消费者**: [[../03-世界生成/002-接口入口|世界生成管线]]（唯一授权调用方）· [[../12-历史/002-接口入口|历史模拟引擎]]（唯一授权调用方）
- **契约**: CHG-025

## 5 通道接触传染

- **Owner**: [[../../信仰系统/004-信仰传播|信仰系统 004]]
- **消费者**: [[../02-NPC活人感/002-接口入口|NPC]] NPC 间信仰传播 · [[../09-文化/002-接口入口|文化]] 文化-信仰互动
- **契约**: CHG-025

## SettlementFaithSnapshot — 聚落信仰快照
- **概念**: 从 NPC 聚合——不存储独立"区域信仰地图"。每游戏日增量更新 (dirty settlements only)。"有人的地方才有信仰"原则的架构实现。
- **Owner**: [[../../信仰系统/010-信仰查询与缓存|信仰系统 010]]（惰性缓存）
- **消费者**: 全部空间查询
- **契约**: CHG-025
