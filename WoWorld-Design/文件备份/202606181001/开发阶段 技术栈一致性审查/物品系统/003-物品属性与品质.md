# 003 — 物品属性与品质

## 一、ItemProperties 数据合同

所有物品共享的核心属性结构——各模块在此基础上叠加领域特定属性：

```rust
/// 物品模板属性——每 ItemDefId 一份，全局恒定
pub struct ItemProperties {
    // === 标识 ===
    pub def_id: ItemDefId,
    pub category: ItemCategory,
    pub name_id: LocalizationKey,       // 本地化名称 key
    pub description_id: LocalizationKey,

    // === 物理属性 ===
    pub weight_grams: u32,              // 实际质量（克）
    pub bulk_factor: f32,               // 体积折重系数（默认 1.0）
    pub volume_liters: f32,             // 体积（升），用于容器容量计算

    // === 品质与稀有度 ===
    pub quality_range: Range<Quality>,  // 该物品可能的品质范围
    pub base_quality: Quality,          // "出厂默认"品质（世界生成用）
    pub rarity: Rarity,                 // 自然稀缺度（5 档）

    // === 堆叠 ===
    pub stack_size: u32,                // 堆叠上限（1 = 不可堆叠）

    // === 经济 ===
    pub base_value_copper: u32,         // 基础价值（铜币）

    // === 使用限制 ===
    pub min_skill: Option<(SkillId, f32)>,  // 需求最低技能等级
    pub min_strength: Option<f32>,      // 需求最低力量（如重型武器）
    pub required_body_part: Option<AppendageLabel>,  // 需要特定身体部位

    // === 耐久 ===
    pub max_durability: f32,            // 最大耐久度（0 = 不可损坏）
    pub durability_loss_per_use: f32,   // 每次使用损耗

    // === 元素 ===
    pub element_affinity: Option<Element>,  // 元素亲和
    pub magic_capacity_ke: u32,         // 魔力容量（刻）—— 0 = 非魔法物品

    // === 标签 ===
    pub tags: Vec<ItemTag>,             // 功能标签
    pub mod_tags: BTreeMap<String, String>,  // Mod 自定义标签

    // === ★ v1.0 [[../../NPC活人感模块/03-基本需求系统|基本需求系统]] — 消耗品效果 ===
    /// 消耗品效果——None = 不可食用/不可饮用
    /// Life 模块定义 ConsumableEffect, 物品模块只存储不解析
    /// 对齐已有模式: Magic → EnchantmentSchema, Assembly → AssemblySchema
    /// 见: [[../../生命/004-身体状态与生命过程|生命/004 §十三]]
    pub consumable: Option<ConsumableEffect>,

    // === ★ v1.0 [[../音频系统/001-音频系统总纲|音频系统]] — 装备声学材质 ===
    /// 装备的声学材质分类——None = 几乎不发出装备声音（如布衣/裸体）
    /// 音频模块定义 AudioMaterial 枚举, 物品模块只存储不解析
    /// 对标 ConsumableEffect schema 模式——音频定义结构，物品只存不解
    /// 见: [[../音频系统/002-声音足迹与发射模型|音频 002]]
    pub audio_material: Option<AudioMaterial>,
}

/// 有效负担——考虑体积折算后的"感觉重量"
pub fn effective_encumbrance_kg(props: &ItemProperties) -> f32 {
    (props.weight_grams as f32 / 1000.0) * props.bulk_factor
}
```

## 二、Quality × Rarity 双维度模型

### 2.1 Quality（工艺品质）——人造属性

来源：[[../魔法/03-实践层/009-触媒与材料|Magic 009（触媒）]] 和 [[../魔法/04-工程层/014-魔法卷轴|Magic 014（卷轴）]]

```rust
pub enum Quality {
    Rough,      // 粗糙——学徒练手/仓促赶工
    Standard,   // 标准——合格匠人的日常产出
    Refined,    // 精良——熟练大师的满意作品
    Perfect,    // 完美——毕生巅峰，可能全世界仅数件
}
```

**品质由制作技能决定**（见 [[006-制造与配方|006 制造系统]]）：

```rust
fn determine_quality(crafter_skill: f32, luck: f32, material_bonus: f32) -> Quality {
    let roll = crafter_skill * 0.6 + luck * 0.2 + material_bonus * 0.2;
    match roll {
        r if r < 0.25 => Quality::Rough,
        r if r < 0.65 => Quality::Standard,
        r if r < 0.90 => Quality::Refined,
        _             => Quality::Perfect,
    }
}
```

**品质影响属性乘数**（不同类别物品不同）：

| 类别 | Rough | Standard | Refined | Perfect |
|------|-------|----------|---------|---------|
| 武器伤害 | ×0.7 | ×1.0 | ×1.2 | ×1.5 |
| 防具防护 | ×0.7 | ×1.0 | ×1.2 | ×1.5 |
| 触媒效率 | 20-40% | 50-70% | 75-90% | 95-100% |
| 卷轴效果 | 40-60% | 70-85% | 90-95% | 99%+ |
| 药水效力 | 40-60% | 70-85% | 90-95% | 99%+ |
| 耐久度上限 | ×0.6 | ×1.0 | ×1.3 | ×1.8 |

### 2.2 Rarity（自然稀有度）——自然属性

来源：[[../世界生成/003-资源与矿产|World Gen 003（资源分布）]]

```rust
pub enum Rarity {
    Common,      // 常见——地表随处可见（铁矿石、松木、砂岩）
    Uncommon,    // 罕见——特定群系/深度（银矿石、紫杉木）
    Rare,        // 稀有——深度>80m/特殊条件（秘银、龙涎香）
    Epic,        // 史诗——单一存档仅几处矿床（山铜、凤凰羽毛）
    Legendary,   // 传说——世界唯一或接近唯一（龙钢、世界树种子）
}
```

**Rarity 由世界生成决定**——不是制作出来的，是地质分布的天然属性。一块秘银矿石永远是 `Rare`，无论它被谁开采、被谁锻造。

### 2.3 两个维度的交叉

```
Quality × Rarity 交叉表——价值由两者共同决定

            Common    Uncommon    Rare      Epic      Legendary
Rough       贱卖品    勉强可卖    ─         ─         ─
Standard    日用品    可接受     贵重品    稀有品    ─
Refined     好货      珍品       传家宝    国宝级    神话级
Perfect     凡物绝品  奇迹之作   神器胚子  不朽杰作  唯一真品
```

**关键涌现**：常见材料 + 完美品质 = "凡铁名剑"——普通铁打成完美品质，证明了**工匠的价值**。这正是 WoWorld 故事生成器想要的东西。

## 五、属性查询路径

```rust
/// 查询物品的全部属性（模板 + 实例状态合并）
fn resolve_properties(
    def_id: ItemDefId,
    state: &ItemState,
    registry: &ItemRegistry,
) -> ResolvedProperties {
    let template = registry.definitions.get(&def_id).unwrap();
    ResolvedProperties {
        // 模板属性
        weight_grams: template.weight_grams,
        base_value: template.base_value_copper,
        // 实例状态覆盖
        current_durability: state.durability * template.max_durability,
        quality: state.quality,
        // 品质乘数应用到各维度
        damage_multiplier: template.quality_multiplier(state.quality),
        armor_multiplier: template.quality_multiplier(state.quality),
        // 连续参数
        dimensions: state.continuous_params.clone(),
    }
}
```

## 六、耐久与修理

### 6.1 设计哲学

**一切终将朽坏——这是设计，不是缺陷。** 物品的有限寿命驱动锻造经济（持续需求）、赋予传奇物品意义（它们活得更久）、并让每件物品的生命周期成为一个迷你故事——从崭新出厂到满身修补痕迹再到退役传承。

### 6.2 耐久消耗

```rust
/// 耐久消耗速率取决于品质（见 §二 Quality 乘数表）
/// Quality × max_durability 决定初始耐久值:
///   Rough:    max × 0.6
///   Standard: max × 1.0
///   Refined:  max × 1.3
///   Perfect:  max × 1.8

fn consume_durability(item: &mut ItemState, props: &ItemProperties, usage: DurabilityUsage) {
    let loss = props.durability_loss_per_use * usage.multiplier();
    item.durability = (item.durability - loss).max(0.0);
}

enum DurabilityUsage {
    NormalUse,       // ×1.0
    HeavyUse,        // ×2.0（砍重型装甲/挖黑曜石）
    CriticalHit,     // ×1.5（被暴击）
    Environmental,   // ×0.5（雨水/沙尘缓慢侵蚀）
}
```

### 6.3 修理机制

修理**恢复当前耐久**，但**永久降低最大耐久**：

```rust
fn repair(item: &mut ItemState, repairer_skill: f32, station_quality: f32) -> RepairResult {
    // 最大耐久衰减率——取决于修理者技能
    let quality_loss = match repairer_skill {
        s if s < 30.0 => 0.10,   // 业余——最大耐久 -10%
        s if s < 60.0 => 0.07,   // 合格铁匠—— -7%
        s if s < 85.0 => 0.05,   // 大师—— -5%
        _              => 0.03,  // 传奇—— -3%
    };

    // 高级工作站可减缓衰减
    let station_bonus = station_quality * 0.02;  // 顶级铁砧减免 2%

    let effective_loss = (quality_loss - station_bonus).max(0.02);  // 最低 2%

    item.max_durability *= (1.0 - effective_loss);
    item.durability = item.max_durability;  // 恢复到新的最大耐久

    // 标记：修理次数影响物品评价
    item.repair_count += 1;

    if item.max_durability < 0.20 {  // 原始最大耐久跌至 20% 以下
        item.degraded = true;  // "这把剑已经满是修补痕迹，不太可靠了"
    }

    RepairResult { new_max: item.max_durability, quality_loss: effective_loss }
}
```

### 6.4 物品生命周期层级

```
普通物品（Common/Iron Sword, max=100）:
  新→修理→max=90→修理→81→修理→73→修理→66→修理→59...
  → max<20 → degraded → 退役（卖掉/熔掉/传给后人当纪念品）
  → 生命周期: 约 10-15 次修理，取决于修理者技能

精良物品（Refined/Mithril Sword, max=130）:
  新→大师修理→max=123→大师修理→117→大师修理→111...
  → 生命周期: 约 30-50 次修理（大师修理衰减 5%）
  → 件传家宝——可以完整陪伴一代冒险者

传奇物品（Perfect/Dragonsteel Sword, max=180）:
  新→传奇修理→max=175→传奇修理→170→传奇修理→165...
  → 生命周期: 约 100+ 次修理（传奇修理衰减 3%）
  → 传世名器——几代人共用一把剑。但它终有一天会老去。
  → 退役时——不是卖掉——是放进家族祠堂/博物馆/传给继承人
```

### 6.5 不修理的后果

耐久 ≤ 20% → 效能 ×0.7（来自 [[../战斗/005-招式积木与流派生成|Combat 005]]）。
耐久 = 0 → 武器不可用——必须切换备用武器。防具=0→防护失效。工具=0→无法采集。

耐久为 0 的物品**不会直接消失**——它们仍然存在于库存中，作为"破损的 XX"存在（ItemDefId 不变，但 ItemState 标记为 broken）。可以被修理（代价更高——从 0 修理的双倍衰减惩罚），也可以被熔掉回收部分材料。

### 6.6 符文自我修复

生命符文（见 [[008-魔法物品与附魔|008]]）提供缓慢的自主耐久修复——不需要找铁匠。但速度极慢（约 1% 每游戏日），不适用于战斗中消耗。

## 七、性能说明

- `ItemProperties` 每 DefId 约 200 bytes。预计 ~5,000 个 DefId（含 Mod）→ 1MB——启动时一次性加载
- `ItemState` 每实例约 128 bytes（不含 provenance 链）。8M 实例 → 1GB 存档
- 品质/稀有度查询 O(1)——枚举映射
- 审美属性在 NPC 感知缓存中预计算——不每次查询
