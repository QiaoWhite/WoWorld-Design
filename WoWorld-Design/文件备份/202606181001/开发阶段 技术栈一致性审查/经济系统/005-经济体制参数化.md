# 005 — 经济体制参数化

## 一、设计哲学：同一套引擎，不同参数

### 1.1 核心原则

经济体制不是"选择哪个代码分支"的问题——是"参数滑块在什么位置"的问题。

```
不是: if mode == "自由市场" { run_free_market() }
                 else if mode == "计划经济" { run_planned_economy() }

而是: 同一套订单簿撮合引擎
      + MarketRegulations 参数集
      → 参数值决定了经济行为
      → 参数是连续的——一个文化可以处于自由市场与计划经济的中间态
```

这意味着：
- 不需要为每种经济体制写独立的代码。`match_orders()` 只有一个版本
- 文化的经济体制可以平滑演变——不是"A/D切换"而是"参数逐渐偏移"
- 极端参数组合会产生体制级涌现效果（黑市、物资短缺、军工景气）
- 同一个世界内，不同城市/区域可以有完全不同的 MarketRegulations

### 1.2 参数的所有权

`MarketRegulations` 挂载在 `Market` 上（见 [[002-市场与订单簿]] 的 `Market` 结构体）。每个 `Market` 有自己的管制参数——这意味着同一世界的不同聚落可以使用完全不同的经济体制。

**参数由谁设定**：权力持有者通过 `PowerAtom` 集合行使管制权（详见 [[006-权力与市场]]）。不是"国王改全局变量"——每个掌权者在自己的管辖范围内设定各自 Market 的 regulations。

## 二、参数框架——MarketRegulations

### 2.1 完整数据结构

```rust
/// 市场管制参数——全部是连续值或可单独配置的集合
/// 同一套订单簿引擎通过这些参数实现从自由市场到军国经济的所有体制
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct MarketRegulations {
    // ═══════════════════════════════════════════════════════════
    // 价格管制
    // ═══════════════════════════════════════════════════════════

    /// 价格上限——挂单价格不得高于此值（铜币/单位）
    /// 空 = 无上限。典型用途：必需品限价（面包、盐、基础药材）
    /// 被触达 → 挂单被拒 → 公开市场有价无市 → 黑市涌现
    pub price_ceiling: BTreeMap<ItemDefId, u32>,

    /// 价格下限——挂单价格不得低于此值（铜币/单位）
    /// 空 = 无下限。典型用途：最低工资的实物等价（保障生产者）
    /// 被触达 → 供给过剩 → 政府托底收购消化过剩
    pub price_floor: BTreeMap<ItemDefId, u32>,

    // ═══════════════════════════════════════════════════════════
    // 数量管制
    // ═══════════════════════════════════════════════════════════

    /// 人均配给限额——每个NPC每月最多购买数量
    /// 空 = 无限制。典型用途：战时限粮、灾年限盐
    /// 消费侧配额——不影响卖方挂单，但限制买方成交
    pub per_capita_quota: BTreeMap<ItemDefId, u32>,

    /// 强制出售配额——生产者每月必须向市场供应的最低数量
    /// 空 = 无强制。典型用途：军国经济——铁匠每月必须交N把剑
    /// 未达标 → 惩罚（罚款/吊销执照/征用店铺）——惩罚机制见 [[006-权力与市场]]
    pub mandatory_sale_quotas: BTreeMap<ItemDefId, u32>,

    // ═══════════════════════════════════════════════════════════
    // 品类管制
    // ═══════════════════════════════════════════════════════════

    /// 禁运物品——完全禁止在本市场交易的物品
    /// 挂单时检查——挂单被拒。但在禁止中创造套利空间 → 地下交易
    /// 典型：军国禁民间买卖铁/钢/硝石；神权禁买卖圣物
    pub trade_ban_items: HashSet<ItemDefId>,

    /// 奢侈品额外税率——在这些物品上叠加额外税
    /// 键: ItemDefId, 值: 额外税率（如0.15 = 在基础sales_tax_rate上加15%）
    /// 典型：珠宝、香料、丝绸——"奢税"
    pub luxury_tax_items: BTreeMap<ItemDefId, f32>,

    // ═══════════════════════════════════════════════════════════
    // 准入管制
    // ═══════════════════════════════════════════════════════════

    /// 谁可以在此市场挂卖单
    pub who_can_sell: MarketAccessRule,

    /// 谁可以在此市场挂买单
    pub who_can_buy: MarketAccessRule,

    // ═══════════════════════════════════════════════════════════
    // 政府市场操作
    // ═══════════════════════════════════════════════════════════

    /// 政府托底收购挂单——价格下限的配套
    /// 当市价跌破price_floor时，政府以floor价无限量收购
    /// 这些挂单以 ListingVisibility::Hidden 存在——不公开显示
    pub government_bid_orders: BTreeMap<ItemDefId, GovernmentStandingOrder>,

    /// 政府抛售挂单——价格上限的配套
    /// 当市价冲破price_ceiling时，政府以ceiling价抛售储备
    /// 同样 Hidden 存在
    pub government_ask_orders: BTreeMap<ItemDefId, GovernmentStandingOrder>,

    /// 政府补贴比例——对特定物品的消费进行补贴
    /// 0.0 = 无补贴，0.5 = 政府承担50%，1.0 = 免费
    /// 补贴在交易结算时自动从买方钱包少扣、政府金库补足差额
    pub government_subsidy_pct: BTreeMap<ItemDefId, f32>,

    // ═══════════════════════════════════════════════════════════
    // 税收
    // ═══════════════════════════════════════════════════════════

    /// 交易税——每笔交易金额中扣除的比例
    /// 卖方承担（从卖方的收入中扣除）
    /// 0.0 = 无税，0.10 = 10%交易税
    pub sales_tax_rate: f32,

    /// 进口关税——从其他Market流入的物品额外征税
    /// 0.0 = 无关税，2.0 = 200%关税（价格×3）
    pub import_tariff_rate: f32,

    // ═══════════════════════════════════════════════════════════
    // 征用权
    // ═══════════════════════════════════════════════════════════

    /// 政府征用能力——0.0到1.0的连续值
    /// 0.0 = 政府不能强制收购任何物品
    /// 0.5 = 政府可以征用50%市价以下的挂单（以强制价收购）
    /// 1.0 = 政府可以在任何价格强制收购任何物品
    /// 征用触发条件：政府军需/基建/救灾——具体由政治系统与NPC心智决定
    pub requisition_power: f32,

    // ═══════════════════════════════════════════════════════════
    // 管制强度元参数
    // ═══════════════════════════════════════════════════════════

    /// 管制执行力度——0.0到1.0
    /// 1.0 = 完美执行——所有违规都被发现和惩罚
    /// 0.0 = 纸面法律——管制规则存在但无人执行
    /// 影响：黑市参与者被发现的概率、违规惩罚的实际执行率
    /// 低 enforcement → 管制参数形同虚设 → "腐败"涌现
    pub enforcement_level: f32,
}

/// 政府常备挂单——用于托底收购和抛售
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct GovernmentStandingOrder {
    /// 挂单价格（铜币/单位）
    pub price_copper: u32,
    /// 每次补充的数量
    pub batch_size: u32,
    /// 政府金库中为此挂单预留的资金上限（仅在收购方向有效）
    pub budget_cap_copper: u64,
    /// 是否当前激活
    pub is_active: bool,
    /// 触发条件——"市价低于多少时激活"
    pub trigger_price_copper: u32,
    /// 触发方向
    pub trigger_direction: TriggerDirection,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
pub enum TriggerDirection {
    /// 市价低于触发价时激活（托底收购用）
    BelowTrigger,
    /// 市价高于触发价时激活（抛售压价用）
    AboveTrigger,
}

/// 市场准入规则
#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum MarketAccessRule {
    /// 任何人（默认自由市场）
    Anyone,

    /// 仅本地居民——非本聚落/区域居民不可交易
    /// 典型：孤立山村、自给自足村落
    OnlyResidents {
        settlement: SettlementId,
    },

    /// 仅行会成员——必须持有特定行会认证
    /// 典型：中世纪行会垄断——不是铁匠行会成员不能在城里卖铁器
    OnlyGuildMembers {
        guild_ids: Vec<GuildId>,
    },

    /// 仅贵族/特权阶层
    /// 典型：奢侈品专卖权——仅有爵位者能买卖特定物品
    OnlyNobles {
        min_noble_rank: NobleRank,
    },

    /// 黑名单——禁止列表中的NPC交易
    /// 典型：驱逐出城的商人、被剥夺行会资格的匠人
    Blacklist {
        banned_npcs: Vec<NpcId>,
    },

    /// 白名单——仅列表中的NPC能交易
    /// 典型：战时特许供应商
    Whitelist {
        allowed_npcs: Vec<NpcId>,
    },
}
```

## 三、管制如何影响同一套订单簿

### 3.1 挂单提交时的管制检查

管制不是"撮合后审查"——是"挂单提交时在入口把关"。这避免了已成交再撤销的复杂性。

```rust
/// 带管制检查的挂单提交——订单簿引擎的唯一入口
/// 所有挂单（无论来自玩家还是NPC）都走此路径
fn submit_listing_with_regulation_check(
    market: &mut Market,
    listing: MarketListing,
    submitter: &NpcEconomicState,
    wallets: &mut dyn WalletAccess,
    inventory: &mut dyn InventoryAccess,
) -> Result<ListingId, RegulationRejection> {
    // ── 第1关：准入检查 ──
    check_access_rule(&market.regulations, &listing, &submitter)?;

    // ── 第2关：禁运检查 ──
    if market.regulations.trade_ban_items.contains(&listing.item_def) {
        return Err(RegulationRejection::ItemBanned {
            item_def: listing.item_def,
            item_name: item_name(listing.item_def),
        });
    }

    // ── 第3关：价格管制检查 ──
    match listing.side {
        OrderSide::Ask => {
            // 价格上限——卖价不能高于上限
            if let Some(&ceiling) = market.regulations.price_ceiling.get(&listing.item_def) {
                if listing.limit_price_copper > ceiling {
                    return Err(RegulationRejection::PriceAboveCeiling {
                        item_def: listing.item_def,
                        asked: listing.limit_price_copper,
                        ceiling,
                    });
                }
            }
            // 价格下限——卖价不能低于下限
            if let Some(&floor) = market.regulations.price_floor.get(&listing.item_def) {
                if listing.limit_price_copper < floor {
                    return Err(RegulationRejection::PriceBelowFloor {
                        item_def: listing.item_def,
                        asked: listing.limit_price_copper,
                        floor,
                    });
                }
            }
        }
        OrderSide::Bid => {
            // 买方也受价格上限约束——不能出价超过上限
            // （防止通过高价买方绕过价格上限）
            if let Some(&ceiling) = market.regulations.price_ceiling.get(&listing.item_def) {
                if listing.limit_price_copper > ceiling {
                    return Err(RegulationRejection::BidAboveCeiling {
                        item_def: listing.item_def,
                        bid: listing.limit_price_copper,
                        ceiling,
                    });
                }
            }
        }
    }

    // ── 第4关：配额检查（消费侧）──
    if listing.side == OrderSide::Bid {
        if let Some(&quota) = market.regulations.per_capita_quota.get(&listing.item_def) {
            let already_bought_this_month =
                monthly_purchase_tracker(submitter.npc_id, listing.item_def);
            let requested = listing.quantity;
            if already_bought_this_month + requested > quota {
                return Err(RegulationRejection::QuotaExceeded {
                    item_def: listing.item_def,
                    requested,
                    quota,
                    already_bought: already_bought_this_month,
                });
            }
        }
    }

    // ── 第5关：征用检查 ──
    if listing.side == OrderSide::Ask
        && market.regulations.requisition_power > 0.0
    {
        if should_requisition(&market.regulations, &listing) {
            // 强制收购——不进入公开订单簿
            execute_requisition(market, listing, wallets, inventory)?;
            // 征用也算成交——返回特殊的 ListingId
            return Ok(ListingId::REQUISITIONED);
        }
    }

    // ── 通过所有检查 → 挂入订单簿 ──
    let listing_id = add_to_order_book(market, listing);

    // ── 第6关：管制后的撮合 ──
    // 管制不禁止撮合——只是在入口把关。通过检查的挂单正常撮合
    // 注意：撮合过程中 government_bid_orders / government_ask_orders 作为
    // 隐式对手方参与（见3.3节）

    Ok(listing_id)
}

/// 管制检查的错误类型
#[derive(Debug, Clone)]
pub enum RegulationRejection {
    /// 准入被拒
    AccessDenied {
        rule: String,
        npc_id: NpcId,
    },
    /// 禁运物品
    ItemBanned {
        item_def: ItemDefId,
        item_name: String,
    },
    /// 卖价超上限
    PriceAboveCeiling {
        item_def: ItemDefId,
        asked: u32,
        ceiling: u32,
    },
    /// 卖价低于限
    PriceBelowFloor {
        item_def: ItemDefId,
        asked: u32,
        floor: u32,
    },
    /// 出价超上限
    BidAboveCeiling {
        item_def: ItemDefId,
        bid: u32,
        ceiling: u32,
    },
    /// 配额超限
    QuotaExceeded {
        item_def: ItemDefId,
        requested: u32,
        quota: u32,
        already_bought: u32,
    },
}
```

### 3.2 准入规则检查

```rust
fn check_access_rule(
    regulations: &MarketRegulations,
    listing: &MarketListing,
    submitter: &NpcEconomicState,
) -> Result<(), RegulationRejection> {
    let rule = match listing.side {
        OrderSide::Ask => &regulations.who_can_sell,
        OrderSide::Bid => &regulations.who_can_buy,
    };

    match rule {
        MarketAccessRule::Anyone => Ok(()),

        MarketAccessRule::OnlyResidents { settlement } => {
            if submitter.residence_settlement != Some(*settlement) {
                return Err(RegulationRejection::AccessDenied {
                    rule: format!("仅{}居民可交易", settlement),
                    npc_id: submitter.npc_id,
                });
            }
            Ok(())
        }

        MarketAccessRule::OnlyGuildMembers { guild_ids } => {
            let npc_guilds = &submitter.guild_memberships; // 需NPC身份系统提供
            if guild_ids.iter().any(|gid| npc_guilds.contains(gid)) {
                Ok(())
            } else {
                Err(RegulationRejection::AccessDenied {
                    rule: "仅行会成员可交易".to_string(),
                    npc_id: submitter.npc_id,
                })
            }
        }

        MarketAccessRule::OnlyNobles { min_noble_rank } => {
            match submitter.noble_rank {
                Some(rank) if rank >= *min_noble_rank => Ok(()),
                _ => Err(RegulationRejection::AccessDenied {
                    rule: format!("仅{:?}及以上爵位可交易", min_noble_rank),
                    npc_id: submitter.npc_id,
                }),
            }
        }

        MarketAccessRule::Blacklist { banned_npcs } => {
            if banned_npcs.contains(&submitter.npc_id) {
                Err(RegulationRejection::AccessDenied {
                    rule: "已被列入黑名单".to_string(),
                    npc_id: submitter.npc_id,
                })
            } else {
                Ok(())
            }
        }

        MarketAccessRule::Whitelist { allowed_npcs } => {
            if allowed_npcs.contains(&submitter.npc_id) {
                Ok(())
            } else {
                Err(RegulationRejection::AccessDenied {
                    rule: "不在白名单中".to_string(),
                    npc_id: submitter.npc_id,
                })
            }
        }
    }
}
```

### 3.3 政府挂单——隐式对手方

政府的托底收购和抛售挂单不在公开订单簿中显示——但撮合引擎在每次撮合时会将它们作为隐式对手方考虑。

```rust
/// 撮合扩展——将政府隐式挂单纳入匹配范围
fn match_orders_with_government(
    market: &mut Market,
    new_listing: &MarketListing,
    inventory: &mut dyn InventoryAccess,
    wallets: &mut dyn WalletAccess,
    government_treasury: &mut dyn GovernmentTreasury,
) -> Vec<TradeRecord> {
    let mut trades = Vec::new();
    let mut remaining_qty = new_listing.quantity;

    // ── 第一优先级：撮合公开订单簿中的对手挂单 ──
    // （标准撮合逻辑，见002-市场与订单簿）
    let public_trades = match_orders(market, new_listing, inventory, wallets);
    for trade in &public_trades {
        remaining_qty -= trade.quantity;
    }
    trades.extend(public_trades);

    // ── 第二优先级：未成交的剩余量 → 检查政府隐式挂单 ──
    if remaining_qty > 0 {
        match new_listing.side {
            OrderSide::Ask => {
                // 新卖单 → 政府是否在托底收购？
                if let Some(gov_order) = market.regulations
                    .government_bid_orders.get(&new_listing.item_def)
                {
                    if gov_order.is_active
                        && should_activate_government_order(gov_order, market, new_listing.item_def)
                    {
                        let qty = remaining_qty.min(gov_order.batch_size);
                        let trade = execute_government_purchase(
                            new_listing, gov_order, qty, government_treasury, wallets, inventory
                        );
                        trades.push(trade);
                    }
                }
            }
            OrderSide::Bid => {
                // 新买单 → 政府是否在抛售压价？
                if let Some(gov_order) = market.regulations
                    .government_ask_orders.get(&new_listing.item_def)
                {
                    if gov_order.is_active
                        && should_activate_government_order(gov_order, market, new_listing.item_def)
                    {
                        let qty = remaining_qty.min(gov_order.batch_size);
                        let trade = execute_government_sale(
                            new_listing, gov_order, qty, government_treasury, wallets, inventory
                        );
                        trades.push(trade);
                    }
                }
            }
        }
    }

    trades
}

/// 判断政府挂单是否应该被激活
fn should_activate_government_order(
    order: &GovernmentStandingOrder,
    market: &Market,
    item_def: ItemDefId,
) -> bool {
    let last_price = market.last_price.get(&item_def).copied().unwrap_or(0);
    match order.trigger_direction {
        TriggerDirection::BelowTrigger => last_price < order.trigger_price_copper,
        TriggerDirection::AboveTrigger => last_price > order.trigger_price_copper,
    }
}
```

### 3.4 税率在交易中的结算

税率不是独立的结算步骤——在交易执行时内嵌计算。

```rust
/// 含税交易结算
fn settle_trade_with_taxes(
    trade: &TradeRecord,
    market: &Market,
    wallets: &mut dyn WalletAccess,
    government_treasury: &mut dyn GovernmentTreasury,
) -> Result<(), TradeError> {
    let regs = &market.regulations;
    let gross_amount = trade.total_copper;

    // ── 计算各项税/费 ──

    // 交易税（卖方承担）
    let sales_tax = (gross_amount as f32 * regs.sales_tax_rate) as u32;

    // 进口关税——仅当买卖双方不在同一Market管辖区
    let import_tariff = if trade.buyer_market != trade.seller_market {
        (gross_amount as f32 * regs.import_tariff_rate) as u32
    } else {
        0
    };

    // 奢侈品税——对luxury_tax_items列表物品叠加
    let luxury_tax = regs.luxury_tax_items.get(&trade.item_def)
        .map(|rate| (gross_amount as f32 * rate) as u32)
        .unwrap_or(0);

    let total_tax = sales_tax + import_tariff + luxury_tax;

    // ── 补贴 ──
    let subsidy = regs.government_subsidy_pct.get(&trade.item_def)
        .map(|pct| (gross_amount as f32 * pct) as u32)
        .unwrap_or(0);

    // ── 结算 ──
    // 买方支付: 物品价格 + 补贴（政府补差价）
    let buyer_pays = gross_amount - subsidy;
    // 卖方实收: 物品价格 - 税
    let seller_receives = gross_amount - total_tax;

    wallets.transfer(trade.buyer, TradeTarget::Escrow, buyer_pays)?;
    wallets.transfer(TradeTarget::Escrow, trade.seller, seller_receives)?;
    // 税金进入政府金库
    government_treasury.deposit(total_tax as u64)?;
    // 补贴从政府金库支出
    government_treasury.withdraw(subsidy as u64)?;

    Ok(())
}
```

**涌现效果：**

| 现象 | 机制 |
|------|------|
| 商人自然涌现 | NPC持股 > 自身需求 → 自动挂卖单。盈余/需求自动产生交易主体 |
| 区域价差自动收敛 | 中间商扫描价差 → 从低价区买入、高价区卖出 → 价差缩小 |

| 信息不对称下套利 | 掌握两地价格信息的商人获利——信息价值的经济变现 |

**适合的文化类型**：商贸共和国、港口自由贸易城、缺乏中央集权的封建领地。

### 4.2 计划经济/管制经济

**参数配置：**

```rust
fn planned_economy_regulations() -> MarketRegulations {
    let mut regs = MarketRegulations::default();

    // —— 必需品严格管制 ——
    // 价格上限——面包、盐、基础药材
    regs.price_ceiling.insert(ITEM_BREAD, 8);      // 面包最高8铜
    regs.price_ceiling.insert(ITEM_SALT, 15);       // 盐最高15铜

    // 价格下限——保护农民
    regs.price_floor.insert(ITEM_WHEAT, 5);         // 小麦至少5铜
    regs.price_floor.insert(ITEM_WOOL, 8);          // 羊毛至少8铜

    // 配给制——每人每月限量
    regs.per_capita_quota.insert(ITEM_BREAD, 60);   // 每月60个面包
    regs.per_capita_quota.insert(ITEM_SALT, 10);    // 每月10单位盐

    // 政府托底收购粮食
    regs.government_bid_orders.insert(ITEM_WHEAT, GovernmentStandingOrder {
        price_copper: 5,
        batch_size: 1000,
        budget_cap_copper: 100_000,
        is_active: true,
        trigger_price_copper: 5,
        trigger_direction: TriggerDirection::BelowTrigger,
    });

    // 政府抛售粮食储备——压价保障民生
    regs.government_ask_orders.insert(ITEM_BREAD, GovernmentStandingOrder {
        price_copper: 8,
        batch_size: 500,
        budget_cap_copper: 0, // 抛售不需要预算（卖东西收钱）
        is_active: true,
        trigger_price_copper: 10,
        trigger_direction: TriggerDirection::AboveTrigger,
    });

    // 战略物资禁民间交易
    regs.trade_ban_items.insert(ITEM_MITHRIL);
    regs.trade_ban_items.insert(ITEM_MAGIC_CRYSTAL);

    // 中等税率
    regs.sales_tax_rate = 0.05;
    regs.import_tariff_rate = 0.10;

    // 执行力度高——管制不是纸面的
    regs.enforcement_level = 0.95;

    regs
}
```

**涌现效果——管制最精彩的部分：**

**公开市场表象**：面包价格稳定在8铜，盐15铜——看起来很健康。

**实际发生的事**：
```
价格上限 → 卖方的真实供给成本 > 8铜 → 没人愿意在公开市场卖
→ 公开市场有价无市——"面包8铜，但货架空着"
→ 供需缺口 = 管制创造的无效率

↓ 黑市自然涌现 ↓
管制参数创造了套利空间：
  公开价8铜 vs 实际需求愿意付15铜 → 价差7铜
  → 有胆量的NPC（低agreeableness + 高risk_tolerance）
    在公开市场以8铜"买"（如果有货）或以管制价收购
    → 拿到黑市卖15铜

黑市的微观机制（全部在订单簿引擎内）：
  - 黑市参与者在公开Market被准入拒绝
  - → 在地下 Market（MarketScope::Underground）创建挂单
  - → 挂单 visibility = Restricted(信得过的人)
  - → 同样的订单簿撮合引擎——但价格是真实的
  - → 黑市价格 = 管制前的真实供需均衡价格

黑市的信号功能：
  - 黑市价格偏离公开价格越远 = 管制越无效率
  - 权力持有者可能选择性执法——收受贿赂、保护黑市
  - 黑市参与者形成关系网——"地下经济阶层"涌现
```

**这就是"同一套引擎"的核心展示**：黑市不是特殊代码路径——是管制参数将被拒挂单"挤"到其他Market的自然结果。

### 4.3 军国经济

**参数配置：**

```rust
fn military_economy_regulations() -> MarketRegulations {

    let strategic_materials = vec![

    regs.mandatory_sale_quotas.insert(ITEM_STEEL_INGOT, 50);  // 每月50钢锭
    regs.mandatory_sale_quotas.insert(ITEM_LEATHER_ARMOR, 10);// 每月10套皮甲

    // 政府大宗军需采购——高价收购以保证供给
    regs.government_bid_orders.insert(ITEM_IRON_SWORD, GovernmentStandingOrder {
        price_copper: 120,   // 政府采购价120铜（高于市价~80）
        batch_size: 100,
        budget_cap_copper: 500_000,
        is_active: true,
        trigger_price_copper: 0, // 总是激活
        trigger_direction: TriggerDirection::BelowTrigger,
    });
    regs.government_bid_orders.insert(ITEM_STEEL_INGOT, GovernmentStandingOrder {
        price_copper: 200,
        batch_size: 200,
        budget_cap_copper: 1_000_000,
        is_active: true,
        trigger_price_copper: 0,
        trigger_direction: TriggerDirection::BelowTrigger,
    });

    // 高征用权——战时可以直接调用民间物资
    regs.requisition_power = 0.6;

    // 高税率——维持军费
    regs.sales_tax_rate = 0.12;
    regs.import_tariff_rate = 0.15;

    // 军需品禁止出口（高进口关税对反向也有威慑）
    // 严格执法
    regs.enforcement_level = 0.98;

    regs
}
```

**涌现效果：**

| 现象 | 机制 |
|------|------|
| 铁/钢价格飙升 | 政府采购抬高出价 + 禁民间交易 → 民间供给缩减 |
| 军工景气 | 铁匠→优先军工订单（价格高 + 强制配额有稳定收入）→ 铁匠阶级富裕 |
| 民间铁器短缺 | 一个农民想买把锄头→铁匠说"这个月要交20把剑的定额，没空打锄头"→锄头价格飙升 |
| 军需品走私 | 禁运品创造套利空间→黑市铁矿石价格3倍→冒险者从邻国走私 |
| 强制配额下的规避 | 铁匠达不到配额→贿赂检查官/虚报产量/转入地下 |
| 征用引发怨恨 | `requisition_power=0.6` → 政府随机征用民间库存 → NPC情绪受损（anger+fear）→ 可能引发反抗 |

**军国经济的周期性特征：**
- 战争期间→军需增大→政府提高配额和征用→经济压力传导至平民
- 战后→军需缩减→军工铁匠收入骤降→转型民用或失业
- 停战时期的"军工转民用"本身就是经济周期的涌现

### 4.4 自给自足

**参数配置：**

```rust
fn self_sufficient_regulations(settlement: SettlementId) -> MarketRegulations {
    let mut regs = MarketRegulations::default();

    // 仅本地居民能买卖
    regs.who_can_sell = MarketAccessRule::OnlyResidents {
        settlement,
    };
    regs.who_can_buy = MarketAccessRule::OnlyResidents {
        settlement,
    };

    // 200%进口关税——外来商品价格×3
    // 使任何进口品在经济上不可行（除非极其稀有且本地产不出）
    regs.import_tariff_rate = 2.00;

    // 内部零交易税——鼓励内部自足循环
    regs.sales_tax_rate = 0.0;

    // 无价格管制——小社区靠口碑和社会规范约束价格
    // 无管制 → 但准入限制已足够隔离外部

    regs
}
```

**涌现效果：**

| 现象 | 机制 |
|------|------|
| 无外部商品 | 200%关税 → 进口铁锭价格 = 本地3倍 → 无人买进口品 |
| 内部循环 | 农夫的粮食卖给本地磨坊主→磨坊主卖给面包师→面包师卖给全村 |
| 货币流速慢 | 每笔交易都在本地人之间→钱在社区内循环→货币总量缓慢增加但不出村 |
| 价格长期稳定 | 无外部供需冲击→粮食丰收/歉收是唯一波动源 |
| 缺乏多样性 | 只能买到本地能产的东西→没有香料、没有丝绸、没有异国武器 |
| 旅行商人的稀缺价值 | 移动商人（Itinerant Storefront）偶尔经过→200%关税但货品唯一→"全村人围着旅行商人" |
| 文化停滞 | 外部思想和商品进不来→文化同质化 |

**自给自足与开放的涌现对比：**
```
自给自足村落 vs 自由港贸易城：
  同一种子铁匠 → 同样的技能、同样的产品
  村落铁匠: 仅卖给同村人 → 月收入50铜 → 勉强维生
  港城铁匠: 出口到远方市场 → 月收入300铜 → 中产富足
  → 但村落铁匠可能更幸福（低物价+无竞争压力+熟悉的生活方式）
  → 经济体制不决定幸福——NPC人格决定主观满意度
```

## 五、混合经济——参数是连续的

### 5.1 连续谱系

经济体制不是离散的枚举类型——是 `MarketRegulations` 中每个字段的连续值在N维空间中的一个点。

```
自由市场 ←──────────────────────────────────────→ 计划经济

价格上限: 0项管制 ← 3项必需品限价 ← 10项限价 → 全部限价
关税:     0%    ← 10%     ← 50%     ← 100%   → 200%
准入:     Anyone ← OnlyNobles ← OnlyGuildMembers ← Whitelist
征用:     0.0   ← 0.2     ← 0.5     ← 0.8    → 1.0
托底:     无    ← 仅粮食   ← 粮+矿+布 → 全覆盖

任何一个具体的 MarketRegulations 实例都是这个谱系上的一个点。
两个相邻聚落可能有略微不同的参数——这正是"文化差异"在经济维度的体现。
```

### 5.2 参数由权力持有者设定

`MarketRegulations` 的字段不是由代码"自动计算"的——由权力持有者主动设定和维护。同一套 PowerAtom 机制：

- **国王**：拥有 `SetPriceControl` PowerAtom → 设定全国 price_ceiling
- **行会会长**：拥有 `SetAccessRule` PowerAtom → 设定 OnlyGuildMembers
- **军事统帅**：拥有 `RequisitionPower` + `MandatoryQuota` → 设定军需配额和征用
- **地方领主**：在自己的封地内复制/修改国王的参数

不同掌权者的人格决定他们如何行使权力：
- 高 conscientiousness + 低 selfishness → 价格管制在合理水平，真正用于保护民生
- 低 conscientiousness + 高 selfishness → 价格管制被用来寻租（黑市保护伞、收取贿赂降低 enforcement_level）

详见 [[006-权力与市场]]。

### 5.3 参数随历史而演变

`MarketRegulations` 不是编译时常量——随游戏内历史事件而动态变化：

```
战争爆发 → military_economy_regulations() 逐步替代原有参数
  → requisition_power 从 0.0 → 0.6
  → trade_ban_items 增加铁/钢/皮革
  → 军需品配额出现

战后恢复 → 参数缓慢回归——但不是回到原点
  → 有些管制被取消（战时的 temporary regulation 到期）
  → 有些管制永久化（"战时临时税"变成"永久国防税"）
  → 权力持有者可能利用战争后经济获取个人利益
  → 新掌权者可能推翻旧管制

饥荒 → 开放进口 + 价格上限 + 配给
  → import_tariff_rate 临时调低甚至归零（紧急粮食进口）
  → price_ceiling 加到粮食上（防止囤积居奇）
  → per_capita_quota 每人口粮

繁荣 → 放松管制
  → 价格上限逐步移除
  → 配额取消
  → 税率温和下降
```

参数的动态演变由历史系统的 `HistoricalEvent` 驱动——经济管制变化本身就是一个值得记入大日志的事件。

### 5.4 参数变化的叙事信号

每一次 `MarketRegulations` 的修改都产生一个 `EconomicEvent` 写入历史系统（见 [[001-经济系统总纲]] 2.1 节），包含：
- 谁修改了什么参数
- 修改前后的值
- 修改原因（战争/灾荒/政治/个人贪婪）
- 受影响的市场和NPC数量

这些事件在几十年后的史书中可能变成"某某王施行暴政，横征暴敛"或"贤相推行新政，平抑物价"——文本由历史系统从事件中生成，而非开发者手写。

### 6.3 黑市与地下经济的模块协同

黑市不是独立模块——是三模块协同的涌现：

```

  → 管制参数创造套利空间（公开价 vs 真实供需价）
  → 挂单被准入/禁运/价格上限拒绝

NPC心智
  → 高 risk_tolerance + 低 agreeableness + 高 greed
  → "我可以从邻村运来禁运的铁矿石，在黑市卖三倍价"
  → 自发创建 Underground Market

语言表达系统
  → 黑市信息通过地下渠道传播（visibility=Restricted）
  → 熟人推荐、暗语接头
  → 信息失真算子在黑市中同样运作——"听说城里铁矿石卖到200铜了"
    （真实黑市价可能150铜——过度放大是黑市的常态）
```

## 七、性能考量

| 维度 | 数据 |
|------|------|
| MarketRegulations 内存 | ~200 bytes/Market（典型配置），最多 ~5KB（极端管制） |
| 挂单提交检查 | ~5次BTreeMap查找 + 1-2次HashSet查找 = <1μs |
| 政府挂单激活检查 | 仅在撮合未完全成交时触发——99%交易直接成交无需检查 |
| 税率结算 | 每笔交易 3-4次浮点乘法 + 1-2次BTreeMap查找 = <0.5μs |
| 配额月度统计 | 每日批量更新——分摊后 <0.01ms/帧 |
| 黑市 Underground Market | 与公开Market相同的数据结构——相同性能 |
| 管制变化广播 | 每个 HistoricalEvent 写入同历史系统的全量写入进度——无额外实时广播 |

---

## 八、明确不做

| 条目 | 理由 |
|------|------|
| 价格管制自动计算（"系统根据供需设定最优限价"） | 参数由权力持有者设定——目的是体现人的意志，不是数学最优。昏君的愚蠢限价也是故事 |
| 黑市的独立代码路径 | 同一套订单簿引擎——Underground Market只是 `MarketScope::Underground` + 准入为 `Restricted` |
| 货币政策的利率工具 | 中世纪奇幻无中央银行——无利率工具。货币调节通过铸币/消费回收/跨区流动 |
| 政府金库的无限预算 | `government_bid_orders` 的 `budget_cap_copper` 是硬上限——政府也可能破产 |
| 税收的自动优化 | 税率由权力持有者设定——不是算法"调优"出来的 |
