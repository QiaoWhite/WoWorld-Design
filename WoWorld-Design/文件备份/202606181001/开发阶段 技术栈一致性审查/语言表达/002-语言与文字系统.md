## 一、LanguageId——语言标识

### 1.1 定义

```rust
/// 语言——锚定到世界种子的全局标识
/// 高 16 位 = 语系 ID（同语系的语言共享）
/// 低 16 位 = 语系内编号
#[derive(Copy, Clone, Hash, Eq, PartialEq, Ord, PartialOrd)]
pub struct LanguageId(u32);

impl LanguageId {
    pub fn family(&self) -> u16 { (self.0 >> 16) as u16 }
    pub fn index(&self) -> u16 { self.0 as u16 }

    /// 两个语言的谱系距离——树上最短路径
    /// 同语系内: |index_a - index_b|
    /// 跨语系: family_distance + depth_a + depth_b
    pub fn distance_to(&self, other: LanguageId, tree: &LanguageFamily) -> u32;
}
```

### 1.2 语言生成

从世界种子确定性生成。每个文化区域获得一种（或多种）语言：

```rust
/// 语言分布——世界生成 P3（文化初始化）阶段确定
struct LanguageDistribution {
    /// 语言 → 使用该语言的区域列表
    regions: HashMap<LanguageId, Vec<RegionId>>,
    /// 区域 → 主要语言
    primary_language: HashMap<RegionId, LanguageId>,
    /// 区域 → 次要语言（贸易语言/殖民语言/学术语言）
    secondary_languages: HashMap<RegionId, Vec<(LanguageId, f32)>>,  // f32 = 普及率
}
```

### 1.3 语言属性

```rust
/// 语言的参数化属性——从世界种子派生，不由设计师手写
struct LanguageAttributes {
    /// 语序类型
    word_order: WordOrder,       // SVO | SOV | VSO | ...
    /// 形态类型
    morphology: MorphologyType,  // Isolating | Agglutinative | Fusional | Polysynthetic
    /// 是否有敬语系统
    has_honorifics: bool,
    /// 是否有性别（语法性别）
    has_grammatical_gender: bool,
    /// 书写方向
    writing_direction: WritingDirection,  // LTR | RTL | TTB
    /// 是否声调语言
    is_tonal: bool,
}

enum WordOrder { SVO, SOV, VSO, OVS, VOS, OSV, Free }
enum MorphologyType { Isolating, Agglutinative, Fusional, Polysynthetic }
enum WritingDirection { LeftToRight, RightToLeft, TopToBottom }
```

这些属性影响：
- 文本渲染方向（Godot UI 的文本组件设置）
- 敬语→对话模板选择（PhaticLayer 中的 formality 层级）
- 语序→翻译时的语法转换复杂度

## 二、ScriptId——文字系统

### 2.1 定义

```rust
/// 文字系统——独立于语言
/// 一种语言可以用多套文字（如日语：汉字 + 假名 + 罗马字）
/// 一种文字可以被多种语言使用（如拉丁字母）
#[derive(Copy, Clone, Hash, Eq, PartialEq)]
pub struct ScriptId(u16);
```

### 2.2 语言↔文字映射

```rust
/// 语言可以写成的文字系统
struct LanguageScriptMapping {
    /// 语言 → 可用文字系统列表（按优先级排序）
    scripts: HashMap<LanguageId, Vec<ScriptId>>,
    /// 语言 → 默认文字（书籍/碑文的主要书写系统）
    default_script: HashMap<LanguageId, ScriptId>,
}
```

### 2.3 文字系统对玩家的影响

- 玩家不认识的文字→文本显示为乱码/符号（除非有翻译魔法/技能）
- 同一种语言+不同文字→玩家需要分别学习（口语法语 vs 书面拉丁字母 vs 书面西里尔字母）
- 文字系统是**视觉层**（Godot 字体选择），语言是**逻辑层**（内容匹配）

## 三、LanguageFamily——谱系系统

### 3.1 有向无环图 (DAG)

```rust
/// 语言谱系——描述语言的演化关系
/// 有向无环图：边 = "派生自"
/// 同一祖先的两种语言在时间上分离后各自演化
struct LanguageFamily {
    /// 根语言——该语系的始祖（可能是已死的古代语）
    root_language: LanguageId,
    /// 谱系图——有向边: 父语言 → 派生语言列表
    descendants: HashMap<LanguageId, Vec<LanguageId>>,
    /// 派生时的分化年代（年）
    divergence_time: HashMap<LanguageId, i32>,
    /// 语言是否已死（无母语者）
    is_extinct: HashMap<LanguageId, bool>,
}
```

### 3.2 谱系距离计算

```rust
impl LanguageFamily {
    /// 两个语言之间的距离——基于最近公共祖先 (LCA)
    pub fn distance(&self, a: LanguageId, b: LanguageId) -> u32 {
        let lca = self.lowest_common_ancestor(a, b);
        // 距离 = (a到LCA的深度) + (b到LCA的深度)
        self.depth_from(a, lca) + self.depth_from(b, lca)
    }

    /// 两种语言的互通度——距离越远互通度越低
    pub fn mutual_intelligibility(&self, a: LanguageId, b: LanguageId) -> f32 {
        let d = self.distance(a, b) as f32;
        // sigmoid 衰减——近距离互通度高，远距离快速下降到接近零
        1.0 / (1.0 + (d / 3.0).powi(2))
        // 距离 0 (同一语言): 1.0
        // 距离 1 (方言): ~0.9
        // 距离 2 (近亲语言): ~0.69
        // 距离 4 (远亲): ~0.2
        // 距离 6+: ~0.02
    }
}
```

## 四、Proficiency——语言能力

### 4.1 定义

```rust
/// 语言熟练度——不再是 `literacy: bool`，而是连续值
#[derive(Copy, Clone, PartialEq, PartialOrd)]
pub struct Proficiency(f32);

impl Proficiency {
    pub const NATIVE: Proficiency = Proficiency(1.0);
    pub const FLUENT: Proficiency = Proficiency(0.7);
    pub const INTERMEDIATE: Proficiency = Proficiency(0.4);
    pub const BASIC: Proficiency = Proficiency(0.15);
    pub const NONE: Proficiency = Proficiency(0.0);

    /// 熟练度对阅读理解的修正
    /// proficiency=1.0 → 完全理解。proficiency=0.3 → 只能理解简单词汇
    pub fn reading_multiplier(&self) -> f32 { self.0 }

    /// 口语和阅读可以分离
    /// 一个 NPC 可能能听懂但看不懂
}

/// NPC 的语言能力集合
struct LanguageProficiencySet {
    proficiencies: HashMap<LanguageId, Proficiency>,
    /// 对文字系统的掌握（可以和口语分离）
    script_proficiencies: HashMap<ScriptId, Proficiency>,
}
```

### 4.2 语言学习

```rust
/// 语言学习——由技能系统的 linguistics 技能驱动
/// 学习速度 = base_rate × aptitude × immersion_bonus × teaching_quality
struct LanguageLearningModel {
    /// 基础学习速率（熟练度增量/游戏日内沉浸学习）
    base_rate: f32,  // 0.05 = 在完全沉浸下每天提升 5%
    /// 天赋修正——NPC 的语言天赋（天生一次 roll）
    aptitude: f32,   // 0.7-1.3
    /// 沉浸加成——在该语言环境中生活 = 大幅加速
    immersion_bonus: f32,  // 1.0(不在该语言区) → 3.0(完全沉浸)
    /// 教学品质——如果有老师在教
    teaching_quality: f32, // 1.0(自学) → 2.5(优秀教师)
}

impl LanguageLearningModel {
    pub fn xp_per_day(&self) -> f32 {
        self.base_rate * self.aptitude * self.immersion_bonus * self.teaching_quality
    }
    // 示例：完全沉浸 + 天赋 1.0 + 无老师 → 每天 +5% proficiency
    //       完全沉浸 + 天赋 1.2 + 优秀老师 → 每天 +15% proficiency
    //       不在语言区 + 自学 → 每天 +0.25% proficiency (非常慢)
}
```

## 五、方言渐变模型

### 5.1 Sigmoid 空间衰减

```rust
/// 方言差异 = f(地理距离)
/// 短距离内几乎无差异，中距离逐渐分化，长距离完全分化
pub fn dialect_divergence(
    distance_km: f32,
    midpoint: f32,    // 分化中点距离（km）——默认 200km
    steepness: f32,   // 分化陡峭度——默认 0.02
) -> f32 {
    // sigmoid: divergence(d) = 1 / (1 + e^(-k * (d - m)))
    1.0 / (1.0 + (-steepness * (distance_km - midpoint)).exp())
}
// 0km:   divergence ≈ 0.02  (几乎相同)
// 100km: divergence ≈ 0.12  (口音差异)
// 200km: divergence ≈ 0.50  (明显方言)
// 400km: divergence ≈ 0.98  (几乎不可互通)
```

### 5.2 文化边界修正

```rust
/// 方言不仅受距离影响——政治/文化边界加速分化
pub fn effective_divergence(
    geographic_distance_km: f32,
    same_polity: bool,
    cultural_similarity: f32,     // 0-1，文化参数余弦相似度
    trade_route_connected: bool,
) -> f32 {
    let base = dialect_divergence(geographic_distance_km, 200.0, 0.02);
    let polity_factor: f32 = if same_polity { 0.5 } else { 1.5 };
    let culture_factor = 2.0 - cultural_similarity;  // 文化越近，分化越慢
    let trade_factor: f32 = if trade_route_connected { 0.7 } else { 1.0 };
    (base * polity_factor * culture_factor * trade_factor).min(1.0)
}
```
