## 一、设计原则

### 1.1 三层回落

```
L1 关键词/实体提取（永远可用，~1µs）
  → 置信度 ≥ 0.6 → 直接使用，不调 L2/L3
  ↓
L2 嵌入向量匹配（离线可用，~1-5ms）
  → 置信度 ≥ 0.3 → 合并结果
  ↓
L3 LLM（在线可用，~1-2s）——可插拔增强件
  → 返回结果或 fallback
```

### 1.2 LLM 立场

99% 对话走 L1 关键词 + 模板。LLM 仅在 L1/L2 都失败且 API key 可用时介入（≤1%/NPC/游戏日，与 NPC 模块 LLM stub 一致）。无 LLM 时系统**完全可用**——这是"非 LLM 回落"原则的落实。

### 1.3 语音管道

```
[玩家语音] → Godot 侧 STT (whisper.cpp 本地) → 原始文本 → Rust InputInterpreter → NLU
```
STT 不在语言表达模块范围内——Godot 侧处理。TTS 同样。

## 二、InputInterpreter——NLU 核心

### 2.1 定义

```rust
pub struct InputInterpreter {
    /// L1: 关键词索引——从世界状态自动构建
    keyword_index: KeywordIndex,
    /// L2: 嵌入匹配器——离线可用
    embedding_matcher: Option<EmbeddingMatcher>,
    /// L3: LLM 客户端——在线可用
    llm_client: Option<LlmClient>,
}

impl InputInterpreter {
    /// 主入口——原始文本 → PlayerIntent
    pub fn interpret(
        &self,
        raw_text: &str,
        ctx: &DialogueContext,
    ) -> PlayerIntent {
        // === L1: 关键词/实体提取 ===
        let entities = self.keyword_index.extract_entities(raw_text, ctx);
        let intent_type = self.classify_intent(raw_text);
        let sentiment = self.sentiment(raw_text);

        let mut intent = PlayerIntent {
            target_npc: entities.primary_npc(),
            topic_label: self.generate_topic_label(&entities, intent_type),
            intent_type,
            entities,
            sentiment,
            confidence: self.l1_confidence(raw_text, &entities),
        };

        // === L2: 嵌入匹配 ===
        if intent.confidence < 0.6 {
            if let Some(ref emb) = self.embedding_matcher {
                if let Some(refined) = emb.match_to_topics(raw_text, ctx) {
                    intent = intent.merge(refined);
                }
            }
        }

        // === L3: LLM ===
        if intent.confidence < 0.3 {
            if let Some(ref llm) = self.llm_client {
                if ctx.can_use_llm() {  // 速率限制 + token 预算检查
                    if let Some(llm_intent) = llm.interpret(raw_text, ctx) {
                        intent = llm_intent;
                    }
                }
            }
        }

        intent
    }
}
```

## 三、KeywordIndex——L1 关键词索引

### 3.1 定义

```rust
/// 关键词索引——前缀树 (trie)，支持 O(n) 扫描提取
/// 从世界状态自动重建，不需要开发者手工维护
pub struct KeywordIndex {
    /// 前缀树——词 → (实体类型, 参考ID可选, 意图提示)
    trie: Trie<IndexEntry>,
    /// 意图模式——正则/关键词 → IntentType
    intent_patterns: Vec<IntentPattern>,
}

struct IndexEntry {
    entity_type: EntityType,
    reference_id: Option<u64>,
    /// 额外标签——如"铁匠"同时匹配 NPC 职业和对话主题
    tags: Vec<String>,
}

struct IntentPattern {
    /// 触发关键词列表
    keywords: Vec<String>,
    /// 映射到的意图类型
    intent_type: IntentType,
    /// 是否需要实体配合（如 trading + item entity → IntentType::Trading）
    requires_entity_type: Option<EntityType>,
}
```

### 3.2 自动重建

```rust
impl KeywordIndex {
    /// 从世界状态完全重建索引
    /// 调用时机: 游戏启动时 + 重大世界事件后（新城市建立/大型战争结束/NPC死亡）
    pub fn rebuild(&mut self, world: &WorldSnapshot) {
        self.trie.clear();

        // ① 所有 NPC 名字 + 绰号 + 职业
        for npc in world.npcs() {
            self.trie.insert(&npc.name, IndexEntry {
                entity_type: EntityType::Npc,
                reference_id: Some(npc.id.as_u64()),
                tags: vec![npc.profession.clone()],
            });
            for nickname in &npc.nicknames {
                self.trie.insert(nickname, IndexEntry {
                    entity_type: EntityType::Npc,
                    reference_id: Some(npc.id.as_u64()),
                    tags: vec![],
                });
            }
        }

        // ② 所有地名 + 地标名
        for location in world.locations() {
            self.trie.insert(&location.name, IndexEntry {
                entity_type: EntityType::Location,
                reference_id: Some(location.id.as_u64()),
                tags: location.tags.clone(),
            });
        }

        // ③ 所有事件标签
        for event in world.major_events() {
            for tag in &event.tags {
                self.trie.insert(tag, IndexEntry {
                    entity_type: EntityType::Event,
                    reference_id: Some(event.id.as_u64()),
                    tags: vec![],
                });
            }
        }

        // ④ 玩家背包物品名
        for item in world.player_inventory() {
            self.trie.insert(&item.display_name, IndexEntry {
                entity_type: EntityType::Item,
                reference_id: Some(item.id.as_u64()),
                tags: item.tags.clone(),
            });
        }

        // ⑤ 常见概念词（手动维护 + 自动派生）
        for concept in COMMON_CONCEPTS {
            self.trie.insert(concept.word, IndexEntry {
                entity_type: EntityType::Concept,
                reference_id: None,
                tags: vec![concept.category.clone()],
            });
        }
    }
}
```

## 四、L2 嵌入向量匹配（v1.2+）

```rust
/// 嵌入匹配器——离线可用，不依赖 LLM API
/// 将玩家输入和所有预设话题的嵌入向量比较
pub struct EmbeddingMatcher {
    /// 预计算的"话题描述 → 嵌入向量"映射
    topic_embeddings: HashMap<TopicId, Vec<f32>>,
    /// 嵌入模型——轻量级本地模型（如 all-MiniLM-L6-v2 的 ONNX 版本）
    model: OnnxModel,
}

impl EmbeddingMatcher {
    pub fn match_to_topics(
        &self,
        text: &str,
        ctx: &DialogueContext,
    ) -> Option<PlayerIntent> {
        // 1. 编码输入文本 → 嵌入向量
        let input_embedding = self.model.encode(text);

        // 2. 计算与所有候选话题的余弦相似度
        let mut best_match: Option<(TopicId, f32)> = None;
        for (topic_id, topic_embedding) in &self.topic_embeddings {
            let similarity = cosine_similarity(&input_embedding, topic_embedding);
            if similarity > best_match.map(|(_, s)| s).unwrap_or(0.5) {
                best_match = Some((*topic_id, similarity));
            }
        }

        // 3. 相似度 > 0.7 → 高置信匹配
        best_match.map(|(topic_id, confidence)| {
            PlayerIntent {
                intent_type: IntentType::Sharing { topic_tags: vec![] },
                confidence,
                // ... 从 topic_id 恢复完整 intent
            }
        })
    }
}
```

## 五、L3 LLM 接口（v2.0+，可插拔）

```rust
/// LLM 客户端——与 NPC 模块的 LLM stub 一致的设计
pub struct LlmClient {
    /// API 端点配置
    endpoint: Option<String>,
    api_key: Option<String>,
    /// 速率限制
    rate_limiter: LlmRateLimiter,
}

struct LlmRateLimiter {
    max_per_npc_per_day: u32,  // 默认 1
    max_per_player_per_day: u32, // 默认 20
    usage: HashMap<EntityId, u32>,
}

impl LlmClient {
    pub fn interpret(
        &self,
        text: &str,
        ctx: &DialogueContext,
    ) -> Option<PlayerIntent> {
        // 1. 速率限制检查
        if !self.rate_limiter.can_use(ctx.current_npc_id) { return None; }
        if !self.rate_limiter.can_use(ctx.player_id) { return None; }

        // 2. 构建 prompt——包含对话上下文
        let prompt = self.build_interpret_prompt(text, ctx);

        // 3. 调用 LLM——要求返回结构化 JSON
        let response = self.call_llm(&prompt)?;

        // 4. 安全网关——验证 action_id 在允许集合内
        if !self.validate_response(&response) { return None; }

        // 5. 解析为 PlayerIntent
        let intent: PlayerIntent = self.parse_response(&response)?;

        // 6. 记录使用量
        self.rate_limiter.record(ctx.current_npc_id);
        self.rate_limiter.record(ctx.player_id);

        Some(intent)
    }
}
```

## 六、Godot 侧集成

### 6.1 语音输入流程

```
玩家按下语音键 (V)
  ↓
Godot 录音 (AudioStreamPlayer 录制)
  ↓
玩家松开 V
  ↓
whisper.cpp 本地推理 → 原始文本 (200ms)
  ↓
GDExtension → Rust InputInterpreter::interpret(text, ctx)
  ↓
PlayerIntent → DialogueSystem::process_turn(intent, conversation, ctx)
  ↓
NpcResponse → Godot UI 显示
  ↓ (可选)
TTS 引擎朗读 NPC 回应
```

### 6.2 搜索框流程

```
玩家在对话中点击搜索框
  ↓
Godot 打开搜索 UI: 输入框 + 搜索结果列表
  ↓
玩家输入 "火龙" → 每次按键触发 JournalSearchIndex::search()
  ↓
搜索结果列表实时更新
  ↓
玩家选择一条大日志条目 → PlayerInput::JournalReference { entry_id }
  ↓
Rust DialogueSystem::process_turn() → TopicResolver::resolve_journal_topic()
  ↓
NPC 回应
```

## 七、性能

| 操作 | 单次耗时 | 说明 |
|------|---------|------|
| KeywordIndex 扫描 | ~1-5 µs | Trie O(n) + HashMap 查找 |
| 意图模式匹配 | ~1 µs | 固定数量的字符串比较 |
| L1 总耗时 | ~2-10 µs | 永远可用 |
| EmbeddingMatcher (L2) | ~1-5 ms | ONNX 本地推理 |
| LLM (L3) | ~1-2 s | 网络延迟 + LLM 推理 |
| KeywordIndex 重建 | ~50-200 ms | 启动时一次性——遍历所有 NPC/地名/事件/物品 |
