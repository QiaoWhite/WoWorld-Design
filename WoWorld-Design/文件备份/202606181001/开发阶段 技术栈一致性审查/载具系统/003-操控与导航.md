## 一、核心原则

> **和战斗系统同款哲学——玩家 AI = NPC AI = 同一套代码，日常全自动，灾害半自动。**

| | 战斗（三层模型） | 载具操控（三层模型） |
|---|---|---|
| **L1 本能**（被动） | 自动格挡/闪避/受击反应 | 紧急修理/排水/弃船——自动触发 |
| **L2 战术**（0.5-10s） | 招式循环，距离管理 | 航向保持，速度调节，避障——自动执行 |
| **L3 战略**（5-30s+） | 切换风格/大招/撤退 | 路线选择/灾害响应——NPC GOAP 或玩家指令 |
| **实现** | 同一套战斗 AI 代码 | VehicleController trait（NPC 和玩家同一 trait） |

## 二、L1-L3 三层操控模型

```
┌──────────────────────────────────────────────────────────────┐
│  L3 战略层 (30s-数小时)                                        │
│  "去哪？走哪条路？遇到风暴怎么办？"                                │
│                                                              │
│  决策频率: 航线开始时 / 遇到灾害时 / 到达航点时                    │
│  驱动: NPC GOAP（人格+记忆+知识） 或 玩家指令（菜单选择）           │
│  内容:                                                       │
│    ├─ route = plan_route(目的地, 主观认知, 路网)                │
│    │     路线不是上帝视角最优解——是 NPC 自认为最好的路线            │
│    └─ response = respond_to_hazard(灾害, 载具状态)              │
│          人格驱动: 高神经质→绕行, 低神经质+高技能→继续             │
├──────────────────────────────────────────────────────────────┤
│  L2 战术层 (1-10s 每决策周期)                                   │
│  "保持航向，调节速度，避开眼前障碍"                                │
│                                                              │
│  决策频率: 每 1-10s（每个载具独立周期）                            │
│  驱动: 自动——不需要 NPC 手动每秒钟转舵                            │
│  内容:                                                       │
│    ├─ 沿航线取下一个航点 (current_route.next_waypoint)           │
│    ├─ 简单的 seek+arrive steering behavior                    │
│    ├─ 速度 = base_speed × crew_efficiency × damage × weather │
│    └─ 技能影响: 舵手 Navigation 技能 → 航线偏差度                 │
│                                                              │
│  玩家例外: 小载具可选手动操控 (Manual mode——WASD 直接输入)        │
├──────────────────────────────────────────────────────────────┤
│  L1 本能层 (被动——每决策周期检查)                                │
│  "船在漏水！火光！有礁石！"                                     │
│                                                              │
│  驱动: 自动触发——不需要任何人的指令                               │
│  内容:                                                       │
│    ├─ hull < 0.3 → 船员自动执行 CrewDuty::Maintain (排水/堵漏)  │
│    ├─ hull < 0.1 → 船长 GOAP 触发 AbandonShip 评估              │
│    ├─ fire on board → 所有船员自动转向灭火                       │
│    └─ 礁石在碰撞路径上 → L2 自动避障（瞭望技能影响检测距离）        │
└──────────────────────────────────────────────────────────────┘
```

## 三、L2 导航更新——核心算法

每载具每决策周期（1-10s）调用一次，每帧成本约 50 次浮点运算。

```rust
/// 载具导航更新——轻量 steering behavior
/// 每个载具每 1-10s 调用一次（不是每帧）
fn update_vehicle_navigation(
    vehicle: &mut VehicleState,
    weather: &dyn WeatherQuery,
    def: &VehicleDef,
) {
    let now = current_game_time();
    let dt = now - vehicle.navigation.last_nav_update;
    vehicle.navigation.last_nav_update = now;
    let dt_s = dt.as_secs_f32();

    // ── 1. 取下一个航点 ──
    let target = match &vehicle.navigation.current_route {
        None => return,  // 无航线——可能正在停泊
        Some(route) => {
            let wp = &route.waypoints[vehicle.navigation.next_waypoint_index];
            // 检查是否已到达当前航点
            if vehicle.world_pos.distance_2d(wp.pos) < wp.arrival_radius_m {
                vehicle.navigation.next_waypoint_index += 1;
                // 到达终点——触发 L3 事件
                if vehicle.navigation.next_waypoint_index >= route.waypoints.len() {
                    on_route_completed(vehicle);
                    return;
                }
            }
            route.waypoints[vehicle.navigation.next_waypoint_index].pos
        }
    };

    // ── 2. 转向——简单的 seek steering ──
    let desired_heading = (target - vehicle.world_pos.to_vec2()).angle();
    let heading_delta = wrap_angle(desired_heading - vehicle.heading);
    let max_turn = def.geometry.turn_rate_rad_per_sec * dt_s;
    vehicle.heading += heading_delta.clamp(-max_turn, max_turn);

    // ── 3. 舵手技能 → 航线偏差 ──
    let deviation = compute_nav_deviation(vehicle, def);
    vehicle.heading += deviation;

    // ── 4. 实际速度 = 基准 × 船员效率 × 损伤减速 × 灾害减速 ──
    let weather_factor = if weather.is_extreme_weather(vehicle.world_pos) {
        compute_weather_penalty(vehicle, weather)
    } else {
        1.0  // 日常——不受自然力影响
    };

    let effective_speed = def.geometry.base_speed_ms
        * crew_efficiency_factor(vehicle, def)
        * vehicle.damage.speed_penalty()
        * weather_factor;

    vehicle.current_speed_ms = effective_speed;

    // ── 5. 位移 ──
    let displacement = Vec2::from_angle(vehicle.heading) * effective_speed * dt_s;
    vehicle.world_pos += displacement;

    // ── 6. 全局变换矩阵更新（移动参考系——Godot 每帧读取） ──
    vehicle.world_transform = Mat4::from_translation_rotation(
        vehicle.world_pos.to_vec3(vehicle.appearance_cache.altitude),
        Quat::from_rotation_y(vehicle.heading),
    );

    // ── 7. 空间索引更新（仅当跨越网格边界时） ──
    let new_cell = GridCell::from_world_pos(vehicle.world_pos.to_vec2());
    if new_cell != vehicle.appearance_cache.current_grid_cell {
        // 标记需要更新空间索引（批量处理——在帧末尾统一更新）
        vehicle.appearance_cache.current_grid_cell = new_cell;
        vehicle.appearance_cache.needs_spatial_update = true;
    }
}

/// 舵手技能 → 航线偏差角（弧度）
fn compute_nav_deviation(vehicle: &VehicleState, def: &VehicleDef) -> f32 {
    // 找到舵手岗位——通过 CrewSlotDef.duties 匹配
    let helm_slot = vehicle.crew.iter().find(|s| {
        def.crew_slots.iter().any(|d|
            d.slot_id == s.slot_id && d.duties.contains(&CrewDuty::Navigate))
    });

    let skill_level = helm_slot
        .and_then(|s| s.occupant)
        .map(|npc_id| query_npc_skill(npc_id, SkillId::Navigation))
        .unwrap_or(0.0);

    // 技能低 → 航线漂移大；技能高 → 几乎不漂移
    let skill_factor = 1.0 - (skill_level / 100.0).min(0.95);
    // 随机漂移——种子确定性，每决策周期一次
    let rng = seeded_rng_from_time_and_vehicle(vehicle);
    let drift = rng.gen_range(-1.0..1.0) * skill_factor * 0.05; // 最大 ~3° 漂移
    drift
}
```

## 四、船员效率——技能如何影响航行

```rust
/// 船员整体效率因子 0-1+
///
/// 每个船员槽位贡献 = staffing × skill_factor × fatigue_penalty
/// 空岗 = 0 贡献（关键岗位如舵手会有额外惩罚）
/// 全满岗+高技能 = >1.0（效率超过基准）
fn crew_efficiency_factor(vehicle: &VehicleState, def: &VehicleDef) -> f32 {
    let total: f32 = vehicle.crew.iter()
        .map(|slot_state| {
            // 从 VehicleDef 获取此槽位的定义
            let slot_def = def.crew_slots.iter()
                .find(|d| d.slot_id == slot_state.slot_id);

            match slot_state.occupant {
                None => {
                    // 空岗——关键岗位惩罚
                    if slot_def.map_or(false, |d| d.critical) { -0.3 } else { 0.0 }
                }
                Some(npc_id) => {
                    let skills = query_npc_skills(npc_id);
                    let vitals = query_npc_vitals(npc_id);

                    // 技能因子: 0.3(完全不会) ~ 1.0(技能100)
                    let skill = slot_def
                        .and_then(|d| skills.get(d.required_skill))
                        .map(|s| s.level)
                        .unwrap_or(0.0);
                    let skill_factor = 0.3 + 0.7 * (skill / 100.0).min(1.0);

                    // 疲劳惩罚: 0.6(极度疲劳) ~ 1.0(精力充沛)
                    let fatigue_penalty = 1.0 - 0.4 * vitals.fatigue;

                    skill_factor * fatigue_penalty
                }
            }
        })
        .sum();

    let count = vehicle.crew.len().max(1) as f32;
    total / count
}
```

**效果示例**：

| 场景 | crew_efficiency |
|------|----------------|
| 全员满岗，技能 80+，精力充沛 | ~1.05 |
| 全员满岗，技能 30 | ~0.51 |
| 缺员 30%（含舵手） | ~0.55 |
| 全员在岗但极度疲劳 | ~0.62 |
| 风暴中——舵手受伤离岗+船漏水+全员疲劳 | ~0.2 → 几乎失控 |

## 五、VehicleController trait —— NPC 和玩家同一接口

```rust
/// 载具操控者——NPC 和 玩家 都实现此 trait
/// 对标战斗系统中"玩家 AI = NPC AI = 同一套代码"
pub trait VehicleController: Send + Sync {
    /// 身份——用于日志
    fn controller_id(&self) -> EntityId;

    // ── L3: 路线规划 ──

    /// 给定目的地和此操控者的主观认知，返回一条路线
    ///
    /// knowledge 不是世界真实路网——是此操控者已知的路网
    /// NPC: 来自记忆（去过的地方） + 语言表达（听过的航线） + 物品（地图）
    /// 玩家: 来自玩家探索的地图
    fn plan_route(
        &self,
        origin: WorldPos,
        destination: WorldPos,
        knowledge: &RouteKnowledge,
        transport_network: &TransportNetwork,
        constraints: &RouteConstraints,  // 载具吃水深度/轨距等
    ) -> Result<Route, RouteError>;

    // ── L3: 灾害响应 ──

    /// 当 detect_hazards() 发现危险时调用
    fn respond_to_hazard(
        &self,
        hazard: &NavigationHazard,
        vehicle: &VehicleState,
        weather: &dyn WeatherQuery,
    ) -> HazardResponse;

    // ── L1/L2 被动 ──

    /// 是否应该弃船？（hull<0.1 时自动调用）
    fn should_abandon(&self, vehicle: &VehicleState) -> bool;

    /// 是否需要临时改道？（瞭望发现礁石/海盗时调用）
    fn should_divert(&self, obstacle: &NavigationObstacle, vehicle: &VehicleState) -> bool;
}

/// NPC 的路线知识——来自记忆 + 语言传播 + 物品
/// 载具系统不拥有此数据——从 NPC 模块 + 语言表达模块查询
struct RouteKnowledge {
    /// 已知的港口/聚落（去过或听说过的）
    known_settlements: HashSet<SettlementId>,
    /// 已知的航线（走过或听过的 SeaRoute/RoadEdge）
    known_routes: HashSet<RouteId>,
    /// 已知的危险区域（"那片海域有海盗"——来自记忆或传闻）
    known_hazards: Vec<(WorldBounds, HazardType, f32)>,  // +置信度
    /// 持有地图物品（ItemEntId）——提供额外的路网知识
    owned_maps: Vec<ItemEntId>,
}
```

## 六、路线规划——NPC 用主观认知，非上帝视角

```
场景: 船长李四是东海镇的商船主，要把货运到南海城

李四的路线知识:
  ├─ 亲身去过: 东海镇→蓬莱港→南海城（这条线他跑过20次）
  ├─ 听商人说过: 有一条"外洋航线"更快（但李四没走过——只有口头信息）
  ├─ 听说: "外洋那片有海盗"（传闻——confidence=0.4）
  └─ 航海图: 有一张旧的航海图，标注了几个未去过但已知坐标的港口

李四的 plan_route():
  → 评估选项:
    A: 沿海近海航线（已知，安全，慢 —— 7天）
    B: 外洋直航（传闻，可能危险，快 —— 3天）
  → 人格: 低神经质，中等开放性，高尽责性
  → 决策: 选 A——安全可靠 > 快而未知
  → 如果货主是加急订单（高运费 + 违约罚金）→ 经济压力可能推动选 B
```

**关键**：这一切是 NPC 心智的自发决策，不是载具系统的代码。载具系统只提供 `Route` 数据结构和 `find_route()` 的查询方法——路线选择权完全在 NPC。
