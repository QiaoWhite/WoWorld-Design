## 一、核心原则

> **所有权在记忆中成立——契书是可选的辅助证据，不是前提条件。**

买卖双方口头达成交易，记忆更新——所有权即已转移。契书是额外的物理证据，用于防止记忆模糊、防止赖账、提供正式记录。

## 二、五条获取路径

```rust
/// 载具所有权记录——客观历史，不含裁决
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct VehicleOwnershipRecord {
    pub vehicle_id: VehicleId,

    /// 所有权变更的事件链——按时间排序
    pub history: Vec<OwnershipEvent>,

    /// 当前活跃的所有权主张（可能有多个——争议中）
    pub active_claims: Vec<OwnershipClaim>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct OwnershipEvent {
    pub when: GameTimestamp,
    pub kind: OwnershipEventKind,
    /// 关联的证据引用
    pub evidence: Vec<EvidenceRef>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum OwnershipEventKind {
    /// 建造完成——建造者即首任所有者
    Constructed { builder: EntityId },

    /// 口头交易——双方记忆更新
    /// 卖方承认 = 最强证据
    VerbalSale {
        seller: EntityId,
        buyer: EntityId,
        price_copper: u64,
        witnessed_by: Vec<EntityId>,
    },

    /// 契书交易——口头交易 + 签署契书
    DeedTransfer {
        seller: EntityId,
        buyer: EntityId,
        deed: ItemEntId,
        price_copper: u64,
    },

    /// 继承——权力系统 SuccessionRule 自动触发
    Inherited {
        from: EntityId,
        to: EntityId,
        rule: SuccessionRule,
    },

    /// 事实夺取——武力/占领
    Seized {
        from: EntityId,
        by: EntityId,
        method: SeizureMethod,
    },

    /// 遗弃——所有者主动放弃
    Abandoned { previous_owner: EntityId },

    /// 法院/仲裁人的裁决（Adjudicate 结果）
    Adjudicated {
        adjudicator: EntityId,
        decision: AdjudicationDecision,
    },
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum SeizureMethod {
    /// 武力夺取（海盗/战争）
    ByForce,
    /// 事实控制积累（长期占有→时效取得）
    AdversePossession { duration_days: u32 },
    /// 债务抵偿（欠债→船被收走）
    DebtSeizure { debt_copper: u64 },
}

/// 所有权主张——每一方声称自己对载具拥有的权利
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct OwnershipClaim {
    pub claimant: EntityId,
    pub basis: ClaimBasis,
    /// 证据质量 0-1（由主张方提供的证据的综合评估）
    pub evidence_strength: f32,
    pub asserted_at: GameTimestamp,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum ClaimBasis {
    /// 我有契书
    Deed { deed: ItemEntId },
    /// 我与卖方口头约定——卖方可以确认
    VerbalAgreement { seller: EntityId },
    /// 我继承了
    Inheritance { from: EntityId, rule: SuccessionRule },
    /// 我造的
    OriginalConstruction,
    /// 我夺的——事实控制
    ByForce { duration_days: u32 },
}
```

## 三、口头交易——最常见路径

```
场景: 渔夫王五把他的小船卖给邻居老张，价格 200 铜

流程:
  1. 王五和老张对话（语言表达系统 005 Conversation）
     王五: "那条船我不用了，200铜卖你"
     老张: "成"

  2. 交易执行（经济系统 Wallet）:
     老张.钱包 -= 200铜
     王五.钱包 += 200铜

  3. 记忆更新（NPC 记忆系统）:
     王五.记忆 += "把船卖给老张了——卖了200铜"
     老张.记忆 += "从王五那买了条船——花了200铜"
     王五.记忆中原本的 "我的船" → 变更为 "以前有过的船（已卖给老张）"

  4. PowerEdge 更新（权力系统）:
     旧的 PowerEdge(王五, VehicleId, Access) → deactivate
     新的 PowerEdge(老张, VehicleId, Access) → activate
     source = Contractual (口头约定也是契约)

  5. VehicleOwnershipRecord 更新:
     history += VerbalSale(seller=王五, buyer=老张, price=200)
     active_claims = [ OwnershipClaim(claimant=老张, basis=VerbalAgreement(seller=王五)) ]

  6. 契书: 无——口头交易不需要契书

如果半年后老张声称船是他的而王五否认:
  裁决者（Adjudicate）查询:
    - 王五的记忆 → "把船卖给老张了" ← 卖家承认！最强证据
    - 老张的记忆 → "从王五那买的"
    - 两人的记忆一致 → 所有权确认归老张
    - 如果王五老了忘了——老张说的"200铜"和王五记忆中的"卖了船"部分吻合 → 倾向老张
```

### 4.1 契书作为物品

```rust
/// 契书（VehicleDeed）—— ItemCategory::Financial
/// 一张纸——和欠条同款
/// 可以放背包/箱子/保险柜
/// 可以走经济系统订单簿交易
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct VehicleDeedState {
    /// 正文——语言表达系统生成的文本
    pub content: ExpressionRef,

    /// 关联的载具
    pub vehicle_id: VehicleId,

    /// 卖方签名
    pub seller: EntityId,
    pub seller_signature: HandwritingSample,

    /// 买方签名
    pub buyer: EntityId,
    pub buyer_signature: HandwritingSample,

    /// 见证人签名（可选——增加可信度）
    pub witnesses: Vec<(EntityId, HandwritingSample)>,

    /// 官方印章/标记（可选——行会/领主/政府烙印）
    pub official_seals: Vec<SealImpression>,

    /// 契书纸张和墨水的品质——决定伪造难度基线
    pub material_quality: f32,  // 0-1

    /// 篡改痕迹（如果有）
    pub tamper_marks: Vec<TamperMark>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct HandwritingSample {
    /// 笔迹特征向量——用于比对
    pub features: HandwritingFeatures,
    /// 签名时的日期
    pub signed_at: GameTimestamp,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct HandwritingFeatures {
    pub pressure_mean: f32,      // 平均笔压
    pub slant_angle: f32,        // 倾斜角
    pub stroke_order: u8,        // 笔顺特征（简化）
    pub letter_spacing: f32,     // 字间距
    pub baseline_drift: f32,     // 基线漂移
}
```

### 4.2 契书伪造与检测

```rust
/// 篡改标记——每次篡改留下痕迹
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct TamperMark {
    /// 篡改的字符偏移范围
    pub position_range: (u32, u32),
    /// 检测难度——篡改者的技能越高，掩藏越好
    pub detection_dc: f32,
    /// 篡改类型
    pub kind: TamperKind,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
pub enum TamperKind {
    /// 刮除改写
    Erasure,
    /// 覆盖原字迹
    Overwrite,
    /// 化学漂洗（墨水褪色后重新书写）
    ChemicalWash,
    /// 魔法篡改——最高级，最难检测
    MagicAlteration,
}
```

| 场景 | 操作 | 走哪个系统 | 后果 |
|------|------|-----------|------|
| **偷契书** | 扒窃→纸被偷走 | 物品系统（盗窃） | PowerEdge 不变——小偷有纸没船 |
| **篡改契书** | 刮掉 buyer 名字，写自己的 | 技能系统(Linguistics+Deception) → ItemState 被修改 | tamper_marks 新增——任何有 ForgeryDetection 技能的人可以检测。检测者的 Linguistics 技能 vs 篡改者的 Deception 技能对抗 |
| **伪造契书** | 从头创建假契书——模仿卖方的签名 | 同上——LanguageExpression 生成假 HandwritingSample | 一张假纸。PowerEdge 不存在→上法庭时：笔迹比对 vs 已知签名样本→技能对抗→识破 |
| **合法签署** | 双方在契书上签名 + PowerEdge 同步更新 | 权力系统 Contract + 经济系统 Wallet + 语言表达生成文本 | 完整签名链 + 记忆一致 + PowerEdge 一致 |

### 4.3 契书检测——NPC 检查一份契书

```rust
fn examine_deed(
    examiner: NpcId,
    deed: &ItemState,  // 物理契书的 ItemState
    known_handwriting_samples: &HashMap<EntityId, HandwritingSample>,
) -> ExaminationResult {
    let examiner_skills = query_npc_skills(examiner);
    let examiner_int = query_npc_intelligence(examiner);

    // 1. 基础阅读——需要语言能力
    let lang_prof = query_npc_language_proficiency(examiner, deed.language);
    if lang_prof < 0.3 {
        return ExaminationResult::CannotRead;
    }

    // 2. 笔迹比对——对照已知签名样本
    let seller_match = compare_handwriting(
        &deed.seller_signature,
        known_handwriting_samples.get(&deed.seller),
        examiner_skills.get(SkillId::Linguistics).map(|s| s.level).unwrap_or(0.0),
    );

    let buyer_match = compare_handwriting(
        &deed.buyer_signature,
        known_handwriting_samples.get(&deed.buyer),
        examiner_skills.get(SkillId::Linguistics).map(|s| s.level).unwrap_or(0.0),
    );

    // 3. 篡改痕迹检测
    let detection_skill = examiner_skills.get(SkillId::ForgeryDetection)
        .map(|s| s.level).unwrap_or(0.0);
    let detected_marks: Vec<&TamperMark> = deed.tamper_marks.iter()
        .filter(|mark| detection_skill + 0.3 * examiner_int > mark.detection_dc)
        .collect();

    // 4. 交叉验证——权力系统查询
    let power_edge = query_power_topology(deed.vehicle_id);
    let deed_matches_power = power_edge.map_or(false, |e| {
        e.active && e.holder == deed.buyer
    });

    ExaminationResult {
        readable: lang_prof >= 0.3,
        seller_handwriting_match: seller_match,
        buyer_handwriting_match: buyer_match,
        tamper_detected: !detected_marks.is_empty(),
        tamper_details: detected_marks.into_iter().cloned().collect(),
        consistent_with_power_topology: deed_matches_power,
        overall_confidence: compute_confidence(seller_match, buyer_match, detected_marks, deed_matches_power),
    }
}
```

## 五、裁决——权力系统 Adjudicate 原子

载具系统**不裁决**所有权争议。载具系统维护 `VehicleOwnershipRecord`——所有客观事实和所有活跃主张。裁决由权力系统的 `Adjudicate` 原子执行：

```
Adjudicate 裁决输入端（从各模块收集）:
  ├─ VehicleOwnershipRecord (载具系统)
  ├─ 各方 NPC 的记忆 (NPC系统)
  ├─ 物理契书 + 笔迹样本 (物品系统 + 语言表达)
  ├─ 证人证词 (语言表达 Conversation)
  ├─ 市场交易记录 (经济系统)
  ├─ 文化法律规范——来自文化系统 + 权力系统 PrescribeRule
  │   ├─ 书面主义: "契书决定一切"
  │   ├─ 口头主义: "口头约定即契约"
  │   ├─ 时效取得: "事实占有满 X 天视为取得所有权"
  │   ├─ 赃物例外: "被偷的不适用时效取得"
  │   └─ 规范冲突时 → 权力系统 005 规范层级规则
  └─ 裁决者自身的 legitimacy → 影响双方对裁决的接受度

裁决输出:
  → OwnershipEventKind::Adjudicated { adjudicator, decision }
  → 败诉方可能继续争议——引发更高层级的 Adjudicate
    或导致裁决者的 legitimacy 下降（如果裁决被认为不公）
```
