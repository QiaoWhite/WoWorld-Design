# 003 — 传播引擎 v1.0

> **核心职责**：给定一个 RawSoundEvent + 听者位置 + 环境条件 → 回答"传到了多少分贝、有什么频率畸变、过了多久"。传播引擎是音频模块的数学核心——全部浮点计算，零 I/O，零分配。

> **可替换性**：通过 `pub(crate) trait PropagationEngine` 在音频 crate 内部隔离。换传播算法 = 重新 impl，不动其他 10 个子插件。衰减/掩蔽模型通过 TOML 选择。

## §一 整体公式

```
SPL_at_listener = SPL_at_source
                - 20·log₁₀(distance)         ← 几何扩散
                - atmospheric_absorption      ← 频率选择性介质吸收
                - occlusion_loss              ← 障碍遮挡
                - medium_boundary_loss        ← 介质边界反射
                + wind_factor                 ← 风调制（可正可负——顺风=负衰减=传更远）
                + temp_gradient_factor        ← 温度梯度弯曲（可正可负）
                - head_medium_filter_loss     ← 听者头部介质过滤

传播延迟 = distance / speed_of_sound(medium, temperature)
```

## §二 几何衰减

```rust
/// 反平方律：每个倍距 -6dB
/// distance clamp 在 0.1m —— 避免 log(0) NaN
fn geometric_attenuation(distance_m: f32) -> f32 {
    20.0 * distance_m.max(0.1).log10()
}
```

三种可选模型（TOML `[propagation.attenuation_model]`）：
- `inverse_square`：经典 1/r²
- `inverse_square_with_absorption`：1/r² + 频率吸收（默认）
- `custom { a, b, c }`：a/r² + b·r + c

## §三 介质吸收（频率选择性）

### 3.1 空气吸收

```rust
fn air_absorption(freq_hz: f32, distance_m: f32, humidity: f32, _temperature: f32) -> f32 {
    let coeff_per_100m = match freq_hz {
        f if f < 125.0 => 0.03,      // 极低频——几乎不衰减
        f if f < 250.0 => 0.12,
        f if f < 500.0 => 0.28,
        f if f < 1000.0 => 0.5,
        f if f < 2000.0 => 1.0,
        f if f < 4000.0 => 2.5,
        f if f < 8000.0 => 7.5,
        _ => 20.0,                    // 极高频——快速衰减
    };
    let humidity_factor = if humidity < 0.2 { 1.5 }
        else if humidity < 0.4 { 1.2 }
        else { 1.0 };
    coeff_per_100m * (distance_m / 100.0) * humidity_factor
}
```

### 3.2 水吸收

```rust
fn water_absorption(freq_hz: f32, distance_m: f32) -> f32 {
    match freq_hz {
        f if f < 250.0 => 0.001 * distance_m / 1000.0,    // 低频：几乎不衰减
        f if f < 1000.0 => 0.06 * distance_m / 1000.0,
        f if f < 4000.0 => 0.5 * distance_m / 1000.0,
        f if f < 8000.0 => 5.0 * distance_m / 1000.0,
        _ => 30.0 * distance_m / 1000.0,                  // 高频：瞬间消失
    }
}
```

### 3.3 涌现效果

| 介质 | 低频 | 中频 | 高频 |
|------|------|------|------|
| 空气 | 几乎不衰减（千米级） | 中等衰减 | 快速衰减 |
| 水 | 几乎不衰减（百公里级·鲸歌） | 中等 | 瞬间消失（几十米） |
| 固体 | 不衰减 | 轻微 | 轻微 |

## §四 风矢量调制

```rust
/// 风改变所有声音的有效衰减——不只是背景风声
fn wind_modulation_factor(wind: &WindVector, source_pos: Vec3, listener_pos: Vec3) -> f32 {
    let sound_dir = (listener_pos - source_pos).normalize();
    let wind_alignment = wind.direction().dot(sound_dir);  // 1=顺风, -1=逆风

    // 强风(15m/s) 顺风 → -0.3 (衰减少 0.3dB/倍距 → 传更远)
    // 强风(15m/s) 逆风 → +0.3 (衰减多 → 传更近)
    -wind_alignment * wind.speed() * 0.02
}

/// 湍流断续——强风(>8m/s) + 高频 → 声音断断续续
fn turbulence_flicker(wind_speed_ms: f32, freq_band: FrequencyBand) -> f32 {
    if wind_speed_ms > 8.0 && freq_band == FrequencyBand::High {
        1.0 - (wind_speed_ms - 8.0) * 0.05
    } else { 1.0 }
}
```

**消费**：`WeatherQuery::sample()` 已提供 `wind_speed`/`wind_direction`。

## §五 温度梯度

```rust
/// 冷空气密度高 → 声波向冷侧弯曲。
/// 冬夜逆温层（地面冷 + 上方暖）→ 声音困在地表，传 3-4 倍远。
fn temperature_gradient_factor(temp_source: f32, temp_listener: f32, vertical_sep_m: f32) -> f32 {
    if vertical_sep_m.abs() < 0.5 { return 0.0; }
    let gradient = (temp_source - temp_listener) / vertical_sep_m.max(1.0);
    // 负梯度 = 地面冷、上方暖 → 声音向下聚焦 → 衰减更少 → 传更远
    (gradient * 0.1).clamp(-0.5, 0.5)
}
```

**涌现**：冬夜地面 -5°C/上方 +3°C → 地面听者听到比白天远 2-3 倍。夜晚野外特别安静但远方钟声却清晰——独特的氛围。

## §六 障碍遮挡

### 6.1 三种算法（TOML 选 `[propagation.occlusion_method]`）

```rust
enum OcclusionMethod {
    /// 简单球体近似——障碍物 = 听者-声源连线上的实体数量
    SimpleSphere,
    /// 射线采样——4 条射线，确定性 jitter，存档可复现
    RaycastSample { ray_count: u32 },
    /// 体素网格——预计算（内存换精度，大室内/洞穴）
    VoxelGrid { cell_size_m: f32 },
}
```

### 6.2 默认：射线采样（4 条射线）

```rust
fn occlusion_loss(source: Vec3, listener: Vec3, world: &dyn SpatialQuery) -> f32 {
    let ray_count = 4;
    let hits = (0..ray_count)
        .filter(|i| {
            let jitter = deterministic_jitter(listener, source, *i);  // 确定性
            world.raycast_hit(listener, jitter)
        })
        .count();
    (hits as f32 / ray_count as f32) * 30.0  // 每 10% 遮挡 = -3dB
}
```

### 6.3 性能优化

先做 cheap 门控：距离 >100m 或 SPL 已 <0dB → 跳过遮挡检测。

## §八 介质边界

### 8.1 介质间传输损失

```rust
fn medium_boundary_loss(source_medium: Medium, head_medium: Medium) -> f32 {
    match (source_medium, head_medium) {
        (a, b) if a == b => 0.0,           // 同一介质

        // 空气 ↔ 水：~99.9% 反射 = 30dB 损失
        (Medium::Air, Medium::Water) | (Medium::Water, Medium::Air) => 30.0,

        // 固体 ↔ 其他：20dB
        (Medium::Solid, _) | (_, Medium::Solid) => 20.0,

        _ => 10.0,
    }
}
```

**例外**：极其响亮的声音（爆炸 150dB+）穿透介质边界——`intensity_db - boundary_loss > MIN_AUDIBLE_SPL`。

### 8.2 听者头部介质过滤

```rust
fn head_medium_filter_loss(freq_band: FrequencyBand, head_medium: Medium) -> f32 {
    match head_medium {
        Medium::Water => match freq_band {
            FrequencyBand::Low => 0.0,     // 水下低频增强
            FrequencyBand::Mid => 5.0,     // 中频衰减
            FrequencyBand::High => 20.0,   // 高频严重衰减
        },
        _ => 0.0,  // 空气基准
    }
}
```

## §九 传播延迟

```rust
fn speed_of_sound(medium: Medium, temperature: f32) -> f32 {
    match medium {
        Medium::Air => 331.3 + 0.606 * temperature,  // ~340 @ 20°C
        Medium::Water => 1500.0,
        Medium::Solid => 3000.0,  // 粗略估计
    }
}

fn propagation_delay(distance_m: f32, medium: Medium, temperature: f32) -> Duration {
    Duration::from_secs_f32(distance_m / speed_of_sound(medium, temperature))
}
```

### 涌现效果表

| 场景 | 距离 | 延迟 | 玩家体验 |
|------|------|------|---------|
| 近处说话 | 3m | 0.009s | 不可感知 |
| 远处钟声 | 500m | 1.47s | 先见钟动，后闻钟声 |
| 龙吼 | 800m | 2.35s | 吼声比嘴巴动作晚——恐怖 |
| 火山爆发 | 5km | 14.7s | 先见烟柱冲天，后闻轰鸣 |
| 水下爆炸 | 2km | 1.33s | 比空气快得多 |
| 闪电 | 3km | 8.8s | 天气系统已有 |

## §十 可听半径

```rust
/// 二分查找：给定声源分贝，找到衰减到听觉阈值的距离。
/// 用于 L2/L3 声音的强度门控判定——"这声音能传到 L1 吗？"
fn audible_radius(
    intensity_db: f32,
    freq: &FrequencyProfile,
    medium: Medium,
    weather: &WeatherSample,
    min_audible: f32,  // 默认 10dB SPL
) -> f32 {
    let mut lo = 0.1;
    let mut hi = 10000.0;  // 10km

    for _ in 0..24 {  // 24 次二分 → 精度 ~0.6mm @ 10km
        let mid = (lo + hi) / 2.0;
        let attenuation = geometric_attenuation(mid)
            + medium_absorption(freq.fundamental_hz.0, mid, medium, weather)
            + medium_boundary_loss(medium, Medium::Air);
        if intensity_db - attenuation > min_audible { lo = mid; }
        else { hi = mid; }
    }
    lo
}
```
