## 一、教育不是模块——是涌现模式

### 1.1 教育是什么

教育是以下既有系统在时间中展开的**互动模式**：

- **技能系统**提供 4 条教学路径（自学/学徒/学院/奥术）
- **GOAP** 提供"我想学 X"和"我想教 Y"的目标
- **认知系统**提供学习率、认知可塑性窗口、MentalModel 继承
- **关系系统**提供师徒/亲子/同辈关系的情感基底
- **文化模块**提供教育规范、技术档案天花板
- **经济模块**提供"能否负担不劳动的学徒期"

### 1.2 教育不是什么

- 不是"教育系统"——没有专用的"教育模块"
- 不保证每个儿童都受到教育
- 不规定"儿童必须上学到 X 岁"
- 不安排统一的课程或进度

**不教育的世界也是涌现出来的合法世界。** 一个出生在贫困家庭的儿童可能从未接触任何系统性的教学——她的技能只能通过观察和模仿自己学。

## 二、教育三层涌现

### 2.1 第一层：家庭

家长不是"教育者"——家长是 NPC，其 GOAP 中可能有"教孩子"的目标，也可能没有。

```
家长 NPC 在孩子 Weaned 后：
  if conscientiousness 高 && 有闲暇 && 有可教的技能：
    → GOAP 目标 "TeachChild" 出现
    → 优先级与工作/休息/社交平等竞争
  
  if 疲于奔命 || 对养育不上心 || 自己也没有什么技能：
    → "TeachChild" 目标优先级很低或根本不出现
```

**尽责的家长自然教得更多。** 母亲在做饭时让孩子在旁边看→孩子通过观察+模仿学习烹饪的基本动作。铁匠父亲在打铁时让儿子拉风箱→孩子在实践中积累力量+技能 XP。这不是"家庭教育系统"——这是 GOAP 中教与学的目标在共处时间中自然交叠。

**不尽责的家长不教——这是涌现的不平等。** 同一条街上的两个孩子，一个每天跟着父亲学手艺，一个被放任自流。到青春期时，技能差距已经拉大。

### 2.2 第二层：社区

当聚落中存在某个高技能工匠，且其工作空间对儿童可见时：

```
铁匠 NPC 在铁匠铺中工作（感官可感知）
→ 多个儿童的感知流中出现了铁匠工作场景
→ 认知系统从重复感知中尝试归纳 → MentalModel "打铁很有意思"
→ 儿童的 GOAP 中 "观察铁匠" 目标优先级上升
→ 铁匠的 GOAP 中——看到几个孩子在围观——
  if 铁匠需要帮工 || 铁匠享受声望：
    → "收学徒" 目标出现
  else if 铁匠嫌孩子碍事：
    → 赶走他们——不给学习机会
```

**社区的介入不由系统规划。** 有的社区自然形成了围绕铁匠铺的"学徒圈"，有的社区工匠冷漠、大门紧闭——都是涌现。

### 2.3 第三层：专职教师

当聚落发展到一定规模、文化中 `education_investment` 高、且有专门的学术/技术传统时：

```
世界生成 P8 创建了"学者/教师"职业的 NPC
教育设施（学院/学堂）存在于聚落建筑布局中
家长有经济能力让孩子不劳动（NpcEconomicState 允许）
→ 学院中聚集了一批学生
→ 教师有稳定收入（学费/聚落资助/神庙拨款）
→ Academy 教学路径被激活
```

这是第三层——只在少数聚落中存在。大多数 NPC 一生中只经历前两层。

## 三、4 条教学路径在童年中的体现

现有技能系统的 4 条路径完全覆盖儿童教育，无需新增：

| 路径 | 童年场景 | 机制 |
|------|---------|------|
| **SelfStudy** | 儿童在无人教的情况下自己观察、模仿、尝试——学会了走路、说话、抓握 | 感知→try_induce→MentalModel→自我尝试→XP |
| **Apprenticeship** | 跟铁匠学打铁、跟猎人学追踪、跟母亲学烹饪 | TeachingSession struct 中的师徒互动 |
| **Academy** | 学院中的系统学习——多位学生同时受教、分科教学 | 只在有教育设施和专职教师的聚落中可用 |
| **ArcaneTransmission** | 魔法/奥术的认知直接灌注——极罕见 | Magic 模块的特定机制 |

**没有"儿童专用"的教学路径。** 8 岁儿童学打铁和 28 岁成人学打铁走同一套代码。差异在：
- 儿童的学习率更高（cognitive_plasticity × LifeStage 加速）
- 儿童的物理属性更低（不能抡大锤→只能打小件→不同难度的技能训练）
- 儿童的 planning_horizon 更短（可能需要师父多次演示同样的东西）

## 四、学习率——童年加速的生物学基础

```rust
/// 认知学习率乘数——不由年龄直接决定，由认知参数派生
fn learning_rate_multiplier(npc: &NpcData) -> f32 {
    let base = 1.0;
    
    // 神经可塑性——童年高，随 mental_age 增加而下降
    let plasticity = 3.0 - 2.0 * npc.cognitive_style.mental_age;
    
    // 认知能力——同样影响学习效率
    let cognition = npc.cognitive_style.abstract_thinking * 0.3
                  + npc.cognitive_style.narrowness * -0.2
                  + 1.0;
    
    // 物理疲劳——太累学不进去
    let fatigue = 1.0 - npc.physiology.fatigue * 0.5;
    
    base * plasticity.clamp(0.5, 3.0) * cognition.clamp(0.5, 1.5) * fatigue.clamp(0.3, 1.0)
}
```

**关键**：plasticity 不由年龄直接决定——由 `mental_age`（经验累积量）决定。一个经历极度丰富、mental_age 已达 0.8 的 10 岁儿童，学习率可能和 30 岁普通成人一样。一个从未经历过任何挑战、mental_age 仍为 0.2 的 40 岁 NPC，学习率可能和一个 5 岁儿童一样高——**但两个人的智力、认知、技能基础完全不同，所以学出来的结果完全不同。**

## 五、游戏——学习的低风险变体

### 5.1 游戏不是独立行为

游戏是 GOAP 选择**低风险变体**执行技能相关动作的自然结果：

```
成人 GOAP 评估 "练习剑术" → 真刀对练，高风险，高 XP 增长
儿童 GOAP 评估 "练习剑术" → 木剑对练（因为力量不够真剑 + 认知评估风险）
                            → 高风险回避 → 选低风险变体
                            → 这就是"游戏打架"

成人 GOAP "社交 → 在酒馆讨论政治"
儿童 GOAP "社交 → 和其他儿童比谁的故事更夸张"
       → 因为词汇量少 + 没有政治 MentalModel
       → 这就是"过家家"
```

### 5.2 游戏的学习效果

游戏动作和练习动作使用完全相同的技能 XP 系统：

```
木剑对练 → Combat 技能 XP += 低风险 × 低难度 × 基础XP率
真刀对练 → Combat 技能 XP += 高风险 × 高难度 × 基础XP率

差异在 XP 的量，不在"这是游戏所以不增加技能"
```

一个每天和伙伴一起练习剑术的儿童，即使从未真刀战斗过，到青春期时已经有了相当的基础剑术 XP——远高于从不"玩"的儿童。这不是"系统鼓励儿童玩游戏"——是儿童从游戏中获得真实技能增长。

## 六、天才儿童——涌现而非预设

### 6.1 天才是什么

天才不是"系统给某个 NPC 打了一个天才标签"。天才是多个参数的有利组合在时间中的自然展开：

```
参数谱系：
  mental_seed 中含高 aptitude 值（受孕时确定）
  + 高 learning_rate（早期丰富环境→mental_age 适度增长但 plasticity 仍保持）
  + 好老师（家族/社区有高技能者愿意教）
  + 文化环境（鼓励学习、有教育资源）
  + 个人动机（BigFive.openness 高→好奇→频繁触发 SelfStudy）
  + 时间（从童年早期开始积累）
  → 到 12 岁时，技能 XP 和 MentalModel 数量远超同龄人
  → 甚至超过一些成年人
```

### 6.2 天才没有年龄门控——有参数优势

```
12 岁天才剑客 vs 40 岁平庸士兵：

物理属性（LifeStage 乘数 × 天赋种子）：
  天才: 0.6(LifeStage) × 1.3(天赋) = 0.78
  士兵: 1.0(LifeStage) × 0.9(天赋) = 0.90
  → 接近，天才稍逊

技能 XP（技能积累）：
  天才: 0.8 × 高学习率 × 好师父 × 5 年训练
  士兵: 0.5 × 中等学习率 × 基础训练 × 20 年
  → 可能反超——质量 > 数量

战术 MentalModel：
  天才: 8 条精妙战术模型（从名师那里学来）
  士兵: 3 条粗糙经验模型（从战场上活下来学的）
  → 天才胜出

GOAP 产出：
  天才的 planning_horizon=5, 8 条战术模型→产出精妙战术
  士兵的 planning_horizon=3, 3 条经验→产出常规打法
```

这不是"系统让儿童可以战斗"——是数学自然产出。

### 6.3 天才不需要标签

**开发者在任何地方不写 `if npc.is_genius { ... }`。** 天才只是参数空间中的一个区域——处于该区域的 NPC 自然表现出众。

## 七、世界生成中的教育集成

### 7.1 教育设施生成

```rust
// 聚落等级 → 教育设施强度
fn education_infrastructure(settlement: &SettlementSnapshot, culture: &CultureCoreParams) -> EducationInfra {
    EducationInfra {
        home_education_quality: culture.long_term_orientation * 0.5 + 0.3,
        apprenticeship_availability: settlement.profession_diversity * 0.8,
        academy_exists: settlement.grade >= SettlementGrade::Town 
                     && culture.long_term_orientation > 0.5,
        library_exists: settlement.grade >= SettlementGrade::City,
    }
}
```

### 7.2 教师 NPC 生成

P8 职业分布中，"学者/教师"作为职业的比例由文化 `education_investment` × 聚落等级决定：
- 高教育投资文化的大城市→专职教师比例高
- 低教育投资文化的村庄→没有专职教师，技能通过家庭和社区传播

### 7.3 初始年龄分布对教育的影响

P8 生成的人口金字塔中，儿童（Juvenile/Adolescent）的比例影响聚落的"教育压力"——如果儿童比例很高（战后婴儿潮），现有工匠能否同时带这么多学徒？如果不能→多数儿童只能靠自学或家庭教学。如果能→更多儿童进入学徒体系。这是涌现的供需关系，不是"教育政策系统"。

## 八、与进阶需求系统的关系——儿童 competence_frustration 修正

当前设计（进阶需求系统）中写：`儿童（< Adolescent）：无 SkillMastery aspiration → competence_frustration = 0.0`。

**这是硬编码——需要修正为涌现**：

```rust
// 修正后：
// competence_frustration 不由 LifeStage 直接归零
// 由认知参数自然派生：
fn competence_frustration(npc: &NpcData) -> f32 {
    if npc.aspiration_count(SkillMastery) == 0 {
        return 0.0;  // 没有技能志向——自然为0
    }
    // 有志向 → 计算志向-现实的差距
    // 儿童可以有志向（"我想成为铁匠"）→ frustration 可以非零
    compute_skill_aspiration_gap(npc) * npc.cognitive_style.investment_intensity
}
```

一个 8 岁儿童说"我想成为铁匠"但因年幼力弱而挫败——这**正是** competence_frustration 的表现。不需要禁止。0 是低参数的自然结果，不是硬编码的门。
