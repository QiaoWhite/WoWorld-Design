## 二、HolyDay 生成

### 2.1 生成规则

```rust
/// P2.5 中为每个 FaithDefinition 生成的 HolyDay 列表。
/// 数量：2 + ritual_formality * 5 —— 仪式越正式，节日越多。
fn generate_holy_days(theology: &FaithTheology, culture_rng: &mut impl Rng) -> Vec<HolyDay> {
    let count = (2.0 + theology.ritual_formality * 5.0) as usize;
    let mut days = Vec::with_capacity(count);
    
    // 1. 必有——一个年度最高庆典（对标圣诞节/开斋节/卫塞节）
    days.push(annual_grand_festival(theology, culture_rng));
    
    // 2. 季节性节日——春分/秋分/夏至/冬至 中选 1-4 个
    let seasonal_count = (1 + theology.nature_sacredness * 3.0) as usize;
    // ...
    
    // 3. 职能节日——基于神祇执掌（丰收→庆祝丰收、战争→战前祭祀）
    // ...
    
    // 4. 固定日节日——从 seed 随机分配 day_of_year
    // ...
    
    days
}
```

### 2.2 日期指定（适应 120 天/年历法）

游戏历法：120 天/年，四季各 30 天。春分 = day 15。HolyDayDate 使用 `Seasonal`/`Computed` 变体而非硬编码 `day_of_year`——使得模组可改变历年长度而不破坏节日逻辑。

- `FixedDay(u16)`: 0-119 固定日——每年此日
- `Seasonal { event, offset_days }`: 如 `(SpringEquinox, +3)` = 第 18 天
- `Lunar { phase, month }`: 如第 3 个月的第 1 个满月（游戏有月相系统吗？有——天气/天空系统提供月相数据）
- `Computed { rule }`: 如 `FirstFullMoonAfterSpringEquinox`——对标复活节算法

### 2.3 与节日系统的对接

```
FaithCalendarQuery::holy_days(faith_id)
  → 节日系统（文化 008 §5.3）消费
  → FestivalType::Religious { holy_day_id }
  → FestivalFramework.bound_faith = Some(FaithId)
  → NPC 参加决策中 piety_factor = participation * 0.4
```

## 三、FastingRule 与遵守

### 3.1 斋戒触发

NPC 模块在每日更新中检查：

```rust
/// NPC 模块调用——检查 NPC 今天是否应该斋戒。
fn should_fast_today(profile: &ReligiousPracticeProfile, day: u16) -> Vec<&FastingRule> {
    let mut applicable = Vec::new();
    for (faith_id, weight) in &profile.participation {
        if *weight < 0.3 { continue; }  // 太冷淡——不遵守斋戒
        
        let rules = faith_calendar_query.fasting_rules(*faith_id, day);
        for rule in rules {
            if rule.applies_to_npc(profile) && !rule.has_exemption(profile) {
                applicable.push(rule);
            }
        }
    }
    applicable
}
```

NPC 决策引擎根据 applicable 列表决定是否实际遵守（受 participation × 人格 conscientiousness 影响——高尽责性更遵守规则）。

### 3.2 斋戒豁免

NPC 自动检测豁免条件：年龄（<12 / >70）、健康状态（HP 不满/疾病标记）、旅行中、怀孕/哺乳、作战中。

## 四、朝圣

### 4.1 朝圣目标

```rust
pub struct PilgrimageTarget {
    /// 目标地理实体——圣地/圣城
    pub destination: GeographicEntityId,
    /// 朝圣类型
    pub pilgrimage_type: PilgrimageType,
    /// 朝圣触发条件
    pub trigger: PilgrimageTrigger,
    /// 完成后的 spiritual_need 重置程度
    pub spiritual_reset: f32,
}

pub enum PilgrimageType {
    OnceInLifetime,     // 一生一次（对标伊斯兰朝觐——哈吉）
    Annual,             // 年度朝圣
    Occasional,         // 在特定条件下触发（疾病/危机/感恩）
    Initiation,         // 入门仪式——入教时必须朝圣
}

pub enum PilgrimageTrigger {
    HolyDay(HolyDayDate),           // 特定神圣日
    LifeEvent(LifeEventType),       // 特定生活事件后
    PersonalDecision,               // 个人自主决定
    Mandatory { min_age: u8 },       // 强制性——所有信徒必须完成
}
```

### 4.2 朝圣执行

NPC GOAP 生成 Pilgrimage 目标 → 出发旅行（TravelerType::Pilgrim——已存在）→ 到达目的地 → 执行仪式（RitualDef）→ spiritual_need 重置 + participation +0.1。

朝圣旅行与普通旅行共享交通网络——NPC 可能加入商队一起走、可能被匪徒攻击、可能在路上生病——所有旅行力学走 NPC 模块已有系统。

## 五、人生过渡仪式（Rites of Passage）

### 5.1 仪式类型

```rust
pub enum RiteOfPassage {
    Birth,              // 出生——命名仪式/洗礼/剃胎发
    ComingOfAge,        // 成年礼——BAR MITZVAH/成人式/割礼
    Marriage,           // 婚礼——宗教婚姻仪式
    Ordination,         // 出家/就职——成为神职的仪式
    Death(FuneralCustoms), // 葬礼——见 §六
}
```

信仰系统定义每种仪式的 RitualDef 填充内容。执行走节日系统的 RitualDef 框架（`CelebrantRole::Clergy` / `RitualLocation::Building(temple)` / `ParticipantRequirement` 等）。

### 5.2 成年礼的特殊意义

成年礼后（约 16 岁）：NPC 的 motivation 可能从 `Habitual` 转变 → `PersonalDevotion`/`SocialCustom`/`NonPracticing`。这是子女信仰继承后的关键转折点。

## 六、葬仪（FuneralCustoms）

```rust
/// 信仰系统定义。生命系统 DeathCustom 消费。
pub struct FuneralCustoms {
    /// 尸体处理方式
    pub body_disposal: BodyDisposal,
    /// 葬礼仪式模板——填充 RitualDef
    pub funeral_ritual: RitualDef,
    /// 陪葬品规则
    pub grave_goods: GraveGoodsPolicy,
    /// 哀悼期（游戏日）
    pub mourning_period_days: u16,
    /// 对亡灵的态度
    pub attitude_toward_undead: UndeadAttitude,
}

pub enum BodyDisposal {
    Burial,             // 土葬
    Cremation,          // 火葬
    SkyBurial,          // 天葬——尸体暴露给鸟兽
    SeaBurial,          // 水葬/海葬
    Mummification,      // 木乃伊化——保存尸体
    NaturalReturn,      // 自然归还——不处理，让自然腐化
    TreeBurial,         // 树葬——在树下埋葬或置于树上
}

pub enum GraveGoodsPolicy {
    None,               // 无陪葬——"空手而来空手而去"
    Symbolic,           // 象征性物品——个人圣徽/经书
    Practical,          // 实用物品——工具/武器（"在来世继续使用"）
    Wealthy,            // 贵重物品——陪葬大量财富
    Sacrificial,        // 殉葬——（历史记录——大多数文化已废除但可能出现在历史中）
}

pub enum UndeadAttitude {
    Abomination,        // 亡灵是亵渎——必须消灭
    Pitiable,           // 亡灵是可怜的存在——应帮其安息
    Sacred,             // 亡灵是神圣的——祖先崇拜文化中与亡灵交流是宗教实践
    Neutral,            // 无特别态度
}
```

**葬仪对接**：
- 生命系统的 DeathCustom 消费 FuneralCustoms → 确定尸体处理方式
- 历史系统的墓碑文本消费 UndeadAttitude + 信仰 afterlife 教义 → 生成墓志铭
- 物品系统的陪葬品消费 GraveGoodsPolicy → 决定在神殿/墓穴中放置什么物品
- 亡灵生命的生成受 UndeadAttitude 影响——神圣 vs 可憎的亡灵来源概率不同（对接生命 008）

## 七、RitualDef 信仰填充

信仰系统不创建新的仪式框架——复用文化 008 的 `RitualDef`。填充规则：

| RitualDef 字段 | 信仰填充 |
|---------------|---------|
| `celebrant` | `Clergy { faith_id }`——祭司主持 |
| `location` | `Building { temple_id }` 或 `OpenAir { sacred_grove }` |
| `speech` | 从经文中选取对应的 RitualSpeechStep 序列 |
| `physical_actions` | 信仰特有的身体动作（跪/匍匐/画圣符/浇圣水/点燃蜡烛） |
| `magic` | 仅当 faith.magic_relation == Unified 或有神迹成分时填充 |
| `eligibility` | `BelieverOf { faith_id }`——仅信徒可参与 |
