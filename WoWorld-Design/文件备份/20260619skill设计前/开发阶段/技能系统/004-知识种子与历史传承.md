## §一、背景

`003-教学与跨模块接口.md` 的 `generate_starting_skills()` 函数接受 `profession_skills` 参数，但从未定义初始知识的来源。知识起源问题被完全留给世界生成模块。

本文件引入 **KnowledgeSeed** — 技术史的时间线元数据——填补这一空白。

## §二、KnowledgeSeed TOML 格式

```toml
# knowledge_seeds/technology_timeline.toml

[[era]]
name = "StoneAge"
start_year = -3000
end_year = -1500

[[era.tech]]
domain = "FireUse"
discovery = "Inevitable"           # 必然发现——任何人类聚落都自然掌握
base_skill_ceiling = 60
propagation = "Fast"
# 对应 SkillId: Survival.FireMaking

[[era.tech]]
domain = "FlintKnapping"
discovery = "Inevitable"
base_skill_ceiling = 50
propagation = "Fast"
# 对应 SkillId: Artisan.WoodStone.PrimitiveStonework

# ---

[[era]]
name = "BronzeAge"
start_year = -1500
end_year = -500

[[era.tech]]
domain = "CopperSmelting"
discovery = "Conditional"          # 条件发现
condition = { biome = ["Mountain"], resource = "CopperOre" }
base_skill_ceiling = 60
propagation = "Slow"
# 对应 SkillId: Artisan.Metalwork.Smelting

[[era.tech]]
domain = "IronSmelting"
discovery = "Breakthrough"         # 突破性——不是每个文明都会独立发现
condition = { prerequisite = "CopperSmelting", tech_level = 0.6 }
base_skill_ceiling = 40            # 早期铁器不成熟
propagation = "VerySlow"
# 对应 SkillId: Artisan.Metalwork.Smelting (Iron sub-variant)
```

### 字段说明

| 字段 | 含义 |
|------|------|
| `discovery` | 发现类型: Inevitable(必然)/Conditional(有条件)/Breakthrough(突破)/Prerequisite(有前置) |
| `base_skill_ceiling` | 该时代该技艺能达到的最高水平 |
| `propagation` | 传播速度: Fast/Slow/VerySlow —— 影响历史引擎的事件传播概率 |
| `condition` | 发现前提: biome/resource/prerequisite/tech_level 的组合 |

## §三、generate_starting_skills() 增强

```rust
    starting_biome: Biome,
    rng: &mut impl Rng,
) -> BTreeMap<SkillId, SkillEntry> {
    // 步骤1: 对每个职业主要技能:
    //   age_factor = ((age_years - 12.0) / 50.0).clamp(0.0, 1.0)
    //   target_level 受 settlement_tech.skill_ceiling(profession_skill) 约束

```

**关键**: `settlement_tech.skill_ceiling(skill_id)` 返回该聚落在该技艺上的当前天花板——由 KnowledgeSeed 的传播结果（该聚落在历史引擎模拟中何时学到了什么）决定。

## §四、与世界生成和历史引擎的集成

```
世界生成 P8/P9 流水线:

1. KnowledgeSeed TOML 加载 → 技术时代时间线
2. 历史引擎（历史系统 002）消费 KnowledgeSeed:
   - 为每个 RootEvent 生成"技艺发现"事件
   - 根据 propagation 速度 + 贸易路线强度 + 文化接触度
   - 自动生成传播事件链:"Settlement_A → Settlement_B"
3. 世界生成 P8 消费传播结果:
   - 每个聚落获得 SettlementTechLevel
   - 调用 generate_starting_skills(settlement_tech=该聚落的技术水平)
   - 生成当代 NPC 的技能分布
```

## §五、与现有技能系统的不变部分

| 不变项 | 说明 |
|--------|------|
| TeachingSession 4种路径 | SelfStudy/Apprenticeship/Academy/ArcaneTransmission 全部不变 |
| 三层天赋 | MentalAccess × InnateAptitude × CrossTraining 全部不变 |
| XP 累积公式 | `total_xp(L) = base × (e^(k×L) - 1) / k` 不变 |
| Cross-Training 规则 | 组内/跨组/天花板 全部不变 |
| SkillId 编码 | 5分类22子组 u64 编码不变 |

## §六、execution_noise_std() — 技能精度的物理映射

技能数值 level（0-100）通过 `execution_noise_std()` 映射到物理原子的执行噪声:

```rust
/// 定义在 woworld_core，由 NPC行动涌现模块 006 权威定义
/// 技能系统仅提供 proficiency() 输入
pub fn execution_noise_std(skill_level: f32) -> f32 {
    let proficiency = skill_level / 100.0;
    BASE_NOISE * (1.0 - proficiency).powi(2)
}
```

详见 [[../../NPC活人感模块/08-NPC行动涌现与分类/006-技能精度与原子执行噪声|物理原子层 006]]。

## §七、性能

- KnowledgeSeed TOML: <10KB, 一次性加载
- `generate_starting_skills()`: 世界生成时离线调用, 不占帧预算
- `execution_noise_std()`: ~10ns 纯函数, 每次原子执行调用一次
