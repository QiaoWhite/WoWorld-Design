## 一、总览

### 1.1 核心主张

**技术不是文化的属性——文化提供创新意愿，物质条件提供创新能力，两者的交互产生技术。**

设计后果：

1. **没有全局"时代"标签**。不预设"石器→青铜→铁器→蒸汽"的单线进化。孤立海岛可能拥有世界顶尖航海术但从未发展冶金——并且这是完全合法的涌现结果。
2. **TechnologyProfile 有 8 个独立领域**。每领域独立增长。矮人 metallurgy=90 但 navigation=15 是正常的——不是 bug。
3. **技术可以从零开始**。如果种子决定世界年龄为 0 年（无 P9 历史模拟），所有文化的 TechnologyProfile 从零开始——玩家将亲历文明从无到有的全部过程。

## 二、TechnologyProfile 完整定义

### 2.1 数据结构

```rust
/// 文化技术画像 —— 第二层派生参数
/// 
/// 8 个独立技术领域。每领域 0-100（但绝大多数文化在 5-85 范围）。
/// 0 = 该领域尚无系统性知识（可能有个人的天才但无集体知识体系）
/// 100 = 理论天花板——任何文化可以触及的最高理论水平
/// 
/// 从核心文化参数 + 人口 + 贸易 + 环境 + 历史时间 综合派生。
/// 重算触发器：核心参数显著变化（|Δ| > 0.03）、人口大幅度变动（±30%）、
///   贸易联系建立或中断、重大历史事件（战争/灾难/发现）
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct TechnologyProfile {
    /// 冶金技术 (0-100)
    /// 控制：金属武器/防具/工具的制造门槛、采矿效率
    /// 关联技能：artisan.metalwork.*, combat.melee.*（高级武器）
    /// 文化驱动：militarism（战争需要好武器）, artistry（金属工艺的美学追求）
    pub metallurgy: f32,

    /// 建筑与工程 (0-100)
    /// 控制：可建造的结构规模与复杂度（桥梁/水渠/城墙/多层建筑）
    /// 关联技能：artisan.woodstone.construction, artisan.woodstone.stonework
    /// 文化驱动：long_term（建造是长线投资）, power_distance（统治者需要宏构）
    pub construction: f32,

    /// 农业技术 (0-100)
    /// 控制：作物产量倍数、可种植作物种类、灌溉系统、轮作/休耕知识
    /// 关联技能：survival.gathering.*, artisan.domestic.*
    /// 文化驱动：long_term（农业是延迟满足的典型）, indulgence（享受美食→驱动农业质量）
    pub agriculture: f32,

    /// 航海技术 (0-100)
    /// 控制：可建造船只大小、航线距离、导航精度、海图绘制
    /// 关联技能：survival.movement.swimming, 未来航海技能
    /// 文化驱动：openness（航行=接触外部）, competition（航海竞赛/贸易竞争）
    pub navigation: f32,

    /// 纺织与服饰 (0-100)
    /// 控制：可用面料种类、染料技术、服饰复杂度
    /// 关联技能：artisan.textile.*
    /// 文化驱动：artistry（追求美的织物）, indulgence（享受精致的穿着）
    pub textiles: f32,

    /// 医学与卫生 (0-100)
    /// 控制：治疗效率、手术成功率、流行病抵抗率、平均寿命
    /// 关联技能：academic.medicine.*, survival.fieldcraft.first_aid
    /// 文化驱动：long_term（投资医疗=长期回报）, (1-militarism)（和平→医疗优先于军事）
    pub medicine: f32,

    /// 书写与学术 (0-100)
    /// 控制：识字率上限、书籍产量、学术机构可行性
    /// 关联技能：academic.humanities.*, academic.natural.*
    /// 文化驱动：artistry + long_term（书写=文化持久化的欲望）,
    ///           uncertainty_avoidance（高规避→记录规则/法律→驱动书写）
    pub writing: f32,

    /// 魔法技术 (0-100)
    /// 控制：该文化对魔法的系统性理解——法阵绘制、附魔工艺、魔法工程的工程化水平
    /// 关联技能：academic.arcane.*, magic.engineering.*
    /// 文化驱动：religiosity（宗教驱动还是抑制魔法研究？取决于信仰系统设定的魔法-宗教关系）,
    ///           artistry（美学驱动的魔法创新）, openness（接触外来魔法传统）
    /// 
    /// 注意：这不是"魔力浓度"——那是群系参数。这是"工程化运用魔法的水平"。
    ///       一个低魔力浓度地区的文化仍然可能有高 magic_tech（通过理论研究）。
    pub magic_tech: f32,
}
```

### 2.2 8 领域与技能系统 5 大类别的映射

```rust
impl SettlementTechQuery for CultureSystem {
    fn tech_ceiling(&self, category: SkillCategory) -> f32 {
        let tech = &self.technology_profile(culture_id);
        match category {
            SkillCategory::Combat => {
                // 战斗技能天花板 = 冶金（武器质量）× 0.7 + 军事化程度 × 0.3
                // 战斗技能不完全依赖技术——口传身授也能出高手
                // 但高品质武器的制造需要冶金
                (tech.metallurgy * 0.7 + self.core_params(culture_id).militarism * 100.0 * 0.3)
                    .clamp(0.0, 100.0)
            }
            SkillCategory::Magic => tech.magic_tech,
            SkillCategory::Artisan => {
                // 工艺天花板 = 冶金 × 0.35 + 建筑 × 0.25 + 纺织 × 0.20 + 农业 × 0.20
                weighted_average(&[
                    (tech.metallurgy, 0.35),
                    (tech.construction, 0.25),
                    (tech.textiles, 0.20),
                    (tech.agriculture, 0.20),
                ])
            }
            SkillCategory::Academic => tech.writing,
            SkillCategory::Survival => 100.0,
            // ↑ 生存技能不受技术限制——追踪和采集不需要"技术基础设施"
        }
    }
    
    fn has_infrastructure_for(&self, skill_id: SkillId) -> bool {
        let tech = &self.technology_profile(culture_id);
        match skill_id.category() {
            1 => true, // Combat: 口传身授，不需要基础设施
            2 => tech.magic_tech >= 10.0, // Magic: 需要最低魔法技术基础
            3 => match skill_id.subgroup() {
                1 => tech.metallurgy >= 15.0,     // Metalwork
                2 => tech.textiles >= 10.0,        // Textile
                3 => tech.construction >= 10.0,    // WoodStone
                4 => tech.metallurgy >= 30.0,      // Precision (珠宝/机械)
                5 => tech.agriculture >= 10.0,     // Domestic
                _ => true,
            },
            4 => tech.writing >= 20.0, // Academic: 需要书写基础设施
            5 => true, // Survival: 不需要基础设施
            _ => true,
        }
    }
}
```

## 三、技术增长模型

### 3.1 逻辑斯谛增长函数

每技术领域使用逻辑斯谛增长：

```
dL/dt = r × L × (1 - L / ceiling)

其中：
  r = 增长率（由创新动力 × 知识阶层 × 贸易接触 × 环境 × 文化权重 决定）
  L = 当前技术水平
  ceiling = 该领域的理论天花板（默认 95.0，可从领先贸易伙伴获得 +5% 溢出）
```

离散版本（每游戏年调用）：

```rust
fn grow_technology_field(
    innovation_drive: f32,       // 来自文化核心参数
    scholar_population: f32,     // 知识阶层人数 ≈ 总人口 × 识字率
    trade_exposure: f32,         // 通过贸易接触外来技术的程度 (0-1)
    environmental_bonus: f32,    // 环境条件对该领域的加成 (0-1)
    cultural_weight: f32,        // 文化对该领域的重视度 (0-1)
    years_active: f32,           // 该技术已发展的年数
    current_level: f32,          // 当前水平
    foreign_tech: f32,           // 最先进贸易伙伴的该领域水平
) -> f32 {
    // 年增长率
    let growth_rate = innovation_drive * 0.25
                    + scholar_population * 0.30
                    + trade_exposure * 0.20
                    + environmental_bonus * 0.10
                    + cultural_weight * 0.15;
    
    // 天花板：基础 95.0 + 可从贸易伙伴学习超出部分（5% 溢出）
    let ceiling = 95.0_f32.min(foreign_tech * 1.05);
    
    // 逻辑斯谛增长一步
    let k = growth_rate * 0.01; // 年增长率系数
    let new_level = ceiling / (1.0 + (ceiling / current_level.max(0.01) - 1.0) * (-k * years_active).exp());
    
    new_level.clamp(0.0, ceiling)
}
```

### 3.2 各领域的文化权重和环境加成

| 领域 | cultural_weight 公式 | environmental_bonus 依赖 |
|------|---------------------|------------------------|
| metallurgy | `militarism × 0.7 + artistry × 0.3` | `biome.mineral_richness` — 周边是否有可开采的金属矿 |
| construction | `long_term × 0.6 + power_distance × 0.4` | `biome.stone_availability` — 可用石材 |
| agriculture | `long_term × 0.5 + (1-militarism) × 0.5` | `biome.soil_fertility` — 土壤肥力 |
| navigation | `openness × 0.6 + competition × 0.4` | `biome.coast_accessibility` — 离海近？ |
| textiles | `artistry × 0.5 + indulgence × 0.5` | `biome.fiber_plant_suitability` — 亚麻/棉花/羊毛 |
| medicine | `long_term × 0.4 + (1-militarism) × 0.6` | `biome.herb_diversity` — 药用植物多样性 |
| writing | `artistry × 0.5 + long_term × 0.5` | 0.5（中性——任何地方都可以发展书写） |
| magic_tech | `religiosity × 0.5 + artistry × 0.5` | `biome.magic_density` — 魔力浓度 |

**关于 magic_tech 的 religiosity 权重**：`religiosity` 在此处的作用取决于信仰系统中"魔法与宗教的关系"——如果信仰视魔法为神赐，高 religiosity 会**加速**魔法技术；如果信仰视魔法为渎神，高 religiosity 会**抑制**魔法技术。此处的 `religiosity × 0.5` 是默认假设（宗教=魔法系统性研究的早期推动者），具体调优由信仰系统的 `MagicReligionRelation` 数据覆盖。详见未来信仰系统模块。

### 3.3 初始技术水平

世界生成 P2.5 时，每个文化根据"世界已经运转的年数"（world_age_years——种子参数，范围 0-5000）计算初始 TechnologyProfile：

```rust
fn initial_technology_profile(
    core: &CultureCoreParams,
    population: u32,
    biome: &BiomeParams,
    world_age_years: u32, // 种子决定——从 0（新生世界）到 5000（古老世界）
    rng: &mut impl Rng,
) -> TechnologyProfile {
    let innovation = core.long_term_orientation
                   * (1.0 - core.uncertainty_avoidance)
                   * (0.5 + core.artistry * 0.5);
    
    let literacy_base = 0.02 + core.artistry * 0.05;
    let scholars = population as f32 * literacy_base;
    
    let initial_trade = 0.0; // 初始无贸易——P9 历史模拟期间会修正
    
    TechnologyProfile {
        metallurgy: grow_tech(innovation, scholars, initial_trade,
            biome.mineral_richness(),
            core.militarism * 0.7 + core.artistry * 0.3,
            world_age_years as f32, 0.0, 0.0),
        
        construction: grow_tech(innovation, scholars, initial_trade,
            biome.stone_availability(),
            core.long_term_orientation * 0.6 + core.power_distance * 0.4,
            world_age_years as f32, 0.0, 0.0),
        
        agriculture: grow_tech(innovation, scholars, initial_trade,
            biome.soil_fertility(),
            core.long_term_orientation * 0.5 + (1.0 - core.militarism) * 0.5,
            world_age_years as f32, 0.0, 0.0),
        
        navigation: grow_tech(innovation, scholars, initial_trade,
            biome.coast_accessibility(),
            core.openness_to_outsiders * 0.6 + core.competition_orientation * 0.4,
            world_age_years as f32, 0.0, 0.0),
        
        textiles: grow_tech(innovation, scholars, initial_trade,
            biome.fiber_plant_suitability(),
            core.artistry * 0.5 + core.indulgence * 0.5,
            world_age_years as f32, 0.0, 0.0),
        
        medicine: grow_tech(innovation, scholars, initial_trade,
            biome.herb_diversity(),
            core.long_term_orientation * 0.4 + (1.0 - core.militarism) * 0.6,
            world_age_years as f32, 0.0, 0.0),
        
        writing: grow_tech(innovation, scholars, initial_trade,
            0.5,
            core.artistry * 0.5 + core.long_term_orientation * 0.5,
            world_age_years as f32, 0.0, 0.0),
        
        magic_tech: grow_tech(innovation, scholars, initial_trade,
            biome.magic_density(),
            core.religiosity * 0.5 + core.artistry * 0.5,
            world_age_years as f32, 0.0, 0.0),
    }
}
```

## 四、技术扩散模型

### 4.1 贸易驱动扩散

每游戏年，对于每对有贸易联系的文化：

```rust
fn technology_diffusion(
    a_tech: &mut TechnologyProfile,
    b_tech: &mut TechnologyProfile,
    trade_volume: f32,         // 0-1 归一化贸易量
    openness_a: f32,
    openness_b: f32,
) {
    let diffusion_rate = trade_volume * 0.01 * openness_a.min(openness_b);
    
    // 对每个技术领域：落后方从领先方学习
    diffuse_field(&mut a_tech.metallurgy, b_tech.metallurgy, diffusion_rate);
    diffuse_field(&mut a_tech.construction, b_tech.construction, diffusion_rate);
    diffuse_field(&mut a_tech.agriculture, b_tech.agriculture, diffusion_rate);
    diffuse_field(&mut a_tech.navigation, b_tech.navigation, diffusion_rate);
    diffuse_field(&mut a_tech.textiles, b_tech.textiles, diffusion_rate);
    diffuse_field(&mut a_tech.medicine, b_tech.medicine, diffusion_rate);
    diffuse_field(&mut a_tech.writing, b_tech.writing, diffusion_rate);
    diffuse_field(&mut a_tech.magic_tech, b_tech.magic_tech, diffusion_rate);
}

fn diffuse_field(my_level: &mut f32, other_level: f32, diffusion_rate: f32) {
    let gap = other_level - *my_level;
    if gap > 0.0 {
        *my_level += gap * diffusion_rate;
        // 不能超过对方的水平（最多到其实不科学，但合理近似——技术转移不会完美）
    }
}
```

### 4.2 征服/殖民驱动的技术转移

被征服方被迫接触征服者的技术——而不取决于 openness：

```rust
fn conquest_technology_transfer(
    conquered_tech: &mut TechnologyProfile,
    conqueror_tech: &TechnologyProfile,
    years_under_rule: f32,
    imposition_intensity: f32,  // 征服者强制程度——由征服者的 militarism + power_distance 决定
) {
    let forced_rate = imposition_intensity * 0.015 * years_under_rule;
    
    // 被征服方的所有领域向征服方靠拢
    // 速度比贸易扩散快 3-5 倍（征服是强制的、全面的）
    for field in TECH_FIELDS {
        let gap = conqueror_tech.get(field) - conquered_tech.get(field);
        if gap > 0.0 {
            conquered_tech.set(field, conquered_tech.get(field) + gap * forced_rate);
        }
    }
}
```

### 4.3 技术停滞与倒退

技术不是单调上升的。以下情况触发停滞或倒退：

```rust
fn check_technology_decline(
    tech: &mut TechnologyProfile,
    population_change: f32,     // 过去 10 年人口变化率
    major_disaster: bool,       // 过去 10 年是否发生重大灾难
    prolonged_war: bool,        // 过去 10 年是否持续战争
    core: &CultureCoreParams,
) -> Vec<TechDeclineEvent> {
    let mut events = Vec::new();
    
    for field in TECH_FIELDS {
        let mut decline = 0.0;
        
        // 人口锐减（>40%）→ 知识分子阶层消失 → 所有领域倒退
        if population_change < -0.4 {
            decline += (-population_change - 0.4) * 10.0; // e.g., -0.6→倒退2%
        }
        
        // 重大灾难（瘟疫/火山/海啸）→ 特定领域的知识体系崩溃
        if major_disaster {
            decline += 1.5;
        }
        
        // 长期战争 → 书写/医疗/农业 → 倒退（社会资源移至军事）
        if prolonged_war {
            match field {
                TechField::Writing | TechField::Medicine | TechField::Agriculture => {
                    decline += core.militarism * 2.0;
                }
                TechField::Metallurgy => {
                    // 战争反而加速冶金——武器技术进步
                    decline = -core.militarism * 1.5; // 负值 = 增长
                }
                _ => {}
            }
        }
        
        if decline > 0.0 {
            let new_level = (tech.get(field) - decline).max(1.0);
            tech.set(field, new_level);
            events.push(TechDeclineEvent { field, amount: decline });
        }
    }
    
    events
}
```

## 六、物品可制造门槛

物品系统 006 的 `SkillRequirement` 不变——它仍检查 NPC 个人技能等级。文化系统提供的 `SettlementTechQuery` 用于检查定居点层面的技术条件：

```rust
// 物品系统 006 中——在定居点层面检查是否可制造
fn can_craft_in_settlement(
    recipe: &CraftingRecipe,
    settlement_tech: &dyn SettlementTechQuery,
    npc_skills: &BTreeMap<SkillId, SkillEntry>,
) -> bool {
    // 1. 定居点有技术基础设施？
    for req in &recipe.skill_requirements {
        if !settlement_tech.has_infrastructure_for(req.skill) {
            return false;
        }
    }
    
    // 2. NPC 个人技能达标？（原有逻辑）
    for req in &recipe.skill_requirements {
        let npc_level = npc_skills.get(&req.skill).map(|e| e.level).unwrap_or(0.0);
        if npc_level < req.min_level {
            return false;
        }
    }
    
    true
}
```

**事例说明**：即使一个 NPC 锻造 Lv90，如果他被困在 `metallurgy=5` 的文化孤岛上，他也无法制造钢剑——因为孤岛没有高炉、没有淬火油、没有焦炭窑。技能代表**个人能力**，TechnologyProfile 代表**可获得的基础设施和集体知识**。

## 七、玩家体验

玩家在不同文化的城市中——可感知的技术差异：

| 城市技术特征 | 玩家在游戏中看到的 |
|------------|-----------------|
| `metallurgy=75, navigation=10` | 市场里有钢剑和板甲卖，但港口只有小渔船——渡海不可能 |
| `metallurgy=20, navigation=80` | 最好的武器是青铜剑，但港口停着三桅远洋帆船 |
| `writing=85, magic_tech=15` | 满街书店、图书馆、学者辩论——但魔法被视为无稽之谈 |
| `magic_tech=90, writing=20` | 法阵路灯、魔力水渠、魔法传送门——但大多数人不会读写（靠口传和实物展示） |
| `agriculture=80, medicine=15` | 城市被农场环绕，粮食丰富——但受伤后只能靠民间偏方 |
| `medicine=75, agriculture=20` | 干净的医院、专业外科医生——但粮食经常短缺、依赖进口 |

这就是涌现的技术多样性——不是"这个地方是铁器时代"，而是"这个文化善于此但拙于彼"的多元格局。
