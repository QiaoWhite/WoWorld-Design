## 一、CultureQuery trait 完整定义

### 1.1 核心查询 trait（高频——每帧数百次调用）

```rust
/// 文化系统只读查询接口 —— 所有消费模块通过此 trait 获取文化数据
/// 
/// 设计原则：
///   - 零反向依赖：此 trait 不引用任何其他模块的类型
///   - 零分配：所有方法返回引用或栈上 Copy 值
///   - Send + Sync：可从多个线程安全调用（Rust 模拟线程 + Godot 渲染线程）
///   - 实现方不透明：消费方不知道文化是 Voronoi 区域还是连续场
pub trait CultureQuery: Send + Sync {
    // ━━━ 空间查询（最热路径——每帧数百次）━━━
    
    /// 查询世界空间中某位置的文化归属
    /// 
    /// 返回：(主导 CultureId, 混合权重 0.0-1.0)
    ///   1.0 = 纯文化区
    ///   0.7 = 70% 此文化 30% 次近文化（边界带）
    /// 
    /// 性能：O(log n) 四叉树查询，~50ns，栈上返回两个 Copy 值
    fn culture_at(&self, pos: WorldPos) -> (CultureId, f32);
    
    /// 批量查询 —— 为合法性重算等批量操作优化
    /// 
    /// 默认实现循环调用 culture_at，但空间索引实现方可以覆盖：
    ///   - 按四叉树节点分组 → BLUK 空间批处理
    ///   - SIMD 多位置并行距离比较
    fn batch_culture_at(&self, positions: &[WorldPos]) -> Vec<(CultureId, f32)> {
        positions.iter().map(|p| self.culture_at(*p)).collect()
    }
    
    // ━━━ 文化身份查询（热路径——每对话回合至少一次）━━━
    
    /// 文化的核心参数 —— 极高频
    /// 返回不可变引用，零拷贝
    fn core_params(&self, id: CultureId) -> Option<&CultureCoreParams>;
    
    /// 沟通规范 —— NPC 对话每回合查询
    fn communication_norms(&self, id: CultureId) -> Option<&CommunicationNorms>;
    
    /// 敬语系统 —— 语言表达每对话回合查询
    fn honorific_system(&self, id: CultureId) -> Option<&HonorificSystem>;
    
    // ━━━ 关系查询（中频——每游戏日/每事件）━━━
    
    /// 两个文化的相似度 —— 0.0-1.0，1.0 = 完全相同
    /// 
    /// 基于 CultureCoreParams 10 维向量的余弦相似度
    /// 预计算懒缓存——首次查询 (A,B) 时计算并缓存，后续 O(1)
    fn similarity(&self, a: CultureId, b: CultureId) -> f32;
    
    /// 两个文化的距离 —— 0.0-1.0，0.0 = 完全相同
    fn distance(&self, a: CultureId, b: CultureId) -> f32 {
        1.0 - self.similarity(a, b)
    }
    
    /// 跨文化态度 —— A 文化对 B 文化的初始接纳度 0.0-1.0
    /// 
    /// 从 openness × similarity × 历史接触经验 派生
    /// 1.0 = 全然接纳，0.0 = 全然敌视
    fn cross_cultural_attitude(&self, observer: CultureId, target: CultureId) -> f32;
    
    // ━━━ 辅助查询 ━━━
    
    /// 所有存在的文化列表
    fn all_cultures(&self) -> &[CultureId];
    
    /// 文化总数
    fn culture_count(&self) -> usize;
}
```

## 二、CultureMut trait —— 文化修改接口

```rust
/// 文化系统修改接口 —— pub(crate)：仅世界生成管线和历史模拟引擎可调用
/// 
/// 消费模块（NPC/战斗/魔法/语言……）只看到 CultureQuery——永远不能修改文化参数。
/// 这保证了文化演变只有一个入口——不会被分散的逻辑意外修改。
pub(crate) trait CultureMut: CultureQuery {
    // ━━━ 代际漂移 ━━━
    
    /// 每年自动调用 —— 所有 10 个核心参数独立微调
    /// σ = 0.003 每维每游戏年
    fn drift_all(&mut self, years: u32, rng: &mut impl Rng);
    
    // ━━━ 贸易影响 ━━━
    
    /// 贸易驱动的参数收敛
    /// influence_flow = trade_volume_norm × openness(A) × openness(B)
    fn apply_trade_influence(&mut self, a: CultureId, b: CultureId, volume_normalized: f32);
    
    // ━━━ 征服强制 ━━━
    
    /// 征服者文化对被征服者文化的强制偏移
    fn apply_conquest_imposition(
        &mut self, 
        conqueror: CultureId, 
        conquered: CultureId, 
        intensity: f32,
        years: u32,
    );
    
    // ━━━ 事件驱动 ━━━
    
    /// 应用 CultureShift 事件 —— 单次显著参数偏移
    /// 返回实际应用的偏移量（可能被钳制）
    fn apply_culture_shift(
        &mut self, 
        culture_id: CultureId, 
        shifts: &[(CultureParam, f32)],  // (哪个参数, 偏移量)
    ) -> Vec<(CultureParam, f32)>;
    
    // ━━━ 文化分裂与融合 ━━━
    
    /// 检查并触发文化分裂
    /// 条件：两个子集的隔离时间 > 200年 且 参数距离 > 0.15
    fn check_divergence(&mut self, current_time: GameTime) -> Vec<CultureDivergenceEvent>;
    
    /// 检查并触发克里奥尔化
    /// 条件：混合区持续 > 300年 且 混合参数与双亲距离 > 0.2
    fn check_creolization(&mut self, current_time: GameTime) -> Vec<CultureCreolizationEvent>;
    
    /// 创建新的子文化（从分裂）
    fn crystallize_subculture(
        &mut self, 
        parent: CultureId, 
        params: CultureCoreParams,
        name: CultureName,
    ) -> CultureId;
    
    /// 创建融合文化（克里奥尔化）
    fn crystallize_creole(
        &mut self,
        parents: (CultureId, CultureId),
        params: CultureCoreParams,
        name: CultureName,
    ) -> CultureId;
    
    // ━━━ 文化消亡 ━━━
    
    /// 标记文化为已消亡（所有载体消失后调用）
    fn mark_extinct(&mut self, id: CultureId, time: GameTime);
    
    // ━━━ 技术更新 ━━━
    
    /// 重算指定文化的 TechnologyProfile（文化参数/人口/贸易变化后触发）
    fn recalculate_technology(&mut self, id: CultureId, population: u32, trade_exposure: f32);
    
    /// 对一对贸易伙伴执行年度技术扩散
    fn technology_diffusion_step(
        &mut self, 
        a: CultureId, 
        b: CultureId, 
        trade_volume: f32,
    );
}
```

## 三、文化演变四路径详解

### 3.1 代际漂移

最慢但最持续的变化来源——模拟文化在无外部干涉下的自然"呼吸"：

```rust
impl CultureMut for CultureSystem {
    fn drift_all(&mut self, years: u32, rng: &mut impl Rng) {
        for culture_id in self.all_cultures().to_vec() {
            let params = self.store.cores[culture_id.0 as usize];
            let mut shifted = params;
            
            // 每个参数独立漂移：N(0, σ=0.003) × 年数
            let effective_sigma = 0.003 * (years as f32).sqrt();
            // 使用 sqrt(years) 因为每年的随机游走累积方差 = σ² × years
            // 标准差 = σ × sqrt(years)
            
            shifted.individualism += gaussian_sample(rng, 0.0, effective_sigma);
            shifted.power_distance += gaussian_sample(rng, 0.0, effective_sigma);
            // ...所有 10 个参数
            
            // 钳制到 [0, 1]
            shifted = shifted.clamp_all();
            
            // 参数变化 > 0.05 → 重新派生第一层参数
            if shifted.distance(&params) > 0.05 {
                self.regenerate_derived(culture_id, &shifted);
            }
            
            self.store.cores[culture_id.0 as usize] = shifted;
        }
    }
}
```

**σ=0.003/年的含义**：100 年内累积漂移约 0.03（一个标准差），1000 年内约 0.10。这意味着在一个玩家存档的游戏时间跨度内（通常数百小时游戏时间=数年游戏时间），文化漂移几乎不可察觉（<0.01）——文化在玩家生命周期内保持稳定，这保证了玩家对一个文化产生归属感。

### 3.2 贸易影响

```rust
fn apply_trade_influence(&mut self, a: CultureId, b: CultureId, volume: f32) {
    let openness_a = self.core_params(a).openness_to_outsiders;
    let openness_b = self.core_params(b).openness_to_outsiders;
    let influence_flow = volume * openness_a * openness_b * 0.001;
    
    let params_a = self.store.cores[a.0 as usize];
    let params_b = self.store.cores[b.0 as usize];
    
    // 对称影响——双方参数互相靠拢
    // 每个参数有独立的"传染性"——有些维度更易受影响
    let contagion = [0.8, 0.3, 0.5, 0.6, 0.5, 0.7, 0.8, 0.4, 0.5, 0.7];
    // individualism, power_distance, uncertainty_avoidance, competition, long_term,
    // indulgence, openness, religiosity, militarism, artistry
    
    for i in 0..10 {
        let move_a = (params_b.get_dim(i) - params_a.get_dim(i)) * influence_flow * contagion[i];
        let move_b = (params_a.get_dim(i) - params_b.get_dim(i)) * influence_flow * contagion[i];
        self.store.cores[a.0 as usize].set_dim(i, params_a.get_dim(i) + move_a);
        self.store.cores[b.0 as usize].set_dim(i, params_b.get_dim(i) + move_b);
    }
}
```

**传染性设计**：`artistry` (0.7) 和 `openness` (0.8) 极易通过贸易传播——一个文化的艺术风格和对外的态度容易被贸易伙伴影响。`power_distance` (0.3) 和 `religiosity` (0.4) 最难通过贸易改变——深植于社会结构和信仰。

### 3.3 征服强制

```rust
fn apply_conquest_imposition(
    &mut self,
    conqueror: CultureId,
    conquered: CultureId,
    intensity: f32,   // 0-1，由征服者的 militarism + power_distance 决定
    years: u32,
) {
    let params_conqueror = self.store.cores[conqueror.0 as usize];
    let params_conquered = self.store.cores[conquered.0 as usize];
    
    // 被征服方的抵抗能力
    let resistance = params_conquered.uncertainty_avoidance * 0.6
                   + (1.0 - params_conquered.openness_to_outsiders) * 0.4;
    
    let effective_rate = intensity * (1.0 - resistance) * 0.005 * years as f32;
    
    for i in 0..10 {
        let shift = (params_conqueror.get_dim(i) - params_conquered.get_dim(i)) * effective_rate;
        self.store.cores[conquered.0 as usize].set_dim(
            i, params_conquered.get_dim(i) + shift
        );
    }
}
```

**高抵抗力示例**：高 `uncertainty_avoidance=0.9` + 低 `openness=0.1` → resistance = 0.9 × 0.6 + 0.9 × 0.4 = 0.90。征服者只能以 10% 的效率强制文化改变——即被征服方只被迫在表面上屈服，核心价值不变。

### 3.4 事件驱动

```rust
/// 历史事件直接修改文化参数
/// 
/// 事件由历史系统发出，文化系统执行修改。
/// 历史系统不直接写文化数据——通过此方法间接修改。
fn apply_culture_shift(
    &mut self,
    culture_id: CultureId,
    shifts: &[(CultureParam, f32)],
) -> Vec<(CultureParam, f32)> {
    let params = &self.store.cores[culture_id.0 as usize];
    let mut applied = Vec::new();
    
    for &(param, requested_delta) in shifts {
        let current = params.get(param);
        // 单次事件最大偏移 ±0.15（防止单事件过度变形文化）
        let clamped_delta = requested_delta.clamp(-0.15, 0.15);
        let new_value = (current + clamped_delta).clamp(0.0, 1.0);
        params.set(param, new_value);
        applied.push((param, new_value - current));
    }
    
    // 参数变化 > 0.03 → 重新派生第一层参数
    if shifts.iter().map(|(_, d)| d.abs()).sum::<f32>() > 0.03 {
        self.regenerate_derived(culture_id, params);
    }
    
    applied
}

/// 文化参数枚举 —— 10 个可被事件修改的维度
pub enum CultureParam {
    Individualism,
    PowerDistance,
    UncertaintyAvoidance,
    CompetitionOrientation,
    LongTermOrientation,
    Indulgence,
    OpennessToOutsiders,
    Religiosity,
    Militarism,
    Artistry,
}
```

## 四、文化分裂与融合机制

### 4.1 亚文化分裂

```rust
fn check_divergence(&mut self, current_time: GameTime) -> Vec<CultureDivergenceEvent> {
    let mut events = Vec::new();
    
    for culture_id in self.all_cultures().to_vec() {
        // 该文化的所有"子集"——被不同政体分割的文化区段
        let segments = self.find_culture_segments(culture_id);
        if segments.len() < 2 { continue; }
        
        for (a, b) in pairs(&segments) {
            let isolation_years = current_time.years_since(a.last_contact_time);
            
            // 隔离 > 200 年 → 检查是否分裂
            if isolation_years >= 200 {
                let params_a = self.core_params(a.culture_id);
                let params_b = self.core_params(b.culture_id);
                let distance = cosine_distance(params_a, params_b);
                
                // 参数距离 > 0.15 → 分裂为新文化
                if distance > 0.15 {
                    let child_params = interpolate(params_a, params_b, 0.5);
                    let child_name = generate_child_culture_name(a, b);
                    let child_id = self.crystallize_subculture(culture_id, child_params, child_name);
                    
                    events.push(CultureDivergenceEvent {
                        parent: culture_id,
                        child: child_id,
                        divergence_time: current_time,
                        parameter_distance: distance,
                        isolation_years,
                    });
                }
            }
        }
    }
    
    events
}
```

### 4.2 克里奥尔化

```rust
fn check_creolization(&mut self, current_time: GameTime) -> Vec<CultureCreolizationEvent> {
    let mut events = Vec::new();
    
    // 检查所有文化边界带中的聚落
    for blend_zone in self.blend_zones.iter() {
        if blend_zone.stable_years(current_time) < 300 { continue; }
        
        let blended_params = blend_zone.compute_blended_params();
        let dist_a = blended_params.distance(self.core_params(blend_zone.a));
        let dist_b = blended_params.distance(self.core_params(blend_zone.b));
        
        // 混合参数与双亲的距离都 > 0.2 → 足够不同 → 新文化
        if dist_a > 0.2 && dist_b > 0.2 {
            let creole_name = generate_creole_name(blend_zone);
            let creole_id = self.crystallize_creole(
                (blend_zone.a, blend_zone.b),
                blended_params,
                creole_name,
            );
            
            events.push(CultureCreolizationEvent {
                parents: (blend_zone.a, blend_zone.b),
                child: creole_id,
                formation_time: current_time,
                zone_center: blend_zone.center,
            });
        }
    }
    
    events
}
```

## 五、跨模块接口契约表

本表是文化系统对外的权威接口注册表。所有消费方引用此表。

| # | 接口 | 位置（提供方） | 消费方 | 数据格式 | 说明 |
|---|------|-------------|--------|---------|------|
| 1 | `CultureQuery` trait | 文化系统 006（本文档） | 全部模块 | trait (Send+Sync) | 所有模块通过此 trait 获取文化数据。实现方不透明——消费方不知道数据来自 Voronoi 还是连续场 |
| 2 | `CultureQueryExt` trait | 文化系统 006 §1.2 | 世界生成、历史、UI | trait | 扩展查询——低频操作。与 `CultureQuery` 分离以优化编译 |
| 3 | `CultureMut` trait | 文化系统 006 §二 | 世界生成管线、历史模拟引擎 | pub(crate) trait | 文化修改的唯一入口。消费模块不可调用 |

| 5 | `CultureCoreParams` | 文化系统 002 §一 | 全部模块 | `struct` (10×f32) | 文化 10 核心参数。从 `core_params(id)` 获取 |
| 6 | `CommunicationNorms` | 文化系统 003 §二 | 语言表达 005/010、NPC | `struct` (8字段) | ★ 所有权从语言表达 010 转移至文化系统 |

## 七、消费方查询代码示例

```rust
// ━━━ NPC GOAP 决策——每帧 ━━━
fn evaluate_talk_action(npc: &NpcData, target: &NpcData, culture: &dyn CultureQuery) -> f32 {
    let my_norms = culture.communication_norms(npc.culture_id).unwrap();
    let their_norms = culture.communication_norms(target.culture_id).unwrap();
    
    // 文化距离 → 轻减交谈意愿（非硬阻，仅温和偏移）
    let culture_penalty = culture.distance(npc.culture_id, target.culture_id) * 0.15;
    
    // directness 差距 > 0.5 → 可能的误解风险 → 进一步轻减
    let directness_gap = (my_norms.directness - their_norms.directness).abs();
    let miscommunication_risk = if directness_gap > 0.5 { 0.2 } else { 0.0 };
    
    1.0 - culture_penalty - miscommunication_risk
}

// ━━━ 权力系统合法性计算——每游戏日 ━━━
fn compute_legitimacy(subject_pos: WorldPos, power_edge: &PowerEdge, culture: &dyn CultureQuery) -> f32 {
    let (culture_id, blend) = culture.culture_at(subject_pos);
    let params = culture.core_params(culture_id).unwrap();
    
    // 文化契合度 = 权力距离与权力执行强度的匹配度
    let culture_fit = 1.0 - (power_edge.enforcement_intensity - params.power_distance).abs();
    culture_fit * 0.20 // 权重 0.20
}

// ━━━ 语言表达对话生成——每对话回合 ━━━
fn generate_greeting(speaker: &NpcData, listener: &NpcData, culture: &dyn CultureQuery) -> String {
    let norms = culture.communication_norms(speaker.culture_id).unwrap();
    let honorifics = culture.honorific_system(speaker.culture_id).unwrap();
    
    let fragment_key = if norms.directness > 0.7 { "greeting_direct" }
                       else if norms.directness > 0.3 { "greeting_neutral" }
                       else { "greeting_indirect" };
    
    if honorifics.status_based && listener.social_status > speaker.social_status {
        format!("{} {}", honorific_prefix(listener), lookup_fragment(fragment_key))
    } else {
        lookup_fragment(fragment_key)
    }
}
```

## 八、内部存储设计

```rust
/// 文化系统的内部存储 —— 列式布局，缓存友好
/// 
/// 消费方只看到 CultureQuery trait——不知道此结构的存在
pub(crate) struct CultureStore {
    /// SoA 存储——索引 = CultureId.0 as usize
    cores: Vec<CultureCoreParams>,              // 10×f32 = 40B each
    communication_norms: Vec<CommunicationNorms>,  // ~40B
    building_styles: Vec<BuildingStylePreferences>, // ~32B
    beauty_standards: Vec<CulturalBeautyStandard>,  // ~48B
    honorific_systems: Vec<HonorificSystem>,        // ~8B
    fertility_norms: Vec<FertilityNorms>,           // ~16B
    dietary_prefs: Vec<DietaryBasePreferences>,     // ~16B
    clothing_prefs: Vec<ClothingPreferences>,       // ~16B
    technology_profiles: Vec<TechnologyProfile>,    // 8×f32 = 32B
    language_ids: Vec<LanguageId>,                  // 4B
    names: Vec<CultureName>,                        // ~128B (含 exonym map)
    
    /// 无效条目标记（已消亡的 CultureId 位置）
    /// 并行 Vec<bool> —— 比 Option 更缓存友好
    valid: Vec<bool>,
}

/// 空间索引 —— 四叉树
/// 每叶节点存储该区域内所有 CultureId 的排序列表
struct CultureSpatialIndex {
    root: QuadTreeNode,
}

struct QuadTreeNode {
    aabb: Aabb2D,
    children: Option<[Box<QuadTreeNode>; 4]>,
    /// 叶节点：该区域内的主导 CultureId（按占比排序）
    dominant_cultures: Vec<(CultureId, f32)>, // (id, 该区域内的最大混合权重)
}

/// 预计算的相似度矩阵（上三角，懒计算）
struct SimilarityCache {
    /// 对于 N 个文化，N*(N-1)/2 个 f32 值
    /// 存储格式：行优先上三角 [idx(0,1), idx(0,2), ..., idx(0,N-1), idx(1,2), ...]
    data: Vec<f32>,
    /// 哪些对已计算
    computed: Vec<bool>,
}

/// 每个文化总计约：40+40+32+48+8+16+16+16+32+4+128 ≈ 380B
/// 100 个文化 → ~38KB —— 极小的内存开销
/// 1000 个文化（未来无限世界）→ ~380KB —— 仍然可忽略
```

## 十一、CHG-013 系列接口契约兼容

文化系统作为**被依赖的纯数据源**，不修改任何已有接口：

- 不修改十元素顺序（金木水火土风雷电血灵）——Magic 002 权威
- 不修改魔力单位（u32 刻）——Life 004 权威
- 不修改部位伤害渐进模型——Combat 012 权威
- 不修改 Physiology ↔ Vitals 派生关系
- `CultureId` / `CultureCoreParams` / `CultureQuery` 等为**新增**命名空间——不作为任何已有 ID/属性的替代
- `CommunicationNorms` 所有权从语言表达 → 文化系统——但此类型在语言表达中本就是"被定义"而非"被使用"的，转移不破坏任何消费方代码
