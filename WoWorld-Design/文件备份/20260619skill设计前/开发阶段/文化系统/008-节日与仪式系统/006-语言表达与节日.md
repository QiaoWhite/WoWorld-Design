## 一、仪式对话——TurnMode::Ritual

### 1.1 节日仪式与 TurnMode::Ritual 的对齐

语言表达 005 中 `TurnMode::Ritual` 已完整设计——有 `RitualStep` 序列。节日系统需要做的就是填充这些步骤的具体内容：

```rust
/// 节日仪式对话——填充 TurnMode::Ritual 的 RitualStep 序列
fn generate_festival_ritual_dialogue(
    festival: &FestivalFramework,
    celebrant: &CelebrantRole,
    culture: &CultureCoreParams,
    lexicon: &CultureToponymyLexicon,
) -> Vec<RitualStep> {
    let emotional_tone = festival.emotional_tone;
    let formality = match emotional_tone {
        Solemn | Contemplative => FormalityLevel::VeryFormal,
        Joyful => FormalityLevel::Formal,
        Ecstatic => FormalityLevel::Casual, // 狂欢节——仪式也不需要太正式
        Competitive => FormalityLevel::Formal,
    };
    
    let mut steps = Vec::new();
    
    // 模板步骤——由节日类型和文化参数填充
    match festival.festival_type {
        FestivalType::Calendar { .. } => {
            steps.push(opening_blessing(celebrant, culture, formality));
            steps.push(seasonal_invocation(&festival.name, lexicon));
            steps.push(communal_response()); // 群众齐声回应
        }
        FestivalType::Religious { .. } => {
            steps.push(sacred_invocation(celebrant, festival.bound_faith));
            steps.push(offering_presentation());
            steps.push(benediction(celebrant, culture));
        }
        // ...
    }
    
    steps
}

fn opening_blessing(celebrant: &CelebrantRole, culture: &CultureCoreParams, formality: FormalityLevel) -> RitualStep {
    RitualStep {
        speaker: celebrant.entity_id(),
        text_template_id: hash_template("festival_opening_blessing"),
        params: hashmap! {
            "festival_name" => "...",
            "season_name" => "...",
            "deity_name" => "...", // 如果有
            "formality" => formality.as_str(),
        },
        duration_seconds: 30.0,
    }
}
```

### 1.2 四种仪式对话模式

| 模式 | TurnMode | 适用场景 |
|------|----------|---------|
| **祭司/统治者独白** | `Speech` | 统治者发表节日演讲——audience 可欢呼/沉默 |
| **祭司↔群众应答** | `Ritual` | "愿神祝福你们！" → 群众："也祝福你！" |
| **多人共祭** | `Ritual`（多个 speaker 轮流） | 祭司A→祭司B→群众代表→闭合 |
| **竞技开场** | `Moderated`（裁判主持） | "第一场比赛——剑术！选手入场！" |

## 二、节日文本片段

### 2.1 需要新增的文本片段类别

语言表达 003 的文本生成引擎基于片段模型（~430 片段）。以下类别需要新增：

| 类别 | 片段数（估计） | 触发上下文 |
|------|--------------|-----------|
| `festival_greeting` | ~15 | NPC 在节日期间对玩家/其他 NPC 的问候（"收获节快乐！"） |
| `festival_invitation` | ~8 | NPC 邀请玩家参加节日 |
| `festival_reminiscence` | ~10 | NPC 回忆去年的节日 |
| `festival_blessing` | ~12 | NPC 在宗教节日中的祝福语 |
| `festival_toast` | ~10 | NPC 在宴饮中的祝酒词 |
| `festival_competitive` | ~8 | NPC 在竞技前/中/后的发言 |
| `festival_complaint` | ~5 | 不参加节日的 NPC 的抱怨（"太吵闹了"） |

### 2.2 片段的文化调节

片段选择由 `FragmentCondition::CultureParam` 过滤——已在语言表达 003 中定义：

```rust
FragmentCondition::CultureParam { param: String, range: (f32, f32) }
```

示例：
- `directness > 0.7` → 节日祝福更直白："祝你收获丰盛！"
- `directness < 0.3` → 节日祝福更委婉："愿今年的果实不辜负你的辛劳。"
- `religiosity > 0.7` → 节日问候包含神祇名："愿火神保佑你的收获。"
- `religiosity < 0.3` → 节日问候世俗："收获节快乐！"

## 三、公开演讲——TurnMode::Speech

### 3.1 演讲者

| 演讲者 | 演讲内容 | 触发条件 |
|--------|---------|---------|
| 统治者 | 节日开幕词、年度总结、宣布来年目标 | 统治者主持的节日 |
| 祭司 | 布道、神圣宣讲、告诫信徒 | 宗教节日的核心仪式 |
| 竞技冠军 | 获奖感言 | 竞技结束后 |
| 长者 | 讲述社区历史 | 纪念性节日 |
| 玩家 | 任何内容 | 如果玩家获得演讲权 |

### 3.2 听众反应

`TurnMode::Speech` 支持 `allowed_interruptions: [Cheer, Boo, WalkOut]`。

节日中的听众反应被 `SocialField` 放大：
- 听众 `approval > 0.7` → 每 30 秒一次 Cheer（SocialField 情绪传染 → 更多人 Cheer）
- 听众 `approval < 0.2` → Boo 或 WalkOut（如果统治者权威低 → 合法性危机触发）
- `EmotionalTone::Ecstatic` → 听众反应频率 ×2

### 4.1 TopicCategory 新增


### 4.2 NPC 自发节日对话的触发

```
宣告期：
  - 两个 NPC 在社交距离内 → 概率提及即将到来的节日
  - 信息传播系统的 Oral 通道携带节日信息
  
庆祝期：
  - TopicCategory::Festival 的权重 ×3
  - NPC 的 ConversationInitiation 概率增加（节日=社交催化剂）
  
余波期：
  - 对话主题回归正常，但"今年的XX节真不错/真糟糕"的概率持续一周
```

## 五、玩家与 NPC 的节日对话

### 5.1 玩家向 NPC 询问节日

```
玩家输入（预设选项）：
  "这是什么节日？"
  "你们每年都这样庆祝吗？"
  "我可以参加吗？"
  "这节日是怎么来的？"

→ InputInterpreter L1 关键词匹配
→ 查询 FestivalQuery → 返回结构化数据
→ 文本生成引擎 用结构化数据 + 片段 + NPC 人格滤镜 → 生成回答
```

### 5.2 节日知识作为对话锚点

NPC 的节日知识来自 `EventMemory`——他们记得的节日经历：

```
NPC: "这是我第 47 个收获节啦。最好的那年是我 20 岁那年——那年的南瓜有车轮那么大！"
     ↑ 该 NPC 的 EventMemory 中确实有一条"收获节·47年前·特大南瓜"
```

这种对话通过 `EventMemory` + `ConversationTopic::Festival` 自然涌现——不需要节日系统的特殊逻辑。
