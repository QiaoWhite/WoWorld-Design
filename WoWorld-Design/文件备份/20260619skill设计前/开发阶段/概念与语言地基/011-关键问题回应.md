## 原始问题

> "当 NPC 发现自己的儿子进入铁匠铺工作，自己不用再给他钱时，他多了一条记忆'我的儿子去铁匠铺工作，我不用给他钱'，那么这个记忆中的词汇'儿子''铁匠铺''钱'，还有从属关系'我的'，还有前后的因果关系，这些都是哪里来的，怎么构成的？NPC 的思考和对话同语言有关的侧面，到底该如何实现？"

---

## 回应

### 词汇的来源

| 词汇 | 来源 | 机制 |
|------|------|------|
| "儿子" | 关系系统 + 文化概念空间 | `EntityIndex::relationship_strength(father, son)` = 亲子关系 → classify_pattern 产出 `ConceptLocalId::SON` → 中原语 lexicon 映射 → "儿子" |
| "铁匠铺" | 世界数据 + 概念组合 | `LocationId(blacksmith_shop)` → 其功能标签 = BLACKSMITH + PLACE → 中原语 morphology 组合规则 → "铁匠" + "铺" = "铁匠铺" |
| "钱" | 经济系统 + 概念空间 | Extract(wage) 边的 resource_type = Currency → 概念 `MONEY` → 中原语 lexicon → "钱" |
| "我的"（从属关系） | 关系系统 | 说话者 perspective = 父亲 → `Possession(father_id)` modifier 附加到 SON 概念上 |
| 因果关系 | 演绎推理 | `deductive_chain()`: 事实"儿子有工作" → MentalModel₁"有工作→有收入" → MentalModel₂"有收入→支持义务解除" → 结论"不用给钱了" |

### 记忆的构成——端到端

```
① 地面真相: PowerTopology 中 Contract+Extract(labor)+Extract(wage) 边创建

② 集成层检测 → compute_pattern_signature(edges) → PatternSignature(0x3F7A...)

③ 父亲感知（社交关联→间接感知）:
   effective_concept_space(父亲) → 中原文化概念空间
   classify_pattern(sig, space) → [(EMPLOYMENT, 0.95), (APPRENTICESHIP, 0.30)]

④ memory_encode_with_deduction:
   premises: [ConceptFact { 儿子, 在铁匠铺 }, ConceptFact { 铁匠铺主, 与儿子, 某种约定 }]
   deductive_chain → "儿子有工作" → "儿子有收入" → "不用给儿子钱了"
   inference_conclusion: { concept: FINANCIAL_SUPPORT_CEASE, confidence: 0.67 }

⑤ CompactConceptEncoding::partial(sig, matches, salience=0.8, zhongyuan, v3)
   → 写入 EventMemory

⑥ 惰性渲染（当父亲对玩家说话时）:
   Utterance { concepts: [SON, EMPLOYMENT, BLACKSMITH_SHOP, FINANCIAL_SUPPORT_CEASE] }
   → TextGenerator::render_utterance()
   → "我的儿子去铁匠铺工作了，我不用给他钱了"
```

### NPC 思考和对话的实现

**思考** = 结构化概念操作。NPC 的"内心"以 ConceptFact + MentalModel + 推理链的形式运作。不是自然语言字符串——是概念图。

**对话** = Utterance（结构化话语）→ TextGenerator（概念→文本渲染）。不是直接拼字符串——是从 `UtteranceConcept` 列表通过片段组合 + 滤镜管道生成自然语言。

**语言表达相关的侧面**：
- 语言 proficiency 决定"能说多好"——从 LISTEN/SPEAK 原子日志涌现
- 文化概念空间决定"怎么理解世界"——从 birth_culture + residences 加权派生
- 推理系统决定"能不能想明白"——归纳+类比+演绎三模式
- 概念→语言渲染决定"说出来是什么样"——TextGenerator 三模式统一管道

## 补充问题：不同语言的玩家如何体验？

> "如果玩家是英语玩家，他最初接触到的文明是俄语模拟文明，那岂不是开头就要学俄语？"

**回应**：玩家 UI 语言独立于角色语言。玩家角色出身于俄语模拟文明 → 角色母语 = 该文明的语言（proficiency 1.0）。UI 语言 = 英语（玩家设置）。角色完全理解周围人说的话（因为那是他的母语）→ 屏幕显示完整英语文本（UI 渲染翻译）。角色旅行到英语模拟文明时 → 角色听不懂（proficiency 0.0）→ 屏幕显示"[这个人说了一种你从未听过的语言]"。

**结论**：玩家不需要学任何语言。语言多样性通过 proficiency 系统体验——角色的语言障碍，不是玩家的语言障碍。

## 补充问题：NPC 能理解逻辑、培养理性思维吗？

**回应**：能。详见 [[005-推理系统|005-推理系统]]。
- **归纳推理**: `try_induce_pattern()` — 从记忆中提取共现模式
- **类比推理**: `creative_leap()` — 跨 domain 结构模板匹配
- **演绎推理**: `deductive_chain()` — MentalModel 链式应用
- **贝叶斯更新**: `micro_digest()` — 证据驱动信念更新
- **理性调控**: `deliberate_reasoning_override()` — 反思型 NPC 可以抑制情绪
- **智慧**: `wisdom_effective()` — 从反思+经验+平衡+年龄派生
