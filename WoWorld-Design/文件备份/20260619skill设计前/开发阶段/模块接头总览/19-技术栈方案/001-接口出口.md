# 技术栈方案 — 接口出口

> **最后验证**: 2026-06-18 | **源文档版本**: 技术栈 v4.0 (CHG-038 审计升级) | **CHG-033 已纳入**
>
> 技术栈方案是所有21个业务模块的"宪法"——定义全局架构、性能预算、技术决策。本模块不定义 trait/struct, 但设定所有模块必须遵守的约束。

## v4.0 权威技术蓝图
- **概念**: 经技术栈全量审计 (CHG-038) 升级后的 v4.0 技术方案. 前身: v1.0(001-005) → v2.0(006-015) → v3.0(016-018)
- **Owner**: [[../../../技术栈方案/001-WoWorld正式技术栈方案v3|技术栈方案v3]]
- **消费者**: 全部21个模块 — 所有模块的技术决策须与此对齐
- **关键约束**:
  - 引擎: Godot 4.6 LTS | 模拟语言: Rust (stable 1.80+) — GDExtension
  - 体素: 自建 Transvoxel (Rust侧) → Godot ArrayMesh. 垂直稀疏Chunk + Clipmap LOD
  - 海洋: Gerstner程序化波, OceanProvider trait 预留FFT升级
  - 天空: 混合体积云 — 2D impostor + 3D体积密度场
  - 画面: 3D低多边形 + flat/cel渲染 (TABS风格). 512²面部图集驱动表情
  - NPC AI: GOAP(9%) + 概率行为树(90%) + 可选LLM(1%). SoA + rayon并行
  - 动画: CPU骨骼矩阵(≤0.5ms) → GPU skinning(双骨). 9层栈 + 38姿态 + 15轨迹
  - 数据库: LMDB — 流式Chunk存储. 双层记忆架构
  - Mod: TOML数据驱动 — 调节涌现乘数, 不编写行为脚本

## 237 + 126 TDI (技术决策项)
- **概念**: 237 已有技术决策项 (TDI) + 126 审计新增. 每个 TDI 记录决策/替代方案/理由/成本
- **Owner**: 技术栈方案 + CHG-038审计
- **消费者**: 全部模块 — 任何技术决策须查阅相关 TDI 或新增 TDI
- **CHG-033 更新**: TDI-032/045/202 物理方案已取代为 Rust 空间查询

## 性能预算 (峰值互斥)
- **概念**: 帧预算 16.7ms (60fps). VRAM ~4.2GB/6GB, 内存 ~1.4GB/32GB
- **Owner**: 技术栈方案 §性能预算
- **消费者**: 全部模块 — 各子系统须在预算内运行. 峰值互斥原则 (不是所有系统同时跑满)
- **关键预算**:
  - CPU动画: ≤0.5ms (rayon 6核, 1000 L1 NPC)
  - Rust模拟总预算: ≤4ms/帧
  - 降水粒子: ≤0.4ms GPU
  - 音频CPU: ≤0.17ms (CHG-030)
  - 天气CPU: ≤0.25ms/帧 (含全部轮询)
  - 面部纹理: 512² RGBA8. 所有角色纹理 VRAM <5MB
  - 硬件目标: GTX 1660 SUPER 6GB VRAM

## LOD 统一架构 (§二十一)
- **概念**: 分层模拟 L1/L2/L3/L4 — 统一LOD调度框架. 8级体素LOD, 3km切换Signed Heightfield
- **Owner**: 技术栈方案 §二十一
- **消费者**: 全部模块 — 各模块按LOD层级分配精度/频率
  - L1: 全精度(玩家15m内) — 每帧
  - L2: 中精度(模拟层) — 1-10s
  - L3: 低精度(远处聚合) — 30-60s
  - L4: 存档级(全局) — 按需/惰性
- **跨L层**: 音频 L1全入/L2强度门控/L3≥120dB; 感知 L1全入/L2主要实体/L3聚合; NPC L1全AI/L2简化/L3统计

## 风险矩阵
- **概念**: 技术风险登记 — 高/中/低三级. 每个风险有缓解方案
- **Owner**: 技术栈方案 §风险
- **消费者**: 全部模块 — 涉及高风险技术的模块须提供缓解证明
- **已知高风险项**: 体素LOD切换接缝 / 100k NPC SoA并行 / GDExtension Rust边界开销 / LMDB并发
