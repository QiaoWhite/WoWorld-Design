## 一、动画 CPU 详细计时

**相位偏移**：L1 NPC 更新周期 0.2s = 12 帧。每帧 ~83 NPC 全更新（pose lookup + interpolation + chain compute）。其余 ~917 NPC 仅层级链乘。

| 操作 | 规模 | 单核耗时 | rayon（6核） |
|------|------|---------|-------------|
| 全更新：pose lookup + 插值 | 83 × 33 bones × slerp | 72µs | — |
| 全更新：层级链乘 | 83 × 33 × Mat4 mul(10ns) | 27µs | — |
| 跟随：层级链乘 | 917 × 33 × 6ns | 180µs | → 30µs |
| 情绪偏移 | 1000 × 33 × 3ns | 100µs | — |
| 脚部 IK | 1000 × 2 feet × 80ns | 160µs | → 27µs |
| 脊柱桥接 | 1000 × 3 spine × 20ns + 查表 | 60µs | → 10µs |
| 写入输出 | 2.1MB write | 100µs | — |
| **总计** | | **699µs** | **~500µs** |

### 重帧

| 场景 | 额外负载 | 帧影响 | 能否 60fps |
|------|---------|--------|-----------|
| 100 NPC 进入战斗 | 100×过渡=等效250全更新 | +160µs | ✅ |
| 火灾（NPC 紧急寻路） | A* 峰值跨3帧分摊 | ~+0.8ms/帧 | ✅ |
| 加载首帧 | 骨骼 LOD 升级 200ms 松弛 | ~+0.5ms 首帧 | ✅ 瞬间 |
| 所有峰值同时 | 理论上 9.7ms——但互斥 | R=7.2ms | ✅ |

**GOAP 高峰 + 战斗高峰互斥**——战斗中 NPC 不换人生目标。
**感官全负荷 + 战斗高峰互斥**——战斗时注意力聚焦，射线减半。

## 三、LOD 体系

### 骨骼 LOD

| | L0 | L1 | L2 | L3 | L4 |
|---|----|----|----|----|----|
| 骨骼数 | 35 | 33 | 28 | 15 | 0 |
| 距离 | <10m | 10-50m | 50-150m | 150-500m | 500m+ |
| 面骨 | 全（jaw+brow×2+eye×2） | jaw only | 无 | 无 | — |
| 手指 | 拇指+食指 | 拇指+食指 | 无 | 无 | — |
| 挂载 | 全部5 | 3（weapon+shield+back） | 2 | 1 | 0 |

LOD 升级过渡：新骨从父骨世界位置开始，指数松弛 200ms。在 48m+ 距离不可见。
LOD 降级：多余骨骼停更新。Mesh LOD 切换用 alpha dither（随机像素丢弃）200ms。

### Mesh LOD

| | L0 | L1 | L2 | L3 | L4 |
|---|----|----|----|----|----|
| 面数 | 1500 | 800 | 300 | 80 | 0 |
| 角色占比 | ~5% | ~60% | ~20% | ~15% | ~数百万 |
| 三角形/帧 | 75K | 480K | 60K | 12K | 0 |

总 NPC 三角形 ≈ 627K/帧。加上地形(500K)+建筑(300K)+植被(200K) ≈ 1.6M/帧。GTX 1660S 可处理 ~1000M/帧 = **0.16% 利用。**

## 六、可扩展性

| 场景 | 分析 | 是否可行 |
|------|------|---------|
| 2000 L1（从 1000） | CPU 动画 0.5→1.0ms，GPU 0.7→1.4ms | ⚠️ 可，LOD 需更激进 |
| 10000 总 NPC | L2+主导，L1 仍 ≤1000。人均成本降 10× | ✅ |
| 500 NPC 同屏战斗 | 战斗过渡峰值 +0.3ms，帧 | ✅ ~58fps 过渡中 |
| GPU compute 骨矩阵（Phase 3+） | CPU 动画 0.5ms→0.25ms | ✅ 远期优化 |

### 优化预留

1. **f16 骨矩阵** → TextureBuffer 减半，GPU 带宽减半。TABS 美学下 ~2% 质量损失
2. **层次化 A*** → 粗格 2m + 细格 0.5m near goal。A* 成本 -80%
3. **SIMD 射线批处理** → 4 条射线同时 SSE。感官射线 -30%
4. **GPU compute 骨矩阵**（Phase 3+） → 链乘移到 compute shader。CPU -0.25ms
5. **自适应射线预算** → L0:3 rays, L1:8, L2a:3。省 ~30% 感官射线
