# 002 — 物品分类与 ID 体系

## 一、统一 ID 命名空间

### 1.1 ItemDefId 编码方案

```
┌─────────────────────────────────────────────────────────┐
│  ItemDefId: u64                                         │
│  ┌──────────────┬────────────────────────────────────┐  │
│  │ 8-bit 类别    │ 56-bit 定义索引                     │  │
│  │ category      │ def_index                          │  │
│  └──────────────┴────────────────────────────────────┘  │
│                                                          │
│  0x00 – 0x7F: 核心游戏类别（128 个，当前约 57 个在用）   │
│  0x80 – 0xFF: Mod 命名空间（128 个）                     │
└─────────────────────────────────────────────────────────┘
```

### 1.2 ItemCategory 分类树

```rust
pub enum ItemCategory {
    // === 0x00-0x0F: 装备 ===
    Weapon,             // 0x00 — 武器（近战/远程/法器——见 004）
    Armor,              // 0x01 — 防具（头盔/胸甲/腿甲/靴/盾）
    Accessory,          // 0x02 — 饰品（戒指/项链/手镯/耳环/头环）

    // === 0x10-0x1F: 消耗品 ===
    Consumable,         // 0x10 — 通用消耗品
        Potion,         // 0x11 — 药水（生命/魔力/效果/灵药）
        Food,           // 0x12 — 食物（果实/肉类/面包/菜肴）
        Scroll,         // 0x13 — 魔法卷轴（一次性施法）
        Ammunition,     // 0x14 — 弹药（箭/弩矢/投掷物）

    // === 0x20-0x2F: 材料 ===
    Material,           // 0x20 — 通用材料
        MineralOre,     // 0x21 — 矿物（煤炭/铜/铁/锡/银/金/秘银/山铜/陨铁）
        Gemstone,       // 0x22 — 宝石（水晶/钻石/红宝石/蓝宝石/翡翠/魔法水晶）
        StoneMat,       // 0x23 — 石料（花岗岩/玄武岩/大理石/砂岩/石灰岩/板岩）
        WoodMat,        // 0x24 — 木材（橡/榉/枫/桦/松/云杉/杉/竹/巨木/魔法木）
        FiberMat,       // 0x25 — 纤维（亚麻/麻/棉/荨麻/藤/芦苇）
        LeatherMat,     // 0x26 — 皮革/骨/角/壳
        LiquidMat,      // 0x27 — 液体（淡水/盐水/油/魔法泉/树脂）
        OrganicMat,     // 0x28 — 有机材料（染料/药草/香料/菌类）
        MagicMat,       // 0x29 — 魔法材料（魔力水晶/元素精华/龙遗物/神遗物/灵质/古遗物）
        SoilMat,        // 0x2A — 土沙（黏土/沙/腐殖土/泥炭/砾石/壤土）

    // === 0x30-0x3F: 工具 ===
    Tool,               // 0x30 — 通用工具
        Pickaxe,        // 0x31 — 镐
        Axe,            // 0x32 — 斧（工具——与武器斧不同 ItemDefId）
        Shovel,         // 0x33 — 铲
        Hammer,         // 0x34 — 锤
        FishingRod,     // 0x35 — 钓竿
        Sickle,         // 0x36 — 镰
        CarvingKnife,   // 0x37 — 刻刀（导魔刻刀等）
        AlchemyKit,     // 0x38 — 炼金套件

    // === 0x40-0x4F: 容器 ===
    Container,          // 0x40 — 通用容器
        Backpack,       // 0x41 — 背包（装备在背槽）
        Pouch,          // 0x42 — 腰包/挎包（装备在腰/肩槽）
        HandCarry,      // 0x43 — 手提容器（麻袋/水桶/工具箱）
        FurnitureChest, // 0x44 — 家具容器（箱子/桶/柜/书架）

    // === 0x50-0x5F: 特殊 ===
    Currency,           // 0x50 — 货币（铜币/银币/金币——但通常走 CharacterWallet）
    QuestItem,          // 0x51 — 任务物品（不掉落/不交易/不消耗）
    Blueprint,          // 0x52 — 蓝图（建造计划书）
    Book,               // 0x53 — 书籍/日记/魔导书
    FurnitureItem,      // 0x54 — 家具（作为物品形态——放置后变成 Furniture 实体）
    KeyItem,            // 0x55 — 钥匙/通行证

    // === 0x60-0x6F: 魔法物品 ===
    MagicItem,          // 0x60 — 通用魔法物品
        EnchantRune,    // 0x61 — 附魔符文（卡槽附魔用——消费品）
        MagicImplement, // 0x62 — 法器（法杖/魔杖/宝珠/魔导书）
        MagicConstruct, // 0x63 — 魔法构造体核心

    // === 0x70-0x7F: 保留 ===
    // 为经济系统/载具系统/通信系统预留
}
```

### 1.3 子类别编码（def_index 内部结构）

对于大类下的子类，使用 def_index 的高 8-bit 编码子类：

```rust
impl ItemDefId {
    /// 构建 ItemDefId
    pub fn new(category: ItemCategory, sub_category: u8, instance: u48) -> Self {
        let cat = category as u64;
        let sub = (sub_category as u64) << 48;
        let inst = instance as u64;
        ItemDefId((cat << 56) | sub | inst)
    }

    pub fn sub_category(&self) -> u8 {
        ((self.0 >> 48) & 0xFF) as u8
    }
}
```

铁矿 = ItemDefId::new(MineralOre, IRON_SUB, 0) = 0x21_01_000000000000
铜矿 = ItemDefId::new(MineralOre, COPPER_SUB, 0) = 0x21_02_000000000000

## 二、现有 ID 映射

### 2.1 World Gen resource_type → ItemDefId

World Gen [[../世界生成/003-资源与矿产|003-资源与矿产]] 定义了数值 ID（20=Coal, 21=Copper...45=Magic Crystal）。映射规则：

```
resource_type (World Gen) → ItemDefId

20=Coal      → MineralOre.COAL    (0x21_00_000...)
21=Copper    → MineralOre.COPPER  (0x21_01_000...)
22=Iron      → MineralOre.IRON    (0x21_02_000...)
23=Tin       → MineralOre.TIN     (0x21_03_000...)
24=Silver    → MineralOre.SILVER  (0x21_04_000...)
25=Gold      → MineralOre.GOLD    (0x21_05_000...)
26=Mithril   → MineralOre.MITHRIL (0x21_06_000...)
27=Orichalcum→ MineralOre.ORICHALCUM (0x21_07_000...)
28=Meteoric  → MineralOre.METEORIC (0x21_08_000...)
30=Sulfur    → MineralOre.SULFUR  (0x21_09_000...)
31=Saltpeter → MineralOre.SALTPETER (0x21_0A_000...)
32=Salt      → MineralOre.SALT    (0x21_0B_000...)
33=Limestone → SoilMat.LIMESTONE  (0x2A_00_000...)

40=Crystal   → Gemstone.CRYSTAL   (0x22_00_000...)
41=Diamond   → Gemstone.DIAMOND   (0x22_01_000...)
42=Ruby      → Gemstone.RUBY      (0x22_02_000...)
43=Sapphire  → Gemstone.SAPPHIRE  (0x22_03_000...)
44=Jade      → Gemstone.JADE      (0x22_04_000...)
45=MagicCrystal → MagicMat.MAGIC_CRYSTAL (0x29_00_000...)
```

映射由 `ItemRegistry` 维护，World Gen 生成资源时通过查表转换为 ItemDefId，然后为每个矿块分配 ItemEntId。

[[../魔法/05-器物层/015-魔法物品与法器|魔法物品与法器]] 中的 MagicItemInfo 映射：

## 三、ID 分配策略

### 3.1 核心游戏（0x00-0x7F）分配

- 0x00-0x5F：当前已使用（约 57 个子类）
- 0x60-0x6F：保留给魔法物品扩展
- 0x70-0x7F：保留给未来系统（经济/载具/通信）

### 3.2 Mod 命名空间（0x80-0xFF）

Mod 作者可自由定义 128 个额外类别。物品系统不验证 Mod ID——仅保证核心 ID 不被 Mod 覆盖。

### 3.3 Instance 分配（ItemEntId）

```rust
pub struct ItemEntIdAllocator {
    next_serial: AtomicU64,  // 全局递增，线程安全
}

impl ItemEntIdAllocator {
    pub fn allocate(&self) -> ItemEntId {
        ItemEntId(self.next_serial.fetch_add(1, Ordering::Relaxed))
    }
}
```

简单全局自增。56-bit 每存档够用 2,300 年（每秒 100 万物品），不回收。

## 四、ID 序列化

```rust
// bincode 编码：ItemDefId 和 ItemEntId 都是 u64——零成本序列化
// 存档中存储格式：
// [u64 ItemDefId] [u64 ItemEntId] [ItemState bytes...]
// LMDB key: ItemEntId (8 bytes)
// LMDB value: (ItemDefId, ItemState) bincode blob
```

性能：100K NPC × 平均 80 物品 = 8,000,000 物品。8M × (8+8+平均200 bytes State) ≈ 1.7 GB 存档物品数据。在 32GB 内存预算内无压力。

## 五、与 CHG-013 的兼容性

- 不修改 CHG-013 建立的任何契约（Physiology/Vitals、魔力刻、部位伤害等）
- ItemDefId 是**新增**命名空间——不影响现有 u32 刻/元素顺序/群系参数场等约定
- 旧 ID 类型（MaterialId/MagicItemId/resource_type）通过映射表桥接——渐进迁移，不强制 break
