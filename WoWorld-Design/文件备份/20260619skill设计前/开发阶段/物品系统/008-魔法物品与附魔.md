# 008 — 魔法物品与附魔

## 一、两条附魔路径

WoWorld 的附魔不是单一系统——是两条有根本不同设计目的的路径：

| | 卡槽附魔 | 直接附魔 |
|---|---|---|
| **本质** | 临时改装 | 永久重生 |
| **可逆性** | 随时替换旧符文 | 不可撤销 |
| **强度** | 中等（标准符文效果） | 强（同符文类型 ×1.5-2.0） |
| **使用者** | 所有人——家常便饭 | 少数——一生可能只有一件 |
| **经济角色** | 符文=消费品，驱动符文工匠行业 | 附魔物品=不朽资产，传承/拍卖/遗产 |
| **战术角色** | 战前调整（火→冰符文对付冰龙） | 不退出的承诺——这件武器就是为此而生 |
| **创作角色** | 工匠职业的日常收入 | 大师一生可能只做数件——名字被历史记住 |
| **History** | 不留痕 | 附魔师名字永久记录在物品 provenance 里 |

## 二、卡槽附魔——日常经济路径

### 2.1 卡槽模型

武器的卡槽是装配框架中的一个组件槽（见 [[001-物品系统总纲|001 §三]]）：

```rust
// 武器装配中的卡槽
ComponentSlot {
    slot_type: CombatSlot::EnchantmentSlot as u16,
    occupant: Option<ItemEntId>,  // 当前插入的符文（符文本身也是物品！）
    params: ItemParams::default(),
}

// 符文是物品——有自己的 ItemDefId 和 ItemEntId
// ItemDefId.category = MagicItem.EnchantRune
// 符文属性:
struct EnchantRuneProps {
    element: Element,              // 火焰/冰霜/雷电/血/...
    effect: RuneEffect,
    uses_remaining: u32,           // 充能次数（0 = 耗尽）
    max_uses: u32,
    rechargeable: bool,            // 能否重新充能
    quality: Quality,              // 符文品质——影响效果强度
}
```

### 2.2 符文作为消耗品经济

```
符文工匠制作符文(消耗魔力水晶+元素精华+工匠技能)
  → 卖给冒险者
  → 冒险者插入卡槽 → 使用
  → 充能耗尽 → 符文变成"耗尽的符文"(ItemDefId 不同——无效果，可回收微量材料)
  → 冒险者回来买新的
  → 符文工匠持续有生意
```

符文是**可替换的战术选择**——不是永久绑定。玩家面对火元素→换冰符文→打完换回日常配置。NPC 同理。

### 2.3 符文类型

| 符文 | 元素 | 效果 |
|------|------|------|
| 火焰符文 | 火 | +火伤害，命中点燃 |
| 冰霜符文 | 水(冰) | +冰伤害，命中减速 |
| 雷电符文 | 雷 | 雷伤害，概率麻痹 |
| 闪电符文 | 电 | 电伤害，连锁跳跃 |
| 吸血符文 | 血 | 伤害的 X% 回复生命 |
| 金刚符文 | 金 | +护甲穿透 |
| 生命符文 | 木 | 缓慢自我修复耐久 |
| 风灵符文 | 风 | +攻速 +移速 |
| 大地符文 | 土 | +格挡效率，击退抗性 |
| 灵元素符文 | 灵 | +魔力恢复速度（法器专用） |

**元素约束**：符文元素不能与武器材质元素排斥（铁+火符文=没事，水元素材料+火符文=效果减半）。见 [[../魔法/02-机制层/003-元素交互|Magic 003]]。

## 三、直接附魔——历史锚点路径

### 3.1 本质

直接附魔不是往武器上"装"东西——是**改写武器的本质**。一旦完成，这把武器就永久获得了附魔属性。卡槽仍可正常使用——两者共存。

### 3.2 高门槛

```rust
struct DirectEnchantmentRequirements {
    min_enchanter_skill: f32,        // 需要附魔技能 ≥ 90
    required_catalyst: ItemDefId,    // 需要稀有以上触媒
    failure_rate: f32,               // 失败率（技能越高越低，最低 5%）
    failure_cost: FailureCost,       // 失败→损失全部材料
    duration_game_hours: u32,        // 仪式需要数小时
    location_required: Option<LocationRequirement>,  // 可能需要特定地点
}
```

**失败率表**：

| 附魔师技能 | 失败率 |
|-----------|--------|
| 90-92 | 40% |
| 93-95 | 25% |
| 96-98 | 12% |
| 99-100 | 5% |

### 3.3 附魔效果强度

```rust
// 同一个符文效果——直接附魔比卡槽强
fn enchantment_power(rune_effect: &RuneEffect, path: EnchantPath) -> f32 {
    match path {
        EnchantPath::CardSlot => rune_effect.base_power,
        EnchantPath::Direct   => rune_effect.base_power * 1.5,  // 强 50%
    }
}
```

### 3.4 Provenance 记录

```rust
// 直接附魔完成后——History 系统记录
history::record_direct_enchantment(
    item: ItemEntId,
    enchanter: NpcId,
    enchantment_name: String,  // "希格尔德的屠龙誓言"
    timestamp: GameTimestamp,
);
```

此后这个附魔名称成为物品的一部分——任何识货的 NPC/玩家鉴定物品时都会看到。这就是"具名物品"的诞生。

## 四、法器——Magic 模块注册的装配组件

法器是 Magic 模块通过装配框架注册的组件类型（见 [[001-物品系统总纲|001 §三]]）：

```rust
// Magic 模块注册的 slot_type:
// 0x1000: ImplementBody (法杖杆/魔杖杆)
// 0x1001: ImplementCore (魔法核心材质)
// 0x1002: ImplementFocus (顶端水晶/宝珠)
// 0x1003: GrimoireBinding (魔导书装订)
// 0x1004: GrimoirePages  (魔导书内页)

// 法杖装配（混种——可近战打击）:
// [ImplementBody(黑檀,1.8m)] → [ImplementCore(龙晶,纯度0.95)] → [Focus(蓝宝石)]
//    + [StrikingHead(铁箍)] ← 可选打击部——有了它就是 Combat+Magic 双用途

// 魔杖装配（纯法器——不可近战）:
// [ImplementBody(冬青木,0.3m)] → [ImplementCore(凤凰羽毛,纯度0.88)]

// 宝珠装配（纯法器——无杆体，手动持有或漂浮施法）:
// [ImplementFocus(水晶球,直径10cm,透明度0.98)] → [ImplementCore(星辉精华,纯度0.99)]

// 魔导书装配:
// [GrimoireBinding(龙皮封面)] → [GrimoirePages(导魔羊皮纸,页数200)]
```

法器的魔力属性由 Magic 模块的 `MagicItemInfo` 派生：

```rust
// Magic 模块查询物品系统
fn get_magic_props(item: &AssembledItem) -> MagicItemInfo {
    // 遍历装配树 → 找到 ImplementCore → 读出材质属性
    // → 计算魔力容量/放大系数/元素亲和
    // 如果找到 StrikingHead → 同时注册为 Combat 可用武器（混种物品）
}
```

> ⚠️ **待对接**：法器组件类型已在物品系统注册，但 Combat 005 尚未消费法器的战斗动作（引导/屏障/增强），Magic 模块的法力计算（魔力放大/元素亲和/施法速度修正）待完全对接装配树查询。物品系统已预留完整接口——见 [[009-跨模块接口与数据合同|009 §三]]。

## 五、魔法物品融入统一 ID 空间

所有魔法物品都映射到 ItemCategory 的魔法子类（见 [[002-物品分类与ID体系|002]]）：

```
MagicItemInfo.id → ItemDefId(MagicItem 类别)

附魔武器  → Weapon(附魔属性挂在 ItemState.enchantments)
附魔防具  → Armor(同上)
附魔饰品  → Accessory(同上)
魔力药水  → Consumable.Potion(MANA)
魔法卷轴  → Consumable.Scroll(品质/法术绑定)
法杖      → MagicItem.MagicImplement.STAFF
魔杖      → MagicItem.MagicImplement.WAND
宝珠      → MagicItem.MagicImplement.ORB
符文      → MagicItem.EnchantRune(元素/效果类型)
构造体核心 → MagicItem.MagicConstruct
```

## 六、性能

- 符文 = 普通 ItemEntId——在库存/容器/卡槽中存储，无特殊处理
- 直接附魔 = ItemState.enchantments 中的一个条目——~16 bytes/附魔
- 附魔效果查询 O(1)——从 ItemState 直接读
- 法器属性计算 = 遍历装配树（最多 ~5 个组件）——微秒级
