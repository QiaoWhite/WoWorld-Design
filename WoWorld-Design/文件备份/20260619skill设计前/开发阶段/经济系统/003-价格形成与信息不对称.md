# 003 — 价格形成与信息不对称

## 一、价格从交易中涌现——非公式计算

### 1.1 核心原则：价格是历史的统计，不是公式的结果

经济系统**不存在中央定价公式**。没有 `fn calculate_price(item: ItemDefId) -> u32` 这样的函数。价格是交易记录的副产品——市场的 `last_price` 是最近成交价的指数移动平均（EMA）。

```
没有公式计算"铁矿石该卖多少钱"
  ↓
供需双方各自挂单 (Bid / Ask)
  ↓
Bid ≥ Ask → 撮合成交（详见 002 §三）
  ↓
成交价写入 TradeRecord → trade_history 滑动窗口
  ↓
从 trade_history 计算 last_price (EMA) 和 price_volatility (变异系数)
  ↓
这个 last_price 就是"当前市场价格"——它只是过去交易的统计回声
```

**为什么不用公式？** 公式意味着开发者按下了某个价格。价格该由那些真实的、带着饥荒恐惧或丰收喜悦的 NPC，用他们手上的铜币和心中的欲望叫出来。公式是故事的终点，但 WoWorld 要的是故事的起点。

### 1.2 last_price — 市场价格的唯一来源

```rust
/// 最近成交价的指数移动平均——"市场价格"的唯一数据来源
/// 没有任何函数"计算"这个价格——它是交易执行的副作用
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct LastPriceSnapshot {
    /// EMA 平滑后的最近价格（铜币/单位）
    pub ema_price_copper: u32,

    /// 最近一笔裸成交价（未经平滑）
    pub raw_last_price_copper: u32,

    /// 最近一笔成交的游戏时间戳
    pub last_trade_at: GameTimestamp,

    /// 最近一笔成交的数量
    pub last_trade_quantity: u32,

    /// 最近一笔成交的模式（现货/远程/合同/中介）
    pub last_trade_mode: TransactionMode,

    /// EMA 平滑系数——默认 0.3
    /// 0.3 = 最近一笔占 30% 权重，历史 EMA 占 70%
    pub ema_alpha: f32,
}
```

```rust
/// 每笔成交后自动更新——这是价格涌现的唯一入口
fn update_last_price(
    market: &mut Market,
    trade: &TradeRecord,
) {
    let entry = market.last_price.get(&trade.item_def);
    let new_raw = trade.price_per_unit_copper;

    let new_ema = match entry {
        Some(old) => {
            let alpha = 0.3_f32;
            let old_ema = old.ema_price_copper as f32;
            let smoothed = alpha * new_raw as f32 + (1.0 - alpha) * old_ema;
            smoothed.round() as u32
        }
        None => {
            // 第一笔交易——EMA = 成交价本身
            new_raw
        }
    };

    market.last_price.insert(trade.item_def, LastPriceSnapshot {
        ema_price_copper: new_ema,
        raw_last_price_copper: new_raw,
        last_trade_at: trade.timestamp,
        last_trade_quantity: trade.quantity,
        last_trade_mode: trade.transaction_mode.clone(),
        ema_alpha: 0.3,
    });
}
```

**EMA 的选择**：不使用简单均价——因为 200 笔交易中最早的 1 笔可能是一年前的僵尸数据。EMA 让最近交易权重更高，价格对供需变化的响应更快（新矿发现→铁矿价格几天内反映，不是几个月）。

**边界情况**：
- 某物品在此 Market 从未有过交易 → `last_price` 中无键——不存在"默认价格"
- 第一笔交易 → EMA = 成交价本身（没有历史可以平滑）
- 长期无交易（超过 30 个游戏日）→ `last_price` 仍然保留但不触发新的更新。消费方（NPC 心智）知道这个价格"可能已过时"

### 1.3 长尾物品与无交易物品

```
常见物品（铁矿石/小麦/布料）：
  last_price 几乎实时更新——每日数十笔交易

稀有物品（龙鳞/贤者之石/国玺）：
  可能数月甚至数年没有交易
  last_price 上一次更新是三个月前
  → NPC 对该物品的"价格认知"停留在三个月前
  → 卖方很可能叫价远高于 last_price
  → 买方很可能出价远低于 last_price
  → 价差 > 30% → 薄市场不撮合（002 §3.3）
  → 双方只能走协商通道（议价×魅力×关系）
  → 稀有物品的"定价"本身就是一个社交事件
```

这就是**价格不是公式的自然结果**——稀有物品没有"应有价格"，每一次交易都是独一无二的谈判。历史系统会记住"那把传奇剑卖了一万铜币"，而这个故事比价格本身更有价值。

## 二、信息不对称——每个 NPC 知道的价不同

### 2.1 设计哲学：信息不对称不是 bug，是故事素材

在真实经济中——尤其是在中世纪奇幻世界——没有人知道所有市场的所有价格。WoWorld 把这个现实原样注入游戏。两个 NPC 对同一物品在同一市场的价格可以有完全不同的认知。这不是系统的缺陷——而是交易利润、套利机会、商队兴起、信息贩子职业涌现的土壤。

```
真实世界：中世纪商人花几天走到邻镇，发现铁矿石便宜 20% → 惊喜套利
WoWorld：  NPC 花几天走到邻镇，发现铁矿石便宜 20% → 惊喜套利
          两者的底层逻辑完全相同——信息不对称
```

### 2.2 PriceKnowledge — NPC 的价格认知片段

`NpcEconomicState.known_prices`（已在 001 §3.1 定义）是每个 NPC 对世界的**主观价格地图**。这个地图可以是残缺的、过时的、扭曲的——每个 NPC 活在一个独一无二的价格宇宙中。

```rust
/// NPC 对某一物品在某一市场的价格认知
/// 这是 NPC 认为的价格——不一定是真实价格
/// 信息不对称的全部奥秘在此结构之中
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct PriceKnowledge {
    /// NPC 认为的当前价格（铜币/单位）
    /// 可能比真实 last_price 高/低/完全离谱
    pub known_price_copper: u32,

    /// 得知此价格的时间
    pub learned_at: GameTimestamp,

    /// 价格信息来源——决定了信息的可靠程度
    pub source: PriceKnowledgeSource,

    /// 信息信心 0-1
    /// 1.0 = 亲眼所见，0.1 = 五手传闻，几乎不可信
    pub confidence: f32,

    /// 价格失真度 0-1
    /// 0 = 准确，1 = 完全失真（已不可用）
    pub distortion: f32,

    /// 信息传播跳数——从原始观察者到当前 NPC 经过了多少人
    /// 0 = 亲眼所见，3 = 经过三个人之口
    pub hops: u8,

    /// 获知此价格时 NPC 所处的位置（用于判断"我那时候在哪"）
    pub learned_location: LocationId,

    /// 此条知识最近一次被 NPC 使用的时间（做决策时刷新）
    /// 用于评估知识的"新鲜度"
    pub last_used_at: Option<GameTimestamp>,

    /// 此条知识已被刷新（确认/更新）的次数
    /// 多次确认 → NPC 对此信息的依赖度更高
    pub confirmation_count: u8,
}
```

### 2.3 信息来源——六种精度等级

```rust
/// 价格知识的来源——直接决定 confidence 的初始值
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub enum PriceKnowledgeSource {
    /// 亲手交易——自己作为卖/买方成交
    /// → confidence = 1.0（不需要再问了——我就是交易方）
    PersonalTransaction {
        trade_id: u64,
        my_side: OrderSide,
    },

    /// 亲眼观察——站在市场中看到别人交易
    /// → confidence = 0.95（亲眼所见，但可能有视觉误差/没看清数量）
    DirectObservation {
        market_id: MarketId,
        observed_trade_id: u64,
    },

    /// 听人说起——与另一 NPC 交谈中获知
    /// → confidence 初始 = 0.7（取决于信息源的可靠性 × 交谈者的智力/宜人性）
    Overheard {
        source_npc: NpcId,
        conversation_id: ConversationId,
        source_npc_reliability: f32,  // 0-1 此人在此事上的可信度
    },

    /// 传闻——道听途说，不确定从谁而来
    /// → confidence 初始 = 0.4（"听说城里铁矿跌了"）
    Rumor {
        rumor_summary: String,       // "有传言说..."
        rumor_source_type: RumorSourceType,
    },

    /// 政府公告——领主/行会发布的官方价格公告
    /// → confidence 初始 = 0.85（官方信息通常可靠，但政府可能操控）
    GovernmentBulletin {
        authority_id: MarketAuthorityId,
        bulletin_id: BulletinId,
        /// 公告可能故意扭曲（如为了压价征购）
        is_suspected_manipulated: bool,
    },

    /// 商人报告——专业信息贩子/商会发布的行情报告
    /// → confidence 初始 = 0.8（商人有记录的专业性，但可能夹带私货）
    MerchantReport {
        reporter_npc: NpcId,
        report_format: MerchantReportType,
        /// 报告是否有利益冲突（如报告者自己持有该物品的大量库存）
        potential_conflict: bool,
    },
}

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub enum RumorSourceType {
    TavernGossip,        // 酒馆流言——最不可靠
    TownSquareChat,      // 广场闲谈
    TravelerStory,       // 旅人故事
    Unknown,             // 完全不知道从哪来的
}

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub enum MerchantReportType {
    WrittenReport,       // 书面行情报告——较正式
    VerbalBriefing,      // 口头简报
    PriceLedger,         // 账本记录
}
```

**信心衰减公式**：

```rust
/// 初始信心 = f(source_type)，然后乘以传播衰减
fn initial_confidence(source: &PriceKnowledgeSource) -> f32 {
    match source {
        PriceKnowledgeSource::PersonalTransaction { .. } => 1.0,
        PriceKnowledgeSource::DirectObservation { .. } => 0.95,
        PriceKnowledgeSource::GovernmentBulletin { is_suspected_manipulated, .. } => {
            if *is_suspected_manipulated { 0.6 } else { 0.85 }
        }
        PriceKnowledgeSource::MerchantReport { potential_conflict, .. } => {
            if *potential_conflict { 0.65 } else { 0.8 }
        }
        PriceKnowledgeSource::Overheard { source_npc_reliability, .. } => {
            0.5 + source_npc_reliability * 0.3  // 范围 0.5-0.8
        }
        PriceKnowledgeSource::Rumor { rumor_source_type, .. } => {
            match rumor_source_type {
                RumorSourceType::TavernGossip => 0.3,
                RumorSourceType::TownSquareChat => 0.4,
                RumorSourceType::TravelerStory => 0.35,
                RumorSourceType::Unknown => 0.2,
            }
        }
    }
}

/// 传播衰减——每经一人之手，信心指数级衰减
/// hops 0 = 亲眼所见（信心不变）
/// hops 1 = 听人说起（initial × 0.8）
/// hops 3 = 三手信息（initial × 0.8³ = initial × 0.51）
/// hops 5+ = 几乎不可靠
fn apply_hop_decay(initial_conf: f32, hops: u8) -> f32 {
    let decay_factor = 0.8_f32.powi(hops as i32);
    (initial_conf * decay_factor * 100.0).round() / 100.0
}
```

**信心随时间衰减**：

```rust
/// 信息的"保鲜期"——超过一定天数后信心逐渐衰减
/// 30 天内信心不变，之后每 60 天衰减 10%
fn apply_time_decay(
    current_confidence: f32,
    learned_at: GameTimestamp,
    now: GameTimestamp,
) -> f32 {
    let days_elapsed = now.days_since(learned_at);
    if days_elapsed <= 30.0 {
        return current_confidence;
    }
    let periods = ((days_elapsed - 30.0) / 60.0).floor() as u32;
    let decay = 0.9_f32.powi(periods as i32);
    (current_confidence * decay * 100.0).round() / 100.0
}
```

### 2.4 价格知识通过语言表达系统传播

经济系统**不自己实现价格传播**——复用 [[../../语言表达/009-信息传播与非语言联动|语言表达 009]] 的五通道（Oral/Letter/Courier/Magical/Official）和五个失真算子（ScaleDistortion/DetailLoss/EmotionalAmplification/AttributionShift/Moralization）。

```
传播流程：

交易发生 → TradeRecord 写入 Market.trade_history
   ↓
Observer 决定"这个价格值得向别人提起"（基于价格变动幅度/自身性格）
   ↓
语言表达系统的信息传播通道接管：
   Oral:    "老张，铁矿跌到 48 铜了！"
   Letter:  商会信函——"本月铁价下行，建议暂缓采购"
   Courier: 信鸽——"东市缺粮速来"
   Magical: 魔法传讯——即时但门槛高
   Official: 领主公告——"本郡铁价官定 50 铜/块"
   ↓
失真算子随传播应用：
   经过 1 人：48 → "大概 50 左右吧"
   经过 3 人：48 → "听说跌到 30 了！"（夸大）
   经过 5 人：48 → "铁矿已经不值钱了"（变成叙事）
   ↓
接收方 NPC 生成一条 PriceKnowledge：
   source = Overheard { source_npc, conversation_id, reliability }
   hops = 发送方的 hops + 1
   confidence = initial_confidence(Overheard) × hop_decay × time_decay
   distortion = f(传播次数, 失真算子累积)
   ↓
存入 npc.eco_state.known_prices[(market_id, item_def_id)]
```

**NPC 更新已知价格的时机**：
- 亲手交易 → 立即更新（confidence=1.0, hops=0）
- 在市场亲自观察 → 立即更新（每次路过市场时随机抽样 1-3 笔交易）
- 对话中获知价格 → 对话结束时更新
- 收到信件/信鸽/公告 → 读取时更新
- 主动套利扫描 → 每 4 游戏小时扫描一次远方市场（仅满足 trader 条件的 NPC）

## 三、锚定效应——记忆中的第一个价格成为参考锚

### 3.1 锚定的心理模型

锚定效应（Anchoring Effect）是行为经济学最稳健的发现之一：人对一个数值的判断会被第一次接触到的数值"锚住"。在 WoWorld 中，每个 NPC 都有锚定效应——这是涌现"比上次便宜就觉得赚了"的心理前提。

```
NPC_A 首次在向阳镇看到铁矿石价格：50 铜/块
  → 存入 memory: "铁矿石 = 50 铜"
  → 同时存入 known_prices: (向阳镇, 铁矿石) → PriceKnowledge { known_price_copper: 50, ... }

三天后再来：
  → 当前真实价格：45 铜（铁矿增产）
  → NPC_A 内心判断："比上次便宜了 5 铜——赚了！"
  → 即使 45 铜仍高于隔壁镇的 40 铜，NPC_A 不知道也不在乎
  → 锚定效应驱动购买决策

三月后再来：
  → 当前真实价格：55 铜（战争导致减产）
  → NPC_A："贵了 5 铜——上次买便宜了，这次不划算"
  → 即使 55 铜已经是最低价（隔壁镇 70 铜），NPC_A 不知道
  → 锚定效应抑制购买——可能选择不买或去更远的地方
```

### 3.2 AnchorPoint——记忆中的价格锚

```rust
/// 价格锚点——NPC 记忆中某个物品的"标准价格"
/// 通常是最早几次接触到的价格
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct PriceAnchor {
    /// 锚定的价格（铜币/单位）
    pub anchor_price_copper: u32,

    /// 锚形成的时间
    pub formed_at: GameTimestamp,

    /// 锚形成时的市场
    pub formed_at_market: MarketId,

    /// 锚形成的方式
    pub formation_type: AnchorFormationType,

    /// 锚的强度 0-1——越高越难挣脱
    /// 从 conscientiousness 派生：高尽责→锚更强（更"固执"）
    pub anchor_strength: f32,

    /// 锚被挑战的次数（看到相反信息但不改变判断）
    /// 高次数 → 确认偏误的指标
    pub challenges_resisted: u32,

    /// NPC 是否已挣脱此锚
    /// 挣脱条件：intelligence × market_understanding 足够高
    pub is_broken: bool,
}

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub enum AnchorFormationType {
    /// 首次接触——对该物品的第一印象
    FirstEncounter,

    /// 首次交易——第一次买卖该物品
    FirstTransaction,

    /// 重大事件锚——在一次重大价格波动中形成（饥荒时的天价粮）
    SignificantEvent { event_summary: String },

    /// 教学锚——从师傅/长辈处学到的"正确价格"
    TaughtAnchor { teacher: NpcId },
}
```

### 3.3 锚的判断逻辑——NPC 如何参考锚

```rust
/// NPC 在评估当前价格时——相对于锚的判断
fn evaluate_price_against_anchor(
    current_price: u32,
    anchor: &PriceAnchor,
    cognition: &EconomicCognition,
    emotion: &EmotionState,
) -> PriceJudgment {
    if anchor.is_broken {
        // 挣脱锚的 NPC：理性判断——综合所有已知价格形成认知
        return rational_price_evaluation(current_price, anchor, cognition);
    }

    let deviation_pct = (current_price as f32 - anchor.anchor_price_copper as f32)
        / anchor.anchor_price_copper as f32;

    // 锚定下的情感反应
    match deviation_pct {
        d if d < -0.30 => PriceJudgment::UnbelievablyCheap,
        d if d < -0.10 => PriceJudgment::Cheaper,
        d if d <= 0.05 => PriceJudgment::Normal,
        d if d <= 0.20 => PriceJudgment::MoreExpensive,
        _ => PriceJudgment::OutrageouslyExpensive,
    }
}

#[derive(Debug, Clone, PartialEq, Eq)]
enum PriceJudgment {
    /// 难以置信的便宜——"这价格是骗人的吧？"
    UnbelievablyCheap,
    /// 比锚便宜——"比上次便宜，值得买"
    Cheaper,
    /// 正常范围——"和平时差不多"
    Normal,
    /// 比锚贵——"贵了，再等等"
    MoreExpensive,
    /// 离谱贵——"疯了！这价格我绝不出手"
    OutrageouslyExpensive,
}
```

### 3.4 挣脱锚定——谁能跳出锚

```rust
/// 检查 NPC 是否能挣脱锚定效应
/// 挣脱条件复合判断——不是单一阈值
fn can_break_anchor(
    npc: &NpcData,
    anchor: &PriceAnchor,
    observed_price: u32,
    known_prices: &BTreeMap<(MarketId, ItemDefId), PriceKnowledge>,
    now: GameTimestamp,
) -> bool {
    let intelligence = npc.mental.attributes.intelligence;
    let market_understanding = npc.eco_state.cognition.market_understanding;
    let anchor_age_days = now.days_since(anchor.formed_at);

    // ── 基础挣脱概率 ──
    let base_prob = (intelligence * 0.4 + market_understanding * 0.4
        + npc.eco_state.cognition.financial_literacy * 0.2)
        .clamp(0.0, 1.0);

    // ── 修正因素 ──
    // 锚越老 → 越容易被质疑（"那是很久以前的价格了"）
    let age_factor = (anchor_age_days / 180.0).min(1.0) * 0.15;

    // 已知价格中至少有 3 个其他市场的价格 → 有对比基础
    let comparison_count = known_prices.iter()
        .filter(|((_, item_id), _)| item_id == &anchor.formed_at_item)
        .count();
    let comparison_factor = (comparison_count.min(5) as f32 / 5.0) * 0.2;

    // 观察到的价格与锚差距很大 → 触发"这不对劲"
    let deviation = (observed_price as f32 - anchor.anchor_price_copper as f32).abs()
        / anchor.anchor_price_copper as f32;
    let surprise_factor = (deviation.min(1.0)) * 0.25;

    // ── 综合判断 ──
    let break_prob = (base_prob + age_factor + comparison_factor + surprise_factor)
        .clamp(0.0, 1.0);

    // 随机 roll（种子 = NpcId + 锚形成时间 → 确定性）
    let roll = deterministic_f32(&[
        npc.identity.npc_id.0,
        anchor.formed_at.as_ticks(),
    ]);
    roll < break_prob
}
```

**挣脱锚后的行为**：NPC 将 `anchor.is_broken = true`。之后的判断不再受锚影响——而是综合所有 `known_prices` 进行加权平均判断。挣脱锚本身的叙事价值：NPC 的自我叙述中可能出现"我曾经以为铁矿石就是 50 铜，直到我去了三座城市"——一个微小的认知成长故事。

### 3.5 锚与 NPC 人格的互动

| 人格维度（BigFive） | 对锚定的影响 |
|---------------------|-------------|
| **开放性（Openness）高** | 更容易挣脱锚——"情况总是在变" |
| **开放性低** | 锚极强——"世界是稳定的，价格就该是这样" |
| **尽责性（Conscientiousness）高** | 锚形成更牢固——"我记得很清楚" |
| **尽责性低** | 锚本身就不稳定——"上次是多少来着？算了随便吧" |
| **神经质（Neuroticism）高** | 偏离锚产生强烈情绪——降价狂喜/涨价焦虑 |
| **宜人性（Agreeableness）高** | 卖方不愿涨价偏离锚——"不能要太多" |

## 四、区域价差与运费壁垒

### 4.1 运费——价差的天然地板

两个市场之间同一物品的价差不会长期超过两地之间的运输成本。因为如果有超过运费的价差，商队就会出现，把便宜的货运到贵的地方卖——这个过程本身就是**套利**，而套利天然地把价差收敛到运费附近。

```
铁矿：矿村 30 铜 vs 城市 60 铜
运费：30 铜/块（距离×地形×安全）
价差：30 铜 = 运费 → 套利利润 = 0
  → 没有商人愿意做这个生意
  → 价差维持在 30 铜（运费壁垒）

如果城市铁矿涨到 70 铜：
价差：40 铜 > 运费 30 铜 → 套利利润 = 10 铜/块
  → 商人涌入 → 矿村需求增加 → 矿村价格上升
  → 城市供给增加 → 城市价格下降
  → 价差收敛到 ~30 铜
```

**运费是价差的天然地板——而不是天花板。** 商队的竞争把价差压在地板上，偶尔跳起（天灾/战争阻断商路），但总是落回。

### 4.2 estimate_travel_cost — 运费估算

```rust
/// 估算两点之间的运输成本（单程·铜币/单位重量）
/// 这个函数是商人套利决策的核心——不精确，但反映客观约束
fn estimate_travel_cost(
    from: (LocationId, Vec3),
    to: (LocationId, Vec3),
    cargo_weight_kg: f32,
    terrain: &dyn TerrainQuery,
    safety: &dyn SafetyQuery,
    weather: &dyn WeatherQuery,
    season: SeasonState,
    now: GameTimestamp,
) -> TravelCostEstimate {
    let distance_km = haversine_distance_km(from.1, to.1);

    // ── 地形系数 ──
    let terrain_cost = terrain.segment_average(from.0, to.0, |segment| {
        match segment.dominant_terrain {
            DominantTerrain::Road        => 1.0,
            DominantTerrain::Plains      => 1.3,
            DominantTerrain::Hills       => 2.0,
            DominantTerrain::Mountains   => 4.0,
            DominantTerrain::Forest      => 2.5,
            DominantTerrain::Swamp       => 5.0,
            DominantTerrain::Desert      => 2.8,
            DominantTerrain::Snowfield   => 3.5,
            DominantTerrain::WaterRoute  => 0.5,   // 水运效率
            DominantTerrain::CoastalRoute=> 0.7,   // 沿海航线
        }
    });

    // ── 安全系数 ──
    // 途经区域的抢匪活动、野兽密度、战区
    let safety_rating = safety.route_safety(from.0, to.0, now);
    let safety_multiplier = match safety_rating {
        SafetyRating::Safe       => 1.0,
        SafetyRating::Caution    => 1.3,
        SafetyRating::Dangerous  => 2.0,
        SafetyRating::WarZone    => 5.0,
    };

    // ── 天气修正 ──
    let weather_mod = season_weather_modifier(weather, season);

    // ── 载重系数 ──
    // 轻载 → 人可以背/骑马 → 成本低
    // 重载 → 需要车/船/驼队 → 成本增加
    let load_multiplier = if cargo_weight_kg < 30.0 {
        1.0                           // 个人携带
    } else if cargo_weight_kg < 500.0 {
        1.0 + (cargo_weight_kg - 30.0) / 470.0 * 1.5   // 骡马驮运
    } else {
        2.5 + (cargo_weight_kg - 500.0) / 5000.0 * 3.0  // 车队/船运（固定成本高但边际低）
    };

    // ── 基础运费率 0.5 铜/kg·km（可调）──
    let base_rate_copper_per_kg_km = 0.5;

    let total_copper = (distance_km
        * cargo_weight_kg
        * base_rate_copper_per_kg_km
        * terrain_cost
        * safety_multiplier
        * weather_mod
        * load_multiplier).round() as u32;

    TravelCostEstimate {
        total_copper,
        per_unit_copper: (total_copper as f32 / cargo_weight_kg.max(0.1)).round() as u32,
        distance_km,
        dominant_terrain_cost: terrain_cost,
        safety_multiplier,
        weather_mod,
        breakdown: TravelCostBreakdown {
            distance_cost: (distance_km * cargo_weight_kg * base_rate_copper_per_kg_km) as u32,
            terrain_surcharge: 0,   // 可在详细计算中填充
            safety_surcharge: 0,
            weather_surcharge: 0,
            load_penalty: 0,
        },
    }
}

/// 季节对运输的修正
fn season_weather_modifier(
    weather: &dyn WeatherQuery,
    season: SeasonState,
) -> f32 {
    match season {
        SeasonState::Spring => 1.15,  // 春雨泥泞——路难走
        SeasonState::Summer => 1.0,   // 最佳运输季节
        SeasonState::Autumn => 1.1,   // 秋雨
        SeasonState::Winter => 1.8,   // 雪封山路——大幅增加成本
    }
}
```

### 4.3 水运优势——海/河运效率远高于陆运

```rust
/// 水运效率是陆运的数倍——这是世界历史的物理约束
/// 海洋为主（海:陆≈7:3）的世界中，水运是经济的动脉
///
/// 具体数值来自地形系数：
///   Road: 1.0    → 陆路最优
///   WaterRoute: 0.5 → 水运效率是陆路的 2 倍
///   CoastalRoute: 0.7 → 沿海航线
///
/// 涌现效果：
///   - 海港城市天然成为贸易枢纽（不是因为"设定"，是因为水运便宜）
///   - 内陆城市物价普遍高于沿海（运费高）
///   - 沿河聚落更繁荣——河水运比陆路便宜
///   - "丝绸之路"类型的跨大陆商路即使陆路也会存在——
///     因为海洋包裹着大陆但不穿越大陆
```

**地理如何决定经济**：在 25 万 km² 的世界中（海:陆≈7:3），大多数聚落距离海岸线不超过 200km。这意味着大多数交易的水运是可行的。偶尔有深入内陆 500km+ 的聚落——它们的物价天然偏高，与沿海之间的价差对应运费壁垒。商队从沿海贩运到内陆是有利可图的——只要道路安全。

### 4.4 套利收敛——商队如何把价差压在地板上

```rust
/// 商人评估套利机会——这是"商人涌现"的微观基础
fn evaluate_arbitrage_opportunity(
    buy_market: &Market,
    sell_market: &Market,
    item_def: ItemDefId,
    trader: &NpcData,
    travel: &dyn TravelQuery,
    known_prices: &BTreeMap<(MarketId, ItemDefId), PriceKnowledge>,
) -> Option<ArbitrageDecision> {
    // ── 使用 NPC 的主观认知价格（不是客观价格）──
    let buy_price = match known_prices.get(&(buy_market.market_id, item_def)) {
        Some(pk) if pk.confidence > 0.5 => pk.known_price_copper,
        _ => return None,  // 不了解买入市场价格——不会去
    };
    let sell_price = match known_prices.get(&(sell_market.market_id, item_def)) {
        Some(pk) if pk.confidence > 0.5 => pk.known_price_copper,
        _ => return None,
    };

    let price_diff = sell_price as i64 - buy_price as i64;
    if price_diff <= 0 { return None; }  // 没有价差或倒挂

    // ── 估算运费 ──
    let cargo_kg = estimate_cargo_for_trader(trader);
    let travel_cost = estimate_travel_cost(
        (buy_market.location(), buy_market.position()),
        (sell_market.location(), sell_market.position()),
        cargo_kg,
        travel.terrain(),
        travel.safety(),
        travel.weather(),
        SeasonState::current(),
        GameTimestamp::now(),
    );

    // ── 套利利润 ──
    let gross_profit_per_unit = price_diff as u32;
    let net_profit_per_unit = if gross_profit_per_unit > travel_cost.per_unit_copper {
        gross_profit_per_unit - travel_cost.per_unit_copper
    } else {
        return None;  // 价差被运费吃光
    };

    // ── NPC 心智修正 ──
    // 风险承受度（神经质高 → 只想高利润的低风险套利）
    let risk_tolerance = 1.0 - trader.personality.big_five.neuroticism;
    let required_profit_margin = 0.30 - risk_tolerance * 0.25;  // 0.05-0.30
    let profit_margin = net_profit_per_unit as f32 / buy_price as f32;
    if profit_margin < required_profit_margin { return None; }

    // ── 决定套利 ──
    Some(ArbitrageDecision {
        buy_market: buy_market.market_id,
        sell_market: sell_market.market_id,
        item_def,
        estimated_buy_price: buy_price,
        estimated_sell_price: sell_price,
        estimated_profit_per_unit: net_profit_per_unit,
        cargo_kg,
        travel_cost,
        npc_confidence: calculate_arbitrage_confidence(trader, profit_margin),
    })
}
```

**涌现效果总结**：

| 情景 | 涌现结果 |
|------|---------|
| 两城价差 50 铜，运费 30 铜 | 套利利润 20 铜/块——商队涌现——价差收敛至 ~30 铜 |
| 两城价差 15 铜，运费 30 铜 | 无套利利润——商队不出现——价差维持 |
| 战争切断商路（安全系数 5.0→运费 150 铜） | 价差短期内扩大到 100+ 铜——城市缺铁——铁匠歇业——剑价暴涨——军队补给困难——战争形势受影响 |
| 新修大路（Road 系数 1.0→0.8） | 运费下降——内陆物价降低——经济繁荣——人口流入——聚落升级 |

## 五、价格波动率

### 5.1 从交易历史中计算波动率

波动率不是预设参数——从交易历史中计算。`Market.price_volatility: BTreeMap<ItemDefId, f32>` 已在 002 §3.4 中定义——变异系数（标准差/均值），基于最近 20 笔交易。

```rust
/// 获取某物品在某市场的波动率分类——便于 NPC 决策
fn volatility_category(market: &Market, item_def: ItemDefId) -> VolatilityCategory {
    let vol = market.price_volatility.get(&item_def).copied()
        .unwrap_or(0.0);  // 无交易 → 零波动（但也是"无数据"）

    match vol {
        v if v < 0.05 => VolatilityCategory::Stable,      // 变异系数 < 5%
        v if v < 0.15 => VolatilityCategory::Low,          // 5-15%
        v if v < 0.30 => VolatilityCategory::Moderate,     // 15-30%
        v if v < 0.50 => VolatilityCategory::High,         // 30-50%
        _             => VolatilityCategory::Extreme,      // > 50%
    }
}

#[derive(Debug, Clone, PartialEq, Eq)]
enum VolatilityCategory {
    Stable,     // 几乎不变——必需品（面包/水/基本工具）
    Low,        // 小幅波动——常规商品
    Moderate,   // 中等波动——季节性商品（农作物/燃料）
    High,       // 高度波动——奢侈品、稀有材料
    Extreme,    // 极度波动——战时的战略物资、囤积品
}
```

### 5.2 波动率如何影响 NPC 行为

**波动率不是"系统参数"——它是 NPC 决策的输入。** 不同人格的 NPC 对同一波动率有不同反应：

```rust
/// NPC 看到当前波动率后，调整自身的交易意愿
fn volatility_impact_on_npc(
    volatility: VolatilityCategory,
    npc: &NpcData,
    is_buyer: bool,
) -> f32 {
    // 基础谨慎度——从 neuroticism + intelligence 派生
    let base_caution = npc.personality.big_five.neuroticism * 0.6
        + npc.mental.attributes.intelligence * 0.4;

    let impact_multiplier = match (volatility, is_buyer) {
        // 稳定 → 所有人都愿意交易
        (VolatilityCategory::Stable, _) => 1.0,

        // 低波动 → 大部分 NPC 照常
        (VolatilityCategory::Low, _) => 0.95,

        // 中等波动 → 谨慎的 NPC 退缩
        (VolatilityCategory::Moderate, true) => {
            // 买方：波动大 → 等价格更低再买（"等等看"）
            0.8 - base_caution * 0.3  // 范围 0.5-0.8
        }
        (VolatilityCategory::Moderate, false) => {
            // 卖方：波动大 → 可能是好事（高价出货的机会！）
            // 高开放性的 NPC 更愿意在波动中卖（"说不定能卖个好价钱"）
            0.85 + npc.personality.big_five.openness * 0.15  // 范围 0.85-1.0
        }

        // 高波动 → 大部分 NPC 退场
        (VolatilityCategory::High, true) => {
            // 买方：大幅退缩（"现在买可能是韭菜"）
            (0.5 - base_caution * 0.4).max(0.1)  // 范围 0.1-0.5
        }
        (VolatilityCategory::High, false) => {
            // 卖方：高开放性 + 高风险承受度 → 可能趁机出货
            // 高神经质 → 害怕价格继续跌 → 现在清仓
            let greed = npc.personality.big_five.openness * 0.6;
            let fear = npc.personality.big_five.neuroticism * 0.4;
            (0.3 + greed - fear).clamp(0.1, 1.0)
        }

        // 极度波动 → 市场近乎停摆
        (VolatilityCategory::Extreme, _) => {
            // 只有最激进（低神经质 + 高开放性）的 NPC 才敢交易
            let extreme_tolerance = npc.personality.big_five.openness
                - npc.personality.big_five.neuroticism;
            if extreme_tolerance > 0.5 { 0.3 } else { 0.0 }
        }
    };

    impact_multiplier
}
```

### 5.3 波动率的涌现故事

| 波动率情景 | 涌现效果 |
|-----------|---------|
| 铁矿石稳定（cv=3%） | 铁匠安心开铺——剑价稳定——冒险者预算可预期——经济健康 |
| 粮食中等波动（cv=18%） | 秋季丰收→粮价低；春荒→粮价高。农民囤粮者获利，先吃后囤者夏季挨饿 |
| 战时铁矿石极度波动（cv=65%） | 铁匠不敢进货（怕明天跌一半）→ 剑产量下降 → 剑价飙升 → 军队装备不足 → 战争局势受影响 |
| 奢侈染料高波动（cv=40%） | 只有最富冒险精神的商人参与——每次交易都是赌博——"染料大王"的发家与破产传奇 |

## 六、价格信息的生命周期——从交易到记忆

### 6.1 一条价格信息的完整旅程

```
T+0: 交易发生——铁矿石以 48 铜成交
  → last_price 更新（EMA = 47.5）——客观事实层

T+1h: 旁观者 NPC_B（intelligence=0.65）看到交易
  → 存入 known_prices: PriceKnowledge {
        known_price_copper: 48, source: DirectObservation, confidence: 0.95, hops: 0
      }

T+2h: NPC_B 在酒馆对 NPC_C 说："铁矿 48 铜了"
  → 语言表达系统应用失真算子
  → NPC_C 存入: PriceKnowledge {
        known_price_copper: 48, source: Overheard{reliability: 0.8}, confidence: 0.68, hops: 1
      }

T+1d: NPC_C 写信告诉远方兄弟 NPC_D
  → Letter 通道（延迟=旅行时间 3 天）
  → NPC_D 在 T+4d 收到，但此时价格可能已变

T+3d: NPC_D 收到信时市场价格已是 52 铜（需求增加）
  → NPC_D 的 known_price 是过时的 48 铜
  → NPC_D 以"便宜价"挂买单 → 买不到（因为没人 48 卖）
  → NPC_D 困惑："信上不是说 48 吗？" → 亲身体验到信息延迟的代价

T+30d: 大部分 NPC 的这条价格知识信心衰减到 0.5 以下
  → 除非有新的确认（confirmation_count++）
  → 不再影响决策
```

### 6.2 价格知识与历史系统的互动

重大价格事件写入历史系统：

- 价格波动率突破 Extreme 阈值 → 触发 EconomicEvent
- 市场长期无交易（dormant/closed） → 记录原因（饥荒/战争/领主暴政）
- 某物品价格创历史新高/新低 → "天价粮"的传说通过语言表达系统传播，成为集体记忆
- 价格知识可存入 NPC 记忆→成为自传的一部分："那年冬天面粉卖到 200 铜一袋"

## 八、伪代码总结——一个完整的价格发现周期

```rust
/// 从交易到 NPC 决策的完整链路——展示价格如何涌入整个系统
fn price_discovery_cycle(
    market: &mut Market,
    npc_registry: &mut NpcRegistry,
    language_system: &mut LanguageSystem,
    now: GameTimestamp,
) {
    // ── 第1步：撮合交易 → 价格涌现 ──
    for listing in market.new_listings_today() {
        let trades = match_orders(market, &listing, inventory, wallets);
        for trade in &trades {
            update_last_price(market, trade);     // EMA 平滑
            update_volatility(market, trade.item_def); // 变异系数
            record_trade_for_history(trade);      // → 历史系统
        }
    }

    // ── 第2步：市场参与者更新已知价格 ──
    for npc_id in &market.participants {
        let npc = npc_registry.get_mut(*npc_id);
        // 随机抽样 1-3 笔今日交易（模拟"路过看到"）
        for trade in sample_recent_trades(market, 2) {
            let knowledge = PriceKnowledge {
                known_price_copper: trade.price_per_unit_copper,
                learned_at: now,
                source: PriceKnowledgeSource::DirectObservation {
                    market_id: market.market_id,
                    observed_trade_id: trade.trade_id,
                },
                confidence: 0.95,
                distortion: 0.0,
                hops: 0,
                learned_location: market.location(),
                last_used_at: None,
                confirmation_count: 0,
            };
            npc.eco_state.known_prices.insert(
                (market.market_id, trade.item_def), knowledge
            );
        }
    }

    // ── 第3步：价格信息传播 ──
    // NPC 在对话中自然提到价格——语言表达系统接管
    // 传播 3-5 跳后失真明显
    // 接收 NPC 存入 known_prices（confidence < 1.0, hops > 0）

    // ── 第4步：NPC 用心智模型做决策 ──
    for npc_id in npc_registry.active_npcs() {
        let npc = npc_registry.get(*npc_id);
        // 查询所有已知价格 → 与锚对比 → 结合情绪/人格 → 决定买/卖/观望
        // 这个决策是 NPC 自己的——经济系统不干涉
    }

    // ── 第5步：商人扫描套利 ──
    for trader_id in npc_registry.active_traders() {
        let trader = npc_registry.get(*trader_id);
        // 扫描已知各市场价差 > 运费 → 发起套利
        // 套利过程本身又产生新交易 → 回到第1步

}
```
