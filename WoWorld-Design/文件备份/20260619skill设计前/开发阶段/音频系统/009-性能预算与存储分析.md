## §一 Rust CPU 预算

### 1.1 每帧分解（目标帧 16.7ms @ 60fps）

| 操作 | 规模 | 成本 | 说明 |
|------|------|------|------|
| L1 SoundFootprint 轮询 | 200 实体 × 60fps | ~15µs | 读 128B struct + 1KB ring_buffer |
| L1 perceived_sounds() | 200 实体 × 空间索引查询 | ~80µs | 每实体听者范围 ~100 事件 × 快速过滤 |
| L1 perceived_speech() | ~10 活跃对话 × 200 实体 | ~20µs | 仅追踪 OngoingSpeech |
| L2+ 强度门控 | 惰性（按需） | ~10µs | 99% L2 声音被门控挡掉 |
| 环境声学混合 | per 有听者的 cell | ~15µs | 广播式——非 per-listener |
| 音乐状态更新 | 1 次 | ~5µs | 上下文切换检测 + 交叉淡入淡出插值 |
| 掩蔽计算 | 广播式 | ~20µs | per-cell，非 per-listener |
| AudioRenderPacket 序列化 | 1 次（玩家） | ~5µs | ~3KB 序列化 |
| 其他（VoiceProfile/EmotionModulation） | 稀疏 | ~5µs | 仅讲话时 |
| **合计** | | **~175µs ≈ 0.17ms** | |

## §四 关键优化技术

### 4.1 空间索引——16m 网格哈希

全局环形缓冲区附带空间索引。查询"附近声音"只查本 cell + 相邻 8 cell（共 9 cell × 256m² = 2,304m²）。空 cell 跳过。

### 4.2 强度门控——过滤 99% L2+ 声音

L2/L3 声音需通过 `can_reach_l1` 判定才入全局缓冲。一个 L2 NPC 走路的 50dB 绝不可能传到 100m 外——衰减 -40dB = 10dB < 阈值。

### 4.3 广播式声场——per-cell，非 per-listener

对有听者的 cell 计算一次声场快照（AmbientNoiseLevel + PerceivedSound），该 cell 内所有听者共享。空 cell 不做——惰性。

### 4.4 掩蔽 early-out

声源 SPL < 环境噪声 - 15dB → 直接丢弃。大多数场景中大多数声音被掩蔽——大幅减少 distinct sounds 数量。

### 4.5 Zero-heap-alloc 热路径

- `SmallVec` 栈上优化（所有 ≤4-7 元素不堆分配）
- 环形缓冲区预分配
- `SoundFootprint`/`PerceivedSound` 栈上 Copy（或浅 clone）

### 4.6 L2 聚落聚合——对标 SettlementFaithSnapshot

L2 NPC 不 per-entity 算音频感知。SettlementAudioAggregate 每游戏日更新一次——惰性。

### 4.7 单线程无锁

Rust 模拟是 SoA + rayon 并行批处理。环形缓冲区单写者（音频模块主循环）——不需要 Mutex/RwLock。

## §六 风险与缓解

| 风险 | 严重度 | 缓解 |
|------|--------|------|
| 密集场景（市场/战场 50+NPC 在一处）→ 空间索引 cell 过载 | 中 | `max_events_per_cell = 64` 限制；超出后按 SPL 剪裁 |
| 程序化生物声音变体过多 → 运行时合成开销 | 低 | 仅 20-30 原型 .ogg + Godot pitch/EQ 调制 |
| 大世界静态环境声源过多（每条河流/瀑布） | 低 | EnvironmentSoundSource 惰性加载——仅 L1 Chunk 激活 |
| Godot AudioServer polyphony 上限 | 低 | 已有 `max_polyphony` 设置 + 同质合并 |
| AudioRenderPacket 跨 GDExtension 带宽 | 低 | ~3KB/帧——对比已有时常 ~1KB/frame NPC 状态同步 |
