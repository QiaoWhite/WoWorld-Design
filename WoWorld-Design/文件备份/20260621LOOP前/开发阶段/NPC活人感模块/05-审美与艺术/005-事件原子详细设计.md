# 005 — 事件原子详细设计

## 一、为什么是 3 个原子而非更多或更少

经过严格归并，审美系统只需要 3 个事件原子：

| # | 原子 | 性质 | 不可归并的理由 |
|---|------|------|--------------|
| 1 | **React** | 内省·不自主·静默 | 判断→内部状态变更的**唯一通道** |
| 2 | **Articulate** | 人际·自愿·外化 | 私密体验→社会事实的**唯一桥梁** |
| 3 | **Adopt** | 人际→内省·半自愿 | 品味传播的**唯一微观机制** |

**被归并掉的候选**：

| 候选 | 为何不是独立原子 |
|------|-----------------|
| **Attend**（感知） | 是 NPC 感知系统的通用功能（眼耳感知），非审美专属 |
| **Ponder**（沉思） | 是记忆系统的通用功能（意义建构 = 记忆编码+联想），非审美专属 |
| **Evaluate**（评估） | 就是 judge()——纯函数，不是事件 |
| **Create**（创作） | = Embellish——但 Embellish 的本质是"审美意图的产出"，与其叫 Create 不如叫 Embellish 更精确（创作是修饰意图的一种极致形式） |
| **Judge**（判断） | = judge()——计算不是动作 |

**三个原子，三个方向，完备正交**：

```
React:    判断 → 内部状态（情绪/记忆/行为倾向）
Articulate: 内部状态 → 外部世界（语言/行为/社交信号）
Adopt:    外部世界 → 内部状态（品味被他人/文化影响）
```

## 二、React 原子

### 2.1 触发

每次 judge() 被调用后，自动执行 React。不可跳过。

### 2.2 输入

```rust
pub struct ReactInput {
    pub agent: NpcId,
    pub target: AestheticTarget,
    pub judgment: AestheticJudgment,
    pub context: AestheticContext,
}
```

### 2.3 内部逻辑

```rust
pub fn react(input: &ReactInput, npc: &mut NpcState) -> Vec<AestheticEffect> {
    let mut effects = Vec::new();
    let pressure = input.context.self_preservation_pressure;
    let j = &input.judgment;
    
    // ── 1. 情绪 delta ──
    let emo_attenuation = (1.0 - pressure * 0.5).max(0.1);
    let emotion_delta = j.valence * emo_attenuation * 0.15;  // BASE_EMO_WEIGHT
    effects.push(AestheticEffect::EmotionDelta(emotion_delta));
    
    // ── 2. 记忆写入 ──
    let somatic = compute_somatic_impact(&input.target.signal, &input.context.stakes);
    let memory_attenuation = (1.0 - pressure * 0.4).max(0.15);
    let depth = (j.interest + somatic) / 2.0 * memory_attenuation;
    
    if depth >= 0.1 {  // 低于阈值不写入记忆
        let memory = AestheticMemory {
            target: input.target.hash(),
            judgment: *j,
            somatic_impact: somatic,
            depth,
            timestamp: WorldTime::now(),
            location: npc.position(),
        };
        effects.push(AestheticEffect::MemoryWrite(memory));
    }
    
    // ── 3. somatic_impact 触发标记 ──
    if somatic > 0.35 {
        effects.push(AestheticEffect::SomaticImpact {
            intensity: somatic,
            // 触发身体反应：心跳变化/起鸡皮疙瘩/脊背发凉——由情绪引擎渲染
        });
        // 如果 memory 已写入 → 标记为高情感权重
        // → 影响记忆检索的优先级和记忆衰退速度
    }
    
    // ── 4. 行为倾向（仅在生存压力 < 0.5 时生效）──
    if pressure < 0.5 {
        if j.valence > 0.5 || j.interest > 0.6 {
            effects.push(AestheticEffect::ApproachBehavior {
                target: input.target.clone(),
                reason: if j.interest > j.valence { "想看仔细" } else { "觉得好看" },
            });
        } else if j.valence < -0.4 {
            effects.push(AestheticEffect::AvoidBehavior {
                target: input.target.clone(),
                reason: "觉得不好看",
            });
        }
    }
    
    // ── 5. 经济意图（进一步的行为倾向）──
    if pressure < 0.4 && j.valence > 0.6 && j.interest > 0.4 {
        // 产生购买/定制/收藏意图 → 由行为树决定是否及如何执行
        effects.push(AestheticEffect::PossessionInterest {
            target: input.target.clone(),
            strength: (j.valence * 0.5 + j.respect * 0.3 + j.interest * 0.2).min(1.0),
        });
    }
    
    effects
}
```

### 2.4 输出

```rust
pub enum AestheticEffect {
    EmotionDelta(f32),                    // → 情绪引擎
    MemoryWrite(AestheticMemory),         // → 记忆系统
    SomaticImpact { intensity: f32 },     // → 情绪引擎 + 记忆标记
    ApproachBehavior { target: AestheticTarget, reason: &'static str },  // → 行为树
    AvoidBehavior { target: AestheticTarget, reason: &'static str },     // → 行为树
    PossessionInterest { target: AestheticTarget, strength: f32 },       // → 行为树→经济系统
}
```

## 三、Articulate 原子

### 3.1 触发决策

```rust
pub fn should_articulate(
    judgment: &AestheticJudgment,
    personality: &BigFive,
    context: &AestheticContext,
    comm_norms: &CommunicationNorms,
) -> (bool, ArticulationMode) {
    // ── 基础表达强度 ──
    let judgment_intensity = (judgment.valence.abs() * 0.8)
        .max(judgment.interest * 0.6)
        .max(judgment.respect * 0.7)
        .max(judgment.arousal * 0.4);
    
    // ── 个性因子 ──
    let personality_factor = personality.extraversion * 0.5
                           + personality.openness * 0.3
                           + (1.0 - personality.neuroticism) * 0.2;
    
    // ── 情境因子 ──
    let mut context_factor = 0.0;
    if !context.social.is_alone { context_factor += 0.6; }
    if context.social.being_asked { context_factor += 2.0; }  // 被问到→几乎必须回应
    if context.social.is_formal_occasion { context_factor -= 0.4; }
    
    // ── 文化因子 ──
    let culture_factor = comm_norms.emotional_expressiveness;
    
    // ── 生存压制 ──
    let pressure_suppression = (1.0 - context.self_preservation_pressure * 0.95).max(0.02);
    
    let expression_level = judgment_intensity 
        * personality_factor 
        * (1.0 + context_factor) 
        * culture_factor 
        * pressure_suppression;
    
    // ── 判定通道 ──
    let mode = if expression_level < 0.2 {
        return (false, ArticulationMode::Silent);
    } else if expression_level < 0.4 {
        ArticulationMode::Exclamation
    } else if expression_level < 0.7 {
        ArticulationMode::Social
    } else {
        // 高智高开放才能产生深度批评
        if personality.openness > 0.6 {
            ArticulationMode::Critique
        } else {
            ArticulationMode::Social
        }
    };
    
    (true, mode)
}
```

### 3.2 五个表达通道

#### 通道 0：Silent（沉默）

无外化。仅 React 原子处理内部状态。

#### 通道 1：Exclamation（简短发声）

```rust
/// 1-5 词评价，无需语言系统重度参与
/// 模板片段覆盖全组合，<10 模板
fn generate_exclamation(judgment: &AestheticJudgment) -> ExpressionRef {
    let template = match () {
        _ if judgment.valence > 0.7 && judgment.respect > 0.7 => "真是杰作！",
        _ if judgment.valence > 0.7 => "真漂亮！",
        _ if judgment.valence > 0.4 => "还不错。",
        _ if judgment.valence < -0.6 => "太难看了……",
        _ if judgment.valence < -0.3 => "不怎么样。",
        _ if judgment.respect > 0.8 => "好手艺！",
        _ if judgment.interest > 0.7 => "有意思……",
        _ => "嗯。",
    };
    ExpressionRegistry::emit(template)
}
```

#### 通道 2：Social（社交性评价）

面向特定听众，语言系统轻度参与（片段组合 + 滤镜，对标 TextGenerator 管道）。产出包括：
- 对朋友推荐：`"你应该去看看那个雕塑"`
- 对创作者赞美：`"你做的这把剑真是……"`
- 对同伴吐槽：`"这家店的家具俗不可耐"`
- 在市场讨价还价：`"这画不值这个价，你看这笔触……"`

#### 通道 3：Critique（深度批评/分析）

仅高 intelligence + 高 openness + 高相关技能的 NPC 能产生。语言系统重度参与（维度→自然语言映射）。产出的文本可作为 ExpressionRef 进入历史/传播管道。

#### 通道 4：Behavioral（行为性表达）

不通过语言，直接产生行为意图：
- **购买**：PossessionInterest → EconomicIntent → 经济系统
- **赠送**：respect 高 → GiftIntent（买下送人）
- **定制**：CommissionIntent（"给我也做一个"）
- **收藏/展示**：DisplayIntent（放在显眼位置）
- **毁坏/丢弃**：DestroyIntent（valence 极低且 arousal 高）
- **靠近细看/触摸**：ApproachIntent
- **回避/离开**：AvoidIntent

### 3.3 输出

```rust
pub enum ArticulateEffect {
    Expression(ExpressionRef),        // → 语言表达系统
    EconomicIntent(EconomicIntent),   // → 经济系统
    GiftIntent { recipient: NpcId, item: ItemEntId },
    CommissionIntent { crafter: NpcId, description: String },
    DisplayIntent { item: ItemEntId, location: DisplayLocation },
    DestroyIntent { item: ItemEntId },

    AvoidIntent { target: AestheticTarget },

    },
}
```

## 四、Embellish 原子（创作）

### 4.1 为什么 Embellish 独立于 Articulate

Embellish 产生**持久的美学信号修改**——它改变对象的 AestheticSignal。而 Articulate 产生**瞬时的社交信号**。

### 4.2 输入

```rust
pub struct EmbellishEvent {
    pub creator: NpcId,
    pub target: EmbellishTarget,
    pub intention: AestheticIntent,
    pub materials: Vec<ItemEntId>,
    pub skill_levels: HashMap<SkillId, f32>,
    pub duration_game_days: u32,
}

pub enum EmbellishTarget {
    NewItem { item_def: ItemDefId },      // 从零创造一个新物品
    ExistingItem { item_id: ItemEntId },  // 装饰/修改一个已有物品
    Performance,                           // 实时表演（音乐/舞蹈）
}

pub struct AestheticIntent {
    /// 创作者希望传达的信号方向（6维目标）
    pub target_dimensions: [f32; 6],
    /// 创作者对自己能达成目标的信心
    pub confidence: f32,
}
```

### 4.3 结果计算

```rust
pub fn embellish_outcome(
    intent: &AestheticIntent,
    skill_levels: &HashMap<SkillId, f32>,
    materials: &[ItemEntId],
    rng: &DeterministicRng,
) -> AestheticSignal {
    // 技能系数：主要技能水平 / 100（或使用加权）
    let skill_factor = compute_skill_factor(skill_levels).min(1.0);
    
    let mut signal = AestheticSignal::default();
    
    for dim in 0..6 {
        // 该维度对技能的要求系数
        let dim_skill_requirement = match dim {
            DIM_FLUENCY => 0.7,        // 流畅度需要中等技能
            DIM_NOVELTY => 0.2,        // 新颖度与技能关系不大（创意≠技术）
            DIM_COMPLEXITY => 0.8,     // 复杂度需要高技能来驾驭
            DIM_HARMONY => 0.6,        // 和谐度需要一定技能
            DIM_EXPRESSIVENESS => 0.3, // 表现力与技能弱相关（真诚>技术）
            DIM_VIRTUOSITY => 0.95,    // 技艺度几乎完全由技能决定
            _ => 0.5,
        };
        
        // 实现度 = skill × dimension_requirement
        let realization = skill_factor * dim_skill_requirement;
        
        // 材料修正（+0 到 +0.15，取决于材料品质）
        let material_bonus = compute_material_bonus(materials, dim);
        
        // 噪声：低技能时噪声更大
        let noise_scale = (1.0 - skill_factor) * 0.15;
        let noise = rng.f32_range(-noise_scale, noise_scale);
        
        // 最终信号 = 意图 × 实现度 + 材料加成 + 噪声
        signal[dim] = (intent.target_dimensions[dim] * realization 
                      + material_bonus + noise).clamp(0.0, 1.0);
    }
    
    signal
}
```

**涌现效果**：
- 大师想画杰作 → skill_factor=0.95 → 结果接近意图 → 杰作
- 学徒想画杰作 → skill_factor=0.20 → 结果严重偏离意图 → "儿童画"
- 但学徒的真诚努力可能在 expressiveness 得到补偿（expressiveness 技能依赖低）
- 大师故意"画烂" → intention.target 低 harmony → 实际结果低 harmony

### 4.4 与物品系统的接口

```rust
// Embellish(NewItem) → 物品系统
let output_signal = embellish_outcome(&intent, &skill_levels, &materials, &rng);
let aesthetic_props = AestheticProps::from_signal(&output_signal);
let item_id = ItemRegistry::create_item(
    item_def_id,
    ItemProperties {
        quality: derive_quality(skill_levels),
        rarity: Rarity::Common,  // 新造物品默认 Common
        aesthetic_props,
        ..Default::default()
    },
    creator_id,
);
// → 历史系统: ArtifactCreatedEvent
```
