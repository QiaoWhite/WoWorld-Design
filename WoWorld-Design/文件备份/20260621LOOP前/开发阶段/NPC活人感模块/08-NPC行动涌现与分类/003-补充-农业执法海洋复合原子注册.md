## 一、农业复合原子 (Agriculture Domain)

### 1.1 TILL — 翻土

```
前提: 有可耕种土地 + 手持工具（锄/铲/手）+ AgentSnapshot.endurance > 0.2
物理序列: POSTURE(弯腰) + STRIKE(土面) × N + POSTURE(起身)
参数:
  - area_m2: 翻土面积
  - soil_type: 从土壤类型推导翻土难度
噪声来源: execution_noise_std(agriculture_Lv) → 翻土均匀度
结果:
  - 土地状态变为"已翻耕" → 影响后续 PLANT 的种子成活率
  - 消耗 stamina (面积×土壤硬度)
  - T3 Delta 写入: LandCultivated
```

### 1.2 PLANT — 播种/移栽

```
前提: 有种子/幼苗 + 已翻耕土地 或 容器（花盆）
物理序列: GRASP(种子) + POSTURE(蹲下) + PLACE(入土) + POUR(浇水)
参数:
  - species_id: 作物物种
  - count: 种子数
  - area: 种植范围
噪声来源: execution_noise_std(agriculture_Lv) → 种子间距均匀度
结果:
  - PlantInstance 创建（T3 持久化）
  - 生长 = growth_per_day × 光照 × 水分 × 肥力 × 温度 × 季节（植物/009 §六）
  - 气候匹配检查: PlantEcology 参数自动判定 → 不适配 → 可能不活
```

### 1.3 HARVEST — 收割/采集

```
前提: 作物成熟（life_stage = Mature/Fruiting）
物理序列: GRASP(作物部位) + CUT 或 PULL + PLACE(容器)
参数:
  - species_id: 作物物种
  - harvest_type: 全株收割 或 部分采集
结果:
  - 产物从 植物/009 §四 HarvestableProduct 表查询
  - 产量 = 基础产量 × skill_bonus(agriculture_Lv) × 工具加成
  - 品质: 品质基础 + 高技能概率提升一级
  - T1 Delta 写入: Harvested → 再生计时器启动
```

### 1.4 MILK — 挤奶

```
前提: 哺乳期雌性牲畜 + 手持容器
物理序列: POSTURE(蹲下) + GRASP(乳头) + SQUEEZE × N + POUR(转移)
参数:
  - animal_id: 牲畜实体
  - container: 容器物品
结果:
  - 产物 ("milk, 牛乳") → 物品栏
  - 产量 = 牛乳量 × skill_bonus
  - 动物情绪微调（舒适=↑，粗暴=↓）
```

### 1.5 SHEAR — 剪毛

```
前提: 毛长足够的羊/骆马等 + 剪刀
物理序列: GRASP(动物) + POSTURE(固定) + CUT(毛, 不伤害皮肤) × N
结果:
  - 产物 ("wool, 羊毛") → 物品栏
  - 毛量减至零 → 再生周期启动（天气×营养影响再生速度）
```

### 1.6 FEED — 喂食

```
前提: 牲畜 + 饲料/草料 + 食槽/地面
物理序列: GRASP(饲料) + POUR(到食槽) 或 SCATTER(撒地面)
结果:
  - 动物获取 food → Physiology.hunger↑
  - 自动打理模式: 小型养殖手动（三五头）→ 中大型 L3 周期统计处理
```

### 1.7 FERTILIZE — 施肥

```
前提: 肥料（动物粪便/堆肥/魔法催长物）+ 土地
物理序列: GRASP(肥料) + SCATTER(洒布) 或 POUR(浇灌)
结果:
  - 土壤肥力 ↑ → growth_per_day ↑ 持续 N 天
  - 魔法催长（TRANSMUTE）→ 更大的肥力 + 但可能土壤灵元素失衡
```

## 二、执法复合原子 (Law Enforcement Domain)

### 2.1 REPORT_TO_AUTHORITY — 告发

```
前提: NPC 记忆中有一条"我目击了 X 违反 PrescribeRule Y" + 告发意愿 > 阈值
物理序列: 
  ① 查 PowerTopology.query_jurisdiction(位置, 违规类型) → 找到有权 holder
  ② MOVE → holder 的所在位置
  ③ 发起对话(DialogueIntent::GoalDriven { action: "report_crime" })
  ④ 提供目击记忆作为对话话题
参数:
  - breach_event: 目击的违规记忆 ID
  - authority: 查到的管辖权 holder
结果:
  - holder 收到报告 → holder 的 GOAP/决策器决定下一步
  - 告发者记忆: 编码"我告发了 X"
```

### 2.2 INVESTIGATE — 调查

```
前提: 权威 holder 收到告发 + 决定调查
物理序列: 
  ① MOVE → 案发现场（如果有）
  ② 查询现场环境（PerceptBatch 高精度——注意焦点在此事件上）
  ③ 对话 × N（访问证人/嫌疑人的记忆比对）
  ④ MentalModel 推理: 整合证据 + 检测矛盾
参数:
  - case: 违规事件
  - witnesses: 已知证人列表
  - suspect: 初始嫌疑人（来自告发者的记忆）
结果:
  - MindAttribution 更新——"我推断 X 是行凶者" confidence 改变
  - 证据充分度: 0-1 → 决定是否 Adjudicate
```

### 2.3 APPREHEND — 逮捕

```
前提: 权威 holder 持有 Sanction 或 Constrain 权力 + 已确认违规者身份
物理序列: 
  ① 战斗半自动（目标=制服, 非致命——战斗系统 CombatCommand::Apprehend）
  ② 如果擒获 → 物理约束（ATTACH 绳索/枷锁）
  ③ 如果逃脱 → 失败 → 传播逃脱事实
结果:
  - 被捕者: 被 Constrain 约束——不可自由移动
  - 逮捕者记忆: 编码"我逮捕了 X 因为 Y"
```

### 2.4 EXECUTE_SANCTION — 执行制裁

```
前提: Adjudicate 裁决已完成 + holder 持有 Sanction 权力
物理序列:
  按 Sanction 类型:
  - Extract(fine) → 从 subject 获取罚金
  - Constrain(imprisonment) → 将 subject 移入 prison 建筑 + ATTACH Constrain 约束
  - Constrain(banishment) → 将 subject 移出领土边界
  - Compel(restitution) → subject 执行赔偿劳动
  - Remit(mercy) → 不执行任何惩罚
结果:
  - Duty 状态更新
  - 事件编码为 subject + 执行者 + 目击者的 EventMemory
  - 信息传播链起点
```

### 2.5 PATROL — 巡逻

```
前提: 守卫/治安官 NPC + 巡逻路线
物理序列: MOVE(巡逻路径) × 循环 + 定期 PerceptBatch 扫描
参数:
  - patrol_route: 路径 waypoints
  - alertness: 感官警觉度——比日常高
结果:
  - 周期性检查 Duty 违约——发现违规 → 触发记忆编码 + 决策
  - 路径可以是硬化路/土径，NPC 的寻路系统自动导航
```

### 2.6 GUARD_PRISONER — 看守囚犯

```
前提: 守卫 NPC + 囚犯 subject + prison 建筑
物理序列: 
  ① 维持与囚犯的空间邻近
  ② 定期 PerceptBatch 扫描——囚犯是否在牢房内？
  ③ 对囚犯的 Constrain 检查——如果出现在禁止区域 → APPREHEND
结果:
  - 囚犯的 Constrain(behavior=leave_cell) 被守卫的 PATROL 周期检查
```

## 三、海洋复合原子 (Maritime Domain)

### 3.1 SCOOP_CANOE — 掏空独木舟

```
前提: 粗树干 + 火源 + 石斧/刮刀
物理序列:
  POSTURE(蹲) + IGNITE(木柴) → 火在树干表面慢烧
  → HEAT(持续) 软化木质
  → SCOOP(刮削炭化层)
  → 循环: 火烧→刮削→火烧→刮削
  → 冷却
参数:
  - log: 树干物品
  - fire_source: 火源
  - tool: 刮削工具
噪声来源: execution_noise_std(woodworking_Lv) → 船舱均匀度 + 壁厚
结果:
  - 独木舟物品产出
  - 品质 = f(noise, 树干材质) → 影响浮力/速度/耐久
  - 木屑副产品（燃料）
```
