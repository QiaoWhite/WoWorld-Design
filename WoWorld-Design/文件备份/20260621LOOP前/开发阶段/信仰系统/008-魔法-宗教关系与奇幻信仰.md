## 一、MagicReligionRelation——per-faith 属性

### 1.1 四种关系

```rust
/// 每个信仰的 FaithDefinition 包含此字段——非全局设置。
/// 覆盖文化系统 005 中的 `religiosity × 0.5` 假设。
pub enum MagicReligionRelation {
    /// 魔法是神的恩赐——施法者是神选者，魔法是祷告的延伸
    /// 效果：高 religiosity → magic_tech 增长加速（+0.1 modifier）
    Gift,

    /// 魔法是亵渎/异端——魔法使用者被宗教迫害
    /// 效果：高 religiosity → magic_tech 增长压制（-0.15 modifier）
    /// 高 participation 信徒拒绝接受魔法治疗/附魔（"这是邪术！"）
    Blasphemy,

    /// 魔法与宗教无关——井水不犯河水
    /// 效果：religiosity 不影响 magic_tech
    /// 信徒对魔法的态度取决于个人，非教义决定
    Independent,

    /// 魔法=宗教——祭司=法师，祈祷=施法，神殿=魔法学校
    /// 效果：magic_tech 增长使用 clergy 数量而非 mage 数量
    /// RitualDef.magic 字段的魔法仪式 = 宗教仪式的自然组成部分
    Unified,
}
```

### 1.2 对技术派生的影响

文化系统 005 中的 `magic_tech` 技术增长公式有 `religiosity × 0.5` 的默认权重。信仰系统覆盖此值：

```rust
// 文化 005 中：
fn compute_magic_tech_growth(culture: &CultureCoreParams, faith_query: &dyn FaithQuery) -> f32 {
    let dominant_faith = faith_query.dominant_faith_in_culture(culture.id);
    let magic_rel = dominant_faith.map(|fid| faith_query.magic_relation(fid))
        .unwrap_or(MagicReligionRelation::Independent);

    match magic_rel {
        MagicReligionRelation::Gift => culture.religiosity * 0.4,        // 正相关
        MagicReligionRelation::Blasphemy => -culture.religiosity * 0.3,   // 负相关
        MagicReligionRelation::Independent => 0.0,                        // 无关
        MagicReligionRelation::Unified => culture.religiosity * 0.6,      // 强正相关
    }
}
```

## 二、魔法与宗教在不同范式下的 NPC 认知

### 2.1 社会角色

| 关系 | 祭司的角色 | 法师的角色 | 社会张力 |
|------|----------|----------|---------|
| Gift | 神选者——唯一的魔法使用者 | 不存在——"法师"=祭司 | 低（统一） |
| Blasphemy | 反魔法的监督者 | 地下/秘密/被迫害 | 极高——可能触发宗教迫害 |
| Independent | 对魔法不关心 | 与祭司平行存在 | 低（各自为政） |
| Unified | 祭司=法师 | 不存在独立法师 | 零（完全同一） |

### 2.2 信仰对 NPC 个人的影响

- `Blasphemy` 信仰的虔诚 NPC → 拒绝魔法治疗（即使重伤——可能导致死亡）；拒绝魔法附魔装备；可能举报使用魔法的邻居
- `Gift` 信仰的虔诚 NPC → 尊重法师如尊重神职；可能寻求法师的祝福；魔法被视为神圣
- `Unified` → NPC 不对"魔法"和"宗教"做区分——神殿=魔法塔，祈祷=施法
- `Independent` → NPC 的态度由个人决定（受 personality + theological_depth 影响），不由教义决定

## 三、奇幻特有信仰的参数空间

### 3.1 DragonGod 龙神信仰

```rust
/// 龙神信仰——非覆盖层，而是 FaithTheology 的特殊参数组合：
///   deity_count = 1-5（龙的数量有限）
///   ancestor_importance = 0.1
///   nature_sacredness = 0.7（龙是自然力量的化身）
///   hierarchy_degree = 0.8（龙>龙裔>凡人——严格的种族等级）
///   exclusivity = 0.6
///   mysticism = 0.4
///
/// 特殊表现：
///   - deity 列表中的每一项 = 一条龙（有自己的颜色/吐息/性格/领地）
///   - 彩色龙/金属龙/元素龙——不同颜色代表不同神格面向
///   - 神殿 = 龙的巢穴/祭坛（建在龙领地内或附近）
///   - 神迹（A 模式）= 龙亲自显现——可能是祝福也可能是毁灭
///   - 信徒可能寻求"龙之血"获得力量（龙裔化）——对接魔法系统的契约魔法
```

### 3.2 ElementalFaith 元素信仰

十元素（金木水火土风雷电血灵——魔法 002 定义）作为崇拜对象：

```rust
/// 元素信仰参数空间：
///   deity_count = 10（或 5（五行）/ 4（四元素）等子变体）
///   nature_sacredness = 1.0
///   mysticism = 0.6
///   ritual_formality = 0.5（元素祭坛/火供/水祝）
///
/// 每种元素有对应的祭司和仪式：
///   - 火之祭司 → 掌管锻造祝福/火供/火山祭祀
///   - 水之祭司 → 掌管求雨/净化/航海祝福
///   - ...
/// 魔法系统（十元素体系）天然支持——元素祭司 = 对应元素的专精法师
```

### 3.3 AscensionFaith 登神信仰

```rust
/// 登神信仰——相信凡人可通过功绩/试炼/仪式升格为神。
///   deity_count = 2-10（登神者加入万神殿）
///   exclusivity = 0.4（登神者不排斥其他登神者）
///   hierarchy_degree = 0.3（动态——新登神者改变万神殿的权力结构）
///
/// 崇拜的对象是"曾经是人"的神——他们的生平故事是教义的核心。
/// A 模式下——NPC 真的可能登神（生命 010 的凡人登神路径）。
/// 一个登神事件可能创建新 FaithDefinition（以登神者为核心神祇的新信仰）。
```

### 3.4 WorldMachine 世界机信仰

```rust
/// 世界机信仰——相信世界是一台巨大的机械，被未知存在/法则驱动。
///   deity_count = 0.0（无神——但有"机械师"/"造物主"概念）
///   mysticism = 0.1（极度理性/机械论——"世界可被理解为齿轮与发条"）
///   ritual_formality = 0.8（维护世界的仪式——"我们必须定期转动圣轮以保持世界运转"）
///
/// 这是 WoWorld 中的"科学前身"——世界机信仰者更倾向于理性解释。
/// 可能成为技术发展的特殊驱动。
```

### 3.5 MagicAsFaith 魔法即信仰

```rust
/// 魔法即信仰——魔法本身被视为神圣力量。不崇拜人格神。
///   deity_count = 0.0（魔法不是神——但法力是神圣的）
///   mysticism = 0.5
///   nature_sacredness = 0.8（魔力流淌于万物）
///
/// 与 Unified 的区别：
///   Unified = "祭司=法师"（有神，神通过魔法工作）
///   MagicAsFaith = "魔法就是神"（没有神，魔法本身是神圣的）
```

## 四、契约魔法与信仰的边界

### 4.1 区别（继承魔法 011 的定义）

| | 信仰关系 | 契约关系 |
|---|---------|---------|
| **方向** | 单方面——信徒崇拜神 | 双方面——交换明确的权利义务 |
| **回应** | 神可能回应也可能不（B/C 模式始终不） | 必须响应——契约有约束力 |
| **本质** | 奉献/服从 | 交易/合作 |
| **终止** | 不可终止（信仰不是契约） | 可终止（违约/到期履行） |

### 4.2 共存

一个神（A 模式）可以同时有：
- **信徒**——one-way 崇拜关系（通过信仰系统）
- **契约者**——two-way 交换关系（通过魔法系统 011 的 Contract 魔法）

信徒 ≠ 契约者。一个虔诚的信徒可能拒绝与他的神签契约（"这是亵渎——你怎能与神做交易？"）。一个无信者（NonPracticing）可能与神签契约（纯功利："给我力量——我给你祭品"）。两种关系完全独立。

## 五、死灵术与祖先崇拜的张力

### 5.1 立场矩阵

| 信仰类型 | 对死灵术的态度 | 对亡灵的定性 |
|---------|-------------|------------|
| 祖先崇拜（ancestor_importance > 0.5） | 与祖先灵魂**沟通** = 神圣实践 | 祖灵 = 保护/祝福 |
| 祖先崇拜 | **操控尸骨** = 亵渎 | 操控祖尸 = 对祖先的极大不敬 |
| 泛灵论（nature_sacredness > 0.7） | 死灵术 = 扰乱自然秩序 | 亡灵 = 失衡的灵 |
| 高 orthodoxy（> 0.7） | 死灵术 = 神学上的错谬 | 亡灵 = 不应存在的 |
| 高 mysticism（> 0.7） | 死灵术 = 危险的灵性实践 | 亡灵 = 可能通向知识的路径 |

### 5.2 对接

信仰系统提供 `UndeadAttitude`（006 §六）→ 生命 008（亡灵）消费。不同信仰对亡灵的态度影响：
- 亡灵在该信仰区域的出现概率
- 信徒对亡灵 NPC 的态度（恐惧/尊重/试图帮助安息）
- 亡灵与信徒的交互可能性（祖先崇拜区域：亡灵可能提供信息/帮助）
