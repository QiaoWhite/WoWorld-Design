# 002 — 天气生成算法

## 一、算法总览

```
═══════════════════════════════════════════════════════════
              天气生成数据流
═══════════════════════════════════════════════════════════

世界生成 P2 产出:
├─ ClimateParamsField (温度基线/湿度基线/季节性幅度/纬度) — 静态，~100m 分辨率
├─ ElevationField (高程)
├─ BiomeMicroclimateField (canopy_closure/albedo/ground_type/...)
└─ OceanCurrentField (洋流温度/方向)

        ↓ 天气系统初始化时降采样

ClimateRegion (~50km, ~100 个区域):
├─ RegionalWeatherBaseline (每游戏日更新)
│   ├─ avg_temperature + season.temp_offset
│   ├─ avg_humidity × season.humidity_mult
│   ├─ prevailing_wind + Coriolis_deflection
│   └─ seasonality_amplitude (控制 Markov 转移速度)
│
└─ → LocalWeather (~1km, 每游戏分钟 tick)
    ├─ 双线性插值 ClimateRegion Baseline
    ├─ TerrainModifier (高程/坡向/山谷风/海岸风/湖效应)
    ├─ BiomeMicroclimateModifier (冠层/雪面/沙地/水体/城市/洞穴)
    ├─ WeatherStateMachine (Markov 6 状态)
    ├─ FogGenerator (独立——湿度+温差触发)
    └─ → WeatherSample (每 query 实时计算)

极端天气:
└─ 参数阈值判断 (非独立状态) → ExtremeWeatherType 标签 + 冷却管理

历史气象异常:
└─ 种子驱动极值采样 (years_to_simulate × ~100 ClimateRegion)
   → HistoricalWeatherAnomaly 序列 → 历史模块消费
```

## 二、ClimateRegion 参数化

### 2.1 降采样

世界生成产出的 `ClimateParamsField` 分辨率 ~100m。天气系统降采样至 ~50km 网格：

```rust
struct ClimateRegion {
    id: ClimateRegionId,
    
    // 从气候参数场降采样
    center_lat: f32,              // 区域中心纬度
    avg_temperature: f32,         // 年均温度 °C
    avg_humidity: f32,            // 年均湿度 0-1
    seasonality_amplitude: f32,   // 季节性幅度 — 热带~5°C，温带~25°C
    
    // 地理特征 (降采样统计)
    terrain_type: TerrainType,    // Plain/Mountain/Coast/Valley/Plateau/Desert
    water_fraction: f32,          // 区域内水域面积比 0-1
    mountain_fraction: f32,       // 区域内山地面积比 0-1
    distance_to_coast_km: f32,    // 中心到最近海岸距离 km
    
    // 洋流影响 (如果临海)
    coastal_current_temp: Option<f32>,  // 沿岸洋流温度偏差
    coastal_current_dir: Option<Vec2>,  // 沿岸洋流方向
    
    // 盛行风 (纬度+科里奥利力+大陆形状推导)
    prevailing_wind_dir: Vec2,
    prevailing_wind_speed: f32,
    
    // 风暴基线
    storm_frequency_base: f32,    // 年均极端天气频率 (0-1)
}
```

### 2.2 季节调制 — 区域天气基线

每游戏日，ClimateModulator 将静态气候基线 × 当前季节 → 动态区域基线：

```rust
/// 每游戏日调用一次
fn compute_regional_baseline(
    climate: &ClimateRegion,
    season: Season,
    season_progress: f32,  // 0-1 季节内进度
) -> RegionalWeatherBaseline {
    // ── 季节参数 ──
    let (temp_offset, humidity_mult, wind_mult, storm_bias) = match season {
        Season::Spring => (
            lerp(-0.3, 0.3, season_progress) * climate.seasonality_amplitude * 0.5,
            1.0,
            1.1,  // 春风略大
            0.8,  // 风暴正常偏少
        ),
        Season::Summer => (
            0.5 * climate.seasonality_amplitude,
            lerp(1.0, 0.7, season_progress), // 盛夏转干
            0.9,  // 夏季风小（除对流外）
            1.5,  // 风暴频率↑（对流强）
        ),
        Season::Autumn => (
            lerp(0.3, -0.3, season_progress) * climate.seasonality_amplitude * 0.5,
            1.1,  // 秋雨
            1.2,  // 秋风大
            1.2,  // 风暴↑（秋台风）
        ),
        Season::Winter => (
            -0.5 * climate.seasonality_amplitude,
            0.8,  // 冬季干燥
            1.0,
            1.1,  // 冬季风暴（温带气旋）
        ),
    };
    
    RegionalWeatherBaseline {
        temperature: climate.avg_temperature + temp_offset,
        humidity: (climate.avg_humidity * humidity_mult).clamp(0.0, 1.0),
        wind_direction: climate.prevailing_wind_dir,
        wind_speed: climate.prevailing_wind_speed * wind_mult,
        storm_probability: climate.storm_frequency_base * storm_bias,
        seasonality_amplitude: climate.seasonality_amplitude,
        // 日长相关——由 SeasonClock 独立提供
    }
}
```

**季节参数表**（[TUNING]——全部可调）：

| 季节 | temp_offset (×seasonality_range) | humidity_mult | wind_mult | storm_bias | day_length_mult (温带) |
|------|----------------------------------|---------------|-----------|------------|------------------------|
| 春 | 0→+0.5（回升中）                | 1.0           | 1.1       | 0.8        | 1.0→1.3                |
| 夏 | +0.5（顶峰）                    | 1.0→0.7（盛夏干）| 0.9    | 1.5        | 1.3→1.0                |
| 秋 | +0.5→0（下降中）                | 1.1           | 1.2       | 1.2        | 1.0→0.7                |
| 冬 | -0.5（低谷）                    | 0.8           | 1.0       | 1.1        | 0.7→1.0                |

## 三、TerrainModifier — 地形修正

地形修正发生在 LocalWeather 层，从 ClimateRegion 插值后叠加：

```rust
fn apply_terrain_modifier(
    baseline: &RegionalWeatherBaseline,
    elevation: f32,           // 本地高程 m
    slope_aspect: Vec2,       // 坡向 (水平法线)
    valley_axis: Option<Vec2>,// 山谷走向
    distance_to_coast_m: f32,
    distance_to_lake_m: f32,
    lake_area_km2: f32,
) -> TerrainCorrection {
    let mut corr = TerrainCorrection::identity();
    
    // ── 1. 气温直减率 ──
    // 每上升100m，温度下降约0.65°C
    corr.temperature_offset -= elevation * 0.0065;
    
    // ── 2. 地形降水（迎风坡/背风坡） ──
    let wind_alignment = baseline.wind_direction.dot(slope_aspect);
    if wind_alignment > 0.3 {
        // 迎风坡 — 湿空气被迫抬升 → 降水增加
        corr.precipitation_mult *= 1.0 + wind_alignment * 0.3;
    } else if wind_alignment < -0.3 {
        // 背风坡 — 干空气下沉 → 雨影效应
        corr.precipitation_mult *= 1.0 + wind_alignment * 0.5; // 最大减50%
        corr.humidity_offset -= 0.1;
    }
    
    // ── 3. 山谷风 ──
    if let Some(valley_dir) = valley_axis {
        // 白天——谷风（沿山谷向上）
        // 夜间——山风（沿山谷向下）
        // 简化: 调制局部风向沿山谷轴线偏转
        let valley_influence = 0.3; // 山谷对风的引导强度
        corr.wind_direction = lerp_vec2(
            baseline.wind_direction,
            valley_dir,
            valley_influence,
        );
    }
    
    // ── 4. 海岸风 ──
    if distance_to_coast_m < 3000.0 {
        let coastal_factor = 1.0 - (distance_to_coast_m / 3000.0);
        // 午后海风——从海面吹向陆地
        corr.afternoon_wind_bias = Some(coastal_factor * 0.4);
        // 海岸更潮湿
        corr.humidity_offset += coastal_factor * 0.1;
    }
    
    // ── 5. 湖效应 ──
    if distance_to_lake_m < 5000.0 && lake_area_km2 > 100.0 {
        let lake_factor = 1.0 - (distance_to_lake_m / 5000.0);
        // 冬季下风岸——湖效应雪
        corr.lake_effect_snow_mult = lake_factor * 1.5;
        // 湖岸湿度
        corr.humidity_offset += lake_factor * 0.08;
    }
    
    // ── 6. 地形雾 ──
    if elevation > 500.0 && baseline.humidity > 0.6 {
        corr.fog_formation_probability *= 3.0; // 山上云雾
    }
    if let Some(valley_dir) = valley_axis {
        corr.fog_formation_probability *= 2.0; // 山谷辐射雾
    }
    
    corr
}
```

### 3.1 关键数值速查

| 修正因子 | 公式 | 最大影响 |
|---------|------|---------|
| 气温直减率 | -0.65°C / 100m | 700m山 ≈ -4.5°C |
| 迎风坡降水 | ×(1 + wind_alignment × 0.3) | +30% |
| 背风坡降水 | ×(1 + wind_alignment × 0.5) | -50% |
| 海岸湿度 | +(1 - dist/3000) × 0.1 | +10% |
| 湖效应雪 | ×(1 - dist/5000) × 1.5 | ×2.5 (紧邻大湖) |

## 四、BiomeMicroclimateModifier — 群系微气候修正

天气系统消费世界生成产出的群系参数场，计算地面温度/湿度/有效风速：

```rust
/// 群系微气候输入 — 来自世界生成 BiomeMicroclimateQuery
struct BiomeMicroclimate {
    canopy_closure: f32,        // 冠层郁闭度 0-1
    albedo: f32,                // 地表反照率 0-1
    ground_type: GroundType,     // BareSoil/Sand/Rock/Urban/Water/Ice
    wetland_fraction: f32,      // 湿地比例 0-1
    urban_fraction: f32,        // 城市建筑覆盖率 0-1
    distance_to_water_km: f32,  // 最近开放水体距离 km
}

enum GroundType {
    BareSoil,       // 裸土 — 基准
    Sand,           // 沙地 — 高日间温度，快速辐射冷却
    Rock,           // 岩漠 — 裸露基岩
    Urban,          // 城市 — 热岛效应
    Water,          // 水体 — 不适用（天气不查询水面）
    Ice,            // 冰面 — 高反照
}

/// 群系微气候修正计算
fn compute_biome_microclimate(
    regional: &RegionalWeatherBaseline,
    biome: &BiomeMicroclimate,
    surface: SurfaceState,       // 当前地表状态（含天气历史累积——如积雪）
    solar: f32,                  // 太阳辐射强度 0-1 (正午=1, 夜间=0)
) -> BiomeCorrection {
    let mut corr = BiomeCorrection::identity();
    
    // ═══════════════════════════════════════════════
    // 地面温度修正
    // ═══════════════════════════════════════════════
    
    // ── 冠层效应 ──
    // 日间遮阳降温，夜间保温
    let canopy_cooling = solar * biome.canopy_closure * 6.0;
    let canopy_insulation = (1.0 - solar) * biome.canopy_closure * 3.0;
    corr.ground_temperature_offset -= canopy_cooling;
    corr.ground_temperature_offset += canopy_insulation;
    
    // ── 地表类型修正 ──
    match surface {
        SurfaceState::Snow => {
            // 雪面高反照(0.7-0.9)→日间不吸热，温度不低过0°C太远（相变锁定）
            let snow_cooling = solar * regional.temperature.abs() * 0.7;
            corr.ground_temperature_offset -= snow_cooling;
            // 但最低不低于 -5°C（极地除外）——雪有隔热性
            // 实际约束在调用方处理
        }
        SurfaceState::Dry => {
            match biome.ground_type {
                GroundType::Sand | GroundType::Rock => {
                    // 低热容 + 直接辐射 → 日间极热，夜间快速冷却
                    corr.ground_temperature_offset += solar * 8.0;
                    corr.ground_temperature_offset -= (1.0 - solar) * 5.0;
                }
                GroundType::Urban => {
                    // 城市热岛——日间吸热+夜间放热
                    corr.ground_temperature_offset += solar * 2.0;
                    corr.ground_temperature_offset += (1.0 - solar) * 3.0;
                }
                _ => {} // BareSoil — 基准
            }
        }
        SurfaceState::Wet | SurfaceState::Mud => {
            // 蒸发冷却——湿地面日间更凉
            corr.ground_temperature_offset -= solar * 2.0;
        }
        SurfaceState::Ice => {
            // 冰面高反照 + 相变锁定
            corr.ground_temperature_offset -= solar * 3.0;
        }
    }
    
    // ── 水体邻近效应 (<1km) ──
    if biome.distance_to_water_km < 1.0 {
        let water_factor = 1.0 - biome.distance_to_water_km; // 0→1
        // 水高热容——日间冷却，夜间加温
        corr.ground_temperature_offset -= solar * water_factor * 4.0;
        corr.ground_temperature_offset += (1.0 - solar) * water_factor * 3.0;
    }
    
    // ═══════════════════════════════════════════════
    // 局地湿度修正
    // ═══════════════════════════════════════════════
    
    corr.ground_humidity_offset += biome.canopy_closure * 0.15;   // 蒸腾
    corr.ground_humidity_offset += biome.wetland_fraction * 0.30; // 沼泽蒸发
    corr.ground_humidity_offset -= biome.urban_fraction * 0.10;   // 城市排水
    
    if biome.distance_to_water_km < 1.0 {
        let water_factor = 1.0 - biome.distance_to_water_km;
        corr.ground_humidity_offset += water_factor * 0.12;
    }
    
    // ═══════════════════════════════════════════════
    // 有效风速修正
    // ═══════════════════════════════════════════════
    
    let blockage = (biome.canopy_closure * 0.7)       // 冠层挡风
        + (biome.urban_fraction * 0.5)                  // 建筑挡风
        + terrain_wind_shadow_factor();                  // 地形背风
    
    corr.wind_speed_mult *= (1.0 - blockage.clamp(0.0, 0.9));
    
    corr
}
```

### 4.1 群系微气候速查表

| 群系特征 | 日间地面温度 | 夜间地面温度 | 局地湿度 | 地表风速 |
|---------|------------|------------|---------|---------|
| **密闭冠层** (>0.7) | -2~6°C | +1~3°C | +10~20% | ×0.2~0.5 |
| **稀疏冠层** (0.3~0.7) | -1~3°C | +0~1°C | +5~10% | ×0.5~0.8 |
| **无植被裸地** | +5~15°C | -5~10°C | 无 | 无修正 |
| **雪盖** | 地表不吸热（反照0.7-0.9），但≥ -5°C | 同左 | 干燥 | 无修正 |
| **沙地** | +8~20°C | 快速冷却 | 极干燥 | 无修正 |
| **沼泽/湿地** | 无显著偏移 | 无显著偏移 | +20~40% | 无修正 |
| **大型水体旁** (<1km) | -2~5°C | +2~5°C | +10~15% | 午后湖/海风 |
| **城市热岛** (>50建筑/km²) | +1~3°C | +2~5°C | -5~15% | 街道风道复杂 |
| **洞穴口** | 恒温 (~年均温) | 恒温 (~年均温) | >80% | ~0 |
| **山脊/山峰** | 每100m -0.65°C（由高程覆盖） | 同左 | 低（绝热冷却已降水） | ×1.5~3.0 |

### 4.2 循环依赖澄清

```
植物生长 → canopy_closure 变化 → 微气候修正 → ground_temperature 变化 → 植物生长

不会振荡——时间尺度拉开:
├─ 天气更新: 每分钟，消费当前 canopy_closure（快）
├─ 植物生长: 每游戏日，产出新 canopy_closure（慢）
└─ canopy_closure 在一次天气更新内是常数

群系突变（森林火灾→冠层清零）→ 通过事件推送通知天气系统刷新缓存。
```

## 五、WeatherStateMachine — Markov 状态机

### 5.1 状态定义

6 个核心状态。每个 ClimateRegion 有独立的状态机实例。

```
状态图 (每游戏分钟一次转移):

    Clear ←→ PartlyCloudy ←→ Overcast ←→ LightPrecip ←→ ModeratePrecip ←→ HeavyStorm

转移约束:
├─ 相邻状态转移概率高（渐变——不突然从晴跳到暴风雨）
├─ 跨一级转移概率低 (~0.005-0.03)——可能但罕见
├─ 跨两级转移概率极低 (~0.001)——几乎不可能
├─ 季节调制转移方向
├─ 地形快车道（山区雾概率×3，海岸午后阵雨↑）
└─ HeavyStorm 冷却期: 退出后 2 游戏日 内 HeavyStorm入边概率=0
```

### 5.2 参数化转移矩阵

```rust
/// 从区域天气基线参数化生成 Markov 转移矩阵
fn build_transition_matrix(
    baseline: &RegionalWeatherBaseline,
    fog_state: &FogState,
) -> MarkovMatrix {
    let mut m = MarkovMatrix::new(6); // Clear..HeavyStorm
    let n = 6;
    
    for from in 0..n {
        for to in 0..n {
            let delta = (to as i32 - from as i32).abs();
            
            // ── 基础概率 ──
            let base_prob = match delta {
                0 => {
                    // 自维持概率——天气倾向于保持当前状态
                    match from {
                        0 => 0.90, // Clear很稳定
                        1 => 0.85, // PartlyCloudy
                        2 => 0.80, // Overcast
                        3 => 0.75, // LightPrecip
                        4 => 0.70, // ModeratePrecip
                        5 => 0.60, // HeavyStorm——最不稳定（会快速消散）
                        _ => 0.80,
                    }
                }
                1 => 0.06, // 相邻状态
                2 => 0.01, // 跨一级
                _ => 0.002, // 跨两级或更多——几乎不可能
            };
            
            // ── 湿度推动 ──
            let humidity_push = if baseline.humidity > 0.7 && to > from {
                1.5 // 高湿区域→降水倾向增加
            } else if baseline.humidity < 0.3 && to < from {
                1.3 // 干燥区域→转晴倾向增加
            } else {
                1.0
            };
            
            // ── 季节倾向 ──
            let season_push = match baseline.current_season {
                Season::Summer if to > from => 1.3, // 夏季对流→暴雨↑
                Season::Winter if to < from && from >= 3 => 1.2, // 冬季降水少
                Season::Autumn if to == 5 => 1.2, // 秋季台风→HeavyStorm↑
                _ => 1.0,
            };
            
            // ── 稳定性 — 季节性幅度越大，天气变化越快 ──
            let stability = 1.0 / (1.0 + baseline.seasonality_amplitude * 0.02);
            
            m.set(from, to, base_prob * humidity_push * season_push * stability);
        }
    }
    
    m.normalize(); // 每行加起来=1
    m
}
```

### 5.3 极端天气冷却

```rust
/// 极端天气冷却管理
struct StormCooldown {
    cooldown_ticks_remaining: u32,  // 游戏分钟
    cooldown_duration: u32,         // 基准冷却期
}

impl StormCooldown {
    fn start(&mut self, extreme_type: ExtremeWeatherType) {
        self.cooldown_duration = match extreme_type {
            ExtremeWeatherType::Storm => 2 * 24 * 60,     // 2 游戏日
            ExtremeWeatherType::Blizzard => 3 * 24 * 60,  // 3 游戏日
            ExtremeWeatherType::Typhoon => 5 * 24 * 60,   // 5 游戏日
            ExtremeWeatherType::DustStorm => 1 * 24 * 60, // 1 游戏日
            ExtremeWeatherType::Hailstorm => 0,            // 无独立冷却
            ExtremeWeatherType::Heatwave => 0,             // 无冷却——自然规律
            ExtremeWeatherType::ColdSnap => 0,
        };
        self.cooldown_ticks_remaining = self.cooldown_duration;
    }
    
    fn tick(&mut self) {
        if self.cooldown_ticks_remaining > 0 {
            self.cooldown_ticks_remaining -= 1;
        }
    }
    
    fn is_active(&self) -> bool {
        self.cooldown_ticks_remaining > 0
    }
}
```

冷却期间——HeavyStorm 的入边概率设置为 0。其他状态正常转移。

### 5.4 雾——独立维度

```rust
/// 雾生成器——独立于 Markov 链
fn update_fog(
    regional: &RegionalWeatherBaseline,
    terrain: &TerrainCorrection,
    biome: &BiomeCorrection,
    time_of_day: f32,  // 0-1 (0=午夜, 0.5=正午)
) -> FogState {
    // ── 辐射雾条件: 晴天 + 高湿 + 夜间/清晨 + 低风速 ──
    let radiation_fog_prob = if regional.temperature > 0.0
        && regional.humidity > 0.7
        && regional.wind_speed < 3.0
        && (time_of_day < 0.25 || time_of_day > 0.85) // 夜间/清晨
    {
        0.15 * terrain.fog_formation_probability
    } else { 0.0 };
    
    // ── 平流雾条件: 暖湿空气 + 冷水面 ──
    let advection_fog_prob = if biome.distance_to_water_km < 1.0
        && regional.temperature > 5.0
        && regional.humidity > 0.8
    {
        0.10
    } else { 0.0 };
    
    // ── 海雾条件: 海岸 + 暖湿 ──
    let sea_fog_prob = if terrain.is_coastal.unwrap_or(false)
        && regional.temperature > 10.0
        && regional.humidity > 0.75
        && time_of_day < 0.3 // 早晨居多
    {
        0.12
    } else { 0.0 };
    
    // ── 上坡雾条件: 山区迎风坡 + 高湿 ──
    let upslope_fog_prob = if terrain.elevation > 500.0
        && regional.humidity > 0.65
        && regional.wind_speed > 2.0
    {
        0.08 * terrain.fog_formation_probability
    } else { 0.0 };
    
    // ── 掷骰子 ──
    let total_prob = (radiation_fog_prob + advection_fog_prob + sea_fog_prob + upslope_fog_prob)
        .clamp(0.0, 0.3); // 封顶30%——不可能总是有雾
    
    let is_foggy = rng.gen::<f32>() < total_prob;
    
    FogState {
        is_foggy,
        fog_density: if is_foggy {
            0.3 + rng.gen::<f32>() * 0.5 // 0.3-0.8
        } else { 0.0 },
        fog_type: if is_foggy {
            // 按各概率比例选最可能的类型
            select_fog_type(radiation_fog_prob, advection_fog_prob, sea_fog_prob, upslope_fog_prob)
        } else { FogType::Radiation }, // 不重要
    }
}
```

## 六、极端天气生成

### 6.1 不新增天气类型——参数组合判定

```rust
/// 从当前天气状态判断是否构成极端天气
fn classify_extreme_weather(
    state: WeatherState,
    precipitation: PrecipitationType,
    precipitation_intensity: f32,
    wind_speed: f32,
    temperature: f32,
    coastal: bool,
    drought_zone: bool,
    duration_minutes: u32,
    historical_95th_temp: f32,     // 当地夏季95%分位温度
    historical_5th_temp: f32,      // 当地冬季5%分位温度
) -> Option<ExtremeWeatherType> {
    match state {
        WeatherState::HeavyStorm => {
            if wind_speed > 30.0 && coastal {
                Some(ExtremeWeatherType::Typhoon)
            } else if precipitation == PrecipitationType::Snow
                && wind_speed > 15.0
                && temperature < -5.0
            {
                Some(ExtremeWeatherType::Blizzard)
            } else if wind_speed > 20.0 {
                Some(ExtremeWeatherType::Storm)
            } else {
                None // HeavyStorm但未达极端阈值
            }
        }
        WeatherState::ModeratePrecip => {
            if precipitation == PrecipitationType::Hail
                && precipitation_intensity > 0.5
            {
                Some(ExtremeWeatherType::Hailstorm)
            } else if precipitation == PrecipitationType::Dust
                && wind_speed > 12.0
            {
                Some(ExtremeWeatherType::DustStorm)
            } else {
                None
            }
        }
        _ => {
            // 非降水极端——温度
            if temperature > historical_95th_temp && duration_minutes > 36 * 60 {
                Some(ExtremeWeatherType::Heatwave)
            } else if temperature < historical_5th_temp && duration_minutes > 24 * 60 {
                Some(ExtremeWeatherType::ColdSnap)
            } else {
                None
            }
        }
    }
}
```

### 6.2 极端天气参数速查

| 极端事件 | 触发条件 | 持续范围 | 冷却期 | 主要影响（消费模块自行实现） |
|---------|---------|---------|--------|--------------------------|
| **暴风雨** | HeavyStorm + 风速>20m/s | 2-8 游戏时 | 2 游戏日 | 航行危险/户外活动受阻/建筑轻微损坏 |
| **暴风雪** | HeavyStorm + 雪 + 风速>15m/s + 温度<-5°C | 2-12 游戏时 | 3 游戏日 | 可见度极低/移动×0.3/冻伤风险/建筑积雪载荷 |
| **台风** | HeavyStorm + 风速>30m/s + 海岸 | 4-16 游戏时 | 5 游戏日 | 建筑损坏/树木倒伏/船只摧毁/海水倒灌 |
| **沙尘暴** | ModeratePrecip + 沙尘 + 风速>12m/s + 干旱带 | 1-4 游戏时 | 1 游戏日 | 可见度极低/露天存储受损/呼吸伤害 |
| **冰雹暴** | ModeratePrecip + 冰雹 + 强度>0.5 | 0.1-0.5 游戏时 | 无 | 果实击落/露天NPC轻伤/窗户碎裂 |
| **热浪** | 温度>夏季95%分位 + 持续>36游戏时 | 36-72 游戏时 | 无冷却 | 干旱/口渴×2/老人NPC死亡风险/草原火概率↑ |
| **寒流** | 温度<冬季5%分位 + 持续>24游戏时 | 24-72 游戏时 | 无冷却 | 冻伤/作物冻死/牲畜死亡/港口冰封 |

互斥与连锁：
- 热浪期间不生成暴风雨（需要冷暖交汇）——但热浪**结束时**可能触发超级暴风雨（冷锋突入高温气团）
- 暴风雨后→沼泽/湿地水位上升→地面 Mud 状态更久维持
- 台风→暴雨→内陆河流泛滥→下游洪泛

### 6.3 极端天气传播

暴风雨不凭空出现——沿盛行风方向传播到相邻 ClimateRegion：

```rust
/// 极端天气空间传播
fn propagate_extreme_weather(
    source_region: ClimateRegionId,
    extreme_type: ExtremeWeatherType,
    wind_dir: Vec2,
    all_regions: &[ClimateRegion],
) -> Vec<(ClimateRegionId, f32)> {
    let mut affected = Vec::new();
    
    for neighbor in adjacent_regions(source_region, all_regions) {
        let dir_to_neighbor = direction(source_region.center, neighbor.center);
        let alignment = wind_dir.dot(dir_to_neighbor);
        
        if alignment > 0.5 {
            // 下风区域——天气系统将传入
            let arrival_delay_hours = match extreme_type {
                ExtremeWeatherType::Typhoon => 2.0 - alignment * 1.5,
                ExtremeWeatherType::Storm => 3.0 - alignment * 2.0,
                _ => 4.0,
            };
            let intensity_decay = 1.0 - alignment * 0.3; // 传播中衰减
            affected.push((neighbor.id, intensity_decay));
        }
    }
    
    affected
}
```

## 七、历史气象异常生成

### 7.1 定位

天气系统不生成"历史事件"——那是历史模块（世界生成 P9）的职责。天气系统只产出 `HistoricalWeatherAnomaly`——纯气象事实：

```rust
/// 历史气象异常 — 纯物理数据
/// 历史模块据此转化为 HistoricalEvent (EventType::Drought/Flood/etc.)
struct HistoricalWeatherAnomaly {
    anomaly_id: u64,
    anomaly_type: HistoricalAnomalyType,
    
    // 时空
    year: u32,                         // 世界诞生后第几年 (1..years_to_simulate)
    season: Season,
    duration_days: u32,
    climate_region: ClimateRegionId,
    affected_area_km2: f32,
    
    // 气象参数 (偏离气候基线的幅度)
    temperature_deviation: f32,        // ±°C 偏离年均值
    precipitation_deviation: f32,      // ×倍偏离年均降水
    wind_max: f32,                     // 峰值风速 m/s
    flooding_extent: f32,              // 0-1 区域内泛滥比例
    
    // 灾害集群
    belongs_to_cluster: Option<u64>,   // 所属灾害集群 ID
    
    // 人口交集 (供历史模块判断是否转化为事件)
    affected_settlement_ids: Vec<SettlementId>, // 波及的聚落
}

enum HistoricalAnomalyType {
    Drought,        // 干旱 — 降水<正常30%
    Flood,          // 洪水 — 降水>正常200% 或 台风引发
    SevereStorm,    // 大风暴 — 风速>极端阈值
    Heatwave,       // 热浪 — 温度>95%分位持续
    ColdSnap,       // 寒流 — 温度<5%分位持续
    ClimateShift,   // 气候转变 — 连续多年异常
}
```

### 7.2 生成算法

```rust
/// 世界生成时调用——种子驱动极值采样
fn generate_historical_anomalies(
    years_to_simulate: u32,        // 来自历史模块配置
    climate_regions: &[ClimateRegion],
    world_seed: u64,
) -> Vec<HistoricalWeatherAnomaly> {
    let mut rng = StdRng::seed_from_u64(world_seed);
    let mut anomalies = Vec::new();
    let mut anomaly_counter: u64 = 0;
    
    // ── 步骤 1: 参数化提取 ──
    // 从每个 ClimateRegion 的气候基线推导年际标准差
    for region in climate_regions {
        let sigma_temp = 0.5 + region.seasonality_amplitude * 0.1;  // 温度年际标准差
        let sigma_precip = 0.15 + region.seasonality_amplitude * 0.05; // 降水年际标准差
        let storm_base = region.storm_frequency_base;                   // 风暴基准频率
        
        // ── 步骤 2: 逐年采样 ──
        for year in 1..=years_to_simulate {
            // 从正态分布采样温度偏离和降水偏离
            let temp_dev = rng.sample(Normal::new(0.0, sigma_temp).unwrap());
            let precip_dev = rng.sample(Normal::new(1.0, sigma_precip).unwrap()); // ×倍——1.0=正常
            
            // 阈值判断: 偏离>2σ → 异常
            if temp_dev.abs() > 2.0 * sigma_temp {
                anomaly_counter += 1;
                anomalies.push(HistoricalWeatherAnomaly {
                    anomaly_id: anomaly_counter,
                    anomaly_type: if temp_dev > 0.0 {
                        HistoricalAnomalyType::Heatwave
                    } else {
                        HistoricalAnomalyType::ColdSnap
                    },
                    year,
                    season: Season::Summer, // 简化——温度异常多发生在夏/冬
                    duration_days: 10 + (temp_dev.abs() * 2.0) as u32,
                    climate_region: region.id,
                    affected_area_km2: 2500.0 * rng.gen_range(0.3..1.0),
                    temperature_deviation: temp_dev,
                    precipitation_deviation: precip_dev,
                    wind_max: 0.0,
                    flooding_extent: 0.0,
                    belongs_to_cluster: None,
                    affected_settlement_ids: vec![], // 由历史模块填入
                });
            }
            
            if precip_dev < 0.3 || precip_dev > 2.0 {
                anomaly_counter += 1;
                anomalies.push(HistoricalWeatherAnomaly {
                    anomaly_id: anomaly_counter,
                    anomaly_type: if precip_dev < 0.3 {
                        HistoricalAnomalyType::Drought
                    } else {
                        HistoricalAnomalyType::Flood
                    },
                    year,
                    season: rng.gen_season(),
                    duration_days: if precip_dev < 0.3 {
                        30 + ((-precip_dev * 100.0) as u32).min(180)
                    } else {
                        3 + (precip_dev * 5.0) as u32
                    },
                    climate_region: region.id,
                    affected_area_km2: 2500.0 * rng.gen_range(0.5..1.0),
                    temperature_deviation: temp_dev,
                    precipitation_deviation: precip_dev,
                    wind_max: 0.0,
                    flooding_extent: if precip_dev > 2.0 { rng.gen_range(0.1..0.5) } else { 0.0 },
                    belongs_to_cluster: None,
                    affected_settlement_ids: vec![],
                });
            }
            
            // 风暴事件
            if rng.gen::<f32>() < storm_base * 0.1 {
                anomaly_counter += 1;
                anomalies.push(HistoricalWeatherAnomaly {
                    anomaly_id: anomaly_counter,
                    anomaly_type: HistoricalAnomalyType::SevereStorm,
                    year,
                    season: rng.gen_season(),
                    duration_days: rng.gen_range(1..5),
                    climate_region: region.id,
                    affected_area_km2: 2500.0 * rng.gen_range(0.2..0.8),
                    temperature_deviation: 0.0,
                    precipitation_deviation: 0.0,
                    wind_max: 20.0 + rng.gen::<f32>() * 30.0,
                    flooding_extent: 0.0,
                    belongs_to_cluster: None,
                    affected_settlement_ids: vec![],
                });
            }
        }
    }
    
    // ── 步骤 3: 时间聚集（灾害集群） ──
    cluster_anomalies(&mut anomalies);
    
    // ── 步骤 4: 筛选只有人口影响的异常（历史模块自行做） ──
    // 天气系统只产出全部异常，不筛选
    
    anomalies
}

/// 时间聚集——相邻年+邻近区域的异常合并为灾害集群
fn cluster_anomalies(anomalies: &mut Vec<HistoricalWeatherAnomaly>) {
    let mut cluster_id = 0u64;
    
    // Group by ClimateRegion
    let mut by_region: HashMap<ClimateRegionId, Vec<usize>> = HashMap::new();
    for (i, a) in anomalies.iter().enumerate() {
        by_region.entry(a.climate_region).or_default().push(i);
    }
    
    for indices in by_region.values() {
        let mut sorted: Vec<usize> = indices.clone();
        sorted.sort_by_key(|&i| anomalies[i].year);
        
        let mut i = 0;
        while i < sorted.len() {
            let mut j = i;
            // 寻找连续异常（间隔≤2年）
            while j + 1 < sorted.len()
                && anomalies[sorted[j + 1]].year - anomalies[sorted[j]].year <= 2
            {
                j += 1;
            }
            
            if j > i {
                // i..=j 属于同一灾害集群
                cluster_id += 1;
                for k in i..=j {
                    anomalies[sorted[k]].belongs_to_cluster = Some(cluster_id);
                }
                i = j + 1;
            } else {
                i += 1;
            }
        }
    }
}
```
