## 一、设计哲学

### 1.1 什么是"文化"

在 WoWorld 中，**文化**是一个人群共享的集体行为模式、价值观、审美偏好和社会规范的集合。它不是种族特性——同一文化下可以有多个种族；同一种族也可以分属不同文化。文化不是预设的"精灵文化""矮人文化"——它是从世界种子生成的初始参数出发，在游戏运行中受 NPC 集体行为、地理隔离、贸易接触和重大历史事件影响而缓慢演变的**涌现系统**。

**文化 ≠ 种族**。种族提供生理差异（视觉范围、力量基数、寿命），文化提供行为差异（怎么打招呼、什么是美、能不能打断长辈）。两者的交集仅在于：一个文化中占多数的种族可能通过审美标准和文化规范产生温和反馈。

**文化 ≠ 信仰**。文化决定"宗教在这个社会中有多重要"（`religiosity`），信仰决定"信什么神、教义是什么、怎么祈祷"。两个共享同一文化的聚落可能信仰不同；两个信仰相同的聚落可能文化迥异。文化与信仰的完整边界见 §四。

**文化 ≠ 政治实体**。政治实体 = 领土 + 统治权力 + 合法性。文化 = 价值观 + 行为规范 + 审美偏好。文化为政治提供合法性来源之一（权力系统中的 `文化契合度` 权重 0.20），但文化区域边界与政治边界不必重合——一个文化区可以包含多个王国，一个帝国可以统治多种文化。

### 1.2 "故事生成器"的文化维度

WoWorld 是故事生成器，不是故事容器。文化系统服务于这一哲学的方式是：

- **不预设任何具体文化**。没有"诺斯文化""希腊文化""中国文化"——只有从种子生成的参数组合
- **定义规则，不定义内容**。定义了"荣誉文化的涌现条件"（7 个参数的特定交集），不定义"武士道"
- **文化差异 = 故事引擎**。文化误解、文化适应、文化融合——从 norms 差异中自然涌现，不手写"文化冲突"
- **玩家是文化参与者**。玩家可以：穿异文化服饰引起反应/充当两个文化之间的翻译/引入外来文化元素/被某个文化接纳或排斥

### 1.3 文化 = 参数场，不是标签

文化不是离散的"这个聚落是 A 类文化"标签。它是一个**连续参数场**——类似群系参数场。世界空间中每一点都有一个文化归属（主导 CultureId）+ 混合权重（在边界带上可能是 70%A / 30%B）。文化特征的边界是渐变的，只在遇到强地理屏障（山脉/大河/海洋）时产生明显分野。

## 二、文化的三层架构

文化由三层组成——从稳定到易变：

```
第一层：核心文化参数（CultureCoreParams）
  10个 f32 参数，种子生成，代际漂移（每游戏年 σ=0.003）
  → 定义"这个文化在价值观上是什么"
  → 消费方：所有模块。最稳定——百年内几乎不变
  
第二层：第一级派生（世界生成时计算，缓存，随核心参数缓慢更新）
  CommunicationNorms、BuildingStylePreferences、CulturalBeautyStandard、
  HonorificSystem、FertilityNorms、DietaryBasePreferences、honor_weight
  → 从核心参数 × 环境上下文 确定性派生
  → 消费方：语言表达、NPC、世界生成（建筑）、经济

第三层：第二级派生（动态计算，受历史事件和运行时状态修正）
  TechnologyProfile、magic_type_affinity、mage_social_status、
  literacy_rate、cross_cultural_attitude
  → 从核心参数 × 人口 × 贸易 × 历史事件 × 其他模块数据 派生
  → 消费方：技能、物品、魔法、权力
```

**为什么分三层？** 核心参数是最原子的文化 DNA——10 个正交维度可以组合出无穷的文化性格。第一级派生是"几乎就是核心"但混合了环境上下文的准核心数据——被消费者大量依赖，但不应独立存储（否则与核心参数失去同步）。第二级派生是"受太多因素影响以至于不该是文化参数"的复杂涌现结果——如技术水平受人口和贸易的强烈影响，不是纯粹的文化属性。

### 2.1 参数不可分原则

10 个核心参数是不可再分的原子。任何文化行为特征——饮食禁忌、尚武精神、艺术风格、权力接受度——都必须能从这 10 个参数的组合中推导出来。**禁止新增核心参数**（除非经过完整跨模块审计验证不可派生）——新增派生字段通过第一/二层派生公式实现，不修改核心参数集合。

## 三、核心类型定义

### 3.1 CultureId

```rust
/// 文化唯一标识符
/// 
/// 全局恒定，存档内持久。从世界种子确定性分配。
/// 与 NpcId(u32)、PolityId(u32) 对齐——同为 4 字节。
/// 
/// 谱系关系（父子/分裂/融合）通过 CultureGenealogy 查询——不编码在 ID 中。
#[derive(Copy, Clone, PartialEq, Eq, Hash, PartialOrd, Ord, Serialize, Deserialize)]
pub struct CultureId(pub u32);

/// 无效/缺失文化的占位值
pub const CULTURE_ID_NONE: CultureId = CultureId(u32::MAX);

/// 文化谱系图 —— 记录文化的父子/分合关系
/// 
/// 与文化存储分离——谱系查询低频，不应污染 CultureId 的 4 字节存储。
pub struct CultureGenealogy {
    /// 每个文化的直系祖先（origin 文化为 None）
    parents: FxHashMap<CultureId, Vec<CultureId>>,
    /// 每个文化的直接子文化
    children: FxHashMap<CultureId, Vec<CultureId>>,
    /// 分裂来源——从哪个现存文化中因隔离而分裂
    diverged_from: FxHashMap<CultureId, (CultureId, GameTime)>,
    /// 融合来源——克里奥尔文化的两个源文化
    fused_from: FxHashMap<CultureId, (CultureId, CultureId, GameTime)>,
    /// 文化消亡标记
    extinct: FxHashSet<CultureId>,
}
```

**设计决策**：`CultureId` 为扁平 u32，不编码谱系信息。原因：CultureId 被大量存储（NPC 记忆中、物品 `era_hint`、历史事件参与者……），4 字节优于 8 字节。谱系查询（"A 和 B 是同一文化家族吗？"）在运行时极低频，不值得为它让每次存储翻倍。

### 3.2 CultureCoreParams

```rust
/// 文化的 10 个核心参数——文化的"人格"
/// 
/// 所有值在 0.0-1.0 范围内。种子生成，代际漂移，不可直接写入。
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct CultureCoreParams {
    /// 个人主义 vs 集体主义
    /// 0 = 集体优先（家族/氏族为社会基本单位）
    /// 1 = 个人优先（核心家庭或个人独立性更强）
    pub individualism: f32,

    /// 权力距离——对等级制度的接受度
    /// 0 = 平等主义（上下级关系松散，挑战权威是正常的）
    /// 1 = 严格的等级（贵族/平民/奴隶分明，下对上有严格礼仪）
    pub power_distance: f32,

    /// 不确定性规避——对变化和未知的容忍度
    /// 0 = 包容变化、灵活应变、对新事物好奇
    /// 1 = 崇尚传统、规避风险、对"规矩"有强烈依赖
    pub uncertainty_avoidance: f32,

    /// 竞争取向——合作 vs 竞争
    /// 0 = 合作/关怀主导（共识决策、互助、社会安全网）
    /// 1 = 竞争/成就主导（个人成就至上、崇尚竞争、排名文化）
    /// 
    /// 注：原名 "masculinity"（霍夫斯泰德），改为 competition_orientation
    /// 原因：WoWorld 中的多智能种族有完全不同的性别动态（单性/双性/三性），
    /// 继续使用性别标签会产生概念混淆。底层维度不变——仍是"竞争 vs 关怀"。
    pub competition_orientation: f32,

    /// 长期取向
    /// 0 = 活在当下、及时行乐、对传统灵活
    /// 1 = 长远规划、节俭积累、投资未来
    pub long_term_orientation: f32,

    /// 放纵 vs 克制
    /// 0 = 克制欲望、自律是美德、享受有罪恶感
    /// 1 = 享受生活、表达情感是自然的、追求快乐
    pub indulgence: f32,

    /// 对外来者的开放度
    /// 0 = 排外/怀疑（陌生人是危险，外来思想应拒斥）
    /// 1 = 开放/好奇（陌生人是机会，外来思想应了解）
    pub openness_to_outsiders: f32,

    /// 宗教性——宗教在社会中的渗透程度
    /// 0 = 世俗化（宗教是个人私事，不参与公共决策）
    /// 1 = 宗教渗透一切（法律从教义派生，日常生活充满宗教仪式）
    /// 
    /// 注意：此参数只决定宗教的渗透程度——不决定信什么神、什么教义。
    /// 具体信仰内容属于信仰系统（独立模块，见 §四）。
    pub religiosity: f32,

    /// 军事化程度
    /// 0 = 和平主义（厌恶暴力、冲突以谈判解决、军事是防御性的）
    /// 1 = 军事优先（崇尚武力、征服是荣誉、军事开支优先于民生）
    pub militarism: f32,

    /// 艺术与审美重视度
    /// 0 = 功能优先（"够用就行"，装饰是浪费）
    /// 1 = 美感优先（建筑/服饰/日常用品都有艺术追求）
    pub artistry: f32,
}
```

### 3.3 CultureName

```rust
/// 文化名称——Endonym（自名）+ Exonym（他称）双层体系
pub struct CultureName {
    /// 自名——该文化自己的语言中对自己的称呼
    /// 从文化种子的音系规则确定性生成
    /// 示例："Elenari"、"河湾人"、"岩之子"
    pub endonym: String,

    /// 自名字面义——endonym 在该文化语言中的含义
    /// 示例："星辰之子"、"河湾的居民"、"岩石的后代"
    pub endonym_meaning: String,

    /// 常见他称——其他文化如何称呼此文化
    /// Key = 观察者 CultureId
    /// 示例：Elenari 在邻文化中被称作 "Silver Ones"（因银色服饰）
    pub common_exonyms: FxHashMap<CultureId, String>,

    /// 生成时兜底——在没有观察到该文化的第三方语境中使用
    /// 通常由地理指向型 exonym 产生
    pub fallback_exonym: String,
}
```

**Endonym 生成**：世界种子 → 文化种子 → 音系规则（`PhonologyRules`：允许的辅音/元音类别、音节模板、重音模式）→ 确定性音节拼接 → endonym。

**Exonym 生成**：观察者文化解释目标文化的方式，5 种来源加权随机：
- 60% 音变：目标 endonym 经由观察者音系规则扭曲（"Kaelim → Kailish"）
- 15% 地理：目标核心区域的地形特征 + 人群后缀（"Hill Folk"）
- 10% 外观：目标审美种子中最显著的视觉特征（"Red Cloaks"）
- 10% 历史：两文化共享的重大历史事件（"The Broken"、"Dragon Slayers"）
- 5% 行为：CommunicationNorms 中最极端的值（`silence_tolerance=0.9` → "The Silent"）

### 4.2 文化与信仰的边界

文化与信仰是两个**独立但交互**的模块：

```
文化系统                              信仰系统
─────────                            ─────────
CultureCoreParams.religiosity        FaithId —— 信仰标识符
  宗教渗透程度(0-1, 单参数)             单神/多神/萨满/泛灵/祖先/无神论/奇幻特有

文化不决定信什么——                     信仰决定：
只决定宗教在生活中的权重                神祇名称、教义内容、经典文本、禁忌清单、
                                       仪式类型、宗教机构结构、传教规则

religiosity 以两种方式影响信仰：
  1. 信仰分配时作为加权因子——高 religiosity → NPC 更可能被分配高虔诚度
  2. 聚落布局时宗教建筑的面积占比

信仰对文化的反馈通过历史事件：
  宗教改革 → CulturalShift 事件修改文化参数（通常是 religiosity、
  uncertainty_avoidance、openness_to_outsiders）
```

**信仰系统是独立模块**——不在文化模块的文档范围内。当前信仰的类型枚举和 NPC 虔诚度定义在 `[[../生命/013-神明与信仰|生命 013]]` 和参考文档 `[[../../想法/WoW World/生命/神明/NPC信仰系统|NPC信仰系统]]` 中。待信仰系统正式设计时，与文化的接口通过文化系统的 `religiosity` 参数和信仰系统的 `FaithId` 互相连接。

### 4.3 文化与语言表达的关系

**`CommunicationNorms` 所有权归文化系统**。文化系统定义 norms 的字段含义、派生公式和存储。语言表达系统通过 `CultureQuery` trait 只读获取 norms 值，并负责 norms 如何影响对话行为（抢话阈值、沉默处理、敬语生成、LLM 提示词构建）。

详见 [[003-文化沟通规范|003]] §一（所有权转移说明）。

## 五、文化区域空间模型

### 5.1 障碍 Voronoi 模型

文化与空间的关系采用**障碍 Voronoi 离散区域 + 渐变边界带**模型：

1. 世界种子在 P2.5 生成 3-8 个文化起源点（数量由种子决定，基于大陆面积和大陆数自动缩放）
2. 对每个起源点，计算到地图上所有点的**旅行代价距离**（非欧氏距离）：
   ```
   cost_distance(p, origin) = shortest_path(p, origin,
       edge_cost_fn = terrain_cost + water_barrier_penalty)
   ```
   山脉/海洋 = 高旅行成本 → 文化边界自然对齐地理边界
3. 每个点归属到代价距离最近的起源点，形成 Voronoi 式区域
4. 在边界带上（距离两个起源的代价比 < 1.5），使用加权混合——该点的文化特征为两侧文化的混合
5. 文化飞地：因移民/征服/流放而在异文化区域中形成的孤立同文化聚落（P8 历史模拟期间产生）

**为什么不用连续噪声场**：离散 CultureId 给 NPC 和玩家一个锚定的文化身份（"我是河湾地的赫拉人"）。纯连续场无法产生"文化边界"的体验——穿越边界时的建筑/服饰/问候语变化是玩家探索世界的关键仪式感。

### 5.2 渐变边界带

| 边界类型 | 条件 | 过渡带宽 | 混合方式 |
|----------|------|---------|---------|
| **硬边界** | 山脉/大河/海洋阻隔 | 0-500m | 几乎无混合（文化飞地除外） |
| **软边界** | 平原/丘陵连续区域 | 2-5km | 70/30 → 50/50 → 30/70 渐变 |
| **城市交汇** | 两个文化中心的聚落群在边界带相邻 | 局部复杂混合 | 点对点按聚落归属 |

渐变带中的文化混合不仅是学术概念——NPC 在混合区可能同时受两个文化 norms 的影响（在 A 文化聚落中用 A 的规则，去 B 文化的市场时用 B 的规则）。

## 六、文化演变概览

文化不是静态的。四种演变路径（详见 [[006-文化演变与跨模块接口|006 §三]]）：

| 路径 | 速度 | 触发 | 影响范围 |
|------|------|------|---------|
| **代际漂移** | 极慢（σ=0.003/年） | 每年自动 | 所有 10 个核心参数微调 |
| **贸易影响** | 慢（百年可感） | 有贸易联系的文化对之间 | 参数向对方收敛 |
| **征服强制** | 中（数十年） | 征服/殖民 | 被征服方向征服方参数偏移 |
| **事件驱动** | 快（数年内） | 重大历史事件（宗教改革/灾难/灭绝） | 特定参数单次显著偏移 |

**亚文化分裂**：隔离 > 200 年 + 参数距离 > 0.15 → 自动创建新 CultureId（子文化）
**克里奥尔化**：边界混合区持续 > 300 年 + 混合参数与双亲距离 > 0.2 → 新文化结晶
**文化消亡**：所有载体消逝 → 标记 extinct，谱系保留

箭头方向 = 数据流方向。所有模块从文化系统**拉取**数据（通过 `CultureQuery` trait），不向文化系统**推送**数据。文化参数的修改仅通过 `CultureMut` trait（`pub(crate)`）——仅世界生成管线和历史模拟引擎可调用。

## 八、关键数据结构汇总

```rust
/// 文化系统的顶层容器
pub struct CultureSystem {
    /// 所有文化的 SoA 存储（索引 = CultureId.0 as usize）
    store: CultureStore,
    /// 空间索引——四叉树
    spatial_index: CultureSpatialIndex,
    /// 谱系图
    genealogy: CultureGenealogy,
    /// 相似度矩阵缓存
    similarity_cache: RefCell<SimilarityCache>,
}

/// 内部存储——列式布局，缓存友好
struct CultureStore {
    cores: Vec<CultureCoreParams>,
    communication_norms: Vec<CommunicationNorms>,
    building_styles: Vec<BuildingStylePreferences>,
    beauty_standards: Vec<CulturalBeautyStandard>,
    honorific_systems: Vec<HonorificSystem>,
    fertility_norms: Vec<FertilityNorms>,
    dietary_prefs: Vec<DietaryBasePreferences>,
    technology_profiles: Vec<TechnologyProfile>,
    language_ids: Vec<LanguageId>,
    names: Vec<CultureName>,
}
```

每文化总计约 364 字节。100 个文化 ≈ 36KB——可忽略。
