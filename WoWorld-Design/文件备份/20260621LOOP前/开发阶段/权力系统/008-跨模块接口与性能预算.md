## 一、Trait 定义

### 1.1 PowerTopologyQuery —— 只读查询

所有模块都可以通过此 trait 安全地查询权力数据，不修改权力拓扑。

```rust
/// 权力拓扑只读查询接口
/// 任何模块都可获取此 trait 的引用
pub trait PowerTopologyQuery: Send + Sync {
    /// 查询某实体拥有的所有活跃权力边（出边）
    fn powers_of(&self, holder: EntityId) -> Vec<PowerEdgeRef>;

    /// 查询某实体承受的所有活跃权力边（入边）
    fn constraints_on(&self, subject: EntityId) -> Vec<PowerEdgeRef>;

    /// 查询某位置适用的领土权力边
    fn territorial_powers_at(&self, pos: WorldPos) -> Vec<PowerEdgeRef>;

    /// 查询某实体对某 holder 在特定 domain 上的合法性感知
    fn perceived_legitimacy(
        &self,
        subject: EntityId,
        holder: EntityId,
        domain: &PowerDomain,
    ) -> f32;

    /// 查询某实体所属的 Polity（惰性快照，可能滞后）
    fn polity_of(&self, entity: EntityId) -> Option<PolityId>;

    /// 查询两个 Polity 之间的外交关系
    fn diplomatic_relation(&self, a: PolityId, b: PolityId) -> Option<DiplomaticState>;

    /// 查询适用于某 subject 的 PrescribeRule（按规范层级排序）
    fn applicable_rules(
        &self,
        subject: EntityId,
        domain: &PowerDomain,
    ) -> Vec<PowerEdgeRef>;

    /// 查询某实体的免疫集合
    fn immunities_of(&self, entity: EntityId) -> ImmunitySet;

    /// 查询某实体负有的所有活跃 Duty
    fn duties_of(&self, obligor: EntityId) -> Vec<DutyRef>;

    /// 检查某实体是否对某 holder 的某原子操作拥有免疫
    fn has_immunity(
        &self,
        subject: EntityId,
        atom_type: AtomTypeTag,
        domain: &PowerDomain,
    ) -> bool;

    /// 查询某 Group 的治理规则
    fn governance_of(&self, group: GroupId) -> Option<&GovernanceRules>;

    /// 查询沿委托链的最终权威
    fn ultimate_authority(
        &self,
        holder: EntityId,
        atom_type: AtomTypeTag,
    ) -> Option<EntityId>;

    /// 查询某 Polity 的政府形式
    fn government_form_of(&self, polity_id: PolityId) -> Option<GovernmentForm>;

    /// 获取两个 Entity 之间的权力关系摘要（用于 UI/NPC 判断）
    fn power_relationship_summary(
        &self,
        a: EntityId,
        b: EntityId,
    ) -> PowerRelationshipSummary;
}

/// 轻量引用——不拥有数据，指向 PowerTopology 内部
#[derive(Debug, Clone, Copy)]
pub struct PowerEdgeRef {
    pub edge_id: EdgeId,
    pub holder: EntityId,
    pub subject: EntityId,
    pub atom_type: AtomTypeTag,
    pub legitimacy: f32,
    pub enforcement: f32,
    pub source: PowerSource,
}
```

### 1.2 PowerTopologyMut —— 可变操作

```rust
/// 权力拓扑可变操作接口
/// 仅权力系统自身和少数经授权的调度代码可获取
pub trait PowerTopologyMut {
    /// 行使权力原子——核心入口
    fn exercise_power(
        &mut self,
        holder: EntityId,
        subject: EntityId,
        atom_type: AtomTypeTag,
        params: AtomParams,
    ) -> Result<(PowerExerciseResult, Vec<PowerEvent>), PowerExerciseError>;

    /// 创建权力边（通常由 Constitute/Delegate/Contract/Pledge 内部调用）
    fn create_edge(&mut self, data: PowerEdgeData) -> Result<EdgeId, PowerError>;

    /// 软删除权力边
    fn deactivate_edge(&mut self, edge_id: EdgeId) -> Result<(), PowerError>;

    /// 创建 Duty（通常由 PrescribeRule/Extract/Compel/Pledge 自动触发）
    fn create_duty(&mut self, duty: Duty) -> DutyId;

    /// 为实体添加免疫
    fn grant_immunity(&mut self, entity: EntityId, immunity: Immunity);

    /// 撤销实体的免疫
    fn revoke_immunity(&mut self, entity: EntityId, immunity_id: ImmunityId);

    /// 处理 holder 死亡——执行继承规则
    fn on_holder_death(&mut self, deceased: EntityId) -> Vec<PowerEvent>;

    /// 触发合法性重算（通常由调度层按游戏日调用）
    fn recalc_legitimacy(&mut self, cache: &LegitimacyInputCache);
}
```

## 二、性能预算总表

### 2.1 CPU 预算

| 子系统 | 每帧 CPU | 触发频率 | 分摊方式 | 备注 |
|--------|---------|---------|---------|------|
| 权力行使 (`exercise_power`) | <0.01ms | 按需（稀疏事件） | 直接 | 每次行使 ~2µs-50µs，频次 <100/帧 |
| 领土查询（NPC 移动触发） | <0.01ms | 每帧 ~167 次边界跨越 | 直接 + Chunk 缓存 | 四叉树查询 ~5µs，LRU 命中 ~1µs |
| Legitimacy 重算 | <0.02ms | 每游戏日，500K 边 | 分摊到 2880 帧 | 每帧 ~174 条边 × ~100ns = ~17µs |
| Duty 条件检查 | <0.001ms | 每游戏日，50K 条件 Duty | 分摊到 2880 帧 | 每帧 ~17 条 × ~50ns = <1µs |
| 非行使衰减检查 | <0.01ms | 每游戏月 | 分摊到 ~43,200 帧 | 衰减原子 × 总边数 → 分摊后 <10µs/帧 |
| Polity 涌现重算 | <0.05ms | 每游戏年一次 | N/A（单帧完成） | 连通分量 O(V+E) ~5ms，仅 1 次/年 |
| DiplomaticRelation 重算 | <0.01ms | 每游戏月，≤5K Polity 对 | 分摊到 ~43,200 帧 | 每对 ~2µs → 10ms 总额，分摊后 <1µs/帧 |
| 边防创建/删除 | <0.005ms | 极罕见 | 直接 | 每次 ~50µs |
| **合计** | **<0.1ms/帧** | | | 占总 Rust 预算 7ms 的 ~1.4% |

### 2.2 内存预算

| 数据结构 | 大小 | 计算依据 |
|---------|------|---------|
| PowerTopology 共有列（10 字段） | ~48 MB | 500K 边 × ~96 bytes |
| 原子专属参数列（分表） | ~80 MB | 500K 边 × ~160 bytes（含 None 占位） |
| 索引（出边 + 入边 + 空间 + 原子类型） | ~20 MB | 500K 边 × ~40 bytes 分摊 |
| Duty 存储 | ~80 MB | 500K Duty × ~160 bytes |
| ContractRecord | ~5 MB | 5K 契约 × ~1KB |
| Polity 缓存 | ~5 MB | 100 Polity × ~50KB |
| DiplomaticRelation 缓存 | ~1 MB | 5K 对 × ~200 bytes |
| ImmunitySet | ~2 MB | 10K 免疫 × ~200 bytes |
| ChunkPowerCache (LRU) | ~4 MB | 1024 Chunk × ~4KB |
| LegitimacyInputCache（临时） | ~5 MB | 批量外部数据缓存 |
| **合计** | **~250 MB** | 占总内存预算 1.4GB 的 ~18% |

**注**：Duty 存储是最大的单项（~80MB）。如果优化，条件型 Duty 和持续型 Duty 可以合并为触发自动生成——不预先存储所有持续型 Duty（如"禁止偷窃"这类 never-trigger 的 Duty 在违反时才计算）。优化后内存可降至 ~180MB。

### 2.3 最坏情况分析

| 场景 | 影响 | 缓解 |
|------|------|------|
| 500 人乘船同时进入新领土 | 500 次领土查询集中在一帧 | 分帧处理——载具上 NPC 延迟绑定到下载具时 |
| 大型 War 宣战（10K×10K 成员 = 100M 临时权力边） | 不可创建全部 | 使用 **lazy evaluation**：不预创建战争权力边——仅在发生 Constrain/Extract 时检查 War 状态 |
| 一个 holder 持有极大量出边（如皇帝有 20K subject） | 出边索引查询 O(20K) | 使用 **分层索引**：按 domain 二次分桶 |
| legitimacy 重算时经济/文化系统不响应 | 重算超时 | 使用 `LegitimacyInputCache` 批量获取——一次调用，本地计算 |

## 三、经济系统桥接——PowerToEconomicBridge

```rust
/// 调度层组件——连接权力系统和经济系统
/// 不隶属任一模块
pub struct PowerToEconomicBridge;

impl PowerToEconomicBridge {
    /// 从权力拓扑向经济系统同步 MarketAuthority
    /// 每游戏日调用一次
    pub fn sync_market_authority(
        topology: &dyn PowerTopologyQuery,
        economy: &mut dyn EconomicSystemMut,
    ) {
        for holder in topology.all_holders() {
            let economic_powers = topology.powers_of(holder)
                .iter()
                .filter_map(|edge| Self::convert_to_economic(edge))
                .collect::<Vec<_>>();
            
            if !economic_powers.is_empty() {
                economy.update_authority(holder, economic_powers);
            }
        }
    }

    /// 将普适权力原子映射为经济系统 PowerAtom
    /// 
    /// 注：13 个经济 PowerAtom 中，4 个有普适原子对应（SetTaxRate, ToggleItemBan, 
    /// GrantLicense, SetPriceCeiling），其余 9 个（SetPriceFloor, SetQuota, Requisition,
    /// Exclude, IssueCurrency, AdjudicateDispute, DelegateAuthority, SetTariff, 
    /// SetMandatoryQuota）保留为经济系统专属操作——它们直接由经济系统 006 
    /// exercise_power 处理，不经普适原子桥接。
    fn convert_to_economic(edge: &PowerEdgeRef) -> Option<EconomicPowerAtom> {
        match (edge.atom_type, &edge.domain) {
            // Extract → 经济征税/收费
            (AtomTypeTag::Extract, PowerDomain::Market { .. })
                | (AtomTypeTag::Extract, PowerDomain::Universal) =>
                Some(EconomicPowerAtom::SetTaxRate),
            
            // Constrain → 经济禁售/限购
            (AtomTypeTag::Constrain, PowerDomain::Behavior { behavior_type: BehaviorType::Trade, .. }) =>
                Some(EconomicPowerAtom::ToggleItemBan),
            
            // Access → 经济许可
            (AtomTypeTag::Access, PowerDomain::Market { .. }) =>
                Some(EconomicPowerAtom::GrantLicense),
            
            // PrescribeRule → 经济体制参数（间接——通过市场规则）
            (AtomTypeTag::PrescribeRule, PowerDomain::Market { .. }) =>
                Some(EconomicPowerAtom::SetPriceCeiling),  // 或其他监管参数
            
            _ => None,
        }
    }
}
```

## 四、事件发射契约

权力系统不直接调用历史系统/语言表达系统——而是返回事件列表，调度层分发。

```rust
/// 权力行使结果——exercise_power 的返回值，也是 PowerEvent 的 data 字段
#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum PowerExerciseResult {
    /// 权力边成功创建
    EdgeCreated { edge_id: EdgeId },
    /// 权力行使成功
    Exercised { edge_id: EdgeId, atom: AtomTypeTag },
    /// Pledge 创建成功
    PledgeCreated { edge_id: EdgeId, duty_id: DutyId },
    /// Contract 创建成功
    ContractCreated { contract_id: ContractId, edge_a: EdgeId, edge_b: EdgeId },
    /// 权力边被软删除
    EdgeDeactivated { edge_id: EdgeId, reason: String },
    /// 权力边恢复活跃
    EdgeReactivated { edge_id: EdgeId },
    /// 制裁已被执行
    SanctionApplied { method: SanctionMethod, target: EntityId },
    /// Adjudicate 裁决完成——为败诉方创建 Duty
    RulingMade { winner: EntityId, loser: EntityId, created_duty: DutyId },
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum PowerEvent {
    // 权力变化
    PowerExercised { holder: EntityId, subject: EntityId, atom: AtomTypeTag, result: PowerExerciseResult },
    PowerInherited { edge_id: EdgeId, from: EntityId, to: EntityId },
    PowerReverted { edge_id: EdgeId, to: EntityId },
    PowerRelinquished { edge_id: EdgeId, holder: EntityId },
    PowerConquered { edge_id: EdgeId, conqueror: EntityId, vanquished: EntityId },
    
    // 结构变化
    EdgeCreated { edge_id: EdgeId },
    EdgeExpired { edge_id: EdgeId, holder: EntityId, subject: EntityId, atom_type: AtomTypeTag },
    EdgeExtinguished { edge_id: EdgeId, reason: String },
    EdgeDecayedToExtinction { edge_id: EdgeId },
    
    // 合法性
    LegitimacyCrisis { holder: EntityId, unenforced_count: u32 },
    RevolutionRisk { risk: RevolutionRisk },
    
    // 涌现
    EmergentPowerDetected { edge_id: EdgeId, holder: EntityId, subject: EntityId },
    PolityChange(PolityChangeEvent),
    
    // 领土
    EntityEnteredTerritory { entity: EntityId, holder: EntityId, territory_edge: EdgeId },
    TerritorialDispute { dispute: TerritorialDispute },
    
    // 继承
    SuccessionCrisis { edge_id: EdgeId, deceased: EntityId },
    ElectionNeeded { edge_id: EdgeId, electorate: EntityId, reason: String },
    
    // 战争
    WarPowerGranted { edge: EdgeId },
    TradeFrozen { edge: EdgeId },
    WarDeclared { polity_a: PolityId, polity_b: PolityId },
    PeaceTreatySigned { contract_id: ContractId, polity_a: PolityId, polity_b: PolityId },

    // 一般性事件
    ContractSigned { contract_id: ContractId, party_a: EntityId, party_b: EntityId },
    RuleConflictAdjudicationNeeded { rule_a: EdgeId, rule_b: EdgeId, subject: EntityId },
    DutyBreached { duty_id: DutyId, obligor: EntityId, breach: DutyBreach },
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum PolityChangeEvent {
    PolityBirthed { polity_id: PolityId, name: String, government_form: GovernmentForm },
    PolityDied { polity_id: PolityId, name: String, reason: String },
    PolitySplit { original: PolityId, successors: Vec<PolityId> },
    PolityMerged { absorbed: PolityId, into: PolityId },
    PolityEvolved { polity_id: PolityId, from: GovernmentForm, to: GovernmentForm },
}
```

## 五-B、世界生成 Bootstrap 接口 ★ CHG-045 新增

> **详细规格**: [[../../../参考文档/037-世界生成重构跨模块补充需求-20260620/004-权力系统补充需求|037-004 权力系统补充需求]]

**PowerEdgeTemplate** — 世界生成 P5 产出的权力边模板（角色级，不引用 NpcId），P10 实例化为 PowerEdge:

```rust
pub struct PowerEdgeTemplate {
    pub holder_role_index: usize,       // P5 RoleTemplate.role_index
    pub subject_role_index: usize,
    pub atom_type: AtomTypeTag,
    pub domain: PowerDomain,
    pub source: PowerSource,
    pub legitimacy_initial: f32,        // <1.0, 留运行时波动空间
    pub cardinality: EdgeCardinality,   // SinglePair/OneToMany/ManyToOne
    pub selection_preference: SelectionPreference,
}
```

**Bootstrap 接口**: `bootstrap_power_topology(skeleton, roles, npcs, families, settlements, culture, rng) → PowerTopology` — 模板→边实例化 + emerge_polities()。

**选择算法**: NPC 加权评分 = `social_standing×w + age_midpoint×w + conscientiousness×w + noise×w`。高责任心者更可能被选为统治者。
