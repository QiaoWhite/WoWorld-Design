## 一、模块定位与设计哲学

### 1.1 为什么需要这个模块

WoWorld 当前有完整的数据层（权力拓扑、经济账本、关系图）和表达层（文本生成、语音输出），但缺少一个**概念中间层**。

当 NPC 形成记忆"我的儿子去铁匠铺工作，我不用给他钱了"——

- 权力拓扑中有客观事实：`Contract(铁匠铺主, 儿子) + Extract(labor) + Extract(wage)`
- 文本生成器可以产出字符串
- 但中间的**概念表示**缺失了：NPC 如何从客观事实中识别"雇佣"这个概念？"儿子""铁匠铺""钱"——这些词汇从何而来？因果关系"因为……所以不用给钱了"是怎么构成的？

本模块填补这个空白。

### 1.2 模块定位

**概念与语言地基系统是 WoWorld 的概念表示层——连接"客观世界数据"和"自然语言表达"之间的空白地带。**

六个核心职责：
1. **概念识别** — 从客观数据模式中识别概念（PatternSignature → ConceptLocalId）
2. **概念空间管理** — 文化相对的概念空间定义、混合概念空间派生
3. **结构化话语** — Utterance 作为 NPC 语言产出的统一数据格式
4. **语言渲染** — TextGenerator 统一管道（对话/自语/书籍三个入口）
5. **语言能力模拟** — proficiency 习得、衰减、文字系统独立维度
6. **跨代知识传递** — 六条传递路径、保真度数学模型、概念漂移

### 1.3 本模块不是什么

- ❌ 不是 LLM 调用层——LLM 在概念层操作（可选），本模块定义确定性管道
- ❌ 不是文本生成器本身——TextGenerator 在语言表达模块，本模块定义概念→文本的接口
- ❌ 不是书籍存储系统——书籍归历史模块，本模块只定义概念引用格式
- ❌ 不是 NPC 记忆系统——记忆归 NPC 模块，本模块定义记忆中的概念编码格式
- ❌ 不是权力/经济系统——本模块只消费它们的客观数据，不修改

### 1.4 设计原则

| 原则 | 含义 |
|------|------|
| **思维是结构化数据，语言是惰性渲染** | NPC 内心以概念思考，自然语言仅在需要外显时生成 |
| **概念先于词语** | 概念是语言无关的，词语是概念在特定语言中的投影 |
| **文化相对概念空间** | 同一客观模式，不同文化可能识别为零个或多个不同概念 |
| **翻译 = 模式重叠搜索** | 源概念的模式覆盖投影到目标文化概念空间，找最大重叠 |
| **语言 proficiency 从原子日志涌现** | LISTEN/SPEAK/READ/WRITE/EXAMINE 物理原子驱动 proficiency |
| **概念习得从暴露累积涌现** | 新概念的暴露累积超过阈值 → 习得，初始低置信 |
| **零新门控，零硬编码路径** | 所有差异从连续参数的数学交互中涌现 |
| **概念识别在集成层** | 各业务模块不知道"概念"的存在——它们只产出客观数据 |
| **玩家 = NPC（贯彻到语言层）** | 同一套 Utterance、proficiency、概念空间 |

## 二、核心架构

### 2.1 三层概念模型

```
Layer 0: 地面真相（客观、全球统一）
  PatternSignature(u64) — 从权力拓扑/经济/关系中数学提取的模式指纹
  不存在"雇佣"这个概念——只存在"Contract+Extract(labor)+Extract(wage)"这个模式
  这是全球客观的，可以被任何模块消费

Layer 1: 文化概念空间（CultureId 相对）
  每个文化有自己的概念清单
  概念 = (CultureId, PatternCover 聚类, 概念边界)
  "在中原文化中，模式 #0x3F7A 被识别为一个独立概念，叫做……"
  "在兽人文化中，模式 #0x3F7A 不被独立识别——它是模式 #0x1A2B 的子集"

Layer 2: 语言词汇（LanguageId × ConceptId 相对）
  概念在特定语言中的词语投影 + 每种语言独立的片段库和构词规则
```

### 2.2 Crate 架构

```
woworld_types/          ← 纯类型，零依赖
  + PatternSignature(u64), ConceptLocalId(u16)
  + CultureConceptDef, CompactConceptEncoding([u8;64])
  + Utterance, UtteranceId, RenderContext
  + ConceptFact, MentalModelRef, DeductiveConclusion
  + ResidenceRecord
  + EntityIndex::close_relations() — 新增方法

woworld_core/           ← 纯数学函数，仅依赖 woworld_types
  + compute_pattern_signature()
  + classify_pattern()
  + pattern_overlap()
  + deductive_chain()
  + transmission_fidelity()
  + format_number()
  + compute_language_exposure()  ← ★ XD1修正: 纯函数，仅依赖 types

woworld_lang/           ← 依赖 woworld_types + TOML，不依赖任何业务 crate
  + TextGenerator (三模式: render_utterance / generate / render_thought)
  + translate_concepts()
  + effective_concept_space()
  + concept_space_drift()
  + FragmentLibrary, MorphologyRegistry, ConceptLexicon
```

**依赖方向（零循环）**：

```
woworld_types ← woworld_core ← woworld_lang
woworld_types ← 业务模块 (NPC/权力/经济/历史/...)
集成层 → 全部
```

### 2.4 词汇架构概览（★ XO6修正）

|------|---------|------|---------|------|

## 三、执行时序

### 3.1 每决策周期 (0.3s, L1 NPC)

```
├─ AgentSnapshot::from_sources() — 派生 116B 快照（含语言/文字 proficiency）
├─ evaluate_deliberation() — 检测是否需要慢思考
├─ GOAP + 行为树 — 选择行动（已有）
├─ 行动执行 → 原子日志追加
└─ 如果有社会事件 (集成层检测):
    ├─ compute_pattern_signature(event)
    ├─ classify_pattern(sig, npc.effective_concept_space())
    ├─ memory_encode_with_deduction() — 浅层演绎 (depth≤2)
    └─ CompactConceptEncoding 写入 EventMemory
```

### 3.2 每游戏日 (L1 + L2 NPC)

```
├─ compute_language_exposure() — 从原子日志计算语言暴露
├─ update_language_proficiency() — 语言 proficiency 增长
├─ update_script_proficiency() — 文字 proficiency 增长
├─ update_concept_acquisition() — 概念习得检查
├─ apply_language_decay() — 语言衰减
├─ life_event_cognitive_shift() — 认知方向偏移（仅当日有重大事件）
└─ concept_space_version bump — 检测概念空间变更
```

### 3.3 每 7 游戏日 (L1 NPC, 随 SelfNarrative::reflect)

```
├─ macro_digest() — 已有
├─ meta_cognitive_review() — 已有
└─ wisdom_effective() 重新评估
```

### 3.4 每年（聚落级）

```
├─ settlement_knowledge_health() — 重算
├─ concept_space_drift() — 代际漂移
└─ knowledge_preservation_impulse() — 检查是否有 NPC 触发
```

P8 人口生成时调用——NPC 语言/文字 proficiency 初始化：
- 母语 proficiency = 0.8 + education×0.2；邻近语言 learn_prob = extraversion×0.5 + trade_contact×0.3 - distance_decay
- 文字 proficiency = education×0.8 + social_complexity×0.2 + profession_mod
- Utterance 风格 = culture formality×0.7 + power_distance×0.3 + BigFive 调制
- 概念空间 = settlement_tech 已知概念 + 职业相关概念（幼儿<5，成人≤20，受 cognitive_damping 调制）
