# 模块接头总览 (Module Interface Hub)

> **定位**: 介于 [[../../../../CLAUDE-INTERFACES.md|CLAUDE-INTERFACES]]（契约宪法）和具体模块文档之间的"生活地图"
> **维护**: 随模块文档修改和新增而**动态更新**——源文档变则接头条目必须同步变
> **创建**: 2026-06-18 技术栈全量审计 ([[../../../参考文档/031-技术栈全量审计-20260618/|审计目录]])

## 使用指南

- **设计新模块时** → 先查接头总览，知道有哪些现成接口可消费
- **修改旧模块时** → 查"变更影响链"，立即知道改动会波及谁
- **发现模块间矛盾时** → 入口消费清单 vs 出口提供清单不一致 = 冲突

## 目录结构

```
00-全局基础设施/     — 跨模块共享的类型和trait (woworld_core)
01-生命/             — 每个模块: 出口/入口/影响链/变更日志
02-NPC活人感/
...
24-概念与语言地基/
25-大气与氛围/       — CHG-053 新建
变更追踪/            — 待处理变更/影响地图/已解决日志
```

参见 [[../../../../CLAUDE.md|CLAUDE.md §节点总览管理]]

## 保鲜规则

1. 模块文档修改 → 接头条目必须同步修改（原子变更）
2. 每条条目有时间戳（最后验证 + 最后修改 + 源文档版本）
3. 每模块维护 `000-变更日志.md`
4. 模块文档的"外部接口"段落应反向链接接头条目
5. 新 CHG 必须填写"接头总览更新"段落
