## 一、文化→家具的参数派生链

```
CultureCoreParams (10参数, P2.5)
  ↓ 第一层派生
CultureFurnishProfile (P2.5e 计算)
  ↓ 第二层: 房间级约束
FurnishEnsemble::roll() (P11.5b 每房间掷一次骰子)
  ↓ 第三层: 逐件采样
FurnitureVariant::sample() (每件家具独立采样, 约束于 Ensemble)
```

**三层的原因**: 第一层（CultureFurnishProfile）是文化级参数——全文化相同。第二层（FurnishEnsemble）是房间级约束——确保同一房间的家具审美协调。第三层是逐件选择——允许轻微个体差异（如品质扰动）。

## 二、CultureFurnishProfile 完整定义

```rust
/// 从 CultureCoreParams 派生的家具文化参数——完全类比 CultureBuildProfile
#[derive(Debug, Clone)]
pub struct CultureFurnishProfile {
    /// 家具材质偏好 — 权重列表
    pub material_preference: Vec<(MaterialClass, f32)>,
    
    /// 家具形态风格 — 直线/曲线/堆叠/编织/雕凿
    pub form_style: FurnishFormStyle,
    
    /// 装饰密度 — 从 artistry 直接派生
    pub decoration_density: f32,
    
    /// 颜色调色板 — 从 CultureCoreParams + 群系色彩 派生
    pub palette: ColorPalette,
    
    /// 座位偏好高度 (m) — 从种族 avg_height × 文化 派生
    pub seat_height_range: (f32, f32),
    
    /// 桌高偏好 (m)
    pub table_height_range: (f32, f32),
    
    /// 排列对称性
    pub symmetry: f32,
    
    /// 房间分隔偏好
    pub room_division: RoomDivisionStyle,
    
    /// 地板生活倾向 — 0.0=全座椅文化, 1.0=全地板文化
    pub floor_living: f32,
    
    /// 床上用品风格
    pub bedding_style: BeddingStyle,
    
    /// 存储偏好 — 柜/箱/壁龛/挂架 的权重
    pub storage_style: Vec<(StorageType, f32)>,
    
    /// 花纹主题
    pub ornament_motif: OrnamentMotif,
    
    /// 器物尺度倾向 — 0.0=紧凑, 1.0=宽大
    pub scale_preference: f32,
    
    /// 手工技艺倾向 — 0.0=粗犷(功能), 1.0=精致(细工)
    pub craftsmanship: f32,
}
```

### 2.1 派生公式（显式、确定性）

每个字段从 `CultureCoreParams` 显式派生——无隐式假设，同种子必同结果：

```rust
impl CultureFurnishProfile {
    pub fn derive(
        params: &CultureCoreParams,
        race: &RaceBodyProfile,
        climate: &ClimateBuildProfile,
    ) -> Self {
        let floor_living = Self::derive_floor_living(params, climate);
        
        Self {
            // ━━ 材质偏好 ━━
            // 从文化 + 气候 + 种族体型综合推导
            material_preference: Self::derive_material_preference(params, climate),
            
            // ━━ 装饰 ━━
            decoration_density: params.artistry,
            craftsmanship: params.artistry * 0.7 + params.long_term_orientation * 0.3,
            // 精致工艺 = 审美追求 + 愿意花时间
            
            // ━━ 形式 ━━
            form_style: Self::derive_form_style(params, race),
            ornament_motif: Self::derive_ornament_motif(params),
            
            // ━━ 颜色 ━━
            palette: Self::derive_palette(params),
            
            // ━━ 尺度 ━━
            scale_preference: params.power_distance * 0.4 + params.artistry * 0.3 + 0.3,
            // 高权力距离→宏大尺度；高审美→宽敞
            
            // ━━ 座高/桌高 —— 种族基础 × 文化修正 ━━
            seat_height_range: (
                race.avg_height * 0.05 * (1.0 + floor_living * 6.0),
                race.avg_height * 0.28,
            ),
            table_height_range: (
                race.avg_height * 0.12 * (1.0 + floor_living * 3.0),
                race.avg_height * 0.48,
            ),
            
            // ━━ 布局 ━━
            symmetry: 1.0 - params.individualism * 0.5,
            room_division: Self::derive_room_division(params, climate),
            floor_living,
            
            // ━━ 卧具 ━━
            bedding_style: Self::derive_bedding(params, climate),
            
            // ━━ 存储 ━━
            storage_style: Self::derive_storage(params, climate),
        }
    }
    
    fn derive_floor_living(params: &CultureCoreParams, climate: &ClimateBuildProfile) -> f32 {
        // 地板生活 = 炎热(+) + 克己(+) - 湿润(−) - 寒冷(−)
        let temp = (climate.temp_range.1 - 20.0) / 30.0; // 20°C→0, 35°C→0.5, -5°C→-0.83
        let indulgence = -params.indulgence;
        let humidity = (climate.precipitation / 2000.0).min(1.0) * -0.3;
        (temp * 0.4 + indulgence * 0.4 + humidity + 0.3).clamp(0.0, 1.0)
    }
    
    fn derive_material_preference(
        params: &CultureCoreParams, 
        climate: &ClimateBuildProfile
    ) -> Vec<(MaterialClass, f32)> {
        let mut prefs = Vec::new();
        
        // 基础: 所有文化都可用木材、石料、织物
        // 调整:
        //   artistry 高 → 偏好精加工材质 (CarvedStone/PolishedWood/Metal)
        //   long_term 高 → 偏好耐久材质 (Stone/Metal > Wood)
        //   floor_living 高 → 偏好软材质 (Fabric/Carpet > Wood)
        //   寒冷气候 → 偏好保温材质 (Wood/Textile > Stone)
        //   炎热气候 → 偏好凉爽材质 (Stone/Marble)
        
        // ...
        prefs
    }
    
    // ... 其他 derive_* 函数 ...
}
```

## 三、FurnishEnsemble — 房间级审美约束

**为什么需要这个**：如果每件家具独立采样，可能产生精灵椅+矮人桌+人类床共处一室。Ensemble 确保同一房间内的所有家具共享一致的审美DNA。

```rust
pub struct FurnishEnsemble {
    /// 主材 — 此房间内多数家具的框架材质
    pub dominant_material: MaterialClass,
    /// 辅材 — 座垫/桌面覆层/装饰嵌件
    pub secondary_material: MaterialClass,
    /// 榫接风格 — 榫卯/钉接/胶合/雕刻连接
    pub joinery_style: JoineryStyle,
    /// 色彩 — 主色+辅色+点缀色
    pub color_scheme: [u8; 3],      // HSL 色相 × 3
    /// 花纹主题
    pub ornament_motif: OrnamentMotif,
    /// 目标品质等级
    pub quality_target: f32,
}

impl FurnishEnsemble {
    /// 一次掷骰——全房间共享
    pub fn roll(
        culture: &CultureFurnishProfile,
        owner: &OwnerBuildPrefs,
        materials: &MaterialAvailability,
        rng: &mut impl Rng,
    ) -> Self {
        Self {
            // 主材 = 从本地可用材料中按文化权重采样
            dominant_material: sample_from_weights(
                culture.material_preference.iter()
                    .filter(|(m, _)| materials.is_available(*m))
            ).unwrap_or(MaterialClass::Wood),
            
            // 辅材 = 与主材兼容的次级材质
            secondary_material: compatible_secondary(
                dominant_material, culture, rng
            ),
            
            // 榫接 = f(技术时代, 材料)
            joinery_style: derive_joinery(dominant_material, culture.craftsmanship),
            
            // 色彩 = 文化调色板 + 微小噪声（±5° 色相）
            color_scheme: culture.palette.sample(rng).with_noise(5.0),
            
            ornament_motif: culture.ornament_motif,
            
            // 品质 = 业主财富 × 文化艺术追求
            quality_target: (owner.wealth_level * 0.7 + culture.decoration_density * 0.3)
                .clamp(0.1, 1.0),
        }
    }
    
    /// 检查家具变体与此 ensemble 的兼容性
    pub fn compatibility_score(&self, variant: &FurnitureVariant) -> f32 {
        let mut score = 1.0;
        
        // 材质兼容
        if variant.dominant_material != self.dominant_material {
            score *= 0.3;  // 降权但不禁止——混搭可以，但极罕见
        }
        
        // 装饰风格兼容
        if variant.ornament_motif != self.ornament_motif {
            score *= 0.5;
        }
        
        // 品质偏差
        let quality_diff = (variant.quality - self.quality_target).abs();
        score *= (1.0 - quality_diff * 0.5);
        
        score
    }
}
```

**涌现**: 精灵可能因个人经历偏好矮人工艺——概率低但不为零。Ensemble 的 `compatibility_score` 用乘积降权而非硬过滤，就是为这种罕见涌现留空间。

## 四、家具变体选择流程

```
逐件家具选择:
1. 从 FurnitureManifest 获取此槽位的候选变体列表
2. 按权重采样一个变体（权重 = 已有权重 × ensemble.compatibility_score(variant)）
3. 如果所有变体的兼容分数 = 0 → 回退到"最兼容的那个"（强行匹配）
4. 品质 = ensemble.quality_target + skill_noise(creator_artisan_skill)
5. 材料 = ensemble.dominant_material——但可因本地稀缺而降级（降级概率 = 1 - availability）
6. 生成 FurnishParams → morph weights → 最终 visual
```

## 六、与 BuildingStylePreferences 的关系

两者来自同一个 CultureCoreParams，但服务不同模块：

| | BuildingStylePreferences | CultureFurnishProfile |
|---|---|---|
| **定义位置** | 文化系统 004 | 文化系统 004 (新增) |
| **消费者** | 建筑模块 WFC 生成 (P6) | 家具 Pass B (P11.5b) |
| **共享参数** | `palette`, `decoration`, `symmetry` | `palette`, `decoration_density`, `symmetry` |
| **特有参数** | `roof` / `wall_material` / `scale` | `floor_living` / `seat_height_range` / `storage_style` |

**不合并**：建筑的 scale（天花板高、门高）和家具的 scale（座高、桌高）虽然都从 avg_height 派生，但建筑模块不应 import 家具概念。
