# 002 — 市场与订单簿

## 一、市场模型：供需匹配机制，不是物理地点

### 1.1 核心洞察

"市场"首先是供需匹配的抽象概念。铁匠在他的铺子里卖剑是市场，两个从未见面的人通过中间商撮合也是市场。物理集市只是市场的一个实现形式。

```
市场 = 订单簿 + 地理范围 + 交易模式组合
店铺 = 交易的原子单位（一个NPC控制的一个销售点）
集市 = 多个店铺在同一地点的聚集（涌现）
```

### 1.2 Market 数据结构

```rust
/// 市场——供需匹配机制的抽象
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Market {
    pub market_id: MarketId,
    pub name: String,

    /// 市场范围——决定覆盖谁
    pub scope: MarketScope,

    /// 订单簿核心
    /// 买方出价——按 limit_price DESC 排序（出价最高的排最前）
    pub bids: BTreeMap<ItemDefId, Vec<MarketListing>>,
    /// 卖方要价——按 limit_price ASC 排序（要价最低的排最前）
    pub asks: BTreeMap<ItemDefId, Vec<MarketListing>>,

    /// 交易历史——滑动窗口，保留最近200笔
    pub trade_history: VecDeque<TradeRecord>,

    /// 最近成交价——"市场价格"的唯一来源
    pub last_price: BTreeMap<ItemDefId, u32>,

    /// 价格波动率——最近20笔交易的变异系数
    pub price_volatility: BTreeMap<ItemDefId, f32>,

    /// 此市场支持的交易模式
    pub supported_modes: Vec<TransactionMode>,

    /// 已知参与者集合
    pub participants: BTreeSet<NpcId>,

    /// 市场参数
    pub params: MarketParameters,

    /// 市场规则（管制）
    pub regulations: MarketRegulations,

    /// 市场状态
    pub status: MarketStatus,
}

/// 市场的地理范围
#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum MarketScope {
    /// 本地集市——覆盖一个聚落或城区的实体交易场所
    LocalBazaar {
        settlement: SettlementId,
        district: Option<DistrictId>,
        /// 此集市的固定营业日（如每旬三六九为赶集日）
        market_days: Vec<DayOfMonth>,
    },

    /// 区域市场——覆盖多个聚落的远程交易
    RegionalMarket {
        region: RegionId,
        hubs: Vec<SettlementId>,
    },

    /// 行业市场——覆盖整个产业的买家和卖家
    /// 如"北方铁矿行业市场"——矿主、铁匠、铁器商人都参与
    TradeNetwork {
        item_categories: Vec<ItemCategory>,
        coverage: RegionId,
    },

    /// 地下市场——无物理位置，纯关系网
    Underground {
        loose_network: Vec<NpcId>,
    },
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum MarketStatus {
    Active,
    Dormant,      // 交易量长期低迷
    Closed,        // 被权力强制关闭
    Emerged,       // 刚从交易中涌现，尚未被权力正式承认
}
```

### 1.3 市场的涌现与衰退

市场不是world gen时被"创建"的——从交易行为中涌现：

```
萌芽: 几个NPC在村口大树下摆摊(CurbSide)
  → 路过的人发现"大树下能买卖"
  → 更多卖家聚集

成熟: 交易密度超过阈值 → 系统识别为 de facto LocalBazaar
  → 领主可能正式承认（发许可证、开始收摊位费）

衰退: 交易密度下降（饥荒/战争/人口流失）
  → 摊贩逐渐减少 → 降级回"几个CurbSide"

关闭: 权力强制关停或经济崩溃
  → 摊贩转到其他地点或转入地下(Underground)
```

## 二、店铺——交易的原子单位

### 2.1 核心概念

绝大多数日常交易不在集市——在铁匠铺、面包房、药水店、农民家门口的菜摊。**开店不需要"去市场找摊位"**——任何NPC，在任何地点，都可以挂出展示物品。

```rust
/// 店铺/摊位——一个NPC个体控制的销售点
/// 交易的原子单位
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Storefront {
    pub storefront_id: StorefrontId,
    pub owner: NpcId,
    pub name: String,                    // "老张铁匠铺"
    pub location: StorefrontLocation,

    /// 店铺自己的订单簿——不依赖任何Market
    pub bids: Vec<MarketListing>,        // 收购单（"长期收铁矿石"）
    pub asks: Vec<MarketListing>,        // 出售单（店铺陈列的商品）

    /// 经营参数
    pub opening_hours: OpeningHours,
    pub price_modifier: f32,            // 店铺相对基准价的系统性偏差
    /// >1.0 = 精品店（贵但品质好）
    /// <1.0 = 平价店（薄利多销）
    pub service_quality: f32,           // 0-1 受店主技能+人格影响
    pub regular_customers: Vec<NpcId>,  // 熟客——享受折扣或优先

    /// 可见性
    pub visibility: StorefrontVisibility,

    /// 物理表现——路过就能看到，不是UI菜单
    pub display_items: Vec<ItemDefId>,  // 展示的商品
    pub signage: Option<String>,        // 招牌文字
    pub visual_state: StorefrontVisual,

    /// 生命周期
    pub founded_at: GameTimestamp,
    pub monthly_revenue_copper: u64,
    pub is_active: bool,
}

/// 店铺位置——灵活多样
#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum StorefrontLocation {
    /// 正式建筑——关联建造系统
    PermanentBuilding { building_id: BuildingId, room_id: Option<RoomId> },

    /// 集市摊位——在LocalBazaar中租的固定摊位
    MarketStall { market_id: MarketId, stall_number: u16 },

    /// 家门口摆摊——不需要任何审批
    CurbSide { address: LocationId, description: String },

    /// 流动摊贩——推车/背包，没有固定位置
    Itinerant { current_position: Vec3, next_destinations: Vec<LocationId> },

    /// 行商——走街串巷，在酒馆/客栈展示商品
    Peddler { frequented_venues: Vec<LocationId> },
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct OpeningHours {
    pub open_hour: f32,     // 0-24 游戏时
    pub close_hour: f32,
    pub market_day_only: bool, // 仅赶集日开张
    pub closed_on_holidays: bool,
    pub currently_open: bool,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum StorefrontVisibility {
    Public,         // 任何过路人都能看到
    RegularsOnly,   // 仅熟客
    Hidden,         // 隐藏（地下交易）
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum StorefrontVisual {
    Open,           // 正常营业——展示商品可见
    Closed,         // 打烊——仍可看到招牌
    Abandoned,      // 废弃——可被新主人认领
    BoardedUp,      // 封门——被强制关闭
}
```

### 2.2 店铺与Market的关系

- 一个 `CurbSide` 店铺不属于任何Market——它是独立存在的交易点。路过的人可以直接买
- 多个CurbSide在同一地点聚集 → 可能涌现为 LocalBazaar
- `MarketStall` 类型的店铺显式关联到一个LocalBazaar——租用集市摊位
- `Itinerant` 和 `Peddler` 可能在多个Market之间移动

**关键：店铺的订单簿是独立的。** 不需要依赖于Market。Market的订单簿可以将范围内所有可见店铺的挂单聚合为统一视图——但撮合仍然发生在每一对订单之间，不管它们来自哪个店铺。

## 三、订单簿撮合引擎

### 3.1 MarketListing —— 单条挂单

```rust
/// 一条市场挂单（订单簿中的一行）
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct MarketListing {
    pub listing_id: ListingId,           // u64
    pub owner: NpcId,
    pub item_def: ItemDefId,
    pub item_ent: Option<ItemEntId>,     // None = 泛指此类物品
    pub quantity: u32,
    pub side: OrderSide,
    pub limit_price_copper: u32,         // 限价铜币/单位
    pub created_at: GameTimestamp,
    pub expires_at: Option<GameTimestamp>,
    pub visibility: ListingVisibility,
    pub listing_type: ListingType,
    pub status: ListingStatus,
    pub filled_quantity: u32,
    /// 此挂单关联的店铺（如果是通过店铺挂出的）
    pub storefront_id: Option<StorefrontId>,
}

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub enum OrderSide { Bid, Ask }

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub enum ListingVisibility {
    Public,                          // 对所有人可见
    Restricted(Vec<NpcId>),          // 仅对列表中NPC可见（黑市/熟人）
    Hidden,                          // 挂单存在但不可见（政府托底买单）
}

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub enum ListingType {
    Normal,            // 普通挂单
    Urgent,            // 紧急挂单——饥饿/重伤驱动，撮合优先，价格向非紧急方倾斜
    StandingOrder,     // 常备挂单——"持续收购铁矿石50铜/块"——长期有效
    GovernmentQuota,   // 政府配给挂单——受配额/限价约束
    BarterOffer,       // 以物易物——bid方列出提供的物品而非铜币
    LoanRequest,       // 借款请求——"我用抵押物换铜币，承诺分期还"
    IOUTrade,          // 欠条交易——买卖已有的还款承诺
}

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub enum ListingStatus {
    Active,
    PartiallyFilled,
    Filled,
    Cancelled { by: NpcId, reason: String, at: GameTimestamp },
    Expired,
}
```

### 3.2 撮合引擎

撮合在每次新挂单提交时触发（事件驱动，不等tick）。撮合流程是 `O(n)` 线性扫描——同类物品队列已预排序。

```rust
/// 撮合引擎——每次新挂单提交时调用
/// 价格涌现的核心：bid >= ask → 即刻撮合成交
fn match_orders(
    market: &mut Market,
    new_listing: &MarketListing,
    inventory: &mut dyn InventoryAccess,
    wallets: &mut dyn WalletAccess,
) -> Vec<TradeRecord> {
    let mut trades = Vec::new();
    let mut remaining_qty = new_listing.quantity;

    match new_listing.side {
        OrderSide::Ask => {
            // 新卖单 vs 现有买单（按价格DESC排序）
            let bids = market.bids.get_mut(&new_listing.item_def);
            if bids.is_none() { return trades; }
            let bids = bids.unwrap();

            for bid in bids.iter_mut() {
                if bid.status != ListingStatus::Active { continue; }
                // 买方出价 < 卖方要价 → 后续所有出价都更低 → 停止
                if bid.limit_price_copper < new_listing.limit_price_copper { break; }

                let (trade_qty, price) =
                    execute_trade(bid, new_listing, remaining_qty);
                trades.push(record_trade(bid, new_listing, trade_qty, price, market));
                remaining_qty -= trade_qty;
                if remaining_qty == 0 { break; }
            }
        }
        OrderSide::Bid => {
            // 新买单 vs 现有卖单（按价格ASC排序）
            let asks = market.asks.get_mut(&new_listing.item_def);
            if asks.is_none() { return trades; }
            let asks = asks.unwrap();

            for ask in asks.iter_mut() {
                if ask.status != ListingStatus::Active { continue; }
                // 卖方要价 > 买方出价 → 后续所有要价都更高 → 停止
                if ask.limit_price_copper > new_listing.limit_price_copper { break; }

                let (trade_qty, price) =
                    execute_trade(new_listing, ask, remaining_qty);
                trades.push(record_trade(new_listing, ask, trade_qty, price, market));
                remaining_qty -= trade_qty;
                if remaining_qty == 0 { break; }
            }
        }
    }

    // 未成交的剩余量 → 挂单留在订单簿
    if remaining_qty > 0 && new_listing.status == ListingStatus::Active {
        let mut leftover = new_listing.clone();
        leftover.quantity = remaining_qty;
        add_to_order_book(market, leftover);
    }

    // 每笔成交后更新市场状态
    for trade in &trades {
        market.last_price.insert(trade.item_def, trade.price_per_unit_copper);
        market.trade_history.push_back(trade.clone());
        if market.trade_history.len() > 200 {
            market.trade_history.pop_front();
        }
        update_volatility(market, trade.item_def);
    }

    trades
}
```

### 3.3 分层定价模型

```rust
/// 成交价格——按情景分层
fn execute_trade(
    bid: &mut MarketListing,
    ask: &mut MarketListing,
    max_qty: u32,
) -> (u32, u32) {
    let trade_qty = bid.remaining_qty()
        .min(ask.remaining_qty())
        .min(max_qty);

    let price = match (&bid.listing_type, &ask.listing_type) {
        // ── 正常撮合 ──
        (ListingType::Normal, ListingType::Normal) |
        (ListingType::StandingOrder, _) | (_, ListingType::StandingOrder) => {
            let spread_pct = (bid.limit_price_copper as f32 - ask.limit_price_copper as f32)
                / ask.limit_price_copper as f32;

            if spread_pct > 0.30 {
                // 薄市场——价差过大，不自动撮合
                // 双方可走协商模式（议价技能+魅力+关系）
                // 返回0 = 不成交
                return (0, 0);
            }
            // 正常中间价
            (bid.limit_price_copper + ask.limit_price_copper) / 2
        }

        // ── 紧急买方 + 正常卖方 → 偏贵（流动性溢价）──
        (ListingType::Urgent, ListingType::Normal) => {
            (bid.limit_price_copper + ask.limit_price_copper * 2) / 3
        }

        // ── 正常买方 + 紧急卖方 → 偏便宜（急售折价）──
        (ListingType::Normal, ListingType::Urgent) => {
            (bid.limit_price_copper * 2 + ask.limit_price_copper) / 3
        }

        // ── 双方都紧急 → 中间价 ──
        (ListingType::Urgent, ListingType::Urgent) => {
            (bid.limit_price_copper + ask.limit_price_copper) / 2
        }

        // ── 政府征购 / 政府抛售 → 政府定价 ──
        (ListingType::GovernmentQuota, _) | (_, ListingType::GovernmentQuota) => {
            // 政府定价强制成交
            bid.limit_price_copper.min(ask.limit_price_copper)
        }

        // ── 以物易物 → 双方自行协商比例 ──
        (ListingType::BarterOffer, _) | (_, ListingType::BarterOffer) => {
            // 以物易物不产生铜币价格——走barter结算通道
            // 此处返回0表示不走铜币撮合
            return (0, 0);
        }

        // ── 借贷相关 → 走借贷结算 ──
        (ListingType::LoanRequest, _) | (_, ListingType::LoanRequest) |
        (ListingType::IOUTrade, _) | (_, ListingType::IOUTrade) => {
            // 借贷/欠条交易有独立的结算逻辑
            // 此处返回0表示不走标准撮合
            return (0, 0);
        }
    };

    // 更新挂单状态
    bid.filled_quantity += trade_qty;
    ask.filled_quantity += trade_qty;
    if bid.filled_quantity >= bid.quantity { bid.status = ListingStatus::Filled; }
    else { bid.status = ListingStatus::PartiallyFilled; }
    if ask.filled_quantity >= ask.quantity { ask.status = ListingStatus::Filled; }
    else { ask.status = ListingStatus::PartiallyFilled; }

    (trade_qty, price)
}
```

**分层定价的设计逻辑**：
- **正常撮合（80%+交易量）**：中间价，双方都有剩余
- **紧急偏斜**：急者多付/贱卖——市场对"提供了流动性"的奖励。涌现效果：NPC囤粮不只为吃——饥荒时卖出可获紧急溢价。这是"囤积居奇"的微观基础
- **薄市场不撮合**：价差>30%时不自动成交——鼓励双方走协商通道（议价技能+魅力+关系），增加社交深度
- **政府强制价**：体现权力对市场的直接干预

### 3.4 价格波动率

```rust
/// 价格波动率——从最近20笔交易的变异系数中涌现
fn update_volatility(market: &mut Marketplace, item_def: ItemDefId) {
    let recent: Vec<u32> = market.trade_history.iter()
        .rev()
        .take(20)
        .filter(|t| t.item_def == item_def)
        .map(|t| t.price_per_unit_copper)
        .collect();

    if recent.len() < 2 { return; }

    let mean = recent.iter().sum::<u32>() as f32 / recent.len() as f32;
    let variance = recent.iter()
        .map(|p| { let d = *p as f32 - mean; d * d })
        .sum::<f32>() / recent.len() as f32;
    let volatility = variance.sqrt() / mean; // 变异系数

    market.price_volatility.insert(item_def, volatility);
}
```

## 四、交易模式

同一套订单簿撮合引擎，不同交易模式影响的是**交割方式和附加成本**——不是撮合逻辑。

```rust
/// 交易模式——交易如何发生
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub enum TransactionMode {
    /// 面对面现货——买卖双方在同一地点
    /// 成本最低，即时交割
    SpotExchange,

    /// 远程订单——买方派信使/代理人下单
    /// 额外成本：信使费 + 运输费
    RemoteOrder {
        messenger_cost_copper: u32,
        transport_cost_copper: u32,
        estimated_days: f32,
    },

    /// 长期供货合同——定期自动交割
    StandingContract {
        contract_id: ContractId,
        period: ContractPeriod,      // Weekly / Monthly / Seasonal
        price_model: ContractPriceModel,
    },

    /// 中间人撮合——买卖双方不见面
    /// 中间人赚取佣金，不持有货物
    BrokeredDeal {
        broker_id: NpcId,
        broker_commission_pct: f32,
    },

    /// 信使拍卖——卖方把样品带到远方展示
    MessengerAuction {
        sample_transport_cost: u32,
        auction_duration_days: u8,
    },
}

#[derive(Debug, Clone, PartialEq, Eq)]
pub enum ContractPeriod { Weekly, Monthly, Seasonal }

#[derive(Debug, Clone, PartialEq, Eq)]
pub enum ContractPriceModel {
    FixedPrice(u32),             // 固定价
    MarketLinked(f32),           // 随市场价 × 系数
    CostPlus { margin_pct: f32 }, // 成本+利润率
}
```

### 4.1 不同模式的价格示例

```
铁匠在矿村订购铁矿石（RemoteOrder）：
  订单簿撮合价 = 50铜/单位（中间价撮合，不变）
  附加成本 = 信使费10铜 + 运输费15铜 = 25铜（一次性）
  本地铁矿石卖60铜 → 远程订单反而便宜！
  → 这就是远程交易存在的经济逻辑

铁匠和矿主签长期合同（StandingContract）：
  合同价格 = 45铜（矿主给了长期客户折扣——高agreeableness）
  不需要每次付信使费
  → 长期关系降低了交易成本
  → 但约束了铁匠不能随意换供应商（现状偏误）
```

### 5.1 交易记录

```rust
/// 交易记录——价格从这些记录中涌现
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct TradeRecord {
    pub trade_id: u64,
    pub item_def: ItemDefId,
    pub item_ent: Option<ItemEntId>,
    pub quantity: u32,
    pub price_per_unit_copper: u32,
    pub total_copper: u32,
    pub buyer: NpcId,
    pub seller: NpcId,
    pub bid_listing_id: ListingId,
    pub ask_listing_id: ListingId,
    pub timestamp: GameTimestamp,
    pub trade_type: TradeType,
    pub transaction_mode: TransactionMode,
}

#[derive(Debug, Clone, PartialEq, Eq)]
```

### 5.2 原子交易执行

```rust
/// 两阶段提交——确保钱包和库存同步
fn execute_atomic_trade(
    trade: &TradeRecord,
    inventory: &mut dyn InventoryAccess,
    wallets: &mut dyn WalletAccess,
) -> Result<(), TradeError> {
    // 阶段1: 验证
    let buyer_wallet = wallets.get(trade.buyer)
        .ok_or(TradeError::BuyerNotFound)?;
    if buyer_wallet.copper < trade.total_copper {
        return Err(TradeError::InsufficientFunds);
    }

    let seller_inv = inventory.get(trade.seller)
        .ok_or(TradeError::SellerNotFound)?;
    if let Some(ent_id) = trade.item_ent {
        if !seller_inv.contains(ent_id, trade.quantity) {
            return Err(TradeError::InsufficientStock);
        }
    }

    // 阶段2: 执行（原子）
    wallets.transfer(trade.buyer, trade.seller, trade.total_copper)?;
    inventory.transfer(trade.seller, trade.buyer, trade.item_def, trade.item_ent, trade.quantity)?;

    Ok(())
}
```
