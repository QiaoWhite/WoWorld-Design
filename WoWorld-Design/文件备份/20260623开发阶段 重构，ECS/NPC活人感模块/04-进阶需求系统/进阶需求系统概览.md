# 04 — 进阶需求系统 v1.0

> **核心设计哲学**：基本需求回答"NPC 缺什么"（deficit → urgency → action）。进阶需求回答两个新问题：**(1)** NPC 在社会中被如何对待？（尊重/认可）**(2)** NPC 的成长之路是否顺畅？（胜任挫折）。再加上已有的"志向"（IntrinsicGoal），三者共同构成 NPC 从"生存"到"生活"到"追求"的完整心理动力链。

## §一 三层需求模型

### 1.1 架构全景

```
┌──────────────────────────────────────────────────────────────┐
│ Layer 3: 成长驱力 (Aspiration-driven)                        │
│ ┌──────────────────────────────────────────────────────────┐ │
│ │ IntrinsicGoal × 6 → intrinsic_motivation_weight()        │ │
│ │ (ExploreUnknown/MasterSkill/ActOnValue/CreateSomething/  │ │
│ │  DeepenConnection/LeaveLegacy)                           │ │
│ │ 数学: commitment × relevance → 偏好偏置, range [1.0, 3.0]│ │
│ └──────────────────────────────────────────────────────────┘ │
│               ↕ frustration_regression (向上)                 │
│               ↕ survival_suppression (向下)                   │
├──────────────────────────────────────────────────────────────┤
│ Layer 2: 基本心理需求 (Deficit-driven + 1 Allostatic)         │
│ ┌──────────────────────────────────────────────────────────┐ │
│ │ 原有6维: hunger/thirst/fatigue/element_balance/libido/   │ │
│ │          social_deficit                                   │ │
│ │ ★新增(deficit): esteem_deficit                           │ │
│ │ ★新增(allostatic): competence_frustration                │ │
│ │ 数学: urgency = deviation × sensitivity → [1.0, 2.0]     │ │
│ └──────────────────────────────────────────────────────────┘ │
│               ↕ survival_suppression (sigmoid 软衰减)         │
├──────────────────────────────────────────────────────────────┤
│ Layer 1: 生存需求 (GOAP 安全网, 阈值触发)                     │
│ ┌──────────────────────────────────────────────────────────┐ │
│ │ hunger<0.3 / thirst<0.25 / fatigue>0.9 / health<0.3     │ │
│ │ combat_threat>0.7 / courtship_pressure>0.3               │ │
│ │ ★ 心理需求不进 GOAP —— 不被认可不会死                     │ │
│ └──────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
```

### 1.2 关键原则

1. **不做硬性需求层次** — 没有"必须先满足 L1 才能追求 L3"的硬规则。抑制是软的、连续的。
2. **GOAP 边界不变** — 只有生存需求 + 配偶压力进安全网。心理需求不致命。
3. **两种数学对象** — Deficit（标量 urgency）和 Aspiration（向量 preference bias）用各自的数学工具，不强求统一。
4. **零新跨模块依赖** — 除 `CultureQuery` 新增 1 个方法外，所有数据通过已有接口查询或 NPC 内部计算。
5. **Emergent, not scripted** — 开发者不定义"铁匠在何时会感到不被认可"——认可信号从社交互动和记忆中涌现。

### 1.3 概念边界

| 概念 | 存储位置 | 类型 | 说明 |
|------|---------|------|------|
| **esteem_deficit** | `NpcData` | f32 [0,1] | Deficit-driven — 认可会褪色，需持续补充 |
| **competence_frustration** | `NpcData` | f32 [0,1] | Allostatic — 设定点 = aspiration vs reality gap |
| social_deficit | `NpcData` | f32 [0,1] | 社交连接的量 — 与 esteem 不同 |
| stagnation_sense | `SelfNarrative` | f32 [0,1] | 人生整体意义匮乏 — 与 competence 不同 |

## §二 尊重/认可维度 (Esteem Deficit)

### 2.1 核心定义

```
esteem_deficit: f32 ∈ [0.0, 1.0]

0.0 = "我被充分看见、充分认可——人们知道我的价值"
1.0 = "没有人在乎我做了什么、我是谁——我像是透明的"

存储: NpcData（心理状态，非生物信号——不入 Life.Vitals）
```

### 2.2 与 social_deficit 的区分

| | social_deficit | esteem_deficit |
|---|---|---|
| **问题** | "有人在我身边吗？" | "人们尊重我吗？" |
| **关注** | 社交连接的**量** | 社交地位的**质** |
| **恢复** | 任何社交互动 | 被钦佩/被称赞/被认可 |
| **比喻** | 孤独 vs 陪伴 | 透明 vs 被看见 |

两者可独立变化：
- 被围绕但不被尊重的弄臣 → social↓, esteem↑
- 远方被传颂的英雄 → social↑, esteem↓

### 2.3 累积与代谢

```
被动累积（认可的"褪色"）:
    daily_accumulation = 0.01 / 游戏日
    → 100天无认可 → deficit = 1.0

对比：
    social_deficit 的累积率 = 0.02/天（孤独来得更快）
    设计理由：社会连接比社会认可更基础——孤独比"不被看见"更紧迫
```

### 2.4 恢复：钦佩检测

esteem_deficit 的恢复**不由其他模块推送事件**。NPC 模块从已有社交管道**内部拉取**钦佩信号。

#### 2.4.1 检测源

```rust
/// 在 NPC 模块内部——每次社交互动后调用
fn detect_admiration(
    interaction: &SocialContext,  // NPC 内部社交处理上下文
    npc: &NpcData,
    culture_query: &dyn CultureQuery,
    power_query: &dyn PowerTopologyQuery,
) -> f32 {
    let mut recovery = 0.0;

    // === 源 1: 直接赞美 ===
    // 检测条件: 互动者在对话中表达了正面评价
    // 实现方式: 通过 NPC 内部的社会认知评估——互动者的语气/措辞是否包含赞赏
    if interaction.contains_appreciation() {
        // 赞美者的"分量"——高 legitimacy 者的赞美更重
        let praiser_weight = power_query
            .perceived_legitimacy(interaction.other_id, npc.id)
            .unwrap_or(0.5);

        // 赞美者与 NPC 的关系亲密度（熟悉的人赞美更有分量）
        let relationship = npc.relationships.get(&interaction.other_id);
        let affection = relationship.map(|r| r.affection.max(0.0)).unwrap_or(0.0);

        recovery += 0.02                           // 基础：被夸就好
                  + praiser_weight * 0.05          // 权威者的赞美 ×5
                  + affection * 0.03;              // 亲密者的赞美 ×3
        // 范围: 0.02 ~ 0.10
    }

    // === 源 2: 感激 ===
    if interaction.contains_gratitude() {
        // 感激 ≈ 认可了你的帮助有价值
        recovery += 0.03;
    }

    // === 源 3: 服从/尊敬（对低宜人性 NPC 有效） ===
    if interaction.contains_deference() || interaction.contains_submission() {
        // 被畏惧/服从——对竞争性强者也是一种"认可"
        let fear_as_esteem = (1.0 - npc.big_five.agreeableness) * 0.5;
        recovery += 0.01 + fear_as_esteem * 0.05;
        // 高宜人性 NPC: 0.01（几乎不从中获得认可）
        // 低宜人性 NPC: 0.06
    }

    // === 源 4: 被谈论（间接认可） ===
    // 不在每次互动中处理——在 §2.4.2 的反思周期中处理

    recovery.clamp(0.0, 0.5) // 单次互动恢复有上限——不会一次互动就消除所有 esteem deficit
}
```

**关键设计**：`contains_appreciation()` 等方法的具体实现取决于 NPC 模块内部的社会认知处理。我们不规定"必须检测到 Praise 这个 SpeechAct"——因为 SpeechAct 枚举在语言表达系统中且可能尚不存在。NPC 模块基于互动中的情绪信号、措辞模式、语调来判断——这是 NPC 内部的实现细节。

#### 2.4.2 反思周期：自我认可 + 被谈论检测

```rust
/// SelfNarrative.reflect() 中调用——每 7 游戏天
fn self_evaluate_esteem(
    npc: &NpcData,
    recent_memories: &[EventMemory],  // 近期记忆（7天内）
    self_concept: &SelfConcept,
) -> f32 {
    let mut recovery = 0.0;

    // 子源 A: 成就记忆（自我认可）
    for memory in recent_memories {
        if memory.event_type.is_achievement() && memory.impact > 0.2 {
            // 这件事符合我的自我概念吗？
            let alignment = self_concept.alignment_with(memory);
            // "战士"击败强敌 → alignment=0.9 → 更大自我认可
            // "战士"种出好庄稼 → alignment=0.2 → 较少的认可

            recovery += memory.impact * 0.15
                      * (0.5 + alignment * 0.5);
        }
    }

    // 自我认可有上限: 最多恢复 0.5
    // 完全的 esteem 满足需要社会认可——隐士也无法完全靠自我认可填满
    recovery = recovery.clamp(0.0, 0.5);

    // 子源 B: 被他人提及（间接认可）
    let mentions = count_mentions_of_npc(recent_memories, npc.id);
    // 注意: count_mentions_of_npc 扫描的是 NPC 自己的记忆——
    // "我听说有人在谈论我"——通过社交互动中的信息传播（谣言/故事）
    recovery += (mentions as f32 * 0.01).min(0.1);
    // 被谈论的感觉——微小的 esteem 恢复，但有上限

    recovery
}
```

### 2.5 文化调制

不同文化对不同领域的认可权重不同。在军事主义文化中，战斗胜利获得尊重。

#### 2.5.1 接口定义

```rust
/// 认可领域编码（NPC 模块定义）
/// 用于 honor_weight_for_domain() 的 domain_code 参数
/// 注意: 文化系统通过 u8 消费——零类型依赖

    /// 纯派生函数——从 CultureCoreParams 已有参数计算
```

#### 2.5.2 文化系统的派生实现

```rust
// 文化系统实现 CultureQueryExt::honor_weight_for_domain()
fn honor_weight_for_domain(&self, culture_id: CultureId, domain_code: u8) -> f32 {
    let p = self.core_params(culture_id)?;

    let raw = match domain_code {
        0 /* Combat */   => p.militarism,
        1 /* Crafting */ =>
            p.long_term_orientation * 0.5 + p.artistry * 0.3 + 0.2,
        2 /* Artistic */ => p.artistry,
        3 /* Scholarly */ =>
            p.long_term_orientation * 0.6 + p.openness_to_outsiders * 0.4,
        4 /* Social */   =>
            (1.0 - p.individualism) * 0.7 + 0.3,
        5 /* Power */    => p.power_distance,
        6 /* Wealth */   =>
            p.indulgence * 0.5 + p.competition_orientation * 0.5,
        7 /* Faith */    => p.religiosity,
        _ => 0.3, // 未知领域 → 保守默认
    };

    raw.clamp(0.05, 1.0) // 没有文化对任何认可领域的权重为 0
}
```

**设计理由**：
- `Crafting` 有 +0.2 基础值——任何文化都尊重好工匠（跨文化普适）
- `Social` 有 +0.3 基础值——社交能力在任何文化中都受一些尊重
- clamp 下限 0.05——即使最不军事的文化，战斗胜利仍有一丝认可价值

#### 2.5.3 涌现效果

| 文化 | militarism | artistry | Combat honor | Artistic honor | 效果 |
|------|-----------|----------|-------------|----------------|------|
| 兽人战士 | 0.9 | 0.1 | 0.9 | 0.1 | 战斗英雄极受尊重，艺术家几乎不被认可 |
| 精灵艺术家 | 0.1 | 0.9 | 0.1 | 0.9 | 艺术大师受崇拜，战士认可度低 |
| 矮人工匠 | 0.5 | 0.7 | 0.5 | 0.7 | 工匠被高度尊重，战斗也被认可 |

→ **NPC 自然倾向于追求本文化认可的成就路径**——不是脚本写的，是数学结果：同样做一个杰作，在精灵文化中 restore 0.9×esteem，在兽人文化中只 restore 0.1×esteem。

### 2.6 极端边界

#### 完全孤立
NPC 困在荒岛：
- 被动累积: +0.01/天
- 社交恢复: 0（无人可钦佩）
- 自我恢复: 最多 0.5（7天周期）
- 平衡点: esteem_deficit ≈ 0.5——"我不需要别人的认可，但我仍想知道我存在过"

#### 被全面憎恨
NPC 被社群排斥：
- 社交互动中充满负面信号而非赞赏
- deficit 无法恢复→持续上升到 1.0
- **上限是 1.0**——不建模"负面尊重"。被憎恨的 NPC 通过关系负值、社交敌意信号来表达，不通过 esteem_deficit。

#### 死后认可（不做）
- NPC 死后的认可不影响 esteem_deficit（死亡时状态冻结）
- 历史声望由历史系统管理

#### 设计理由：为何用 affection 而非 sincerity？

早期的讨论草稿（[[../../参考文档/025-NPC进阶需求系统设计探讨-20260615/002-尊重认可维度详细设计|参考/025-002]]）中使用 `estimate_sincerity(interaction, relationship)` ——基于 trust × agreeableness 估算赞美者的真诚度——来确定赞美的分量。

正式规格改为使用 `affection`（好感度）。理由：

1. **Sincerity 检测需要心智理论**：判断一个人是在真诚赞美还是奉承/讽刺，需要社会认知推理——这在 WoWorld 中尚未建模（且成本极高）
2. **Affection 已有精确追踪**：`npc.relationships` 中的 `affection` 字段是已有数据，零新增查询开销
3. **心理近似合理**：来自喜欢之人的赞美主观上"感觉"更有分量——即使这混淆了真诚度和好感度，对 NPC 的主观体验而言是合理的近似
4. **未来升级路径**：v2.0 可引入基于 intelligence + agreeableness 的 sincerity 检测器，届时可将 `affection` 替换为 `sincerity` 或两者混合。当前设计为这一升级预留了接口——`contains_appreciation()` 的返回值可扩展为包含 sincerity 估计

#### 设计理由：被畏惧作为认可

低宜人性 NPC 可以从他人的服从/畏惧中获得 esteem 恢复（§2.4.1 源 3）。这建模了"暴君从臣民的恐惧中获得满足"的心理动态。

- 高宜人性（A > 0.8）：几乎不从中获益（`fear_as_esteem ≈ 0.1`）
- 低宜人性（A < 0.3）：显著获益（`fear_as_esteem ≈ 0.35`）
- 这与"被憎恨"不同——畏惧者仍然服从，憎恨者则反抗。被憎恨（上限=1.0 的 deficit）不带来认可

#### 设计理由：单次互动恢复上限（clamp 0.5）

单次社交互动中的 esteem 恢复被限制在 0.5。即使三个恢复源（赞美+感激+服从）同时触发，理论最大值也仅约 0.165（远超实际触发概率）。0.5 的上限是一个安全阀——防止单次互动异常恢复所有 esteem。实际上，这个上限几乎不可能触及。

### 2.7 NeedSensitivity::esteem

```rust
// ★ 从 BigFive 派生——初始一次性计算。BigFive 在极端冲击(impact>0.9)后漂移时重算
impl NeedSensitivity {
    pub fn from_big_five(bf: &BigFive) -> Self {
        Self {
            // ... 已有 6 项不变 (来自 03-基本需求系统) ...

            esteem: (0.2

        }
```

| 高E, 低A (竞争型) | 0.2+0.5+0.3=1.0 | 极其在意他人看法和地位竞争 |
| 高E, 高A (亲和型) | 0.2+0.5+0.0=0.7 | 在意被喜欢，但不追求竞争性认可 |
| 低E (内向型) | 0.2+0.0+0.15=0.35 | 不太在乎外界评价 |
| 极低E, 极高A | 0.2+0.0+0.0=0.2 | 几乎不在乎——淡然无求 |

## §三 胜任挫折维度 (Competence Frustration)

### 3.1 核心定义

```
competence_frustration: f32 ∈ [0.0, 1.0]

0.0 = "我的能力在预期轨道上进步，或我没有任何技能志向"
1.0 = "我卡住了——我的努力没有回报，我与目标的距离没有缩小"

存储: NpcData
类型: Allostatic — 设定点 = aspiration.target vs current skill gap
更新: 每游戏日一次（非热路径）
```

### 3.2 为什么叫"胜任挫折"而非"胜任需求"

SDT 的胜任需求不是"我需要变强"——它是**"我需要在与环境互动中感到有效能"**。它不是消耗-恢复循环，而是一种**持续的心理状态**。

"胜任挫折"更精确地描述了我们要建模的东西：
- 不是来自绝对技能水平（Lv.80 的大师也可能有更高的 aspiration）
- 而是来自 **aspiration 与 reality 的 gap**
- gap 小→ frustration 低; gap 大且持续→ frustration 高

### 3.3 与 stagnation_sense 的区分

| 维度 | competence_frustration | stagnation_sense |
|------|----------------------|-------------------|
| **粒度** | 技能特定 | 人生整体 |
| **频率** | 每游戏日 | 每 7 游戏天 |
| **来源** | aspiration gap | 有意义的记忆数量 |
| **解决** | 提升技能或降低志向 | 改变生活轨迹或找到新意义 |
| **类比** | "我在铁匠技艺上卡住了" | "我的人生缺乏意义" |

两者可同时高 = **中年危机**。两者可分离 = 正常。

### 3.4 每日更新公式

```rust
/// 每游戏日调用一次
fn update_competence_frustration(
    npc: &mut NpcData,
    aspirations: &[Aspiration],  // 来自 SelfNarrative
) {
    // Step 1: 收集活跃的 SkillMastery 志向
    let skill_aspirations: Vec<(&Aspiration, SkillId, u32, f32)> = aspirations.iter()
        .filter_map(|a| match &a.category {
            AspirationCategory::SkillMastery { skill_id, target_level } => {
                Some((a, *skill_id, *target_level, a.priority))
            }
            _ => None,
        })
        .collect();

    // 无技能志向 → frustration 归零
    if skill_aspirations.is_empty() {
        npc.competence_frustration = 0.0;
        npc.competence_frustration_chronic_days = 0;
        return;
    }

    // Step 2: 计算加权 gap
    let mut total_weighted_gap = 0.0;
    let mut total_priority = 0.0;

    for (_, skill_id, target, priority) in &skill_aspirations {
        let current = npc.skills.get(skill_id)
            .map(|s| s.level)
            .unwrap_or(0) as f32;

        // gap = 归一化的距离: (target - current) / target
        // current >= target → gap = 0（已达标，无挫折）
        let gap = ((*target as f32 - current) / *target as f32).max(0.0);

        total_weighted_gap += gap * priority;
        total_priority += priority;
    }

    let avg_gap = total_weighted_gap / total_priority;

    // Step 3: 慢性累积
    if avg_gap > 0.3 {
        npc.competence_frustration_chronic_days =
            npc.competence_frustration_chronic_days.saturating_add(1);
    } else {
        // gap 缩小 → 慢性快速消退（×2 速度）
        npc.competence_frustration_chronic_days =
            npc.competence_frustration_chronic_days.saturating_sub(2);
    }

    let chronic_factor = 1.0
        + (npc.competence_frustration_chronic_days as f32 / 30.0).min(1.0) * 0.5;
    // Day 0:  ×1.0
    // Day 15: ×1.25
    // Day 30+: ×1.5 (饱和——30天持续挫败后慢性放大达到最大)

    // Step 4: 人格调制
    let neuroticism_amp = 0.7 + npc.big_five.neuroticism * 0.6;
    // N=0 → 0.7, N=1 → 1.3 —— 神经质放大挫败感 86%

    let conscientiousness_buffer = 1.0 - npc.big_five.conscientiousness * 0.3;
    // C=0 → 1.0, C=1 → 0.7 —— 尽责者相信坚持能突破，缓冲挫败感 30%

    npc.competence_frustration = (avg_gap
        * chronic_factor
        * neuroticism_amp
        * conscientiousness_buffer
    ).clamp(0.0, 1.0);
}
```

### 3.5 人格调制的涌现效果

| 人格标签 | N | C | amp×buffer | 行为表现 |
|----------|---|---|-----------|---------|
| 淡定大师 | 0.2 | 0.8 | 0.82×0.76=0.62 | 遇到瓶颈很冷静，耐心坚持 |
| 焦虑学徒 | 0.9 | 0.2 | 1.24×0.94=1.17 | 容易挫败，频繁换方向或放弃 |
| 完美主义者 | 0.9 | 0.9 | 1.24×0.73=0.90 | 痛苦地坚持——高标准+高韧性 |
| 佛系 | 0.1 | 0.1 | 0.76×0.97=0.74 | 不太在意 gap |

### 3.6 恢复路径

**积极恢复：技能突破**
- Skill level up 事件 → gap 缩小 → 下次每日更新时 frustration 自然下降
- chronic_days 重置 → 慢性因子归 1.0

**消极恢复：降低志向**
- SelfNarrative 反思时，如果 `frustration > 0.8 && chronic_days > 90`，可能触发 aspiration 降级或放弃
- 放弃后 frustration 归零——但 SelfConcept 可能受损
- 概率公式（来自 SelfNarrative 反思逻辑）：
  ```
  give_up_probability = competence_frustration × neuroticism × (1.0 - conscientiousness) × 0.3
  // 高N+低C → 最高 ~30% 概率放弃当前 aspiration
  // 低N+高C → 接近 0% — "我一定能突破"
  ```

**日常恢复：慢性消退**
- 当 gap 降到 0.3 以下，chronic_days 每天 -2 → 快速恢复

**技能突破的愉悦**
- Skill level up 事件 → `emotion.pleasure += 0.05`（一次性 spike——"突破的喜悦"）
- 这是情绪引擎中的独立牵引，不属于 competence_frustration 公式本身
- 幅度 0.05 ≈ 相当于一次愉快的社交互动——足以被玩家感知

### 3.7 边缘情况

#### 儿童与青少年
- 儿童（< Adolescent 生命阶段）：无 SkillMastery aspiration（尚未形成技能志向）
- → `competence_frustration = 0.0`（永远为 0）
- 青少年（Young Adult 初期）：开始形成简单的志向，frustration 开始有非零值

#### 老年 NPC
- LifeTheme::Decline 阶段：aspiration 从 SkillMastery → LeaveLegacy 自然过渡
- → competence_frustration 自然下降（SkillMastery aspiration 被移除或被降低优先级）
- LifeTheme::Legacy 阶段：关注点从"我有多强"转向"我留下了什么"
- → competence_frustration 被 transcendence 驱力取代

#### 无技能志向的 NPC（"过日子的人"）
- NPC 无 SkillMastery aspiration → `competence_frustration = 0.0`
- 这些 NPC 的行为完全由基本需求 + 习惯 + 文化驱动
- 他们构成社会的稳定性——这是正确且期望的行为

#### 慢性挫折触发 LifeTheme 转换
- 当 `competence_frustration > 0.7 && chronic_days > 60` 时，SelfNarrative 反思可能触发：
  ```
  LifeTheme::Building 或 LifeTheme::Mastery → LifeTheme::Crisis
  ```
- 这是"中年危机"的机制实现——不是硬编码的"45岁触发"，而是从技能挫败→慢性心理压力→人生主题转变的涌现链

### 3.8 NeedSensitivity::competence

```rust
competence: (0.2
    + bf.conscientiousness * 0.5   // 尽责者标准高，落差感强
    + bf.neuroticism * 0.3          // 神经质放大能力焦虑
).clamp(0.2, 1.0),
// 范围: 0.2 ~ 1.0
```

**设计理由**：尽责性设定内在标准（更努力→对停滞更敏感），神经质放大焦虑感。

## §四 IntrinsicGoal 形式化

### 4.1 现状

`IntrinsicGoal` 枚举（6 种）和 `intrinsic_motivation_weight` 乘数已存在于 NPC ver2.0。但：
- `intrinsic_motivation_weight` 如何从 goal 列表计算——**未指定**
- 不同 goal 对不同 action 的相关性——**未指定**
- goal 的"投入度"如何量化——**未指定**

### 4.2 核心公式

```
intrinsic_motivation_weight(goals, action) =
    ∏_{g ∈ active_goals}
        (1.0 + relevance(g.category, action) × commitment(g) × 0.5)

最终 clamp 到 [1.0, 3.0]
```

- 无 active goals → 1.0（不影响行为权重）
- 1 个 goal 完全相关 + 完全投入 → 1.0 + 1.0×1.0×0.5 = 1.5
- 3 个 goal 都高相关 → 1.5³ = 3.375 → clamp 到 3.0

**为什么是乘法而非加法？**
- 加法：弱关联可以累积出强影响（1 + 0.1×5 = 1.5）——不合理
- 乘法：只有真正高度相关且高度投入的 goal 才能显著提升权重

### 4.3 commitment(goal) — 投入度

每个 goal 的 `commitment_strength()` 表示 NPC 对此目标的当前投入度 [0, 1]：

| Goal | 公式 | 范围 | 设计理由 |
|------|------|------|---------|
| **ExploreUnknown** | `openness × 0.8` | 0~0.8 | 好奇心几乎完全由开放性决定 |
| **MasterSkill** | `progress×0.6 + (1-progress)×0.4` | 0.4~1.0 | 已投入越多→越坚持（沉没成本）。progress=0→0.4, progress=1→1.0 |
| **ActOnValue** | `max_aligned_value_strength` | 0~1.0 | 由对齐的 CoreValue 强度决定 |
| **CreateSomething** | `openness×0.6 + (1-progress)×0.4` | 0~1.0 | 类似 MasterSkill 但更依赖灵感 |
| **DeepenConnection** | `|affection|×0.3 + (1-trust)×0.5 + familiarity×0.2` | 0~1.0 | 情感深+信任缺(修复)+熟悉 |
| **LeaveLegacy** | `closeness_to_death×0.7 + stagnation_sense×0.3` | 0~1.0 | 离死越近+人生越空虚→传承欲越强 |

年龄因子：`age_factor = 1.0 + (1.0 - age_proportion) × 0.3` [1.0, 1.3]
- 年轻人对目标有更强的情绪投入

### 4.4 relevance(goal, action) — 相关性

**纯函数**——不依赖 NPC 状态，可编译时优化为查找表。

| Goal | 强相关 (≥0.7) | 中相关 (0.3~0.6) |
|------|-------------|-----------------|
| **ExploreUnknown** | Explore(0.9), Travel(0.8), Investigate(0.8) | Read(0.5), AskQuestions(0.4) |
| **MasterSkill(s)** | TrainSkill(s)(1.0), SeekMentor(s)(0.8) | Practice(0.6), TrainSkill(other)(0.3) |
| **ActOnValue** | (动态——与 CoreValue 对齐的 action) | (默认 0.5 保守估计) |
| **CreateSomething** | CraftItem(0.9), Build(0.8), Compose(0.9) | GatherMaterials(0.4), Design(0.6) |
| **DeepenConnection(t)** | Visit(t)(1.0), Gift(t)(0.8), Help(t)(0.9) | Socialize(0.5), WriteLetter(t)(0.7) |
| **LeaveLegacy** | Teach(0.9), Write(0.8), MentorChild(1.0) | Build(0.5), CreateHeirloom(0.7) |

### 4.5 Goal 生命周期

```rust
enum GoalState {
    Active,      // 正常 → 贡献权重
    Dormant,     // 暂停 → commitment × 0.3
    Abandoned,   // 放弃 → 从列表中移除
    Achieved,    // 完成 → 从列表中移除
}
```

**目标空白期**：goal 完成后至下一个反思周期（≤7天），intrinsic_motivation_weight = 1.0。这是正确的——完成目标后有自然的"消化期"。

## §五 跨层桥接

### 5.1 桥接 1：生存抑制 (Survival Suppression)

#### 5.1.1 公式

```
survival_suppression(physiology) = 1.0 / (1.0 + e^(10.0 × (max_urgency - 0.7)))

其中:
    max_urgency = max(
        1.0 - physiology.hunger,
        1.0 - physiology.thirst,
        physiology.fatigue,
        1.0 - physiology.health,
    )
```

#### 5.1.2 数值行为

| max_urgency | suppression | 含义 |
|-------------|-------------|------|
| 0.3 | 0.982 | 几乎无影响——GOAP-hunger 阈值附近 |
| 0.5 | 0.881 | 轻微——高层需求开始让步 |
| **0.7** | **0.500** | **拐点——高层需求减半** |
| 0.9 | 0.119 | 严重——几乎只剩生存驱动 |
| 0.95 | 0.076 | 濒死——只有生存本能 |

#### 5.1.3 生效范围

**仅影响概率引擎权重链**——不影响 GOAP。GOAP 在阈值触发时仍然硬接管行为选择。

survival_suppression 在权重链中的位置：
```
need_action_match × intrinsic_motivation_weight
    → × survival_suppression  ← 统一软衰减
    → × time/weather/vehicle
```

**为什么是 sigmoid？**
- 线性衰减 → 中度饥饿就砍掉一半高层需求——太剧烈
- 硬阈值 → 0.69→1.0 vs 0.71→0.2 的二分——违背涌现
- sigmoid → 正常状态几乎无抑制，危机状态急剧抑制，中间平滑

### 5.2 桥接 2：挫折回归 (Frustration-Regression)

#### 5.2.1 理论基础

ERG 理论的核心机制：当**成长需求**持续受挫时，人会向下回归到**关系需求**和**生理舒适**寻求补偿。

#### 5.2.2 触发条件与公式

```rust
/// 每次决策前调用——修改临时 NeedSensitivity（不持久化）
fn apply_frustration_regression(
    goals: &[IntrinsicGoal],
    sensitivity: &mut NeedSensitivity,  // 临时副本
) {
    let active_goals: Vec<_> = goals.iter()
        .filter(|g| g.state == GoalState::Active)
        .collect();

    if active_goals.is_empty() {
        return;  // 无目标 → 无挫折来源
    }

    let avg_frustration: f32 = active_goals.iter()
        .map(|g| 1.0 - g.progress)
        .sum::<f32>() / active_goals.len() as f32;

    if avg_frustration < 0.6 {
        return;  // 进展尚可，不触发回归
    }

    let r = ((avg_frustration - 0.6) / 0.4).clamp(0.0, 1.0);
    // r ∈ [0.0, 1.0] — 回归强度

    // 敏感度临时提升（仅本次决策）
    sensitivity.social  *= 1.0 + r * 0.50;   // 社交补偿——最强路径
    sensitivity.esteem  *= 1.0 + r * 0.40;   // 认可替代成就感
    sensitivity.hunger  *= 1.0 + r * 0.30;   // 舒适进食
    sensitivity.libido  *= 1.0 + r * 0.25;   // 性作为出口

    // thirst, fatigue, element_balance — 不变（非舒适型需求）
    // competence — 不变（不自指）
}
```

#### 5.2.3 为什么这些调制系数？

| 需求 | 增幅(max) | 理由 |
|------|----------|------|
| social | +50% | ERG 核心路径——关系需求是成长受挫最自然的回归方向 |
| esteem | +40% | 外部认可替代内部成就感 |
| hunger | +30% | 情绪性进食是跨文化的心理调节行为 |
| libido | +25% | 性作为情绪出口——幅度最低受生物约束 |

#### 5.2.4 临时性

**不持久化**。每次决策前重新计算。当 goal 的 progress 改善或 goal 被放弃时，回归自动解除。这确保了 NPC 的行为变化是**情境性的**，不是永久性的人格漂移。

|------|--------|------|---------|---------|

| `honor_weight_for_domain()` | **文化系统**（CultureQuery 方法） | NPC 模块（esteem 计算） | ★ **唯一新跨模块方法**。从已有 CultureCoreParams 派生。纯函数——不新增存储 |
| `NeedSensitivity::esteem` | **NPC 模块**（from_big_five） | NPC 决策器 | `0.2 + E×0.5 + (1-A)×0.3`。BigFive 漂移后重算 |
| `NeedSensitivity::competence` | **NPC 模块**（from_big_five） | NPC 决策器 | `0.2 + C×0.5 + N×0.3`。BigFive 漂移后重算 |
| `survival_suppression()` | **NPC 模块**（概率权重链内部） | 仅概率引擎 | sigmoid `1/(1+e^(-10(x-0.7)))`。**不影响 GOAP** |
| `frustration_regression()` | **NPC 模块**（决策预处理） | NeedSensitivity 临时调制 | avg_frustration > 0.6 触发。**不持久化** |
| `intrinsic_motivation_weight()` | **NPC 模块**（权重链） | 仅概率引擎 | `∏(1 + relevance×commitment×0.5)`, clamp [1,3] |
| `NeedTag::Esteem` / `NeedTag::Competence` | **NPC 模块** | ActionType 定义 | 新增 2 个需求标签 |

### 7.3 与其他模块的边界澄清

| 其他模块 | 可能误解 | 本模块澄清 |
|---------|---------|-----------|
| Life.Vitals | 可能被误认为应包含 esteem/competence | ❌ 不入 Vitals。心理需求存储在 NpcData |
| Life.Physiology | 可能被认为要包含 esteem_urgency | ❌ Physiology 是 Vitals→主观感知的桥。心理需求从 NpcData 直接进入决策器 |
| 技能系统 | 可能自己建模"挫败感" | ❌ competence_frustration 以本模块为权威 |
| SelfNarrative | stagnation_sense 可能被混淆 | 明确区分：stagnation = 人生意义匮乏，competence = 技能 gap 挫折 |
| 文化系统 | 可能试图在 honor_weight 中重复建模领域差异 | honor_weight 保持为通用标量。honor_weight_for_domain 为独立新方法 |
| 经济系统 | 可能将 esteem 驱动的消费建模为新的需求来源 | ❌ 交易主体涌现的"需求侧"已经涵盖。esteem 只影响 NPC 决策权重，不改变经济算法 |

### 7.4 本模块的权威声明

以下概念以**本模块（进阶需求系统）为权威 Owner**。其他模块在设计或修改涉及这些概念时，应以本模块的定义为准，不得重复建模：

| 概念 | 说明 | 其他模块不得 |
|------|------|------------|
| `esteem_deficit` | 尊重/认可匮乏——心理状态 | 不得在 Life.Vitals 中自行定义或重复建模 |
| `competence_frustration` | 胜任挫折——心理状态 | 不得在技能系统中重复建模。技能系统提供 gap 数据，本模块消费 |
| `RecognitionDomain` (domain_code: u8) | 认可领域的 8 分类编码 | 不得在其他模块中定义重叠的分类体系 |
| `survival_suppression()` | sigmoid 软衰减函数 | 不得在 GOAP 或其他决策代码中硬编码等效逻辑替代 |
| `frustration_regression()` | ERG 挫折回归——敏感度临时调制 | 不得在其他模块中重复建模（如情绪系统中硬编码"受挫后暴食"规则） |
| `intrinsic_motivation_weight()` | IntrinsicGoal → action_weight 的正式公式 | 不得在 SelfNarrative 或决策器中重复定义等效公式 |
| `NeedSensitivity` 的完整 8 字段 | 大五派生——含 esteem 和 competence | 其他模块不得自行定义 sensitivity。如果需要在其他上下文中使用 sensitivity 概念，引用 NPC 模块的定义 |
| `NeedTag::Esteem` / `NeedTag::Competence` | 需求标签枚举变体 | 其他模块如果需要扩展 ActionType→satisfied_needs 映射，必须以 NPC 模块的 NeedTag 定义为权威 |

## §八 性能预算

### 8.1 存储

| 项目 | 每 NPC 增量 | 100K NPC | 占比 |
|------|------------|----------|------|
| NpcData 新字段 | ~12 bytes | 1.2 MB | +0.6% NpcData |
| NeedSensitivity 扩展 | 8 bytes | 0.8 MB | — |
| **总计** | **~20 bytes** | **2.0 MB** | **<0.15% Rust 模拟总内存** |

### 8.2 热路径（每决策周期）

| 操作 | 成本 | 说明 |
|------|------|------|
| need_action_match (+2维) | +6 ns | 原来 7 维 ~24ns |
| intrinsic_motivation_weight | ~25 ns | 平均 3 个 active goals |
| survival_suppression | ~20 ns | 含 1 次 exp() |
| frustration_regression | ~5 ns | 通常不触发 |
| emotion 心理牵引 | ~10 ns | 3 次比较 + 乘加 |
| **总计** | **~66 ns** | **+1.3% 决策器预算** |

### 8.3 非热路径

- competence_frustration 每日更新：每游戏日一次 → 摊还成本 ≈ 0（day_length [TUNING] 不影响摊销结论）
- SelfNarrative 反思扩展：每 7 游戏天一次 → 摊还成本 ≈ 0

### 8.4 VRAM

**零** — 全部 CPU 侧。

## §九 扩展预留

### 9.1 v2.0 候选扩展

1. **地位锚定 (Esteem Baseline)**
   - 长期被尊重的 NPC 的 esteem "设定点"上移
   - 突然失去地位 → 比一直不受尊重的人感到更严重
   - 解释"曾经的大人物落魄后比普通人更痛苦"

2. **多领域独立追踪**
   - 不同领域的认可独立：战士可能在战斗中备受尊重，但在社交中感到不被认可
   - 需要 `[RecognitionDomain; 8]` 而非单 f32
   - 成本: 8× 存储 vs 涌现丰富度提升

3. **社会比较加剧**
   - 同行的技能远超自己 → competence_frustration 放大
   - "不仅我进步慢，我同龄人都在超越我"

4. **教学缓解**
   - 教导他人可缓解自己的 competence_frustration
   - "虽然我卡在 Lv.75，但我能教出 Lv.60 的弟子"

5. **挫折回归的持久化痕迹**
   - 记录 NPC 经历过多少次重大挫折回归
   - 用于 LifeChapter 叙事："饱经挫折的中年"

### 9.2 RecognitionDomain 扩展

当前 8 领域 — 枚举可扩展，新增领域不需要改架构：
- 预留：Legal（法律认可）、Political（政治认可）、Educational（学术认可）
