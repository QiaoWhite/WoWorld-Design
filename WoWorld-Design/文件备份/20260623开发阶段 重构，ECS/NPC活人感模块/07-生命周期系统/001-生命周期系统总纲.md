## 一、模块定位

### 1.1 本模块是什么

NPC 生命周期系统是 NPC 活人感模块的第 7 个子模块。它负责 NPC 从受孕到死亡——以及死后痕迹留存——的完整生命历程中**行为层的编排**。

### 1.2 本模块不是什么

- **不是生物学事实的权威来源**——生命阶段定义、身体属性乘数、死亡原因的生物学分类、受孕/妊娠的生理过程——这些由**生命模块**拥有
- **不是教育系统**——教育是既有系统（技能/认知/GOAP/关系）之间的涌现互动模式，本模块描述这种模式但不创建"教育模块"
- **不是生育欲望系统**——生育欲望和吃饭欲望走同一条通用认知管线，没有专用通道
- **不是临终关怀系统**——临终行为从 GOAP + 身体衰退 + 认知-情绪管线自然涌现
- **不是"老化编排器"**——本模块不主动编排任何模块的行为。它定义事件协议，各模块独立订阅响应

### 1.3 三层归属分离

```
生命模块（生物学事实）      → LifeStage, DeathCause, AgeClock, GestationState, Vitals
NPC 生命周期（行为协调）    → 阶段转换响应, InfantDependency, 能力渐进, 亲子行为涌现
其余模块（各自响应）        → 认知/审美/技能/权力/历史/信仰 独立订阅生命周期事件
```

### 1.4 与生命模块的边界

| 生命模块拥有 | 本模块拥有 |
|------------|-----------|
| LifeStage enum 定义和生物学属性乘数 | NPC 在各阶段的典型行为模式（描述性，非规定性） |
| AgeClock::advance_age() 纯函数 | stage 转换时的行为门控过渡（渐进，非开关） |
| DeathCause 30 种 6 类分类 | 死亡后各模块的响应协调（事件协议） |
| GestationState + FetusBlueprint | PrenatalAccumulator 产前影响积累 |
| InfantDependency 营养依赖状态机 | 母亲/照料者 GOAP 中的育幼行为 |
| FertilityPotential 生物学曲线 | 生育欲望如何进入通用认知管线 |
| DeathSummary 内存压缩 | 死后痕迹的类型和归属 |

## 二、核心设计原则

### 原则 1：涌现优先——零年龄门控

**没有任何代码写 `if age < X { return CantDoThis }`。**

能力的每个差异都从系统内部参数自然派生。一个 NPC 不能做某事是因为他的物理属性不够、认知参数不够、技能积累不够、动作库中不知道这个动作——不是因为"年龄没到"。

一个 12 岁天才少年的战斗/政治/学术能力可能超过 40 岁平庸成人——因为他的内部参数（天赋种子 × 高学习率 × 高密度经验暴露）在某些维度上已经超过成人。一个 70 岁文盲老人的认知能力可能不如一个 10 岁受过教育的儿童——因为他的 MentalModel 网络、词汇量、抽象思维从未被充分训练。

**年龄本身不出现。年龄只是参数自然增长所耗费的时间。**

### 原则 2：零系统开关——所有系统从出生起持续运行

GOAP、审美、语言、认知、技能、关系——没有任何一个系统在某个年龄"被激活"或在某个年龄"被关闭"。新生儿的行为贫乏是因为系统内部参数处于初始值：

| 系统 | 新生儿状态 | 成人状态 | 差异来源 |
|------|----------|---------|---------|
| GOAP | planning_horizon=1, action_repertoire≈["cry","suck","sleep"] | horizon=5-7, repertoire=数百 | 认知 × 动作库积累 |
| 语言 | vocabulary≈0, grammar≈0 | vocabulary=数千, grammar≈1.0 | 暴露量 × 学习率 × 时间 |
| 审美 | taste=流动性极高，未结晶 | taste=结晶（青春期后） | 青春期 derive_taste() 是稳定化过程，不是开启 |
| 认知 | mental_age≈0, MentalModel≈0 条 | mental_age→1.0, MentalModel≤20 | 经验累积 + 睡眠正则化 |
| 技能 | XP=0, aptitude 从 mental_seed 埋入 | XP>0, 4 条路径均可 | 学习率 × 教学 × 时间 |

### 原则 3：连续模型——无阈值开关

所有能力是连续的渐变曲线。没有"从不可生育到可生育"的瞬间切换，没有"从儿童到成人"的单帧切换：

```rust
// fertility 从 0 平滑升到 1 再平滑降到 0
// 没有"今天不能生，明天就能生"的阈值
fn fertility_potential(age_pct: f32) -> f32 { /* sigmoid 连续曲线 */ }

// 认知学习率从高（童年）平滑过渡到基线（成人）
// 没有"13 岁学习率减半"的阈值
fn learning_rate_multiplier(mental_age: f32) -> f32 { /* 连续衰减 */ }
```

### 原则 4：中性事件——事件不携带情感建议

所有生命周期事件（出生、死亡、阶段转换、妊娠终止）都是**中性的生物学事实**。事件本身不携带：

- 情感建议（"此人死了→你应该悲伤"）
- 行为期望（"此人出生了→你应该庆祝"）
- 人际通知指令（"通知所有亲属"）
- 仪式触发（"自动安排葬礼"）

每个消费方基于自己的既有状态——人格、关系历史、MentalModel 网络、当前处境、感知范围——独立决定自己对事件的反应。同一个 DeathEvent，遗孀可能悲痛欲绝，债主可能暗自高兴，路过的陌生人毫无感觉。

### 原则 5：平等性——生育欲望无专用通道

生育欲望和吃饭欲望、学技能欲望、社交欲望走**完全相同的**认知管线：感知→认知→MentalModel→Aspiration→GOAP 竞争。libido 是身体信号输入的一端（和 hunger 一样是 Physiology 信号），不是"生育欲望"本身。

### 原则 6：统一时间流速——不可分层加速

所有 LOD 层共用完全相同的时间流速。NPC 的年龄在所有层以相同速度推进。L3/L4 的"批处理"是渲染粒度的批处理——把 30 天的变化一次性算完，不是让时间加速。

## 三、全生命周期概览

```
受孕 ──→ 孕期 ──→ 出生 ──→ 婴幼儿 ──→ 儿童 ──→ 青壮年 ──→ 老年 ──→ 死亡 ──→ 死后
  │        │        │         │          │         │          │        │        │
  │        │        │         │          │         │          │        │        │
Fetus   Gestation  Birth  InfantDep.  参数持续   参数峰值   衰退开始  Gompertz  LifeTrace
Blueprint State    Event  Nursing→    增长中     区间的      认知三    触发     Thought-
+种子   +产前影响  +各模  Weaning→    GOAP渐     漫长稳定    路径      死亡事件  Imprint
确定    积累      块响应  Weaned      扩展       期          社会角色   发布      亡灵潜力
```

### 3.1 各阶段概要

| 阶段 | LifeStage | 约占寿命 | 关键特征 |
|------|-----------|---------|---------|
| 孕期 | `Pregnant` 标签 | 物种特定 | 胎儿不独立存在。母体 GOAP 自然倾向低风险。产前影响积累（可开关）。FetusBlueprint 在受孕时确定 |
| 婴幼儿 | `Juvenile` 前半 | 0~10% | `InfantDependency::Nursing→Weaning→Weaned`。完全依赖照料者。L1 由母亲 GOAP 驱动；L3/L4 统计近似 |
| 儿童 | `Juvenile` 后半 + `Adolescent` | 10~25% | GOAP 持续运行。planning_horizon 随认知增长。教育从家庭→社区→专职教师三层涌现。游戏=低风险学习 |
| 青壮年 | `YoungAdult` + `Adult` + `MiddleAge` | 25~85% | 一生中最长的阶段。所有参数达峰值区间。生育、事业、创造的核心时期。MiddleAge 末期轻微身体衰退 |
| 老年 | `Elder` | 85~100% | 物理属性持续衰退。认知老化三路径（健康/病理/超常）——由 `crystallized_factor()` + `cognitive_engagement_score()` + `health_burden()` 三函数联合派生（CHG-058）。高负担触发独立致病通路 `pathological_annual_degradation()`。社会角色从生产者向传承者过渡。Gompertz 死亡率指数上升 |
| 死亡 | — | 瞬间 | 30 种死因 6 类。事件发布→各模块独立响应。ThoughtImprint 自动保存。DeathSummary 三阶段压缩 |
| 死后 | — | 持续 | LifeTrace（历史）、亡灵（生命）、记忆在他人脑中（NPC）、墓碑（世界空间） |

## 四、关键数据结构

### 4.1 生命模块新增/修改的类型

```rust
// —— woworld_life ——

/// 年龄时钟——纯函数，无状态
pub struct AgeClock;

impl AgeClock {
    /// 推进实体的年龄，返回新状态和在此期间发生的事件
    /// 纯函数：同输入 + 同种子 → 同输出。与调用频率无关。
    pub fn advance_age(
        entity: &LifeEntity,
        delta_days: f32,
        rng: &mut impl RngCore,
    ) -> (LifeEntity, Vec<LifeEvent>);
}

/// 生命周期事件——中性事实
/// 不携带情感建议、行为期望、人际通知指令
pub enum LifeEvent {
    StageTransition { entity: EntityId, from: LifeStage, to: LifeStage, at_age: f32 },
    Birth { entity: EntityId, mother: EntityId, father: EntityId, location: Vec3 },
    Death { entity: EntityId, cause: DeathCauseId, location: Vec3, spirit_remaining: f32 },
    GestationEnded { mother: EntityId, outcome: GestationOutcome },
}

pub enum GestationOutcome {
    LiveBirth { child_ids: Vec<EntityId> },
    Miscarriage { at_progress: f32 },
    Stillbirth { child_blueprints: Vec<FetusBlueprint> },
    MaternalDeath { at_progress: f32 },
}

/// 妊娠状态——在母体 LifeEntity 上
pub struct GestationState {
    pub progress: f32,              // 0.0 → 1.0
    pub duration_days: u32,
    pub fetuses: Vec<FetusBlueprint>,  // 多胞胎
}

/// 胎儿蓝图——受孕时确定
/// 编码了此 NPC 在 YoungAdult 阶段的标准模型种子
/// 出生前的外观通过 age_factor 从此蓝图插值派生
pub struct FetusBlueprint {
    pub physical_seed: [u8; 32],
    pub mental_seed: [u8; 32],
    pub elemental_seed: [u8; 32],
    pub mother_id: EntityId,
    pub father_id: EntityId,
    pub race: RaceId,
    pub race_rule: MixedRaceRule,
}

/// 营养依赖状态——在婴儿 LifeEntity 上
pub enum InfantDependency {
    Nursing { mother_id: EntityId, last_fed: GameTime },
    Weaning { mother_id: EntityId, independence: f32 },
    Weaned,
}

/// 死亡摘要——供内存压缩和后代查询
pub struct DeathSummary {
    pub entity_id: EntityId,
    pub name: String,
    pub birth_day: GameDay,
    pub death_day: GameDay,
    pub death_cause: DeathCauseId,
    pub epitaph: String,
    pub key_achievements: [AchievementRef; 5],
    pub lingering_spirit: f32,
    pub data_archive_key: Option<u64>,
}

/// 生育能力——连续生物学曲线
pub fn fertility_potential(age_pct: f32, race: &RaceTraits) -> f32;

/// 衰老生存概率——Gompertz 闭式积分
pub fn senescence_survival(
    age_pct: f32, delta_pct: f32,
    constitution: f32, health_history: f32,
) -> f32;
```

### 4.2 NPC 模块新增类型

```rust
// —— woworld_npc::lifecycle ——

/// 产前影响——在母亲身上积累，出生时转移
pub struct PrenatalAccumulator {
    pub influences: Vec<PrenatalInfluence>,
    pub enabled: bool,
}

/// 单条产前影响
pub struct PrenatalInfluence {
    pub time: GameDay,
    pub category: PrenatalInfluenceCategory,
    pub intensity: f32,
    pub source: Option<EntityId>,
}

/// 认知老化路径 ★ v1.1 (CHG-058): 由 crystallized_factor + cognitive_engagement_score + health_burden 联合派生
/// 高健康负担(>0.3)有独立致病通路——不仅是 adjusted 低加随机门
pub enum CognitiveAgingPath {
    Healthy,
    Pathological,
    SuperAging,
}

/// 丧偶事实——在中性数据，非情感预设
pub struct Widowhood {
    pub deceased_partner_id: EntityId,
    pub date: GameDay,
    pub death_cause: DeathCauseId,
    // 注意：无 grief_intensity、无 remarriage_stance
    // 情感从认知-情绪管线实时产生
}

/// 玩家控制模式
pub enum ControlMode {
    Auto,
    Manual { action_override: Action },
    DomainDelegated { manual_domains: HashSet<ActionDomain> },
}
```

## 五、事件流架构

```
Life::advance_age() 或 Life::tick_reproduction()
  │
  ├─► 产生 LifeEvent
  │
  └─► EventBus::publish(event)    ← woworld_core 的薄分发层
        │
        ├─► NPC::cognition     → 感知→解释→情绪→记忆→MentalModel 更新
        ├─► NPC::relationship  → 关系网更新（丧偶/新生收养/断交）
        ├─► NPC::goap          → GOAP 世界状态更新（母亲离世→婴儿无人照料）
        ├─► History            → LifeTrace 创建/更新、事件日志
        ├─► Power              → 继位检查（SuccessionRule 触发）
        ├─► Economy            → 遗产分配（InheritanceLaw）
        ├─► Faith              → 葬礼调度、亡灵判定、成人礼
        ├─► Model/Animation    → 骨骼缩放更新（LifeStage→骨骼乘数）
        ├─► Spatial            → 死亡: velocity→0, entity保留
        └─► Combat             → 死亡: 终止战斗中状态
```

**各模块互不感知。** EventBus 不知道谁在订阅。订阅方不知道事件来自生命模块。
