## 一、受孕

### 1.1 纯生物学触发——无社会仲裁

受孕是一个**概率事件**，不由 GOAP 直接触发，不由文化审批，不由法律批准。欲望影响了行为，行为改变了受孕所需的物理条件（伴侣共处时间），但受孕本身是生命模块在月度 tick 中评估的纯生物学概率。

```rust
fn probability_of_conception(
    female: &LifeEntity,
    male: &LifeEntity,
    cohabitation_days_this_month: f32,
) -> f32 {
    // 零概率条件（纯生物学——不受文化/法律/个人意愿影响）：
    if female.fertility_potential() < 0.001 { return 0.0; }
    if male.fertility_potential() < 0.001 { return 0.0; }
    if female.gestation_state.is_some() { return 0.0; }
    if female.is_dead || male.is_dead { return 0.0; }
    
    let base = female.race.fertility.conception_base_rate;
    let health = female.vitals.health_factor().min(male.vitals.health_factor());
    let frequency = (cohabitation_days_this_month / 30.0).min(1.0);
    
    // 哺乳期抑制（生物学事实，非文化选择）
    let lactation_suppression = if female.has_tag(Tag::Nursing) {
        female.race.fertility.lactation_amenorrhea_factor  // ~0.1-0.3
    } else { 1.0 };
    
    base * health * frequency * lactation_suppression
}
```

### 1.2 受孕＝确定 FetusBlueprint

受孕发生时，在母体的 LifeEntity 上创建 GestationState。FetusBlueprint 在**此瞬间确定**——包含该 NPC 一生中所有物理、心智、元素参数的基础种子。

```rust
fn create_fetus_blueprint(
    mother: &LifeEntity,
    father: &LifeEntity,
    rng: &mut Rng,
) -> FetusBlueprint {
    FetusBlueprint {
        physical_seed: blend_seeds(mother.physical_seed, father.physical_seed, rng),
        mental_seed: blend_seeds(mother.mental_seed, father.mental_seed, rng),
        elemental_seed: blend_seeds(mother.elemental_seed, father.elemental_seed, rng),
        mother_id: mother.id,
        father_id: father.id,
        race: determine_child_race(mother.race, father.race, rng),
        race_rule: mother.race.cross_race_rules.get(&father.race),
    }
}
```

"青壮年标准模型"的种子在此确定。出生前、儿童期、老年期的外观通过 age_factor 从此种子插值派生。不存储"多个阶段的模型"——只存储一个种子。

## 二、孕期

### 2.1 GestationState

```rust
pub struct GestationState {
    pub progress: f32,                  // 0.0 → 1.0
    pub duration_days: u32,             // 种族妊娠天数（人类 ~280 天）
    pub fetuses: Vec<FetusBlueprint>,   // 单胎或多胎
    pub start_date: GameDay,
}
```

### 2.2 孕期进程

advance_age() 在每个 tick 中推进妊娠进度。妊娠期是相对的——不同物种的 gestation 天数不同。

```rust
fn advance_gestation(
    mother: &mut LifeEntity,
    events: &mut Vec<LifeEvent>,
    delta_days: f32,
    rng: &mut Rng,
) {
    let gestation = mother.gestation_state.as_mut().unwrap();
    gestation.progress += delta_days / gestation.duration_days as f32;
    
    // 流产/死胎风险（详见 2.3）
    if gestation.progress < 1.0 {
        let loss_prob = pregnancy_loss_probability(gestation, &mother.vitals, mother.health_history, delta_days);
        if rng.gen::<f32>() < loss_prob {
            let outcome = if gestation.progress > 0.75 {
                GestationOutcome::Stillbirth {
                    child_blueprints: std::mem::take(&mut gestation.fetuses),
                }
            } else {
                GestationOutcome::Miscarriage {
                    at_progress: gestation.progress,
                }
            };
            events.push(LifeEvent::GestationEnded {
                mother: mother.id,
                outcome,
            });
            mother.gestation_state = None;
            mother.tags.remove(Tag::Pregnant);
            return;
        }
    }
    
    // 足月→分娩
    if gestation.progress >= 1.0 {
        let child_ids: Vec<EntityId> = gestation.fetuses.iter().map(|_| world.allocate_entity_id()).collect();
        events.push(LifeEvent::Birth {
            entity: child_ids[0], // 第一个孩子——多胞胎各自发出独立 BirthEvent
            mother: mother.id,
            father: gestation.fetuses[0].father_id,
            location: mother.position,
        });
        // 多胞胎的后续 BirthEvent...
        
        events.push(LifeEvent::GestationEnded {
            mother: mother.id,
            outcome: GestationOutcome::LiveBirth { child_ids },
        });
        mother.gestation_state = None;
        mother.tags.remove(Tag::Pregnant);
        mother.tags.insert(Tag::Nursing);  // 哺乳期开始
    }
}
```

### 2.3 流产与死胎——风险是累积的

怀孕不等于活产。整个孕期有累积风险——来自营养、年龄、外伤、母体健康历史。

```rust
fn pregnancy_loss_probability(
    gestation: &GestationState,
    mother_vitals: &Vitals,
    mother_health_history: f32,
    delta_days: f32,
) -> f32 {
    let base_daily = 0.001;  // 每日基础风险 0.1%
    
    // 营养修正
    let nutrition = if mother_vitals.hunger < 0.3 { 3.0 }
        else if mother_vitals.hunger < 0.5 { 1.5 }
        else { 1.0 };
    
    // 年龄修正——太年轻和太老都增加风险
    let age_pct = mother.age_years / mother.max_lifespan;
    let age = if age_pct > 0.85 { 2.5 }
        else if age_pct < 0.18 { 1.5 }
        else { 1.0 };
    
    // 孕早期风险最高（前 25%），晚期次之
    let trimester = if gestation.progress < 0.25 { 2.0 }
        else if gestation.progress > 0.75 { 1.3 }
        else { 1.0 };
    
    // 健康历史
    let health = 1.0 + mother_health_history * 2.0;
    
    let daily_risk = base_daily * nutrition * age * trimester * health;
    1.0 - (1.0 - daily_risk).powf(delta_days)
}
```

### 2.4 GestationEnded 事件是中性事实

流产/死胎事件**不携带情感预设**。母亲如何反应取决于她自己的认知加工：
- 她是否知道自己怀孕了？（生理变化是否被她的 Physiology 感知到？）
- 她的 MentalModel 中"流产"意味着什么？（教义：未出生的孩子不算人 vs 生命的开始）
- 她是否有其他孩子？是否渴望孩子多年？
- 她的信仰是否将此解读为"神的惩罚"？
- 她的人格（神经质高→更易产生痛苦/愧疚的认知归因）

同样的生理事实——两个不同的 NPC，两个完全不同的情绪轨迹。

## 三、产前影响——可开关的孕期积累

### 3.1 设计意图

产前影响是胎儿在母体内通过母亲感知到的世界。这些影响**不改变** FetusBlueprint 的物理/心智种子——那在受孕时已最终确定。产前影响只影响新生儿出生后的**初始记忆和初始 MentalModel 偏差**。

这是纯风味层，性能可开关。

### 3.2 PrenatalAccumulator

```rust
// 存储在母体的 NpcData 上（不在 LifeEntity 上——LifeEntity 只存储生物学事实）
pub struct PrenatalAccumulator {
    pub influences: Vec<PrenatalInfluence>,
    pub enabled: bool,  // 性能开关——关闭时完全不积累
}

pub struct PrenatalInfluence {
    pub time: GameDay,
    pub category: PrenatalInfluenceCategory,
    pub intensity: f32,           // 0-1
    pub source: Option<EntityId>,
}

pub enum PrenatalInfluenceCategory {
    MaternalVoice { estimated_words: u32 },
    MaternalEmotion { emotion: EmotionId, intensity: f32 },
    ExternalMusic { style: MusicStyle },
    MaternalStress { duration_minutes: f32 },
    RitualBlessing { faith_id: FaithId },
}
```

### 3.3 积累规则

- 仅在母体处于 Pregnant 标签时积累
- 积累在母体的 NPC 数据上（不是胎儿上）——出生时转移至新生儿
- 最高积累条数有上限（~32 条）
- 超上限时淘汰最旧或强度最低的记录

### 3.4 出生时的消费

出生时，PrenatalAccumulator 的 influences 转移到新生儿的初始记忆种子中。影响方式：

```
MaternalVoice(estimated_words=50000) → 新生儿对母亲声音的熟悉感
MaternalEmotion(fear, 0.8) → 新生儿初始 MentalModel "这个世界是危险的" confidence 略高
ExternalMusic(lullaby) → 新生儿对某种旋律模式的偏好
RitualBlessing → 新生儿与特定信仰的微弱连接
```

**所有偏差都是统计性的、小量的。** 不是决定性的。一个孕期母亲充满恐惧的婴儿不会"必然"成为一个焦虑的人——只是起点略微不同。后天的经历可以完全覆盖这些偏差。

## 四、出生

### 4.1 出生事件

```rust
pub struct BirthEvent {
    pub child_id: EntityId,
    pub mother_id: EntityId,
    pub father_id: EntityId,
    pub blueprint: FetusBlueprint,
    pub prenatal_influences: Vec<PrenatalInfluence>,
    pub location: Vec3,
    pub time: GameDay,
}
```

### 4.2 出生时各模块的独立响应

| 模块 | 响应 |
|------|------|
| **NPC 活人感** | 用 mental_seed 生成 BigFive；用 physical_seed 生成标准模型外观；初始化 Identity{age:0, birth_day}；消费 prenatal_influences → 初始记忆种子 |
| **认知系统** | `inherit_initial_models(parent1, parent2, settlement, cultural_baseline)` — 已有 |
| **信仰** | `ChildFaithProfile` 继承规则 — 已有 (CHG-025) |
| **空间查询** | `SpatialEntity::register(entity_kind=Creature)` — 已有 |
| **历史** | 记录 `HistoricalEvent::NpcBirth` — 新增事件类型 |
| **权力** | 检查继承管线 — 若父母持有头衔，出生触发继承权登记 |
| **生命** | 母亲移除 GestationState + Pregnant 标签，添加 Nursing 标签 |
| **模型物理** | 新生儿骨骼 = 标准模型的 ~0.4×（按 Juvenile 比例派生） |
| **关系** | 创建 Parent-Child Relationship（双向） |

### 4.3 出生事件也是中性事实

BirthEvent 不携带"这是一个值得庆祝的事件"的信号。附近 NPC 的反应取决于他们的关系、处境、文化：

| NPC | 可能的涌现反应 |
|-----|-------------|
| 母亲，婴儿健康，她一直想要 | 她自己的期待得到满足→喜悦从内部产生 |
| 父亲，婴儿是女儿但在重男轻女文化中 | MentalModel "家族断了香火"被激活→失望/焦虑 |
| 父亲，已经养不起更多孩子 | NpcEconomicState 感知→焦虑 |
| 大孩子，父母注意力全部转向婴儿 | Relationship 感知到注意力的转移→嫉妒 |
| 奴隶主，女奴生了孩子=多一个奴隶 | 算计——婴儿是财产增量 |
| 路过的陌生人 | 毫无感觉——不在关系网中，不过是一声婴儿啼哭 |

**没有"附近所有 NPC 都高兴"的系统。** 每个人提取自己的关系、记忆、MentalModel，形成自己的结论。

## 五、避孕——无专门系统

没有"避孕系统"。避孕效果从以下途径自然涌现：

| 途径 | 机制 |
|------|------|
| 分居两地 | 空间距离→共处时间为零→受孕概率为零 |
| 贫困/饥荒 | 身体条件差→fertility_potential 下降→受孕概率降低 |
| 禁欲教义 + 虔诚 | GOAP 中与伴侣共处的目标被压制 |
| 长哺乳期 | 哺乳期 amenorrhea→受孕概率极低（生物学事实） |
| 避孕物品（未来 mod） | ConsumableEffect 临时降低 fertility_potential——一个物品，不构成系统 |
