## 〇、总览

感官与知觉系统是 WoWorld NPC 感知世界的**统一入口**。它介于世界物理真值与 NPC 心智之间——不做决策、不存记忆、不辨识实体，只做信息压缩与变形。

**一句话**：感官系统将世界真值转化为每个 NPC 独一无二的知觉流（`PerceptBatch`），供给日常决策、战斗情报、审美判断和大日志。

### 核心设计承诺

1. **无状态** — 感官系统是纯函数管线。不拥有世界状态。所有状态通过参数传入
2. **不辨识** — 感官只提取物理特征，不做"这是龙"或"他受伤了"的语义判断
3. **不整合跨模态** — 视觉和听觉保持独立输出。听到脚步声+看到影子→是不是同一个人→是信念推导层的事。★ v1.1: 基础 `perceive()` 保持不整合。跨模态绑定移至 `enrich()` 桥梁层 (`PerceptBatch::bind_modalities()`)，由消费者按需调用。
4. **不定义事件边界** — 感官输出知觉片段流。记忆系统自己决定什么时候编码事件
5. **统一产出** — 同一份 `PerceptBatch` 同时供日常决策、战斗情报、审美判断、大日志消费

## 一、架构定位

### 1.1 在 NPC 六层模型中的位置

NPC ver2.0 定义了六层架构。感官系统占据**第 3 层**——"感官外壳"：

```
Layer 1: NpcData — 数据本体
Layer 2: Mind Core — 情绪+记忆+决策+社会+文化+空间
Layer 3: Sensory Shell — ★ 本系统。视觉+听觉+嗅觉+注意+内感受+社会感知
Layer 4: Physical Body — Godot 侧物理体
Layer 5: Social Projection — 其他 NPC 如何记住/看待此 NPC
Layer 6: Statistical Layer (L4) — 超远距纯统计 NPC
```

### 1.2 与已有感知通道的关系

感官系统**不重新实现**已有感知通道。它做编排+整合，并在缺失的通道（视觉、嗅觉）上提供新实现：

| 通道 | 状态 | 感官系统的角色 |
|------|------|-------------|
| 听觉 | ✅ 音频系统完成 | 编排消费 `AudioQuery` |
| 天象 | ✅ NPC crate 完成 | 编排消费 `perceive_sky()` |
| 内感受 | ✅ Life crate 完成 | 编排消费 `Physiology::from_vitals()` |
| 情绪感知 | ✅ NPC crate 完成 | 编排消费情绪传染引擎 |
| 非语言信号 | ✅ 语言表达完成 | 编排消费 `NonVerbalPerception` |
| 流言/声望 | ✅ 语言表达+信息传播完成 | 编排消费 `SettlementInformationPool` |
| **视觉** | ❌ **缺失** | **感官 crate 从零实现 `VisionQuery`** |
| **嗅觉** | ❌ **缺失** | **感官 crate 从零实现 `ScentQuery`** |
| 魔法感知 | ⚠️ 预留 | v1.0 延迟——预留 `Option<&dyn MagicPerception>` |

### 4.4 DarkAdaptation

```rust
/// 暗适应状态——指数松弛模型
pub struct DarkAdaptation {
    pub current_level: f32,   // 0=未适应暗, 1=完全适应
    pub target_level: f32,    // 由环境光决定
}

impl DarkAdaptation {
    pub fn update(&mut self, ambient_light: f32, dt: TimeDelta) {
        self.target_level = if ambient_light < 0.1 { 1.0 } else { ambient_light };
        let rate = if self.target_level > self.current_level { 0.0005 } else { 0.05 };
        let alpha = 1.0 - (-rate * dt.as_seconds() as f32).exp();
        self.current_level += (self.target_level - self.current_level) * alpha;
    }
}
```

```rust

/// NPC 对周围人群情绪的感知。从 PerceptEntry 聚合，在 perceive() 期间一次计算。
pub struct EnvironmentPerception {
    // ... 已有字段（温度、风速、光照等）...
    
    /// 感知到的周围人群情绪强度
    /// 0.0 = 冷静/空旷的人群，1.0 = 强烈的集体情绪
    /// 聚合自 PerceptEntry.emotional_charge — 加权距离和显著性
    /// 3 人以下不算"人群"
    pub crowd_emotional_field: f32,
    
    /// 人群的主导情绪（None = 无明确人群或无主导情绪）
    pub crowd_dominant_emotion: Option<BasicEmotion>,
}

/// 在 perceive() 末尾计算
fn compute_crowd_field(entities: &[PerceptEntry]) -> (f32, Option<BasicEmotion>) {
    let mut intensity_sum = 0.0f32;
    let mut count = 0u32;
    let mut emotion_counts: HashMap<BasicEmotion, f32> = HashMap::new();
    
    for e in entities.iter().filter(|e| {
        e.classification >= EntityIdentification::Categorized
        && e.detail_level >= PerceptDetail::MotionAndShape
    }) {
        let w = e.salience_raw * (1.0 - e.distance_estimate / 15.0).max(0.0);
        intensity_sum += e.emotional_charge.arousal * w;
        if let Some(emo) = e.emotional_charge.primary_emotion {
            *emotion_counts.entry(emo).or_default() += w;
        }
        count += 1;
    }
    
    if count < 3 { return (0.0, None); }
    
    let field = (intensity_sum / count as f32).clamp(0.0, 1.0);
    let dominant = emotion_counts.iter()
        .max_by(|a, b| a.1.partial_cmp(b.1).unwrap())
        .map(|(e, _)| *e);
    
    (field, dominant)
}
```

## 十一、特殊感官通道

### 11.1 痛觉——唯一绕过决策周期的感官通道

痛觉不走决策周期。它是**中断**——对标战斗系统中的攻击判定。攻击命中→`DamageEvent`→感官系统的事件通道（`interrupt_check()`）立即处理：

```
攻击命中 NPC（任何时间点）:
  → DamageEvent 推送至 event_bus
  → 感官系统的 interrupt_check() 被触发（事件通道——不在决策周期内）
  → pain_intensity = damage / max_hp × pain_sensitivity
  → 如果 pain_intensity > threshold:
      → 标记 npc.urgent_percept（缩短下次决策周期到 0.05s）
      → 强制注意力转移：清空 attention_focus → 锁定到伤害来源方向
      → 直接写入情绪引擎：pain → fear/anger
      → 直接写入 PerceptualCache：伤害来源实体 + "疼痛"标签
```

**痛觉不可被注意力过滤。** 不论 NPC 在做什么——睡觉、打铁、专注看书——剧痛无条件打断。这是生理学的硬编码。

### 11.2 触觉——分散在已有系统中，感官编排整合

| 触觉类型 | 信息来源 | 进入感官的路径 |
|---------|---------|-------------|
| 温度（冷/热） | `Physiology::perceived_temperature()` + `WeatherSample.ground_temperature` | 编排消费→`BodyPerception` |
| 接触（被推/被碰/被击中） | 物理引擎碰撞事件→`SpatialEvent` buffer | 事件通道→`PerceptBatch` |
| 表面质感（粗糙/光滑/湿滑） | `SpatialQuery::surface_material_at()` | 编排消费→`VisualScene.dominant_surface` |
| 风（被风吹到） | `WeatherSample.ground_wind_speed + gust_factor` | 编排消费→`EnvironmentPerception` |
| 伤害痛觉 | 战斗系统 `DamageEvent` | §11.1 的事件中断通道 |

### 11.3 味觉——进食时触发，不在常规感官周期内

味觉仅在 NPC **进食**时激活——不作为常规感知周期的一部分。对标 `ConsumableEffect` 的模式：

```
NPC 进食:
  → 物品系统提供 ConsumableEffect（Life 004 §十三定义 schema，物品存储）
  → taste_profile = ConsumableEffect.taste  // 甜/咸/酸/苦/鲜/辣
  → 感官系统的 taste_evaluate(taste_profile, npc) → TastePerception
  → 情绪引擎: 美味→pleasure↑, 难吃→disgust↑
  → 记忆编码: 特别好吃/难吃→写入记忆
```

## 十二、空间氛围——VisualScene→情绪的直连通路

某些空间属性**直接**触发情绪反应——不经审美系统，不经信念推导。这是生理性、跨文化的感官→情绪硬连线，对标已有的"血腥气味→恐惧"的嗅觉→情绪映射：

```rust
/// 空间氛围——感官系统直接从 VisualScene 映射到情绪调制
/// 不经审美判断，不经信念推导
/// 对标已有的"气味→情绪"硬编码映射
pub fn spatial_mood_impact(scene: &VisualScene, npc: &NpcData) -> EmotionDelta {
    let mut delta = EmotionDelta::zero();

    // 封闭+暗 → 压抑/恐惧（幽闭恐惧的生理基础）
    if scene.openness < 0.3 && scene.light_level < 0.2 {
        delta.pleasure -= 0.05 * (1.0 - npc.personality.openness);
        delta.arousal += 0.03;  // 警觉
    }

    // 开阔+高+多层深度 → 壮阔/眩晕
    if scene.openness > 0.8 && scene.depth_layers > 3 {
        delta.pleasure += 0.03;
        delta.arousal += 0.02;
    }

    // 拥挤+高密度 → 焦虑（感官过载）
    if scene.entity_density > 0.7 {
        delta.pleasure -= 0.02 * npc.personality.neuroticism;
    }

    // 完全寂静+孤立 → 不安或平和（取决于人格）
    if scene.entity_density < 0.03 && scene.openness > 0.6 {
        if npc.personality.neuroticism > 0.6 {
            delta.pleasure -= 0.03;  // 高神经质→孤独=不安
        } else {
            delta.pleasure += 0.02;  // 低神经质→孤独=宁静
        }
    }

    delta
}
```

**注意**：这不是审美。一个完全没有审美能力的 NPC——文化 `artistry=0`、个性 `openness=0`——仍然会在黑暗封闭空间中感到不安。这是生理性的。

## 十三、prepare_facts() — 信念推导的辨识三层回落

感官系统输出 `PerceptEntry`（含视觉特征+身体状态+装备特征——全是物理特征，无语义标签）。`prepare_facts()` 将特征转化为 NPC 能理解的**事实**。辨识过程分三层回落：

```
percept: "在位置(10,0,5)的实体，视觉特征=[大型,有翼,深色,鳞片,嘴喷火]"

辨识过程:

L1 直接记忆匹配: percept.entity_id 存在 → 查 NPC 记忆中的识别记录
    → "entity_id=12347 → 这是铁匠艾隆" ← 不需要 Knowledge，记忆就够
    → confidence = percept.confidence × memory_record.familiarity

L2 原型匹配: entity_id 未知 → 提取视觉特征 → 查三个知识源:
    2a. 个人 Knowledge: entries.find_by_features(&features)
        → "这种蘑菇我见过——特征是伞盖边缘锯齿状"
    2b. 文化基线: culture_kb.category_prototype(culture_id, features)
        → "我所属的文化知道龙长什么样"
    2c. 替代文化框架: 遍历 adopted AestheticFrameworks 中带知识条目的文化
        → "南方文化说这个叫'火翼兽'"
    → 最佳匹配: CategoryTag::Dragon (similarity=0.87)
    → 如果 similarity < 0.7 → 标记为 Vague（不确定但最接近的猜测）
    → novelty = 1.0 - similarity → 驱动惊奇
    → 如果全不匹配 → 进L3

L3 基础特征分类: 硬编码基线——所有 SapientMind NPC 都有的能力
    → 取实体最显著特征: "大型"+"有翼"+"飞行"+"嘴喷火"
    → classify_basic(&features) → "巨大的有翼飞行生物,会喷火"
    → confidence = percept.confidence × 0.5
    → novelty = 1.0 (完全不认识 → 最大惊奇)

惊奇驱动的知识创建:
    如果 novelty > 0.8 且 consecutive_observations > 20:
    → 自发创建个人知识原型:
        tag = CategoryTag::from_features(&features, npc.npc_id)
        // 个人临时标签——不是全局CategoryTag
        npc.knowledge.add(tag, LearnedFromExperience, confidence=0.3)
    → "这个我不认识的东西，我给它记个名字——至少记住它的特征"
    → 涌现的是个人的"民间分类"——对标现实中的"那个长得像鹿但不是鹿的东西"
```

**注意**：L3 的 `classify_basic()` 返回的是自然语言描述——不是预设的枚举。它从特征中组合出最大可能的描述："人形" / "四足动物" / "飞行生物" / "蛇形" / "水生" / "未知"。这是所有智能 NPC 的基线——不需要 Knowledge 条目。

## 十四、期望验证与实体消失检测

NPC 需要知道"刚才还在那里的东西现在不在了"——这不靠置信度衰减到零。靠**期望验证失败**：

```rust
/// prepare_facts() 中的交叉比对——期望验证
fn verify_expectations(cache: &PerceptualCache, batch: &PerceptBatch, npc: &NpcData) -> Vec<Fact> {
    let mut facts = Vec::new();

    for cached in &cache.entries {
        // 检查: 缓存中的实体是否仍在当前视野中？
        let still_visible = batch.entities.iter().any(|p| p.entity_id == cached.entity_id);

        if !still_visible {
            // 东西不在当前知觉中了
            if was_in_expected_fov(&cached.position_estimate, npc.position, npc.facing, npc.sensory.vision.fov_horizontal_deg) {
                // 它本应该在视野里但没看到 → 消失了/被移动了/被挡住了
                facts.push(Fact::Vanished {
                    entity_id: cached.entity_id,
                    last_seen_at: cached.last_perceived,
                    last_known_position: cached.position_estimate,
                    confidence: cached.confidence * 0.7,  // "可能还在...不确定"
                });
                // 注意: 缓存中的条目保留，但标记为"未确认"
                // 下次决策周期如果重新出现→恢复；如果连续多周期消失→移除
            }
            // 如果不在当前视野 → 不触发——"可能还在，只是我没看那边"
        }
    }

    // 反过来: 当前知觉中有但缓存中没有 → 新东西 → 注意
    for percept in &batch.entities {
        let is_new = !cache.entries.iter().any(|c| c.entity_id == percept.entity_id);
        if is_new {
            facts.push(Fact::Appeared {
                entity_id: percept.entity_id,
                position: percept.position_estimate,
                classification: percept.classification.clone(),
                confidence: percept.confidence,
            });
        }
    }

    facts
}
```

**关键设计**："消失"和"没看见"的区别。如果 NPC 转头了——不再看向那个方向——没看到的东西不触发消失检测。只有当 NPC 看着的方向上，之前存在的实体不再可见时——才触发"消失了"。这防止了"转身后所有东西都标记为消失"的误报。

## 十五、ConcealmentState — 潜行的统一隐蔽模型

潜行不只是声音（SilenceIntent 已有）。视觉和嗅觉也需要压制：

```rust
/// NPC/玩家的隐蔽姿态——影响所有感官模态的"可被感知度"
/// 写入 SpatialEntity.concealment，由感官系统在感知时读取
pub struct ConcealmentState {
    /// 声音压制——已有（音频系统的 SilenceIntent）
    pub silence: Option<SilenceIntent>,

    /// 视觉隐蔽——感官系统新增
    pub visual: VisualConcealment,

    /// 气味压制 (0-1, 0=正常, 1=完全无味)
    pub scent_suppression: f32,
}

pub enum VisualConcealment {
    Normal,                                         // 正常站立/行走——不可见性=0
    Crouching { silhouette_reduction: f32 },         // 蹲下——降低轮廓。0.3=减少30%
    BehindCover { cover_quality: f32 },              // 掩体后——依赖掩体质量 0-1
    InShadow { darkness_factor: f32 },               // 暗处——依赖光照水平 0-1
    Camouflaged { effectiveness: f32 },              // 主动伪装（魔法/自然）0-1
    FullCover,                                       // 完全不可见——墙后
}
```

**VisionQuery 消费方式**：在做可见性判定时，`ConcealmentState` 作为实体属性传入：

```
判定逻辑:
    Normal → 标准可见性公式
    Crouching → effective_range *= (1.0 - silhouette_reduction × 0.5)
    BehindCover → if !spatial.line_of_sight(to_entity_center) → 不可见
    InShadow → light_level_threshold += darkness_factor × 0.5
    Camouflaged → detail_level 降一级（Full→Blurred, Blurred→MotionOnly）
    FullCover → 无条件不可见（无视线）
```

**SilenceIntent 已有——音频系统在处理 SoundFootprint 衰减时自动应用。感官不需要额外处理。**

## 十六、digest_aesthetic_experience() — 审美消化器

对标语言表达系统的 `digest_conversation_to_memory()`——对话结束后将对话内容消化为记忆。审美消化器在高情绪事件发生后将经历的审美维度提取出来，修改个人审美框架：

```rust
/// 审美经历消化——从已编码的记忆事件中检测审美维度并修改框架
/// 由 NPC 决策循环在记忆编码后调用（见 §六 步骤5）
fn digest_aesthetic_experience(
    memory_event: &EventMemory,
    percept_batch: &PerceptBatch,   // 事件发生时的知觉快照
    frameworks: &mut AestheticFrameworks,
    npc: &NpcData,
) {
    // 只处理高情绪事件——低情绪不值得修改框架
    if memory_event.emotional_impact < 0.5 { return; }

    let scene = &percept_batch.visual_scene;

    // 从 VisualScene 的物理事实映射到审美维度
    let primary_dimension = map_scene_to_dimension(scene);
    // 规则:
    // sky_exposure>0.5 + light_direction==Horizontal → Twilight (黄昏/黎明)
    // openness>0.7 + depth_layers>3 → Vastness (辽阔)
    // light_level<0.2 → Darkness
    // entity_density<0.05 + openness>0.5 → Silence (寂静/空旷)
    // 否则 → General

    let experience = AestheticExperience {
        emotional_intensity: memory_event.emotional_impact,
        emotional_valence: memory_event.emotional_valence,
        primary_dimension,
        experience_type: classify_experience_type(memory_event),
        // ChangeEvent (改变人生的事件→大幅修改框架)
        // SignificantEvent (显著事件→中等修改)
        // MildImpression (温和印象→微小修改)
    };

    // 修改个人框架
    frameworks.integrate_experience(&experience, npc);

    // 如果事件强烈+有其他文化的NPC参与→可能接触替代审美框架
    if experience.emotional_intensity > 0.8 {
        for participant in memory_event.other_participants {
            if participant.culture_id != npc.culture_id {
                frameworks.try_adopt_framework(
                    participant.culture_id,
                    exposure_quality = experience.emotional_intensity * 0.6,
                    npc,
                );
            }
        }
    }
}
```

**关键**：这不是在事件发生时实时运行的——是在**记忆编码后**运行的。事件已进入长期记忆——消化器决定"这个事件值得修改我对美的理解吗？"这保证了审美框架只被真正有影响的事件改变——对标"你不会因为吃了一顿普通的饭而改变对食物的偏好，但一顿米其林三星可能会"。
