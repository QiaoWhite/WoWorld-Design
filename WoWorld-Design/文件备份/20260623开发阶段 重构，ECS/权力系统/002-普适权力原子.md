## 一、核心理念

权力的最小不可分单元——一个实体对另一个实体能执行的**最基础控制操作**。任何"角色"（国王、族长、行会会长、父亲）都可以表示为这 17 个原子的特定子集。

### 设计约束

1. **原子不可再分**：每个原子是最小操作——不能被进一步拆解为更基础的操作序列
2. **原子可组合**：任意原子组合产生复合权力。"国王" = `[Extract + Constrain + Compel + PrescribeRule + Delegate + ...]`
3. **domain 参数化**：同一原子在不同 domain 上有不同效果（`Extract(tax)` ≠ `Extract(labor)`）
4. **原子间有联动**：某些原子组合有预设的联动规则（如 `ConferRank` 可批量附带 `Delegate` 边）

---

## 二、五分类

| 类别 | 操作对象 | 特征 |
|------|---------|------|
| **结构原子** (5) | 权力拓扑图本身的节点和边 | 创建/修改关系的基础设施 |
| **自指原子** (1) | holder 自身 | holder = subject |
| **关系原子** (6) | subject 的行为/资源/状态 | 单向，通过已存在的边作用于 subject |
| **规范原子** (2) | subject 群体的义务集合 | 创建或豁免通用规则 |
| **裁决原子** (3) | 争议与结果 | 处理违规和纠纷 |

## 三、AtomTypeTag 与 PowerDomain

### 3.1 AtomTypeTag

```rust
/// 原子类型标签——u8 编码，17 种
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, Serialize, Deserialize)]
#[repr(u8)]
pub enum AtomTypeTag {
    // 结构 (0-4)
    Constitute = 0,
    DefineMembership = 1,
    Delegate = 2,
    Relinquish = 3,
    Contract = 4,
    
    // 自指 (5)
    Pledge = 5,
    
    // 关系 (6-11)
    Constrain = 6,
    Compel = 7,
    Extract = 8,
    Access = 9,
    ConferRank = 10,
    Represent = 11,
    
    // 规范 (12-13)
    PrescribeRule = 12,
    Derogate = 13,
    
    // 裁决 (14-16)
    Adjudicate = 14,
    Sanction = 15,
    Remit = 16,
}
```

### 3.2 PowerDomain

权力的适用范围——定义"在什么领域、对什么事务"行使权力。

```rust
#[derive(Debug, Clone, PartialEq, Eq, Hash, Serialize, Deserialize)]
pub enum PowerDomain {
    /// 领土域——在特定地理范围内自动适用
    Territory {
        bounds: WorldBounds,       // 经纬度多边形
        auto_apply: bool,          // 进入此区域的人是否自动受约束
    },
    /// 市场域——经济交易
    Market {
        market_id: Option<MarketId>,
        item_category: Option<ItemCategory>,
    },
    /// 行为域——对 subject 某类行为的约束
    Behavior {
        behavior_type: BehaviorType,
        scope: BehaviorScope,      // Always / InPublic / InPrivate / InSpecificPlace
    },
    /// 信息域——对信息的访问/控制
    Information {
        info_type: InfoType,       // PersonalData / StateSecret / Archive / Correspondence
    },
    /// 身份域——对成员资格/等级的控制
    Identity {
        group_id: Option<GroupId>,
        rank_tier: Option<u8>,
    },
    /// 通用域——无特定范围限制
    Universal,
}
```

## 四、17 个原子完整定义

### 4.1 结构原子

#### 4.1.1 Constitute — 创建/解散权力结构

**含义**：创建、重组或解散一个权力结构（团体、部门、国家、条约组织）。这是"创世"原子——它改变拓扑图的节点和边集合。

**行使者**：任何持有 `Constitute` 权力的实体，或任何实体（对于自组织团体）。

**参数**：
```rust
struct ConstituteParams {
    /// 要创建的结构类型
    structure_type: StructureType,
    /// 结构的初始治理规则
    governance: GovernanceRules,
    /// 初始成员及其角色
    initial_members: Vec<(EntityId, MemberRole)>,
    /// 结构名称
    name: String,
}
```

**效果**：
- 如果 `action = Create`：创建 `EntityId::Group` 节点 → 为每个初始成员创建 `DefineMembership` 边 → 按 governance 规则创建内部权力边
- 如果 `action = Dissolve`：软删除该结构的所有出边和入边 → 释放相关 Duty
- 如果 `action = Reorganize`：修改 governance 规则，重新计算内部权力边

**性能**：低频操作（团体创立/解散是罕见事件）。O(N) 其中 N = 成员数，通常 <100。

---

#### 4.1.2 DefineMembership — 决定成员资格

**含义**：授予、撤销、或暂停某实体在权力结构中的成员资格。

**行使者**：持有对目标结构的 `DefineMembership` 权力的 holder。

**参数**：
```rust
struct MembershipParams {
    group_id: EntityId,       // 目标结构
    target: EntityId,         // 被操作者
    action: MembershipAction, // Admit / Expel / Suspend / Restore
    reason: Option<String>,   // 操作原因（叙事用）
}
```

**效果**：
- `Admit`：创建 subject 对 group 的入边，按 group 的 governance 分配初始角色
- `Expel`：软删除该成员在 group 内的所有权力边（入边+出边）
- `Suspend`：暂停成员资格但不删除边（边保持但 active=false）
- `Restore`：恢复暂停的边

**与 NPC 系统的联动**：`Expel` → subject 失去 `SocialIdentity.group_roles` 中对应条目。`Admit` → 创建对应条目。

---

#### 4.1.3 Delegate — 转移权力给他人

**含义**：将自己的一条或多条权力边转移给另一个实体行使。Delegator 仍保有原权力（除非 `Relinquish`）——Delegate 是"借出使用权"，不是"放弃所有权"。

**行使者**：持有某权力边的 holder。

**参数**：
```rust
struct DelegateParams {
    receiver: EntityId,                  // 接收权力的实体
    delegated_atoms: Vec<AtomTypeTag>,   // 委托哪些原子
    domain_filter: Option<PowerDomain>,  // 限缩 domain（可选——不填则全委托）
    revocable: bool,                     // 是否可撤销
    valid_until: Option<GameTimestamp>,  // 委托期限
    constraints: Vec<PowerConstraint>,   // 附加约束（如"不能 Extract 超过 X"）
}
```

**效果**：
- 为 receiver 创建一条或多条权力边，边的 `source = PowerSource::Appointed(delegator)`
- 新增的边的 `valid_until` 为委托期限，`succession_rule = SuccessionRule::RevertToSuperior`
- Delegator 的原边不受影响（他仍可并行行使）

**循环防护**：
```rust
fn validate_delegate_edge(&self, granter: EntityId, receiver: EntityId) -> Result<(), PowerError> {
    let mut visited = HashSet::new();
    let mut current = receiver;
    while let Some(next) = self.delegation_target(current) {
        if next == granter {
            return Err(PowerError::CircularDelegation);
        }
        if !visited.insert(next) {
            return Err(PowerError::CircularDelegationChain);
        }
        current = next;
    }
    Ok(())
}
```

---

#### 4.1.4 Relinquish — 主动放弃权力

**含义**：Holder 主动永久放弃自己持有的某条权力边。无接收方——权力消失或回归上级委托链。

**行使者**：持有该权力边的 holder（只有 holder 自己可以放弃自己的权力）。

**参数**：
```rust
struct RelinquishParams {
    edge_id: EdgeId,        // 要放弃的边
    reason: Option<String>, // 放弃原因（叙事用）
}
```

**效果**：
- 软删除目标边（`active = false`）
- 如果是 `source = Appointed(superior)` 的边 → 权力回归 superior
- 触发所有从该边派生的 Delegate 边的级联失效
- 生成 `PowerEvent::PowerRelinquished`

---

#### 4.1.5 Contract — 创建双边契约 ★唯一双边原子★

**含义**：两个实体互相约定——彼此对对方创建一组受限的权力边。Contract 需要**双方同意**才能成立（区别于其他单边原子的 holder 宣告即可成立）。

**行使者**：两个愿意的实体。

**参数**：
```rust
struct ContractParams {
    party_a: EntityId,
    party_b: EntityId,
    /// A 对 B 承担的义务（→ B 获得对 A 的权力）
    obligations_a_to_b: Vec<ObligationSpec>,
    /// B 对 A 承担的义务（→ A 获得对 B 的权力）
    obligations_b_to_a: Vec<ObligationSpec>,
    /// 相互依存：如果一方严重违约，另一方的义务是否自动解除
    interdependent: bool,
    /// 终止条款
    termination: TerminationClause,
    /// 期限（无限期 = None）
    valid_until: Option<GameTimestamp>,
    /// 见证人（可选——增强 enforceability）
    witnesses: Vec<EntityId>,
}
```

**效果**：
- 创建一对对称的受限权力边（`mutual_consent: true` 标记）
- 创建 `ContractRecord` 关联两条边
- 如果 `interdependent = true`：任一边因违约被撤销 → 另一边自动失效
- 每条 ObligationSpec 映射到具体的原子类型（`Extract`/`Compel`/`Constrain`/`Access`）

**区别**：
- `Contract`：双方互有权力和义务，需要双方同意
- `Delegate`：单向转移，不需要接收方同意
- `Pledge`：自我约束，不涉及他人

---

### 4.2 自指原子

#### 4.2.1 Pledge — 自我约束

**含义**：Holder 对自己施加一个有约束力的承诺。Holder = Subject。这是唯一一个 holder 和 subject 重合的原子。

**为什么独立**：
- 不能归入 `Constrain`（Constrain 需要 holder ≠ subject）
- 不能归入 `Contract`（Contract 需要两方）
- Pledge 是社会契约的原子基础——国王的加冕誓言、骑士的效忠誓言、婚姻誓言、信托义务

**行使者**：任何实体（对自己）。

**参数**：
```rust
struct PledgeParams {
    content: PledgeContent,      // "我承诺 X"
    witnesses: Vec<EntityId>,    // 见证人——增强约束力
    sanction_on_breach: Option<SanctionSpec>,  // 违约制裁
    valid_until: Option<GameTimestamp>,
    /// 对谁的承诺（可选——对神/对人民/对抽象原则）
    pledged_to: Option<EntityId>,
}
```

**效果**：
- 创建一条指向 holder 自身的 Duty
- 如果指定了 `pledged_to`，该实体获得"期待 holder 履约"的 soft expectation
- 如果指定了 `sanction_on_breach`，违约时 guardian（或见证人）获得执行制裁的合法依据
- 违约 → Duty breach → legitimacy 下降 + 可能的制裁

**涌现现象**：
- 国王加冕时 `Pledge`"公正统治" → 后续 Extract 过高 → 违反 Pledge → legitimacy 暴跌 + 贵族/人民获得反抗的道德依据
- 骑士 `Pledge`"效忠领主" → 战斗中逃跑 → 违反 Pledge → 被剥夺骑士头衔（ConferRank 的逆操作）
- 婚姻 `Contract` + 双方各自的 `Pledge` → 完整的婚姻法律/社会结构

---

### 4.3 关系原子

#### 4.3.1 Constrain — 约束行为范围

**含义**：禁止或限制 subject 的某些行为类型。

**行使者**：持有对 subject 的 `Constrain` 权力的 holder。

**参数**：
```rust
struct ConstrainParams {
    behavior_type: BehaviorType,  // 被约束的行为类型
    restriction: RestrictionType, // Prohibit / Limit(freq) / RequirePermission
    scope: BehaviorScope,        // Always / InPublic / InPrivate / InSpecificPlace
}
```

**效果**：
- subject 尝试执行被约束的行为 → `PowerExerciseError::BehaviorConstrained`
- 创建对应的 Duty（subject 负有"不执行 X 行为"的义务）
- 可用于：宵禁、武器禁令、出境限制、言论限制、职业限制

**与 Access 的区别**：
- `Constrain` → 约束**行为类型**。不依赖特定位置。"你不许离开这个城"
- `Access` → 控制**物理/信息空间准入**。依赖具体地点或数据。"这把钥匙能开这个门"

判据：如果限制只在一个特定位置/信息源生效 → `Access`；如果限制跟随 subject 到处生效 → `Constrain`。

---

#### 4.3.2 Compel — 强制行为

**含义**：强制 subject 做出特定行为。

**行使者**：持有对 subject 的 `Compel` 权力的 holder。

**参数**：
```rust
struct CompelParams {
    behavior_type: BehaviorType,     // 被强制的行为
    duration: Option<GameDuration>,  // 持续时间（如兵役 2 年）
    frequency: Option<u32>,          // 频率（如每月巡逻 4 次）
}
```

**效果**：
- 创建对应的 Duty（subject 负有"执行 X 行为"的义务）
- subject 不执行 → Duty breach → Sanction 塌缩链
- 可用于：兵役、劳役、出庭、纳税（但纳税通常用 Extract 更精确）、巡逻

---

#### 4.3.3 Extract — 强制获取资源

**含义**：从 subject 获取资源——税收、贡品、劳役产物、罚款、征用。

**行使者**：持有对 subject 的 `Extract` 权力的 holder。

**参数**：
```rust
struct ExtractParams {
    resource_type: ResourceType,     // Copper / Item / Labor / Service
    amount: u64,                     // 数量
    frequency: ExtractFrequency,     // Once / Daily / Weekly / Monthly / Yearly
    purpose: Option<ExtractPurpose>, // Tax / Tribute / Fine / Requisition / Fee
}
```

**效果**：
- 从 subject 的钱包/库存/劳动时间中扣除对应资源
- 资源转移到 holder（或 holder 的 treasury）
- 创建对应的 Duty（持续型——每次到期自动评估）
- subject 不足 → Duty breach → Sanction 塌缩链

**Extract 频率**：持续型 Extract（如月税）通过 Duty trigger 自动执行，不需要 holder 每次手动 `exercise_power`。

---

#### 4.3.4 Access — 控制物理/信息准入

**含义**：授予或撤销 subject 进入特定物理位置或获取特定信息的权限。

**行使者**：持有对某个位置/信息源的 `Access` 权力的 holder。

**参数**：
```rust
struct AccessParams {
    target: AccessTarget,        // PhysicalEntrance / DigitalRecord / Archive / RestrictedArea
    level: AccessLevel,          // Full / ReadOnly / TimeLimited / Escorted
}
```

**效果**：
- subject 尝试访问受限区域/信息 → 检查 Access 边 → 不通过 → `PowerExerciseError::AccessDenied`
- `level = ReadOnly` 时，subject 可看但不可修改/带走

---

#### 4.3.5 ConferRank — 赋予/调整等级地位

**含义**：在群体内部赋予、提升、降低或撤销 subject 的等级地位。

**行使者**：持有对群体内成员 `ConferRank` 权力的 holder。

**参数**：
```rust
struct ConferRankParams {
    rank_title: RankTitle,           // 头衔名称（如"公爵"、"上校"、"长老"）
    rank_tier: u8,                   // 等级数字（0=最低）
    attached_powers: Vec<EdgeId>,    // 此 rank 自动附带的权力边（创建时一并授予）
    revocable: bool,                 // 是否可被撤销
}
```

**效果**：
- 授予 rank → 自动创建附带的权力边（`attached_powers` 中的边都以 subject 为 holder 创建）
- 撤销 rank → 级联软删除所有附带的权力边
- 提升/降低 → 调整 `rank_tier`，可能改变附带的权力边范围

**关键设计**：`ConferRank` 不直接创建权力边——它通过 `attached_powers` 声明该 rank 附带哪些权力。当 rank 被授予时，系统在这个原子事务中**一并创建**附带的 Delegate 边。这避免"有爵位无权力"或"有权力无爵位"的数据不一致。

---

#### 4.3.6 Represent — 代表群体对外互动

**含义**：以 subject 的名义与外部互动。Represent 的结果直接绑定 principal（被代表者）。

**行使者**：持有对某群体/个人的 `Represent` 权力的 holder。

**参数**：
```rust
struct RepresentParams {
    principal: EntityId,           // 被代表者
    binds_principal: bool,         // 代表的承诺是否约束 principal
    scope: RepresentScope,         // Diplomatic / Commercial / Military / Legal / Full
}
```

**效果**：
- `binds_principal = true`：代表签署的条约/合同 → 为 principal 创建 Duty 和权力边
- `binds_principal = false`：代表只是传达、交涉，不产生约束力的结果

**与 Delegate 的区别**：
- `Delegate`：B 以**自己的名义**行使 A 的权力
- `Represent`：B 以 **A 的名义**行事，结果绑定 A

---

### 4.4 规范原子

#### 4.4.1 PrescribeRule — 制定通用规范

**含义**：制定适用于特定 subject 群体的**通用行为规范**。这是"法律"的基础原子。

**行使者**：持有对目标群体的 `PrescribeRule` 权力的 holder。

**参数**：
```rust
struct PrescribeRuleParams {
    scope: RuleScope,               // 规则适用的 subject 群体
    content: RuleContent,           // 行为规范的具体内容
    sanction_on_violation: SanctionSpec,  // 违反后触发的制裁
    priority: RulePriority,         // 与已有规则冲突时的优先级
    valid_until: Option<GameTimestamp>,
}
```

**效果**：
- 创建一条 `PrescribeRule` 边 → 为 scope 内的每个 subject 生成对应的 Duty
- 规则自动适用——subject 不需要被逐个"通知"（虽然语言表达系统会传播）
- 违反规则 → Duty breach → Adjudicate 或 Sanction（取决于执行者）
- 多个 holder 的 PrescribeRule 冲突时 → 规范层级决定优先级（见 [[005-义务、免疫与规范层级]]）

**法律涌现**：`PrescribeRule` + `Sanction` + `Adjudicate` + `Delegate` + legitimacy → 法律体系。法律不是预设的——它从规则的制定、执行、被认可中涌现。

---

#### 4.4.2 Derogate — 豁免规则适用

**含义**：使特定 subject **不受**某条 `PrescribeRule` 的约束——临时或永久豁免。

**行使者**：持有对目标规则的 `Derogate` 权力的 holder（通常是规则的制定者或更高级 holder）。

**参数**：
```rust
struct DerogateParams {
    rule_edge_id: EdgeId,          // 被豁免的规则边
    beneficiary: EntityId,         // 受益者（谁被豁免）
    scope: DerogateScope,          // FullExemption / PartialExemption(sections)
    valid_until: Option<GameTimestamp>,  // 临时豁免的到期时间
    reason: Option<String>,
}
```

**效果**：
- 为 beneficiary 创建一条覆盖对应规则 Duty 的豁免
- beneficiary 违反被豁免的规则 → 不触发 Sanction
- 撤销 Derogate → 豁免失效，规则恢复适用

**区别**：
- `Derogate`：holder 主动豁免 subject
- `Immunity`：subject 自带的豁免属性（可能来自身份、文化、魔法等）
- `Remit`：对**已判惩罚**的豁免（已经 Adjudicate 并 Sanction 之后的赦免）
- `Pardon`（未独立为原子）：是 Remit 的特例——赦免特定人的已判罪罚

---

### 4.5 裁决原子

#### 4.5.1 Adjudicate — 有约束力的裁决

**含义**：对涉及 subject 的争议做出终局性判定。

**行使者**：持有对 subject 的 `Adjudicate` 权力的 holder（可能通过 Delegate 链获得）。

**参数**：
```rust
struct AdjudicateParams {
    dispute: Dispute,              // 争议的内容和涉及方
    evidence: Vec<EvidenceItem>,   // 各方提交的证据
    ruling: Ruling,                // 裁决结果
    based_on: Option<Vec<EdgeId>>, // 裁决依据的法律/规则边
}
```

**效果**：
- 裁决结果对 dispute 的双方有约束力 → 为败诉方创建 Duty（如赔偿）
- 裁决可能触发 `Sanction`（如败诉方被罚款/监禁）
- Adjudicate 权威来自 holder 对所有方的 `Adjudicate` 权力。如果某方不在 holder 的 subject 范围内 → `PowerExerciseError::NotSubject`

---

#### 4.5.2 Sanction — 施加奖励或惩罚

**含义**：对 subject 施加后果——正向（奖励）或负向（惩罚）。

**行使者**：持有对 subject 的 `Sanction` 权力的 holder。

**参数**：
```rust
struct SanctionParams {
    sanction_type: SanctionType,   // Reward / Punishment
    method: SanctionMethod,        // Fine / Imprisonment / Corporal / Exile / Death / Grant / Honor
    magnitude: f32,               // 0-1 强度
    reason: String,               // 原因（叙事用 + legitimacy 记录）
}
```

**效果**：
- `Punishment`：`Fine` → Extract(铜币) / `Imprisonment` → Constrain(人身自由) / `Corporal` → 部位伤害 / `Exile` → Constrain(领土排除) + DefineMembership(expel) / `Death` → 生命系统执行
- `Reward`：`Grant` → Extract(反向——holder 给 subject 资源) / `Honor` → ConferRank(提升)
- 惩罚的 severity 受 legality 和文化规范约束——过度惩罚可能降低 holder 的 legitimacy

---

#### 4.5.3 Remit — 豁免已判惩罚

**含义**：豁免或撤销 subject 已经受到的惩罚——不是豁免规则（Derogate），而是对**已经执行的 Sanction 后果**的撤销或停止。

**行使者**：持有对 subject 的 `Remit` 权力的 holder。

**参数**：
```rust
struct RemitParams {
    sanction_edge_id: EdgeId,      // 要豁免的 Sanction 边
    scope: RemitScope,             // FullPardon / Commute(减刑) / Suspension(暂停)
    reason: Option<String>,
}
```

**效果**：
- `FullPardon`：完全撤销 Sanction 的效果（释放囚犯、撤销罚款记录）
- `Commute`：降低惩罚强度（死刑改终身监禁）
- `Suspension`：暂停执行（保留恢复的可能）

## 五、原子行使通用流程

```rust
/// 行使任意权力原子的通用入口
fn exercise_power(
    topology: &mut PowerTopology,
    holder: EntityId,
    subject: EntityId,
    atom_type: AtomTypeTag,
    params: AtomParams,
) -> Result<(PowerExerciseResult, Vec<PowerEvent>), PowerExerciseError> {
    // 1. 权限检查——holder 是否持有对 subject 的该原子边？
    let edge_id = topology.find_edge(holder, subject, atom_type, &params.domain())?;
    let edge = topology.get_edge(edge_id);
    
    // 2. 活跃性检查——边是否 active？
    if !edge.active {
        return Err(PowerExerciseError::EdgeInactive);
    }
    
    // 3. 时效检查——边是否过期？
    if let Some(expiry) = edge.valid_until {
        if current_time() > expiry {
            return Err(PowerExerciseError::EdgeExpired);
        }
    }
    
    // 4. 免疫检查——subject 是否对此原子的此 domain 有免疫？
    if topology.has_immunity(subject, atom_type, &params.domain()) {
        return Err(PowerExerciseError::SubjectHasImmunity);
    }
    
    // 5. 执法能力检查——enforcement 是否足够？
    if edge.enforcement < MIN_ENFORCEMENT_THRESHOLD {
        return Err(PowerExerciseError::InsufficientEnforcement);
    }
    
    // 6. 利益冲突检查——holder 是否在自己裁决自己？
    let conflict = detect_self_dealing(&edge, atom_type, &params);

    // 7. 执行原子效果
    let result = dispatch_atom(topology, edge_id, atom_type, &params)?;
    
    // 8. 更新 last_exercised_at
    topology.set_last_exercised(edge_id, current_time());
    
    // 9. 收集副作用事件（不发射——返回给调度层）
    let events = collect_power_events(&edge, atom_type, &result, conflict);
    
    Ok((result, events))
}
```

### 错误类型

```rust
#[derive(Debug)]
pub enum PowerExerciseError {
    /// 无此权力边
    NotAuthorized,
    /// 边已被软删除
    EdgeInactive,
    /// 边已过期
    EdgeExpired,
    /// subject 对此原子的此 domain 有免疫
    SubjectHasImmunity,
    /// 执法能力不足
    InsufficientEnforcement,
    /// 循环委托
    CircularDelegation,
    /// subject 不在 holder 的管辖范围内
    NotSubject,
    /// 原子所需的资源不足（如 Extract 余额不足）
    InsufficientResources,
    /// 行为被已有 Constrain 边禁止
    BehaviorConstrained,
    /// Access 被拒绝
    AccessDenied,
    /// 一般性失败
    GeneralFailure(String),
}
```

## 六、原子组合：从原子到角色

任何社会角色都可以表示为 17 个原子的特定子集。这不是硬编码映射——而是权力拓扑边的自然组合。

```
"父亲"     = [Constrain(child_behavior) + Compel(chores) + Access(home)]
"行会会长"  = [DefineMembership(guild_members) + Constrain(quality_standards) 
               + Extract(membership_fee) + Represent(guild_external) + PrescribeRule(guild_regulations)]
"领主"     = [Constrain(domain=Territory) + Extract(tax) + Compel(militia_service)
               + Adjudicate(local_disputes) + Constrain(exit_territory)]
"国王"     = 上面组合的最大集合（通常是 12-15 个原子，取决于政体）
"议会"     = Group 作为 holder，通过内部选举获得 [PrescribeRule + Extract(budget) 
               + Delegate(executive_powers) + Constrain(monarch_power)]（如果是君主立宪）
```

同一个 NPC 可以同时是多个角色——作为"父亲"对其孩子有 Constrain 边，作为"领主"对领民有 Extract 边，作为"国王的臣子"对国王有入边（subject）。**权力拓扑图同时容纳所有这些尺度。**
