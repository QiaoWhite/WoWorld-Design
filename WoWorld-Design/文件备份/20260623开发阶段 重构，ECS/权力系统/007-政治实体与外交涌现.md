## 一、核心洞察

Polity（政治实体）不是被"创建"的对象——它是从权力拓扑图的稳定连通分量的**涌现标签**。定义 Polity 的不是"有人称王"——而是权力关系在某一区域、某一群体上形成了**稳定的、被承认的、持续的**结构。

## 二、Polity 涌现条件

### 2.1 四个必要条件

```rust
/// 检查一个连通分量是否满足 Polity 涌现条件
fn qualifies_as_polity(
    component: &ConnectedComponent,
    topology: &PowerTopology,
) -> bool {
    // === 条件 1：领土连续性 ===
    // 存在至少一个 ClaimTerritory 边，且这些边的 territory 形成连通块
    let territory_edges: Vec<&PowerEdge> = component.edges.iter()
        .filter(|e| matches!(e.domain, PowerDomain::Territory { .. }))
        .collect();
    let has_contiguous_territory = territory_edges.len() >= 1
        && are_territories_contiguous(&territory_edges);
    
    // === 条件 2：统一的权威结构 ===
    // 所有 ClaimTerritory 边指向同一个 holder，或 holder 之间通过 Delegate 链连接
    let authorities: HashSet<EntityId> = territory_edges.iter()
        .map(|e| e.holder)
        .collect();
    let has_unified_authority = authorities.len() == 1
        || are_connected_by_delegation(&authorities, topology);
    
    // === 条件 3：合法性阈值 ===
    // 连通分量内所有权力边的平均 legitimacy > 阈值
    let avg_legitimacy: f32 = component.edges.iter()
        .map(|e| e.legitimacy)
        .sum::<f32>() / component.edges.len() as f32;
    let meets_legitimacy_threshold = avg_legitimacy >= POLITY_LEGITIMACY_THRESHOLD;  // [TUNING] 0.30
    
    // === 条件 4：持续性 ===
    // 此连通分量已存在超过一定时间
    let meets_duration = component.age() >= POLITY_MIN_DURATION;  // [TUNING] 1 游戏年 (365天)
    
    has_contiguous_territory && has_unified_authority && meets_legitimacy_threshold && meets_duration
}
```

### 2.3 涌现算法

```rust
/// 每游戏年执行一次——从拓扑图中检测 Polity
fn emerge_polities(topology: &PowerTopology) -> Vec<Polity> {
    // 1. 提取所有活跃的 ClaimTerritory 边及其 holder
    let territory_edges = topology.edges_of_type(AtomTypeTag::Constrain)
        .filter(|e| matches!(e.domain, PowerDomain::Territory { .. }) && e.active);
    
    // 2. 按领土连接性 + holder 委托关系 → 连通分量
    let components = compute_connected_components(territory_edges, topology);
    
    // 3. 对每个连通分量，检查 4 个涌现条件
    let mut polities = Vec::new();
    for component in components {
        if qualifies_as_polity(&component, topology) {
            let gov_form = infer_government_form(&component, topology);
            let name = generate_polity_name(&component, gov_form, topology);
            let bounds = compute_territorial_bounds(&component);
            
            polities.push(Polity {
                polity_id: PolityId::new(),
                constituent_edges: component.edge_ids().to_vec(),
                average_legitimacy: component.average_legitimacy(),
                recognized_at: current_time(),
                last_recomputed: current_time(),
                government_form: gov_form,
                territorial_bounds: bounds,
                name,
                authority_hierarchy: AuthorityHierarchy::from_component(&component, topology),
            });
        }
    }
    
    // 4. 与上次重算结果比较——检测 Polity 变化
    let changes = diff_polities(&topology.polities, &polities);
    for change in changes {
        topology.emit_event(PowerEvent::PolityChange(change));
    }
    
    polities
}
```

## 三、政府形式推断

从权力拓扑图的边模式中推断政府形式——不预设枚举。

```rust
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub enum GovernmentForm {
    /// 绝对君主制：单一 holder 持有全套原子，没有对他有效的 Check-and-Balance 边
    AbsoluteMonarchy { monarch: EntityId },
    /// 立宪君主制：monarch + 议会的 Constrain 边限制 monarch 的权力
    ConstitutionalMonarchy { monarch: EntityId, parliament: EntityId },
    /// 寡头制：少数几个 holder 共同持有权力的核心集合，互相无约束
    Oligarchy { council: Vec<EntityId> },
    /// 民主共和制：holder 通过 Elected source 获得权力，有固定任期
    DemocraticRepublic { head_of_state: EntityId, legislature: EntityId },
    /// 神权制：holder 的 source = Divine，PrescribeRule 引用宗教文本
    Theocracy { religious_leader: EntityId, deity: EntityId },
    /// 军事独裁：holder 的 source = Conquered，权力以 Constrain + Compel 为主
    MilitaryDictatorship { dictator: EntityId },
    /// 部落联盟：多个 group 通过 Contract 边联合，领土不连续但松散关联
    TribalConfederation { tribes: Vec<EntityId> },
    /// 城邦：单一聚落 = 单一 Polity，领土紧密
    CityState { ruling_body: EntityId },
    /// 无政府/自治：不存在满足 Polity 条件的连通分量
    Stateless,
}

fn infer_government_form(
    component: &ConnectedComponent,
    topology: &PowerTopology,
) -> GovernmentForm {
    let holders = component.holders();
    let edges = component.edges();
    
    // 单一 holder → 检查哪种单一制
    if holders.len() == 1 {
        let holder = holders[0];
        
        // 检查是否有 Elected 来源
        if edges.iter().any(|e| matches!(e.source, PowerSource::Elected { .. })) {
            return GovernmentForm::DemocraticRepublic {
                head_of_state: holder,
                legislature: component.legislature().unwrap_or(holder),
            };
        }
        
        // 检查是否有 Divine 来源
        if edges.iter().any(|e| matches!(e.source, PowerSource::Divine { .. })) {
            return GovernmentForm::Theocracy {
                religious_leader: holder,
                deity: component.deity().unwrap_or(EntityId::Npc(0)),
            };
        }
        
        // 检查是否有 Conquered 来源
        if edges.iter().any(|e| matches!(e.source, PowerSource::Conquered { .. })) {
            return GovernmentForm::MilitaryDictatorship { dictator: holder };
        }
        
        // 默认：绝对君主制
        return GovernmentForm::AbsoluteMonarchy { monarch: holder };
    }
    
    // 两个 holder → 检查是否立宪君主制（一个 monarch 被另一个 parliament Constrain 边制约）
    if holders.len() == 2 {
        let (monarch, parliament) = if has_constraint_edge(holders[0], holders[1], topology) {
            (holders[0], holders[1])
        } else if has_constraint_edge(holders[1], holders[0], topology) {
            (holders[1], holders[0])
        } else {
            (EntityId::Npc(0), EntityId::Npc(0))  // 不会被使用
        };
        if monarch != EntityId::Npc(0) {
            // 检查 parliament 对 monarch 是否有 Constrain 边（Check-and-Balance 特征）
            return GovernmentForm::ConstitutionalMonarchy { monarch, parliament };
        }
    }
    
    // 多个 holder → 寡头制或联盟
    if holders.len() <= 5 {
        // 检查是否有互相的 Constrain 边（Check-and-Balance）
        if has_mutual_constraints(component, topology) {
            return GovernmentForm::Oligarchy { council: holders.to_vec() };
        }
    }
    
    // 多个 holder + 多个 group → 部落联盟
    let groups = component.groups();
    if groups.len() >= 3 && component.territory().is_fragmented() {
        return GovernmentForm::TribalConfederation { tribes: groups.to_vec() };
    }
    
    // 单一聚落
    if component.settlements().len() == 1 {
        return GovernmentForm::CityState { ruling_body: holders[0] };
    }
    
    // 无法推断 → 寡头制的广义形式
    GovernmentForm::Oligarchy { council: holders.to_vec() }
}
```

## 四、Polity 的惰性反馈

Polity 不锁定其内部权力边。但提供弱惯性反馈：

```rust
/// Polity 对内部边的唯一反馈：legitimacy 惯性加成
fn polity_inertia_bonus(edge: &PowerEdge, polity: Option<&Polity>) -> f32 {
    match polity {
        Some(p) if p.constituent_edges.contains(&edge.edge_id) => {
            // 每年 +0.03，上限 +0.15
            (p.age_years() * 0.03).min(0.15)
        }
        _ => 0.0,
    }
}
```

**Polity 变化检测**：每次重算后，与上次 Polity 列表比较：
- **PolityBirthed**：上次不存在，本次涌现 → 历史系统记录"X 王国成立"
- **PolityDied**：上次存在，本次不再满足条件 → "X 王国灭亡"
- **PolitySplit**：上次一个 Polity，本次变成两个（内战/分裂）→ 历史事件
- **PolityMerged**：上次两个，本次合并（征服/联姻合并）
- **PolityEvolved**：政府形式变化（君主制→共和制）→ 历史事件

pub struct DiplomaticRelation {
    pub polity_a: PolityId,
    pub polity_b: PolityId,
    /// 连续关系分数：-1.0 ～ +1.0
    pub score: f32,
    /// 从 score 阈值派生的离散状态
    pub state: DiplomaticState,
    /// 计算时间
    pub computed_at: GameTimestamp,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
pub enum DiplomaticState {
    /// 同盟——score > 0.6。有 mutual defense 义务
    Allied,
    /// 友好——0.2 < score ≤ 0.6
    Friendly,
    /// 中立—— -0.2 ≤ score ≤ 0.2（默认状态）
    Neutral,
    /// 冷淡—— -0.5 ≤ score < -0.2
    Cold,
    /// 敌对—— -0.8 ≤ score < -0.5
    Hostile,
    /// 战争——score < -0.8，或有显式宣战
    War,
}

### 5.2 关系分数计算

```rust
fn compute_relation_score(
    a: PolityId,
    b: PolityId,
    topology: &PowerTopology,
    economy: &dyn EconomyQuery,
    culture: &dyn CultureQuery,
    history: &dyn HistoryQuery,
) -> f32 {
    // === 因子 1：活跃契约 [权重 0.25，范围 0~+0.25] ===
    let treaty_bonus = active_contracts_between(a, b, topology)
        .map(|c| c.strength * 0.25)
        .sum::<f32>().clamp(0.0, 0.25);
    
    // === 因子 2：领土争议 [权重 0.20，范围 -0.20~0] ===
    let territorial_tension = overlapping_claims(a, b, topology)
        .map(|o| -o.severity * 0.20)
        .unwrap_or(0.0);
    
    // === 因子 3：贸易依存 [权重 0.15，范围 0~+0.15] ===
    let trade_interdependence = economy.trade_volume_between(a, b)
        / economy.total_trade_volume(a).max(1.0)
        * 0.15;
    
    // === 因子 4：近期冲突 [权重 0.20，范围 -0.20~0] ===
    let recent_conflict = history.recent_events_between(a, b, 365)  // 1 游戏年
        .filter(|e| e.is_hostile())
        .map(|e| -e.severity * 0.04)
        .sum::<f32>().clamp(-0.20, 0.0);
    
    // === 因子 5：文化亲和 [权重 0.10，范围 -0.10~+0.10] ===
    let cultural_affinity = (1.0 - culture.distance(a, b)) * 0.20 - 0.10;
    
    // === 因子 6：历史深度 [权重 0.10，范围 -0.05~+0.10] ===
    let historical_depth = compute_history_depth(a, b, history) * 0.10;
    // compute_history_depth 内部已 clamp 到 [-0.05, 0.10]
    
    treaty_bonus + territorial_tension + trade_interdependence
        + recent_conflict + cultural_affinity + historical_depth
}
```

### 5.3 War 状态——唯一有硬效果的外交状态

```rust
/// War 状态的特殊效果
fn apply_war_effects(
    topology: &mut PowerTopology,
    war: &DiplomaticRelation,
) -> Vec<PowerEvent> {
    let mut events = Vec::new();
    
    // 效果 1：对敌对 Polity 成员的 Constrain/Extract/Sanction 不再需要合法性
    for member_a in topology.members_of_polity(war.polity_a) {
        for member_b in topology.members_of_polity(war.polity_b) {
            // 自动授予临时战争权力边
            // member_a 对 member_b：Constrain(人身自由) + Extract(战利品)
            let edge_ab = topology.create_war_power(member_a, member_b);
            events.push(PowerEvent::WarPowerGranted { edge: edge_ab });
        }
    }
    
    // 效果 2：敌对 Polity 成员的 Immunity 被撤销
    for member in topology.members_of_polity(war.polity_b) {
        topology.revoke_immunity_for_war(member, war.polity_a);
    }
    
    // 效果 3：Conscript 的门槛降低
    topology.set_conscription_threshold(war.polity_a, CONSCRIPTION_WAR_THRESHOLD);
    topology.set_conscription_threshold(war.polity_b, CONSCRIPTION_WAR_THRESHOLD);
    
    // 效果 4：贸易冻结——所有经济契约边被暂停
    let trade_edges = topology.find_trade_edges_between(war.polity_a, war.polity_b);
    for edge in trade_edges {
        topology.suspend_edge(edge);
        events.push(PowerEvent::TradeFrozen { edge });
    }
    
    events
}
```

### 5.4 和平

War → 和平通过 Contract（和平条约）实现：
- 双方各行使 `Contract` 原子，约定：停止敌对行为 + 可能割让领土（`ClaimTerritory` 边的 holder 变更）+ 可能赔偿（`Extract` obligation）
- 和平条约 = 一个 `Contract`，其中 `obligations` 包含：A 对 B 的 Constrain(not attack) + B 对 A 的 Constrain(not attack)
- 如果和平条约被违反 → 回到 War（由 Contract 违约处理逻辑驱动）

### compute_history_depth

```rust
/// 计算两个 Polity 之间的历史深度——影响外交关系分数
/// 正值 = 有长期和平/合作历史，负值 = 长期敌对历史
fn compute_history_depth(
    a: PolityId,
    b: PolityId,
    history: &dyn HistoryQuery,
) -> f32 {
    // 统计两个 polity 之间的历史事件
    let events = history.events_between(a, b, HISTORY_DEPTH_WINDOW);  // [TUNING] 10 游戏年
    if events.is_empty() { return 0.0; }
    
    let total: f32 = events.iter()
        .map(|e| match e.nature {
            EventNature::Hostile => -0.02,
            EventNature::Friendly => 0.02,
            _ => 0.0,
        })
        .sum();
    
    // 加权（近期事件权重高）+ clamp 到 [-0.05, 0.10]
    (total * 0.10).clamp(-0.05, 0.10)
}

/// 检测两个 holder 之间是否存在 Constrain 边（用于立宪君主制推断）
fn has_constraint_edge(holder_a: EntityId, holder_b: EntityId, topology: &PowerTopology) -> bool {
    topology.powers_of(holder_a).iter()
        .any(|e| e.atom_type == AtomTypeTag::Constrain && e.subject == holder_b)
}
```
