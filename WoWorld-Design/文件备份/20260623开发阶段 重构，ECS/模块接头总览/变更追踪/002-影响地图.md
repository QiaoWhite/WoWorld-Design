# 模块变更影响地图

> **最后验证**: 2026-06-20 | **定位**: 跨模块依赖关系概览——任一模块变更时快速识别需检查的下游模块
>
> 当修改某个模块的接口出口（trait 签名、数据合同、类型定义）时，必须先查阅本影响地图，对所有消费方进行兼容性检查。

---

## 消费方快速反查

| 如果修改了... | 必须检查... |
|-------------|-----------|
| woworld_core trait 签名 | 所有实现方 + 所有消费方（查 003-通用trait索引） |
| NpcData 字段 | NPC(02)·世界生成(03)·经济(14)·概念地基(24)·历史(12) |
| Vitals/DeathCause | NPC(02)·战斗(06)·经济(14)·历史(12)·感官(05)·世界生成(03) |
| 世界生成管线阶段 | 该阶段的产出消费方（查 03-世界生成/001-接口出口） |
| 性能预算 | 22-性能优化 + 受影响模块的性能文档 |
| 物理架构 | 17-模型物理 + 所有 TerrainQuery/EntityIndex 消费方 |
| LOD 架构 | 全模块——LODCoordinator 聚合 7 维输入 |
