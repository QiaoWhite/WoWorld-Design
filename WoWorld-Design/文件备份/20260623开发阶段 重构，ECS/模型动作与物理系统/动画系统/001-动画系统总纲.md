## 〇、核心设计原则

1. **模块化骨架**——Lower/Upper/Head/Face-Hands 四个独立动画模块。下半身和上半身各自独立驱动。脊柱作为髋↔肩的约束桥接。
2. **Keyframe 只做必需的手工姿态**——人体微妙的劳作/社交/施法姿态。一切周期运动（步态、游泳、飞行）和约束求解（IK、轨迹跟随）都是程序化的。
3. **涌现式动画**——步态风格 9 参数从大五人格×情绪×疲劳度纯数学派生。战斗流派从 CombatStyleParams 8 参数涌现。零手写"文化变体"动画。
4. **任何动画可在任何帧被中断**——受击、死亡、强制位移在任何姿态中立即生效。中断优先级系统（0-4）。

## 二、9 层动画栈

每层是独立的 `AnimationLayer` trait 实现。按优先级从低到高叠加：

```rust
pub trait AnimationLayer: Send + Sync {
    fn apply(&self, local: &mut [Mat4; 50], skeleton: &SkeletonDef,
             state: &AnimationBodyState, dt: f32);
    fn priority(&self) -> LayerPriority;
}
```

| 层 | 名称 | 做什么 | 并 |
|----|------|--------|----|
| **Layer 0** | 基础姿态 | 关键帧混合（Daily mode）或战斗轨迹求解（Combat mode）。Layer 0 之上全部是叠加 | 100% NPC |
| **Layer 1** | 情绪偏移 | 13 参数从 EmotionState 映射→骨骼旋转。身体(7参)+面部(6参)。零 keyframe | 100% |
| **Layer 2** | 过渡混合 | from→to slerp，接触锁定，关节指数收敛。仅过渡期运行 | ~15% NPC |
| **Layer 3** | 根运动 | 攻击/施法的重心位移提取→叠加步态位移 | 战斗 10% |
| **Layer 4** | 注视 | 头骨→注意目标。眼球 L0 旋转或 L1 UV 偏移 | 100% |
| **Layer 5** | 脚部 IK | 着地脚锁定世界空间→踝/膝/髋 IK 链 | 着地中 ~90% |
| **Layer 6** | 武器手 IK | 基元轨迹跟随→手部精确定位到握持点 | 战斗/劳作 |
| **Layer 7** | 脊柱桥接 | 髋/肩角度差→查 SpineTable(80条目,10°步长)→分配各节脊椎 | 100% |
| **Layer 8** | 中断恢复 | 受击/踉跄/死亡。最高优先级覆盖 | <1% |

### Layer 2 过渡——五项自然法则

1. **接触锁定**：过渡期间着地脚世界位置锁定——不 blend，就是不动
2. **指数收敛**：每关节独立收敛速度。肩=14（快），髋=5（慢），踝=4（最慢），指=20（飞快）
3. **相位匹配**：循环→循环先在 to_cycle 中搜索和当前脚状态的最近相位→从该相位开始 blend
4. **动量继承**：重心速度二次衰减 `carried_vel × (1-t)²`
5. **Ease 曲线**：所有 blend 走 ease-in-out——永不线性

```toml
# TOML 过渡定义示例
[[transitions]]
from = "walk"
to = "stand_idle"
duration_ms = 350
ease = "ease_out_exp"

[[transitions]]

[[transitions]]
```

## 三、Keyframe 系统——38 个模块姿态

每个姿态定义**一个** AnimationModule 中所有骨骼的 local transforms。

| 模块 | 姿态 ID | 数量 | 举例 |
|------|--------|------|------|
| **Lower（静态）** | StandIdle, CrouchIdle, SitGround, SitChair, Kneel, LieSide | 6 | |
| **Upper（劳作）** | SwingOverhead, SwingHorizontal, PushPull, DigLow, WorkBench, CarryHeavy | 6 | 打铁→U01, 锯木→U03 |
| **Upper（交互）** | EatDrink, PickUpLow, OpenPush, HoldPresent | 4 | |
| **Upper（社交）** | WaveHand, PointArm, ArmsOpen, ArmsCross, HandsOnHips, ClaspHands | 6 | |
| **Upper（表演）** | PlayInstrument | 1 | 吟游诗人 |
| **Upper（武器）** | DrawWeapon, SheathWeapon | 2 | 拔剑/收剑 |
| **Combat** | CombatGuard | 1 | 战斗→日常的桥 |
| **Spell** | CastProject, CastChannel, CastRitual | 3 | 投射/引导/仪式 |
| **非人形** | QuadHitFront, QuadHitBack, QuadDeath, AvianTakeoff, AvianLanding | 5 | Phase 3+ |
| **总计** | | **38** | ~30KB 数据 |

**数据格式**：
```rust
pub struct ModulePose {
    pub id: ModulePoseId,
    pub module: AnimationModule,  // 属于哪个模块
    pub bone_transforms: Vec<BoneLocalTransform>,  // 仅此模块的骨
    pub sound_markers: Vec<AnimationSoundMarker>,  // 音效标记
}

pub struct BoneLocalTransform {
    pub translation: Vec3,   // 通常 ≈ 0（除 root bones）
    pub rotation: Quat,      // 相对父骨
}
// 每姿态约 800 字节。38 × 800B ≈ 30KB
```

## 四、战斗轨迹系统——15 条基元

10 个武器基元：刺 斩 撩 劈 扫 点 崩 缠 引 化
5 个空手基元：点 崩 缠 引 化（无斩/劈/扫——空手不可用）

```rust
pub struct PrimitiveStrike {
    pub primitive: ActionPrimitive,
    pub trajectory: TrajectoryCurve,         // 武器手在空间中的路径
    pub key_bone_frames: Vec<StrikeKeyFrame>, // 打击段的骨骼快照（3-5 帧）
    pub duration_ms: (f32, f32),             // （min, max），修饰符调整
    pub residual_energy: f32,                // 0-1，打击后剩余能量
    pub body_commitment: f32,                // 0=手腕, 1=全身突刺
}

pub enum TrajectoryCurve {
    Linear { direction: Vec3, length: f32 },                      // 刺/点
    Arc { plane_normal: Vec3, radius: f32, sweep_angle_deg: f32 }, // 斩/劈/扫/撩
    LinearThenArc { linear: LinearSeg, arc: ArcSeg },              // 佯攻
    Spline { control_points: Vec<Vec3> },                           // 缠/引/化
}
// 15 条 ≈ 9KB 数据
```

### 武器适配——程序化

```rust
fn adapt_for_weapon(base: &PrimitiveStrike, weapon: &WeaponPhysicalParams) -> AdaptedTrajectory {
    AdaptedTrajectory {
        reach_scale: weapon.length / BASE_WEAPON_LENGTH,
        speed_scale: (BASE_WEAPON_WEIGHT / weapon.weight).sqrt(),
        body_commitment: base.body_commitment * (weapon.weight / BASE_WEAPON_WEIGHT),
        grip_offset: weapon.grip_position,
    }
}
```

### 基元链——串联求解

```rust
fn chain_primitives(seq: &[ActionPrimitive], weapon: &WeaponPhysicalParams, style: &CombatStyleParams) -> ChainedMove {
    let mut carry_momentum = 0.0;
    for (i, prim) in seq.iter().enumerate() {
        let traj = get_trajectory(*prim);
        let adapted = adapt_for_weapon(&traj, weapon);
        // 动量继承：前基元余能→后基元前摇缩短或跳过
        let start = if carry_momentum > 0.3 && i > 0 { current_hand_pos } else { style.guard_position() };
        let segment = adapted.solve(start, carry_momentum);
        carry_momentum = traj.residual_energy * adapted.speed_scale;
        // 力学约束：重慢弧线→短快直线被战斗系统拒绝——动画系统永不见非法序列
    }
}
```

```rust
pub struct GaitStyle {
    pub hip_sway: f32,            // 0.5=僵硬, 1.0=正常, 2.0=风骚
    pub stride_length: f32,       // 0.6=碎步, 1.0=正常, 1.4=大步
    pub arm_swing: f32,           // 0.3=垂手, 1.0=正常, 1.6=甩臂
    pub bounce: f32,              // 0.4=滑行, 1.0=正常, 1.5=弹跳
    pub forward_lean_deg: f32,    // 负=后仰(傲), 正=前倾(急/老)

    pub foot_drag: f32,           // 0=利落, 1.0=擦地

impl GaitStyle {
    pub fn from_npc(p: &BigFive, e: &EmotionState, fatigue: f32) -> Self {
        Self {
            hip_sway: (1.0 + p.extraversion*0.6 + p.openness*0.4 - p.conscientiousness*0.3).clamp(0.3, 2.5),
            stride_length: (1.0 + p.extraversion*0.3 - e.fear*0.4 - fatigue*0.3).clamp(0.5, 1.5),
            // ... 其余 7 个参数同理
        }
    }
```

## 六、中断系统

```rust
pub enum InterruptType {
    Voluntary,                                  // 优先级 0
    Whiffed,                                    // 1
    Dodge { direction: Vec3 },                  // 2
    Deflected { deflect_dir: Vec3 },            // 2
    HitReaction { from: Vec3 },                 // 3
    ForcedDisplacement { displacement: Vec3 },  // 3
    Death { cause: DeathCause },               // 4
}
```

- **HitReaction**：力的方向决定脊椎后仰角度。下肢尽力站稳。重心出支撑域→踉跄。武器手放松。
- **Death**：骨架松弛——肌肉刚度 1.0→0.02 指数衰减。关节限制强制。完全松弛后→COM 物理。成本 ~0.03ms/NPC（仅死亡 NPC）。
- **ForcedDisplacement**：重心位移。双脚离地→着地检测→三级响应（致命/重伤/硬着陆）。

## 十、animation_lod 五档退化契约 ★CHG-049

> **关联**: [[../../../../Change/CHG-049-LOD架构全面深化-20260620|CHG-049 §三]] · [[../../模块接头总览/00-全局基础设施/001-核心类型注册表|LodPrescription]]

animation_lod 是 `LodPrescription` 的 7 个维度之一，由 LODCoordinator 下发，动画系统按此值选择 9 层栈的计算子集。

### 10.1 退化预设

| animation_lod | 参与计算的层 | CPU 预算 | 对外契约 |
|---------------|------------|---------|---------|
| 0 | L0+L1+L2+L3+L4+L5+L6+L7+L8 (9层全栈) | ~0.5ms | 完整动画产出。行为类型/情感状态/社交意图/个体特征全可辨 |
| 1 | L0+L1+L3+L4(嘴眉)+L6+L7 (6核心层) | ~0.25ms | 简化动画。行为类型/情感/社交意图可辨。精细运动(手指/眼睑/微表情)不可用 |
| 2 | L0+L1+L3+L6 (4行为层) | ~0.10ms | 基础动画。姿态(站/坐/战斗/躺)+位移方向可辨。个体识别信息(步态/手势)不可靠 |
| 3 | L0+L6 (2层:姿态+战斗状态) | ~0.02ms | 仅静态姿态+战斗状态。无帧间动画变化。仅能判断"那方向上有一个站着的/战斗中的NPC" |
| 4 | 无 | 0 | 无动画产出。位置由AI统计模型维护 |

### 10.2 面部 L4 分层退化

面部表情（512² 图集, CHG-033）不在 animation_lod 退化中作为整层开关，而是独立降级：

| animation_lod | 面部精度 | 保留语义 | 丢失信息 |
|---------------|---------|---------|---------|
| 0 | 512² 全 (16嘴×16眉×8眼) | 全部 | — |
| 1 | 64² (8嘴×4眉, 无眼睑) | 在说话/沉默, 基本情绪 | 精细眼睑/微表情 |
| 2+ | 关闭 | — | 全部。200m 外嘴型角分辨率 <0.014°——低于人眼极限 (1'≈0.017°)。视觉不可辨 |

### 10.4 模块内部约束

```
animation_lod ≥ skeleton_lod
// 0骨无法执行任何动画层——此约束在动画模块内部 clamp
// 例: LODCoordinator 下发 animation=1, skeleton=3
//      → 模块内部 clamp 到 animation=max(1,3)=3 → 仅2层
```

### 10.5 过渡时序

| 方向 | 延迟 | 机制 |
|------|------|------|
| 升级 (靠近) | 200-500ms | 逐层 blend_weight 从 0 渐增至 1.0。层按优先级顺序开启: L0→L6→L1→L3→L7→L4→L2→L5→L8 |
| 降级 (远离) | 0ms | 直接关闭目标层。无可见 pop——距离已超出细节感知范围 |
