## 〇、核心决策——放弃 PhysicsServer3D 作为模拟侧物理依赖

WoWorld 需要的"物理"全部是空间查询，不需要刚体动力学求解器。场景枚举：

| 需求 | 本质 | 查询类型 |
|------|------|---------|
| 站在地形上 | 高度查询→设置 y | `TerrainQuery::height_at` |
| 不穿过其他实体 | 胶囊体重叠避让→迭代分离 | `EntityIndex::entities_in_aabb` |
| 箭/法术命中 | 射线投射 | `TerrainQuery::terrain_raycast` |
| NPC 看到 X | 视线检测 | `VisibilityQuery::line_of_sight` |
| 船浮在水上 | 水面高度采样 | `TerrainQuery::height_at`（水面） |
| 物品掉到地上 | 地面高度查询 | `TerrainQuery::height_at` |
| 被击飞飞行 | COM 抛物体 | `TerrainQuery::terrain_raycast` + 重力积分 |

**PhysicsServer3D 仅保留用于玩家 CharacterBody3D。** 玩家需要帧完美的手感、斜坡处理、`move_and_slide()`。Transvoxel 地形碰撞体只给玩家用——NPC 不需要碰撞体。

## 一、trait 定义（woworld_core::spatial）

### 1.1 TerrainQuery

```rust
/// 地形表面查询——纯函数。数据源=世界生成密度场。
/// 实现: world_gen crate
pub trait TerrainQuery: Send + Sync {
    /// 地表高度——对悬挑(overhang)的上面
    fn height_at(&self, x: f64, z: f64) -> f64;

    /// 表面法线
    fn normal_at(&self, pos: Vec3) -> Vec3;

    /// DDA 射线行进——在密度场上步进
    fn terrain_raycast(&self, origin: Vec3, dir: Vec3, max_dist: f32) -> Option<TerrainHit>;

    /// 密度+材质——两个值可从一个 voxel 读出
    fn density_at(&self, pos: Vec3) -> (f32, u8 /* material_id */);

    /// 可通行：脚下是固体、头顶没有悬岩
    fn is_walkable(&self, pos: Vec3) -> bool;

    /// 表面材质——用于脚部声音/粒子
    fn surface_material_at(&self, pos: Vec3) -> SurfaceMaterial;

    /// 介质——空气/水/岩浆/虚空
    fn medium_at(&self, pos: Vec3) -> Medium;

    /// 环境光（遮蔽+时间+天气）
    fn light_level_at(&self, pos: Vec3, time: GameInstant) -> f32;

    /// 地平线采样——用于远景自然奇观检测
    fn sample_horizon(&self, origin: Vec3, azimuth: f32) -> HorizonSample;
}

struct TerrainHit {
    point: Vec3,
    normal: Vec3,
    material_id: u8,
    distance: f32,
}

enum Medium {
    Air,
    Water { depth: f32, surface_y: f32 },
    Lava,
    Void,
}
```

### 1.2 EntityIndex

```rust
/// 实体空间索引——可变数据结构。每个有物理存在的实体注册于此。
/// 实现: woworld_spatial crate
pub trait EntityIndex: Send + Sync {
    fn register(&mut self, entity: SpatialEntity);
    fn unregister(&mut self, id: EntityId);
    fn update_transform(&mut self, id: EntityId, transform: Mat4);

    /// AABB 范围内所有匹配过滤器的实体
    fn entities_in_aabb(&self, aabb: Aabb, filter: EntityFilter) -> &[SpatialEntity];
    fn entity_aabb(&self, id: EntityId) -> Option<Aabb>;

    /// 空间位置的声学属性——室内/室外/洞穴/走廊
    fn acoustic_tag_at(&self, pos: Vec3) -> AcousticTag;
}

struct EntityFilter {
    layer_mask: u32,       // Terrain | Npc | Item | Vehicle | Effect | ...
    include_dead: bool,    // 是否包含尸体
    include_sleeping: bool,
}
```

### 1.3 SpatialEventBus

```rust
/// 世界事件黑板——ephemeral append-only 缓冲，自动过期。
/// 实现: woworld_spatial crate
pub trait SpatialEventBus: Send + Sync {
    /// 查询范围内的近期事件
    fn recent_events_in(&self, aabb: Aabb, since: GameInstant) -> &[SpatialEvent];

enum SpatialEventType {
```

### 1.4 VisibilityQuery

```rust
/// 视线检测——跨 terrain + entity 的 DDA march。
/// 需要同时访问密度场和实体 AABB——两者来自不同数据源。
/// 实现: woworld_spatial crate（持有 Arc<dyn TerrainQuery> + &dyn EntityIndex）
pub trait VisibilityQuery: Send + Sync {
    /// from 到 to 之间有没有不透明障碍？
    fn line_of_sight(&self, from: Vec3, to: Vec3, max_dist: f32) -> bool;

    /// 返回命中点——用于弹道模拟
    fn line_of_sight_hit(&self, from: Vec3, to: Vec3, max_dist: f32) -> Option<VisibilityHit>;
}

struct VisibilityHit {
    point: Vec3,
    hit_type: VisibilityHitType,
}

enum VisibilityHitType {
    Terrain { material_id: u8 },
    Entity { entity_id: EntityId, body_part: Option<BodyPart> },
    WaterSurface,
}
```

## 二、物理飞行系统（COM 抛物体）

### 2.1 PhysicsBodyState

```rust
struct PhysicsBodyState {
    com: Vec3,              // 重心世界位置
    velocity: Vec3,         // 速度向量
    angular_velocity: Vec3, // 身体角速度（翻滚）
    initial_facing: Vec3,   // 离地时的面朝方向
    flight_time: f32,       // 已在空中多久
    airborne: bool,
}
```

### 2.2 每帧求解

```rust
fn tick_physics_body(
    state: &mut AnimationBodyState,
    terrain: &dyn TerrainQuery,
    dt: f32,
) {
    // 1. 重力
    state.com_velocity.y -= 9.81 * dt;

    // 2. 预测下一步位置
    let predicted = state.center_of_mass + state.com_velocity * dt;

    // 3. DDA 射线：COM → 预测 COM。有没有中途障碍？
    let ray_dir = state.com_velocity.normalize();
    let ray_len = state.com_velocity.length() * dt;

    if let Some(hit) = terrain.terrain_raycast(state.center_of_mass, ray_dir, ray_len) {
        // 中途截断——
        handle_collision(state, hit);
    } else {
        // 自由飞行
        state.center_of_mass = predicted;
    }

    // 4. 空中姿态：四肢张开指数松弛
    if state.balance == BalanceState::Airborne {
        state.limb_splay += (1.0 - state.limb_splay) * dt * 3.0;
        state.angular_velocity *= 0.999;
    }
}
```

### 2.3 碰撞处理——三种结果

| 碰撞法线 | 行为 | 公式 |
|---------|------|------|
| 法线朝上 >0.3 | 着地→三级响应 | 致命(>20m/s 等效)、重伤(>8m/s)、硬着陆(<8m/s) |
| 法线水平/朝下 | 撞壁→弹开 | `new_vel = recoil * 0.4`，沿崖壁滑落 |
| 着地——致命 | 触发 Death 中断→骨架松弛 | tick_relaxed_skeleton |
| 着地——重伤 | crumple→恢复 | interrupt_recovery，duration=impact_speed×0.15 |
| 着地——硬着陆 | 膝盖深弯吸收→手撑地→恢复 | interrupt_recovery，duration=impact_speed×0.08 |

### 2.4 骨架松弛死亡——布娃娃替代方案

```rust
fn tick_relaxed_skeleton(
    state: &mut AnimationBodyState,
    skeleton: &SkeletonDef,
    terrain: &dyn TerrainQuery,
    dt: f32,
) {
    let stiffness = (1.0 - state.death_progress * 2.0).max(0.02);

    for bone_idx in 1..skeleton.bone_count {
        if stiffness < 0.05 { break; }

        let natural_dir = (Vec3::NEG_Y + state.bone_velocities[bone_idx] * 0.1).normalize();
        let current_dir = state.bone_world_transforms[bone_idx].w_axis().truncate()
                        - state.bone_world_transforms[skeleton.parent(bone_idx)].w_axis().truncate();
        let target_dir = current_dir.lerp(natural_dir, dt * (1.0 - stiffness) * 5.0);

        // 关键——钳制到关节生理限制
        let clamped_dir = clamp_to_joint_limits(bone_idx, target_dir, current_dir, skeleton);
        // ... 更新骨骼 transform
    }

    state.death_progress = (state.death_progress + dt * 0.3).min(1.0);
}
```

**效果**: 死亡瞬间试图稳住→逐关节放松→完全瘫倒。关节不反折。成本 ~0.03ms/NPC（仅死亡 NPC）。

### 3.2 每帧求解——被动方 IK 到约束目标

```rust
fn solve_two_person_interaction(
    leader_state: &AnimationBodyState,
    follower_state: &mut AnimationBodyState,
    interaction: &TwoPersonInteraction,
    skeleton: &SkeletonDef,
    dt: f32,
) {
    for constraint in &interaction.relative_constraints {
        match constraint {
            Coincident { a, b, tolerance } => {
                let a_world = leader_state.anchor_world(a);
                let b_world = follower_state.anchor_world(b);
                if (a_world - b_world).length() > *tolerance {
                    ik_to_target(follower_state, skeleton, b.bone_index, a_world, dt);
                }
            }
            // ...
        }
    }
}
```

### 3.3 预定义模板

## 四、水下/骑乘模式

### 4.1 水下

`medium_at(pos) == Medium::Water` → 自动切换到水模式。浮力将重心推向水面，水阻使速度指数衰减，游泳步态替换陆地步态。死亡在水中=浮→慢沉→沉底。COM 物理解方程加了浮力项——零额外代码。

### 4.2 骑乘

骑手骨盆 world transform = 马鞍 world transform。骑手下肢 IK 锁定马镫。骑手上半身完全独立（可战斗/施法）。马受惊→重心速度突变→骑手和马鞍相对位置破坏→自动触发骑手 ForcedDisplacement 中断→COM 物理→摔下马。

## 五、性能

| 组件 | 每帧成本 |
|------|---------|
| TerrainQuery(height_at × 200) | ~0.02ms |
| TerrainQuery(raycast × 136 sensory + 5 combat) | ~1.4ms（含在感官预算中） |
| EntityIndex(register/unregister × 100) | ~0.01ms |
| EntityIndex(entities_in_aabb × 17 sensory) | ~0.085ms |
| SpatialEventBus(push × 5 + query × 17) | ~0.05ms |
| VisibilityQuery(line_of_sight × 85) | ~2.1ms（含在感官预算中） |
| COM飞行 × 3 NPC | ~0.06ms |
| 骨架松弛 × 2 NPC | ~0.06ms |
| 双人交互 × 1 | ~0.01ms |
| **总计（独立于感官系统）** | **~0.2ms** |

空间查询的战术层成本远小于感官系统的感知层成本。**~0.2ms 空间查询维护 + 物理响应，在 7.0ms Rust 模拟预算中完全可行。**
