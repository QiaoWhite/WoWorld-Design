## 〇、核心定位

植物是群系的**肉身**——不是贴给地形的标签，不是独立于地形之外的"渲染对象"。群系的名字（"温带阔叶林"）本身就是对植物群落的**统计概括**。先有植物群落，再从这个群落反推群系标签。

植物占世界生物量的绝大多数，植物占玩家视野的绝大多数，植物占 GPU 帧预算的比例必须极小（<7%）。

## 一、架构定位——植物继承什么

植物**不继承 Life 完整基类**。与动物的关键差异：

| | 动物/怪物 | 植物 |
|---|---|---|
| 运动 | 能动——AI/寻路/行为树 | 不可动——极少数魔法植物有缓慢运动 |
| 智能 | 有本能→完全智能 | 无 |
| 行为AI | 需要 | 不需要——仅"生长+反应" |
| 身体状态 | 完整 BodyState(饥饿/疲劳/体温/…) | **无**——仅 `health + spirit` |
| 交互 | 战斗/驯服/捕猎/训练 | 采集/种植/收割/砍伐 |
| 性能 | L1-L4 分层模拟 | 种子驱动瞬态生成 + MultiMesh 批量渲染 |
| 灵元素 | 有(所有动物) | 有(所有植物)——但低(0.05-0.15) |

植物继承极简 `LifeBase`：

```rust
/// 植物继承的极简 Life 子集——不继承完整 LifeEntity
struct PlantLifeBase {
    id: EntityId,
    species_id: SpeciesId,
    display_name: String,
    health: f32,                // 1.0(健康)→0.0(枯死)——简单健康度, 代替完整 BodyState
    spirit: f32,                // 当前灵元素量——补上(之前方案遗漏)
    life_stage: PlantLifeStage, // 种子→幼苗→成熟→衰老→枯死
    tags: HashSet<Tag>,
    ecology: PlantEcology,      // 群系参数范围——让植物能被群系筛选
}
```

**不继承**：Vitals（无饥饿/口渴/疲劳）、PhysicalAttributes（无力量/敏捷）、Mind（无 AI）、Combat（不战斗）。

## 二、8 维程序化生成体系

植物的程序化生成用 **8 个维度**的积木拼装——类比动物的 12 维体系，但针对植物特性裁剪。

### 维度 1：体型尺寸

```
苔藓(Moss)            — 贴地, <5cm, 地表纹理
草本(Herbaceous)       — <1m, 一年生或多年生
灌木(Shrub)            — 1-3m, 多干基部
小乔木(SmallTree)      — 3-15m
乔木(CanopyTree)       — 15-30m, 冠层主导
巨树(GiantTree)        — 30-80m, 地标级——独立生成路径
世界树(WorldTree)      — 100-300m, 大陆传说级——独立生成路径
```

### 维度 2：茎干结构

包含**木质类型**和**分枝模式**两个子维度：

**木质类型**：
```
木本(乔木/灌木) — 多年生, 形成木材
草本              — 一年生/多年生, 柔软
藤本              — 攀援, 依赖其他植物/结构支撑
多肉              — 储水组织, 干旱适应
空心(竹型)        — 节状结构, 轻而坚韧
匍匐              — 茎贴地横向扩展, 节上生根——地被植物
```

**分枝模式**（决定树冠剪影——群系天际线的直接视觉特征）：

| 分枝模式 | 树冠形态 | 代表树种 | 视觉效果 |
|---------|---------|---------|---------|
| **单轴分枝** (Monopodial) | 塔形/圆锥——主干通直 | 云杉/冷杉/落叶松 | 针叶林尖塔天际线 |
| **合轴分枝** (Sympodial) | 球形/伞形/不规则——顶芽停长 | 橡树/榉木/枫树 | 阔叶林圆冠天际线 |
| **二叉分枝** (Dichotomous) | 每节点等分两枝 | 蕨类树木/某些热带树 | 独特的分叉剪影 |
| **基部分枝** (Basal) | 多干从基部并出——灌丛团块 | 多数灌木/榛子 | 灌木群圆块剪影 |
| **不分枝** (Unbranched) | 单杆通顶 | 棕榈/竹子/龙血树 | 伞状冠或棒状剪影 |
| **匍匐蔓延** (Creeping) | 茎贴地横向扩展 | 地被植物 | 地面覆盖 |

### 维度 3：叶系统

```
叶型:
  阔叶 — 大面积, 宽频率沙沙声
  针叶 — 细长, 高频咝咝声
  鳞叶 — 鳞片状叠覆(柏树/桧)
  多汁叶 — 厚肉质, 储水, 几乎无声
  无叶   — 以茎光合(仙人掌/麻黄), 或冬季落叶树
  变色叶 — 秋季变色(黄/红/橙)——落叶树的标志
  发光叶 — 魔法植物专有——恒定微光

叶的季节性:
  常绿(Evergreen)       — 全年有叶——针叶/热带阔叶
  落叶(Deciduous)       — 冬季落叶——温带阔叶
  半落叶(Semi-deciduous) — 干旱季节部分落叶
```

### 维度 4：色彩系统

颜色是植物最直观的视觉特征。花颜色、叶颜色、季节变色、果实颜色、树皮颜色——直接影响群系风貌。

```rust
struct ColorProfile {
    leaf_summer: HslRange,          // 夏季叶色范围——从嫩绿到深绿
    leaf_autumn: Option<HslRange>,  // 秋色——落叶树有(红/黄/橙), 常绿=None
    leaf_spring: Option<HslRange>,  // 春色——嫩绿/红(新生叶常为红色——防御虫害)
    flower_color: Option<HslRange>, // 花色范围——8色调: 白/黄/红/蓝/紫/粉/橙/绿
    fruit_color: Option<HslRange>,  // 果色——吸引传播者
    bark_color: HslRange,           // 树皮色——灰/褐/白/红褐/黑
}

// 花色 + 花形 → 传粉者信号
// 果色 → 成熟度信号: 青→黄→红(鸟类感知红色=成熟)
// 秋色 → 叶绿素分解→类胡萝卜素(黄/橙)+花青素(红/紫)暴露
// 同纬度同季节同物种→同步变色
```

### 维度 5：繁殖系统

```
花型:
  花色(HslRange) + 花形(轮状/钟状/蝶形/穗状/伞形/头状/管状)
  + 花期(哪个季节/持续多久) + 传粉方式

  传粉方式:
    虫媒 — 花大/鲜艳/香
    鸟媒 — 长管状/红色
    风媒 — 小花密集/无花被(禾本科/松柏)
    夜媒 — 夜间开花/白/浓香——蝙蝠/夜蛾
    自花授粉 — 闭花受精——不需要传粉者

果型:
  浆果/核果/坚果/荚果/翅果/球果/蒴果
  + 可食用(bool) + 有毒(bool) + 成熟季节 + 是否为 T1 可采集物

种子传播:
  Wind(风播)          — 轻种子/翅果——长途扩散
  Water(水播)         — 种子漂浮——跨洋传播
  AnimalInternal      — 果实被吃→种子随粪便传播(1-20km)
  AnimalExternal      — 种子钩附皮毛(0.1-2km)
  AnimalCache         — 松鼠埋藏→遗忘=新树(10-200m)
  Explosive           — 爆炸播种(数米)
  Fire                — 火适应——火后释放种子
```

### 维度 6：生命节奏

```
年度类型:
  一年生 — 种子→生长→开花→种子→死亡——一个生长季
  二年生 — 第一年营养生长→第二年开花→死亡
  多年生 — 活多年, 每年生长+开花——绝大多数木本植物

叶周期: 落叶/常绿/半落叶

寿命:
  一年生草本: 数周-数月
  多年生草本: 2-10 年
  灌木: 10-50 年
  乔木: 50-500 年
  巨树: 500-3000 年
  世界树: 3000-10000+ 年

生长速度(米/年):
  快速(竹型/速生树种): 2-5m/年
  中等(多数乔木): 0.3-1m/年
  慢速(橡树/巨木): 0.1-0.3m/年
  极慢(世界树): <0.05m/年——但寿命以千年计
```

### 维度 7：生态策略

```
光照需求:
  喜光(Heliophyte)     — 需要充足光照——先锋物种/草原树木
  耐阴(Sciophyte)      — 能在林下生存——林下灌木/地被
  中间型                 — 大多数树种

水分需求:
  耐旱(Xerophyte)      — 沙漠/半干旱——多肉/深根
  中生(Mesophyte)       — 适中——大多数
  喜湿(Hygrophyte)      — 沼泽/湿地/河岸
  水生(Hydrophyte)      — 挺水/浮叶/沉水/漂浮——独立 4 维生成

温度耐受:
  热带(最低温>15°C)
  亚热带(最低温>5°C)
  温带(最低温>-20°C)
  寒带(最低温>-40°C)
  极寒(最低温<-40°C)

演替位置:
  先锋物种 — 先占领空地——速生/喜光/短寿
  中期物种 — 先锋后进入——中速/中间型
  顶极物种 — 最终占据——慢生/耐阴/长寿

边缘偏好:
  边缘特化 — 在群系边界密度更高——果实多/动物活动热点
  核心种   — 仅在群系内部存活——对边缘环境敏感

根系标签(简化——不作为独立维度):
  浅根 / 深主根 / 块根(采集产物) / 气生根 / 寄生根 / 共生根
```

### 维度 8：特殊能力

```rust
enum PlantFeature {
    Luminescent,       // 发光——吸引传粉者/警告毒性/纯粹美化
    Thorny,            // 荆棘——物理防御——降低啃食量 40-80%
    Toxic,             // 毒性——化学防御——降低啃食量 60-100%
    Carnivorous,       // 食肉——主动捕捉小动物/昆虫——补充氮素

    ExplosiveSeeds,    // 爆炸播种

    WaterDispersed,    // 水播
    AnimalDispersed,   // 动物传播

}
```

## 三、种子驱动 + 稀疏持久化

### 核心原则

**99.9% 的植物不需要存储。** 位置由种子+群系+噪声决定——查询时才生成，用完就扔。只有被玩家/NPC 改变的植物才写入稀疏修改层。

### 多层噪声生态群落生成

一个 Chunk（32m×32m）的植物由五层叠噪决定：

```
第1层: 大尺度密度噪声 (~50m 波长)
  → 这个 Chunk 是"密林"还是"林间空地"还是"过渡带"？
  → density_forest: 0.0(空地) ~ 1.0(密不透光)

第2层: 演替阶段噪声 (~100m 波长)
  → 成熟林(冠层封闭) / 再生林(灌丛+幼树) / 林窗
  → 决定树龄分布和垂直结构

第3层: 物种混合噪声 (~30m 波长)
  → 哪个物种在这里占主导？
  → 每个物种产生自己的密度子图

第4层: Poisson盘个体放置 (物种自适应半径)
  → 冠层树半径大(3-5m间距) / 灌木半径小(1-2m)
  → 自然结果: 密林区树多间隙小、空地无树
  → 采样域扩展 5m——邻 Chunk 边界处确保间距一致

第5层: 个体变异 (per-instance seed)
  → 同物种每棵略有不同: 高度±15%、旋转±22.5°步进、色相±5%
  → 老树=粗干+宽冠、幼树=细干+窄冠
  → 枯立木(1-5%概率)=无叶+灰褐色干
```

### 垂直分层——光竞争自动塑造林下

```
冠层树(canopy, >15m): 先放置 — 树冠半径投射"光锥阴影"
      │
      ▼ 阴影区域内
林下树(understory, 5-15m): 仅在光照 > 40% 的位置放置
      │
      ▼ 进一步筛选
灌木层(shrub, 1-5m): 仅在光照 > 20% 的位置放置
      │
      ▼
地被层(herb/grass): 密度图 = f(光照, 土壤湿度) — 不是个体, 是密度
```

**效果**：
- 密林深处地面干净（地被被遮光抑制）
- 林缘/林窗处灌木丛生（光进来了）
- 疏林草原的地被层茂密
- 新树在倒下老树留下的林窗中萌发

### 选择性持久化——四级

| 层级 | 内容 | 存储方式 | 举例 |
|------|------|---------|------|
| **T0 纯装饰** | 99.9%的草/野花/普通灌木/普通树 | **零存储**——纯种子驱动 | 随地可见的杂草/蕨类/橡树 |
| **T1 可采集资源** | 有产物的草本/灌木/果树 | 存"采集状态"+ 最后采集时间 | 浆果丛/药草/蘑菇/果树 |
| **T2 特殊植物** | 魔法/稀有/巨树/世界树 | 存完整状态——位置+生长阶段+健康+灵元素 | 魔法花/石化树/巨树 |
| **T3 玩家/NPC创建/修改** | 种植/砍伐/烧毁/清地 | 写入稀疏修改层——非递归状态快照 | 农田/果园/伐木区/火灾迹地 |

**T1 草本特殊处理**：药草和浆果丛同时是地被密度图的一部分（不渲染独立 Mesh）+ 有虚拟实例 ID（用于追踪采集状态）：

```rust
/// T1 草本——不渲染独立 Mesh 但需要追踪采集状态
struct HerbInstance {
    instance_id: PlantInstanceId,  // seed-derived: hash(seed, chunk, herb_index)
    species_id: SpeciesId,
    position: Vec3,                // 精确位置——Poisson 盘采样
}
```

**InstanceId 的确定性**：`PlantInstanceId = hash(world_seed, chunk_pos, instance_index)` ——种子推导，不存储。T0 实例被砍后写入 T3 delta；下次查询该 Chunk 时查 Delta 层掩码→该 ID 在"已移除集合"中→不生成该实例。

## 四、植物产物

### 五类产物

| 产物类型 | 内容 | 采集方式 | 再生 | 举例 |
|---------|------|---------|------|------|
| **木材** | 树干——按树种分硬度/纹理/耐久 | 砍伐——移除整棵树 | ❌ 不自动再生 | 橡木(硬)/松木(软)/巨木(船龙骨) |
| **纤维** | 树皮/藤蔓/叶片/草茎 | 采集——不破坏植株 | ✅ 5-10 游戏天 | 亚麻纤维(纺织)/藤条(编织)/芦苇(造纸) |
| **食物** | 果实/坚果/种子/块根/嫩叶/树汁 | 采集——按季节有或无 | ✅ 3-7 游戏天 | 野果/橡子/松子/野莓/枫糖浆 |
| **药物/炼金** | 药草/香料/毒草/魔法植物精华 | 采集——需识别技能 | ✅ 7-15 游戏天 | 金创药草/曼德拉草/夜光菇/龙息椒 |
| **染料/装饰** | 花瓣/色素/香精 | 采集——不破坏植株 | ✅ 按花期 | 靛蓝(蓝色)/茜草(红色)/薰衣草(紫色/香料) |

### 采集机制

```
采集(必定成功——不卡成功率):
  ① 检查物种植物的产物类型 + 季节 + 成熟度 → 可采集？产出什么？产量多少？
  ② 检查采集者技能 → 产量加成(0.5x ~ 1.5x)、品质加成
  ③ 检查工具 → 斧=木材加成、镰=纤维加成、手=食物/药草(base)
  ④ 更新持久化:
     - 食物/纤维/药草 → 写入 T1 Delta(Harvested) → 再生计时器启动
     - 木材 → 写入 T3 Delta(TreeChopped) → 树从渲染移除
  ⑤ 产物进入采集者 inventory

再生:
  食物: 3-7 游戏天后可再生——一个季节可采 2-3 轮
  纤维: 5-10 游戏天
  药草: 7-15 游戏天——稀有=慢
  树苗: 砍伐后 1-3 游戏年原地出现新苗——玩家能看到变化
  成材: 5-10 游戏年树苗长大——一代玩家能见证森林更新
  草/地被: 3-5 游戏天快速恢复——保持群系底色
```

## 五、生态链——植物与动物

### 仅保留一个运行时通道：啃食

```
啃食压力(每 L3 周期计算一次, <0.001ms/Chunk):

  grazing_pressure = 植食动物密度 × 平均体型 × diet_plant_ratio
  plant_density_loss = grazing_pressure × palatability × (1 - defense)
  
  defense = f(荆棘, 毒素, 高度)
  palatability = 物种可口度, 0.1(难吃) ~ 1.0(美味)

效果:
  轻啃 → 植物健康微降(可恢复)
  重啃 → 生长停滞、当季不结果
  持续过度 → 种群密度下降 → 草地→裸土→荒漠化风险
  掠食者减少 → 植食动物爆发 → 植被崩溃(生态连锁)
```

### 全烘培进世界生成（零运行时成本）

```
✅ 传粉配对 — 群系生成时查表, 传粉者不存在=依赖它的植物自动淘汰
✅ 共生关系 — 同上, 生成时配对约束
✅ 种子传播基线 — 群系噪声已隐含扩散范围
```

### 统一底层

```

                 × 光照因子(0-1, 从群系光照+遮阴计算)
```

### 三层灵元素

**第一层：普通植物（0.05-0.10）——被动效果**

```
四个被动效果:

1. 生命感知: 植物"知道"附近有生物——不是意识, 是灵元素微弱共振
   → 游戏表现: 某些植物在生物靠近时微动(合拢/颤抖)——纯粹的氛围

2. 环境记录(读树): 植物长时间处于同一位置→灵元素缓慢吸收环境信息
   → 游戏表现: 老树可被"读"——获取该地历史的情报碎片(见 §二十)

3. 魔力原料: 所有植物的灵元素可被提取——但效率极低
   → 大量焚烧→微弱魔力释放。不是有效方式, 但理论上可行

4. 死后滋养: 植物枯死→灵元素散逸→滋养土壤
   → 腐殖土灵元素含量略高于裸土
```

```
灵元素 > 0.10 → 可感知的魔法性质:

  · 药效增强: 治疗草药的有效成分 × 灵元素含量
  · 炼金亲和: 可被炼金术提取——灵元素+有效成分→炼金产物
```

```
特征(至少具备一项):
  · 发光(恒定——不依赖外部光源)
```

## 九、巨树与世界树——全程序化生成

巨树和世界树由**同一条程序化管线**生成，高度是连续参数——不做硬分类。

### 三级连续谱

```
生成条件叠加:
  灵元素浓度高 + 古老演替阶段(数千年未扰动) + 地脉节点 + 长期稳定
  条件满足越多 → 高度上限越高

┌──────────┬──────────┬──────────────┬──────────┐
│          │ 高度     │ 概率(陆域)    │ 核心差异 │
├──────────┼──────────┼──────────────┼──────────┤
│ 大树      │ 30-50m   │ ~每10km² 1棵  │ 地标级   │
│ 巨树      │ 50-100m  │ ~每500km² 1棵 │ 聚落核心 │
│ 世界树    │ 100-300m │ 每存档0-5棵   │ 大陆传说 │
└──────────┴──────────┴──────────────┴──────────┘

世界树概率 = 种子决定——有些存档 0 棵、有些 3 棵、有些 5 棵
             0棵 → 该世界没有世界树——NPC 传说也不会有"世界树"概念
             多棵 → 不同大陆各一棵→大陆间"谁的世界树是真的"→信仰分歧
```

### 生态效应

```
巨树:
  · 自身是一个"微群系"——树冠上/中/下三层光照不同→不同植物附生
  · 树干上长满附生植物(苔藓/蕨类/兰花)——垂直生态梯
  · 树洞中有独特生物(巨树特有鸟类/昆虫/灵兽)
  · 根系范围→周围土壤灵元素浓度↑
  · 天然地标——NPC 用它导航
  · 可攀爬——树冠顶=天然瞭望台
  · 不可砍伐——但可采集树皮/树脂/附生药草(T2 持久化)

世界树:
  · 穿透云层——树冠顶部可能有独立微气候
  · 树枝上不同高度=不同生态带(底层温带、中层云雾森林、顶层高山苔藓)
  · 树根系统形成天然地下城(根系间空洞=洞穴网络)
  · 树冠上可能有聚落(精灵城市/隐士村/龙巢)
  · 整棵树是活的魔力源(spirit_capacity > 0.7)
  · 如果世界树生病→周边大范围魔力下降
  · 世界树的种子=游戏中最稀有物品之一
  · 世界树的死亡=大陆级历史事件——所有 NPC 都会记住
```

## 十一、群系过渡带

过渡带不是两个群系的"混合区"——是**物种逐个退出/进入的排序梯度**。

### 四种基本形态

```
类型 A: 气候渐变带 (宽数百米至数千米)
  · 温度/降水连续变化 → 物种逐个退出/进入
  · 不是"A和B混合"——是"A物种逐一被B物种替代"

类型 B: 地形突变带 (窄, 陡峭)
  · 山脊/悬崖/河岸 → 参数在短距离内剧变
  · 边缘锐利但两侧有"溢出"

类型 C: 斑块镶嵌带 (离散, 多尺度)
  · 参数在临界值→小地形扰动决定局部归属
  · 南坡=草斑、北坡=树斑——同一 Chunk 内交替

类型 D: 动态边缘 (火/洪水维持——接口预留)
  · 周期性干扰→边缘停留在演替早期
  · 生成时用噪声模拟
```

### 三尺度噪声嵌套

```
一段 200m 宽的森林→草原过渡带:

大尺度趋势 (~200m 波长):
  树密度: 0.7 → 0.5 → 0.3 → 0.1 → 0    (sigmoid 曲线, 非线性)
  草密度: 0   → 0.2 → 0.5 → 0.7 → 0.9
  → 由群系参数梯度决定——确定性

中尺度斑块 (~30-60m 波长):
  在树密度 0.3 的区域: 微地形凸起→排水好→树密度跳到 0.5
  在树密度 0.3 的区域: 微凹地→积水→树密度跌到 0.1
  → 创造"森林伸入草原的指状延伸"和"草原吞入森林的凹入"

小尺度点缀 (~5-15m 波长):
  孤树: 在树密度<0.05 区域随机出现
  灌丛: 在草密度>0.7 区域随机出现
  枯立木: 过渡带中随机分布(1-3%)
```

### 物种排序——逐个退出

```
森林→草原过渡: 树种不是一起消失的

冠层树种退出顺序(随降水减少):
  热带硬木(需>1800mm) → 最先退出
  橡树(需>800mm)       → 中间退出
  松树(需>400mm)       → 最后退出——延伸到稀树草原深处

灌木/草种进入顺序:
  灌丛                   → 最早——光多了
  高草(>1m)              → 紧随
  短草(<30cm)            → 降水<500mm 后才成为主导

每个物种的"存活带"由 PlantEcology 定义:
  查询当前点的连续参数 → 筛选能存活的物种 → 按存活概率排序
  → 自然结果: 不同物种在梯度上不同位置"退出"或"进入"
```

### sigmoid 密度曲线——不是线性

```
树密度过渡:

  密度
  1.0 ┤          ╱
      │        ╱
  0.7 ┤───────────────
      │      ╱        ╲
  0.5 ┤    ╱            ╲
      │  ╱                ╲
  0.2 ┤╱                    ╲
      │                      ╲────
  0.0 ┤────────────────────────────
      ├────┬────┬────┬────┬────┬───
```

## 十三、水生植物——4 维生成 + 系统标签

水生植物走独立生成路径——陆地 8 维不适用。水下是**光衰减深度**和**水体参数**决定一切。

### 维度 1：水体定位——这是"群系"

```
盐度:
  淡水(0-0.5‰) — 河流/湖泊/溪流
  半咸水(0.5-30‰) — 河口/三角洲/潟湖
  海水(30-40‰) — 海洋
  超盐(>40‰) — 盐湖/蒸发潟湖

流速:
  静水(still) / 缓流(slow) / 急流(fast) / 潮汐(tidal) / 洋流(ocean_current)

透光层深度(取决于浊度+季节):
  透光层 0-xm — 光够光合作用
  弱光层 x-2xm — 仅沉水耐阴植物
  无光层 >2xm — 无光合植物——仅真菌/魔法依赖
```

### 维度 2：生长形态（视觉核心）

```
挺水型(Emergent)     — 茎叶出水——芦苇/香蒲/荷花/纸莎草
                      玩家看到: 水面上高耸的杆穗/带状叶丛/大圆叶

浮叶型(Floating-leaf) — 叶贴水面——睡莲/菱角/荇菜
                       玩家看到: 水面上圆盘叶覆盖→水面纹理变化

沉水型(Submerged)     — 全在水下——海草(带状)/金鱼藻(细裂)/海藻(气囊直立)
                      水面看: 水下深色阴影
                      潜水时: 水下"草原"或"森林"

漂浮型(Free-floating) — 随水漂——浮萍(微粒)/水葫芦(叶丛+须根)
                      玩家看到: 水面绿色覆盖→缓慢移动

潮间型(Intertidal)    — 周期性暴露——红树林(支柱根+胎生苗)/盐沼草
                      玩家看到: 支柱根迷宫——退潮可见/涨潮淹没
```

### 维度 3：视觉特征

```
水面可见:
  花色+花期 + 叶色 + 秋冬枯黄倒伏

水面覆盖度(视觉关键):
  散点(<10%) → 斑块(10-40%) → 覆盖(40-80%) → 全覆盖(>80%)

潜水可见(仅潜水时):
  叶形(带状/细裂/藻体/圆叶) + 透光性 + 气囊
```

### 维度 4：与玩家交互

```
采集产物: 挺水→纤维/嫩茎、浮叶→花/种子、沉水→药草/炼金、潮间→耐水腐木材(造船)

移动影响: 挺水密丛→不可步行通过、浮叶→游泳速度微降、红树林→支柱根迷宫

钓鱼关联(★ 最重要):
  挺水植物边缘=鲤鱼/鲫鱼
  睡莲下方=黑鱼/鲈鱼
  海草床=海马/热带鱼
  海藻林=龙虾/海胆
  → 植物分布=天然钓鱼点标记
```

### 系统标签（驱动底层系统，不展示给玩家）

```
  鱼群吸引力(fish_attraction) —— 0-1
  水质影响(water_clarity_effect) —— 负=净水, 正=浊水
  波浪衰减(wave_damping) —— 0-1 —— 芦苇带→浪小
```

## 十五、渲染管线——从密度场到像素

### 总管线

```
玩家移动到新区域
  → Chunk 加载
  → Rust: 种子→多层噪声→光竞争→Poisson盘→PlantCommunitySnapshot(瞬态)
  → Rust: 查询 T1-T3 Delta 层→移除被砍、加入被种
  → Rust: LOD 筛选→按物种分组→MultiMesh 批次
  → GDExtension: PackedByteArray→Godot
  → Godot: MultiMeshInstance3D + GPU skinning shader (季节+风+雪)
  → 屏幕
```

### 跨 Chunk 全局合批

```
每帧:
  ① 确定所有视锥内 Chunk(已有——terrain 渲染系统)
  ② 对每个 Chunk: 查询 PlantCommunitySnapshot(瞬态)
  ③ 合并: 所有 Chunk 的实例→按 species_id 分组
  ④ 对每个物种: 1 个 MultiMesh, 1 draw call
  
实例估算(可见 500m × 90° FOV ≈ ~100 Chunk):
  冠层树: 1000-3000 棵
  灌木: 3000-8000 丛
  总实例: 4000-11000
  跨 30-60 物种 → 30-60 draw calls
  GPU 成本: ~0.3ms
```

### LOD 四级

```
LOD0: 全 3D 实例 (<200m)
  冠层树: 全 3D + CSM 阴影 + 风动画
  灌木/小树: 全 3D + blob shadow + 风动画
  草本 T1: 虚拟实例——不渲染, 仅追踪采集状态
  草 T0: compute shader 草片——不生成实例

LOD1: 4 角度 Impostor (200-800m)
  对每个物种: 4 角度(0°/90°/180°/270°)预烘焙 RGB+Alpha
  运行时: 相机方向→选最近角度→1 个四边形
  4 角度 × ~600 地表物种 × 128² RGBA ≈ ~150MB VRAM——可行
  阴影: 简化代理——竖直柱体+顶部球体

LOD2: 群落级 (800m-3km)
  整个 Chunk 冠层→1-2 个大四边形
  从代表性树种烘焙森林顶部纹理
  赋给 Signed Heightfield 的高度偏移

LOD3: 色彩高度场 (>3km)
  完全融入 terrain LOD 6-7 Signed Heightfield
  群系色×季节=赋给 Heightfield 颜色通道
  冠层顶面烘焙进 Heightfield→远山天际线有森林起伏

LOD 过渡: Dithered Crossfade
  用屏幕空间 Bayer 矩阵渐变——不透明, 无 alpha blend overdraw
```

### 草渲染——Compute Shader + Indirect Draw

```
不生成草实例——GPU compute shader 驱动:

Step 1: Compute Shader
  输入: GroundCoverMap (128×128 per Chunk)
  输出: AppendStructuredBuffer——每草片 [position, height, bend_phase, color_variant]

Step 2: ExecuteIndirect
  顶点 shader 从 StructuredBuffer 读草片数据
  → 生成三角形条带(每草片 3-5 顶点)
  → 无 CPU 往返——全部在 GPU

Step 3: Vertex Shader 风动画
  amplitude = f(草高, 风速, bend_phase)
  动物经过→密度图修改(density=0)→compute shader 不生成那里草片→自动"压平"
  恢复: 随时间 density 回到原值→草"长回来"

GPU 成本: ~0.1-0.2ms per Chunk
```

### 每实例 GPU 数据（64 字节）

```
mat4 transform:        48 字节 (3×vec4, 第4行隐式)
uint packed_data:      4 字节
  [0:7]   年龄百分位 (0-255)
  [8:15]  色相偏移 (0-255→-15°~+15°)
  [16:23] 高度乘数 (0-255→0.85~1.15)
  [24:27] 旋转偏移 (0-15→0°/22.5°/...337.5°)
  [28:31] 风 phase 偏移 (0-15→0-2π)
uint species_data:     4 字节 (species_id)
float health:          4 字节
uint reserved:         4 字节 (灾害/修改标记)

总计: 64 字节 × 11,000 实例 ≈ 704KB——完全在 GPU 缓存内
```

### 季节与天气——Shader 参数, 零额外存储

```
全局 uniform → vertex shader:
  time_of_year, weather_state, wind_vector

Shader 计算:
  叶色:      summer_color → autumn_color (lerp, autumn_factor)
             autumn_factor = f(time_of_year, species, latitude)
             同纬度同季节同物种→同步变色
  花:        bloom_visibility = f(time_of_year, bloom_season, bloom_duration)
  落叶:      leaf_drop = f(time_of_year, leaf_cycle)
  风:        草=高位移低刚度, 灌木=中位移, 大树=极低位移高刚度
             阵风 gust→所有植物同步→回弹各异(物种阻尼)
  雨后  specular↑; 霜冻 叶色灰暗; 雪重 vertex 下移
```

### 水体植物渲染

```
挺水植物(芦苇/香蒲):
  水面以下: 水下 fog 颜色
  水面以上: 正常光照
  交界线: 水面波动→植物茎上水线波动(vertex shader)

浮叶植物(睡莲):
  在水面 shader 中采样浮叶密度图→替换水面纹理
  不需要独立 3D mesh

漂浮植物(浮萍):
  共享水面 mesh 顶点位移, 偏移在水面之上 1-3cm
```

### 阴影

```
LOD0 (<200m):
  冠层树(>15m): 投射 CSM(Cascade Shadow Maps)
    C0: 0-30m(高精度)——玩家+最近树
    C1: 30-80m(低精度)——中距离树
  灌木/小树: 仅 SSAO + 虚拟阴影(树下半径内地表微暗)
```

## 二十二、植物 × 天气交互——七个通道

### 风——最核心

```
不同植物的风响应(从 PlantSpecies 的 morphology 预计算, shader 常量):

  迎风面积 = f(leaf_shape, canopy_width, leaf_seasonality, season)
  弹性模量 = f(stem.wood_type, stem.max_height)
  阻尼系数 = f(leaf_shape, leaf_seasonality)
  自然频率 = f(stem.max_height, stem.wood_type)

  草:          高位移, 高频率, 低刚度 → 细碎摆动
  灌木:        中位移, 中频率, 中刚度 → 蓬松摇曳
  小树(<10m):  低位移, 低频率, 中刚度 → 慢摇
  大树(>15m):  极低位移, 极低频率, 高刚度 → 几乎不动
               树冠边缘嫩枝=高频微动
  针叶:        位移 < 阔叶(小迎风面)
  枯枝:        位移 > 活枝(更脆)

阵风(gust):
  t=0 → 所有植物同方向猛倾(方向一致性→真实感)
  t=0.5s → 回弹→过冲→反方向微倾
  t=1-2s → 残余振荡(阻尼决定衰减)
  t=5-10s → 恢复基线

风在森林中的变异:
  林外: 全速风
  林缘: 风加速(狭窄效应)
  林内: 风速骤降(≈林外×canopy_closure×0.4)
  树冠背风面: 涡流→枝叶不规则乱晃

叶落/花粉/果实坠落: 强风+秋季→落叶加速; 成熟果实→概率吹落→采集机会
```

### 雨

```
冠层截留(Chunk 级, 每降水周期一次):
  截留率 = canopy_closure × 0.5 × f(leaf_seasonality) × f(leaf_shape)
  林下实际降水 = 总降水 × (1-截留率)
  截留水→蒸发(30%)+沿树干流下(40%)+叶缘滴落(30%)

不同雨型:
  细雨: 叶面均匀湿润→几乎无声
  中雨: 叶面雨滴击打→叶片微振→滴答声(阔叶)/细密咝声(针叶)
  暴雨: 叶剧烈振颤→叶子翻面→可见白背→树冠在雨中模糊
  雨后: specular↑ + 持续滴水(数十分钟到数小时) + 蘑菇冒出(采集)

冠层阴影: 密林树冠截留30-50%→林下雨比开阔地小→地被更干→雨后地面更早干燥
```

### 雪

```
冠层积雪(per Chunk):
  积雪速率 = snowfall × 冠层截雪率
  截雪率 = canopy_closure × f(leaf_seasonality) × f(leaf_shape) × f(branch_pattern)
  针叶塔形(单轴)=最高截雪(雪滑到枝腋堆积)
  阔叶落叶冬季=0截雪

  积雪量累积 → 超过承雪阈值 → 自发滑落(大量)
  温度>0°C → 开始融化滴水
  阵风 gust > 阈值 → 抖掉薄雪
  玩家/NPC 撞树干 → 惊喜被雪淋

视觉:
  针叶塔形: 层层白环嵌套——标志性"圣诞树"外观
  落叶裸枝: 枝腋少量雪——白点散布
  常绿阔叶: 宽叶大面积雪→叶被压弯
  灌木: 完全被雪埋→蓬松雪丘
  草: 消失(weather temporary 层覆盖)
  苔藓: 雪下存活→雪融后最先绿
```

### 极端温度

```
霜冻渐进:
  轻度(-0~-3°C): 不耐寒植物叶缘开始枯萎(色偏褐)
  中度(-3~-8°C): 叶下垂(白天回暖可恢复), 农作物→损失
  严重(<-8°C): 树皮开裂(冰冻膨胀)→树可能死亡, 一年生=死(种子过冬)
  视觉: 晨霜=叶缘白结晶, 雾凇=针叶树白晶被覆(极美)

热浪/干旱渐进:
  D1-3: 草生长减速, 阔叶叶缘微卷(减少蒸腾)→可观察!
  D4-10: 草开始变黄, 灌木叶下垂, 树木老叶黄化脱落(应激弃叶)
  D10-20: 草枯黄=可燃物, 灌木永久萎蔫, 树木嫩枝枯死
  D20+: 最脆弱个体死亡, 耐旱物种存活, 景观绿→黄→褐→灰
  恢复: 草3-5天返绿, 灌木1-2周新芽, 枯死枝不可恢复
```

### 冰雹、雾露、闪电

```
冰雹: 树叶被打烂(嫩叶穿孔), 果实全被打落, 极罕见——年一遇
      发生后 NPC 会记住("二十年前那场雹灾")

雾/露:
  雾: 植物表面水膜, 蛛丝在雾中反光, 苔藓饱满
  露: 清晨叶面水珠, 逆光闪烁, 草地 walk through→湿脚

闪电:
  击中最高树→树被劈开/燃烧→T3写入
  雷击木=稀有炼金/魔法原料
  目击者→记忆+传播("上帝在发怒")
```

### 天气组合效应

```
大风+暴雨:   最戏剧化——树剧烈摇摆+雨帘横扫+叶翻面, 枝断风险最高
大雪+微风:   最静美——树冠均匀积雪+安静(雪吸音), NPC 室内烤火
暴风雪:      雪水平飞, 树冠积雪不均, 可见度极低——最危险天气
雨后+晴+晨:  叶面水珠+晨光逆光=万点闪光, 蘑菇爆发(采集高峰)
霜冻+雾+晨:  雾凇——针叶树白晶被覆+逆光→"发光", 日出后雾散→滴水→魔幻半小时
暖风+干旱:   植物加速脱水, 可燃物急剧增加→火灾窗口
```

### 数据流

```
天气系统(WeatherState 输出)
    │
    ├→ Rust L3 周期: 温度累积×物种耐受→健康度, 降水×冠层→地被湿度, 降雪×冠层→积雪
    ├→ Rust 每帧: wind_params→GPU uniform buffer
    └→ GPU 顶点shader: 风位移+雪下移+叶色+specular
```
