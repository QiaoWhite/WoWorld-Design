## 〇、核心定位

本文档定义玩家角色的**创建参数、双角色机制、死亡与继承、衰老规则**——是对 [[011-玩家]] 的补充扩展。

玩家角色和 NPC 的本质关系（[[009 玩家角色本质|Change/hand/009]]）：
- 玩家操控的角色本质上和 NPC 一样——共享 BodyState、RaceTraits、部位伤害、死亡流程
- 玩家可以开启完全托管模式（角色按自己的意愿行事）
- 手操模式中可以部分托管（如只控制移动，其他任由角色自主）
- 两个可控角色随时切换

## 一、角色创建

### 1.1 两种进入模式

| 模式 | 描述 | 来源 |
|------|------|------|
| **原住民** | 接管既有一个 NPC 的人生——继承其身份、家庭、工作、人际关系、财产 | [[Change/hand/011 玩家进入游戏]] |
| **穿越者** | 玩家自定义全部参数——角色从前与这个世界毫不相干 | [[Change/hand/011 玩家进入游戏]] |

**原住民模式**：
- 世界生成完成 → 玩家从大日志中选择一个 NPC → 接管
- 角色已经在做某事（铁匠在工作、农夫在田间、士兵在巡逻）——玩家切入的第一刻听到 NPC 的日常对话（"师父，今天打什么？"）
- 角色的家庭关系、社会义务、财产、名声——全部继承。是否甩掉这些——玩家自己决定，后果由系统涌现

**穿越者模式**：
- 玩家在进入游戏前完成角色创建的参数填写
- 参考日式异世界奇幻冒险的进入感——从零开始，一无所有（或含金汤匙）
- 随机出生点在野外——第一个交互可能是树、兔子、石头。最近聚落可能要走两天

### 1.2 可自定义参数

穿越者模式下的角色创建参数。以下字段从 `LifeEntity` + `SapientMind`（NPC 活人感模块）+ 技能/魔法/物品系统的完整数据模型中导出，零纯文本"背景故事"框——所有参数都对应到已有数据结构的实际字段。

#### 一、身份层

| 参数 | 类型 | 来源 |
|------|------|------|
| `display_name` | String | 生命/001 |
| `species_id` | SpeciesId | 生命/011 §五 — 人类·精灵·矮人·程序化涌现种族 |
| `biological_sex` | BiologicalSex | NPC活人感 §1.1.7 — Male/Female/Hermaphroditic/Sequential/Neuter |
| 出生坐标 | WorldPos | 随机或指定 |

#### 二、身体层

| 参数 | 类型 | 来源 |
|------|------|------|
| 力量 / 敏捷 / 体质 / 感知 / 灵巧 / 耐久 | f32 (0-1) | 生命/001 PhysicalAttributes |
| 身高 (相对种族均值偏差) | f32 (-1~1) | NPC活人感 §1.1.7 |
| 体型 | BuildType | Slender/Lean/Average/Muscular/Stocky/Heavy |
| 脸宽 / 下巴形状 / 颧骨高度 | f32 | NPC活人感 §1.1.7 (FaceParams 8维) |
| 眼大小 / 眼距 / 鼻梁 / 鼻宽 / 唇厚 | f32 | 同上 |
| 肤色 / 发色 / 发型 / 胡型 | 枚举 | NPC活人感 §1.1.7 |
| 声线 / 音色 | f32 (0-1) | NPC活人感 §1.1.7 |
| 疤痕 / 纹身 / 特征标记 | 可选列表 | NPC活人感 §1.1.7 |

#### 三、心智层

| 参数 | 类型 | 来源 |
|------|------|------|
| 智力 / 智慧 / 意志力 / 魅力 | f32 (0-1) | 生命/001 MentalAttributes |
| 记忆容量 | u16 (0-2000) | 生命/001 |
| 开放性 / 尽责性 / 外向性 / 宜人性 / 神经质 | f32 (0-1) | NPC活人感 §1.1.2 BigFive (至少一个突出维度 \|Δ\|>0.25) |
| 认知风格 4 维 | f32 | NPC活人感 CognitiveStyle — 直觉-分析/冲动-反思/具象-抽象/顽固-灵活 |
| 审美品味 | AestheticTaste (32B) | NPC活人感 §1.1 |
| 吸引力偏好模板 | AttractionTemplate | NPC活人感 §1.1 |

**人格锁定规则**：
- 创建时随便填
- 进游戏后和 NPC 一样——终身极少变动
- 仅极高冲击力创伤 (>0.9) 或长期环境冲突 (>100 游戏日) 时微调 (Δt ≤ 0.05)
- 玩家不能打开面板修改。除非改后台数据

#### 四、技能层

| 参数 | 类型 | 来源 |
|------|------|------|
| 任意 SkillId 初始经验值/等级 | (SkillId, xp, level) | 技能系统/002 |
| 天生才能乘数 | f32 (0.7-1.3) | 技能系统 MentalAccess trait |
| 跨训练非递归上限 | SkillGroup 映射 | 技能系统 |

#### 五、魔法层

| 参数 | 类型 | 来源 |
|------|------|------|
| 十元素亲和 (金木水火土风雷电血灵) | [f32; 10] | 生命/001 SpiritState |
| 魔力强度 / 控制 / 抗性 / 恢复速度 | f32 (0-1) | 生命/001 MagicAttributes |
| 初始已知法术 | Vec\<SpellId\> | 魔法模块 |
| 初始法术书/卷轴 | Vec\<ItemId\> | 物品系统 |

#### 六、语言层

| 参数 | 类型 | 来源 |
|------|------|------|
| 第一语言 | LanguageId (proficiency 1.0) | 语言表达/002 |
| 额外语言及 proficiency | Vec\<(LanguageId, f32)\> | 语言表达/002 |

#### 七、物品/装备层

| 参数 | 类型 | 来源 |
|------|------|------|
| 初始衣物 | ItemId | 默认：一套平平无奇的衣服 |
| 初始货币 | u32 | 经济系统 |
| 初始额外物品 | Vec\<ItemId\> | 可选 |

### 1.3 自定义范围

- 玩家可以从零开始（全部默认值）——一整套平平无奇的衣服、随机坐标
- 也可以填成龙傲天——全部属性拉满、十元素亲和全高、初始携带神器
- 数值范围由各数据结构的字段类型决定——不添加"角色创建专属限制层"
- 玩家填的值直接写入 LifeEntity + SapientMind 的对应字段——没有翻译层

## 二、双角色系统

### 2.1 核心机制

| 项目 | 规格 |
|------|------|
| 可控角色数 | 2 |
| 切换 | 随时 |
| 未被操控的角色 | 完全自主——GOAP + 概率决策引擎驱动，自己做自己的事 |
| 两个角色都睡/无法活动 | 可选时间跳过——直到至少一人醒来 |
| 两人关系 | 和 NPC 间关系一样涌现——不被系统特殊对待 |
| 大日志 | 共享——两个角色共用同一本日志 |

### 2.2 两个角色的关系

- 两个角色之间有一条 `Relationship`（和所有 NPC 间关系同一数据结构）
- affection / trust / familiarity / attraction —— 全是这两个角色互动中涌现出来的
- 角色 A 对角色 B 的感情不是玩家手动设定的——是 A 和 B 相处时间、对话质量、共同经历（战斗/合作/背叛）的结果
- 玩家可以试图让 A 喜欢 B —— 但 B 喜不喜欢 A 不是玩家决定的，是 B 的大五人格 + 情绪 + 关系涌现的

### 2.3 活动模式

- **一起活动**：两个角色在物理空间中结伴——一起打铁、一起航海、一起战斗。未被操控的那个跟随 + 自主行动
- **分开活动**：一个在北山屠龙、一个在苍山城打铁。切换时——你看到的已经是另一个人的生活流

### 2.4 大日志的共享

- 大日志像两个角色共享的外置大脑
- 角色 A 在北地发现霜铃兰 → 大日志自动记录 → 角色 B 在南海立刻可见此条目
- 时间线镜头的双角色混排模式：角色 A 的条目和角色 B 的条目用颜色区分来源
- 大日志不随个体死亡消亡——旧的条目在这本共享日志里永久保留（除非玩家手动清除）

## 三、死亡与继承

### 3.1 死亡瞬间

- 角色 HP 归零 → 进入和 NPC 相同的濒死窗口（抢救/魔法干预）
- 没人救 → 死亡
- 如果玩家正在操控该角色：
  - 身体倒地。摄像机留在原地约 3-5 秒（告别间隙）
  - 屏幕边缘微暗（不明原因），一声低沉的安静音效
  - 屏幕底部文字：`[角色名] 死亡 · 死因：XX · 地点：XX · 享年：XX 岁`
  - 底部提示：`[另一个角色名] 正在 XX。切换？`（如果另一个活了）
- 如果玩家正在操控另一个角色：
  - 大日志里多了一条死亡条目——另一个角色点开日志就看到了
  - 会不会悲伤——取决于角色的大五人格 × 情绪 × 两者之间的关系。强硬的战士可能沉默，高神经质的恋人可能崩溃。全是涌现——不是系统推送"你感到不安"
- NPC 的反应：全由 NPC 自主。熟人 → 哀悼、传播、遗忘。路人 → "又死了一个兵"

### 3.2 尸体

- 和 NPC 的尸体完全一致：物理存在、可搬运/掩埋/焚烧/抢夺、按生命模块规则腐烂/风化
- 多年后，该地点的 AetherImprint 多了一条"一个人死在这里"
- 读树的人可能感受到模糊碎片——不完整、有诗意

### 3.3 继承规则

和 NPC 继承一致——走生命模块的继承管道。物品 → 血亲/债权人/同僚 → 无人则留在尸体上。

**继承顺序**：

1. **血亲继承**（一代以内：父母/子女/兄弟姐妹）——选择一个大日志中的血亲接管。该角色的物品按继承规则转移
2. **大日志继承**——无血亲时，选择大日志中的任何人接管
3. **全新角色**——无合适继承对象时，创建新角色加入世界

**大日志的继承选择**：
- **继承全部**：旧角色条目保留，新角色延续同一本日志
- **封印**：旧条目标灰但仍可查——"前世"的记忆还在但被标记为过去的
- **清除**：全部移除，新角色从空白大日志重新开始

### 3.4 技能与关系的不可继承性

- 旧角色四十年的技能——没了。新角色有自己的 Skills
- 旧角色的人际关系——新角色和那些人**不认识**
- 大日志知道你旧角色的人生——但你现在是另一个人。你记得一切，但你不会那些手艺、朋友不再是你的朋友
- 你可以选择告诉旧角色的朋友"我是他的兄弟"——也可以沉默。系统不强推"你长得像他"

### 3.5 双角色全死

- 两个角色都死了 + 都无血亲 → 弹出结算
- 存档保留——下次加载时再次弹出继承提示
- 玩家可以选择结算（查看大日志回顾一生）或从大日志中选一个新角色接续

## 四、衰老

### 4.1 基本规则

- 玩家角色和 NPC 一样——每年长一岁。`age_years` 递增
- `life_stage` 按种族 max_lifespan 自动推进：Young → Adult → MiddleAge → Old → Ancient
- 精灵活 500 人类活 80——年龄体验取决于种族选择

### 4.2 衰老的物理影响（待正式设计）

本节为预留框架。具体参数变化待独立设计。

```rust
/// 衰老修正——从 life_stage 派生（预留）
struct AgingModifiers {
    /// 身体属性随年龄的变化乘数
    strength_mod: f32,       // Old: ~0.7-0.8, Ancient: ~0.5-0.6
    endurance_mod: f32,      // 耐力下降最明显
    perception_mod: f32,     // 感官迟钝——但 wisdom 可能上升
    /// 魔力——精灵等长寿种族可能越老越强
    spirit_mod: f32,
    /// 技能经验的获取速率
    learning_rate_mod: f32,  // 老了学新东西慢
}
```

**设计原则**：
- 衰老不是惩罚——是人生阶段。老铁匠可能体力不如从前，但他的剑因为几十年经验（技能 Lv 极高）仍然是最好的
- "我练了 40 年手艺因为老了握不稳工具"——这不是 Bug，是人世的一部分
- 精灵 500 年→ 衰老极慢。人类 80 年→ 中年后加速。不同种族的选择对游戏体验有深远影响

## 五、后代（待正式设计）

### 5.1 预留框架

- 玩家角色如果是智能种族 + 满足繁衍条件（生命模块繁衍系统）——可以生育
- 孩子是 NPC（SapientMind）——有自己的人格、技能、属性
- 孩子的 traits = 父母 traits 加权 + 种子随机变异（遗传 × 变异）
- 孩子长大后可成为继任可控角色——"玩自己的后代"
- 大日志在家族线上跨代延续

### 5.2 待决定

- 如果两个可控角色都是玩家控制的——两人生的孩子由谁控制？
- 孩子的成长速度——需要和时间流速对齐

## 六、离线时间处理（待正式设计）

### 6.1 问题

WoWorld 是单机游戏。玩家关闭游戏 → 世界停止。NPC 的"40 年人生"只在玩家在线时推进——实际游戏时间如何定义？

### 6.2 待选方案

| 方案 | 描述 |
|------|------|
| **暂停** | 关闭 = 世界冻结。下次打开时从关闭时刻继续。40 年 = 40 游戏年 × 游戏时间流速 |
| **摘要推演** | 关闭 = 低精度模拟。关键事件统计（谁死了/谁出生/战争结果）。玩家回到时看到变化 |
| **混合** | 短离线暂停（< 一周现实时间）→ 长离线的摘要推演 |
