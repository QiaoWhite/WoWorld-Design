## 一、模块定位与设计哲学

### 1.1 模块定位

**本模块是NPC活人感模块的子模块（编号06），为NPC提供结构化认知加工能力。**

六个核心职责：
1. **思考** — NPC如何从记忆中产生关联链、从经历中归纳MentalModel
2. **消化** — NPC如何将外来信念内化为"自己的"信念
3. **浮现** — 思考如何在NPC的外在行为中可见（自语/书写/表情/暂停）
4. **正则化** — 睡眠期间的认知维护（记忆巩固/情绪复位/过度拟合校正）
5. **创新** — 跨界结构类比→新MentalModel→验证→形式化为领域产物
6. **传递** — 通过已有通道（对话/书籍/师徒/观察）将智慧传给其他NPC

### 1.2 本模块不是什么

- ❌ 不是"人工大脑"——不取代已存在的决策引擎/情绪引擎/记忆系统/SelfNarrative
- ❌ 不是新crate——全部代码在NPC crate内部，零新trait定义
- ❌ 不是新调参旋钮——一切从已有维度派生（BigFive/wisdom/经历/文化参数）
- ❌ 不是脚本——所有认知行为从底层参数的数学组合中涌现
- ❌ 不是LLM——思考文本使用确定性片段组合模型，不对接LLM增强层

### 1.3 设计原则

| 原则 | 含义 | 对标已有模式 |
|------|------|------------|
| **基于模式的向前投射** ★ v1.2 | NPC 心智的唯一原料是已编码经验。评估未发生路径时——查询已有 MentalModel 产生基于模式的向前投射。投射是单查询的（O(log n)，零 World 引用），输出带不确定性（variance/confidence）。与"世界仿真"的区分不在方向而在机制——模式匹配 vs 因果链仿真。附带三问判定标准——详见 [[003-MentalModel与智慧积累#十六-判定标准|003 §十六 判定标准]] | `PatternExpression` 从 EventMemory 归纳。deliberation 是向前投射——模式应用，不是世界仿真 |
| **零新trait** | 所有新增代码在NPC crate内部；跨模块通过已有enum variant扩展或已有API调用 | `HasAestheticSignal` （CHG-029）——各领域crate各自impl |
| **零新调参旋钮** | 不给设计者暴露"确认偏误强度""消化速度"等参数。一切从BigFive/wisdom/经历/文化参数派生 | `NeedSensitivity` 从BigFive派生 |
| **纯函数优先** | CognitiveBiases, EmbodiedCognition, CognitiveNorms, generalization_quality全部是纯函数，惰性计算，零存储 | `Physiology` 从`Vitals` 派生 |
| **惰性求值** | 不主动轮询。思考触发被其他系统的正常事件携带。文本生成仅在玩家15m内 | `PerceptBatch` 统一产出 |
| **可见行为涌现** | 不弹认知面板、不显示UI标签。玩家感知到的是行为变化：皱眉、自语、停手、写字、说"我觉得..." | `NonVerbalSignal` 非语言通道 |
| **对标已有模式** | 每个新概念都有对标概念：CognitiveStyle↔AestheticTaste, CognitiveTide↔Physiology, MentalModel↔Knowledge | 见具体章节 |
| **涌现领域** ★ v1.1 | 学科不来自枚举。NPC 群体的 MentalModel 在 DomainSignature 空间聚类 → 学科涌现。设计者只定义原子类型 | `DomainSignature` 替代 `MentalModelDomain` |

## 二、NPC认知体系全景图

### 2.1 五层认知架构

```
Layer 0: 认知基础（已有系统——本模块不修改）
  ├─ 记忆系统 → 事件记忆/情绪编码/衰减/重建过滤器 → 提供检索
  ├─ 情绪引擎 → PAD三轴/33复合情绪/MoodState → 提供情绪调制
  ├─ 自我叙事 → self_concept/life_chapters/aspirations → 提供价值观与目标
  ├─ 知识 → ~64条个人知识 → 提供事实基础
  └─ 感官管线 → PerceptEntry/VisualScene/AuditoryScene → 提供触发原料

Layer 1: 认知风格（新增——表征NPC"如何思考"）
  ├─ CognitiveStyle — 4维认知风格（直觉-分析/冲动-反思/具象-抽象/顽固-灵活）
  ├─ CognitiveTide — 3维认知潮汐（负载/反刍压力/安静度）
  ├─ CognitiveBiases — 7种认知偏误（完全从已有数据惰性派生）
  ├─ EmbodiedCognition — 身体状态→认知调制（对标Physiology模式）
  └─ CognitiveNorms — 文化→认知风格（对标CommunicationNorms模式）

Layer 2: MentalModel（新增——表征NPC"相信什么"）
  ├─ 归纳 — try_induce_pattern：从记忆中提取共现模式
  ├─ 微消化 — micro_digest：每记忆编码时用贝叶斯更新置信度
  ├─ 宏消化 — macro_digest：每7天source转移/融合/重构/边界扩展
  ├─ 种子继承 — 对标ChildFaithProfile：从父母/聚落/文化获取初始模型
  ├─ 创造性飞跃 — creative_leap：跨界结构模板匹配
  ├─ 跨代传递 — 走已有通道（WisdomSharing/书籍/师徒/观察/ThoughtImprint）
  └─ Shareability — 模型的传播边界（Public/InGroup/SpecificTarget/Private）

Layer 3: 思考涌现（新增——表征NPC"在想什么"）
  ├─ ThoughtTrigger — 6类触发（感知关联/地点/情绪/失调/空闲漫游/Zeigarnik）
  ├─ ThoughtFragment — 3级清晰度（模糊感觉→半成形→清晰命题）
  ├─ SurfacingModality — 6种浮现方式（内心/微表情/自语/暂停/书写/说话）
  └─ 对话微思考 — micro_think_during_dialogue：听着→脑中加工→回应

Layer 4: 认知维护与创新（新增——表征NPC心智的"后台任务"）
  ├─ 睡眠认知加工 — 经验窄度→自适应正则化→记忆巩固/情绪复位/模型重整/梦境
  ├─ 创新管线 — CreativeLeap→验证→形式化→领域系统存储→传播
  ├─ Theory of Mind — MindAttribution（NPC关于其他NPC心智的归因）
  ├─ Meta-Cognition — 反思性自指：反思型NPC能"反思自己的反思"
  ├─ CognitiveDistress — 认知危机指数（完全派生）
  └─ CognitiveExceptionality — 天才涌现（罕见参数组合）
```

### 2.2 从已有维度到认知产出的完整派生链

```

MentalAttributes.wisdom ─┤
Life Event Count ────────┼──→ CognitiveStyle (4f32) ──→ CognitiveBiases (7f32)

                                                        ↓

                                                    ThoughtFragment 清晰度

          所有向下箭头最终汇入：
          ↓
  1. 决策引擎（行为变化）
  2. Godot渲染（可见表情/自语/暂停）
```

## 三、核心数据结构概览

### 3.1 NpcData新增字段

```rust
// 对标已有：Knowledge (~2.5KB), AestheticFrameworks (~2KB), SelfNarrative (~1.5KB)
// 认知数据总计：<4KB/L1 NPC

pub struct NpcData {
    // === 已有字段（略）===
    
    // === NPC认知 v1.0 新增 ===
    
    /// 认知风格——NPC"如何思考"
    /// 对标 AestheticTaste（青春期派生→年度成熟）
    pub cognitive_style: CognitiveStyle,            // 16B
    
    /// 认知潮汐——心智此刻的状态
    /// 对标 Physiology（从已有数据每决策周期派生）
    pub cognitive_tide: CognitiveTide,              // 16B
    
    /// 世界运作模型——NPC持有的信念
    /// 对标 Knowledge（~64条个人知识）
    /// 上限20条，约1.6KB
    pub mental_models: ArrayVec<MentalModel, 20>,
    
    /// 他人心智归因——Theory of Mind
    /// 对标 MentalModel
    /// 上限16条，约960B
    pub mind_attributions: ArrayVec<MindAttribution, 16>,
    
    /// 信念演变里程碑——回顾性叙事的基础
    /// 对标 SelfNarrative.life_chapters
    /// 上限16条，约768B
    pub belief_history: ArrayVec<BeliefMilestone, 16>,
    
    /// 认知压力——0=健康，1=崩溃边缘
    /// 对标 self_narrative.stagnation_sense
    pub cognitive_distress: f32,                    // 4B
}
```

## 四、执行时机与调用顺序

### 4.1 每决策周期（L1 NPC, ~0.1-0.3s）

```
1. 感官管线（已有）：perceive() → PerceptBatch
2. 情绪引擎（已有）：emotion_step()
3. CognitiveTide（新增）：update_tide()
   └─ 3个f32更新，<6次浮点乘加，<0.001ms
4. 思考背景层（新增·惰性）：
   ├─ check_perceptual_trigger()    ← 附着在PerceptEntry处理中
   ├─ check_location_trigger()       ← 附着在位置更新中
   ├─ check_emotional_trigger()      ← 附着在emotion_step()后
   ├─ check_dissonance_trigger()     ← 附着在信息接收中
   └─ check_wander_trigger()         ← 概率触发，仅在cognitive_load<0.4时
5. 思考浮现（新增·仅玩家15m内~5 NPC）：
   └─ generate_thought_fragment() → ThoughtFragment + SurfacingType
6. Intent候选池排序（新增薄调度层）：
   ├─ GOAP（生存硬约束）
   ├─ MentalModelTest（主动验证信念）
   ├─ Aspiration（长期目标→今日行动）
   ├─ ThoughtSurfacing（书写/说话冲动）
   └─ 概率权重链（日常行为~85%）
7. 选中Intent → 执行行动
8. 行动结果 → micro_digest()（如有新记忆编码）
```

### 4.2 每游戏日（L1+L2 NPC）

```
1. MentalModel微消化：对所有未消化的外来模型
2. cognitive_distress更新（完全派生）
3. CognitiveStyle阻尼更新（如果距上次>30天）
4. 技能→MentalModel关联更新（如果技能等级变化>5）
5. SettlementWisdomAggregate惰性重算（仅dirty settlement）
```

### 4.3 每7游戏日（L1 NPC, 随SelfNarrative::reflect()触发）

```
1. SelfNarrative::reflect()（已有）
2.   ├─ meta_cognitive_review()（新增·嵌入）
3.   └─ macro_digest()（新增·嵌入）
4.     ├─ source转移（消化深度>0.8）
5.     ├─ 模型融合（domain相同+结构相似）
6.     ├─ 边界扩展（高openness对高confidence模型附加条件）
7.     ├─ 放弃（confidence<0.1）
8.     └─ WorldviewShift→SelfNarrative
9. CognitiveStyle重派生（含阻尼）
```

### 4.4 睡眠结束时

```
1. sleep_cognitive_processing()（新增·一次性运行）
   ├─ 计算经验窄度→确定正则化需求
   ├─ 记忆三阶段处理（标签→巩固→筛选）
   ├─ 情绪复位（睡眠质量×回归速度）
   ├─ MentalModel宏观重组（深睡期跨界关联）
   ├─ 梦境生成（如果quality>0.3）
   │   ├─ 荒诞度 = f(经验窄度, 开放性, 年龄)
   │   ├─ 存储确定性seed
   │   └─ 惰性文本生成（仅在被询问时）
   ├─ 睡眠洞察→新MentalModel
   └─ CognitiveTide次日重置
```

### 4.5 长期/事件驱动

| 触发条件 | 操作 |
|---------|------|
| Apprenticeship > 365游戏日 | DeepMentorship升级 |
| NPC死亡 | 可能遗留ThoughtImprint（高confidence+高emotional_charge模型） |
| NPC阅读书籍 | 创建ReadingSession→assess_and_integrate() |
| 群体对话（WisdomSharing） | 群体认知处理：SocialField×MentalModel |
| 创伤事件（impact>0.95） | 可能触发CognitiveBreak前驱条件 |
| 创新形式化 | confidence>0.7+supporting>3→formalize_innovation() |

## 五、LOD差异化策略

| 层 | 模拟频率 | 运行内容 | 存储内容 | 思考可见性 |
|----|---------|---------|---------|-----------|
| **L1** | 每决策周期 | 全部认知功能 | 全部字段（<4KB） | 全可见（ThoughtFragment生成） |
| **L2** | 每30s~分钟 | CognitiveTide + 低频MentalModel归纳 | 全部字段（不删除） | 不生成（无player观察） |
| **L2→L1 promote** | — | 补算消化管线 + 思考浮现 | — | 从L2快照唤醒 |
| **L3** | 统计抽象 | 不跑个体 | 不存个体 | 仅通过书籍/流言/结晶进入L1 |
| **L4** | 纯统计 | 不跑个体 | 不存个体 | 仅通过宏观统计影响世界 |

**L3/L4"幽灵思考"**：L3/L4 NPC个体不存在，但其代表性NPC的思考积淀通过书籍（历史系统）、流言（信息传播）、MentalModel结晶（SettlementWisdomAggregate）进入L1世界。对标P9历史模拟引擎的稀疏代表性NPC抽样。
