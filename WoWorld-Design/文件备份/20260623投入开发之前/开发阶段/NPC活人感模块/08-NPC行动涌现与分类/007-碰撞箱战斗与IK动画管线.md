## §一、设计原则

**动画是物理的输出，不是物理的输入。** — 不做任何预设战斗动画。所有攻击/格挡/闪避/受伤动画由碰撞检测+IK解算自动生成。

**碰撞扫掠驱动命中判定。** — 武器碰撞体沿轨迹扫掠→与目标身体碰撞体求交→命中部位+角度+穿透→伤害。不做 O(1)查表，做物理几何。

### 2.2 武器碰撞体

从 ItemParams（装配树组件的连续参数）在**持有实例化时**派生，缓存在 GripHandle 中：

| 武器类型 | 碰撞体形状 | 从哪些 ItemParams 派生 |
|---------|-----------|---------------------|
| 直剑 (Rigid) | 刃部 capsule(length, radius=thickness/2) + 尖端 sphere | StrikingHead.length, width, thickness |
| 弯刀 (Rigid) | 弯曲 capsule（曲率=curvature param） | StrikingHead.length, curvature |
| 锤 (Rigid) | 头部 sphere(radius) 或 capsule(radius, length) | StrikingHead.diameter, length |
| 矛 (Rigid) | 矛头 small capsule + 矛杆 long thin capsule | 两个组件的尺寸参数 |
| 匕首 (Rigid) | short capsule | StrikingHead.length, thickness |
| 空手 | fist sphere(r=0.03m) | 固定 |
| 连枷 (Chain) | 柄 capsule + N链段 micro-capsules + 锤头 sphere | ChainLink.links, link_length, StrikingHead |
| 鞭 (Flexible) | N micro-capsules taper from root→tip | stiffness, taper_ratio, length |
| 双节棍 (Hinge) | 两段 capsules + hinge constraint | 两段尺寸, limits_deg |

### 2.3 格挡碰撞体

```
盾牌: 大 capsule 或 oriented box, 覆盖角 ~120° 正前方
武器格挡: 武器碰撞体本身在防御姿态下成为格挡判定
  格挡窗口: 0.1-0.2s（精准弹反窗口）
  判定: 攻击武器碰撞体 ∩ 防御武器/盾牌碰撞体 = 格挡成功
```

## §三、轨迹扫掠算法

### 3.1 算法

```
输入: 武器碰撞体(capsule), 轨迹曲线(Linear/Arc/Spline), 起止时间, 扫掠步数(4-8)

for each timestep in [0, steps):
    t = timestep / steps
    weapon_pos = trajectory.evaluate(t)
    weapon_capsule = transform(weapon_template, weapon_pos, weapon_orientation)
    
    for each target_body_capsule in nearby_targets:
        if intersect(weapon_capsule, target_body_capsule):
            hit_time = t
            hit_body_part = target_body_capsule.bone_name
            hit_angle = angle(weapon_velocity, body_surface_normal)
            penetration = compute_penetration(force, weapon_sharpness, armor_hardness, hit_angle)
            return HitResult { hit_time, hit_body_part, hit_angle, penetration, impulse }
    
    // 无命中 → 继续下一步

return Miss
```

### 3.3 多目标扫掠

```
1v1: 4步 × 15身体胶囊 = 60 tests = 300ns
1v3 (围攻): 4步 × 45胶囊 = 180 tests = 900ns ≈ 0.9μs
100 NPC 战斗: 30攻击者 × 2目标 × 60 tests = ~18μs → 可忽略
```

### 3.4 轨迹类型

| 轨迹 | 曲线函数 | 适用基元 | 扫掠步数 |
|------|---------|---------|---------|
| Linear | P(t) = start + (end - start) × t | 刺、点 | 4 |
| Arc | P(t) = start × cos + axis × sin + ... | 斩、劈、撩、扫 | 6 |
| LinearThenArc | 前半 Linear, 后半 Arc | 佯攻 | 6 |
| Spline | Catmull-Rom 控制点 | 缠、引、化 | 8 |

## §四、命中判定与部位伤害

### 4.1 命中计算

```
命中角度 → 伤害类型:
  angle < 30° (接近垂直): 穿刺/切割 (剑尖, 刀刃)
  angle 30-60°: 混合
  angle > 60° (接近平行): 钝击 (刀背, 锤面)

穿透深度 = f(force, weapon_sharpness, armor_hardness, angle):
  weapon_sharpness = 1.0 / (weapon_edge_radius + 0.01) // 越薄越锋利
  armor_resistance = armor_hardness × armor_thickness × armor_coverage
  penetration = max(0, force × weapon_sharpness / armor_resistance - min_penetration)
```

### 4.2 部位伤害映射

与战斗系统 012 的渐进三档模型一致:

```
命中部位 → 伤害效果:
  Head:     暴击 ×2.0, 可能眩晕
  Neck:     致命 ×3.0
  Chest:    标准 ×1.0, 可能失衡
  Arm:      攻击力下降 (轻-3%→重-15%)
  Leg:      移动速度下降 (轻-3%→重-15%), 闪避成功率下降
  Hand:     握持力下降, GRASP稳定性↓
```

**不是二值"该肢不可用"——是连续三档渐进。** 与 CHG-013 契约一致。

## §五、IK 动画管线

### 5.1 完整链路

```
┌──────────────────────────────────────────────────────────────┐
│ 输入: STRIKE 轨迹关键帧 (position + rotation at each timestep)│
│      + AgentSnapshot (身体参数)                               │
│      + CombatStyleParams (6维风格)                            │
│      + 防御者碰撞响应 (如果被格挡/闪避, 轨迹偏移/提前终止)    │
└──────────────────────┬───────────────────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────────────────┐
│ Hand IK (Layer 6):                                           │
│   目标: 手的位置 = 武器握持位置                                │
│   方法: 2-bone analytical IK                                  │
│   → Wrist position = weapon grip position                     │
│   → Elbow position = solve(wrist_target, shoulder, lengths)  │
│   → Shoulder position = from spine top                        │
└──────────────────────┬───────────────────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────────────────┐
│ Spine Bridge (Layer 7):                                       │
│   肩旋转量 → 分配到 7 节脊椎 → 每节 slerp 一点旋转            │
│   distribute(hip_rotation, shoulder_rotation, spine_bones)    │
└──────────────────────┬───────────────────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────────────────┐
│ Hip/重心调制:                                                 │
│   CombatStyleParams.body_commitment × strike_force × weapon_weight │
│   → COM 前移量 (0=仅手臂动, 1=全身压上)                        │

                       ▼
```

### 5.2 2-Bone Analytical IK（数学细节）

```rust
fn solve_2bone_ik(shoulder: Vec3, target: Vec3, l1: f32, l2: f32) -> (Quat, Quat) {
    let d = distance(shoulder, target);
    let d_clamped = d.clamp((l1 - l2).abs(), l1 + l2);
    
    // 余弦定理
    let elbow_angle = acos((l1*l1 + l2*l2 - d_clamped*d_clamped) / (2.0 * l1 * l2));
    let base_angle = atan2(target.y - shoulder.y, target.x - shoulder.x);
    let offset = acos((l1*l1 + d_clamped*d_clamped - l2*l2) / (2.0 * l1 * d_clamped));
    let shoulder_angle = base_angle - offset; // 肘的朝向决定 +/-
    
    let shoulder_quat = Quat::from_axis_angle(rotation_axis, shoulder_angle);
    let elbow_quat = Quat::from_axis_angle(rotation_axis, elbow_angle);
    
    (shoulder_quat, elbow_quat)
}
// ~70 FMA 指令, ~17.5ns per limb on 4GHz CPU
```

### 5.3 IK vs 预设动画 — 性能对比

| 动画方法 | 每骨骼成本 | 33骨骼总成本 |
|---------|-----------|------------|
| 预设动画 (解压+9层混合+插值) | ~180ns | ~6μs |
| 2-bone Analytical IK | ~17.5ns/limb | ~0.67μs (双臂+双腿+脊柱+头) |
| **结论** | **IK 比预设动画快 ~9倍** | |

**IK 不是额外的性能负担——它是骨骼计算预算的优化方案。**

## §六、非 Rigid 武器物理

### 6.1 链武器 (Chain JointType)

使用 Position-Based Dynamics (PBD) 迭代约束求解:

```
连枷 (links=6, link_length=15mm):

每帧:
  1. 柄端 = hand IK 位置 (固定)
  2. 链段 1-6: 独立微刚体, 质量 = link_length × material.density
  3. PBD 约束:
     - 距离约束: |pos[i] - pos[i-1]| = link_length
     - 碰撞约束: 链段不与自身/NPC身体穿插
  4. 3-5 次 Gauss-Seidel 迭代 (通常 3 次即收敛)
  5. 锤头段: 最大的 sphere 碰撞体
     → 锤头速度 = Σ(链段角速度 × 半径) → 连枷的真正威力

每次迭代: 6 distance + 7 collision = 13 constraint solves
每次 solve: ~10ns (Gauss-Seidel)
3 iterations: ~0.4μs
加上锤头扫掠: ~0.3μs → 总计 ~0.7μs
```

### 6.2 鞭武器 (Flexible JointType)

```
鞭 (20 segments, stiffness=0.1, taper=0.05):

  1. 根段 = hand IK 位置
  2. B样条曲线控制点 → 弯曲度由 stiffness 决定
  3. 沿长度采样 N 个微胶囊体 (radius 从 0.01 递减到 0.003)
  4. PBD: 距离约束 + 弯曲约束 + 碰撞约束
  5. 3 iterations × ~610ns = ~1.8μs + 0.7μs 扫掠 = ~2.5μs

鞭的"卷/锁/绞":
  当鞭段碰撞体包络目标肢体:
    → 鞭段数量 × 包络弧度 → "卷"的约束数
    → 鞭段穿过缝隙+反向拉紧 → "锁"的力矩
    → 鞭段围绕颈部 → "绞"的窒息判定
  这些不是预设动作——是碰撞体几何 × 约束求解的自然结果
```

### 6.3 动态降级策略

```rust
fn adjust_pbd_iterations(active_chain_weapon_count: usize) -> u32 {
    if active_chain_weapon_count <= 50 { 3 }       // 正常精度
    else if active_chain_weapon_count <= 200 { 1 }  // 降级
```
