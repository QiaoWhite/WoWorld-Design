> **版本**: v1.0

---

## 核心定位

小精灵是玩家的可选辅助伙伴。定位类似《原神》的派蒙——但只有玩家角色可以感知到她。可被随时关闭。

---

## 关键设计决策

- **仅玩家可见** — NPC 完全感知不到她的存在
- **可随时消失/出现** — 3D 模型存在于空间场景中，但可立即隐身，以纯"声音"形式交流
- **3D 模型** — 小型飞行实体。手搓模型（造物）
- **熟知世界基本运作机制** — 但不了解此存档内的具体内容（人物/事件/地形）
- **可读大日志** — 能检索玩家已记录的信息
- **LLM 可选增强** — 和 NPC 接入 LLM 同机制。非 LLM 模式用预定义模板

## 行为逻辑

```
玩家状态感知 → 触发条件检测 → 行为选择 (LLM/模板) → 对话/提示输出
    ↑                                                   ↓
    └── 大日志查询 (已记录信息) ←── 语言表达 ContentResolver
```

## 性能

| 项目 | 数值 |
|------|------|
| 非 LLM 模式 | <0.01ms/帧——纯模板匹配 |
| LLM 模式 | <100ms/请求——异步，不阻塞帧 |
| 内存 | <1MB (模板+状态) |
| 模型 | 手搓低多边形 ~500 tris——GPU 可忽略 |

## 玩家体验设计

- **首次接触**: 角色创建完成后——小精灵出现，简短自我介绍 (非 LLM 模板)
- **新手引导**: 分阶段触发——第一次采集/战斗/交易/死亡时提供上下文提示
- **情感陪伴**: 读表情+身体状态→产生关切反应 (LLM模式)。非 LLM 模式使用预定义模板库
- **大日志联动**: "你之前在XX见过类似的花"、"这是你第N次死亡"
- **可关闭**: 玩家随时可在设置中完全关闭——零 gameplay 影响
