## 二、模块间调用协议

### 2.1 NPC决策循环 → 感官系统

```
NPC crate 每决策周期调用一次 SensoryProvider::perceive()。
传入: NpcData.sensory + 位置朝向 + 基础设施trait引用 + 注意状态 + 调制器 + goal_relevance闭包
返回: PerceptBatch → 更新 PerceptualCache → 传给 prepare_facts() 或 combat_intelligence.assess()
```

### 2.2 大日志 → 感官系统

```
历史 005 在 tick_journal() 中调用 SensoryProvider::perceive_lightweight()。
轻量版不做: 注意筛选、噪声注入、内感受、暗适应
轻量版只做: 实体发现 + 分类 + 场景上下文
```

### 2.3 战斗情报层 → PerceptBatch

```
战斗 004 combat_intelligence.assess(percept_batch, npc, memory) → CombatAssessment
HpEstimate: 从 PerceptEntry.body_state 特征+combat_experience派生
threat_level: 从武器特征+距离+已知战斗风格派生
observed_style: 从连续观察的攻击模式派生
```

### 2.4 审美系统 → VisualScene

```
审美系统从 PerceptBatch.visual_scene 派生 AestheticSignal（6维）:
    fluency:     sky_exposure × chromaticity_gradient_smoothness
    novelty:     1.0 - familiarity (from memory)
    complexity:  depth_layers × entity_density
    harmony:     complementary_colors × golden_ratio_composition
    expressiveness: light_direction_dramatic × spectacle_present
    virtuosity:  0.0（自然景观）

然后: aesthetic_system.judge(signal, npc.aesthetic_taste, active_framework) → AestheticJudgment
```

### 2.5 状态效果 → PerceptualModifiers

```
NPC crate 在决策循环前:
    mods = PerceptualModifiers::default()
    for status in npc.status_effects:
        let impact = status.perceptual_impact()  // NPC/life crate 定义
        mods.vision_range_mult *= impact.vision_range_mult
        mods.noise_mult *= impact.noise_mult
        // ...
    传入感官管线
```

### 2.6 认知管线 → EnrichedPerceptBatch ★ v1.1 (CHG-057)

```
认知系统在决策循环的 prepare_facts() 之前调用 enrich():

    // 1. 获得原始知觉
    let batch = sensory_provider.perceive(/* ... */);

    // 2. 富化 —— 纯函数，零状态
    let enriched = batch.enrich(
        &npc.sensory.perceptual_cache,
        npc.position,  // observer position for gaze estimates
    );

    // 3. 分发富化结果到各消费者
    // 认知系统: enriched.emotional_readings → 情绪情境推理
    // 认知系统: enriched.gaze_estimates → Theory of Mind 推理
    // 认知系统: enriched.bound_entities → 多模态共现推理
    // 战斗: enriched.emotional_readings → 士气评估
    // 经济: crowd_emotional_field → 市场情绪判断
    // 权力: enriched.gaze_estimates + emotional_readings → 权力符号感知
```

**性能**: `enrich()` 每决策周期每 L1 NPC < 0.01ms。1000 L1 → < 0.1ms/frame。在帧预算的噪声层。

## 三、SpatialEntity 写入协议

各模块创建/更新实体时→写入世界空间管理的 SpatialIndex：

| 模块 | 写入的 SpatialEntity | 关键字段 |
|------|---------------------|---------|
| Life | 所有生物/NPC | entity_kind=Creature, velocity, center |
| Items | 掉落/放置的物品 | entity_kind=Item, center |
| World Gen | 建筑/地形特征 | entity_kind=Building/Terrain |
| Vehicles | 载具 | entity_kind=Vehicle, velocity |
| Magic | 法术效果（火球/冰墙/屏障） | entity_kind=Effect, light_emission |
| Combat | 投射物 | entity_kind=Effect, velocity |

**NPC/玩家死亡时不注销 SpatialEntity——velocity→0, entity_kind保持Creature。尸体处理由生命模块负责。**

## 四、SpatialEvent 写入协议

任何模块产生显著物理事件→写入空间 Chunk 的事件 buffer：

```rust
spatial.push_event(SpatialEvent {
    event_type: SpatialEventType::LoudSound { peak_db: 95.0, frequency_profile: ... },
    position: explosion_center,
    intensity: 0.8,
    timestamp: now,
    source_entity: Some(bomb.id),
    duration: TimeDelta::from_millis(500),
    propagation_medium: Medium::Air,
});
```

事件自动过期（`max(intensity × 10s, 5s)`）。Chunk ring buffer（64条，LRU）。
