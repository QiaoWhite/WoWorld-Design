## 〇、核心定位

战争不是新模块——是已有系统在"大规模"上的并发运行。宣战（权力 GOAP）、动员（Compel 原子）、后勤（经济系统）、组织（Delegate 链）、交锋（战斗系统）、伤亡（生命模块）、记忆（历史模块）——全由已有模块覆盖。

**本文档只做一件事**：为大规模战场（数百至数万人）定义**方阵统计优化层 + 结晶/溶解算法**。这是性能优化设计，不是新模块。~200 行 Rust 设计规格。

---

## 一、背景——为什么不能全量模拟

现有 L1-L4 分层模拟可以覆盖日常战斗和中小规模冲突。但大规模战场（几千至几万人）中的问题：

- L1-L2 全战斗 AI 实例化每个士兵：CPU 成本不可接受（~0.5ms/个体 × 10000 = 5000ms）
- L4 统计层对日常 NPC 够用——但对战场来说太粗糙（方阵推进、士气崩塌是动态过程）
- 玩家在战场中——他身边的几十人需要全精度（L1），远处几千人只需要统计（L3），**两者之间的过渡需要无缝**

**解决**：在 L2（简化战斗 AI）和 L3（统计）之间插入**方阵统计层**。方阵 = 100-300 个士兵的空间聚合。方阵内部用统计（伤亡率/士气/方向），方阵边缘接触区域用详细计算。

## 二、方阵数据模型

### 2.1 结构

```rust
/// 一个方阵——约 100-300 个士兵的空间聚合
pub struct Phalanx {
    pub id: PhalanxId,

    /// 空间参数
    pub centroid: Vec3,            // 方阵质心（世界坐标）
    pub radius: f32,               // 方阵半径（m）
    pub direction: Vec3,           // 面对方向（单位向量）
    pub width: f32,                // 正面宽度（m）

    /// 构成
    pub soldier_count: u32,        // 当前士兵数
    pub original_count: u32,       // 初始士兵数
    pub formation: FormationType,
    pub composition: UnitComposition,  // 兵种构成

    /// 状态
    pub state: PhalanxState,       // 推进中/相持中/溃败中
    pub mean_hp: f32,              // 平均 HP (0-1, 归一化)
    pub mean_morale: f32,          // 平均士气 (0-1)
    pub morale_cohesion: f32,      // 士气凝聚度 (0-1) — 1=万众一心, 0=一盘散沙
    pub fatigue_level: f32,        // 疲劳程度 (0-1)

    /// 战斗参数
    pub attack_rating: f32,        // 攻击力评分 (装备 × 技能)
    pub defense_rating: f32,       // 防御力评分
    pub discipline: f32,           // 纪律性 (0-1) — 影响溃败阈值

    /// 指挥
    pub commander: Option<EntityId>,  // 百夫长/校尉
    pub command_chain_depth: u8,      // 到最高指挥官的距离

    /// 更新标记
    pub last_updated: GameTimestamp,
}
```

```rust
pub enum FormationType {
```

## 三、方阵 vs 方阵 交锋计算

### 3.1 推进速率

```rust
/// 每游戏分钟更新一次
fn phalanx_advance_rate(
    attacker: &Phalanx,
    defender: &Phalanx,
    terrain: &TerrainFactors,
    weather: &WeatherFactors,
) -> f32 {
    // 基础推进力
    let push = attacker.attack_rating * attacker.mean_morale
              * attacker.state.momentum_factor();

    // 阻力
    let resist = defender.defense_rating * defender.mean_morale
               * defender.formation.defensive_multiplier()
               * terrain.defensive_bonus();

    // 兵力比修正
    let ratio = (attacker.soldier_count as f32 / defender.soldier_count.max(1) as f32).min(2.0);

    // 推进速率 (m/分钟)
    let base_rate = (push - resist).max(-5.0).min(15.0) * ratio;
    base_rate * weather.movement_modifier()
}
```

### 3.2 伤亡速率

```rust
/// 双方在接触面上的伤亡速率
fn phalanx_casualty_rate(
    a: &Phalanx,
    b: &Phalanx,
    contact_width: f32,    // 接触面宽度（m）
) -> (f32, f32) {          // (a的伤亡/分钟, b的伤亡/分钟)
    let engagement_intensity = contact_width * 0.2; // 每米接触面 ~0.2人/分钟的基础交锋密度

    // A的伤亡 = B的攻击力 × 接触强度 ÷ A的防御
    let a_loss = b.attack_rating * engagement_intensity / a.defense_rating.max(0.1)
               * (1.0 - a.mean_morale * 0.3);  // 士气高的伤亡少

    let b_loss = a.attack_rating * engagement_intensity / b.defense_rating.max(0.1)
               * (1.0 - b.mean_morale * 0.3);

    (a_loss, b_loss)
}
```

### 3.3 士气变化

```rust
fn phalanx_morale_change(
    phalanx: &Phalanx,
    casualties_this_minute: f32,
    ally_morale_avg: f32,          // 周围友军平均士气（情绪感染）
    commander_charisma: f32,       // 指挥官魅力
    enemy_size_visible: u32,       // 可见敌军数量
) -> f32 {
    let mut delta = 0.0;

    // 伤亡冲击 — 伤亡率 > 5%/分钟 → 士气暴跌
    let casualty_rate = casualties_this_minute / phalanx.soldier_count.max(1) as f32;
    delta -= casualty_rate * 15.0;

    // 情绪感染 — 周围友军士气拉或压
    delta += (ally_morale_avg - phalanx.mean_morale) * 0.1;

    // 指挥官加成 — 高魅力指挥官稳定军心
    delta += commander_charisma * 0.05;

    // 兵力差距 — 看到敌军人多势众
    let ratio = enemy_size_visible as f32 / phalanx.soldier_count.max(1) as f32;
    if ratio > 2.0 {
        delta -= (ratio - 2.0) * 0.05;
    }

    // 纪律修正——高纪律减缓情绪波动
    delta *= (1.0 - phalanx.discipline * 0.5);

    delta.clamp(-0.3, 0.1) // 每分钟变化上限
}
```

### 3.4 溃败阈值

```rust
fn phalanx_rout_check(phalanx: &Phalanx) -> bool {
    // 多条件累积 → 方阵溃败
    let mut rout_score = 0.0;

    // 士气极低
    if phalanx.mean_morale < 0.15 { rout_score += 1.5; }
    else if phalanx.mean_morale < 0.3 { rout_score += 1.0; }
    else if phalanx.mean_morale < 0.5 { rout_score += 0.5; }

    // 伤亡过半
    let loss_ratio = 1.0 - phalanx.soldier_count as f32 / phalanx.original_count.max(1) as f32;
    rout_score += loss_ratio * 2.0;

    // 指挥官阵亡
    if phalanx.commander.is_none() && phalanx.command_chain_depth > 3 {
        rout_score += 1.0;
    }

    // 纪律对抗溃败
    rout_score *= (1.0 - phalanx.discipline);

    rout_score > 2.0
}
```

## 四、结晶化与溶解——L2↔L3 边界过渡

### 4.1 核心机制

玩家在战场中移动时，方阵的远缘在统计中消失、近缘结晶为实际战斗个体。这个过渡必须是**无缝的**——玩家不应该看到个体"突然出现"或"突然消失"。

```rust
/// 结晶化——方阵边缘进入 L2 范围时，个体从统计中实例化
fn crystallize(
    phalanx: &Phalanx,
    player_pos: Vec3,
    l2_range: f32,           // 500m
    l1_range: f32,           // 100m
) -> Vec<CombatantInstance> {
    let dist = (phalanx.centroid - player_pos).length();

    // 超出 L2 范围 → 完全统计，不实例化
    if dist > l2_range { return vec![]; }

    // 在 L2 范围内 → 实例化方阵中离玩家最近的部分
    let edge_section = phalanx.edge_facing(player_pos);

    // 在 L1 范围内 → 实例化全精度（全 Combat AI + 全 3D）
    // 在 L2 范围（100-500m）→ 实例化简化（仅本能层 AI + LOD1 模型）
    let precision = if dist < l1_range { InstancePrecision::Full }
                    else { InstancePrecision::Simplified };

    // 只实例化此方阵的 ~10-15% 士兵（离玩家最近的那部分）
    let instance_count = (phalanx.soldier_count as f32 * 0.12).min(50.0) as u32;

    (0..instance_count)
        .map(|i| CombatantInstance {
            position: phalanx.sample_position_in_edge(edge_section, i),
            phalanx_id: phalanx.id,
            precision,
            ..phalanx.sample_soldier(i)
        })
        .collect()
}

/// 溶解——方阵远离玩家时，个体回归统计
fn dissolve(
    instances: &[CombatantInstance],
    phalanx: &mut Phalanx,
) {
    // 取实例的平均 HP 和士气 → 更新方阵统计
    phalanx.mean_hp = instances.iter().map(|i| i.hp).sum::<f32>() / instances.len().max(1) as f32;
    phalanx.mean_morale = instances.iter().map(|i| i.morale).sum::<f32>() / instances.len().max(1) as f32;

    // 实例中的伤亡数 → 从方阵总数中扣除
    let survivors = instances.iter().filter(|i| i.hp > 0.0).count() as u32;
    let casualties = instances.len() as u32 - survivors;
    phalanx.soldier_count = phalanx.soldier_count.saturating_sub(casualties);

    // 溶解后的实例从世界移除
}
```

### 4.2 过渡频率

- 结晶化/溶解触发：玩家每移动 ~50m 或方阵每 L3 更新周期（游戏小时）
- 不是每帧——按需在 Rust L3 周期中异步完成
- 每次结晶化 < 0.5ms（生成 15-50 个实例）

## 五、四层分级总览

| 层级 | 范围 | 模拟精度 | 渲染 | 实例数 |
|------|------|---------|------|--------|
| **L1** | <100m | 全 Combat AI（本能+节奏+战略） | 全 3D + CSM 阴影 | ~20-50 |
| **L2** | 100-500m | 简化 Combat AI（仅本能层 + 粗糙物理） | LOD1 模型/Impostor | ~200-1000 |
| **L3** | 500m-3km | 方阵级统计（每游戏分钟更新） | 方阵贴片/粒子群 | ~数十方阵 |
| **L4** | >3km | 纯军队级统计（胜率/伤亡率） | 不渲染个体 | 不实例化 |

## 六、性能预算

```
L1 全 AI:   ~0.5ms × 50   = ~25ms → rayon 并行分摊到 8 核 = ~3ms
L2 简化 AI: ~0.01ms × 1000 = ~10ms → rayon 并行 = ~1.3ms
L3 方阵更新: 每游戏分钟 ~0.5ms（后台——不在帧预算内）
结晶化/溶解: 每次 ~0.5ms（异步）
─────────────────────────────
战场总 CPU: ~5ms（在 16.7ms 帧内，通过 rayon 并行）
```
