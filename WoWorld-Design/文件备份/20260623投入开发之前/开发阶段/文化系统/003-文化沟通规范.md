## 一、CommunicationNorms 所有权转移

### 1.1 转移说明

`CommunicationNorms` 结构体目前在 [[../语言表达/010-非语言表达与语言联动|语言表达 010]] 中定义。这是历史原因——语言表达模块先于文化系统设计，在文化系统缺失时临时承担了 norms 的定义职责。

**从 v1.0 起，所有权正式转移至文化系统**。变动范围：

| 项目 | 原位置 | 新位置 |
|------|--------|--------|
| `CommunicationNorms` 结构体定义 | [[../语言表达/010-非语言表达与语言联动|语言表达 010]] | 本文档 §二 |
| 各字段语义描述 | 语言表达 010 | 本文档 §二 |
| norms 的派生公式（从 CultureCoreParams 计算） | 不存在（此前 norms 被手动假设） | 本文档 §三 |
| `HonorificSystem` 结构体定义 | 语言表达 010 | 本文档 §四 |
| `GestureCultureMapping` | 语言表达 010 | 本文档 §五 |
| `TouchNorms` 类型体系 | 语言表达 010 | 本文档 §六 |
| norms 如何调制对话行为（抢话阈值/沉默处理/敬语生成/LLM提示） | 语言表达 010 | **保留在** 语言表达 010（消费逻辑——文化系统不拥有） |
| `dialogue_ctx.culture_norms` 字段 | 语言表达 005 | **保留在** 语言表达 005（依赖注入——文化系统不拥有） |

**语言表达模块的角色变化**：从 `CommunicationNorms` 的**定义者和拥有者** → **只读消费者**。语言表达通过 `CultureQuery` trait 获取 norms 值，并负责如何将 norms 应用于对话行为调制。

### 1.2 向后兼容

已有的 `DialogueContext` 依赖注入模式不变——调用者在创建 `DialogueContext` 时从 `CultureQuery` 获取 norms 并注入 `dialogue_ctx.culture_norms` 字段。语言表达模块仍然不反向查询文化系统。

## 二、CommunicationNorms 完整定义

### 2.1 数据结构

```rust
/// 文化沟通规范 —— 一个文化中人际交流的行为准则
/// 
/// 作为第一层派生参数——从 CultureCoreParams 计算，世界生成时缓存，
/// 随核心参数的代际漂移而缓慢更新。
/// 
/// 所有权：文化系统（v1.0 起）
/// 消费者：语言表达（对话行为调制）、NPC（社交行为决策）
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct CommunicationNorms {
    /// 打断容忍度 —— 在对方说话时插话的可接受程度
    /// 0.0 = 从不打断别人说话——打断是严重的冒犯
    /// 1.0 = 打断是热情参与的表现——不打断反而显得冷漠
    /// 
    /// 消费方：语言表达 TurnMode::FreeForm 的抢话阈值
    pub interruption_tolerance: f32,

    /// 视线接触规范 —— 对话中目光接触的规则
    /// 
    /// 消费方：NPC 社交行为（注视/回避行为）、语言表达 010 非语言信号合成
    pub eye_contact_norm: EyeContactNorm,

    /// 个人空间半径 —— 两个非亲密关系的人之间感到舒适的最近距离（米）
    /// 典型值：北欧 ~1.2m，地中海 ~0.5m，东亚 ~0.8m
    /// 
    /// 消费方：NPC 社交距离计算、有效听觉半径修正（文化因子）
    pub personal_space_radius_m: f32,

    /// 直接性 —— 沟通中是否直言不讳
    /// 0.0 = 极其委婉——批评需要通过隐喻和暗示表达
    /// 1.0 = 极其直接——有话直说，不拐弯抹角
    /// 
    /// 消费方：语言表达文本片段选择（direct/neutral/indirect 词库）、
    ///        LLM 提示词构建（"Be indirect. Never criticize directly."）
    pub directness: f32,

    /// 沉默容忍度 —— 对话中延长的沉默的社交意义
    /// 0.0 = 沉默是尴尬——需要尽快填补
    /// 1.0 = 沉默是深思——暂停是思考的标志，不是不适
    /// 
    /// 消费方：语言表达 TurnMode——沉默时长超过阈值时 NPC 是否主动发言填补
    pub silence_tolerance: f32,

    /// 情感表达度 —— 在公共场合表达情感的开放程度
    /// 0.0 = 克制——面瘫是稳重，"喜怒不形于色"是美德
    /// 1.0 = 奔放——情感写在脸上，掩藏情感被视为虚伪
    /// 
    /// 消费方：NPC 非语言信号合成（面部表情强度）、LLM 提示词、
    ///        语言表达文本片段选择（情感形容词数量）
    pub emotional_expressiveness: f32,

    /// 敬语系统 —— 该文化是否使用正式的敬语体系
    /// 
    /// 消费方：语言表达敬语文本生成、LLM 提示词
    pub honorifics: HonorificSystem,

    /// 触觉规范 —— 身体接触的社交规则
    /// 
    /// 消费方：NPC 非语言信号合成（触摸是否允许）、
    ///        跨文化误解检测（拍肩在某些文化是友好，在某些文化是冒犯）
    pub touch_norms: TouchNorms,
}

/// 视线接触规范
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
pub enum EyeContactNorm {
    /// 直视 = 尊重和关注。回避视线 = 不尊重或欺瞒。
    Direct,
    /// 直视 = 挑衅或不敬（特别是对上级）。回避视线 = 尊重。
    Averted,
    /// 对视规则取决于社会地位——上位者有权直视下位者，反之则失礼。
    StatusBased,
}

/// 敬语体系
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct HonorificSystem {
    /// 语言本身是否有语法敬语（如日语 desu/masu、韩语 합쇼체/해요체）
    pub has_grammatical_honorifics: bool,

    /// 是否基于年龄使用敬语——长辈必须用敬语称呼
    pub age_based: bool,

    /// 是否基于社会地位使用敬语——高地位者必须用敬语称呼
    pub status_based: bool,

    /// 亲密关系是否可以省略敬语——如果为 true，家人/密友间省略敬语
    pub intimacy_overrides: bool,
}

/// 触觉社交规范
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct TouchNorms {
    /// 握手：0=不允许，1=标准问候
    pub handshake: f32,

    /// 拥抱：0=仅最亲密关系，1=朋友间正常
    pub hug: f32,

    /// 异性间身体接触的限制：0=完全禁止，1=无限制
    pub opposite_gender: f32,

    /// 公共场合身体接触的接受度：0=不可接受，1=完全正常
    pub public: f32,

    /// 触摸作为支配姿态（拍肩表示地位更高）：0=不被接受，1=正常
    pub dominance: f32,
}
```

### 2.2 每字段的语义锚定点

| 字段 | 0.0 文化示例 | 0.5 文化示例 | 1.0 文化示例 |
|------|-------------|-------------|-------------|
| `interruption_tolerance` | 北欧式——等待对方完全说完，打断是粗鲁 | 美式——适度打断可接受 | 拉丁式——多人同时说话是热情的体现 |
| `eye_contact_norm` | Averted（东亚式——直视长辈是冒犯） | StatusBased（根据地位调节） | Direct（北欧式——直视=诚实） |
| `personal_space_radius_m` | 0.3m（近距离——地中海南美） | 0.8m | 1.5m（远距离——北欧/北美） |
| `directness` | 极度委婉——"也许我们可以考虑其他方案" | 中等 | 极度直接——"这是错的"。 |
| `silence_tolerance` | 沉默=尴尬必须填充（美式） | 中等 | 沉默=深思（芬兰式——"不说话不代表不开心"） |
| `emotional_expressiveness` | 高度克制（英式——"保持冷静"） | 中等 | 高度表达（意式/拉美——激情外露） |
| `honorifics` | 无敬语（英语——"you" 通用于所有人） | 部分敬语（法语 tu/vous） | 完全敬语系统（日语/韩语——多层敬语体系） |
| `touch_norms` | 极少接触（北欧——个人空间大） | 中度（英美——握手+偶尔轻拍） | 频繁接触（地中海/拉美——拥抱+触摸+近距站立） |

## 三、CommunicationNorms 的派生公式

每字段从 CultureCoreParams 通过确定性公式计算——不独立存储：

```rust
impl CommunicationNorms {
    /// 从核心文化参数派生沟通规范
    /// 
    /// 此为第一层派生——世界生成时计算一次并缓存。
    /// 后续仅在核心参数发生显著变化（|Δ| > 0.05）时重算。
    pub fn derive_from(core: &CultureCoreParams) -> Self {
        Self {
            interruption_tolerance: derive_interruption_tolerance(core),
            eye_contact_norm: derive_eye_contact_norm(core),
            personal_space_radius_m: derive_personal_space(core),
            directness: derive_directness(core),
            silence_tolerance: derive_silence_tolerance(core),
            emotional_expressiveness: derive_emotional_expressiveness(core),
            honorifics: derive_honorific_system(core),
            touch_norms: derive_touch_norms(core),
        }
    }
}

/// 打断容忍度 = f(竞争取向, 权力距离, 个人主义)
/// 
/// 高竞争 + 低个人主义 → "集体内部的热烈讨论"——打断容忍度高
/// 高权力距离 + 低个人主义 → "下不能犯上"——打断容忍度低
fn derive_interruption_tolerance(core: &CultureCoreParams) -> f32 {
    let raw = core.competition_orientation * 0.40
            + core.individualism * 0.30
            + (1.0 - core.power_distance) * 0.30;
    raw.clamp(0.0, 1.0)
}

/// 视线接触规范 = f(权力距离, 开放度, 直接性)
/// 
/// 高权力距离 → StatusBased（地位决定目光规则）
/// 低权力距离 + 高开放度 → Direct（人人平等=互视为尊重）
/// 低权力距离 + 低开放度 → Averted（不互视但也不是挑衅）
fn derive_eye_contact_norm(core: &CultureCoreParams) -> EyeContactNorm {
    if core.power_distance > 0.6 {
        EyeContactNorm::StatusBased
    } else if core.openness_to_outsiders > 0.5 {
        EyeContactNorm::Direct
    } else {
        EyeContactNorm::Averted
    }
}

/// 个人空间半径 = f(个人主义, 权力距离, 放纵度)
/// 
/// 高个人主义 → 个人空间大（个体边界强）
/// 高放纵度 → 个人空间小（亲密接触是正常的享受）
/// 高权力距离 → 个人空间因地位而异——但平均值中等
fn derive_personal_space(core: &CultureCoreParams) -> f32 {
    let raw = core.individualism * 0.40
            + (1.0 - core.indulgence) * 0.35
            + (1.0 - core.power_distance) * 0.25;
    // 映射到物理距离: 0.3m - 1.5m
    0.3 + raw * 1.2
}

/// 直接性 = f(个人主义, 权力距离, 不确定性规避)
/// 
/// 高个人主义 → 直接（"我说我的想法，不依赖群体暗示"）
/// 低权力距离 → 直接（"没有权威需要绕过"）
/// 高不确定性规避 → 间接（"不能直接说不——太粗暴了"）
fn derive_directness(core: &CultureCoreParams) -> f32 {
    let raw = core.individualism * 0.45
            + (1.0 - core.power_distance) * 0.30
            + (1.0 - core.uncertainty_avoidance) * 0.25;
    raw.clamp(0.0, 1.0)
}

/// 沉默容忍度 = f(不确定性规避, 长期取向, 个人主义)
/// 
/// 高不确定性规避 → 沉默容忍低（"不确定对方在沉默中的意思——尽快打破"）
/// 高长期取向 → 沉默容忍高（沉默=思考，思考=长远决策的一部分）
/// 高个人主义 → 沉默容忍较低（"沉默浪费了表达自我的时间"）
fn derive_silence_tolerance(core: &CultureCoreParams) -> f32 {
    let raw = (1.0 - core.uncertainty_avoidance) * 0.40
            + core.long_term_orientation * 0.35
            + (1.0 - core.individualism) * 0.25;
    raw.clamp(0.0, 1.0)
}

/// 情感表达度 = f(放纵度, 个人主义, 艺术性)
/// 
/// 高放纵度 → 情感自由表达（"享受生活——情感是生活的一部分"）
/// 高个人主义 → 情感表达（"表达自我是真实的"）
/// 高艺术性 → 情感表达（"情感是艺术的来源"）
fn derive_emotional_expressiveness(core: &CultureCoreParams) -> f32 {
    let raw = core.indulgence * 0.50
            + core.individualism * 0.25
            + core.artistry * 0.25;
    raw.clamp(0.0, 1.0)
}

/// 敬语系统 = f(权力距离, 长期取向, 不确定性规避)
/// 
/// 高权力距离 → 必须通过语言标记等级——敬语的核心驱动力
/// 高长期取向 → 传统语法结构更倾向于被保留——敬语更可能存续
/// 高不确定性规避 → 固定规则的社会礼仪——敬语提供了"安全"的社交脚本
fn derive_honorific_system(core: &CultureCoreParams) -> HonorificSystem {
    let has_grammatical = core.power_distance > 0.4;
    HonorificSystem {
        has_grammatical_honorifics: has_grammatical,
        age_based: core.power_distance > 0.3 || core.long_term_orientation > 0.5,
        status_based: core.power_distance > 0.25,
        intimacy_overrides: core.individualism > 0.3,
    }
}

/// 触觉规范 = f(个人主义, 放纵度, 权力距离, 开放度)
fn derive_touch_norms(core: &CultureCoreParams) -> TouchNorms {
    // 身体接触的文化接受度 = 低个人主义（集体文化更接受接触）
    //                       + 高放纵度（享受包括身体接触）
    let touch_acceptance = (1.0 - core.individualism) * 0.40
                         + core.indulgence * 0.30
                         + core.openness_to_outsiders * 0.20
                         + (1.0 - core.power_distance) * 0.10;
    
    TouchNorms {
        handshake: (touch_acceptance * 1.2).clamp(0.0, 1.0),
        hug: ((touch_acceptance - 0.3) * 1.5).clamp(0.0, 1.0),
        opposite_gender: if core.power_distance > 0.7 { 0.1 } else { touch_acceptance },
        public: touch_acceptance,
        dominance: if core.power_distance > 0.5 { 0.6 } else { 0.2 },
    }
}
```

## 四、HonorificSystem 的独立消费

虽然 `HonorificSystem` 是 `CommunicationNorms` 的字段，但它被语言表达系统高频独立使用。`CultureQuery` trait 提供直接访问方法（避免每次从完整 `CommunicationNorms` 中提取）：

```rust
// CultureQuery trait 中
fn honorific_system(&self, id: CultureId) -> Option<&HonorificSystem>;
```

语言表达 `005` 中的敬语生成逻辑使用此接口：

```rust
// 语言表达 005 对话系统中——每对话回合调用
fn select_address_form(
    speaker: &NpcData,
    listener: &NpcData,
    culture: &dyn CultureQuery,
) -> AddressForm {
    let honorifics = culture.honorific_system(speaker.culture_id).unwrap();
    
    if !honorifics.has_grammatical_honorifics {
        return AddressForm::Neutral; // 无敬语系统→无需选择
    }
    
    // 亲密关系覆盖
    if honorifics.intimacy_overrides && speaker.relationship_to(listener).intimacy > 0.7 {
        return AddressForm::Intimate;
    }
    
    // 年龄规则
    if honorifics.age_based && listener.age > speaker.age + 15 {
        return AddressForm::Honorific;
    }
    
    // 地位规则
    if honorifics.status_based && listener.social_status() > speaker.social_status() + 10 {
        return AddressForm::Honorific;
    }
    
    AddressForm::Casual
}
```

## 五、GestureCultureMapping

### 5.1 数据结构

```rust
/// 跨文化手势含义映射
/// 
/// 同一手势在不同文化中有不同的含义。
/// 例如：点头在北欧="同意"，在部分中东文化="我在听（不一定同意）"。
/// 
/// 所有权转移至文化系统（与 CommunicationNorms 同步转移）
pub struct GestureCultureMapping {
    /// 外键：GestureId → (CultureId → GestureMeaning)
    /// 
    /// 两重 HashMap 在空间上可能较大——优化：
    ///   - 只有"有文化特定含义"的手势才在此存储（不同文化的 default 含义不存在此）
    ///   - 只有一个文化中存在的手势使用单键存储
    pub meanings: FxHashMap<GestureId, FxHashMap<CultureId, GestureMeaning>>,
}

/// 某个文化中特定手势的含义
pub struct GestureMeaning {
    /// 含义描述（用于 LLM 提示和文本生成）
    pub interpretation: String,
    /// 情感效价——正向或负向
    /// -1.0 = 严重冒犯, 0.0 = 中性, 1.0 = 高度赞赏
    pub valence: f32,
    /// 适用于哪个形式层级——正式场合使用的手势可能与日常不同
    pub formality_appropriate: FormalityLevel,
}
```

### 5.2 手势含义的生成

手势含义不像 CommunicationNorms——不是从核心参数派生的。它们是**从文化种子随机生成的**——与语言分配同属于"种子驱动的随机文化特征"：

```rust
fn generate_gesture_meanings(
    seed: u64,
    cultures: &[CultureId],
    gesture_defs: &[GestureDef],  // 所有可能的手势列表
) -> GestureCultureMapping {
    let mut mapping = GestureCultureMapping::default();
    
    for gesture in gesture_defs {
        for &culture_id in cultures {
            let gesture_seed = hash_2(seed, "gesture", 
                (gesture.id.0 as u64) << 32 | culture_id.0 as u64);
            let mut rng = StdRng::seed_from_u64(gesture_seed);
            
            // 手势要么有"默认通用含义"（不存储——节省空间），
            // 要么有"文化特定含义"（存储——偏离默认）
            // 大约 30% 的手势有文化特定含义
            if rng.gen_bool(0.3) {
                let meaning = generate_gesture_meaning(gesture, culture_id, &mut rng);
                mapping.meanings
                    .entry(gesture.id)
                    .or_default()
                    .insert(culture_id, meaning);
            }
        }
    }
    
    mapping
}
```

### 5.3 跨文化误解检测

语言表达 `010` 保留此方法的实现——它消费文化数据，但不拥有数据：

```rust
/// 跨文化手势误解检测
/// 
/// 当 actor 以 culture A 的含义做一个手势，
/// 但 observer 以 culture B 的含义解读它时——可能产生误解。
/// 
/// 此方法在 语言表达 010 中实现——它是使用逻辑，非定义逻辑。
fn cross_cultural_misinterpretation(
    gesture: GestureId,
    actor_culture: CultureId,
    observer_culture: CultureId,
    culture: &dyn CultureQuery,
) -> Option<GestureMisinterpretation> {
    let actor_meaning = culture.gesture_meaning(gesture, actor_culture)?;
    let observer_meaning = culture.gesture_meaning(gesture, observer_culture)?;
    
    // 如果 valence 符号不同 → 严重误解
    if actor_meaning.valence.signum() != observer_meaning.valence.signum() {
        Some(GestureMisinterpretation {
            gesture,
            intended: actor_meaning.clone(),
            perceived: observer_meaning.clone(),
            severity: (actor_meaning.valence - observer_meaning.valence).abs() / 2.0,
        })
    } else {
        None // 含义有差异但情感方向相同 → 轻微差异，不构成"误解"
    }
}
```

## 六、TouchNorms 在 NPC 行为中的应用

`TouchNorms` 被 NPC 的非语言行为系统消费——NPC 在社交互动中选择是否触摸、如何触摸。此消费逻辑保留在语言表达 `010` 和 NPC 模块中：

```
NPC 社交互动 → 查询 TouchNorms：
  触摸对手臂→ norms.public > 0.3 ✓
  拥抱 → norms.hug > 0.5 ✓
  对异性触摸 → norms.opposite_gender > 0.4 ✓
  拍下属肩膀 → norms.dominance > 0.3 且 culture.power_distance > 0.5 ✓
  
  如果 norms 值低于阈值 → NPC 不做此动作
  如果对方来自 norms 差异 > 0.5 的文化 → 可能触发"文化误解"社交事件
```

## 七、CultureQuery 中的 norms 访问方法

```rust
// CultureQuery trait 中
fn communication_norms(&self, id: CultureId) -> Option<&CommunicationNorms>;

// 批量获取 norms——用于 NPC 群组对话等场景
fn batch_communication_norms(&self, ids: &[CultureId]) -> Vec<Option<&CommunicationNorms>> {
    ids.iter().map(|id| self.communication_norms(*id)).collect()
}
```

`CommunicationNorms` 是**高频查询**——每对话回合至少一次。文化 Store 使用 SoA 布局（`Vec<CommunicationNorms>` 按 CultureId 索引），零拷贝返回引用。

## 八、HonorificSystem 与语言表达 LanguageAttributes 的对齐

语言表达 `002` 中 `LanguageAttributes.has_honorifics` 字段与 `HonorificSystem.has_grammatical_honorifics` 在概念上相关但不同：

| 字段 | 含义 | 所有者 |
|------|------|--------|
| `LanguageAttributes.has_honorifics` | 语言本身是否包含敬语语法结构 | [[../语言表达/002-语言与文字系统|语言表达 002]] |
| `HonorificSystem.has_grammatical_honorifics` | 文化是否期望使用语法敬语 | 本文档（文化系统） |

两者应一致——如果语言没有敬语语法，文化不可能期望使用它。实现上：语言分配时查询文化的 `HonorificSystem.has_grammatical_honorifics` → 如果为 true → 分配给该文化的语言必须有敬语语法。这是文化→语言的单向约束。
