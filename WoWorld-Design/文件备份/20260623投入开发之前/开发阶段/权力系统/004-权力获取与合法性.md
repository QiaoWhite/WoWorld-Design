## 一、核心洞察

权力不是"凭空存在"的——每条权力边都有一个**获取路径**（PowerSource）。路径决定**初始 legitimacy**、**继承行为**和**被挑战的方式**。

8 条获取路径对应 8 种不同的社会叙事："他是国王的儿子"、"她击败了前任"、"他们投票选了她"、"他花钱买通了掌权者"、"他靠自己建立起统治"。

## 二、8 条权力获取路径

### 2.1 Inherited — 继承

**含义**：从父辈/前任 holder 通过继承规则获得。最常见的正式权力转移。

**适用原子**：任何。但通常用于 ClaimTerritory（统治权）、Extract（税收权）、ConferRank 附带权力。

**初始 legitimacy**：高（0.7-0.9）——取决于继承规则是否被广泛接受。

**触发条件**：
- Holder 死亡 → `on_holder_death` 处理 → 按 SuccessionRule 决定继承
- 如果 `SuccessionRule::Unspecified` → 触发继承危机事件（PowerEvent::SuccessionCrisis）

```rust
fn on_holder_death(&mut self, deceased: EntityId) -> Vec<PowerEvent> {
    let mut events = Vec::new();
    let edges: Vec<EdgeId> = self.outgoing.get(&deceased)
        .cloned().unwrap_or_default();
    
    for edge_id in edges {
        let edge = self.get_edge(edge_id);
        if !edge.active { continue; }
        
        match &edge.succession_rule {
            SuccessionRule::Designated { heir } => {
                self.transfer_edge(edge_id, deceased, *heir, PowerSource::Inherited { from: Some(deceased) });
                events.push(PowerEvent::PowerInherited { edge_id, from: deceased, to: *heir });
            }
            SuccessionRule::Primogeniture { dynasty, order } => {
                if let Some(heir) = self.find_heir_in_dynasty(*dynasty, *order) {
                    self.transfer_edge(edge_id, deceased, heir, PowerSource::Inherited { from: Some(deceased) });
                    events.push(PowerEvent::PowerInherited { edge_id, from: deceased, to: heir });
                } else {
                    // 无继承人 → 边失效 + 继承危机
                    self.deactivate_edge(edge_id);
                    events.push(PowerEvent::SuccessionCrisis { edge_id, deceased });
                }
            }
            SuccessionRule::ElectedBy { electorate } => {
                // 触发选举事件——由选举路径处理
                events.push(PowerEvent::ElectionNeeded { edge_id, electorate: *electorate, reason: "holder_died" });
                self.set_edge_pending_succession(edge_id);
            }
            SuccessionRule::RevertToSuperior => {
                if let Some(superior) = self.delegation_source_for_edge(edge_id) {
                    // 权力回归上级——上级决定重新委派或保留
                    self.revert_to_superior(edge_id, superior);
                    events.push(PowerEvent::PowerReverted { edge_id, to: superior });
                }
            }
            SuccessionRule::ExtinguishWithHolder => {
                self.deactivate_edge(edge_id);
                events.push(PowerEvent::EdgeExtinguished { edge_id, reason: "holder_died" });
            }
            SuccessionRule::Unspecified => {
                self.set_edge_pending_succession(edge_id);
                events.push(PowerEvent::SuccessionCrisis { edge_id, deceased });
            }
        }
    }
    events
}
```

### 2.2 Appointed — 任命

**含义**：上级 holder 通过 `Delegate` 或 `ConferRank` 将权力授予他人。最常见的官僚路径。

**适用原子**：任何（受限于上级 holder 拥有什么）。

**初始 legitimacy**：中高（0.5-0.8）——取决于上级 holder 的合法性。如果上级合法性低 → 任命的合法性也低（`legitimacy = superior_legitimacy × 0.85`）。

**触发条件**：Delegate 原子行使。

### 2.3 Elected — 选举

**含义**：由 subject 群体（或代表群体）通过投票产生。最常见于共和制、行会会长选举。

**适用原子**：任何（选举范围由 electorate 的治理规则决定）。

**初始 legitimacy**：高（0.6-0.9）——取决于选举程序的公正性 + 当选者的声望。

**选举机制**（简化——不模拟逐票计数）：
```rust
fn simulate_election(
    electorate: EntityId,  // Group
    candidates: Vec<EntityId>,
    topology: &PowerTopology,
) -> ElectionResult {
    let voters = topology.get_members(electorate);
    let mut votes = HashMap::new();
    
    for voter in voters {
        // 每个投票者的投票 = f（候选人声望 × 关系 × 文化偏好）
        let vote = choose_candidate(voter, &candidates, topology);
        *votes.entry(vote).or_default() += 1;
    }
    
    let winner = votes.iter().max_by_key(|(_, count)| *count).map(|(c, _)| *c);
    ElectionResult { winner, votes, turnout: voters.len() }
}
```

### 2.4 Purchased — 购买

**含义**：用资源（通常是铜币）向掌权者交换权力行使权。不是"买头衔"——是"购买权力的行使承诺"。

**适用原子**：任何持有者可转让的原子。

**初始 legitimacy**：低（0.1-0.3）——取决于文化对权力买卖的态度。`power_distance` 高的文化可能认"花钱买的权力"，低的则不认。

**触发条件**：经济系统产生 PowerDeal → 权力系统消费。不能由玩家在权力系统内"点按钮"——需要先在经济系统侧完成交易。

**行贿意愿**（在 NPC 侧，由经济系统 007 已有公式覆盖）：`(1-agreeableness) × intelligence × (1-neuroticism)`。

### 2.5 Conquered — 征服

**含义**：通过暴力或武力击败前任 holder 或 subjugator，直接夺取权力。

**适用原子**：主要限于 ClaimTerritory、Extract（征收贡品）、Constrain（占领限制）、DefineMembership（强制并入）。

**初始 legitimacy**：极低（0-0.15）——"征服不能让人心悦诚服"。随时间 + 治理表现逐步回升。

**触发条件**：战斗系统返回征服结果 → 权力系统消费。征服者必须在战斗系统中明确"击败"前任 holder（或其军事力量）。

**Legitimacy 回升曲线**：
```
legitimacy(t) = initial + (1 - e^(-λt)) × recovery_cap
其中 λ = f(governance_quality) × f(cultural_fit)
recovery_cap = 0.6（征服永远不能达到选举的合法性高度——总有"篡位者"的阴影）
```

### 2.6 Divine — 神授/魔法

**含义**：通过信仰仪式、神谕、魔法证明获得权力。通常需要信仰系统或魔法系统的支持。

**适用原子**：任何。通常用于 ClaimTerritory + PrescribeRule（神权政体）。

**初始 legitimacy**：在宗教性高的文化中极高（0.7-0.95）——在世俗文化中中低（0.3-0.5）。

**触发条件**：**信仰系统**（[[../../信仰系统/007-信仰与权力的桥接|信仰 007]]）或魔法系统发出 Divine/Magic authorization 事件 → 权力系统创建对应边。

### 2.7 Emergent — 涌现

**含义**：Holder 在事实上对 subject 持续行使控制，但没有任何正式权力来源。系统自动检测并创建"事实权力"边。这是"地头蛇"、"事实领袖"的产生机制。

**适用原子**：Constrain、Compel、Extract（事实控制的最初形式）。

**初始 legitimacy**：低（0-0.2）——事实权力缺乏正式认可。但如果 holder 持续提供保护/秩序/福利，随时间自然上升。

**触发条件**——涌现检测算法：
```rust
/// 每游戏月执行一次
fn detect_emergent_power(&mut self) -> Vec<PowerEvent> {
    let mut events = Vec::new();
    
    for subject in self.all_entities() {
        // 统计在过去 90 天内，谁对 subject 行使了 ≥5 次关系原子
        let de_facto_holders = self.recent_exercise_stats(subject, 90)  // 天
            .filter(|(holder, stats)| {
                stats.total_exercises >= 5                  // ≥5 次行使
                && stats.compliance_rate >= 0.7             // subject 服从率 ≥70%
                && !self.has_formal_edge(*holder, subject)  // 没有正式权力边
            });
        
        for (holder, stats) in de_facto_holders {
            // 创建涌现边
            let atom = self.infer_dominant_atom(&stats);
            let edge_id = self.create_edge(PowerEdgeData {
                holder: *holder,
                subject,
                atom_type: atom,
                source: PowerSource::Emergent {
                    recognized_at: current_time(),
                    basis: EmergenceBasis { compliance_rate: stats.compliance_rate, exercise_count: stats.total_exercises },
                },
                legitimacy: 0.1,
                enforcement: stats.compliance_rate * 0.8,
                succession_rule: SuccessionRule::Unspecified,
                ..Default::default()
            });
            events.push(PowerEvent::EmergentPowerDetected { edge_id, holder: *holder, subject });
        }
    }
    events
}
```

### 2.8 Contractual — 契约

**含义**：双方通过 `Contract` 原子互相授予对方的权力。唯一需要另一方同意的获取路径。

**适用原子**：任何两个实体通过 Contract 约定的原子。

**初始 legitimacy**：高（0.7-0.9）——契约 = 双方同意，天然具有合法性。但如果一方被迫签约（威胁/欺诈），legitimacy 从签约时就被污染。

**触发条件**：Contract 原子行使（需双方同意确认）。

## 三、Legitimacy 计算模型

### 3.1 定义

`legitimacy` = subject 对 holder 对 subject 行使某原子的**主观认可度**。不同的 subject 可能对同一个 holder 的同一个权力边有不同的 legitimacy（但实际上只存储 subject 侧的感知值）。

**为什么是主观的**：同一个国王，税率 5%。农民 A 认为合理 → legitimacy 0.8。地主 B 认为太重 → legitimacy 0.4。这两个合法性共存——如果足够多 subject 的 legitimacy 降到 0 以下 → 集体反抗。

**存储方案**：`legitimacy` 存储在每条边的 subject 侧——即 subject 对 holder 此条权力边的认可。如果 subject 是 Group（一个群体），legitimacy = 群体成员的平均认可度。

### 3.2 五因子可配置公式

```rust
/// legitimacy 变化的五因子模型
fn compute_legitimacy_delta(
    edge: &PowerEdge,
    holder: &NpcData,
    subject: &NpcData,
    topology: &PowerTopology,
    economy: &dyn EconomyQuery,
    culture: &dyn CultureQuery,
) -> f32 {
    // === 因子 1: 程序正当性 [权重 0.35] ===
    // 权力是否按照已有规则获得？
    let procedural = match &edge.source {
        PowerSource::Inherited { .. } if is_established_dynasty(edge, topology) => 0.8,
        PowerSource::Inherited { .. } => 0.6,                // 新 dynasty
        PowerSource::Appointed { appointer, .. } => {
            topology.perceived_legitimacy(*appointer, edge.subject) * 0.85
        }
        PowerSource::Elected { .. } => 0.85,                  // 选举程序本身即正当
        PowerSource::Purchased { .. } => 0.15,                // 购买的程序正当性低
        PowerSource::Conquered { .. } => 0.05,               // 征服的程序正当性极低
        PowerSource::Divine { .. } => 0.7,                    // 取决于文化 religiosity
        PowerSource::Emergent { .. } => 0.1,                 // 无程序，靠事实
        PowerSource::Contractual { .. } => 0.75,             // 双方同意 = 正当
    };

    // === 因子 2: 结果满意度 [权重 0.20] ===
    // holder 行使此权力是否让 subject 受益？
    let outcome = match edge.atom_type {
        AtomTypeTag::Extract => {
            // 税收被用于公共工程 → 满意度高
            // 税收被用于挥霍 → 满意度低
            let tax_usage_score = economy.tax_usage_rating(edge.holder, edge.subject);
            tax_usage_score * 0.8 + 0.1  // [0.1, 0.9] 范围
        }
        AtomTypeTag::Constrain => {
            // 如果约束减少了犯罪/混乱 → 满意度中高
            // 如果约束被认为过严 → 满意度低
            let crime_change = economy.crime_rate_change(edge.subject, 90);
            if crime_change < -0.1 { 0.7 } else if crime_change > 0.1 { 0.2 } else { 0.5 }
        }
        _ => 0.5,  // 默认中性
    };

    // === 因子 3: 文化契合度 [权重 0.20] ===
    // 权力结构是否符合本地文化的期望？
    let cultural = {
        let cd = culture.params_at(edge.subject.position());
        let power_gap = 1.0 - (edge.enforcement - cd.power_distance).abs();
        power_gap * 0.8 + 0.2  // [0.2, 1.0]
    };

    // === 因子 4: 时间惯性 [权重 0.15] ===
    // 存在的越久 → 越被认为是"理所当然的"
    let inertia = (edge.age_years() * INERTIA_RATE).min(INERTIA_CAP);
    // INERTIA_RATE = 0.04/年, INERTIA_CAP = 0.3

    // === 因子 5: 仪式加持 [一次性, 权重 0.10] ===
    // 最近的加冕/宣誓/祭祀带来的合法性提升
    let ritual = if let Some(last_ritual) = edge.last_ritual_at {
        let days_since = (current_time() - last_ritual).as_days();
        if days_since < 30 { 0.15 }                          // 一个月内
        else if days_since < 365 { 0.08 }                    // 一年内
        else if days_since < 3650 { 0.03 }                   // 十年内
        else { 0.0 }
    } else { 0.0 };

    // === 综合公式 ===
    procedural * 0.35 + outcome * 0.20 + cultural * 0.20 + inertia * 0.15 + ritual * 0.10
}
```

### 3.3 Legitimacy 重算频率与优化

**频率**：每游戏日重算一次。分摊到帧 → 500K 边 / ~200K帧(每游戏日) ≈ 2-3 条边/帧。可忽略。

**批量重算优化**：
```rust
fn prepare_legitimacy_cache(
    edges: &[EdgeId],
    economy: &dyn EconomyQuery,
    culture: &dyn CultureQuery,
) -> LegitimacyInputCache {
    // 批量收集所有需要的外部数据——一次跨模块调用
    LegitimacyInputCache {
        tax_usage: economy.batch_tax_usage_rating(edges),
        crime_rates: economy.batch_crime_rates(edges),
        culture_params: culture.batch_params_for_positions(edges),
    }
}

fn recalc_legitimacy_batch(
    topology: &mut PowerTopology,
    cache: &LegitimacyInputCache,
    chunk: &[EdgeId],
) {
    for &edge_id in chunk {
        let edge = topology.get_edge_mut(edge_id);
        let delta = compute_legitimacy_delta_cached(edge, cache);
        let new_legitimacy = (edge.legitimacy + delta * LEGITIMACY_SMOOTHING).clamp(0.0, 1.0);
        edge.legitimacy = new_legitimacy;
        
        // legitimacy 归零 → 触发危机
        if new_legitimacy < LEGITIMACY_CRISIS_THRESHOLD {  // 0.05
            topology.flag_for_revolution_check(edge_id);
        }
    }
}
```

### 3.4 正反馈循环的阻尼

**问题**：高 legitimacy → 更易行使权力 → 好治理结果 → 更高 legitimacy（正反馈）

**阻尼机制**：
1. 结果满意度权重设低（0.20）——不是主要因子
2. 外部冲击（天气灾害）不因 governance quality 而改变频率——好国王也会遭遇天灾
3. 信息失真会衰减"好治理 → 高满意度"的传导——国王的好政策如果下级执行走样，subject 的满意度对应下降
4. Legitimacy 的 `SMOOTHING` 系数：增量乘以 0.3（也就是说每次最多改变 30% 的计算增量）——legitimacy 像一艘大船，转动缓慢

## 四、非行使衰减

如果一个 holder 长时间不行使某原子——权力自然衰退。

```rust
/// 每游戏月检查一次
fn check_exercise_decay(&mut self) -> Vec<PowerEvent> {
    let now = current_time();
    let mut events = Vec::new();
    
    // 会衰减的原子类型
    const DECAYING_ATOMS: &[AtomTypeTag] = &[
        AtomTypeTag::Constrain,
        AtomTypeTag::Compel,
        AtomTypeTag::Extract,
        AtomTypeTag::Adjudicate,
    ];
    
    for atom in DECAYING_ATOMS {
        for &edge_id in &self.atom_type_index[*atom as usize] {
            let edge = self.get_edge(edge_id);
            if !edge.active { continue; }
            
            if let Some(last_ex) = edge.last_exercised_at {
                let days_since_exercise = (now - last_ex).as_days();
                let decay_rate = match edge.atom_type {
                    // Constrain/Compel 衰减最快——"不用的铁链会生锈"
                    AtomTypeTag::Constrain | AtomTypeTag::Compel => 0.002,
                    // Extract 衰减中——"中断征税重新开征有阻力"
                    AtomTypeTag::Extract => 0.001,
                    // Adjudicate 衰减慢——"法院不判案但还在那儿"
                    AtomTypeTag::Adjudicate => 0.0005,
                    _ => 0.0,  // 其他原子不衰减
                };
                
                let new_enforcement = edge.enforcement - decay_rate * days_since_exercise as f32;
                edge.enforcement = new_enforcement.max(0.0);
                
                if new_enforcement <= ENFORCEMENT_EXTINCTION_THRESHOLD {  // 0.02
                    self.deactivate_edge(edge_id);
                    events.push(PowerEvent::EdgeDecayedToExtinction { edge_id });
                }
            }
        }
    }
    events
}
```

**不衰减的原子**：Delegate（转让出的权力，原 holder 是否行使不影响被委托者）、ConferRank（头衔持续有效除非被撤销）、DefineMembership（成员资格不因不用而丧失）、Contract（契约有双方约束，不靠单方行使维持）。

## 五、玩家获取权力的入口

8 条路径对玩家全部开放：

| 路径 | 玩家操作 | 需要的跨模块支持 |
|------|---------|----------------|
| Inherited | 通过死亡传承系统继承前角色的权力 | 生命系统（死亡传承） |
| Appointed | 与掌权者建立高关系 → 被授予职位 | NPC 社交系统 |
| Elected | 在允许选举的团体内积累声望 → 参选 | 名声系统（声望数据） |
| Purchased | 向腐败掌权者行贿（消耗铜币） | 经济系统（PowerDeal） |
| Conquered | 击败持有者（战斗）→ 宣称权力 | 战斗系统（决斗/战争结果） |
| Divine | 完成神祇/魔法的仪式/任务 | 信仰系统 + 魔法系统 |
| Emergent | 在事实上保护/服务一群人 → 被系统识别 | 时间 + 持续行为记录 |
| Contractual | 与另一方签订契约互授权力 | NPC 社交 / 法律系统 |

没有任何一条路径是"玩家在菜单中选'成为国王'"——每条都需要跨模块的持续行为投入。
