# 游戏概述 — 接口入口

> **最后验证**: 2026-06-18 | **源文档版本**: 游戏概述 v1.0
>
> 游戏概述是高层愿景文档——它不消费技术接口, 但依赖技术栈方案确认愿景的技术可行性。

## 技术栈方案
- **v4.0 技术蓝图**: 确认"25万km²体素世界 + 100k NPC + 涌现交互"的技术可行性 — [[../19-技术栈方案/001-接口出口|技术栈方案]]
- **性能预算**: 16.7ms/帧 → 确认"完整的人"模型可以在目标硬件上运行
- **风险矩阵**: 高风险项 (体素LOD/100k NPC并行/GDExtension边界) → 影响游戏概述中承诺的规模

## 设计约束传递
游戏概述设定的愿景 → 向下约束各模块设计:

| 愿景 | 约束 | 影响模块 |
|------|------|---------|
| 100k 活NPC | 内存≤1.4GB, CPU≤4ms模拟 | NPC/生命/技能/感官 |
| 25万km²世界 | VRAM≤4.2GB, 体素8级LOD | 世界生成/载具 |
| 涌现交互 | 概率系统为主(90%), GOAP安全网(9%) | NPC/经济/权力/文化 |
| 全球多元文化 | 参数化文化(非枚举), 障碍Voronoi | 文化/信仰/语言表达/世界生成 |
| 故事生成器 | 所有行为=参数+随机+统计涌现 | NPC/战斗/经济/历史 |
| "完整的人" | NPC记忆2000条, 人格BigFive | NPC/生命/认知/审美 |
| 冒险=生活 | 战斗/经济/技能/权力平等的系统深度 | 战斗/经济/技能/权力 |

## 不消费的技术接口
游戏概述是纯设计文档 — 不定义也不消费任何 Rust trait/struct. 所有约束通过设计审查传递, 非编译时检查.
