# 004 — 装备系统

## 一、装备槽位——BodyPlan 自动派生

### 1.1 设计原则

**不预定义物种类型。** 不给"人类""龙""蛇人"硬编码槽位集合。槽位从 BodyPlan 的解剖结构自动派生。

```rust
/// BodyPlan（Life 模块生成）→ 物品系统查询可用装备槽位
impl BodyPlan {
    pub fn equipment_slots(&self) -> Vec<EquipmentSlot> {
        let mut slots = Vec::new();

        // === 战斗/防具槽位 ===

        // 头 → 头盔（每头 1 槽）
        for i in 0..self.head_count {
            slots.push(EquipmentSlot {
                id: SlotId::Head(i),
                label: "头",
                accepts: SlotFilter::Category(ItemCategory::Armor, ArmorType::Helmet),
            });
        }

        // 躯干 → 躯干甲
        slots.push(EquipmentSlot {
            id: SlotId::Torso(self.torso_type),
            label: "躯干",
            accepts: SlotFilter::Category(ItemCategory::Armor, ArmorType::TorsoArmor),
        });

        // ★ 肩部 → 围巾/斗篷/披风（躯干有肩部悬挂结构则有）
        // shoulder_drape 从 TorsoType 派生：肩部能否悬挂织物/轻甲
        if self.torso_type.shoulder_drape() {
            slots.push(EquipmentSlot {
                id: SlotId::Shoulder,
                label: "肩部",
                accepts: SlotFilter::Category(ItemCategory::Armor, ArmorType::ShoulderArmor),
            });
        }

        // 腿 → 腿甲
        for group in &self.appendages {
            if matches!(group.label, AppendageLabel::Leg | AppendageLabel::HoofLeg) {
                for i in 0..group.count {
                    slots.push(EquipmentSlot {
                        id: SlotId::Leg(group.label, i),
                        label: "腿",
                        accepts: SlotFilter::Category(ItemCategory::Armor, ArmorType::LegArmor),
                    });
                }
            }
        }

        // 足 → 靴（Leg/HoofLeg/ClawLeg 有足则有靴）
        for group in &self.appendages {
            if group.has_foot_equivalent() {
                for i in 0..group.count {
                    slots.push(EquipmentSlot {
                        id: SlotId::Foot(group.label, i),
                        label: "足",
                        accepts: SlotFilter::Category(ItemCategory::Armor, ArmorType::Boots),
                    });
                }
            }
        }

        // 握持槽 → 武器/盾/工具
        for group in &self.appendages {
            if group.can_grasp {
                for i in 0..group.count {
                    slots.push(EquipmentSlot {
                        id: SlotId::Grasp(group.label, i),
                        label: &format!("{}", group.label),
                        accepts: SlotFilter::Graspable,  // 武器/盾/工具均可
                        two_hand_capable: group.strength_ratio > 0.6,
                    });
                }
            }
        }

        // 翼 → 翼甲
        for group in &self.appendages {
            if matches!(group.label, AppendageLabel::Wing) {
                for i in 0..group.count {
                    slots.push(EquipmentSlot {
                        id: SlotId::Wing(group.label, i),
                        label: "翼",
                        accepts: SlotFilter::Category(ItemCategory::Armor, ArmorType::WingArmor),
                    });
                }
            }
        }

        // 尾 → 尾甲/尾刺
        for group in &self.appendages {
            if matches!(group.label, AppendageLabel::Tail) {
                for i in 0..group.count {
                    slots.push(EquipmentSlot {
                        id: SlotId::Tail(group.label, i),
                        label: "尾",
                        accepts: SlotFilter::TailSlot,  // 尾甲或尾武器
                    });
                }
            }
        }

        // === 饰品槽（解剖特征派生） ===
        slots.extend(self.accessory_slots());

        // === 便携容器槽（纯数据层，无 3D 实体建模） ===
        slots.extend(self.container_slots());

        slots
    }
}
```

### 1.2 TorsoType 解剖学属性

`shoulder_drape` 是 `TorsoType`（Life 模块定义）上的派生方法——描述躯干是否具有可悬挂织物的肩部结构：

```rust
impl TorsoType {
    /// 躯干是否有肩部悬挂结构——可用于披挂斗篷/围巾/披风
    pub fn shoulder_drape(&self) -> bool {
        matches!(
            self,
            TorsoType::HumanoidMedium
                | TorsoType::HumanoidLarge
                | TorsoType::HumanoidLong
                | TorsoType::HumanoidSlender
        )
    }
}
```

**设计意图**：不引入"类人/非类人"物种标签。从解剖学事实——肩部形态——自动派生。龙/蛇人/半人马（马躯干）/史莱姆/元素生物均无此解剖结构，自然无肩部槽位。

### 1.3 具体生物示例

```
人类: Head×1 / Torso(HumanoidMedium)×1 / Arm×2 / Leg×2
  → 1头盔 / 1肩部(斗篷/围巾) / 1胸甲 / 2腿甲 / 2靴 / 2握持(可双持)
     / 饰品+容器(纯数据)

龙: Head×1 / Torso(DraconicLarge)×1 / ClawLeg×4 / Wing×2 / Tail×1
  → 1头甲 / 1龙铠 / 4爪甲 / 2翼甲 / 1尾刺 / 0握持 / 0肩部(无悬挂结构)
     / 饰品+容器(纯数据)

半人马: Head×1 / Torso(HumanoidMedium)×1 / Torso(Equine)×1 / Arm×2 / HoofLeg×4
  → 1头盔 / 1肩部(人躯干有肩) / 1人胸甲 + 1马铠 / 4蹄铁 / 2握持
     / 饰品+容器(纯数据)

蛇人: Head×1 / Torso(Serpentine)×1 / Arm×2 / Tail×1
  → 1头盔 / 1蛇形躯干甲 / 2握持 / 1尾环 / 0靴(无腿) / 0肩部(无悬挂结构)
     / 饰品+容器(纯数据)
```

## 二、双套 Outfit —— BG3 式即时切换

**设计意图**：双套服饰不仅是 UX 便利——它驱动**双轨经济**。战甲路径消费锻造/制甲（金属+皮革），常服路径消费纺织/裁缝（纤维+布料+染料）。两个制造业并行运转，互不替代。一个繁荣的城镇需要铁匠**和**裁缝。NPC 也要买两套衣服——这就是对经济系统的天然需求信号。

### 2.2 切换规则

**谁做决定：NPC 自己，不是开发者。**

```rust
/// NPC 自主评估当前应该穿什么
/// 此函数在 NPC 模块中实现——物品系统只定义数据模型
fn evaluate_outfit_choice(npc: &Npc, world: &World) -> OutfitMode {
    // 多个驱动力博弈——没有"if combat → wear combat"的固定规则

    let danger = npc.combat.self_assess_threat_level(world);  // Combat 自查
    let social = npc.social.current_context();
    let p = &npc.personality;
    let mood = npc.emotion.current_mood();

    let mut weights: BTreeMap<OutfitMode, f32> = BTreeMap::new();

    // 危险评估
    weights[Combat] += danger * 2.0;

    // 社交场景
    if social.is_tavern() && p.extraversion > 0.7 {
        // 虚荣的人穿最好看的
        let combat_score = aesthetic_score(&equipment.combat, &npc.taste);
        let civilian_score = aesthetic_score(&equipment.civilian, &npc.taste);
        if combat_score > civilian_score + 0.3 {
            weights[Combat] += 0.4;  // "这件甲确实帅"
        }
    }

    // 人格
    if p.neuroticism > 0.8 && danger < 0.2 {
        weights[Combat] += 0.3;  // 偏执狂——安全村庄也穿甲
    }
    if p.conscientiousness < 0.2 && danger > 0.6 {
        weights[Civilian] += 0.3;  // 懒人——危险也不换
    }

    // 经验
    if npc.knowledge.combat_experience > 100 {
        weights[Combat] += 0.15;  // 老兵——不穿反而不自在
    }

    // 天气
    if world.weather.is_freezing() {
        // 选最暖的 outfit
        let combat_warmth = thermal_score(&equipment.combat);
        let civilian_warmth = thermal_score(&equipment.civilian);
        if combat_warmth > civilian_warmth {
            weights[Combat] += 0.2;
        }
    }

    weights.into_iter().max_by_key(|(_, w)| *w).unwrap().0
}
```

**切换开销**：0 秒。Baldur's Gate 3 式——瞬间切换，无动画延迟，无冷却。理由：
- 10 万 NPC 每天切 4 次如果有 3s 动画 = 120 万秒/天的纯等待
- 这是模拟 + 渲染的双重浪费
- 玩家也不想等

### 2.3 饰品不随 Outfit 切换

戒指/项链/手镯/耳环/头环——战斗/日常/睡眠都一样戴着。除非 NPC **自己愿意**换（见决策 3 结论）。

**戒指手指映射**：四个戒指槽位绑定到具体手指——

| 槽位 | 手指 | 说明 |
|------|------|------|
| `ring_right_middle` | 右手中指 | 手背朝向人脸时从内往外数第三指 |
| `ring_right_ring` | 右手无名指 | 手背朝向人脸时从内往外数第四指 |
| `ring_left_middle` | 左手中指 | 同上，左手 |
| `ring_left_ring` | 左手无名指 | 同上，左手 |

3D 渲染时，戒指骨骼挂点直接查 `SlotId` → 手指骨骼映射表，不区分左右手特效差异。

饰品的战斗属性和审美价值在任何 Outfit 模式下都生效。

## 三、便携容器槽（纯数据层）

```rust
pub struct ContainerSet {
    pub back: Option<ItemEntId>,         // 背槽 ×1 —— 背包/背囊/次元袋
    pub shoulder: Option<ItemEntId>,     // 肩槽 ×1 —— 挎包/肩挂包
    pub waist_left: Option<ItemEntId>,   // 腰槽左 ×1 —— 腰包/采集袋
    pub waist_right: Option<ItemEntId>,  // 腰槽右 ×1 —— 弹药包/药剂腰包
    pub hand_left: Option<ItemEntId>,    // 手提左 ×1 —— 麻袋/水桶/工具箱
    pub hand_right: Option<ItemEntId>,   // 手提右 ×1 —— 同上
}
```

**★ v1.2 变更——容器无 3D 实体建模**：所有容器槽位仅为数据层概念。背包、挎包、腰包、手提容器均无角色模型上的 3D 渲染。存储容量、负重修正等 gameplay 效果照常生效。

**手提槽战斗规则**：
- 非战斗 → 移速惩罚生效
- 进入战斗 → 移速惩罚消除，手恢复自由
- 离开战斗 5s → 移速惩罚恢复
- 容器内容物在战斗期间仍属于角色库存——不会丢失
- 容器内容物在战斗中仍可访问——但触发 0.8s "翻找"延迟（从背包直接拿是 0s）

**BodyPlan 派生**：
- 背槽：有躯干则有
- 肩槽：躯干有肩部结构则有
- 腰槽：躯干有腰部分段则有（蛇=0，人=2，半人马=4）
- 手提槽：每个可握持附属提供 1 个手提槽（最多 2 个——双手各一）

## 四、装备作为物品

装备槽位中的每件装备**同时也是库存物品**——拥有自己的 ItemEntId 和 ItemState。从槽位卸下 = 回到库存。从库存装备 = 进入槽位。

```rust
impl CharacterEquipment {
    /// 装备一件物品到指定槽位
    pub fn equip(&mut self, slot: SlotId, item: ItemEntId, inventory: &mut CategorizedInventory) {
        // 如果槽位已有装备，先卸下
        if let Some(old) = self.unequip(slot) {
            inventory.add(old);  // 旧装备回库存
        }
        // 从库存移除
        inventory.remove(item);
        // 装备
        self.set_slot(slot, Some(item));
    }
}
```

## 五、槽位视觉独立开关

**★ v1.2 新增。** 玩家可独立控制每个装备槽位的 3D 渲染显隐，不影响装备属性/附魔/战斗效果。

```rust
/// 每槽位视觉显隐——纯客户端开关，不影响 gameplay
/// 仅对玩家角色生效。NPC 始终全显示。
pub struct EquipmentVisualToggles {
    pub head: bool,         // true = 显示
    pub shoulder: bool,
    pub torso: bool,
    pub legs: bool,
    pub feet: bool,
    pub mainhand: bool,
    pub offhand: bool,
    pub rings: bool,        // 全部 4 枚戒指统一开关
    pub necklace: bool,
    pub bracelets: bool,
    pub earrings: bool,
    pub circlet: bool,
}

impl Default for EquipmentVisualToggles {
    fn default() -> Self {
        Self {
            head: true, shoulder: true, torso: true,
            legs: true, feet: true, mainhand: true, offhand: true,
            rings: true, necklace: true, bracelets: true,
            earrings: true, circlet: true,
        }
    }
}
```

**设计意图**：参考大量 RPG 的"隐藏头盔"功能。许多玩家不希望头盔遮挡角色面部，但不影响头盔的防御和附魔。肩部斗篷同理——可能遮挡精心设计的躯干甲外观。

**约束**：
- 仅玩家角色——NPC 不受此开关影响（NPC 的显隐由 OutfitMode 和审美自主决定）
- 纯客户端——不影响存档、不影响网络同步、不影响战斗计算
- 无性能影响——只是跳过某个骨骼挂点的 `instance` 提交

## 七、与战斗模块的接口

- **WeaponType**：Combat 定义的属性标签——挂在 Blade/BowLimb 等组件的 `ItemProperties.tags` 中
- **动作基元**：由 Combat 注册的 slot_type 派生——物品系统不管
- **武器切换时间**：Combat 005 定义——物品系统只提供"装备/卸下"的槽位操作
- **双持规则**：两个握持槽各持一件（单持）或两个槽共同持一件（双持）——由 Combat 检查 `two_hand_capable`

## 八、性能

- 每 NPC 装备数据 ~320 bytes（Combat + Civilian + Accessories + Containers + shoulder 字段）
- 100K NPCs → ~32 MB 装备数据——存档中占比 ~0.1%
- 切换 outfit = 数据指针交换——0 计算开销
- 视觉开关 = bool 数组读取——零性能开销
