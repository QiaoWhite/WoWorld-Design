## 一、定位——智能种族的特例

```rust
struct Player {
    body: BodyState,          // ← 与同种族 NPC 完全相同
    race: RaceTraits,         // ← 共享的种族参数
    mind: PlayerMind,         // ← 唯一差异——由人类玩家输入驱动
}
```

| 系统 | 玩家 | NPC |
|------|:--:|:--:|
| BodyState(HP/饥饿/疲劳/体温/灵元素) | ✅ 共用同一套 | ✅ |
| RaceTraits(种族参数——从历史涌现) | ✅ | ✅ |
| 部位伤害 | ✅ | ✅ |
| 伤病/状态效果 | ✅ | ✅ |
| 心智层(GOAP/记忆/情绪) | ❌ 人类直接控制 | ✅ AI驱动 |
| 战斗AI | ❌ 半自动——玩家选风格+关键指令 | ✅ 同套Rust AI |
| 灵兽驯服/训练 | ✅ 5种驯服路径 | ✅ NPC也可拥有灵兽 |
| 死亡 | ✅ 走相同死亡流程 | ✅ |

## 二、玩家死亡

玩家和 NPC 走完全相同死亡流程——HP归零→濒死(抢救窗口)→死亡原因记录。**死亡后如何处理**(重生/继承者/永久死亡/……)本次不最终决定——留给游戏总体设计。

---

## 三、半自动战斗中的玩家

战斗系统已由 `开发阶段/战斗/` 定义：玩家的战斗 AI = NPC 的战斗 AI = 同一套 Rust 代码。

**自动执行**(无需玩家操作)：目标选择、移动(接近/保持距离/利用掩体)、普通攻击节奏、防御时机(格挡/躲避)、体力管理。

**玩家手动干预**：战斗风格切换(狂暴↔战术↔支援↔施法)、特殊技能释放(魔法大招/必杀技/物品)、队伍命令(集火/保护/撤退/散开)、关键决策(投降/谈判/逃跑/追击)、**灵兽指令**(攻击/回来/使用技能)。

---

## 四、灵兽与玩家

- 捕获：5种驯服路径(见 [[006-灵兽]])
- 训练：解锁/强化协作技能
- 纽带：bond_level 随时间和共同经历提升
- 忠诚：得不到照顾→下降→可能逃跑
- 死亡：灵兽走同样的死亡流程——**灵兽死亡=永久损失**——不能"复活"(除非极特殊魔法/神明干预)
- 上限：同一时间可携带的灵兽数量有限(由角色等级/技能/魅力决定)

---

## 五、玩家种族选择

创建角色时可选：人类(基准——中性适应力)、精灵(灵元素容量偏高+木/灵元素亲和)、矮人(力量/耐久高+火/金元素亲和)、程序化涌现的种族(如果该种子世界有)。

所有可选种族共享同一套底层系统——差异**仅在于 RaceTraits**——而这些特质不是手写表，是从该种族在世界生成+历史模拟中的涌现结果。
