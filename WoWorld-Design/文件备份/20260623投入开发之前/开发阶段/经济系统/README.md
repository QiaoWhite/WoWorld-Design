## 模块定位

经济系统是 WoWorld 的**价值交换物理引擎**——定义供需匹配、价格涌现、货币流动、市场权力的底层规则。它不定义"什么东西值多少钱"或"NPC该买什么"——这些是从底层规则和NPC心智中涌现出来的。

**核心原则**：经济系统 = 客观事实层。NPC心智 = 主观决策层。经济系统提供价格/供给/钱包余额；NPC用自己的人格×情绪×记忆×认知做消费决策。

## 设计哲学

1. **价格从交易中涌现，非公式计算** — 不存在中央定价函数。价格是撮合交易的统计副产品
2. **角色从条件中涌现，非指定** — 不需要"指定商人NPC"。满足四条件的任何NPC都可以自发开始套利
3. **同一套引擎，参数化体制** — 自由市场到军国经济，一套订单簿引擎 + MarketRegulations参数差异
4. **权力是原子的** — PowerAtom。任何人（包括玩家）的权力 = 一组可对市场执行的原子操作集合
5. **借贷是特化交易** — 不需要银行系统。贷款 = 赊购铜币。欠条 = 金融类物品
6. **玩家AI = NPC AI** — 交易、挂单、套利、行使权力——全部走同一代码路径

## 关键设计决策

| 决策 | 结论 |
|------|------|
| 价格如何形成 | 限价订单簿撮合——价格从交易中涌现，非公式计算 |
| 成交价模式 | 分层定价：正常中间价 + 紧急偏移 + 薄市场协商 + 政府强制价 |
| 市场是什么 | 供需匹配机制——不要求物理共处。Market = 订单簿 + 范围 + 交易模式。Storefront = 交易原子单位 |
| 商人怎么产生 | 四条件涌现（信息+资本+心理+物流）——不指定"商人NPC" |
| 不同经济体制 | 同一引擎 × MarketRegulations参数——自由市场到军国经济 |
| 银行/金融系统 | 不做独立系统。借贷=特化交易（赊购铜币）。欠条=物品系统ItemDefId |
| NPC消费决策 | NPC心智驱动（人格×情绪×记忆×认知）——不是财富档位规定 |
| 权力怎么行使 | PowerAtom集合 + exercise_power()——任何人（包括玩家）走同一代码路径 |
| 没有LLM时 | 100%功能正常。LLM只是经济解说员 |
