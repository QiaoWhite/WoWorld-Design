## 一、所有 trait 汇集

### 1.1 语言表达模块定义的 trait

```rust
// ===== woworld_types/readable.rs =====

/// 内容解析——每个拥有"可表达内容"的模块实现
pub trait ContentResolver: Send + Sync {
    fn carrier_type(&self) -> u8;
    fn resolve(&self, local_id: u64, ctx: &ReaderContext) -> Option<ResolvedContent>;
}

/// 位置查询扩展——仅物理位置可读物实现
pub trait LocatableReadable: ContentResolver {
    fn query_at_location(&self, location: LocationId, radius_m: f32) -> Vec<ExpressionRef>;
    fn query_at_location_filtered(&self, location: LocationId, radius_m: f32, carrier_types: &[u8]) -> Vec<ExpressionRef>;
}

// ===== lang_expression/text_generator.rs =====

/// 文本滤镜——后处理管道
pub trait TextFilter {
    fn apply(&self, text: String, ctx: &FilterContext) -> String;
}
```

### 1.2 语言表达模块消费的 trait（来自其他模块）

```rust
// 来自 NPC 模块——通过 DialogueContext 依赖注入，不直接调用
// NpcSnapshot, EventMemory, Relationship —— 纯数据类型

// 来自历史模块——通过 DialogueContext 依赖注入
// JournalEntry, JournalEntryRef —— 纯数据类型

// 来自物品模块——通过 DialogueContext 依赖注入
// ItemId, item display names —— 纯数据类型

// 来自世界生成——通过 DialogueContext 依赖注入
// LocationId → 地名字符串
```

**关键性质**：lang_expression 不需要任何来自其他模块的 trait。所有跨模块数据通过 `DialogueContext` 纯数据注入。

## 二、各模块 ContentResolver 实现规范

### 2.1 历史模块

```rust
// history/src/resolver.rs

/// PhysicalBook → ExpressionRef (carrier_type = 0x01)
impl ContentResolver for PhysicalBookResolver {
    fn carrier_type(&self) -> u8 { 0x01 }

    fn resolve(&self, local_id: u64, ctx: &ReaderContext) -> Option<ResolvedContent> {
        // LMDB: get("books/physical/{local_id}")
        let book: PhysicalBook = self.lmdb.get_physical_book(local_id)?;
        // LMDB: get("books/work/{book.work_id}")
        let work: Work = self.lmdb.get_work(book.work_id)?;

        // 取第一个 TextSegment 的 params——长篇内容分页由 UI 处理
        let segment = work.contents.first()?;

        Some(ResolvedContent {
            language: work.language,
            template_id: segment.content.template_id(),
            params: segment.content.text_params.clone(),
            encoding: Encoding::Text,
            condition: book.condition,
            metadata: ContentMetadata {
                title: Some(work.title.clone()),
                author: work.primary_author_name(),
                date: Some(self.format_year(work.creation_date)),
                encoding: Encoding::Text,
                language_display: self.language_name(work.language),
                script_display: self.script_name(work.language),
                carrier_description: Some(format!("{}·{}", book.copy_generation_display(), book.material_display())),
            },
        })
    }
}

impl LocatableReadable for PhysicalBookResolver {
    fn query_at_location(&self, location: LocationId, radius_m: f32) -> Vec<ExpressionRef> {
        self.lmdb.scan_books_at_location(location, radius_m)
            .map(|book| ExpressionRef::new(0x01, book.id.as_u64()))
            .collect()
    }
}

/// LifeTrace → ExpressionRef (carrier_type = 0x02)
impl ContentResolver for LifeTraceResolver {
    fn carrier_type(&self) -> u8 { 0x02 }
    fn resolve(&self, local_id: u64, ctx: &ReaderContext) -> Option<ResolvedContent> {
        let trace: LifeTrace = self.lmdb.get_trace(local_id)?;
        // LifeTrace 的可读性受 visibility + text_completeness 影响
        let condition = trace.visibility_factor() * trace.content.text_completeness;
        Some(ResolvedContent {
            language: self.region_language(trace.location),
            template_id: trace.template_id(),
            params: trace.content.text_params.clone(),
            encoding: Encoding::Text,
            condition,
            metadata: ContentMetadata {
                title: Some(trace.trace_type.display_name()),
                author: trace.owner_name(),
                date: Some(self.format_year(trace.created_at)),
                encoding: Encoding::Text,
                language_display: self.region_language_name(trace.location),
                script_display: None,
                carrier_description: Some(trace.trace_type.carrier_description()),
            },
        })
    }
}

impl LocatableReadable for LifeTraceResolver {
    fn query_at_location(&self, location: LocationId, radius_m: f32) -> Vec<ExpressionRef> {
        self.lmdb.scan_traces_at_location(location, radius_m)
            .map(|trace| ExpressionRef::new(0x02, trace.id.as_u64()))
            .collect()
    }
}
```

### 2.2 魔法模块

```rust
// magic/src/resolver.rs

/// Grimoire → ExpressionRef (carrier_type = 0x10)
impl ContentResolver for GrimoireResolver {
    fn carrier_type(&self) -> u8 { 0x10 }
    fn resolve(&self, local_id: u64, ctx: &ReaderContext) -> Option<ResolvedContent> {
        let grimoire = self.lmdb.get_grimoire(local_id)?;
        let entry = grimoire.primary_entry()?;

        Some(ResolvedContent {
            language: grimoire.language,
            template_id: Self::template_for_entry_type(entry.entry_type),
            params: entry.text_params(),  // 魔导书6字段 → 标准参数名
            encoding: Encoding::Text,     // 魔导书=文本编码
            condition: grimoire.condition,
            metadata: ContentMetadata {
                title: Some(grimoire.title.clone()),
                author: Some(grimoire.author_name.clone()),
                date: Some(self.format_year(grimoire.creation_date)),
                encoding: Encoding::Text,
                language_display: self.language_name(grimoire.language),
                script_display: self.script_name(grimoire.script),
                carrier_description: Some(grimoire.binding_description()),
            },
        })
    }
}

/// MagicScroll → ExpressionRef (carrier_type = 0x11)
impl ContentResolver for ScrollResolver {
    fn carrier_type(&self) -> u8 { 0x11 }
    fn resolve(&self, local_id: u64, ctx: &ReaderContext) -> Option<ResolvedContent> {
        let scroll = self.lmdb.get_scroll(local_id)?;
        let formation = scroll.formation()?;

        Some(ResolvedContent {
            language: LanguageId::MAGIC_UNIVERSAL,  // 法阵语言 = 跨文化通用
            template_id: TemplateId::MAGIC_SCROLL_DESCRIPTION,
            params: formation.to_text_params(),  // 法阵参数 → 标准参数名
            encoding: Encoding::Geometric,       // 卷轴=几何编码 → UI 应额外渲染 Formation 图
            condition: scroll.condition,
            metadata: ContentMetadata {
                title: Some(scroll.name.clone()),
                author: scroll.creator_name(),
                date: Some(self.format_year(scroll.creation_date)),
                encoding: Encoding::Geometric,
                language_display: "法阵几何语言".to_string(),
                script_display: None,
                carrier_description: Some(scroll.material_description()),
            },
        })
    }
}
```

## 三、LMDB key 约定

```rust
// 语言表达模块不创建自己的 LMDB 存储
// 所有数据属于各模块。约定如下:

// ---- 历史模块 ----
// books/work/<WorkId>              → Work (bincode)
// books/physical/<PhysicalBookId>  → PhysicalBook (bincode)
// books/location/<LocationId>/<PhysicalBookId> → empty (key only, 用于位置扫描)
// traces/<TraceId>                 → LifeTrace (bincode)
// traces/location/<LocationId>/<TraceId> → empty (key only)

// ---- 魔法模块 ----
// magic/grimoire/<GrimoireId>            → Grimoire (bincode)
// magic/grimoire/location/<LocId>/<Id>   → empty
// magic/scroll/<ScrollId>                → Scroll (bincode)
// magic/scroll/location/<LocId>/<Id>     → empty
// magic/inscription/<InscriptionId>      → MagicInscription (bincode)

// ---- 物品模块 ----
// items/entity/<ItemEntId>               → ItemEnt + ItemState (bincode)
// items/sign/<SignId>                    → Sign (bincode)
// items/sign/location/<LocId>/<SignId>   → empty
```

## 四、性能预算汇总

| 组件 | 预算 | 说明 |
|------|------|------|
| ExpressionRef 拷贝 | ~1 ns | 8B Copy |
| Resolver 分发 | ~10 ns | HashMap u8 key |
| LMDB 查询 | ~1-10 µs | 索引 key-value |
| TextGenerator::generate() | ~10-50 µs | 交互触发 |
| L1 NLU (关键词) | ~2-10 µs | 交互触发 |
| L2 NLU (嵌入) v1.2+ | ~1-5 ms | ONNX 本地 |
| L3 LLM v2.0+ | ~1-2 s | 网络+推理 |
| SocialField::absorb_utterance() | ~100 ns | 浮点计算 |
| TurnAllocator::allocate_freeform() | ~1 µs | ≤10 参与者评估 |
| PhaticLayer 片段选择 | ~1-5 µs | 条件检查 + 确定性随机 |
| 片段库加载 | ~10 ms | 启动时一次性 TOML 解析 |
| **片段库内存** | **~86 KB** | ~430 片段 × 200B |
| **修辞化模板库内存** | **~8 KB** | ★ v1.1 — ~40 修辞模板 × 200B，与基座片段共存于同一 TOML |
| **rhetorical_ability() 调用** | **~100 ns** | ★ v1.1 — 纯浮点计算，无 DB 查询 |
| **domain_similarity_raw() 调用** | **~50 ns** | ★ v1.1 — u64 XOR + popcount，woworld_core 提供 |
| **KeywordIndex 内存** | **~1-5 MB** | 所有 NPC/地名/事件/物品名的 Trie |
| KeywordIndex 重建 | ~50-200 ms | 启动时或重大事件后一次性 |

**文本生成不在每帧性能预算中**——所有文本操作都是交互触发。真正的帧级瓶颈是体素、路径寻找、物理。

## 五、安全边界

```rust
// 1. ReadResult.text 是纯渲染数据——永不解析执行
//    Godot 侧直接渲染为 Label/RichTextLabel 的 text 属性

// 2. LLM 生成文本经安全网关:
//    - action_id 必须在预定义的允许集合内
//    - target entity 必须存在
//    - 不得违反 NPC 核心人格
//    - 速率限制: ≤1次/NPC/游戏日

// 3. 大日志搜索只匹配文本——不执行文本中的指令
//    JournalSearchIndex 是纯字符串匹配，不解析内容

// 4. 文本内容不经过 eval/exec/interpret
//    所有参数填充是纯字符串替换——不涉及代码执行
```
