## 数据流架构

```
Rust 模拟核心                    GDExtension              Godot UI 层
┌──────────────────┐        ┌────────────────┐        ┌──────────────┐
│ woworld_core     │────────│ gdext_bridge   │────────│ Control 节点  │
│ (类型定义)       │        │ (序列化+缓存)  │        │              │
│                  │        │                │        │ HUD.tscn     │
│ woworld_life     │──Vitals│ VitalsSnapshot │──var──│ 状态条       │
│ woworld_npc      │──Dialog│ DialogOption[] │──arr──│ 对话界面     │
│ woworld_items    │──Items │ InventorySnap  │──dict─│ 物品栏       │
│ woworld_skills   │──Skill │ SkillSnapshot  │──dict─│ 技能面板     │
│ woworld_worldgen │──Map   │ TerrainSample  │──tex──│ 地图纹理     │
│ woworld_history  │──Log   │ HistoryLog     │──dict─│ 日志面板     │
│ woworld_economy  │──Trade │ TradeSession   │──obj──│ 交易界面     │
└──────────────────┘        └────────────────┘        └──────────────┘
```

- Rust 侧：所有模拟数据以纯数据 struct 形式暴露
- GDExtension 桥接层：序列化为 Godot Variant 类型（`Dictionary`/`Array`/`PackedByteArray`）
- Godot UI 层：Control 节点消费数据，Rust 侧不参与布局/渲染

## 性能预算

### 帧预算（Godot 侧）

| 子系统 | 预算 | 说明 |
|--------|------|------|
| HUD 更新 | < 0.3ms | 状态条 + 准心 + Bark |
| 小地图更新 | < 0.5ms | 1Hz，非每帧 |
| 物品栏渲染 | < 1.0ms | 按需打开时一次性 |
| 对话界面 | < 0.2ms | 4 选项 + 文本 |
| 地图面板 | < 2.0ms | 全屏时采样 + 纹理生成 |
| 日志面板 | < 1.0ms | 查询驱动 |
| **总计** | **< 2ms/帧** | Godot 侧 UI 逻辑 |

### 内存预算

| 资源 | 预算 |
|------|------|
| UI 纹理（图标/字体/面板） | ~50 MB VRAM |
| 缓存地形纹理（小地图） | ~4 MB（256² × 4 通道） |
| 缓存物品图标 | ~10 MB（~1,000 物品 × 10KB） |

### GDExtension 桥接开销

| 操作 | 开销 | 说明 |
|------|------|------|
| Vitals 查询 | ~1μs | 5 个 f32 字段 |
| 物品栏快照 | ~50μs | 30 格 + N 容器 |
| 对话选项 | ~5μs | ≤4 选项 |
| 地形采样 | ~2ms | 256² 点 × height_at（每帧只更新脏区域） |

## UI 数据合同

所有 UI 消费数据遵循以下合同：

| 约束 | 说明 |
|------|------|
| **只读** | UI 不写入模拟数据。所有修改通过 `PlayerCommand` 消息 |
| **异步** | UI 查询不阻塞模拟。如数据未就绪则显示占位/缓存值 |
| **可降级** | 帧预算不足时 UI 自然降级——减少采样分辨率/降低刷新率/使用缓存 |
| **不缓存 NPC 内心状态** | UI 不缓存 NPC 的情绪/意图——每次查询都是最新值（保证信息诚实的边界） |
