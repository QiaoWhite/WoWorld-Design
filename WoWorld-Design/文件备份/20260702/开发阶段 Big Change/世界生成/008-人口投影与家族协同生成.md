## §〇、设计目标

100K+ NPC，每人拥有:
- 年龄、性别、BigFive 人格、技能等级、职业标签
- 在 FamilyTree（有向无环图）中的位置
- 绑定的物理锚点（`NpcAnchors`: 家/工作场所/产权）
- 初始社会地位和钱包（钱包在 P11 填充）
- 可供查询但**不预存文本**的个人史

**约束**:
- 总人口 = Σ settlement.population（来自 P4 承载容量×等级系数）
- 人口金字塔符合 Gompertz 死亡率 + 文化生育率
- 职业分布满足 P5 `RoleTemplate` 的数量约束
- 家族关系跨代一致（无时间悖论）
- 跨聚落婚姻在小聚落中自动启用

## §一、四阶段生成算法

```
Phase A — 骨架:   年龄/性别/rough_standing/gender_attraction。职业=未分配。
Phase B — 家族:   ★ 五 Pass 混合构造——祖先创建→配偶池合并→社会关系初始化→已故子女→DAG验证。
Phase C — 职业世系: 全局整数规划。满足 RoleTemplate 约束。消费完整 FamilyTree。
Phase D — 实例化: 技能+BigFive+AttractionTemplate+NpcAnchors+初始动作状态+婚姻审计。
```

### 1.1 Phase A：骨架生成

对每个聚落，从承载容量→人口总数→年龄分布直方图→按桶采样。

```rust
fn generate_skeletons(
    settlement: &SettlementData,
    culture: &CultureCoreParams,
    fertility_norms: &FertilityNorms,
    relationship_norms: &RelationshipNorms,
    rng: &mut DeterministicRng,
) -> Vec<NpcSkeleton> {
    let total_pop = settlement.population;

    // 1. 稳态人口金字塔
    let fertility_rate = fertility_norms.annual_fertility_rate();
    let (marriage_age_f, _) = FertilityNorms::marriage_age_typical(culture);
    let age_pyramid = gompertz_age_distribution(
        total_pop, fertility_rate, marriage_age_f,
        GOMPERTZ_ALPHA, GOMPERTZ_BETA, rng,
    );

    // 2. Rough standing + 取向分布
    let standing_dist = rough_standing_distribution(culture, settlement.grade);
    let orientation_dist = orientation_distribution(relationship_norms);

    // 3. 按桶采样——年龄/性别/rough_standing/gender_attraction
    let mut skeletons = Vec::with_capacity(total_pop as usize);
    for (bucket_idx, count) in age_pyramid.iter().enumerate() {
        let age_min = bucket_idx as u8 * 5;
        let age_max = age_min + 4;
        for _ in 0..*count {
            let age = rng.gen_range(age_min..=age_max);
            skeletons.push(NpcSkeleton {
                id: NpcId::from_hash(hash(seed, settlement.id, skeleton_index)),
                age,
                gender: if rng.gen::<f32>() < 0.51 { Female } else { Male },
                birth_settlement: settlement.id,
                birth_year: -1 * age as i32,
                rough_standing: sample_standing(&standing_dist, rng),       // ★ 新增
                gender_attraction: sample_orientation(&orientation_dist, rng), // ★ 新增
                profession_tags: Vec::new(),  // Phase C 填充
            });
        }
    }
    skeletons
}
```

**Gompertz 年龄分布**：`μ(x) = α × exp(β × x)`。在中世纪农业社会参数下，底部宽（高出生率+高婴儿死亡率）、顶部窄（极少活到 80+）。

### 1.2 Phase B：家族结构——五 Pass 混合构造（★ v2.1 重构）

★ v2.1 重构为五 Pass 混合构造（详见 §二～§六）：Pass 1 祖先创建（聚类合并+年龄约束）→ Pass 2 配偶池合并（小聚落跨聚落婚）→ Pass 3 社会关系初始化（统计贪心、同性伴侣、多配偶制、非婚生子女）→ Pass 4 已故子女生成 → Pass 5 DAG 验证。消除原设计的 social_standing 循环依赖和 blood_distance 不可用问题。消费 `FertilityNorms`、`RelationshipNorms`（文化系统）和 `ReligiousReproductionNorms`（信仰系统）。

```
算法: Reverse Population Projection

INPUT: 每个聚落的 Vec<NpcSkeleton> (按年龄从老到幼排序)
OUTPUT: FamilyTree (FatherOf/MotherOf/MarriedTo 边), Vec<NpcSkeleton> (含祖先)

FOR each settlement:
  1. 按年龄降序排列活人 NPC（老人先配对）

  2. 婚姻配对:
     FOR each NPC (年龄 ≥ marriage_age_min, 未配对):
       候选配偶 = 同龄±15岁, 异性, 同聚落(小聚落扩展到邻近聚落),
                  social_standing ±1 tier, blood_distance ≥ 2
       评分 = age_compatibility × standing_match × 随机噪声
       选择最高分→创建 MarriedTo 边(start_year = min(birth_year)+marriage_age)
       已婚 NPC 标记为 paired

  3. 生成子女:
     FOR each 已婚配对:
       子女数 = 泊松(文化生育率 × 结婚年数 / fertility_window)
       FOR each 子女:
         如果子女年龄 < 0 → 还未出生, 跳过
         如果子女年龄 ≥ 0 且不在活人列表中 → 已死, 加入 dead_ancestors
         如果子女在活人列表中 → 已有, 创建 FatherOf/MotherOf 边

  4. 反推祖先:
     FOR each NPC (已死或活着):
       以概率(1 - mortality_rate[parent_age_at_child_birth]) → 父母还活着
       如果活着 → 创建 FatherOf/MotherOf 边
       如果已死 → 加入 dead_ancestors（仅保留最近 2-3 代）
       递归直到: 父母年龄 < settlement_founding_year 或 递归深度 > 3

  5. 跨聚落婚姻:
     小聚落(<200人) → 30-60% 配偶来自邻近聚落
     P(跨聚落) = exp(-distance / marriage_range) × culture.exogamy_rate
     社会阶层匹配: ±1 tier
```

**跨聚落婚姻**的关键：先完成所有聚落的 Phase B.2（内部配对），然后对剩余未配对的 NPC 做跨聚落匹配。这避免了小聚落的基因瓶颈。

### 1.3 Phase C：职业世系——整数规划

**问题**: 将 N 个 NPC 分配到 M 种职业，满足 P5 RoleTemplate 的数量约束，最大化匹配质量。

```
约束:
  ∀ occupation: Σ_i assigned[i][occ] ∈ [role.count_min, role.count_max]
  ∀ npc: assigned[npc] 恰好为一个职业标签集合 (≤4个标签)
  
目标: 最大化 Σ 匹配质量
  匹配质量[npc][occ] =
    family_inheritance_bonus[npc][occ] × 0.40   // 父亲是铁匠→子女优先
    + big_five_fit[npc][occ] × 0.35              // 人格适配度
    + settlement_need_urgency[occ] × 0.25         // 聚落缺人程度
```

**算法**: 最小费用最大流。

```rust
fn big_five_fit(npc: &NpcSkeleton, occupation: ProfessionTagId) -> f32 {
    // 铁匠: conscientiousness↑, openness↓ → 高分
    // 商人: extraversion↑, agreeableness↓ → 高分  
    // 冒险者: openness↑, neuroticism↓ → 高分
    // 映射表在 ProfessionTag TOML 中定义 (big_five_profile 字段)
}

fn family_inheritance_bonus(npc: &NpcSkeleton, occupation: ProfessionTagId) -> f32 {
    if let Some(father) = family_tree.father_of(npc.id) {
        if father.has_tag(occupation) { return 0.8; }  // 强继承
    }
    if let Some(mother) = family_tree.mother_of(npc.id) {
        if mother.has_tag(occupation) { return 0.5; }  // 弱继承（母系）
    }
    0.0
}
```

**继承强度调制**: `actual_inheritance = base_inheritance × culture.occupation_inheritance_strength`。高传统社会→严格世袭。低传统社会→子女自由选择。

**已故祖先的职业查询**（★ v2.1 注意）：Phase C 仅给活人 NPC + 存活祖先分配 `profession_tags`。Pass 1 创建的已故祖先没有职业标签。`family_inheritance_bonus` 对已故祖先使用 fallback：从此人 `rough_standing` 对应的聚落历史职业分布中加权采样（与懒推导 `query_personal_history()` 逻辑一致）。即父亲是已故祖先时，`family_inheritance_bonus` = `sample_ancestor_profession_match(father.rough_standing, occupation, settlement, rng)`。不新增数据结构——纯函数查询。

## §五、大五人格分配

```rust
fn generate_big_five(
    culture: &CultureCoreParams,
    occupations: &[(ProfessionTagId, f32)],
    rng: &mut DeterministicRng,
) -> BigFive {
    // 1. 文化基线 (CultureCoreParams → 该文化人群的人格均值)
    let mut base = culture.big_five_baseline();

    // 2. 个体偏移 (高斯噪声 σ=0.15)
    for dim in base.iter_mut() {
        *dim = (*dim + rng.gen_normal(0.0, 0.15)).clamp(0.0, 1.0);
    }

    // 3. 职业修正 (从 ProfessionTag TOML 的 big_five_profile 字段)
    for (tag, _) in occupations {
        if let Some(profile) = tag_def(*tag).big_five_profile {
            base.openness            += profile.openness_mod * 0.1;
            base.conscientiousness   += profile.conscientiousness_mod * 0.1;
            base.extraversion        += profile.extraversion_mod * 0.1;
            base.agreeableness       += profile.agreeableness_mod * 0.1;
            base.neuroticism         += profile.neuroticism_mod * 0.1;
        }
    }

    // 4. 年龄修正: young→openness↑, old→conscientiousness↑
    //    所有修正乘法叠加，重归一化到 0-1

    base.normalize()
}
```

**无硬编码**: 职业→人格映射全在 ProfessionTag TOML 中。

## §六、技能分配

```rust
fn generate_skills(
    occupations: &[(ProfessionTagId, f32)],
    age: u8,
    settlement_tech: &SettlementTechState,
    big_five: &BigFive,
    rng: &mut DeterministicRng,
) -> Vec<SkillEntry> {
    // 1. 从 ProfessionTag TOML 的 related_skills → 确定关联技能
    // 2. 技能天花板 = settlement_tech.tech_levels[domain] / 100.0
    //    （该聚落该技艺的最高水平——来自 KnowledgeSeed 传播）
    // 3. 年龄进度曲线:
    //    学徒期(12-18): 0.1 → 0.4
    //    熟练期(18-35): 0.4 → 0.85
    //    维持期(35-55): 0.85 ± 0.05
    //    下降期(55+):   0.85 → 0.7
    // 4. 天赋修正 (三层天赋: 天生×交叉训练×MentalAccess)
    // 5. 天花板裁剪: skill = min(raw_skill, ceiling)

    occupations.iter().flat_map(|(tag, proficiency_base)| {
        let def = tag_def(*tag);
        def.related_skills.iter().map(|(skill_id, weight)| {
            let raw = proficiency_base * weight * age_progress_curve(age, skill_id);
            let ceiling = settlement_tech.tech_levels
                .get(&def.tech_domain).unwrap_or(&30) / 100.0;
            let talent_mod = compute_talent(big_five, skill_id, rng);
            SkillEntry {
                skill_id: *skill_id,
                level: (raw * talent_mod).clamp(0.0, ceiling as f32),
            }
        }).collect::<Vec<_>>()
    }).collect()
}
```

## §九、Phase B Pass 1-5 详细设计（★ v2.1 新增）

### Pass 1：祖先创建

核心思想：对每个缺少父母的活人 NPC，向上创建父母实体。同胞的父母通过聚类合并共享同一组父母。递归深度 ≤ 3 代（~75 年），截止于 `settlement.founding_year`。

**步骤**：
1. **收集父母签名**：对每个无父母的活人 NPC，计算预期母亲出生年份 = `NPC.birth_year - marriage_age_typical.female`，预期父亲出生年份同理。收录为 `(child_idx, ParentSignature { birth_year_approx, settlement_id, rough_standing, gender })`。
2. **聚类合并**：动态聚类半径（小聚落 3 年 / 大聚落 1 年），合并 `birth_year_approx` 相近、同性别、同 rough_standing 的父母签名。合并后验证子女数 ≤ 15（生物学上限），超过则分裂。
3. **创建实体**：对每组合并签名，创建父母 NPC 实体（`birth_year` ± 2 年抖动以去重，`death_age` 由 Gompertz 预采样）。创建 `FatherOf`/`MotherOf` 边。
4. **年龄验证**：`parent.birth_year < child.birth_year - 14`。失败 → 放弃该祖先（链终止）。
5. **递归**：对父母的父母重复上述过程。停止条件：`birth_year < settlement.founding_year` 或递归深度 > 3。

**修复的原有 Bug**：Bug 2（blood_distance→Pass 1 后可用）、Bug 3（子女认领→父母在 Pass 1 创建，不被 Pass 4 覆盖）、Bug 4（死亡率→查当前年龄非出生时年龄）、Bug 5（递归停止→`birth_year < founding_year` 非年龄 < 年份）、Bug 7（祖先年龄→Gompertz 预采样）。

### Pass 2：配偶池合并

小聚落（<200 人）将邻近聚落的适龄成年人（≥16 岁）合并入本聚落候选池。合并概率 `P = exp(-distance_km / 20) × (200 - pop) / 200`。"邻近"定义：同一贸易路线相邻节点，或直线距离 < 20km（来自 P7 TransportNetwork）。

### Pass 3：社会关系初始化

贪心 O(N) 统计分配。**不优化个体匹配质量**——运行时 GOAP 求爱/离婚/再婚会覆盖生成时的初始婚姻。

**输入**：Pass 1 FamilyTree（血亲关系） + Pass 2 扩展池 + `RelationshipNorms`（文化系统） + `ReligiousReproductionNorms`（信仰系统） + `rough_standing` + `gender_attraction`。

**规则**（按优先级）：
1. **生物学父母结婚**：`conjugal_birth_norm ≥ 0.8` → 确定性 MarriedTo。`< 0.8` → 概率性。
2. **异性婚配**：基线 70% 成年人已婚 [TUNING]。候选条件：异性、年龄 ±15、rough_standing ±1、`blood_distance ≥ 2`、跨信仰兼容。
3. **同性伴侣**：仅取向兼容者（Homo/Bi）。概率 = `homosexuality_acceptance × 0.3`。如果 `same_sex_marriage_legal > 0.5` → MarriedTo，否则 → PartneredTo。
4. **多配偶制**：已婚 NPC 按 `polygyny_acceptance`（男）/`polyandry_acceptance`（女）概率添加额外配偶，最多 `max_spouses` 个。
5. **配偶池枯竭处理**：最后未配对者，扩展阶层约束至 ±2，放松取向约束。
6. **跨聚落迁移**：跨聚落配偶 → 按 `ResidencePattern` 决定迁移方向 → 更新聚落人口计数。
7. **独身要求**：不在生成时处理（交予运行时——Phase C 后才知祭司等独身职业）。

**更新的人口公式**（P13 消费）：`实际人口 = count(NPC where home.settlement == settlement)`，非 `count(NPC where birth_settlement == settlement)`。

### Pass 4：已故子女生成

消费 Pass 1 的生物学父母对。对每对父母，生成已故子女（死产/夭折/已故同胞）。

**步骤**：
1. 对每对生物学父母，计算婚姻窗口：`start = max(mother.birth_year, father.birth_year) + 18`；`end = min(mother.death_year, father.death_year, 0)`。
2. 总预期子女数 = Poisson(`fertility_rate × (end - start)`)。
3. 已故子女数 = 总预期 - 已存活的子女数（从 FamilyTree 查询）。
4. 对每个已故子女，采样出生年份 ∈ `[start, end)`。
   **★ 约束**：出生年份必须 ≤ 母亲 death_year 且 ≤ 父亲 death_year（严格小于）。出生年份必须 ≥ 母亲 birth_year + 14 且 ≥ 父亲 birth_year + 14。
5. 创建已故子女实体（`birth_year` + `death_age` 由 Gompertz 预采样，`death_cause = Natural`——P13 后处理可覆盖）。
6. 创建 `FatherOf`/`MotherOf`/`SiblingOf` 边。

### Pass 5：DAG 验证

1. **无环检查**：`family_tree.detect_cycle()` → 致命错误。
2. **对称性**：每条 `MarriedTo`/`PartneredTo` 边必须有反向边。
3. **父母年龄**：`FatherOf(a,b)` 或 `MotherOf(a,b)` → `a.birth_year < b.birth_year - 14`。

失败处理：致命错误 → 加载画面显示"世界种子无效"，建议换种子。

## §十、Phase D 婚姻质量审计（★ v2.1 新增）

Phase C 后，`social_standing`（精确值）已知。对比 Phase A 的 `rough_standing`（估计值）：

```rust
fn audit_marriage_quality(skeletons: &[NpcSkeleton], family_tree: &FamilyTree) -> Vec<MarriageFlag> {
    let mut flags = Vec::new();
    for edge in family_tree.edges_of_type(FamilyRelation::MarriedTo) {
        let a = find_skeleton(skeletons, edge.from);
        let b = find_skeleton(skeletons, edge.to);
        let gap = (a.social_standing as i32 - b.social_standing as i32).abs();
        if gap > 1 {
            flags.push(MarriageFlag::CrossTier {
                npc_a: edge.from, npc_b: edge.to, gap,
                narrative: "跨阶层婚姻——叙事素材"
            });
        }
    }
    flags
}
```

`MarriageFlag` 作为 WorldState 元数据写入 LMDB，供传说系统和八卦系统消费。80% 保留、20% 由 Phase D 重配（选择 ±1 tier 内的替代配偶）。
