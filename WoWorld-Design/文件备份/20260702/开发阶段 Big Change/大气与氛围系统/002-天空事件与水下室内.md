## 一、SkyEvent —— 叙事性天空覆盖

### 1.1 设计哲学

SkyEvent 不由中心化的"事件管理器"调度。它由**实体的存在**决定——任何实体可以携带 `AtmosphericPresence`，氛围系统在每帧扫描这些实体，取最强影响者作为天空覆盖。

龙出现在天空 → 天空自动变色。魔法系统不需要调用大气系统。零模块依赖。

### 1.2 数据流

```
产生方（魔法/信仰/战斗模块）:
  实体携带 AtmosphericPresence { event_type, intensity, position }
  → atmosphere.register_sky_event(entity_id, event_type, intensity, position)
  → 实体消失时: atmosphere.unregister_sky_event(entity_id)

消费方（大气系统）:
  每帧: 扫描已注册的 SkyEvent 源
  → 按 intensity × falloff(distance) × priority 排序
  → 取最强 → TOML 查参数 → lerp 到 ResolvedAtmosphere
```

### 1.4 实现

```rust
pub struct SkyEventManager {
    sources: Vec<SkyEventSource>,
    config: SkyEventsConfig,  // sky_events.toml
}

struct SkyEventSource {
    entity_id: EntityId,
    position: Vec3,
    event_type: SmolStr,
    intensity: f32,           // 0-1
}

impl SkyEventManager {
    pub fn register(&mut self, entity_id: EntityId, event_type: &str,
                    intensity: f32, position: Vec3) {
        if let Some(s) = self.sources.iter_mut().find(|s| s.entity_id == entity_id) {
            s.intensity = intensity;
            s.position = position;
            return;
        }
        self.sources.push(SkyEventSource { entity_id, position,
            event_type: event_type.into(), intensity });
    }

    pub fn unregister(&mut self, entity_id: EntityId) {
        self.sources.retain(|s| s.entity_id != entity_id);
    }

    pub fn resolve(&self, player_pos: Vec3) -> Option<ResolvedSkyEvent> {
        let mut best: Option<(f32, &SkyEventDef)> = None;
        for source in &self.sources {
            let def = match self.config.get(&source.event_type) {
                Some(d) => d,
                None => {
                    log::warn!("SkyEvent '{}' not found in sky_events.toml", source.event_type);
                    continue;
                }
            };
            let dist = (source.position - player_pos).length();
            if dist > def.falloff_m { continue; }
            let effective = source.intensity * (1.0 - dist / def.falloff_m) * def.priority;
            if best.map_or(true, |(s, _)| effective > s) {
                best = Some((effective, def));
            }
        }
        best.map(|(strength, def)| ResolvedSkyEvent {
            params: def.params.clone(),
            blend_factor: strength.min(1.0),
        })
    }
}
```

### 1.5 调用方式

SkyEvent 是氛围/天气模块的**内部方法**——不是独立 trait。其他模块直接调用 `register`/`unregister`。具体结构体依赖——solo 开发可接受。如果将来需要解耦，提取 trait 到 woworld_core——两小时的事。

### 1.6 过渡

SkyEvent 不是硬切。`blend_factor` 由 `resolve()` 计算——在 LODCoordinator 过渡时序中 lerp。龙出现 → blend 从 0 到 1（跟随龙的出现动画）。龙离开 → blend 从 1 到 0（跟随龙的消失）。和已有的 LOD 过渡机制一致。

## 二、水下深度分层

### 2.1 已消费的接口

已有 `OceanProvider` trait（世界生成 §4.7）：
```rust
fn wave_height(&self, x: f64, z: f64, time: f64) -> f64;
fn depth_query(&self, x: f64, z: f64) -> f64;
```

已有海洋深度分层（Life 003/005）：epipelagic(0-200m)、mesopelagic(200-1000m)、bathypelagic(1000-4000m)、abyssopelagic(4000m+)。

### 2.2 水下 TOML

```toml
# underwater.toml
[depth.epipelagic]     # 0-200m
fog_color = [0.15, 0.35, 0.55]
fog_density = 0.6
saturation_mod = 0.7

[depth.mesopelagic]    # 200-1000m
fog_density = 0.85
saturation_mod = 0.3
ambient_tint = [0.02, 0.05, 0.12]

[depth.bathypelagic]   # 1000-4000m
fog_density = 0.98
saturation_mod = 0.0
ambient_tint = [0.0, 0.01, 0.03]
bloom_threshold = 0.3

[depth.abyssopelagic]  # 4000m+
fog_density = 1.0
floor_brightness = 0.001
```

### 2.3 过渡

```
水面高度 = ocean.wave_height(x, z, time)  // 动态——波浪起伏

如果玩家在水面上方 → 正常室外氛围
如果玩家在水面下方:
  depth = water_height - pos.y
  transition_zone = 0.5m
  t = (depth / 0.5).min(1.0)  // 0→1 穿过水面近端
  
  depth > 0.5m → 完全水下 → 选择深度带 → 带间线性插值
  depth < 0.5m → 过渡带 → lerp(室外, 水下, t)
```

水面是动态的（波浪）→ 进出水面时氛围平滑过渡。和已有的 Gerstner 波视觉同步。

## 三、室内氛围

### 3.1 数据流

```
Godot 每帧:
  camera_pos → Rust BuildingQuery::building_at(camera_pos) → Option<BuildingId>
  
  if camera inside building.bounds:
    取 building 内部氛围参数
    indoor_atm = derive_from_lights(active_interior_lights)
    atmosphere = lerp(outdoor_atm, indoor_atm, transition_t)
    // transition_t = 门口距离 / 2.0m
```

### 3.2 室内 ≠ TOML——光源派生

室内氛围不由 TOML 静态定义。Godot 已有所有室内光源数据——AreaLight3D（窗户光）、OmniLight3D（壁炉/蜡烛）。室内氛围从光源**自动派生**：

```
indoor_atm:
  ambient_tint    = 光源平均色温 × 平均强度 → [暖橙, 暖白]
  shadow_tint     = 主光源色温的暗端（暖棕/暗红）
  fog_density     = 0.005（仅微尘——室内清晰）
  aerial_blur     = 0.0（室内无远景）
  sky_*           = 不适用
  bloom_threshold = 0.45（烛光更容易 bloom——比室外 0.7 低）
  rim_strength    = 0.05（微弱——室内多方向光线消减边缘光）
```

### 3.3 Rust 参与的边界

Rust 仅做几何判定——"玩家在哪个建筑里"（已有 `BuildingQuery::building_at`）。光照派生完全在 Godot。
