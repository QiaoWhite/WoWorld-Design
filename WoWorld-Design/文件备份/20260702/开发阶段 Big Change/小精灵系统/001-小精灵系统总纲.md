## 〇、定位

小精灵是玩家的**可选辅助伙伴**。她不是教程——如果玩家选择关闭，她完全消失。她不是"任务系统"的替代品——她不给任务、不给目标、不奖励。她只是**一个不离不弃的、只能由玩家感知到的小帮手**。

设计参考：类似《原神》的派蒙——但更轻量、更可退场。只有玩家角色可以看到/听到她。

---

## 一、存在形式

### 1.1 物理模式 vs 纯声模式

```rust
pub enum FairyPresence {
    /// 3D 模型存在于空间场景中，占物理位置，有碰撞（但不影响 NPC/敌人）
    Physical { position: Vec3, visible: bool },
    /// 纯"声音"——无 3D 模型，以 UI 文字或音频形式交流
    VoiceOnly,
    /// 完全消失——不交流、不提示
    Hidden,
}
```

- 玩家可以随时切换：`Physical` → `VoiceOnly` → `Hidden`
- `Physical` 模式下：小型飞行实体跟随玩家，可停在玩家肩上/背包上/附近
- `VoiceOnly` 模式下：只有 UI 文字或音频（如果玩家启用了语音）
- NPC 完全不可感知——不能看到、不能听到、不能互动

### 1.2 3D 模型

- 小型飞行实体
- 手搓模型（造物——开发者制作的特别生物，每个世界都有）
- 位置：跟随玩家飞行——不是固定位置，有随机的微小浮动动画

## 二、数据模型

```rust
/// 小精灵的完整状态
pub struct FairyCompanion {
    pub enabled: bool,
    pub presence: FairyPresence,

    /// 行为配置
    pub config: FairyConfig,

    /// 当前状态
    pub state: FairyState,
}

#[derive(Debug, Clone)]
pub struct FairyConfig {
    /// 是否主动提供帮助（vs 仅在玩家询问时回应）
    pub proactive_help: bool,
    /// 提示频率 0-1（0=极其沉默，1=话痨）
    pub verbosity: f32,
    /// 是否使用 LLM（和 NPC 的 LLM 开关共享主开关 + 独立控制）
    pub use_llm: bool,
    /// 提示类型开关
    pub survival_reminders: bool,      // "你饿了"
    pub combat_warnings: bool,         // "这个敌人太强"
    pub environmental_observations: bool, // "这里好像很冷"
    pub social_hints: bool,            // "那个人在找你"
}

#[derive(Debug, Clone)]
pub struct FairyState {
    /// 上次提示的时间戳——避免重复提醒
    pub last_survival_reminder: Option<GameTimestamp>,
    pub last_danger_warning: Option<GameTimestamp>,
    /// 当前关注的事物
    pub attention_focus: Option<FairyAttention>,
}

pub enum FairyAttention {
    PlayerVitals,         // 关注玩家生理状态
    NearbyNpc(EntityId), // 关注某个 NPC
    EnvironmentalThreat,  // 关注环境危险
    JournalEntry(JournalEntryId), // 提到了大日志中的某条记录
}
```

## 三、知识边界

### 3.1 她知道的事

- **世界基本运作机制**：物理规则、魔法体系、经济原理、文化概念、语言结构。相当于"读了游戏说明书"但没读过"这个存档的历史书"
- **大日志内容**：可以检索玩家角色已知的一切信息。"你三个月前在北山见过一个铁匠。"

### 3.2 她不知道的事

- **这个存档内的具体内容**：不知道铁炉镇在哪个方向、不认识老王、不知道龙还在不在北山。只能引用大日志中的已有记录
- **未探索的区域**：不知道山那边有什么
- **NPC 的内心想法**：不知道老王对玩家的 trust 是多少——除非玩家角色自己观察到了
- **未来的事**：不知道明天的天气、不知道敌人什么时候来

## 四、行为触发

### 4.1 主动提示类型

小精灵在以下条件触发生成提示（如果 `proactive_help` 开启）：

| 触发条件 | 提示类型 | 冷却 |
|---------|---------|------|
| 玩家饥饿 < 0.3 | "你该找点吃的了" | 30 分钟 |
| 玩家口渴 < 0.25 | "你该喝点水了" | 30 分钟 |
| 玩家疲劳 > 0.85 | "你需要休息" | 30 分钟 |
| 附近怪物强度 > 玩家战斗力的 1.5 倍 | "这个对手太强了" | 15 分钟 |
| 有 NPC 的 DialogueIntent 指向玩家（且未被玩家感知） | "那边那个人好像想跟你说什么" | 15 分钟 |
| 极端天气逼近（雷暴/暴风雪） | "天要变了，最好找个地方避一避" | 每小时 |
| 玩家在极冷/极热区 > 10 分钟 | "这里太冷/热了" | 20 分钟 |
| 玩家在有毒气区域 | "呼吸...有点困难" | 5 分钟 |
| 潜水时氧气不足 | "该上去了" | 5 分钟 |
| 玩家已经 3 天没和任何人说话 | "你很久没跟人说话了" | 每天 |

提示文本在非 LLM 模式下从预定义模板拼装。LLM 模式按玩家的 LlmSceneConfig 调用 `FairyDialogueEngine`。

### 4.2 被动回应——玩家问问题

玩家的对话输入（和 NPC 对话同一套 `PlayerInput` 机制）发送给小精灵。小精灵能回答：

| 问题类型 | 回答来源 |
|---------|---------|
| "这附近有铁匠吗" | 大日志查询——"你三个月前在北山见过一个" |
| "这是什么植物" | 大日志查询——如果识别过 → 答名。未识别 → "我也不认识" |
| "我该去哪里" | 不答——不做目标引导。最多"你想去哪里？" |
| "你觉得老王这人怎么样" | 大日志查询——玩家和老王的互动记录。不评价——"从你的记录来看，他帮过你三次，有一次骗了你" |

### 4.3 始终不会做的事

- 不给目标、不给任务、不给奖励
- 不评价玩家行为（"你做得对""你不该这样"）
- 不透露 meta 信息（"把镜头转过去""按 X 键"）
- 不说话时保持安静——不刷存在感

## 五、对话接口

### 5.1 和 NPC 对话共用管道

小精灵的对话走的和 NPC 对话同一套 `PlayerInput` → `TopicResolver` → `NpcResponse` 管道：

```rust
/// 和小精灵对话——复用对话系统
impl FairyCompanion {
    pub fn process_turn(
        &mut self,
        player_input: PlayerInput,
        journal: &GrandJournal,
        llm: &Option<LlmOrchestrator>,
        ctx: &FairyDialogueContext,
    ) -> FairyResponse {
        match player_input {
            PlayerInput::FreeText { text, .. } => {
                // 检查 LLM 是否可用 + 玩家是否启用了小精灵 LLM
                if let Some(llm) = llm {
                    if self.config.use_llm && llm.is_active() {
                        return self.llm_response(text, journal, ctx);
                    }
                }
                self.template_response(text, journal, ctx)
            }
            // 预设选项（和 NPC 对话共用同一套）
            _ => self.template_response_from_option(player_input, journal, ctx),
        }
    }
}
```

### 5.2 FairyDialogueContext

```rust
pub struct FairyDialogueContext {
    /// 玩家角色快照
    pub player_snapshot: LifeEntitySnapshot,
    /// 玩家附近环境

    pub nearby_threats: Vec<(EntityId, f32)>, // (实体, 强度)
    pub weather: WeatherSnapshot,
```

## 六、LLM 增强

### 6.1 和 NPC LLM 共享基础设施

小精灵的 LLM 对话走语言表达/011 的同一套 `LlmOrchestrator`：

- 后端：和 NPC 共享 `LlmBackendRegistry`
- Token 预算：和玩家对话场景共享每日限额
- 场景标签：`"fairy_companion"`
- 失败回退：LLM 不可用 → 模板化回应

### 6.2 LLM System Prompt 结构

```rust
fn build_fairy_system_prompt(journal: &GrandJournal, ctx: &FairyDialogueContext) -> String {
    format!(
        "你是一个小精灵，只有玩家可以感知到你。\
         你知道这个世界的运作方式，但你对这个世界当前的具体状态了解有限。\
         你可以访问玩家的大日志。\
         \
         你的个性：温暖、有点俏皮、偶尔开玩笑，但不做评价。\
         你不会给玩家目标或任务。你不会评价玩家的选择。\
         你不会提到任何'游戏机制'或'meta'概念。\
         \
         当前环境：{}\
         玩家状态：饥饿{}、疲劳{}、健康{}\
         附近有{}个人",
        ctx.weather_description(),
        ctx.player.hunger, ctx.player.fatigue, ctx.player.health,
        ctx.nearby_npcs.len()
    )
}
```

### 6.3 非 LLM 模式

预定义模板 + 大日志索引。从 ~50 个 Fragments 中按触发条件 + 确定性选择器拼装模板文本。和 PhaticFragment 同构——只是提示类型不同。

## 七、和大日志的接口

小精灵对大日志是**只读消费者**：

```rust
impl FairyCompanion {
    /// 查询大日志——玩家问"这附近有铁匠吗"
    fn query_journal(
        &self,
        query: &str,
        filters: SearchFilters,
        journal: &GrandJournal,
    ) -> Vec<JournalEntrySummary> {
        journal.search(query, filters)
            .into_iter()
            .take(3)  // 最多返回 3 条
            .map(|id| journal.summary(id))
            .collect()
    }
}
```

## 八、性能

- 不启用（Hidden 模式）→ 零成本
- 模板模式（无 LLM）→ < 0.01ms/触发检查
- LLM 模式 → 和 NPC 对话共享 Token 预算。玩家可控开关

## 九、和 NPC 对话系统的区别

| 维度 | NPC | 小精灵 |
|------|-----|--------|
| 物理存在 | 实体、可被所有角色感知 | 仅玩家可见 |
| SocialField | 参与群体场 | 不影响任何 SocialField |
| 情绪感染 | 双向感染 | 不参与 |
| 关系数据 | 有 Relationship | 无——恒定温暖 |
| LLM | 按 NpcDialogueWillingness 控制深度 | 玩家开关控制 |
| 话题 | 来自记忆/个性/当前情绪 | 来自大日志 + 环境 + 玩家生理状态 |
