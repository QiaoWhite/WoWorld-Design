## §一、从 O(1) 查表到碰撞扫掠

### 1.1 变更

当前 005/012 的命中判定: "攻击方向+闪避方向→O(1)几何查表→击中部位"。替换为:

**武器碰撞体沿轨迹扫掠 → 与目标身体碰撞体求交 → 命中判定。**

### 1.2 变更动机

O(1) 查表无法处理:
- 非人形目标（龙、巨型蜘蛛、史莱姆——身体碰撞体形状差异巨大）
- 非 Rigid 武器（连枷的链段可能绕过盾牌打到后背）
- 环境交互（武器碰到墙壁/柱子→弹刀/卡刀）
- 不同武器同一基元的物理差异（长剑刺和匕首刺的碰撞体完全不同）

### 1.3 与现有基元的关系

10个招式基元（刺/斩/撩/劈/扫/点/崩/缠/引/化）**全部保留**——它们仍然是 STRIKE 原子的参数化标签。碰撞扫掠只改变"怎么判定命中"——不改变"选什么基元"。

## §二、武器碰撞体规范

武器碰撞体由本文件（战斗系统）权威定义。物理原子层 007 提供碰撞扫掠的算法实现。

### 2.1 从 ItemParams 派生

武器碰撞体的形状和尺寸从 ItemAssembly 组件树的 `ItemParams` 自动派生:

| 组件类型 | 碰撞体形状 | 派生公式 |
|---------|-----------|---------|
| **StrikingHead** (0x0101) | capsule(length, radius) | length = `params["length"]`, radius = `params["thickness"] / 2` |
| **StrikingHead** (锤) | sphere(radius) | radius = `params["diameter"] / 2` |
| **BowLimb** (0x0201) | thin capsule | 不参与攻击碰撞（弓的伤害来自弹药） |
| **WhipBody** | N micro-capsules | N = length / segment_size, radius 从 root 向 tip taper |
| **ChainLink** | N micro-capsules | N = `params["links"]`, radius = `params["link_length"] * 0.3` |

**关键**: `ItemParams` 的数值本身由物品系统在物品实例化时根据 Quality × 材料密度 × 配方生成。

### 2.2 武器类型→可用基元表（不变）

现有 005 §三 的映射表保持不变——碰撞体形状从组件自动派生，不改变"斧不能用刺"的物理约束。

### 2.3 非 Rigid 武器基元——首次正式定义

005 §四 标记为"尚无动作基元映射"的非 Rigid 武器基元:

| 基元 | 对应 JointType | 物理实现 |
|------|---------------|---------|
| **甩** (Centrifugal) | Hinge, Chain | 柄端加速→链段角速度累积→锤头高动能。PBD 约束+碰撞扫掠 |
| **缠** (Entwine) | Chain, Flexible | 链/鞭段碰撞体包络目标肢体→约束数+包络弧度→缠绕判定 |
| **卷** (Coil) | Flexible | 鞭段连续接触面→卷曲约束→持续压力 |
| **锁** (Lock) | Chain | 链段穿过缝隙+反向拉紧→力矩约束→关节锁定 |
| **绞** (Strangle) | Flexible, Chain | 鞭/链段围绕颈部→窒息判定(O₂ deficit accumulation) |
| **抡** (Circular Momentum) | Chain | 持续旋转→角动量累积→离心力放大锤头速度 |

这些基元加入 005 §三 的武器→基元映射表。

## §三、CombatStyleParams（6维）→ IK 调制

战斗系统定义 6 个风格维度（来自 007 §三）。每个维度连续调制 IK 解算:

| 维度 | 范围 | IK 调制 |
|------|------|---------|
| **进攻性** | 保守→压制 | 攻击频率↑ → 轨迹时间缩放 ×0.7 |
| **风险** | 谨慎→冒险 | body_commitment ↑ → COM 前移量 ×2 |
| **移动** | 固守→机动 | Foot IK 步频↑ → MOVE 原子消耗↑ |
| **目标** | 最近/最弱/最威胁/指挥官 | gaze IK 锁定方向 → OBSERVE 焦点 |
| **资源** | 节约→全力 | STRIKE force ×1.5, stamina 消耗 ×2 |
| **队内角色** | 前线/侧翼/后卫 | guard_height 偏移, 身体朝向约束 |

**关键**: 6 个维度不是"6 种预设动画"——是 6 维连续参数空间。每个维度驱动 IK 解算的不同参数，任何组合产生独特的视觉风格。

## §四、动画管线

**零预设战斗动画。** 完整 IK 链路:

1. STRIKE 轨迹关键帧 → Hand IK(2-bone analytical)
2. Hand 位置 → Elbow → Shoulder (2-bone analytical ×2, 双臂)
3. Shoulder rotation → Spine Bridge(7 bones slerp distribution)
4. COM shift = body_commitment × strike_force × weapon_weight
5. Foot IK → 腿 2-bone analytical → 脚世界锁定

详见 [[../../NPC活人感模块/08-NPC行动涌现与分类/007-碰撞箱战斗与IK动画管线#§五、IK 动画管线|物理原子层 007 §五]] 的完整链路。

**CombatStyleParams 的调制**: flourish（手腕旋转 sin 波振幅）、guard_height（肘部+肩部默认高度）、body_commitment（COM偏移量）、speed_over_power（轨迹时间缩放）——全部从 6 维度连续派生。

## §五、性能

- 碰撞扫掠: ≤0.1ms/帧（100 NPC 战斗）
- IK 解算: 零增量（在骨骼 ≤0.5ms 预算内，比预设动画快 ~9×）
- 非 Rigid 武器 PBD: ≤0.3ms（动态降级）
