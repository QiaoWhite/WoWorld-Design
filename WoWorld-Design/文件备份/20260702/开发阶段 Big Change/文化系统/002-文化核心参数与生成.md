## 一、CultureCoreParams 完整定义

### 1.1 参数表

10 个核心参数（0.0-1.0 范围）是文化系统的不可再分原子。所有其他文化特征从这些参数派生。

| # | 参数 | 低值 | 高值 | 消费者示例 |
|---|------|------|------|-----------|
| 1 | `individualism` | 集体优先——家族/氏族为基本单位 | 个人优先——个体成就和责任至上 | 权力（合法性中的个人/集体责任观）、NPC（GOAP 目标中家族 vs 个人权重）、繁衍系统（家族压力强度） |
| 2 | `power_distance` | 平等主义——质疑权威是正常的 | 严格等级——对上级有严格的礼仪和服从 | 权力（合法性文化契合度公式中最重要参数）、语言表达（敬语系统的触发元）、NPC（对权力者的社会反馈） |
| 3 | `uncertainty_avoidance` | 包容变化——新事物=机会 | 崇尚传统——新事物=威胁 | 权力（法律规范的刚性）、经济（市场规制的弹性）、魔法（魔法禁忌的严格度）、TechnnologyProfile（创新阻力） |
| 4 | `competition_orientation` | 合作/关怀主导 | 竞争/成就主导 | NPC（社交行为中的竞争 vs 合作倾向）、权力（权力争夺的积极性）、战斗（流派文化特性——竞争高→喜欢挑战强者） |
| 5 | `long_term_orientation` | 活在当下——及时行乐 | 长远规划——延迟满足 | TechnologyProfile（所有技术领域的创新动力基数）、经济（储蓄率偏好）、NPC（生命目标规划跨度） |
| 6 | `indulgence` | 克制欲望——自律是美德 | 享受生活——追求快乐 | NPC（休闲行为频率）、经济（奢侈品消费偏好）、语言表达（情感表达度不受抑制）、饮食偏好（享乐 vs 简朴） |
| 7 | `openness_to_outsiders` | 排外/怀疑——陌生人是威胁 | 开放/好奇——陌生人是机会 | NPC（对外来 NPC/玩家的初始态度）、权力（外交中的开放度）、经济（贸易开放度）、语言表达（对外来口音的容忍度） |
| 8 | `religiosity` | 世俗化——宗教是个人私事 | 宗教渗透一切 | 生命（神明/信仰分布加权）、权力（神权合法性的基础）、魔法（魔法师的宗教身份——法师=祭司 vs 法师=学者）、建筑（神殿面积占比） |
| 9 | `militarism` | 和平主义——暴力是最后手段 | 军事优先——征服是荣誉 | 战斗（流派文化特性——军事高→实用高效招式）、权力（征服合法性恢复速度）、历史（战争事件频率修正） |
| 10 | `artistry` | 功能优先——"够用就行" | 美感优先——建筑/服饰/日用品都有艺术追求 | 世界生成（建筑装饰密度、色彩丰富度）、物品（工艺品的审美属性权重）、家具（装饰复杂度）、NPC（艺术相关行为的权重） |

### 1.2 参数正交性

10 个参数被设计为近似正交——任意两个参数的组合不应有"不可能出现"的限制：

- 一个文化可以同时是 `individualism=0.9` 且 `religiosity=0.9`（高度个人化的宗教体验——如新教伦理）
- 一个文化可以同时是 `power_distance=0.1` 且 `militarism=0.9`（平等的军事民主——如维京式全民皆兵但商议决策）
- 一个文化可以同时是 `uncertainty_avoidance=0.9` 且 `openness_to_outsiders=0.9`（对外来者友好但内部高度传统——如"客人至上"文化）

**外部约束**（环境、历史）在派生层施加——核心参数本身不设互斥规则。

## 二、文化种子生成

### 2.1 世界生成管线中的位置

文化生成在**世界生成 P2.5** 执行——在 P2（陆地自然基底）完成之后、P3（资源分布）之前：

```
P0: 大陆架构 → P1: 海洋系统 → P2: 自然基底
  → ★ P2.5: 文化生成
    2.5a: 起源点生成
    2.5b: 障碍 Voronoi 区域划分
    2.5c: CultureCoreParams 生成
    2.5d: 文化边界细化
    2.5e: 第一级派生计算
    2.5f: 语言分配
  → P3: 资源分布 → P4: 聚落选址 → ...
```

### 2.2 P2.5a：起源点生成

```rust
/// 文化起源点生成
/// 输入: world_seed, LandSeaMask, TerrainBase（P2 产物）
/// 输出: 3-8 个文化起源点的世界坐标
fn generate_culture_origins(
    seed: u64,
    mask: &LandSeaMask,
    terrain: &TerrainBase,
) -> Vec<WorldPos> {
    let mut rng = stage_rng(seed, "culture_origins");
    
    // 起源点数量 = f(大陆面积, 大陆数)
    let land_area_km2 = mask.land_area();
    let continent_count = mask.continent_count();
    let base_count = (land_area_km2 / 15000.0).ceil() as u32;  // 每约15,000km²一个起源点
    let count = base_count.clamp(3, 8).min(continent_count * 3);
    // 每个大陆至少1个，最多3个起源点
    
    let mut origins = Vec::with_capacity(count as usize);
    let mut attempts = 0;
    
    while origins.len() < count as usize && attempts < 100 {
        let candidate = WorldPos {
            x: rng.gen_range(0.0..mask.width_km),
            y: rng.gen_range(0.0..mask.height_km),
        };
        
        // 必须在陆地上
        if !mask.is_land(candidate) { attempts += 1; continue; }
        
        // 不能太靠近已有起源点（最小间距 = 大陆最短边 / 起源点数的 0.6）
        let min_separation = mask.min_continent_dimension_km() / (count as f32 * 0.6);
        if origins.iter().any(|o| o.distance_km(candidate) < min_separation) {
            attempts += 1; continue;
        }
        
        // 偏好：适中的高程（不是山顶，不是沼泽）
        let elev = terrain.elevation_at(candidate);
        if elev < 0.05 || elev > 0.85 { attempts += 1; continue; }
        
        origins.push(candidate);
        attempts = 0; // 重置——找到了一个好点
    }
    
    origins
}
```

起源点偏好人类宜居的位置——不是山巅、不是沼泽（低海拔积水区）、不是沙漠深处。这反映了"文化起源于人类聚集地"的现实预设。

### 2.3 P2.5b：障碍 Voronoi 区域划分

```rust
/// 障碍 Voronoi —— 用地形旅行代价替代欧氏距离
fn obstacle_voronoi_partition(
    origins: &[WorldPos],
    mask: &LandSeaMask,
    terrain: &TerrainBase,
    ocean: &OceanSystem,
) -> CultureRegionMap {
    // 对每个陆地格点，计算到每个起源点的旅行代价距离
    // 海洋格点：归属到最近海岸线的文化区域
    
    let mut region_map = CultureRegionMap::new(mask.width_cells, mask.height_cells);
    
    // 使用 Dijkstra 式洪水填充——从每个起源点同时出发
    // travel_cost(edge) = terrain_cost(elevation, biome) + water_barrier(is_water)
    // 
    // terrain_cost:
    //   平原: 1.0, 丘陵: 1.5, 森林: 2.0, 沼泽: 4.0, 山脉: 10.0
    //   (与 世界生成 006 交通网络的地形代价一致)
    //
    // water_barrier:
    //   河流(>3级): ×3.0, 海洋: ×100.0 (几乎不可跨越),
    //   湖泊(中型+): ×5.0 (可绕过)
    
    let mut pq = BinaryHeap::new(); // (Reverse(cost), cell, origin_index)
    for (i, origin) in origins.iter().enumerate() {
        let cell = region_map.cell_at(origin);
        pq.push((Reverse(0.0), cell, i));
        region_map.set_origin_cell(cell, i);
    }
    
    while let Some((Reverse(cost), cell, origin_idx)) = pq.pop() {
        for neighbor in region_map.neighbors(cell) {
            if !mask.is_land(neighbor.to_world_pos()) { continue; }
            
            let edge_cost = travel_cost(cell, neighbor, terrain, mask);
            let new_cost = cost + edge_cost;
            
            if region_map.try_set(neighbor, origin_idx, new_cost) {
                pq.push((Reverse(new_cost), neighbor, origin_idx));
            }
        }
    }
    
    // 对于海洋格点：归属到最近海岸线的文化区域
    assign_ocean_cells(&mut region_map, mask, ocean);
    
    region_map
}

/// 旅行代价——复用 世界生成 006 交通网络的地形代价表
fn travel_cost(from: Cell, to: Cell, terrain: &TerrainBase, mask: &LandSeaMask) -> f32 {
    let base = from.distance(to);
    let terrain_factor = match terrain.biome_at(to) {
        Biome::Plains | Biome::Grassland => 1.0,
        Biome::Hills => 1.5,
        Biome::Forest(_) => 2.0,
        Biome::Swamp | Biome::Wetland => 4.0,
        Biome::Mountain(_) => 10.0,
        _ => 2.0,
    };
    // 大型河流：×3——与 006 一致（水域不可通行"除非有桥"的逻辑——但文化生成不使用桥，
    // 所以河流作为强障碍但不是绝对的——绕路成本高但可绕）
    if mask.is_major_river(to) { terrain_factor *= 3.0; }
    
    base * terrain_factor
}
```

### 2.4 P2.5c：CultureCoreParams 生成

```rust
/// 为每个文化区域生成初始 CultureCoreParams
fn generate_culture_params(
    seed: u64,
    origins: &[WorldPos],
    region_map: &CultureRegionMap,
    terrain: &TerrainBase,
) -> Vec<CultureCoreParams> {
    let mut params = Vec::with_capacity(origins.len());
    let mut rng = stage_rng(seed, "culture_params");
    
    for (i, origin) in origins.iter().enumerate() {
        // 步骤 1: 从种子生成底层的随机参数（80-95% 的权重——种子为主）
        let culture_seed = hash_2(seed, "culture_core", i as u64);
        let mut culture_rng = StdRng::seed_from_u64(culture_seed);
        
        let base = CultureCoreParams {
            individualism:          culture_rng.gen_range(0.0..1.0),
            power_distance:         culture_rng.gen_range(0.0..1.0),
            uncertainty_avoidance:  culture_rng.gen_range(0.0..1.0),
            competition_orientation: culture_rng.gen_range(0.0..1.0),
            long_term_orientation:  culture_rng.gen_range(0.0..1.0),
            indulgence:             culture_rng.gen_range(0.0..1.0),
            openness_to_outsiders:  culture_rng.gen_range(0.0..1.0),
            religiosity:            culture_rng.gen_range(0.0..1.0),
            militarism:             culture_rng.gen_range(0.0..1.0),
            artistry:               culture_rng.gen_range(0.0..1.0),
        };
        
        // 步骤 2: 群系温和修正（±0.05 级别——环境提供温和的适应性压力）
        let biome_params = terrain.biome_params_at(*origin);
        let modifier = biome_modifier(&biome_params);
        
        let adjusted = apply_biome_modifier(base, modifier);
        
        // 步骤 3: 参数聚类——生成文化个性差异
        // 随机文化倾向于在特定维度上形成"极点"而非均匀分布
        let polarized = apply_polarization(adjusted, &mut culture_rng);
        
        params.push(polarized);
    }
    
    params
}

/// 群系对文化参数的温和偏移
fn biome_modifier(biome: &BiomeParams) -> CultureCoreParams {
    CultureCoreParams {
        // 沙漠 → 略高的 militarism（资源稀缺→竞争）、略高的 long_term（生存压力→规划）
        // 沿海 → 略高的 openness（接触外来者多）、略高的 navigation 倾向
        // 肥沃平原 → 略高的 long_term（农业文明→长远规划）、略低的 militarism（资源充足→减少冲突）
        // 寒带 → 略高的 collectivism（抱团取暖）、略高的 long_term（漫长的冬天→规划储备）
        individualism:          if biome.is_coastal { 0.03 } else { 0.0 },
        power_distance:         0.0, // 群系不显著影响等级制度
        uncertainty_avoidance:  0.0,
        competition_orientation: if biome.is_desert { 0.05 } else { 0.0 },
        long_term_orientation:  if biome.is_fertile_plain { 0.04 } else if biome.is_arctic { 0.05 } else { 0.0 },
        indulgence:             if biome.is_tropical { 0.04 } else { 0.0 }, // 热带→丰富的自然资源→享受
        openness_to_outsiders:  if biome.is_coastal { 0.05 } else { 0.0 },
        religiosity:            0.0,
        militarism:             if biome.is_desert { 0.04 } else if biome.is_arctic { 0.03 } else if biome.is_fertile_plain { -0.03 } else { 0.0 },
        artistry:               0.0,
    }
}

fn apply_biome_modifier(mut base: CultureCoreParams, modifier: CultureCoreParams) -> CultureCoreParams {
    // 每个参数的修正钳制在 ±0.05
    let apply = |b: f32, m: f32| -> f32 { (b + m.clamp(-0.05, 0.05)).clamp(0.0, 1.0) };
    base.individualism = apply(base.individualism, modifier.individualism);
    base.power_distance = apply(base.power_distance, modifier.power_distance);
    base.uncertainty_avoidance = apply(base.uncertainty_avoidance, modifier.uncertainty_avoidance);
    base.competition_orientation = apply(base.competition_orientation, modifier.competition_orientation);
    base.long_term_orientation = apply(base.long_term_orientation, modifier.long_term_orientation);
    base.indulgence = apply(base.indulgence, modifier.indulgence);
    base.openness_to_outsiders = apply(base.openness_to_outsiders, modifier.openness_to_outsiders);
    base.religiosity = apply(base.religiosity, modifier.religiosity);
    base.militarism = apply(base.militarism, modifier.militarism);
    base.artistry = apply(base.artistry, modifier.artistry);
    base
}
```

**关键设计决策：群系修正仅 ±0.05**。文化的性格主要来自种子随机性——不是地理决定论。环境提供温和的适应性压力，但两个同处沙漠的文化可以有完全不同的价值观。

### 2.5 参数极化

均匀分布在 0-1 区间的 10 个参数产生"平均文化"——所有值接近 0.5。但真实世界的文化倾向于在某些维度上形成极端值。极化算法：

```rust
fn apply_polarization(mut base: CultureCoreParams, rng: &mut impl Rng) -> CultureCoreParams {
    // 随机选择 2-4 个维度推向极端
    let polarized_count = rng.gen_range(2..=4);
    let mut indices: Vec<usize> = (0..10).collect();
    indices.shuffle(rng);
    
    for &idx in indices.iter().take(polarized_count) {
        let direction: f32 = if rng.gen_bool(0.5) { 1.0 } else { -1.0 };
        let magnitude = rng.gen_range(0.15..0.35);
        set_param(&mut base, idx, get_param(&base, idx) + direction * magnitude);
    }
    
    base
}
```

这确保大多数文化在 2-4 个维度上形成鲜明的"性格"——其他维度保持中等。比如一个文化可能是"高权力距离 + 高宗教性 + 低开放度"（神权王国），但 individualism 和 long_term 可能接近 0.5。

## 三、文化区域的空间属性

### 3.1 区域规模分类

| 规模 | 覆盖聚落数 | 典型面积 | 出现条件 |
|------|-----------|---------|---------|
| 大型文化区 | 30-100+ | 20,000-50,000 km² | 广阔平原或沿海连续地带，起源点周围无强地理屏障 |
| 中型文化区 | 10-30 | 5,000-20,000 km² | 典型规模——被河流/山丘部分限制 |
| 小型文化区 | 3-10 | 1,000-5,000 km² | 被山脉/海洋/大河围合的半隔离区域 |
| 微型文化区 | 1-3 | 100-1,000 km² | 孤岛/深山峡谷/沙漠绿洲——高度隔离 |

规模由障碍 Voronoi 的天然结果决定——不预设。

### 3.2 文化飞地

## 四、边界混合

### 4.1 边界带定义

```
对于每个格点 p：
  设 d1 = 到最近起源点的代价距离（归属文化 C1）
  设 d2 = 到次近起源点的代价距离（归属文化 C2）
  
  如果 d2 / d1 < 1.5：
    p 在混合带中
    blend_weight = 1.0 - (d1 / d2) × 0.5  // -> [0.25, 0.5]
    p 的文化混合 = C1 × (1 - blend_weight) + C2 × blend_weight
  否则：
    p 为纯 C1
```

边界带中的位置同时受两个文化的 norms 影响。NPC 在边界带中选择哪个文化的规则取决于上下文（在自己聚落中使用聚落的文化规则，在异文化聚落中使用异文化的规则）。

### 4.2 混合带中的 NPC 行为

NPC 在混合带中出生时，默认归属主导文化（d1 的 C1）。但 NPC 的 `CulturalSystem`（见 [[../NPC活人感模块/NPC活人感开发文档ver2.0|NPC ver2.0]] §2.6）可以内化来自相邻文化的 norms——特别是如果 NPC 的工作涉及跨文化互动（商人、信使、边境守卫），他们可能发展出混合文化认同。

## 五、确定性保证

文化生成必须是确定性的——给定相同的 world_seed，总是生成相同的文化区域和参数。这是整个世界生成管线确定性保证的一部分。

```rust
fn stage_rng(seed: u64, stage: &str) -> StdRng {
    let stage_seed = hash(seed, stage);
    StdRng::seed_from_u64(stage_seed)
}

fn culture_rng(seed: u64, culture_index: u32) -> StdRng {
    let culture_seed = hash_2(seed, "culture_core", culture_index as u64);
    StdRng::seed_from_u64(culture_seed)
}
```

所有随机源从 world_seed 经由确定性哈希派生——不使用系统时间、硬件状态或任何外部熵源。

## 六、参数验证规则

初始化后的文化参数集必须通过以下验证：

```rust
fn validate_culture_params(cultures: &[CultureCoreParams]) -> Result<(), Vec<String>> {
    let mut errors = Vec::new();
    
    // 1. 范围检查：所有值在 [0, 1]
    for (i, c) in cultures.iter().enumerate() {
        if c.individualism < 0.0 || c.individualism > 1.0 {
            errors.push(format!("Culture {}: individualism={} out of [0,1]", i, c.individualism));
        }
        // ... 每个参数
    }
    
    // 2. 多样性检查：至少两个文化的参数距离 > 0.3
    //    （确保世界中有真正不同的文化——不是所有文化趋同）
    if cultures.len() >= 2 {
        let max_distance = max_pairwise_distance(cultures);
        if max_distance < 0.3 {
            errors.push(format!(
                "Warning: All cultures too similar (max pairwise distance={:.2})", 
                max_distance
            ));
        }
    }
    
    // 3. 多样性检查：至少有一个文化的 openness > 0.5
    //    （如果所有文化都极度排外——世界不存在跨文化互动）
    if !cultures.iter().any(|c| c.openness_to_outsiders > 0.5) {
        errors.push("Warning: No culture has openness > 0.5 — cross-cultural interaction will be rare".into());
    }
    
    if errors.is_empty() { Ok(()) } else { Err(errors) }
}
```

**注意**：这些是验证警告——不是硬约束。如果随机种子产生了一个"所有文化都相似"的世界，这本身是涌现的——不会重新生成。但种子验证规则提供开发阶段的合理性检查。
