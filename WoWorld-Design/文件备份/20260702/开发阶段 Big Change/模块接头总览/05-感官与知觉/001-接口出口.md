# 感官与知觉模块 — 接口出口

> **最后验证**: 2026-06-22 | **源文档版本**: v1.1 | **CHG-031 权威 (CHG-033 SpatialQuery→4trait 更新) + CHG-057/058 感知富化桥梁层**

## SensoryProvider trait (统一感知入口)
- **概念**: 顶层trait—NPC决策循环的唯一感知入口. `perceive()` → `PerceptBatch`. `perceive_lightweight()` → `LightweightPerceptBatch` (大日志用, 不做注意筛选/噪声/内感受)
- **Owner**: [[../../../感官与知觉系统/001-感官系统总纲|感官 §三]] + [[../../../感官与知觉系统/跨模块接口与数据合同|合同]]
- **类型**: Rust trait (woworld_core定义, woworld_senses实现)
- **消费者**: [[../02-NPC活人感/002-接口入口|NPC]] 决策循环 · [[../12-历史/002-接口入口|历史]] 大日志perceive_lightweight
- **契约依据**: CHG-031 (⚠️ CHG-033: perceive()签名已更新—spatial拆分为4 trait)

## PerceptBatch (统一知觉产出)
- **概念**: 感官系统对所有消费者的统一输出. 包含: visual_entities[] + visual_scene + scents[] + olfactory_landscape + audio_events[] + weather_snapshot + interoception + social_percepts + non_verbal_signals. PerceptEntry 含 emotional_charge 字段 (★ v1.1: Option\<EmotionalCharge\>). 同份PerceptBatch供: 日常决策(prepare_facts) + 战斗情报(combat_intelligence.assess) + 审美判断(aesthetic_system.judge) + 大日志
- **Owner**: [[../../../感官与知觉系统/001-感官系统总纲|感官 §二]]
- **类型**: Rust struct, woworld_senses
- **消费者**: [[../02-NPC活人感/002-接口入口|NPC]] prepare_facts/GOAP · [[../06-战斗/002-接口入口|战斗]] CombatPerception→CombatAssessment · [[../12-历史/002-接口入口|历史]] 大日志 · [[../14-经济/002-接口入口|经济]] 市场感知 · [[../11-权力/002-接口入口|权力]] 权力符号感知 · ★ v1.1: EnrichedPerceptBatch → 认知与智慧（认知管线输入）

## ★ v1.1 感知富化桥梁层 (CHG-057/058)

- **EmotionalCharge** (CHG-057): 跨模态提取的外部情绪信号 — valence/arousal/dominance/primary_emotion/confidence — 由感官系统生产、认知系统消费。定义在 woworld_core。
- **GazeEstimate** (CHG-057): 从被感知实体朝向/骨架推测的视线信息 — 纯几何计算 — direction + confidence + target_entity。定义在 woworld_core。
- **BoundEntity** (CHG-057): 跨模态绑定结果 — "这个声音属于哪个视觉实体？" — PerceptBatch::bind_modalities() 的产出。
- **EnrichedPerceptBatch** (CHG-057): 感知富化的统一产物 — bound_entities[] + gaze_estimates[] + emotional_readings[] — PerceptBatch::enrich() 的产出。
- **EmotionalReading** (CHG-057): read_emotions() 的产出 — entity + emotional_charge + reading_confidence。
- **crowd_emotional_field** (CHG-057): 在 EnvironmentPerception 中 — 人群情绪场 (valence/arousal/dominance 加权聚合) + crowd_dominant_emotion。
- **PerceptBatch 扩展方法** (CHG-058): enrich() / bind_modalities() / estimate_gaze() / read_emotions() — 感知富化桥梁层，供认知/战斗/经济/权力模块消费。

## VisionQuery trait (视觉主动扫描)
- **概念**: 视觉感知—NPC/玩家视野内实体发现+分类. 输出: (Vec<PerceivedVisualEntity>, VisualScene). 消费: TerrainQuery(材质/光照) + VisibilityQuery(视线) + EntityIndex(AABB) + WeatherSample + PerceptualCache + DarkAdaptation + AttentionState. 不做语义辨识
- **Owner**: [[../../../感官与知觉系统/001-感官系统总纲|感官 §三]] + [[../../../感官与知觉系统/视觉/001-视觉感知-VisionQuery|视觉 §001]]
- **类型**: Rust trait (woworld_core定义, woworld_senses实现)
- **消费者**: [[../02-NPC活人感/002-接口入口|NPC]] 决策循环 · [[../06-战斗/002-接口入口|战斗]] 战斗感知
- **契约依据**: CHG-031 (⚠️ CHG-033: VisibilityQuery替代旧line_of_sight)

## ScentQuery trait (嗅觉懒采样)
- **概念**: 嗅觉感知—懒采样(非主动扫描). 输出: (Vec<PerceivedScent>, OlfactoryLandscape). 消费: SpatialEventBus(scent_sources_in) + WindVector(wind_at→WeatherQuery). 确定性噪声种子
- **Owner**: [[../../../感官与知觉系统/001-感官系统总纲|感官 §三]] + [[../../../感官与知觉系统/嗅觉/001-嗅觉感知-ScentQuery|嗅觉 §001]]
- **类型**: Rust trait (woworld_core定义, woworld_senses实现)
- **消费者**: [[../02-NPC活人感/002-接口入口|NPC]] 决策循环 · [[../01-生命/002-接口入口|生命]] 动物觅食
- **契约依据**: CHG-031 (⚠️ CHG-033: SpatialEventBus::scent_sources_in替代旧查询)

## PerceptualCache (LRU 64)
- **概念**: 短期感知缓存—64条LRU. 不持久化(存储于NpcData.sensory). 用途: 变化盲视检测/熟悉度判断/视觉持续性
- **Owner**: [[../../../感官与知觉系统/001-感官系统总纲|感官 §四]]
- **类型**: Rust struct (LRU cache), woworld_senses
- **消费者**: [[../02-NPC活人感/002-接口入口|NPC]] 变化检测 · [[../06-战斗/002-接口入口|战斗]] 敌人追踪

## DarkAdaptation (暗适应)
- **概念**: 指数松弛模型—进入黑暗后视觉灵敏度指数级提升. 对标视觉生理
- **Owner**: [[../../../感官与知觉系统/视觉/002-视觉参数与种族差异|视觉 §002]]
- **类型**: Rust struct, woworld_senses
- **消费者**: [[../02-NPC活人感/002-接口入口|NPC]] VisionQuery输入

## AttentionState / 显著性引擎
- **概念**: 注意状态(当前焦点/分散度/疲劳) + 显著性计算(运动/对比度/新异/目标相关). 注意切换+变化盲视
- **Owner**: [[../../../感官与知觉系统/注意与显著性/001-显著性引擎|注意 §001]] + [[../../../感官与知觉系统/注意与显著性/002-注意切换与变化盲视|§002]]
- **类型**: Rust struct, woworld_senses
- **消费者**: [[../02-NPC活人感/002-接口入口|NPC]] 决策循环 · [[../06-战斗/002-接口入口|战斗]] 战斗注意

## PerceptualModifiers (知觉调制器)
- **概念**: 状态效果对感知的修正: vision_range_mult/noise_mult/hearing_range_mult/scent_range_mult/attention_shift_penalty. 每决策周期从NPC status_effects聚合
- **Owner**: [[../../../感官与知觉系统/001-感官系统总纲|感官 §四]]
- **类型**: Rust struct, woworld_senses
- **消费者**: [[../02-NPC活人感/002-接口入口|NPC]] 决策循环前聚合

## 确定性噪声种子
- **概念**: 每种感官每个NPC一条独立噪声序列—seed=hash(npc_id, sense_modality, tick). 保证同一NPC在同一时刻对同一刺激的感知偏差一致(可重现)
- **Owner**: [[../../../感官与知觉系统/知觉噪声与偏差/001-噪声函数族与种子管理|噪声 §001]]
- **类型**: Rust Rng种子管理, woworld_senses
- **消费者**: VisionQuery · ScentQuery · 注意筛选

## L0-L4 感知退化
- **概念**: L0(玩家全精度)→L1(全个体)→L2(简化)→L3(代理统计)→L4(纯统计). 每级精度递减—L4无个体感知
- **Owner**: [[../../../感官与知觉系统/性能预算与L分层|感官 性能]]
- **类型**: Rust架构分层, woworld_senses
- **消费者**: [[../02-NPC活人感/002-接口入口|NPC]] 分层模拟

## 疼痛中断通道
- **概念**: 重大伤害→强制注意切换→打断当前感知焦点. 疼痛强度>阈值→注意强制重定向到伤害源
- **Owner**: [[../../../感官与知觉系统/001-感官系统总纲|感官 §六]]
- **类型**: Rust中断信号, woworld_senses
- **消费者**: [[../06-战斗/002-接口入口|战斗]] 本能层反应 · [[../02-NPC活人感/002-接口入口|NPC]] 注意中断

## 三级识别回落
- **概念**: L1精确识别(近距离/熟悉)→L2类别识别(中等距离/常见类别)→L3模糊检测(远距离/不熟悉—"那边有个东西"). PerceptBatch统一产出
- **Owner**: [[../../../感官与知觉系统/001-感官系统总纲|感官 §五]]
- **类型**: Rust识别管线, woworld_senses
- **消费者**: [[../02-NPC活人感/002-接口入口|NPC]] 决策 · [[../06-战斗/002-接口入口|战斗]] 敌我识别

## ConcealmentState (隐藏状态)
- **概念**: 实体隐藏状态查询—潜行/伪装/隐身. 影响VisionQuery的实体发现概率
- **Owner**: [[../../../感官与知觉系统/视觉/003-可见性与射线策略|视觉 §003]]
- **类型**: Rust struct, woworld_senses
- **消费者**: [[../06-战斗/002-接口入口|战斗]] 潜行判定

## digest_aesthetic_experience()
- **概念**: 审美体验消化—VisualScene→AestheticSignal(6维)派生. fluency/novelty/complexity/harmony/expressiveness/virtuosity. 喂入NPC审美系统judge()
- **Owner**: [[../../../感官与知觉系统/001-感官系统总纲|感官 §七]]
- **类型**: Rust fn, woworld_senses
- **消费者**: [[../02-NPC活人感/002-接口入口|NPC]] AestheticSystem.judge()

## CulturalKnowledgeBase trait
- **概念**: 文化基线知识查询—knows_category() + category_prototype(). TOML数据驱动. 感官系统通过此trait判定NPC文化背景下的分类能力
- **Owner**: [[../../../感官与知觉系统/001-感官系统总纲|感官 §三]] + [[../../../感官与知觉系统/跨模块接口与数据合同|合同 §一]]
- **类型**: Rust trait (woworld_core定义, TOML数据实现)
- **消费者**: [[../02-NPC活人感/002-接口入口|NPC]] 文化感知

## CategoryTag 全局分类标签
- **概念**: TOML数据驱动—物品/生物/建筑ID→全局CategoryTag映射. 感官系统据此做"这是剑"/"这是面包"的分类
- **Owner**: [[../../../感官与知觉系统/跨模块接口与数据合同|感官合同 §六]]
- **类型**: TOML映射表
- **消费者**: [[../18-物品/002-接口入口|物品]] 映射注册 · [[../01-生命/002-接口入口|生命]] 物种映射 · [[../03-世界生成/002-接口入口|世界生成]] 建筑映射

## SensoryState (NPC 感官参数聚合)
- **概念**: 存储于NpcData.sensory—hearing+voice+action_log+silence(音频) + vision+perceptual_cache+dark_adaptation(感官) + current_speech(语言). NPC crate定义结构, 各子模块定义字段
- **Owner**: [[../../../感官与知觉系统/001-感官系统总纲|感官 §四]]
- **类型**: Rust struct, woworld_npc(存储) + woworld_senses(字段定义)
- **消费者**: [[../02-NPC活人感/002-接口入口|NPC]] NpcData.sensory

## ScentSource 写入协议
- **概念**: 各模块写入气味源到SpatialEventBus: 尸体腐烂(Life)→焚香(Faith)→奥术残留(Magic)→烹饪(Items/NPC)→篝火(WorldGen). 感官系统通过scent_sources_in()查询
- **Owner**: [[../../../感官与知觉系统/跨模块接口与数据合同|感官合同 §五]]
- **类型**: SpatialEventBus::push_event 协议
- **消费者**: ScentQuery 消费
