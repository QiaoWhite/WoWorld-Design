# 性能优化 — 变更影响链

> **最后验证**: 2026-06-18 | **源文档版本**: 性能优化分析 20260603 | **CHG-033/CHG-038 已纳入**
>
> 性能优化分析变更本身影响有限. 但其**结论**会影响所有模块的性能预算分配。

## 瓶颈分析结论变更
→ **全部模块**: 新识别的瓶颈 → 相关模块须优化或接受预算削减
→ **技术栈方案**: 性能预算重新分配

## 测试基准变更
→ **全部模块**: 各模块须提供更新的性能数据
→ **技术栈方案**: 新的预算分配依据

## Rust/Godot 边界建议变更
→ **全部模块**: Rust crate 归属/交互模式调整
→ **GDExtension 接口**: 共享内存 vs trait调用 vs 事件总线 的选择
→ **CHG-033**: 物理方案改为 Rust 空间查询 — 减少跨边界调用

## 预算分配建议变更 ⚠️ 高影响
→ **各子系统预算调整**: 一方的预算增加 → 另一方预算减少 (零和)
→ **技术栈方案**: 重新仲裁各模块预算
→ **受影响最大的模块**:
  - NPC模拟 (最大CPU消费者)
  - 世界生成 (最大VRAM消费者)
  - 动画系统 (0.5ms硬上限 — CHG-033)
  - 音频系统 (0.17ms硬上限 — CHG-030)

## 优化优先级矩阵变更
→ **全部模块**: 各模块的优化投入优先级重排

## 体素渲染分析变更
→ **世界生成**: Chunk大小/LOD策略/VRAM占用
→ **技术栈方案**: 体素渲染管线参数

## NPC模拟分析变更
→ **NPC**: GOAP调度频率/SoA布局/rayon任务粒度
→ **技术栈方案**: 100k NPC可行性确认或下调

## ⚠️ CHG-033 架构影响 (Wave2)
→ **物理方案**: 仅玩家 PhysicsServer3D — 大量减少 Godot 物理开销
→ **空间查询四trait**: 性能 profile 确认 — DDA射线 ~10µs
→ **双骨蒙皮**: GPU skinning VRAM 确认 — INSTANCE_CUSTOM 16B
→ **骨架松弛**: ~0.03ms/死亡NPC — 替代布娃娃的性能收益确认
