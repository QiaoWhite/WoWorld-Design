## 提供给其他模块的接口

### SaveableModule trait v2.0

存档系统定义 `SaveableModule` trait——所有需要持久化的模块实现此 trait。存档系统不提供数据给其他模块——它只消费模块的 trait 实现。

**14 方法** (4 必覆 + 10 默认实现):

```rust
trait SaveableModule: Send + Sync {
    // === 4 必覆方法 ===
    fn module_name(&self) -> &'static str;
    fn current_version(&self) -> u32;
    fn named_dbs(&self) -> &[&str];           // ★ v2.0 新增——声明本模块的 LMDB named_db 列表
    fn snapshot_dirty(&self) -> Result<DirtySnapshot>;

    // === 10 默认方法 ===
    fn is_critical(&self) -> bool { false }
    fn has_dirty(&self) -> bool { false }
    fn estimate_dirty_bytes(&self) -> usize { 0 }
    fn write_dirty(&mut self, snap: &DirtySnapshot) -> Result<()> {}   // 流式增量写入
    fn write_initial(&mut self, txn: &RwTransaction) -> Result<()> {}  // Initial存档直写
    fn confirm_snapshot_written(&mut self, snapshot: &DirtySnapshot) {}
    fn load(&mut self, ctx: &LoadContext) -> Result<()>;  // ★ v2.0 签名变更 (txn→LoadContext)
    fn reset_to_default(&mut self) {}       // ★ v2.0 新增——半加载恢复
    fn migrate(&self, from_version: u32, txn: &RwTransaction) -> Result<()> {}
    fn validate_after_migration(&self) -> Result<()> { Ok(()) }  // ★ v2.0 新增——迁移后验证
}
```

### LoadContext — 渐进加载上下文

```rust
struct LoadContext<'env> {
    txn: RoTransaction<'env>,
    create_txn: Box<dyn Fn() -> Result<RoTransaction<'env>> + 'env>,
}
```

模块在 `load()` 中通过 `ctx.txn` 读取主数据，通过 `ctx.create_txn()` 惰性打开额外 named_db 进行渐进加载。零 SaveSystem 耦合。

### Initial 存档直写路径

P13 产出 `ValidatedWorldState`（纯内存）→ `SaveSystem::create_initial_save()` 逐个模块调用 `write_initial()` 流式写入。世界生成不持有 LMDB 写入权。

### 崩溃恢复三件套

1. 临时文件 + 原子重命名（`.woworld.tmp` → `.woworld`）
2. 覆盖前自动备份（`.woworld.bak`）
3. `session.lock` 会话检测
