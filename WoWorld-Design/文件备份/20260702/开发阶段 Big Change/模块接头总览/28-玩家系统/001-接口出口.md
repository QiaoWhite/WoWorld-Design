## 玩家系统导出的接口概念

### ControlMode

```rust
enum ControlMode {
    Auto,
    Manual { action_override: Action },
    DomainDelegated { manual_domains: HashSet<ActionDomain> },
}

enum ActionDomain {
    Movement, Combat, Speech, ItemUse, Interaction, MagicUse,
}
```

**消费方**: NPC生命周期 008、输入系统

### PlayerGoal（长期计划槽）

```rust
struct PlayerGoal {
    id: GoalId,
    description: String,
    priority: f32,              // 0-1
    state: GoalState,           // Draft/Active/Paused/Completed/Abandoned
    sub_goals: Vec<GoapGoal>,   // GOAP 自动展开
}
```

**消费方**: NPC活人感 GOAP 规划器

### 两种进入模式

- 原住民模式：接管既有 NPC
- 穿越者模式：7步向导创建新角色

**消费方**: 角色创建流程、UI/UX 系统

### 信息展示边界

自身 Vitals/技能/大日志 ✅ | 他人内心/好感度/记忆/头衔 ❌

**消费方**: UI/UX 系统

### 死亡继承链

血亲 → 大日志选择 → 全新角色。技能/关系不可继承。

**消费方**: NPC生命周期 007、存档系统

### 输入动作清单

~40 动作（移动/战斗/交互/对话/角色与视角/系统），手柄+键鼠平等，可全改键。

**消费方**: UI/UX 系统
