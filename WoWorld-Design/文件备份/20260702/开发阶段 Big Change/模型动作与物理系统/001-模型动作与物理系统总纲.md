## 〇、模块定位

模型动作与物理系统是 WoWorld 所有"物理存在"的完整表达层。一个实体在世界中有一个位置、一个外形、一个骨架、一套动作、以及和世界互动的物理行为——本系统统一管理这全部三个维度。

**三个子系统，一个管线**：

```
Rust 模拟核心（物理现实的唯一权威）
  ├─ 空间查询（TerrainQuery + EntityIndex + SpatialEventBus + VisibilityQuery）
  ├─ 模型定义（SkeletonDef, ModelVariant, BodyPlan, PhysicalAppearance）
  ├─ 动画生成（关键帧 + 程序化轨迹 + 9 层动画栈）
  ├─ 物理响应（COM 飞行 + 着地 + 水下 + 死亡）
  └─ 输出：每帧骨骼矩阵 [Mat4; N] + per-instance 数据

         │ GDExtension（双缓冲共享内存）
         ▼

Godot 客户端（纯渲染器 + 玩家输入）
  ├─ MultiMeshInstance3D × GPU skinning shader（双骨蒙皮）
  ├─ TABS 式 flat/cel 渲染——仅 diffuse，512² 面部图集共享
  ├─ 玩家 CharacterBody3D（唯一保留 PhysicsServer3D 的东西）
  └─ 音频播放 + UI
```

**核心设计原则**：

1. **模型定义归 Rust 所有** — 骨架结构、身体计划、外观参数在 `woworld_core`。Godot 只持有渲染资产（网格 RID、纹理）。
2. **动画归 Rust 所有** — 骨骼矩阵在 Rust CPU 计算→rayon 并行→推送 Godot GPU skinning。
3. **物理规则归 Rust 所有** — 空间查询纯 Rust 实现。仅玩家角色控制器保留 Godot PhysicsServer3D。
4. **涌现式动画** — 步态风格从大五人格派生。战斗流派从参数差异涌现。面部表情从情绪状态数学映射。开发者定义底层规则，具体动画自然涌现。
5. **全 TOML 自定义** — 步态/战斗风格/过渡参数/面部图集/双人交互——全部数据驱动。Mod 社区可扩展。

### 1.2 帧内执行顺序

```
Rust 模拟核心（每帧）:
  1. 感官系统 — 消费 TerrainQuery + EntityIndex → PerceptBatch
  2. NPC AI — 决策 → 输出 ActionRequest[]
  3. 动画系统 — AnimationBodyState::tick() × N NPC
       ├─ 中断检查（hit/death/forced_displacement）
       ├─ 基础姿态计算（keyframe 混合 or 战斗轨迹）
       ├─ 9 层动画栈叠加
       ├─ 过渡系统
       ├─ 脚部 IK + 脊柱桥接
       └─ 输出 bone_matrices: [Mat4; 50]
  4. 空间索引更新 — EntityIndex::update_transform() 批量提交
  5. GDExtension 推送 — PackedByteArray → Godot 双缓冲

Godot 渲染线程（每帧，异步）:
  6. TextureBuffer 上传骨骼矩阵
  7. MultiMesh draw call
  8. GPU skinning shader
```

## 二、人形骨架规范

| 部位 | 骨骼数 | 说明 |
|------|--------|------|
| Lower Body | 10 | pelvis, pelvis_secondary, L/R thigh/shin/foot/toe |
| Upper Body | 15 | spine_01~03, L/R clavicle, L/R upper_arm/forearm/hand, L/R thumb/index |
| Head/Face | 7 | neck, head, jaw, L/R brow, L/R eye |
| Attachment | 5 | weapon_r, shield_l, back_slot, L/R hand_grip |
| **L1 总计** | **33** | 不含 brow/eye（L1 面部用纹理） |
| **L0 总计** | **35** | 含 L/R brow + L/R eye（L0 近距离面部骨骼） |
| **L2 总计** | **28** | 砍掉面骨 + 手指骨 + 部分挂载 |
| **L3 总计** | **15** | 仅腿+脊柱+头+武器挂载 |

### 面部表情方案

面部表情不靠骨骼驱动——用共享 512² 纹理图集。

| 图集区域 | 变体数 | 内容 |
|---------|--------|------|
| 嘴型 | 16 | 闭/微开/大笑/噘/咬/吐舌/说话口型(5个phoneme)/惊/怒 |
| 眉型 | 16 | 平/抬/皱/八字/挑/压 |
| 眼态 | 8 | 全开/半眯/闭/怒/惊 |
| 虹膜 | 8 | 棕/蓝/绿/灰/金/红/白/异色 |
| 叠加层 | 变量 | 伤疤/胎记/战纹/纹身 |

每 NPC 表情 = **6 字节**（mouth_shape + brow_shape + eye_state + iris_offset_x/y + blush），塞入 MultiMesh `INSTANCE_CUSTOM`。shader 中通过 UV 偏移采样对应图集行。

## 三、动画系统核心

### 3.1 关键帧姿态

**38 个模块姿态**（日常 + 施法 + 社交 + 拔剑/收剑），4 阶段包络（Anticipation → Strike → Follow-through → Recovery），存储在 TOML 或 PoseDatabase。

**15 条基元轨迹**（10 武器 + 5 空手），仅保留 Strike 阶段的关键骨骼帧（3-5 帧），前后由身体求解器自动计算。

### 3.2 9 层动画栈

| 层 | 名称 | 功能 |
|----|------|------|
| Layer 0 | 基础姿态 | keyframe 混合 or 战斗轨迹求解 |
| Layer 1 | 情绪偏移 | 13 参数情绪→骨骼旋转/纹理索引 映射 |
| Layer 2 | 过渡混合 | from→to slerp，关节指数收敛 |
| Layer 3 | 根运动 | 攻击根运动提取，叠加到步态位移 |
| Layer 4 | 注视 | 头骨→注意目标 slerp |
| Layer 5 | 脚部 IK | 着地脚锁定世界空间 + IK 链 |
| Layer 6 | 武器手 IK | 基元轨迹跟随 + 握持点精确定位 |
| Layer 7 | 脊柱桥接 | 髋↔肩旋转差分配到各节脊椎 |
| Layer 8 | 中断恢复 | 受击/踉跄/死亡响应 |

### 3.3 步态风格——从人格涌现

步态不是"选一个动画"。是 9 个连续参数：髋摆幅、步幅、手臂摆动、弹跳、前倾角、节奏规整度、拖地量、肩稳定性、视线水平。全部从 NPC 大五人格 × 情绪 × 疲劳度 纯数学派生。零手写步态数据。

### 3.4 战斗轨迹——积木式动画

动作基元定义打击段的空间轨迹曲线。武器物理参数（长度/重量/重心）程序化缩放轨迹。身体求解器从轨迹反算全身骨骼。基元串联通过动量继承自然衔接。流派差异来自 CombatStyleParams 的连续参数空间。

### 3.5 动画衔接——五项自然法则

1. **接触锁定**：过渡期间着地脚死锁世界位置——不滑步
2. **指数收敛**：每关节独立的收敛速度——肩快、髋慢、踝最慢
3. **相位匹配**：循环→循环过渡前先对齐双脚着地状态
4. **动量继承**：重心速度不归零——前动作的余势自然流入后动作
5. **Ease 曲线**：所有 blend 走 ease-in-out，永不线性

过渡参数存储在 TOML——开发者/玩家可调每对动作间的过渡行为。

## 四、物理响应系统

### 4.1 COM 抛物体飞行

被击飞/被投掷/跳崖——统一由 `PhysicsBodyState` 处理。重力加速度 + DDA 射线投射检测障碍 + 崖壁碰撞反弹/滑落 + 三级着地响应（致命/重伤/硬着陆）。

**不做布娃娃。做"骨架松弛物理"**：死亡后肌肉刚度指数衰减→骨骼在重力+惯性+关节限制下自然塌落→完全松弛后退化为 COM 物理。成本 ~0.03ms/NPC（仅死亡 NPC）。

### 4.2 双人交互

接触点锚定 + 相对空间约束。被动方 IK 跟随主动方。拥抱/搀扶/推搡——TOML 交互模板定义锚点和约束。

### 4.3 水下/骑乘

水模式：浮力 + 水阻 + 水流 + 游泳步态自动切换。
骑乘：骑手骨盆锚定马鞍 world transform。马上半身独立动作。马受惊→骑手跌落（涌现行为）。

## 五、空间查询架构

四个 trait（woworld_core 定义，world_gen + woworld_spatial 实现）：

| Trait | 职责 | 实现方 |
|-------|------|--------|
| `TerrainQuery` | 高度/法线/射线投射/可通行/材质/介质/光照/地平线 | world_gen |
| `EntityIndex` | 实体注册/注销/变换更新/AABB 查询/声学标签 | woworld_spatial |
| `SpatialEventBus` | 空间事件推送/范围查询/气味源注册 | woworld_spatial |
| `VisibilityQuery` | 多源视线检测（密度场 + 实体 AABB） | woworld_spatial |

仅玩家 CharacterBody3D 保留 Godot PhysicsServer3D。其余全部物理在 Rust 侧。
