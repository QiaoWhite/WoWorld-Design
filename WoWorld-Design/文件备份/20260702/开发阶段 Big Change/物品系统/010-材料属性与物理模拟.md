## §一、概述

MaterialDef 是 WoWorld 中所有材料物理属性的 TOML 注册表。它使物理原子（STRIKE、HEAT、IGNITE、CONDUCT 等）能够根据材料的实际属性差异化行为，而非依赖硬编码的材料规则。

**核心原则**: 物品系统**存储和提供查询** MaterialDef，但**不解释**其字段含义。字段的物理语义由 NPC 行动涌现模块定义，由物理原子引擎消费。此模式与 `ConsumableEffect`（Life 定义，物品存储）和 `AudioMaterial`（音频定义，物品存储）一致。

---

## §二、MaterialDefId

```rust
/// 材料定义的全局唯一标识符
/// 布局: [16-bit namespace][16-bit category][32-bit index]
///   namespace: 0x0000 = Core, 0x0001-0xFFFF = Mod
///   category:  0x0001 = Metal, 0x0002 = Stone, 0x0003 = Wood, ...
///   index:     该类别内的顺序编号
pub struct MaterialDefId(pub u64);
```

命名空间: Core(0x0000) 预留给官方材料。Mod 可从 0x0001-0xFFFF 中选择命名空间。

## §三、MaterialDef TOML 完整格式

### 3.1 力学属性

```toml
[mechanics]
density_kgm3 = 7870          # 密度 (kg/m³)。铁=7870, 水=1000, 木≈600, 金=19300
hardness = 0.45              # 莫氏硬度归一化 0-1。钻石=1.0, 铁≈0.45, 滑石≈0.01
toughness = 0.55             # 断裂韧性 0-1。高=抗断裂(钢0.8), 低=脆(玻璃0.05)
elasticity = 0.3             # 弹性模量归一化 0-1。0=完全塑性(黏土), 1=完美弹性(橡胶)
friction = 0.6               # 表面摩擦系数 0-1。冰≈0.05, 橡胶≈0.9, 铁≈0.6
```

**物理意义**: 
- `hardness` 驱动 STRIKE 的"谁划伤谁"判定。硬度差 > 0.3 → 硬者基本无损
- `toughness` 驱动 BREAK 阈值。force / toughness > 阈值 → 碎裂
- `elasticity` 驱动 STRIKE 的变形类型。低弹性=永��变形（锻打），高弹性=弹回
- `friction` 驱动 GRASP 稳定性、MOVE 抓地力、STACK 堆叠稳定性

### 3.2 热学属性

```toml
[thermal]
specific_heat = 450           # 比热容 J/(kg·K)。水=4184, 铁=450, 金=129
thermal_conductivity = 80     # 热导率 W/(m·K)。铜=401, 铁=80, 木≈0.15, 空气=0.026
melting_point_k = 1811        # 熔点 (K)。铁=1811, 金=1337, 钨=3695, 冰=273
ignition_point_k = 9999       # 燃点 (K)。f32::MAX = 不燃。木≈520, 纸≈506
combustibility = 0.0          # 燃烧强度 0-1。0=不燃, 0.5=木材, 0.9=油脂
```

**物理意义**:
- `specific_heat` 驱动 HEAT 能量需求。高比热容=加热慢（水难烧开）
- `thermal_conductivity` 驱动热传导速度。金属传热快→手握热铁需钳子
- `melting_point_k` 驱动 HEAT 超过熔点→液化。锻造窗口在熔点的 60-80%
- `ignition_point_k` 和 `combustibility` 驱动 IGNITE。温度 > 燃点 + 可燃性 → 着火
- `thermal_softening_rate` 不在 TOML 中——从 `hardness` 和 `melting_point` 自动派生: `hardness_at_temp = hardness × (1 - (temp/melting_point)²)`

### 3.3 电学与魔导属性

```toml
[electrical]
electrical_conductivity = 0.15  # 电导率 0-1。铜=1.0, 铁≈0.15, 橡胶≈0, 水=0.01
mana_conductivity = 0.3         # 魔力传导率 0-1。秘银=0.95, 铁=0.3, 木=0.1, 石=0.05
```

**物理意义**:
- `electrical_conductivity` 驱动 CONDUCT(电)。水+电→范围传导
- `mana_conductivity` 驱动 CONDUCT(魔力) 和 INFUSE。高魔导=法器材料

### 3.4 化学与表面属性

```toml
[chemical]
flammability = 0.0            # 可燃性 0-1。油=0.9, 干草=0.7, 湿木=0.3, 铁=0
solubility = 0.0              # 水溶性 0-1。盐=1.0, 铁=0, 糖=1.0
acidity = 0.0                 # 酸碱度。-1(强碱)~1(强酸)。柠檬汁≈0.3, 铁=0
toxicity = 0.0                # 毒性 0-1。铅=0.6, 砒霜=1.0, 铁=0
porosity = 0.05               # 孔隙率 0-1。海绵=0.9, 铁=0.05, 玻璃=0
oxidizes_when_heated = true   # 加热时是否氧化。铁=true(生锈), 金=false
```

**物理意义**:
- `flammability` 和 `combustibility` 共同驱动 IGNITE→火势蔓延
- `porosity` 驱动 SPREAD(液体吸收率)、DIP(附着持久性)、WIPE(清洁难度)
- `oxidizes_when_heated` 驱动 HEAT + 空气 → 氧化皮（锻造时需 WIPE 去除）

### 3.5 感官属性

```toml
[sensory]
reflectivity = 0.6            # 反光率 0-1。镜面=1.0, 抛光铁=0.6, 炭=0.02
sound_damping = 0.1           # 声阻尼 0-1。橡胶=0.9, 铁=0.1(响亮), 木=0.5
scent_intensity = 0.0         # 气味强度 0-1。铁=0, 松木=0.4, 麝香=0.8
```

### 3.6 魔法属性

```toml
[magic]
elemental_affinity = [0.6, 0.0, 0.1, 0.0, 0.2, 0.0, 0.1, 0.0, 0.05, 0.0]
# 金木水火土风雷电血灵 — 10元素亲和度，各 0-1
# 铁: 金0.6(金属性), 火0.2(锻造关联), 水0.1(淬火关联), 电0.1(导电)
enchantment_capacity = 0.5    # 附魔容量 0-1。秘银=1.0, 铁=0.5, 木=0.3, 石=0.1
aether_retention = 0.4        # 灵元素印记保留率 0-1。石=0.8, 铁=0.4, 水=0.05
```

## §五、装配树材质继承

```rust
/// 遍历装配树，解析任意物品的有效 MaterialProperties
/// 用于物理原子执行时查询——结果缓存在 GripHandle 中避免重复遍历
pub fn resolve_material(item: &ItemEntId, registry: &MaterialRegistry) -> MaterialDef {
    match item.assembly() {
        ItemAssembly::Simple { def } => {
            def.properties.material
                .map(|mid| registry.get(mid))
                .unwrap_or_else(|| registry.default())
        }
        ItemAssembly::Composite { components, .. } => {
            // 找到"干活的部分":
            //   武器 → StrikingHead (slot_type 0x0101)
            //   工具 → ToolHead (slot_type 0x0300 范围)
            //   法器 → ImplementCore (slot_type 0x1001)
            let working_part = find_working_component(components);
            resolve_material(&working_part.occupant?, registry)
        }
    }
}
```

## §六、对制造配方的影响

在 `006-制造与配方.md` 的 `RecipeInput` 中新增可选的材料约束字段:

```rust
pub struct RecipeInput {
    // ... 现有字段保留 ...
    
    /// 材料约束（CHG-042 新增）
    /// None = 不限制材料（使用 item_def 直接指定）
    /// Some = 使用属性范围匹配（支持材料替换涌现）
    pub acceptable_material: Option<MaterialConstraint>,
}

pub struct MaterialConstraint {
    pub category: Option<MaterialCategory>,  // Metal/Stone/Wood/Fiber/...
    pub hardness_min: Option<f32>,
    pub hardness_max: Option<f32>,
    pub density_min: Option<f32>,
    pub density_max: Option<f32>,
    pub melting_point_min_k: Option<f32>,
    pub melting_point_max_k: Option<f32>,
    pub mana_conductivity_min: Option<f32>,
    pub toxicity_max: Option<f32>,
    // 可扩展更多约束维度
}
```

TOML 示例:

```toml
[[recipe.input]]
role = "BladeMaterial"
acceptable_material = { 
    category = "Metal", 
    hardness_min = 0.3, 
    melting_point_max = 2500 
}
quantity = 4
```

## §七、性能

- MaterialDef 注册表: ~200条目 × ~120B = 24KB。启动时一次性加载，常驻 L2 缓存
- `resolve_material()` 遍历: ≤10 节点，<200ns。结果缓存在 GripHandle 中——热路径零查表
- ItemProperties 新增 8B: 5000 DefId × 8B = 40KB 额外内存
- 详见 [[../../NPC活人感模块/08-NPC行动涌现与分类/009-性能预算与LOD联动|物理原子层 009-性能]]

## §八、权威性声明

MaterialDef 的 TOML 格式和注册表由物品系统管理。**字段的物理语义和公式**由 [[../../NPC活人感模块/08-NPC行动涌现与分类/001-NPC行动涌现总纲|NPC行动涌现模块]] 权威定义。消费模块（Combat/Magic/Physics）通过物品系统查询 MaterialDef，但各自读取关心的字段——物品系统不解释材料属性的物理含义。

与现有委托模式一致: `ConsumableEffect`（Life 定义，物品存储）· `AudioMaterial`（音频定义，物品存储）· `MaterialDef`（物理原子层定义，物品存储）。
