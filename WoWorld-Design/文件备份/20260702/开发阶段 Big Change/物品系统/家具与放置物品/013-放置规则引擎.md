## 一、纯函数放置验证——七步管线

单个物品的放置验证是纯函数——不依赖 PlacementStore 或其他模块的内部状态。所有输入通过参数显式传入：

```rust
/// 纯函数——零隐藏状态。可独立单元测试
pub fn validate_furniture_placement(
    item: &ItemPlacementProps,       // 物品自身的物理放置属性
    surface: &dyn Surface,           // 目标表面（woworld_core trait）
    desired_offset: Vec2,            // 表面局部坐标上的期望位置
    desired_rotation: f32,           // 绕表面法线的旋转角
    obstacles: &[PlacedObstacle],    // 已有障碍物（由调用方提供）
    region_portals: Option<&[Portal]>, // 仅运行时放置——用于可达性检查
) -> Result<ApprovedPlacement, PlaceError>
```

**为什么是纯函数**：调用方自己负责从 ItemRegistry/BuildingQuery/EntityIndex 获取输入。物品系统不 import 外部模块——只依赖 `Surface` trait (woworld_core) 和 `PlacedObstacle` (自定 struct)。

### Step 1: 方向匹配

```rust
fn check_orientation(item: &ItemPlacementProps, surface: &dyn Surface) 
    -> Result<(), PlaceError> 
{
    match (item.orientation, surface.orientation()) {
        (PlacementOrientation::Floor, SurfaceOrientation::Floor) => Ok(()),
        (PlacementOrientation::Wall, SurfaceOrientation::Wall) => Ok(()),
        (PlacementOrientation::Ceiling, SurfaceOrientation::Ceiling) => Ok(()),
        (PlacementOrientation::Any, _) => Ok(()),
        _ => Err(PlaceError::WrongOrientation),
    }
}
```

### Step 2: 坡度容差

```rust
fn check_slope(item: &ItemPlacementProps, surface: &dyn Surface) 
    -> Result<(), PlaceError> 
{
    let angle = surface.normal().angle_between(item.preferred_up());
    if angle > item.max_slope_deg {
        return Err(PlaceError::TooSteep { 
            surface_angle_deg: angle, 
            max_allowed_deg: item.max_slope_deg 
        });
    }
    Ok(())
}
```

**典型值**: 高花瓶/立烛台 → max 5°；椅子/桌子/床 → max 15°；地毯/扁平石 → max 30°。

### Step 3: 承载能力

```rust
fn check_load_bearing(item: &ItemPlacementProps, surface: &dyn Surface) 
    -> Result<(), PlaceError> 
{
    let pressure_kpa = (item.mass_kg * 9.81) / (item.footprint.0 * item.footprint.1);
    if pressure_kpa > surface.load_bearing_kpa() {
        return Err(PlaceError::SurfaceTooWeak { 
            required_kpa: pressure_kpa, 
            available_kpa: surface.load_bearing_kpa() 
        });
    }
    Ok(())
}
```

**涌现**: 铁砧放玻璃桌上 → TooWeak → 被拒绝。铁砧放石台上 → 通过。

### Step 4: 足迹碰撞

```rust
fn check_footprint(
    item: &ItemPlacementProps,
    desired_offset: Vec2,
    rotation: f32,
    surface: &dyn Surface,
    obstacles: &[PlacedObstacle],
) -> Result<Aabb2D, PlaceError> {
    let footprint = item.footprint_2d(desired_offset, rotation);
    let clearance = footprint.expand(item.clearance_margin); // 椅子后面需要 0.3m
    
    // 必须在表面范围内
    if !surface.contains_footprint(&clearance) {
        return Err(PlaceError::OffSurface);
    }
    
    // 不能与任何已有障碍物碰撞
    for obs in obstacles {
        if clearance.intersects(&obs.footprint) {
            return Err(PlaceError::Occupied { by: obs.entity_id });
        }
    }
    
    Ok(footprint)
}
```

### Step 5: 垂直净空

```rust
fn check_vertical_clearance(
    item: &ItemPlacementProps,
    surface: &dyn Surface,
    footprint: &Aabb2D,
    obstacles: &[PlacedObstacle],
) -> Result<(), PlaceError> {
    let required_height = item.height_above_surface;
    let base_elev = surface.elevation_at(footprint.center());
    let column = footprint.extrude_3d(base_elev, base_elev + required_height);
    
    for obs in obstacles {
        if obs.bounds_3d.intersects(&column) {
            return Err(PlaceError::CeilingTooLow { blocked_by: obs.entity_id });
        }
    }
    Ok(())
}
```

### Step 6: 通行保留

**世界生成时（Pass B）**——使用"走廊预留法"（快速）：

```rust
fn check_walk_space_generation(
    walk_corridor: &WalkCorridor,     // 预计算——门到关键区的必经路径
    footprint: &Aabb2D,
) -> Result<(), PlaceError> {
    if walk_corridor.intersects(footprint) {
        return Err(PlaceError::BlocksAccess);
    }
    Ok(())
}
```

**运行时**——使用完整 A* 可达性：

```rust
fn check_walk_space_runtime(
    region_portals: &[Portal],
    critical_targets: &[Vec3],       // 床/箱子/工作站——必须可达
    footprint: &Aabb2D,
    all_obstacles: &[PlacedObstacle],
) -> Result<(), PlaceError> {
    let temp_obstacles: Vec<_> = 
        all_obstacles.iter().chain(std::iter::once(&footprint.to_obstacle())).collect();
    
    for portal in region_portals {
        if !has_path_to_any(portal.position, critical_targets, &temp_obstacles) {
            return Err(PlaceError::BlocksAccess);
        }
    }
    Ok(())
}
// 房间 ≤ 50m², 网格 0.25m → ≤ 800 A* 节点 → < 100μs
```

### Step 7: Snap 终定

```rust
fn snap_to_surface(
    item: &ItemPlacementProps,
    surface: &dyn Surface,
    footprint: Aabb2D,
    rotation: f32,
) -> Transform {
    let snapped_pos = match item.snap_mode {
        SnapMode::ToGrid { resolution } => footprint.center().snap(resolution),
        SnapMode::ToAnchor { point } => point,
        SnapMode::Free => footprint.center(),
    };
    
    let up = surface.normal();
    let forward = rotate_around_axis(Vec3::FORWARD, up, rotation);
    Transform::from_origin(snapped_pos, up, forward)
}
```

### 2.2 四阶段分层放置

```
Phase 1: 锚点物品 — 大件，位置由房间功能+文化决定
  → 床（床头靠墙，床尾朝房间中央）
  → 大餐桌（房间中央 或 靠墙——文化 symmetry 决定）
  → 衣柜（靠墙，远离门）

Phase 2: 关系物品 — 相对于锚点位置推导
  → 椅绕桌 (chair_count = f(household_size), 均匀分布+噪声±8.6°)
  → 床头柜靠床 (床头两侧)
  → 灯在床头柜上 (表面链第一步)

Phase 3: 表面链 — 物品叠放在已有物品的表面上
  → 桌布 → 餐桌顶面
  → 主盘 → 桌布中心
  → 餐具 × N → 均匀分布绕主盘
  → 食物 → 餐盘上
  → 中央烛台 → 餐桌顶面

Phase 4: 散落装饰 — Poisson 盘采样填充剩余空间
  → 花瓶/书堆/酒杯 — 最小间距 0.3m
  → 如果放不下 → 不是 bug——"这个房间比较挤"
  → 记录 skipped_items → BuildingHistory
```

### 2.3 家具清单推导

```rust
pub fn derive_manifest(
    region_function: RegionFunction,
    occupant: &NpcFurnishProfile,
    culture: &CultureFurnishProfile,
    materials: &MaterialAvailability,
    rng: &mut impl Rng,
) -> FurnitureManifest {
    let base = FURNISH_TABLE.get(&region_function);
    let mut manifest = FurnitureManifest::new();
    
    for slot in &base.slots {
        // 财富过滤：稻草垫 (wealth < 0.3) vs 四柱床 (wealth > 0.8)
        // 文化覆盖：和风 → 床→布団
        // 材料过滤：本地无硬木 → 去除需要硬木的变体
        // 采样：按权重随机选择一个变体
        let variant = slot.sample(
            occupant.wealth_level,
            culture,
            materials,
            rng.gen_range(0.0..1.0),
        );
        if let Some(v) = variant {
            manifest.add(v);
        }
    }
    
    manifest
}
```

**`FURNISH_TABLE` 是纯数据 (TOML)**:

```toml

slots = [
    { id = "bed", required = true, count = [1, 1] },
    { id = "nightstand", required = false, count = [0, 2], per_occupant = 1 },
    { id = "wardrobe", required = true, count = [1, 1] },
    { id = "lamp", required = false, count = [0, 2] },
    { id = "rug", required = false, count = [0, 1] },
    { id = "decor", required = false, count = [1, 5] },
]
```

## 三、Pass A：附着家具（P11.5a 胶水代码）

```
遍历所有 Building 的 ComponentInstance
  → 筛选 ConnectionFace::PassThrough, usage_tag = Some("chimney_opening")
  → 此口位于 Component 的 PosY 面，world_position = component.pos + face.offset
  → 查 BuildContext.faith/culture → 选择壁炉变体
    火塘文化(fireplace_type=OpenFirePit) → 不在此处——地板火塘走 Pass B
    石砌壁炉文化 → FireplaceFamily + StoneHearth 模板
  → 分配 ItemEntId(种子确定性: hash(world_seed || "fireplace_at_{building_id}_{component_id}"))
  → Place(wall_opening_surface, offset=(0,0))
  → PlacementStore.place() 执行七步验证

同样处理:
  → usage_tag = "builtin_shelf" → StorageFamily + OpenShelf 模板
  → usage_tag = "wall_shrine"   → DecorationFamily, 信仰参数驱动
```

**BuildingGenerator 零改动**——它只是标记了 `usage_tag`。

## 四、表面链级联物理

### 4.1 SurfaceDestroyed 的同帧处理

```rust
impl PlacementStore {
    fn process_surface_destroyed(&self, destroyed: SurfaceId) {
        let mut pending = vec![destroyed];
        
        while let Some(sid) = pending.pop() {
            // 在所有受影响的 chunk 中查找 surface_ref 匹配的条目
            let affected = self.find_items_on_surface(sid);
            
            for item_id in affected {
                // 从当前 item 位置向下做射线检测
                let new_surface = self.find_surface_below(
                    self.transform_of(item_id).position
                );
                
                match new_surface {
                    Some(surface) => {
                        self.relocate(item_id, surface.id(), surface.offset_at(pos));
                        // 如果此 item 自身提供表面 → 其表面也可能被摧毁
                        if self.provides_surface(item_id) {
                            pending.push(SurfaceId::furniture(item_id, FaceDir::PosY));
                        }
                    }
                    None => self.fall_out_of_world(item_id),
                }
            }
        }
        // while 循环——整条表面链在同一帧内收束
    }
}
```

### 4.2 SurfaceEvent::Moved（桌子被推开）

```rust
fn process_surface_moved(&self, moved: SurfaceId, new_transform: Transform) {
    let affected = self.find_items_on_surface(moved);
    for item_id in affected {
        // 重新计算 item 在新表面上的绝对位置
        let new_pos = recompute_absolute_position(item_id, &new_transform);
        // 重新验证至少 Step 2（坡度容差）
        let new_surface = self.get_surface(moved, new_transform);
        if check_slope(&item_props, &new_surface).is_err() {
            // 滑落 → 找下一个表面
            let below = self.find_surface_below(new_pos);
            if let Some(s) = below {
                self.relocate(item_id, s.id(), s.offset_at(new_pos));
            } else {
                self.fall_out_of_world(item_id);
            }
        } else {
            // 留在新位置
            self.update_position(item_id, new_pos);
        }
    }
}
```

## 五、运行时交互

### 5.1 place() — 玩家/NPC 放置

```rust
impl PlacementStore {
    pub fn place(
        &self,
        item_id: ItemEntId,
        surface: &dyn Surface,
        offset: Vec2,
        rotation: f32,
        obstacles: &[PlacedObstacle],
        placer: Option<EntityId>,
        time: GameTime,
    ) -> Result<PlacedEntry, PlaceError> {
        // 1. 查 ItemPlacementProps
        let props = ITEM_REGISTRY.get_placement_props(item_id)
            .ok_or(PlaceError::ItemNotPlaceable)?;

        // 2. 七步验证（纯函数）
        let approved = validate_furniture_placement(
            &props, surface, offset, rotation, obstacles,
            Some(region_portals), // 运行时 → A* 可达性
        )?;

        // 3. 确定目标 chunk
        let chunk_key = ChunkKey::from_world(approved.transform.position);

        // 4. 获取目标 chunk 的写锁（per-chunk RwLock——无全局阻塞）
        let chunk = self.get_or_create_chunk(chunk_key);
        let mut hot = chunk.hot.write();

        // 5. 再次检查碰撞（chunk 级并发安全）
        if chunk.any_intersects(&approved.footprint_2d, &hot) {
            return Err(PlaceError::Occupied { by: /* ... */ });
        }

        // 6. 构建条目
        let entry = PlacedEntry::new(item_id, &approved, surface.id());
        let cold = PlacedCold { placed_at: time, placed_by: placer,
            original_owner: placer, conjured_ttl: None };

        // 7. 插入保持排序（binary_search 找插入点）
        let idx = hot.binary_search_by_key(&item_id, |e| e.item_id)
            .expect_err("item already placed elsewhere");
        hot.insert(idx, entry);
        drop(hot);
        chunk.cold.write().insert(idx, cold);
        chunk.dirty.store(true, Ordering::Release);

        // 8. 更新派生索引
        if props.provides_surface.is_some() {
            chunk.rebuild_surfaces(&ITEM_REGISTRY);
        }
        if props.is_container {
            self.container_map.write().insert(item_id.0, ContainerId::new());
        }

        // 9. 发布事件
        SPATIAL_EVENT_BUS.publish(WorldEvent::ItemPlaced {
            item_id: EntityId::from_item(item_id),
            surface: surface.id(),
            placed_by: placer,
        });

        Ok(entry)
    }
}
```

### 5.2 pick_up() — 拾取

```rust
pub fn pick_up(
    &self,
    item_id: ItemEntId,
    picker: Option<EntityId>,
) -> Result<PlacedEntry, PickUpError> {
    // 1. 如果此物品的表面上有子物品 → 触发 SurfaceDestroyed 级联
    if self.provides_surface(item_id) {
        let furniture_surface = SurfaceId::furniture(item_id.0, FaceDir::PosY);
        self.process_surface_destroyed(furniture_surface);
    }

    // 2. 从 chunk 移除
    let chunk_key = self.item_chunk_index.read().get(&item_id.0)
        .ok_or(PickUpError::NotPlaced)?;
    let chunk = self.chunk_table.read().get(chunk_key)?;
    let mut hot = chunk.hot.write();
    let idx = hot.binary_search_by_key(&item_id, |e| e.item_id)
        .map_err(|_| PickUpError::NotPlaced)?;
    let entry = hot.remove(idx);
    drop(hot);
    chunk.cold.write().remove(idx);
    chunk.dirty.store(true, Ordering::Release);

    // 3. 清理容器
    if entry.flags & FLAG_HAS_CONTAINER != 0 {
        self.container_map.write().remove(&item_id.0);
    }

    // 4. 发布事件
    SPATIAL_EVENT_BUS.publish(WorldEvent::ItemPickedUp {
        item_id: EntityId::from_item(item_id),
        from_surface: SurfaceId::from_raw(entry.surface_ref),
        picked_by: picker.unwrap_or(EntityId(0)),
    });

    Ok(entry)
}
```

## 六、并发放置仲裁

放置是极低频操作（每秒 < 10 次），采用 per-chunk `RwLock`：

```
place() → 写锁目标 chunk 的 hot/cold/surfaces
pick_up() → 写锁目标 chunk
entities_in_radius() → 读锁相关 chunk（每帧数十次——零延迟抖动）
```

**无需乐观锁**——write 持有时间 < 100μs（不含 A*），竞争可忽略。

**批量放置** (`batch_place`): 所有验证在获取写锁前完成（纯函数）→ 获取写锁 → 一次性插入 → 解锁。部分冲突只标记失败而非全部回滚。

## 七、PlaceError 枚举

```rust
pub enum PlaceError {
    WrongOrientation,
    TooSteep { surface_angle_deg: f32, max_allowed_deg: f32 },
    SurfaceTooWeak { required_kpa: f32, available_kpa: f32 },
    OffSurface,
    Occupied { by: EntityId },
    CeilingTooLow { blocked_by: EntityId },
    BlocksAccess,
    ItemNotPlaceable,
    SurfaceDestroyed,
}
```

**每个变体对 UI 的意义**: `Occupied` → 红色 ghost (碰撞位置)；`TooSteep` → 黄色 ghost ("表面太斜")；`SurfaceTooWeak` → 表面材质高亮闪烁；`BlocksAccess` → 橙色 ghost。
