## 〇、定位

本文档定义穿越者模式下的**角色创建流程**和**外观定制参数**，以及所有模式下外观的**动态变化**（着装/伪装/身份隐藏）。原住民模式走[[003-双角色与托管模式|大日志选择]]，不经过本文档的创建向导。

---

## 一、创建流程

### 1.1 呈现形式

**线性 7 步向导 + "快速开始"按钮。**

- 每一步为一个独立页面，包含本步所有参数
- 每步可**返回上一步**、**跳过**（使用默认值）、或**使用随机值**
- 任何时候可用"快速开始"按钮——全部参数默认值 + 随机出生点 + 一套衣服 → 立即进入游戏
- 完成后进入**确认总览页**——展示所有选择的摘要，可返回任意步骤修改

角色创建很重要——一个角色可能玩上百小时。向导给玩家节奏感、不遗漏关键选择。但同时也尊重有时间压力的玩家——快速开始一键跳过。

### 1.2 七步向导

| 步骤 | 内容 | 来源文档 |
|------|------|----------|
| **第一步：身份** | 姓名 / 种族 / 性别 / 出生坐标 | 生命/001 · 生命/002 |
| **第二步：身体** | 体型 + 面部 + 肤色发色 + 伤疤纹身 | NPC活人感 §1.1.7 |
| **第三步：心智** | 属性 + 大五人格 + 认知风格 + 审美 + 吸引力偏好 | 生命/001 · NPC活人感 §1.1.2 |
| **第四步：技能** | 初始技能经验值 / 天生才能乘数 | 技能系统/002 |
| **第五步：魔法** | 十元素亲和 / 魔力属性 / 已知法术 | 生命/001 · 魔法模块 |
| **第六步：语言** | 第一语言 + 额外语言 | 语言表达/002 |
| **第七步：物品** | 初始衣物 / 初始货币 / 可选额外物品 | 经济系统 · 物品系统 |

## 二、可自定义参数详情

以下字段从 `LifeEntity` + `SapientMind` + 技能/魔法/物品系统的完整数据模型中导出。零纯文本"背景故事"框——所有参数都对应到已有数据结构的实际字段。玩家填的值直接写入对应字段——没有翻译层。

### 2.1 第一步：身份

| 参数 | 类型 | 来源 |
|------|------|------|
| `display_name` | String | 生命/001 |
| `species_id` | SpeciesId | 生命/002 —— 人类·精灵·矮人·程序化涌现种族 |
| `biological_sex` | BiologicalSex | NPC活人感 §1.1.7 — Male/Female/Hermaphroditic/Sequential/Neuter |
| 出生坐标 | WorldPos | 随机（野外）或指定 |

**种族选择影响**：
- 精灵：灵元素容量偏高 + 木/灵元素亲和 + 寿命 ~500 年
- 矮人：力量/耐久高 + 火/金元素亲和 + 寿命 ~200 年
- 人类：中性适应力（基准） + 寿命 ~80 年
- 程序化涌现种族：该种子世界的特有智能种族——参数从世界生成+历史涌现
- 所有可选种族共享同一套底层系统——差异**仅在于 RaceTraits**

### 2.2 第二步：身体

| 参数 | 类型 | 来源 |
|------|------|------|
| 力量 / 敏捷 / 体质 / 感知 / 灵巧 / 耐久 | f32 (0-1) | 生命/001 PhysicalAttributes |
| 身高（相对种族均值偏差） | f32 (-1~1) | NPC活人感 §1.1.7 |
| 体型 | BuildType | Slender/Lean/Average/Muscular/Stocky/Heavy |
| 脸宽 / 下巴形状 / 颧骨高度 | f32 | NPC活人感 §1.1.7 (FaceParams 8维) |
| 眼大小 / 眼距 / 鼻梁 / 鼻宽 / 唇厚 | f32 | 同上 |
| 肤色 / 发色 / 发型 / 胡型 | 枚举 | NPC活人感 §1.1.7 |
| 声线 / 音色 | f32 (0-1) | NPC活人感 §1.1.7 |
| 疤痕 / 纹身 / 战纹 / 特征标记 | 可选列表 | NPC活人感 §1.1.7 |

> ⚠️ **外观参数状态标记**：面部和身体的外观设计仍处于设计意图阶段。低多边形风格 + 512² 面部图集 + shader 合成的最终表现力需要在模型建立后验证。以下参数标记为：
> - ✅ **锁定**：参数映射到 LifeEntity 字段、外观=身体能力线索（肌肉量→力量视觉线索、疤痕→战斗历史视觉线索）、无独立"捏脸数据文件"
> - 🟡 **意图·待模型验证**：8 维面部 blend shape、6 种 BuildType、身高偏差 ±1
> - 🔵 **预留**：年龄视觉变化（白发/皱纹/驼背——从 life_stage 派生）、动态疤痕（战斗受伤→自动添加）、纹身/战纹（文化模板库 + 自定义）

### 2.3 第三步：心智

| 参数 | 类型 | 来源 |
|------|------|------|
| 智力 / 智慧 / 意志力 / 魅力 | f32 (0-1) | 生命/001 MentalAttributes |
| 记忆容量 | u16 (0-2000) | 生命/001 |
| 开放性 / 尽责性 / 外向性 / 宜人性 / 神经质 | f32 (0-1) | NPC活人感 §1.1.2 BigFive（至少一个突出维度 \|Δ\|>0.25） |
| 认知风格 4 维 | f32 | NPC活人感 CognitiveStyle — 直觉-分析/冲动-反思/具象-抽象/顽固-灵活 |
| 审美品味 | AestheticTaste (32B) | NPC活人感 §1.1 |
| 吸引力偏好模板 | AttractionTemplate | NPC活人感 §1.1 |

**人格锁定规则**：
- 创建时随便填
- 进游戏后和 NPC 一样——终身极少变动
- 仅极高冲击力创伤 (>0.9) 或长期环境冲突 (>100 游戏日) 时微调 (Δt ≤ 0.05)
- 玩家不能打开面板修改（除非改后台数据）

### 2.4 第四步：技能

| 参数 | 类型 | 来源 |
|------|------|------|
| 任意 SkillId 初始经验值/等级 | (SkillId, xp, level) | 技能系统/002 |
| 天生才能乘数 | f32 (0.7-1.3) | 技能系统 MentalAccess trait |
| 跨训练非递归上限 | SkillGroup 映射 | 技能系统 |

**初始技能的方向引导**（向导页内的小精灵提示）：
- 不强制任何技能
- 小精灵可提示："你想成为怎样的人？"→ 提供模糊方向（"擅长战斗"/"精通手艺"/"学识渊博"/"不设限"）→ 自动分配对应的初始技能经验值
- 也可以手动逐项填写

### 2.5 第五步：魔法

| 参数 | 类型 | 来源 |
|------|------|------|
| 十元素亲和（金木水火土风雷电血灵） | [f32; 10] | 生命/001 SpiritState |
| 魔力强度 / 控制 / 抗性 / 恢复速度 | f32 (0-1) | 生命/001 MagicAttributes |
| 初始已知法术 | Vec\<SpellId\> | 魔法模块 |
| 初始法术书/卷轴 | Vec\<ItemId\> | 物品系统 |

### 2.6 第六步：语言

| 参数 | 类型 | 来源 |
|------|------|------|
| 第一语言 | LanguageId (proficiency 1.0) | 语言表达/002 |
| 额外语言及 proficiency | Vec\<(LanguageId, f32)\> | 语言表达/002 |

### 2.7 第七步：物品/装备

| 参数 | 类型 | 来源 |
|------|------|------|
| 初始衣物 | ItemId | 默认：一套平平无奇的衣服 |
| 初始货币 | u32 | 经济系统 |
| 初始额外物品 | Vec\<ItemId\> | 可选 |

### 2.8 自定义范围

- 玩家可以从零开始（全部默认值）——随机坐标、一套衣服
- 也可以填成龙傲天——全部属性拉满、十元素亲和全高、初始携带神器
- 数值范围由各数据结构的字段类型决定——不添加"角色创建专属限制层"
- 玩家填的值直接写入 LifeEntity + SapientMind 的对应字段——没有翻译层

## 三、外观动态

### 3.1 着装与装备可见性

外观随装备变化自动更新——不存储"外观快照"。换装备 = 换模型引用。

| 触发 | 视觉变化 |
|------|----------|
| 更换头盔/帽子 | 头部模型替换——可能遮住发型/面部 |
| 更换身体护甲 | 躯干模型替换——覆盖体型线条 |
| 手持武器/工具 | 手部持握模型 |
| 背负盾牌/弓箭 | 背部挂载 |

**原则**：装备 = 外观。没有"装备A属性 + 幻化成装备B外观"的幻化系统——别人看到你穿什么就是什么。这是"信息诚实"——NPC 从你的着装判断你的职业/财富/威胁。

### 3.2 伪装与身份隐藏

玩家可以手动切换**伪装模式**：

| 伪装维度 | 效果 |
|----------|------|
| 隐藏姓名 | NPC 不直接"知道"你的名字——除非你告诉他们或被认出来 |
| 隐藏装备 | 披风/斗篷覆盖装备外观——但战斗时不生效 |
| 身份伪装 | 声称自己是另一个人——NPC 是否相信取决于你的演技（Speech skill）+ NPC 的智慧/感知 + 是否有矛盾信息 |

**伪装不是魔法隐身**：NPC 仍然能看到你的身体特征（种族/体型/面部）。熟人仍可能认出你——"这个人虽然用兜帽遮住了脸……但他的体型和走路方式很像老王打铁坊的学徒。"

### 3.3 无头衔/名声标签

**系统不显示任何头衔或名声标签。** 不存在"XX 的征服者"/"YY 的铁匠大师"之类的 UI 文字。

别的 NPC 怎么看你——完全从他们的言行中涌现。他说"老朋友"可能是真诚的，也可能是骗你。他说"久仰大名"可能是真听说过你，也可能是在试探你的反应。

这是"玩家 = NPC"的贯彻——NPC 也没有头顶标签。你的名声在他们的记忆里，不在 UI 里。

## 四、原住民模式的差异

原住民模式不经过 7 步向导。玩家从大日志中选择 NPC 后直接进入游戏。

- 角色的所有参数——身份/身体/心智/技能/魔法/语言/物品——**已经是该 NPC 现有的**
- 进入后如需了解角色擅长什么 → 大日志 + 小精灵
- 角色的人际关系/财产/义务/名声——全部继承（Q4）
