## 〇、定位

本文档是"玩家 = NPC"的**差异清单**——逐系统列出玩家和 NPC 走不同路径的地方。它不设计新东西，而是把分散在 CLAUDE-INTERFACES.md 和各种模块文档中的差异条目**汇总、系统化、验证一致性**。

原则：**每一处差异必须有明确理由。没有理由的差异 = 应该消除。**

## 一、逐系统差异清单

### 1.1 物理

| 维度 | NPC 路径 | 玩家路径 | 差异理由 |
|------|----------|----------|----------|
| 物理引擎 | Rust 空间查询四 trait（TerrainQuery/EntityIndex/SpatialEventBus/VisibilityQuery） | Godot PhysicsServer3D (CharacterBody3D) | 玩家需要精确碰撞响应 + 手柄/键鼠输入 → 物理引擎直接处理。仅玩家保留 PhysicsServer3D |
| 碰撞响应 | 空间查询返回布尔/距离——不产生物理反作用力 | 完整刚体物理——推/被推/滑/跳 | 玩家体感需要 |
| 移动解析 | GOAP 输出目标位置 → 寻路 → 逐帧插值 | 输入 → CharacterBody3D.move_and_slide() | 输入延迟 < 1 帧 |

> 来源：技术栈方案 v4.0 §物理架构、CLAUDE-INTERFACES.md TDI-032/045/202

### 1.2 心智

| 维度 | NPC 路径 | 玩家路径 | 差异理由 |
|------|----------|----------|----------|
| 决策引擎 | GOAP + 概率行为树 | 人类玩家输入（ControlMode 覆盖） | 本质差异——玩家有心智 |
| GOAP 状态 | 持续运转，产出动作 | Auto/DomainDelegated 时正常运转；Manual 时后台运转但不执行 | 人格塑造——GOAP 持续产生"假设"，供自我叙事合理化 |
| 记忆写入 | 完整感官管线 → 记忆编码 | 同 NPC（轻量感官 → 大日志） + 玩家亲眼看到的 | 玩家感知 = 轻量感官管线（perceive_lightweight） |
| 情绪 | 情绪引擎自主运转 | 同 NPC——情绪不受 ControlMode 影响 | 玩家的角色仍然有情绪 |

> 来源：[[../NPC活人感模块/07-生命周期系统/008-玩家生命周期|NPC生命周期 008]]

### 1.3 战斗

| 维度 | NPC 路径 | 玩家路径 | 差异理由 |
|------|----------|----------|----------|
| 战斗 AI | 同一套 Rust 代码（三层模型：本能→节奏→战略） | 同一套 + 可选手动干预 | 玩家可选择干预 |
| 自动执行 | 目标选择/移动/普通攻击节奏/防御时机/体力管理 | 同 NPC | — |
| 手动干预 | — | 战斗风格切换/特殊技能释放/队伍命令/关键决策/灵兽指令 | 玩家专属决策 |

> 来源：[[../生命/011-玩家|生命 011]] §三

### 1.4 感官

| 维度 | NPC 路径 | 玩家路径 | 差异理由 |
|------|----------|----------|----------|
| 感官管线 | 完整——注意筛选 + 噪声注入 + 内感受 | 轻量（perceive_lightweight）——实体发现 + 分类，不做注意筛选/噪声注入 | 玩家自己看屏幕——不需要模拟"看" |
| 信息记录 | 记忆系统编码（有损、有偏见） | 大日志自动记录（玩家的经历——相对完整） | 大日志是玩家外置大脑 |

> 来源：CLAUDE-INTERFACES.md 大日志感官输入

### 1.5 对话

| 维度 | NPC 路径 | 玩家路径 | 差异理由 |
|------|----------|----------|----------|
| 发起方 | GOAP 决定是否发起对话 | 玩家按键发起（按键行为 NPC 不可感知） | hand/011 |
| 对话选项 | GOAP 生成 utterance | 预设选项（滚轮/数字键选择）+ 自由文本输入 + 语音输入 + 托管一句话 | hand/011 |
| 声量 | GOAP 决定 | 玩家鼠标滚轮调节 | — |
| 对话中活动 | 通常专注对话 | 可一边对话一边做其他事，可突兀打断/被打断 | hand/011 |
| 对话渲染 | 四层可见性（Full/Partial/Barely/Invisible） | 玩家听到的取决于与声源的距离 + PlayerPerception 衰减 | CLAUDE-INTERFACES.md |

## 二、输入映射

### 2.1 设计原则

- **手柄 + 键鼠平等支持**（hand/011 裁决）
- **可全改键**——所有映射可由玩家重定义
- **默认键位由 UI/UX 系统定义**（轨B.1）——本文档只列动作分类和约束
- **输入不直接触发游戏逻辑**——输入 → InputAction（平台无关枚举）→ 游戏逻辑

### 2.2 玩家动作完整清单

以下动作需要输入映射。具体键位在 UI/UX 系统中定义。

### 2.3 手柄适配约束

- 所有战斗动作需在 **不离开摇杆** 的情况下可达（肩键/扳机 + 面键组合）
- 对话选项选择：方向键 / 摇杆 + 确认键
- 轮盘选择：右摇杆——释放触发
- UI 面板导航：方向键 / 左摇杆

## 三、信息展示边界

### 3.1 原则

秉承"不泄露 NPC 不知道的信息"原则。信息展示分为三类：

| 类别 | 规则 | 例 |
|------|------|-----|
| ✅ 可见 | 玩家角色自身能感知的信息 | 自身 Vitals、自身技能、自身装备、自己的大日志 |
| ✅ 可见 | 物理世界可直接观察的信息 | 天气、时间、地图（已探索区域）、物品外观 |
| ❌ 不可见 | 其他 NPC 的内部状态数值 | NPC 的 HP/情绪/好感度/GOAP 目标/对你的真实评价 |
| ❌ 不可见 | 系统 meta 信息 | 谁在背后议论你、未探索区域的内容 |

### 3.2 详细边界

| 信息 | 可见性 | 理由 |
|------|--------|------|
| 自身 HP / 饥饿 / 口渴 / 疲劳 / 体温 / 灵元素 | ✅ HUD | 你的身体你能感知 |
| 自身部位伤害 | ✅ 身体图示（模糊——"左臂受伤"，非"左臂 0.37/1.0"） | — |
| 自身技能列表 + 等级 | ✅ 技能面板 | 你知道你会什么 |
| 自身装备 + 物品栏 | ✅ 物品栏 UI | — |
| 自身大日志 | ✅ 大日志面板 | — |
| 自身目标（PlayerGoal） | ✅ 置顶笔记 / 大日志条目 | 你自己写的 |
| 小精灵提示 | ✅（可关闭） | 仅玩家可感知 |
| **其他 NPC 的 Vitals** | ❌ | 你看不到别人的 HP 条 |
| **其他 NPC 的技能/属性** | ❌ | 你只能从观察推断——"他打铁很熟练" |
| **其他 NPC 对你的好感度/信任值** | ❌ | 认知仅从 NPC 言行涌现。他说"老朋友"可能是真诚的，也可能是骗你 |
| **其他 NPC 的 GOAP 当前目标** | ❌ | 你不知道别人在想什么 |
| **其他 NPC 的记忆内容** | ❌ | — |
| **头衔/名声标签** | ❌ | 不显示"XX 的征服者"——别人怎么看你，从他的言行推断 |
| 地图（已探索区域） | ✅ 世界地图 | 你去过的地方你记得 |
| 地图（未探索区域） | ❌ 知识迷雾（灰色） | — |
| 天气/时间/季节 | ✅ HUD（可隐藏） | 你能感知 |
| 当前操控角色的 ControlMode | ✅ 小图标提示 | DomainDelegated 时显示哪些域在托管 |
| 另一个角色的位置/状态 | 仅切换时摘要 | 不推送通知——去看看才知道（Q14） |

## 四、视线之外的非操控角色

当玩家操控角色 A 时，角色 B（未被操控）：

- **距离近**（< L1 范围）：和任何其他 NPC 一样——L1 全模拟。大日志持续记录其经历。
- **距离远**（> L1 范围）：退回 L2/L3 模拟——统计层面（在做什么、关键事件）。和任何其他 NPC 相同。
- **切换时加载**：如果 B 在远处 → 加载 B 所在区域 → **游戏暂停** → 加载完摄像机切过去（Q9）。

角色 B 永远不会因"是玩家的另一个角色"而被特殊对待——LOD 系统平等对待所有实体。
