## 一、定位

生命周期时钟与事件系统是**生命模块的事件发射层**——不定义生命过程本身（那是 004 的职责），而是将生命过程中的离散状态转换为跨模块事件，供各消费模块独立订阅。

**核心设计原则**：
- **中性事实**：所有事件不携带情感建议、行为期望、人际通知——消费模块自行解读
- **确定性**：`AgeClock::advance_age(entity, delta_days, rng)` 同输入+同种子→同输出，与调用频率无关
- **EventBus 分发**：所有事件通过 EventBus 发射，零轮询

## 二、AgeClock — 生命周期时钟

### 2.1 签名

```rust
/// 纯函数——推进实体年龄，返回生命周期事件列表
/// 同输入 + 同种子 → 同输出。与调用频率无关。
fn advance_age(
    entity: &LifeEntity,
    delta_days: f32,
    rng: &mut impl Rng,
) -> (LifeEntity, Vec<LifeEvent>);
```

### 2.2 阶段判定

| 阶段 | 年龄范围（占 max_lifespan 比例） | 属性乘数 |
|------|-------------------------------|---------|
| `Infant` | 0 ~ 2% | 0.2× 全属性 |
| `Juvenile` | 2% ~ 15% | 0.5× → 0.8×（渐变） |
| `Adolescent` | 15% ~ 25% | 0.8× → 0.95× |
| `YoungAdult` | 25% ~ 45% | 1.0×（峰值） |
| `Adult` | 45% ~ 75% | 1.0× → 0.9× |
| `MiddleAge` | 75% ~ 95% | 0.9× → 0.7× |
| `Elder` | 95% ~ 100% | 0.7× → 0.4× |

### 2.3 阶段转换规则

阶段间的过渡为**渐进式**（属性乘数线性插值），而非阶跃。`LifeStageTransition` 事件在阶段边界发射（年龄恰好跨越阶段阈值的那一 tick）。

```rust
fn life_stage(age: f32, max_lifespan: f32) -> LifeStage {
    let ratio = age / max_lifespan;
    match ratio {
        r if r < 0.02 => LifeStage::Infant,
        r if r < 0.15 => LifeStage::Juvenile,
        r if r < 0.25 => LifeStage::Adolescent,
        r if r < 0.45 => LifeStage::YoungAdult,
        r if r < 0.75 => LifeStage::Adult,
        r if r < 0.95 => LifeStage::MiddleAge,
        _            => LifeStage::Elder,
    }
}
```

## 三、LifeEvent 枚举

```rust
/// 生命周期事件——中性事实
/// 不携带情感建议/行为期望/人际通知。
/// 各消费模块通过 EventBus 独立订阅并自行解读。
#[derive(Clone, Debug, Serialize, Deserialize)]
pub enum LifeEvent {
    /// 出生事件
    Birth(BirthEvent),

    /// 生命阶段转换
    StageTransition(StageTransitionEvent),

    /// 死亡事件
    Death(DeathEvent),

    /// 妊娠结束（成功分娩或流产）
    GestationEnded(GestationEndedEvent),

    /// 受伤事件（部位 damage 超过阈值）
    Injury(InjuryEvent),

    /// 疾病状态变更
    Disease(DiseaseEvent),

    /// 灵元素枯竭临界（spirit 进入危险区间）
    SpiritCritical(SpiritCriticalEvent),

    /// 状态效果变更（中毒/流血/眩晕等标签附加或移除）
    StatusEffectChanged(StatusEffectChangedEvent),

    /// 驯服完成（灵兽 bond_level 突破阈值）
    CreatureTamed(CreatureTamedEvent),
}
```

## 四、事件负载定义

### 4.1 BirthEvent

```rust
pub struct BirthEvent {
    pub entity_id: EntityId,
    pub birth_timestamp: GameTime,
    pub location: WorldPosition,
    pub mother_id: EntityId,
    pub father_id: Option<EntityId>,
    pub gestation_duration_days: f32,
    pub birth_order: u8,          // 多胞胎序号（1-based）
    pub attendants: Vec<EntityId>, // 接生/见证者
    pub initial_health: f32,      // 0-1，初始健康度
    pub race: RaceTraitsId,
}
```

### 4.2 StageTransitionEvent

```rust
pub struct StageTransitionEvent {
    pub entity_id: EntityId,
    pub from_stage: LifeStage,
    pub to_stage: LifeStage,
    pub age_years: f32,
    pub timestamp: GameTime,
}
```

### 4.3 DeathEvent

```rust
pub struct DeathEvent {
    pub entity_id: EntityId,
    pub death_cause: DeathCause,
    pub death_location: WorldPosition,
    pub death_timestamp: GameTime,
    pub final_spirit: f32,        // 死亡时刻灵元素残留
    pub witnesses: Vec<EntityId>,  // 在场见证者
    pub summary: DeathSummary,     // → 历史模块 LifeTrace
}
```

### 4.4 InjuryEvent

```rust
pub struct InjuryEvent {
    pub entity_id: EntityId,
    pub body_part: BodyPart,
    pub previous_damage: f32,
    pub new_damage: f32,
    pub severity: InjurySeverity,  // Light / Moderate / Severe
    pub cause: InjuryCause,        // 战斗/事故/环境/魔法
    pub source_entity: Option<EntityId>, // 施加方
}
```

### 4.5 SpiritCriticalEvent

```rust
pub struct SpiritCriticalEvent {
    pub entity_id: EntityId,
    pub spirit_ratio: f32,        // 当前 spirit/max_spirit
    pub stage: SpiritDangerStage,  // Warning(30%) / Critical(10%) / Fatal(0%)

}
```

### 4.6 其他事件（DiseaseEvent, StatusEffectChangedEvent, CreatureTamedEvent）

结构体定义与上类似——包含 entity_id、时间戳、变更前后状态、关联来源。详细字段在对应子系统文档中展开。

## 七、性能

- `advance_age()` 纯函数——单次调用 < 0.001ms
- 事件发射走 EventBus 无锁队列
- 阶段判定为 O(1) 查表（7级 if-else）
- 建议调用频率：每游戏日一次（非每帧）。L1 距离内实体每游戏日，L2/L3 批量处理
