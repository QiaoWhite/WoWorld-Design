# 006 — 权力与市场

## 一、核心洞察

`MarketRegulations` 不是静态配置——它们是人设定的，带着意图、人格和局限设定的。国王、宰相、领主、行会会长、地头蛇、垄断商人、玩家——各自有权力、有动机、有信息局限。

**关键设计原则**：掌权者和其他NPC一样——用自己的心智模型做决策，受限于自己的信息。国王不是神——他根据汇报给他（可能失真）的信息来制定经济政策。

**为什么放在经济系统而非政治系统？**
市场权力是对市场参数的修改能力——这是经济系统的内核机制。权力的来源（继承/选举/政变/任命）是政治系统的事——但权力对市场的作用接口属于经济系统。经济系统定义"此人能否动这个参数"——政治系统定义"此人怎么获得这个资格"。

## 二、PowerAtom——权力的原子单位

### 2.1 从复合身份到原子能力

"国王"、"领主"、"行会会长"——这些是权力的复合身份，不是权力的原子。就像"铁器商"是 `[锻造+交易]` 的复合，"国王"是多种权力原子的复合。

```rust
/// 权力原子——对市场能执行的单项操作
/// 不是"角色"——是"能力"
#[derive(Debug, Clone, PartialEq, Eq, Hash, Serialize, Deserialize)]
pub enum PowerAtom {
    /// 设定某物品的价格上限
    SetPriceCeiling { item: ItemDefId, market: MarketId },
    /// 设定某物品的价格下限
    SetPriceFloor { item: ItemDefId, market: MarketId },
    /// 设定某实体在某市场的交易配额
    SetQuota { entity: EntityId, item: ItemDefId, market: MarketId },
    /// 调整交易税率
    SetTaxRate { market: MarketId },
    /// 禁止/解禁某物品在某市场交易
    ToggleItemBan { item: ItemDefId, market: MarketId },
    /// 征用某实体的物品
    Requisition { entity: EntityId, item: ItemEntId },
    /// 发放/撤销营业许可（MarketLicense）
    GrantLicense { entity: EntityId, market: MarketId },
    /// 将某实体驱逐出某市场
    Exclude { entity: EntityId, market: MarketId },
    /// 发行货币（铸币权）
    IssueCurrency { amount_copper: u64 },
    /// 裁决交易纠纷（最终裁定权）
    AdjudicateDispute { trade_id: TradeId },
    /// 授予/撤销他人的某个PowerAtom（权力的权力）
    DelegateAuthority { entity: EntityId, power: Box<PowerAtom> },
    /// 设立/取消关税
    SetTariff { item_category: ItemCategory, destination: MarketId, rate: f32 },
    /// 强制收购配额
    SetMandatoryQuota { entity: EntityId, item: ItemDefId, quantity: u32, price: u32 },
}
```

### 2.2 MarketAuthority——谁拥有哪些原子

```rust
/// 市场权力持有记录——谁在什么市场有什么权力
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct MarketAuthority {
    pub authority_id: AuthorityId,
    /// 权力持有者（NPC 或 Player）
    pub holder: EntityId,

    /// 此持有者拥有的权力原子集合
    pub powers: BTreeSet<PowerAtom>,

    /// 权力的合法性/认可度 0-1
    /// 影响 enforcement_capacity——低合法性→政策推不动
    pub legitimacy: f32,

    /// 执法能力 0-1
    /// 取决于：下属数量+忠诚度、武力资源、法治传统
    pub enforcement_capacity: f32,

    /// 权力来源类型——用于历史记录和叙事
    pub source: AuthoritySource,
}

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub enum AuthoritySource {
    /// 继承/任命——正式权力（政治系统的领域）
    Inherited { from: Option<EntityId>, title: String },
    /// 购买——用铜币换权力行使权
    Purchased { from: EntityId, price_copper: u64 },
    /// 事实产生——市场占有率达垄断水平
    DeFacto { item_def: ItemDefId, market_share: f32 },
    /// 暴力/胁迫——没人敢违抗
    Coercion { enforcer_count: u32, territory: RegionId },
    /// 选举/推举
    Elected { term_start: GameTimestamp, term_end: Option<GameTimestamp> },
}
```

## 三、权力来源的四条路径

### 3.1 正式权力（继承/任命/选举）

政治系统负责（继承法/官僚制/选举制）。经济系统只查询结果："此人是否拥有此PowerAtom？"不关心怎么获得的。

### 3.2 财富→权力（购买）

这是经济系统直接处理的部分。本质上是**用铜币向掌权者购买他手中的一个PowerAtom行使权**。

```rust
/// 权力交易——用铜币换权力的行使承诺
/// 不需要"贿赂系统"——这就是一笔交易
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct PowerDeal {
    pub deal_id: PowerDealId,
    /// 卖方：掌权者（承诺行使权力）
    pub authority: EntityId,
    /// 买方：出钱的人
    pub buyer: EntityId,
    /// 承诺行使什么权力
    pub promised_action: PowerAtom,
    /// 买方付了多少铜币
    pub price_copper: u64,
    /// 交易状态
    pub status: PowerDealStatus,
    pub created_at: GameTimestamp,
}

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub enum PowerDealStatus {
    /// 钱已付，等待兑现 → 买方风险：掌权者可能赖账
    Paid,
    /// 已兑现
    Fulfilled,
    /// 收钱不办事 → 买方怎么处理？由买家人格驱动
    Breached,
    /// 买方反悔（撤回了）
    Cancelled,
}
```

**掌权者是否愿意"出售"权力行使权**——由他的心智驱动：
- 低宜人性 → 更愿意受贿
- 高智慧 → 懂得合理定价、评估被抓风险
- 高神经质 → 怕被更高层发现 → 不敢
- 低尽责性 → 收了钱可能不办事

**买方是否愿意行贿**——由他的心智驱动：
- 低宜人性 → 不介意行贿
- 低神经质 → 敢冒险
- 高智慧 → 懂得选时机和对象

### 3.3 事实权力（经济力量→定价权）

不靠任何人的授权——靠市场占有率。

```rust
/// 事实垄断检测——单一卖家控制市场供给超过阈值
fn detect_de_facto_monopoly(
    market: &Market,
    item_def: ItemDefId,
) -> Option<DeFactoMonopoly> {
    let shares = calculate_market_shares(market, item_def, 30); // 30天窗口

    // 单一卖家 > 40% → 事实垄断
    if let Some((seller_id, share)) = shares.largest() {
        if share > 0.40 {
            return Some(DeFactoMonopoly {
                monopolist: seller_id,
                item_def,
                market_share: share,
                /// 定价权 = 超出竞争阈值的比例
                pricing_power: (share - 0.40) / 0.60, // 0-1
                is_cartel: false,
            });
        }
    }

    // 多个卖家合计 > 60% + 有派系/团体关系 → 卡特尔
    if let Some(cartel) = detect_cartel(&shares) {
        return Some(DeFactoMonopoly {
            monopolist: cartel.ringleader,
            item_def,
            market_share: cartel.combined_share,
            pricing_power: cartel.cohesion * 0.8,
            is_cartel: true,
        });
    }

    None
}
```

垄断检测结果 → 自动写入 `MarketAuthority`（来源 = `DeFacto`）→ 其他NPC（包括权力持有者）可能对此做出反应。

### 3.4 暴力/胁迫

地头蛇、黑帮——靠暴力维持的市场控制权。经济系统只记录此人实际拥有的PowerAtom——不管手段是否合法。法律系统（待设计）负责定义哪些PowerAtom的获取方式是"犯罪"。

## 四、权力效果的全光谱——从毁灭到繁荣

权力对市场的影响不是好坏二分。**人格×能力×信息 = 六个典型组合**：

```rust
fn power_impact_profile(
    holder: &NpcData,
    authority: &MarketAuthority,
) -> PowerImpactProfile {
    let competence = holder.mental.intelligence * 0.3
        + holder.mental.wisdom * 0.4
        + holder.economic_state.cognition.financial_literacy * 0.3;

    let public_spiritedness = holder.personality.agreeableness * 0.45
        + holder.personality.conscientiousness * 0.20
        + holder.self_narrative.core_values
            .iter()
            .filter(|v| matches!(v.value.as_str(), "公正" | "责任" | "服务"))
            .map(|v| v.strength)
            .sum::<f32>().min(1.0) * 0.35;

    let information_quality = estimate_report_accuracy(holder, authority);

    PowerImpactProfile { competence, public_spiritedness, information_quality }
}
```

| 能力 | 公益心 | 信息 | 掌权者类型 | 市场涌现效果 |
|------|--------|------|-----------|------------|
| **高** | **高** | **高** | **贤明治理者** | 精准政策→适度减税+保护商路+饥荒预储。**市场持续繁荣** |
| **高** | **高** | **低** | **好心办坏事** | 政策意图对但力度失准（信息被下级扭曲）。**阶段性市场紊乱** |
| **高** | **低** | **高** | **高效寻租者** | "聪明的腐败"——保护费合理、维持秩序、打击竞争者。**短期经济扩张** |
| **高** | **低** | **低** | **受蒙蔽的掠夺者** | 过度压榨+信息闭塞→杀鸡取卵。**经济快速崩溃** |
| **低** | **高** | — | **善良的庸官** | 政策要么太松要么太死。市场自顾自运转。**≈无干预** |
| **低** | **低** | — | **昏庸的掠夺者** | 税制混乱+贪婪→收不上税+更狠压榨→经济崩溃+税收也无增长。**最烂组合** |

**关键涌现现象**：
- 高效寻租者——腐败但高效的中世纪语境：行会交保护费→获得垄断经营权的保护→行会利润增长→经济扩张（短期）。长期天花板：新进入者被排斥→创新停滞
- 权力从"好心办坏事"→"贤明治理"的转变需要一个信息校准事件——如玩家亲自汇报实情、正直官员被提拔

## 五、权力行使流程

```rust
/// 掌权者尝试修改市场规则
fn exercise_power(
    authority: &MarketAuthority,
    holder: &NpcData,
    action: PowerAtom,
    target_market: &mut Market,
) -> Result<RegulationChange, PowerExerciseError> {
    // 1. 资格检查——此人是否拥有执行此操作的PowerAtom？
    if !authority.powers.contains(&action) {
        return Err(PowerExerciseError::NotAuthorized);
    }

    // 2. 执法能力检查——政策能否落地？
    if authority.enforcement_capacity < 0.2 {
        // 政令不出官府——但仍然可以"宣布"
        // 宣布但无法执行 → legitimacy下降
        return Err(PowerExerciseError::InsufficientEnforcement);
    }

    // 3. 利益冲突检查——掌权者是否在利用权力自肥？
    let conflict = detect_self_dealing(holder, &action, target_market);

    // 4. 执行——修改 MarketRegulations
    let change = apply_regulation_change(target_market, &action, holder);

    // 5. 记录——写入历史系统
    record_regulation_change(&change, conflict);

    Ok(change)
}
```

**信息局限 → 政策失当的涌现**：
国王得到的汇报可能失真（汇报者自己有动机夸大/隐瞒）。国王根据失真信息做决策 → 政策可能过激或不足。这不是bug——是叙事的来源。"国王被奸臣蒙蔽"不是脚本，是信息传播失真+权力结构的自然产物。

## 六、玩家作为权力持有者

玩家通过多条路径获得 PowerAtom：

| 路径 | 获得什么 | 示例 |
|------|---------|------|
| **被任命**（政治系统） | 地方官——管理本地市场 | 玩家被国王任命为边境领主→获得领地内市场的税率/许可权 |
| **加入/创立行会** | 行会会长——行业准入+价格规范 | 玩家铁匠行会会长→设定本镇铁制品最低价 |
| **财富购买**（经济系统） | 贿赂/政治献金→特定PowerAtom行使权 | 玩家资助国王战争→被封男爵→附带税收权 |
| **市场垄断**（经济系统） | 事实定价权 | 控制60%铁矿石供应→你的要价就是市场价 |
| **继承/政变**（政治系统） | 最高权力 | 玩家政变成功→获得前任国王的全部PowerAtom集合 |

**核心原则**：玩家行使权力和NPC行使权力走**同一套代码路径**。`exercise_power()` 不区分调用者是玩家还是NPC。

## 七、与政治系统的接口（预留）

经济系统提供：
- `MarketAuthority` 数据结构和查询接口
- `PowerAtom` 的完整枚举
- `exercise_power()` 的执行和验证流程
- 权力冲突检测（两个authority对同一市场有矛盾诉求）

政治系统（Phase 13.4·待设计）负责：
- 正式权力的初始分配（世界生成时谁当国王/领主）
- 正式权力的转移规则（选举制/世袭制/任命制/政变机制）
- 权力合法性的判定（革命/篡位→legitimacy变化）
- 权力斗争（两个权力持有者冲突的高层仲裁）
