# 007 — 提示音系统 v1.0

> **核心设计**：提示音是纯系统→玩家通信——NPC 听不到、不进物理模拟、不被环境噪声掩蔽、始终清晰 2D 播放。各业务模块通过 `world_core` 事件总线发射 `PlayerNotificationEvent` 枚举——不知道音频存在。音频模块订阅事件总线→查 `notification_sounds.toml` 规则表→播放。对标已有模式：大日志订阅事件（写历史）+ UI 层订阅事件（浮动文字）。

## §一 提示音与世内声音的本质区别

| 维度 | 世内声音 (WorldSound/Voice) | 提示音 (Notification) |
|------|---------------------------|----------------------|
| 物理模拟 | ✅ 传播/衰减/掩蔽/打断 | ❌ 不进入模拟 |
| NPC 感知 | ✅ NPC 可听到 | ❌ NPC 完全不知 |
| 空间化 | 3D | 2D（直接播放） |
| 掩蔽 | 受环境噪声影响 | ❌ 始终清晰 |
| 音量控制 | World SFX / Voice 滑条 | Notification 独立滑条 |
| 资产组织 | sfx/ 目录 | sfx/ui/ 目录 |
| 进入方式 | SoundFootprint/HasSoundEmitter | world_core 事件总线 |

## §二 PlayerNotificationEvent 枚举（woworld_types）

```rust
/// 各模块发射这些事件 —— 不知道音频存在。
/// 大日志也订阅（写历史）、UI 也订阅（浮动文字）。
/// 音频模块订阅 → 查规则表 → 播放。
#[derive(Clone)]
pub enum PlayerNotificationEvent {
    // === 战斗 ===
    ThreatDetected { direction: Vec3, threat_type: ThreatType, distance: f32 },
    SkillCooldownReady { skill_id: SkillId },
    EquipmentBroken { item_id: ItemEntId, slot: EquipmentSlot },
    EquipmentNearBreak { item_id: ItemEntId, durability_fraction: f32 },
    LowHealth { current: f32, max: f32 },
    CombatEnter,
    CombatExit,

    // === 任务 ===
    QuestAccepted { quest_id: u32 },
    QuestCompleted { quest_id: u32 },
    QuestFailed { quest_id: u32 },
    ObjectiveUpdated { quest_id: u32, objective_index: u8 },

    // === 物品 ===
    ItemAcquired { item_id: ItemDefId, rarity: Rarity, count: u32 },
    InventoryFull,
    ItemDropped { item_id: ItemDefId },
    CraftingComplete { recipe_id: u32 },
    CraftingFailed { recipe_id: u32 },
    EnchantSuccess { item_id: ItemEntId },
    EnchantFail { item_id: ItemEntId },

    // === 生存 ===
    HungerWarning { severity: f32 },
    ThirstWarning { severity: f32 },
    TemperatureWarning { temp: f32, trend: TempTrend },
    DrowningImminent { oxygen_remaining_pct: f32 },
    ElementImbalance { element: u8, severity: f32 },
    FatigueWarning { stamina_pct: f32 },
    EncumbranceWarning { weight_ratio: f32 },

    // === 技能 ===
    SkillLevelUp { skill_id: SkillId, new_level: u16 },
    SkillUnlocked { skill_id: SkillId },
    TalentTriggered { talent_id: u32 },

    // === 社交 ===
    NpcGreeting { npc_id: EntityId, relationship_level: f32 },
    NpcHostile { npc_id: EntityId },
    LetterReceived,
    RumourHeard,

    // === 经济 ===
    TradeComplete { value: u64 },
    AuctionWon { item_id: ItemDefId, price: u64 },
    AuctionOutbid,
    DebtDue { amount: u64 },

    // === 载具 ===
    VehicleDamaged { severity: f32 },
    FuelLow { remaining_pct: f32 },
    DestinationNear,
    VehicleDocking,

    // === 天气 ===
    ExtremeWeatherWarning { weather_type: u8, eta_seconds: f32 },
    SeasonChange { new_season: u8 },

    // === UI/系统 ===
    MenuNavigate,
    MenuConfirm,
    MenuBack,
    SaveComplete,
    SaveFailed,
    ScreenshotTaken,
    AchievementUnlocked { achievement_id: u32 },

    // === 其他 ===
    Discovery { discovery_type: u8 },  // 隐藏区域/秘密/地标
    PuzzleHint { puzzle_id: u32 },
    MusicTuningComplete,               // 玩家乐器调音完成
}
// ~52 个变体。各模块可申请新增——不改现有代码，只需在枚举后追加。
```

## §三 触发机制：事件总线

```
业务模块（战斗/物品/技能/...）
    ↓ 发射 PlayerNotificationEvent（不知道音频存在）
world_core 事件总线
    ├→ 音频模块订阅 → 查 notification_sounds.toml → 播放
    ├→ 大日志订阅 → 写入历史事件
    └→ UI 模块订阅 → 显示浮动文字/通知提示
```

**零耦合**：业务模块不知道谁在消费。新增消费者不需要改业务代码。

## §四 规则表（notification_sounds.toml）

```toml
[item_acquired]
event = "ItemAcquired"
sound_variants = [
    { rarity = "Common",    sound = "ui/item_pickup_common",  volume = 0.7 },
    { rarity = "Uncommon",  sound = "ui/item_pickup_uncommon", volume = 0.8 },
    { rarity = "Rare",      sound = "ui/item_pickup_rare",     volume = 0.9 },
    { rarity = "Legendary", sound = "ui/item_pickup_legendary", volume = 1.0, interrupt = true },
]
max_overlap = 2                  # 最多同时 2 个
min_interval_ms = 200            # 此间隔内不重复播

[low_health]
event = "LowHealth"
sound = "ui/heartbeat_warning"
loop_while_active = true         # 持续播放直到血量恢复
fade_in_ms = 300
fade_out_ms = 500
pitch_range = [0.8, 1.3]        # 随 severity 自动 pitch
volume_range = [0.6, 1.0]       # 随 severity 自动 volume
interrupts = []                  # 不打断其他音

[drowning_imminent]
event = "DrowningImminent"
sound = "ui/underwater_countdown"
loop_while_active = true
pitch_range = [0.9, 1.5]        # 越接近溺水 pitch 越高
volume_range = [0.7, 1.0]
interrupts = ["LowHealth"]       # 比低血量警告更优先

[threat_detected]
event = "ThreatDetected"
sound = "ui/threat_alert"
priority = "Critical"            # 打断一切
interrupts = ["*"]               # * = 全部
min_interval_ms = 3000           # 3 秒内不重复
```

**规则驱动**：新提示音 = 在 TOML 加规则 + 放音频文件。零代码。

## §五 冷却/打盹/音量闪避

### 5.1 冷却管理

```rust
struct NotificationCooldown {
    last_played: HashMap<NotificationKey, GameInstant>,
    consecutive_skipped: HashMap<NotificationKey, u8>,
    last_any_ui_sound: GameInstant,
}

fn should_play(&mut self, event: &PlayerNotificationEvent, config: &NotificationRule) -> NotificationDecision {
    let key = event.type_key();

    // 最小间隔
    if let Some(last) = self.last_played.get(&key) {
        let elapsed = now - *last;
        if elapsed < config.min_interval {
            self.consecutive_skipped.inc(&key);
            return NotificationDecision::Skip;
        }
    }

    // 连续跳过过多 → 强制播一次
    let skipped = self.consecutive_skipped.get(&key).copied().unwrap_or(0);
    if skipped >= 5 {
        self.consecutive_skipped.reset(&key);
        return NotificationDecision::Play { volume: 1.0, force: true };
    }

    // 音量闪避——密集提示时降低音量
    let ducking = if (now - self.last_any_ui_sound).as_secs_f32() < 0.3 {
        0.6  // 降至 60%
    } else {
        1.0
    };

    self.consecutive_skipped.reset(&key);
    NotificationDecision::Play { volume: ducking, force: false }
}
```

### 5.2 多提示音叠加

最多同时 3 个（可配置）。新提示音到来时的行为：

| 当前播放数 | 新提示音优先级 | 行为 |
|-----------|-------------|------|
| <3 | 任意 | 播放 |
| =3 | Critical | 替换最低优先级 |
| =3 | High | 替换最低（如果最低不是 Critical/High） |
| =3 | Normal/Low | 丢弃 |
