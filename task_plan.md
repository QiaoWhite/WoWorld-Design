# Task Plan: WoWorld 设计文档工作

## Goal
管理和推进 WoWorld 游戏的设计文档——在 `开发阶段/` 中产出正式开发规格，逐步将 `想法/` 中的概念设计成熟化、正式化。

## 仓库结构理解
- **`开发阶段/`** — 正式开发规格。经过讨论确定的、预备投入开发的设计。权威。
  - 技术栈方案/ · NPC活人感模块/ · 战斗/ · 魔法/ · 世界生成/ · 生命/ · 历史/
- **`想法/`** — 概念设计、脑暴、早期草稿。允许矛盾、模糊、未决。
- **`参考文档/`** — 参考性设计文档，编号归档。
- **`Change/`** — 设计变更追踪（CHG-XXX 格式）。

## Current Phase
**CHG-047 全模块系统性交叉审计 (Phase 1: 基础设施清理)** — 修复CHG编号重复/模块槽位冲突/缺失CHG文档/过时文档。后续 Phases 2-5 将覆盖模块接头交叉验证、跨模块内容深度审计、设计哲学一致性审计、修复与文档。
