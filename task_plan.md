# Task Plan: WoWorld 设计文档工作

## Goal
管理和推进 WoWorld 游戏的设计文档——在 `开发阶段/` 中产出正式开发规格，逐步将 `想法/` 中的概念设计成熟化、正式化。

## 仓库结构理解
- **`开发阶段/`** — 正式开发规格。经过讨论确定的、预备投入开发的设计。权威。
  - 技术栈方案/ · NPC活人感模块/ · 战斗/ · 魔法/ · 游戏概述.md
- **`想法/`** — 概念设计、脑暴、早期草稿。允许矛盾、模糊、未决。
- **`参考文档/`** — 参考性设计文档，编号归档。
- **`Change/`** — 设计变更追踪（CHG-XXX 格式）。

## Decisions Made
| Decision | Rationale |
|----------|-----------|
| 开发阶段 = 权威规格 | `想法/` 是草稿，`开发阶段/` 是正式结果 |
| 技术决策以 技术栈方案 v3.0 为准 | 所有模块技术方案的最高依据 |
| 变更用 CHG 文档追踪 | 结构化修改需要变更记录 + 交叉引用 |
