## §1 阶段目标

**一句话**：Layer 0 和 Layer 1 的全部模块达到 🟢 稳定状态——世界能生成、玩家能走、天会亮、海有浪、物品技能生命天气存档全部就位。

## §6 冲刺执行规则

1. **冲刺粒度**: 每冲刺 ≤3 个目标。一个里程碑可以拆分为多个冲刺。
2. **并行策略**: 里程碑 1B-1I 无互相依赖——可以交错冲刺（例如：1B 做累了换 1F 换口味）。但单个冲刺内保持专注。
3. **研究先行**: 每个新 crate 的架构设计（数据流、trait 拆分、模块边界）属于 🟠 级——必须先研究行业方案再编码。
4. **Godot 桥接同步**: 每个里程碑完成后，更新 `woworld_godot` 桥接代码，确保 Godot 场景能验证。
5. **接口冻结**: 里程碑完成 = 该模块的 pub trait 签名进入「冻结」状态。后续修改必须走场景 I 升级协议。
6. **冲刺提案**: 每个冲刺前必须按 [附录F-冲刺模板](../附录F-冲刺模板.md) 写提案。用户审批后执行。

## §9 偏差与重规划协议

```
同一目标连续 2 冲刺未完成 → 🟡 偏差：AI 自主调整时间线
同一目标连续 3 冲刺未完成 → 🔴 重评估：AI 分析 + 用户裁决
                             选项: A) 继续投入 B) 降级验收标准 C) 拆分为更小粒度
同一里程碑 2 次重评估 → 🔴🔴 重规划：用户主导重新设计该里程碑
发现架构级不可行 → 立即停止，不等 3 次——走场景 I 技术升级协议
```

**Phase 1 特殊规则**：里程碑 1B-1I 之间无互相依赖。如果某个里程碑持续卡住，可以暂时搁置，集中火力推进其他里程碑。Phase 1 出口要求**全部 10 个里程碑完成**，但推进顺序可以灵活调整。
