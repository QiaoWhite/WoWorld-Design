### 09:30-15:00 — 诊断绿色多边形根因（5 个半小时，3 个 bug）

#### 尝试 1: 以为是新增 LOD 导致 SH→SH 重叠 z-fighting
修了 compute_transition_faces 和 desired_keys。无效。

#### 尝试 2: 二分法回退
- 回退 LOD 6-7 → 问题仍在
- 回退 Part A 去重 → 问题仍在
- 回退地形参数 → 问题仍在
- **结论: 基线代码（Sprint-016 提交 `5c18b8d`）就有这个 bug**

#### 尝试 3: 全 SH 诊断
把所有 LOD 从 Transvoxel 改为 SH → 问题消失。
**结论: bug 在 Transvoxel 渲染路径。**

#### 尝试 4: 隔离 Transvoxel 子模块
- 移除洞穴密度层 → 无效
- MC 参考实现替代 Transvoxel → 问题消失（加载极慢）
- 禁用 transition cells → **大面积多边形消失**，剩 bumps 和 cracks
- **Bug #1: transition cells 几何错误**

#### 尝试 5: 修复 bumps 和 cracks
禁用 edge cache → 恶化（条纹状半透明）。
**发现根因**: `marching_cubes.rs` 和 `transvoxel.rs` 的角落密度数组访问（d数组）和 `global_corners` 函数中，y 和 z 的 stride 被**互换**了。

- x 步进 = 1
- y 步进 = **nx**（代码用了 ny·nx）
- z 步进 = **ny·nx**（代码用了 nx）

这导致：密度值从错误位置读取 + edge cache 键值计算错误 → 山体 bumps + 裂缝。
**Bug #2: y↔z stride 互换**

#### 尝试 6: 恢复 transition cells 验证
stride 修复后恢复 transition cells → 绿色多边形复现。
**确认 Bug #3: transition cells 必须禁用（暂时）**
