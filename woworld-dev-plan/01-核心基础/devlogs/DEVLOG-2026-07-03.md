### 关键架构洞察

**Perlin 格点陷阱**：`noise` crate 的 `Perlin::get([0.0, 0.0])` 对所有 seed 返回 0.0——数学性质，非 bug。`continent_scale × 0 + 0 = 0` 对任何 scale 都是格点。修复：无理数相位偏移 `Perlin(x·s + φ⁻¹, z·s + 1−φ⁻¹)` 使原点 seed-dependent。

**cargo test ≠ cargo build**：`cargo test` 产出测试可执行文件，`cargo build` 产出 cdylib。Godot 加载 `target/debug/woworld_godot.dll`——只有 `cargo build --workspace` 更新它。此后每个冲刺末尾必须 `cargo build` 才能验证。

**Cross-sampling 纹理引用生命周期**：`Gd<ImageTexture>` 在 Godot 中是指针。swap 后旧纹理被引用计数回收——持有旧句柄的 shader 读到零纹理。必须每帧从 Rust 字段重新绑定 `fine_heightmap` / `coarse_heightmap`。

### LOD 过渡质量判定

双层 morphing 就位后，用户评估："当前 LOD 质量已经够好"。判断依据：
- 30m 处 0.5m→1.0m 间距变化在 1080p 下约 3-4 像素
- 6m 宽过渡带将变化分散到 2-3 秒步行时间
- 继续投入 ROI 极低——真正消除需法线贴图或多 LOD 层，均为大工程

**决策**：停止 LOD polish。将精力转向海洋渲染、植被、或玩家交互。
