# 开发日志 — 2026-07-04

## Sprint 029: 海洋地基 + 地形修复

**承接**：Sprint 028 LOD 接缝修复完成，用户判定质量"够好"。

### OceanProvider trait — woworld_core 海洋查询接口

在 `woworld_core` 新增 `ocean` 模块，定义 `OceanProvider` trait（4 方法）：
- `sea_level_at(pos)` — 海平面高度（当前恒为 0.0）
- `wave_height_at(pos, time)` — Gerstner 波面高度
- `water_depth_at(pos)` — 水深
- `is_underwater(pos, time)` — 是否在水面以下

`HeightfieldOcean`（`woworld_worldgen`）为噪声驱动实现，持有 `Arc<WorldNoise>`——与 `HeightfieldTerrain` 共享同一实例。Rust 侧 Gerstner 波数学与 shader 同步。

### 海洋视觉升级

**DEPTH_TEXTURE per-fragment 深度色变**：`ocean.gdshader` 使用 Godot 4.x 的 `hint_depth_texture` uniform 声明，在 fragment shader 中重建海底世界坐标。每个像素独立计算水深，海岸绿松石渐变到深海暗蓝。删除了旧的单 scalar `water_depth` uniform。

**透明度修复**：新增 `ALPHA = 1.0 - smoothstep(-2.0, 2.0, seabed_y)`——陆地（seabed_y > 0）上方海面透明，真正海洋区域不透明。

### 地形光照修复

`terrain.gdshader` 去掉 `unshaded` render_mode + 新增梯度法线计算 + `NORMAL = mat3(VIEW_MATRIX) * world_normal`。地形现在响应 `DirectionalLight3D`（正午亮、午夜暗）。

### 背面剔除修复

根因：`clipmap.rs` 中每个 quad 的两个三角形绕序不一致——`[tl,bl,tr]` 是 CCW，`[tr,bl,br]` 是 CW。经 `cull_front` 对照测试确认管线翻转绕序后，将全部 5 处索引反转为 CW，配合 `cull_back` 正常工作。

### 大陆尺寸 + 雾修复

- `continent_scale`: 0.001 → 0.0001（10 倍波长，小岛变大陆）
- `sea_threshold`: 0.3 → 0.2（更多陆地）
- 水下 Environment 调制加 `is_underwater` gate——水上关雾，不抢昼夜 ambient

## Sprint 030: Transvoxel 骨架

**背景**：设计文档（007 §1.2, 015 §4.1）规定近处 LOD 0-3 使用 Transvoxel 3D 等值面提取。P2a DensityStack 地基 Sprint 029 已就位，前置条件满足。

### Step 1: CaveDensity — 3D 洞穴密度层

新建 `cave.rs`，消费 `worley_3d_f2f1()`（Worley 3D |F₂−F₁| 脊状噪声）产生 3D 隧道网络。

累加模式下洞穴层返回负密度（`-amplitude`）抵消地形正密度：
- `TerrainBaseDensity`: `h - y` → 正值=实体
- `CaveDensity`: `worley < thr → -80.0, else → 0.0`
- 总和: 地下实体区域被洞穴空洞打断 → Transvoxel 提取出连通隧道

参数（`noise_params.toml` `[cave]` section）：frequency=0.04, threshold=0.15, amplitude=80.0。

### Step 2: 恢复过渡表

从 git 历史恢复 `transition_tables.rs`（615 行）——Lengyel 算法自动生成的 6 个查找表。纯数据，零修改。

### Step 3: transvoxel_extract()

新建 `transvoxel.rs`，从 `DensityStack` 消费密度数据，输出 `TerrainMeshData`。

关键适配（vs 旧 `DensityField` [0,1] 体系）：
- 等值面阈值 0.5 → **0.0**（正值=实体，负值=空）
- `density.sample(x,y,z)` → `stack.density_at(WorldPos{x,y,z})`
- 材质颜色由 `material_colors` HashMap 驱动（与 `generate_heightmap_data()` 一致）
- 顶点共享（edge_key 全局缓存）——减少约 2/3 冗余顶点
- 过渡单元（半厚度偏移）——消除 LOD 接缝

MC EDGE_TABLE / TRI_TABLE / EDGE_ENDPOINTS 从 git 历史提取，存入 `tri_table_data.rs`（302 行）通过 `include!` 内联。

4 个单元测试：空栈→空 mesh、正密度球体→球面三角形、确定性验证、顶点共享验证。全部通过。
