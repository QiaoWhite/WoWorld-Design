# DEVLOG — 2026-07-05 (Sprint 031 性能+海洋+体素)

## 摘要

从 Sprint 031 Transvoxel Godot 集成起步，完成了三轮性能优化、海洋视觉质量修复、VoxelChunk LOD 0 替换（部分）。Release mode 验证流畅，72 测试全绿。VoxelChunk 渲染有未解决问题——chunks 呈碎片化。

## 性能优化

### Round 1: CPU 纹理上传 (3 commits)
- **纹理池**：8 层各预分配 2 套 Image+ImageTexture，harvest 改用 `set_data()+update()` → 零 VRAM 运行时分配
- **L3 margin**：`hm_size 256→512, hm_extent 1024→2048`，margin 8m→520m，飞行时 Perlin/s **16x↓**
- **帧预算**：`try_recv()` 每帧 max 1 harvest → 消除批量纹理上传尖峰

### Round 2: GPU shader (3 commits)
- **Depth Pre-Pass 移除**：Ocean `hint_depth_texture`→`seabed_y` uniform（Rust 每帧传入）→ 地形从渲染 2x→1x
- **冗余 h_c 删除**：terrain shader 647K tex fetch 消除
- **Dirty flags**：28 次 cross-sampling `set_shader_parameter` → 按需更新
- **Sun 常量移除**：`set_sun_angle_max(5.0) + set_sun_curve(0.5)` 移出每帧热路径

### Round 3: Release + 配置 (3 commits)
- `[profile.dev] opt-level=1`：debug 构建轻度优化
- Ocean `cull_disabled`→`cull_back`：80K 三角形不再渲染双面
- `project.godot`：显式关闭 SDFGI/SSAO/SSIL/volumetric_fog
- **Release mode 验证**：`cargo build --release` → 飞行完全流畅

## 海洋视觉质量

- 深度曲线 `sea_factor.powf(0.4)`：大洋中部 -196m（原 -67m）
- 海岸线平滑：`land_factor * detail/mountain_height`——消除 100m+ 大陆边缘凸起
- 海底材质分层：`h<-100m→Stone, -100<h<-10m→Gravel, -10<h<0→Sand`
- 新增 detail noise 到海洋分支：`ocean_detail * sea_factor * 50.0`

# DEVLOG — 2026-07-05 (Sprint 033: MC绕序 + LODCoordinator + 天气Phase1 + PBR法线修复)

## 摘要

承接 Sprint 032 VoxelChunk 修复，在新工作流程下完成四项工作。最关键成果：**PBR 法线修复**——10 轮隔离诊断定位 Floating Origin→PBR 后暴露的视角依赖光照异常。84 测试全绿。

## 完成内容

### MC 绕序 → cull_back
- 根因: Paul Bourke MC 表 256 cases 绕序不一致，全局 swap 一刀切无效
- 修复: `winding_is_godot_front()` 运行时面法线×顶点法线点积检测
- 过渡单元 double-flip bug 同时修复
- 文件: transvoxel.rs, voxel_terrain.gdshader

### LOD 6-7 范围对齐 CHG-049

- LOD 7: 7-10km → 10-15km
- cull_margin: 10km → 15km

### 天气系统 Phase 1

- ⚠️ 架构债: 离散状态硬编码 → 需升级为连续参数驱动 (WeatherVisualAggregator)

### PBR 法线修复 ★
- 现象: 太阳当空时摄像机朝下，全地形视角依赖"多边形阴影"
- 10 轮诊断: unshaded→SPECULAR=0→shadow=false→ambient_light_disabled→normalize(NORMAL)→世界法线可视化→自定义light()→per-fragment heightmap法线
- 根因: clipmap 16m+ 三角形顶点法线透视插值漂移 → dot(N,L) 错误
- 修复: terrain.gdshader fragment 中 heightmap 重算法线, voxel_terrain.gdshader dFdx/dFdy
- 保留 Godot 标准 PBR, SPECULAR=0, ROUGHNESS=1.0

## 诊断教训

- sed 破坏 shader 语法 → 编译失败 → 假阳性诊断结果。始终用 Write 工具写 shader
- ensure_correct_normals 在 Godot 4.7 是 unimplemented 空操作——文档明确标注
- 隔离变量一次只改一个，保持回退基线
- const/static f64 数组迭代在某些 Rust 优化级别下行为异常
