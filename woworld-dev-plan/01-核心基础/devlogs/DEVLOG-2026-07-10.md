## 一、grill 深挖

用户以 `/grill-me` 逐叉拷问视距/世界尺度/海洋方案，跨 10 维工程权衡逐一定夺。

| 轮次 | 决策 |
|------|------|
| 终态 | 默认视距 20km，极限 40km，连续滑块 1-40km 步长 1km，确认后生效 |
| 世界尺度 | UK (~500km²) → 欧洲 (~3200km²)，异世界群岛拓扑——几块大陆 + 大岛链 + 无数小岛 + 汪洋 |
| 海:陆 | 6:4（旧 7:3） |
| 海陆噪声 | 双层 AND：super_continent (3333km) × region (333km)，涌现大陆/群岛/汪洋，替换旧单层 sea_threshold |
| LOD 扩展 | 8→10 层：+L8(20-30km/128m SH) + L9(30-40km/256m SH)，LEVELS 常量化 |
| 远景采样 | FP2 "Rough Generator" 落地——L8/L9 仅采 super_continent+region，跳过 detail+mountain |
| 海洋架构 | 2-tier：near 5km/512²/Gerstner (3 涌浪 25/35/50m) + far 50km/200²/Flat Fresnel |
| 波参数管理 | ocean_waves.toml 单一真相源——Rust HeightfieldOcean + GLSL shader 共同消费，替换双边硬编码 |
| 大气远景 | Shader 距离指数雾 + L8/L9 跳过 per-fragment normal |
| ViewDistance | `woworld_core` 新类型，单一真相源，各子系统独立消费。GDExtension #[export] property |
| 世界噪声 | seed[5]→[6]，continent_scale→region_scale 重命名，sea_threshold 删除（被双层阈值替代） |
| 海深公式 | 加权 deficit：`(sc_deficit + rc_deficit) * 0.5`，替代旧 sea_factor |
| 控制台 | `/viewdistance <km>` 作为可选调试辅助（非主要交互） |

## 二、外部技术研究

5 个子代理并行研究外部实践，结果记录于 grill 过程中：

| 来源 | 技术 | WoWorld 落地 |
|------|------|-------------|
| FP2 (FarPlaneTwo) | Rough Generator——跳过中间计算直接估高 | `sample_height_coarse()`——L8/L9 仅双层噪声 |
| Voxy | 独立渲染层（天空盒类比） | L9 `is_distant_layer` uniform——跳过 per-fragment normal |
| CDLOD/UDLOD | 逐顶点 morphing 消 pop | **已有**——terrain.gdshader fine/coarse smoothstep blend，L7→L8→L9 纯配置 |
| Crest Ocean | 相机相对 LOD 网格 | 2-tier 海洋设计 |
| DH (Distant Horizons) | 渐变球体过渡 | → 替换为物理大气散射消隐（shader 距离指数雾） |
| Decima (地平线) | GPU compute 批量剔除 | 远期参考——当前 CPU 管线够用 |
| MSFS 2024 | 屏幕比例 LOD + SimProp 容器 | 地标系统参考——远期 |
| Bruneton 2017 | 预计算大气散射 LUT | 远期升级路径——当前用经验近似 |
| Nvidium | GPU Mesh Shader | GTX 1660 SUPER 支持——远期 L8/L9 mesh GPU 生成 |
