## 当前架构

```
Godot 场景:
  WorldDriver (GDExtension Rust)
    ├── LOD_0..7 MeshInstance3D (8 层环形 clipmap 网格)
    │   ├── 每层: 静态 ArrayMesh + ShaderMaterial + R32F heightmap 纹理
    │   ├── Vertex shader: VTF 采样 heightmap → Y 位移
    │   └── 每帧: camera_pos uniform + node_pos uniform (按需)
    ├── 昼夜循环 (DirectionalLight3D + ProceduralSkyMaterial)
    └── 海洋 (OceanPlane, Gerstner wave)
  
  Player (GDScript, WASD + 鼠标 + Space/G)
```

### 数据流
```
启动: generate_clipmap_grid() → ArrayMesh (一次性)
      创建空 heightmap 纹理 (512×512 R32F × 8 层)

每帧: 收割 ray on 完成的 heightmap → ImageTexture::update()
      玩家移动 → 按需提交新 heightmap job → rayon::spawn()
```

## 仍然存在的问题

### 主要问题: FPS 个位数
地形可见但性能极差。已完成但未奏效的修复：
- 坐标空间 mismatch (node_pos uniform) — 已修复
- 全盘 overdraw → 环形网格 — 已修复
- 冗余 per-frame set_global_position — 已修复
- 异步 heightmap 生成 (rayon) — 已实施

### 最可能的原因
1. VTF 在 Godot Forward+ 的 vertex shader 中性能极差（最可疑）
2. Shader 有运行时错误（`set_code` 不返回 Result）
3. Heightmap 纹理上传路径导致 GPU stall
4. 环形网格拓扑有覆盖空洞

## 下一步：二分调试

建议从最小可行 VTF 平面开始，逐步叠加到当前完整代码：
1. 1 个平面 + VTF shader（无 heightmap 纹理）
2. + 1 张 heightmap 纹理
3. + 8 层网格
4. + 环形拓扑

每步编译测试，FPS 骤降的那一步即为瓶颈。
