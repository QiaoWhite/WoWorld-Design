## 完成的工作

### Sprint 036: ECS Phase 1 生命系统（上）— 死亡→掉落涌现链

**新增 5 Component + 3 System + 1 Resource**

#### 5 生命 Component (`components/vitals.rs`)

| Component | 字段 | 语义 |
|-----------|------|------|
| `Vitals` | hp, max_hp, stamina, hunger, thirst, body_temp, oxygen (7×f32) | "这个实体还活着" |
| `DeathCause` | category (DeathCategory), specific (u8), source (Option\<EntityId\>) | "怎么死的、谁杀的" |
| `Corpse` | death_tick (u64), corpse_temperature (f32) | "死多久了、尸温" |
| `PendingLoot` | _(零字段 tag)_ | "等待掉落判定" |
| `LootResult` | [Option\<ItemDefId\>; 8], count (u8) | "掉落了什么" |

**关键设计决策**：
- **Corpse 带数据**（非 tag）——death_tick 一处写入多处读取，corpse_temperature 支持涌现式感官线索
- **PendingLoot 零字段 tag**——DeathWatch 不知道"掉什么"，LootRoll 读 EntityKind 自决
- **LootResult 固定数组 [8]** ——≤8 物品，72B，铁律 2 合规

#### 3 生命 System (`systems/life/`)

| System | 触发条件 | 操作 |
|--------|---------|------|
| `death_watch` | Vitals.hp <= 0 | cmd: remove Vitals, insert Corpse+PendingLoot+DeathCause |
| `loot_roll` | Corpse + PendingLoot + EntityKind | 查 LootTableRegistry → cmd: remove PendingLoot, insert LootResult |
| `item_spawn` | Corpse + LootResult + Position | 每物品 cmd.spawn(DroppedItem Entity), remove LootResult |

**涌现链**（一帧延迟是设计特性）：
```
帧 N:   Vitals{hp:0} → DeathWatch → cmd(装 Corpse+PendingLoot)
帧 N+1: Corpse+PendingLoot → LootRoll → cmd(装 LootResult)
帧 N+2: Corpse+LootResult → ItemSpawn → 物品 Entity 诞生
```
