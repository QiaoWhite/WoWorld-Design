**核心架构决策**：
- **零外部依赖**：过渡查找表从 transvoxel-data npm 包提取（MIT/Apache 2.0），Python 脚本生成 Rust 静态数组，`transition_tables.rs` 自动生成
- **架构 A**：过渡单元嵌入 `extract_isosurface_transvoxel()` 内部，复用顶点缓存和输出缓冲区——API 不变
- **transition_faces 语义**：纯空间信息——"这个面需要过渡单元"。Transvoxel 不关心邻居是什么算法
- **高分辨率侧负责**：过渡三角形由高分辨率 tile 在其边界面上生成（算法唯一正确实现）
- **L1 Transvoxel 参数**：voxel_size=2.0m，不硬编码 LOD 级别假设
- **查找表格式**：4 张 `[[i8; 32]; 512]` × 4 面——与 `TRI_TABLE` 格式统一
- **邻居检测**：`compute_transition_faces()` 纯函数——基于 LOD tile 坐标推导线，不依赖 active set
