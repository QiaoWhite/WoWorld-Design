## 🎯 目标（≤3 个）

### 目标 1: 缓冲机制核心 — 满容量淘汰 + pop_if 物理重检（I1+I2）
- **验收标准**: `cargo test -p woworld_ecs -p woworld_core` 通过，且以下各有单测：
  - **I1 满容量淘汰**（008 §二）: `CInputBuffer::push_bounded(entry)` — 容量 4，满时淘汰最低优先级旧条目；新条目优先级 ≤ 最低现存 → 丢弃。`action_resolver_system` 的裸 `entries.push` 换成 `push_bounded`。测试：塞 5 条 → 长度仍 4，被淘汰的是最低优先级条目。
  - **I2 过期清理 + pop_if 物理重检**（008 §二）: `input_buffer_system` drain 前先剔过期条目（`is_expired(now)`，签名加 `now`）；drain 改为**只弹出物理可行条目**——`is_physically_possible(req)` = `registry.get(req.action_id).physics_req.is_satisfied_by(effective_loco)`（**复用** `action_controller_tick` 同款谓词，单一权威，F3）。`effective_loco` = `compute_locomotion(pos, terrain)` + 土狼时间上调（**不是** CMovementState，F2）——抽 `resolve_effective_loco(pos, terrain, coyote_remaining)` 共享 helper，消解 movement_system/action_system 两处现有重复。空中 `Jump`（physics_req=Grounded）不弹出、留缓冲直到落地或过期。测试：空中缓冲 Jump→不 drain；grounded→drain；过期→被清。
- **涉及模块**: 角色控制器 008 §二 `WoWorld-Design/Happy Game/开发阶段/模型动作与物理系统/角色控制器/008-手感系统.md`
- **涉及代码**: `woworld_ecs/.../components/input_state.rs`（`push_bounded` + `CInputFeelConfig` 补 ledge 字段）；`.../systems/input/input_buffer_system.rs`（签名加 `terrain`/`registry`/`now`，query `Position`+`CCoyoteTime`，过期 + physics pop_if）；`.../systems/input/action_resolver_system.rs`（`push`→`push_bounded`）；共享 `resolve_effective_loco` helper（movement 或 kinematics 模块）；`woworld_godot/src/terrain_chunk.rs`（input_buffer_system 调用点补参数）

## 🔍 计划↔文档审计修正记录（2026-07-11·对照 008/003/004/002 + 代码勘查）

> 提案初稿经逐条对照设计文档与现有代码后的修正。分三类：不吻合（已自修）/ 遗漏（已补记）/ 意外增补（已消除或呈裁决）。

**不吻合 → 已自修：**
- **F1 双权威**：初稿拟给 `CInputFeelConfig` 加 `jump_buffer_secs`。实测缓冲窗口权威已是 action registry `def.buffer_window_ms`（per-action，`action_resolver` 已读并写入 `BufferedInput.expires_at`）。再加全局字段 = 双权威（违反设计门 #13）→ **删除，仅加 `ledge_snap_distance`/`ledge_snap_angle`**。
- **F2 loco 来源错**：初稿称 I2 物理重检"查 `CMovementState`"。实际 `effective_loco` = `compute_locomotion(pos, terrain)` + 土狼上调（action_system.rs:69-74）→ **改为 position+terrain+coyote，抽 `resolve_effective_loco` 共享 helper**（同时消解 movement_system/action_system 两处现有重复，净简化）。
- **F3 谓词单一权威**：I2 的 `is_physically_possible` **复用** `physics_req.is_satisfied_by(loco)`（action_controller.rs:91 同款），不自造第二套物理判定。
- **F5 I3 过度耦合 CJustLanded**：初稿把落地消费挂 `CJustLanded`。实际 loco 转 Grounded 即令缓冲 Jump 可行→drain→起跳，`CJustLanded` 非必需 → **I3 解耦为 I2 的纯行为验证**，`CJustLanded` 降为可选精修（零延迟同帧起跳）留待需要时。

**意外增补 → 已消除/呈裁决：**
- **F6 ledge_snap 玩家门控**：008 §七 未限定仅玩家（〇/§九 只说 `CInputBuffer` 玩家专属）。此门控是我方推断 → **改为 Q4 呈用户裁决**，不擅定。

**遗漏 → 已补记（本冲刺不做，透明声明）：**
- **§三消费链 vs BufferPriority**：§三"空闲时按语义类别消费"由 `action_controller_tick` 的 `max_by_key(ActionRequest.priority)` 实现（已在线）；`BufferPriority` 仅管**淘汰**（I1）。两套优先级各司其职，非冲突，无需改。
- **§六连招缓冲已就位**：`cancel_window_open` + `cancel_set_ids` 中断路径（Sprint-065 已修比对 bug），本冲刺 I1-5 不含 §六，正确。
- **008 §一 turn_smoothing / acceleration_curves**：转身平滑/加减速曲线是 008 §一 手感参数，**不在 I1-5 范围**（I1-5 = 缓冲淘汰/物理重检/落地预输入/边缘吸附/空闲门控）→ 留待后续手感冲刺。
- **死数据存疑（低优先，非本冲刺）**：`CInputBuffer.prev_frame_inputs`（"刚按下vs持续按住"疑与 `InputState.was_pressed` 重复）、`InputFeelConfig` 的 `dodge_buffer_ms`/`combo_buffer_ms`（窗口已 per-action 到 registry，疑 vestigial）→ 记入待观察，非本冲刺清理项。
- **文档↔代码命名漂移（doc-sync 待办，非本冲刺）**：008 用 `max_snap_distance`/`max_snap_angle_deg`（§一/§七）与 `CPlayerTag`（§九），代码用 `ledge_snap_distance`/`ledge_snap_angle` 与 `PlayerComponent` → 属既有文档漂移，留 `/woworldidea-design` doc-sync。
