## 🎭 语言涌现定位（回应"话语是否硬编码/真涌现"·本冲刺哲学红线）

**结论：预写片段库 = 设计规格本身（非偷懒）；但涌现必须活在"选择/组合"层——初稿的硬阈值选句偏硬编码，本冲刺已强化为状态驱动的涌现选择。**

- **词是预写的，且这是设计明文**：`003-文本生成引擎` §一"**片段组合，非单体模板**" + §1.3"**~430 片段全部 TOML 数据驱动**"；`006` §1.4 ~210 应酬片段。WoWorld **刻意不凭空生成语言**——生成式造句是**可选 1% LLM 层**（技术栈方案），非基线。TOML 片段库 = 规格，非绕过涌现（真人打招呼也用"早安"定式）。
- **涌现在"选择"不在"造词"**：WoWorld 语言涌现 = **涌现状态驱动下对预写片段的涌现式选择+组合**。设计 `FragmentCondition`（人格阈值/`MinTrust`/情绪/`TimeOfDay`/文化/`All`·`Any`·`Not`）× `FragmentSelection`（`RandomMatch`/`FirstMatch`）× 参数填槽 → 线性片段生指数组合。涌现落在"**谁·什么心情·什么关系·什么时段·对谁·说哪句·什么语气**"——即游戏精神"定义底层规则→故事概率涌现"。
- **初稿缺口（认账）**：`pick_bubble` 用**硬阈值**（`hunger>0.6→`固定整句）+ 固定 5% 概率门 = "把硬编码从 Rust 挪进 if"，**非**设计的富条件涌现选择。
- **本冲刺修正（强制）**：① 选句走 003 真 `FragmentCondition`（人格/情绪/信任/时段条件过滤）+ 概率加权 `RandomMatch`；② **说话驱动从涌现状态派生**——`speak_drive`（005）/`frequency_modifier`（006：外向高频·内向少言·坏心情短促）取代固定概率门；③ **问候不是"每次遭遇必触发"**——是否问候/热情或闷哼，从 extraversion×agreeableness×mood×relationship 涌现。"确定性"= 种子 `hash(pair,tick)` 从**加权候选集**抽，与"概率加权"不矛盾（可复现的涌现）。
- **仍不做（超 A1 探针·诚实边界）**：完整 003 引擎（多片段 `CompositeTemplate` 拼装+填槽+`JoinStrategy`）、概念→句子真生成（`概念与语言地基`）、LLM 层。MVP = 003 **真子集**（富条件+概率选择+按情绪微调语气，暂不做多片段填槽拼装）。

### 🔑 "问候本身"是否涌现——两层诚实划界（回应"是否总先说 hello"）

- **第一层（authored·非涌现·认账）**：**"见面存在一个寒暄开场槽"是结构规则**，设计明写（`005` §process_turn"① 首轮→Opening Phatic" + `006` PhaticLayer authored 子系统）。最底层"为何人类会寒暄"（礼貌从重复博弈涌现）**WoWorld 不建模**，直接把"寒暄 act"当**文化原语给定**——匹配真实社会语言学（寒暄是义务仪式，非 NPC 重新发现的优化）。这是诚实地板。
- **第二层（涌现·本冲刺强制）**：**是否/如何/是否被抢占 = 涌现**。**红线：禁止"遭遇→硬塞 Greeting"或建语音决策 silo**。改为——"见到人做什么"由**既有 `ActionWeight`→`ActionIntent`** 在 `ActionCategory`（`Socialize`/`Fight`/`Flee`/`Work`/…已平级）上**涌现竞争**（001 §二 三层原子 + §1.2 原则2）；问候只是 `Socialize`/reactive 反射赢时的**语音表达**，被 `Fight`/`Flee`/高紧急需求 `ActionIntent` 否决。于是"总先 hello"**不是绝对律**：怒者打斗、忙者无视、急者先办事。V4a 即体现（问候 vs 忙碌/饥饿由 ActionIntent 涌现决定）；"寒暄先于正事"完整排序待 V3b/V4b 有"正事"后浮现。**打斗/沉默/单方不回应 = 同一 ActionCategory 竞争落在别处，零壁障**。

## 🏛️ 架构分析（长线工程视角·10 维权衡）

> 用户裁决：**设计已覆盖的严格遵设计；未覆盖的从长线开发角度求最优。**

**设计已覆盖（严格遵守）**：
- 问候的语义模型 = **006 `PhaticLayer`**（Opening/Closing/Reactive 片段 + formality-from-trust + time_of_day）——直接照 006。
- 触发概念 = `Reactive::SomeoneEntered/SomeoneLeft { who }`（遭遇边沿），非轮询硬编码。
- 类型对齐 = **006 PhaticLayer 主导 + 005 `DialogueIntentType` 辅**（问候/告别属 Phatic；需求/情绪自言自语属 DialogueIntent 投影）。轮替 = 005 §二 TurnMode。

**设计未覆盖 → 长线求最优**：ECS 里"谁计算遭遇边沿事件"。设计 006 只规定语义（`SomeoneEntered` 触发），未规定 ECS 布线。权衡：

| 维度 | 结论 |
|------|------|
| **①解耦/ECS** | 遭遇检测（空间原语）与"头顶说什么"（气泡语义）应分离。`social_system` 已拥有邻居扫描——**不复制 O(n²)**（避免信息孤岛/双源真相）。抽共享 `neighbors_within` 原语，单点扫描。 |
| **③性能** | 村庄 3-5 NPC，O(n²) 可忽略；但**单点原语**为 Phase-3 SpatialGrid（social.rs 已注释此升级路径）留唯一改造点。 |
| **④可维护** | `SpeechAct`（语义）⊥ `BubbleType`（渲染色）正交，类型系统降耦；短语进 TOML，改文案不碰 Rust。 |
| **⑦涌现** | 问候按 formality/time/情绪过滤后**确定性选句**，非"见面必同一句"脚本——简单规则涌现自然寒暄。 |
| **⑧社区实践** | 沿用本仓 `Registry + include_str! + load_from_toml` 既有成熟模式，不造新加载轮子。遭遇边沿 = 帧间邻居集合差分 + **迟滞（施密特触发，enter@3m/leave@4m）**防边界抖动，标准做法。 |

**概念所有权核查（2026-07-12，已验代码）**：`SpatialEvent`（`types.rs:86`）是**广播式点事件**（combat_hit/footstep，供感官感知）；`RelationStorage` 是**长期关系值**（trust/affection）。NPC 邻近遭遇是**成对、对称、有状态的关系边**——二者皆不建模它 → **独立 `EncounterEvent` 资源不构成平行抽象**（不违反概念所有权铁律）。

**架构定位（barrier-free·对齐 001-行动涌现总纲）——问候是"社交行动的表达"，非"语音决策"**：
```
感知层   EncounterState/EntityIndex：NPC 感知附近 agent（通用·非语音专属·未来战斗/逃跑/交易共用）
   ↓ 喂
行动涌现  既有 ActionWeight → ActionIntent：Socialize/Fight/Flee/Work/... 按效用竞争（001 §二 三层原子）
   ↓ 产出 ActionIntent（已含 category+weight，movement_system 已消费）
表达层   speech_bubble = 众多表达通道之一：
   ActionIntent∈社交 且感知到人 → 问候气泡；==Fight → 攻击（未来）；自身需求/情绪 → 自言自语
```
- **红线（原则2 零硬编码禁止）**：**禁止建"遭遇→语音驱动优先级"平行决策 silo**。"见到人做什么"由**既有 ActionWeight 在 `ActionCategory` 上涌现**（Fight/Flee/Socialize 已平级），问候只是 `Socialize` 赢时的**语音表达**；气泡系统是**表达渲染器**，不做行为决策。
- **问候 = reactive 反射**（006 `Reactive::SomeoneEntered`）：occurrence 从 personality×mood×relationship 涌现，**被不相容 `ActionIntent`（Fight/Flee/紧急需求）否决**。→ 打斗/沉默/单方不回应**全自然涌现**（各 NPC 独立算 ActionIntent），无需脚本、无壁障。
- **共用 `neighbors_within()` 单原语**（Phase-3 SpatialGrid 唯一改造缝）；`social_system` 重构调用它（纯重构·测试守门）。**不建**完整 `SpatialEventBus`/`TextGenerator`/`SocialField`（超范围，A1 宁薄而真）。

## ✅ 新审查落定 + 长线接缝（第 4 轮·以最终态为参照）

**已落定（2 项·10 维分析后·比原选项更优）**：
- **D4. 问候社会效果 = 表达层零改写、效果归关系层**（G1）：**气泡/表达层不改 trust/familiarity/情绪**（barrier-free 铁律：表达不得越界改关系数据，否则语音变带副作用的决策，送礼/打斗效果无法复用）。社会效果**归 `social_system`/`RelationStorage` 独占**（已在邻近累积 familiarity + 情绪传染，问候正发生于邻近期、效果已计入，气泡再加=双账）。片段 TOML **携带声明式 `social_effect` 字段**（紧贴 006、数据驱动），**V4a 不从气泡施加**——留作未来关系层消费。未来"主动交谈>被动共处"由 **social_system 读 `EncounterEvent` 加 bonus**（关系层拥有），非气泡写。
- **D5. 感知 = proximity + 便宜朝向门（临时感知赝品）**（G8）：感知归 `感官系统`/`VisibilityQuery`（未建）→ V4a 用 **3m proximity（基）+ 朝向门（glam 点积·FOV≈±120°，读 `Rotation`/移动方向）** 作**显式临时赝品**，杀掉"背后/隔墙问候"穿帮（活人感）；只在遭遇 Enter 边沿算（非每帧·近零成本）。**完整 LOS 光线步进延后**（开阔村庄遮挡罕见）。`encounter` 设计成可换为"消费感官感知"——感官系统就位后**替换非扩展**，不成永久 silo。

**长线接缝（本冲刺不做，但架构须留缝、不锁死）**：
- **LOD 门控（G4）**：现有 npc 系统**无 LOD 门**（social.rs 注释"仅 L1 视野内"未实现）。规模化后 encounter/speech 应只跑 L1/可见 NPC。V4a 村庄小可全跑，但 `encounter_system` 须设计成**可按 LOD/可见性过滤输入**（未来接 `LodCoordinator`），不写死全量。
- **非语言联动（G5）**：设计 006 §三——每次发话应co-产 `nonverbal_expression`（点头/微笑/注视，010 模块）。V4a 只做气泡文字；问候的**手势/表情**留通道（气泡是"众多表达通道之一"，不锁死为唯一）。
- **记忆形成（G7·活人感命门）**：真人记得"今早遇见谁"。设计 NPC 有记忆（≤2000 条）。V4a 遭遇**暂不写 episodic 记忆**（关系累积已由 social_system 承担），但遭遇事件须**可被记忆系统消费**（`EncounterEvent` 是未来记忆/认知的输入源），不做成气泡私有死数据。
- **文化/语言标签（G9）**：`FragmentCondition` 已含 `CultureParam`；片段表须**预留 language/culture 键**（V4a 用单一默认语言），未来跨文化问候/听不懂不返工。
- **瞬时不存档（G11）**：`EncounterState`/气泡是瞬时物，**不进 V6 快照**（重载从位置重算）——与"UtteranceId 瞬时车道"一致，无存档负担。

- `SpeechAct` 不得成为 `DialogueIntentType` 的平行竞品——是其 MVP 子集投影，命名/语义对齐。
- **禁止"遭遇→硬塞 Greeting"，禁止建"语音驱动优先级"平行决策 silo**（违 001 §1.2 原则2 + 壁障）：问候是 `Socialize`/reactive 行动的**语音表达**，"见到人做什么"由既有 `ActionWeight`→`ActionIntent` 在 `ActionCategory`（Fight/Flee/Socialize/…）上涌现决定；气泡系统只表达不决策。Fight/Flee/紧急需求 `ActionIntent` **否决**问候——打斗/沉默/单方不回应皆经此自然涌现。
- 资源查询纪律延续 Sprint-067：不新建"需求满足点"平行 trait。
- **表达层不改关系（D4·barrier-free）**：`speech_bubble_system` **禁止改写 `RelationStorage`/`Emotion`**——社会效果归 `social_system` 独占；片段 `social_effect` 仅存不施加。
- **感知归感官系统（D5）**：朝向门是**临时赝品**，`encounter_system` 须可替换为消费 `VisibilityQuery`/感官感知，不把 FOV/LOS 逻辑长期焊死在遭遇层。
