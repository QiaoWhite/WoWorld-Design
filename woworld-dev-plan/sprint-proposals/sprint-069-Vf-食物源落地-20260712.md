## 📐 实现设计

### 架构：确定性函数式 Poisson disc

```
query_harvestable(pos, radius)
  │
  ├─ 0. 若 classifier is None → 返回 vec![]（无群系数据·海洋/未初始化）
  │
  ├─ 1. 计算查询圆覆盖的 TC 列表
  │     tc_at(world_pos) = (floor(x/32), floor(z/32))
  │     覆盖半径 = radius + r (跨TC边界·确保边缘点不遗漏)
  │
  ├─ 2. 对每个 TC：
  │     tc_center = ((tc_x+0.5)*32, (tc_z+0.5)*32)
  │     biome = classifier.classify(tc_center)
  │     //   ↑ None（海洋/气候空档）→ 跳过此 TC（无采集物）
  │     seed_tc = hash_2(world_seed ^ hash_str("vegetation"), tc_x, tc_z)
  │     //   ↑ 对齐设计 012 §八 chunk_vegetation_seed 模式
  │     points = poisson_disc_tc(seed_tc, tc_x, tc_z)
  │     //   ↑ 确定性——同 seed + 同 TC → 同点集
  │     for each point:
  │       biome → 加权抽取产物类别
  │       y = classifier.sample_height(point.x, point.z)
  │       // ↑ 从 WorldNoise 采样地形高度（1 行委托）
  │       instance_id = hash_4(world_seed, "plant_instance", tc_x, tc_z, point_idx)
  │       // ↑ 对齐设计 012 §八 plant_instance_id 公式
  │       yield_base = deterministic_range(point_seed, SALT_YIELD, biome_min, biome_max)
  │
  └─ 3. 过滤 points 到 radius 内 → 组装 HarvestableInfo → 返回
```

### Poisson disc 参数（Bridson 2007）

| 参数 | 值 | 理由 |
|------|-----|------|
| 采样域 | TC (32m×32m) + 4m margin = 40m×40m | 跨 TC 边界覆盖（margin=r） |
| 最小间距 `r` | 4.0m | **采集物是全集子集**——设计 012 灌木间距 1-2m（全植被）；Vf 只放可采集种（~25% 灌木是浆果/草药），有效间距 ≈ 2m / √0.25 ≈ 4m。全量 P2.25 将先放全集再筛选，Vf 直接以等效间距放子集 |
| 尝试上限 `k` | 30 | Bridson 标准值 |
| 背景网格 cell | `r / √2 ≈ 2.828m` | 加速邻域查询 |
| 每 TC 预期点数 | ~40-80 | 40²/4²=100 格·Poisson 饱和 ~50-70% |

### 群系→产物映射（硬编码·MVP）

```rust
fn harvestable_species_for_biome(biome: &str, rng: &mut impl FnMut() -> f32)
  -> (SpeciesId, ProductCategory, f32)  // (species, category, yield_base)
```

| 群系 | 产物分布（权重） | yield_base |
|------|-----------------|------------|
| **Forest** | Berry 40% / Mushroom 35% / Nut 25% | 1.5-3.0 |
| **Grassland** | Herb 50% / Berry 30% / Flower 20% | 0.8-2.0 |
| **Swamp** | Mushroom 60% / Herb 25% / Fiber 15% | 1.0-2.5 |
| **Desert** | Herb 60%（稀疏·仅 20% TC 有点）/ Fiber 40% | 0.3-1.0 |
| **Snowfield** | 无采集物 | — |

每个 Poisson 点独立加权抽取产物类别。

### `HarvestableInfo` 对齐

```rust
// 修改后（core/vegetation.rs）
pub struct HarvestableInfo {
    pub instance_id: u64,        // hash_4(world_seed, "plant_instance", tc_x, tc_z, point_idx)
    pub species_id: SpeciesId,   // 哨兵值 1(BerryBush)/2(Mushroom)/3(NutTree)/4(Herb)/5(Fiber)
    pub position: Vec3,          // Poisson 点世界坐标 + classifier.sample_height(x,z) 地形 y
    pub product_category: ProductCategory,
    pub yield_base: f32,         // 确定性：pseudo_random_f32_range(point_seed, SALT_YIELD, biome_min, biome_max)
    pub season_optimal: bool,    // MVP 恒 true（无季节系统接线）
    pub regen_state: RegenState, // ★ 新增字段·全部 Full（采集消耗是 V3a 的事）
}
```

> ⚠️ **代码 vs 设计分歧**：设计 010 §1.2 的 `HarvestableInfo` 含 `regen_state: RegenState` 且 `instance_id: PlantInstanceId`。Vf 补上 `regen_state`（已定义的枚举），`instance_id` 保持 `u64`（`PlantInstanceId` 待 `woworld_life` crate 统一引入）。

### 地形高度采样（新增 `BiomeClassifier` 方法）

```rust
// woworld_worldgen/src/biome.rs — BiomeClassifier 新增 1 行方法
pub fn sample_height(&self, x: f64, z: f64) -> f64 {
    self.noise.sample_height(x, z)  // 委托 WorldNoise
}
```

> `BiomeClassifier.noise` 是 `Arc<WorldNoise>`（已有），`WorldNoise::sample_height()` 是已有 pub 方法。此修改为 0 成本委托。

## ⚠️ 诚实边界（本冲刺不做）

- **不建 `woworld_life` / `woworld_vegetation` crate**（`PlantSpeciesRegistry` 零代码，Vf 硬编码）
- **不实现 VMC 双层架构**（直接 TC 粒度——VMC 是 P2.25 完整实现的事）
- **不做 `RegenState` 消耗/再生**（全部 `Full`——采集→消耗是 V3a 的事）
- **不做季节连线**（`season_optimal` 恒 true）
- **不做 L6.5 密度场**（那是 canopy tree 的事，采集物不进入密度场）
- **不做远距离懒展开**（`query_harvestable` 实时计算——NPC L3 每 5s 查一次，TC 内 ~50 点，Bridson O(n) 可忽略）
- **不做 splitmix 跨 crate 收敛**（复用同 crate `noise_gen::mix64`·零重复·跨 crate 收敛是独立轻活）
- **不做 `woworld_godot` 可视化**（食物点不可见——只有 NPC 知道在哪；可视化是 V5 旁观工具的事）
- **不修 Godot 桥 `set_vegetation_provider` 从未调用**（预存 gap——`terrain_chunk.rs:1959` 方法存在但无调用点，V2/V3a 集成时自然会接通；Vf 仅确保 `BiomeVegetation` 自身正确，测试覆盖）
