## 📐 实现设计

### 架构：修改 `goal_resolution_system`——Goal 创建时同步填充 target_pos

**为什么改 goal_resolution 而非新建 system**：Block A4 中所有 system 共享同一个 `CommandBuffer`——`goal_resolution` 通过 `cmd.insert_one(goal)` 排队插入的 Goal 在 `cmd.run_on()` 前对 `world.query()` 不可见。若分拆为独立 system，新 system 只能看到**上一帧**的旧 Goal（1 帧延迟，逻辑不一致）。合并后，Goal 创建时即带着坐标，无需事后修补。

```
Frame N:
  need_evaluation_system  → Desire(Eat, urgency=0.9)
  goal_resolution_system  → Goal(FindFood, target_pos: Some(berry_pos))  ← 同步填充
  action_weight_system    → ActionIntent（含 time_modifier）

Frame N+1:
  movement_system         → 读到 target_pos=Some → 走向浆果丛 → 到达 satisfy_goal()
```

### Pass 1 新增逻辑（Desire → Goal 时同步填充 target_pos）

```
for each (entity, Desire{kind, urgency}) in world:
  goal_type = goal_for_desire(kind)
  target_pos = resolve_target(entity, goal_type, world, vegetation, ocean, search_radius)
  cmd.insert_one(entity, Goal{goal_type, urgency, target_pos})
  cmd.remove_one::<Desire>(entity)
```

### `resolve_target` 核心逻辑

```
fn resolve_target(npc_pos: Vec3, goal_type: GoalType, ...):
  match goal_type:
    FindFood if vegetation.is_some():
      harvestables = vegetation.unwrap().query_harvestable(npc_xz, search_radius)
      if harvestables not empty → return Some(nearest.position)
      else → return None

    FindWater:
      water_pos = find_nearest_water_xz(npc_xz, ocean, search_radius)
      // 返回最近的 XZ 有水点的 (x, 0, z)——movement 到达判定忽略 Y，
      // 地形跟随自动将 NPC 放在水面边缘。深水/陡岸由 movement 坡度门阻挡（诚实涌现）
      return water_pos

    FindRest / FindSafePlace / FindSocialContact / BalanceElements / ExpressLibido:
      return None  // Phase 3+

    Idle:
      return None
```

### 最近水源搜索——XZ 平面 `water_depth > 0` 最近点

```
find_nearest_water_xz(origin_xz: Vec2, ocean: &dyn OceanProvider, radius: f32) -> Option<Vec3>
  // 极坐标网格搜索: 8 angles × r/8 step × expanding rings
  // 返回最近一个 water_depth_at(sample_worldpos) > 0.0 的 XZ 坐标
  // Y 固定为 0.0——movement 的 XZ 到达判定 + 地形跟随自然停在岸边
  best: Option<(dist, pos)>
  for each (x, z) in polar grid:
    let wp = WorldPos { x: origin.x + x, y: 0.0, z: origin.z + z };
    if ocean.water_depth_at(wp) > 0.0:
      dist = (x² + z²).sqrt()
      if dist < best.dist → best = Some((dist, Vec3(wp.x as f32, 0.0, wp.z as f32)))
  return best.pos
```

> **选点策略说明**: `water_depth_at` 接受 `WorldPos`（忽略 y 字段）——内部查询地形高度与海平面 (y=0) 比较。选中点的 Y 恒为 0.0——movement_system 的到达判定比较 XZ 距离并忽略 Y（`movement.rs:95-96`），地形跟随自动将 NPC 放在地面高度。平缓海岸线处 NPC 自然站在水边；陡岸/悬崖由 `movement_system` 45° 坡度门阻挡（"走不到"=诚实涌现）。
>
> **A1 纪律说明**: 8-angle expanding ring 扫描比简单 4 方向检查更可靠（海边方向可能不在正方位上），且 `water_depth_at` 是一次内部查表（O(1)），总开销可忽略。
>
> **诚实边界**: 不检查路径可达性（体素 A* 是 Phase 3 的事）；不区分淡水/咸水（饮水类型是 V3a 的事）。
