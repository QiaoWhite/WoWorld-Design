## 📐 实现设计

### 架构：三件事——EventChannel 暴露成交 + priority 显式仲裁 + TradeShout 片段

1. **成交事件走 `EventChannel<TradeRecord>`**（设计 `06-跨System通信` §通道3 规定）
2. **`ActiveBubble` 加 `priority: u8` 字段**（设计 `UI与UX系统/005` 规定）+ 三级优先级常量
3. **新增 `SpeechAct::TradeShout`** + `trade_bubble_system` 薄函数

### 改动 1: `ActiveBubble` 加 `priority` 字段 + 三级优先级常量

```rust
// speech_bubble_state.rs — ActiveBubble +1 字段
pub struct ActiveBubble {
    pub text: String,
    pub bubble_type: BubbleType,
    pub expiry_tick: u64,
    pub priority: u8,  // ★ 优先级——高可抢占低（设计 UI与UX系统/005）
}

// speech_bubble.rs — 三级优先级常量（对齐设计 UI与UX系统/005 §Bark 气泡）
pub const PRIORITY_SOCIAL: u8 = 3;     // 问候/告别——社交定向·可抢占一切
pub const PRIORITY_TRADE: u8 = 2;      // 交易吆喝——成交信号·可抢占自言自语
pub const PRIORITY_SELF_TALK: u8 = 1;  // 自言自语——需求嘟囔/情绪宣泄·最低
```

**仲裁规则**: 新气泡写入条件 = slot 为空 **或** `new_priority >= existing_priority`。即：
- greeting (3) 可抢占 trade (2) 和 self-talk (1)
- trade (2) 可抢占 self-talk (1)，不可抢占 greeting (3)
- self-talk (1) 永不抢占任何已有气泡

> ⚠️ 这改变了 V4a `speech_bubble_system` 的行为：Pass 1 从"无条件覆盖"变为"priority 比较后覆盖"（实际等效——greeting priority=3 > self-talk priority=1，永远覆盖）；Pass 2 从"有气泡就跳过"变为"priority 比较后跳过"（实际等效——self-talk priority=1 < 任何已有气泡，永远跳过）。行为等价，但机制从隐式变为显式。


**时序验证**:
```
Frame N:
  视觉相位: trade_bubble_system drain Frame N-1 的 trades → 冒泡
  Block A0:  trade_events.begin_frame() → 清 read_buf（Frame N-1 trades 已消费）
  Block A5:  market_matching → send_all(trades) → mid_phase_flush → read_buf 有 Frame N trades

  ...
```
✅ 跨帧消费正确——`begin_frame` 在视觉相位**之后**执行，不会清除未消费的 trades。

### 改动 3: `speech_bubble_system` — Pass 2 改为 priority 仲裁

V4a 原有逻辑（`systems/npc/speech_bubble.rs` line 278）:
```rust
// 旧: 有气泡就跳过（隐式——Pass 1 之后任何活跃气泡阻塞自言自语）
if slot.active.is_some() { continue; }
```

改为:
```rust
// 新: priority 比较——self-talk(1) < trade(2)/greeting(3)，永不抢占高优先级气泡
// 需新增 import: use woworld_core::speech_bubble::PRIORITY_SELF_TALK;
let blocked = slot.active.as_ref()
    .map(|a| a.priority > PRIORITY_SELF_TALK)
    .unwrap_or(false);
if blocked { continue; }
```

> ⚠️ **行为等价性验证**: 旧代码 `is_some()` 等价于新代码 `priority > 1`，因为 V4a 只有 greeting(3) 和 self-talk(1) 两种气泡。greeting(3) > 1 → blocked，self-talk(1) ≯ 1 → not blocked（但 cooldown=600 > duration=180，旧 self-talk 在 cooldown 结束时已过期，blocked 与否均无影响）。V4b 加入 trade(2) 后: 2 > 1 → blocked ✅。

**Pass 1（问候/告别·line 248-255）无需修改**——greeting 设置 `slot.active` 时不做 priority 比较（无条件覆盖）。greeting priority=3 ≥ 任何现有 priority，结果总是正确。

**`BUBBLE_DURATION_TICKS` / `BUBBLE_COOLDOWN_TICKS` 可见性**：当前为模块私有（`speech_bubble.rs` line 41-42）。`trade_bubble_system` 在 `systems/economy/mod.rs` 需引用它们 → 改为 `pub(crate) const`（同一 crate 内可见·2 字符改动）。

### 改动 5: 新增 `trade_bubble_system`

```rust
/// 交易气泡系统——成交事件→吆喝气泡（薄出口，复用 SpeechBubbleState）。
///
/// - 消费 EventChannel<TradeRecord>::drain()（双缓冲·跨帧）
/// - 买卖双方各独立选句（per-entity seed → 非对称·涌现）
/// - Priority=2——不覆盖 greeting(3)，可覆盖 self-talk(1)
/// - 受 cooldown 约束（不连刷）
/// - 玩家实体不冒泡
#[allow(clippy::too_many_arguments)]
pub fn trade_bubble_system(
    world: &hecs::World,
    trade_events: &mut EventChannel<TradeRecord>,
    state: &mut SpeechBubbleState,
    fragments: &SpeechFragmentRegistry,
    current_tick: u64,
    day_progress: f32,
    player_entity: Option<hecs::Entity>,
) {
    let trades = trade_events.drain();
    if trades.is_empty() {
        return;
    }

    let time_of_day = TimeOfDay::from_progress(day_progress as f64);

    for trade in &trades {
        for &trader_id in &[trade.buyer_id, trade.seller_id] {
            let entity = hecs::Entity::from_bits(trader_id.0);

            // 玩家实体不冒泡（一致性）
            if player_entity == Some(entity) {
                continue;
            }

            // 实体存在性检查（可能已 despawn）
            if world.entity(entity).is_err() {
                continue;
            }

            let slot = state.slots.entry(entity).or_default();

            // Priority 仲裁: PRIORITY_TRADE=2 ——不覆盖 greeting(3)，可覆盖 self-talk(1)
            let blocked = slot.active.as_ref()
                .map(|a| a.priority > PRIORITY_TRADE)
                .unwrap_or(false);
            if blocked {
                continue;
            }

            if current_tick < slot.next_allowed_tick {
                continue;
            }

            // 从片段库选句（per-entity seed → 买卖双方可能不同话·涌现非对称）
            let pleasure = world.get::<&Emotion>(entity).map(|e| e.pleasure).unwrap_or(0.0);
            let extraversion = world.get::<&BigFive>(entity).map(|b| b.extraversion).unwrap_or(0.5);
            let ctx = SpeechContext {
                time_of_day,
                trust: 0.0,
                pleasure,
                extraversion,
                topic: None,
            };
            let seed = trader_id.0.wrapping_mul(31).wrapping_add(current_tick);
            if let Some((text, bt)) = fragments.select(SpeechAct::TradeShout, &ctx, seed) {
                slot.active = Some(ActiveBubble {
                    text,
                    bubble_type: bt,
                    expiry_tick: current_tick + BUBBLE_DURATION_TICKS,
                    priority: PRIORITY_TRADE,
                });
                slot.next_allowed_tick = current_tick + BUBBLE_COOLDOWN_TICKS;
            }
        }
    }
}
```

## 🚫 诚实边界

- **不碰撮合引擎**：`execute_trade`·`match_orders`·订单簿——零改动
- **不建新气泡框架**：复用 `SpeechBubbleState` + `SpeechFragmentRegistry` + `ActiveBubble`
- **成交事件走设计规定的 EventChannel**：`EventChannel<TradeRecord>`（设计 `06-跨System通信` §通道3 `EventBus { trade_events }` 的最小实现）
- **priority 按设计规定实现**：`SpeechBubbleEvent { priority }` 三级制（social=3 > trade=2 > self-talk=1），匹配 `UI与UX系统/005` 数据合同
- **不实现交易对话**：吆喝≠讨价还价对话——后者属 `语言表达/006` 的 DialogueIntent 体系，Phase 3+
- **不做 buyer/seller 差异化片段**：双方从同一候选池选句，涌现非对称（per-entity seed）
- **不接入情绪/人格全部门控**：`SpeechContext` 仅填 `pleasure`+`extraversion`
- **不实现"最大5气泡同时"限制**：`UI与UX系统/002` 规定上限 5——村庄 ~3-5 NPC 不触发此边界
- **不迁移 `market_matching_system` 调用方**（tests 调 `market_matching_system(&mut reg, tick)`）：仅改返回类型为 `-> Vec<TradeRecord>`，tests 忽略返回值——零 breakage
