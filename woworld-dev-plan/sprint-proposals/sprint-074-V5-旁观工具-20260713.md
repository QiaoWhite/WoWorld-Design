## 🎯 目标（3 个）

### 目标 1: 模拟速度控制（F3 控制台命令）

- **验收标准**:
  1. `WorldClock` 新增 `time_scale: f32` 字段（默认 1.0，范围 [0.0, 100.0]）+ `set_time_scale(f32)` 方法（自动 clamp）✅ 已完成
  2. `WorldClock.advance()` 内部乘以 `time_scale`——时钟推进自动跟随速度 ✅ 已完成
  3. `WorldDriver.process()` 计算 `sim_delta = delta * time_scale`，ECS System（年龄/需求衰减/情绪漂移/社交/移动/天气）统一使用 `sim_delta`——完整模拟加速
  4. F3 控制台新增 `speed` 命令：`speed 1`/`speed 10`/`speed 60`——设置模拟速度。不带参数 = 显示当前速度。`speed 0` = 暂停
  5. 控制台输出确认信息（如 `[speed] 10x`）
  6. `cargo test --workspace` 全绿——time_scale 测试 ✅ 已完成（+7 tests）
- **涉及模块**: `woworld_core::time`（✅ 完成）+ `woworld_godot::debug_console`（speed 命令）+ `woworld_godot::terrain_chunk`（sim_delta 接线）
- **涉及代码**: `time.rs`（✅ 完成·+~30 行·+7 tests）, `debug_console.rs`（+~25 行 speed 命令）, `terrain_chunk.rs`（+~5 行 sim_delta 接线）
- **设计决策**: 速度控制归入调试控制台（而非全局热键）——调试功能集中管理，避免占用未来快捷栏。玩家正式版的速度控制（如睡觉/等待）走独立 gameplay 路径
- **A1 纪律**: time_scale 是纯乘法器——不碰系统逻辑、不伪造时间

### 目标 2: 点击 NPC 检视面板

- **验收标准**:
  1. **扩展 `entity_debug_system`**：新增以下 section：
     - **Action**（`ActionIntent` Component）——category + weight。回答"NPC 正在做什么"
     - **Wallet**（`Wallet` Component）——铜/银/金 + 总铜币值
     - **Economy**（`EconomicCognition` Component）——金融素养/市场理解/时间偏好/搜索广度
     - **Growth**（`GrowthNeeds` Component）——尊重赤字/胜任挫折/慢性天数
     - **Inventory**（`InventoryRegistry` 查询）——持有物品数 + 装备槽占用数
     - 函数签名扩展为 `entity_debug_system(world, entity, inventory_registry: Option<&InventoryRegistry>)` —— console `info` 命令传 `None`，检视面板传 `Some(&self.inventory_registry)`
  2. **WorldDriver 统一处理点击选中**：复用现有 `raycast_select` 逻辑（AABB slab），在非控制台模式下左键点击也触发选中（现有 console 模式的点击逻辑抽为独立方法，两个调用点共享）
  3. **检视面板**：Godot Control CanvasLayer（layer 100，低于 debug_console 的 128），屏幕右上角停靠。读取 `#[func] get_inspect_data(entity_bits: u64) -> Dictionary` 渲染字段。面板只读——不写模拟数据
  4. 点击空白 → 面板清空。ESC → 面板清空（不关 mouse capture）
  5. `cargo test --workspace` 全绿，含新增 entity_debug_system 扩展测试（Wallet/Economy/Growth/ActionIntent section 存在性 + Inventory 查询）
- **涉及模块**: `woworld_ecs::systems::entity_visual`（扩展快照 + 新签名）+ `woworld_godot::terrain_chunk`（#[func] + 点击逻辑重构）+ 新建 Godot Control 场景
- **涉及代码**: `entity_visual.rs`（+~60 行新 section）+ `terrain_chunk.rs`（+~40 行 #[func] + ~20 行点击重构）+ 新建 `godot/scenes/inspect_panel.tscn` + `godot/scripts/inspect_panel.gd`（~80 行）
- **架构原则**:
  - `entity_debug_system` 是**唯一数据源**——console `info` 和检视面板消费同一 `EntityDebugSnapshot`
  - Inventory 数据通过可选 `&InventoryRegistry` 参数获取（Resource 非 Component）
  - 面板不缓存 NPC 内心状态（设计 `005-跨模块接口` §UI 数据合同）
  - Godot Control 纯表现——不包含游戏逻辑（宪法 §2）

## 附录 A：十大维度权衡

### 1. 架构与解耦
- `time_scale` 在 `woworld_core::WorldClock`（core 是时间权威），`sim_delta` 计算在 WorldDriver（Godot 侧）——core 不知道"模拟速度"概念，只知道 "delta 可以不同"
- `entity_debug_system` 签名扩展为可选 `&InventoryRegistry`——保持纯函数，不引入 WorldDriver 依赖。Console 传 None 不受影响
- Godot Control 纯表现——Rust 侧通过 Dictionary 提供数据，不含布局逻辑

### 2. 模块间协作
- 检视面板打通 NPC 内心状态 ↔ 玩家视觉：Vitals/Emotion/BigFive/Needs/Goal/ActionIntent 全部可读
- Wallet/Inventory/EconomicCognition 打通经济层 ↔ 旁观层
- 复用 V4a/V4b 气泡框架的 `ActiveBubble` 展示模式（头顶文字）
- 复用 Vf `VegetationProvider::query_harvestable` 做场景验证

### 3. 运行性能
- 检视面板：冷路径（点击触发），单次 Dictionary 序列化 ~20 字段 < 50μs
- time_scale：纯 f32 乘法，热路径零开销
- debug 标记球体：一次性 spawn，非每帧

### 4. 可维护性与代码品质
- 不硬编码速度档位——`[1.0, 10.0, 60.0]` 常量数组
- entity_debug_system 新增 section 用相同 `DebugSection` 结构——与现有代码风格一致
- InventoryRegistry 通过 `Option<&>` 传入——类型系统保证调用者显式处理"无 registry"情况

### 5. 数学工具
- 本冲刺无重数学——time_scale 是简单乘法

### 6. 模块特色与游玩感受
- 检视面板是"活着的村庄"的透明窗口——证明 NPC 有真实内心生活
- 速度 60x 下一天 ≈ 24 秒——旁观者可在 30 秒内看完完整日循环（日出→活动→社交→日落→休息倾向）

### 7. 涌现式交互
- 面板不创造涌现——它**揭示**涌现。玩家看到 NPC 的 Needs/Goal/ActionIntent 随时间和环境变化
- 速度加速让因果链变得肉眼可见：太阳落山→疲劳上升→FindRest goal→移动

### 8. 社区智慧
- Godot Control + CanvasLayer 是 Godot 社区 UI 标准方案
- Dictionary 跨 GDExtension 边界是 godot-rust 0.5.x 推荐模式
- 模拟速度控制参考 RimWorld/DF 的 `sim_speed` 模式

### 9. Godot 4.7 特点
- `CanvasLayer.layer` 控制 UI 层级：检视面板 100 < debug_console 128
- `Control.mouse_filter = IGNORE` 让检视面板不拦截 3D 视口点击
- Godot 4.7 的 `InputEventMouseButton` 提供 `.pressed` 和 `.double_click`——用于区分 click vs drag

### 10. 玩家体验与可达性
- `-`/`=`/`Backspace` 三键在 WASD 左手区附近，无需移手
- 检视面板右上角——不遮挡游戏视野中心（准心区）+ 不挡 HUD 左侧状态条
- 点击 NPC → 面板弹出，点击空白 → 关闭——零学习成本
