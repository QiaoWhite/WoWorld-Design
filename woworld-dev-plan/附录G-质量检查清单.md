## 🔧 一、机械门检查清单

> **触发**: 每个冲刺结束。**不可跳过**。必须粘贴真实命令输出到 Handoff。

```
□ cargo build --workspace 通过（零错误）
□ cargo test --workspace 全部绿（粘贴测试数量）
□ cargo clippy --workspace -- -D warnings 零警告
□ cargo fmt --check 通过
```

**附加 Rust 病检查**（每 5 个冲刺或阶段结束时执行一次）:
```
□ 无 HashMap 默认哈希（必须使用 FxHashMap 或 seeded hasher）
□ 无 SystemTime::now()（确定性——用 WorldTime）
□ 无 rayon 非确定归约（parallel fold 需要确定性的 combine 函数）
□ 所有 pub enum 实现 strum::EnumIter + EnumString
□ 所有 pub 类型有 Debug + Clone 派生
□ 无 unwrap() 在非 Fatal 路径（用 expect() 或 proper error handling）
```

## 🏥 五、分诊协议速查

> 收到用户问题反馈时，先分诊再行动。

| 类别 | 判定标准 | 动作 |
|------|---------|------|
| 🔴 回归 | 之前可工作、现在坏了。测试失败/编译错误/crash。 | **立刻修** |
| 🟡 已知偏离 | 已在 附录E 或冲刺提案中追踪。 | 引用追踪 ID，不重复调查 |
| 🟢 参数调优 | 数值/视觉效果不满意，不是正确性错误。 | 附成本估算，用户决定 |
| 🔵 结构性依赖 | 根因依赖未来冲刺的重构/新模块。 | 标注 Sprint-N，不现在修 |

**回复格式**:
```
> **分诊**: [🔴🟡🟢🔵] — [一句话判定]
> **如果现在修**: 影响 ~N 文件, 预计 ~X-Y 分钟, 阻塞风险: [无 / 依赖 Sprint-N]
> **建议**: [现在修 / 记入已知问题 / 等待 Sprint-N 自然解决]
```

## 🔄 六、偏差升级速查

```
同一目标连续 2 冲刺未完成 → 🟡 偏差：AI 自主调整时间线
同一目标连续 3 冲刺未完成 → 🔴 重评估：AI 分析 + 用户裁决
                             选项: A) 继续投入 B) 降级验收标准 C) 拆分为更小粒度
同一里程碑 2 次重评估 → 🔴🔴 重规划：用户主导重新设计阶段
发现架构级不可行 → 立即停止，不等 3 次
```

## 🧪 七、研究先行速查

| 级别 | 触发 | 研究深度 |
|------|------|---------|
| 🟢 已知模式 | hashmap、builder pattern 等常规操作 | 跳过 |
| 🟡 新算法/新技术 | 未见过的算法、不熟悉的 crate | 搜索 2-3 个现有方案，保留 URL |
| 🟠 性能关键路径 | 地形采样、寻路、网格生成、渲染管线 | 行业标准 + 现有 Rust 实现 |
| 🔴 架构级决策 | 自研 vs 第三方、数据流设计 | 深度调研 + ADR 记录 |

**研究四步法**: 识别问题类别 → WebSearch 行业标准 → 2-3 方案决策矩阵 → 记录决策+引用 URL
