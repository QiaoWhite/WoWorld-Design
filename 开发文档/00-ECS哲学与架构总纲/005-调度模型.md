## 两阶段调度

```
每帧 process():

  Phase 0 — 输入与基础设施 (必须最先，< 0.2ms)
    ├─ InputGatherSystem      读取 Godot Input → Resource<InputState>
    ├─ LodCoordinatorSystem   距离 → LodLevel Component
    └─ ClockAdvanceSystem     推进 WorldClock

  Phase 1 — 游戏逻辑 (所有 System 平等，< 6ms)
    ├─ 全部游戏逻辑 System ——无顺序依赖，可 rayon 并行
    ├─ CleanupSystem ——标记删除执行（必须最后）
    └─ CommandBuffer flush ——所有拆装操作统一应用

  Phase 2 — Godot 同步 (必须最后，< 0.2ms)
    ├─ RenderSyncSystem       可见性 → Godot 渲染
    ├─ AudioDispatchSystem    音频事件 → Godot AudioServer
    └─ AnimationSyncSystem    动画状态 → Godot AnimationTree
```

## Phase 1 内 System 的注册

```rust
struct SystemMeta {
    name: &'static str,
    func: SystemFn,  // fn(&hecs::World, &mut CommandBuffer, &Resources)
    reads: &'static [TypeId],
    writes: &'static [TypeId],
}

// WorldDriver 初始化
fn register_systems(phase1: &mut Vec<SystemMeta>) {
    phase1.push(SystemMeta::new("hunger_decay", hunger_decay_system, 
        &[TypeId::of::<Needs>()], &[TypeId::of::<Needs>()]));
    phase1.push(SystemMeta::new("need_eval", need_evaluation_system,
        &[TypeId::of::<Needs>()], &[]));  // 只通过 CommandBuffer 写
    phase1.push(SystemMeta::new("combat_damage", combat_damage_system,
        &[TypeId::of::<IncomingDamage>()], &[TypeId::of::<Vitals>()]));
}
```

## 并行安全

注册时验证：

```rust
fn validate_phase1(systems: &[SystemMeta]) -> Result<(), Vec<Conflict>> {
    for (i, a) in systems.iter().enumerate() {
        for b in &systems[i+1..] {
            for w_a in a.writes {
                if b.writes.contains(w_a) {
                    // 两个 System 写同一个 Component → 不能并行
                    return Err(Conflict { a: a.name, b: b.name, component: w_a });
                }
            }
        }
    }
    Ok(())
}
```

通过验证的 System → rayon 并行执行。
未通过验证的 → 标记为顺序执行（少数例外）。

## 一帧延迟

System A 写入的 Component，System B 在**下一帧**看到。

```
帧 N:
  NeedEvaluationSystem 执行 → cmd.insert(Desire) → 记录到 CommandBuffer
  GoalResolutionSystem 执行 → 查询 Desire → 查不到（还没 flush）
  CommandBuffer flush → Desire 正式装上

帧 N+1:
  GoalResolutionSystem 执行 → 查询 Desire → 查到了！
  → cmd.remove(Desire), cmd.insert(Goal{kind: Eat})
```

**这是特性，不是 bug**：
- 消除了 System 之间的顺序耦合
- 每个 System 独立并行
- 跨帧延迟在游戏玩法中不可感知
