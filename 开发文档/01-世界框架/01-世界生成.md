## 不进 ECS（内核）

世界生成的**内核数据**（地形高度场、群系参数、海洋深度）不进入 ECS Archetype 存储。

**原因**：
- 这些都是**世界级单例**，不是"有生命周期和状态变化"的实体
- 放在 ECS 的 Archetype 表中零收益
- 业界一致：Veloren（MMO 体素 RPG）将 TerrainGrid 作为 ECS Resource，非 Entity
- 15 阶段管线（P0-P13）在世界初始化时执行，不是每帧

**但**：★ CHG-065 新增**地形修改编排层**——修改请求验证、dirty 传播、重 mesh 调度、NPC 感知通知走 ECS Component + System + Event 编排。内核不动，编排入 ECS。

---

## 作为 Resource 提供给 ECS

世界生成的输出作为 Resource 被 ECS System 消费：

| Resource | 提供者 | 消费者 |
|----------|--------|--------|
| `TerrainQuery` (trait) | `HeightfieldTerrain` | `MovementSystem`, `CombatSystem`, `PerceptionSystem` |
| `OceanProvider` (trait) | `HeightfieldOcean` | `MovementSystem`(游泳), `CombatSystem`(水面战斗) |
| `VegetationProvider` (trait) | 植被模块 | `GatheringSystem`, `PerceptionSystem` |
| `DensityProvider` (trait) | `DensityStack` | `BuildingSystem`(地基检测) |
| `EditTerrainSnapshot` (CoW) | `woworld_core::edit_terrain` | `HeightfieldTerrain`(查询), VoxelChunk(提取) |
| `EditDensityLayer` (DensityProvider) | `woworld_core::edit_terrain` | `DensityStack`(插入), Transvoxel(提取) |

---

## 修改编排层（CHG-065 新增）

运行时地形修改的**编排层**走 ECS：

| 层 | ECS 角色 | 说明 |
|----|---------|------|
| `ModificationBatch` | Resource | 帧内批量积累修改请求（Veloren BlockChange 模式） |
| `DirtyChunkQueue` | Resource | Chunk 脏标记 + 两级索引（消费后触发重 mesh） |
| `EditDensityLayer` | DensityProvider 桥接 | 将 CoW 快照插入 DensityStack——Transvoxel 提取自动包含修改 |
| `modification_collector_system` | ECS System（待实现） | 验证请求 + flush 到 EditTerrainBuilder + 标记 DirtyChunkQueue |
| `chunk_invalidation_system` | ECS System（待实现） | Transvoxel 边界邻居 dirty 传播 |
| `SpatialEventBus` | Resource（待实现） | 推送 `TerrainModified` 事件——NPC 感知系统消费 |

### 2D/3D 分裂修复

原 `HeightfieldTerrain` 的 `height_at()` 走裸噪声，`density_at()` 走 DensityStack——修改后两者不一致。CHG-065 修复：所有 9 个 `TerrainQuery` 方法均通过 `actual_height_at()` 或手动密度叠加感知 EditTerrain。`HeightfieldOcean` 同样修复。
