## 这个模块关心什么"细节"

生命系统管理所有活物的**生命体征**（血量/体力/法力/精神）、**死亡**（30种死因6大类）、**生命周期**（年龄/生育/衰老）。

## Component 定义

### Vitals

> body_temp/oxygen — Sprint 中合理新增，回溯同步至设计文档。
> max_stamina/spirit — Sprint 中遗漏，从设计文档恢复。hunger/thirst 已移除——归属 Needs Component。

```rust
struct Vitals {
    hp: f32,            // f32: 毒/燃烧/流血是连续伤害
    max_hp: f32,
    stamina: f32,
    max_stamina: f32,   // 物种/职业上限可配置
    spirit: f32,        // 0-1, 精神完整度。0=精神崩溃 → DeathCategory::SpiritExhaustion
    body_temp: f32,     // 摄氏度, 37.0 正常
    oxygen: f32,        // 0=窒息, 100=正常
}
impl Component for Vitals {}
```

| 字段 | 读写 System |
|------|-----------|
| hp | CombatDamageSystem(W), RegenSystem(W), DeathWatchSystem(R) |
| stamina | MovementSystem(R+W), CombatActionSystem(R), RegenSystem(W) |
| max_stamina | RegenSystem(R) — 恢复上限 |
| spirit | MagicSystem(W), TraumaSystem(W), DeathWatchSystem(R) |
| body_temp | WeatherSystem(W), DeathWatchSystem(R→corpse_temperature) |
| oxygen | EnvironmentSystem(W·水下), DeathWatchSystem(R) |

### DeathCause

#[derive(Clone, Copy)]
struct DeathCause {

## System 定义

### DeathWatchSystem

- **触发**: Entity 有 `Vitals` 且 `hp == 0`
- **读**: `&Vitals`
- **CommandBuffer**: 
  - `remove: Vitals, CombatState, Goal, Desire, …`（所有"活着"的组件）
  - `insert: Corpse, PendingLoot, DeathCause`
- **与其他 System 的关系**: 无

### LootRollSystem

- **触发**: Entity 有 `Corpse` + `PendingLoot`
- **读**: `&Corpse`, `&PendingLoot`, `&EntityKind`
- **读 Resource**: `&LootTableRegistry`
- **CommandBuffer**: `remove: PendingLoot`, `insert: LootResult`
- **与其他 System 的关系**: 无

### ItemSpawnSystem

- **触发**: Entity 有 `Corpse` + `LootResult` + `&Position`
- **读**: `&LootResult`, `&Position`
- **CommandBuffer**: 
  - `spawn: 为每个物品创建新 Entity`
  - `remove: LootResult`, `insert: CorpseLooted`
- **与其他 System 的关系**: 无

### CorpseDecaySystem

- **触发**: Entity 有 `Corpse` + `CorpseLooted`，且 `time_since_death > 5分钟`
- **读**: `&Corpse`
- **CommandBuffer**: `remove: Corpse`, `insert: DecayingRemains`

### CleanupSystem（Phase 1 最后执行）

- **触发**: Entity 有 `PendingDespawn` 或 `DecayingRemains{decay_progress >= 1.0}`
- **CommandBuffer**: `despawn`

## 完整生命周期链条

```
帧 N:   [Health{hp:0}] → DeathWatchSystem → 拆Health, 装Corpse+PendingLoot
帧 N+1: [Corpse, PendingLoot] → LootRollSystem → 拆PendingLoot, 装LootResult
帧 N+2: [Corpse, LootResult] → ItemSpawnSystem → 生成物品Entity, 拆LootResult
帧 N+~18000: [Corpse, CorpseLooted] → CorpseDecaySystem → 拆Corpse, 装DecayingRemains
帧 N+~36000: [DecayingRemains{1.0}] → CleanupSystem → despawn
```

**每个 System 独立工作。链条从 Component 变化中涌现。**
