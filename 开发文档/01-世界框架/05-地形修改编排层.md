## 定位

地形修改编排层是 WoWorld 运行时地形修改系统的 ECS 架构实现。它遵循 **"内核不转 ECS，编排层入 ECS"** 原则：

- **内核**（高度场、密度场、海洋）保持为 ECS Resource——不走 Archetype 存储
- **编排层**（修改请求→验证→dirty 传播→重 mesh 调度→NPC 感知）走 ECS Component + System + Event

---

## 核心类型

| 类型 | 位置 | ECS 角色 | 说明 |
|------|------|---------|------|
| `EditDensitySnapshot` | `woworld_core::edit_terrain` | Resource（CoW 读者） | 3D 稀疏密度修改——`Arc<HashMap<IVec3, (f32, u8)>>`，零锁读取 |
| `EditDensityBuilder` | `woworld_core::edit_terrain` | Resource 构建器 | 由 ECS System 构建，`freeze()` 后原子交换给读者 |
| `EditHeightfieldSnapshot` | `woworld_core::edit_terrain` | Resource（CoW 读者） | 2D 表面高度投影——`Arc<HashMap<(i32,i32), f32>>`，O(1) 查询 |
| `EditHeightfieldBuilder` | `woworld_core::edit_terrain` | Resource 构建器 | 与 EditDensityBuilder 同步构建，保持 2D/3D 一致性 |
| `EditTerrainSnapshot` | `woworld_core::edit_terrain` | Resource | 组合密度 + 高度的完整快照（DirtyChunkQueue 为独立 Resource） |
| `EditTerrainBuilder` | `woworld_core::edit_terrain` | Resource 构建器 | 帧内修改累积 + dirty_chunks 追踪，freeze() → EditTerrainSnapshot |
| `ModificationKind` | `woworld_core::edit_terrain` | 枚举 | Dig/Fill/Smooth/Paint/Flatten |
| `ModificationRequest` | `woworld_core::edit_terrain` | 值类型 | 单次修改描述符（kind + center + radius + material + intensity + requester） |
| `ModificationBatch` | `woworld_core::edit_terrain` | Resource | 帧内批量积累修改请求（Veloren BlockChange 模式） |
| `DirtyChunkQueue` | `woworld_core::edit_terrain` | Resource | Chunk 脏标记 + 两级索引（ChunkCoord → HashSet<局部体素>） |
| `EditDensityLayer` | `woworld_core::edit_terrain` | DensityProvider 桥接 | 实现 `DensityProvider` trait，插入 VoxelChunk 临时 DensityStack |

## Copy-on-Write 设计

**问题**：一次 Transvoxel 提取做 ~36,000 次密度采样。若每次采样获取 `RwLock` 读锁，性能不可接受。

**方案**：CoW——读者持 `Arc<HashMap>` 快照（零锁），写者构建新 HashMap 后原子交换。

```
ECS System（写者）                    Terrain Query（读者）
      │                                      │
      ├─ EditTerrainBuilder                  │
      │    ├─ density.set() × N                │
      │    ├─ heightfield.set_height() × N            │
      │    └─ freeze() → EditTerrainSnapshot  │
      │         │                              │
      │    atomic swap ────────────────────→ EditTerrainSnapshot (CoW)
      │                                      │
      │                                      ├─ height_at(x,z) → heightfield.height_at()
      │                                      ├─ density_at(pos) → stack + edit.delta
      │                                      └─ material_at(pos) → edit.material ?? stack.material_at()
```

## 2D/3D 分裂修复

原 `HeightfieldTerrain` 存在 2D/3D 分裂：`height_at()` 走裸噪声（`noise.sample_height`），`density_at()` 走 `DensityStack`。修改后两者不一致——NPC 站在被挖过的地面上会浮空或沉入地下。

**修复**：所有 9 个 `TerrainQuery` 方法均可感知 EditTerrain：

| 方法 | 修改路径 | 说明 |
|------|---------|------|
| `height_at` | `actual_height_at` → EditHeightfield 优先，回退 `terrain_height` | O(1) CoW 查询 |
| `normal_at` | `actual_height_at` ×3 点 → 修改感知表面法线 | 有限差分计算 |
| `density_at` | `density_stack.density_at` + EditDensity 增量 | 手动叠加，避免锁 |
| `terrain_raycast` | 走 `density_at` → 自动包含修改 | DDA 步进 |
| `is_walkable` | `actual_height_at` + 修改感知法线 | 坡度和表面检测 |
| `surface_material_at` | EditDensity 材质 → DensityStack 材质 → 噪声回退 | 三级优先级 |
| `medium_at` | `actual_height_at` 判断水/空 | 2D 高度限制 |
| `sample_vertex` | `actual_height_at` + `actual_material_at` | Clipmap 消费 |
| `sample_horizon` | 走 `density_at` → 自动包含修改 | 天空可见性 |

`HeightfieldOcean` 同样修复：`terrain_height` → 先查 EditHeightfield，回退噪声。

## 修改流水线

```
[Phase 0: 积累]
  Player/NPC/Explosion → ModificationBatch::push()
  
[Phase 1: 应用]（modification_collector_system·待实现）
  验证权限 → EditDensityBuilder + EditHeightfieldBuilder
  → freeze() → EditTerrainSnapshot
  → DirtyChunkQueue 标记受影响 Chunk（含边界邻居）
  
[Phase 2: 同步]（WorldDriver·待实现）
  terrain.set_edit_terrain(snapshot)
  ocean.set_edit_heightfield(snapshot)
  
[Phase 3: 重提取]
  drain DirtyChunkQueue → submit_voxel_job(rayon)
  → DensityStack + EditDensityLayer → Transvoxel 提取
  → Godot ArrayMesh 更新
  
[Phase 4: 通知]（SpatialEventBus·待实现）
  push(TerrainModified) → NPC 感知系统 → 更新世界模型
```

## 查询路径集成

```
世界查询调用方
    │
    ├── HeightfieldTerrain（所有 TerrainQuery 方法）
    │       │
    │       ├── height_at / normal_at / is_walkable / medium_at
    │       │       └── actual_height_at()
    │       │               ├── EditHeightfield (CoW·O(1)) → 命中
    │       │               └── terrain_height() (噪声) → 回退
    │       │
    │       ├── density_at / terrain_raycast / sample_horizon
    │       │       └── density_stack.density_at() + edit.density_delta_at()
    │       │
    │       ├── surface_material_at / sample_vertex
    │       │       └── edit.density.material_at() ?? density_stack.material_at() ?? 噪声
    │       │
    │       └── sample_vertex (Clipmap 消费)
    │               └── actual_height_at() + actual_material_at()
    │
    ├── HeightfieldOcean（OceanProvider 方法）
    │       └── terrain_height() → EditHeightfield 优先，回退噪声
    │
    └── VoxelChunk::submit_voxel_job()（rayon 线程）
            └── 临时 DensityStack + EditDensityLayer (DensityProvider 桥接)
```

## 性能特征

| 指标 | 值 | 说明 |
|------|-----|------|
| 密度查询 | O(1) | CoW HashMap 查找，无锁 |
| 高度查询 | O(1) | HashMap 查找或噪声计算 |
| 材质查询 | O(layers) | 最多 ~5 层遍历（基底层→EditDensity） |
| 增量构建 | O(N) | N = 本帧修改体素数 |
| Transvoxel 提取 | 已有 + 0.1-1ms | 32³ chunk mesh 提取本身不变 |
| DirtyChunkQueue 去重 | O(1) | HashSet 插入 |
| 内存（稀疏） | ~48B/修改体素 | HashMap 条目开销，与修改量成正比 |

## 与现有 ECS 系统的集成点

### 已受影响的系统（CHG-065 修改感知）

| 系统 | 模块 | 影响 | 说明 |
|------|------|------|------|
| `movement_system` | `npc/movement.rs` | `height_at()` 现在走 `actual_height_at` | NPC 移动后 Y 钳制感知修改后的地表 |
| `lod_coordinator_system` | `lod_coordinator.rs` | 无直接影响 | 未来可通过 `LODCoordinatorInput.broadcasts` 接收修改事件触发 LOD 升级 |
| 所有 `TerrainQuery` 消费方 | 多个 | 9 个方法全部修改感知 | 透明升级，无需改调用方代码 |

### 将产生修改的系统（后续冲刺）

## 与 Veloren 的设计对比

| 方面 | Veloren (SPECS ECS) | WoWorld (hecs ECS) |
|------|---------------------|---------------------|
| 地形存储 | Chunk 独立稀疏结构（4×4×4 group） | CoW EditDensity + DensityStack |
| 修改积累 | `BlockChange` Resource（parking_lot::Mutex） | `ModificationBatch` Resource |
| 调度策略 | 每帧 dispatch 1 chunk，receive up to 240/s | 待实现（参考 Veloren 模式） |
| 网络同步 | +2 chunk buffer 启发式 | N/A（单机优先） |
| 材质系统 | BlockKind 4B 编码 | SurfaceMaterial 21 变体 |
| Mesh 算法 | Greedy meshing | Transvoxel（等值面提取） |
