## 三层审美管线

```
AestheticSignal (6D) → AestheticTaste (6D权重) → AestheticJudgment (4D)
   ↑ 被感知实体发出           ↑ 观察者偏好            ↑ 观察者反应
```

## System 定义

### AestheticEvaluationSystem
- **查询**: `(&AestheticTaste, &PerceptBatch)` + 被感知实体的 `HasAestheticSignal`
- **核心函数**: `judge(signal, taste, context) → AestheticJudgment`
  - Step 1: 感知抖动 (perceptual noise)
  - Step 2: 维度映射 (6D signal × 6D taste → 4D judgment)
  - Step 3: 上下文调制 (StakesContext, SocialAestheticContext)
- **衍生属性**: somatic_impact, symbolic_depth, cultural_fit
- **涌现体验分类**: beauty / stunning / sublime / interesting / cute / boring / cheap / offensive / pretentious

### TasteDriftSystem
- **触发**: 低频（月级/事件驱动）
- **derive_taste()**: BigFive + CultureCoreParams + CulturalBeautyStandard + parent taste + personal_seed
  — 青春期反叛因子: openness > 0.7 → taste 偏离父母 ±0.3
- **mature_taste()**: 年更新——年龄效应 + 技能成长 + 旅行效应 + 自然漂移
- **traumatic_taste_shift()**: 极端审美事件触发

### AestheticEventSystem（4 种事件原子）
| 原子 | 触发 | 效果 |
|------|------|------|
| React | 感知到 AestheticSignal | 情绪变化 + 记忆写入 + 行为倾向 + 占有欲 |
| Articulate | 5 表达通道 | 输出到语言/经济/行为系统 |
| Adopt | 社会影响 (6 源) | AestheticTaste 漂移 |
| Embellish | 创作意图 | 成品→物品系统接口 |

### FineArtsSkillSystem
- **6th 技能类别**: 4 子组——视觉艺术/音乐/表演/文学
- **跨类别训练**: 与 Crafting/Scholarship 有交叉训练权重

## HasAestheticSignal trait（12 实现者）

trait HasAestheticSignal {
    fn aesthetic_signal(&self, observer: EntityId) -> AestheticSignal;
    // 同一实体对不同的观察者可能产生不同的信号（文化差异）

## 4 条实现约束（C01-C04）

1. **禁止缓存 judge() 结果**——每次调用必须实时计算（上下文可能已变化）
2. **禁止无中介的 taste 突变**——所有 taste 变化必须经过 TasteDriftSystem
3. **禁止 judgment 持久化**——AestheticJudgment 是临时 Component, 帧末清除
4. **禁止 NpcData impl 块**——审美逻辑全部在 System 中

## 性能预算

- **judge()**: <0.002ms/次——纯数学, 零 VRAM
- **TasteColumn SoA**: 40B/NPC——L2 缓存友好
