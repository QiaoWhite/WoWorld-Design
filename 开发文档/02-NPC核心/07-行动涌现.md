## 核心机制

35 个物理原子 + ~40 个复合原子 + ~25 个社会原子 = ~100 个 System。

每个原子是一个独立的 System——查询特定的 Component 组合，执行物理/社会操作，修改 Component 状态。

---

## Component

| Component | 字段 | 内存 |
|-----------|------|------|
| `ActionQueue` | actions: [ActionAtom; 4], count: u8 | ~64B |
| `ActionCooldown` | cooldowns: [(AtomId, u16); 8] | ~32B |

## System（示例）

### MoveAtomSystem
- **查询**: `(&mut Position, &Velocity, &Goal{kind: Move})`
- **动作**: 沿路径移动实体

### StrikeAtomSystem
- **查询**: `(&CombatState, &EquipmentSlots, &AgentSnapshot, &Position)`
- **读 Resource**: `MaterialProperties` 表
- **动作**: 力 × 硬度差 = 变形 / 碎���

### CraftAtomSystem
- **查询**: `(&Goal{kind: Craft}, &SkillHandle, &Position)`
- **读 Resource**: `CraftingRecipeRegistry`
- **动作**: 执行锻造流程（加热→打击×N→观察→淬火→冷却→擦拭）

## 原子注册表

```rust
trait AtomRegistry {
    fn register_atoms(registry: &mut ActionRegistry);
}

// 战斗模块注册自己的原子
impl AtomRegistry for CombatModule {
    fn register_atoms(r: &mut ActionRegistry) {
        r.register(STRIKE, strike_execute);
        r.register(PARRY, parry_execute);
        r.register(DODGE, dodge_execute);
    }
}
// 新创意只需 impl 这个 trait → 不改已有代码
```
