## 这个模块关心什么"细节"

移动系统驱动 NPC 在 3D 世界中的**物理位移**：有目标时朝目标直线移动，无目标时漫游。系统负责**地形感知**——Y 轴跟随地表高度、坡度减速、陡坡阻挡。

## Component 定义

### Movement

```rust
struct Movement {
    speed: f32,           // 基础移动速度 (m/s), 默认 3.0
    arrival_radius: f32,  // 到达判定半径 (m), 默认 0.5
}
impl Component for Movement {}
```

| 字段 | 读写 System |
|------|-----------|
| speed | MovementSystem(R) — 后续可从 BigFive 外向性 + urgency 派生 |
| arrival_radius | MovementSystem(R) |

### Wander

```rust
struct Wander {
    direction: Vec3,  // 归一化移动方向（XZ 平面）
    remaining: f32,   // 剩余漫游时间 (s), 到期后重选方向
}
impl Component for Wander {}
```

| 字段 | 读写 System |
|------|-----------|
| direction | MovementSystem(R+W) — 漫游时定期重选 |
| remaining | MovementSystem(R+W) — 倒计时耗尽后重选 |

## System 定义

### MovementSystem

- **签名**: `movement_system(world, cmd, dt, tick, terrain: &dyn TerrainQuery)`
- **触发**: Entity 有 `Position` + `Movement` + `Goal` + `Needs`，`Wander` 可选
- **逻辑**:
  1. **有目标** (`Goal.target_pos = Some(pos)`): 朝目标 XZ 方向直线移动。距离 < `arrival_radius` 时判定到达→调用 `satisfy_goal()` 恢复需求→移除 Goal + Wander
  2. **无目标** (`Goal.target_pos = None`): 漫游——沿 `Wander.direction` 移动，每 3s 重选随机方向
  3. **到达判定忽略 Y 轴**——XZ 距离 < arrival_radius 即到达
- **★ 地形感知** (Phase 2):
  1. 计算 XZ 新位置
  2. 查 `terrain.height_at(new_xz)` → 设 Y
  3. 查 `terrain.normal_at(new_xz)` → 若 `normal.y < MAX_WALKABLE_SLOPE_COS` (0.707 = 45°) → **陡坡阻挡**，不动
  4. 若 `normal.y < SLOPE_FULL_SPEED_COS` (0.966 = 15°) → **坡度减速**，`actual_speed = speed × slope_factor`
- **CommandBuffer**: `remove: Goal` (到达时), `remove: Wander` (到达时), `insert: Wander` (首次漫游)

### 与 GoalSystem 的关系

MovementSystem **不决定去哪里**——它只消费 `Goal.target_pos`。目标选择由 GoalResolutionSystem（上层认知）负责。MovementSystem 是机械执行层。

---

## 地形感知参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `MAX_WALKABLE_SLOPE_COS` | 0.707 (≈45°) | 超过此坡度原地不动 |
| `SLOPE_FULL_SPEED_COS` | 0.966 (≈15°) | 低于此坡度全速，15°-45° 平滑减速 |
| `WANDER_CHANGE_INTERVAL` | 3.0s | 漫游方向重选间隔 |

> ⚠️ 以上数值为工程判断，未经设计文档确认。调参时应验证不同地形（平原/丘陵/山地）下的 NPC 移动观感。

## 完整移动流程

```
MovementSystem (每帧)
  │
  ├─ Goal.target_pos = Some(pos)
  │   ├─ XZ 距离 < arrival_radius → 到达
  │   │   ├─ satisfy_goal(goal, needs)   // 恢复对应需求
  │   │   └─ remove Goal + remove Wander
  │   └─ XZ 距离 ≥ arrival_radius
  │       ├─ 计算方向 = (target - current).normalize_xz()
  │       ├─ 计算 new_xz = current + direction × speed × dt
  │       ├─ new_y = terrain.height_at(new_xz)
  │       ├─ normal = terrain.normal_at(new_xz)
  │       ├─ normal.y < 0.707 → 陡坡, 不动
  │       ├─ normal.y < 0.966 → 坡度减速
  │       └─ pos = (new_xz.x, new_y, new_xz.z)
  │
  └─ Goal.target_pos = None
      ├─ 无 Wander → 创建 Wander (随机方向, remaining=3s)
      └─ 有 Wander → 沿 direction 移动 + 地形感知
```

每个 System 独立工作。**MovementSystem 不知道"为什么"移动**——那是上层认知的事。

## Resource 依赖

| Resource | 用途 |
|----------|------|
| `TerrainQuery` (trait) | height_at / normal_at — 地形感知的数据源 |

MovementSystem 通过参数注入 `&dyn TerrainQuery` 而非存储为 Resource——避免 ECS World 持有 trait object 的生命周期问题。
