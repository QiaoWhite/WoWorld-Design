## 核心哲学

NPC 没有"神学立场"——只有 `ReligiousPracticeProfile`。信仰通过 NPC 移动/交谈/死亡传播。"有人的地方就有信仰。"

---

## Component

| Component | 字段 | 内存 | 说明 |
|-----------|------|------|------|
| `FaithId` | u32 | 4B | 扁平 ID |
| `ReligiousPracticeProfile` | participation:f32, motivation:u8(3种), theological_depth:f32 | 12B | 热路径本地读取——~40B/NPC |
| `FaithTheology` | 10 参数: piety_weight:f32, orthodoxy_tolerance:f32, proselytizing_drive:f32, asceticism:f32, mysticism:f32, ritualism:f32, exclusivity:f32, syncretism:f32, apocalypticism:f32, wealth_attitude:f32 | 40B | 每个信仰的定义 (~500B/faith) |

## System

| System | 触发 | 动作 |
|--------|------|------|
| `FaithPracticeSystem` | 日级 | NPC 宗教实践更新——participation 衰减 + 仪式恢复 |
| `FaithPropagationSystem` | NPC 交互时 | 信仰在 NPC 间传播——接触感染 + 讲道 + 死亡见证 |
| `SchismSystem` | 年级 | 分裂/融合/改革/复兴条件检测 |
| `DivineEventSystem` | 事件驱动 | 神迹/天启/宗教战争触发 |

## Resource

| Resource | 内容 |
|----------|------|
| `FaithRegistry` | FaithId → FaithDefinition (含 FaithTheology + 神圣建筑参数 + 葬礼习俗) |
| `FaithQuery` (trait) | 30 个只读方法——零分配查询 |
| `SettlementFaithSnapshot` | NPC 群体惰性聚合 |

## 实践先于教义

```
NPC 的 ReligiousPracticeProfile {
    participation: 从父母/社区观察学习的行为频率
    motivation: 3种——Communal/Quest/Conformity
    theological_depth: 后期反思→加深——不是前提条件
}
// 先有"做"→后有"信"→再有"深"
```

## 魔法-宗教关系 (4 种)

| 关系 | 说明 |
|------|------|
| Gift | 魔法是神的恩赐——宗教实践增强魔法 |
| Blasphemy | 魔法是对神的亵渎——宗教实践抑制魔法 |
| Independent | 魔法和宗教各走各路 |
| Unified | 魔法实践 = 宗教仪式 |
