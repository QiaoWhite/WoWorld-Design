## 核心哲学

10 元素体系(金木水火土风雷电气灵)——顺序固定, 不可混淆(雷=声音/振动, 电=闪电/电荷)。常规消耗 Mana (安全), 过载消耗 raw spirit (渐进症状→系统阻断→不可逆死亡)。

---

## Component

| Component | 字段 | 内存 | 说明 |
|-----------|------|------|------|
| `ManaPool` | current:u32, max:u32(ke), regen_rate:f32 | 12B | u32 刻——避免浮点精度误差 |
| `SpiritState` | is_overloading:bool, overload_progress:f32, irreversible_threshold:f32 | 12B | 过载追踪 |
| `SpellSlots` | slots:[Option<SpellId>; 8], count:u8 | ~32B | 已装备法术 |
| `ActiveEffects` | effects:[Option<EffectInstance>; 4], count:u8 | ~64B | 活跃效果 |
| `ElementAffinity` | affinities:[f32; 10] | 40B | 10 元素亲和度——种族+训练 |
| `AetherImprint` | (shared with 历史系统) | 12B | 灵元素印记 |

## System

| System | 查询 | 动作 |
|--------|------|------|
| `MagicCastSystem` | `(&SpellSlots, &ManaPool, &AgentSnapshot)` | 施法——消耗法力→产生效果→AetherImprint |
| `EffectTickSystem` | `(&ActiveEffects)` | 效果持续时间推进 + 衰减 |
| `ElementInteractionSystem` | `(&ActiveEffects)` + Resource<ElementTable> | 元素反应——火+水=蒸汽, 风+火=扩散, 雷+水=传导 |
| `SpiritOverloadSystem` | `(&SpiritState, &ManaPool)` | 过载监控——渐进症状→系统阻断 |
| `MagicRegenSystem` | `(&ManaPool)` | Mana 恢复——活跃 1-3%/h, 冥想 5-10%/h |

## Resource

| Resource | 内容 |
|----------|------|
| `SpellRegistry` | SpellId → SpellDef (元素/消耗/效果/施法时间) |
| `ElementTable` | 10×10 元素交互矩阵 |

## 调优参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `MANA_REGEN_ACTIVE` | 0.02/h | 活跃恢复率 |
| `MANA_REGEN_MEDITATE` | 0.08/h | 冥想恢复率 |
| `SPIRIT_IRREVERSIBLE` | 0.95 | 不可逆阈值 |

## 约束检查清单

- [x] 雷/电不可混淆
- [x] Mana 全 u32——无浮点精度误差
- [x] 魔法效果不直接修改 Vitals——通过 CombatDamageSystem
