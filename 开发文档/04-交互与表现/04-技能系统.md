## 核心哲学

**技能是事后记录标签, 不是事前门控规则。** 门槛从物理涌现（MaterialProperties × noise）——不是 `if skill_level < X { cannot_do }`.

## Component 定义

| Component | 字段 | 内存 | 说明 |
|-----------|------|------|------|
| `SkillHandle` | storage_key:u64, count:u16 | 10B | → SkillStorage Resource |
| `SkillCooldown` | cooldowns:[(SkillId, u16); 8] | ~32B | 技能冷却计时器 |
| `InnateAptitude` | aptitudes:[(SkillId, f32); 16] | ~128B | 0.7-1.3, 创角时 roll |

## System 定义

### SkillUseSystem
- **查询**: `(&SkillHandle, &AgentSnapshot)` + `Resource<SkillStorage>`
- **公式**: `execution_precision → execution_noise_std` (f32 签名——零耦合)
- **XP 公式**: `total_xp(L) = 100 × (e^(0.04×L) - 1) / 0.04` — 指数减速
- **原则**: 用进不退——无衰减

### SkillTrainingSystem
- **4 种教学路径**:
  - 自学: 1.0× 效率
  - 师承: 1.2-3.0× (取决于师傅 skill_level / 徒弟 skill_level)
  - 学院: 1.3× 基准
  - 秘传: → Magic::add_xp_direct()
- **3 层天赋模型** (乘法叠加):
  - Layer 1: MentalAccess trait (0.7-1.0, 由认知系统提供)
  - Layer 2: InnateAptitude (0.7-1.3, 创角 roll)
  - Layer 3: CrossTraining (同组 0.04-0.20, 天花板 min(40, 0.5×source_level))

### TeachingSystem
- **查询**: `(&SkillHandle)` where NPC is teaching
- **逻辑**: 知识传递——师傅 SkillEntry → 徒弟 XP 累积

## SkillId 布局

```
u64: [8bit category][8bit subgroup][16bit reserved][32bit serial]

5 类别: Combat / Magic / Artisan / Academic / Survival
22 子组 (跨类别交叉训练权重定义在子组级别)
```

## Resource

| Resource | 内容 |
|----------|------|
| `SkillStorage` | EntityId → HashMap<SkillId, SkillEntry> (稀疏存储) |
| `SkillRegistry` | SkillId → SkillDef (TOML 数据驱动) |

## 交叉训练

```rust
fn cross_training_bonus(source: SkillId, target: SkillId) -> f32 {
    let group = source.subgroup();
    if group == target.subgroup() {
        min(0.20, 0.04 + group_similarity * 0.16)  // 同组: 0.04-0.20
    } else {
        0.0  // 非递归——不跨组传递
    }
}
```
