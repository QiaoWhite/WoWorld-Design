## Hybrid 架构

Rust 侧: 骨骼矩阵计算 (CPU, ≤0.5ms)。Godot 侧: GPU skinning + 渲染。

---

## Component (Rust 侧)

| Component | 字段 | 内存 | 说明 |
|-----------|------|------|------|
| `SkeletonState` | bone_count:u8, pose_data_handle:u64 | 12B | 骨骼配置 |
| `AnimationState` | active_layers:u8, blend_weights:[f32; 9], gait_params:[f32; 9] | 44B | 9 层动画栈 |
| `BodyPlan` | skeleton_template:u8, bone_mapping:[u8; 35] | 36B | 身体蓝图——通用 |
| `PhysicalCollider` | capsule_count:u8, capsule_data_handle:u64 | 12B | 物理碰撞体 (LOD 驱动) |

## 涌现式步态

9 个参数从 BigFive 派生——非手工预设:
- stride_length, arm_swing, hip_sway, shoulder_rotation
- head_bob, foot_placement, weight_transfer, posture_tilt, tempo

## System

| System | 动作 |
|--------|------|
| `SkeletonAnimationSystem` (Rust) | CPU 批量骨骼矩阵——≤0.5ms (6 核 rayon) |
| `AnimationDispatchSystem` (Phase 2) | 骨骼矩阵→Godot GPU skinning |

## 物理

- **仅玩家**保留 Godot PhysicsServer3D
- **NPC/生物/载具**全部走 Rust 侧空间查询 (TerrainQuery/EntityIndex)

## IK 管线

```
碰撞扫掠 (capsule-capsule: ~5ns) →
2-bone 解析 IK (~17.5ns/limb) →
完整 IK 链 (双臂+双腿+脊椎+头部: ~0.67μs/NPC)
// 比预置动画混合 (~6μs/NPC) 快 ~9×
```
