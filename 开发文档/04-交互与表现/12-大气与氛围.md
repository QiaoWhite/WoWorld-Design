# 大气与氛围 — ECS 架构

> **关联原文档**: [[开发阶段/大气与氛围系统/README]]

## 不进 ECS

大气与氛围系统**整个不进 ECS**。

**原因**：已有独立的 `woworld_atmosphere` crate——纯计算，无实体管理。

## 作为 Resource

```rust
struct ResolvedAtmosphere {
    sky_zenith_color: [f32;3],
    sky_horizon_color: [f32;3],
    ambient_color: [f32;3],
    sun_color: [f32;3],
    sun_energy: f32,
    fog_color: [f32;3],
    fog_density: f32,
    // … 17 参数
}
```

`AtmosphereSynthesizer` 作为 Resource 被 WorldDriver 持有，每帧更新 `ResolvedAtmosphere`。

## 消费方式

Godot shader 直接消费 `ResolvedAtmosphere` 的 `PackedFloat32Array` 输出——不经过 ECS。
