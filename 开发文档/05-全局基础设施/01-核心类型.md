## 这个模块关心什么"细节"

定义所有 ECS 模块共享的基础类型：EntityId、WorldPos、SurfaceMaterial、时间类型、ID 类型。

---

## 类型清单

### EntityId

```rust
// woworld_core::types
pub struct EntityId(pub u64);
// 与 hecs::Entity 互转：EntityId(id.0) ↔ hecs::Entity::from_bits(id.0)
```

不移入 ECS Component——保留为基础类型。

### WorldPos / Vec3 / DVec3 / Aabb

保留在 `woworld_core::types`，用于 Component 字段和空间查询。

### SurfaceMaterial (21 变体)

```rust
enum SurfaceMaterial { Grass, Sand, Rock, Stone, Wood, Metal, Water, Ice, Mud, Snow, Gravel, Clay, Moss, LeafLitter, Cobblestone, Marble, Glass, Fabric, Thatch, Bone, Flesh }
```

保留为基础枚举。不进 ECS——地形查询返回此类型。

### WorldTime / WorldClock

```rust
struct WorldTime { day_progress: f64, day_number: u64, season_progress: f64, phase: TimeOfDay, light_level: f32, sun_elevation: f64, sun_azimuth: f64 }
struct WorldClock { current: WorldTime, seconds_per_day: f64, … }
```

保留为 Resource。不进 ECS 的 Entity-Component 体系。

### ID 类型

`ItemDefId(u64)`, `ItemEntId(u64)`, `SkillId(u64)`, `ProfessionTagId(u32)`, `SpeciesId(u64)`, `ChunkCoord`

全部保留——用于 Component 字段。

---

## 不进 ECS 的原因

这些是**值类型**和**全局单例**——不是"有生命周期、有状态变化"的实体。放在 ECS 的 Archetype 表中没有意义，反而增加查询成本。

---

## 与 hecs 的互操作

```rust
// EntityId → hecs::Entity
fn to_hecs(id: EntityId) -> hecs::Entity {
    hecs::Entity::from_bits(id.0)
}

// hecs::Entity → EntityId
fn from_hecs(e: hecs::Entity) -> EntityId {
    EntityId(e.to_bits())
}
```
