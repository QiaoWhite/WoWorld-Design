## 这个模块关心什么"细节"

空间查询提供 5 个全局 Resource——地形采样、实体索引、事件总线、可见性检测、名声查询。它们不是 ECS Component，而是**被 System 查询的全局服务**。

---

## Resource 定义

### TerrainQuery

```rust
trait TerrainQuery: Send + Sync {
    fn height_at(&self, pos: WorldPos) -> f32;
    fn normal_at(&self, pos: WorldPos) -> Vec3;
    fn terrain_raycast(&self, origin: WorldPos, dir: Vec3, max_dist: f32) -> Option<TerrainHit>;
    fn density_at(&self, pos: WorldPos) -> f32;
    fn is_walkable(&self, pos: WorldPos) -> bool;
    fn surface_material_at(&self, pos: WorldPos) -> SurfaceMaterial;
    fn medium_at(&self, pos: WorldPos) -> Medium;
    fn light_level_at(&self, pos: WorldPos) -> f32;
    fn sample_horizon(&self, pos: WorldPos, dirs: &[Vec3]) -> Vec<f32>;
}
```

**实现者**: `woworld_worldgen::HeightfieldTerrain`（不进 ECS）
**消费者**: `MovementSystem`, `CombatSystem`, `PerceptionSystem`, `BuildingSystem`
**★ CHG-065**: 全部 9 方法已感知 EditTerrain——`height_at` 查 EditHeightfield，`density_at` 叠加 EditDensity 增量，`surface_material_at` 查编辑材质。消费者代码无需修改。

**消费者**: 几乎所有需要空间查询的 System

### SpatialEventBus

```rust
trait SpatialEventBus: Send + Sync {
    fn recent_events_in(&self, aabb: &Aabb, time_window: Duration) -> Vec<SpatialEvent>;

    fn scent_sources_in(&self, aabb: &Aabb) -> Vec<ScentSource>;

```

**维护者**: 各 System 通过 `push_event()` 写入
**消费者**: `PerceptionSystem`, `AudioSystem`, `HistoryRecorderSystem`
**★ CHG-065**: 地形修改完成后将推送 `TerrainModified` 事件（待实现）——NPC 感知系统消费

### ReputationQuery (NEW)

```rust
/// 名声查询——从分散式 NPC 记忆中涌现，不建集中式声望数据库。
trait ReputationQuery: Send + Sync {
    /// observer 个人对 target 的评价: -1.0 ~ +1.0, 0=中性/无印象
    fn reputation_toward(&self, observer: EntityId, target: EntityId) -> f32;
    /// target 在区域内所有知情 NPC 中的共识评价
    fn local_consensus(&self, target: EntityId, center: WorldPos, radius: f32) -> f32;
}
```

**实现者**: `woworld_ecs`（stub·返回 0.0——等 MemoryStore 就位后连上）
**消费者**: 战斗系统(名声传播) / 权力系统(合法性) / 玩家 UI(NPC 态度) / 文化系统(仪式资格)
**涌现原则**: 无全局名声值。每个 observer 独立视角。共识从知情 NPC 的独立评价加权平均涌现。

## System 定义

### SpatialIndexSystem

- **触发**: Phase 1 每帧
- **读**: `&Position`, `&Collider`, `&LayerMask`（所有有这些 Component 的 Entity）
- **写**: `&mut EntityIndex`（Resource）
- **逻辑**: 从 Position Component 重建空间索引
- **与其他 System 的关系**: 无

## 消费模式

```rust
// MovementSystem 中使用 TerrainQuery
fn movement_system(world: &hecs::World, terrain: &dyn TerrainQuery) {
    for (e, (pos, target)) in world.query::<(&Position, &PathTarget)>().iter() {
        let height = terrain.height_at(pos.into());
        // 不需要 terrain 是 Component——它是全局 Resource
    }
}

// PerceptionSystem 中使用 EntityIndex
fn perception_system(world: &hecs::World, index: &dyn EntityIndex) {
    for (e, (pos, eyes)) in world.query::<(&Position, &Eyes)>().iter() {
        let nearby = index.entities_in_aabb(&aabb, LAYER_NPC);
        // 不需要 index 是 Component——它是全局 Resource
    }
}
```

**关键**：空间查询是全局服务——EntityIndex 内部管理自己的数据结构，不依赖 Entity-Component 体系。System 通过 Resource 访问它，就像访问 WorldClock 一样。
