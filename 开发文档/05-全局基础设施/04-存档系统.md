## 这个模块关心什么"细节"

存档系统负责所有 ECS 状态的序列化与反序列化。核心挑战：ECS 的 Archetype 表 → LMDB 键值对 的双向转换。

---

## Component 定义

### SaveMarker

```rust
struct SaveMarker {
    last_saved_tick: u64,
    is_dirty: bool,
}
impl Component for SaveMarker {}
```

标记 Entity 是否需要序列化。`is_dirty` 由任何修改了该 Entity 的 System 设置为 true。

---

## Resource 定义

### SaveDb

```rust
struct SaveDb {
    env: lmdb::Environment,
    // named_dbs: terrain / entities / economy / power / culture / faith / ...
}
```

`SaveDb` 是 LMDB 句柄——存档系统只提供 IO 流，不拥有模拟数据。

---

## System 定义

### SaveSerializationSystem

- **触发**: 手动保存 / 自动保存触发 / 退出保存触发
- **读**: 所有有 `SaveMarker{is_dirty: true}` 的 Entity + 其全部 Component
- **写**: `SaveDb`（LMDB 写入）
- **逻辑**:
  1. 遍历所有 dirty Entity
  2. 对每个 Entity，序列化其全部 Component 到 LMDB key: `entities/{entity_id}/{component_type}`
  3. 更新 `SaveMarker.last_saved_tick`

### LoadDeserializationSystem

- **触发**: 加载存档
- **读**: `SaveDb`
- **写**: `hecs::World`（重建所有 Entity + Component）
- **逻辑**:
  1. 从 LMDB 读取所有 `entities/` 键
  2. 对每个 Entity，重建其 Component 集合
  3. 验证 Archetype 一致性

---

## 序列化策略

| Component 类型 | 序列化方式 |
|---------------|----------|
| 纯值 Component（Vitals, Position, …） | 直接 bincode 序列化（固定大小，快速） |
| Handle + Resource（MemoryHandle → MemoryStorage） | Handle 序列化 + Resource 独立序列化 |
| Tag Component | 不序列化（重建时根据状态重新添加） |
