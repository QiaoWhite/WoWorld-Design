## 核心哲学

**玩家 = NPC 的 Component 集合 + PlayerTag + ControlMode**

玩家不是新的 Entity 类型。玩家是"一个 NPC 被玩家接管了控制权"这件事。

---

## Component 定义

### PlayerTag

```rust
struct PlayerTag;  // 零数据标签——标记"这个 Entity 正在被玩家控制"
impl Component for PlayerTag {}
```

### ControlMode

```rust
enum ControlMode {
    Auto,                                    // NPC 自主（GOAP 驱动）
    Manual { action_override: Option<Action> },  // 玩家直接输入
    DomainDelegated { manual_domains: HashSet<ActionDomain> },  // 部分手动（如玩家只控制战斗）
}
impl Component for ControlMode {}
```

---

## System 定义

### PlayerInputSystem

- **触发**: Phase 0（必须先执行）
- **读**: `Resource<InputState>`（Godot 输入）
- **写**: `ControlMode` Component
- **与其他 System 的关系**: 无——ActionResolutionSystem 下帧读取 ControlMode

---

## 玩家 = NPC 的证明

```
一个"被玩家控制的铁匠 NPC"的 Component 集合：

  [NpcCore, Position, Vitals, Needs, Goal, SkillHandle, ProfessionTagId]  ← NPC 的部分
  [PlayerTag, ControlMode::Manual]                                          ← 玩家的部分

当玩家释放控制（切换角色或死亡）：
  → 拆 PlayerTag, ControlMode
  → Entity 回归为普通 NPC
  → GOAP System 下帧自然接管
```

**不需要"把玩家变成 NPC"或"把 NPC 变成玩家"——只是 Component 的拆和装。**

---

## 双角色系统

玩家可以控制 1-2 个角色：

```
Entity A: [NpcCore{铁匠}, PlayerTag, ControlMode{战斗=Manual}]
Entity B: [NpcCore{法师}, PlayerTag, ControlMode{Auto}]  ← 非活跃角色 = AI控制

切换时：
  Entity A: 拆 ControlMode → GOAP 接管
  Entity B: 装 ControlMode → 玩家接管
```
