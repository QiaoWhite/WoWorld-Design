## 这是什么

本文件夹是 WoWorld 项目的 **ECS（Entity Component System）架构视角**。

原始设计文档（`WoWorld-Design/Happy Game/开发阶段/`）以游戏领域模块组织，描述了 27 个系统的完整设计——NPC 认知、世界生成、经济、战斗等。这些文档是**权威设计规格**，不动。

本文档从 ECS 的视角重新组织相同的设计内容：每个领域模块的 Component 是什么、System 如何工作、Resource 依赖哪些。这是**补充视角**，用于指导 Rust 侧 ECS 架构的代码实现。

## 为什么 ECS

- **100+ NPC 全模拟**：需要 SoA 缓存友好的内存布局
- **20+ 系统并行**：经济/权力/文化/战斗/感官等系统需要无冲突并发
- **新创意接入**：新功能 = 新 Component + 新 System，不改已有代码
- **现在改架构 = 改规划，以后改 = 重写代码**

## 技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
| ECS 存储 | `hecs` crate | Archetype SoA，轻量，不绑调度器 |
| System 模型 | 普通 Rust 函数 | 不定义 trait，不设 before/after |
| 调度 | WorldDriver 两阶段 | Phase 0(输入/LOD) + Phase 1(游戏逻辑·无顺序) + Phase 2(Godot同步) |
| 组件拆装 | Component add/remove | ECS 核心机制——状态变化 = 组件集变化 |
